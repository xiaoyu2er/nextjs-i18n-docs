欢迎来到 Next.js 文档！

## Next.js 是什么？

Next.js 是一个用于构建全栈 Web 应用程序的 React 框架。您可以使用 React 组件来构建用户界面，并使用 Next.js 获得额外的功能和优化。

在底层，Next.js 还会抽象并自动配置 React 所需的工具，如打包、编译等。这让你能够专注于构建应用，而不必花费时间在配置上。

无论您是独立开发者还是大型团队的一员，Next.js 都可以帮助您构建交互式、动态且快速的 React 应用程序。

## 主要功能

Next.js 的一些主要功能包括：

| 功能 | 描述 |
| --- | --- |
| [路由](/docs/app/building-your-application/routing) | 基于文件系统的路由器，构建于 服务器组件 (Server Components) 之上，支持布局、嵌套路由、加载状态、错误处理等。
| [渲染](/docs/app/building-your-application/rendering) | 使用 客户端和服务器组件 (Client and Server Components) 进行客户端和服务器端渲染。利用 Next.js 在服务器端进一步优化静态和动态渲染。支持 边缘 (Edge) 和 Node.js 运行时的 流式传输 (Streaming)。
| [数据获取](/docs/app/building-your-application/data-fetching) | Server Components 中简化 async/await 数据获取，以及扩展的 `fetch` API 用于请求缓存、数据缓存和重新验证。
| [样式](/docs/app/building-your-application/styling) | 支持您首选的样式方法，包括 CSS Modules、Tailwind CSS 和 CSS-in-JS。
| [优化](/docs/app/building-your-application/optimizing) | 图像、字体和脚本优化，以改善应用的 核心 Web 指标 和用户体验。
| [TypeScript](/docs/app/building-your-application/configuring/typescript) | 增强对 TypeScript 的支持，提供更好的类型检查和更高效的编译，以及自定义 TypeScript 插件和类型检查器。

## 如何使用这些文档

屏幕左侧，您将找到文档导航栏。文档页面按顺序组织，从基础到高级，以便您在构建应用时一步步跟随。不过，您可以按任何顺序阅读，或直接跳转到适用于您用例的页面。

屏幕右侧，您将看到一个目录，可更容易地在页面各部分之间导航。如果您需要快速查找页面，可以使用顶部的搜索栏，或者使用搜索快捷键（`Ctrl+K` 或 `Cmd+K`）。

要开始使用，请查看 [安装](/docs/getting-started/installation) 指南。

## App Router 与 Pages Router

Next.js 有两种不同的路由系统：App Router 和 Pages Router。App Router 是一种更新的路由器，允许您使用 React 的最新功能，例如 服务器组件 (Server Components) 和 流式传输 (Streaming)。Pages Router 是原始 Next.js 路由系统，它允许您构建服务器端渲染的 React 应用，并继续支持较旧的 Next.js 应用。

边栏顶部，您会看到一个下拉菜单，允许您在 **App Router** 和 **Pages Router** 功能之间切换。由于每个目录都有特有的功能，因此跟踪选择了哪个标签非常重要。

页面顶部的 面包屑导航 也会指示您正在查看 App Router 文档还是 Pages Router 文档。
