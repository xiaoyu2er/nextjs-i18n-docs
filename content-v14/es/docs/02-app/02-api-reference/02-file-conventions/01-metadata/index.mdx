Esta sección de la documentación cubre **convenciones de archivos de metadatos**. Los metadatos basados en archivos se pueden definir añadiendo archivos especiales de metadatos a segmentos de ruta.

Cada convención de archivo puede definirse usando un archivo estático (por ejemplo, `opengraph-image.jpg`), o una variante dinámica que utiliza código para generar el archivo (por ejemplo, `opengraph-image.js`).

Una vez definido un archivo, Next.js lo servirá automáticamente (con hashes en producción para caché) y actualizará los elementos head relevantes con los metadatos correctos, como la URL del recurso, el tipo de archivo y el tamaño de la imagen.
