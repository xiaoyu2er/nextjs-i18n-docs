2016 年，谷歌开始对使用 [AMP](https://amp.dev/) 技术的网页给予[搜索排名优先](https://developers.google.com/search/docs/guides/about-amp) —— 这项技术虽能让开发者创建在移动设备上加载更快的网页，但需要长期投入开发和维护成本。

随着 [Core Web Vitals](https://web.dev/vitals/) 页面体验更新，谷歌[取消了 AMP 页面作为出现在搜索轮播中的必要条件](https://developers.google.com/search/blog/2021/04/more-details-page-experience#details)。这是谷歌在 SEO 方面为 AMP 保留的最后一项主要优势。

自 AMP 推出以来，诸如 Next.js 等新技术已证明其能在不牺牲开发者体验 (DX) 的前提下提升网站体验。

虽然 Next.js 提供 [AMP 支持](https://nextjs.org/docs/advanced-features/amp-support/introduction)，但如果您的网站已具备优秀的 Core Web Vitals 评分，建议权衡实施 AMP 方案的成本与收益。
