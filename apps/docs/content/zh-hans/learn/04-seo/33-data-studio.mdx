另一种免费开源的性能测量方法是使用 [Chrome 用户体验报告](https://developers.google.com/web/tools/chrome-user-experience-report) 数据集。

[Chrome 用户体验报告](https://developers.google.com/web/tools/chrome-user-experience-report) 提供了真实用户在使用 Chrome 访问热门网站时的体验指标。

该数据集可通过 [BigQuery](https://console.cloud.google.com/bigquery?project=chrome-ux-report) 公开获取，也可以完全免费地在 [Google Data Studio](https://datastudio.google.com/overview) 中使用。

幸运的是，已有现成的 [开源仪表板模板](https://g.co/chromeuxdash) 可供直接使用，用于跟踪网站性能表现。

该数据集的唯一缺点是：若要使网站被纳入 CrUX 报告，需要达到一定的访问量级，否则会因数据不足而被排除。因此对于开发中或新创建的网站可能不是最佳选择。

此外，数据每月更新一次，通常在上个月结束后的 15 天左右，因此您看到的数据会有一定延迟。如果您计划优化核心网页指标 (Core Web Vitals)，这可能不够实时。
