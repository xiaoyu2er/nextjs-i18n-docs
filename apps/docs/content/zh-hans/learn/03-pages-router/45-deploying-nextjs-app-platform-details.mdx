[Vercel](https://vercel.com) 由 Next.js 的创造者打造，为 Next.js 提供一流的支持。当您将 Next.js 应用部署至 Vercel 时，默认会发生以下情况：

*   使用 [静态生成 (Static Generation)](/docs/basic-features/pages#static-generation-recommended) 的页面和资源（JS、CSS、图片、字体等）将自动通过 [Vercel CDN](https://vercel.com) 提供服务，速度极快。
*   使用 [服务端渲染 (Server-Side Rendering)](/docs/basic-features/pages#server-side-rendering) 的页面和 [API 路由](/docs/api-routes/introduction) 将自动转换为独立的 [无服务器函数 (Serverless Functions)](https://vercel.com/docs/serverless-functions/introduction)，从而实现页面渲染和 API 请求的无限扩展。

Vercel 还提供更多功能，例如：

*   **自定义域名**：部署至 Vercel 后，您可为 Next.js 应用分配自定义域名。请参阅 [相关文档](https://vercel.com/docs/concepts/projects/custom-domains)。
    
*   **环境变量**：您可在 Vercel 上设置环境变量。请查阅 [环境变量文档](https://vercel.com/docs/build-step#environment-variables)，然后在 Next.js 应用中 [使用这些环境变量](/docs/basic-features/environment-variables#loading-environment-variables)。
    
*   **自动 HTTPS**：HTTPS 默认启用（包括自定义域名）且无需额外配置，SSL 证书会自动续期。
    

您可通过 [Vercel 文档](https://vercel.com/docs) 了解更多平台功能。

### [每次推送的预览部署](#preview-deployment-for-every-push)

> 以下步骤为**可选**——您可以选择尝试或仅阅读了解。

部署至 Vercel 后，建议尝试以下操作（如条件允许）：

*   在您的应用中创建新**分支**。
*   进行更改并推送至 GitHub。
*   创建新**拉取请求**（[GitHub 帮助文档](https://help.github.com/en/github/collaborating-with-issues-and-pull-requests/creating-a-pull-request)）。

您将在拉取请求页面看到 `vercel` 机器人的评论。

![预览部署 URL](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/pages-router/deploying-nextjs-app/vercel-bot.png)

点击评论中的**预览** URL，即可查看您刚做的更改。

当拉取请求处于开启状态时，Vercel 会为每次推送自动创建该分支的**预览部署**。预览 URL 始终指向最新的预览部署。

您可分享此预览 URL 给协作者，即时获取反馈。

若预览部署符合预期，**将其合并至 `main` 分支**。合并后，Vercel 将自动创建生产环境部署。

### [开发、预览、发布](#develop-preview-ship)

我们刚刚体验了称为 **DPS** 的工作流：**D**evelop（开发）、**P**review（预览）和 **S**hip（发布）。

*   **开发**：我们在 Next.js 中编写代码，并利用 Next.js 开发服务器的热重载功能。
*   **预览**：我们将更改推送至 GitHub 分支，Vercel 会创建可通过 URL 访问的预览部署。除了进行_代码审查_，您还可以进行_部署预览_，将此预览 URL 分享给他人获取反馈。
*   **发布**：我们将拉取请求合并至 `main` 分支以发布至生产环境。

我们强烈建议在开发 Next.js 应用时采用此工作流——它将帮助您更快地迭代应用。
