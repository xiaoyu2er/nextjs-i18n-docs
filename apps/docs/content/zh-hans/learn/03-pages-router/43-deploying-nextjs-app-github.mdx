在部署之前，如果您尚未操作，请将您的 Next.js 应用推送至 [GitHub](https://github.com/vercel/next.js)。这将使部署过程更加便捷。

*   在您的个人 GitHub 账户中，创建一个名为 `nextjs-blog` 的新仓库。
*   仓库可设为公开或私有。您**无需**使用 README 或其他文件初始化仓库。
*   如需设置仓库的帮助，请参阅 [GitHub 上的这份指南](https://help.github.com/en/github/getting-started-with-github/create-a-repo)。

接下来：

*   如果您尚未在本地为 Next.js 应用初始化 Git 仓库，请立即执行此操作。
*   将 Next.js 应用推送至您的 GitHub 仓库。

要推送至 GitHub，您可以运行以下命令（请将 `<username>` 替换为您的 GitHub 用户名）：

```
git remote add origin https://github.com/<username>/nextjs-blog.git
git push -u origin main
```

当您的 GitHub 仓库准备就绪后，请继续至下一页。
