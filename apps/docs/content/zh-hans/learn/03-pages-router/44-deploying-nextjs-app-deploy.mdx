将 Next.js 应用部署到生产环境的最简单方式是使用由 Next.js 创建者开发的 **[Vercel](https://vercel.com)** 平台。

Vercel 是一个为静态和混合应用打造的无服务器平台，可与您的内容管理系统 (headless content)、电商平台或数据库无缝集成。我们让前端团队能够轻松开发、预览和交付卓越的用户体验，且默认具备高性能。您可以免费开始使用——无需信用卡。

### [创建 Vercel 账户](#create-a-vercel-account)

首先，访问 [Vercel](https://vercel.com/signup) 创建账户。选择 **通过 GitHub 继续** 并完成注册流程。

### [导入您的 `nextjs-blog` 仓库](#import-your-nextjs-blog-repository)

注册完成后，在 Vercel 上 **导入** 您的 `nextjs-blog` 仓库。您可以通过此链接操作：[Vercel](https://vercel.com/import)。

*   您需要 **安装 Vercel for GitHub**。可以授予其访问 **所有仓库** 的权限。
*   安装 Vercel 后，导入 `nextjs-blog` 项目。

以下设置可直接使用 _默认值_ —— 无需修改任何配置。Vercel 会自动检测到您拥有 Next.js 应用，并为您选择最优构建设置。

*   项目名称 (Project Name)
*   根目录 (Root Directory)
*   构建命令 (Build Command)
*   输出目录 (Output Directory)
*   开发命令 (Development Command)

部署时，您的 Next.js 应用将开始构建。通常在一分钟内即可完成。

> **获取帮助**：如果部署失败，您随时可以在 [GitHub Discussions](https://github.com/vercel/next.js/discussions) 寻求帮助。要了解更多部署信息，请查阅 [我们的文档](/docs/deployment)。

完成后，您将获得 **部署 URL**。点击任意 URL 即可看到 Next.js 初始页面已上线。

恭喜！您已成功将 Next.js 应用部署至生产环境。在下一页，我们将详细介绍 Vercel 平台及其推荐的工作流程。
