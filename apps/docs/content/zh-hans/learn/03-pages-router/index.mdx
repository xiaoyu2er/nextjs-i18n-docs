要从零开始使用 React 构建完整的 Web 应用程序，您需要考虑许多重要细节：

*   需要使用 webpack 等打包工具对代码进行打包，并通过 Babel 等编译器进行转换
*   需要实施生产环境优化措施，如代码分割 (code splitting)
*   可能需要对部分页面进行静态预渲染 (pre-render) 以提升性能和 SEO，同时可能还需要使用服务端渲染 (SSR) 或客户端渲染 (CSR)
*   可能需要编写一些服务端代码来将 React 应用连接到数据存储

**框架**可以解决这些问题。但这样的框架必须具备适当的抽象级别——否则它将不太实用。同时还需要具备出色的"开发者体验"，确保您和团队在编写代码时获得卓越体验。

### [Next.js：React 框架](#nextjs-the-react-framework)

**Next.js** 应运而生——这个 React 框架能解决上述所有问题。更重要的是，它能让您和团队在构建 React 应用时获得[成功保障](https://blog.codinghorror.com/falling-into-the-pit-of-success/)。

Next.js 致力于提供一流的开发者体验和众多内置功能，包括：

*   直观的[基于页面](/docs/basic-features/pages)的路由系统（支持[动态路由](/docs/routing/dynamic-routes)）
*   [预渲染](/docs/basic-features/pages#pre-rendering)支持，每个页面均可选择[静态生成](/docs/basic-features/pages#static-generation-recommended) (SSG) 或[服务端渲染](/docs/basic-features/pages#server-side-rendering) (SSR)
*   自动代码分割以加速页面加载
*   具备优化预取功能的[客户端路由](/docs/routing/introduction#linking-between-pages)
*   [内置 CSS](/docs/basic-features/built-in-css-support) 和 [Sass 支持](/docs/basic-features/built-in-css-support#sass-support)，兼容所有 [CSS-in-JS](/docs/basic-features/built-in-css-support#css-in-js) 方案
*   开发环境支持[快速刷新](/docs/basic-features/fast-refresh)
*   通过[API 路由](/docs/api-routes/introduction)使用 Serverless Functions 构建 API 端点
*   完全可扩展

Next.js 已被数以万计的面向生产环境的网站和 Web 应用采用，其中包括许多世界知名品牌。
