在 React 和 Next.js 中，您可以编写几种不同类型的测试，每种测试都有其特定的目的和用例。本页概述了可用于测试应用程序的类型及常用工具。

## 测试类型

- **单元测试 (Unit Testing)** 涉及隔离测试独立的代码单元（或代码块）。在 React 中，一个单元可以是一个函数、钩子或组件。
  - **组件测试 (Component Testing)** 是单元测试的一种更聚焦的形式，主要测试对象是 React 组件。这可能包括测试组件的渲染方式、与 props 的交互以及对用户事件的响应行为。
  - **集成测试 (Integration Testing)** 涉及测试多个单元如何协同工作。这可以是组件、钩子和函数的组合。
- **端到端测试 (End-to-End Testing / E2E Testing)** 在模拟真实用户场景（如浏览器环境）中对用户流程进行测试。这意味着需要在类生产环境中测试特定任务（例如注册流程）。
- **快照测试 (Snapshot Testing)** 通过捕获组件的渲染输出并将其保存为快照文件。测试运行时，会将组件当前的渲染输出与保存的快照进行比较。快照的变化用于指示行为中的意外变更。

## 异步服务端组件 (Async Server Components)

由于 `async` 服务端组件是 React 生态系统中的新特性，部分工具尚未完全支持它们。在此期间，我们建议对 `async` 组件使用**端到端测试 (End-to-End Testing)** 而非**单元测试 (Unit Testing)**。
