错误可分为两类：**预期错误**和**未捕获异常**：

- **将预期错误建模为返回值**：避免在服务端操作 (Server Actions) 中对预期错误使用 `try`/`catch`。使用 [`useActionState`](https://react.dev/reference/react/useActionState) 管理这些错误并将其返回至客户端。
- **对意外错误使用错误边界**：通过 `error.tsx` 和 `global-error.tsx` 文件实现错误边界，处理意外错误并提供备用 UI。

## 处理预期错误

预期错误指应用正常运行过程中可能出现的错误，例如来自[服务端表单验证](/docs/app/building-your-application/data-fetching/server-actions-and-mutations#server-side-form-validation)或失败请求的错误。这些错误应被显式处理并返回至客户端。

### 处理服务端操作的预期错误

使用 `useActionState` 钩子管理服务端操作的状态，包括错误处理。此方法避免对预期错误使用 `try`/`catch` 块，这类错误应建模为返回值而非抛出异常。

```ts filename="app/actions.ts" switcher
'use server'

import { redirect } from 'next/navigation'

export async function createUser(prevState: any, formData: FormData) {
  const res = await fetch('https://...')
  const json = await res.json()

  if (!res.ok) {
    return { message: '请输入有效的邮箱地址' }
  }

  redirect('/dashboard')
}
```

```js filename="app/actions.js" switcher
'use server'

import { redirect } from 'next/navigation'

export async function createUser(prevState, formData) {
  const res = await fetch('https://...')
  const json = await res.json()

  if (!res.ok) {
    return { message: '请输入有效的邮箱地址' }
  }

  redirect('/dashboard')
}
```

然后，您可以将操作传递给 `useActionState` 钩子，并使用返回的 `state` 显示错误信息。

```tsx filename="app/ui/signup.tsx" highlight={11,18-20} switcher
'use client'

import { useActionState } from 'react'
import { createUser } from '@/app/actions'

const initialState = {
  message: '',
}

export function Signup() {
  const [state, formAction, pending] = useActionState(createUser, initialState)

  return (
    <form action={formAction}>
      <label htmlFor="email">邮箱</label>
      <input type="text" id="email" name="email" required />
      {/* ... */}
      <p aria-live="polite">{state?.message}</p>
      <button disabled={pending}>注册</button>
    </form>
  )
}
```


您也可以使用返回的状态在客户端组件中显示 toast 通知。

### 处理服务端组件的预期错误

在服务端组件内获取数据时，您可以使用响应条件式渲染错误信息或执行[重定向](/docs/app/building-your-application/routing/redirecting#redirect-function)。

```tsx filename="app/page.tsx" switcher
export default async function Page() {
  const res = await fetch(`https://...`)
  const data = await res.json()

  if (!res.ok) {
    return '发生错误。'
  }

  return '...'
}
```

```jsx filename="app/page.js" switcher
export default async function Page() {
  const res = await fetch(`https://...`)
  const data = await res.json()

  if (!res.ok) {
    return '发生错误。'
  }

  return '...'
}
```

## 未捕获异常

未捕获异常是意外错误，表明应用中不应出现的 bug 或问题。这些错误应通过抛出异常来处理，随后会被错误边界捕获。

- **常见方式**：使用 `error.js` 处理根布局下方的未捕获错误。
- **可选方式**：通过嵌套的 `error.js` 文件（如 `app/dashboard/error.js`）处理细粒度未捕获错误。
- **不常见方式**：使用 `global-error.js` 处理根布局中的未捕获错误。

### 使用错误边界

Next.js 使用错误边界处理未捕获异常。错误边界会捕获子组件中的错误，并显示备用 UI 而非崩溃的组件树。

通过在路由段内创建 `error.tsx` 文件并导出 React 组件来建立错误边界：

```tsx filename="app/dashboard/error.tsx" switcher
'use client' // 错误边界必须是客户端组件

import { useEffect } from 'react'

export default function Error({
  error,
  reset,
}: {
  error: Error & { digest?: string }
  reset: () => void
}) {
  useEffect(() => {
    // 将错误记录至错误报告服务
    console.error(error)
  }, [error])

  return (
    <div>
      <h2>出错了！</h2>
      <button
        onClick={
          // 尝试通过重新渲染该段来恢复
          () => reset()
        }
      >
        重试
      </button>
    </div>
  )
}
```

```jsx filename="app/dashboard/error.js" switcher
'use client' // 错误边界必须是客户端组件

import { useEffect } from 'react'

export default function Error({ error, reset }) {
  useEffect(() => {
    // 将错误记录至错误报告服务
    console.error(error)
  }, [error])

  return (
    <div>
      <h2>出错了！</h2>

      </button>

```

若希望错误冒泡至父级错误边界，可在渲染 `error` 组件时使用 `throw`。

### 处理嵌套路由中的错误

错误会冒泡至最近的父级错误边界。通过在[路由层级](/docs/app/getting-started/project-structure#component-hierarchy)的不同位置放置 `error.tsx` 文件，可实现细粒度的错误处理。

### 处理全局错误

虽然较少见，但您可以使用位于根应用目录的 `app/global-error.js` 处理根布局中的错误，即使在使用[国际化](/docs/app/building-your-application/routing/internationalization)时也是如此。全局错误 UI 必须定义自己的 `<html>` 和 `<body>` 标签，因为它在激活时会替换根布局或模板。

```tsx filename="app/global-error.tsx" switcher
'use client' // 错误边界必须是客户端组件

export default function GlobalError({
  error,
  reset,
}: {
  error: Error & { digest?: string }
  reset: () => void
}) {
  return (
    // global-error 必须包含 html 和 body 标签
    <html>
      <body>
        <h2>出错了！</h2>
        <button onClick={() => reset()}>重试</button>
      </body>
    </html>
  )
}
```

