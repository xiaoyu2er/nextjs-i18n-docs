`default.js` 文件用于在 [并行路由 (Parallel Routes)](/docs/app/api-reference/file-conventions/parallel-routes) 中提供回退渲染，当 Next.js 在整页加载后无法恢复某个 [插槽 (slot)](/docs/app/api-reference/file-conventions/parallel-routes#slots) 的活动状态时使用。

在 [软导航 (soft navigation)](/docs/app/building-your-application/routing/linking-and-navigating#5-soft-navigation) 过程中，Next.js 会跟踪每个插槽的活动 _状态_（子页面）。然而，对于硬导航（整页加载），Next.js 无法恢复活动状态。在这种情况下，可以为不匹配当前 URL 的子页面渲染 `default.js` 文件。

考虑以下文件夹结构。`@team` 插槽有一个 `settings` 页面，但 `@analytics` 没有。

当导航到 `/settings` 时，`@team` 插槽会渲染 `settings` 页面，同时保持 `@analytics` 插槽当前的活动页面。

刷新页面时，Next.js 会为 `@analytics` 渲染 `default.js`。如果 `default.js` 不存在，则会渲染 `404` 页面。

此外，由于 `children` 是一个隐式插槽，您也需要创建一个 `default.js` 文件，以便在 Next.js 无法恢复父页面活动状态时为 `children` 提供回退渲染。

## 参考

### `params`（可选）

一个解析为包含从根段到插槽子页面的 [动态路由参数 (dynamic route parameters)](/docs/app/api-reference/file-conventions/dynamic-routes) 对象的 Promise。例如：

```tsx filename="app/[artist]/@sidebar/default.js" switcher
export default async function Default({
  params,
}: {
  params: Promise<{ artist: string }>
}) {
  const { artist } = await params
}
```

```jsx filename="app/[artist]/@sidebar/default.js" switcher
export default async function Default({ params }) {
  const { artist } = await params
}
```

| 示例                                      | URL           | `params`                                     |
| ---------------------------------------- | ------------- | -------------------------------------------- |
| `app/[artist]/@sidebar/default.js`       | `/zack`       | `Promise<{ artist: 'zack' }>`                |
| `app/[artist]/[album]/@sidebar/default.js` | `/zack/next`  | `Promise<{ artist: 'zack', album: 'next' }>` |

- 由于 `params` 属性是一个 Promise，您必须使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函数来访问其值。
  - 在版本 14 及更早版本中，`params` 是同步属性。为了向后兼容，在 Next.js 15 中仍然可以同步访问它，但这种行为将在未来被弃用。
