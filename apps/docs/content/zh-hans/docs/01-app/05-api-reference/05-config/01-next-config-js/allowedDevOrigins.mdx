Next.js 在开发过程中不会自动阻止跨域请求，但在未来的主要版本中默认会阻止，以防止未经授权请求开发模式下可用的内部资源/端点。

要配置 Next.js 应用允许来自服务器初始化主机名（默认为 `localhost`）以外的来源请求，您可以使用 `allowedDevOrigins` 配置选项。

`allowedDevOrigins` 允许您设置在开发模式下可使用的额外来源。例如，要使用 `local-origin.dev` 而不仅限于 `localhost`，请打开 `next.config.js` 并添加 `allowedDevOrigins` 配置：

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```
