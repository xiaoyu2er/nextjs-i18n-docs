当项目中存在 TypeScript 错误时，Next.js 会使您的 **生产环境构建** (`next build`) 失败。

如果您希望 Next.js 即使在应用存在错误时也强制生成生产环境代码（存在风险），可以禁用内置的类型检查步骤。

请注意：禁用后请确保在构建或部署流程中运行类型检查，否则可能造成严重后果。

打开 `next.config.js` 文件，在 `typescript` 配置中启用 `ignoreBuildErrors` 选项：

```js filename="next.config.js"
module.exports = {
  typescript: {
    // !! 警告 !!
    // 即使项目存在类型错误，也允许生产构建成功完成
    // !! 警告 !!
    ignoreBuildErrors: true,
  },
}
```
