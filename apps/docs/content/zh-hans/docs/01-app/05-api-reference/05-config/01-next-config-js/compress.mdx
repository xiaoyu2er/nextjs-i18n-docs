默认情况下，当使用 `next start` 或自定义服务器时，Next.js 会使用 `gzip` 压缩渲染内容和静态文件。这是针对未配置压缩的应用程序的优化。如果您的应用程序已通过自定义服务器配置了压缩，Next.js 将不会重复添加压缩功能。

您可以通过检查响应头中的 [`Accept-Encoding`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Accept-Encoding)（浏览器接受的编码选项）和 [`Content-Encoding`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Encoding)（当前使用的编码）来确认是否启用了压缩以及使用的压缩算法。

## 禁用压缩

要禁用**压缩**功能，请将 `compress` 配置选项设置为 `false`：

```js filename="next.config.js"
module.exports = {
  compress: false,
}
```

我们**不建议禁用压缩**，除非您已在服务器上配置了压缩功能，因为压缩可以减少带宽使用并提升应用程序性能。例如，如果您正在使用 [nginx](https://nginx.org/) 并希望改用 `brotli` 压缩，可以将 `compress` 选项设为 `false` 以让 nginx 处理压缩。
