Next.js 提供了两种可在应用程序中使用的服务器运行时：

- **Node.js 运行时**（默认），可访问所有 Node.js API，用于渲染应用程序。
- **Edge 运行时**，包含更有限的 [API 集合](#reference)，用于 [中间件](/docs/app/api-reference/file-conventions/middleware)。

## 注意事项

- Edge 运行时不支持所有 Node.js API，某些包可能无法按预期工作。
- Edge 运行时不支持增量静态再生 (ISR)。
- 根据您的部署适配器，两种运行时都可以支持 [流式传输](/docs/app/api-reference/file-conventions/loading)。

### 不支持的 API

边缘运行时 (Edge Runtime) 存在以下限制：

- **不支持**原生 Node.js API。例如无法读写文件系统。
- 可以使用 `node_modules`，但必须实现 ES 模块规范且不使用原生 Node.js API。
- **不允许**直接调用 `require`，请改用 ES 模块。

以下 JavaScript 语言特性已被禁用，**无法正常工作**：

| API                                                                                                                             | 描述                                                                 |
| ------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------- |
| [`eval`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/eval)                                       | 执行以字符串形式表示的 JavaScript 代码                                |
| [`new Function(evalString)`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Function)               | 通过代码字符串创建新函数                                              |
| [`WebAssembly.compile`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/compile)         | 从缓冲区源码编译 WebAssembly 模块                                     |
| [`WebAssembly.instantiate`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/instantiate) | 从缓冲区源码编译并实例化 WebAssembly 模块                             |

在极少数情况下，你的代码可能包含（或导入）某些动态代码执行语句，这些语句_在运行时无法被访问_且无法通过摇树优化 (treeshaking) 移除。可以通过中间件配置允许特定文件绕过检查：

```javascript filename="middleware.ts"
export const config = {
  unstable_allowDynamic: [
    // 允许单个文件
    '/lib/utilities.js',
    // 使用通配符允许第三方模块 function-bind 中的所有内容
    '**/node_modules/function-bind/**',
  ],
}
```

`unstable_allowDynamic` 支持 [glob](https://github.com/micromatch/micromatch#matching-features) 模式或 glob 数组，用于忽略特定文件的动态代码执行检查。glob 路径相对于应用程序根目录。

请注意：如果这些语句在边缘运行时中执行，_将会抛出异常并导致运行时错误_。
