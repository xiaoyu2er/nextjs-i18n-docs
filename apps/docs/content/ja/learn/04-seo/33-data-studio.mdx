パフォーマンスを測定するもう1つの優れた無料オープンソース方法として、[Chrome ユーザーエクスペリエンスレポート](https://developers.google.com/web/tools/chrome-user-experience-report)データセットを使用する方法があります。

[Chrome ユーザーエクスペリエンスレポート](https://developers.google.com/web/tools/chrome-user-experience-report)は、実際のChromeユーザーがウェブ上の主要なサイトをどのように体験しているかに関するユーザーエクスペリエンスメトリクスを提供します。

このデータセットは[BigQuery](https://console.cloud.google.com/bigquery?project=chrome-ux-report)で公開されており、[Google Data Studio](https://datastudio.google.com/overview)でも完全無料で利用できます。

幸いなことに、[オープンソースのダッシュボード](https://g.co/chromeuxdash)がテンプレートとして利用可能で、ウェブサイトのパフォーマンス追跡に使用できます。

このデータセットの唯一の欠点は、CrUXレポートにサイトが含まれるためには一定量の訪問が必要であり、データ不足のためレポートに含まれない可能性があることです。そのため、制作途中や新しく作成されたサイトには最適な選択肢ではないかもしれません。

また、データは月次で更新されます。通常は月終了後約15日後に更新されるため、データにはある程度の遅延が生じます。コアウェブバイタルスコアの改善に取り組む場合、これが実用的でない場合があります。
