[Web Vitals](https://web.dev/vitals/) は、Googleが作成したイニシアチブで、ウェブ上のエンドユーザーページエクスペリエンスを測定するための統一されたガイダンスとメトリクスを提供します。

[コアウェブバイタル (Core Web Vitals)](https://developers.google.com/search/blog/2020/11/timing-for-page-experience) はWeb Vitalsのサブセットで、現在は読み込み速度、インタラクティブ性、視覚的安定性を測定する3つのメトリクスで構成されています。これらのメトリクスは [Largest Contentful Paint (LCP)](/learn/seo/lcp)、[First Input Delay (FID)](/learn/seo/fid)、[Cumulative Layout Shift (CLS)](/learn/seo/cls) です。

これら3つのメトリクスで高いスコアを達成することで、ユーザーにとってよりスムーズなウェブサイト体験を提供できます。

各コアウェブバイタルメトリクスで低いスコアのウェブサイトは、Googleが検索アルゴリズムの[ランキング要素としてコアウェブバイタルを使用](https://developers.google.com/search/blog/2020/11/timing-for-page-experience)し始めたため、検索エンジンのランキングに影響を与えます。バイタルが低いと、ウェブトラフィックやビジネスに影響を及ぼす可能性があります。

このレッスンでは以下を学びます:

* コアウェブバイタルの簡単な背景
* SEOとUXにおけるコアウェブバイタルの意味合い、およびウェブサイトへの影響
* 開発プロセスでコアウェブバイタルを重視すべき理由とその測定方法
* Next.jsを使用してコアウェブバイタルを改善し、変更を監視する方法
