2016年、Googleは[AMP](https://amp.dev/)を使用したウェブページに[検索ランキングの優先度](https://developers.google.com/search/docs/guides/about-amp)を与え始めました。AMPはモバイルデバイスでより速く読み込まれるウェブページを作成できる技術ですが、構築と維持にはコストがかかります。

[Core Web Vitals](https://web.dev/vitals/)のページエクスペリエンス更新により、Googleは検索カルーセルに表示されるための[AMPページの要件を撤廃](https://developers.google.com/search/blog/2021/04/more-details-page-experience#details)しました。これはSEO目的でGoogleがAMPに与えていた最後の主要な利点の1つでした。

AMPの導入以来、Next.jsのような新しい技術は、開発者エクスペリエンス（DX）を犠牲にすることなくウェブサイト体験を向上させる能力を証明しています。

Next.jsは[AMPサポート](https://nextjs.org/docs/advanced-features/amp-support/introduction)を提供していますが、すでに優れたCore Web Vitalsスコアを持っているウェブサイトでAMPを実装するコストと利点を検討することをお勧めします。
