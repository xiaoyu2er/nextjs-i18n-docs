フィールドデータ収集には以下のツールが利用できます:

*   [PageSpeed Insights](https://developers.google.com/speed/pagespeed/insights/): Googleのページ速度測定ツール
*   [Chrome User Experience Report](https://developers.google.com/web/tools/chrome-user-experience-report): フィールドデータのオープンソースデータセット
*   [Search Console](https://support.google.com/webmasters/answer/9205520): Google SearchのCore Web Vitalsレポート

ラボデータが必要な場合、Googleは以下の測定ツールも提供しています:

*   [Lighthouse](https://developers.google.com/web/tools/lighthouse?hl=en): Webページの品質向上のためのGoogleのオープンソース自動化ツール
*   [Chrome DevTools](https://developer.chrome.com/docs/devtools/): Google Chromeブラウザに直接組み込まれたWeb開発者向けツールセット

両ツールにおいては、[First Input Delay (FID)](https://web.dev/fid/)の代わりに[Total Blocking Time (TBT)](https://web.dev/tbt/)を使用する必要があることに注意してください。FIDはフィールドデータを通じてのみ測定可能なためです。
