ソースマップは開発時にはデフォルトで有効化されています。本番ビルド時には、クライアント側にソースコードが漏洩するのを防ぐため、設定フラグで明示的に有効化しない限り無効化されます。

Next.jsでは、本番ビルド時にブラウザ向けソースマップ生成を有効化する設定フラグを提供しています:

```js filename="next.config.js"
module.exports = {
  productionBrowserSourceMaps: true,
}
```

`productionBrowserSourceMaps`オプションが有効化されている場合、ソースマップはJavaScriptファイルと同じディレクトリに出力されます。Next.jsはリクエスト時にこれらのファイルを自動的に提供します。

- ソースマップを追加すると`next build`の時間が増加する可能性があります
- `next build`時のメモリ使用量が増加します
