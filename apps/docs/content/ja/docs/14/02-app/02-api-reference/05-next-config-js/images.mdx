Next.jsの組み込み画像最適化APIの代わりにクラウドプロバイダーを使用して画像を最適化したい場合、以下のように`next.config.js`を設定できます:

```js filename="next.config.js"
module.exports = {
  images: {
    loader: 'custom',
    loaderFile: './my/image/loader.js',
  },
}
```

この`loaderFile`はNext.jsアプリケーションのルートからの相対パスで指定する必要があります。ファイルは文字列を返すデフォルト関数をエクスポートする必要があります。例:

<AppOnly>

```js filename="my/image/loader.js"
'use client'

export default function myImageLoader({ src, width, quality }) {
  return `https://example.com/${src}?w=${width}&q=${quality || 75}`
}
```

または、[`loader` prop](/docs/app/api-reference/components/image#loader)を使用して、各`next/image`インスタンスに関数を渡すこともできます。

> **知っておくと便利**: 関数を受け取る画像ローダーファイルをカスタマイズするには、提供された関数をシリアライズするために[クライアントコンポーネント](/docs/app/building-your-application/rendering/client-components)を使用する必要があります。

組み込みの[画像最適化API](/docs/app/building-your-application/optimizing/images)と[画像コンポーネント](/docs/app/api-reference/components/image)の動作設定について詳しくは、[画像設定オプション](/docs/app/api-reference/components/image#configuration-options)で利用可能なオプションを確認してください。

</AppOnly>

### Akamai

```js
// ドキュメント: https://techdocs.akamai.com/ivm/reference/test-images-on-demand
export default function akamaiLoader({ src, width, quality }) {
  return `https://example.com/${src}?imwidth=${width}`
}
```

### AWS CloudFront

```js
// ドキュメント: https://aws.amazon.com/developer/application-security-performance/articles/image-optimization
export default function cloudfrontLoader({ src, width, quality }) {
  const url = new URL(`https://example.com${src}`)
  url.searchParams.set('format', 'auto')
  url.searchParams.set('width', width.toString())
  url.searchParams.set('quality', (quality || 75).toString())
  return url.href
```

### Cloudinary

```js
// デモ: https://res.cloudinary.com/demo/image/upload/w_300,c_limit,q_auto/turtles.jpg
export default function cloudinaryLoader({ src, width, quality }) {
  const params = ['f_auto', 'c_limit', `w_${width}`, `q_${quality || 'auto'}`]
  return `https://example.com/${params.join(',')}${src}`
}
```

### Cloudflare

```js
// ドキュメント: https://developers.cloudflare.com/images/url-format
export default function cloudflareLoader({ src, width, quality }) {

  return `https://example.com/cdn-cgi/image/${params.join(',')}/${src}`
}
```

### ImageEngine

```js
// ドキュメント: https://support.imageengine.io/hc/en-us/articles/360058880672-Directives
export default function imageengineLoader({ src, width, quality }) {
  const compression = 100 - (quality || 50)
  const params = [`w_${width}`, `cmpr_${compression}`)]
  return `https://example.com${src}?imgeng=/${params.join('/')`
}
```

### Imgix

```js
// デモ: https://static.imgix.net/daisy.png?format=auto&fit=max&w=300
export default function imgixLoader({ src, width, quality }) {
  const url = new URL(`https://example.com${src}`)
  const params = url.searchParams
  params.set('auto', params.getAll('auto').join(',') || 'format')
  params.set('fit', params.get('fit') || 'max')
  params.set('w', params.get('w') || width.toString())
  params.set('q', (quality || 50).toString())
  return url.href
}
```

### Sanity

```js
// ドキュメント: https://www.sanity.io/docs/image-urls
export default function sanityLoader({ src, width, quality }) {
  const prj = 'zp7mbokg'
  const dataset = 'production'
  const url = new URL(`https://cdn.sanity.io/images/${prj}/${dataset}${src}`)
  url.searchParams.set('auto', 'format')
  url.searchParams.set('fit', 'max')
  url.searchParams.set('w', width.toString())
  if (quality) {
    url.searchParams.set('q', quality.toString())
  }
  return url.href
}
```
