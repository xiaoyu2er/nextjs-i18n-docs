ソースマップは開発時にはデフォルトで有効になっています。プロダクションビルド時には、設定フラグで明示的に有効にしない限り、ソースコードがクライアントに漏れるのを防ぐために無効化されます。

Next.jsでは、プロダクションビルド時にブラウザ用ソースマップを生成するための設定フラグを提供しています：

```js filename="next.config.js"
module.exports = {
  productionBrowserSourceMaps: true,
}
```

`productionBrowserSourceMaps` オプションが有効な場合、ソースマップはJavaScriptファイルと同じディレクトリに出力されます。Next.jsはリクエスト時にこれらのファイルを自動的に提供します。

- ソースマップを追加すると `next build` の時間が増加する可能性があります
- `next build` 時のメモリ使用量が増加します
