Eine weitere großartige, kostenlose und quelloffene Möglichkeit, Ihre Performance zu messen, ist die Verwendung des [Chrome User Experience Report](https://developers.google.com/web/tools/chrome-user-experience-report)-Datensatzes.

Der [Chrome User Experience Report](https://developers.google.com/web/tools/chrome-user-experience-report) liefert Nutzererfahrungsmetriken darüber, wie reale Chrome-Nutzer beliebte Webseiten erleben.

Dieser Datensatz ist öffentlich verfügbar [über BigQuery](https://console.cloud.google.com/bigquery?project=chrome-ux-report) und kann auch komplett kostenlos über [Google Data Studio](https://datastudio.google.com/overview) genutzt werden.

Glücklicherweise gibt es [quelloffene Dashboards](https://g.co/chromeuxdash), die Sie als Vorlage für die Performance-Verfolgung Ihrer Website verwenden können.

Der einzige Nachteil dieses Datensatzes ist, dass Ihre Website, um in den CrUX-Report aufgenommen zu werden, eine ausreichende Anzahl an Besuchen haben muss. Andernfalls wird sie aufgrund unzureichender Daten nicht in den Report aufgenommen. Daher ist dies möglicherweise nicht die beste Option für Websites in Arbeit oder neu erstellte Websites.

Außerdem werden die Daten monatlich aktualisiert, üblicherweise etwa 15 Tage nach Monatsende. Sie sehen die Daten also mit einer gewissen Verzögerung, was nicht besonders praktisch ist, wenn Sie an der Verbesserung Ihrer Core Web Vitals-Werte arbeiten möchten.
