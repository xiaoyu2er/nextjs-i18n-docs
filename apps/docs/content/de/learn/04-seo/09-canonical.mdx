Eine **kanonische** URL ist die URL der Seite, die Suchmaschinen als die repräsentativste aus einer Gruppe doppelter Seiten auf Ihrer Website betrachten.

Während Sie kanonische URLs direkt an Suchmaschinen kommunizieren können, können diese auch entscheiden, mehrere URLs ohne Ihre Benachrichtigung zu gruppieren. Dies kann automatisch geschehen, wenn Google eine URL unter verschiedenen Pfaden findet.

Obwohl Google gute Arbeit bei der Erkennung solcher Fälle leistet, arbeiten seine Systeme in enormem Maßstab und decken nicht alle Randfälle ab. Canonical Tags sind ein wichtiger Aspekt für Ihre Website, um eine optimale Performance sicherzustellen.

Wenn Google mehrere URLs mit demselben Inhalt findet, könnte es diese in den Suchergebnissen herabstufen, da sie als Duplikate betrachtet werden können.

Dies passiert auch domainsübergreifend: Wenn Sie zwei verschiedene Websites betreiben und denselben Inhalt auf beiden veröffentlichen, können Suchmaschinen entscheiden, eine davon für das Ranking auszuwählen oder beide herabzustufen.

Hier kommen Canonical Tags ins Spiel. Sie teilen Google mit, welche URLs die ursprüngliche Quelle sind und welche Duplikate darstellen. Viele doppelte Seiten innerhalb derselben oder verschiedener Domains können zu schlechten Rankings oder sogar Strafen führen.

Stellen wir uns vor, unser E-Shop ermöglicht den Zugriff auf Produkte über [example.com/products/phone](https://example.com/products/phone) und [example.com/phone](http://example.com/phone).

Beides sind gültige, funktionierende URLs, aber wir verwenden Canonical Tags, um die Erkennung von doppeltem Inhalt, den wir besitzen, zu verhindern. Wenn wir entscheiden, dass `https://example.com/products/phone` für das Ranking berücksichtigt werden soll, würden wir einen Canonical Tag erstellen:

```
<link rel="canonical" href="https://example.com/products/phone" />
```

Canonical Tags sind grundlegend für die SEO-Performance, denn nicht nur Sie können verschiedene URLs erstellen, sondern auch Benutzer oder Marketing-Tools.

Stellen Sie sich vor, Sie führen einige Marketingkampagnen auf Google durch, dann entscheidet Google, einige [UTM-Parameter](https://ga-dev-tools.appspot.com/campaign-url-builder/) hinzuzufügen. Es ist möglich, dass diese neue, eindeutige URL von Googlebot indexiert wird, daher sollten Sie sicherstellen, dass Sie weiterhin Ihre Canonical Tags anzeigen, um doppelte Seiten zu vereinheitlichen.

#### [Beispiel](#example)

```
import Head from 'next/head';
 
function IndexPage() {
  return (
    <div>
      <Head>
        <title>Canonical Tag Example</title>
        <link
          rel="canonical"
          href="https://example.com/blog/original-post"
          key="canonical"
        />
      </Head>
      <p>This post exists on two URLs.</p>
    </div>
  );
}
 
export default IndexPage;
```
