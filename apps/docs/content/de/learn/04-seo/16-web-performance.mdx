[Web Vitals](https://web.dev/vitals/) ist eine Initiative von Google, die einheitliche Richtlinien und Metriken bereitstellt, um das Nutzererlebnis auf Webseiten zu messen.

[Core Web Vitals](https://developers.google.com/search/blog/2020/11/timing-for-page-experience) sind eine Teilmenge der Web Vitals und bestehen derzeit aus drei Metriken, die Ladezeit, Interaktivität und visuelle Stabilität messen. Diese Metriken sind [Largest Contentful Paint (LCP)](/learn/seo/lcp), [First Input Delay (FID)](/learn/seo/fid) und [Cumulative Layout Shift (CLS)](/learn/seo/cls).

Ein gutes Ergebnis in diesen drei Metriken führt zu einer reibungsloseren Nutzererfahrung auf Ihrer Website.

Webseiten mit schlechten Werten in den Core Web Vitals-Metriken werden in den Suchmaschinenrankings beeinträchtigt, da Google [Core Web Vitals als Rankingfaktor](https://developers.google.com/search/blog/2020/11/timing-for-page-experience) für seinen Suchalgorithmus nutzt. Schlechte Vitals können sich auf Ihren Webverkehr und Ihr Geschäft auswirken.

In dieser Lektion lernen Sie:

*   Einen kurzen Hintergrund zu Core Web Vitals.
*   Die Auswirkungen von Core Web Vitals auf SEO und UX sowie deren Einfluss auf Ihre Website.
*   Warum Sie Core Web Vitals in Ihrem Entwicklungsprozess berücksichtigen sollten und wie Sie sie messen können.
*   Wie Sie Ihre Core Web Vitals mit Next.js verbessern und Änderungen überwachen können.
