Das Hauptziel der Google-Suchmaschine besteht darin, Nutzern die bestmöglichen Ergebnisse zu liefern, wobei Lokalisierung und Tippfehler berücksichtigt werden. In jedem Fall legt Google Wert darauf, Nutzern schnelle und reibungslose Ergebnisse mit einer hervorragenden Website-Erfahrung zu bieten. Die [Ladegeschwindigkeit von Webseiten](https://developers.google.com/web/updates/2018/07/search-ads-speed) auf Mobilgeräten ist seit 2018 ein Ranking-Faktor. Bis jetzt war jedoch nicht klar, welche spezifischen Leistungsmetriken der Google-Suchalgorithmus für das Ranking verwendet.

Dies änderte sich im Juni 2021, als Google [eine Reihe spezifischer Metriken und Bereiche](https://developers.google.com/search/blog/2021/04/more-details-page-experience) zur Analyse und Optimierung der Leistung bereitstellte.

### [Gewichtung der Web Vitals in Lighthouse (v6)](#lighthouse-v6-weights-for-vitals)

Die drei Metriken werden nicht unbedingt gleich gewichtet. In [Lighthouse](https://developers.google.com/web/tools/lighthouse) werden den Core Web Vitals unterschiedliche Gewichtungen zugewiesen:

*   **Largest Contentful Paint**: 25%
*   **Total Blocking Time**: 25%
*   **First Contentful Paint**: 15%
*   **Speed Index**: 15%
*   **Time to Interactive**: 15%
*   **Cumulative Layout Shift**: 5%

Dies ähnelt dem [First Input Delay](/learn/seo/fid).

**Hinweis:** Die Auswirkungen auf das Google-Ranking sind für alle Seiten im guten Bereich aller Core Web Vitals gleich, unabhängig von ihren individuellen Core Web Vitals-Werten.

### [Auswirkungen auf die Nutzererfahrung](#ux-impact)

Die meisten Diskussionen über Core Web Vitals konzentrieren sich primär auf SEO.

Zwar sind Core Web Vitals eine Initiative, die darauf abzielt, die Seitenqualität und Suchrankings zu messen und zu verbessern, aber letztendlich profitieren die Nutzer von diesen Veränderungen.

Core Web Vitals helfen dabei, die bestmögliche Nutzererfahrung zu erreichen. Laut einer [von Amazon im Jahr 2012 durchgeführten Studie](https://www.fastcompany.com/1825005/how-one-second-could-cost-amazon-16-billion-sales) könnte eine zusätzliche Sekunde Ladezeit das Unternehmen potenziell 1,6 Milliarden US-Dollar kosten. Studien wie diese zeigen die Bedeutung einer hervorragenden Seitenqualität und einer schnellen Website, beides Ziele, die durch Core Web Vitals gefördert werden.
