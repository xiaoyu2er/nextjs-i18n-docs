Der einfachste Weg, Next.js in der Produktion bereitzustellen, ist die Verwendung der **[Vercel](https://vercel.com)**-Plattform, die von den Entwicklern von Next.js erstellt wurde.

Vercel ist eine serverlose Plattform für statische und hybride Anwendungen, die entwickelt wurde, um mit Ihrem Headless-Content, Commerce oder Datenbanken zu integrieren. Wir machen es Frontend-Teams leicht, ansprechende Benutzererlebnisse zu entwickeln, in der Vorschau zu betrachten und auszuliefern, wobei Leistung der Standard ist. Sie können es kostenlos nutzen – keine Kreditkarte erforderlich.

### [Vercel-Konto erstellen](#create-a-vercel-account)

Gehen Sie zunächst zu [Vercel](https://vercel.com/signup), um ein Vercel-Konto zu erstellen. Wählen Sie **Mit GitHub fortfahren** und durchlaufen Sie den Anmeldeprozess.

### [Ihr `nextjs-blog`-Repository importieren](#import-your-nextjs-blog-repository)

Nach der Anmeldung können Sie Ihr `nextjs-blog`-Repository auf Vercel **importieren**. Dies können Sie hier tun: [Vercel](https://vercel.com/import).

*   Sie müssen **Vercel für GitHub installieren**. Sie können ihm Zugriff auf **Alle Repositories** gewähren.
*   Nach der Installation von Vercel können Sie `nextjs-blog` importieren.

Sie können für die folgenden Einstellungen die _Standardwerte_ verwenden – es ist keine Änderung erforderlich. Vercel erkennt automatisch, dass Sie eine Next.js-App haben, und wählt optimale Build-Einstellungen für Sie aus.

*   Projektname
*   Stammverzeichnis
*   Build-Befehl
*   Ausgabeverzeichnis
*   Entwicklungsbefehl

Wenn Sie die Bereitstellung starten, beginnt der Build Ihrer Next.js-App. Dieser sollte in weniger als einer Minute abgeschlossen sein.

> **Hilfe ist verfügbar**: Falls Ihre Bereitstellung fehlschlägt, können Sie jederzeit Hilfe in den [GitHub Discussions](https://github.com/vercel/next.js/discussions) erhalten. Um mehr über die Bereitstellung zu erfahren, werfen Sie einen Blick in [unsere Dokumentation](/docs/deployment).

Wenn der Build abgeschlossen ist, erhalten Sie **Bereitstellungs-URLs**. Klicken Sie auf eine der URLs und Sie sollten die Next.js-Startseite live sehen.

Glückwunsch! Sie haben gerade Ihre Next.js-App in der Produktion bereitgestellt. Auf der nächsten Seite gehen wir näher auf Vercel und den empfohlenen Workflow ein.
