Bevor wir die Anwendung bereitstellen, pushen wir unsere Next.js-App auf [GitHub](https://github.com/vercel/next.js), falls Sie dies noch nicht getan haben. Dies erleichtert die Bereitstellung.

*   Erstellen Sie in Ihrem persönlichen GitHub-Account ein neues Repository mit dem Namen `nextjs-blog`.
*   Das Repository kann öffentlich oder privat sein. Sie müssen es **nicht** mit einer README oder anderen Dateien initialisieren.
*   Falls Sie Hilfe bei der Einrichtung Ihres Repositories benötigen, lesen Sie [diese Anleitung auf GitHub](https://help.github.com/en/github/getting-started-with-github/create-a-repo).

Dann:

*   Falls Sie das Git-Repository lokal für Ihre Next.js-App noch nicht initialisiert haben, tun Sie dies jetzt.
*   Pushen Sie die Next.js-App in Ihr GitHub-Repository.

Um auf GitHub zu pushen, können Sie die folgenden Befehle ausführen (ersetzen Sie `<username>` mit Ihrem GitHub-Benutzernamen):

```
git remote add origin https://github.com/<username>/nextjs-blog.git
git push -u origin main
```

Sobald Ihr GitHub-Repository bereit ist, fahren Sie mit der nächsten Seite fort.
