Next.js kann auf jedem Hosting-Anbieter bereitgestellt werden, der Node.js unterstützt.

Wenn Sie den bisherigen Anweisungen gefolgt sind, sollte Ihre `package.json` die folgenden `build`- und `start`-Skripte enthalten:

```
{
  "scripts": {
    "dev": "next",
    "build": "next build",
    "start": "next start"
  }
}
```

Führen Sie auf Ihrem eigenen Hosting-Anbieter einmal das `build`-Skript aus, welches die Produktionsanwendung im `.next`-Ordner erstellt.

```
npm run build
```

Nach dem Build startet das `start`-Skript einen Node.js-Server, der hybride Seiten unterstützt, sowohl statisch generierte als auch serverseitig gerenderte Seiten sowie API-Routen bereitstellt.

```
npm run start
```

> **Tipp**: Sie können das `start`-Skript in `package.json` anpassen, um einen `PORT`-Parameter zu akzeptieren, indem Sie es wie folgt aktualisieren: `"start": "next start -p $PORT"`.
