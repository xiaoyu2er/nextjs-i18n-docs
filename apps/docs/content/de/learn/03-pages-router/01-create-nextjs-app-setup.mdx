Zunächst stellen wir sicher, dass Ihre Entwicklungsumgebung bereit ist.

*   Falls Sie **Node.js** noch nicht installiert haben, [installieren Sie es hier](https://nodejs.org/en/). Sie benötigen Node.js Version **18** oder höher.
*   Für dieses Tutorial verwenden Sie Ihren eigenen Texteditor und Terminal.

> Falls Sie Windows nutzen, empfehlen wir [Git für Windows herunterzuladen](https://gitforwindows.org/) und das mitgelieferte Git Bash zu verwenden, das die UNIX-spezifischen Befehle in diesem Tutorial unterstützt. [Windows Subsystem for Linux (WSL)](https://docs.microsoft.com/en-us/windows/wsl/install-win10) ist eine weitere Option.

### [Erstellen einer Next.js-App](#create-a-nextjs-app)

Um eine Next.js-App zu erstellen, öffnen Sie Ihr Terminal, navigieren Sie mit `cd` in das gewünschte Verzeichnis und führen Sie folgenden Befehl aus:

```
npx create-next-app@latest nextjs-blog --use-npm --example "https://github.com/vercel/next-learn/tree/main/basics/learn-starter"
```

> Intern verwendet dieser Befehl das Tool [`create-next-app`](/docs/api-reference/create-next-app), das eine Next.js-App für Sie vorbereitet. Es nutzt [diese Vorlage](https://github.com/vercel/next-learn/tree/main/basics/learn-starter) über das `--example`-Flag.
> 
> Falls es nicht funktioniert, lesen Sie bitte [diese Seite](https://github.com/vercel/next-learn/blob/main/basics/errors/install.md).

### [Starten des Entwicklungsservers](#run-the-development-server)

Sie haben nun ein neues Verzeichnis namens `nextjs-blog`. Wechseln Sie mit `cd` hinein:

```
cd nextjs-blog
```

Führen Sie anschließend folgenden Befehl aus:

```
npm run dev
```

Dies startet den "Entwicklungsserver" Ihrer Next.js-App (mehr dazu später) auf Port **3000**.

Prüfen wir, ob es funktioniert. Öffnen Sie [http://localhost:3000](http://localhost:3000) in Ihrem Browser.
