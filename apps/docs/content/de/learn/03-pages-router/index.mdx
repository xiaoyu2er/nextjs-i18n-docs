Um eine komplette Webanwendung mit React von Grund auf zu erstellen, gibt es viele wichtige Details zu beachten:

*   Code muss mit einem Bundler wie webpack gebündelt und mit einem Compiler wie Babel transformiert werden.
*   Produktionsoptimierungen wie Code-Splitting sind erforderlich.
*   Möglicherweise möchten Sie einige Seiten statisch vor-rendern, um Performance und SEO zu verbessern. Sie könnten auch serverseitiges Rendering oder clientseitiges Rendering nutzen.
*   Eventuell müssen Sie serverseitigen Code schreiben, um Ihre React-App mit Ihrem Datenspeicher zu verbinden.

Ein _Framework_ kann diese Probleme lösen. Aber ein solches Framework muss das richtige Abstraktionsniveau haben - sonst ist es nicht sehr nützlich. Es muss auch eine großartige "Developer Experience" bieten, damit Sie und Ihr Team eine hervorragende Erfahrung beim Schreiben von Code haben.

### [Next.js: Das React-Framework](#nextjs-the-react-framework)

Hier kommt **Next.js**, das React-Framework. Next.js bietet eine Lösung für all diese Probleme. Noch wichtiger ist, dass es Sie und Ihr Team beim Erstellen von React-Anwendungen in die [Pit of Success](https://blog.codinghorror.com/falling-into-the-pit-of-success/) führt.

Next.js zielt darauf ab, eine erstklassige Developer Experience und viele integrierte Funktionen zu bieten, wie:

*   Ein intuitives [seitenbasiertes](/docs/basic-features/pages) Routing-System (mit Unterstützung für [dynamische Routen](/docs/routing/dynamic-routes))
*   [Pre-Rendering](/docs/basic-features/pages#pre-rendering), sowohl [statische Generierung](/docs/basic-features/pages#static-generation-recommended) (SSG) als auch [serverseitiges Rendering](/docs/basic-features/pages#server-side-rendering) (SSR) werden pro Seite unterstützt
*   Automatisches Code-Splitting für schnellere Seitenladezeiten
*   [Clientseitiges Routing](/docs/routing/introduction#linking-between-pages) mit optimiertem Prefetching
*   [Integrierte CSS-](/docs/basic-features/built-in-css-support) und [Sass-Unterstützung](/docs/basic-features/built-in-css-support#sass-support) sowie Unterstützung für jede [CSS-in-JS](/docs/basic-features/built-in-css-support#css-in-js)-Bibliothek
*   Entwicklungsumgebung mit [Fast Refresh](/docs/basic-features/fast-refresh)-Unterstützung
*   [API-Routen](/docs/api-routes/introduction) zum Erstellen von API-Endpunkten mit Serverless Functions
*   Vollständig erweiterbar

Next.js wird in zehntausenden produktiven Websites und Webanwendungen eingesetzt, darunter viele der weltweit größten Marken.
