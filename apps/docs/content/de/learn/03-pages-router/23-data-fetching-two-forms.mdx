Next.js bietet zwei Formen des Pre-rendering: [**Static Generation**](/docs/basic-features/pages#static-generation-recommended) und [**Server-side Rendering**](/docs/basic-features/pages#server-side-rendering). Der Unterschied liegt darin, **wann** das HTML für eine Seite generiert wird.

*   [**Static Generation**](/docs/basic-features/pages#static-generation-recommended) ist die Pre-rendering-Methode, die das HTML **zur Build-Zeit** generiert. Das vorgerenderte HTML wird dann bei _jeder Anfrage wiederverwendet_.
*   [**Server-side Rendering**](/docs/basic-features/pages#server-side-rendering) ist die Pre-rendering-Methode, die das HTML bei **jeder Anfrage** neu generiert.

![Static Generation](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/pages-router/data-fetching/static-generation.png)

![Server-side Rendering](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/pages-router/data-fetching/server-side-rendering.png)

> Im Entwicklungsmodus (wenn Sie `npm run dev` oder `yarn dev` ausführen), werden Seiten bei [jeder Anfrage vorgerendert](/docs/basic-features/pages#pre-rendering). Dies gilt auch für die [Static Generation](https://nextjs.org/docs/basic-features/data-fetching/get-static-props#runs-on-every-request-in-development), um die Entwicklung zu vereinfachen. In der Produktionsumgebung erfolgt Static Generation nur einmal zur Build-Zeit und **nicht** bei jeder Anfrage.

### [Pro-Seite-Basis](#per-page-basis)

Wichtig ist, dass Next.js Ihnen die **Wahl** lässt, welche Pre-rendering-Methode für jede Seite verwendet wird. Sie können eine "hybride" Next.js-App erstellen, indem Sie für die meisten Seiten [Static Generation](/docs/basic-features/pages#static-generation-recommended) und für andere [Server-side Rendering](/docs/basic-features/pages#server-side-rendering) verwenden.

### [Wann sollte man](#when-to-use-static-generation-vs-server-side-rendering) [Static Generation](/docs/basic-features/pages#static-generation-recommended) vs. [Server-side Rendering](/docs/basic-features/pages#server-side-rendering) verwenden?

Wir empfehlen, wann immer möglich [**Static Generation**](/docs/basic-features/pages#static-generation-recommended) (mit oder ohne Daten) zu verwenden, da Ihre Seite einmal gebaut und von einer CDN bereitgestellt werden kann, was sie viel schneller macht als eine Seite, die bei jeder Anfrage vom Server gerendert wird.

Sie können [Static Generation](/docs/basic-features/pages#static-generation-recommended) für viele Arten von Seiten verwenden, einschließlich:

*   Marketing-Seiten
*   Blog-Beiträge
*   E-Commerce-Produktlisten
*   Hilfe- und Dokumentationsseiten

Sie sollten sich fragen: "Kann ich diese Seite **vor** der Anfrage eines Nutzers vorrendern?" Wenn die Antwort ja lautet, sollten Sie [Static Generation](/docs/basic-features/pages#static-generation-recommended) wählen.

Andererseits ist [Static Generation](/docs/basic-features/pages#static-generation-recommended) **keine** gute Idee, wenn Sie eine Seite nicht vor der Anfrage eines Nutzers vorrendern können. Vielleicht zeigt Ihre Seite häufig aktualisierte Daten an, und der Seiteninhalt ändert sich bei jeder Anfrage.

In diesem Fall können Sie [**Server-side Rendering**](/docs/basic-features/pages#server-side-rendering) verwenden. Es wird langsamer sein, aber die vorgerenderte Seite wird immer auf dem neuesten Stand sein. Alternativ können Sie das Pre-rendering überspringen und clientseitiges JavaScript verwenden, um häufig aktualisierte Daten zu laden.
