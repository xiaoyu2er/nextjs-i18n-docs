Hier sind einige wesentliche Informationen, die Sie über [API-Routen](/docs/api-routes/introduction) wissen sollten.

### [API-Routen nicht aus `getStaticProps` oder `getStaticPaths` abfragen](#do-not-fetch-an-api-route-from-getstaticprops-or-getstaticpaths)

Sie sollten **nicht** eine API-Route aus [`getStaticProps`](/docs/basic-features/data-fetching#getstaticprops-static-generation) oder [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) abfragen. Schreiben Sie Ihren serverseitigen Code stattdessen direkt in [`getStaticProps`](/docs/basic-features/data-fetching#getstaticprops-static-generation) oder [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) (oder rufen Sie eine Hilfsfunktion auf).

Hier ist der Grund: [`getStaticProps`](/docs/basic-features/data-fetching#getstaticprops-static-generation) und [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) laufen nur serverseitig und niemals clientseitig. Darüber hinaus werden diese Funktionen nicht in das JS-Bundle für den Browser aufgenommen. Das bedeutet, Sie können Code wie direkte Datenbankabfragen schreiben, ohne ihn an Browser zu senden. Lesen Sie die Dokumentation [Serverseitigen Code schreiben](/docs/basic-features/data-fetching/get-static-props#write-server-side-code-directly), um mehr zu erfahren.

### [Ein guter Anwendungsfall: Formulareingaben verarbeiten](#a-good-use-case-handling-form-input)

Ein guter Anwendungsfall für API-Routen ist die Verarbeitung von Formulareingaben. Sie können beispielsweise ein Formular auf Ihrer Seite erstellen und es eine `POST`-Anfrage an Ihre API-Route senden lassen. Dann können Sie Code schreiben, um die Daten direkt in Ihrer Datenbank zu speichern. Der API-Routen-Code wird nicht Teil Ihres Client-Bundles sein, sodass Sie sicher serverseitigen Code schreiben können.

```
export default function handler(req, res) {
  const email = req.body.email;
  // Dann E-Mail in Ihrer Datenbank speichern usw...
}
```

### [Vorschaumodus](#preview-mode)

[Statische Generierung](/docs/basic-features/pages#static-generation-recommended) ist nützlich, wenn Ihre Seiten Daten von einem Headless-CMS abrufen. Sie ist jedoch nicht ideal, wenn Sie einen Entwurf in Ihrem Headless-CMS schreiben und ihn sofort auf Ihrer Seite **vorschauen** möchten. In diesem Fall möchten Sie, dass Next.js diese Seiten zur **Laufzeit** statt zur Build-Zeit rendert und den Entwurfsinhalt statt des veröffentlichten Inhalts abruft. Sie möchten, dass Next.js die Statische Generierung nur für diesen speziellen Fall umgeht.

Next.js hat eine Funktion namens **Vorschaumodus**, um das obige Problem zu lösen, und nutzt dabei [API-Routen](/docs/api-routes/introduction). Um mehr darüber zu erfahren, lesen Sie unsere Dokumentation zum [Vorschaumodus](/docs/advanced-features/preview-mode).

### [Dynamische API-Routen](#dynamic-api-routes)

API-Routen können dynamisch sein, genau wie reguläre Seiten. Werfen Sie einen Blick in unsere Dokumentation zu [Dynamischen API-Routen](/docs/api-routes/dynamic-api-routes), um mehr zu erfahren.
