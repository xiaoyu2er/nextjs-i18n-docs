Next.js blockiert Cross-Origin-Anfragen während der Entwicklung nicht automatisch, wird dies jedoch in einer zukünftigen Hauptversion standardmäßig tun, um unbefugte Anfragen an interne Assets/Endpunkte zu verhindern, die im Entwicklungsmodus verfügbar sind.

Um eine Next.js-Anwendung so zu konfigurieren, dass Anfragen von anderen Ursprüngen als dem Hostnamen, mit dem der Server initialisiert wurde (standardmäßig `localhost`), zugelassen werden, können Sie die Konfigurationsoption `allowedDevOrigins` verwenden.

`allowedDevOrigins` ermöglicht es Ihnen, zusätzliche Ursprünge festzulegen, die im Entwicklungsmodus verwendet werden können. Um beispielsweise `local-origin.dev` anstelle von nur `localhost` zu verwenden, öffnen Sie `next.config.js` und fügen die `allowedDevOrigins`-Konfiguration hinzu:

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```
