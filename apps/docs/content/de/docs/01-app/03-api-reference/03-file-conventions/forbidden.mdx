Die **forbidden**-Datei wird verwendet, um die Benutzeroberfläche anzuzeigen, wenn die [`forbidden`](/docs/app/api-reference/functions/forbidden)-Funktion während der Authentifizierung aufgerufen wird. Neben der Möglichkeit, die Benutzeroberfläche anzupassen, gibt Next.js auch einen `403`-Statuscode zurück.

```tsx filename="app/forbidden.tsx" switcher
import Link from 'next/link'

export default function Forbidden() {
  return (
    <div>
      <h2>Forbidden</h2>
      <p>You are not authorized to access this resource.</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

```jsx filename="app/forbidden.jsx" switcher
import Link from 'next/link'

export default function Forbidden() {
  return (
    <div>
      <h2>Forbidden</h2>
      <p>You are not authorized to access this resource.</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

## Referenz

### Props

`forbidden.js`-Komponenten akzeptieren keine Props.
