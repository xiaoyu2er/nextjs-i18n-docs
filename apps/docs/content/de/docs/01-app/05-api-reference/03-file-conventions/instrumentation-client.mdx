Die Datei `instrumentation-client.js|ts` ermöglicht es Ihnen, Monitoring- und Analysecode hinzuzufügen, der ausgeführt wird, bevor der Frontend-Code Ihrer Anwendung startet. Dies ist nützlich für die Einrichtung von Performance-Tracking, Fehlerüberwachung oder anderen clientseitigen Observability-Tools.

Um sie zu verwenden, platzieren Sie die Datei im **Stammverzeichnis** Ihrer Anwendung oder in einem `src`-Ordner.

## Verwendung

Im Gegensatz zur [serverseitigen Instrumentierung](/docs/app/guides/instrumentation) müssen Sie keine spezifischen Funktionen exportieren. Sie können Ihren Monitoring-Code direkt in der Datei schreiben:

```ts filename="instrumentation-client.ts" switcher
// Performance-Monitoring einrichten
performance.mark('app-init')

// Analytics initialisieren
console.log('Analytics initialized')

// Fehlertracking einrichten
window.addEventListener('error', (event) => {
  // An Ihren Fehlertracking-Service senden
  reportError(event.error)
})
```
