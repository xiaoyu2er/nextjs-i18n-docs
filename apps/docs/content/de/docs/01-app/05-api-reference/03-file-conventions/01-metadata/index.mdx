Dieser Abschnitt der Dokumentation behandelt **Konventionen für Metadata-Dateien**. Dateibasierte Metadaten können durch das Hinzufügen spezieller Metadata-Dateien zu Routensegmenten definiert werden.

Jede Dateikonvention kann entweder als statische Datei (z.B. `opengraph-image.jpg`) oder als dynamische Variante, die Code zur Generierung der Datei verwendet (z.B. `opengraph-image.js`), definiert werden.

Sobald eine Datei definiert ist, wird Next.js die Datei automatisch bereitstellen (in der Produktionsumgebung mit Hashes für das Caching) und die relevanten Kopf-Elemente mit den korrekten Metadaten aktualisieren, wie z.B. die URL der Ressource, den Dateityp und die Bildgröße.

> **Wissenswert**:
>
> - Spezielle Route Handler wie [`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap), [`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image) und [`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons) sowie andere [Metadata-Dateien](/docs/app/api-reference/file-conventions/metadata) werden standardmäßig gecached.
> - Bei Verwendung zusammen mit [`middleware.ts`](/docs/app/api-reference/file-conventions/middleware) sollte der [Matcher konfiguriert werden](/docs/app/building-your-application/routing/middleware#matcher), um die Metadata-Dateien auszuschließen.
