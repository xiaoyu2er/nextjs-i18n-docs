> **Gut zu wissen**: Seit Next.js 13.5.1 ist der Strict Mode standardmäßig auf `true` gesetzt, wenn der `app`-Router verwendet wird. Daher ist die obige Konfiguration nur für `pages` notwendig. Sie können den Strict Mode dennoch deaktivieren, indem Sie `reactStrictMode: false` setzen.

> **Empfehlung**: Wir empfehlen dringend, den Strict Mode in Ihrer Next.js-Anwendung zu aktivieren, um Ihre Anwendung besser auf die Zukunft von React vorzubereiten.

Der [Strict Mode](https://react.dev/reference/react/StrictMode) von React ist ein Feature, das nur im Entwicklungsmodus verfügbar ist und potenzielle Probleme in einer Anwendung hervorhebt. Es hilft dabei, unsichere Lebenszyklen, veraltete API-Verwendung und eine Reihe anderer Features zu identifizieren.

Die Next.js-Laufzeitumgebung ist Strict Mode-kompatibel. Um den Strict Mode zu aktivieren, konfigurieren Sie die folgende Option in Ihrer `next.config.js`:

```js filename="next.config.js"
module.exports = {
  reactStrictMode: true,
}
```

Falls Sie oder Ihr Team noch nicht bereit sind, den Strict Mode in Ihrer gesamten Anwendung zu verwenden, ist das in Ordnung! Sie können schrittweise eine Migration auf Seitenbasis durchführen, indem Sie `<React.StrictMode>` verwenden.
