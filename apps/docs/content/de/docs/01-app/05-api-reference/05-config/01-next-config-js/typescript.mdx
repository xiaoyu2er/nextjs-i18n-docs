Next.js bricht Ihren **Produktionsbuild** (`next build`) ab, wenn TypeScript-Fehler in Ihrem Projekt vorhanden sind.

Falls Sie möchten, dass Next.js gefährlicherweise Produktionscode erzeugt, selbst wenn Ihre Anwendung Fehler enthält, können Sie den integrierten Typüberprüfungsschritt deaktivieren.

Wenn dies deaktiviert ist, stellen Sie sicher, dass Sie Typüberprüfungen als Teil Ihres Build- oder Bereitstellungsprozesses durchführen, da dies sonst sehr riskant sein kann.

Öffnen Sie `next.config.js` und aktivieren Sie die Option `ignoreBuildErrors` in der `typescript`-Konfiguration:

```js filename="next.config.js"
module.exports = {
  typescript: {
    // !! WARNUNG !!
    // Erlaubt gefährlicherweise, dass Produktionsbuilds erfolgreich abgeschlossen werden,
    // selbst wenn Ihr Projekt Typfehler enthält.
    // !! WARNUNG !!
    ignoreBuildErrors: true,
  },
}
```
