Wenn ESLint in Ihrem Projekt erkannt wird, bricht Next.js Ihren **Produktions-Build** (`next build`) ab, wenn Fehler vorhanden sind.

Falls Sie möchten, dass Next.js auch bei ESLint-Fehlern Produktionscode erzeugt, können Sie den integrierten Linting-Schritt komplett deaktivieren. Dies wird nicht empfohlen, es sei denn, Sie haben ESLint bereits so konfiguriert, dass es in einem separaten Teil Ihres Workflows läuft (z.B. in CI oder einem Pre-Commit-Hook).

Öffnen Sie `next.config.js` und aktivieren Sie die Option `ignoreDuringBuilds` in der `eslint`-Konfiguration:

```js filename="next.config.js"
module.exports = {
  eslint: {
    // Warnung: Dies ermöglicht die erfolgreiche Fertigstellung von Produktions-Builds,
    // selbst wenn Ihr Projekt ESLint-Fehler aufweist.
    ignoreDuringBuilds: true,
  },
}
```
