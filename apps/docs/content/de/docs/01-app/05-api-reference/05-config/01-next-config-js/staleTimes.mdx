`staleTimes` ist ein experimentelles Feature, das das Caching von Seiten-Segmenten im [Client-seitigen Router-Cache](/docs/app/deep-dive/caching#client-side-router-cache) ermöglicht.

Sie können dieses experimentelle Feature aktivieren und individuelle Revalidierungszeiten festlegen, indem Sie das experimentelle `staleTimes`-Flag setzen:

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

Die Eigenschaften `static` und `dynamic` entsprechen dem Zeitraum (in Sekunden) basierend auf verschiedenen Arten von [Link-Prefetching](/docs/app/api-reference/components/link#prefetch).

- Die `dynamic`-Eigenschaft wird verwendet, wenn die Seite weder statisch generiert noch vollständig geprefetched wurde (z.B. mit `prefetch={true}`).
  - Standardwert: 0 Sekunden (nicht gecached)
- Die `static`-Eigenschaft wird für statisch generierte Seiten verwendet oder wenn die `prefetch`-Prop von `Link` auf `true` gesetzt ist oder wenn [`router.prefetch`](/docs/app/deep-dive/caching#routerprefetch) aufgerufen wird.
  - Standardwert: 5 Minuten

> **Wissenswert:**
>
> - [Loading-Boundaries](/docs/app/api-reference/file-conventions/loading) gelten für den in dieser Konfiguration definierten `static`-Zeitraum als wiederverwendbar.
> - Dies beeinflusst nicht das [Partielle Rendering](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering), **was bedeutet, dass gemeinsame Layouts nicht bei jeder Navigation automatisch neu abgerufen werden, sondern nur das sich ändernde Seiten-Segment.**
> - Dies ändert nicht das [Vorwärts/Rückwärts-Caching](/docs/app/deep-dive/caching#client-side-router-cache)-Verhalten, um Layout-Shifts zu verhindern und die Browser-Scroll-Position beizubehalten.

Weitere Informationen zum Client-Router-Cache finden Sie [hier](/docs/app/deep-dive/caching#client-side-router-cache).
