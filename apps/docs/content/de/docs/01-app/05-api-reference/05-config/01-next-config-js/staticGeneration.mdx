Die `staticGeneration*`-Optionen ermöglichen es Ihnen, den Prozess der statischen Generierung (Static Generation) für fortgeschrittene Anwendungsfälle zu konfigurieren.

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    staticGenerationRetryCount: 1,
    staticGenerationMaxConcurrency: 8,
    staticGenerationMinPagesPerWorker: 25,
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
const nextConfig = {
  experimental: {
    staticGenerationRetryCount: 1,
    staticGenerationMaxConcurrency: 8,
    staticGenerationMinPagesPerWorker: 25,
  },
}

export default nextConfig
```

## Konfigurationsoptionen

Folgende Optionen sind verfügbar:

- `staticGenerationRetryCount`: Die Anzahl der Versuche, eine fehlgeschlagene Seitengenerierung erneut durchzuführen, bevor der Build als fehlgeschlagen markiert wird.
- `staticGenerationMaxConcurrency`: Die maximale Anzahl von Seiten, die pro Worker verarbeitet werden sollen.
- `staticGenerationMinPagesPerWorker`: Die minimale Anzahl von Seiten, die verarbeitet werden sollen, bevor ein neuer Worker gestartet wird.
