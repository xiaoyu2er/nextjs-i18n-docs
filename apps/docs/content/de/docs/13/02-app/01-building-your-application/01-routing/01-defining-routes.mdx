Diese Seite führt Sie durch das Definieren und Organisieren von Routen in Ihrer Next.js-Anwendung.

## Routen erstellen

Next.js verwendet einen dateisystembasierten Router, bei dem **Ordner** zur Definition von Routen verwendet werden.

Jeder Ordner repräsentiert ein [**Route-Segment**](/docs/app/building-your-application/routing#route-segments), das einem **URL-Segment** zugeordnet ist. Um eine [verschachtelte Route](/docs/app/building-your-application/routing#nested-routes) zu erstellen, können Sie Ordner ineinander verschachteln.

Eine spezielle [`page.js`-Datei](/docs/app/building-your-application/routing/pages-and-layouts#pages) wird verwendet, um Routensegmente öffentlich zugänglich zu machen.

In diesem Beispiel ist der URL-Pfad `/dashboard/analytics` _nicht_ öffentlich zugänglich, da er keine entsprechende `page.js`-Datei hat. Dieser Ordner könnte verwendet werden, um Komponenten, Stylesheets, Bilder oder andere zusammengehörige Dateien zu speichern.

> **Gut zu wissen**: Die Dateiendungen `.js`, `.jsx` oder `.tsx` können für spezielle Dateien verwendet werden.

## UI erstellen

[Spezielle Dateikonventionen](/docs/app/building-your-application/routing#file-conventions) werden verwendet, um UI für jedes Routensegment zu erstellen. Die gebräuchlichsten sind [Seiten](/docs/app/building-your-application/routing/pages-and-layouts#pages) zur Anzeige von UI, die für eine Route einzigartig ist, und [Layouts](/docs/app/building-your-application/routing/pages-and-layouts#layouts) zur Anzeige von UI, die über mehrere Routen hinweg geteilt wird.

Um beispielsweise Ihre erste Seite zu erstellen, fügen Sie eine `page.js`-Datei in das `app`-Verzeichnis ein und exportieren Sie eine React-Komponente:

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```
