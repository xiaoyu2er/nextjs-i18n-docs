## Wie Server Actions funktionieren

Mit Server Actions m√ºssen Sie keine manuellen API-Endpunkte erstellen. Stattdessen definieren Sie asynchrone Serverfunktionen, die direkt aus Ihren Komponenten aufgerufen werden k√∂nnen.

> **üé• Ansehen:** Erfahren Sie mehr √ºber Formulare und Mutationen mit dem App-Router ‚Üí [YouTube (10 Minuten)](https://youtu.be/dDpZfOQBMaU?si=cJZHlUu_jFhCzHUg).

Server Actions k√∂nnen in Server-Komponenten definiert oder aus Client-Komponenten aufgerufen werden. Die Definition der Aktion in einer Server-Komponente erm√∂glicht es dem Formular, ohne JavaScript zu funktionieren, was eine progressive Verbesserung bietet.

Aktivieren Sie Server Actions in Ihrer `next.config.js`-Datei:

```js filename="next.config.js"
module.exports = {
  experimental: {
    serverActions: true,
  },
}
```

> **Gut zu wissen:**
>
> - Formulare, die Server Actions aus Server-Komponenten aufrufen, k√∂nnen ohne JavaScript funktionieren.
> - Formulare, die Server Actions aus Client-Komponenten aufrufen, werden bei noch nicht geladenem JavaScript in eine Warteschlange gestellt, wobei die Client-Hydration priorisiert wird.
> - Server Actions erben die [Laufzeitumgebung](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes) von der Seite oder dem Layout, in dem sie verwendet werden.
> - Server Actions funktionieren mit vollst√§ndig statischen Routen (einschlie√ülich der Revalidierung von Daten mit ISR).

## Revalidierung von gecachten Daten

Server Actions sind tief in die Next.js [Caching- und Revalidierungsarchitektur](/docs/app/building-your-application/caching) integriert. Wenn ein Formular √ºbermittelt wird, kann die Server Action gecachte Daten aktualisieren und alle Cache-Schl√ºssel revalidieren, die sich √§ndern sollten.

Im Gegensatz zu traditionellen Anwendungen, die auf ein Formular pro Route beschr√§nkt sind, erm√∂glichen Server Actions mehrere Aktionen pro Route. Dar√ºber hinaus muss der Browser bei einer Formular√ºbermittlung nicht aktualisiert werden. In einer einzigen Netzwerkrunde kann Next.js sowohl die aktualisierte Benutzeroberfl√§che als auch die aktualisierten Daten zur√ºckgeben.

Sehen Sie sich die folgenden Beispiele f√ºr die [Revalidierung von Daten aus Server Actions](#revalidating-data) an.

### Optimistische Updates

Verwenden Sie `useOptimistic`, um die Benutzeroberfl√§che optimistisch zu aktualisieren, bevor die Server Action abgeschlossen ist, anstatt auf die Antwort zu warten:

```tsx filename="app/page.tsx" switcher

  message: string
}

export function Thread({ messages }: { messages: Message[] }) {
  const [optimisticMessages, addOptimisticMessage] = useOptimistic<Message[]>(
    messages,
    (state: Message[], newMessage: string) => [
      ...state,
      { message: newMessage },
    ]
  )

  return (
    <div>
      {optimisticMessages.map((m, k) => (
        <div key={k}>{m.message}</div>
      ))}
      <form
        action={async (formData: FormData) => {
          const message = formData.get('message')
          addOptimisticMessage(message)
          await send(message)
        }}
      >
        <input type="text" name="message" />
        <button type="submit">Send</button>
      </form>
    </div>
  )
}
```
