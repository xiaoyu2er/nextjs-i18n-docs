Dieser Abschnitt der Dokumentation behandelt **Metadata-Dateikonventionen**. Dateibasierte Metadaten können durch Hinzufügen spezieller Metadata-Dateien zu Routensegmenten definiert werden.

Jede Dateikonvention kann entweder als statische Datei (z.B. `opengraph-image.jpg`) oder als dynamische Variante, die Code zur Generierung der Datei verwendet (z.B. `opengraph-image.js`), definiert werden.

Sobald eine Datei definiert ist, serviert Next.js die Datei automatisch (mit Hashes in der Produktionsumgebung für Caching-Zwecke) und aktualisiert die relevanten Head-Elemente mit den korrekten Metadaten, wie z.B. der URL des Assets, dem Dateityp und der Bildgröße.
