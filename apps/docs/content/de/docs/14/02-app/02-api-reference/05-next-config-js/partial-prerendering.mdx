> **Warnung**: Partielles Prerendering ist ein experimentelles Feature und aktuell **nicht für Produktionsumgebungen geeignet**.

Partielles Prerendering ist ein experimentelles Feature, das statische Teile einer Route vorrendern und aus dem Cache bereitstellen kann, während dynamische Abschnitte in einem einzigen HTTP-Request nachgeladen werden.

Partielles Prerendering ist in `next@canary` verfügbar:

```bash filename="Terminal"
npm install next@canary
```

Sie können partielles Prerendering aktivieren, indem Sie das experimentelle `ppr`-Flag setzen:

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    ppr: true,
  },
}

module.exports = nextConfig
```

> **Gut zu wissen:**
>
> - Partielles Prerendering gilt noch nicht für clientseitige Navigationen. Wir arbeiten aktiv daran.
> - Partielles Prerendering ist nur für die [Node.js Runtime](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes) konzipiert. Die Verwendung einer Teilmenge der Node.js Runtime ist nicht nötig, wenn die statische Hülle sofort bereitgestellt werden kann.

Erfahren Sie mehr über partielles Prerendering im [Next.js Learn-Kurs](/learn/dashboard-app/partial-prerendering).
