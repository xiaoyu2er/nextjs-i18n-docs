<AppOnly>

Wenn Sie Ihren Code bearbeiten und Next.js die Anwendung kompiliert, erscheint ein Kompilierungsindikator in der unteren rechten Ecke der Seite.

> **Gut zu wissen**: Dieser Indikator ist nur im Entwicklungsmodus vorhanden und erscheint nicht beim Bauen und Ausführen der App im Produktionsmodus.

In einigen Fällen kann dieser Indikator auf Ihrer Seite falsch positioniert sein, zum Beispiel wenn er mit einem Chat-Launcher kollidiert. Um seine Position zu ändern, öffnen Sie `next.config.js` und setzen Sie `buildActivityPosition` im `devIndicators`-Objekt auf `bottom-right` (Standard), `bottom-left`, `top-right` oder `top-left`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivityPosition: 'bottom-right',
  },
}
```

In manchen Fällen ist dieser Indikator möglicherweise nicht nützlich für Sie. Um ihn zu entfernen, öffnen Sie `next.config.js` und deaktivieren Sie die `buildActivity`-Konfiguration im `devIndicators`-Objekt:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivity: false,
  },
}
```

</AppOnly>

<PagesOnly>

> **Gut zu wissen**: Dieser Indikator wurde in Next.js Version 10.0.1 entfernt. Wir empfehlen ein Upgrade auf die neueste Version von Next.js.

Wenn eine Seite für die [Automatische statische Optimierung](/docs/pages/building-your-application/rendering/automatic-static-optimization) qualifiziert ist, zeigen wir einen Indikator an.

Dies ist hilfreich, da die automatische statische Optimierung sehr vorteilhaft sein kann und es nützlich ist, sofort in der Entwicklung zu wissen, ob die Seite qualifiziert ist.

In einigen Fällen ist dieser Indikator möglicherweise nicht nützlich, zum Beispiel bei der Arbeit an Electron-Anwendungen. Um ihn zu entfernen, öffnen Sie `next.config.js` und deaktivieren Sie die `autoPrerender`-Konfiguration in `devIndicators`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    autoPrerender: false,
  },
}
```

</PagesOnly>
