Una URL **canónica** es la URL de la página que los motores de búsqueda consideran más representativa dentro de un conjunto de páginas duplicadas en tu sitio.

Aunque puedes comunicar directamente las URLs canónicas a los motores de búsqueda, estos también pueden decidir agrupar varias URLs sin que tú lo notifiques. Esto puede ocurrir automáticamente si Google encuentra una URL bajo diferentes rutas.

Si bien Google hace un buen trabajo detectando esto, sus sistemas operan a gran escala y no cubren todos los casos extremos. Las etiquetas canónicas son un aspecto importante para garantizar un buen rendimiento de tu sitio web.

Si Google encuentra varias URLs con el mismo contenido, podría decidir degradarlas en los resultados de búsqueda porque pueden considerarse duplicadas.

Esto también ocurre entre dominios: si tienes dos sitios web diferentes y publicas el mismo contenido en cada uno, los motores de búsqueda pueden elegir uno para posicionar o degradar ambos directamente.

Aquí es donde las etiquetas canónicas son extremadamente útiles. Le indican a Google qué URLs son la fuente original de verdad y cuáles son duplicadas. Muchas páginas duplicadas en el mismo dominio o en dominios diferentes pueden llevar a malas clasificaciones o incluso penalizaciones.

Imaginemos que nuestra tienda de comercio electrónico permite acceder a los productos mediante [example.com/products/phone](https://example.com/products/phone) y [example.com/phone](http://example.com/phone).

Ambas son URLs válidas y funcionales, pero usamos la etiqueta canónica para evitar la detección de contenido duplicado que nos pertenece. Si decidimos que `https://example.com/products/phone` debe ser considerada para el posicionamiento, crearíamos una etiqueta canónica:

```
<link rel="canonical" href="https://example.com/products/phone" />
```

Las etiquetas canónicas son fundamentales para el rendimiento SEO, no solo porque puedes crear diferentes URLs, sino también porque los usuarios o herramientas de marketing pueden generarlas.

Imagina que estás ejecutando campañas de marketing en Google, entonces Google decide añadir algunos [parámetros UTM](https://ga-dev-tools.appspot.com/campaign-url-builder/). Es posible que esta nueva URL única sea indexada por Googlebot, por lo que debes asegurarte de seguir mostrando tus etiquetas canónicas para unificar las páginas duplicadas.

#### [Ejemplo](#ejemplo)

```
import Head from 'next/head';
 
function IndexPage() {
  return (
    <div>
      <Head>
        <title>Ejemplo de Etiqueta Canónica</title>
        <link
          rel="canonical"
          href="https://example.com/blog/original-post"
          key="canonical"
        />
      </Head>
      <p>Esta publicación existe en dos URLs.</p>
    </div>
  );
}
 
export default IndexPage;
```
