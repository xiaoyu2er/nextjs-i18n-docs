Otra excelente forma gratuita y de código abierto para medir el rendimiento es utilizar el conjunto de datos del [Informe de Experiencia de Usuario de Chrome](https://developers.google.com/web/tools/chrome-user-experience-report).

El [Informe de Experiencia de Usuario de Chrome](https://developers.google.com/web/tools/chrome-user-experience-report) proporciona métricas de experiencia de usuario sobre cómo los usuarios reales de Chrome experimentan los destinos populares en la web.

Este conjunto de datos está disponible públicamente [en BigQuery](https://console.cloud.google.com/bigquery?project=chrome-ux-report) y también puede consumirse en [Google Data Studio](https://datastudio.google.com/overview) completamente gratis.

Afortunadamente, existen [paneles de código abierto disponibles](https://g.co/chromeuxdash) que puede utilizar como plantilla para rastrear el rendimiento de su sitio web.

El único inconveniente de este conjunto de datos es que, para que su sitio web se incluya en el informe CrUX, necesitará tener un número significativo de visitas; de lo contrario, no se incluirá en el informe debido a la falta de datos. Por lo tanto, puede que no sea la mejor opción para sitios web en desarrollo o recién creados.

Además, los datos se actualizan mensualmente. Generalmente alrededor de 15 días después de que finaliza el mes, por lo que verá los datos con cierto retraso y eso podría no ser lo más práctico si planea trabajar en mejorar sus puntuaciones de Core Web Vitals.
