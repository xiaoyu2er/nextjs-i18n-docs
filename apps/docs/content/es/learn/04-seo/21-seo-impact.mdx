El principal objetivo del motor de búsqueda de Google es proporcionar a los usuarios los mejores resultados posibles, teniendo en cuenta la localización y los errores ortográficos. En todos los casos, a Google le importa ofrecer a los usuarios un resultado rápido y fluido con una gran experiencia de sitio web. La [velocidad de las páginas web](https://developers.google.com/web/updates/2018/07/search-ads-speed) en dispositivos móviles ha sido un factor de clasificación desde 2018. Sin embargo, hasta ahora no estaba claro qué métricas de rendimiento específicas utiliza el algoritmo de Búsqueda de Google como parte de la clasificación.

Esto cambió en junio de 2021, cuando Google proporcionó [un conjunto de métricas y rangos específicos](https://developers.google.com/search/blog/2021/04/more-details-page-experience) para analizar y optimizar el rendimiento.

![Detalles de las señales de experiencia de búsqueda para la actualización del algoritmo de Google de junio de 2021.](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/pages-router/seo/page-experience.png)

### [Ponderaciones de Lighthouse (v6) para las Web Vitals](#lighthouse-v6-weights-for-vitals)

Las tres métricas no necesariamente tienen el mismo valor. En [Lighthouse](https://developers.google.com/web/tools/lighthouse), se asignan diferentes ponderaciones a cada una de las Core Web Vitals:

*   **Largest Contentful Paint**: 25%
*   **Total Blocking Time**: 25%
*   **First Contentful Paint**: 15%
*   **Speed Index**: 15%
*   **Time to Interactive**: 15%
*   **Cumulative Layout Shift**: 5%

Esto es similar al [First Input Delay](/learn/seo/fid).

**Nota:** El impacto en el ranking de Google será el mismo para todas las páginas que estén en el rango bueno para todas las Core Web Vitals, independientemente de sus puntuaciones individuales.

### [Impacto en la Experiencia de Usuario (UX)](#ux-impact)

La mayoría de las conversaciones sobre Core Web Vitals se centran principalmente en SEO.

Si bien es cierto que Core Web Vitals es una iniciativa diseñada para medir y promover la mejora de la experiencia de página y el ranking de búsqueda, son los usuarios quienes finalmente se benefician de estos cambios.

Core Web Vitals ayuda a esforzarse por la mejor experiencia de página. Según un [estudio realizado por Amazon en 2012](https://www.fastcompany.com/1825005/how-one-second-could-cost-amazon-16-billion-sales), un segundo adicional de tiempo de carga podría costarle a la empresa 1.600 millones de dólares. Estudios como este muestran la importancia de una gran experiencia de página y un sitio web rápido, ambos objetivos que Core Web Vitals ayuda a lograr.
