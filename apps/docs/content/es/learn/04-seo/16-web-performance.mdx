[Web Vitals](https://web.dev/vitals/) es una iniciativa creada por Google para proporcionar orientación unificada y métricas que midan la experiencia del usuario final en la web.

[Core Web Vitals](https://developers.google.com/search/blog/2020/11/timing-for-page-experience) es un subconjunto de Web Vitals, y actualmente consta de tres métricas que miden la carga, interactividad y estabilidad visual. Estas métricas son [Largest Contentful Paint (LCP)](/learn/seo/lcp), [First Input Delay (FID)](/learn/seo/fid) y [Cumulative Layout Shift (CLS)](/learn/seo/cls).

Lograr una buena puntuación en estas tres métricas creará una experiencia de sitio web más fluida para tus usuarios.

Los sitios web con malas puntuaciones en cada una de las métricas de Core Web Vitals afectarán su posicionamiento en los motores de búsqueda, ya que Google comienza a usar [Core Web Vitals como factor de clasificación](https://developers.google.com/search/blog/2020/11/timing-for-page-experience) en su algoritmo de búsqueda. Un rendimiento deficiente puede tener impacto en tu tráfico web y negocio.

En esta lección, aprenderás:

* Un breve contexto sobre Core Web Vitals.
* Implicaciones de Core Web Vitals en SEO y UX, y su impacto en tu sitio web.
* Por qué deberías considerar Core Web Vitals en tu proceso de desarrollo y cómo medirlos.
* Cómo mejorar tus Core Web Vitals con Next.js y monitorear cambios.
