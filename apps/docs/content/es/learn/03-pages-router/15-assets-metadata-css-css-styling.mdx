Hablemos ahora sobre **estilos CSS**.

Como puedes ver, nuestra página de inicio ([http://localhost:3000](http://localhost:3000)) ya tiene algunos estilos. Si observas la estructura de archivos, verás una carpeta llamada `styles` con dos archivos CSS: una hoja de estilos global (`global.css`) y un módulo CSS (`Home.module.css`).

Si tu proyecto no tiene esos archivos, puedes descargar el código inicial aquí:

```
npx create-next-app nextjs-blog --use-npm --example "https://github.com/vercel/next-learn/tree/main/basics/assets-metadata-css-starter"
```

### [Módulos CSS](#css-modules)

Los [módulos CSS](/docs/basic-features/built-in-css-support) permiten aplicar CSS de forma local a nivel de componente generando automáticamente nombres de clase únicos. Esto te permite usar el mismo nombre de clase CSS en diferentes archivos sin preocuparte por colisiones de nombres.

Además de los módulos CSS, puedes estilizar tu aplicación Next.js de varias formas, incluyendo:

* Sass, que permite importar archivos `.css` y `.scss`.
* Bibliotecas PostCSS como [Tailwind CSS](https://github.com/vercel/next.js/tree/canary/examples/with-tailwindcss).
* Bibliotecas CSS-in-JS como [styled-jsx](https://github.com/vercel/styled-jsx), [styled-components](https://github.com/vercel/next.js/tree/canary/examples/with-styled-components) y [emotion](https://github.com/vercel/next.js/tree/canary/examples/with-emotion)

En esta lección, hablaremos sobre cómo usar [Módulos CSS](/docs/basic-features/built-in-css-support#adding-component-level-css) y [Sass](/docs/basic-features/built-in-css-support#sass-support) en Next.js. ¡Comencemos!
