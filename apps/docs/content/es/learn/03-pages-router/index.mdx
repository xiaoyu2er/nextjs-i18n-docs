Para construir una aplicación web completa con React desde cero, hay muchos detalles importantes que debes considerar:

*   El código debe ser empaquetado usando un empaquetador como webpack y transformado usando un compilador como Babel.
*   Necesitas realizar optimizaciones para producción como la división de código (code splitting).
*   Es posible que desees pre-renderizar estáticamente algunas páginas para mejorar el rendimiento y SEO. También podrías querer usar renderizado del lado del servidor (SSR) o renderizado del lado del cliente (CSR).
*   Podrías necesitar escribir código del lado del servidor para conectar tu aplicación React con tu almacén de datos.

Un _framework_ puede resolver estos problemas. Pero dicho framework debe tener el nivel adecuado de abstracción, de lo contrario no será muy útil. También necesita ofrecer una gran "Experiencia de Desarrollo", asegurando que tú y tu equipo tengan una experiencia excepcional al escribir código.

### [Next.js: El Framework de React](#nextjs-the-react-framework)

Presentamos **Next.js**, el Framework de React. Next.js proporciona una solución a todos los problemas mencionados anteriormente. Pero lo más importante, te coloca a ti y a tu equipo en el ["pozo del éxito"](https://blog.codinghorror.com/falling-into-the-pit-of-success/) al construir aplicaciones React.

Next.js tiene como objetivo ofrecer la mejor experiencia de desarrollador e incluye muchas características integradas, como:

*   Un sistema de [enrutamiento basado en páginas](/docs/basic-features/pages) intuitivo (con soporte para [rutas dinámicas](/docs/routing/dynamic-routes))
*   [Pre-renderizado](/docs/basic-features/pages#pre-rendering), tanto [generación estática](/docs/basic-features/pages#static-generation-recommended) (SSG) como [renderizado del lado del servidor](/docs/basic-features/pages#server-side-rendering) (SSR) soportados por página
*   División de código automática para cargas de página más rápidas
*   [Enrutamiento del lado del cliente](/docs/routing/introduction#linking-between-pages) con prefetching optimizado
*   [Soporte integrado para CSS](/docs/basic-features/built-in-css-support) y [Sass](/docs/basic-features/built-in-css-support#sass-support), y compatibilidad con cualquier biblioteca [CSS-in-JS](/docs/basic-features/built-in-css-support#css-in-js)
*   Entorno de desarrollo con soporte para [Actualización Rápida (Fast Refresh)](/docs/basic-features/fast-refresh)
*   [Rutas API](/docs/api-routes/introduction) para construir endpoints con Funciones Serverless
*   Totalmente extensible

Next.js se utiliza en decenas de miles de sitios web y aplicaciones en producción, incluyendo muchas de las marcas más grandes del mundo.
