Las [Rutas API](/docs/api-routes/introduction) permiten crear un endpoint de API dentro de una aplicación Next.js. Puedes hacerlo creando una **función** dentro del directorio `pages/api` con el siguiente formato:

```
// req = mensaje HTTP entrante, res = respuesta del servidor HTTP
export default function handler(req, res) {
  // ...
}
```

> Obtén más información sobre el manejador de solicitudes anterior en la [documentación de Rutas API](/docs/api-routes/introduction).

Estas rutas pueden implementarse como Funciones Serverless (también conocidas como Lambdas).

### [Creación de un endpoint API simple](#creating-a-simple-api-endpoint)

Probemos esto. Crea un archivo llamado `hello.js` en `pages/api` con el siguiente código:

```
export default function handler(req, res) {
  res.status(200).json({ text: 'Hello' });
}
```

Intenta acceder a él en [http://localhost:3000/api/hello](http://localhost:3000/api/hello). Deberías ver `{"text":"Hello"}`. Ten en cuenta que:

*   `req` es una instancia de [http.IncomingMessage](https://nodejs.org/api/http.html#http_class_http_incomingmessage), más algunos [middlewares](/docs/api-routes/api-middlewares) preconstruidos.
*   `res` es una instancia de [http.ServerResponse](https://nodejs.org/api/http.html#http_class_http_serverresponse), más algunas [funciones auxiliares](/docs/api-routes/response-helpers).

¡Eso es todo! Antes de concluir esta lección, hablemos sobre algunos consejos para usar [Rutas API](/docs/api-routes/introduction) en la siguiente página.
