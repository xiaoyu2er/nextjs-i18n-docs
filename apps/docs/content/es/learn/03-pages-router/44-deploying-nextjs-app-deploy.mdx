La forma más sencilla de desplegar Next.js en producción es utilizando la plataforma **[Vercel](https://vercel.com)**, desarrollada por los creadores de Next.js.

Vercel es una plataforma sin servidor (serverless) para aplicaciones estáticas e híbridas diseñada para integrarse con tu contenido headless, comercio electrónico o base de datos. Facilitamos a los equipos de frontend el desarrollo, vista previa e implementación de experiencias de usuario excepcionales, donde el rendimiento es el estándar. Puedes comenzar a usarlo de forma gratuita — no se requiere tarjeta de crédito.

### [Crear una cuenta en Vercel](#create-a-vercel-account)

Primero, dirígete a [Vercel](https://vercel.com/signup) para crear una cuenta. Selecciona **Continuar con GitHub** y completa el proceso de registro.

### [Importar tu repositorio `nextjs-blog`](#import-your-nextjs-blog-repository)

Una vez registrado, **importa** tu repositorio `nextjs-blog` en Vercel. Puedes hacerlo desde aquí: [Vercel](https://vercel.com/import).

*   Necesitarás **Instalar Vercel para GitHub**. Puedes otorgarle acceso a **Todos los repositorios**.
*   Una vez instalado Vercel, importa `nextjs-blog`.

Puedes usar los _valores predeterminados_ para las siguientes configuraciones — no es necesario cambiar nada. Vercel detecta automáticamente que tienes una aplicación Next.js y elige las configuraciones óptimas de compilación.

*   Nombre del proyecto
*   Directorio raíz
*   Comando de compilación
*   Directorio de salida
*   Comando de desarrollo

Cuando despliegues, tu aplicación Next.js comenzará a compilarse. Debería terminar en menos de un minuto.

> **Ayuda disponible**: Si tu despliegue falla, siempre puedes obtener ayuda en [GitHub Discussions](https://github.com/vercel/next.js/discussions). Para aprender más sobre despliegue, consulta [nuestra documentación](/docs/deployment).

Cuando termine, obtendrás **URLs de despliegue**. Haz clic en una de las URLs y deberías ver la página de inicio de Next.js en vivo.

¡Felicidades! Acabas de desplegar tu aplicación Next.js en producción. En la siguiente página, profundizaremos en los detalles de Vercel y el flujo de trabajo recomendado.
