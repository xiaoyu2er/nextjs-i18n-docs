Next.js puede ser desplegado en cualquier proveedor de alojamiento que soporte Node.js.

Si has seguido las instrucciones hasta ahora, tu archivo `package.json` debería tener los siguientes scripts `build` y `start`:

```
{
  "scripts": {
    "dev": "next",
    "build": "next build",
    "start": "next start"
  }
}
```

En tu propio proveedor de alojamiento, ejecuta el script `build` una vez, el cual construye la aplicación de producción en la carpeta `.next`.

```
npm run build
```

Después de construir, el script `start` inicia un servidor Node.js que soporta páginas híbridas, sirviendo tanto páginas generadas estáticamente como renderizadas en el servidor (SSR), además de Rutas API.

```
npm run start
```

> **Consejo**: Puedes personalizar el script `start` en `package.json` para aceptar un parámetro `PORT` actualizándolo como: `"start": "next start -p $PORT"`.

¡Eso es todo! Si tienes preguntas sobre cómo desplegar Next.js, puedes consultar a nuestra comunidad en [GitHub Discussions](https://github.com/vercel/next.js/discussions).
