Next.js ofrece dos formas de pre-renderizado: [**Generación Estática**](/docs/basic-features/pages#static-generation-recommended) y [**Renderizado del Lado del Servidor (SSR)**](/docs/basic-features/pages#server-side-rendering). La diferencia radica en **cuándo** se genera el HTML para una página.

*   [**Generación Estática**](/docs/basic-features/pages#static-generation-recommended) es el método de pre-renderizado que genera el HTML en el **momento de la construcción**. El HTML pre-renderizado se _reutiliza_ en cada solicitud.
*   [**Renderizado del Lado del Servidor (SSR)**](/docs/basic-features/pages#server-side-rendering) es el método de pre-renderizado que genera el HTML en **cada solicitud**.

![Generación Estática](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/pages-router/data-fetching/static-generation.png)

![Renderizado del Lado del Servidor](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/pages-router/data-fetching/server-side-rendering.png)

> En modo de desarrollo (cuando ejecutas `npm run dev` o `yarn dev`), las páginas se [pre-renderizan](/docs/basic-features/pages#pre-rendering) en cada solicitud. Esto también aplica a la [Generación Estática](https://nextjs.org/docs/basic-features/data-fetching/get-static-props#runs-on-every-request-in-development) para facilitar el desarrollo. Al pasar a producción, la Generación Estática ocurrirá una vez, durante la construcción, y **no** en cada solicitud.

### [Base por Página](#per-page-basis)

Es importante destacar que Next.js te permite **elegir** qué forma de pre-renderizado usar para cada página. Puedes crear una aplicación Next.js "híbrida" utilizando [Generación Estática](/docs/basic-features/pages#static-generation-recommended) para la mayoría de las páginas y [Renderizado del Lado del Servidor (SSR)**](/docs/basic-features/pages#server-side-rendering) para otras.

### [Cuándo Usar](#when-to-use-static-generation-vs-server-side-rendering) [Generación Estática](/docs/basic-features/pages#static-generation-recommended) vs. [Renderizado del Lado del Servidor (SSR)**](/docs/basic-features/pages#server-side-rendering)

Recomendamos usar [**Generación Estática**](/docs/basic-features/pages#static-generation-recommended) (con y sin datos) siempre que sea posible, porque tu página puede construirse una vez y servirse desde una CDN, lo que la hace mucho más rápida que tener un servidor renderizando la página en cada solicitud.

Puedes usar [Generación Estática](/docs/basic-features/pages#static-generation-recommended) para muchos tipos de páginas, incluyendo:

*   Páginas de marketing
*   Publicaciones de blog
*   Listados de productos de comercio electrónico
*   Ayuda y documentación

Debes preguntarte: "¿Puedo pre-renderizar esta página **antes** de que un usuario la solicite?" Si la respuesta es sí, entonces deberías elegir [Generación Estática](/docs/basic-features/pages#static-generation-recommended).

Por otro lado, [Generación Estática](/docs/basic-features/pages#static-generation-recommended) **no** es una buena idea si no puedes pre-renderizar una página antes de que un usuario la solicite. Tal vez tu página muestra datos que se actualizan frecuentemente, y el contenido cambia en cada solicitud.

En ese caso, puedes usar [**Renderizado del Lado del Servidor (SSR)**](/docs/basic-features/pages#server-side-rendering). Será más lento, pero la página pre-renderizada siempre estará actualizada. O puedes omitir el pre-renderizado y usar JavaScript del lado del cliente para poblar datos que se actualizan frecuentemente.
