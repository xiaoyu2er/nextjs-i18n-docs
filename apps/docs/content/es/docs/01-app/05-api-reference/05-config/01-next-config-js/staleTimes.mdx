`staleTimes` es una característica experimental que permite el almacenamiento en caché de segmentos de página en la [caché del enrutador del lado del cliente](/docs/app/deep-dive/caching#client-side-router-cache).

Puede habilitar esta característica experimental y proporcionar tiempos de revalidación personalizados configurando la bandera experimental `staleTimes`:

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

Las propiedades `static` y `dynamic` corresponden con el período de tiempo (en segundos) basado en diferentes tipos de [precarga de enlaces (link prefetching)](/docs/app/api-reference/components/link#prefetch).

- La propiedad `dynamic` se utiliza cuando la página no se genera estáticamente ni se precarga completamente (por ejemplo, con `prefetch={true}`).
  - Valor predeterminado: 0 segundos (no se almacena en caché)
- La propiedad `static` se utiliza para páginas generadas estáticamente, o cuando la propiedad `prefetch` en `Link` está configurada como `true`, o al llamar a [`router.prefetch`](/docs/app/deep-dive/caching#routerprefetch).
  - Valor predeterminado: 5 minutos

> **Es bueno saber:**
>
> - Los [límites de carga (Loading boundaries)](/docs/app/api-reference/file-conventions/loading) se consideran reutilizables durante el período `static` definido en esta configuración.
> - Esto no afecta el [renderizado parcial (partial rendering)](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering), **lo que significa que los diseños compartidos no se volverán a cargar automáticamente en cada navegación, solo el segmento de página que cambia.**
> - Esto no modifica el comportamiento del [almacenamiento en caché de avance/retroceso (back/forward caching)](/docs/app/deep-dive/caching#client-side-router-cache) para evitar cambios en el diseño y prevenir la pérdida de la posición de desplazamiento del navegador.

Puede obtener más información sobre la caché del enrutador del lado del cliente [aquí](/docs/app/deep-dive/caching#client-side-router-cache).

| Versión   | Cambios                                                    |
| --------- | ---------------------------------------------------------- |
| `v15.0.0` | El valor predeterminado de `dynamic` en `staleTimes` cambió de 30s a 0s. |
