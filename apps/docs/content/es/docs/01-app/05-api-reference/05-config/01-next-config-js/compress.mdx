Por defecto, Next.js usa `gzip` para comprimir el contenido renderizado y archivos estáticos cuando se utiliza `next start` o un servidor personalizado. Esta es una optimización para aplicaciones que no tienen configurada la compresión. Si la compresión ya está configurada en tu aplicación a través de un servidor personalizado, Next.js no agregará compresión.

Puedes verificar si la compresión está habilitada y qué algoritmo se utiliza revisando los encabezados [`Accept-Encoding`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Accept-Encoding) (opciones aceptadas por el navegador) y [`Content-Encoding`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Encoding) (actualmente en uso) en la respuesta.

## Deshabilitar la compresión

Para deshabilitar la **compresión**, establece la opción de configuración `compress` en `false`:

```js filename="next.config.js"
module.exports = {
  compress: false,
}
```

**No recomendamos deshabilitar la compresión** a menos que ya tengas configurada la compresión en tu servidor, ya que la compresión reduce el uso de ancho de banda y mejora el rendimiento de tu aplicación. Por ejemplo, si estás usando [nginx](https://nginx.org/) y quieres cambiar a `brotli`, establece la opción `compress` en `false` para permitir que nginx maneje la compresión.
