Next.js no bloquea automáticamente las solicitudes de origen cruzado (cross-origin) durante el desarrollo, pero lo hará por defecto en una futura versión principal de Next.js para evitar solicitudes no autorizadas de recursos/endpoints internos que están disponibles en modo desarrollo.

Para configurar una aplicación Next.js y permitir solicitudes desde orígenes distintos al nombre de host con el que se inicializó el servidor (`localhost` por defecto), puede usar la opción de configuración `allowedDevOrigins`.

`allowedDevOrigins` le permite establecer orígenes adicionales que pueden usarse en modo desarrollo. Por ejemplo, para usar `local-origin.dev` en lugar de solo `localhost`, abra `next.config.js` y agregue la configuración `allowedDevOrigins`:

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```
