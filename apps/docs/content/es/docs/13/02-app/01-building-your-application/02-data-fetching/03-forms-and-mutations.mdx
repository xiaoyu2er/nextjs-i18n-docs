## C√≥mo Funcionan las Acciones de Servidor

Con las Acciones de Servidor, no necesitas crear manualmente endpoints de API. En su lugar, defines funciones as√≠ncronas en el servidor que pueden ser llamadas directamente desde tus componentes.

> **üé• Mira:** Aprende m√°s sobre formularios y mutaciones con el Enrutador de Aplicaci√≥n ‚Üí [YouTube (10 minutos)](https://youtu.be/dDpZfOQBMaU?si=cJZHlUu_jFhCzHUg).

Las Acciones de Servidor pueden definirse en Componentes de Servidor o llamarse desde Componentes de Cliente. Definir la acci√≥n en un Componente de Servidor permite que el formulario funcione sin JavaScript, proporcionando mejora progresiva.

Habilita las Acciones de Servidor en tu archivo `next.config.js`:

```js filename="next.config.js"
module.exports = {
  experimental: {
    serverActions: true,
  },
}
```

> **Es bueno saber:**
>
> - Los formularios que llaman Acciones de Servidor desde Componentes de Servidor pueden funcionar sin JavaScript.
> - Los formularios que llaman Acciones de Servidor desde Componentes de Cliente encolar√°n los env√≠os si JavaScript no est√° cargado a√∫n, priorizando la hidrataci√≥n del cliente.
> - Las Acciones de Servidor heredan el [entorno de ejecuci√≥n (runtime)](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes) de la p√°gina o layout donde se usan.
> - Las Acciones de Servidor funcionan con rutas completamente est√°ticas (incluyendo revalidaci√≥n de datos con ISR).

## Revalidando Datos en Cach√©

Las Acciones de Servidor se integran profundamente con la arquitectura de [cach√© y revalidaci√≥n](/docs/app/building-your-application/caching) de Next.js. Cuando se env√≠a un formulario, la Acci√≥n de Servidor puede actualizar datos en cach√© y revalidar cualquier clave de cach√© que deba cambiar.

A diferencia de las aplicaciones tradicionales limitadas a un solo formulario por ruta, las Acciones de Servidor permiten tener m√∫ltiples acciones por ruta. Adem√°s, el navegador no necesita refrescarse al enviar un formulario. En un solo viaje de red, Next.js puede devolver tanto la UI actualizada como los datos refrescados.

Consulta los ejemplos a continuaci√≥n para [revalidar datos desde Acciones de Servidor](#revalidando-datos).

### Actualizaciones Optimistas

}

  const [optimisticMessages, addOptimisticMessage] = useOptimistic<Message[]>(
    messages,

      ...state,
      { message: newMessage },

      <form
        action={async (formData: FormData) => {
          const message = formData.get('message')
          addOptimisticMessage(message)
          await send(message)
        }}
      >
        <input type="text" name="message" />
        <button type="submit">Send</button>
      </form>
    </div>
  )
}
