Cuando ESLint es detectado en su proyecto, Next.js falla su **compilación de producción** (`next build`) cuando hay errores presentes.

Si desea que Next.js produzca código de producción incluso cuando su aplicación tenga errores de ESLint, puede desactivar completamente el paso de linting integrado. Esto no se recomienda a menos que ya tenga ESLint configurado para ejecutarse en una parte separada de su flujo de trabajo (por ejemplo, en CI o un hook pre-commit).

Abra `next.config.js` y habilite la opción `ignoreDuringBuilds` en la configuración de `eslint`:

```js filename="next.config.js"
module.exports = {
  eslint: {
    // Advertencia: Esto permite que las compilaciones de producción
    // se completen exitosamente incluso si su proyecto tiene errores de ESLint.
    ignoreDuringBuilds: true,
  },
}
```
