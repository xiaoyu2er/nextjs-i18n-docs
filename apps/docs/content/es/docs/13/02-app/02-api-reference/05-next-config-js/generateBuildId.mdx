Next.js utiliza un ID constante generado en tiempo de compilación para identificar qué versión de su aplicación se está sirviendo. Esto puede causar problemas en implementaciones multi-servidor cuando `next build` se ejecuta en cada servidor. Para mantener un ID de compilación consistente entre compilaciones, puede proporcionar su propio ID de compilación.

Abra `next.config.js` y agregue la función `generateBuildId`:

```js filename="next.config.js"
module.exports = {
  generateBuildId: async () => {
    // Puede, por ejemplo, obtener aquí el último hash de commit de git
    return 'my-build-id'
  },
}
```
