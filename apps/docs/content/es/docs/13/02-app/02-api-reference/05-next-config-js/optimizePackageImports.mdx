Algunos paquetes pueden exportar cientos o miles de módulos, lo que puede causar problemas de rendimiento tanto en desarrollo como en producción.

Al agregar un paquete a `experimental.optimizePackageImports`, solo se cargarán los módulos que realmente estés utilizando, manteniendo la conveniencia de escribir sentencias de importación con múltiples exportaciones nombradas.

```js filename="next.config.js"
module.exports = {
  experimental: {
    optimizePackageImports: ['package-name'],
  },
}
```

Bibliotecas como `@mui/icons-material`, `@mui/material`, `date-fns`, `lodash`, `lodash-es`, `react-bootstrap`, `@headlessui/react`, `@heroicons/react` y `lucide-react` ya están optimizadas por defecto.
