Esta sección de la documentación cubre las **convenciones de archivos de metadatos**. Los metadatos basados en archivos pueden definirse añadiendo archivos especiales de metadatos a segmentos de ruta.

Cada convención de archivo puede definirse usando un archivo estático (ej. `opengraph-image.jpg`), o una variante dinámica que utiliza código para generar el archivo (ej. `opengraph-image.js`).

Una vez definido un archivo, Next.js lo servirá automáticamente (con hashes en producción para almacenamiento en caché) y actualizará los elementos head relevantes con los metadatos correctos, como la URL del recurso, tipo de archivo y tamaño de imagen.
