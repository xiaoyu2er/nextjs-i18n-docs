Los mapas de origen (source maps) están habilitados por defecto durante el desarrollo. Durante las construcciones en producción, se deshabilitan para evitar que filtres tu código fuente al cliente, a menos que habilites específicamente esta opción mediante el indicador de configuración.

Next.js proporciona un indicador de configuración que puedes usar para habilitar la generación de mapas de origen del navegador durante la construcción en producción:

```js filename="next.config.js"
module.exports = {
  productionBrowserSourceMaps: true,
}
```

Cuando la opción `productionBrowserSourceMaps` está habilitada, los mapas de origen se generarán en el mismo directorio que los archivos JavaScript. Next.js servirá automáticamente estos archivos cuando se soliciten.

- Agregar mapas de origen puede aumentar el tiempo de `next build`
- Incrementa el uso de memoria durante `next build`
