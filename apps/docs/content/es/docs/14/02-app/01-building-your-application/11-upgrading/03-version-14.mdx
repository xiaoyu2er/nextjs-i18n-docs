## Actualización de la versión 13 a la 14

Para actualizar a la versión 14 de Next.js, ejecute el siguiente comando usando su gestor de paquetes preferido:

```bash filename="Terminal"
npm i next@latest react@latest react-dom@latest eslint-config-next@latest
```

```bash filename="Terminal"
yarn add next@latest react@latest react-dom@latest eslint-config-next@latest
```

```bash filename="Terminal"
pnpm up next react react-dom eslint-config-next --latest
```

```bash filename="Terminal"
bun add next@latest react@latest react-dom@latest eslint-config-next@latest
```

> **Nota importante:** Si está usando TypeScript, asegúrese de actualizar también `@types/react` y `@types/react-dom` a sus últimas versiones.

### Resumen de la versión 14

- La versión mínima de Node.js ha sido actualizada de 16.14 a 18.17, ya que la versión 16.x ha llegado al fin de su ciclo de vida.
- El comando `next export` ha sido eliminado en favor de la configuración `output: 'export'`. Consulte la [documentación](https://nextjs.org/docs/app/building-your-application/deploying/static-exports) para más información.
- La importación `next/server` para `ImageResponse` ha sido renombrada a `next/og`. Hay disponible un [codemod](/docs/app/building-your-application/upgrading/codemods#next-og-import) para renombrar sus importaciones de manera segura y automática.
- El paquete `@next/font` ha sido completamente eliminado en favor del `next/font` integrado. Hay disponible un [codemod](/docs/app/building-your-application/upgrading/codemods#built-in-next-font) para renombrar sus importaciones de manera segura y automática.
- El objetivo WASM para `next-swc` ha sido eliminado.
