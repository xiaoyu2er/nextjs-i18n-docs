<AppOnly>

Cuando editas tu código y Next.js está compilando la aplicación, aparece un indicador de compilación en la esquina inferior derecha de la página.

> **Es bueno saber**: Este indicador solo está presente en modo desarrollo y no aparecerá al construir y ejecutar la aplicación en modo producción.

En algunos casos este indicador puede aparecer en una posición incorrecta en tu página, por ejemplo, cuando entra en conflicto con un lanzador de chat. Para cambiar su posición, abre `next.config.js` y configura `buildActivityPosition` en el objeto `devIndicators` con los valores `bottom-right` (predeterminado), `bottom-left`, `top-right` o `top-left`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivityPosition: 'bottom-right',
  },
}
```

En algunos casos este indicador puede no ser útil para ti. Para eliminarlo, abre `next.config.js` y desactiva la configuración `buildActivity` en el objeto `devIndicators`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivity: false,
  },
}
```

</AppOnly>

<PagesOnly>

> **Es bueno saber**: Este indicador fue eliminado en la versión 10.0.1 de Next.js. Recomendamos actualizar a la última versión de Next.js.

Cuando una página califica para [Optimización Estática Automática](/docs/pages/building-your-application/rendering/automatic-static-optimization) mostramos un indicador para informarte.

Esto es útil ya que la optimización estática automática puede ser muy beneficiosa y saber inmediatamente en desarrollo si la página califica puede ser práctico.

En algunos casos este indicador puede no ser útil, como cuando trabajas en aplicaciones de electron. Para eliminarlo abre `next.config.js` y desactiva la configuración `autoPrerender` en `devIndicators`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    autoPrerender: false,
  },
}
```

</PagesOnly>
