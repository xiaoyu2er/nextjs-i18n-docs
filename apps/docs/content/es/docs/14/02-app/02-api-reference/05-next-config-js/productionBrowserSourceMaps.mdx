Los mapas de origen (source maps) están habilitados por defecto durante el desarrollo. Durante las compilaciones de producción, se deshabilitan para evitar que se filtre el código fuente al cliente, a menos que se habilite explícitamente con la bandera de configuración.

Next.js proporciona una bandera de configuración que puedes usar para habilitar la generación de mapas de origen del navegador durante la compilación de producción:

```js filename="next.config.js"
module.exports = {
  productionBrowserSourceMaps: true,
}
```

Cuando la opción `productionBrowserSourceMaps` está habilitada, los mapas de origen se generarán en el mismo directorio que los archivos JavaScript. Next.js servirá automáticamente estos archivos cuando se soliciten.

- Agregar mapas de origen puede aumentar el tiempo de `next build`
- Incrementa el uso de memoria durante `next build`
