Cuando ESLint es detectado en tu proyecto, Next.js falla tu **compilación de producción** (`next build`) cuando hay errores presentes.

Si deseas que Next.js produzca código de producción incluso cuando tu aplicación tenga errores de ESLint, puedes desactivar completamente el paso de linting integrado. Esto no se recomienda a menos que ya tengas ESLint configurado para ejecutarse en una parte separada de tu flujo de trabajo (por ejemplo, en CI o un hook pre-commit).

Abre `next.config.js` y habilita la opción `ignoreDuringBuilds` en la configuración de `eslint`:

```js filename="next.config.js"
module.exports = {
  eslint: {
    // Advertencia: Esto permite que las compilaciones de producción
    // se completen exitosamente incluso si tu proyecto tiene errores de ESLint.
    ignoreDuringBuilds: true,
  },
}
```
