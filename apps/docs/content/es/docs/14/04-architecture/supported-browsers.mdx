Next.js soporta **navegadores modernos** sin necesidad de configuración adicional.

- Chrome 64+
- Edge 79+
- Firefox 67+
- Opera 51+
- Safari 12+

## Browserslist

Si necesita compatibilidad con navegadores o características específicas, Next.js soporta configuración de [Browserslist](https://browsersl.ist) en su archivo `package.json`. Next.js usa la siguiente configuración de Browserslist por defecto:

```json filename="package.json"
{
  "browserslist": [
    "chrome 64",
    "edge 79",
    "firefox 67",
    "opera 51",
    "safari 12"
  ]
}
```

## Polyfills

Inyectamos [polyfills ampliamente utilizados](https://github.com/vercel/next.js/blob/canary/packages/next-polyfill-nomodule/src/index.js), incluyendo:

- [**fetch()**](https://developer.mozilla.org/docs/Web/API/Fetch_API) — Reemplaza: `whatwg-fetch` y `unfetch`.
- [**URL**](https://developer.mozilla.org/docs/Web/API/URL) — Reemplaza: el [paquete `url` (API de Node.js)](https://nodejs.org/api/url.html).
- [**Object.assign()**](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Object/assign) — Reemplaza: `object-assign`, `object.assign` y `core-js/object/assign`.

Si alguna de sus dependencias incluye estos polyfills, se eliminarán automáticamente del build de producción para evitar duplicaciones.

Además, para reducir el tamaño del bundle, Next.js solo cargará estos polyfills para navegadores que los requieran. La mayoría del tráfico web global no descargará estos polyfills.

### Polyfills personalizados

Si su código o alguna dependencia externa de npm requiere características no soportadas por sus navegadores objetivo (como IE 11), deberá agregar polyfills manualmente.

En este caso, debe agregar una importación de nivel superior para el **polyfill específico** que necesite en su [Custom `<App>`](/docs/pages/building-your-application/routing/custom-app) o en el componente individual.

## Características de JavaScript

Next.js le permite usar las últimas características de JavaScript sin configuración adicional. Además de [características ES6](https://github.com/lukehoban/es6features), Next.js también soporta:

- [Async/await](https://github.com/tc39/ecmascript-asyncawait) (ES2017)
- [Propiedades Rest/Spread de objetos](https://github.com/tc39/proposal-object-rest-spread) (ES2018)
- [`import()` dinámico](https://github.com/tc39/proposal-dynamic-import) (ES2020)
- [Encadenamiento opcional](https://github.com/tc39/proposal-optional-chaining) (ES2020)
- [Fusión nula](https://github.com/tc39/proposal-nullish-coalescing) (ES2020)
- [Campos de clase](https://github.com/tc39/proposal-class-fields) y [propiedades estáticas](https://github.com/tc39/proposal-static-class-features) (ES2022)
- ¡y más!
