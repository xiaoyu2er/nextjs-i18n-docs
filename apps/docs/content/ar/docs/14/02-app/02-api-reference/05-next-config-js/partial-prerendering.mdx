> **تحذير**: التصيير الجزئي المسبق ميزة تجريبية وهي حاليًا **غير مناسبة لبيئات الإنتاج**.

التصيير الجزئي المسبق هو ميزة تجريبية تسمح بتصيير الأجزاء الثابتة من المسار مسبقًا وعرضها من ذاكرة التخزين المؤقت مع دفق الأجزاء الديناميكية، كل ذلك في طلب HTTP واحد.

التصيير الجزئي المسبق متاح في `next@canary`:

```bash filename="Terminal"
npm install next@canary
```

يمكنك تمكين التصيير الجزئي المسبق عن طريق تعيين علم `ppr` التجريبي:

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    ppr: true,
  },
}

module.exports = nextConfig
```

> **معلومة مفيدة:**
>
> - التصيير الجزئي المسبق لا ينطبق بعد على التنقلات من جانب العميل. نحن نعمل بنشاط على هذا.
> - تم تصميم التصيير الجزئي المسبق ليعمل فقط مع [بيئة تشغيل Node.js](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes). لا حاجة لاستخدام مجموعة فرعية من بيئة تشغيل Node.js عندما يمكنك تقديم القشرة الثابتة على الفور.

تعلم المزيد عن التصيير الجزئي المسبق في [دورة تعلم Next.js](/learn/dashboard-app/partial-prerendering).
