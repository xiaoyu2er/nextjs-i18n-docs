<AppOnly>

عند تعديل الكود الخاص بك، وعندما يقوم Next.js بتحويل التطبيق، يظهر مؤشر التحويل في الزاوية اليمنى السفلية للصفحة.

> **من الجيد معرفة**: هذا المؤشر موجود فقط في وضع التطوير ولن يظهر عند بناء وتشغيل التطبيق في وضع الإنتاج.

في بعض الحالات قد يكون هذا المؤشر في موضع غير مناسب في صفحتك، على سبيل المثال، عند تعارضه مع مشغل الدردشة. لتغيير موضعه، افتح ملف `next.config.js` وقم بتعيين `buildActivityPosition` في كائن `devIndicators` إلى `bottom-right` (الافتراضي)، `bottom-left`، `top-right` أو `top-left`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivityPosition: 'bottom-right',
  },
}
```

في بعض الحالات قد لا يكون هذا المؤشر مفيدًا لك. لإزالته، افتح ملف `next.config.js` وقم بتعطيل إعداد `buildActivity` في كائن `devIndicators`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivity: false,
  },
}
```

</AppOnly>

<PagesOnly>

> **من الجيد معرفة**: تمت إزالة هذا المؤشر في إصدار Next.js 10.0.1. ننصح بالترقية إلى أحدث إصدار من Next.js.

عندما تكون الصفحة مؤهلة لـ[التحسين الثابت التلقائي](/docs/pages/building-your-application/rendering/automatic-static-optimization) نعرض مؤشرًا لإعلامك بذلك.

هذا مفيد لأن التحسين الثابت التلقائي يمكن أن يكون مفيدًا جدًا، ومعرفة ما إذا كانت الصفحة مؤهلة لذلك أثناء التطوير يمكن أن يكون مفيدًا.

في بعض الحالات قد لا يكون هذا المؤشر مفيدًا، مثل عند العمل على تطبيقات إلكترون. لإزالته افتح ملف `next.config.js` وقم بتعطيل إعداد `autoPrerender` في كائن `devIndicators`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    autoPrerender: false,
  },
}
```

</PagesOnly>
