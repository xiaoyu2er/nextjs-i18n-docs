`useSearchParams` هو hook لمكون **العميل (Client Component)** يتيح لك قراءة **سلسلة الاستعلام (query string)** للرابط الحالي.

`useSearchParams` يُرجع نسخة **للقراءة فقط** من واجهة [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams).

```tsx filename="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // URL -> `/dashboard?search=my-project`
  // `search` -> 'my-project'
  return <>Search: {search}</>
}
```

```jsx filename="app/dashboard/search-bar.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // URL -> `/dashboard?search=my-project`
  // `search` -> 'my-project'
  return <>Search: {search}</>
}
```

## المعاملات

```tsx
const searchParams = useSearchParams()
```

`useSearchParams` لا يأخذ أي معاملات.

## القيم المُرجعة

`useSearchParams` يُرجع نسخة **للقراءة فقط** من واجهة [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams)، والتي تتضمن طرقًا مساعدة لقراءة سلسلة الاستعلام للرابط:

- [`URLSearchParams.get()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/get): تُرجع القيمة الأولى المرتبطة بمعامل البحث. على سبيل المثال:

  | الرابط                  | `searchParams.get("a")`                                                                                         |
  | -------------------- | --------------------------------------------------------------------------------------------------------------- |
  | `/dashboard?a=1`     | `'1'`                                                                                                           |
  | `/dashboard?a=`      | `''`                                                                                                            |
  | `/dashboard?b=3`     | `null`                                                                                                          |
  | `/dashboard?a=1&a=2` | `'1'` _- استخدم [`getAll()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/getAll) للحصول على جميع القيم_ |

- [`URLSearchParams.has()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/has): تُرجع قيمة منطقية تشير إلى وجود المعامل المحدد. على سبيل المثال:

  | الرابط              | `searchParams.has("a")` |
  | ---------------- | ----------------------- |
  | `/dashboard?a=1` | `true`                  |
  | `/dashboard?b=3` | `false`                 |

- تعرّف على المزيد حول الطرق الأخرى **للقراءة فقط** في [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams)، بما في ذلك [`getAll()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/getAll)، [`keys()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/keys)، [`values()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/values)، [`entries()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/entries)، [`forEach()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/forEach)، و [`toString()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/toString).

> **معلومة مفيدة**:
>
> - `useSearchParams` هو hook لمكون [العميل (Client Component)](/docs/app/getting-started/server-and-client-components) و **غير مدعوم** في [مكونات الخادم (Server Components)](/docs/app/getting-started/server-and-client-components) لمنع قيم قديمة أثناء [التصيير الجزئي (partial rendering)](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering).
> - إذا كان التطبيق يتضمن دليل `/pages`، فإن `useSearchParams` سيرجع `ReadonlyURLSearchParams | null`. القيمة `null` هي لأغراض التوافق أثناء الترحيل حيث لا يمكن معرفة معلمات البحث أثناء التصيير المسبق لصفحة لا تستخدم `getServerSideProps`.

## السلوك

### التصيير الثابت (Static Rendering)

إذا كان المسار [مُصيَّرًا بشكل ثابت](/docs/app/getting-started/partial-prerendering#static-rendering)، فإن استدعاء `useSearchParams` سيتسبب في تصيير شجرة مكون العميل حتى أقرب [حد `Suspense`](/docs/app/building-your-application/routing/loading-ui-and-streaming#example) على جانب العميل.

هذا يسمح لجزء من المسار أن يكون مُصيَّرًا بشكل ثابت بينما الجزء الديناميكي الذي يستخدم `useSearchParams` يتم تصييره على جانب العميل.

نوصي بلف مكون العميل الذي يستخدم `useSearchParams` داخل حدود `<Suspense/>`. هذا سيسمح لأي مكونات عميل فوقه أن تكون مُصيَّرة بشكل ثابت وإرسالها كجزء من HTML الأولي. [مثال](/docs/app/api-reference/functions/use-search-params#static-rendering).

على سبيل المثال:

```tsx filename="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // هذا لن يتم تسجيله على الخادم عند استخدام التصيير الثابت
  console.log(search)

  return <>Search: {search}</>
}
```

```jsx filename="app/dashboard/search-bar.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

  const searchParams = useSearchParams()

  const search = searchParams.get('search')

}
```

```tsx filename="app/dashboard/page.tsx" switcher

import SearchBar from './search-bar'

// هذا المكون الممرر كاحتياطي لحدود Suspense
// سيتم تصييره مكان شريط البحث في HTML الأولي.
// عندما تصبح القيمة متاحة أثناء تهيئة React، سيتم استبدال
// الاحتياطي بمكون `<SearchBar>`.
function SearchBarFallback() {
  return <>placeholder</>
}

export default function Page() {
  return (
    <>
      <nav>
        <Suspense fallback={<SearchBarFallback />}>
          <SearchBar />
        </Suspense>
      </nav>
      <h1>Dashboard</h1>
    </>
  )
}
```

### التصيير الديناميكي (Dynamic Rendering)

إذا كان المسار [مُصيَّرًا بشكل ديناميكي](/docs/app/getting-started/partial-prerendering#dynamic-rendering)، فإن `useSearchParams` سيكون متاحًا على الخادم أثناء التصيير الأولي لمكون العميل.

على سبيل المثال:

```tsx filename="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // هذا سيتم تسجيله على الخادم أثناء التصيير الأولي
  // وعلى العميل أثناء التنقلات اللاحقة.
  console.log(search)

  return <>Search: {search}</>
}
```

```jsx filename="app/dashboard/search-bar.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {

  // وعلى العميل أثناء التنقلات اللاحقة.
  console.log(search)

  return <>Search: {search}</>
}
```

### مكونات الخادم (Server Components)

#### الصفحات (Pages)

للوصول إلى معلمات البحث في [الصفحات (Pages)](/docs/app/api-reference/file-conventions/page) (مكونات الخادم)، استخدم خاصية [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional).

#### التخطيطات (Layouts)

على عكس الصفحات، [التخطيطات (Layouts)](/docs/app/api-reference/file-conventions/layout) (مكونات الخادم) **لا** تتلقى خاصية `searchParams`. هذا لأن التخطيط المشترك [لا يتم إعادة تصييره أثناء التنقل](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering) مما قد يؤدي إلى `searchParams` قديمة بين التنقلات. شاهد [التفسير المفصل](/docs/app/api-reference/file-conventions/layout#query-params).

بدلاً من ذلك، استخدم خاصية [`searchParams`](/docs/app/api-reference/file-conventions/page) للصفحة أو hook [`useSearchParams`](/docs/app/api-reference/functions/use-search-params) في مكون عميل، والذي يتم إعادة تصييره على العميل بأحدث `searchParams`.

## أمثلة

### تحديث `searchParams`

يمكنك استخدام [`useRouter`](/docs/app/api-reference/functions/use-router) أو [`Link`](/docs/app/api-reference/components/link) لتعيين `searchParams` جديدة. بعد إجراء التنقل، ستتلقى الصفحة الحالية [`page.js`](/docs/app/api-reference/file-conventions/page) خاصية [`searchParams` محدثة](/docs/app/api-reference/file-conventions/page#searchparams-optional).

```tsx filename="app/example-client-component.tsx" switcher
'use client'

export default function ExampleClientComponent() {
  const router = useRouter()
  const pathname = usePathname()
  const searchParams = useSearchParams()

  // الحصول على سلسلة searchParams جديدة عن طريق دمج
  // searchParams الحالية مع زوج مفتاح/قيمة معطى
  const createQueryString = useCallback(
    (name: string, value: string) => {
      const params = new URLSearchParams(searchParams.toString())
      params.set(name, value)

      return params.toString()
    },
    [searchParams]
  )

  return (
    <>
      <p>Sort By</p>

      {/* باستخدام useRouter */}
      <button
        onClick={() => {
          // <pathname>?sort=asc
          router.push(pathname + '?' + createQueryString('sort', 'asc'))
        }}
      >
        ASC
      </button>

      {/* باستخدام <Link> */}
      <Link
        href={
          // <pathname>?sort=desc
          pathname + '?' + createQueryString('sort', 'desc')
        }
      >
        DESC
      </Link>
    </>
  )
}
```
