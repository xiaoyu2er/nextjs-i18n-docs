`staleTimes` هي ميزة تجريبية تمكن من تخزين مقاطع الصفحة في [ذاكرة التخزين المؤقت لموجه العميل](/docs/app/deep-dive/caching#client-side-router-cache).

يمكنك تمكين هذه الميزة التجريبية وتوفير أوقات إعادة تحقق مخصصة عن طريق تعيين علامة `staleTimes` التجريبية:

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

تتوافق خصائص `static` و `dynamic` مع الفترة الزمنية (بالثواني) بناءً على أنواع مختلفة من [الجلب المسبق للروابط](/docs/app/api-reference/components/link#prefetch).

- تُستخدم خاصية `dynamic` عندما لا تكون الصفحة مُولدة بشكل ثابت أو غير مسبوقة الجلب بالكامل (مثل `prefetch={true}`).
  - القيمة الافتراضية: 0 ثانية (غير مخزنة مؤقتًا)
- تُستخدم خاصية `static` للصفحات المولدة بشكل ثابت، أو عند تعيين خاصية `prefetch` في مكون `Link` إلى `true`، أو عند استدعاء [`router.prefetch`](/docs/app/deep-dive/caching#routerprefetch).
  - القيمة الافتراضية: 5 دقائق

> **معلومة مفيدة:**
>
> - تعتبر [حدود التحميل](/docs/app/api-reference/file-conventions/loading) قابلة لإعادة الاستخدام لفترة `static` المحددة في هذا التكوين.
> - لا يؤثر هذا على [التصيير الجزئي](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering)، **بمعنى أن التخطيطات المشتركة لن يتم إعادة جلبها تلقائيًا في كل تنقل، فقط مقطع الصفحة الذي يتغير.**
> - لا يغير هذا سلوك [ذاكرة التخزين المؤقت للخلف/الأمام](/docs/app/deep-dive/caching#client-side-router-cache) لمنع تغيير التخطيط ولمنع فقدان موضع التمرير في المتصفح.

يمكنك معرفة المزيد عن ذاكرة التخزين المؤقت لموجه العميل [هنا](/docs/app/deep-dive/caching#client-side-router-cache).
