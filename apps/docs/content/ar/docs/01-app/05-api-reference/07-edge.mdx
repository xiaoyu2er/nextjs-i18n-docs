يحتوي Next.js على وقتين تشغيل للخادم يمكنك استخدامهما في تطبيقك:

- **وقت تشغيل Node.js** (الافتراضي)، الذي يتيح الوصول إلى جميع واجهات برمجة تطبيقات Node.js ويستخدم لعرض تطبيقك.
- **وقت تشغيل Edge** الذي يحتوي على [مجموعة محدودة من واجهات برمجة التطبيقات](#reference)، ويستخدم في [الوسيط (Middleware)](/docs/app/api-reference/file-conventions/middleware).

## محاذير

- لا يدعم وقت تشغيل Edge جميع واجهات برمجة تطبيقات Node.js. قد لا تعمل بعض الحزم كما هو متوقع.
- لا يدعم وقت تشغيل Edge التجديد الثابت التدريجي (ISR).
- يمكن لكلا وقت التشغيل دعم [البث (streaming)](/docs/app/building-your-application/routing/loading-ui-and-streaming) اعتمادًا على محول النشر الخاص بك.

### واجهات برمجة التطبيقات غير المدعومة (Unsupported APIs)

يحتوي وقت تشغيل الحافة (Edge Runtime) على بعض القيود بما في ذلك:

- واجهات برمجة تطبيقات Node.js الأصلية **غير مدعومة**. على سبيل المثال، لا يمكنك القراءة أو الكتابة في نظام الملفات.
- يمكن استخدام `node_modules` طالما أنها تنفذ وحدات ES (ES Modules) ولا تستخدم واجهات برمجة تطبيقات Node.js الأصلية.
- استدعاء `require` مباشرة **غير مسموح به**. استخدم وحدات ES بدلاً من ذلك.

ميزات لغة JavaScript التالية معطلة، **ولن تعمل:**

| واجهة برمجة التطبيق (API)                                                                                                                             | الوصف                                                         |
| ------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| [`eval`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/eval)                                       | يقيم كود JavaScript ممثلًا كسلسلة نصية                   |
| [`new Function(evalString)`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Function)               | ينشئ دالة جديدة بالكود المقدم كوسيطة        |
| [`WebAssembly.compile`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/compile)         | يترجم وحدة WebAssembly من مصدر مخزن في buffer                  |
| [`WebAssembly.instantiate`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/instantiate) | يترجم وينشئ وحدة WebAssembly من مصدر مخزن في buffer |

في حالات نادرة، قد يحتوي الكود الخاص بك (أو يستورد) بعض عبارات تقييم الكود الديناميكي التي _لا يمكن الوصول إليها أثناء التشغيل_ ولا يمكن إزالتها عن طريق تقليل الشجرة (treeshaking).
يمكنك تخفيف الفحص للسماح بملفات محددة من خلال تكوين Middleware الخاص بك:

```javascript filename="middleware.ts"
export const config = {
  unstable_allowDynamic: [
    // يسمح بملف واحد
    '/lib/utilities.js',
    // استخدم النمط العام (glob) للسماح بأي شيء في وحدة function-bind من طرف ثالث
    '**/node_modules/function-bind/**',
  ],
}
```

`unstable_allowDynamic` هو [نمط عام (glob)](https://github.com/micromatch/micromatch#matching-features)، أو مصفوفة من الأنماط العامة، تتجاهل تقييم الكود الديناميكي لملفات محددة. الأنماط العامة مرتبطة بمجلد جذر التطبيق الخاص بك.

تحذير: إذا تم تنفيذ هذه العبارات على الحافة (Edge)، _فسوف تسبب خطأ أثناء التشغيل_.
