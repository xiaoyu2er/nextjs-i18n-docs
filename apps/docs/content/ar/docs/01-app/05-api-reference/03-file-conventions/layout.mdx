يُستخدم ملف `layout` لتعريف تخطيط في تطبيق Next.js الخاص بك.

```tsx filename="app/dashboard/layout.tsx" switcher
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return <section>{children}</section>
}
```

```jsx filename="app/dashboard/layout.js" switcher
export default function DashboardLayout({ children }) {
  return <section>{children}</section>
}
```

**التخطيط الجذري** هو أعلى تخطيط في دليل `app` الجذري. يُستخدم لتعريف علامات `<html>` و `<body>` وواجهة المستخدم المشتركة عالميًا.

```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

## المرجع

### الخصائص

#### `children` (مطلوب)

يجب أن تقبل مكونات التخطيط وتستخدم خاصية `children`. أثناء التصيير، سيتم تعبئة `children` بمقاطع المسار التي يحيط بها التخطيط. ستكون هذه في الغالب مكون [التخطيط](/docs/app/api-reference/file-conventions/page) الفرعي (إذا كان موجودًا) أو [الصفحة](/docs/app/api-reference/file-conventions/page)، ولكن يمكن أن تكون أيضًا ملفات خاصة أخرى مثل [التحميل](/docs/app/building-your-application/routing/loading-ui-and-streaming) أو [الخطأ](/docs/app/getting-started/error-handling) عند الاقتضاء.

#### `params` (اختياري)

وعد يحل إلى كائن يحتوي على كائن [معلمات المسار الديناميكي](/docs/app/api-reference/file-conventions/dynamic-routes) من المقطع الجذري حتى ذلك التخطيط.

```tsx filename="app/dashboard/[team]/layout.tsx" switcher
export default async function Layout({
  params,
}: {
  params: Promise<{ team: string }>
}) {
  const { team } = await params
}
```

```jsx filename="app/dashboard/[team]/layout.js" switcher
export default async function Layout({ params }) {
  const { team } = await params
}
```

| مثال المسار                     | URL            | `params`                           |
| ------------------------------- | -------------- | ---------------------------------- |
| `app/dashboard/[team]/layout.js`  | `/dashboard/1` | `Promise<{ team: '1' }>`           |
| `app/shop/[tag]/[item]/layout.js` | `/shop/1/2`    | `Promise<{ tag: '1', item: '2' }>` |
| `app/blog/[...slug]/layout.js`    | `/blog/1/2`    | `Promise<{ slug: ['1', '2'] }>`    |

- نظرًا لأن خاصية `params` هي وعد، يجب عليك استخدام `async/await` أو دالة [`use`](https://react.dev/reference/react/use) من React للوصول إلى القيم.
  - في الإصدار 14 وما قبله، كانت `params` خاصية متزامنة. للمساعدة في التوافق مع الإصدارات السابقة، لا يزال بإمكانك الوصول إليها بشكل متزامن في Next.js 15، ولكن هذا السلوك سيتم إهماله في المستقبل.

### التخطيط الجذري

**يجب** أن يتضمن دليل `app` تخطيطًا جذريًا `app/layout.js`.

```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html>
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html>
      <body>{children}</body>
    </html>
  )
}
```

- **يجب** أن يحدد التخطيط الجذري علامات `<html>` و `<body>`.
  - **لا يجب** إضافة علامات `<head>` يدويًا مثل `<title>` و `<meta>` إلى التخطيطات الجذرية. بدلاً من ذلك، يجب استخدام [واجهة برمجة تطبيقات البيانات الوصفية (Metadata API)](/docs/app/getting-started/metadata-and-og-images) التي تتعامل تلقائيًا مع المتطلبات المتقدمة مثل البث وإزالة تكرار عناصر `<head>`.
- يمكنك استخدام [مجموعات المسار (route groups)](/docs/app/api-reference/file-conventions/route-groups) لإنشاء تخطيطات جذرية متعددة.
  - التنقل **بين تخطيطات جذرية متعددة** سيؤدي إلى **تحميل كامل للصفحة** (بدلاً من التنقل من جانب العميل). على سبيل المثال، التنقل من `/cart` الذي يستخدم `app/(shop)/layout.js` إلى `/blog` الذي يستخدم `app/(marketing)/layout.js` سيؤدي إلى تحميل كامل للصفحة. هذا **ينطبق فقط** على التخطيطات الجذرية المتعددة.

## محاذير

### كائن الطلب

يتم تخزين التخطيطات في ذاكرة التخزين المؤقت للعميل أثناء التنقل لتجنب طلبات الخادم غير الضرورية.

[التخطيطات](/docs/app/api-reference/file-conventions/layout) لا يتم إعادة تصييرها. يمكن تخزينها مؤقتًا وإعادة استخدامها لتجنب الحسابات غير الضرورية عند التنقل بين الصفحات. من خلال تقييد وصول التخطيطات إلى الطلب الخام، يمكن لـ Next.js منع تنفيذ التعليمات البرمجية للمستخدم التي قد تكون بطيئة أو مكلفة داخل التخطيط، مما قد يؤثر سلبًا على الأداء.

للوصول إلى كائن الطلب، يمكنك استخدام واجهات برمجة تطبيقات [`headers`](/docs/app/api-reference/functions/headers) و [`cookies`](/docs/app/api-reference/functions/cookies) في [مكونات الخادم (Server Components)](/docs/app/getting-started/server-and-client-components) والوظائف.

```tsx filename="app/shop/layout.tsx" switcher
import { cookies } from 'next/headers'

export default async function Layout({ children }) {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

```jsx filename="app/shop/layout.js" switcher
import { cookies } from 'next/headers'

export default async function Layout({ children }) {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

### معلمات الاستعلام

لا يتم إعادة تصيير التخطيطات عند التنقل، لذا لا يمكنها الوصول إلى معلمات البحث التي قد تصبح قديمة.

للوصول إلى معلمات الاستعلام المحدثة، يمكنك استخدام خاصية [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) للصفحة، أو قراءتها داخل مكون العميل باستخدام خطاف [`useSearchParams`](/docs/app/api-reference/functions/use-search-params). نظرًا لأن مكونات العميل يتم إعادة تصييرها عند التنقل، فإن لديها إمكانية الوصول إلى أحدث معلمات الاستعلام.

```tsx filename="app/ui/search.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function Search() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  return '...'
}
```

```jsx filename="app/ui/search.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function Search() {

  const search = searchParams.get('search')

  return '...'
```
```jsx filename="app/shop/layout.js" switcher
}
```

### مسار الصفحة

لا يتم إعادة تصيير التخطيطات عند التنقل، لذا لا يمكنها الوصول إلى مسار الصفحة الذي قد يصبح قديمًا.

للوصول إلى مسار الصفحة الحالي، يمكنك قراءته داخل مكون العميل باستخدام خطاف [`usePathname`](https://nextjs.org/docs/app/api-reference/functions/use-pathname). نظرًا لأن مكونات العميل يتم إعادة تصييرها أثناء التنقل، فإن لديها إمكانية الوصول إلى أحدث مسار للصفحة.

```tsx filename="app/ui/breadcrumbs.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'

// منطق تتبع المسار المبسط
export default function Breadcrumbs() {
  const pathname = usePathname()
  const segments = pathname.split('/')

  return (
    <nav>
      {segments.map((segment, index) => (
        <span key={index}>
          {' > '}
          {segment}
        </span>
      ))}
    </nav>
  )
}
```


### جلب البيانات

لا يمكن للتخطيطات تمرير البيانات إلى `children` الخاصة بها. ومع ذلك، يمكنك جلب نفس البيانات في المسار أكثر من مرة، واستخدام [`cache`](https://react.dev/reference/react/cache) من React لإزالة تكرار الطلبات دون التأثير على الأداء.

بدلاً من ذلك، عند استخدام [`fetch`](https://nextjs.org/docs/app/api-reference/functions/fetch) في Next.js، يتم إزالة تكرار الطلبات تلقائيًا.

```tsx filename="app/lib/data.ts" switcher
export async function getUser(id: string) {
  const res = await fetch(`https://.../users/${id}`)
  return res.json()
}
```

```tsx filename="app/dashboard/layout.tsx" switcher
import { getUser } from '@/app/lib/data'
```

### الوصول إلى المقاطع الفرعية

لا يمكن للتخطيطات الوصول إلى مقاطع المسار الموجودة أسفلها. للوصول إلى جميع مقاطع المسار، يمكنك استخدام [`useSelectedLayoutSegment`](/docs/app/api-reference/functions/use-selected-layout-segment) أو [`useSelectedLayoutSegments`](/docs/app/api-reference/functions/use-selected-layout-segments) في مكون العميل.

```tsx filename="app/ui/nav-link.tsx" switcher
'use client'

import Link from 'next/link'
import { useSelectedLayoutSegment } from 'next/navigation'

export default function NavLink({
  slug,
  children,
}: {
  slug: string
  children: React.ReactNode
}) {
  const segment = useSelectedLayoutSegment()
  const isActive = slug === segment

  return (
    <Link
      href={`/blog/${slug}`}
      // تغيير النمط حسب ما إذا كان الرابط نشطًا
      style={{ fontWeight: isActive ? 'bold' : 'normal' }}
    >
      {children}
    </Link>
  )
}
```

```jsx filename="app/ui/nav-link.js" switcher
'use client'

import Link from 'next/link'

export default function NavLinks({ slug, children }) {

    <Link
      href={`/blog/${slug}`}
      style={{ fontWeight: isActive ? 'bold' : 'normal' }}
    >
      {children}
    </Link>
  )
}
```



## أمثلة

### البيانات الوصفية

يمكنك تعديل عناصر `<head>` في HTML مثل `title` و `meta` باستخدام [كائن `metadata`](/docs/app/api-reference/functions/generate-metadata#the-metadata-object) أو [دالة `generateMetadata`](/docs/app/api-reference/functions/generate-metadata#generatemetadata-function).

```tsx filename="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Next.js',
}

export default function Layout({ children }: { children: React.ReactNode }) {
  return '...'
}
```

```jsx filename="app/layout.js" switcher
export const metadata = {
  title: 'Next.js',
}

export default function Layout({ children }) {
  return '...'
}
```

> **ملاحظة جيدة**: **لا يجب** إضافة علامات `<head>` يدويًا مثل `<title>` و `<meta>` إلى التخطيطات الجذرية. بدلاً من ذلك، استخدم [واجهات برمجة تطبيقات البيانات الوصفية (Metadata APIs)](/docs/app/api-reference/functions/generate-metadata) التي تتعامل تلقائيًا مع المتطلبات المتقدمة مثل البث وإزالة تكرار عناصر `<head>`.

### روابط التنقل النشطة

يمكنك استخدام خطاف [`usePathname`](/docs/app/api-reference/functions/use-pathname) لتحديد ما إذا كان رابط التنقل نشطًا.

نظرًا لأن `usePathname` هو خطاف للعميل، فأنت بحاجة إلى استخراج روابط التنقل إلى مكون عميل، والذي يمكن استيراده إلى تخطيطك:

```tsx filename="app/ui/nav-links.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'
import Link from 'next/link'

export function NavLinks() {
  const pathname = usePathname()

  return (
    <nav>
      <Link className={`link ${pathname === '/' ? 'active' : ''}`} href="/">
        الرئيسية
      </Link>

      <Link
        className={`link ${pathname === '/about' ? 'active' : ''}`}
        href="/about"
      >
        حول
      </Link>
    </nav>
  )
}
```

### عرض المحتوى بناءً على `params`

باستخدام [شرائح المسارات الديناميكية (dynamic route segments)](/docs/app/api-reference/file-conventions/dynamic-routes)، يمكنك عرض أو جلب محتوى محدد بناءً على خاصية `params`.

```tsx filename="app/dashboard/layout.tsx" switcher
export default async function DashboardLayout({
  children,
  params,
}: {
  children: React.ReactNode
  params: Promise<{ team: string }>
}) {
  const { team } = await params

  return (
    <section>
      <header>
        <h1>Welcome to {team}'s Dashboard</h1>
      </header>
      <main>{children}</main>
    </section>
  )
}
```

```jsx filename="app/dashboard/layout.js" switcher
export default async function DashboardLayout({ children, params }) {
  const { team } = await params

  return (
    <section>
      <header>
        <h1>Welcome to {team}'s Dashboard</h1>
      </header>
      <main>{children}</main>
    </section>
  )
}
```

### قراءة `params` في مكونات العميل (Client Components)

لاستخدام `params` في مكون العميل (Client Component) (الذي لا يمكن أن يكون `async`)، يمكنك استخدام دالة [`use`](https://react.dev/reference/react/use) من React لقراءة الـ promise:

```tsx filename="app/page.tsx" switcher
'use client'

import { use } from 'react'

export default function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = use(params)
}
```

```js filename="app/page.js" switcher
'use client'

import { use } from 'react'

export default function Page({ params }) {
  const { slug } = use(params)
}
```
