يغطي هذا القسم من الوثائق **اتفاقيات ملفات البيانات الوصفية**. يمكن تعريف البيانات الوصفية المعتمدة على الملفات عن طريق إضافة ملفات بيانات وصفية خاصة إلى مقاطع المسار (route segments).

يمكن تعريف كل اتفاقية ملف إما باستخدام ملف ثابت (مثل `opengraph-image.jpg`)، أو متغير ديناميكي يستخدم الكود لإنشاء الملف (مثل `opengraph-image.js`).

بمجرد تعريف الملف، سيقوم Next.js تلقائيًا بخدمة الملف (مع تجزئة في بيئة الإنتاج للتخزين المؤقت) وتحديث عناصر الرأس ذات الصلة بالبيانات الوصفية الصحيحة، مثل عنوان URL للأصل، ونوع الملف، وحجم الصورة.

> **معلومة مفيدة**:
>
> - معالجات المسار الخاصة مثل [`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap)، و[`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)، و[`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons)، و[ملفات البيانات الوصفية الأخرى](/docs/app/api-reference/file-conventions/metadata) يتم تخزينها مؤقتًا افتراضيًا.
> - إذا كنت تستخدمها مع [`middleware.ts`](/docs/app/api-reference/file-conventions/middleware)، [قم بتكوين المطابق (matcher)](/docs/app/building-your-application/routing/middleware#matcher) لاستثناء ملفات البيانات الوصفية.
