يسمح لك ملف `instrumentation-client.js|ts` بإضافة أكواد مراقبة وتحليلات تعمل قبل بدء تنفيذ كود الواجهة الأمامية لتطبيقك. هذا مفيد لإعداد تتبع الأداء، مراقبة الأخطاء، أو أي أدوات أخرى لمراقبة جانب العميل.

لاستخدامه، ضع الملف في **الجذر** لتطبيقك أو داخل مجلد `src`.

## الاستخدام

على عكس [أدوات التتبع من جانب الخادم](/docs/app/guides/instrumentation)، لا تحتاج إلى تصدير أي دوال محددة. يمكنك كتابة كود المراقبة مباشرة في الملف:

```ts filename="instrumentation-client.ts" switcher
// إعداد مراقبة الأداء
performance.mark('app-init')

// تهيئة التحليلات
console.log('تم تهيئة التحليلات')

// إعداد تتبع الأخطاء
window.addEventListener('error', (event) => {
  // إرسال إلى خدمة تتبع الأخطاء الخاصة بك
  reportError(event.error)
})
```
