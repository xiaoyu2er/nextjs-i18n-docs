يستخدم Next.js معرّفًا ثابتًا يتم إنشاؤه أثناء عملية البناء لتحديد أي نسخة من التطبيق يتم تقديمها. قد يسبب هذا مشاكل في عمليات النشر متعددة الخوادم عند تشغيل `next build` على كل خادم. للحفاظ على معرّف بناء متسق بين عمليات البناء المختلفة، يمكنك توفير معرّف البناء الخاص بك.

افتح ملف `next.config.js` وأضف دالة `generateBuildId`:

```js filename="next.config.js"
module.exports = {
  generateBuildId: async () => {
    // يمكنك هنا على سبيل المثال الحصول على آخر تجزئة (hash) لالتزام git
    return 'my-build-id'
  },
}
```
