يفشل Next.js في عملية **بناء الإنتاج** (`next build`) عندما تكون هناك أخطاء TypeScript في مشروعك.

إذا كنت ترغب في أن يقوم Next.js بإنشاء كود إنتاج بشكل خطير حتى عندما يكون هناك أخطاء في تطبيقك، يمكنك تعطيل خطوة التحقق من الأنواع المدمجة.

إذا تم تعطيل هذه الميزة، تأكد من أنك تقوم بإجراء فحوصات الأنواع كجزء من عملية البناء أو النشر، وإلا قد يكون هذا خطيرًا جدًا.

افتح ملف `next.config.js` وقم بتمكين خيار `ignoreBuildErrors` في إعدادات `typescript`:

```js filename="next.config.js"
module.exports = {
  typescript: {
    // !! تحذير !!
    // السماح بشكل خطير بإكمال عمليات بناء الإنتاج بنجاح حتى إذا
    // كان مشروعك يحتوي على أخطاء في الأنواع.
    // !! تحذير !!
    ignoreBuildErrors: true,
  },
}
```
