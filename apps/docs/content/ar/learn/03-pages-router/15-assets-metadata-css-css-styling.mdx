لنتحدث الآن عن **تنسيق CSS**.

كما ترى، صفحتنا الرئيسية ([http://localhost:3000](http://localhost:3000)) تحتوي بالفعل على بعض التنسيقات. إذا نظرت إلى بنية الملفات، ستجد مجلدًا باسم `styles` يحتوي على ملفين CSS: ورقة أنماط عامة (`global.css`)، ووحدة CSS (`Home.module.css`).

إذا لم يكن مشروعك يحتوي على هذه الملفات، يمكنك تنزيل الكود المبدئي من هنا:

```
npx create-next-app nextjs-blog --use-npm --example "https://github.com/vercel/next-learn/tree/main/basics/assets-metadata-css-starter"
```

### [وحدات CSS](#css-modules)

[وحدات CSS](/docs/basic-features/built-in-css-support) تتيح لك تحديد نطاق CSS محليًا على مستوى المكون عن طريق إنشاء أسماء فريدة للفئات تلقائيًا. هذا يسمح لك باستخدام نفس اسم فئة CSS في ملفات مختلفة دون القلق عن تعارض أسماء الفئات.

بالإضافة إلى وحدات CSS، يمكنك تنسيق تطبيق Next.js الخاص بك بعدة طرق، بما في ذلك:

*   Sass الذي يسمح لك باستيراد ملفات `.css` و `.scss`.
*   مكتبات PostCSS مثل [Tailwind CSS](https://github.com/vercel/next.js/tree/canary/examples/with-tailwindcss).
*   مكتبات CSS-in-JS مثل [styled-jsx](https://github.com/vercel/styled-jsx)، [styled-components](https://github.com/vercel/next.js/tree/canary/examples/with-styled-components)، و [emotion](https://github.com/vercel/next.js/tree/canary/examples/with-emotion)

في هذا الدرس، سنتحدث عن كيفية استخدام [وحدات CSS](/docs/basic-features/built-in-css-support#adding-component-level-css) و [Sass](/docs/basic-features/built-in-css-support#sass-support) في Next.js. دعونا نبدأ!
