قبل أن نقوم بالنشر، لنقم بدفع تطبيق Next.js الخاص بنا إلى [GitHub](https://github.com/vercel/next.js) إذا لم تكن قد فعلت ذلك بالفعل. هذا سيجعل عملية النشر أسهل.

* على حساب GitHub الشخصي الخاص بك، قم بإنشاء مستودع جديد باسم `nextjs-blog`.
* يمكن أن يكون المستودع عامًا أو خاصًا. **لا** تحتاج إلى تهيئته بملف README أو ملفات أخرى.
* إذا كنت بحاجة إلى مساعدة في إعداد المستودع الخاص بك، يمكنك الاطلاع على [هذا الدليل على GitHub](https://help.github.com/en/github/getting-started-with-github/create-a-repo).

ثم:

* إذا لم تكن قد قمت بتهيئة مستودع git محليًا لتطبيق Next.js الخاص بك، قم بذلك الآن.
* ادفع تطبيق Next.js إلى مستودع GitHub الخاص بك.

لدفع التغييرات إلى GitHub، يمكنك تنفيذ الأوامر التالية (استبدل `<username>` باسم مستخدم GitHub الخاص بك):

```
git remote add origin https://github.com/<username>/nextjs-blog.git
git push -u origin main
```

بمجرد أن يصبح مستودع GitHub الخاص بك جاهزًا، انتقل إلى الصفحة التالية.
