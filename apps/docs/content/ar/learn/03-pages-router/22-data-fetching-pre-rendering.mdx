قبل أن نتحدث عن [جلب البيانات](/docs/basic-features/data-fetching)، دعونا نناقش أحد أهم المفاهيم في Next.js: [**التصيير المسبق**](/docs/basic-features/pages#pre-rendering).

بشكل افتراضي، يقوم Next.js بتصيير كل صفحة مسبقًا. هذا يعني أن Next.js _يولد HTML لكل صفحة مقدمًا_، بدلاً من القيام بكل ذلك عبر JavaScript من جانب العميل. يمكن أن يؤدي التصيير المسبق إلى تحسين الأداء و[تحسين محركات البحث (SEO)](https://en.wikipedia.org/wiki/Search_engine_optimization).

يرتبط كل HTML تم إنشاؤه بأقل كود JavaScript ضروري لتلك الصفحة. عندما يتم تحميل الصفحة بواسطة المتصفح، يتم تشغيل كود JavaScript الخاص بها مما يجعل الصفحة تفاعلية بالكامل. (تسمى هذه العملية **الترطيب (hydration)**).

### [التحقق من حدوث التصيير المسبق](#check-that-pre-rendering-is-happening)

يمكنك التحقق من حدوث التصيير المسبق عن طريق اتباع الخطوات التالية:

1.  قم بتعطيل JavaScript في متصفحك. ([إليك الطريقة في Chrome](https://developer.chrome.com/docs/devtools/javascript/disable/)).
2.  [حاول الوصول إلى هذه الصفحة](https://next-learn-starter.vercel.app/) (النتيجة النهائية لهذا البرنامج التعليمي).

يجب أن ترى أن تطبيقك يتم عرضه بدون JavaScript. هذا لأن Next.js قام بتصيير التطبيق مسبقًا إلى HTML ثابت، مما يسمح لك برؤية واجهة المستخدم للتطبيق دون تشغيل JavaScript.

> **ملاحظة**: يمكنك أيضًا تجربة الخطوات المذكورة أعلاه على `localhost`، ولكن لن يتم تحميل CSS إذا قمت بتعطيل JavaScript.

إذا كان تطبيقك عبارة عن تطبيق React.js عادي (بدون Next.js)، فلا يوجد [تصيير مسبق](/docs/basic-features/pages#pre-rendering)، لذلك لن تتمكن من رؤية التطبيق إذا قمت بتعطيل JavaScript. على سبيل المثال:

*   قم بتمكين JavaScript في متصفحك و[تحقق من هذه الصفحة](https://create-react-template.vercel.app/). هذا تطبيق React.js عادي تم إنشاؤه باستخدام [Create React App](https://create-react-app.dev/).
*   الآن، قم بتعطيل JavaScript وقم بالوصول إلى [نفس الصفحة](https://create-react-template.vercel.app/) مرة أخرى.
*   لن ترى التطبيق بعد الآن - بدلاً من ذلك، ستظهر رسالة "You need to enable JavaScript to run this app". هذا لأن التطبيق لم يتم تصييره مسبقًا إلى HTML ثابت.
