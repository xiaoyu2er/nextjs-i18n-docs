[建置效能](#build-performance)
---------------------------------------

Next.js 建置現在更具確定性，這意味著如果程式碼未更改，每次建置輸出都會保持相同結果。

這使得建置過程中的更多工作可以被快取，從而實現生產程式碼在首次建置後更快的重新建置速度。

我們的測量顯示，大量建置時間花在 JavaScript 最小化上。由於建置輸出現在更具確定性，最小化檔案已存在快取中的機率更大。

[部署到 Vercel](https://vercel.com/guides/deploying-nextjs-with-vercel) 的用戶將自動體驗到這些快取改進。

我們簡化了伺服器端的 `next/dynamic` 清單，僅包含非同步載入的模組。這個簡化後的清單計算速度更快，產生的 JavaScript 也更少。

CircleCI 用戶應該會注意到更快的建置時間。之前由於 CircleCI 環境未能準確反映可用 CPU 數量，導致工作程序過度排程。Next.js 現在會檢測 CircleCI 並根據可用資源設置適當的 CPU 數量。

[更小的運行時佔用空間](#smaller-runtime-footprint)
-------------------------------------------------------

Next.js 現在生成的客戶端 JavaScript **減少了 5.58KB**，HTTP 負載也比前一版本更小。

[`withRouter`](/docs#using-a-higher-order-component) 不再依賴 `hoist-non-react-statics`，減少了 **3KB** 的打包體積。`withRouter` 仍會提升 `getInitialProps`，但不會提升其他靜態屬性。

`next/babel` 預設值已優化，可產生更精簡、更快的 JavaScript。

移除了 `X-Powered-By` 標頭，減小了 HTTP 負載大小。我們調查了社群，發現該標頭在生產環境中經常被禁用，因此決定移除它。這也意味著如果您的專案中啟用了該選項，可以從 `next.config.js` 中移除 `poweredByHeader`。

我們對 Next.js 的依賴關係樹和整體程式碼庫進行了許多優化，使每個無伺服器函式減小了 **44KB (5.44KB gzip)**。

無伺服器函式的大小直接影響無伺服器啟動效能，更小的函式意味著更快的啟動速度。

在 Next.js 8 發布後，我們收到少數用戶報告在 Next.js 之外導入 React 元件時遇到問題，例如在測試套件中。這是因為導入 `next` 時會被重寫為 Next.js 程式碼庫中的正確檔案，但此優化應用於所有使用 `next/babel` 預設值的用戶。該優化現已移至 Next.js 建置流程本身，因此不會再與用戶的 babel 設置衝突。

[預設 Viewport Meta 標籤](#default-viewport-meta-tag)
-------------------------------------------------------

Next.js 的目標之一是為開發網頁應用提供最佳預設值。為了減少在 Next.js 中實現 CSS 媒體查詢時所需的設置工作。

預設情況下，瀏覽器處理 CSS `@media` 查詢和縮放的方式與預期不同，這可能導致編寫 CSS `@media` 查詢時出現意外的不一致。

在幾乎所有情況下，Next.js 用戶都會在應用程式中添加 `viewport` meta 標籤來解決這些不一致。

從 8.0.4 版本開始，在大多數情況下不再需要設置此 `viewport` 標籤。如果應用程式未設置 `viewport`，將應用預設 viewport：

```
<meta
  name="viewport"
  content="width=device-width,minimum-scale=1,initial-scale=1"
/>
```

仍可通過 `next/head` 覆蓋 viewport 標籤：

```js filename="pages/index.js"
import Head from 'next/head';
 
function HomePage() {
  return (
    <>
      <Head>
        <meta
          name="viewport"
          content="width=device-width,minimum-scale=0.5,initial-scale=1"
        />
      </Head>
    </>
  );
}
 
export default HomePage;
```

感謝 [Jason Miller](https://twitter.com/_developit) 合作並實現了此變更。

[新的 `@next/mdx` 插件](#new-nextmdx-plugin)
---------------------------------------------

[MDX](https://mdxjs.com/) 允許您在 Markdown 文件中編寫 JSX。您可以使用常規的 markdown 語法編寫內容，並導入 React 元件來增強文件的互動性和動態內容。

為 Next.js 提供 MDX 支援的插件 `@zeit/next-mdx` 已移至 [Next.js GitHub 存儲庫](https://github.com/vercel/next.js/tree/canary/packages/next-mdx)，現在以 [`@next/mdx`](https://www.npmjs.com/package/@next/mdx) 提供。

安裝：

```bash filename="終端機"
npm i @next/mdx @mdx-js/loader
```

要使其在 Next.js 應用中可用，創建一個 `next.config.js` 文件並包含：

```
const withMDX = require('@next/mdx')();
module.exports = withMDX();
```

隨著時間推移，更多 Next.js 插件將被移至 Next.js 存儲庫，以便與 Next.js 核心一起發布並由 Next.js 測試套件進行測試。這樣我們可以確保熱模組替換、生產建置和其他功能與插件良好配合。
