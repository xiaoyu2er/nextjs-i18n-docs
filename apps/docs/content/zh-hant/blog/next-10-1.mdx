我們很高興推出 Next.js 10.1，主要功能包括：

*   **[3 倍更快的熱更新](#3x-faster-refresh)**：無需任何變更即可實現 200 毫秒更快的熱更新。
*   **[安裝時間改進](#improved-installation-time)**：安裝體積減少 58%，依賴項減少 56%。
*   **[`next/image` 改進](#nextimage-improvements)**：支援 Apple Silicon (M1) 處理器，並增加更多佈局和載入器選項。
*   **[Next.js Commerce Shopify 整合](#nextjs-commerce-shopify-integration)**：為可組合式電商應用提供靈活的數據層。
*   **[自訂 500 頁面](#custom-500-page)**：在錯誤頁面添加您的品牌標誌。
*   **[嚴格的 PostCSS 配置載入](#strict-postcss-configuration-loading)**：透過 Webpack 5 改進快取機制。
*   **[支援 `tsconfig.json` 中的 `extends`](#support-for-extends-in-tsconfigjson)**：為大型 TypeScript 應用提供可擴展配置。
*   **[檢測預覽模式啟用狀態](#detect-when-preview-mode-is-enabled)**：在預覽時條件式顯示內容。
*   **[路由方法自動滾動至頂部](#router-methods-scroll-to-top)**：現在所有路由導航都會自動滾動至頂部。
*   **[文件改進](#documentation-improvements)**：增量採用指南、遷移教學和 Docker 部署說明。

[3 倍更快的熱更新](#3x-faster-refresh)
---------------------------------------

自 [v9.4](https://nextjs.org/blog/next-9-4) 版本起，Next.js 將 Facebook 的 [快速熱更新 (Fast Refresh)](/docs/architecture/fast-refresh) 技術開源，讓您編輯 React 元件時能立即獲得反饋。

這意味著 Next.js 只會更新您編輯的文件代碼，並僅重新渲染該元件，同時保持元件狀態不變。這包括樣式（行內、CSS-in-JS 或 CSS/Sass Modules）、標記、事件處理程序和效果（透過 `useEffect`）。

現在，我們讓每次編輯的熱更新速度 **再提升 200 毫秒**，**無需您進行任何代碼變更**。平均而言，這使快速熱更新的速度提升了 3 倍。

想要更快的性能和更好的建構時間嗎？現在就可以透過新的 `future` 標誌啟用 Webpack 5。此標誌目前是候選版本 (RC)，Webpack 5 將很快成為預設選項。啟用後，請[與我們分享您的反饋](https://github.com/vercel/next.js/discussions/23498)。

```js filename="next.config.js"
module.exports = {
  future: {
    webpack5: true,
  },
};
```

要了解 Webpack 5 如何以向後兼容的方式應用於您的應用程式，請參閱[相關文件](/docs/messages/webpack5)。

[`next/image` 改進](#nextimage-improvements)
------------------------------------------

### [Apple Silicon 支援](#apple-silicon-support)

當 [Next.js 10](https://nextjs.org/blog/next-10) 發布時，圖片優化功能依賴於原生套件。這導致安裝時間較長，原生套件佔了 Next.js 總安裝體積的 50%。此外，這還移除了安裝後執行腳本和按需編譯的需求。

我們很高興地宣布，透過 `next/image` 的自動圖片優化現在由 [WebAssembly](https://web.dev/squoosh-v2/) 提供支援。這項優化開箱即用，適用於 `next dev` 和 `next start`。透過使用 WASM，我們減少了約 30MB 的安裝體積，並**新增了對 [Apple Silicon](https://www.apple.com/mac/m1/) M1 MacBook 的支援**。

我們正在透過利用 [SIMD](https://v8.dev/features/simd) 擴展和多線程繼續改進 WebAssembly 性能。

#### [新增佈局選項](#additional-layouts)

根據您的反饋，我們為 `next/image` 新增了多種佈局和選項：

*   `layout=fill`：無需提供 `width` 和 `height`。[示範](https://image-component.nextjs.gallery/layout-fill)
*   `layout=fixed`：原生 `img` 行為。[示範](https://image-component.nextjs.gallery/layout-fixed)
*   `layout=responsive`：彈性寬度與固定長寬比。[示範](https://image-component.nextjs.gallery/layout-responsive)
*   `layout=intrinsic`：不會放大，但可以縮小並保持固定長寬比。[示範](https://image-component.nextjs.gallery/layout-intrinsic)
*   `objectFit=cover`：背景圖片。[示範](https://image-component.nextjs.gallery/background)

更多資訊，請參閱 [`layout` 文件](/docs/pages/api-reference/components/image#layout)。

#### [改進的圖片優化支援](#improved-image-optimization-support)

我們擴展了 `next/image` 的[內建圖片載入器](/docs/pages/building-your-application/optimizing/images#loader)，現在您可以新增任何自訂載入器。透過新的 `loader` 屬性，您可以使用**任何供應商或 CDN 來優化圖片**。

```js filename="components/my-image.js"
import Image from 'next/image';
 
const myLoader = ({ src, width, quality }) => {
  return `https://example.com/${src}?w=${width}&q=${quality || 75}`;
};
 
const MyImage = (props) => {
  return (
    <Image
      loader={myLoader}
      src="/me.png"
      alt="作者照片"
      width={500}
      height={500}
    />
  );
};
```

更多資訊，請參閱 [`next/image` 文件](/docs/pages/api-reference/components/image#loader)。

[Next.js Commerce Shopify 整合](#nextjs-commerce-shopify-integration)
--------------------------------------------------------------------

電子商務是採用 Next.js 增長最快的行業之一。Apple、Wal-Mart、McDonald's 和 Nike 等公司都信任 Next.js。

我們希望為您提供一個高性能的起點來建構電子商務應用程式，因此我們發布了 [Next.js Commerce](https://nextjs.org/commerce)。在收到數千個 Shopify 請求後，我們更新了 Next.js Commerce，提供**供應商無關的 UI**——讓您可以選擇任何無頭電子商務解決方案。

切換供應商只需在配置中進行[單行變更](https://github.com/vercel/commerce#configuration)。我們還建立了功能 API，讓您可以根據使用案例切換功能——同時保持一流的性能。

[自訂 500 頁面](#custom-500-page)
---------------------------------

您現在可以建立自訂的 `pages/500.js`，在顯示錯誤時添加自己的標誌和品牌。與 `pages/404.js` 類似，此文件在建構時會靜態生成。

```js filename="pages/500.js"
export default function Custom500() {
  return <h1>500 - 伺服器端發生錯誤</h1>;
}
```

[嚴格的 PostCSS 配置載入](#strict-postcss-configuration-loading)
---------------------------------------------------------------

如果您使用 PostCSS 和 Webpack 5，可以透過新的 `future` 標誌來提高性能（透過快取機制）。

```js filename="next.config.js"
module.exports = {
  future: {
    webpack5: true,
    strictPostcssConfiguration: true,
  },
};
```

立即試用並告訴我們您的反饋。更多資訊，請參閱 [PostCSS 配置文件](/docs/pages/building-your-application/configuring/post-css)。

[支援 `tsconfig.json` 中的 `extends`](#support-for-extends-in-tsconfigjson)
-------------------------------------------------------------------------

您現在可以在 Next.js 使用的 `tsconfig.json` 中使用 `extends` 來擴展專案中的其他 `tsconfig.json`。例如，您可以從專案中的 `tsconfig.base.json` 擴展如下：

```json filename="tsconfig.json"
{
  "extends": "./tsconfig.base.json"
}
```

更多資訊，請參閱 [TypeScript 文件](/docs/pages/building-your-application/configuring/typescript)。

[檢測預覽模式啟用狀態](#detect-when-preview-mode-is-enabled)
-----------------------------------------------------------

您現在可以檢測預覽模式是否啟用，從而僅為編輯者渲染特定元件。

```js filename="components/MyComponent.js"
import { useRouter } from 'next/router';
 
function MyComponent() {
  const { isPreview } = useRouter();
  return <>{isPreview ? <h1>預覽模式已啟用</h1> : null}</>;
}
```

更多資訊，請參閱 [預覽模式文件](/docs/pages/building-your-application/configuring/preview-mode)。

[路由方法自動滾動至頂部](#router-methods-scroll-to-top)
-----------------------------------------------------

使用 `next/link` 導航時，Next.js 會自動滾動至頂部。在 Next.js 10.1 中，`router.push` 和 `router.replace` 也會自動滾動至頂部，以與 `next/link` 行為保持一致。您可以使用 `scroll` 選項選擇退出自動滾動。

更多資訊，請參閱 [`next/router` 文件](/docs/pages/api-reference/functions/use-router)。
