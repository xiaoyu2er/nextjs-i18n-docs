title: 路由群組
description: 路由群組 (Route Groups) 可用於將您的 Next.js 應用程式劃分為不同區塊。
---

在 `app` 目錄中，巢狀資料夾通常會對應到 URL 路徑。然而，您可以將資料夾標記為 **路由群組 (Route Group)**，以防止該資料夾被包含在路由的 URL 路徑中。

這讓您可以將路由區段和專案檔案組織成邏輯群組，而不影響 URL 路徑結構。

路由群組在以下情況特別有用：

- 將路由按網站區塊、用途或團隊分組
- 在同一路由區段層級啟用巢狀佈局：
  - 在同一區段建立多個巢狀佈局，包括多個根佈局
  - 將特定區段加入某個佈局
- 為特定路由選擇載入骨架畫面 (loading skeletons)

## 慣例

建立路由群組的方法是將資料夾名稱用括號包起來：`(folderName)`

> **須知事項**：
>
> - 路由群組的命名僅用於組織目的，沒有特殊意義，不會影響 URL 路徑。
> - 包含路由群組的路由 **不應** 解析為與其他路由相同的 URL 路徑。例如，由於路由群組不影響 URL 結構，`(marketing)/about/page.js` 和 `(shop)/about/page.js` 都會解析到 `/about` 並導致錯誤。
> - 如果您在沒有頂層 `layout.js` 檔案的情況下使用多個根佈局，您的首頁 `page.js` 檔案應定義在其中一個路由群組中，例如：`app/(marketing)/page.js`。
> - 在 **多個根佈局之間** 導航會導致 **完整頁面重新載入**（與客戶端導航相反）。例如，從使用 `app/(shop)/layout.js` 的 `/cart` 導航到使用 `app/(marketing)/layout.js` 的 `/blog` 會觸發完整頁面載入。此情況 **僅** 適用於多個根佈局。
