`create-next-app` CLI 工具讓您能使用預設範本或從公開 GitHub 儲存庫的 [範例](https://github.com/vercel/next.js/tree/canary/examples) 建立新的 Next.js 應用程式。這是開始使用 Next.js 最簡單的方式。

基本用法：

```bash filename="Terminal"
npx create-next-app@latest [專案名稱] [選項]
```

## 參考選項

以下為可用選項：

| 選項                                  | 說明                                                                 |
| ------------------------------------- | ------------------------------------------------------------------- |
| `-h` 或 `--help`                     | 顯示所有可用選項                                                   |
| `-v` 或 `--version`                  | 輸出版本號                                                         |
| `--no-*`                             | 取消預設選項。例如 `--no-eslint`                                   |
| `--ts` 或 `--typescript`             | 初始化為 TypeScript 專案 (預設值)                                  |
| `--js` 或 `--javascript`             | 初始化為 JavaScript 專案                                           |
| `--tailwind`                         | 初始化包含 Tailwind CSS 設定 (預設值)                              |
| `--eslint`                           | 初始化包含 ESLint 設定                                             |
| `--app`                              | 初始化為 App Router 專案                                           |
| `--api`                              | 初始化僅包含路由處理器的專案                                       |
| `--src-dir`                          | 在 `src/` 目錄內初始化                                             |
| `--turbopack`                        | 預設啟用 Turbopack 進行開發                                        |
| `--import-alias <要設定的別名>`      | 指定使用的導入別名 (預設為 "@/*")                                  |
| `--empty`                            | 初始化空白專案                                                     |
| `--use-npm`                          | 明確告知 CLI 使用 npm 初始化應用程式                               |
| `--use-pnpm`                         | 明確告知 CLI 使用 pnpm 初始化應用程式                              |
| `--use-yarn`                         | 明確告知 CLI 使用 Yarn 初始化應用程式                              |
| `--use-bun`                          | 明確告知 CLI 使用 Bun 初始化應用程式                               |
| `-e` 或 `--example [名稱] [github-url]` | 使用範例初始化應用程式                                           |
| `--example-path <範例路徑>`          | 單獨指定範例的路徑                                                 |
| `--reset-preferences`                | 明確告知 CLI 重置所有儲存的偏好設定                                |
| `--skip-install`                     | 明確告知 CLI 跳過安裝套件                                          |
| `--yes`                              | 使用先前的偏好設定或所有選項的預設值                               |

## 範例

### 使用預設範本

要使用預設範本建立新應用程式，請在終端機執行以下指令：

```bash filename="Terminal"
npx create-next-app@latest
```

接著會出現以下提示問題：

```txt filename="Terminal"
您的專案名稱是？ my-app
要使用 TypeScript 嗎？ 否 / 是
要使用 ESLint 嗎？ 否 / 是
要使用 Tailwind CSS 嗎？ 否 / 是
要將程式碼放在 `src/` 目錄內嗎？ 否 / 是
要使用 App Router 嗎？(推薦) 否 / 是
要在 `next dev` 中使用 Turbopack 嗎？ 否 / 是
要自訂導入別名嗎？(預設為 `@/*`) 否 / 是
```

回答完問題後，系統將根據您的選擇建立新專案。

### 使用官方 Next.js 範例

要使用官方 Next.js 範例建立新應用程式，請使用 `--example` 參數。例如：

```bash filename="Terminal"
npx create-next-app@latest --example [範例名稱] [您的專案名稱]
```

您可以在 [Next.js 儲存庫](https://github.com/vercel/next.js/tree/canary/examples) 查看所有可用範例列表及設定說明。

### 使用任何公開 GitHub 範例

要使用任何公開 GitHub 範例建立新應用程式，請搭配 GitHub 儲存庫 URL 使用 `--example` 選項。例如：

```bash filename="Terminal"
npx create-next-app@latest --example "https://github.com/.../" [您的專案名稱]
```
