`favicon`、`icon` 或 `apple-icon` 檔案慣例允許您為應用程式設定圖示。

這些圖示可用於添加出現在網頁瀏覽器分頁、手機主畫面及搜尋引擎結果等處的應用程式圖示。

設定應用程式圖示有兩種方式：

- [使用圖片檔案 (.ico, .jpg, .png)](#使用圖片檔案-ico-jpg-png)
- [使用程式碼生成圖示 (.js, .ts, .tsx)](#使用程式碼生成圖示-js-ts-tsx)

## 使用圖片檔案 (.ico, .jpg, .png)

透過在 `/app` 目錄中放置 `favicon`、`icon` 或 `apple-icon` 圖片檔案來設定應用程式圖示。  
`favicon` 圖片只能位於 `app/` 的頂層目錄中。

Next.js 會評估檔案並自動將適當的標籤添加到應用程式的 `<head>` 元素中。

| 檔案慣例             | 支援的檔案類型                    | 有效位置       |
| -------------------- | --------------------------------- | -------------- |
| [`favicon`](#favicon) | `.ico`                           | `app/`         |
| [`icon`](#icon)       | `.ico`, `.jpg`, `.jpeg`, `.png`, `.svg` | `app/**/*`     |
| [`apple-icon`](#apple-icon) | `.jpg`, `.jpeg`, `.png`          | `app/**/*`     |

### `favicon`

在根目錄 `/app` 路由區段中添加 `favicon.ico` 圖片檔案。

```html filename="<head> 輸出"
<link rel="icon" href="/favicon.ico" sizes="any" />
```

### `icon`

添加 `icon.(ico|jpg|jpeg|png|svg)` 圖片檔案。

```html filename="<head> 輸出"
<link
  rel="icon"
  href="/icon?<generated>"
  type="image/<generated>"
  sizes="<generated>"
/>
```

### `apple-icon`

添加 `apple-icon.(jpg|jpeg|png)` 圖片檔案。

```html filename="<head> 輸出"
<link
  rel="apple-touch-icon"
  href="/apple-icon?<generated>"
  type="image/<generated>"
  sizes="<generated>"
/>
```

> **須知事項**:
> 
> - 您可以透過在檔案名稱中添加數字後綴來設定多個圖示。例如 `icon1.png`、`icon2.png` 等。編號檔案會按字母順序排序。
> - Favicon 只能在根目錄 `/app` 區段中設定。如果需要更細緻的控制，可以使用 [`icon`](#icon)。
> - 適當的 `<link>` 標籤和屬性（如 `rel`、`href`、`type` 和 `sizes`）由圖示類型和評估檔案的元數據決定。
> - 例如，32x32 像素的 `.png` 檔案會有 `type="image/png"` 和 `sizes="32x32"` 屬性。
> - 當圖示的副檔名為 `.svg` 或無法確定檔案圖片大小時，會添加 `sizes="any"`。更多細節請參閱 [favicon 手冊](https://evilmartians.com/chronicles/how-to-favicon-in-2021-six-files-that-fit-most-needs)。

## 使用程式碼生成圖示 (.js, .ts, .tsx)

除了使用[圖片檔案](#使用圖片檔案-ico-jpg-png)外，您還可以使用程式碼**生成**圖示。

透過建立一個預設導出函數的 `icon` 或 `apple-icon` 路由來生成應用程式圖示。

| 檔案慣例 | 支援的檔案類型 |
| -------- | -------------- |
| `icon`   | `.js`, `.ts`, `.tsx` |
| `apple-icon` | `.js`, `.ts`, `.tsx` |

生成圖示最簡單的方法是使用 `next/og` 中的 [`ImageResponse`](/docs/app/api-reference/functions/image-response) API。

```tsx filename="app/icon.tsx" switcher
import { ImageResponse } from 'next/og'

// 圖片元數據
export const size = {
  width: 32,
  height: 32,
}
export const contentType = 'image/png'

// 圖片生成
export default function Icon() {
  return new ImageResponse(
    (
      // ImageResponse JSX 元素
      <div
        style={{
          fontSize: 24,
          background: 'black',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
          color: 'white',
        }}
      >
        A
      </div>
    ),
    // ImageResponse 選項
    {
      // 為方便起見，我們可以重複使用導出的圖示大小元數據
      // 配置來設定 ImageResponse 的寬度和高度。
      ...size,
    }
  )
}
```

```jsx filename="app/icon.js" switcher
import { ImageResponse } from 'next/og'

// 圖片元數據
export const size = {
  width: 32,
  height: 32,
}
export const contentType = 'image/png'

// 圖片生成
export default function Icon() {
  return new ImageResponse(
    (
      // ImageResponse JSX 元素
      <div
        style={{
          fontSize: 24,
          background: 'black',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
          color: 'white',
        }}
      >
        A
      </div>
    ),
    // ImageResponse 選項
    {
      // 為方便起見，我們可以重複使用導出的圖示大小元數據
      // 配置來設定 ImageResponse 的寬度和高度。
      ...size,
    }
  )
}
```

```html filename="<head> 輸出"
<link rel="icon" href="/icon?<generated>" type="image/png" sizes="32x32" />
```

> **須知事項**:
> 
> - 預設情況下，生成的圖示是[**靜態優化**](/docs/app/getting-started/partial-prerendering#static-rendering)的（在構建時生成並緩存），除非它們使用[動態 API](/docs/app/getting-started/partial-prerendering#dynamic-rendering#dynamic-apis) 或未緩存的數據。
> - 您可以使用 [`generateImageMetadata`](/docs/app/api-reference/functions/generate-image-metadata) 在同一個檔案中生成多個圖示。
> - 您無法生成 `favicon` 圖示。請改用 [`icon`](#icon) 或 [favicon.ico](#favicon) 檔案。
> - 應用程式圖示是特殊的路由處理程序，預設情況下會被緩存，除非它使用[動態 API](/docs/app/deep-dive/caching#dynamic-apis) 或[動態配置](/docs/app/deep-dive/caching#segment-config-options)選項。

### 屬性

預設導出函數接收以下屬性：

#### `params` (選用)

一個包含從根區段到 `icon` 或 `apple-icon` 所在區段的[動態路由參數](/docs/app/building-your-application/routing/dynamic-routes)物件。

```tsx filename="app/shop/[slug]/icon.tsx" switcher
export default function Icon({ params }: { params: { slug: string } }) {
  // ...
}
```

```jsx filename="app/shop/[slug]/icon.js" switcher
export default function Icon({ params }) {
  // ...
}
```

| 路由                           | URL         | `params`                  |
| ------------------------------ | ----------- | ------------------------- |
| `app/shop/icon.js`             | `/shop`     | `undefined`               |
| `app/shop/[slug]/icon.js`      | `/shop/1`   | `{ slug: '1' }`           |
| `app/shop/[tag]/[item]/icon.js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |

### 返回值

預設導出函數應返回 `Blob` | `ArrayBuffer` | `TypedArray` | `DataView` | `ReadableStream` | `Response`。

> **須知事項**：`ImageResponse` 符合此返回類型。

### 配置導出

您可以選擇性地透過從 `icon` 或 `apple-icon` 路由導出 `size` 和 `contentType` 變數來配置圖示的元數據。

| 選項                        | 類型                                                                                                            |
| --------------------------- | --------------------------------------------------------------------------------------------------------------- |
| [`size`](#size)             | `{ width: number; height: number }`                                                                             |
| [`contentType`](#contenttype) | `string` - [圖片 MIME 類型](https://developer.mozilla.org/docs/Web/HTTP/Basics_of_HTTP/MIME_types#image_types) |

#### `size`

```tsx filename="icon.tsx | apple-icon.tsx" switcher
export const size = { width: 32, height: 32 }

export default function Icon() {}
```

```jsx filename="icon.js | apple-icon.js" switcher
export const size = { width: 32, height: 32 }

export default function Icon() {}
```

```html filename="<head> 輸出"
<link rel="icon" sizes="32x32" />
```

#### `contentType`

```tsx filename="icon.tsx | apple-icon.tsx" switcher
export const contentType = 'image/png'

export default function Icon() {}
```

```jsx filename="icon.js | apple-icon.js" switcher
export const contentType = 'image/png'

export default function Icon() {}
```

```html filename="<head> 輸出"
<link rel="icon" type="image/png" />
```

#### 路由區段配置

`icon` 和 `apple-icon` 是專門的[路由處理程序](/docs/app/building-your-application/routing/route-handlers)，可以使用與頁面和佈局相同的[路由區段配置](/docs/app/api-reference/file-conventions/route-segment-config)選項。
