**template** 檔案與 [layout](/docs/app/building-your-application/routing/layouts-and-templates#layouts) 類似，都是用來包裹佈局或頁面。不同之處在於，layout 會跨路由保持狀態，而 template 會被賦予一個獨特的 key，這意味著其中的 Client Components 會在導航時重置其狀態。

```tsx filename="app/template.tsx" switcher
export default function Template({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>
}
```

```jsx filename="app/template.jsx" switcher
export default function Template({ children }) {
  return <div>{children}</div>
}
```

雖然較不常見，但在以下情況您可能會選擇使用 template 而非 layout：

- 需要使用依賴 `useEffect` 的功能（例如記錄頁面瀏覽）或 `useState`（例如每頁面的回饋表單）。
- 需要變更預設框架行為。例如，在 layout 中的 Suspense Boundaries 只會在首次載入 Layout 時顯示 fallback，而在切換頁面時不會顯示。對於 template，fallback 會在每次導航時顯示。

## Props

### `children` (必填)

Template 接受一個 `children` prop。例如：

```jsx filename="Output"
<Layout>
  {/* 請注意 template 會自動被賦予一個獨特的 key */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```

> **須知**：
>
> - 預設情況下，`template` 是一個 [Server Component](/docs/app/getting-started/server-and-client-components)，但也可以透過 `"use client"` 指令作為 [Client Component](/docs/app/getting-started/server-and-client-components) 使用。
> - 當使用者在共享同一個 `template` 的路由之間導航時，會掛載一個新的元件實例，DOM 元素會被重新建立，Client Components 中的狀態**不會**被保留，且 effects 會重新同步。
