當您的專案中存在 TypeScript 錯誤時，Next.js 會使您的**正式環境建置** (`next build`) 失敗。

如果您希望 Next.js 即使在應用程式存在錯誤時仍「危險地」產生正式環境程式碼，您可以停用內建的型別檢查步驟。

請注意，若停用此功能，請確保在建置或部署流程中執行型別檢查，否則可能導致嚴重問題。

開啟 `next.config.js` 並在 `typescript` 設定中啟用 `ignoreBuildErrors` 選項：

```js filename="next.config.js"
module.exports = {
  typescript: {
    // !! 警告 !!
    // 危險地允許正式環境建置即使專案存在型別錯誤仍能成功完成
    // !! 警告 !!
    ignoreBuildErrors: true,
  },
}
```
