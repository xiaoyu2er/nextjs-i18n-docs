`<Form>` 元件擴展了 HTML `<form>` 元素的功能，提供 <AppOnly>[**預載 (prefetching)**](/docs/app/building-your-application/routing/linking-and-navigating#2-prefetching) [載入 UI](/docs/app/building-your-application/routing/loading-ui-and-streaming)、</AppOnly> 提交時的 **客戶端導航 (client-side navigation)**，以及 **漸進增強 (progressive enhancement)**。

對於需要更新 URL 搜尋參數的表單特別有用，可減少實現上述功能所需的樣板程式碼。

基本用法：

<AppOnly>

```tsx filename="/app/ui/search.tsx" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      {/* 提交時，輸入值會被附加到 URL，例如 /search?query=abc */}
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

```jsx filename="/app/ui/search.js" switcher
import Form from 'next/form'

export default function Search() {
  return (
    <Form action="/search">
      {/* 提交時，輸入值會被附加到 URL，例如 /search?query=abc */}
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

</AppOnly>

## 參考

`<Form>` 元件的行為取決於 `action` prop 傳遞的是 `string` 還是 `function`。

<AppOnly>

- 當 `action` 為 **字串** 時，`<Form>` 的行為類似於使用 **`GET`** 方法的原生 HTML 表單。表單資料會被編碼為 URL 的搜尋參數，提交表單時會導航到指定的 URL。此外，Next.js 還會：
  - 在表單可見時[預載 (prefetch)](/docs/app/building-your-application/routing/linking-and-navigating#2-prefetching) 路徑，預先載入共享 UI（例如 `layout.js` 和 `loading.js`），從而實現更快的導航。
  - 提交表單時執行[客戶端導航 (client-side navigation)](/docs/app/building-your-application/routing/linking-and-navigating#5-soft-navigation)，而非整頁重新載入，保留共享 UI 和客戶端狀態。
- 當 `action` 為 **函式**（伺服器動作 (Server Action)）時，`<Form>` 的行為類似於 [React 表單](https://react.dev/reference/react-dom/components/form)，在提交表單時執行該動作。

</AppOnly>

<PagesOnly>

- 當 `action` 為 **字串** 時，`<Form>` 的行為類似於使用 **`GET`** 方法的原生 HTML 表單。表單資料會被編碼為 URL 的搜尋參數，提交表單時會導航到指定的 URL。此外，Next.js 還會：
  - 提交表單時執行[客戶端導航 (client-side navigation)](/docs/app/building-your-application/routing/linking-and-navigating#5-soft-navigation)，而非整頁重新載入，保留共享 UI 和客戶端狀態。

</PagesOnly>

### `action` (字串) Props

<PagesOnly>

當 `action` 為字串時，`<Form>` 元件支援以下 props：

| Prop      | 範例               | 類型                             | 必填     |
| --------- | ------------------ | -------------------------------- | -------- |
| `action`  | `action="/search"` | `string` (URL 或相對路徑)        | 是       |
| `replace` | `replace={false}`  | `boolean`                        | -        |
| `scroll`  | `scroll={true}`    | `boolean`                        | -        |

- **`action`**：提交表單時要導航到的 URL 或路徑。
  - 空字串 `""` 會導航到相同路由並更新搜尋參數。
- **`replace`**：替換當前的歷史狀態，而非在[瀏覽器歷史記錄](https://developer.mozilla.org/en-US/docs/Web/API/History_API)堆疊中新增一筆。預設為 `false`。
- **`scroll`**：控制導航時的滾動行為。預設為 `true`，表示會滾動到新路由的頂部，並在向前/向後導航時保持滾動位置。

</PagesOnly>

<AppOnly>

當 `action` 為字串時，`<Form>` 元件支援以下 props：

| Prop       | 範例               | 類型                             | 必填     |
| ---------- | ------------------ | -------------------------------- | -------- |
| `action`   | `action="/search"` | `string` (URL 或相對路徑)        | 是       |
| `replace`  | `replace={false}`  | `boolean`                        | -        |
| `scroll`   | `scroll={true}`    | `boolean`                        | -        |
| `prefetch` | `prefetch={true}`  | `boolean`                        | -        |

- **`action`**：提交表單時要導航到的 URL 或路徑。
  - 空字串 `""` 會導航到相同路由並更新搜尋參數。
- **`replace`**：替換當前的歷史狀態，而非在[瀏覽器歷史記錄](https://developer.mozilla.org/en-US/docs/Web/API/History_API)堆疊中新增一筆。預設為 `false`。
- **`scroll`**：控制導航時的滾動行為。預設為 `true`，表示會滾動到新路由的頂部，並在向前/向後導航時保持滾動位置。
- **`prefetch`**：控制表單在用戶視窗中可見時是否預載路徑。預設為 `true`。

### `action` (函式) Props

當 `action` 為函式時，`<Form>` 元件支援以下 prop：

| Prop     | 範例                | 類型                          | 必填     |
| -------- | ------------------- | ----------------------------- | -------- |
| `action` | `action={myAction}` | `function` (伺服器動作)       | 是       |

- **`action`**：提交表單時要呼叫的伺服器動作 (Server Action)。詳見 [React 文件](https://react.dev/reference/react-dom/components/form#props)。

> **須知**：當 `action` 為函式時，`replace` 和 `scroll` props 會被忽略。

</AppOnly>

### 注意事項

<AppOnly>

- **`formAction`**：可在 `<button>` 或 `<input type="submit">` 欄位中使用，以覆寫 `action` prop。Next.js 會執行客戶端導航，但此方法不支援預載 (prefetching)。
  - 使用 [`basePath`](/docs/app/api-reference/config/next-config-js/basePath) 時，必須在 `formAction` 路徑中包含它，例如 `formAction="/base-path/search"`。
- **`key`**：不支援對字串 `action` 傳遞 `key` prop。如需觸發重新渲染或執行變更，請考慮改用函式 `action`。

</AppOnly>

- **`onSubmit`**：可用於處理表單提交邏輯。但呼叫 `event.preventDefault()` 會覆寫 `<Form>` 的行為（例如導航到指定 URL）。
- **[`method`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#method)、[`encType`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#enctype)、[`target`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#target)**：不支援，因為它們會覆寫 `<Form>` 的行為。
  - 類似地，`formMethod`、`formEncType` 和 `formTarget` 可用於分別覆寫 `method`、`encType` 和 `target` props，使用它們會回退到原生瀏覽器行為。
  - 如需使用這些 props，請改用 HTML `<form>` 元素。
- **`<input type="file">`**：當 `action` 為字串時使用此輸入類型，會符合瀏覽器行為，提交檔案名稱而非檔案物件。

<AppOnly>

## 範例

### 導向搜尋結果頁面的搜尋表單

您可以透過傳遞路徑作為 `action` 來建立導向搜尋結果頁面的搜尋表單：

```tsx filename="/app/page.tsx" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

```jsx filename="/app/page.js" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

當用戶更新查詢輸入欄位並提交表單時，表單資料會被編碼為 URL 的搜尋參數，例如 `/search?query=abc`。

> **須知**：如果您傳遞空字串 `""` 給 `action`，表單會導航到相同路由並更新搜尋參數。

在結果頁面，您可以使用 [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) `page.js` prop 來存取查詢，並用它從外部來源取得資料。

```tsx filename="/app/search/page.tsx" switcher
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const results = await getSearchResults((await searchParams).query)

  return <div>...</div>
}
```

```jsx filename="/app/search/page.js" switcher
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({ searchParams }) {
  const results = await getSearchResults((await searchParams).query)

  return <div>...</div>
}
```

當 `<Form>` 在用戶視窗中可見時，`/search` 頁面的共享 UI（例如 `layout.js` 和 `loading.js`）會被預載。提交時，表單會立即導航到新路由，並在取得結果時顯示載入 UI。您可以使用 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 來設計後備 UI：

```tsx filename="/app/search/loading.tsx" switcher
export default function Loading() {
  return <div>Loading...</div>
}
```

```jsx filename="/app/search/loading.js" switcher
export default function Loading() {
  return <div>Loading...</div>
}
```

為了涵蓋共享 UI 尚未載入的情況，您可以使用 [`useFormStatus`](https://react.dev/reference/react-dom/hooks/useFormStatus) 向用戶顯示即時反饋。

首先，建立一個在表單處理中顯示載入狀態的元件：

```tsx filename="/app/ui/search-button.tsx" switcher
'use client'
import { useFormStatus } from 'react-dom'

export default function SearchButton() {
  const status = useFormStatus()
  return (
    <button type="submit">{status.pending ? '搜尋中...' : '搜尋'}</button>
  )
}
```

```jsx filename="/app/ui/search-button.js" switcher
'use client'
import { useFormStatus } from 'react-dom'

export default function SearchButton() {
  const status = useFormStatus()
  return (
    <button type="submit">{status.pending ? '搜尋中...' : '搜尋'}</button>
  )
}
```

然後，更新搜尋表單頁面以使用 `SearchButton` 元件：

```tsx filename="/app/page.tsx" switcher
import Form from 'next/form'
import { SearchButton } from '@/ui/search-button'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <SearchButton />
    </Form>
  )
}
```

```jsx filename="/app/ui/search-button.js" switcher
import Form from 'next/form'
import { SearchButton } from '@/ui/search-button'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <SearchButton />
    </Form>
  )
}
```

### 使用伺服器動作執行變更

您可以透過傳遞函式給 `action` prop 來執行變更。

```tsx filename="/app/posts/create/page.tsx" switcher
import Form from 'next/form'
import { createPost } from '@/posts/actions'

export default function Page() {
  return (
    <Form action={createPost}>
      <input name="title" />
      {/* ... */}
      <button type="submit">建立文章</button>
    </Form>
  )
}
```

```jsx filename="/app/posts/create/page.js" switcher
import Form from 'next/form'
import { createPost } from '@/posts/actions'

export default function Page() {
  return (
    <Form action={createPost}>
      <input name="title" />
      {/* ... */}
      <button type="submit">建立文章</button>
    </Form>
  )
}
```

變更後，通常會重新導向到新資源。您可以使用 `next/navigation` 的 [`redirect`](/docs/app/building-your-application/routing/redirecting) 函式來導向到新文章頁面。

> **須知**：由於表單提交的「目的地」在動作執行前未知，`<Form>` 無法自動預載共享 UI。

```tsx filename="/app/posts/actions.ts" switcher
'use server'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  // 建立新文章
  // ...

  // 導向到新文章
  redirect(`/posts/${data.id}`)
}
```

```jsx filename="/app/posts/actions.js" switcher
'use server'
import { redirect } from 'next/navigation'

export async function createPost(formData) {
  // 建立新文章
  // ...

  // 導向到新文章
  redirect(`/posts/${data.id}`)
}
```

然後，在新頁面中，您可以使用 `params` prop 來取得資料：

```tsx filename="/app/posts/[id]/page.tsx" switcher
import { getPost } from '@/posts/data'

export default async function PostPage({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const data = await getPost(id)

  return (
    <div>
      <h1>{data.title}</h1>
      {/* ... */}
    </div>
  )
}
```

```jsx filename="/app/posts/[id]/page.js" switcher
import { getPost } from '@/posts/data'

    </div>

```
