您可以使用 `metadata` 物件或 `generateMetadata` 函數來定義元數據。

## `metadata` 物件

要定義靜態元數據，請從 `layout.js` 或 `page.js` 檔案匯出一個 [`Metadata` 物件](#metadata-fields)。

```tsx filename="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: '...',
  description: '...',
}

export default function Page() {}
```

```jsx filename="layout.js | page.js" switcher
export const metadata = {
  title: '...',
  description: '...',
}

export default function Page() {}
```

> 查看 [元數據欄位](#metadata-fields) 以獲取完整支援的選項列表。

## `generateMetadata` 函數

依賴於**動態資訊**的元數據，例如當前路由參數、外部數據或父區段中的 `metadata`，可以通過匯出一個返回 [`Metadata` 物件](#metadata-fields) 的 `generateMetadata` 函數來設定。

```tsx filename="app/products/[id]/page.tsx" switcher
import type { Metadata, ResolvingMetadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  // 讀取路由參數
  const { id } = await params

  // 獲取數據
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // 可選地訪問並擴展（而非替換）父元數據
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }: Props) {}
```

```jsx filename="app/products/[id]/page.js" switcher
export async function generateMetadata({ params, searchParams }, parent) {
  // 讀取路由參數
  const { id } = await params

  // 獲取數據
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // 可選地訪問並擴展（而非替換）父元數據
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }) {}
```

> **須知事項**:
>
> - 元數據可以添加到 `layout.js` 和 `page.js` 檔案中。
> - Next.js 會自動解析元數據，並為頁面創建相關的 `<head>` 標籤。
> - `metadata` 物件和 `generateMetadata` 函數匯出**僅在伺服器元件中支援**。
> - 您不能從同一路由區段同時匯出 `metadata` 物件和 `generateMetadata` 函數。
> - `generateMetadata` 中的 `fetch` 請求會自動[記憶化](/docs/app/deep-dive/caching#request-memoization)，以便在 `generateMetadata`、`generateStaticParams`、Layouts、Pages 和 Server Components 之間共享相同數據。
> - 如果無法使用 `fetch`，可以使用 React 的 [`cache` 函數](/docs/app/deep-dive/caching#react-cache-function)。
> - [基於檔案的元數據](/docs/app/api-reference/file-conventions/metadata)具有更高的優先級，並會覆蓋 `metadata` 物件和 `generateMetadata` 函數。

## 參考

### 參數

`generateMetadata` 函數接受以下參數：

- `props` - 包含當前路由參數的物件：
  - `params` - 包含從根區段到調用 `generateMetadata` 的區段的[動態路由參數](/docs/app/building-your-application/routing/dynamic-routes)物件。例如：

    | 路由                           | URL         | `params`                  |
    | ------------------------------- | ----------- | ------------------------- |
    | `app/shop/[slug]/page.js`       | `/shop/1`   | `{ slug: '1' }`           |
    | `app/shop/[tag]/[item]/page.js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |
    | `app/shop/[...slug]/page.js`    | `/shop/1/2` | `{ slug: ['1', '2'] }`    |

  - `searchParams` - 包含當前 URL 的[搜尋參數](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters)的物件。例如：

    | URL             | `searchParams`       |
    | --------------- | -------------------- |
    | `/shop?a=1`     | `{ a: '1' }`         |
    | `/shop?a=1&b=2` | `{ a: '1', b: '2' }` |
    | `/shop?a=1&a=2` | `{ a: ['1', '2'] }`  |

- `parent` - 父路由區段解析後的元數據的 Promise。

### 返回值

`generateMetadata` 應返回一個包含一個或多個元數據欄位的 [`Metadata` 物件](#metadata-fields)。

> **須知事項**:
>
> - 如果元數據不依賴於運行時資訊，應使用靜態的 [`metadata` 物件](#the-metadata-object)而非 `generateMetadata` 來定義。
> - `fetch` 請求會自動[記憶化](/docs/app/deep-dive/caching#request-memoization)，以便在 `generateMetadata`、`generateStaticParams`、Layouts、Pages 和 Server Components 之間共享相同數據。如果無法使用 `fetch`，可以使用 React 的 [`cache` 函數](/docs/app/deep-dive/caching#react-cache-function)。
> - `searchParams` 僅在 `page.js` 區段中可用。
> - Next.js 的 [`redirect()`](/docs/app/api-reference/functions/redirect) 和 [`notFound()`](/docs/app/api-reference/functions/not-found) 方法也可以在 `generateMetadata` 中使用。

### 元數據欄位

以下欄位受到支援：

#### `title`

`title` 屬性用於設定文件的標題。可以定義為簡單的[字串](#string)或可選的[模板物件](#template)。

##### 字串

```jsx filename="layout.js | page.js"
export const metadata = {
  title: 'Next.js',
}
```

```html filename="<head> output" hideLineNumbers
<title>Next.js</title>
```

##### `default`

`title.default` 可用於為未定義 `title` 的子路由區段提供**後備標題**。

```tsx filename="app/layout.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    default: 'Acme',
  },
}
```

```tsx filename="app/about/page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {}

// 輸出: <title>Acme</title>
```

##### `template`

`title.template` 可用於為**子**路由區段中定義的 `titles` 添加前綴或後綴。

```tsx filename="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    template: '%s | Acme',
    default: 'Acme', // 創建模板時必須提供 default
  },
}
```

```jsx filename="app/layout.js" switcher
export const metadata = {
  title: {
    template: '%s | Acme',
    default: 'Acme', // 創建模板時必須提供 default
  },
}
```

```tsx filename="app/about/page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'About',
}

// 輸出: <title>About | Acme</title>
```

```jsx filename="app/about/page.js" switcher
export const metadata = {
  title: 'About',
}

// 輸出: <title>About | Acme</title>
```

> **須知事項**:
>
> - `title.template` 應用於**子**路由區段，而非定義它的區段。這意味著：
>
>   - 添加 `title.template` 時，必須提供 `title.default`。
>   - `layout.js` 中定義的 `title.template` 不會應用於同一路由區段的 `page.js` 中定義的 `title`。
>   - `page.js` 中定義的 `title.template` 無效，因為頁面始終是路由的終止區段（它沒有任何子路由區段）。
>
> - 如果路由未定義 `title` 或 `title.default`，`title.template` **無效**。

##### `absolute`

`title.absolute` 可用於提供一個**忽略**父區段中設定的 `title.template` 的標題。

```tsx filename="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    template: '%s | Acme',
  },
}
```

```jsx filename="app/layout.js" switcher
export const metadata = {
  title: {
    template: '%s | Acme',
  },
}
```

```tsx filename="app/about/page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    absolute: 'About',
  },
}

// 輸出: <title>About</title>
```

```jsx filename="app/about/page.js" switcher
export const metadata = {
  title: {
    absolute: 'About',
  },
}

// 輸出: <title>About</title>
```

> **須知事項**:
>
> - `layout.js`
>
>   - `title` (字串) 和 `title.default` 為未定義自己 `title` 的子區段定義默認標題。如果存在，它將擴展最近父區段的 `title.template`。
>   - `title.absolute` 為子區段定義默認標題。它忽略父區段的 `title.template`。
>   - `title.template` 為子區段定義新的標題模板。
>
> - `page.js`
>   - 如果頁面未定義自己的標題，將使用最近父區段的解析標題。
>   - `title` (字串) 定義路由的標題。如果存在，它將擴展最近父區段的 `title.template`。
>   - `title.absolute` 定義路由標題。它忽略父區段的 `title.template`。
>   - `title.template` 在 `page.js` 中無效，因為頁面始終是路由的終止區段。

#### `metadataBase`

`metadataBase` 是一個方便的選項，用於為需要完整 URL 的元數據欄位設置基礎 URL 前綴。

- `metadataBase` 允許**當前路由區段及以下**的基於 URL 的元數據欄位使用**相對路徑**，而非原本需要的絕對 URL。
- 該欄位的相對路徑將與 `metadataBase` 組合形成完整的 URL。

```jsx filename="layout.js | page.js"
export const metadata = {
  metadataBase: new URL('https://acme.com'),
  alternates: {
    canonical: '/',
    languages: {
      'en-US': '/en-US',
      'de-DE': '/de-DE',
    },
  },
  openGraph: {
    images: '/og-image.png',
  },
}
```

```html filename="<head> output" hideLineNumbers
<link rel="canonical" href="https://acme.com" />
<link rel="alternate" hreflang="en-US" href="https://acme.com/en-US" />
<link rel="alternate" hreflang="de-DE" href="https://acme.com/de-DE" />
<meta property="og:image" content="https://acme.com/og-image.png" />
```

> **須知事項**:
>
> - `metadataBase` 通常設置在根 `app/layout.js` 中，以應用於所有路由的基於 URL 的元數據欄位。
> - 所有需要絕對 URL 的基於 URL 的元數據欄位都可以通過 `metadataBase` 選項進行配置。
> - `metadataBase` 可以包含子域名，例如 `https://app.acme.com` 或基礎路徑，例如 `https://acme.com/start/from/here`。
> - 如果元數據欄位提供絕對 URL，`metadataBase` 將被忽略。
> - 在不配置 `metadataBase` 的情況下在基於 URL 的元數據欄位中使用相對路徑將導致建構錯誤。
> - Next.js 會將 `metadataBase`（例如 `https://acme.com/`）和相對欄位（例如 `/path`）之間的重複斜線標準化為單一斜線（例如 `https://acme.com/path`）。

#### URL 組合

URL 組合優先考慮開發者意圖而非默認的目錄遍歷語義。

- `metadataBase` 和元數據欄位之間的尾隨斜線會被標準化。
- 元數據欄位中的「絕對」路徑（通常會替換整個 URL 路徑）被視為「相對」路徑（從 `metadataBase` 的末尾開始）。

例如，給定以下 `metadataBase`：

```tsx filename="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

```jsx filename="app/layout.js" switcher
export const metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

任何繼承上述 `metadataBase` 並設定自己值的元數據欄位將按以下方式解析：

| 元數據欄位                 | 解析後的 URL                     |
| -------------------------------- | -------------------------------- |
| `/`                              | `https://acme.com`               |
| `./`                             | `https://acme.com`               |
| `payments`                       | `https://acme.com/payments`      |
| `/payments`                      | `https://acme.com/payments`      |
| `./payments`                     | `https://acme.com/payments`      |
| `../payments`                    | `https://acme.com/payments`      |
| `https://beta.acme.com/payments` | `https://beta.acme.com/payments` |

### 資源提示

`<link>` 元素有許多 `rel` 關鍵字，可用於提示瀏覽器可能需要外部資源。瀏覽器根據這些關鍵字應用預載優化。

雖然元數據 API 不直接支援這些提示，但您可以使用新的 [`ReactDOM` 方法](https://github.com/facebook/react/pull/26237) 安全地將它們插入到文件的 `<head>` 中。

```tsx filename="app/preload-resources.tsx" switcher
'use client'

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

```jsx filename="app/preload-resources.js" switcher
'use client'

import ReactDOM from 'react-dom'

export function PreloadResources() {
  ReactDOM.preload('...', { as: '...' })
  ReactDOM.preconnect('...', { crossOrigin: '...' })
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

#### `<link rel="preload">`

在頁面渲染（瀏覽器）生命週期早期開始載入資源。[MDN 文檔](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preload)。

```tsx
ReactDOM.preload(href: string, options: { as: string })
```

```html filename="<head> output" hideLineNumbers
<link rel="preload" href="..." as="..." />
```

##### `<link rel="preconnect">`

預先初始化與來源的連接。[MDN 文檔](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/preconnect)。

```tsx

```

```html filename="<head> output" hideLineNumbers

```

#### `<link rel="dns-prefetch">`

在資源被請求之前嘗試解析域名。[MDN 文檔](https://developer.mozilla.org/docs/Web/HTML/Attributes/rel/dns-prefetch)。



> **須知**:
>
> - 這些方法目前僅在客戶端組件中支援，這些組件在初始頁面載入時仍會進行伺服器端渲染。
> - Next.js 內建功能如 `next/font`、`next/image` 和 `next/script` 會自動處理相關的資源提示。

## 行為

### 預設字段

即使路由未定義元數據，也會始終添加兩個預設的 `meta` 標籤：

- [meta charset 標籤](https://developer.mozilla.org/docs/Web/HTML/Element/meta#attr-charset) 設置網站的字符編碼。
- [meta viewport 標籤](https://developer.mozilla.org/docs/Web/HTML/Viewport_meta_tag) 設置網站的視口寬度和縮放比例，以適應不同設備。

```html
<meta charset="utf-8" />
<meta name="viewport" content="width=device-width, initial-scale=1" />
```

> **須知**：您可以覆蓋預設的 [`viewport`](/docs/app/api-reference/functions/generate-metadata#viewport) meta 標籤。

### 流式元數據

由 `generateMetadata` 返回的元數據會流式傳輸到客戶端。這使得 Next.js 可以在元數據解析後立即將其注入 HTML。

由於頁面元數據主要針對機器人和爬蟲，Next.js 會為可以執行 JavaScript 並檢查完整頁面 DOM 的機器人（如 `Googlebot`）流式傳輸元數據。然而，對於 **HTML 受限** 的機器人（如 `Twitterbot`），元數據仍會阻塞頁面渲染，因為這些機器人在爬取時無法執行 JavaScript。

Next.js 會自動檢測傳入請求的用戶代理，以決定是提供流式元數據還是回退到阻塞元數據。

如果需要自定義此列表，您可以在 `next.config.js` 中使用 `htmlLimitedBots` 選項手動定義它們。Next.js 會確保匹配此正則表達式的用戶代理在請求您的網頁時接收阻塞元數據。

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const config: NextConfig = {
  htmlLimitedBots: /MySpecialBot|MyAnotherSpecialBot|SimpleCrawler/,
}

export default config
```

```js filename="next.config.js" switcher
module.exports = {
  htmlLimitedBots: /MySpecialBot|MyAnotherSpecialBot|SimpleCrawler/,
}
```

指定 `htmlLimitedBots` 配置將覆蓋 Next.js 的預設列表，讓您可以完全控制哪些用戶代理應選擇此行為。這是高級行為，預設值在大多數情況下已足夠。

### 順序

元數據按順序評估，從根區段開始，一直到最接近最終 `page.js` 區段的區段。例如：

1. `app/layout.tsx` (根佈局)
2. `app/blog/layout.tsx` (嵌套的部落格佈局)
3. `app/blog/[slug]/page.tsx` (部落格頁面)

### 合併

遵循[評估順序](#ordering)，從同一路由的多個區段導出的元數據物件會**淺層合併**在一起，形成路由的最終元數據輸出。重複的鍵會根據它們的順序**替換**。

這意味著在較早區段中定義的嵌套字段（如 [`openGraph`](/docs/app/api-reference/functions/generate-metadata#opengraph) 和 [`robots`](/docs/app/api-reference/functions/generate-metadata#robots)）會被最後定義它們的區段**覆蓋**。

#### 覆蓋字段

```jsx filename="app/layout.js"
export const metadata = {
  title: 'Acme',
  openGraph: {
    title: 'Acme',
    description: 'Acme is a...',
  },
}
```

```jsx filename="app/blog/page.js"
export const metadata = {
  title: 'Blog',
  openGraph: {
    title: 'Blog',
  },
}

// 輸出:
// <title>Blog</title>
// <meta property="og:title" content="Blog" />
```

在上面的例子中：

- `app/layout.js` 中的 `title` 被 `app/blog/page.js` 中的 `title` **替換**。
- `app/layout.js` 中的所有 `openGraph` 字段在 `app/blog/page.js` 中被**替換**，因為 `app/blog/page.js` 設置了 `openGraph` 元數據。注意缺少 `openGraph.description`。

如果您希望在區段之間共享一些嵌套字段同時覆蓋其他字段，可以將它們提取到單獨的變量中：

```jsx filename="app/shared-metadata.js"
export const openGraphImage = { images: ['http://...'] }
```

```jsx filename="app/page.js"
import { openGraphImage } from './shared-metadata'

export const metadata = {
  openGraph: {
    ...openGraphImage,
    title: 'Home',
  },
}
```

```jsx filename="app/about/page.js"
import { openGraphImage } from '../shared-metadata'

export const metadata = {
  openGraph: {
    ...openGraphImage,
    title: 'About',
  },
}
```

在上面的例子中，OG 圖片在 `app/layout.js` 和 `app/about/page.js` 之間共享，而標題則不同。

#### 繼承字段

```jsx filename="app/layout.js"
export const metadata = {
  title: 'Acme',
  openGraph: {
    title: 'Acme',
    description: 'Acme is a...',
  },
}
```

```jsx filename="app/about/page.js"

}

// 輸出:
// <title>About</title>
// <meta property="og:title" content="Acme" />
// <meta property="og:description" content="Acme is a..." />
```

**注意**

- `app/layout.js` 中的 `title` 被 `app/about/page.js` 中的 `title` **替換**。
- `app/layout.js` 中的所有 `openGraph` 字段在 `app/about/page.js` 中被**繼承**，因為 `app/about/page.js` 未設置 `openGraph` 元數據。
