Next.js 可以作為 Node.js 伺服器、Docker 容器、靜態匯出或透過適配器在不同平台上運行。

| 部署選項                | 功能支援   |
| -------------------------------- | ----------------- |
| [Node.js 伺服器](#nodejs-server) | 全部               |
| [Docker 容器](#docker)      | 全部               |
| [靜態匯出](#static-export)  | 有限           |
| [適配器](#adapters)            | 平台特定 |

## Node.js 伺服器

Next.js 可以部署到任何支援 Node.js 的供應商。請確保你的 `package.json` 包含 `"build"` 和 `"start"` 腳本：

```json filename="package.json"
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start"
  }
}
```

然後，執行 `npm run build` 來建置你的應用程式，並執行 `npm run start` 來啟動 Node.js 伺服器。此伺服器支援所有 Next.js 功能。如有需要，你也可以切換到 [自訂伺服器](/docs/app/guides/custom-server)。

Node.js 部署支援所有 Next.js 功能。學習如何為你的基礎設施 [配置它們](/docs/app/guides/self-hosting)。

## Docker

Next.js 可以部署到任何支援 [Docker](https://www.docker.com/) 容器的供應商。這包括像 Kubernetes 這樣的容器編排器或運行 Docker 的雲端供應商。

Docker 部署支援所有 Next.js 功能。學習如何為你的基礎設施 [配置它們](/docs/app/guides/self-hosting)。

## 靜態匯出

Next.js 可以作為靜態網站或 [單頁應用程式 (SPA)](/docs/app/guides/single-page-applications) 開始，之後可選擇升級使用需要伺服器的功能。

由於 Next.js 支援 [靜態匯出](/docs/app/guides/static-exports)，它可以部署並託管在任何能提供 HTML/CSS/JS 靜態資源的網頁伺服器上。這包括像 AWS S3、Nginx 或 Apache 這樣的工具。

以 [靜態匯出](/docs/app/guides/static-exports) 運行 **不支援** 需要伺服器的 Next.js 功能。[了解更多](/docs/app/guides/static-exports#unsupported-features)。

### 範本

- [GitHub Pages](https://github.com/nextjs/deploy-github-pages)

## 適配器

Next.js 可以透過適配器在不同平台上運行，以支援它們的基礎設施能力。

請參考各供應商的文件以了解支援的 Next.js 功能：

- [AWS Amplify Hosting](https://docs.amplify.aws/nextjs/start/quickstart/nextjs-app-router-client-components)
- [Cloudflare](https://developers.cloudflare.com/workers/frameworks/framework-guides/nextjs)
- [Deno Deploy](https://docs.deno.com/examples/next_tutorial)
- [Netlify](https://docs.netlify.com/frameworks/next-js/overview/#next-js-support-on-netlify)
- [Vercel](https://vercel.com/docs/frameworks/nextjs)

> **注意：** 我們正在開發一個 [部署適配器 API](https://github.com/vercel/next.js/discussions/77740) 供所有平台採用。完成後，我們將新增如何編寫自訂適配器的文件。
