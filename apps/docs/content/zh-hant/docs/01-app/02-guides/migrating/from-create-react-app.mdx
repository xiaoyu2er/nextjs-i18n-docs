本指南將協助您將現有的 Create React App (CRA) 網站遷移至 Next.js。

## 為什麼要切換？

從 Create React App 切換到 Next.js 有以下幾個原因：

### 初始頁面載入時間緩慢

Create React App 僅使用客戶端 React。純客戶端應用程式，也稱為 [單頁應用程式 (SPAs)](/docs/app/guides/single-page-applications)，通常會遇到初始頁面載入時間緩慢的問題。這是由於以下幾個原因造成的：

1. 瀏覽器需要等待 React 代碼和整個應用程式套件下載並運行後，您的代碼才能發送請求載入數據。
2. 您的應用程式代碼隨著新增的功能和依賴項而不斷增長。

### 沒有自動代碼分割

上述載入時間緩慢的問題可以通過代碼分割來稍微緩解。然而，如果您嘗試手動進行代碼分割，可能會無意中引入網絡瀑布效應。Next.js 在其路由器和建構管道中內建了自動代碼分割和樹搖 (tree-shaking) 功能。

### 網絡瀑布效應

性能不佳的一個常見原因是應用程式為了獲取數據而進行連續的客戶端-伺服器請求。在 [SPA](/docs/app/guides/single-page-applications) 中獲取數據的一種模式是先渲染一個佔位符，然後在組件掛載後獲取數據。不幸的是，子組件只能在其父組件完成載入自己的數據後才能開始獲取數據，從而導致請求的「瀑布」效應。

雖然 Next.js 支援客戶端數據獲取，但它也允許您將數據獲取移至伺服器端。這通常可以完全消除客戶端-伺服器的瀑布效應。

### 快速且可控的載入狀態

通過內建的 [透過 React Suspense 進行串流](/docs/app/building-your-application/routing/loading-ui-and-streaming#streaming-with-suspense) 支援，您可以定義哪些部分的 UI 先載入以及載入順序，而不會產生網絡瀑布效應。

這使您能夠構建載入速度更快的頁面，並消除 [版面偏移](https://vercel.com/blog/how-core-web-vitals-affect-seo)。

### 選擇數據獲取策略

根據您的需求，Next.js 允許您在頁面或組件級別選擇數據獲取策略。例如，您可以從 CMS 獲取數據並在建構時 (SSG) 渲染博客文章以實現快速載入速度，或者在必要時在請求時 (SSR) 獲取數據。

### 中介軟體 (Middleware)

[Next.js 中介軟體](/docs/app/building-your-application/routing/middleware) 允許您在請求完成之前在伺服器上運行代碼。例如，您可以通過在中介軟體中將用戶重定向到登入頁面來避免未驗證內容的閃爍，從而保護僅限驗證用戶訪問的頁面。您還可以將其用於 A/B 測試、實驗和 [國際化](/docs/app/guides/internationalization) 等功能。

### 內建優化

[圖片](/docs/app/api-reference/components/image)、[字體](/docs/app/api-reference/components/font) 和 [第三方腳本](/docs/app/guides/scripts) 通常對應用程式的性能有很大影響。Next.js 包含專門的組件和 API，可以自動為您優化這些資源。

## 遷移步驟

我們的目標是盡快獲得一個可運行的 Next.js 應用程式，以便您可以逐步採用 Next.js 的功能。首先，我們將您的應用程式視為純客戶端應用程式 ([SPA](/docs/app/guides/single-page-applications))，而不立即替換現有的路由器。這減少了複雜性和合併衝突。

> **注意**：如果您使用高級 CRA 配置，例如 `package.json` 中的自定義 `homepage` 字段、自定義 service worker 或特定的 Babel/webpack 調整，請參閱本指南末尾的 **其他注意事項** 部分，以獲取在 Next.js 中複製或調整這些功能的提示。

### 步驟 1：安裝 Next.js 依賴項

在現有專案中安裝 Next.js：

```bash filename="Terminal"
npm install next@latest
```

### 步驟 2：建立 Next.js 配置檔案

在專案的根目錄（與 `package.json` 同一級別）建立一個 `next.config.ts`。此檔案包含您的 [Next.js 配置選項](/docs/app/api-reference/config/next-config-js)。

```js filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  output: 'export', // 輸出一個單頁應用程式 (SPA)
  distDir: 'build', // 將建構輸出目錄更改為 `build`
}

export default nextConfig
```

> **注意**：使用 `output: 'export'` 表示您正在進行靜態導出。您將 **無法** 使用伺服器端功能，如 SSR 或 API。您可以移除這一行以利用 Next.js 的伺服器功能。

### 步驟 3：建立根佈局 (Root Layout)

Next.js [App Router](/docs/app) 應用程式必須包含一個 [根佈局](/docs/app/building-your-application/routing/layouts-and-templates#root-layout-required) 檔案，這是一個 [React 伺服器組件](/docs/app/getting-started/server-and-client-components)，將包裹您的所有頁面。

在 CRA 應用程式中，最接近根佈局檔案的是 `public/index.html`，其中包含您的 `<html>`、`<head>` 和 `<body>` 標籤。

1. 在您的 `src` 資料夾內建立一個新的 `app` 目錄（或者如果您希望 `app` 位於根目錄，則直接在專案根目錄建立）。
2. 在 `app` 目錄內，建立一個 `layout.tsx`（或 `layout.js`）檔案：

```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return '...'
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return '...'
}
```

現在將舊的 `index.html` 的內容複製到這個 `<RootLayout>` 組件中。將 `body div#root`（和 `body noscript`）替換為 `<div id="root">{children}</div>`。

```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <head>
        <meta charSet="UTF-8" />
        <link rel="icon" href="%PUBLIC_URL%/favicon.ico" />
        <meta name="viewport" content="width=device-width, initial-scale=1" />
        <title>React App</title>
        <meta name="description" content="Web site created..." />
      </head>
      <body>
        <div id="root">{children}</div>
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <head>
        <meta charset="UTF-8" />
        <link rel="icon" href="%PUBLIC_URL%/favicon.ico" />
        <meta name="viewport" content="width=device-width, initial-scale=1" />
        <title>React App</title>
        <meta name="description" content="Web site created..." />
      </head>
      <body>
        <div id="root">{children}</div>
      </body>
    </html>
  )
}
```

> **小知識**：Next.js 預設會忽略 CRA 的 `public/manifest.json`、額外的圖標和 [測試配置](/docs/app/guides/testing)。如果您需要這些，Next.js 提供了 [Metadata API](/docs/app/getting-started/metadata-and-og-images) 和 [測試](/docs/app/guides/testing) 設置的支援。

### 步驟 4：元數據 (Metadata)

Next.js 自動包含 `<meta charset="UTF-8" />` 和 `<meta name="viewport" content="width=device-width, initial-scale=1" />` 標籤，因此您可以將它們從 `<head>` 中移除：

```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <head>
        <link rel="icon" href="%PUBLIC_URL%/favicon.ico" />
        <title>React App</title>
        <meta name="description" content="Web site created..." />
      </head>
      <body>
        <div id="root">{children}</div>
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <head>
        <link rel="icon" href="%PUBLIC_URL%/favicon.ico" />
        <title>React App</title>
        <meta name="description" content="Web site created..." />
      </head>
      <body>
        <div id="root">{children}</div>
      </body>
    </html>
  )
}
```

任何 [元數據檔案](/docs/app/getting-started/metadata-and-og-images#file-based-metadata)，例如 `favicon.ico`、`icon.png`、`robots.txt`，只要將它們放置在 `app` 目錄的頂層，就會自動添加到應用程式的 `<head>` 標籤中。將 [所有支援的檔案](/docs/app/getting-started/metadata-and-og-images#file-based-metadata) 移動到 `app` 目錄後，您可以安全地刪除它們的 `<link>` 標籤：

```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <head>
        <title>React App</title>
        <meta name="description" content="Web site created..." />
      </head>
      <body>
        <div id="root">{children}</div>
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <head>
        <title>React App</title>
        <meta name="description" content="Web site created..." />
      </head>
      <body>
        <div id="root">{children}</div>
      </body>
    </html>
  )
}
```

最後，Next.js 可以通過 [Metadata API](/docs/app/getting-started/metadata-and-og-images) 管理您的最後一個 `<head>` 標籤。將您的最終元數據信息移動到一個導出的 [`metadata` 物件](/docs/app/api-reference/functions/generate-metadata#metadata-object) 中：

```tsx filename="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'React App',
  description: 'Web site created with Next.js.',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        <div id="root">{children}</div>
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export const metadata = {
  title: 'React App',
  description: 'Web site created with Next.js.',
}

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <div id="root">{children}</div>
      </body>
    </html>
  )
}
```

通過上述更改，您從在 `index.html` 中聲明所有內容轉變為使用 Next.js 內建於框架的基於約定的方法 ([Metadata API](/docs/app/getting-started/metadata-and-og-images))。這種方法使您能夠更輕鬆地提高頁面的 SEO 和網絡共享性。

### 步驟 5：樣式

與 CRA 一樣，Next.js 開箱即用地支援 [CSS 模組](/docs/app/getting-started/css#css-modules)。它還支援 [全局 CSS 導入](/docs/app/getting-started/css#global-css)。

如果您有一個全局 CSS 檔案，請將其導入到您的 `app/layout.tsx` 中：

```tsx filename="app/layout.tsx" switcher
import '../index.css'

export const metadata = {
  title: 'React App',
  description: 'Web site created with Next.js.',
}

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        <div id="root">{children}</div>
      </body>
    </html>
  )
}
```

如果您使用 Tailwind CSS，請參閱我們的 [安裝文檔](/docs/app/guides/tailwind-css)。

### 步驟 6：建立入口頁面

Create React App 使用 `src/index.tsx`（或 `index.js`）作為入口點。在 Next.js (App Router) 中，`app` 目錄內的每個資料夾對應一個路由，每個資料夾應包含一個 `page.tsx`。

由於我們希望暫時將應用程式保持為 SPA 並攔截 **所有** 路由，我們將使用 [可選的全域路由](/docs/app/building-your-application/routing/dynamic-routes#optional-catch-all-segments)。

1. **在 `app` 內建立一個 `[[...slug]]` 目錄。**

```bash
app
 ┣ [[...slug]]
 ┃ ┗ page.tsx
 ┣ layout.tsx
```

2. **將以下內容添加到 `page.tsx`**：

```tsx filename="app/[[...slug]]/page.tsx" switcher
export function generateStaticParams() {
  return [{ slug: [''] }]
}

export default function Page() {
  return '...' // 我們稍後會更新此處
}
```

```jsx filename="app/[[...slug]]/page.js" switcher
export function generateStaticParams() {
  return [{ slug: [''] }]
}

export default function Page() {
  return '...' // 我們稍後會更新此處
}
```

這告訴 Next.js 為空 slug (`/`) 生成單一路由，有效地將 **所有** 路由映射到同一頁面。此頁面是一個 [伺服器組件](/docs/app/getting-started/server-and-client-components)，預渲染為靜態 HTML。

### 步驟 7：添加僅客戶端入口點

接下來，我們將把您的 CRA 的根 App 組件嵌入到一個 [客戶端組件](/docs/app/getting-started/server-and-client-components) 中，以便所有邏輯保持客戶端。如果您是第一次使用 Next.js，值得知道的是客戶端組件（預設情況下）仍然會在伺服器上預渲染。您可以將它們視為具有運行客戶端 JavaScript 的額外能力。

在 `app/[[...slug]]/` 中建立一個 `client.tsx`（或 `client.js`）：

```tsx filename="app/[[...slug]]/client.tsx" switcher
'use client'

import dynamic from 'next/dynamic'

const App = dynamic(() => import('../../App'), { ssr: false })

export function ClientOnly() {
  return <App />
}
```

```jsx filename="app/[[...slug]]/client.js" switcher
'use client'

import dynamic from 'next/dynamic'

const App = dynamic(() => import('../../App'), { ssr: false })

export function ClientOnly() {
  return <App />
}
```

- `'use client'` 指令使此檔案成為 **客戶端組件**。
- 帶有 `ssr: false` 的 `dynamic` 導入會禁用 `<App />` 組件的伺服器端渲染，使其成為真正的僅客戶端 (SPA)。

現在更新您的 `page.tsx`（或 `page.js`）以使用您的新組件：

```tsx filename="app/[[...slug]]/page.tsx" switcher
import { ClientOnly } from './client'

export function generateStaticParams() {
  return [{ slug: [''] }]
}

export default function Page() {
  return <ClientOnly />
}
```

```jsx filename="app/[[...slug]]/page.js" switcher
import { ClientOnly } from './client'

export function generateStaticParams() {
  return [{ slug: [''] }]
}

export default function Page() {
  return <ClientOnly />
}
```

### 步驟 8：更新靜態圖片導入方式

在 CRA 中，導入圖片檔案會返回其公開 URL 作為字串：

```tsx
import image from './img.png'

export default function App() {
  return <img src={image} />
}
```

使用 Next.js 時，靜態圖片導入會返回一個物件。該物件可以直接用於 Next.js 的 [`<Image>` 元件](/docs/app/api-reference/components/image)，或者你也可以使用物件的 `src` 屬性搭配現有的 `<img>` 標籤。

`<Image>` 元件具有[自動圖片優化](/docs/app/api-reference/components/image)的額外優勢。該元件會根據圖片的尺寸自動設定結果 `<img>` 的 `width` 和 `height` 屬性，這能防止圖片載入時的版面位移。但若你的應用中包含僅設定單一維度（另一維度未設定為 `auto`）的圖片，可能會導致問題。當未設定為 `auto` 時，該維度會預設為 `<img>` 維度屬性的值，可能導致圖片顯示變形。

保留 `<img>` 標籤可以減少應用程式中的變更量，並避免上述問題。之後你可以選擇性地遷移到 `<Image>` 元件，透過[設定 loader](/docs/app/api-reference/components/image#loader) 來優化圖片，或改用預設支援自動圖片優化的 Next.js 伺服器。

**將從 `/public` 導入圖片的絕對路徑轉換為相對導入：**

```tsx
// 之前
import logo from '/logo.png'

// 之後
import logo from '../public/logo.png'
```

**將圖片的 `src` 屬性（而非整個圖片物件）傳遞給 `<img>` 標籤：**

```tsx
// 之前
<img src={logo} />

// 之後
<img src={logo.src} />
```

或者，你也可以根據檔案名稱直接引用圖片的公開 URL。例如，`public/logo.png` 會在你的應用程式中透過 `/logo.png` 提供圖片，這將是 `src` 的值。

> **警告：** 如果你使用 TypeScript，在存取 `src` 屬性時可能會遇到類型錯誤。要解決此問題，你需要在 `tsconfig.json` 檔案的 [`include` 陣列](https://www.typescriptlang.org/tsconfig#include)中加入 `next-env.d.ts`。當你在步驟 9 執行應用程式時，Next.js 會自動生成此檔案。

### 步驟 9：遷移環境變數

Next.js 支援[環境變數](/docs/app/guides/environment-variables)，方式與 CRA 類似，但**要求**任何要在瀏覽器中暴露的變數必須加上 `NEXT_PUBLIC_` 前綴。

主要差異在於暴露客戶端環境變數的前綴。將所有帶有 `REACT_APP_` 前綴的環境變數更改為 `NEXT_PUBLIC_`。

### 步驟 10：更新 `package.json` 中的腳本

更新你的 `package.json` 腳本以使用 Next.js 指令。同時，將 `.next` 和 `next-env.d.ts` 加入 `.gitignore`：

```json filename="package.json"
{
  "scripts": {
    "dev": "next dev --turbopack",
    "build": "next build",
    "start": "npx serve@latest ./build"
  }
}
```

```txt filename=".gitignore"
# ...
.next
next-env.d.ts
```

現在你可以執行：

```bash
npm run dev
```

開啟 [http://localhost:3000](http://localhost:3000)。你應該會看到你的應用程式現在以 Next.js 運行（SPA 模式）。

### 步驟 11：清理

現在你可以移除 Create React App 特有的檔案：

- `public/index.html`
- `src/index.tsx`
- `src/react-app-env.d.ts`
- `reportWebVitals` 設定
- `react-scripts` 依賴（從 `package.json` 中解除安裝）

## 其他注意事項

### 在 CRA 中使用自訂 `homepage`

如果你在 CRA 的 `package.json` 中使用 `homepage` 欄位來讓應用程式在特定子路徑下運行，可以在 Next.js 中使用 [`basePath` 設定](/docs/app/api-reference/config/next-config-js/basePath)來達成相同效果：

```ts filename="next.config.ts"
import { NextConfig } from 'next'

const nextConfig: NextConfig = {
  basePath: '/my-subpath',
  // ...
}

export default nextConfig
```

### 處理自訂 `Service Worker`

如果你使用 CRA 的 service worker（例如來自 `create-react-app` 的 `serviceWorker.js`），可以學習如何在 Next.js 中建立[漸進式網路應用程式 (PWA)](/docs/app/guides/progressive-web-apps)。

### 代理 API 請求

如果你的 CRA 應用程式使用 `package.json` 中的 `proxy` 欄位來轉發請求到後端伺服器，可以在 `next.config.ts` 中使用 [Next.js rewrites](/docs/app/api-reference/config/next-config-js/rewrites) 來達成相同效果：

```ts filename="next.config.ts"
import { NextConfig } from 'next'

const nextConfig: NextConfig = {
  async rewrites() {
    return [
      {
        source: '/api/:path*',
        destination: 'https://your-backend.com/:path*',
      },
    ]
  },
}
```

### 自訂 Webpack / Babel 設定

如果你在 CRA 中有自訂的 webpack 或 Babel 設定，可以在 `next.config.ts` 中擴展 Next.js 的設定：

```ts filename="next.config.ts"
import { NextConfig } from 'next'

const nextConfig: NextConfig = {
  webpack: (config, { isServer }) => {
    // 在此修改 webpack 設定
    return config
  },
}

export default nextConfig
```

> **注意：** 這需要停用 Turbopack，方法是從 `dev` 腳本中移除 `--turbopack`。

### TypeScript 設定

Next.js 會自動設定 TypeScript（如果你有 `tsconfig.json`）。請確保 `next-env.d.ts` 列在你的 `tsconfig.json` 的 `include` 陣列中：

```json
{
  "include": ["next-env.d.ts", "app/**/*", "src/**/*"]
}
```

## 打包工具相容性

Create React App 和 Next.js 預設都使用 webpack 進行打包。Next.js 還提供 [Turbopack](/docs/app/api-reference/config/next-config-js/turbopack) 以獲得更快的本地開發體驗：

```bash
next dev --turbopack
```

如果需要從 CRA 遷移進階的 webpack 設定，你仍然可以提供[自訂的 webpack 設定](/docs/app/api-reference/config/next-config-js/webpack)。

## 後續步驟

如果一切順利，你現在已經有一個以單頁應用程式模式運行的 Next.js 應用程式。雖然尚未利用 Next.js 的伺服器渲染或檔案式路由等功能，但你可以逐步開始使用：

- **從 React Router 遷移**到 [Next.js App Router](/docs/app/building-your-application/routing)，以獲得：
  - 自動程式碼分割
  - [串流伺服器渲染](/docs/app/building-your-application/routing/loading-ui-and-streaming)
  - [React 伺服器元件](/docs/app/getting-started/server-and-client-components)
- 使用 [`<Image>` 元件](/docs/app/api-reference/components/image) **優化圖片**
- 使用 [`next/font`](/docs/app/api-reference/components/font) **優化字體**
- 使用 [`<Script>` 元件](/docs/app/guides/scripts) **優化第三方腳本**
- 執行 `npx next lint` **啟用 ESLint** 並設定 Next.js 推薦規則以符合專案需求

> **注意：** 使用靜態導出 (`output: 'export'`) [目前不支援](https://github.com/vercel/next.js/issues/54393) `useParams` 鉤子或其他伺服器功能。若要使用所有 Next.js 功能，請從 `next.config.ts` 中移除 `output: 'export'`。
