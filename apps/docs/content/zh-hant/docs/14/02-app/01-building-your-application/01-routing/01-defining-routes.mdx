> 建議在繼續之前先閱讀[路由基礎](/docs/app/building-your-application/routing)頁面。

本頁將引導你如何在 Next.js 應用程式中定義和組織路由。

## 建立路由

Next.js 使用基於檔案系統的路由器，其中**資料夾**被用來定義路由。

每個資料夾代表一個[路由區段 (route segment)](/docs/app/building-your-application/routing#route-segments)，對應到一個 **URL** 區段。要建立[巢狀路由 (nested route)](/docs/app/building-your-application/routing#nested-routes)，你可以將資料夾相互嵌套。

特殊的 [`page.js` 檔案](/docs/app/building-your-application/routing/pages-and-layouts#pages)用於讓路由區段可公開存取。

在此範例中，`/dashboard/analytics` URL 路徑_無法_公開存取，因為它沒有對應的 `page.js` 檔案。此資料夾可用於儲存元件、樣式表、圖片或其他共置檔案。

> **須知**：特殊檔案可使用 `.js`、`.jsx` 或 `.tsx` 副檔名。

## 建立使用者介面

[特殊檔案慣例](/docs/app/building-your-application/routing#file-conventions)用於為每個路由區段建立使用者介面。最常見的是[頁面 (pages)](/docs/app/building-your-application/routing/pages-and-layouts#pages)用於顯示路由專屬的使用者介面，以及[佈局 (layouts)](/docs/app/building-your-application/routing/pages-and-layouts#layouts)用於顯示跨多個路由共享的使用者介面。

例如，要建立你的第一個頁面，在 `app` 目錄內新增一個 `page.js` 檔案並匯出一個 React 元件：

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```
