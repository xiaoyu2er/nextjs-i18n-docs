## 從 13 升級至 14

要更新至 Next.js 第 14 版，請使用您偏好的套件管理工具執行以下指令：

```bash filename="終端機"
npm i next@latest react@latest react-dom@latest eslint-config-next@latest
```

```bash filename="終端機"
yarn add next@latest react@latest react-dom@latest eslint-config-next@latest
```

```bash filename="終端機"
pnpm up next react react-dom eslint-config-next --latest
```

```bash filename="終端機"
bun add next@latest react@latest react-dom@latest eslint-config-next@latest
```

> **小提示：** 如果您使用 TypeScript，請確保同時將 `@types/react` 和 `@types/react-dom` 升級至最新版本。

### v14 更新摘要

- 最低 Node.js 版本要求從 16.14 提升至 18.17，因為 16.x 已結束生命週期。
- `next export` 指令已被移除，改用 `output: 'export'` 設定。詳情請參閱 [文件](https://nextjs.org/docs/app/building-your-application/deploying/static-exports)。
- `next/server` 中的 `ImageResponse` 導入已更名為 `next/og`。我們提供 [codemod 工具](/docs/app/building-your-application/upgrading/codemods#next-og-import) 可安全自動替換導入路徑。
- `@next/font` 套件已完全移除，改用內建的 `next/font`。我們提供 [codemod 工具](/docs/app/building-your-application/upgrading/codemods#built-in-next-font) 可安全自動替換導入路徑。
- `next-swc` 的 WASM 目標編譯選項已被移除。
