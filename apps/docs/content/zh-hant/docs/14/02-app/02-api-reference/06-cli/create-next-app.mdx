`create-next-app` CLI 讓您能快速建立新的 Next.js 應用程式，使用預設範本或來自公開 Github 儲存庫的[範例](https://github.com/vercel/next.js/tree/canary/examples)。這是開始使用 Next.js 最簡單的方式。

基本用法：

```bash filename="Terminal"
npx create-next-app@latest [專案名稱] [選項]
```

## 參考選項

以下為可用選項：

| 選項                                  | 說明                                                          |
| ------------------------------------- | ------------------------------------------------------------ |
| `-h` 或 `--help`                     | 顯示所有可用選項                                             |
| `-v` 或 `--version`                  | 輸出版本號碼                                                 |
| `--no-*`                             | 取消預設選項。例如 `--no-eslint`                             |
| `--ts` 或 `--typescript`             | 初始化為 TypeScript 專案 (預設)                              |
| `--js` 或 `--javascript`             | 初始化為 JavaScript 專案                                     |
| `--tailwind`                         | 初始化包含 Tailwind CSS 設定 (預設)                          |
| `--eslint`                           | 初始化包含 ESLint 設定                                       |
| `--app`                              | 初始化為應用程式路由器專案                                   |
| `--src-dir`                          | 在 `src/` 目錄內初始化                                       |
| `--turbo`                            | 預設啟用 Turbopack 進行開發                                  |
| `--import-alias <設定別名>`          | 指定使用的導入別名 (預設 "@/*")                             |
| `--empty`                            | 初始化空專案                                                 |
| `--use-npm`                          | 明確告知 CLI 使用 npm 初始化應用程式                         |
| `--use-pnpm`                         | 明確告知 CLI 使用 pnpm 初始化應用程式                        |
| `--use-yarn`                         | 明確告知 CLI 使用 Yarn 初始化應用程式                        |
| `--use-bun`                          | 明確告知 CLI 使用 Bun 初始化應用程式                         |
| `-e` 或 `--example [名稱] [github-url]` | 用於初始化應用程式的範例                                     |
| `--example-path <範例路徑>`          | 單獨指定範例的路徑                                           |
| `--reset-preferences`                | 明確告知 CLI 重置所有儲存的偏好設定                          |
| `--skip-install`                     | 明確告知 CLI 跳過安裝套件                                    |
| `--yes`                              | 使用先前的偏好設定或所有選項的預設值                         |

## 範例

## 使用預設範本

要使用預設範本建立新應用程式，請在終端機執行以下命令：

```bash filename="Terminal"
npx create-next-app@latest
```

接著會出現以下提示問題：

```txt filename="Terminal"
請輸入專案名稱？  my-app
是否要使用 TypeScript？  否 / 是
是否要使用 ESLint？  否 / 是
是否要使用 Tailwind CSS？  否 / 是
是否要將程式碼放在 `src/` 目錄中？  否 / 是
是否要使用應用程式路由器？(推薦)  否 / 是
是否要為 `next dev` 使用 Turbopack？  否 / 是
是否要自訂導入別名？(預設為 `@/*`)  否 / 是
```

回答完問題後，將根據您的選擇建立新專案。

## 使用官方 Next.js 範例

要使用官方 Next.js 範例建立新應用程式，請使用 `--example` 參數執行以下命令：

```bash filename="Terminal"
npx create-next-app@latest --example [您的專案名稱] [範例名稱]
```

您可以在 [Next.js 儲存庫](https://github.com/vercel/next.js/tree/canary/examples) 查看所有可用範例列表及設定說明。

## 使用任何公開 Github 範例

要使用任何公開 Github 範例建立新應用程式，請使用 `--example` 選項加上 Github 儲存庫網址。例如：

```bash filename="Terminal"
npx create-next-app@latest --example [您的專案名稱] "https://github.com/.../"
```
