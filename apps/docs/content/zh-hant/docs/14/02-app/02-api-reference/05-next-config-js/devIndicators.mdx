<AppOnly>

當您編輯程式碼且 Next.js 正在編譯應用程式時，頁面右下角會出現一個編譯指示器。

> **須知**：此指示器僅在開發模式下出現，在生產模式下建置和執行應用程式時不會顯示。

在某些情況下，此指示器可能會在頁面上位置不當，例如與聊天啟動器衝突。若要變更其位置，請開啟 `next.config.js` 並在 `devIndicators` 物件中將 `buildActivityPosition` 設定為 `bottom-right` (預設值)、`bottom-left`、`top-right` 或 `top-left`：

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivityPosition: 'bottom-right',
  },
}
```

在某些情況下，此指示器可能對您沒有用處。若要移除它，請開啟 `next.config.js` 並在 `devIndicators` 物件中停用 `buildActivity` 設定：

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivity: false,
  },
}
```

</AppOnly>

<PagesOnly>

> **須知**：此指示器已在 Next.js 10.0.1 版本中移除。我們建議升級至最新版本的 Next.js。

當頁面符合[自動靜態最佳化](/docs/pages/building-your-application/rendering/automatic-static-optimization)條件時，我們會顯示一個指示器讓您知道。

這非常有用，因為自動靜態最佳化可能帶來很大好處，在開發時立即知道頁面是否符合條件會很有幫助。

在某些情況下，此指示器可能沒有用處，例如在開發 electron 應用程式時。若要移除它，請開啟 `next.config.js` 並在 `devIndicators` 中停用 `autoPrerender` 設定：

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    autoPrerender: false,
  },
}
```

</PagesOnly>
