某些套件可能會匯出數百甚至數千個模組，這可能會導致開發和生產環境中的效能問題。

將套件加入 `experimental.optimizePackageImports` 後，只會載入您實際使用的模組，同時仍可方便地使用含有多個命名匯入的 import 語句。

```js filename="next.config.js"
module.exports = {
  experimental: {
    optimizePackageImports: ['package-name'],
  },
}
```

預設已針對以下函式庫進行優化：`@mui/icons-material`、`@mui/material`、`date-fns`、`lodash`、`lodash-es`、`react-bootstrap`、`@headlessui/react`、`@heroicons/react` 和 `lucide-react`。
