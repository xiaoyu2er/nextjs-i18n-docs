當專案中檢測到 ESLint 時，Next.js 會在出現錯誤時使您的**正式環境構建** (`next build`) 失敗。

如果您希望 Next.js 即使在應用程式存在 ESLint 錯誤時也能產生正式環境程式碼，可以完全停用內建的 linting 步驟。除非您已將 ESLint 配置在工作流程的其他部分執行（例如在 CI 或 pre-commit 鉤子中），否則不建議這樣做。

開啟 `next.config.js` 並在 `eslint` 配置中啟用 `ignoreDuringBuilds` 選項：

```js filename="next.config.js"
module.exports = {
  eslint: {
    // 警告：這允許即使專案存在 ESLint 錯誤，也能成功完成正式環境構建
    ignoreDuringBuilds: true,
  },
}
```
