2016 年，Google 開始對使用 [AMP](https://amp.dev/) 技術的網頁給予[搜尋排名優先權](https://developers.google.com/search/docs/guides/about-amp)。AMP 能讓開發者建立行動裝置上載入更快的網頁，但代價是需要長期維護這些頁面。

隨著 [Core Web Vitals](https://web.dev/vitals/) 頁面體驗更新，Google [取消了 AMP 頁面作為出現在搜尋輪播中的必要條件](https://developers.google.com/search/blog/2021/04/more-details-page-experience#details)。這曾是 Google 在 SEO 方面為 AMP 提供的主要優勢之一。

自 AMP 推出以來，像 Next.js 這樣的新技術已證明其能在不犧牲開發者體驗 (DX) 的情況下改善網站體驗。

雖然 Next.js 提供 [AMP 支援](https://nextjs.org/docs/advanced-features/amp-support/introduction)，但如果您的網站已有優異的 Core Web Vitals 分數，建議您權衡實施 AMP 的成本與效益。
