將 Next.js 部署至生產環境最簡單的方式，就是使用由 Next.js 開發團隊打造的 **[Vercel](https://vercel.com)** 平台。

Vercel 是一個為靜態與混合應用設計的無伺服器 (serverless) 平台，可與您的無頭內容 (headless content)、電子商務或資料庫整合。我們讓前端團隊能輕鬆開發、預覽並部署令人愉悅的使用者體驗，且預設就具備高效能。您可以免費開始使用 — 無需信用卡。

### [建立 Vercel 帳號](#create-a-vercel-account)

首先，前往 [Vercel](https://vercel.com/signup) 建立帳號。選擇 **Continue with GitHub** 並完成註冊流程。

### [匯入您的 `nextjs-blog` 儲存庫](#import-your-nextjs-blog-repository)

註冊完成後，在 Vercel 上**匯入**您的 `nextjs-blog` 儲存庫。您可以從此處操作：[Vercel](https://vercel.com/import)。

*   您需要**安裝 Vercel for GitHub**。您可以授予它**所有儲存庫**的存取權限。
*   安裝 Vercel 後，匯入 `nextjs-blog`。

以下設定您可以使用_預設值_ — 無需變更任何內容。Vercel 會自動偵測到您有一個 Next.js 應用程式，並為您選擇最佳建置設定。

*   專案名稱 (Project Name)
*   根目錄 (Root Directory)
*   建置指令 (Build Command)
*   輸出目錄 (Output Directory)
*   開發指令 (Development Command)

部署時，您的 Next.js 應用程式會開始建置。通常會在一分鐘內完成。

> **取得協助**：如果部署失敗，您隨時可以在 [GitHub Discussions](https://github.com/vercel/next.js/discussions) 尋求幫助。想了解更多部署相關資訊，請參考[我們的文件](/docs/deployment)。

完成後，您會獲得**部署網址**。點擊任一網址，您應該就能看到 Next.js 起始頁面正式上線。

恭喜！您已成功將 Next.js 應用程式部署至生產環境。在下一頁，我們將深入探討 Vercel 與推薦的工作流程。
