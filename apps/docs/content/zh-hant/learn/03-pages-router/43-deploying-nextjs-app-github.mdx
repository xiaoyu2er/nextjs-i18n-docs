在我們部署之前，如果您尚未這麼做，請先將我們的 Next.js 應用程式推送至 [GitHub](https://github.com/vercel/next.js)。這將使部署過程更加容易。

*   在您的個人 GitHub 帳戶中，建立一個名為 `nextjs-blog` 的新儲存庫。
*   儲存庫可以是公開或私有的。您**不需要**使用 README 或其他檔案來初始化它。
*   如果您需要設定儲存庫的協助，請參考 [GitHub 上的這份指南](https://help.github.com/en/github/getting-started-with-github/create-a-repo)。

接著：

*   如果您尚未在本地為您的 Next.js 應用程式初始化 git 儲存庫，請現在進行。
*   將 Next.js 應用程式推送至您的 GitHub 儲存庫。

要推送至 GitHub，您可以執行以下指令（將 `<username>` 替換為您的 GitHub 使用者名稱）：

```
git remote add origin https://github.com/<username>/nextjs-blog.git
git push -u origin main
```

當您的 GitHub 儲存庫準備就緒後，請繼續前往下一頁。
