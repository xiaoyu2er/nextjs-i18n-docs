Next.js предоставляет несколько опций, которые позволяют контролировать, как сервер будет освобождать или сохранять в памяти собранные страницы в процессе разработки.

Для изменения настроек по умолчанию откройте файл `next.config.js` и добавьте конфигурацию `onDemandEntries`:

```js filename="next.config.js"
module.exports = {
  onDemandEntries: {
    // период (в мс), в течение которого сервер будет хранить страницы в буфере
    maxInactiveAge: 25 * 1000,
    // количество страниц, которые должны храниться одновременно без освобождения
    pagesBufferLength: 2,
  },
}
```
