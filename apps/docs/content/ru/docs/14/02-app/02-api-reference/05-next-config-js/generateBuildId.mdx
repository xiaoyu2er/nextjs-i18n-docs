Next.js генерирует идентификатор во время выполнения `next build`, чтобы определить, какая версия вашего приложения обслуживается. Одна и та же сборка должна использоваться и запускаться на нескольких контейнерах.

Если вы выполняете пересборку для каждого этапа вашего окружения, вам потребуется сгенерировать постоянный идентификатор сборки (build ID) для использования между контейнерами. Используйте команду `generateBuildId` в файле `next.config.js`:

```jsx filename="next.config.js"
module.exports = {
  generateBuildId: async () => {
    // Это может быть что угодно, например, последний хэш git
    return process.env.GIT_HASH
  },
}
```
