<AppOnly>

Когда вы редактируете код и Next.js компилирует приложение, в правом нижнем углу страницы появляется индикатор компиляции.

> **Полезно знать**: Этот индикатор отображается только в режиме разработки и не появляется при сборке и запуске приложения в production-режиме.

В некоторых случаях этот индикатор может некорректно отображаться на странице, например, при конфликте с виджетом чата. Чтобы изменить его положение, откройте `next.config.js` и задайте `buildActivityPosition` в объекте `devIndicators` одно из значений: `bottom-right` (по умолчанию), `bottom-left`, `top-right` или `top-left`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivityPosition: 'bottom-right',
  },
}
```

Если индикатор вам не нужен, его можно отключить. Для этого в `next.config.js` установите параметр `buildActivity` в объекте `devIndicators` в значение `false`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivity: false,
  },
}
```

</AppOnly>

<PagesOnly>

> **Полезно знать**: Этот индикатор был удалён в Next.js версии 10.0.1. Рекомендуем обновиться до последней версии Next.js.

Когда страница соответствует требованиям [Автоматической статической оптимизации](/docs/pages/building-your-application/rendering/automatic-static-optimization), отображается соответствующий индикатор.

Это полезно, поскольку автоматическая статическая оптимизация может быть очень выгодной, и возможность сразу увидеть в режиме разработки, соответствует ли страница требованиям, может пригодиться.

В некоторых случаях этот индикатор может быть не нужен, например при работе с Electron-приложениями. Чтобы отключить его, откройте `next.config.js` и установите параметр `autoPrerender` в объекте `devIndicators` в значение `false`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    autoPrerender: false,
  },
}
```

</PagesOnly>
