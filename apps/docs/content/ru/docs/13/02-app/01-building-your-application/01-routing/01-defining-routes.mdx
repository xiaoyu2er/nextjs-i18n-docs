Эта страница расскажет, как определять и организовывать маршруты в вашем приложении Next.js.

## Создание маршрутов

Next.js использует файловую систему для маршрутизации, где **папки** определяют маршруты.

Каждая папка представляет [сегмент маршрута](/docs/app/building-your-application/routing#route-segments), который соответствует **сегменту URL**. Для создания [вложенного маршрута](/docs/app/building-your-application/routing#nested-routes) вы можете вкладывать папки друг в друга.

Специальный файл [`page.js`](/docs/app/building-your-application/routing/pages-and-layouts#pages) делает сегменты маршрута общедоступными.

В этом примере путь URL `/dashboard/analytics` _не_ является общедоступным, так как для него нет соответствующего файла `page.js`. Эта папка может использоваться для хранения компонентов, таблиц стилей, изображений или других сопутствующих файлов.

> **Полезно знать**: для специальных файлов можно использовать расширения `.js`, `.jsx` или `.tsx`.

## Создание интерфейса

[Специальные соглашения о файлах](/docs/app/building-your-application/routing#file-conventions) используются для создания интерфейса каждого сегмента маршрута. Наиболее распространённые — [страницы](/docs/app/building-your-application/routing/pages-and-layouts#pages) для отображения уникального интерфейса маршрута и [макеты](/docs/app/building-your-application/routing/pages-and-layouts#layouts) для отображения интерфейса, общего для нескольких маршрутов.

Например, чтобы создать свою первую страницу, добавьте файл `page.js` в директорию `app` и экспортируйте React-компонент:

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```
