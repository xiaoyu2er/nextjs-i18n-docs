Next.js использует постоянный идентификатор, генерируемый во время сборки, чтобы определить, какая версия вашего приложения обслуживается. Это может вызвать проблемы в мультисерверных развертываниях, когда `next build` выполняется на каждом сервере. Для сохранения единого идентификатора сборки между разными сборками вы можете предоставить собственный идентификатор.

Откройте `next.config.js` и добавьте функцию `generateBuildId`:

```js filename="next.config.js"
module.exports = {
  generateBuildId: async () => {
    // Здесь вы можете, например, получить последний хеш коммита git
    return 'my-build-id'
  },
}
```
