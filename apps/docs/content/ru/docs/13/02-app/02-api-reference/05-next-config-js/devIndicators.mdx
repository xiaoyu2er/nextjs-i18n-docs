<AppOnly>

Когда вы редактируете код и Next.js компилирует приложение, в правом нижнем углу страницы появляется индикатор компиляции.

> **Полезно знать**: Этот индикатор отображается только в режиме разработки и не появляется при сборке и запуске приложения в продакшн-режиме.

В некоторых случаях индикатор может некорректно отображаться на странице, например, при конфликте с чат-виджетом. Чтобы изменить его положение, откройте `next.config.js` и задайте `buildActivityPosition` в объекте `devIndicators` одно из значений: `bottom-right` (по умолчанию), `bottom-left`, `top-right` или `top-left`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivityPosition: 'bottom-right',
  },
}
```

Если индикатор вам не нужен, его можно отключить. Для этого в `next.config.js` установите параметр `buildActivity` в объекте `devIndicators` в значение `false`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivity: false,
  },
}
```

</AppOnly>

<PagesOnly>

> **Полезно знать**: Этот индикатор был удалён в Next.js версии 10.0.1. Рекомендуем обновиться до последней версии Next.js.

Когда страница соответствует требованиям [Автоматической статической оптимизации](/docs/pages/building-your-application/rendering/automatic-static-optimization), отображается соответствующий индикатор.

Это полезно, поскольку автоматическая статическая оптимизация может значительно улучшить производительность, и важно сразу видеть в процессе разработки, подходит ли страница для этой оптимизации.

В некоторых случаях индикатор может быть не нужен, например, при разработке Electron-приложений. Чтобы отключить его, откройте `next.config.js` и установите параметр `autoPrerender` в объекте `devIndicators` в значение `false`:

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    autoPrerender: false,
  },
}
```

</PagesOnly>
