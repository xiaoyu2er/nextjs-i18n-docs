Когда ESLint обнаружен в вашем проекте, Next.js прерывает **продакшен-сборку** (`next build`), если присутствуют ошибки.

Если вы хотите, чтобы Next.js создавал продакшен-код даже при наличии ошибок ESLint в вашем приложении, вы можете полностью отключить встроенный этап линтинга. Это не рекомендуется, если у вас уже не настроен ESLint для запуска в отдельной части рабочего процесса (например, в CI или pre-commit хуке).

Откройте `next.config.js` и включите опцию `ignoreDuringBuilds` в конфигурации `eslint`:

```js filename="next.config.js"
module.exports = {
  eslint: {
    // Внимание: Это позволяет успешно завершать продакшен-сборки,
    // даже если в вашем проекте есть ошибки ESLint.
    ignoreDuringBuilds: true,
  },
}
```
