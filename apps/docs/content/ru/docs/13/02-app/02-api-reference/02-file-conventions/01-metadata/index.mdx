Этот раздел документации посвящён **соглашениям по файлам метаданных**. Файловые метаданные можно определить, добавив специальные файлы метаданных в сегменты маршрутов.

Каждое соглашение может быть определено либо как статический файл (например, `opengraph-image.jpg`), либо как динамический вариант с использованием кода для генерации файла (например, `opengraph-image.js`).

После определения файла Next.js автоматически обслуживает файл (с хешами в production-режиме для кэширования) и обновляет соответствующие элементы head правильными метаданными, такими как URL ресурса, тип файла и размер изображения.
