Next.js не блокирует автоматически межсайтовые запросы во время разработки, но начиная с будущей основной версии Next.js будет блокировать их по умолчанию, чтобы предотвратить несанкционированные запросы внутренних ресурсов/конечных точек, доступных в режиме разработки.

Чтобы настроить приложение Next.js для разрешения запросов от источников, отличных от имени хоста, с которым был инициализирован сервер (по умолчанию `localhost`), вы можете использовать параметр конфигурации `allowedDevOrigins`.

`allowedDevOrigins` позволяет задать дополнительные источники, которые можно использовать в режиме разработки. Например, чтобы использовать `local-origin.dev` вместо только `localhost`, откройте `next.config.js` и добавьте конфигурацию `allowedDevOrigins`:

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```
