`devIndicators` позволяет настроить индикатор на экране, который предоставляет информацию о текущем маршруте во время разработки.

```ts filename="Types"
  devIndicators: false | {
    position?: 'bottom-right'
    | 'bottom-left'
    | 'top-right'
    | 'top-left', // по умолчанию 'bottom-left',
  },
```

Установка `devIndicators` в значение `false` скроет индикатор, однако Next.js продолжит отображать любые ошибки сборки или выполнения, которые возникли.

## Устранение проблем

### Индикатор не отмечает маршрут как статический

Если вы ожидаете, что маршрут должен быть статическим, а индикатор отмечает его как динамический, вероятно, маршрут отказался от статического рендеринга.

Вы можете проверить, является ли маршрут [статическим](/docs/app/getting-started/partial-prerendering#static-rendering) или [динамическим](/docs/app/getting-started/partial-prerendering#dynamic-rendering), собрав приложение с помощью `next build --debug` и проверив вывод в терминале. Статические (или предварительно отрендеренные) маршруты будут отмечены символом `○`, тогда как динамические маршруты — символом `ƒ`. Например:

```bash filename="Build Output"
Route (app)                              Size     First Load JS
┌ ○ /_not-found                          0 B               0 kB
└ ƒ /products/[id]                       0 B               0 kB

○  (Static)   prerendered as static content
ƒ  (Dynamic)  server-rendered on demand
```

<AppOnly>

Есть две причины, по которым маршрут может отказаться от статического рендеринга:

- Наличие [динамических API](/docs/app/getting-started/partial-prerendering#dynamic-rendering), которые зависят от информации во время выполнения.
- [Не кэшированный запрос данных](/docs/app/getting-started/fetching-data), например вызов ORM или драйвера базы данных.

Проверьте ваш маршрут на наличие этих условий, и если вы не можете статически отрендерить маршрут, рассмотрите возможность использования [`loading.js`](/docs/app/api-reference/file-conventions/loading) или [`<Suspense />`](https://react.dev/reference/react/Suspense) для использования [стриминга](/docs/app/building-your-application/routing/loading-ui-and-streaming#what-is-streaming).

</AppOnly>

<PagesOnly>

При экспорте [`getServerSideProps`](/docs/pages/building-your-application/data-fetching/get-server-side-props) или [`getInitialProps`](/docs/pages/api-reference/functions/get-initial-props) из страницы она будет помечена как динамическая.

</PagesOnly>
