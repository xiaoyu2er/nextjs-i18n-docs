`staleTimes` — это экспериментальная функция, которая позволяет кешировать сегменты страниц в [клиентском кеше роутера](/docs/app/deep-dive/caching#client-side-router-cache).

Вы можете включить эту экспериментальную функцию и задать собственное время ревалидации, установив экспериментальный флаг `staleTimes`:

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

Свойства `static` и `dynamic` соответствуют периоду времени (в секундах) для разных типов [префетчинга ссылок](/docs/app/api-reference/components/link#prefetch).

- Свойство `dynamic` используется, когда страница не статически сгенерирована и не полностью префетчена (например, с `prefetch={true}`).
  - По умолчанию: 0 секунд (не кешируется)
- Свойство `static` используется для статически сгенерированных страниц или когда свойство `prefetch` компонента `Link` установлено в `true`, либо при вызове [`router.prefetch`](/docs/app/deep-dive/caching#routerprefetch).
  - По умолчанию: 5 минут

> **Полезно знать:**
>
> - [Границы загрузки](/docs/app/api-reference/file-conventions/loading) считаются переиспользуемыми в течение периода `static`, определённого в этой конфигурации.
> - Это не влияет на [частичный рендеринг](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering), **то есть общие лэйауты не будут автоматически перезапрашиваться при каждой навигации — только изменяемый сегмент страницы.**
> - Это не меняет поведение [кеширования назад/вперёд](/docs/app/deep-dive/caching#client-side-router-cache) для предотвращения сдвигов лэйаута и потери позиции прокрутки в браузере.

Подробнее о клиентском кеше роутера можно узнать [здесь](/docs/app/deep-dive/caching#client-side-router-cache).
