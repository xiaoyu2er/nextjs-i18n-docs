Next.js прерывает **сборку для продакшена** (`next build`), если в вашем проекте присутствуют ошибки TypeScript.

Если вы хотите, чтобы Next.js опасно продолжал сборку продакшен-кода даже при наличии ошибок в приложении, вы можете отключить встроенную проверку типов.

При отключении убедитесь, что проверка типов выполняется как часть процесса сборки или деплоя, иначе это может быть очень опасно.

Откройте `next.config.js` и включите опцию `ignoreBuildErrors` в конфигурации `typescript`:

```js filename="next.config.js"
module.exports = {
  typescript: {
    // !! ВНИМАНИЕ !!
    // Опасное разрешение на успешное завершение сборки для продакшена,
    // даже если в вашем проекте есть ошибки типов.
    // !! ВНИМАНИЕ !!
    ignoreBuildErrors: true,
  },
}
```
