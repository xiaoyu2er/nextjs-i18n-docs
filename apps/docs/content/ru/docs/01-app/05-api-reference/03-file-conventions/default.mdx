Файл `default.js` используется для отображения резервного содержимого в [параллельных маршрутах](/docs/app/api-reference/file-conventions/parallel-routes), когда Next.js не может восстановить активное состояние [слота](/docs/app/api-reference/file-conventions/parallel-routes#slots) после полной загрузки страницы.

При [мягкой навигации](/docs/app/building-your-application/routing/linking-and-navigating#5-soft-navigation) Next.js отслеживает активное _состояние_ (подстраницу) для каждого слота. Однако при жесткой навигации (полной загрузке страницы) Next.js не может восстановить активное состояние. В этом случае для подстраниц, не соответствующих текущему URL, может быть отображен файл `default.js`.

Рассмотрим следующую структуру папок. Слот `@team` имеет страницу `settings`, а `@analytics` — нет.

При переходе на `/settings` слот `@team` отобразит страницу `settings`, сохраняя текущую активную страницу для слота `@analytics`.

При обновлении страницы Next.js отобразит `default.js` для `@analytics`. Если `default.js` не существует, вместо него будет отображена ошибка `404`.

Кроме того, поскольку `children` является неявным слотом, вам также необходимо создать файл `default.js` для отображения резервного содержимого для `children`, когда Next.js не может восстановить активное состояние родительской страницы.

## Справочник

### `params` (опционально)

Promise, который разрешается в объект, содержащий [динамические параметры маршрута](/docs/app/api-reference/file-conventions/dynamic-routes) от корневого сегмента до подстраниц слота. Например:

```tsx filename="app/[artist]/@sidebar/default.js" switcher
export default async function Default({
  params,
}: {
  params: Promise<{ artist: string }>
}) {
  const { artist } = await params
}
```

```jsx filename="app/[artist]/@sidebar/default.js" switcher
export default async function Default({ params }) {
  const { artist } = await params
}
```

| Пример                                     | URL          | `params`                                     |
| ------------------------------------------ | ------------ | -------------------------------------------- |
| `app/[artist]/@sidebar/default.js`         | `/zack`      | `Promise<{ artist: 'zack' }>`                |
| `app/[artist]/[album]/@sidebar/default.js` | `/zack/next` | `Promise<{ artist: 'zack', album: 'next' }>` |

- Поскольку `params` является Promise, для доступа к значениям необходимо использовать `async/await` или функцию [`use`](https://react.dev/reference/react/use) из React.
  - В версии 14 и ранее `params` был синхронным свойством. Для обеспечения обратной совместимости в Next.js 15 его по-прежнему можно использовать синхронно, но это поведение будет устаревать в будущих версиях.
