Функция `redirect` позволяет перенаправлять пользователя на другой URL. `redirect` может использоваться в [Серверных Компонентах](/docs/app/getting-started/server-and-client-components), [Обработчиках Маршрутов](/docs/app/building-your-application/routing/route-handlers) и [Серверных Действиях](/docs/app/building-your-application/data-fetching/server-actions-and-mutations).

При использовании в [контексте потоковой передачи (streaming)](/docs/app/building-your-application/routing/loading-ui-and-streaming#what-is-streaming) функция вставляет мета-тег для выполнения перенаправления на стороне клиента. При использовании в серверном действии она возвращает HTTP-ответ с кодом 303 для перенаправления. В остальных случаях возвращается HTTP-ответ с кодом 307.

Если ресурс не существует, вместо этого можно использовать функцию [`notFound`](/docs/app/api-reference/functions/not-found).

> **Полезно знать**:
>
> - В Серверных Действиях и Обработчиках Маршрутов `redirect` следует вызывать после блока `try/catch`.
> - Если требуется вернуть HTTP-перенаправление с кодом 308 (Постоянное) вместо 307 (Временное), можно использовать функцию [`permanentRedirect`](/docs/app/api-reference/functions/permanentRedirect).

## Параметры

Функция `redirect` принимает два аргумента:

```js
redirect(path, type)
```

| Параметр | Тип                                                          | Описание                                                 |
| --------- | ------------------------------------------------------------- | ----------------------------------------------------------- |
| `path`    | `string`                                                      | URL для перенаправления. Может быть относительным или абсолютным. |
| `type`    | `'replace'` (по умолчанию) или `'push'` (по умолчанию в Серверных Действиях) | Тип перенаправления.                            |

По умолчанию `redirect` использует `push` (добавляет новую запись в стек истории браузера) в [Серверных Действиях](/docs/app/building-your-application/data-fetching/server-actions-and-mutations) и `replace` (заменяет текущий URL в стеке истории браузера) во всех остальных случаях. Это поведение можно переопределить, указав параметр `type`.

Параметр `type` не имеет эффекта при использовании в Серверных Компонентах.

## Возвращаемое значение

Функция `redirect` не возвращает значение.

## Пример

### Серверный Компонент

Вызов функции `redirect()` вызывает ошибку `NEXT_REDIRECT` и прекращает рендеринг сегмента маршрута, в котором она была вызвана.

```tsx filename="app/team/[id]/page.tsx" switcher
import { redirect } from 'next/navigation'

async function fetchTeam(id: string) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const team = await fetchTeam(id)

  if (!team) {
    redirect('/login')
  }

  // ...
}
```

```jsx filename="app/team/[id]/page.js" switcher
import { redirect } from 'next/navigation'

async function fetchTeam(id) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({ params }) {
  const { id } = await params
  const team = await fetchTeam(id)

  if (!team) {
    redirect('/login')
  }

  // ...
}
```

> **Полезно знать**: `redirect` не требует использования `return redirect()`, так как использует тип TypeScript [`never`](https://www.typescriptlang.org/docs/handbook/2/functions.html#never).

### Клиентский Компонент

`redirect` можно использовать напрямую в Клиентском Компоненте.

```tsx filename="components/client-redirect.tsx" switcher
'use client'

import { redirect, usePathname } from 'next/navigation'

export function ClientRedirect() {
  const pathname = usePathname()

  if (pathname.startsWith('/admin') && !pathname.includes('/login')) {
    redirect('/admin/login')
  }

  return <div>Login Page</div>
}
```

```jsx filename="components/client-redirect.jsx" switcher
'use client'

import { redirect, usePathname } from 'next/navigation'

export function ClientRedirect() {
  const pathname = usePathname()

  if (pathname.startsWith('/admin') && !pathname.includes('/login')) {
    redirect('/admin/login')
  }

  return <div>Login Page</div>
}
```

> **Полезно знать**: При использовании `redirect` в Клиентском Компоненте во время первоначальной загрузки страницы при SSR (рендеринге на стороне сервера) будет выполнено серверное перенаправление.

`redirect` можно использовать в Клиентском Компоненте через Серверное Действие. Если требуется перенаправить пользователя с помощью обработчика событий, можно использовать хук [`useRouter`](/docs/app/api-reference/functions/use-router).

```tsx filename="app/client-redirect.tsx" switcher
'use client'

import { navigate } from './actions'

export function ClientRedirect() {
  return (
    <form action={navigate}>
      <input type="text" name="id" />
      <button>Submit</button>
    </form>
  )
}
```

```jsx filename="app/client-redirect.jsx" switcher
'use client'

import { navigate } from './actions'

export function ClientRedirect() {
  return (
    <form action={navigate}>
      <input type="text" name="id" />

```

## Часто задаваемые вопросы

### Почему `redirect` использует коды 307 и 308?

При использовании `redirect()` можно заметить, что применяются коды состояния `307` для временного перенаправления и `308` для постоянного. Хотя традиционно использовались `302` для временного и `301` для постоянного перенаправления, многие браузеры изменяли метод запроса с `POST` на `GET` при использовании `302`, независимо от исходного метода запроса.

Рассмотрим пример перенаправления с `/users` на `/people`. Если сделать `POST`-запрос к `/users` для создания нового пользователя и получить `302` (временное перенаправление), метод запроса изменится с `POST` на `GET`. Это нелогично, так как для создания пользователя должен использоваться `POST`-запрос к `/people`, а не `GET`.

Введение кода состояния `307` означает, что метод запроса сохраняется как `POST`.

- `302` - Временное перенаправление, изменяет метод запроса с `POST` на `GET`
- `307` - Временное перенаправление, сохраняет метод запроса как `POST`

Функция `redirect()` по умолчанию использует `307` вместо `302`, что гарантирует сохранение метода запроса как `POST`.

[Подробнее](https://developer.mozilla.org/docs/Web/HTTP/Redirections) о HTTP-перенаправлениях.
