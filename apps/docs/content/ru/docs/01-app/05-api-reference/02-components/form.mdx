Компонент `<Form>` расширяет HTML-элемент `<form>`, предоставляя <AppOnly>[**префетчинг**](/docs/app/building-your-application/routing/linking-and-navigating#2-prefetching) для [UI загрузки](/docs/app/building-your-application/routing/loading-ui-and-streaming),</AppOnly> **клиентскую навигацию** при отправке и **прогрессивное улучшение**.

Он особенно полезен для форм, обновляющих параметры поиска в URL, так как сокращает количество шаблонного кода, необходимого для реализации вышеуказанного.

Базовое использование:

<AppOnly>

```tsx filename="/app/ui/search.tsx" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      {/* При отправке значение input будет добавлено к
          URL, например /search?query=abc */}
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

```jsx filename="/app/ui/search.js" switcher
import Form from 'next/form'

export default function Search() {
  return (
    <Form action="/search">
      {/* При отправке значение input будет добавлено к
          URL, например /search?query=abc */}
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

</AppOnly>

## Справочник

Поведение компонента `<Form>` зависит от того, передается ли в проп `action` строка (`string`) или функция (`function`).

<AppOnly>

- Когда `action` является **строкой**, `<Form>` ведет себя как нативная HTML-форма с методом **`GET`**. Данные формы кодируются в URL как параметры поиска, и при отправке формы происходит переход по указанному URL. Дополнительно Next.js:
  - [Префетчит](/docs/app/building-your-application/routing/linking-and-navigating#2-prefetching) путь, когда форма становится видимой, предварительно загружая общий UI (например, `layout.js` и `loading.js`), что ускоряет навигацию.
  - Выполняет [клиентскую навигацию](/docs/app/building-your-application/routing/linking-and-navigating#5-soft-navigation) вместо полной перезагрузки страницы при отправке формы. Это сохраняет общий UI и клиентское состояние.
- Когда `action` является **функцией** (Server Action), `<Form>` ведет себя как [React-форма](https://react.dev/reference/react-dom/components/form), выполняя действие при отправке формы.

</AppOnly>

<PagesOnly>

- Когда `action` является **строкой**, `<Form>` ведет себя как нативная HTML-форма с методом **`GET`**. Данные формы кодируются в URL как параметры поиска, и при отправке формы происходит переход по указанному URL. Дополнительно Next.js:

</PagesOnly>

### Пропсы `action` (string)

<PagesOnly>

Когда `action` является строкой, компонент `<Form>` поддерживает следующие пропсы:

| Пропс     | Пример              | Тип                            | Обязательный |
| --------- | ------------------- | ------------------------------ | ------------ |
| `action`  | `action="/search"`  | `string` (URL или относительный путь) | Да          |
| `replace` | `replace={false}`   | `boolean`                      | -            |
| `scroll`  | `scroll={true}`     | `boolean`                      | -            |

- **`action`**: URL или путь для перехода при отправке формы.
  - Пустая строка `""` перейдет на тот же маршрут с обновленными параметрами поиска.
- **`replace`**: Заменяет текущее состояние истории вместо добавления нового в [стек истории браузера](https://developer.mozilla.org/en-US/docs/Web/API/History_API). По умолчанию `false`.
- **`scroll`**: Управляет поведением прокрутки при навигации. По умолчанию `true`, что означает прокрутку к верху нового маршрута и сохранение позиции прокрутки при навигации назад/вперед.

</PagesOnly>

<AppOnly>

Когда `action` является строкой, компонент `<Form>` поддерживает следующие пропсы:

| Пропс      | Пример              | Тип                            | Обязательный |
| ---------- | ------------------- | ------------------------------ | ------------ |
| `action`   | `action="/search"`  | `string` (URL или относительный путь) | Да          |
| `replace`  | `replace={false}`   | `boolean`                      | -            |
| `scroll`   | `scroll={true}`     | `boolean`                      | -            |
| `prefetch` | `prefetch={true}`   | `boolean`                      | -            |

- **`action`**: URL или путь для перехода при отправке формы.
  - Пустая строка `""` перейдет на тот же маршрут с обновленными параметрами поиска.
- **`replace`**: Заменяет текущее состояние истории вместо добавления нового в [стек истории браузера](https://developer.mozilla.org/en-US/docs/Web/API/History_API). По умолчанию `false`.
- **`scroll`**: Управляет поведением прокрутки при навигации. По умолчанию `true`, что означает прокрутку к верху нового маршрута и сохранение позиции прокрутки при навигации назад/вперед.
- **`prefetch`**: Управляет префетчингом пути, когда форма становится видимой в области просмотра пользователя. По умолчанию `true`.

### Пропсы `action` (function)

Когда `action` является функцией, компонент `<Form>` поддерживает следующий пропс:

| Пропс    | Пример               | Тип                       | Обязательный |
| -------- | -------------------- | ------------------------- | ------------ |
| `action` | `action={myAction}`  | `function` (Server Action) | Да          |

- **`action`**: Server Action, которая будет вызвана при отправке формы. Подробнее см. в [документации React](https://react.dev/reference/react-dom/components/form#props).

> **Полезно знать**: Когда `action` является функцией, пропсы `replace` и `scroll` игнорируются.

</AppOnly>

### Ограничения

<AppOnly>

- **`formAction`**: Может использоваться в `<button>` или `<input type="submit">` для переопределения пропса `action`. Next.js выполнит клиентскую навигацию, однако этот подход не поддерживает префетчинг.
  - При использовании [`basePath`](/docs/app/api-reference/config/next-config-js/basePath) его также необходимо включать в путь `formAction`. Например: `formAction="/base-path/search"`.
- **`key`**: Передача пропса `key` для строкового `action` не поддерживается. Если требуется вызвать повторный рендеринг или выполнить мутацию, рассмотрите использование функционального `action`.

</AppOnly>

- **`onSubmit`**: Может использоваться для обработки логики отправки формы. Однако вызов `event.preventDefault()` переопределит поведение `<Form>`, такое как переход по указанному URL.
- **[`method`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#method), [`encType`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#enctype), [`target`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#target)**: Не поддерживаются, так как переопределяют поведение `<Form>`.
  - Аналогично, `formMethod`, `formEncType` и `formTarget` могут использоваться для переопределения пропсов `method`, `encType` и `target` соответственно, и их использование приведет к возврату к нативному поведению браузера.
  - Если вам необходимо использовать эти пропсы, используйте HTML-элемент `<form>`.
- **`<input type="file">`**: Использование этого типа input при строковом `action` приведет к поведению браузера по умолчанию - отправке имени файла вместо объекта файла.

<AppOnly>

## Примеры

### Форма поиска, ведущая на страницу результатов

Вы можете создать форму поиска, которая переходит на страницу результатов, передав путь в качестве `action`:

```tsx filename="/app/page.tsx" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

```jsx filename="/app/page.js" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

Когда пользователь обновляет поле ввода запроса и отправляет форму, данные формы будут закодированы в URL как параметры поиска, например `/search?query=abc`.

> **Полезно знать**: Если передать пустую строку `""` в `action`, форма перейдет на тот же маршрут с обновленными параметрами поиска.

На странице результатов вы можете получить доступ к запросу через пропс [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) `page.js` и использовать его для получения данных из внешнего источника.

```tsx filename="/app/search/page.tsx" switcher
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const results = await getSearchResults((await searchParams).query)

  return <div>...</div>
}
```

```jsx filename="/app/search/page.js" switcher
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({ searchParams }) {
  const results = await getSearchResults((await searchParams).query)

  return <div>...</div>
}
```

Когда `<Form>` становится видимой в области просмотра пользователя, общий UI (такой как `layout.js` и `loading.js`) на странице `/search` будет префетчен. При отправке форма немедленно перейдет на новый маршрут и покажет UI загрузки во время получения результатов. Вы можете настроить резервный UI с помощью [`loading.js`](/docs/app/api-reference/file-conventions/loading):

```tsx filename="/app/search/loading.tsx" switcher
export default function Loading() {
  return <div>Loading...</div>
}
```

```jsx filename="/app/search/loading.js" switcher
export default function Loading() {
  return <div>Loading...</div>
}
```

Для случаев, когда общий UI еще не загрузился, вы можете показать мгновенную обратную связь пользователю с помощью [`useFormStatus`](https://react.dev/reference/react-dom/hooks/useFormStatus).

Сначала создайте компонент, отображающий состояние загрузки, когда форма находится в ожидании:

```tsx filename="/app/ui/search-button.tsx" switcher
'use client'
import { useFormStatus } from 'react-dom'

export default function SearchButton() {
  const status = useFormStatus()
  return (
    <button type="submit">{status.pending ? 'Searching...' : 'Search'}</button>
  )
}
```

```jsx filename="/app/ui/search-button.js" switcher
'use client'
import { useFormStatus } from 'react-dom'

export default function SearchButton() {
  const status = useFormStatus()
  return (
    <button type="submit">{status.pending ? 'Searching...' : 'Search'}</button>
  )
}
```

Затем обновите страницу формы поиска, чтобы использовать компонент `SearchButton`:

```tsx filename="/app/page.tsx" switcher
import Form from 'next/form'
import { SearchButton } from '@/ui/search-button'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <SearchButton />
    </Form>
  )
}
```

```jsx filename="/app/ui/search-button.js" switcher
import Form from 'next/form'
import { SearchButton } from '@/ui/search-button'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <SearchButton />
    </Form>
  )
}
```

### Мутации с Server Actions

Вы можете выполнять мутации, передавая функцию в проп `action`.

```tsx filename="/app/posts/create/page.tsx" switcher
import Form from 'next/form'
import { createPost } from '@/posts/actions'

export default function Page() {
  return (
    <Form action={createPost}>
      <input name="title" />
      {/* ... */}
      <button type="submit">Create Post</button>
    </Form>
  )
}
```

```jsx filename="/app/posts/create/page.js" switcher
import Form from 'next/form'
import { createPost } from '@/posts/actions'

export default function Page() {
  return (
    <Form action={createPost}>
      <input name="title" />
      {/* ... */}
      <button type="submit">Create Post</button>
    </Form>
  )
}
```

После мутации обычно выполняется перенаправление на новый ресурс. Вы можете использовать функцию [`redirect`](/docs/app/guides/redirecting) из `next/navigation` для перехода на страницу нового поста.

> **Полезно знать**: Поскольку "назначение" отправки формы неизвестно до выполнения действия, `<Form>` не может автоматически префетчить общий UI.

```tsx filename="/app/posts/actions.ts" switcher
'use server'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  // Создание нового поста
  // ...

  // Перенаправление на новый пост
  redirect(`/posts/${data.id}`)
}
```

```jsx filename="/app/posts/actions.js" switcher
'use server'
import { redirect } from 'next/navigation'

export async function createPost(formData) {
  // Создание нового поста
  // ...

  // Перенаправление на новый пост
  redirect(`/posts/${data.id}`)
}
```

Затем на новой странице вы можете получить данные с помощью пропса `params`:

```tsx filename="/app/posts/[id]/page.tsx" switcher
import { getPost } from '@/posts/data'

export default async function PostPage({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const data = await getPost(id)

  return (
    <div>
      <h1>{data.title}</h1>
      {/* ... */}
    </div>
  )
}
```

```jsx filename="/app/posts/[id]/page.js" switcher
import { getPost } from '@/posts/data'

```
