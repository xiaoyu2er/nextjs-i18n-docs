Next.js генерирует идентификатор во время выполнения `next build`, чтобы определить, какая версия вашего приложения используется. Одна и та же сборка должна использоваться и запускаться в нескольких контейнерах.

Если вы пересобираете приложение для каждого этапа вашего окружения, вам потребуется сгенерировать постоянный идентификатор сборки для использования между контейнерами. Используйте команду `generateBuildId` в файле `next.config.js`:

```jsx filename="next.config.js"
module.exports = {
  generateBuildId: async () => {
    // Это может быть что угодно, например, последний хеш git
    return process.env.GIT_HASH
  },
}
```
