Еще один отличный бесплатный и открытый способ измерения производительности — использование набора данных [Chrome User Experience Report](https://developers.google.com/web/tools/chrome-user-experience-report).

[Chrome User Experience Report](https://developers.google.com/web/tools/chrome-user-experience-report) предоставляет метрики пользовательского опыта, показывающие, как реальные пользователи Chrome взаимодействуют с популярными веб-ресурсами.

Этот набор данных общедоступен [в BigQuery](https://console.cloud.google.com/bigquery?project=chrome-ux-report), а также его можно бесплатно использовать в [Google Data Studio](https://datastudio.google.com/overview).

К счастью, существуют [доступные открытые дашборды](https://g.co/chromeuxdash), которые можно использовать как шаблоны для отслеживания производительности вашего сайта.

Единственный недостаток этого набора данных в том, что для включения вашего сайта в отчет CrUX требуется значительное количество посещений — в противном случае он не будет включен в отчет из-за недостатка данных. Поэтому это может быть не лучшим вариантом для сайтов в разработке или недавно созданных.

Кроме того, данные обновляются ежемесячно. Обычно примерно через 15 дней после окончания месяца, поэтому вы увидите данные с определенной задержкой, что может быть не самым практичным, если вы планируете работать над улучшением показателей Core Web Vitals.
