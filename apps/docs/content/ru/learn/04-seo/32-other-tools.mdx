Сбор полевых данных доступен в следующих инструментах:

*   [PageSpeed Insights](https://developers.google.com/speed/pagespeed/insights/): Инструмент Google для измерения скорости страниц.
*   [Chrome User Experience Report](https://developers.google.com/web/tools/chrome-user-experience-report): Открытый набор данных о пользовательском опыте в Chrome.
*   [Search Console](https://support.google.com/webmasters/answer/9205520): Отчет о Core Web Vitals в Google Search.

Если вам нужны лабораторные данные, Google также предлагает несколько инструментов для измерений:

*   [Lighthouse](https://developers.google.com/web/tools/lighthouse?hl=en): Открытый автоматизированный инструмент Google для улучшения качества веб-страниц.
*   [Chrome DevTools](https://developer.chrome.com/docs/devtools/): Набор инструментов для разработчиков, встроенный непосредственно в браузер Google Chrome.

Обратите внимание, что в обоих инструментах вам потребуется использовать [Total Blocking Time (TBT)](https://web.dev/tbt/) в качестве альтернативы [First Input Delay (FID)](https://web.dev/fid/), поскольку FID можно измерить только через полевые данные.
