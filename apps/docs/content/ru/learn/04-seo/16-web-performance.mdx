[Web Vitals](https://web.dev/vitals/) — это инициатива Google, предоставляющая унифицированные рекомендации и метрики для измерения пользовательского опыта на веб-страницах.

[Core Web Vitals](https://developers.google.com/search/blog/2020/11/timing-for-page-experience) — это подмножество Web Vitals, которое в настоящее время включает три метрики, измеряющие скорость загрузки, интерактивность и визуальную стабильность. Эти метрики: [Largest Contentful Paint (LCP)](/learn/seo/lcp) (время загрузки самого большого контента), [First Input Delay (FID)](/learn/seo/fid) (задержка первого ввода) и [Cumulative Layout Shift (CLS)](/learn/seo/cls) (кумулятивное смещение макета).

Достижение высоких показателей по этим трём метрикам обеспечит более плавный пользовательский опыт на вашем сайте.

Сайты с низкими показателями Core Web Vitals могут пострадать в поисковом рейтинге, так как Google начал использовать [Core Web Vitals как фактор ранжирования](https://developers.google.com/search/blog/2020/11/timing-for-page-experience) в своём алгоритме поиска. Плохие показатели могут негативно сказаться на веб-трафике и бизнесе.

В этом уроке вы узнаете:

*   Краткую историю Core Web Vitals.
*   Влияние Core Web Vitals на SEO и пользовательский опыт (UX), а также их воздействие на ваш сайт.
*   Почему важно учитывать Core Web Vitals в процессе разработки и как их измерять.
*   Как улучшить показатели Core Web Vitals с помощью Next.js и отслеживать изменения.
