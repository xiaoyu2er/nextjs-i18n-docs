В 2016 году Google начал отдавать [приоритет в поисковой выдаче](https://developers.google.com/search/docs/guides/about-amp) веб-страницам, использующим [AMP](https://amp.dev/) — технологию, позволяющую разработчикам создавать страницы, которые быстрее загружаются на мобильных устройствах, ценой дополнительных усилий по их разработке и поддержке.

С обновлением метрик [Core Web Vitals](https://web.dev/vitals/), связанным с пользовательским опытом, Google [отменил требование](https://developers.google.com/search/blog/2021/04/more-details-page-experience#details) использовать AMP-страницы для отображения в поисковых каруселях. Это было одним из последних основных преимуществ AMP с точки зрения SEO.

С момента появления AMP такие технологии, как Next.js, доказали свою способность улучшать пользовательский опыт без ущерба для опыта разработчиков (DX).

Хотя Next.js поддерживает [режим AMP](https://nextjs.org/docs/advanced-features/amp-support/introduction), стоит тщательно оценить затраты и выгоды от внедрения AMP на вашем сайте, если он уже демонстрирует отличные показатели Core Web Vitals.
