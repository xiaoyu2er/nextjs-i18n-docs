[API-маршруты](/docs/api-routes/introduction) позволяют создавать API-эндпоинты внутри Next.js приложения. Для этого нужно создать **функцию** в директории `pages/api` следующего формата:

```
// req = HTTP входящее сообщение, res = HTTP ответ сервера
export default function handler(req, res) {
  // ...
}
```

> Подробнее об обработчике запросов можно узнать в [документации по API-маршрутам](/docs/api-routes/introduction).

Эти маршруты могут быть развернуты как серверные функции (Serverless Functions, также известные как Lambdas).

### [Создание простого API-эндпоинта](#creating-a-simple-api-endpoint)

Давайте попробуем. Создайте файл `hello.js` в `pages/api` со следующим кодом:

```
export default function handler(req, res) {
  res.status(200).json({ text: 'Hello' });
}
```

Попробуйте обратиться к нему по адресу [http://localhost:3000/api/hello](http://localhost:3000/api/hello). Вы должны увидеть `{"text":"Hello"}`. Обратите внимание:

*   `req` является экземпляром [http.IncomingMessage](https://nodejs.org/api/http.html#http_class_http_incomingmessage) с добавлением предустановленных [мидлваров](/docs/api-routes/api-middlewares).
*   `res` является экземпляром [http.ServerResponse](https://nodejs.org/api/http.html#http_class_http_serverresponse) с добавлением [вспомогательных функций](/docs/api-routes/response-helpers).

Вот и всё! Прежде чем завершить этот урок, давайте обсудим несколько советов по использованию [API-маршрутов](/docs/api-routes/introduction) на следующей странице.
