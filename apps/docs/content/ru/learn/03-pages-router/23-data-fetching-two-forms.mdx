Next.js поддерживает два вида предварительного рендеринга: [**Статическую генерацию**](/docs/basic-features/pages#static-generation-recommended) и [**Рендеринг на стороне сервера (SSR)**](/docs/basic-features/pages#server-side-rendering). Разница заключается в том, **когда** генерируется HTML для страницы.

*   [**Статическая генерация**](/docs/basic-features/pages#static-generation-recommended) — это метод предварительного рендеринга, при котором HTML генерируется во время **сборки**. Затем предварительно сгенерированный HTML _переиспользуется_ при каждом запросе.
*   [**Рендеринг на стороне сервера (SSR)**](/docs/basic-features/pages#server-side-rendering) — это метод предварительного рендеринга, при котором HTML генерируется **при каждом запросе**.

![Статическая генерация](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/pages-router/data-fetching/static-generation.png)

![Рендеринг на стороне сервера](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/pages-router/data-fetching/server-side-rendering.png)

> В режиме разработки (при запуске `npm run dev` или `yarn dev`) страницы [предварительно рендерятся](/docs/basic-features/pages#pre-rendering) при каждом запросе. Это также относится к [Статической генерации](https://nextjs.org/docs/basic-features/data-fetching/get-static-props#runs-on-every-request-in-development) для упрощения разработки. В продакшене Статическая генерация происходит один раз во время сборки и **не** выполняется при каждом запросе.

### [Выбор на уровне страницы](#per-page-basis)

Важно отметить, что Next.js позволяет **выбирать** метод предварительного рендеринга для каждой страницы. Вы можете создать "гибридное" приложение Next.js, используя [Статическую генерацию](/docs/basic-features/pages#static-generation-recommended) для большинства страниц и [Рендеринг на стороне сервера (SSR)](/docs/basic-features/pages#server-side-rendering) для остальных.

![Выбор на уровне страницы](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/pages-router/data-fetching/per-page-basis.png)

### [Когда использовать](#when-to-use-static-generation-vs-server-side-rendering) [Статическую генерацию](/docs/basic-features/pages#static-generation-recommended) или [Рендеринг на стороне сервера (SSR)](/docs/basic-features/pages#server-side-rendering)

Мы рекомендуем по возможности использовать [**Статическую генерацию**](/docs/basic-features/pages#static-generation-recommended) (с данными и без), так как страница может быть собрана один раз и обслуживаться через CDN, что делает её гораздо быстрее, чем рендеринг на сервере при каждом запросе.

Статическую генерацию можно использовать для многих типов страниц, включая:

*   Маркетинговые страницы
*   Посты в блоге
*   Списки товаров в интернет-магазине
*   Справку и документацию

Задайте себе вопрос: "Могу ли я предварительно отрендерить эту страницу **до** запроса пользователя?" Если ответ "да", то следует выбрать [Статическую генерацию](/docs/basic-features/pages#static-generation-recommended).

С другой стороны, [Статическая генерация](/docs/basic-features/pages#static-generation-recommended) **не** подходит, если вы не можете предварительно отрендерить страницу до запроса пользователя. Например, если страница отображает часто обновляемые данные, и её содержимое меняется при каждом запросе.

В таком случае можно использовать [**Рендеринг на стороне сервера (SSR)**](/docs/basic-features/pages#server-side-rendering). Он будет медленнее, но предварительно отрендеренная страница всегда будет актуальной. Или можно отказаться от предварительного рендеринга и использовать клиентский JavaScript для загрузки часто обновляемых данных.
