Сначала убедимся, что ваша среда разработки готова.

*   Если у вас не установлен **Node.js**, [установите его отсюда](https://nodejs.org/en/). Вам понадобится Node.js версии **18** или выше.
*   В этом руководстве вы будете использовать собственный текстовый редактор и терминал.

> Если вы работаете на Windows, мы рекомендуем [скачать Git для Windows](https://gitforwindows.org/) и использовать Git Bash, который входит в его состав, так как он поддерживает UNIX-специфичные команды из этого руководства. Другой вариант — [Windows Subsystem for Linux (WSL)](https://docs.microsoft.com/en-us/windows/wsl/install-win10).

### [Создание приложения Next.js](#create-a-nextjs-app)

Чтобы создать приложение Next.js, откройте терминал, перейдите (`cd`) в директорию, где вы хотите создать приложение, и выполните следующую команду:

```
npx create-next-app@latest nextjs-blog --use-npm --example "https://github.com/vercel/next-learn/tree/main/basics/learn-starter"
```

> Под капотом эта команда использует инструмент [`create-next-app`](/docs/api-reference/create-next-app), который создаёт заготовку приложения Next.js. Она использует [этот шаблон](https://github.com/vercel/next-learn/tree/main/basics/learn-starter) через флаг `--example`.
> 
> Если команда не работает, ознакомьтесь с [этой страницей](https://github.com/vercel/next-learn/blob/main/basics/errors/install.md).

### [Запуск сервера разработки](#run-the-development-server)

Теперь у вас есть новая директория `nextjs-blog`. Перейдём в неё:

```
cd nextjs-blog
```

Затем выполните следующую команду:

```
npm run dev
```

Это запустит "сервер разработки" вашего приложения Next.js (подробнее об этом позже) на порту **3000**.

Проверим, работает ли он. Откройте [http://localhost:3000](http://localhost:3000) в браузере.
