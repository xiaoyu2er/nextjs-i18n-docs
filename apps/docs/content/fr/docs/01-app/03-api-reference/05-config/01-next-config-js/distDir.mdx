Vous pouvez spécifier un nom à utiliser pour un répertoire de build personnalisé au lieu de `.next`.

Ouvrez `next.config.js` et ajoutez la configuration `distDir` :

```js filename="next.config.js"
module.exports = {
  distDir: 'build',
}
```

Maintenant, si vous exécutez `next build`, Next.js utilisera `build` au lieu du dossier `.next` par défaut.

> `distDir` **ne doit pas** quitter le répertoire de votre projet. Par exemple, `../build` est un répertoire **invalide**.
