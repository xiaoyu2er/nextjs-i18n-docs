Cette section de la documentation couvre **les conventions des fichiers de métadonnées**. Les métadonnées basées sur des fichiers peuvent être définies en ajoutant des fichiers de métadonnées spéciaux aux segments de route.

Chaque convention de fichier peut être définie en utilisant un fichier statique (par exemple `opengraph-image.jpg`), ou une variante dynamique qui utilise du code pour générer le fichier (par exemple `opengraph-image.js`).

Une fois qu'un fichier est défini, Next.js servira automatiquement le fichier (avec des hachages en production pour la mise en cache) et mettra à jour les éléments head pertinents avec les métadonnées correctes, telles que l'URL de la ressource, le type de fichier et la taille de l'image.

> **Bon à savoir** :
>
> - Les gestionnaires de route spéciaux comme [`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap), [`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image), et [`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons), ainsi que d'autres [fichiers de métadonnées](/docs/app/api-reference/file-conventions/metadata) sont mis en cache par défaut.
> - Si vous les utilisez avec [`middleware.ts`](/docs/app/api-reference/file-conventions/middleware), [configurez le matcher](/docs/app/building-your-application/routing/middleware#matcher) pour exclure les fichiers de métadonnées.
