Next.js expose certaines options qui vous permettent de contrôler comment le serveur gère la suppression ou la conservation en mémoire des pages construites en développement.

Pour modifier les paramètres par défaut, ouvrez `next.config.js` et ajoutez la configuration `onDemandEntries` :

```js filename="next.config.js"
module.exports = {
  onDemandEntries: {
    // période (en ms) pendant laquelle le serveur conserve les pages dans le buffer
    maxInactiveAge: 25 * 1000,
    // nombre de pages qui doivent être conservées simultanément sans être supprimées
    pagesBufferLength: 2,
  },
}
```
