Next.js fait échouer votre **build de production** (`next build`) lorsque des erreurs TypeScript sont présentes dans votre projet.

Si vous souhaitez que Next.js produise dangereusement du code de production même lorsque votre application contient des erreurs, vous pouvez désactiver l'étape intégrée de vérification des types.

Si cette option est désactivée, assurez-vous d'exécuter les vérifications de type dans le cadre de votre processus de build ou de déploiement, sinon cela peut être très risqué.

Ouvrez `next.config.js` et activez l'option `ignoreBuildErrors` dans la configuration `typescript` :

```js filename="next.config.js"
module.exports = {
  typescript: {
    // !! ATTENTION !!
    // Autorise dangereusement la réussite des builds de production même si
    // votre projet contient des erreurs de type.
    // !! ATTENTION !!
    ignoreBuildErrors: true,
  },
}
```
