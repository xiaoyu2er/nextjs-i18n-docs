La configuration `htmlLimitedBots` vous permet de spécifier une liste d'agents utilisateurs qui doivent recevoir des métadonnées de blocage au lieu de [métadonnées de streaming (streaming metadata)](/docs/app/api-reference/functions/generate-metadata#streaming-metadata).

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const config: NextConfig = {
  htmlLimitedBots: /MySpecialBot|MyAnotherSpecialBot|SimpleCrawler/,
}

export default config
```

```js filename="next.config.js" switcher
module.exports = {
  htmlLimitedBots: /MySpecialBot|MyAnotherSpecialBot|SimpleCrawler/,
}
```

## Liste par défaut

Next.js inclut [une liste par défaut de bots HTML limités (HTML limited bots)](https://github.com/vercel/next.js/blob/canary/packages/next/src/shared/lib/router/utils/html-bots.ts).

La spécification d'une configuration `htmlLimitedBots` remplacera la liste par défaut de Next.js, vous donnant un contrôle total sur les agents utilisateurs qui doivent adopter ce comportement. Cependant, il s'agit d'un comportement avancé, et la valeur par défaut devrait suffire dans la plupart des cas.
