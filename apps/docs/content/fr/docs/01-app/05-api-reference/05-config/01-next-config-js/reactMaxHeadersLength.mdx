title: reactMaxHeadersLength
description: Longueur maximale des en-têtes émis par React et ajoutés à la réponse.
---

Lors du rendu statique, React peut émettre des en-têtes qui peuvent être ajoutés à la réponse. Ceux-ci peuvent être utilisés pour améliorer les performances en permettant au navigateur de précharger des ressources comme des polices, des scripts et des feuilles de style. La valeur par défaut est `6000`, mais vous pouvez remplacer cette valeur en configurant l'option `reactMaxHeadersLength` dans `next.config.js` :

```js filename="next.config.js"
module.exports = {
  reactMaxHeadersLength: 1000,
}
```

> **Bon à savoir** : Cette option n'est disponible que dans le routeur App.

Selon le type de proxy entre le navigateur et le serveur, les en-têtes peuvent être tronqués. Par exemple, si vous utilisez un proxy inverse qui ne prend pas en charge les en-têtes longs, vous devriez définir une valeur plus faible pour vous assurer que les en-têtes ne sont pas tronqués.
