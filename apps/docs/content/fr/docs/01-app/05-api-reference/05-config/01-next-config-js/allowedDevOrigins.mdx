Next.js ne bloque pas automatiquement les requêtes cross-origin pendant le développement, mais bloquera par défaut dans une future version majeure de Next.js pour empêcher les requêtes non autorisées vers des ressources/points de terminaison internes disponibles en mode développement.

Pour configurer une application Next.js afin d'autoriser les requêtes provenant d'origines autres que le nom d'hôte avec lequel le serveur a été initialisé (`localhost` par défaut), vous pouvez utiliser l'option de configuration `allowedDevOrigins`.

`allowedDevOrigins` vous permet de définir des origines supplémentaires pouvant être utilisées en mode développement. Par exemple, pour utiliser `local-origin.dev` au lieu de seulement `localhost`, ouvrez `next.config.js` et ajoutez la configuration `allowedDevOrigins` :

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```
