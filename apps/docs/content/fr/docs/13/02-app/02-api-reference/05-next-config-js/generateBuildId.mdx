Next.js utilise un identifiant constant généré au moment du build pour identifier quelle version de votre application est servie. Cela peut poser problème dans les déploiements multi-serveurs lorsque `next build` est exécuté sur chaque serveur. Pour maintenir un identifiant de build cohérent entre les builds, vous pouvez fournir votre propre identifiant.

Ouvrez `next.config.js` et ajoutez la fonction `generateBuildId` :

```js filename="next.config.js"
module.exports = {
  generateBuildId: async () => {
    // Vous pouvez par exemple récupérer le dernier hash de commit git ici
    return 'my-build-id'
  },
}
```
