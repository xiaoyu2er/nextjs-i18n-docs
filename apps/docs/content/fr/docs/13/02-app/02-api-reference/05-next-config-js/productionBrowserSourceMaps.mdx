Les Source Maps sont activées par défaut pendant le développement. Lors des builds de production, elles sont désactivées pour éviter de divulguer votre code source côté client, sauf si vous optez explicitement pour leur activation via le flag de configuration.

Next.js fournit un flag de configuration que vous pouvez utiliser pour activer la génération de source maps dans le navigateur pendant la build de production :

```js filename="next.config.js"
module.exports = {
  productionBrowserSourceMaps: true,
}
```

Lorsque l'option `productionBrowserSourceMaps` est activée, les source maps seront générées dans le même répertoire que les fichiers JavaScript. Next.js servira automatiquement ces fichiers lorsqu'ils seront demandés.

- L'ajout de source maps peut augmenter le temps de `next build`
- Augmente l'utilisation mémoire pendant `next build`
