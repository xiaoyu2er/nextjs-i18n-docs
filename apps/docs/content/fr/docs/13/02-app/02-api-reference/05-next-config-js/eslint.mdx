Lorsqu'ESLint est détecté dans votre projet, Next.js interrompt votre **build de production** (`next build`) en présence d'erreurs.

Si vous souhaitez que Next.js produise du code pour la production même lorsque votre application contient des erreurs ESLint, vous pouvez désactiver complètement l'étape de linting intégrée. Ceci n'est pas recommandé à moins que vous n'ayez déjà configuré ESLint pour s'exécuter dans une autre partie de votre workflow (par exemple, dans l'intégration continue ou un hook pre-commit).

Ouvrez `next.config.js` et activez l'option `ignoreDuringBuilds` dans la configuration `eslint` :

```js filename="next.config.js"
module.exports = {
  eslint: {
    // Attention : Ceci permet aux builds de production de se terminer avec succès même si
    // votre projet contient des erreurs ESLint.
    ignoreDuringBuilds: true,
  },
}
```
