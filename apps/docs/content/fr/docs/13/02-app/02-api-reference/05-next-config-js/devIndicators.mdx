<AppOnly>

Lorsque vous modifiez votre code et que Next.js compile l'application, un indicateur de compilation apparaît dans le coin inférieur droit de la page.

> **Bon à savoir** : Cet indicateur est uniquement présent en mode développement et n'apparaîtra pas lors de la construction et de l'exécution de l'application en mode production.

Dans certains cas, cet indicateur peut être mal positionné sur votre page, par exemple en conflit avec un lanceur de chat. Pour modifier sa position, ouvrez `next.config.js` et définissez `buildActivityPosition` dans l'objet `devIndicators` avec les valeurs `bottom-right` (par défaut), `bottom-left`, `top-right` ou `top-left` :

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivityPosition: 'bottom-right',
  },
}
```

Dans certains cas, cet indicateur peut ne pas vous être utile. Pour le supprimer, ouvrez `next.config.js` et désactivez la configuration `buildActivity` dans l'objet `devIndicators` :

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    buildActivity: false,
  },
}
```

</AppOnly>

<PagesOnly>

> **Bon à savoir** : Cet indicateur a été supprimé dans la version 10.0.1 de Next.js. Nous recommandons de mettre à jour vers la dernière version de Next.js.

Lorsqu'une page est éligible à l'[Optimisation statique automatique](/docs/pages/building-your-application/rendering/automatic-static-optimization), nous affichons un indicateur pour vous en informer.

Ceci est utile car l'optimisation statique automatique peut être très bénéfique et savoir immédiatement en développement si la page est éligible peut être pratique.

Dans certains cas, cet indicateur peut ne pas être utile, comme lors du travail sur des applications Electron. Pour le supprimer, ouvrez `next.config.js` et désactivez la configuration `autoPrerender` dans `devIndicators` :

```js filename="next.config.js"
module.exports = {
  devIndicators: {
    autoPrerender: false,
  },
}
```

</PagesOnly>
