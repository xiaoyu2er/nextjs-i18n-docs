Cette page vous guidera dans la définition et l'organisation des routes dans votre application Next.js.

## Création de routes

Next.js utilise un routeur basé sur le système de fichiers où les **dossiers** sont utilisés pour définir les routes.

Chaque dossier représente un [segment de route](/docs/app/building-your-application/routing#route-segments) qui correspond à un segment d'**URL**. Pour créer une [route imbriquée](/docs/app/building-your-application/routing#nested-routes), vous pouvez imbriquer des dossiers les uns dans les autres.

Un fichier spécial [`page.js`](/docs/app/building-your-application/routing/pages-and-layouts#pages) est utilisé pour rendre les segments de route accessibles publiquement.

Dans cet exemple, le chemin URL `/dashboard/analytics` n'est _pas_ accessible publiquement car il ne possède pas de fichier `page.js` correspondant. Ce dossier pourrait être utilisé pour stocker des composants, des feuilles de style, des images ou d'autres fichiers colocalisés.

> **Bon à savoir** : Les extensions de fichier `.js`, `.jsx` ou `.tsx` peuvent être utilisées pour les fichiers spéciaux.

## Création d'interface utilisateur

Des [conventions de fichiers spéciaux](/docs/app/building-your-application/routing#file-conventions) sont utilisées pour créer l'interface utilisateur de chaque segment de route. Les plus courantes sont les [pages](/docs/app/building-your-application/routing/pages-and-layouts#pages) pour afficher une interface spécifique à une route, et les [mises en page](/docs/app/building-your-application/routing/pages-and-layouts#layouts) pour afficher une interface partagée entre plusieurs routes.

Par exemple, pour créer votre première page, ajoutez un fichier `page.js` dans le répertoire `app` et exportez un composant React :

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```
