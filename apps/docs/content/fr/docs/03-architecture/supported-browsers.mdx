Next.js prend en charge **les navigateurs modernes** sans configuration requise.

- Chrome 64+
- Edge 79+
- Firefox 67+
- Opera 51+
- Safari 12+

## Browserslist

Si vous souhaitez cibler des navigateurs ou des fonctionnalités spécifiques, Next.js prend en charge la configuration [Browserslist](https://browsersl.ist) dans votre fichier `package.json`. Next.js utilise par défaut la configuration Browserslist suivante :

```json filename="package.json"
{
  "browserslist": [
    "chrome 64",
    "edge 79",
    "firefox 67",
    "opera 51",
    "safari 12"
  ]
}
```

## Polyfills

Nous injectons des [polyfills couramment utilisés](https://github.com/vercel/next.js/blob/canary/packages/next-polyfill-nomodule/src/index.js), notamment :

- [**fetch()**](https://developer.mozilla.org/docs/Web/API/Fetch_API) — Remplace : `whatwg-fetch` et `unfetch`.
- [**URL**](https://developer.mozilla.org/docs/Web/API/URL) — Remplace : le [package `url` (API Node.js)](https://nodejs.org/api/url.html).
- [**Object.assign()**](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Object/assign) — Remplace : `object-assign`, `object.assign` et `core-js/object/assign`.

Si l'une de vos dépendances inclut ces polyfills, ils seront automatiquement éliminés de la build de production pour éviter les doublons.

De plus, pour réduire la taille du bundle, Next.js ne chargera ces polyfills que pour les navigateurs qui en ont besoin. La majorité du trafic web mondial ne téléchargera pas ces polyfills.

### Polyfills personnalisés

Si votre propre code ou des dépendances npm externes nécessitent des fonctionnalités non prises en charge par vos navigateurs cibles (comme IE 11), vous devez ajouter vous-même des polyfills.

Dans ce cas, vous devez ajouter une importation de haut niveau pour le **polyfill spécifique** dont vous avez besoin dans votre [Custom `<App>`](/docs/pages/building-your-application/routing/custom-app) ou dans le composant individuel.

## Fonctionnalités JavaScript

Next.js vous permet d'utiliser les dernières fonctionnalités JavaScript sans configuration. En plus des [fonctionnalités ES6](https://github.com/lukehoban/es6features), Next.js prend également en charge :

- [Async/await](https://github.com/tc39/ecmascript-asyncawait) (ES2017)
- [Propriétés Rest/Spread d'objets](https://github.com/tc39/proposal-object-rest-spread) (ES2018)
- [Importation dynamique `import()`](https://github.com/tc39/proposal-dynamic-import) (ES2020)
- [Chaînage optionnel](https://github.com/tc39/proposal-optional-chaining) (ES2020)
- [Opérateur de coalescence des nuls](https://github.com/tc39/proposal-nullish-coalescing) (ES2020)
- [Champs de classe](https://github.com/tc39/proposal-class-fields) et [propriétés statiques](https://github.com/tc39/proposal-static-class-features) (ES2022)
- et plus encore !
