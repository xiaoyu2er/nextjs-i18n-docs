## Mise à niveau de la version 13 à la 14

Pour mettre à jour vers Next.js version 14, exécutez la commande suivante avec votre gestionnaire de paquets préféré :

```bash filename="Terminal"
npm i next@latest react@latest react-dom@latest eslint-config-next@latest
```

```bash filename="Terminal"
yarn add next@latest react@latest react-dom@latest eslint-config-next@latest
```

```bash filename="Terminal"
pnpm up next react react-dom eslint-config-next --latest
```

```bash filename="Terminal"
bun add next@latest react@latest react-dom@latest eslint-config-next@latest
```

> **Bon à savoir :** Si vous utilisez TypeScript, assurez-vous de également mettre à jour `@types/react` et `@types/react-dom` vers leurs dernières versions.

### Résumé de la v14

- La version minimale de Node.js a été augmentée de 16.14 à 18.17, car la 16.x a atteint sa fin de vie.
- La commande `next export` a été supprimée au profit de la configuration `output: 'export'`. Consultez la [documentation](https://nextjs.org/docs/app/building-your-application/deploying/static-exports) pour plus d'informations.
- L'import `next/server` pour `ImageResponse` a été renommé en `next/og`. Un [codemod est disponible](/docs/app/building-your-application/upgrading/codemods#next-og-import) pour renommer automatiquement et en toute sécurité vos imports.
- Le package `@next/font` a été complètement supprimé au profit du `next/font` intégré. Un [codemod est disponible](/docs/app/building-your-application/upgrading/codemods#built-in-next-font) pour renommer automatiquement et en toute sécurité vos imports.
- La cible WASM pour `next-swc` a été supprimée.
