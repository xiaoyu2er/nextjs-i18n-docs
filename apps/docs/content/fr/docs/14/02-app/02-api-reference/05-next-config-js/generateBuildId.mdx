Next.js génère un identifiant lors de `next build` pour identifier quelle version de votre application est en cours de service. Le même build doit être utilisé et démarré sur plusieurs conteneurs.

Si vous reconstruisez pour chaque étape de votre environnement, vous devrez générer un identifiant de build cohérent à utiliser entre les conteneurs. Utilisez la commande `generateBuildId` dans `next.config.js` :

```jsx filename="next.config.js"
module.exports = {
  generateBuildId: async () => {
    // Cela peut être n'importe quoi, en utilisant par exemple le dernier hash git
    return process.env.GIT_HASH
  },
}
```
