Les source maps (cartes sources) sont activées par défaut pendant le développement. Lors des constructions en production, elles sont désactivées pour éviter de divulguer votre code source côté client, sauf si vous optez explicitement pour leur activation via le flag de configuration.

Next.js fournit un flag de configuration que vous pouvez utiliser pour activer la génération de source maps (cartes sources) pour le navigateur lors de la construction en production :

```js filename="next.config.js"
module.exports = {
  productionBrowserSourceMaps: true,
}
```

Lorsque l'option `productionBrowserSourceMaps` est activée, les source maps (cartes sources) seront générées dans le même répertoire que les fichiers JavaScript. Next.js servira automatiquement ces fichiers lorsqu'ils seront demandés.

- L'ajout de source maps peut augmenter le temps de construction avec `next build`
- Augmente l'utilisation mémoire pendant `next build`
