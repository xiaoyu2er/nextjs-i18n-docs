> **Avertissement** : Le prérendu partiel est une fonctionnalité expérimentale et n'est actuellement **pas adapté aux environnements de production**.

Le prérendu partiel est une fonctionnalité expérimentale qui permet de prérendre les parties statiques d'une route et de les servir depuis le cache, tout en diffusant les parties dynamiques dans un même appel HTTP.

Le prérendu partiel est disponible dans `next@canary` :

```bash filename="Terminal"
npm install next@canary
```

Vous pouvez activer le prérendu partiel en configurant le flag expérimental `ppr` :

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    ppr: true,
  },
}

module.exports = nextConfig
```

> **Bon à savoir :**
>
> - Le prérendu partiel ne s'applique pas encore aux navigations côté client. Nous travaillons activement sur cette fonctionnalité.
> - Le prérendu partiel est conçu uniquement pour le [runtime Node.js](/docs/app/building-your-application/rendering/edge-and-nodejs-runtimes). L'utilisation d'un sous-ensemble du runtime Node.js n'est pas nécessaire lorsque vous pouvez servir instantanément l'enveloppe statique.

Apprenez-en plus sur le prérendu partiel dans le [cours Next.js Learn](/learn/dashboard-app/partial-prerendering).
