[Web Vitals](https://web.dev/vitals/) est une initiative créée par Google pour fournir des directives unifiées et des métriques permettant de mesurer l'expérience utilisateur sur le web.

[Core Web Vitals](https://developers.google.com/search/blog/2020/11/timing-for-page-experience) est un sous-ensemble de Web Vitals, et comprend actuellement trois métriques qui mesurent le chargement, l'interactivité et la stabilité visuelle. Ces métriques sont [Largest Contentful Paint (LCP)](/learn/seo/lcp), [First Input Delay (FID)](/learn/seo/fid) et [Cumulative Layout Shift (CLS)](/learn/seo/cls).

Obtenir un bon score sur ces trois métriques permettra de créer une expérience plus fluide pour vos utilisateurs.

Les sites web ayant de mauvais résultats sur chacune des métriques Core Web Vitals verront leur classement dans les moteurs de recherche impacté, car Google commence à utiliser [Core Web Vitals comme facteur de classement](https://developers.google.com/search/blog/2020/11/timing-for-page-experience) dans son algorithme de recherche. De mauvaises métriques peuvent avoir un impact sur votre trafic web et votre activité.

Dans cette leçon, vous apprendrez :

*   Un bref aperçu de Core Web Vitals.
*   Les implications de Core Web Vitals sur le SEO et l'UX, et son impact sur votre site web.
*   Pourquoi vous devriez vous soucier de Core Web Vitals dans votre processus de développement et comment les mesurer.
*   Comment améliorer vos Core Web Vitals avec Next.js et surveiller les changements.
