Une autre excellente méthode gratuite et open-source pour mesurer vos performances consiste à utiliser le jeu de données du [Rapport d'expérience utilisateur Chrome](https://developers.google.com/web/tools/chrome-user-experience-report).

Le [Rapport d'expérience utilisateur Chrome](https://developers.google.com/web/tools/chrome-user-experience-report) fournit des métriques d'expérience utilisateur montrant comment les utilisateurs réels de Chrome perçoivent les destinations populaires sur le web.

Ce jeu de données est accessible publiquement [via BigQuery](https://console.cloud.google.com/bigquery?project=chrome-ux-report) et vous pouvez également l'utiliser gratuitement avec [Google Data Studio](https://datastudio.google.com/overview).

Heureusement pour vous, il existe [des tableaux de bord open-source disponibles](https://g.co/chromeuxdash) que vous pouvez utiliser comme modèles pour suivre les performances de votre site web.

Le seul inconvénient de ce jeu de données est que, pour que votre site soit inclus dans le rapport CrUX, il doit avoir un nombre significatif de visites, sinon il ne sera pas inclus dans le rapport en raison d'un manque de données. Par conséquent, ce n'est peut-être pas la meilleure option pour les sites en cours de développement ou récemment créés.

De plus, les données sont mises à jour mensuellement. Généralement environ 15 jours après la fin du mois, vous verrez donc les données avec un certain délai, ce qui pourrait ne pas être très pratique si vous prévoyez d'améliorer vos scores Core Web Vitals.
