L'objectif principal du moteur de recherche de Google est de fournir aux utilisateurs les meilleurs résultats possibles tout en tenant compte de la localisation et des fautes de frappe. Dans tous les cas, Google s'efforce de fournir aux utilisateurs un résultat rapide et fluide avec une excellente expérience de site web. La [vitesse des pages web](https://developers.google.com/web/updates/2018/07/search-ads-speed) sur les appareils mobiles est un facteur de classement depuis 2018. Cependant, jusqu'à présent, il n'était pas clair quelles étaient les métriques de performance spécifiques utilisées par l'algorithme de recherche Google dans le cadre du classement.

Cela a changé en juin 2021, lorsque Google a fourni [un ensemble de métriques et de plages spécifiques](https://developers.google.com/search/blog/2021/04/more-details-page-experience) pour analyser et optimiser vos performances.

![Détails des signaux d'expérience de recherche pour la mise à jour de l'algorithme Google de juin 2021.](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/pages-router/seo/page-experience.png)

### [Pondérations Lighthouse (v6) pour les Web Vitals](#lighthouse-v6-weights-for-vitals)

Les trois métriques ne sont pas nécessairement valorisées de manière égale. Dans [Lighthouse](https://developers.google.com/web/tools/lighthouse), différents poids sont attribués à chacune des Core Web Vitals :

*   **Largest Contentful Paint** : 25 %
*   **Total Blocking Time** : 25 %
*   **First Contentful Paint** : 15 %
*   **Speed Index** : 15 %
*   **Time to Interactive** : 15 %
*   **Cumulative Layout Shift** : 5 %

Cela est similaire au [First Input Delay](/learn/seo/fid).

**Remarque :** L'impact sur le classement Google sera le même pour toutes les pages dans la bonne plage pour toutes les Core Web Vitals, quel que soit leur score individuel.

### [Impact sur l'expérience utilisateur](#ux-impact)

La plupart des conversations autour des Core Web Vitals se concentrent principalement sur le SEO.

Bien qu'il soit vrai que les Core Web Vitals sont une initiative conçue pour mesurer et pousser à l'amélioration de l'expérience de page et du classement dans les résultats de recherche, ce sont les utilisateurs qui bénéficient ultimement de ces changements.

Les Core Web Vitals aident à viser la meilleure expérience de page possible. Selon une [étude réalisée par Amazon en 2012](https://www.fastcompany.com/1825005/how-one-second-could-cost-amazon-16-billion-sales), une seconde supplémentaire de temps de chargement pourrait coûter à l'entreprise 1,6 milliard de dollars. Des études comme celle-ci mettent en lumière l'importance d'une excellente expérience de page et d'un site web rapide, deux aspects que les Core Web Vitals aident à atteindre.
