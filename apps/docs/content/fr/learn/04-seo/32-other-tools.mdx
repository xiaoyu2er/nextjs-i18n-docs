Vous pouvez trouver la collecte de données terrain dans les outils suivants :

*   [PageSpeed Insights](https://developers.google.com/speed/pagespeed/insights/) : L'outil de mesure de vitesse de page de Google.
*   [Chrome User Experience Report](https://developers.google.com/web/tools/chrome-user-experience-report) : Jeu de données open-source sur l'expérience utilisateur.
*   [Search Console](https://support.google.com/webmasters/answer/9205520) : Google Search, rapport Core Web Vitals.

Si vous recherchez des données de laboratoire, Google propose également plusieurs outils de mesure :

*   [Lighthouse](https://developers.google.com/web/tools/lighthouse?hl=en) : Outil open-source et automatisé de Google pour améliorer la qualité des pages web.
*   [Chrome DevTools](https://developer.chrome.com/docs/devtools/) : Ensemble d'outils pour développeurs intégrés directement dans le navigateur Google Chrome.

Notez que pour ces deux outils, vous devrez utiliser le [Temps de blocage total (TBT)](https://web.dev/tbt/) comme alternative au [Délai de première entrée (FID)](https://web.dev/fid/), car le FID ne peut être mesuré qu'à travers des données terrain.
