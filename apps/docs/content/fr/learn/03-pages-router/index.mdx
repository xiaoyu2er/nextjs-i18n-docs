Pour construire une application web complète avec React à partir de zéro, il y a de nombreux détails importants à prendre en compte :

*   Le code doit être regroupé à l'aide d'un bundler comme webpack et transformé à l'aide d'un compilateur comme Babel.
*   Vous devez effectuer des optimisations de production telles que le découpage du code (code splitting).
*   Vous pourriez vouloir pré-rendre statiquement certaines pages pour des raisons de performance et de SEO. Vous pourriez aussi vouloir utiliser le rendu côté serveur (SSR) ou le rendu côté client (CSR).
*   Vous pourriez avoir à écrire du code côté serveur pour connecter votre application React à votre base de données.

Un _framework_ peut résoudre ces problèmes. Mais un tel framework doit avoir le bon niveau d'abstraction — sinon il ne sera pas très utile. Il doit aussi offrir une excellente "Expérience Développeur", garantissant que vous et votre équipe ayez une expérience exceptionnelle lors de l'écriture du code.

### [Next.js : Le Framework React](#nextjs-the-react-framework)

Voici **Next.js**, le Framework React. Next.js fournit une solution à tous les problèmes mentionnés ci-dessus. Mais plus important encore, il vous place, vous et votre équipe, dans la [zone de succès](https://blog.codinghorror.com/falling-into-the-pit-of-success/) lors de la construction d'applications React.

Next.js vise à offrir une expérience développeur de premier ordre et de nombreuses fonctionnalités intégrées, telles que :

*   Un système de [routage basé sur les pages](/docs/basic-features/pages) intuitif (avec prise en charge des [routes dynamiques](/docs/routing/dynamic-routes))
*   [Pré-rendu](/docs/basic-features/pages#pre-rendering), avec prise en charge à la fois de la [génération statique](/docs/basic-features/pages#static-generation-recommended) (SSG) et du [rendu côté serveur](/docs/basic-features/pages#server-side-rendering) (SSR) page par page
*   Découpage automatique du code pour des chargements de page plus rapides
*   [Routage côté client](/docs/routing/introduction#linking-between-pages) avec préchargement optimisé
*   [Prise en charge intégrée du CSS](/docs/basic-features/built-in-css-support) et [de Sass](/docs/basic-features/built-in-css-support#sass-support), ainsi que prise en charge de n'importe quelle bibliothèque [CSS-in-JS](/docs/basic-features/built-in-css-support#css-in-js)
*   Environnement de développement avec prise en charge du [Rafraîchissement Instantané](/docs/basic-features/fast-refresh)
*   [Routes API](/docs/api-routes/introduction) pour construire des points d'API avec des fonctions Serverless
*   Entièrement extensible

Next.js est utilisé dans des dizaines de milliers de sites web et applications web en production, y compris parmi les plus grandes marques mondiales.
