La manière la plus simple de déployer Next.js en production est d'utiliser la plateforme **[Vercel](https://vercel.com)** développée par les créateurs de Next.js.

Vercel est une plateforme serverless pour les applications statiques et hybrides conçue pour s'intégrer à votre contenu headless, votre commerce ou votre base de données. Nous facilitons le développement, la prévisualisation et le déploiement d'expériences utilisateur exceptionnelles pour les équipes frontend, où la performance est la norme. Vous pouvez commencer à l'utiliser gratuitement — sans carte de crédit.

### [Créer un compte Vercel](#create-a-vercel-account)

Commencez par vous rendre sur [Vercel](https://vercel.com/signup) pour créer un compte. Choisissez **Continuer avec GitHub** et suivez le processus d'inscription.

### [Importer votre dépôt `nextjs-blog`](#import-your-nextjs-blog-repository)

Une fois inscrit, **importez** votre dépôt `nextjs-blog` sur Vercel. Vous pouvez le faire depuis cette page : [Vercel](https://vercel.com/import).

*   Vous devrez **Installer Vercel pour GitHub**. Vous pouvez lui donner accès à **Tous les dépôts**.
*   Après avoir installé Vercel, importez `nextjs-blog`.

Vous pouvez utiliser les _valeurs par défaut_ pour les paramètres suivants — aucun changement n'est nécessaire. Vercel détecte automatiquement que vous avez une application Next.js et choisit les paramètres de build optimaux pour vous.

*   Nom du projet
*   Répertoire racine
*   Commande de build
*   Répertoire de sortie
*   Commande de développement

Lors du déploiement, votre application Next.js commencera à se construire. Cela devrait prendre moins d'une minute.

> **De l'aide est disponible** : Si votre déploiement échoue, vous pouvez toujours obtenir de l'aide sur [GitHub Discussions](https://github.com/vercel/next.js/discussions). Pour en savoir plus sur le déploiement, consultez [notre documentation](/docs/deployment).

Une fois terminé, vous obtiendrez **des URL de déploiement**. Cliquez sur l'une des URL et vous devriez voir la page de démarrage Next.js en ligne.

Félicitations ! Vous venez de déployer votre application Next.js en production. Dans la page suivante, nous aborderons en détail Vercel et le workflow recommandé.
