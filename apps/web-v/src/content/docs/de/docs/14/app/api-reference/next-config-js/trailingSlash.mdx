Standardmäßig leitet Next.js URLs mit abschließendem Schrägstrich zu ihren Entsprechungen ohne Schrägstrich um. Zum Beispiel wird `/about/` zu `/about` weitergeleitet. Sie können dieses Verhalten so konfigurieren, dass URLs ohne Schrägstrich zu ihren Entsprechungen mit Schrägstrich umgeleitet werden.

Öffnen Sie `next.config.js` und fügen Sie die `trailingSlash`-Konfiguration hinzu:

```js filename="next.config.js"
module.exports = {
  trailingSlash: true,
}
```

Mit dieser Option werden URLs wie `/about` zu `/about/` weitergeleitet.

Bei Verwendung mit der [`output: "export"`](/docs/14/app/building-your-application/deploying/static-exports)-Konfiguration gibt die Seite `/about` die Datei `/about/index.html` aus (anstelle der Standarddatei `/about.html`).

| -------- | ------------------------- |
| `v9.5.0` | `trailingSlash` hinzugefügt. |
