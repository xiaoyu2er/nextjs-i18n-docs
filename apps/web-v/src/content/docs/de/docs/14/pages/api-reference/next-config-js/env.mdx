> Seit der Veröffentlichung von [Next.js 9.4](https://nextjs.org/blog/next-9-4) bieten wir ein intuitiveres und ergonomischeres Erlebnis für das [Hinzufügen von Umgebungsvariablen](/docs/14/pages/building-your-application/configuring/environment-variables). Probieren Sie es aus!

</PagesOnly>

<AppOnly>

> **Wissenswert**: Auf diese Weise angegebene Umgebungsvariablen werden **immer** im JavaScript-Bundle enthalten sein. Das Präfix `NEXT_PUBLIC_` vor dem Variablennamen hat nur dann eine Wirkung, wenn die Variablen [über die Umgebung oder .env-Dateien](/docs/14/app/building-your-application/configuring/environment-variables) angegeben werden.

</AppOnly>

<PagesOnly>

> **Wissenswert**: Auf diese Weise angegebene Umgebungsvariablen werden **immer** im JavaScript-Bundle enthalten sein. Das Präfix `NEXT_PUBLIC_` vor dem Variablennamen hat nur dann eine Wirkung, wenn die Variablen [über die Umgebung oder .env-Dateien](/docs/14/pages/building-your-application/configuring/environment-variables) angegeben werden.

</PagesOnly>

Um Umgebungsvariablen zum JavaScript-Bundle hinzuzufügen, öffnen Sie `next.config.js` und fügen Sie die `env`-Konfiguration hinzu:

```js filename="next.config.js"
module.exports = {
  env: {
    customKey: 'my-value',
  },
}
```

Jetzt können Sie in Ihrem Code auf `process.env.customKey` zugreifen. Zum Beispiel:

```jsx
function Page() {
  return <h1>Der Wert von customKey ist: {process.env.customKey}</h1>
}

export default Page
```

Next.js ersetzt `process.env.customKey` zur Build-Zeit durch `'my-value'`. Der Versuch, `process.env`-Variablen zu destrukturieren, funktioniert aufgrund der Natur des Webpack [DefinePlugin](https://webpack.js.org/plugins/define-plugin/) nicht.

Die folgende Zeile:

```jsx
return <h1>Der Wert von customKey ist: {process.env.customKey}</h1>
```

Wird letztendlich zu:

```jsx
return <h1>Der Wert von customKey ist: {'my-value'}</h1>
```
