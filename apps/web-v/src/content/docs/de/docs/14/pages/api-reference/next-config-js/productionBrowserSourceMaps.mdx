Source-Maps sind standardmäßig während der Entwicklung aktiviert. Während Produktions-Builds sind sie deaktiviert, um zu verhindern, dass Sie Ihren Quellcode auf dem Client preisgeben, es sei denn, Sie aktivieren sie explizit mit dem Konfigurations-Flag.

Next.js bietet ein Konfigurations-Flag, mit dem Sie die Generierung von Browser-Source-Maps während des Produktions-Builds aktivieren können:

```js filename="next.config.js"
module.exports = {
  productionBrowserSourceMaps: true,
}
```

Wenn die Option `productionBrowserSourceMaps` aktiviert ist, werden die Source-Maps im selben Verzeichnis wie die JavaScript-Dateien ausgegeben. Next.js serviert diese Dateien automatisch, wenn sie angefordert werden.

- Das Hinzufügen von Source-Maps kann die Build-Zeit von `next build` erhöhen
- Erhöht die Speichernutzung während `next build`
