عند اكتشاف ESLint في مشروعك، يفشل Next.js في **بناء الإنتاج** (`next build`) عند وجود أخطاء.

إذا كنت ترغب في أن يقوم Next.js بإنتاج كود الإنتاج حتى عندما يحتوي تطبيقك على أخطاء ESLint، يمكنك تعطيل خطوة الفحص المضمنة تمامًا. لا يُنصح بهذا إلا إذا كنت قد قمت بالفعل بتكوين ESLint ليعمل في جزء منفصل من سير العمل الخاص بك (على سبيل المثال، في CI أو خطاف ما قبل الالتزام).

افتح `next.config.js` وقم بتمكين خيار `ignoreDuringBuilds` في تكوين `eslint`:

```js filename="next.config.js"
module.exports = {
  eslint: {
    // تحذير: هذا يسمح لعمليات بناء الإنتاج بالإكمال بنجاح حتى إذا
    // كان مشروعك يحتوي على أخطاء ESLint.
    ignoreDuringBuilds: true,
  },
}
```
