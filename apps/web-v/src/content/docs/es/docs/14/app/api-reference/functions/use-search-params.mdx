`useSearchParams` es un hook de **Componente Cliente** que permite leer la **cadena de consulta (query string)** de la URL actual.

`useSearchParams` devuelve una versiĂłn **de solo lectura** de la interfaz [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams).

```tsx filename="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // URL -> `/dashboard?search=my-project`
  // `search` -> 'my-project'
  return <>Buscar: {search}</>
}
```


```tsx

```

## Retorno

`useSearchParams` devuelve una versiĂłn **de solo lectura** de la interfaz [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams), que incluye mĂ©todos Ăştiles para leer la cadena de consulta de la URL:

- [`URLSearchParams.get()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/get): Devuelve el primer valor asociado al parĂˇmetro de bĂşsqueda. Por ejemplo:

  | URL                  | `searchParams.get("a")`                                                                                         |
  | -------------------- | --------------------------------------------------------------------------------------------------------------- |
  | `/dashboard?a=1`     | `'1'`                                                                                                           |
  | `/dashboard?a=`      | `''`                                                                                                            |
  | `/dashboard?b=3`     | `null`                                                                                                          |
  | `/dashboard?a=1&a=2` | `'1'` _- usa [`getAll()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/getAll) para todos los valores_ |

- [`URLSearchParams.has()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/has): Devuelve un valor booleano indicando si existe el parĂˇmetro. Por ejemplo:

  | URL              | `searchParams.has("a")` |
  | ---------------- | ----------------------- |
  | `/dashboard?a=1` | `true`                  |
  | `/dashboard?b=3` | `false`                 |

- MĂˇs informaciĂłn sobre otros mĂ©todos **de solo lectura** de [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams), incluyendo [`getAll()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/getAll), [`keys()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/keys), [`values()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/values), [`entries()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/entries), [`forEach()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/forEach), y [`toString()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/toString).

> **Importante**:
>
> - `useSearchParams` es un hook de [Componente Cliente](/docs/14/app/building-your-application/rendering/client-components) y **no es compatible** con [Componentes Servidor](/docs/14/app/building-your-application/rendering/server-components) para evitar valores obsoletos durante el [renderizado parcial](/docs/14/app/building-your-application/routing/linking-and-navigating#4-partial-rendering).
> - Si una aplicaciĂłn incluye el directorio `/pages`, `useSearchParams` devolverĂˇ `ReadonlyURLSearchParams | null`. El valor `null` es para compatibilidad durante la migraciĂłn, ya que los parĂˇmetros de bĂşsqueda no pueden conocerse durante el pre-renderizado de una pĂˇgina que no usa `getServerSideProps`.

### Renderizado estĂˇtico

Esto permite que parte de la ruta se renderice estĂˇticamente mientras la parte dinĂˇmica que usa `useSearchParams` se renderiza en el cliente.

Recomendamos envolver el Componente Cliente que usa `useSearchParams` en un lĂ­mite `<Suspense/>`. Esto permitirĂˇ que los Componentes Cliente superiores se rendericen estĂˇticamente y se envĂ­en como parte del HTML inicial. [Ejemplo](/docs/14/app/api-reference/functions/use-search-params#static-rendering).

Por ejemplo:

```tsx filename="app/dashboard/search-bar.tsx" switcher

  const searchParams = useSearchParams()

  const search = searchParams.get('search')
```

### Componentes Servidor

#### PĂˇginas

Para acceder a los parĂˇmetros de bĂşsqueda en [PĂˇginas](/docs/14/app/api-reference/file-conventions/page) (Componentes Servidor), usa la prop [`searchParams`](/docs/14/app/api-reference/file-conventions/page#searchparams-optional).

#### Layouts

A diferencia de las PĂˇginas, los [Layouts](/docs/14/app/api-reference/file-conventions/layout) (Componentes Servidor) **no reciben** la prop `searchParams`. Esto se debe a que un layout compartido [no se vuelve a renderizar durante la navegaciĂłn](/docs/14/app/building-your-application/routing/linking-and-navigating#4-partial-rendering), lo que podrĂ­a generar `searchParams` obsoletos entre navegaciones. Ver [explicaciĂłn detallada](/docs/14/app/api-reference/file-conventions/layout#layouts-do-not-receive-searchparams).

En su lugar, usa la prop [`searchParams`](/docs/14/app/api-reference/file-conventions/page) de la PĂˇgina o el hook [`useSearchParams`](/docs/14/app/api-reference/functions/use-search-params) en un Componente Cliente, que se vuelve a renderizar en el cliente con los `searchParams` mĂˇs recientes.

## Ejemplos

### Actualizando `searchParams`

Puedes usar [`useRouter`](/docs/14/app/api-reference/functions/use-router) o [`Link`](/docs/14/app/api-reference/components/link) para establecer nuevos `searchParams`. DespuĂ©s de una navegaciĂłn, la [`page.js`](/docs/14/app/api-reference/file-conventions/page) actual recibirĂˇ una prop [`searchParams`](/docs/14/app/api-reference/file-conventions/page#searchparams-optional) actualizada.

```tsx filename="app/example-client-component.tsx" switcher
export default function ExampleClientComponent() {
  const router = useRouter()
  const pathname = usePathname()
  const searchParams = useSearchParams()

  // ObtĂ©n una nueva cadena searchParams fusionando los
  // searchParams actuales con un par clave/valor proporcionado
  const createQueryString = useCallback(
    (name: string, value: string) => {
      const params = new URLSearchParams(searchParams.toString())
      params.set(name, value)

      return params.toString()
    },
    [searchParams]
  )

  return (
    <>
      <p>Ordenar por</p>

      
      <button
        onClick={() => {
          // <pathname>?sort=asc
          router.push(pathname + '?' + createQueryString('sort', 'asc'))
        }}
      >
        ASC
      </button>

      
      <Link
        href={
          // <pathname>?sort=desc
          pathname + '?' + createQueryString('sort', 'desc')
        }
      >
        DESC
      </Link>
    </>
  )
}
```
