{/* md5:2c859097c2820c91cf44c9fb74b7bc27 */}
Next.js 編譯器採用 Rust 編寫並使用 [SWC](https://swc.rs/)，能為生產環境轉換和壓縮 JavaScript 程式碼。這取代了原本用於單一檔案的 Babel 和用於壓縮輸出包的 Terser。

{/* md5:ae10600e4962d260ba10eca63ef8253e */}
自 Next.js 12 版起預設啟用此編譯器，其編譯速度比 Babel 快 17 倍。若您現有 Babel 配置或使用[不支援的功能](#不支援的功能)，應用程式將自動切回使用 Babel。

{/* md5:9aae3160708ba98a215c4b67287a8d8f */}
## 為何選擇 SWC？

{/* md5:e224441823cc9fe1a38e79a70a2938c8 */}
[SWC](https://swc.rs/) 是一個基於 Rust 的可擴展平台，專為新一代快速開發工具而設計。

{/* md5:bc662488f8d2a5424af402458470836c */}
SWC 可用於編譯、壓縮、打包等，並支援擴展功能。您可呼叫它來執行程式碼轉換（內建或自訂）。這些轉換透過 Next.js 等高階工具執行。

{/* md5:cd91eb130ab9f8797d10249bb147e5e8 */}
選擇 SWC 的原因如下：

{/* md5:54b45ec36321e4183171e7a293c7d406 */}
- **可擴展性**：SWC 可作為 Crate 整合至 Next.js，無需分叉函式庫或設計變通方案。
- **效能**：切換至 SWC 後，Next.js 的快速重新整理速度提升約 3 倍，建置速度提升約 5 倍，且仍有優化空間。
- **WebAssembly**：Rust 對 WASM 的支援，讓 Next.js 能跨平台開發。
- **社群**：Rust 社群和生態系統蓬勃發展。

{/* md5:67d4ff6d268b6a4cc6f4143253ba9131 */}
## 支援的功能

{/* md5:b873f4b566d727465ce8782fe75edd76 */}
### 樣式化元件 (Styled Components)

{/* md5:e87de51fbd801f8b56c67c0ce1f72dbf */}
我們正將 `babel-plugin-styled-components` 移植至 Next.js 編譯器。

{/* md5:55dc169933d12e5fc0d103fa08d4c6fe */}
首先更新至最新版 Next.js：`npm install next@latest`。接著更新 `next.config.js` 檔案：

```js filename="next.config.js"
module.exports = {
  compiler: {
    styledComponents: true,
  },
}
```

{/* md5:d21d857edb0fc0d6d437226e436dbcbb */}
進階使用可配置樣式化元件的個別屬性。

{/* md5:fa41ddf1946bbeae79ba68c2076c31ec */}
> 注意：在 Next.js 中使用 `styled-components` 主要需 `ssr` 和 `displayName` 轉換。

```js filename="next.config.js"
module.exports = {
  compiler: {
    // see https://styled-components.com/docs/tooling#babel-plugin for more info on the options.
    styledComponents: {
      // Enabled by default in development, disabled in production to reduce file size,
      // setting this will override the default for all environments.
      displayName?: boolean,
      // Enabled by default.
      ssr?: boolean,
      // Enabled by default.
      fileName?: boolean,
      // Empty by default.
      topLevelImportPaths?: string[],
      // Defaults to ["index"].
      meaninglessFileNames?: string[],
      // Enabled by default.
      minify?: boolean,
      // Enabled by default.
      transpileTemplateLiterals?: boolean,
      // Empty by default.
      namespace?: string,
      // Disabled by default.
      pure?: boolean,
      // Enabled by default.
      cssProp?: boolean,
    },
  },
}
```

### Jest

{/* md5:badb4f0699fce14e3587bc31e40ab2bf */}
Next.js 編譯器會轉譯測試程式碼，並簡化與 Next.js 整合的 Jest 配置，包括：

{/* md5:01f896d8907f317d9da89b70052a4c52 */}
- 自動模擬 `.css`、`.module.css`（及其 `.scss` 變體）和圖片匯入
- 使用 SWC 自動設定 `transform`
- 將 `.env`（及所有變體）載入 `process.env`
- 忽略 `node_modules` 的測試解析和轉換
- 忽略 `.next` 的測試解析
- 載入 `next.config.js` 以啟用實驗性 SWC 轉換

{/* md5:c4559bc0ff7a23aa1a819c041bf996a2 */}
首先更新至最新版 Next.js：`npm install next@latest`。接著更新 `jest.config.js` 檔案：

```js filename="jest.config.js"
const nextJest = require('next/jest')

// Providing the path to your Next.js app which will enable loading next.config.js and .env files
const createJestConfig = nextJest({ dir: './' })

// Any custom config you want to pass to Jest
const customJestConfig = {
  setupFilesAfterEnv: ['<rootDir>/jest.setup.js'],
}

// createJestConfig is exported in this way to ensure that next/jest can load the Next.js configuration, which is async
module.exports = createJestConfig(customJestConfig)
```

### Relay

{/* md5:5dc009b5a7354b3b3c221103dd3b3114 */}
啟用 [Relay](https://relay.dev/) 支援：

```js filename="next.config.js"
module.exports = {
  compiler: {
    relay: {
      // This should match relay.config.js
      src: './',
      artifactDirectory: './__generated__',
      language: 'typescript',
      eagerEsModules: false,
    },
  },
}
```

{/* md5:10b130febc4eabb58e45648f53f2755a */}
> **須知**：在 Next.js 中，`pages` 目錄下所有 JavaScript 檔案均視為路由。因此 `relay-compiler` 需指定 `artifactDirectory` 配置於 `pages` 外，否則產生的檔案會被視為路由而導致生產建置失敗。

{/* md5:8a0efbcda828d9a8318f941fcc1ffef1 */}
### 移除 React 屬性

{/* md5:45d72338666294d9aa2071eac228ee9f */}
可移除 JSX 屬性，常用於測試。類似 `babel-plugin-react-remove-properties`。

{/* md5:0041d54e9577cab42af44ea00b2f0187 */}
移除符合預設正則式 `^data-test` 的屬性：

```js filename="next.config.js"
module.exports = {
  compiler: {
    reactRemoveProperties: true,
  },
}
```

{/* md5:6c66cea3174a801c6e538352c4e07a1c */}
移除自訂屬性：

```js filename="next.config.js"
module.exports = {
  compiler: {
    // The regexes defined here are processed in Rust so the syntax is different from
    // JavaScript `RegExp`s. See https://docs.rs/regex.
    reactRemoveProperties: { properties: ['^data-custom$'] },
  },
}
```

{/* md5:98411d1b88bfd0901a6a81425d3e9723 */}
### 移除 Console

{/* md5:a28f6071d0e13e15d893133f364b8286 */}
此轉換可移除應用程式碼中所有 `console.*` 呼叫（不包括 `node_modules`）。類似 `babel-plugin-transform-remove-console`。

{/* md5:b842e87e11862f0942c024ee9270d3cd */}
移除所有 `console.*` 呼叫：

```js filename="next.config.js"
module.exports = {
  compiler: {
    removeConsole: true,
  },
}
```

{/* md5:98a1c96035856a3ce30d51f57f11b527 */}
僅保留 `console.error`：

```js filename="next.config.js"
module.exports = {
  compiler: {
    removeConsole: {
      exclude: ['error'],
    },
  },
}
```

{/* md5:82717a186dfc56c79ef1410fef7ba328 */}
### 傳統裝飾器 (Legacy Decorators)

{/* md5:a51e26ea09590900855836ce020882ea */}
Next.js 會自動偵測 `jsconfig.json` 或 `tsconfig.json` 中的 `experimentalDecorators`。傳統裝飾器常用於舊版函式庫如 `mobx`。

{/* md5:bf0abc8fec0c48588f26a4183597c9f9 */}
此標記僅為相容現有應用，不建議在新專案中使用。

{/* md5:55df77d872889c885e6dc6ecc9338383 */}
首先更新至最新版 Next.js：`npm install next@latest`。接著更新設定檔：

```js
{
  "compilerOptions": {
    "experimentalDecorators": true
  }
}
```

{/* md5:edd3f88647c66141d0119495e585191f */}
### importSource

{/* md5:93ef793708f3baac67d2ac5dd54af5b8 */}
Next.js 會自動偵測 `jsconfig.json` 或 `tsconfig.json` 中的 `jsxImportSource` 並套用。常用於 [Theme UI](https://theme-ui.com) 等函式庫。

{/* md5:55df77d872889c885e6dc6ecc9338383 */}
首先更新至最新版 Next.js：`npm install next@latest`。接著更新設定檔：

```js
{
  "compilerOptions": {
    "jsxImportSource": "theme-ui"
  }
}
```

### Emotion

{/* md5:06e33c730d0434865252277e71a19486 */}
我們正將 `@emotion/babel-plugin` 移植至 Next.js 編譯器。

{/* md5:55dc169933d12e5fc0d103fa08d4c6fe */}
首先更新至最新版 Next.js：`npm install next@latest`。接著更新 `next.config.js` 檔案：

```js filename="next.config.js"

module.exports = {
  compiler: {
    emotion: boolean | {
      // default is true. It will be disabled when build type is production.
      sourceMap?: boolean,
      // default is 'dev-only'.
      autoLabel?: 'never' | 'dev-only' | 'always',
      // default is '[local]'.
      // Allowed values: `[local]` `[filename]` and `[dirname]`
      // This option only works when autoLabel is set to 'dev-only' or 'always'.
      // It allows you to define the format of the resulting label.
      // The format is defined via string where variable parts are enclosed in square brackets [].
      // For example labelFormat: "my-classname--[local]", where [local] will be replaced with the name of the variable the result is assigned to.
      labelFormat?: string,
      // default is undefined.
      // This option allows you to tell the compiler what imports it should
      // look at to determine what it should transform so if you re-export
      // Emotion's exports, you can still use transforms.
      importMap?: {
        [packageName: string]: {
          [exportName: string]: {
            canonicalImport?: [string, string],
            styledBaseImport?: [string, string],
          }
        }
      },
    },
  },
}
```

### 模組轉譯

{/* md5:aa8db8098e8a050f1757296d9219a2bc */}
Next.js 可自動轉譯並打包本地套件（如 monorepo）或外部依賴（`node_modules`）的模組。這取代了 `next-transpile-modules` 套件。

```js filename="next.config.js"
module.exports = {
  transpilePackages: ['@acme/ui', 'lodash-es'],
}
```

### Define（建置時替換變數）

{/* md5:e1337f5cd4fbd01f67275eba3821418a */}
`define` 選項允許在建置時靜態替換程式碼中的變數。
該選項接受鍵值對物件，鍵為需替換的變數，值為替換內容。

{/* md5:bd4dc021a6c0b2b3767cf5a7951276d7 */}
在 `next.config.js` 中使用 `compiler.define` 為所有環境（伺服器、edge 和客戶端）定義變數。或使用 `compiler.defineServer` 僅為伺服器端程式碼定義變數：

```js filename="next.config.js"
module.exports = {
  compiler: {
    define: {
      MY_VARIABLE: 'my-string',
      'process.env.MY_ENV_VAR': 'my-env-var',
    },
    defineServer: {
      MY_SERVER_VARIABLE: 'my-server-var',
    },
  },
}
```

{/* md5:1a0e3569af6f3a9cc099dff85190d12c */}
### 建置生命週期鉤子 (Build Lifecycle Hooks)

{/* md5:98ab7a42893f2060fbba84ec646ff505 */}
Next.js 編譯器支援生命週期鉤子，允許在建置過程特定階段執行自訂程式碼。目前支援以下鉤子：

{/* md5:8391e355aaa8ff95a12cc9ab02b2a3e9 */}
#### runAfterProductionCompile

{/* md5:31be69c4ccc7a32ba12bf1a8ffdeb146 */}
此鉤子函式在生產建置編譯完成後執行，但在執行類型檢查和靜態頁面產生等後編譯任務之前。該鉤子提供專案元資料存取，包括專案目錄和建置輸出目錄，適合第三方工具收集如 sourcemaps 等建置輸出。

```js filename="next.config.js"
module.exports = {
  compiler: {
    runAfterProductionCompile: async ({ distDir, projectDir }) => {
      // Your custom code here
    },
  },
}
```

{/* md5:0e652f71be038855d5f6b159fed0072e */}
鉤子接收包含以下屬性的物件：

{/* md5:c8582dcadeae7da63849ecc6362fb4bf */}
- `distDir`：建置輸出目錄（預設為 `.next`）
- `projectDir`：專案根目錄

{/* md5:934a45fa4a1c1a296b381e5529e32a6b */}
## 實驗性功能

{/* md5:2a2c1ccf72e5ae55145d34bd447e9300 */}
### SWC 追蹤分析 (SWC Trace profiling)

{/* md5:d53a35698f264a8c742ed71ecc975b66 */}
可產生 SWC 內部轉換追蹤，格式為 Chromium 的 [trace event format](https://docs.google.com/document/d/1CvAClvFfyA5R-PhYUmn5OOQtYMH4h6I0nSsKchNAySU/preview?mode=html#%21=)。

```js filename="next.config.js"
module.exports = {
  experimental: {
    swcTraceProfiling: true,
  },
}
```

{/* md5:f52260e780a2adfc7f22e00135c1c107 */}
啟用後，SWC 會在 `.next/` 下產生名為 `swc-trace-profile-${timestamp}.json` 的追蹤檔案。可使用 Chromium 的追蹤檢視器（chrome://tracing/、https://ui.perfetto.dev/）或相容的火焰圖檢視器（https://www.speedscope.app/）載入並視覺化追蹤結果。

{/* md5:f43f0bc0d85aced5af46565ee4baa7e7 */}
### SWC 插件（實驗性）

{/* md5:6ded96b8a1d895347e125fd549ad7f85 */}
可配置 SWC 轉換以使用實驗性的 WASM 插件支援來自訂轉換行為。

```js filename="next.config.js"
module.exports = {
  experimental: {
    swcPlugins: [
      [
        'plugin',
        {
          ...pluginOptions,
        },
      ],
    ],
  },
}
```

{/* md5:8f3f7dc11735a43f67e4bfd749bec3ba */}
`swcPlugins` 接受由元組組成的陣列來配置插件。每個插件元組包含插件路徑和配置物件。插件路徑可以是 npm 模組名稱或 `.wasm` 二進位檔案的絕對路徑。

{/* md5:624f4348d31228db3defd0a919f8c0bc */}
## 不支援的功能

{/* md5:48e669e3ab4deeea930a326f19cc7e90 */}
當應用程式存在 `.babelrc` 檔案時，Next.js 會自動切回使用 Babel 轉換單一檔案。這確保了與現有使用自訂 Babel 插件的應用程式相容。

{/* md5:3ccdfd5bdd8b7c5c453aafde04fac393 */}
若您使用自訂 Babel 配置，[請分享您的設定](https://github.com/vercel/next.js/discussions/30174)。我們正努力移植常見的 Babel 轉換，並在未來支援插件功能。
