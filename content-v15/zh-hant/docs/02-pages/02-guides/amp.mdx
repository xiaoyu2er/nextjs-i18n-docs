> **警告**：內建的 AMP 支援將在 Next.js 16 中移除。

使用 Next.js，您可以使用少量配置，無需離開 React，即可將任何 React 頁面轉換為 AMP 頁面。

您可以在官方的 [amp.dev](https://amp.dev/) 網站上閱讀更多關於 AMP 的內容。

## 啟用 AMP

要為頁面啟用 AMP 支援，並瞭解不同的 AMP 配置，請參閱 [`next/amp`](/docs/pages/guides/amp) 的 [API 文件](/docs/pages/guides/amp)。

## 注意事項

- 僅支援 CSS-in-JS。[CSS Modules](/docs/app/getting-started/css) 目前暫不支援於 AMP 頁面。您可以 [協助貢獻 CSS Modules 支援至 Next.js](https://github.com/vercel/next.js/issues/10549)。

## 新增 AMP 元件

AMP 社群提供了 [眾多元件](https://amp.dev/documentation/components/)，讓 AMP 頁面更具互動性。Next.js 會自動匯入頁面上使用的所有元件，無需手動匯入 AMP 元件腳本：

```jsx
export const config = { amp: true }

function MyAmpPage() {
  const date = new Date()

  return (
    <div>
      <p>Some time: {date.toJSON()}</p>
      <amp-timeago
        width="0"
        height="15"
        datetime={date.toJSON()}
        layout="responsive"
      >
        .
      </amp-timeago>
    </div>
  )
}

export default MyAmpPage
```

上述範例使用了 [`amp-timeago`](https://amp.dev/documentation/components/amp-timeago/?format=websites) 元件。

預設情況下，總是會匯入元件的最新版本。若要自訂版本，您可以使用 `next/head`，如下所示：

```jsx
import Head from 'next/head'

export const config = { amp: true }

function MyAmpPage() {
  const date = new Date()

  return (
    <div>
      <Head>
        <script
          async
          key="amp-timeago"
          custom-element="amp-timeago"
          src="https://cdn.ampproject.org/v0/amp-timeago-0.1.js"
        />
      </Head>

      <p>Some time: {date.toJSON()}</p>
      <amp-timeago
        width="0"
        height="15"
        datetime={date.toJSON()}
        layout="responsive"
      >
        .
      </amp-timeago>
    </div>
  )
}

export default MyAmpPage
```

## AMP 驗證

AMP 頁面在開發期間會透過 [amphtml-validator](https://www.npmjs.com/package/amphtml-validator) 進行自動驗證。錯誤和警告將出現在您啟動 Next.js 的終端機中。

頁面在 [靜態 HTML 匯出](/docs/pages/guides/static-exports) 期間也會進行驗證，任何警告/錯誤都會列印至終端機。任何 AMP 錯誤都會導致匯出以狀態碼 `1` 結束，因為此匯出不符合 AMP 標準。

### 自訂驗證器

您可以在 `next.config.js` 中設定自訂 AMP 驗證器，如下所示：

```js
module.exports = {
  amp: {
    validator: './custom_validator.js',
  },
}
```

### 跳過 AMP 驗證

若要關閉 AMP 驗證，請將以下程式碼新增至 `next.config.js`

```js
experimental: {
  amp: {
    skipValidation: true
  }
}
```

### 靜態 HTML 匯出中的 AMP

當使用 [靜態 HTML 匯出](/docs/pages/guides/static-exports) 靜態預先處理頁面時，Next.js 會偵測頁面是否支援 AMP，並根據該情況變更匯出行為。

例如，混合式 AMP 頁面 `pages/about.js` 將輸出：

- `out/about.html` - 具有客戶端 React 執行時之 HTML 頁面
- `out/about.amp.html` - AMP 頁面

如果 `pages/about.js` 是僅 AMP 頁面，則它將輸出：

- `out/about.html` - 最佳化 AMP 頁面

Next.js 將自動在 HTML 版本中插入指向您頁面 AMP 版本的連結，您無需手動操作，如下所示：

```jsx
<link rel="amphtml" href="/about.amp.html" />
```

而您頁面的 AMP 版本將包含指向 HTML 頁面的連結：

```jsx
<link rel="canonical" href="/about" />
```

當啟用 [`trailingSlash`](/docs/pages/api-reference/config/next-config-js/trailingSlash) 時，`pages/about.js` 的匯出頁面將為：

- `out/about/index.html` - HTML 頁面
- `out/about.amp/index.html` - AMP 頁面

## TypeScript

AMP 目前尚未內建提供 TypeScript 的類型，但這是他們願景清單中的一環 ([#13791](https://github.com/ampproject/amphtml/issues/13791))。

作為解決方法，您可以在專案內部手動建立一個名為 `amp.d.ts` 的檔案，並新增這些 [自訂類型](https://stackoverflow.com/a/50601125)。
