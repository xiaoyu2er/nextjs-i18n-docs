## Tailwind CSS

[Tailwind CSS](https://tailwindcss.com/) 是一個基於工具類的 CSS 框架，提供低階工具類別以建置自訂設計。

<AppOnly>

安裝 Tailwind CSS:

```bash package="pnpm"
pnpm add -D tailwindcss @tailwindcss/postcss
```

```bash package="npm"
npm install -D tailwindcss @tailwindcss/postcss
```

```bash package="yarn"
yarn add -D tailwindcss @tailwindcss/postcss
```

```bash package="bun"
bun add -D tailwindcss @tailwindcss/postcss
```

將 PostCSS 插件新增到您的 `postcss.config.mjs` 檔案中：

```js filename="postcss.config.mjs"
export default {
  plugins: {
    '@tailwindcss/postcss': {},
  },
}
```

在全域 CSS 檔案中匯入 Tailwind：

```css filename="app/globals.css"
@import 'tailwindcss';
```

在您的根層級佈局中匯入 CSS 檔案：

```tsx filename="app/layout.tsx" switcher
import './globals.css'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import './globals.css'

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

現在您可以在應用程式中開始使用 Tailwind 的工具類別：

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return (
    <main className="flex min-h-screen flex-col items-center justify-between p-24">
      <h1 className="text-4xl font-bold">Welcome to Next.js!</h1>
    </main>
  )
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return (
    <main className="flex min-h-screen flex-col items-center justify-between p-24">
      <h1 className="text-4xl font-bold">Welcome to Next.js!</h1>
    </main>
  )
}
```

</AppOnly>

## CSS 模組

CSS 模組透過產生唯一的類別名稱來為 CSS 進行本地範圍限制。這允許您在不同的檔案中使用相同的類別，而無需擔心命名衝突。

<AppOnly>

要開始使用 CSS 模組，請建立一個副檔名為 `.module.css` 的新檔案，並將其匯入 `app` 目錄中的任何元件：

```css filename="app/blog/blog.module.css"
.blog {
  padding: 24px;
}
```

```tsx filename="app/blog/page.tsx" switcher
import styles from './blog.module.css'

export default function Page() {
  return <main className={styles.blog}></main>
}
```

```jsx filename="app/blog/page.js" switcher
import styles from './blog.module.css'

export default function Layout() {
  return <main className={styles.blog}></main>
}
```

</AppOnly>

<PagesOnly>

要開始使用 CSS 模組，請建立一個副檔名為 `.module.css` 的新檔案，並將其匯入 `pages` 目錄中的任何元件：

```css filename="/styles/blog.module.css"
.blog {
  padding: 24px;
}
```

```tsx filename="pages/blog/index.tsx" switcher
import styles from './blog.module.css'

export default function Page() {
  return <main className={styles.blog}></main>
}
```

```jsx filename="pages/blog/index.js" switcher
import styles from './blog.module.css'

export default function Page() {
  return <main className={styles.blog}></main>
}
```

</PagesOnly>

## 全域 CSS

您可以使用全域 CSS 來為您的應用程式套用樣式。

<AppOnly>

建立 `app/global.css` 檔案，並在根佈局中匯入它，以將樣式套用至應用程式中的 **每一個路徑**：

```css filename="app/global.css"
body {
  padding: 20px 20px 60px;
  max-width: 680px;
  margin: 0 auto;
}
```

```tsx filename="app/layout.tsx" switcher
// These styles apply to every route in the application
import './global.css'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
// These styles apply to every route in the application
import './global.css'

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

> **貼心提醒：** 全域樣式可以匯入 `app` 目錄中的任何佈局、頁面或元件。然而，由於 Next.js 使用 React 內建支援的樣式表來整合 Suspense，這目前不會在您在各路徑間導覽時移除樣式表，這可能會導致衝突。我們建議使用全域樣式作為真正的全域 CSS（例如 Tailwind 的基礎樣式），[Tailwind CSS](#tailwind-css) 作為元件樣式，並在需要時使用 [CSS 模組](#css-modules) 作為自訂範圍 CSS。

</AppOnly>

<PagesOnly>

在 `pages/_app.js` 檔案中匯入樣表單式，以將樣式套用至應用程式中的 **每一個路徑**：

```tsx filename="pages/_app.js"
import '@/styles/global.css'

export default function MyApp({ Component, pageProps }) {
  return <Component {...pageProps} />
}
```

由於樣表單式的全域性質，且為了避免衝突，您應該在 [`pages/_app.js`](/docs/pages/building-your-application/routing/custom-app) 內部匯入它們。

</PagesOnly>

## 外部樣式表

<AppOnly>

外部套件發布的樣式表可以在 `app` 目錄的任何位置匯入，包含共用位置的元件：

```tsx filename="app/layout.tsx" switcher
import 'bootstrap/dist/css/bootstrap.css'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body className="container">{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import 'bootstrap/dist/css/bootstrap.css'

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body className="container">{children}</body>
    </html>
  )
}
```

> **貼心提醒：** 在 React 19 中， `<link rel="stylesheet" href="..." />` 也可以使用。詳細資訊請參見 [React `link` 文件](https://react.dev/reference/react-dom/components/link)。

</AppOnly>

<PagesOnly>

Next.js 允許您從 JavaScript 檔案匯入 CSS 檔案。這是因為 Next.js 擴充了 [`import`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Statements/import) 的概念，使其不僅限於 JavaScript。

### 從 `node_modules` 匯入樣式

自 Next.js **9.5.4** 起，在應用程式的任何位置匯入來自 `node_modules` 的 CSS 檔案皆屬允許範圍。

對於全域樣式表（例如 `bootstrap` 或 `nprogress`），您應該在 `pages/_app.js` 內部匯入該檔案。例如：

```jsx filename="pages/_app.js"
import 'bootstrap/dist/css/bootstrap.css'

export default function MyApp({ Component, pageProps }) {
  return <Component {...pageProps} />
}
```

若要匯入第三方元件所需的 CSS，您可以在您的元件中進行此操作。例如：

```jsx filename="components/example-dialog.js"
import { useState } from 'react'
import { Dialog } from '@reach/dialog'
import VisuallyHidden from '@reach/visually-hidden'
import '@reach/dialog/styles.css'

function ExampleDialog(props) {
  const [showDialog, setShowDialog] = useState(false)
  const open = () => setShowDialog(true)
  const close = () => setShowDialog(false)

  return (
    <div>
      <button onClick={open}>Open Dialog</button>
      <Dialog isOpen={showDialog} onDismiss={close}>
        <button className="close-button" onClick={close}>
          <VisuallyHidden>Close</VisuallyHidden>
          <span aria-hidden>×</span>
        </button>
        <p>Hello there. I am a dialog</p>
      </Dialog>
    </div>
  )
}
```

</PagesOnly>

## 排序與合併

Next.js 會在生產環境建置期間，透過自動分割（合併）樣式表來最佳化 CSS。**您的 CSS 順序**取決於**您在程式碼中匯入樣式的順序**。

例如，`base-button.module.css` 的順序會排在 `page.module.css` 之前，因為 `<BaseButton>` 是在匯入 `page.module.css` 之前匯入的：

```tsx filename="page.tsx" switcher
import { BaseButton } from './base-button'
import styles from './page.module.css'

export default function Page() {
  return <BaseButton className={styles.primary} />
}
```

```jsx filename="page.js" switcher
import { BaseButton } from './base-button'
import styles from './page.module.css'

export default function Page() {
  return <BaseButton className={styles.primary} />
}
```

```tsx filename="base-button.tsx" switcher
import styles from './base-button.module.css'

export function BaseButton() {
  return <button className={styles.primary} />
}
```

```jsx filename="base-button.js" switcher
import styles from './base-button.module.css'

export function BaseButton() {
  return <button className={styles.primary} />
}
```

### 建議

為了保持 CSS 順序可預測：

- 嘗試將 CSS 匯入限制在單一 JavaScript 或 TypeScript 入口檔案中。
- 在應用程式的根目錄匯入全域樣式和 Tailwind 樣式表。
- **使用 Tailwind CSS** 處理大多數樣式需求，因為它涵蓋了使用工具類別的常見設計模式。
- 當 Tailwind 功能不充足時，使用 CSS 模組處理元件專屬樣式。
- 對 CSS 模組使用一致的命名慣例。例如，使用 `<name>.module.css` 而非 `<name>.tsx`。
- 將共用樣式提取至共用元件中，以避免重複匯入。
- 關閉會自動排序匯入的程式碼檢查器或格式化工具（例如 ESLint 的 [`sort-imports`](https://eslint.org/docs/latest/rules/sort-imports)）。
- 您可以在 `next.config.js` 中使用 [`cssChunking`](/docs/app/api-reference/config/next-config-js/cssChunking) 選項來控制 CSS 如何分割。

## 開發環境與生產環境

- 在開發環境中（`next dev`），使用 [Fast Refresh](/docs/architecture/fast-refresh) 進行 CSS 更新會立即套用。
- 在生產環境中（`next build`），所有 CSS 檔案都會自動連接為 **多個經過壓縮和代碼分割的** `.css` 檔案，確保路由載入最少的 CSS 量。
- 在生產環境中，即使停用 JavaScript，CSS 仍會載入，但在開發環境中，Fast Refresh 需要 JavaScript。
- 開發環境中的 CSS 順序行為可能不同，務必檢查建置（`next build`）以驗証最終的 CSS 順序。
