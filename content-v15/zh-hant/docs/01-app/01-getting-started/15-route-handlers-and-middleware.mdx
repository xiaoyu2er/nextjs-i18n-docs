## 路由處理器

路由處理器允許您使用 Web [Request](https://developer.mozilla.org/docs/Web/API/Request) 和 [Response](https://developer.mozilla.org/docs/Web/API/Response) API 為指定路由建立自訂請求處理器。

> **小知識**：路由處理器僅限於 `app` 目錄內使用。它們等同於 [API 路由](/docs/pages/building-your-application/routing/api-routes) 在 `pages` 目錄內部，這表示您**不需要**同時使用 API 路由和路由處理器。

### 慣例

路由處理器在 `app` 目錄中的 [`route.js|ts` 檔案](/docs/app/api-reference/file-conventions/route) 中定義：

```ts filename="app/api/route.ts" switcher
export async function GET(request: Request) {}
```

```js filename="app/api/route.js" switcher
export async function GET(request) {}
```

路由處理器可以像 `page.js` 和 `layout.js` 一樣，在 `app` 目錄內的任何位置嵌套。但在與 `page.js` 相同的路由片段級別上，**不可以**有 `route.js` 檔案。

### 支援的 HTTP 方法

支援以下 [HTTP 方法](https://developer.mozilla.org/docs/Web/HTTP/Methods)：`GET`、`POST`、`PUT`、`PATCH`、`DELETE`、`HEAD` 和 `OPTIONS`。如果調用不受支援的方法，Next.js 將返回 `405 Method Not Allowed` 響應。

### 擴充的 `NextRequest` 和 `NextResponse` API

除了支援原生的 [請求 (Request)](https://developer.mozilla.org/docs/Web/API/Request) 和 [回應 (Response)](https://developer.mozilla.org/docs/Web/API/Response) API 之外，Next.js 還使用 [`NextRequest`](/docs/app/api-reference/functions/next-request) 和 [`NextResponse`](/docs/app/api-reference/functions/next-response) 對其進行擴充，以提供方便協助處理高級使用案例的工具。

### 快取

路由處理器預設不會被快取。不過，您可以選擇對 `GET` 方法啟用快取。其他支援的 HTTP 方法**不會**被快取。若要快取 `GET` 方法，請在路由處理器檔案中使用 [路由配置選項](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)，例如 `export const dynamic = 'force-static'`。

```ts filename="app/items/route.ts" switcher
export const dynamic = 'force-static'

export async function GET() {
  const res = await fetch('https://data.mongodb-api.com/...', {
    headers: {
      'Content-Type': 'application/json',
      'API-Key': process.env.DATA_API_KEY,
    },
  })
  const data = await res.json()

  return Response.json({ data })
}
```

```js filename="app/items/route.js" switcher
export const dynamic = 'force-static'

export async function GET() {
  const res = await fetch('https://data.mongodb-api.com/...', {
    headers: {
      'Content-Type': 'application/json',
      'API-Key': process.env.DATA_API_KEY,
    },
  })
  const data = await res.json()

  return Response.json({ data })
}
```

> **小知識**：其他支援的 HTTP 方法**不會**被快取，即使它們與同一檔案中快取的 `GET` 方法放在一起。

### 特殊路由處理器

特殊路由處理器，例如 [`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap)、[`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image) 和 [`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons)，以及其他 [元數據文件](/docs/app/api-reference/file-conventions/metadata) 預設保持靜態，除非它們使用動態 API 或動態配置選項。

### 路由解析

您可以將 `route` 視為最低層級的路由原語。

- 它們**不會**參與佈局或像 `page` 那樣的客戶端導航。
- 在與 `page.js` 相同的路由處，**不可以**有 `route.js` 檔案。

| Page                 | Route              | Result                       |
| -------------------- | ------------------ | ---------------------------- |
| `app/page.js`        | `app/route.js`     | <Cross size={18} /> 衝突 |
| `app/page.js`        | `app/api/route.js` | <Check size={18} /> 有效    |
| `app/[user]/page.js` | `app/api/route.js` | <Check size={18} /> 有效    |

每個 `route.js` 或 `page.js` 檔案會接管該路由的所有 HTTP 動詞。

```ts filename="app/page.ts" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}

// Conflict
// `app/route.ts`
export async function POST(request: Request) {}
```

```js filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}

// Conflict
// `app/route.js`
export async function POST(request) {}
```

請閱讀更多關於路由處理器如何 [補充您的前端應用](/docs/app/guides/backend-for-frontend) 的內容，或瀏覽 Route Handlers 的 [API 參考](/docs/app/api-reference/file-conventions/route)。

### 路由上下文助手

在 TypeScript 中，您可以使用全域可用的 [`RouteContext`](/docs/app/api-reference/file-conventions/route#route-context-helper) 助手，為路由處理器的 `context` 參數指定類型：

```ts filename="app/users/[id]/route.ts" switcher
import type { NextRequest } from 'next/server'

export async function GET(_req: NextRequest, ctx: RouteContext<'/users/[id]'>) {
  const { id } = await ctx.params
  return Response.json({ id })
}
```

> **小知識**
>
> - 類型是在 `next dev`、`next build` 或 `next typegen` 期間生成的。

## 中間件

中間件允許您在請求完成前執行程式碼。接著，基於輸入的請求，您可以修改回應，包括重新編寫、重導向、修改請求或回應標頭，或直接回應。

### 使用案例

中間件有效的常見情境包括：

- 讀取輸入請求的部分內容後快速重導向
- 根據 A/B 測試或實驗重新編寫至不同頁面
- 為所有頁面或部分頁面修改標頭

中間件**不**適合用於：

- 緩慢的資料檢索
- 會話管理

在 Middleware 中使用 `fetch` 搭配 `options.cache`、`options.next.revalidate` 或 `options.next.tags` 無效。

### 慣例

在專案根目錄或 `src` 目錄（若適用）建立一個 `middleware.ts`（或 `.js`）檔案，使其位於與 `pages` 或 `app` 相同的層級。

> **注意**：雖然每個專案僅支援一個 `middleware.ts` 檔案，您仍需將中間件的邏輯組織為模組。將中間件功能拆分成獨立的 `.ts` 或 `.js` 檔案，並匯入至主 `middleware.ts` 檔案。這可以為集中在 `middleware.ts` 中的特定路由中間件實現更清潔的管理。透過強制實施單一中間件檔案，可簡化配置、防止潛在衝突，並透過避免多層中間件來最佳化效能。

### 範例

```ts filename="middleware.ts" switcher
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

// This function can be marked `async` if using `await` inside
export function middleware(request: NextRequest) {
  return NextResponse.redirect(new URL('/home', request.url))
}

// See "Matching Paths" below to learn more
export const config = {
  matcher: '/about/:path*',
}
```

```js filename="middleware.js" switcher
import { NextResponse } from 'next/server'

// This function can be marked `async` if using `await` inside
export function middleware(request) {
  return NextResponse.redirect(new URL('/home', request.url))
}

// See "Matching Paths" below to learn more
export const config = {
  matcher: '/about/:path*',
}
```
