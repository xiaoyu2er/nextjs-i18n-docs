建立新的 Next.js 應用程式並在本地執行。

## 快速開始

1. 建立名為 `my-app` 的新的 Next.js 應用程式
2. `cd my-app` 並啟動開發伺服器。
3. 造訪 `http://localhost:3000`。

```bash package="pnpm"
pnpm create next-app@latest my-app --yes
cd my-app
pnpm dev
```

```bash package="npm"
npx create-next-app@latest my-app --yes
cd my-app
npm run dev
```

```bash package="yarn"
yarn create next-app@latest my-app --yes
cd my-app
yarn dev
```

```bash package="bun"
bun create next-app@latest my-app --yes
cd my-app
bun dev
```

- `--yes` 會跳過提示，使用已保存的偏好設定或預設值。預設設定會啟用 TypeScript、Tailwind、App Router 和 Turbopack，並使用匯入別名 `@/*`。

</AppOnly>

## 系統要求

在開始之前，請確保您的系統符合以下要求：

- [Node.js 18.18](https://nodejs.org/) 或更新版本。
- macOS、Windows（包括 WSL）或 Linux。

## 使用 CLI 建立

建立新的 Next.js 應用程式最快的方式是使用 [`create-next-app`](/docs/app/api-reference/cli/create-next-app)，它會為您自動設定所有內容。要建立專案，請執行：

```bash filename="Terminal"
npx create-next-app@latest
```

安裝過程中，您將看到以下提示：

```txt filename="Terminal"
What is your project named? my-app
Would you like to use TypeScript? No / Yes
Would you like to use ESLint? No / Yes
Would you like to use Tailwind CSS? No / Yes
Would you like your code inside a `src/` directory? No / Yes
Would you like to use App Router? (recommended) No / Yes
Would you like to use Turbopack? (recommended) No / Yes
Would you like to customize the import alias (`@/*` by default)? No / Yes
What import alias would you like configured? @/*
```

提示結束後，[`create-next-app`](/docs/app/api-reference/cli/create-next-app) 會建立包含您專案名稱的資料夾並安裝所需的相依套件。

## 手動安裝

若要手動建立新的 Next.js 應用程式，請安裝所需的套件：

```bash package="pnpm"
pnpm i next@latest react@latest react-dom@latest
```

```bash package="npm"
npm i next@latest react@latest react-dom@latest
```

```bash package="yarn"
yarn add next@latest react@latest react-dom@latest
```

```bash package="bun"
bun add next@latest react@latest react-dom@latest
```

> **補充說明**：App Router 內建使用 [React 開發版本](https://react.dev/blog/2023/05/03/react-canaries)，其中包含所有穩定的 React 19 變更，以及正在框架中驗證的新功能。Pages Router 使用您在 `package.json` 中安裝的 React 版本。

接著，將以下指令腳本新增到您的 `package.json` 檔案中：

```json filename="package.json"
{
  "scripts": {
    "dev": "next dev --turbopack",
    "build": "next build",
    "start": "next start",
    "lint": "eslint"
  }
}
```

這些腳本參照應用程式開發的不同階段：

- `next dev --turbopack`：使用 Turbopack 啟動開發伺服器。
- `next build`：建置應用程式以供生產使用。
- `next start`：啟動生產伺服器。
- `eslint`：執行 ESLint。

Turbopack 在 `dev` 模式下穩定。對於生產建置，Turbopack 處於測試版階段。若要嘗試，請執行 `next build --turbopack`。請參閱 [Turbopack 文件](/docs/app/api-reference/turbopack) 以了解狀態和注意事項。

### 建立 `app` 目錄

Next.js 使用檔案系統路由，這意味著應用程式中的路由取決於您如何結構化檔案。

建立 `app` 資料夾。接著，在 `app` 內部建立 `layout.tsx` 檔案。此檔案是 [根佈局](/docs/app/api-reference/file-conventions/layout#root-layout)。這是必需的，且必須包含 `<html>` 和 `<body>` 標籤。

```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

建立首頁 `app/page.tsx` 並加入一些初始內容：

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

`layout.tsx` 和 `page.tsx` 在用戶造訪應用程式根目錄 (`/`) 時都會被渲染。

### 建立 `pages` 目錄

Next.js 使用檔案系統路由，這意味著應用程式中的路由取決於您如何結構化檔案。

在您的專案根目錄建立 `pages` 目錄。接著，在您的 `pages` 資料夾內新增 `index.tsx` 檔案。這將是您的首頁 (`/`)：

```tsx filename="pages/index.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="pages/index.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

接著，在 `pages/` 內新增 `_app.tsx` 檔案以定義全域佈局。請參閱 [自訂 App 檔案](/docs/pages/building-your-application/routing/custom-app) 以了解更多。

```tsx filename="pages/_app.tsx" switcher
import type { AppProps } from 'next/app'

export default function App({ Component, pageProps }: AppProps) {
  return <Component {...pageProps} />
}
```

```jsx filename="pages/_app.js" switcher
export default function App({ Component, pageProps }) {
  return <Component {...pageProps} />
}
```

最後，在 `pages/` 內新增 `_document.tsx` 檔案以控制伺服器的初始回應。請參閱 [自訂 Document 檔案](/docs/pages/building-your-application/routing/custom-document) 以了解更多。

```tsx filename="pages/_document.tsx" switcher
import { Html, Head, Main, NextScript } from 'next/document'

export default function Document() {
  return (
    <Html>
      <Head />
      <body>
        <Main />
        <NextScript />
      </body>
    </Html>
  )
}
```

```jsx filename="pages/_document.js" switcher
import { Html, Head, Main, NextScript } from 'next/document'

export default function Document() {
  return (
    <Html>
      <Head />
      <body>
        <Main />
        <NextScript />
      </body>
    </Html>
  )
}
```

</PagesOnly>

### 建立 `public` 資料夾（可選）

在您的專案根目錄建立 [`public` 資料夾](/docs/app/api-reference/file-conventions/public-folder) 以儲存靜態資源，例如圖片、字型等。接著，程式碼可以從根 URL (`/`) 開頭引用 `public` 內的檔案。

接著，您可以使用根路徑 (`/`) 引用這些資源。例如，`public/profile.png` 可以引用為 `/profile.png`：

```tsx filename="app/page.tsx" highlight={4} switcher
import Image from 'next/image'

export default function Page() {
  return <Image src="/profile.png" alt="Profile" width={100} height={100} />
}
```

```jsx filename="app/page.js" highlight={4} switcher
import Image from 'next/image'

export default function Page() {
  return <Image src="/profile.png" alt="Profile" width={100} height={100} />
}
```

## 執行開發伺服器

1. 執行 `npm run dev` 以啟動伺服器。
2. 造訪 `http://localhost:3000` 以檢視您的應用程式。
3. 編輯 <AppOnly>`app/page.tsx`</AppOnly><PagesOnly>`pages/index.tsx`</PagesOnly> 檔案並儲存，即可在瀏覽器中看到更新結果。

## 設定 TypeScript

> 最低 TypeScript 版本：`v4.5.2`

Next.js 內建 TypeScript 支援。要在專案中加入 TypeScript，請將檔案重新命名為 `.ts` / `.tsx` 並執行 `next dev`。Next.js 會自動安裝必要的相依套件，並新增一個包含建議組態選項的 `tsconfig.json` 檔案。

## 設定 ESLint

Next.js 內建 ESLint。當您使用 `create-next-app` 建立新專案時，它会自动安裝必要的套件並設定正確的設定。

若要手動將 ESLint 新增到現有專案，請將 `next lint` 新增為 `package.json` 的指令腳本：

```json filename="package.json"
{
  "scripts": {
    "lint": "next lint"
  }
}
```

接著，執行 `npm run lint`，您將被引導完成安裝和設定程序。

```bash filename="Terminal"
npm run lint
```

您將看到類似以下的提示：

> ? How would you like to configure ESLint?
>
> ❯ Strict (recommended)
> Base
> Cancel

- **Strict**：包含 Next.js 的基礎 ESLint 設定以及更嚴格的 Core Web Vitals 規則集。這是首次設定 ESLint 的開發人員建議的設定。
- **Base**：包含 Next.js 的基礎 ESLint 設定。
- **Cancel**：跳過設定。如果您計劃設定自己的自訂 ESLint 設定，請選擇此選項。

如果選擇了 `Strict` 或 `Base`，Next.js 會自動在您的應用程式中安裝 `eslint` 和 `eslint-config-next` 作為相依套件，並在專案根目錄建立設定檔案。

由 `next lint` 生成的 ESLint 設定使用舊版的 `.eslintrc.json` 格式。ESLint 支援 [舊版 `.eslintrc.json` 和新的 `eslint.config.mjs` 格式](https://eslint.org/docs/latest/use/configure/configuration-files#configuring-eslint)。

您可以使用 [ESLint API 參考](/docs/app/api-reference/config/eslint#with-core-web-vitals) 中建議的組態，手動將 `.eslintrc.json` 替換為 `eslint.config.mjs` 檔案，並安裝 [`@eslint/eslintrc`](https://www.npmjs.com/package/@eslint/eslintrc) 套件。這更貼近 `create-next-app` 使用的 ESLint 設定。

現在您可以在每次要執行 ESLint 以捕捉錯誤時執行 `next lint`。一旦 ESLint 已完成設定，它也會在每次建置 (`next build`) 期間自動執行。錯誤會導致建置失敗，而警告則不會。

請參閱 [ESLint 外掛](/docs/app/api-reference/config/next-config-js/eslint) 頁面以獲取更多信息。

## 設定絕對匯入和模組路徑別名

Next.js 內建支援 `tsconfig.json` 和 `jsconfig.json` 檔案的 `"paths"` 和 `"baseUrl"` 選項。

這些選項允許您將專案目錄別名為絕對路徑，使匯入模組更方便且更清潔。例如：

```jsx
// Before
import { Button } from '../../../components/button'

// After
import { Button } from '@/components/button'
```

若要設定絕對匯入，請將 `baseUrl` 組態選項新增到您的 `tsconfig.json` 或 `jsconfig.json` 檔案中。例如：

```json filename="tsconfig.json or jsconfig.json"
{
  "compilerOptions": {
    "baseUrl": "src/"
  }
}
```

除了設定 `baseUrl` 路徑外，您還可以使用 `"paths"` 選項來 `"alias"` 模組路徑。

例如，以下設定將 `@/components/*` 映射到 `components/*`：

```json filename="tsconfig.json or jsconfig.json"
{
  "compilerOptions": {
    "baseUrl": "src/",
    "paths": {
      "@/styles/*": ["styles/*"],
      "@/components/*": ["components/*"]
    }
  }
}
```

每個 `"paths"` 均是相對於 `baseUrl` 位置。
