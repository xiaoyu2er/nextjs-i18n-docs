部分預渲染 (Partial Prerendering) (PPR) 是一種渲染策略，允許您在同一路由中結合靜態和動態內容。這改善了初始頁面效能，同時仍支援個人化、動態資料。

  srcDark="/learn/dark/thinking-in-ppr.png"

/>

當使用者造訪路由時：

- 伺服器發送一包含靜態內容的 **外殼 (shell)**，確保快速初始載入。
- 外殼留下 **孔洞 (holes)**，用於非同步載入的動態內容。
- 動態的 **孔洞 (holes)** 會 **並行串流 (streamed in parallel)**，減少頁面的整體載入時間。

> **🎥 觀看：** 為何使用 PPR 以及如何運作 → [YouTube (10 分鐘)](https://www.youtube.com/watch?v=MTcPrTIBkpA)。

## 部分預渲染 (Partial Prerendering) 是如何運作的？

要瞭解部分預渲染 (Partial Prerendering)，熟悉 Next.js 提供的渲染策略會比較幫助。

### 靜態渲染 (Static Rendering)

使用靜態渲染 (Static Rendering) 時，HTML 會預先生成——即在建置時期透過 [重新驗證 (revalidation)](/docs/app/guides/incremental-static-regeneration) 生成。結果會被快取並在使用者和請求之間共用。

在部分預渲染 (Partial Prerendering) 中，Next.js 會為路由預渲染 **靜態外殼 (static shell)**。這可能包含版面配置以及任何不依賴請求時資料的其他元件。

### 動態渲染 (Dynamic Rendering)

使用動態渲染 (Dynamic Rendering) 時，HTML 會在 **請求時 (request time)** 生成。這允許您基於請求時的資料提供個人化內容。

如果元件使用了以下 API，它就變成動態的：

- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`connection`](/docs/app/api-reference/functions/connection)
- [`draftMode`](/docs/app/api-reference/functions/draft-mode)
- [`searchParams` prop](/docs/app/api-reference/file-conventions/page#searchparams-optional)
- [`unstable_noStore`](/docs/app/api-reference/functions/unstable_noStore)
- [`fetch`](/docs/app/api-reference/functions/fetch) with `{ cache: 'no-store' }`

在部分預渲染 (Partial Prerendering) 中，使用這些 API 會引發特殊的 React 錯誤，告知 Next.js 該元件無法靜態渲染，從而導致建置錯誤。您可以使用 [Suspense](#suspense) 邊界來包裝您的元件，以延遲渲染至執行期。

### Suspense

React [Suspense](https://react.dev/reference/react/Suspense) 用於延遲渲染應用程式的一部分，直到滿足某些條件。

在部分預渲染 (Partial Prerendering) 中，Suspense 用於標記元件樹中的動態邊界。

在建置時期，Next.js 會預渲染靜態內容和候補 (fallback) UI。內容會在請求路由時延遲顯示。

將元件包裝在 Suspense 中不會使元件本身變成動態的（您的 API 使用會），而是 Suspense 用作邊界來匯總動態內容並啟用串流。

```jsx filename="app/page.js"
import { Suspense } from 'react'
import StaticComponent from './StaticComponent'
import DynamicComponent from './DynamicComponent'
import Fallback from './Fallback'

export const experimental_ppr = true

export default function Page() {
  return (
    <>
      <StaticComponent />
      <Suspense fallback={<Fallback />}>
        <DynamicComponent />
      </Suspense>
    </>
  )
}
```

### 串流

串流 (Streaming) 將路由分成區塊，並在這些區塊就緒時，逐步將其串流到用戶端。這允許使用者在整頁內容完成渲染之前，立即看到部分頁面。

為了降低網路開銷，完整的回應——包括靜態 HTML 和串流的動態部分——會在 **單一 HTTP 請求** 中發送。這避免了額外的往返次數，改善了初始載入和整體效能。

## 啟用部分預渲染 (Enabling Partial Prerendering)

您可以透過在 `next.config.ts` 檔案中加入 [`ppr`](https://rc.nextjs.org/docs/app/api-reference/next-config-js/ppr) 選項來啟用 PPR：

```ts filename="next.config.ts" highlight={5} switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    ppr: 'incremental',
  },
}

export default nextConfig
```

```js filename="next.config.js" highlight={4} switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    ppr: 'incremental',
  },
}
```

`'incremental'` 值允許您針對特定路由採用 PPR：

```tsx filename="/app/dashboard/layout.tsx" switcher
export const experimental_ppr = true

export default function Layout({ children }: { children: React.ReactNode }) {
  // ...
}
```

```jsx filename="/app/dashboard/layout.jsx" switcher
export const experimental_ppr = true

export default function Layout({ children }) {
  // ...
}
```

沒有 `experimental_ppr` 的路由將預設為 `false`，並且不會使用 PPR 進行預渲染。您必須針對每個路由明確加入 PPR。

> **值得瞭解**：
>
> - `experimental_ppr` 將應用於路由區塊的所有子要素，包括巢狀的版面配置和頁面。您不必將它加入每個檔案，只需要在路由的頂層區塊即可。
> - 要為子區塊停用 PPR，您可以在子區塊中將 `experimental_ppr` 設定為 `false`。

## 範例

### 動態 API

當使用需要查看請求的動態 API 時，Next.js 將針對路由選擇使用動態渲染。要繼續使用 PPR，請用 Suspense 包裝元件。例如，`<User />` 元件是動態的，因為它使用了 `cookies` API：

```jsx filename="app/user.js" switcher
import { cookies } from 'next/headers'

export async function User() {
  const session = (await cookies()).get('session')?.value
  return '...'
}
```

```tsx filename="app/user.tsx" switcher
import { cookies } from 'next/headers'

export async function User() {
  const session = (await cookies()).get('session')?.value
  return '...'
}
```

`<User />` 元件將被串流，而 `<Page />` 內的其他任何內容將被預渲染並成為靜態外殼的一部分。

```tsx filename="app/page.tsx" switcher
import { Suspense } from 'react'
import { User, AvatarSkeleton } from './user'

export const experimental_ppr = true

export default function Page() {
  return (
    <section>
      <h1>This will be prerendered</h1>
      <Suspense fallback={<AvatarSkeleton />}>
        <User />
      </Suspense>
    </section>
  )
}
```

```jsx filename="app/page.js" switcher
import { Suspense } from 'react'
import { User, AvatarSkeleton } from './user'

export const experimental_ppr = true

export default function Page() {
  return (
    <section>
      <h1>This will be prerendered</h1>
      <Suspense fallback={<AvatarSkeleton />}>
        <User />
      </Suspense>
    </section>
  )
}
```

### 傳遞動態屬性 (Props)

元件僅在存取值時才選擇動態渲染。例如，如果您要從 `<Page />` 元件讀取 `searchParams`，您可以將此值作為屬性傳遞給另一個元件：

```tsx filename="app/page.tsx" switcher
import { Table, TableSkeleton } from './table'
import { Suspense } from 'react'

export default function Page({
  searchParams,
}: {
  searchParams: Promise<{ sort: string }>
}) {
  return (
    <section>
      <h1>This will be prerendered</h1>
      <Suspense fallback={<TableSkeleton />}>
        <Table searchParams={searchParams} />
      </Suspense>
    </section>
  )
}
```

```jsx filename="app/page.js" switcher
import { Table, TableSkeleton } from './table'
import { Suspense } from 'react'

export default function Page({ searchParams }) {
  return (
    <section>
      <h1>This will be prerendered</h1>
      <Suspense fallback={<TableSkeleton />}>
        <Table searchParams={searchParams} />
      </Suspense>
    </section>
  )
}
```

在表格元件內部，存取 `searchParams` 的值將使元件變成動態，而頁面的其餘部分將被預渲染。

```tsx filename="app/table.tsx" switcher
export async function Table({
  searchParams,
}: {
  searchParams: Promise<{ sort: string }>
}) {
  const sort = (await searchParams).sort === 'true'
  return '...'
}
```

```jsx filename="app/table.js" switcher

```
