快取是一種儲存資料擷取和其他運算結果的技術，以便日後請求相同資料時可以更快提供服務，而無需重做工作。而重新驗證則允許您在不重建整個應用程式的情況下更新快取條目。

Next.js 提供了一些 APIs 來處理快取和重新驗證。本指南將引導您了解何時以及如何使用的它們。

- [`fetch` (擷取)](#fetch)
- [`unstable_cache`](#unstable_cache)
- [`revalidatePath`](#revalidatepath)
- [`revalidateTag`](#revalidatetag)

## `fetch` (擷取)

預設情況下，[`fetch`](/docs/app/api-reference/functions/fetch) 請求不會被快取。您可以將 `cache` 選項設為 `'force-cache'` 來快取個別請求。

```tsx filename="app/page.tsx" switcher
export default async function Page() {
  const data = await fetch('https://...', { cache: 'force-cache' })
}
```

```jsx filename="app/page.jsx" switcher
export default async function Page() {
  const data = await fetch('https://...', { cache: 'force-cache' })
}
```

> **值得了解**：儘管 `fetch` 請求預設不會被快取，Next.js 仍會 [預先渲染 (prerender)](/docs/app/getting-started/partial-prerendering#static-rendering) 包含 `fetch` 請求的路由並快取 HTML。如果您想確保路由是 [動態渲染 (dynamic)](/docs/app/getting-started/partial-prerendering#dynamic-rendering)，請使用 [`connection` API](/docs/app/api-reference/functions/connection)。

若要重新驗證 `fetch` 請求回傳的資料，您可以使用 `next.revalidate` 選項。

```tsx filename="app/page.tsx" switcher
export default async function Page() {
  const data = await fetch('https://...', { next: { revalidate: 3600 } })
}
```

```jsx filename="app/page.jsx" switcher
export default async function Page() {
  const data = await fetch('https://...', { next: { revalidate: 3600 } })
}
```

這會在指定秒數後重新驗證資料。

請參閱 [`fetch` API 參考](/docs/app/api-reference/functions/fetch) 以了解更多。

## `unstable_cache` (暫定快取)

`unstable_cache` 允許您快取資料庫查詢和其他非同步函數的結果。要使用它，請以 `unstable_cache` 包覆函式。例如：

```tsx filename="app/lib/data.ts swichter
import { db } from '@/lib/db'
export async function getUserById(id: string) {
  return db
    .select()
    .from(users)
    .where(eq(users.id, id))
    .then((res) => res[0])
}
```

```jsx filename="app/lib/data.js" switcher
import { db } from '@/lib/db'

export async function getUserById(id) {
  return db
    .select()
    .from(users)
    .where(eq(users.id, id))
    .then((res) => res[0])
}
```

```tsx filename="app/page.tsx" highlight={2,11,13} switcher
import { unstable_cache } from 'next/cache'
import { getUserById } from '@/app/lib/data'

export default async function Page({
  params,
}: {
  params: Promise<{ userId: string }>
}) {
  const { userId } = await params

  const getCachedUser = unstable_cache(
    async () => {
      return getUserById(userId)
    },
    [userId] // add the user ID to the cache key
  )
}
```

```jsx filename="app/page.jsx" highlight={2,7,9} switcher
import { unstable_cache } from 'next/cache';
import { getUserById } from '@/app/lib/data';

export default async function Page({ params } }) {
  const { userId } = await params

  const getCachedUser = unstable_cache(
    async () => {
      return getUserById(userId)
    },
    [userId] // add the user ID to the cache key
  );
}
```

函式接受第三個可選物件來定義快取應如何重新驗證。它接受：

- `tags`：Next.js 用來重新驗證快取的標籤陣列。
- `revalidate`：快取應重新驗證後的秒數。

```tsx filename="app/page.tsx" highlight={6-9} switcher
const getCachedUser = unstable_cache(
  async () => {
    return getUserById(userId)
  },
  [userId],
  {
    tags: ['user'],
    revalidate: 3600,
  }
)
```

```jsx filename="app/page.js" highlight={6-9} switcher
const getCachedUser = unstable_cache(
  async () => {
    return getUserById(userId)
  },
  [userId],
  {
    tags: ['user'],
    revalidate: 3600,
  }
)
```

請參閱 [`unstable_cache` API 參考](/docs/app/api-reference/functions/unstable_cache) 以了解更多。

## `revalidateTag` (重新驗證標籤)

`revalidateTag` 用於根據標籤和觸發事件來重新驗證快取條目。要將其與 `fetch` 配合使用，請先用 `next.tags` 選項標記函式：

```tsx filename="app/lib/data.ts" highlight={3-5} switcher
export async function getUserById(id: string) {
  const data = await fetch(`https://...`, {
    next: {
      tags: ['user'],
    },
  })
}
```

```jsx filename="app/lib/data.js" highlight={3-5} switcher
export async function getUserById(id) {
  const data = await fetch(`https://...`, {
    next: {
      tags: ['user'],
    },
  })
}
```

或者，您可以使用 `tags` 選項標記 `unstable_cache` 函式：

```tsx filename="app/lib/data.ts" highlight={6-8} switcher
export const getUserById = unstable_cache(
  async (id: string) => {
    return db.query.users.findFirst({ where: eq(users.id, id) })
  },
  ['user'], // Needed if variables are not passed as parameters
  {
    tags: ['user'],
  }
)
```

```jsx filename="app/lib/data.js" highlight={6-8} switcher
export const getUserById = unstable_cache(
  async (id) => {
    return db.query.users.findFirst({ where: eq(users.id, id) })
  },
  ['user'], // Needed if variables are not passed as parameters
  {
    tags: ['user'],
  }
)
```

接著，在 [路線處理程序 (Route Handler)](/docs/app/api-reference/file-conventions/route) 或伺服器動作 (Server Action) 中呼叫 `revalidateTag`：

```tsx filename="app/lib/actions.ts" highlight={1} switcher
import { revalidateTag } from 'next/cache'

export async function updateUser(id: string) {
  // Mutate data
  revalidateTag('user')
}
```

```jsx filename="app/lib/actions.js" highlight={1} switcher
import { revalidateTag } from 'next/cache'

export async function updateUser(id) {
  // Mutate data
  revalidateTag('user')
}
```

您可以在多個函式中重複使用相同的標籤，一次重新驗證所有內容。

請參閱 [`revalidateTag` API 參考](/docs/app/api-reference/functions/revalidateTag) 以了解更多。

## `revalidatePath` (重新驗證路由)

`revalidatePath` 用於根據觸發事件重新驗證路由。要使用它，請在 [路線處理程序 (Route Handler)](/docs/app/api-reference/file-conventions/route) 或伺服器動作 (Server Action) 中呼叫它：

```tsx filename="app/lib/actions.ts" highlight={1} switcher
import { revalidatePath } from 'next/cache'

export async function updateUser(id: string) {
  // Mutate data
  revalidatePath('/profile')
```

```jsx filename="app/lib/actions.js" highlight={1} switcher
import { revalidatePath } from 'next/cache'

export async function updateUser(id) {
  // Mutate data
  revalidatePath('/profile')
```

請參閱 [`revalidatePath` API 參考](/docs/app/api-reference/functions/revalidatePath) 以了解更多。
