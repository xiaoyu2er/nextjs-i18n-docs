---
title: 如何在您的 Next.js 應用程序中實施 JSON-LD
nav_title: JSON-LD
description: 學習如何將 JSON-LD 添加到您的 Next.js 應用程序中，以便向搜索引擎和 AI 描述您的內容。
---

[JSON-LD](https://json-ld.org/) 是一種結構化數據 (Structured Data) 格式，可由搜索引擎 (Search Engines) 和人工智能 (AI) 使用，以幫助它們了解頁面結構，而不僅僅是純內容。例如，您可以使用它來描述個人、活動、組織、電影、書籍、食譜以及許多其他類型的實體 (Entities)。

我們目前對於 JSON-LD 的建議是將結構化數據 (Structured Data) 渲染為 `<script>` 標籤，置於您的 `layout.js` 或 `page.js` 組件中。

以下代碼片段使用 `JSON.stringify`，它不會清洗用於 XSS 注入 (XSS Injection) 的惡意字串。為了防止此類型漏洞，您可以從 `JSON-LD` 有效負載 (Payload) 中移除 `HTML` 標籤，例如，通過將字元 `<` 替換為其 Unicode 等價物 `\u003c`。

請審查您組織推薦的清洗潛在危險字串的方法，或使用社群維護的 `JSON.stringify` 替代方案，例如，[serialize-javascript](https://www.npmjs.com/package/serialize-javascript)。

```tsx filename="app/products/[id]/page.tsx" switcher
export default async function Page({ params }) {
  const { id } = await params
  const product = await getProduct(id)

  const jsonLd = {
    '@context': 'https://schema.org',
    '@type': 'Product',
    name: product.name,
    image: product.image,
    description: product.description,
  }

  return (
    <section>
      {/* Add JSON-LD to your page */}
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{
          __html: JSON.stringify(jsonLd).replace(/</g, '\\u003c'),
        }}
      />
      {/* ... */}
    </section>
  )
}
```

您可以使用 Google 的 [Rich Results Test](https://search.google.com/test/rich-results) 或通用的 [Schema Markup Validator](https://validator.schema.org/) 來驗證和測試您的結構化數據 (Structured Data)。

您可以使用社群套件 [`schema-dts`](https://www.npmjs.com/package/schema-dts)，配合 TypeScript 為您的 JSON-LD 進行類型定義 (Typing)。

```tsx
import { Product, WithContext } from 'schema-dts'

const jsonLd: WithContext<Product> = {
  '@context': 'https://schema.org',
  '@type': 'Product',
  name: 'Next.js Sticker',
  image: 'https://nextjs.org/imgs/sticker.png',
  description: 'Dynamic at the speed of static.',
}
```
