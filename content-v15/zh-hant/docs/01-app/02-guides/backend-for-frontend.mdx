Next.js 支援「前端後端 (Backend for Frontend)」模式。這讓您可以建立公開端點來處理 HTTP 請求並返回任何內容類型－不只是 HTML。您還可以存取資料來源並執行副作用，例如更新遠端資料。

若您開始新專案，使用 `create-next-app` 配合 `--api` 旗標會在新專案的 `app/` 資料夾中自動包含範例 `route.ts`，展示如何建立 API 端點。

```bash filename="Terminal"
npx create-next-app@latest --api
```

> **好知識**：Next.js 後端功能並非完整的後端替代品。它們作為一 API 層，具備以下特性：
>
> - 公開可達
> - 處理任何 HTTP 請求
> - 可返回任何內容類型

要實作此模式，請使用：

- [Route Handlers](/docs/app/api-reference/file-conventions/route)
- [`middleware`](/docs/app/api-reference/file-conventions/middleware)
- 在 Pages Router，[API Routes](/docs/pages/building-your-application/routing/api-routes)

## 公開端點

Route Handlers 是公開的 HTTP 端點。任何用戶端皆可存取它們。

使用 `route.ts` 或 `route.js` 檔案範例建立 Route Handler：

```ts filename="/app/api/route.ts" switcher
export function GET(request: Request) {}
```

```js filename="/app/api/route.js" switcher
export function GET(request) {}
```

這處理發送至 `/api` 的 `GET` 請求。

對於可能拋出異常的操作，請使用 `try/catch` 區塊：

```ts filename="/app/api/route.ts" switcher
import { submit } from '@/lib/submit'

export async function POST(request: Request) {
  try {
    await submit(request)
    return new Response(null, { status: 204 })
  } catch (reason) {
    const message =
      reason instanceof Error ? reason.message : 'Unexpected error'

    return new Response(message, { status: 500 })
  }
}
```

```js filename="/app/api/route.js" switcher
import { submit } from '@/lib/submit'

export async function POST(request) {
  try {
    await submit(request)
    return new Response(null, { status: 204 })
  } catch (reason) {
    const message =
      reason instanceof Error ? reason.message : 'Unexpected error'

    return new Response(message, { status: 500 })
  }
}
```

避免暴露敏感資訊到傳送給用戶端的錯誤訊息中。

若要限制存取，請實作身分驗證和授權。請參閱 [Authentication](/docs/app/guides/authentication)。

## 內容類型

Route Handlers 允許您提供非 UI 回應，包括 JSON、XML、圖片、檔案和純文字。

Next.js 使用檔案範例來處理常用端點：

- [`sitemap.xml`](/docs/app/api-reference/file-conventions/metadata/sitemap)
- [`opengraph-image.jpg`, `twitter-image`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)
- [favicon, app icon, and apple-icon](/docs/app/api-reference/file-conventions/metadata/app-icons)
- [`manifest.json`](/docs/app/api-reference/file-conventions/metadata/manifest)
- [`robots.txt`](/docs/app/api-reference/file-conventions/metadata/robots)

您也可以定義自訂的端點，例如：

- `llms.txt`
- `rss.xml`
- `.well-known`

例如，`app/rss.xml/route.ts` 建立 `rss.xml` 的 Route Handler。

```ts filename="/app/rss.xml/route.ts" switcher
export async function GET(request: Request) {
  const rssResponse = await fetch(/* rss endpoint */)
  const rssData = await rssResponse.json()

  const rssFeed = `<?xml version="1.0" encoding="UTF-8" ?>
<rss version="2.0">
<channel>
 <title>${rssData.title}</title>
 <description>${rssData.description}</description>
 <link>${rssData.link}</link>
 <copyright>${rssData.copyright}</copyright>
 ${rssData.items.map((item) => {
   return `<item>
    <title>${item.title}</title>
    <description>${item.description}</description>
    <link>${item.link}</link>
    <pubDate>${item.publishDate}</pubDate>
    <guid isPermaLink="false">${item.guid}</guid>
 </item>`
 })}
</channel>
</rss>`

  const headers = new Headers({ 'content-type': 'application/xml' })

  return new Response(rssFeed, { headers })
}
```

### 處理請求載入

使用 Request [實例方法](https://developer.mozilla.org/en-US/docs/Web/API/Request#instance_methods) 如 `.json()`, `.formData()`, 或 `.text()` 來存取請求載入。

`GET` 和 `HEAD` 請求不攜帶載入。

```ts filename="/app/api/echo-body/route.ts" switcher
export async function POST(request: Request) {
  const res = await request.json()
  return Response.json({ res })
}
```

```js filename="/app/api/echo-body/route.js" switcher
export async function POST(request) {
  const res = await request.json()
  return Response.json({ res })
}
```

> **好知識**：轉送前驗證資料

```ts filename="/app/api/send-email/route.ts" switcher
import { sendMail, validateInputs } from '@/lib/email-transporter'

export async function POST(request: Request) {
  const formData = await request.formData()
  const email = formData.get('email')
  const contents = formData.get('contents')

  try {
    await validateInputs({ email, contents })
    const info = await sendMail({ email, contents })

    return Response.json({ messageId: info.messageId })
  } catch (reason) {
    const message =
      reason instanceof Error ? reason.message : 'Unexpected exception'

    return new Response(message, { status: 500 })
  }
}
```

## 操作資料

Route Handlers 可以從一個或多個來源轉換、篩選和彙總資料。這能將邏輯保持在前端之外，並避免暴露內部系統。

您也可以將繁重計算卸載至伺服器，減少用戶端的電池和資料使用量。

```ts file="/app/api/weather/route.ts" switcher
import { parseWeatherData } from '@/lib/weather'

export async function POST(request: Request) {
  const body = await request.json()
  const searchParams = new URLSearchParams({ lat: body.lat, lng: body.lng })

  try {
    const weatherResponse = await fetch(`${weatherEndpoint}?${searchParams}`)

    if (!weatherResponse.ok) {
      /* handle error */
    }

    const weatherData = await weatherResponse.text()
    const payload = parseWeatherData.asJSON(weatherData)

    return new Response(payload, { status: 200 })
  } catch (reason) {
    const message =
      reason instanceof Error ? reason.message : 'Unexpected exception'

    return new Response(message, { status: 500 })
  }
}
```


## 代理至後端

您可以將 Route Handler 用作至另一個後端的代理。轉送請求前加入驗證邏輯。

```ts filename="/app/api/[...slug]/route.ts" switcher
import { isValidRequest } from '@/lib/utils'

export async function POST(request: Request, { params }) {
  const clonedRequest = request.clone()
  const isValid = await isValidRequest(clonedRequest)

  if (!isValid) {
    return new Response(null, { status: 400, statusText: 'Bad Request' })
  }

  const { slug } = await params
  const pathname = slug.join('/')
  const proxyURL = new URL(pathname, 'https://nextjs.org')
  const proxyRequest = new Request(proxyURL, request)

  try {
    return fetch(proxyRequest)
  } catch (reason) {
    const message =
      reason instanceof Error ? reason.message : 'Unexpected exception'

    return new Response(message, { status: 500 })
  }
}
```

## NextRequest 和 NextResponse

Next.js 擴充 [`Request`](https://developer.mozilla.org/en-US/docs/Web/API/Request) 和 [`Response`](https://developer.mozilla.org/en-US/docs/Web/API/Response) Web APIs 的內建方法，簡化常見操作。這些擴充功能在 Route Handlers 和 Middleware 中皆可使用。

兩者提供讀取和操作 Cookie 的方法。

`NextRequest` 包含 [`nextUrl`](/docs/app/api-reference/functions/next-request#nexturl) 屬性，揭露來自輸入請求的解析值，例如，這讓存取請求路徑和搜尋參數更輕鬆。

`NextResponse` 提供如 `next()`, `json()`, `redirect()`, 和 `rewrite()` 等輔助工具。

您可以將 `NextRequest` 傳遞給任何預期 `Request` 的函式。同樣地，您可以在預期 `Response` 的地方回報 `NextResponse`。

```ts filename="/app/echo-pathname/route.ts" switcher
import { type NextRequest, NextResponse } from 'next/server'

export async function GET(request: NextRequest) {
  const nextUrl = request.nextUrl

  if (nextUrl.searchParams.get('redirect')) {
    return NextResponse.redirect(new URL('/', request.url))
  }

  if (nextUrl.searchParams.get('rewrite')) {
    return NextResponse.rewrite(new URL('/', request.url))
  }

  return NextResponse.json({ pathname: nextUrl.pathname })
}
```


## Webhooks 和呼叫返回 URL

使用 Route Handlers 接收第三方應用程式的事件通知。

例如，當 CMS 中的內容變更時重新驗證端點。設定 CMS 在變更時呼叫特定端點。

```ts filename="/app/webhook/route.ts" switcher
import { type NextRequest, NextResponse } from 'next/server'

export async function GET(request: NextRequest) {
  const token = request.nextUrl.searchParams.get('token')

  if (token !== process.env.REVALIDATE_SECRET_TOKEN) {
    return NextResponse.json({ success: false }, { status: 401 })
  }

  const tag = request.nextUrl.searchParams.get('tag')

  if (!tag) {
    return NextResponse.json({ success: false }, { status: 400 })
  }

  revalidateTag(tag)

  return NextResponse.json({ success: true })
}
```


呼叫返回 URL 是另一個使用案例。當用戶完成第三方流程時，第三方會將他們傳送回呼叫返回 URL。使用 Route Handler 來驗證回應並決定如何重新導向用戶。

```ts filename="/app/auth/callback/route.ts" switcher
import { type NextRequest, NextResponse } from 'next/server'

export async function GET(request: NextRequest) {
  const token = request.nextUrl.searchParams.get('session_token')
  const redirectUrl = request.nextUrl.searchParams.get('redirect_url')

  const response = NextResponse.redirect(new URL(redirectUrl, request.url))

  response.cookies.set({
    value: token,
    name: '_token',
    path: '/',
    secure: true,
    httpOnly: true,
    expires: undefined, // session cookie
  })

  return response
}
```

## 中間件

每個專案僅允許一個 middleware 檔案。使用 `config.matcher` 針對特定路徑。深入了解 [middleware](/docs/app/api-reference/file-conventions/middleware)。

使用 `middleware` 在請求到達路由路徑前生成回應。

```ts filename="middleware.ts" switcher
import { isAuthenticated } from '@lib/auth'

export const config = {
  matcher: '/api/:function*',
}

export function middleware(request: Request) {
  if (!isAuthenticated(request)) {
    return Response.json(
      { success: false, message: 'authentication failed' },
      { status: 401 }
    )
  }
}
```

```js filename="middleware.js" switcher
import { isAuthenticated } from '@lib/auth'
```


## 安全性

### 處理標頭

審慎處理標頭去向，避免直接將輸入請求標頭傳遞至輸出回應。

- **上遊請求標頭**：在 Middleware 中，`NextResponse.next({ request: { headers } })` 修改您的伺服器接收的標頭，且不向用戶端暴露它們。
- **回應標頭**：`new Response(..., { headers })`, `NextResponse.json(..., { headers })`, `NextResponse.next({ headers })`, 或 `response.headers.set(...)` 將標頭傳送給用戶端。若敏感值已附加至這些標頭，用戶端將可看見它們。

深入了解 [NextResponse headers in Middleware](/docs/app/api-reference/functions/next-response#next)。

### 速率限制

您可以在 Next.js 後端實作速率限制。除了程式碼檢查外，請啟用宿主提供的任何速率限制功能。

```ts filename="/app/resource/route.ts" switcher
import { NextResponse } from 'next/server'
import { checkRateLimit } from '@/lib/rate-limit'

export async function POST(request: Request) {
  const { rateLimited } = await checkRateLimit(request)

  if (rateLimited) {
    return NextResponse.json({ error: 'Rate limit exceeded' }, { status: 429 })
  }

  return new Response(null, { status: 204 })
}
```

```js filename="/app/resource/route.js" switcher
import { NextResponse } from 'next/server'

```

### 驗證載入

切勿信任輸入的請求資料。驗證內容類型和大小，並在用於前進行 XSS 清潔。

使用超時時間來防止濫用並保護伺服器資源。

將使用者產生的靜態資產儲存於專用服務中。若可能，從瀏覽器上載它們並將回傳的 URI 儲存至您的資料庫以減少請求大小。

### 存取受保護的資源

授予存取權之前始終驗證憑證。不要單獨依賴 middleware 進行身分驗證和授權。

從回應和後端記錄中移除敏感或不需要資料。

定期輪替憑證和 API 金鑰。

## 預檢請求

預檢請求使用 `OPTIONS` 方法詢問伺服器，根據來源、方法和標頭是否允許請求。

若未定義 `OPTIONS`，Next.js 會自動設定並根據其他定義的方法設定 `Allow` 標頭。

- [CORS](/docs/app/api-reference/file-conventions/route#cors)

## 函式庫模式

社群函式庫常為 Route Handlers 使用工廠模式。

```ts filename="/app/api/[...path]/route.ts"
import { createHandler } from 'third-party-library'

const handler = createHandler({
  /* library-specific options */
})

export const GET = handler
// or
export { handler as POST }
```

這為 `GET` 和 `POST` 請求建立共用 handler。函式庫根據請求中的 `method` 和 `pathname` 自訂行為。

函式庫也可以提供 `middleware` 工廠。

```ts filename="middleware.ts"
import { createMiddleware } from 'third-party-library'

export default createMiddleware()
```

## 注意事項

### 伺服器元件

在 Server Components 中直接從其來源存取資料，不要透過 Route Handlers。

針對在構建時預先封裝的 Server Components，使用 Route Handlers 會導致構建步驟失敗。這是因為在構建時沒有伺服器監聽這些請求。

針對隨時運作的 Server Components，從 Route Handlers 存取會因 handler 和渲染行程之間的額外 HTTP 往返而較慢。

> 伺服器端 `fetch` 請求使用絕對 URL。這意味著需與外部伺服器進行 HTTP 往返。開發期間，您的開發伺服器作為外部伺服器。構建時沒有伺服器，而在執行時間，伺服器透過您公開的網域提供。

Server Components 涵蓋大多數的資料存取需求。然而，在用戶端存取資料可能是必要的，例如：

- 依賴用戶端專屬 Web APIs 的資料：
  - Geo-location API
  - Storage API
  - Audio API
  - File API
- 經常輪詢的資料

針對這些情況，請使用社群函式庫，如 [`swr`](https://swr.vercel.app/) 或 [`react-query`](https://tanstack.com/query/latest/docs/framework/react/overview)。

### 伺服器操作

Server Actions 讓您可以從用戶端執行伺服器端程式碼。其主要目的是從您的前端用戶端變更資料。

Server Actions 會排隊。使用它們進行資料存取會引入遞延執行。

### `export` 模式

`export` 模式輸出沒有執行時伺服器的靜態網站。需要 Next.js 執行時的功能 [不支援](/docs/app/guides/static-exports#unsupported-features)，因為此模式產生靜態網站，且沒有執行時伺服器。

在 `export` 模式下，僅支援 `GET` Route Handlers，配合 [`dynamic`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic) 路線區域組態，設定為 `'force-static'`。

這可用於產生靜態 HTML、JSON、TXT 或其他檔案。

```js filename="app/hello-world/route.ts"
export const dynamic = 'force-static'

export function GET() {
  return new Response('Hello World', { status: 200 })
}
```

### 部署環境

一些主機部署 Route Handlers 作為 lambda 函式。這意味著：

- Route Handlers 無法在請求間共享資料。
- 環境可能不支援撰寫至檔案系統。
- 長時間執行的 handlers 可能因超時而終止。
- WebSockets 不適用於執行，因為連線會在超時或產生回應後關閉。
