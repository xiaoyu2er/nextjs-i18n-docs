預先載入功能讓您的應用程式中在不同路由之間導航變得即刻響應。Next.js 預設會根據應用程式代碼中使用的連結，嘗試智慧地進行預先載入。

本指南將解釋預先載入的工作原理，並展示常見的實作模式：

- [自動預先載入](#automatic-prefetch)
- [手動預先載入](#manual-prefetch)
- [滑鼠懸停觸發的預先載入](#hover-triggered-prefetch)
- [擴充或彈出連結](#extending-or-ejecting-link)
- [停用預先載入](#disabled-prefetch)

## 預先載入如何運作？

當在路由之間導航時，瀏覽器會請求頁面所需的資產，例如 HTML 和 JavaScript 檔案。預先載入是在導航到新路由之前，提前獲取這些資源的過程。

Next.js 會根據路由自動將您的應用程式分割成較小的 JavaScript 資料區塊。不像傳統 SPA 那樣一次性載入所有程式碼，只有當前路由所需的程式碼會被載入。這減少了初始載入時間，同時應用程式的其他部分在背景中載入。當您點擊連結時，新路由的資源已經載入瀏覽器的快取中。

導航到新頁面時，不會進行完整的頁面重新整理或出現瀏覽器的載入指示器。取而代之地，Next.js 執行 [客戶端轉換](/docs/app/getting-started/linking-and-navigating#client-side-transitions)，使頁面導航感覺即刻響應。

## 靜態與動態路由的預先載入

|                                                                   | **靜態頁面** | **動態頁面**                                                                                               |
| ----------------------------------------------------------------- | ------------ | ---------------------------------------------------------------------------------------------------------- |
| **已預先載入**                                                    | 是，完整路由 | 否，除非 [`loading.js`](/docs/app/api-reference/file-conventions/loading)                                  |
| [**客戶端快取 TTL**](/docs/app/guides/caching#full-route-cache) | 5 分鐘 (預設) | 關閉，除非 [啟用](/docs/app/api-reference/config/next-config-js/staleTimes)                                |
| **點擊時的伺服器往返**                                       | 否              | 是，在 [Shell](/docs/app/getting-started/partial-prerendering#how-does-partial-prerendering-work) 後串流 |

> **值得了解：** 在初始導航期間，瀏覽器會獲取 HTML、JavaScript 和 React 伺服器元件 (RSC) 負載。對於後續導航，瀏覽器將獲取伺服器元件的 RSC 負載和客戶端元件的 JS 捆綁包。

## 自動預先載入

```tsx filename="app/ui/nav-link.tsx" switcher
import Link from 'next/link'

export default function NavLink() {
  return <Link href="/about">About</Link>
}
```

```jsx filename="app/ui/nav-link.js" switcher
import Link from 'next/link'

export default function NavLink() {
  return <Link href="/about">About</Link>
}
```

| **Context**       | **預先載入負載**           | **客戶端快取 TTL**                                                           |
| ----------------- | -------------------------------- | ------------------------------------------------------------------------------ |
| 未使用 `loading.js`   | 整個頁面                      | 直到應用程式重新整理                                                               |
| 使用 `loading.js` | 佈局至第一個載入邊界 | 30 秒 ([可配置](/docs/app/api-reference/config/next-config-js/staleTimes)) |

自動預先載入僅在生產環境運行。使用 `prefetch={false}` 停用，或在 [停用預先載入](#disabled-prefetch) 中使用封裝函式。

## 手動預先載入

```tsx
'use client'

import { useRouter } from 'next/navigation'

const router = useRouter()
router.prefetch('/pricing')
```

呼叫 `router.prefetch()` 以預載視窗範圍外的連結，或針對分析、滑鼠懸停、捲動等事件進行預載。

## 滑鼠懸停觸發的預先載入

> **謹慎操作：** 擴充 `Link` 元素代表您需自行維護預先載入、快取無效及無障礙功能。僅在預設值不足時執行。

Next.js 預設會嘗試進行正確的預先載入，但進階使用者可以彈出並根據需求進行修改。您將擁有在效能與資源消耗之間的控制權。

例如，您可能需要在滑鼠懸停時觸發預先載入，而不是當進入視窗範圍時（預設行為）：

```tsx
'use client'

import Link from 'next/link'
import { useState } from 'react'

export function HoverPrefetchLink({
  href,
  children,
}: {
  href: string
  children: React.ReactNode
}) {
  const [active, setActive] = useState(false)

  return (
    <Link
      href={href}
      prefetch={active ? null : false}
      onMouseEnter={() => setActive(true)}
    >
      {children}
    </Link>
  )
}
```

`prefetch={null}` 會在用戶顯示意圖後恢復預設（靜態）預先載入。

## 擴充或彈出連結

您可以擴充 `<Link>` 元件以建立自訂的預先載入策略。例如，使用 [ForesightJS](https://foresightjs.com/docs/integrations/nextjs) 函式庫，它通過預測用戶游標的方向來預先載入連結。

另外，您也可以使用 [`useRouter`](/docs/app/api-reference/functions/use-router) 重現部分原生 `<Link>` 行為。但請注意，這代表您需要自行維護預先載入和快取無效。

```tsx
'use client'

import { useRouter } from 'next/navigation'
import { useEffect } from 'react'

function ManualPrefetchLink({
  href,
  children,
}: {
  href: string
  children: React.ReactNode
}) {
  const router = useRouter()

  useEffect(() => {
    let cancelled = false
    const poll = () => {
      if (!cancelled) router.prefetch(href, { onInvalidate: poll })
    }
    poll()
    return () => {
      cancelled = true
    }
  }, [href, router])

  return (
    <a
      href={href}
      onClick={(event) => {
        event.preventDefault()
        router.push(href)
      }}
    >
      {children}
    </a>
  )
}
```

當 Next.js 推測快取資料已過期時會呼叫 [`onInvalidate`](/docs/app/api-reference/functions/use-router#userouter)，允許您刷新預先載入。

> **值得了解：** 使用 `a` 標籤將導致前往目標路由的完整頁面導航，您可以使用 `onClick` 防止完整頁面導航，然後呼叫 `router.push` 導航至目標。

## 停用預先載入

您可以針對特定路由完全停用預先載入，以更精細地控制資源消耗。

```tsx
'use client'

import Link, { LinkProps } from 'next/link'

function NoPrefetchLink({
  prefetch,
  ...rest
}: LinkProps & { children: React.ReactNode }) {
  return <Link {...rest} prefetch={false} />
}
```

例如，您可能仍希望在應用程式中一致地使用 `<Link>`，但位於頁尾的連結在進入視窗範圍時可能不需要預先載入。

## 預先載入最佳化

> **值得了解：** 佈局去重與預先載入排程是即將推出的最佳化功能。目前可在 Next.js canary 版中透過 `experimental.clientSegmentCache` 旗標使用。

### 客戶端快取

Next.js 將預先載入的 React 伺服器元件負載儲存於記憶體中，路由段作為索引。當在兄弟路由之間導航時（例如 `/dashboard/settings` → `/dashboard/analytics`），它會重用父層佈局，僅獲取更新的葉片頁面。這減少了網路流量並提高了導航速度。

### 預先載入排程

Next.js 維護一個小型任務佇列，按以下順序進行預先載入：

1. 視窗範圍內的連結
2. 顯示用戶意圖的連結 (懸停或觸控)
3. 較新的連結會取代較老的連結
4. 已捲動出螢幕的連結會被丟棄

排程器優先處理可能的導航，同時最大限度地減少未使用的下載。

### 部分預先渲染 (PPR)

啟用 PPR 時，頁面被分為靜態 Shell 和串流的動態區塊：

- 可預先載入的 Shell 會立即串流
- 動態資料在準備就緒時串流
- 資料無效化 (`revalidateTag`, `revalidatePath`) 會靜默地刷新關聯的預先載入

## 疑難排解

### 預先載入期間觸發不想要的事後效果

如果您的佈局或頁面不是 [純](https://react.dev/learn/keeping-components-pure#purity-components-as-formulas) 的並且具有副作用（例如追蹤分析），可能會在路由預先載入時觸發這些，而不是當用戶訪問頁面時。

要避免這個問題，您應該將副作用移至 `useEffect` 鉤子，或在客戶端元件中觸發的伺服器動作。

**之前**：

```tsx filename="app/dashboard/layout.tsx" switcher
import { trackPageView } from '@/lib/analytics'

export default function Layout({ children }: { children: React.ReactNode }) {
  // This runs during prefetch
  trackPageView()

  return <div>{children}</div>
}
```

```jsx filename="app/dashboard/layout.js" switcher
import { trackPageView } from '@/lib/analytics'

export default function Layout({ children }) {
  // This runs during prefetch
  trackPageView()

  return <div>{children}</div>
}
```

**之後**：

```tsx filename="app/ui/analytics-tracker.tsx" switcher
'use client'

import { useEffect } from 'react'
import { trackPageView } from '@/lib/analytics'

export function AnalyticsTracker() {
  useEffect(() => {
    trackPageView()
  }, [])

  return null
}
```

```jsx filename="app/ui/analytics-tracker.js" switcher
'use client'

import { useEffect } from 'react'
import { trackPageView } from '@/lib/analytics'

export function AnalyticsTracker() {
  useEffect(() => {
    trackPageView()
  }, [])

  return null
}
```

```tsx filename="app/dashboard/layout.tsx" switcher
import { AnalyticsTracker } from '@/app/ui/analytics-tracker'

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <AnalyticsTracker />
      {children}
    </div>
  )
}
```

```jsx filename="app/dashboard/layout.js" switcher
import { AnalyticsTracker } from '@/app/ui/analytics-tracker'

export default function Layout({ children }) {
  return (
    <div>
      <AnalyticsTracker />

    </div>

```

### 防止過多預先載入

使用 `<Link>` 元件時，Next.js 會自動預先載入視窗範圍內的連結。

可能有情況您希望防止這樣做，以避免不必要的資源使用，例如渲染大量連結列表時（例如無限捲動表格）。

您可以將 `<Link>` 元件的 `prefetch` Prop 設定為 `false` 來停用預先載入。

```tsx filename="app/ui/no-prefetch-link.tsx" switcher
<Link prefetch={false} href={`/blog/${post.id}`}>
  {post.title}
</Link>
```

然而，這表示靜態路由僅會在點擊時獲取，而動態路由會在導航前等待伺服器渲染。

為了在不完全停用預先載入的情況下減少資源使用，您可以將預先載入延遲至用戶在連結上懸停時進行。這僅針對用戶可能訪問的連結。

```tsx filename="app/ui/hover-prefetch-link.tsx" switcher
'use client'

import Link from 'next/link'
import { useState } from 'react'

export function HoverPrefetchLink({
  href,
  children,
}: {
  href: string
  children: React.ReactNode
}) {
  const [active, setActive] = useState(false)

  return (
    <Link
      href={href}
      prefetch={active ? null : false}
      onMouseEnter={() => setActive(true)}
    >
      {children}
    </Link>
  )
}
```

