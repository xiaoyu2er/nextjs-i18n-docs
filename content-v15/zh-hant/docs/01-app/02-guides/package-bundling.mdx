打包外部套件可以顯著提升應用程式的效能。<AppOnly>預設情況下，在伺服器組件 (Server Components) 和路由處理器 (Route Handlers) 中匯入的套件會由 Next.js 自動打包。本頁將引導您分析並進一步優化套件打包。</AppOnly> <PagesOnly>預設情況下，匯入應用程式的套件不會進行打包。如果外部套件未預先打包（例如從 monorepo 或 `node_modules` 匯入），這可能會影響效能或無法正常運作。本頁將引導您分析並配置套件打包。</PagesOnly>

## 分析 JavaScript 打包

[`@next/bundle-analyzer`](https://www.npmjs.com/package/@next/bundle-analyzer) 是 Next.js 的插件，可協助您管理應用程式打包的大小。它會生成一份視覺報告，顯示每個套件及其相依套件的規模。您可以利用此資訊來移除大型相依套件、分割，或 [延遲載入](/docs/app/guides/lazy-loading) 您的程式碼。

### 安裝

執行以下命令以安裝插件：

```bash
npm i @next/bundle-analyzer
# or
yarn add @next/bundle-analyzer
# or
pnpm add @next/bundle-analyzer
```

然後，在您的 `next.config.js` 中加入打包分析器的設定。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {}

const withBundleAnalyzer = require('@next/bundle-analyzer')({
  enabled: process.env.ANALYZE === 'true',
})

module.exports = withBundleAnalyzer(nextConfig)
```

### 生成報告

執行以下命令以分析您的打包：

```bash
ANALYZE=true npm run build
# or
ANALYZE=true yarn build
# or
ANALYZE=true pnpm build
```

報告會在瀏覽器開啟三個新分頁，供您檢視。定期評估應用程式的打包可協助您長期維持應用程式效能。

## 優化套件匯入

某些套件（例如圖示程式庫）可匯出數百個模組，這可能會導致開發和生產環境的效能問題。

您可以透過在 `next.config.js` 中加入 [`optimizePackageImports`](/docs/app/api-reference/config/next-config-js/optimizePackageImports) 選項，來優化這些套件的匯入方式。此選項僅會載入您實際使用的模組，同時仍提供您以具多個具名匯出 (named exports) 的語法撰寫匯入陳述式的便利性。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    optimizePackageImports: ['icon-library'],
  },
}

module.exports = nextConfig
```

Next.js 也會自動優化某些程式庫，因此無需將其包含在 optimizePackageImports 清單中。請參見 [完整清單](/docs/app/api-reference/config/next-config-js/optimizePackageImports)。

## 打包特定套件

若要打包特定套件，您可以在 `next.config.js` 中使用 [`transpilePackages`](/docs/app/api-reference/config/next-config-js/transpilePackages) 選項。此選項對於打包未預先打包的外部套件非常有用，例如從 monorepo 匯入或從 `node_modules` 匯入時。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  transpilePackages: ['package-name'],
}

module.exports = nextConfig
```

## 打包所有套件

若要自動打包所有套件（App Router 的預設行為），您可以在 `next.config.js` 中使用 [`bundlePagesRouterDependencies`](/docs/pages/api-reference/config/next-config-js/bundlePagesRouterDependencies) 選項。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  bundlePagesRouterDependencies: true,
}

module.exports = nextConfig
```

## 選擇將特定套件排除在打包之外

如果您啟用了 [`bundlePagesRouterDependencies`](/docs/pages/api-reference/config/next-config-js/bundlePagesRouterDependencies) 選項，可以使用 `next.config.js` 中的 [`serverExternalPackages`](/docs/pages/api-reference/config/next-config-js/serverExternalPackages) 選項，將特定套件選擇性地排除在自動打包之外：

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  // Automatically bundle external packages in the Pages Router:
  bundlePagesRouterDependencies: true,
  // Opt specific packages out of bundling for both App and Pages Router:
  serverExternalPackages: ['package-name'],
}

module.exports = nextConfig
```

</PagesOnly>

## 選擇將特定套件排除在打包之外

由於在伺服器組件 (Server Components) 和路由處理器 (Route Handlers) 中匯入的套件已由 Next.js 自動打包，因此您可以透過 `next.config.js` 中的 [`serverExternalPackages`](/docs/app/api-reference/config/next-config-js/serverExternalPackages) 選項，選擇將特定套件排除在打包之外。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  serverExternalPackages: ['package-name'],
}

module.exports = nextConfig
```

Next.js 包含目前正進行相容性處理並自動排除的熱門套件清單。請參見 [完整清單](/docs/app/api-reference/config/next-config-js/serverExternalPackages)。

</AppOnly>
