title: 碼模組 (Codemods)
description: 使用碼模組 (Codemods) 在有新功能釋出時升級您的 Next.js 程式碼庫。
---

碼模組 (Codemods) 是針對您的程式碼庫進行的程式化轉換。這使得大量更改能夠程式化套用，而無需手動逐檔案檢查。

Next.js 提供 Codemod 轉換功能，以協助在 API 更新或棄用時升級您的 Next.js 程式碼庫。

## 使用方法

在終端機中，切換 (`cd`) 到專案資料夾，然後執行：

```bash filename="Terminal"
npx @next/codemod <transform> <path>
```

替換 `<transform>` 和 `<path>` 為適當的值。

- `transform` - 轉換的名稱
- `path` - 要轉換的檔案或目錄
- `--dry` 執行模擬運行，不編輯任何程式碼
- `--print` 列印變更後的輸出以供比對

## 代碼轉換 (Codemods)

### 16.0

#### 從 `next lint` 遷移至 ESLint CLI

##### `next-lint-to-eslint-cli`

```bash filename="Terminal"
npx @next/codemod@canary next-lint-to-eslint-cli .
```

此碼模組將專案從使用 `next lint` 遷移至使用 ESLint CLI 和您的本地 ESLint 設定。它：

- 建立 `eslint.config.mjs` 檔案，包含 Next.js 推薦設定
- 更新 `package.json` 中的腳本，將 `next lint` 改為 `eslint .`
- 將必要的 ESLint 相依套件新增到 `package.json` 中
- 若找到現有的 ESLint 設定，則予以保留

例如：

```json filename="package.json"
{
  "scripts": {
    "lint": "next lint"
  }
}
```

變成：

```json filename="package.json"
{
  "scripts": {
    "lint": "eslint ."
  }
}
```

並建立：

```js filename="eslint.config.mjs"
import { dirname } from 'path'
import { fileURLToPath } from 'url'
import { FlatCompat } from '@eslint/eslintrc'

const __filename = fileURLToPath(import.meta.url)
const __dirname = dirname(__filename)

const compat = new FlatCompat({
  baseDirectory: __dirname,
})

const eslintConfig = [
  ...compat.extends('next/core-web-vitals', 'next/typescript'),
  {
    ignores: [
      'node_modules/**',
      '.next/**',
      'out/**',
      'build/**',
      'next-env.d.ts',
    ],
  },
]

export default eslintConfig
```

### 15.0

#### 將 App Router 路由區段 Config 的 `runtime` 值從 `experimental-edge` 轉換為 `edge`

##### `app-dir-runtime-config-experimental-edge`

> **注意**: 此碼模組專為 App Router 設計。

```bash filename="Terminal"
npx @next/codemod@latest app-dir-runtime-config-experimental-edge .
```

此碼模組轉換 [路由區段 Config `runtime`](https://nextjs.org/docs/app/api-reference/file-conventions/route-segment-config#runtime) 值 `experimental-edge` 為 `edge`。

例如：

```ts
export const runtime = 'experimental-edge'
```

轉換為：

```ts
export const runtime = 'edge'
```

#### 遷移至異步動態 API

採用動態渲染且原先支援同步存取的 API 現在已改為異步運作。您可以在 [升級指南](/docs/app/guides/upgrading/version-15) 中閱讀有關此重大變更的更多詳細資訊。

##### `next-async-request-api`

```bash filename="Terminal"
npx @next/codemod@latest next-async-request-api .
```

此碼模組將現在已改為異步運作的動態 APIs (從 `next/headers` 的 `cookies()`、`headers()` 和 `draftMode()`) 轉換為正確地使用 `await` 或用 `React.use()` 包裝 (若適用)。在無法自動遷移的情況下，碼模組會新增類型轉換 (若是 TypeScript 檔案) 或註解，以通知使用者需要手動檢視與更新。

例如：

```tsx
import { cookies, headers } from 'next/headers'
const token = cookies().get('token')

function useToken() {
  const token = cookies().get('token')
  return token
}

export default function Page() {
  const name = cookies().get('name')
}

function getHeader() {
  return headers().get('x-foo')
}
```

轉換為：

```tsx
import { use } from 'react'
import {
  cookies,
  headers,
  type UnsafeUnwrappedCookies,
  type UnsafeUnwrappedHeaders,
} from 'next/headers'
const token = (cookies() as unknown as UnsafeUnwrappedCookies).get('token')

function useToken() {
  const token = use(cookies()).get('token')
  return token
}

export default async function Page() {
  const name = (await cookies()).get('name')
}

function getHeader() {
  return (headers() as unknown as UnsafeUnwrappedHeaders).get('x-foo')
}
```

當我們在頁面 / 路由項目 (`page.js`、`layout.js`、`route.js` 或 `default.js`) 或 `generateMetadata` / `generateViewport` APIs 中偵測到 `params` 或 `searchParams` 屬性存取時，它會嘗試將調用站點從同步函數轉換為異步函數，並等待屬性存取。如果無法改為異步 (例如客戶端元件)，它將使用 `React.use` 來解包裝 Promise 。

例如：

```tsx
// page.tsx
export default function Page({
  params,
  searchParams,
}: {
  params: { slug: string }
  searchParams: { [key: string]: string | string[] | undefined }
}) {
  const { value } = searchParams
  if (value === 'foo') {
    // ...
  }
}

export function generateMetadata({ params }: { params: { slug: string } }) {
  const { slug } = params
  return {
    title: `My Page - ${slug}`,
  }
}
```

轉換為：

```tsx
// page.tsx
export default async function Page(props: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const searchParams = await props.searchParams
  const { value } = searchParams
  if (value === 'foo') {
    // ...
  }
}

export async function generateMetadata(props: {
  params: Promise<{ slug: string }>
}) {
  const params = await props.params
  const { slug } = params
  return {
    title: `My Page - ${slug}`,
  }
}
```

> **好消息**: 當此碼模組識別到可能需要手動介入的位置，但我們無法確定確切修復方法時，它會新增註解或類型轉換至程式碼，通知使用者需要手動更新。這些註解會以 **@next/codemod** 為前綴，類型轉換會以 `UnsafeUnwrapped` 為前綴。
> 直到這些註解被明確移除之前，您的建置將出錯。[閱讀更多](/docs/messages/sync-dynamic-apis)。

#### 將 `NextRequest` 的 `geo` 和 `ip` 屬性轉換為 `@vercel/functions`

##### `next-request-geo-ip`

```bash filename="Terminal"
npx @next/codemod@latest next-request-geo-ip .
```

此碼模組會安裝 `@vercel/functions`，並與對應的 `@vercel/functions` 功能一併轉換 `NextRequest` 的 `geo` 和 `ip` 屬性。

例如：

```ts
import type { NextRequest } from 'next/server'

export function GET(req: NextRequest) {
  const { geo, ip } = req
}
```

轉換為：

```ts
import type { NextRequest } from 'next/server'
import { geolocation, ipAddress } from '@vercel/functions'

export function GET(req: NextRequest) {
  const geo = geolocation(req)
  const ip = ipAddress(req)
}
```


### 13.0

#### 重新命名 Next Image 匯入

##### `next-image-to-legacy-image`

```bash filename="Terminal"
npx @next/codemod@latest next-image-to-legacy-image .
```

安全地重新命名現有 Next.js 10、11 或 12 應用程式中的 `next/image` 匯入為 Next.js 13 的 `next/legacy/image`。同時將 `next/future/image` 重新命名為 `next/image`。

例如：

```jsx filename="pages/index.js"
import Image1 from 'next/image'
import Image2 from 'next/future/image'

export default function Home() {
  return (
    <div>
      <Image1 src="/test.jpg" width="200" height="300" />
      <Image2 src="/test.png" width="500" height="400" />
    </div>
  )
}
```

轉換為：

```jsx filename="pages/index.js"
// 'next/image' becomes 'next/legacy/image'
import Image1 from 'next/legacy/image'
// 'next/future/image' becomes 'next/image'
import Image2 from 'next/image'

export default function Home() {
  return (
    <div>
      <Image1 src="/test.jpg" width="200" height="300" />
      <Image2 src="/test.png" width="500" height="400" />
    </div>
  )
}
```

#### 從 Link 元件中移除 `<a>` 標籤

##### `new-link`

```bash filename="Terminal"
npx @next/codemod@latest new-link .
```

<AppOnly>

移除 [Link 元件](/docs/app/api-reference/components/link) 內的 `<a>` 標籤，或為無法自動修復的 Link 新增 `legacyBehavior` prop。

</AppOnly>

<PagesOnly>

移除 [Link 元件](/docs/pages/api-reference/components/link) 內的 `<a>` 標籤，或為無法自動修復的 Link 新增 `legacyBehavior` prop。

</PagesOnly>

例如：

```jsx
<Link href="/about">
  <a>About</a>
</Link>
// transforms into
<Link href="/about">
  About
</Link>

<Link href="/about">
  <a onClick={() => console.log('clicked')}>About</a>
</Link>
// transforms into
<Link href="/about" onClick={() => console.log('clicked')}>
  About
</Link>
```

在無法套用自動修復的情況下，會新增 `legacyBehavior` prop。這允許您的應用程式針對特定連結沿用舊版行為繼續運作。

```jsx
const Component = () => <a>About</a>

<Link href="/about">
  <Component />
</Link>
// becomes
<Link href="/about" legacyBehavior>
  <Component />
</Link>
```

### 9

#### 將匿名元件轉換為命名元件

##### `name-default-component`

```bash filename="Terminal"
npx @next/codemod name-default-component
```

**版本 9 及以上。**

將匿名元件轉換為命名元件，以確保它們能與 [Fast Refresh](https://nextjs.org/blog/next-9-4#fast-refresh) 運作。

例如：

```jsx filename="my-component.js"
export default function () {
  return <div>Hello World</div>
}
```

轉換為：

```jsx filename="my-component.js"
export default function MyComponent() {
  return <div>Hello World</div>
}
```

元件將根據檔案名稱擁有小駝峰命名法，且也支援箭頭函數。

```js
export default function Home() {
```

### 6

#### 使用 `withRouter`

##### `url-to-withrouter`

```bash filename="Terminal"
npx @next/codemod url-to-withrouter
```

將頂級頁面中已棄用的自動注入 `url` 屬性轉換為使用 `withRouter` 及其注入的 `router` 屬性。閱讀更多：[https://nextjs.org/docs/messages/url-deprecated](/docs/messages/url-deprecated)

例如：

```js filename="From"
import React from 'react'
export default class extends React.Component {
  render() {
    const { pathname } = this.props.url
    return <div>Current pathname: {pathname}</div>
  }
}
```

```js filename="To"
import React from 'react'
import { withRouter } from 'next/router'
export default withRouter(
  class extends React.Component {
    render() {
      const { pathname } = this.props.router
      return <div>Current pathname: {pathname}</div>
    }
  }
)
```

只是其中一種情況。所有已轉換 (且已測試) 的情況均可在 [`__testfixtures__` 目錄](https://github.com/vercel/next.js/tree/canary/packages/next-codemod/transforms/__testfixtures__/url-to-withrouter) 中找到。
