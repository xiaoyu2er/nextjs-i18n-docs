> **值得注意**：從 Next.js 13.5.1 開始，嚴格模式 (Strict Mode) 在使用 `app` 路由時預設為 `true`，因此上述配置僅適用於 `pages`。您仍可透過設定 `reactStrictMode: false` 來停用嚴格模式 (Strict Mode)。

> **建議**：我們強烈建議您在 Next.js 應用程式中啟用嚴格模式 (Strict Mode)，以便更好地為 React 的未來做好準備。

React 的 [嚴格模式 (Strict Mode)](https://react.dev/reference/react/StrictMode) 是僅限開發模式的功能，強調應用程序中的潛在問題。這有助於識別不安全的生命週期、舊版 API 使用情況以及其他許多功能。

Next.js 執行環境已符合嚴格模式 (Strict Mode) 標準。要選擇加入嚴格模式 (Strict Mode)，請在您的 `next.config.js` 中配置以下選項：

```js filename="next.config.js"
module.exports = {
  reactStrictMode: true,
}
```

如果您或您的團隊尚未準備好在整個應用程式中使用嚴格模式 (Strict Mode)，沒關係！您可以使用 `<React.StrictMode>` 以頁面為基礎逐步進行轉換。
