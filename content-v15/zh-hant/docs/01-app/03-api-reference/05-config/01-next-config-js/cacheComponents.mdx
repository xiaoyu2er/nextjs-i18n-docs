cacheComponents 標誌是 Next.js 中的一個實驗性功能，它會導致 App Router 中的數據獲取操作在未經明確快取的情況下被排除在預渲染之外。這對於優化伺服器元件 (Server Components) 中的動態數據獲取效能很有幫助。

如果您的應用程式需要在執行階段進行即時數據獲取，而非從預渲染快取中提供內容，這將非常有用。

預計應與 [`use cache`](/docs/app/api-reference/directives/use-cache) 搭配使用，使得預設情況下您的數據獲取在執行階段發生，除非您以頁面、函式或元件層級使用 use cache 定義應用程式的特定部分來進行快取。

## 使用方法

要啟用 cacheComponents 標誌，請在 next.config.ts 檔案的 experimental 部分將其設定為 true ：

```ts filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    cacheComponents: true,
  },
}

export default nextConfig
```

啟用 cacheComponents 後，您可以使用下列快取函式與設定：

- [使用快取 (use cache) 指令](/docs/app/api-reference/directives/use-cache)
- [`cacheLife` 函式](/docs/app/api-reference/config/next-config-js/cacheLife) 搭配 `use cache` 使用
- [`cacheTag` 函式](/docs/app/api-reference/functions/cacheTag)

## 註記

- 雖然 cacheComponents 可透過確保執行期間進行即時數據獲取來優化效能，但與提供預渲染內容相比，它也可能會引入額外的延遲。
