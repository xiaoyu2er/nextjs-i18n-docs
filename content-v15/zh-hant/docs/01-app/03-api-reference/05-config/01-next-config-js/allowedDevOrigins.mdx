Next.js 在開發時不會自動阻止跨來源請求 (cross-origin requests)，但將預設在 Next.js 的未來主要版本中阻止，以防止開發模式下可用的內部資源/端點遭未經授權請求。

若要配置 Next.js 應用程式允許來自伺服器初始化主機名稱（預設為 `localhost`）以外的來源請求，可使用 `allowedDevOrigins` 配置選項。

`allowedDevOrigins` 可設定開發模式下可使用的額外來源。例如：將 `local-origin.dev` 替代 `localhost`，請開放 `next.config.js` 並添加 `allowedDevOrigins` 配置：

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```
