漸進式預渲染 (PPR) 允許您在同一個路由中結合靜態和動態組件。了解更多關於 [漸進式預渲染 (PPR)](/docs/app/getting-started/partial-prerendering)。

## 使用漸進式預渲染 (PPR)

### 增量採用 (版本 15)

在 Next.js 15 中，您可以透過在 `next.config.js` 中將 `ppr` 選項設定為 `incremental`，並在檔案頂部匯出 `experimental_ppr` [路由配置選項](/docs/app/api-reference/file-conventions/route-segment-config)，來在 [佈局 (Layout)](/docs/app/api-reference/file-conventions/layout) 和 [頁面 (Page)](/docs/app/api-reference/file-conventions/page) 中漸進式採用漸進式預渲染 (PPR)：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    ppr: 'incremental',
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    ppr: 'incremental',
  },
}

module.exports = nextConfig
```

```tsx filename="app/page.tsx" switcher
import { Suspense } from "react"
import { StaticComponent, DynamicComponent, Fallback } from "@/app/ui"

export const experimental_ppr = true

export default function Page() {
  return {
     <>
      <StaticComponent />
      <Suspense fallback={<Fallback />}>
        <DynamicComponent />
      </Suspense>
     </>
  };
}
```

```jsx filename="app/page.js" switcher
import { Suspense } from "react"
import { StaticComponent, DynamicComponent, Fallback } from "@/app/ui"

export const experimental_ppr = true

export default function Page() {
  return {
     <>
      <StaticComponent />
      <Suspense fallback={<Fallback />}>
        <DynamicComponent />
      </Suspense>
     </>
  };
}
```

> **小提示**:
>
> - 若路由未設定 `experimental_ppr`，預設值為 `false`，且不會使用 PPR 進行預渲染。您需為每個路由明確選擇加入 PPR。
> - `experimental_ppr` 將套用於路由片段的所有子項，包括嵌套的佈局 (Layouts) 和頁面 (Pages)。您無需為每個檔案新增它，只需在路由的頂層片段新增即可。
> - 若要在子片段中停用 PPR，您可以在子片段中將 `experimental_ppr` 設定為 `false`。

| 版本   | 變更                                     |
| --------- | ------------------------------------------- |
| `v15.0.0` | 引入實驗性 `incremental` 值              |
| `v14.0.0` | 引入實驗性 `ppr`                       |
