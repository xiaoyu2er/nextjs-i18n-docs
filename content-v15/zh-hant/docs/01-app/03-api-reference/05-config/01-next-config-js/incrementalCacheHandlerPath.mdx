如果您希望將快取頁面和資料持久化到耐久儲存，或在多個容器或您的 Next.js 應用程式的實體之間共用快取，您可以配置 Next.js 快取位置。

```js filename="next.config.js"
module.exports = {
  cacheHandler: require.resolve('./cache-handler.js'),
  cacheMaxMemorySize: 0, // disable default in-memory caching
}
```

查看 [自訂快取處理器](/docs/app/guides/self-hosting#configuring-caching) 的範例，並了解更多關於實作的資訊。

## API 參考

快取處理器可實作以下方法：`get`、`set`、`revalidateTag` 和 `resetRequestCache`。

### `get()`

| 參數 | 類型       | 說明                  |
| --------- | -------- | ---------------------------- |
| `key`     | `string` | 快取值的鍵。 |

若找不到則回傳快取值或 `null`。

### `set()`

| 參數 | 類型             | 說明                              |
| --------- | -------------- | -------------------------------- |
| `key`     | `string`       | 儲存資料的鍵。 |

回傳 `Promise<void>`。

### `revalidateTag()`

| 參數 | 類型                   | 說明                   |
| --------- | ---------------------- | ----------------------------- |
| `tag`     | `string` or `string[]` | 要重新驗證的快取標籤。 |

回傳 `Promise<void>`。了解更多關於 [重新驗證資料](/docs/app/guides/incremental-static-regeneration) 或 [`revalidateTag()`](/docs/app/api-reference/functions/revalidateTag) 函式。

### `resetRequestCache()`

此方法會在下一個請求之前，重置單一請求的臨時記憶體快取。

回傳 `void`。

**小知識：**

- `revalidatePath` 是快取標籤之上的便利層。呼叫 `revalidatePath` 將呼叫您的 `revalidateTag` 函式，您可以選擇是否要根據路徑為快取鍵加上標籤。

## 平台支援

| 部署選項                                                           | 支援狀態          |
| ------------------------------------------------------------------- | ----------------- |
| [Node.js 伺服器](/docs/app/getting-started/deploying#nodejs-server) | 是               |
| [Docker 容器](/docs/app/getting-started/deploying#docker)      | 是               |
| [靜態匯出](/docs/app/getting-started/deploying#static-export)  | 否                |
| [配接器](/docs/app/getting-started/deploying#adapters)            | 依平台而定 |

了解如何 [設定 ISR](/docs/app/guides/self-hosting#caching-and-isr) 時自行託管 Next.js。
