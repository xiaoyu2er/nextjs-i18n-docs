預設情況下，Next.js 會將帶有尾隨斜線的 URL 重定向到不帶尾隨斜線的對應 URL。例如 `/about/` 會重定向到 `/about`。您可以配置此行為以相反方式運作，即不帶尾隨斜線的 URL 會被重定向到帶有尾隨斜線的對應 URL。

開啟 `next.config.js` 並新增 `trailingSlash` 配置:

```js filename="next.config.js"
module.exports = {
  trailingSlash: true,
}
```

設定此選項後，類似 `/about` 的 URL 會重定向到 `/about/`。

使用 `trailingSlash: true` 時，某些 URL 是例外且將不會新增尾隨斜線：

- 靜態文件 URL，例如帶有副檔名的文件。
- 任何位於 `.well-known/` 下的路徑。

例如，以下 URL 將保持不變： `/file.txt`, `images/photos/picture.png`, 和 `.well-known/subfolder/config.json`。

使用 [`output: "export"`](/docs/app/guides/static-exports) 配置時，`/about` 頁面將輸出 `/about/index.html`（而非預設的 `/about.html`）。
