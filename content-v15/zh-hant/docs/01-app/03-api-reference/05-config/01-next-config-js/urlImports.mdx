URL 導入 (urlImports) 是一種實驗性功能，允許您直接從外部伺服器匯入模組（而非從本機磁碟）。

> **警告**：僅使用您信任能在您的機器上下載並執行的網域。在該功能標記為穩定之前，請謹慎行事，並保持警覺。

若要參與，請在 `next.config.js` 中加入允許的 URL 前綴：

```js filename="next.config.js"
module.exports = {
  experimental: {
    urlImports: ['https://example.com/assets/', 'https://cdn.skypack.dev'],
  },
}
```

接著，您就可以直接從網址匯入模組：

```js
import { a, b, c } from 'https://example.com/assets/some/module.js'
```

URL 導入 (urlImports) 可在正常使用套件匯入的任何地方使用。

## 安全模型 (Security Model)

此功能設計時將 **安全性視為首要任務**。一開始，我們加入了一個實驗性旗標，強制您明確允許接受 URL 導入的網域。我們正致力於進一步限制 URL 導入，使其使用 [Edge 執行環境 (Edge Runtime)](/docs/app/api-reference/edge) 在瀏覽器沙盒中執行。

## 鎖定檔案 (Lockfile)

使用 URL 導入時，Next.js 會建立包含鎖定檔案及抓取資源的 `next.lock` 目錄。
此目錄**必須提交至 Git**，不應被 `.gitignore` 忽略。

- 執行 `next dev` 時，Next.js 會下載並將所有新發現的 URL 導入新增至您的鎖定檔案。
- 執行 `next build` 時，Next.js 將僅使用鎖定檔案來建置生產環境的應用程式。

通常不需要網路請求，且過時的鎖定檔案會致使建置失敗。
一項例外是回應 `Cache-Control: no-cache` 的資源。
這些資源在鎖定檔案中會有 `no-cache` 項目，並且每次建置時都將來自網路抓取。

## 範例

### Skypack

```js
import confetti from 'https://cdn.skypack.dev/canvas-confetti'
import { useEffect } from 'react'

export default () => {
  useEffect(() => {
    confetti()
  })
  return <p>Hello</p>
}
```

### 靜態圖片匯入 (Static Image Imports)

```js
import Image from 'next/image'
import logo from 'https://example.com/assets/logo.png'

export default () => (
  <div>
    <Image src={logo} placeholder="blur" />
  </div>
)
```

### CSS 中的網址

```css
.className {
  background: url('https://example.com/assets/hero.jpg');
}
```

### 資產匯入 (Asset Imports)

```js
const logo = new URL('https://example.com/assets/file.txt', import.meta.url)

console.log(logo.pathname)

// prints "/_next/static/media/file.a9727b5d.txt"
```
