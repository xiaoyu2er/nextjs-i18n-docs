CSS 分塊 (CSS Chunking) 是一種透過將 CSS 檔案拆分為分塊並重新排序來提升網頁應用程式效能的策略。這允許您僅加載特定路由所需的 CSS，而不是一次加載整個應用程式的 CSS。

您可以使用 `next.config.js` 檔案中的 `experimental.cssChunking` 選項來控制 CSS 檔案的分塊方式：

```tsx filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig = {
  experimental: {
    cssChunking: true, // default
  },
} satisfies NextConfig

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    cssChunking: true, // default
  },
}

module.exports = nextConfig
```

## 選項

- `**true**`（預設）: Next.js 會盡可能嘗試合併 CSS 檔案，根據匯入順序判斷檔案間的顯式與隱式相依性，以減少分塊數量，進而減少請求數。
- `**false**`: Next.js 不會嘗試合併或重新排序 CSS 檔案。
- `**'strict'**`: Next.js 會依照檔案匯入您的程式碼中的正確順序加載 CSS 檔案，這可能會導致更多的分塊和請求。

如果遇到非預期的 CSS 行為，您可以考慮使用 `'strict'`。例如，如果您在不同檔案中使用不同的匯入順序 (`a` 在 `b` 之前，或 `b` 在 `a` 之前) 匯入 `a.css` 和 `b.css`，`true` 會以任何順序合並檔案並假設它們之間沒有相依性。然而，如果 `b.css` 依賴於 `a.css`，您可能希望使用 `'strict'` 來防止檔案被合併，改為依匯入順序加載它們 - 這可能會導致更多的分塊和請求。

對於大多數應用程式，我們建議使用 `true`，因為它會導致較少的請求和更好的效能。
