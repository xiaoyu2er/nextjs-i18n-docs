「turbopack」選項可讓您自訂 [Turbopack](/docs/app/api-reference/turbopack)，以轉換不同的檔案並更改模組解析方式。

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  turbopack: {
    // ...
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  turbopack: {
    // ...
  },
}

module.exports = nextConfig
```

> **要知道**：
>
> - 針對 Next.js 的 Turbopack 不需要裝載器 (loader) 或裝載器配置即可支援內建功能。Turbopack 內建支援 CSS 和編譯現代 JavaScript，因此如果您使用 `@babel/preset-env`，則不需要 `css-loader`、`postcss-loader` 或 `babel-loader`。


## 參考

### 選項

下列選項可用於 `turbo` 配置：

| 選項 | 描述 |
| ------------------- | ----------------------------------------------------------------------- |
| `root` | 設定應用程式根目錄。應為絕對路徑。 |
| `rules` | 列出支援 webpack 裝載器 (loader) 清單，在執行 Turbopack 時套用。 |
| `resolveAlias` | 將別名匯入對應至要載入的模組。 |
| `resolveExtensions` | 匯入檔案時解析的副檔名列表。 |

### 支援的裝載器 (loader)

下列裝載器已測試可與 Turbopack 的 webpack 裝載器 (loader) 實作配合運作，但即使未列於此處，許多其他 webpack 裝載器也應能正常運作：

- [`babel-loader`](https://www.npmjs.com/package/babel-loader)
- [`@svgr/webpack`](https://www.npmjs.com/package/@svgr/webpack)
- [`svg-inline-loader`](https://www.npmjs.com/package/svg-inline-loader)
- [`yaml-loader`](https://www.npmjs.com/package/yaml-loader)
- [`string-replace-loader`](https://www.npmjs.com/package/string-replace-loader)
- [`raw-loader`](https://www.npmjs.com/package/raw-loader)
- [`sass-loader`](https://www.npmjs.com/package/sass-loader)
- [`graphql-tag/loader`](https://www.npmjs.com/package/graphql-tag)

## 範例

### 根目錄

Turbopack 使用根目錄來解析模組。專案根目錄之外的檔案將不會被解析。

Next.js 會自動偵測您的專案根目錄。這是透過尋找以下檔案之一來完成的：

- `pnpm-lock.yaml`
- `package-lock.json`
- `yarn.lock`
- `bun.lock`
- `bun.lockb`

如果您有不同的專案結構，例如如果您不使用工作區 (workspaces)，您可以手動設定 `root` 選項：

```js filename="next.config.js"
const path = require('path')
module.exports = {
  turbopack: {
    root: path.join(__dirname, '..'),
  },
}
```

### 設定 webpack 裝載器 (loader)

如果您需要超過內建能力的裝載器 (loader) 支援，許多 webpack 裝載器已經與 Turbopack 配合運作。目前有一些限制：

- 僅實現了 webpack 裝載器 API 的核心子集。目前，對於一些熱門裝載器已足夠覆蓋範圍，我們將擴展未來的 API 支援。
- 僅支援傳回 JavaScript 程式碼的裝載器。轉換樣式表或圖片等檔案的裝載器目前並不支援。
- 傳遞給 webpack 裝載器的選項必須是純 JavaScript 原始值、物件和陣列。例如，無法將 `require()` 外掛模組作為選項值傳遞。

要設定裝載器，請在 `next.config.js` 中加入您已安裝的裝載器名稱及任何選項，並將副檔名對應至裝載器清單。

下方為使用 [`@svgr/webpack`](https://www.npmjs.com/package/@svgr/webpack) 裝載器的範例，此功能可讓您在匯入 `.svg` 檔案時將其渲染為 React 元件。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    rules: {
      '*.svg': {
        loaders: ['@svgr/webpack'],
        as: '*.js',
      },
    },
  },
}
```

對於需要配置選項的裝載器，您可以使用物件格式來替代字串：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    rules: {
      '*.svg': {
        loaders: [
          {
            loader: '@svgr/webpack',
            options: {
              icon: true,
            },
          },
        ],
        as: '*.js',
      },
    },
  },
}
```

> **要知道**：在 Next.js 13.4.4 版本之前，`turbo.rules` 名稱是 `turbo.loaders`，且僅接受 `.mdx` 之類的副檔名，而非 `*.mdx`。


### 解析別名

Turbopack 可透過別名設定來修改模組解析，類似於 webpack 的 [`resolve.alias`](https://webpack.js.org/configuration/resolve/#resolvealias) 設定。

要配置解析別名，請在 `next.config.js` 中將匯入模式對應至其新目的地：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    resolveAlias: {
      underscore: 'lodash',
      mocha: { browser: 'mocha/browser-entry.js' },
    },
  },
}
```

這將 `underscore` 程式集的匯入別名設定為 `lodash` 程式集。換句話說，`import underscore from 'underscore'` 將載入 `lodash` 模組而非 `underscore`。

Turbopack 也透過此欄位支援條件式別名設定，類似於 Node.js 的 [條件式匯出](https://nodejs.org/docs/latest-v18.x/api/packages.html#conditional-exports)。目前僅支援 `browser` 條件。在上述情況下，當 Turbopack 目標為瀏覽器環境時，`mocha` 模組的匯入將被別名設定為 `mocha/browser-entry.js`。

### 解析自定義副檔名

Turbopack 可配置解析具有自定義副檔名的模組，類似於 webpack 的 [`resolve.extensions`](https://webpack.js.org/configuration/resolve/#resolveextensions) 設定。

要配置解析副檔名，請在 `next.config.js` 中使用 `resolveExtensions` 欄位：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    resolveExtensions: ['.mdx', '.tsx', '.ts', '.jsx', '.js', '.mjs', '.json'],
  },
}
```

這會以提供的清單覆蓋原始解析副檔名。請務必包含預設副檔名。

關於如何將應用程式從 webpack 迁移至 Turbopack 的更多資訊和指導，請參閱 [Turbopack 關於 webpack 相容性的文件](https://turbo.build/pack/docs/migrating-from-webpack)。
