當專案中偵測到 ESLint 時，如果存在錯誤，Next.js 會使 **生產環境構建 (production build)** (`next build`) 失敗。

如果您希望即使應用程式存在 ESLint 錯誤，Next.js 仍能產生生產代碼，您可以完全停用內建的 **代碼检查 (linting)** 步驟。除非您已將 ESLint 配置為在工作流程的其他部分運行（例如，在 CI 或 pre-commit hook），否則不建議此做法。

開啟 `next.config.js` 並在 `eslint` 配置中啟用 `ignoreDuringBuilds` 選項：

```js filename="next.config.js"
module.exports = {
  eslint: {
    // Warning: This allows production builds to successfully complete even if
    // your project has ESLint errors.
    ignoreDuringBuilds: true,
  },
}
```
