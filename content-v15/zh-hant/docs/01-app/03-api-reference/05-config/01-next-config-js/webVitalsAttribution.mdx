當除錯與 Web Vitals 相關的議題時，如果能定位問題的來源通常很有幫助。
例如，在累積佈局偏移 (CLS) 的情況下，我們可能想知道在發生單次最大佈局偏移時第一個發生位移的元素。
或者，在最大內容繪製 (LCP) 的情況下，我們可能想識別對應頁面 LCP 的元素。
如果 LCP 元素是圖片，知道該圖片資源的 URL 可以幫助我們定位需要優化的資源。

定位 Web Vitals 分數的最大貢獻者，亦即 [歸因 (attribution)](https://github.com/GoogleChrome/web-vitals/blob/4ca38ae64b8d1e899028c692f94d4c56acfc996c/README.md#attribution)，
讓我們能夠取得更深入的資訊，例如 [PerformanceEventTiming](https://developer.mozilla.org/docs/Web/API/PerformanceEventTiming)、[PerformanceNavigationTiming](https://developer.mozilla.org/docs/Web/API/PerformanceNavigationTiming) 與 [PerformanceResourceTiming](https://developer.mozilla.org/docs/Web/API/PerformanceResourceTiming) 的條目。

歸因 (Attribution) 預設在 Next.js 中是停用狀態，但可以在 `next.config.js` 中透過指定以下內容來**針對每項指標**啟用。

```js filename="next.config.js"
module.exports = {
  experimental: {
    webVitalsAttribution: ['CLS', 'LCP'],
  },
}
```

有效的歸因值為 [`NextWebVitalsMetric`](https://github.com/vercel/next.js/blob/442378d21dd56d6e769863eb8c2cb521a463a2e0/packages/next/shared/lib/utils.ts#L43) 型別中指定的所有 `web-vitals` 指標。
