## 設置內容傳遞網路 (CDN)

要設置 [內容傳遞網路 (CDN)](https://en.wikipedia.org/wiki/Content_delivery_network) ，您可以設置 assetPrefix 並配置 CDN 的 Origin，使其解析為 Next.js 託管的網域。

打開 `next.config.mjs` 並基於 [phase](/docs/app/api-reference/config/next-config-js#async-configuration) 添加 `assetPrefix` 設定：

```js filename="next.config.mjs"
// @ts-check
import { PHASE_DEVELOPMENT_SERVER } from 'next/constants'

export default (phase) => {
  const isDev = phase === PHASE_DEVELOPMENT_SERVER
  /**
   * @type {import('next').NextConfig}
   */
  const nextConfig = {
    assetPrefix: isDev ? undefined : 'https://cdn.mydomain.com',
  }
  return nextConfig
}
```

Next.js 將自動使用您的 assetPrefix，針對其從 `/_next/` 路徑 (` .next/static/` 資料夾 ) 載入的 JavaScript 和 CSS 文件。例如，對於上述配置，以下針對 JS 程式碼塊的請求：

```
/_next/static/chunks/4b9b41aaa062cbbfeff4add70f256968c51ece5d.4d708494b3aed70c04f0.js
```

則會變成：

```
https://cdn.mydomain.com/_next/static/chunks/4b9b41aaa062cbbfeff4add70f256968c51ece5d.4d708494b3aed70c04f0.js
```

將您的檔案上傳至給定 CDN 的確切配置將取決於您選擇的 CDN。您唯一需要在 CDN 上託管的資料夾是 `.next/static/` 的內容，根據上述 URL 請求，應該上傳為 `/_next/static/`。**請勿上傳您 `.next/` 資料夾的其餘部分**，因為您不應將伺服器代碼和其他配置暴露在公眾面前。

雖然 `assetPrefix` 涵蓋了對 `/_next/static` 的請求，但它不影響以下路徑：

<AppOnly>

- [public](/docs/app/api-reference/file-conventions/public-folder) 資料夾中的檔案；如果您希望透過 CDN 提供這些資源，您必須自己引入前綴

</AppOnly>

<PagesOnly>

- [public](/docs/pages/api-reference/file-conventions/public-folder) 資料夾中的檔案；如果您希望透過 CDN 提供這些資源，您必須自己引入前綴
- 針對 `getServerSideProps` 頁面的 `/_next/data/` 請求。由於這些請求不是靜態的，它們將總是針對主網域進行。
- 針對 `getStaticProps` 頁面的 `/_next/data/` 請求。為了支援 [遞增式靜態生成 (Incremental Static Generation)](/docs/pages/guides/incremental-static-regeneration)，即使您不使用它 (為了保持一致性)，這些請求將總是針對主網域進行。

</PagesOnly>
