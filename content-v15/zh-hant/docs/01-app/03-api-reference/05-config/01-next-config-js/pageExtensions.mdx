<AppOnly>

預設情況下，Next.js 接受具有下列副檔名的檔案： `.tsx`, `.ts`, `.jsx`, `.js`。這可以進行修改以允許其他副檔名，例如 markdown（ `.md`, `.mdx` ）。

```js filename="next.config.js"
const withMDX = require('@next/mdx')()

/** @type {import('next').NextConfig} */
const nextConfig = {
  pageExtensions: ['js', 'jsx', 'ts', 'tsx', 'md', 'mdx'],
}

module.exports = withMDX(nextConfig)
```

</AppOnly>

<PagesOnly>

您可以擴充 Next.js 所使用的預設頁面副檔名（ `.tsx`, `.ts`, `.jsx`, `.js` ）。在 `next.config.js` 內，新增 `pageExtensions` 設定：

```js filename="next.config.js"
module.exports = {
  pageExtensions: ['mdx', 'md', 'jsx', 'js', 'tsx', 'ts'],
}
```

修改這些值會影響 _所有_ Next.js 頁面，包含以下：

- [`middleware.js`](/docs/pages/api-reference/file-conventions/middleware)
- [`instrumentation.js`](/docs/pages/guides/instrumentation)
- `pages/_document.js`
- `pages/_app.js`
- `pages/api/`

例如，如果您將 `.ts` 頁面副檔名重新設定為 `.page.ts`，您需要重新命名頁面，例如 `middleware.page.ts`, `instrumentation.page.ts`, `_app.page.ts`。

## 在 `pages` 目錄中包含非頁面檔案

您可以在 `pages` 目錄中放置測試檔案或其他由組件使用的檔案。在 `next.config.js` 內，新增 `pageExtensions` 設定：

```js filename="next.config.js"
module.exports = {
  pageExtensions: ['page.tsx', 'page.ts', 'page.jsx', 'page.js'],
}
```

接著，重新命名您的頁面使其具有包含 `.page` 的檔案副檔名（例如：將 `MyPage.tsx` 重新命名為 `MyPage.page.tsx`）。請確保您重新命名 _所有_ Next.js 頁面，包含上述提到的檔案。

</PagesOnly>
