預設情況下，Next.js 使用 `gzip` 來壓縮渲染內容和靜態文件，在使用 `next start` 或自訂服務器時適用。這是針對未配置壓縮功能的應用程式的優化。如果透過自訂服務器在您的應用程式中_已經_配置了壓縮，Next.js 將不會添加壓縮。

您可以通過查看響應中的 [`Accept-Encoding`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Accept-Encoding)（瀏覽器接受的選項）和 [`Content-Encoding`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Encoding)（當前使用的）標頭，來檢查壓縮是否已啟用以及使用了哪種算法。

## 停用壓縮 (disabling compression)

要停用**壓縮**，請將 `compress` 配置選項設置為 `false`：

```js filename="next.config.js"
module.exports = {
  compress: false,
}
```

除非您在服務器上配置了壓縮，否則我們**不建議停用壓縮**，因為壓縮可降低頻寬使用量並提升應用程式效能。例如，如果您正在使用 [nginx](https://nginx.org/) 並希望切換到 `brotli`，請將 `compress` 選項設置為 `false`，以允許 nginx 處理壓縮。
