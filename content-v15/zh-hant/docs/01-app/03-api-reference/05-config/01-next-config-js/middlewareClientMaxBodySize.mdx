---
title: experimental.middlewareClientMaxBodySize
description: 設定在使用中間件時的最大請求體大小
version: experimental
---

當使用中間件時，Next.js 會自動複製請求體並將其緩衝在記憶體中，以便在中間件和底層路由處理器中都能進行多次讀取。為防止記憶體使用過多，此配置選項對緩衝的請求體設置了大小限制。

預設情況下，最大請求體大小為 **10MB**。如果請求體超過此限制，請求體將僅緩衝至該限制，並記錄警告指出哪個路由超過了限制。

## 選項

### 字串格式（建議）

使用可讀取的字串格式指定大小：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    middlewareClientMaxBodySize: '1mb',
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    middlewareClientMaxBodySize: '1mb',
  },
}

module.exports = nextConfig
```

支援的單位：`b`, `kb`, `mb`, `gb`

### 數字格式

或者，以數字形式指定以位元組為單位的大小：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    middlewareClientMaxBodySize: 1048576, // 1MB in bytes
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    middlewareClientMaxBodySize: 1048576, // 1MB in bytes
  },
}

module.exports = nextConfig
```

## 行為

當請求體超過配置的限制時：

1. Next.js 將僅緩衝前 N 個位元組（直至限制）
2. 控制台將記錄警告，指出哪個路由超過了限制
3. 請求將繼續正常處理，但僅提供部分請求體
4. 請求**不會**失敗或向客戶端返回錯誤

如果您的應用需要處理完整的請求體，您應該執行以下操作之一：

- 增加 `middlewareClientMaxBodySize` 限制
- 在您的應用邏輯中優雅地處理部分請求體

## 範例

```ts filename="middleware.ts"
import { NextRequest, NextResponse } from 'next/server'

export async function middleware(request: NextRequest) {
  // Next.js automatically buffers the body with the configured size limit
  // You can read the body in middleware...
  const body = await request.text()

  // If the body exceeded the limit, only partial data will be available
  console.log('Body size:', body.length)

  return NextResponse.next()
}
```

```ts filename="app/api/upload/route.ts"
import { NextRequest, NextResponse } from 'next/server'

export async function POST(request: NextRequest) {
  // ...and the body is still available in your route handler
  const body = await request.text()

  console.log('Body in route handler:', body.length)

  return NextResponse.json({ received: body.length })
}
```

## 小知識

- 此設定僅在您的應用中使用中間件時適用
- 預設的 10MB 限制旨在平衡記憶體使用與典型用例
- 限制按請求適用，而非在所有並發請求中全局適用
- 對於處理大型檔案上傳的應用，請考慮相應增加限制
