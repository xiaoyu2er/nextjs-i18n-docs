## 使用方法

taint 選項啟用對實驗性 React APIs 的支援，用於標記對象和值。此功能有助於防止敏感資料意外傳遞至客戶端。啟用後，您可以：

- [`experimental_taintObjectReference`](https://react.dev/reference/react/experimental_taintObjectReference) 標記對象引用。
- [`experimental_taintUniqueValue`](https://react.dev/reference/react/experimental_taintUniqueValue) 用於標記唯一值。

> **小知識**：啟用此標誌也會啟用 `app` 目錄的 React `experimental` 通道。

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    taint: true,
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    taint: true,
  },
}

module.exports = nextConfig
```

> **警告**：不要將 taint API 作為防止敏感資料洩露至客戶端的唯一機制。請參閱我們的 [安全建議](/blog/security-nextjs-server-components-actions)。

taint APIs 讓您能夠設定防禦機制，透過聲明式和明確地標記不允許通過伺服器端與客戶端邊界的資料。當對象或值通過伺服器端與客戶端邊界傳遞時，React 會拋出錯誤。

這對於以下情況非常有用：

- 讀取資料的方法不在您的控制範圍內
- 您必須處理非您定義的敏感資料結構
- 在 Server Component 渲染期間存取敏感資料

建議建立您的資料和 API 模型，避免將敏感資料返回到不需要它的上下文中。

## 注意事項

- 標記功能僅能透過引用追蹤對象。複製對象會建立未經標記的版本，這會失去 API 提供的所有保證。您必須標記副本。
- 標記功能無法追蹤源自已標記值的衍生數據。您也需要標記該衍生值。
- 值在其生命週期引用範圍內時會被標記。請參閱 [`experimental_taintUniqueValue` 參數參考](https://react.dev/reference/react/experimental_taintUniqueValue#parameters) 以獲取更多資訊。

## 範例

### 標記對象引用

在這種情況下，`getUserDetails` 函數返回有關特定使用者的資料。我們標記使用者對象引用，使其無法跨越伺服器端與客戶端邊界。例如，假設 `UserCard` 是 Client Component。

```ts switcher
import { experimental_taintObjectReference } from 'react'

function getUserDetails(id: string): UserDetails {
  const user = await db.queryUserById(id)

  experimental_taintObjectReference(
    'Do not use the entire user info object. Instead, select only the fields you need.',
    user
  )

  return user
}
```

### 標記唯一值

在這種情況下，我們可以透過等候 `config.getConfigDetails` 的呼叫來存取伺服器設定。然而，系統設定包含不希望揭露給客戶端的 `SERVICE_API_KEY`。

我們可以標記 `config.SERVICE_API_KEY` 的值。

```ts switcher
import { experimental_taintUniqueValue } from 'react'

function getSystemConfig(): SystemConfig {
  const config = await config.getConfigDetails()

  experimental_taintUniqueValue(
    'Do not pass configuration tokens to the client',
    config,
    config.SERVICE_API_KEY
  )

  return config
}
```

我們仍然可以存取 `systemConfig` 物件的其他屬性。

```tsx
export async function Dashboard() {
  const systemConfig = await getSystemConfig()

  return <ClientDashboard version={systemConfig.SERVICE_API_VERSION} />
}
```

然而，將 `SERVICE_API_KEY` 傳遞給 `ClientDashboard` 會拋出錯誤。

```tsx
export async function Dashboard() {
  const systemConfig = await getSystemConfig()
  // Someone makes a mistake in a PR
  const version = systemConfig.SERVICE_API_KEY

  return <ClientDashboard version={version} />
}
```

```tsx

  const systemConfig = await getSystemConfig()
  // Someone makes a mistake in a PR

```

請注意，即使 `systemConfig.SERVICE_API_KEY` 被重新指派給新變數，將它傳遞至 Client Component 仍會拋出錯誤。

不過，源自已標記唯一值的值將會揭露給客戶端。

```tsx
export async function Dashboard() {
  const systemConfig = await getSystemConfig()
  // Someone makes a mistake in a PR
  const version = `version::${systemConfig.SERVICE_API_KEY}`

  return <ClientDashboard version={version} />
}
```

```tsx

}
```

更好的做法是從 `getSystemConfig` 回傳的資料中移除 `SERVICE_API_KEY`。
