---
title: serverComponentsHmrCache
description: 配置 Server Components 中的 fetch 回應是否在 HMR 重新整理請求之間保持緩存
version: 實驗性
---

實驗性 `serverComponentsHmrCache` 選項允許您在本地開發期間的熱模組替換 (HMR) 重新整理之間緩存 Server Components 中的 `fetch` 回應。這導致更快的回應和減少已計費 API 呼叫的成本。
---

預設情況下，HMR 緩存適用於所有 `fetch` 請求，包括那些帶有 `cache: 'no-store'` 選項的請求。這意味著未緩存的請求在 HMR 重新整理之間將不會顯示最新資料。然而，在導航或全頁重新載入時，緩存將會被清除。

您可以在 `next.config.js` 檔案中將 `serverComponentsHmrCache` 設定為 `false` 以停用 HMR 緩存：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    serverComponentsHmrCache: false, // defaults to true
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    serverComponentsHmrCache: false, // defaults to true
  },
}

module.exports = nextConfig
```

> **值得了解：** 為了獲得更好的可觀測性，我們建議使用 [`logging.fetches`](/docs/app/api-reference/config/next-config-js/logging) 選項，該選項會在開發期間於主控台中記錄 fetch 快取命中與未命中。
