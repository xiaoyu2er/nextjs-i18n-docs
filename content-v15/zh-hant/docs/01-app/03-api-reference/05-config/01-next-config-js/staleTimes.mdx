version: 實驗性
---

`staleTimes` 是一個實驗性功能，可啟用 [ 客戶端路由快取 (client-side router cache) ](/docs/app/guides/caching#client-side-router-cache) 中的網頁區段快取。

您可以透過設定實驗性 `staleTimes` 旗標來啟用此實驗性功能，並提供自訂的重新驗證時間：

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

`static` 和 `dynamic` 屬性對應於基於不同類型 [連結預取 (link prefetching)](/docs/app/api-reference/components/link#prefetch) 的時間區間（以秒為單位）。

- 當頁面未以靜態方式生成或未完全預取時（例如 `prefetch={true}`），`dynamic` 屬性適用。
  - 預設值：0 秒（不儲存快取）
- `static` 屬性用於靜態生成的頁面，或當 `Link` 的 `prefetch` prop 設定為 `true`，或呼叫 [`router.prefetch`](/docs/app/guides/caching#routerprefetch) 時適用。
  - 預設值：5 分鐘

> **了解重點：**
>
> - [載入邊界 (Loading boundaries)](/docs/app/api-reference/file-conventions/loading) 在此配置定義的 `static` 期間內視為可重用於。
> - 這不會影響 [部分渲染 (partial rendering)](/docs/app/getting-started/linking-and-navigating#client-side-transitions)，**意思是共用佈局不會在每次導航時自動重新抓取，只有變更的網頁區段會。**
> - 這不會改變 [後退/前進快取 (back/forward caching)](/docs/app/guides/caching#client-side-router-cache) 行為，以避免版面移位並防止遺失瀏覽器捲動位置。

您可以在 [ 此處 ](/docs/app/guides/caching#client-side-router-cache) 了解更多關於客戶端路由快取 (Client Router Cache) 的資訊。

### 版本歷史

| 版本 | 變更 |
| --------- | ---------------------------------------------------------- |
| `v15.0.0` | `dynamic` `staleTimes` 預設值從 30 秒改為 0 秒。|
| `v14.2.0` | 引入實驗性 `staleTimes`。|
