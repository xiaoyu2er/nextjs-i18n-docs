Next.js 提供了一個 ESLint 外掛程式，[`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)，該外掛程式已封裝在基礎配置中，可讓您偵測 Next.js 應用程式中的常見問題和錯誤。

## 參考

下列 ESLint 外掛程式的建議規則集都在 `eslint-config-next` 中使用：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

這將優先於 `next.config.js` 中的設定。

## 範例

### 自訂目錄和檔案的程式碼檢查

預設情況下，Next.js 會在 `pages/`, `app/`, `components/`, `lib/`, 和 `src/` 目錄中的所有檔案上執行 ESLint。然而，您可以在 `next.config.js` 中的 `eslint` 設定下使用 `dirs` 選項指定要檢查的目錄，用於生產環境建置：

```js filename="next.config.js"
module.exports = {
  eslint: {
    dirs: ['pages', 'utils'], // Only run ESLint on the 'pages' and 'utils' directories during production builds (next build)
  },
}
```

同樣地，`--dir` 和 `--file` 旗標可用於 `next lint` 來檢查特定的目錄和檔案：

```bash filename="Terminal"
next lint --dir pages --dir utils --file bar.js
```

### 指定 monorepo 內的根目錄

如果您在 Next.js 未安裝在根目錄的專案中使用 `eslint-plugin-next`（例如 monorepo），您可以在 `.eslintrc` 設定中使用 `settings` 屬性告訴 `eslint-plugin-next` 在哪裡找到您的 Next.js 應用程式：

```js filename="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname is available after Node.js v20.11.0
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  }),
]

export default eslintConfig
```

`rootDir` 可以是路徑（相對或絕對）、萬用字元（Glob）（即 `"packages/*/"`），或是路徑和/或萬用字元的陣列。

### 停用快取

為了提升效能，ESLint 處理過的檔案資訊預設會被快取。這儲存於 `.next/cache` 或您設定的 [建置目錄](/docs/app/api-reference/config/next-config-js/distDir)。如果您包含任何依賴單一原始檔內容以外的 ESLint 規則，並需要停用快取，請使用 `--no-cache` 旗標搭配 `next lint`。

```bash filename="Terminal"
next lint --no-cache
```

### 停用規則

如果您想要修改或由支援的外掛程式 (`react`, `react-hooks`, `next`) 提供的任何規則，您可以在 `.eslintrc` 中使用 `rules` 屬性直接更動它們：

```js filename="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname is available after Node.js v20.11.0
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next'],
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  }),
]

export default eslintConfig
```

### 搭配 Core Web Vitals

當為首次執行 `next lint` 並選擇 **strict** 選項時，會啟用 `next/core-web-vitals` 規則集。

```js filename="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname is available after Node.js v20.11.0
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals'],
  }),
]

export default eslintConfig
```

`next/core-web-vitals` 會將 `eslint-plugin-next` 更新為在數個規則上發出錯誤，這些規則預設為警告，但會影響 [Core Web Vitals](https://web.dev/vitals/)。

> `next/core-web-vitals` 入口點會自動包含在使用 [Create Next App](/docs/app/api-reference/cli/create-next-app) 建立的新應用程式中。

### 使用 TypeScript

除了 Next.js ESLint 規則外，`create-next-app --typescript` 也會將 TypeScript 相關的檢查規則 `next/typescript` 加到您的設定中：

```js filename="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname is available after Node.js v20.11.0
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next/core-web-vitals', 'next/typescript'],
  }),
]

export default eslintConfig
```

這些規則基於 [`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)。更多詳情請參閱 [typescript-eslint > Configs](https://typescript-eslint.io/linting/configs)。

### 搭配 Prettier

ESLint 也包含程式碼格式化規則，這可能與您現有的 [Prettier](https://prettier.io/) 設定衝突。我們建議將 [eslint-config-prettier](https://github.com/prettier/eslint-config-prettier) 包含在您的 ESLint 設定中，讓 ESLint 和 Prettier 能協同工作。

首先，安裝相依套件：

```bash filename="Terminal"
npm install --save-dev eslint-config-prettier

yarn add --dev eslint-config-prettier

pnpm add --save-dev eslint-config-prettier

bun add --dev eslint-config-prettier
```

接著，將 `prettier` 添加到您現有的 ESLint 設定：

```js filename="eslint.config.mjs"
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname is available after Node.js v20.11.0
  baseDirectory: import.meta.dirname,
})

const eslintConfig = [
  ...compat.config({
    extends: ['next', 'prettier'],
  }),
]

export default eslintConfig
```

### 在暫存檔案上執行檢查

如果您想要使用 `next lint` 搭配 [lint-staged](https://github.com/okonet/lint-staged) 來檢查暫存的 Git 檔案，您必須將以下內容添加到專案根目錄中的 `.lintstagedrc.js` 檔案，以指定使用 `--file` 旗標。

```js filename=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `next lint --fix --file ${filenames
    .map((f) => path.relative(process.cwd(), f))
    .join(' --file ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## 在生產環境建置期間停用檢查

如果您不想要在 `next build` 期間執行 ESLint，您可以在 `next.config.js` 中將 `eslint.ignoreDuringBuilds` 選項設定為 `true`：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  eslint: {
    // Warning: This allows production builds to successfully complete even if
    // your project has ESLint errors.
    ignoreDuringBuilds: true,
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
const nextConfig = {
  eslint: {
    // Warning: This allows production builds to successfully complete even if
    // your project has ESLint errors.
    ignoreDuringBuilds: true,
  },
}

export default nextConfig
```

### 遷移既有設定

如果您應用程式中已設定 ESLint，我們建議直接擴充此外掛程式，而不是包含 `eslint-config-next`，除非滿足下列條件。

#### 推薦的外掛程式規則集

如果符合以下條件：

- 您已安裝以下其中一個或多個外掛程式（分別安裝或透過其他設定如 `airbnb` 或 `react-app`）：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- 您已定義與 Next.js 內部 Babel 設定不同的特定 `parserOptions`（除非您 [自訂了您的 Babel 設定](/docs/pages/guides/babel)，否則不建議這麼做）
- 您已安裝 `eslint-plugin-import` 並定義了 JavaScript 和/或 TypeScript [解析器](https://github.com/benmosher/eslint-plugin-import#resolvers) 來處理匯入

那麼我們建議：如果您偏好 [`eslint-config-next`](https://github.com/vercel/next.js/blob/canary/packages/eslint-config-next/index.js) 內配置的這些屬性，則移除這些設定；或選擇直接擴充 Next.js ESLint 外掛程式：

```js
module.exports = {
  extends: [
    //...
    'plugin:@next/next/recommended',
  ],
}
```

此外掛程式可在您的專案中正常安裝，無需執行 `next lint`：

```bash filename="Terminal"
npm install --save-dev @next/eslint-plugin-next

yarn add --dev @next/eslint-plugin-next

pnpm add --save-dev @next/eslint-plugin-next

bun add --dev @next/eslint-plugin-next
```

這消除了因在多組設定中匯入相同外掛程式或解析器而可能發生的衝突或錯誤風險。

#### 其他設定

如果您已使用单独的 ESLint 設定，並想要包含 `eslint-config-next`，請確保它在其他設定之後最後擴充。例如：

```js filename="eslint.config.mjs"
import js from '@eslint/js'
import { FlatCompat } from '@eslint/eslintrc'

const compat = new FlatCompat({
  // import.meta.dirname is available after Node.js v20.11.0
  baseDirectory: import.meta.dirname,
  recommendedConfig: js.configs.recommended,
})

const eslintConfig = [
  ...compat.config({
    extends: ['eslint:recommended', 'next'],
  }),
]

export default eslintConfig
```

`next` 設定已經處理了 `parser`, `plugins` 和 `settings` 屬性的預設值設定。除非您需要不同的配置來配合您的使用情境，否則無需手動重新宣告任何這些屬性。

如果您包含任何其他公用設定，**您必須確保這些屬性未被覆寫或修改**。否則，我們建議移除與 `next` 設定共享行為的設定，或選擇直接擴充 Next.js ESLint 外掛程式，如上所述。
