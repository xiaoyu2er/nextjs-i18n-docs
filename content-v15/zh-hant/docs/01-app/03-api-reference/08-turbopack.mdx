Turbo pack 是一個針對 JavaScript 和 TypeScript 優化的 **增量打包器 (incremental bundler)**，由 Rust 編寫，並內建於 **Next.js**。您可以在 Pages 和 App Router 中使用 Turbopack，獲得 **更快速 (much faster)** 的本地開發體驗。

## 為什麼使用 Turbopack？

我們開發 Turbopack 是為了提升 Next.js 的效能，包括：

- **統一模組圖 (Unified Graph):** Next.js 支援多輸出環境（例如：客戶端和伺服器端）。管理多個編譯器和縫合 bundles 可能十分繁瑣。Turbopack 對於所有環境使用 **單一統一模組圖 (single, unified graph)**。
- **打包與原生 ESM (Bundling vs Native ESM):** 一些工具在開發期間跳過打包並依賴瀏覽器的原生 ESM。對於小型應用程式這工作良好，但因過多網路請求可能會降低大型應用程式的速度。Turbopack 在開發期間 **打包 (bundles)**，但採用優化方式以保持大型應用程式速度。
- **增量計算 (Incremental Computation):** Turbopack 跨核心平行化工作，並 **快取 (caches)** 結果至函式級別。一旦完成部分工作，Turbopack 不會重複它。
- **延遲打包 (Lazy Bundling):** Turbopack 僅打包開發伺服器實際請求的內容。這種延遲方法可以降低初始編譯時間和記憶體使用量。

## 開始使用

要在 Next.js 專案中啟用 Turbopack，請在您 `package.json` 檔案的 `dev` 和 `build` 腳本中加入 `--turbopack` 旗標：

```json filename="package.json" highlight={3}
{
  "scripts": {
    "dev": "next dev --turbopack",
    "build": "next build --turbopack",
    "start": "next start"
  }
}
```

目前，Turbopack 的 `dev` 功能已穩定，而 `build` 功能則處於 beta 階段。我們正在積極推動產線支援，因為 Turbopack 正逐漸趨向穩定。

## 支援的功能

Next.js 中的 Turbopack 對共用案例提供了 **零配置 (zero-configuration)**。以下是預設支援的功能摘要，以及當需要時如何進一步配置 Turbopack 的參考資訊。

### 語言功能

| 功能 | 狀態 | 備註 |
| --- | --- | --- |
| **JavaScript 與 TypeScript** | **支援** | 底層使用 SWC。Turbopack 不進行類型檢查（執行 `tsc --watch` 或依賴您的 IDE 進行類型檢查）。 |
| **ECMAScript (ESNext)** | **支援** | Turbopack 支援最新的 ECMAScript 功能，與 SWC 的覆蓋範圍匹配。 |
| **CommonJS** | **支援** | `require()` 語法預設處理。 |
| **ESM** | **支援** | 靜態和動態 `import` 完全支援。 |
| **Babel** | 部分不支援 | Turbopack 預設不包含 Babel。但是，您可以 [透過 Turbopack 配置配置 `babel-loader`](/docs/app/api-reference/config/next-config-js/turbopack#configuring-webpack-loaders)。 |

### Framework 和 React 功能

| 功能 | 狀態 | 備註 |
| --- | --- | --- |
| **JSX / TSX** | **支援** | SWC 處理 JSX/TSX 編譯。 |
| **Fast Refresh** | **支援** | 無需配置。 |
| **React Server Components (RSC)** | **支援** | 用於 Next.js App Router。Turbopack 確保正確的服務器/客戶端打包。 |
| **Root layout creation** | 不支援 | 在 App Router 中自動創建根佈局不獲支援。Turbopack 會指示您手動創建。 |

### CSS 和樣式

| 功能 | 狀態 | 備註 |
| --- | --- | --- |
| **Global CSS** | **支援** | 直接在應用程式中匯入 `.css` 檔案。 |
| **CSS Modules** | **支援** | `.module.css` 檔案原生工作（Lightning CSS）。 |
| **CSS Nesting** | **支援** | Lightning CSS 支援 [現代 CSS 嵌套](https://lightningcss.dev/)。 |
| **@import syntax** | **支援** | 組合多個 CSS 檔案。 |
| **PostCSS** | **支援** | 在 Node.js 工作員池自動處理 `postcss.config.js`。適用於 Tailwind、Autoprefixer 等。 |
| **Sass / SCSS** | **支援**（Next.js） | 對於 Next.js，Sass 預設支援。未來，Turbopack 獨立使用可能需要載入器配置。 |
| **Less** | 透過外掛計劃 | 預設尚不支援。一旦自訂載入器穩定，可能需要載入器配置。 |
| **Lightning CSS** | **使用中** | 處理 CSS 轉換。一些低使用率的 CSS Modules 功能（如 `:local/:global` 為獨立偽類別）目前尚不支援。[詳情請見下方]。(#unsupported-and-unplanned-features) |

### 資源

| 功能 | 狀態 | 備註 |
| --- | --- | --- |
| **靜態資源**（圖片，字體） | **支援** | 匯入 `import img from './img.png'` 預設支援。在 Next.js 中，傳回 `<Image />` 物件。 |
| **JSON 匯入** | **支援** | 支援 `.json` 的具名或預設匯入。 |

### 模組解析

| 功能 | 狀態 | 備註 |
| --- | --- | --- |
| **路徑別名** | **支援** | 讀取 `tsconfig.json` 的 `paths` 和 `baseUrl`，與 Next.js 行為匹配。 |
| **手動別名** | **支援** | [在 `next.config.js` 中配置 `resolveAlias`](/docs/app/api-reference/config/next-config-js/turbopack#resolving-aliases)（類似 `webpack.resolve.alias`）。 |
| **自訂副檔名** | **支援** | [在 `next.config.js` 中配置 `resolveExtensions`](/docs/app/api-reference/config/next-config-js/turbopack#resolving-custom-extensions)。 |
| **AMD** | 部分支援 | 基本轉換工作；進階 AMD 使用受限。 |

### 效能與 Fast Refresh

| 功能 | 狀態 | 備註 |
| --- | --- | --- |
| **Fast Refresh** | **支援** | 更新 JavaScript、TypeScript 和 CSS 而不需完整重新整理。 |
| **Incremental Bundling** | **支援** | Turbopack 延遲建構開發伺服器請求的內容，加快大型應用程式速度。 |

## 與 webpack 的已知差距

在遷移應用程式時，webpack 和 Turbopack 之間存在一些重要的非微不足道行為差異。一般來說，這些對新應用程式較無影響。

### CSS Module 排序

Turbopack 將遵循 JS 匯入順序來排序 [CSS modules](/docs/app/getting-started/css#css-modules)（ otherwise 無排序）。例如：

```jsx filename="components/BlogPost.jsx"
import utilStyles from './utils.module.css'
import buttonStyles from './button.module.css'
export default function BlogPost() {
  return (
    <div className={utilStyles.container}>
      <button className={buttonStyles.primary}>Click me</button>
    </div>
  )
}
```

在這個範例中，Turbopack 將確保 `utils.module.css` 的輸出順序在 `button.module.css` 之前，遵循匯入順序。

Webpack 通常也會這樣做，但有些情況下它會忽略 JS 推斷的排序，例如如果它推斷 JS 檔案沒有副作用。

這可能會導致採用 Turbopack 時產生微妙的渲染變更，如果應用程式依賴於任意排序。通常，解決方案很簡單，例如讓 `button.module.css` `@import utils.module.css` 來強制排序，或識別衝突規則並更改它們以避免針對相同的屬性。

### 打包大小

Turbopack 目前還沒有 webpack 的 [內圖最佳化 (Inner Graph Optimization)](https://webpack.js.org/configuration/optimization/#optimizationinnergraph) 的等效功能。此最佳化對於 tree shake 大型模組很有用。例如：

```js filename=large.module.js
import heavy from 'some-heavy-dependency.js'

export function usesHeavy() {
  return heavy.run()
}

export const CONSTANT_VALUE = 3
```

如果應用程式僅使用 `CONSTANT_VALUE`，Turbopack 會檢測到並刪除 `usesHeavy` 匯出，但不刪除相應的 `import`。然而，在啟用 `optimization.innerGraph = true` 選項後，webpack 也能刪除 `import`。

我們計畫在 Turbopack 中提供 `innerGraph` 最佳化的等效功能，但目前仍在開發中。如果您受此差距影響，請考慮手動分割這些模組。

### 建置快取

Webpack 支援 [磁碟建置快取 (disk build caching)](https://webpack.js.org/configuration/cache/#cache) 以加速建置。我們計畫在 Turbopack 中支援類似的功能，但目前尚未就緒。在 `next@canary` 版本中，您可以透過啟用 [`experimental.turbopackPersistentCaching` 旗標](/docs/app/api-reference/config/next-config-js/turbopackPersistentCaching) 來實驗我們的解決方案。

> **小提示：** 為此，在比較 webpack 和 Turbopack 效能時，請確保在建置之間刪除 `.next` 資料夾以獲得公平的比較。

### Webpack 外掛

Turbopack 不支援 webpack 外掛。這會影響依賴 webpack 外掛系統進行整合的第三方工具。我們支援 [webpack loaders](/docs/app/api-reference/config/next-config-js/turbopack#configuring-webpack-loaders)。如果您依賴 webpack 外掛，您需要尋找 Turbopack 相容的替代方案，或繼續使用 webpack 直到擁有功能等效的功能。

## 不支援且未規劃的功能

某些功能尚未實作或尚未規劃：

- **舊版 CSS Modules 功能**
  - 獨立 `:local` 和 `:global`偽類（仅支援函式變體 `:global(...)`）。
  - `@value` 規則（已由 CSS 變數取代）。
  - `:import` 和 `:export` ICSS 規則。
  - `composes` 在 `.module.css` 中組合 `.css` 檔案。在 webpack 中這會將 `.css` 檔案視為 CSS Module，在 Turbopack 中 `.css` 檔案將始終是全域的。這意味著如果您想在 CSS Module 中使用 `composes`，您需要將 `.css` 檔案更改為 `.module.css` 檔案。
  - CSS Modules 中的 `@import` 匯入 `.css` 作為 CSS Module。在 webpack 中這會將 `.css` 檔案視為 CSS Module，在 Turbopack 中 `.css` 檔案將始終是全域的。這意味著如果您想在 CSS Module 中使用 `@import`，您需要將 `.css` 檔案更改為 `.module.css` 檔案。
- **`webpack()` 配置** 在 `next.config.js` 中
  Turbopack 取代了 webpack，因此 `webpack()` 配置不受識別。請改用 [`turbopack` 配置](/docs/app/api-reference/config/next-config-js/turbopack)。
- **AMP**
  未在 Next.js 中規劃 Turbopack 支援。
- **Yarn PnP**
  未在 Next.js 中規劃 Turbopack 支援。
- **`experimental.urlImports`**
  未規劃供 Turbopack 使用。
- **`experimental.esmExternals`**
  未規劃。Turbopack 不支援 Next.js 中的舊版 `esmExternals` 配置。
- **部分 Next.js 實驗性旗標**
    - `experimental.nextScriptWorkers`
    - `experimental.sri.algorithm`
    - `experimental.fallbackNodePolyfills`
    我們計畫未來實作這些功能。

欲查看每個功能旗標及其狀態的完整詳細清單，請參閱 [Turbopack API 參考文獻](/docs/app/api-reference/config/next-config-js/turbopack)。

## 配置

Turbopack 可以透過 `turbopack` 鍵下的 `next.config.js`（或 `next.config.ts`）進行配置。配置選項包括：

- **`rules`**
  定義額外的 [webpack loaders](/docs/app/api-reference/config/next-config-js/turbopack#configuring-webpack-loaders) 用於檔案轉換。
- **`resolveAlias`**
  建立手動別名（類似 Webpack 中的 `resolve.alias`）。
- **`resolveExtensions`**
  變更或擴充模組解析的副檔名。
- **`moduleIds`**
  設定模組 ID 的生成方式（`'named'` 與 `'deterministic'` 之間）。
- **`memoryLimit`**
  設定 Turbopack 的記憶體上限（以位元組為單位）。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    // Example: adding an alias and custom file extension
    resolveAlias: {
      underscore: 'lodash',
    },
    resolveExtensions: ['.mdx', '.tsx', '.ts', '.jsx', '.js', '.json'],
  },
}
```

欲查看更多深入的配置範例，請參閱 [Turbopack 配置文件](/docs/app/api-reference/config/next-config-js/turbopack)。

## 產生效能除錯的追蹤檔案

如果您遇到效能或記憶體問題並希望協助 Next.js 團隊診斷，您可以在開發命令中追加 `NEXT_TURBOPACK_TRACING=1` 來產生追蹤檔案：

```bash
NEXT_TURBOPACK_TRACING=1 next dev --turbopack
```

這將產生 `.next/trace-turbopack` 檔案。建立 [Next.js repo](https://github.com/vercel/next.js) 上的 GitHub 問題時包含該檔案，以幫助我們調查。

## 摘要

Turbopack 是一個基於 **Rust (Rust-based)**、**增量 (incremental)** 的打包器，旨在加快本地開發和建置速度——尤其是大型應用程式。它整合至 Next.js 中，提供零配置的 CSS、React 和 TypeScript 支援。

繼續關注更多更新，隨著我們持續改進 Turbopack 並增加生產建置支援。與此同時，請使用 `next dev --turbopack` 嘗試一下，並告訴我們您的反饋。
