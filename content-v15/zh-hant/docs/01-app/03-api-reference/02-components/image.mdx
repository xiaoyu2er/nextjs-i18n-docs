Next.js 的 Image 元件擴充了 HTML `<img>` 元素，用於自動最佳化圖片。

```jsx filename="app/page.js"
import Image from 'next/image'

export default function Page() {
  return (
    <Image
      src="/profile.png"
      width={500}
      height={500}
      alt="Picture of the author"
    />
  )
}
```

<PagesOnly>

> **小提示**：如果您使用的是 13 之前的 Next.js 版本，由於元件已更名，您需要使用 [next/legacy/image](/docs/pages/api-reference/components/image-legacy) 文件。

</PagesOnly>

#### `src`

圖片來源。可以是以下其中一種：

內部路徑字串。

```jsx
<Image src="/profile.png" />
```

絕對的外部 URL (必須透過 [remotePatterns](#remotepatterns) 進行配置)。

```jsx
<Image src="https://example.com/profile.png" />
```

靜態導入。

```jsx
import profile from './profile.png'

export default function Page() {
  return <Image src={profile} />
}
```

> **小提示**：出於安全考慮，使用預設 [loader](#loader) 的 Image Optimization API 在取得 `src` 圖片時將*不*轉發-Headers。
> 如果 `src` 圖片需要認證，請考慮使用 [unoptimized](#unoptimized) 屬性來停用 Image Optimization。

#### `alt`

`alt` 屬性用於為螢幕閱讀器和搜尋引擎描述圖片。如果圖片被停用或圖片載入時發生錯誤，它也是備用文字。

它應包含可替換圖片的文字 [但不改變頁面的含義](https://html.spec.whatwg.org/multipage/images.html#general-guidelines)。它不旨在補充圖片，也不應重複已經在圖片上方或下方的標題中提供的訊息。

如果圖片是 [純粹裝飾性](https://html.spec.whatwg.org/multipage/images.html#a-purely-decorative-image-that-doesn't-add-any-information) 或 [不打算提供給使用者](https://html.spec.whatwg.org/multipage/images.html#an-image-not-intended-for-the-user)，`alt` 屬性應為空字串 (`alt=""`)。

> 更多資訊請參閱 [圖片无障碍指南](https://html.spec.whatwg.org/multipage/images.html#alt)。

#### `width` 和 `height`

`width` 和 `height` 屬性代表 [內建](https://developer.mozilla.org/en-US/docs/Glossary/Intrinsic_Size) 圖片的像素大小。此屬性用於推斷瀏覽器用於保留圖片空間和避免載入期間佈局移位時所使用的正確 **aspect ratio**，它不決定圖片的 _Rendered size_，這是由 CSS 控制的。

```jsx
<Image src="/profile.png" width={500} height={500} />
```

您**必須**設定 `width` 和 `height` 屬性，除非：

- 圖片是靜態導入的。
- 圖片具有 [`fill` 屬性](#fill)

如果高度和寬度未知，我們建議使用 [`fill` 屬性](#fill)。

#### `fill`

一個布林值，導致圖片擴展到父元素的大小。

```js
<Image src="/profile.png" fill={true} />
```

**定位 (Positioning)**：

- 父元素**必須**分配 `position: "relative"`, `"fixed"`, `"absolute"`。
- 預設情況下，`<img>` 元素使用 `position: "absolute"`。

**物件調整 (Object Fit)**：

如果沒有對圖片應用樣式，圖片將拉伸以適合容器。您可以使用 `objectFit` 来控制裁剪和縮放。

- `"contain"`: 圖片將縮小以適合容器並保留比例。
- `"cover"`: 圖片將填滿容器並被裁剪。

> 更多資訊請參閱 [`position`](https://developer.mozilla.org/en-US/docs/Web/CSS/position) 和 [`object-fit`](https://developer.mozilla.org/docs/Web/CSS/object-fit)。

#### `loader`

用於產生圖片 URL 的自定義函數。此函數接收以下參數，並回傳圖片的 URL 字串：

- [`src`](#src)
- [`width`](#width-and-height)
- [`quality`](#quality)

<AppOnly>

```js
'use client'

import Image from 'next/image'

const imageLoader = ({ src, width, quality }) => {
  return `https://example.com/${src}?w=${width}&q=${quality || 75}`
}

export default function Page() {
  return (
    <Image
      loader={imageLoader}
      src="me.png"
      alt="Picture of the author"
      width={500}
      height={500}
    />
  )
}
```

> **小提示**：使用接受函數的屬性 (如 `onLoad`)，需要使用 [Client Components](https://react.dev/reference/rsc/use-client) 來序列化提供的函數。

</AppOnly>

<PagesOnly>

```js
import Image from 'next/image'

const imageLoader = ({ src, width, quality }) => {
  return `https://example.com/${src}?w=${width}&q=${quality || 75}`
}

```

Alternatively, you can use the [loaderFile](#loaderfile) configuration in `next.config.js` to configure every instance of `next/image` in your application, without passing a prop.
#### `sizes`
Define the sizes of the image at different breakpoints. Used by the browser to choose the most appropriate size from the generated `srcset`.
```jsx
import Image from 'next/image'

export default function Page() {
  return (
    <div className="grid-element">
      <Image
        fill
        src="/example.png"
        sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
      />
    </div>
  )
}
```
`sizes` 應在以下情況下使用：

- 圖片正在使用 [`fill`](#fill) 屬性。
- CSS 用於使圖片響應式。

如果缺少 `sizes`，瀏覽器會假設圖片與視窗寬度一樣寬 (`100vw`)。這會導致下載不必要的大量圖片。

此外，`sizes` 會影響 `srcset` 的產生方式：

- 無 `sizes`：Next.js 會產生有限的 `srcset` (例如 1x, 2x)，適用於固定大小的圖片。
- 有 `sizes`：Next.js 會產生完整的 `srcset` (例如 640w, 750w 等)，適用於響應式佈局。

> 請在 [web.dev](https://web.dev/learn/design/responsive-images/#sizes) 和 [mdn](https://developer.mozilla.org/docs/Web/HTML/Element/img#sizes) 上了解更多關於 `srcset` 和 `sizes` 的資訊。

#### `quality`
設定最佳化圖片品質的整數，介於 `1` 和 `100` 之間。較高的值會增加檔案大小和視覺保真度。較低的值會減少檔案大小，但可能會影響清晰度。

```jsx
// 預設品質為 75
<Image quality={75} />
```

如果您已在 `next.config.js` 中設定了 [qualities](#qualities)，則值必須與允許的條目之一匹配。

> **小提示**：如果原始圖片已經品質較低，設定較高的品質值會增加檔案大小，但外觀不會改善。

#### `style`
允許傳遞 CSS 樣式給底層的圖片元素。

```jsx
const imageStyle = {
  borderRadius: '50%',
  border: '1px solid #fff',
  width: '100px',
  height: 'auto',
}

export default function ProfileImage() {
  return <Image src="..." style={imageStyle} />
}
```

> **小提示**：如果您使用 `style` 屬性來設定自定義寬度，請務必也設定 `height: 'auto'` 以保持圖片的長寬比。

#### `priority`
一個布林值，指示是否應預載圖片。

```jsx
// 預設優先級為 false
<Image priority={false} />
```

- `true`: [預載](https://web.dev/preload-responsive-images/) 圖片。停用延遲載入。
- `false`: 延遲載入圖片。

**使用時機：**

- 圖片位於 "Above the fold" 區域 (顯示器範圍)。
- 圖片是 [Largest Contentful Paint (LCP)](https://nextjs.org/learn/seo/web-performance/lcp) 元素。
- 您希望提高頁面的初始載入效能。

**不當使用時機：**

- 當 `loading` 屬性已在使用時 (會觸發警告)。

#### `loading`
控制圖片應開始載入的時機。

```jsx
// 預設為 lazy
<Image loading="lazy" />
```

- `lazy`: 延遲載入圖片直到它達到距離視窗的計算距離。
- `eager`: 無論圖片位置如何，立即載入圖片。

使用 `eager` 僅當您希望確保圖片立即載入時。

> 了解更多關於 [`loading` 屬性](https://developer.mozilla.org/docs/Web/HTML/Element/img#loading) 的資訊。

#### `placeholder`
指定在圖片載入期間使用的佔位符，以改善感知的載入效能。

```jsx
// 預設為 empty
<Image placeholder="empty" />
```

- `empty`: 圖片載入期間無佔位符。
- `blur`: 使用圖片的模糊版本作為佔位符。必須與 [`blurDataURL`](#blurdataurl) 屬性一起使用。
- `data:image/...`: 使用 [Data URL](https://developer.mozilla.org/docs/Web/HTTP/Basics_of_HTTP/Data_URIs) 作為佔位符。

**範例：**

- [`blur` 佔位符](https://image-component.nextjs.gallery/placeholder)
- [使用 Data URL `placeholder` 屬性的閃爍效果](https://image-component.nextjs.gallery/shimmer)
- [使用 `blurDataURL` 屬性的色彩效果](https://image-component.nextjs.gallery/color)

> 了解更多關於 [`placeholder` 屬性](https://developer.mozilla.org/docs/Web/HTML/Element/img#placeholder) 的資訊。

#### `blurDataURL`
一個 [Data URL](https://developer.mozilla.org/docs/Web/HTTP/Basics_of_HTTP/Data_URIs)，
用於圖片成功載入之前作為佔位圖圖片。可以自動設定或使用 [`placeholder="blur"`](#placeholder) 屬性。

```jsx
<Image placeholder="blur" blurDataURL="..." />
```

圖片會被自動放大和模糊，因此建議使用非常小的圖片 (10px 或更少)。

**自動 (Automatic)**
如果 `src` 是 `jpg`、`png`、`webp` 或 `avif` 檔案的靜態導入，則會自動新增 `blurDataURL` - 除非圖片是動態的。

**手動設定 (Manually set)**
如果圖片是動態的或遠端，您必須自行提供 `blurDataURL`。要產生一個，您可以使用：

- [像 png-pixel.com 的線上工具](https://png-pixel.com)
- [像 Plaiceholder 的函式庫](https://github.com/joe-bell/plaiceholder)

大型 blurDataURL 可能會影響效能。保持簡單和小型化。

**範例：**

- [預設 `blurDataURL` 屬性](https://image-component.nextjs.gallery/placeholder)
- [使用 `blurDataURL` 屬性的色彩效果](https://image-component.nextjs.gallery/color)

#### `onLoad`
一個當圖片完全載入且 [placeholder](#placeholder) 已移除時被呼叫的回調函數。

```jsx
<Image onLoad={(e) => console.log(e.target.naturalWidth)} />
```

回調函數將被呼叫一次，作為參數，是一個具有 `target` 參照底層 `<img>` 元素的事件。

<AppOnly>

> **小提示**：使用接受函數的屬性 (如 `onLoad`)，需要使用 [Client Components](https://react.dev/reference/rsc/use-client) 來序列化提供的函數。

</AppOnly>

#### `onError`
一個當圖片載入失敗時被呼叫的回調函數。

```jsx
<Image onError={(e) => console.error(e.target.id)} />
```

<AppOnly>

> **小提示**：使用接受函數的屬性 (如 `onError`)，需要使用 [Client Components](https://react.dev/reference/rsc/use-client) 來序列化提供的函數。

</AppOnly>

#### `unoptimized`
一個布林值，指示是否應最佳化圖片。這對於不會從最佳化中受益的圖片非常有幫助，例如小圖片 (小於 1KB)、向量圖片 (SVG) 或動畫圖片 (GIF)。

```js
import Image from 'next/image'

const UnoptimizedImage = (props) => {
  // 預設為 false
  return <Image {...props} unoptimized />
}
```

- `true`: 源圖片將以原樣从 `src` 提供，而不是更改品質、大小或格式。
- `false`: 源圖片將被最佳化。

自 Next.js 12.3.0 起，此屬性可以透過更新 `next.config.js` 來指派給所有圖片：

```js filename="next.config.js"
module.exports = {
  images: {
    unoptimized: true,
  },
}
```

#### `overrideSrc`
當向 `<Image>` 元件提供 `src` 屬性時，會為結果產生的 `<img>` 自動產生 `srcset` 和 `src` 屬性。

```jsx filename="input.js"
<Image src="/profile.jpg" />
```

```html filename="output.html"
<img
  srcset="
    /_next/image?url=%2Fprofile.jpg&w=640&q=75 1x,
    /_next/image?url=%2Fprofile.jpg&w=828&q=75 2x
  "
  src="/_next/image?url=%2Fprofile.jpg&w=828&q=75"
/>
```

在某些情況下，產生 `src` 屬性是不理想的，您希望使用 `overrideSrc` 屬性來覆蓋它。

例如，當從 `<img>` 升級現有網站到 `<Image>` 時，您可能希望維持相同的 `src` 屬性以用於 SEO 目的，例如圖片排名或避免重新爬蟲。

```jsx filename="input.js"
<Image src="/profile.jpg" overrideSrc="/override.jpg" />
```

```html filename="output.html"
<img
  srcset="
    /_next/image?url=%2Fprofile.jpg&w=640&q=75 1x,
    /_next/image?url=%2Fprofile.jpg&w=828&q=75 2x
  "
  src="/override.jpg"
/>
```

#### `decoding`
一個提示給瀏覽器的指示，表明它是否應等待圖片解碼后才呈現其他內容更新。

```jsx
// 預設為 async
<Image decoding="async" />
```

- `async`: 非同步解碼圖片，允許在圖片完成之前呈現其他內容。
- `sync`: 同步解碼圖片，以便與其他內容原子呈現。
- `auto`: 無偏好設定。瀏覽器選擇最佳方法。

> 了解更多關於 [`decoding` 屬性](https://developer.mozilla.org/docs/Web/HTML/Element/img#decoding) 的資訊。

### 配置選項
您可以在 `next.config.js` 中配置 Image Component。以下選項可用：

#### `localPatterns`
在 `next.config.js` 檔案中使用 `localPatterns` 以允許來自特定本地路徑的圖片被最佳化，並封鎖其他所有路徑。

```js filename="next.config.js"
module.exports = {
  images: {
    localPatterns: [
      {
        pathname: '/assets/images/**',
        search: '',
      },
    ],
  },
}
```
上述範例將確保 `next/image` 的 `src` 屬性必須以 `/assets/images/` 開頭，且不得有查詢字串。嘗試最佳化任何其他路徑將回應 `400` Bad Request 錯誤。

#### `remotePatterns`
在 `next.config.js` 檔案中使用 `remotePatterns` 以允許來自特定外部路徑的圖片，並封鎖其他所有路徑。這確保只有來自帳戶的外部圖片可以被提供。

```js filename="next.config.js"
module.exports = {
  images: {
    remotePatterns: [new URL('https://example.com/account123/**')],
  },
}
```

您也可以使用物件來配置 `remotePatterns`：

```js filename="next.config.js"
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'example.com',
        port: '',
        pathname: '/account123/**',
        search: '',
      },
    ],
  },
}
```
上述範例將確保 `next/image` 的 `src` 屬性必須以 `https://example.com/account123/` 開頭，且不得有查詢字串。任何其他協議、主機名、端口或不匹配的路徑將回應 `400` Bad Request。

**萬用字元模式：**

萬用字元模式可用於 `pathname` 和 `hostname`，具有以下的語法：

- `*` 匹配單一路徑段或子網域
- `**` 匹配末尾的任何路徑段或開頭的子網域。該語法無法在模式的末尾工作。

```js filename="next.config.js"
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: '**.example.com',
        port: '',
        search: '',
      },
    ],
  },
}
```
這允許子網域如 `image.example.com`。查詢字串和自訂端口仍然被封鎖。

> **小提示**：當省略 `protocol`、`port`、`pathname` 或 `search` 時，則隱含萬用字元 `**`。这不推薦，因為它可能會允許惡意行為者最佳化您不想要的 url。

**查詢字串：**

您也可以使用 `search` 屬性限制查詢字串：

```js filename="next.config.js"
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'assets.example.com',
        search: '?v=1727111025337',
      },
    ],
  },
}
```
上述範例將確保 `next/image` 的 `src` 屬性必須以 `https://assets.example.com` 開頭，且必須有確切的查詢字串 `?v=1727111025337`。任何其他協議或查詢字串將回應 `400` Bad Request。

#### `loaderFile`
`loaderFiles` 允許您使用自定義的圖片最佳化服務，而不是 Next.js。

```js filename="next.config.js"
module.exports = {
  images: {
    loader: 'custom',
    loaderFile: './my/image/loader.js',
  },
}
```

路徑必須是針對專案根目錄的相對路徑。檔案必須導入回傳 URL 字串的預設函數：

<AppOnly>

```js filename="my/image/loader.js"
'use client'

export default function myImageLoader({ src, width, quality }) {
  return `https://example.com/${src}?w=${width}&q=${quality || 75}`
}
```

</AppOnly>

#### `path`
如果您想更改或預設 Image Optimization API 的預設路徑，可以使用 `path` 屬性來完成。`path` 的預設值為 `/_next/image`。

```js filename="next.config.js"
module.exports = {
  images: {
    path: '/my-prefix/_next/image',
  },
}
```

#### `deviceSizes`
`deviceSizes` 允許您指定裝置寬度斷點列表。當 `next/image` 元件使用 [`sizes`](#sizes) 屬性時，會使用這些寬度，以確保為使用者的裝置提供正確的圖片。

如果沒有提供配置，則使用下方的預設值：

```js filename="next.config.js"
module.exports = {
  images: {
    deviceSizes: [640, 750, 828, 1080, 1200, 1920, 2048, 3840],
  },
}
```

#### `imageSizes`
`imageSizes` 允許您指定圖片寬度列表。這些寬度會與 [裝置大小](#devicesizes) 陣列串接，以形成用於產生圖片 [srcset](https://developer.mozilla.org/docs/Web/API/HTMLImageElement/srcset) 的全尺寸陣列。

如果沒有提供配置，則使用下方的預設值：

```js filename="next.config.js"
module.exports = {
  images: {
    imageSizes: [16, 32, 48, 64, 96, 128, 256, 384],
  },
}
```

`imageSizes` 僅用於提供 [`sizes`](#sizes) 屬性的圖片，這表示圖片小於螢幕的完整寬度。因此，`imageSizes` 中的所有大小都應小於 `deviceSizes` 中的最小大小。

#### `qualities`
`qualities` 允許您指定圖片品質值列表。

```js filename="next.config.js"
module.exports = {
  images: {
    qualities: [25, 50, 75],
  },
}
```

上述範例中，只允許三種品質：25、50 和 75。如果 [`quality`](#quality) 屬性不與此陣列中的值匹配，則圖片會因 `400` Bad Request 而失敗。

#### `formats`
`formats` 允許您指定要使用的圖片格式列表。

```js filename="next.config.js"
module.exports = {
  images: {
    // 預設
    formats: ['image/webp'],
  },
}
```
Next.js 會自動透過請求的 `Accept` 標頭偵測瀏覽器支援的圖片格式，以決定最佳輸出格式。

如果匹配的 `Accept` 標頭超過一個配置格式，則使用陣列中的第一個匹配項。因此，陣列順序很重要。如果沒有匹配 (或是源圖片是動態的)，它將使用源圖片的格式。

您可以啟用 AVIF 支援，如果瀏覽器 [不支援 AVIF](https://caniuse.com/avif)，則會回退到源圖片的原始格式：

```js filename="next.config.js"
module.exports = {
  images: {
    formats: ['image/avif'],
  },
}
```

> **小提示**：

> - 我們仍然建議大多數情況下使用 WebP。
> - AVIF 通常编码时间比 WebP 長 50% 左右，但壓縮後比 WebP 小 20%。這意味著初次請求圖片時通常會較慢，但後續請求的快取會較快。
> - 如果您在 Next.js 前面自行託管 Proxy/CDN，則必須配置 Proxy 以轉發 `Accept` 標頭。

#### `minimumCacheTTL`
`minimumCacheTTL` 允許您配置快取最佳化圖片的生存時間 (TTL)，單位為秒。在許多情況下，使用 [靜態圖片導入](/docs/app/getting-started/images#local-images) 更好，它將自動雜湊檔案內容，並使用 `Cache-Control` 標頭 `immutable` 永久快取圖片。

如果沒有提供配置，則使用下方的預設值。

```js filename="next.config.js"
module.exports = {
  images: {
    minimumCacheTTL: 60, // 1 分鐘
  },
}
```

您可以增加 TTL 以減少重新驗證的次數，並可能降低成本：

```js filename="next.config.js"
module.exports = {
  images: {
    minimumCacheTTL: 2678400, // 31 天
  },
}
```

最佳化圖片的過期 (或更準確地說是 Max Age) 由 `minimumCacheTTL` 或上游圖片 `Cache-Control` 標頭定義，以較大者為準。

如果您需要針對每張圖片更改快取行為，您可以配置 [`headers`](/docs/app/api-reference/config/next-config-js/headers) 以在上游圖片 (例如 `/some-asset.jpg`，而不是 `/_next/image` 本身) 上設定 `Cache-Control` 標頭。

目前沒有機制使快取失效，因此最好保持 `minimumCacheTTL` 較低。否則，您可能需要手動變更 [`src`](#src) 屬性或刪除快取檔案 `<distDir>/cache/images`。

#### `disableStaticImages`
`disableStaticImages` 允許您停用靜態圖片導入。

預設行為允許您導入靜態檔案 (例如 `import icon from './icon.png') 然後將其傳遞給 `src` 屬性。在某些情況下，如果您希望停用此功能以與預期的導入行為不同的其他外掛發生衝突。

您可以在 `next.config.js` 內停用靜態圖片導入：

```js filename="next.config.js"
module.exports = {
  images: {
    disableStaticImages: true,
  },
}
```

#### `maximumResponseBody`
預設圖片最佳化載入器會 fetch 源圖片，大小不超過 50 MB。

```js filename="next.config.js"
module.exports = {
  images: {
    maximumResponseBody: 50_000_000,
  },
}
```

如果您知道所有源圖片都很小，您可以透過將其減少到較小的值 (例如 5MB) 來保護記憶體受限的伺服器。

```js filename="next.config.js"
module.exports = {
  images: {
    maximumResponseBody: 5_000_000,
  },
}
```

#### `dangerouslyAllowSVG`
`dangerouslyAllowSVG` 允許您提供 SVG 圖片。

```js filename="next.config.js"
module.exports = {
  images: {
    dangerouslyAllowSVG: true,
  },
}
```
預設情況下，Next.js 不大會最佳化 SVG 圖片，原因如下：

- SVG 是向量格式意味著它可以無損縮放。
- SVG 具有與 HTML/CSS 相同的許多功能，這可能會導致安全漏洞，而沒有適當的 [Content Security Policy (CSP) 標頭](/docs/app/api-reference/config/next-config-js/headers#content-security-policy)。

我們建議在使用 [`src`](#src) 屬性已知是 SVG 時使用 [`unoptimized`](#unoptimized) 屬性。當 `src` 以 `".svg"` 結尾時，這會自動發生。

```jsx
<Image src="/my-image.svg" unoptimized />
```

此外，強烈建議將 `contentDispositionType` 設定為強制瀏覽器下載圖片，並將 `contentSecurityPolicy` 設定為防止嵌入在圖片中的腳本執行。

```js filename="next.config.js"
module.exports = {
  images: {
    dangerouslyAllowSVG: true,
    contentDispositionType: 'attachment',
    contentSecurityPolicy: "default-src 'self'; script-src 'none'; sandbox;",
  },
}
```

#### `contentDispositionType`
`contentDispositionType` 允許您配置 [`Content-Disposition`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Disposition#as_a_response_header_for_the_main_body) 標頭。

```js filename="next.config.js"
module.exports = {
  images: {
    contentDispositionType: 'inline',
  },
}
```

#### `contentSecurityPolicy`
`contentSecurityPolicy` 允許您為圖片配置 [`Content-Security-Policy`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Guides/CSP) 標頭。當使用 [`dangerouslyAllowSVG`](#dangerouslyallowsvg) 防止嵌入在圖片中的腳本執行時，這特別重要。

```js filename="next.config.js"
module.exports = {
  images: {
    contentSecurityPolicy: "default-src 'self'; script-src 'none'; sandbox;",
  },
}
```

預設情況下，[loader](#loader) 對於 [Content-Disposition](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Disposition#as_a_response_header_for_the_main_body) 標頭設定為 `attachment` 以增加保護效果，因為 API 可以提供任意遠端圖片。

預設值為 `attachment`，強制瀏覽器在直接檢視時下載圖片。當 [`dangerouslyAllowSVG`](#dangerouslyallowsvg) 為 true 時，這特別重要。

您可以選擇配置 `inline` 以允許瀏覽器在直接檢視時呈現圖片而不會下載它。

### 已棄用的配置選項
#### `domains`

> **警告**：自 Next.js 14 起已棄用，改用嚴格的 [`remotePatterns`](#remotepatterns) 以保護應用程式免受惡意使用者侵害。僅當您擁有從網域提供的所有內容時才使用 `domains`。

與 [`remotePatterns`](#remotepatterns) 類似，`domains` 配置可用於提供外部圖片的允許主機名列表。然而，`domains` 配置不支援萬用字元模式匹配，且無法限制協議、端口或路徑。

以下是 `next.config.js` 檔案中 `domains` 屬性的範例：

```js filename="next.config.js"
module.exports = {
  images: {
    domains: ['assets.acme.com'],
  },
}
```

## 函式 (Functions)

### `getImageProps`
`getImageProps` 函式可用於取得將傳遞給底層 `<img>` 元素的屬性，並將其傳遞給另一個元件、樣式、畫布等。

```jsx
import { getImageProps } from 'next/image'

const { props } = getImageProps({
  src: 'https://example.com/image.jpg',
  alt: 'A scenic mountain view',
  width: 1200,
  height: 800,
})

function ImageWithCaption() {
  return (
    <figure>
      <img {...props} />
      <figcaption>A scenic mountain view</figcaption>
    </figure>
  )
}
```

這也避免了呼叫 React `useState()`，因此可能導致更好的效能，但它不能與 [`placeholder`](#placeholder) 屬性一起使用，因為佔位符永遠不會被移除。

## 已知瀏覽器問題 (Known browser bugs)
此 `next/image` 元件使用瀏覽器原生 [延遲載入](https://caniuse.com/loading-lazy-attr)，對於較舊的瀏覽器 (Safari 15.4 之前) 可能會回退到即時載入。當使用模糊佔位符時，Safari 12 之前的較舊瀏覽器會回退到空白佔位符。當使用寬度/高度為 `auto` 的樣式時，它可能會在 Safari 15 之前的較舊瀏覽器上造成 [Layout Shift](https://web.dev/cls/)，因為它們不 [保留長寬比](https://caniuse.com/mdn-html_elements_img_aspect_ratio_computed_from_attributes)。更多詳情，請參見 [MDN 影片](https://www.youtube.com/watch?v=4-d_SoCHeWE)。

- [Safari 15 - 16.3](https://bugs.webkit.org/show_bug.cgi?id=243601) 在載入時顯示灰色邊框。Safari 16.4 [已修正此問題](https://webkit.org/blog/13966/webkit-features-in-safari-16-4/#:~:text=Now%20in%20Safari%2016.4%2C%20a%20gray%20line%20no%20longer%20appears%20to%20mark%20the%20space%20where%20a%20lazy%2Dloaded%20image%20will%20appear%20once%20it%E2%80%99s%20been%20loaded.)。可能的解決方案：

  - 使用 CSS `@supports (font: -apple-system-body) and (-webkit-appearance: none) { img[loading="lazy"] { clip-path: inset(0.6px) } }`
  - 如果圖片在 "Above the fold" 區域，則使用 [`priority`](#priority)
- [Firefox 67+](https://bugzilla.mozilla.org/show_bug.cgi?id=1556156) 在載入時顯示白色背景。解決方案：

  - 啟用 [AVIF `formats`](#formats)
  - 使用 [`placeholder`](#placeholder)

## 範例

### 圖片樣式 (Styling images)

對 Image 元件進行樣式設定，與對 normal `<img>` 元素進行樣式設定類似，但有幾點指南必須留意：

使用 `className` 或 `style`，不支援 `styled-jsx`。在大多數情況下，我們建議使用 `className` 屬性。這可以是導入的 [CSS Module](/docs/app/getting-started/css)、[全域樣式表](/docs/app/getting-started/css#global-css) 等。

```jsx
import styles from './styles.module.css'

export default function MyImage() {
  return <Image className={styles.image} src="/my-image.png" alt="My Image" />
}
```

您也可以使用 `style` 屬性來指派內聯樣式。

```jsx
export default function MyImage() {
  return (
    <Image style={{ borderRadius: '8px' }} src="/my-image.png" alt="My Image" />
  )
}
```

當使用 `fill` 時，父元素必須具有 `position: relative` 或 `display: block`。這對於在佈局模式中正確呈現圖片元素是必要的。

```jsx
<div style={{ position: 'relative' }}>

  <Image fill src="/my-image.png" alt="My Image" />

</div>

```

您無法使用 [styled-jsx](/docs/app/guides/css-in-js)，因為它限制在目前的元件中 (除非您將樣式標記為 `global`)。

### 使用靜態導出的響應式圖片 (Responsive images with a static export)
當您導入靜態圖片時，Next.js 會根據檔案自動設定其寬度和高度。您可以透過設定樣式來使圖片響應式：

```jsx

import mountains from '../public/mountains.jpg'

        // 並保留其長寬比
```

### 遠端圖片 (Remote images)
要使用遠端圖片，`src` 屬性應為 URL 字串。

```jsx filename="app/page.js"
import Image from 'next/image'

export default function Page() {
  return (
    <Image
      src="https://s3.amazonaws.com/my-bucket/profile.png"
      alt="Picture of the author"
      width={500}
      height={500}
    />
  )
}
```

由於 Next.js 在建置過程中無法存取遠端檔案，您必須手動提供 [`width`](/docs/app/api-reference/components/image#width-and-height)、[`height`](/docs/app/api-reference/components/image#width-and-height) 和選填 [`blurDataURL`](/docs/app/api-reference/components/image#blurdataurl) 屬性。

`width` 和 `height` 屬性用於推斷圖片的正確長寬比，並避免因圖片載入而產生的佈局移位。`width` 和 `height` 並**不**決定圖片檔案的渲染大小。

為了安全地允許最佳化圖片，請在 `next.config.js` 中定義支援的 URL 模式列表。儘可能精確以防止惡意使用。例如，以下配置將僅允許來自特定 AWS S3 儲存區的圖片：

```js filename="next.config.js"
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 's3.amazonaws.com',
        port: '',
        pathname: '/my-bucket/**',
        search: '',
      },
    ],
  },
}
```

### 主題偵測 (Theme detection)
如果您想要為輕模式和暗模式顯示不同的圖片，您可以建立一個新元件來包裹兩個 `<Image>` 元件，並根據 CSS 媒體查詢顯示正確的圖片。

```css filename="components/theme-image.module.css"
.imgDark {
  display: none;
}

@media (prefers-color-scheme: dark) {
  .imgLight {
    display: none;
  }
  .imgDark {
    display: unset;
  }
}
```

```tsx filename="components/theme-image.tsx" switcher
import styles from './theme-image.module.css'
import Image, { ImageProps } from 'next/image'

type Props = Omit<ImageProps, 'src' | 'priority' | 'loading'> & {
  srcLight: string
  srcDark: string
}

const ThemeImage = (props: Props) => {
  const { srcLight, srcDark, ...rest } = props

  return (
    <>
      <Image {...rest} src={srcLight} className={styles.imgLight} />
      <Image {...rest} src={srcDark} className={styles.imgDark} />
    </>
  )
}
```

```jsx filename="components/theme-image.js" switcher
import styles from './theme-image.module.css'
import Image from 'next/image'

const ThemeImage = (props) => {
  const { srcLight, srcDark, ...rest } = props

  return (
    <>
      <Image {...rest} src={srcLight} className={styles.imgLight} />
      <Image {...rest} src={srcDark} className={styles.imgDark} />
    </>
  )
}
```

> **小提示**：`loading="lazy"` 的預設行為確保只會載入正確的圖片。您無法使用 `priority` 或 `loading="eager"`，因為這會導致兩張圖片都載入。相反地，您可以使用 [`fetchPriority="high"`](https://developer.mozilla.org/docs/Web/API/HTMLImageElement/fetchPriority)。

### 圖片方向調整 (Art direction)
如果您想要為手機和桌面顯示不同的圖片，有時稱為 [圖片方向調整 (Art Direction)](https://developer.mozilla.org/en-US/docs/Learn/HTML/Multimedia_and_embedding/Responsive_images#art_direction)，您可以為 `getImageProps()` 提供不同的 `src`、`width`、`height` 和 `quality` 屬性。

```jsx filename="app/page.js"
import { getImageProps } from 'next/image'

export default function Home() {
  const common = { alt: 'Art Direction Example', sizes: '100vw' }
  const {
    props: { srcSet: desktop },
  } = getImageProps({
    ...common,
    width: 1440,
    height: 875,
    quality: 80,
    src: '/desktop.jpg',
  })
  const {
    props: { srcSet: mobile, ...rest },
  } = getImageProps({
    ...common,
    width: 750,
    height: 1334,
    quality: 70,
    src: '/mobile.jpg',
  })

  return (
    <picture>
      <source media="(min-width: 1000px)" srcSet={desktop} />
      <source media="(min-width: 500px)" srcSet={mobile} />
      <img {...rest} style={{ width: '100%', height: 'auto' }} />
    </picture>
  )
}
```

### 背景 CSS (Background CSS)
您甚至可以將 `srcSet` 字串轉換為 [`image-set()`](https://developer.mozilla.org/en-US/docs/Web/CSS/image/image-set) CSS 函式以最佳化背景圖片。

```jsx filename="app/page.js"
import { getImageProps } from 'next/image'

function getBackgroundImage(srcSet = '') {
  const imageSet = srcSet
    .split(', ')
    .map((str) => {
      const [url, dpi] = str.split(' ')
      return `url("${url}") ${dpi}`
    })
    .join(', ')
  return `image-set(${imageSet})`
}

export default function Home() {
  const {
    props: { srcSet },
  } = getImageProps({ alt: '', width: 128, height: 128, src: '/img.png' })
  const backgroundImage = getBackgroundImage(srcSet)
  const style = { height: '100vh', width: '100vw', backgroundImage }

  return (
    <main style={style}>
      <h1>Hello World</h1>
    </main>
  )
}
```
