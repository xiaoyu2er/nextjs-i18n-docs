[`next/font`](/docs/app/api-reference/components/font) 會自動優化您的字體（包括自訂字體），並移除外部網路請求，以提升隱私性和效能。

它包含**內建的自動自託管**功能，適用於任何字體檔案。這意味著您可以最優化地載入網頁字體，而無 [佈局偏移 (Layout Shift)](https://web.dev/articles/cls)。

您也可以在方便地使用所有 [Google 字體 (Google Fonts)](https://fonts.google.com/)。CSS 和字體檔案會在建置時下載，並與其他靜態資產一起自託管。**瀏覽器不會向 Google 發送任何請求**。

<AppOnly>

```tsx filename="app/layout.tsx" switcher
import { Inter } from 'next/font/google'

// If loading a variable font, you don't need to specify the font weight
const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
})

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher

// If loading a variable font, you don't need to specify the font weight

  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  )
}
```

</AppOnly>

### src

作為字串或物件陣列（其類型為 `Array<{path: string, weight?: string, style?: string}>`）的字體檔案路徑，相對於呼叫字體載入器函式的目錄。

用於 `next/font/local`

- 必要

範例：

- `src:'./fonts/my-font.woff2'` 其中 `my-font.woff2` 放置在位於 `app` 目錄內的名為 `fonts` 的資料夾中
- `src:[{path: './inter/Inter-Thin.ttf', weight: '100',},{path: './inter/Inter-Regular.ttf',weight: '400',},{path: './inter/Inter-Bold-Italic.ttf', weight: '700',style: 'italic',},]`
- 如果字體載入器函式在 `app/page.tsx` 中使用 `src:'../styles/fonts/my-font.ttf'` 呼叫，那麼 `my-font.ttf` 放置在專案根目錄的 `styles/fonts` 中

### `weight`

字體的 [`weight`](https://fonts.google.com/knowledge/glossary/weight)，具有以下可能性：

- 字串，具有特定字體可能可用的粗細值，或如果是 [變數 (variable)](https://fonts.google.com/variablefonts) 字體則為值範圍
- 如果字體不是 [變數 Google 字體 (variable google font)](https://fonts.google.com/variablefonts)，則為粗細值陣列。這僅適用於 `next/font/google`。

用於 `next/font/google` 和 `next/font/local`

- 若所使用的字體**不是** [變數 (variable)](https://fonts.google.com/variablefonts) 則為必要

範例：

- `weight: '400'`：單一粗細值的字串 - 對於字體 [`Inter`](https://fonts.google.com/specimen/Inter?query=inter)，可能值為 `'100'`, `'200'`, `'300'`, `'400'`, `'500'`, `'600'`, `'700'`, `'800'`, `'900'` 或 `'variable'`（其預設值）
- `weight: '100 900'`：對於變數字體，介於 `100` 和 `900` 之間的字串
- `weight: ['100','400','900']`：針對非變數字體的 3 個可能值的陣列

### `style`

字體的 [`style`](https://developer.mozilla.org/docs/Web/CSS/font-style)，具有以下可能性：

- 字串 [值 (value)](https://developer.mozilla.org/docs/Web/CSS/font-style#values)，預設值為 `'normal'`
- 如果字體不是 [變數 Google 字體 (variable google font)](https://fonts.google.com/variablefonts)，則為類型值陣列。這僅適用於 `next/font/google`。

用於 `next/font/google` 和 `next/font/local`

- 選擇性

範例：

- `style: 'italic'`：字串 - 對於 `next/font/google` 它可以是 `normal` 或 `italic`
- `style: 'oblique'`：字串 - 對於 `next/font/local` 它可以取任何值，但預期來自 [標準字體樣式 (standard font styles)](https://developer.mozilla.org/docs/Web/CSS/font-style)
- `style: ['italic','normal']`：針對 `next/font/google` 包含 2 個值的陣列 - 值來自 `normal` 和 `italic`

### `subsets`

字體的 [`subsets`](https://fonts.google.com/knowledge/glossary/subsetting)，由包含您想要 [預載 (preloaded)](/docs/app/api-reference/components/font#specifying-a-subset) 的子集名稱的字串值陣列定義。透過 `subsets` 指定的字體將具有在 `<head>` 中注入的 link preload 標籤，當 [`preload`](#preload) 選項為 `true`（預設值）時。

用於 `next/font/google`

- 選擇性

範例：

- `subsets: ['latin']`：包含 `latin` 子集的陣列

您可以在 Google 字體頁面找到所有子集清單。

### `axes`

某些變數字體具有可包含的額外 `axes`。預設情況下，僅包含字體粗細以減少檔案大小。`axes` 的可能值取決於特定字體。

用於 `next/font/google`

- 選擇性

範例：

- `axes: ['slnt']`：具有 `slnt` 作為額外 `axes` 的 `Inter` 變數字體的陣列值，請參考此處 [此頁](https://fonts.google.com/variablefonts?vfquery=inter#font-families)。您可以透過使用 [Google 變數字體頁面](https://fonts.google.com/variablefonts#font-families) 上的篩選器並尋找 `wght` 以外的軸，找到您的字體的可能的 `axes` 值

### `display`

字體的 [`display`](https://developer.mozilla.org/docs/Web/CSS/@font-face/font-display)，具有可能字串 [值 (values)](https://developer.mozilla.org/docs/Web/CSS/@font-face/font-display#values) 為 `'auto'`, `'block'`, `'swap'`, `'fallback'` 或 `'optional'`，其預設值為 `'swap'`。

用於 `next/font/google` 和 `next/font/local`

- 選擇性

範例：

- `display: 'optional'`：分配給 `optional` 值的字串

### `preload`

指定字體是否應該 [預載 (preloaded)](/docs/app/api-reference/components/font#preloading) 布林值。預設值為 `true`。

用於 `next/font/google` 和 `next/font/local`

- 選擇性

範例：

- `preload: false`

### `fallback`

如果無法載入字體時要使用的置換字體。無預設值的置換字體字串陣列。

- 選擇性

用於 `next/font/google` 和 `next/font/local`

範例：

- `fallback: ['system-ui', 'arial']`：將置換字體設定為 `system-ui` 或 `arial` 的陣列

### `adjustFontFallback`

- 對於 `next/font/google`：設定是否應使用自動置換字體以減少 [累積佈局偏移 (Cumulative Layout Shift)](https://web.dev/cls/) 布林值。預設值為 `true`。
- 對於 `next/font/local`：設定是否應使用自動置換字體以減少 [累積佈局偏移 (Cumulative Layout Shift)](https://web.dev/cls/) 字串或布林 `false` 值。可能值為 `'Arial'`, `'Times New Roman'` 或 `false`。預設值為 `'Arial'`。

用於 `next/font/google` 和 `next/font/local`

- 選擇性

範例：

- `adjustFontFallback: false`：用於 `next/font/google`
- `adjustFontFallback: 'Times New Roman'`：用於 `next/font/local`

### `variable`

如果字體與 [CSS 變數方法](#css-variables) 套用的字串值，用於定義要使用的 CSS 變數名稱。

用於 `next/font/google` 和 `next/font/local`

- 選擇性

範例：

- `variable: '--my-font'`：已宣告 CSS 變數 `--my-font`

### `declarations`

定義所產生 `@font-face` 的更多字體樣式表 [描述 (descriptor)](https://developer.mozilla.org/docs/Web/CSS/@font-face#descriptors) 鍵值對的陣列。

用於 `next/font/local`

- 選擇性

範例：

- `declarations: [{ prop: 'ascent-override', value: '90%' }]`

### 在 `<head>` 中套用字體

您也可以在不使用包裝器和 `className` 的情況下使用字體，如下所示將它注入 `<head>` 中：

```jsx filename="pages/_app.js"
import { Inter } from 'next/font/google'

const inter = Inter({ subsets: ['latin'] })

export default function MyApp({ Component, pageProps }) {
  return (
    <>
      <style jsx global>{`
        html {
          font-family: ${inter.style.fontFamily};
        }
      `}</style>
      <Component {...pageProps} />
    </>
  )
}
```

## 使用多個字體

您可以在應用程式中匯入並使用多個字體。您可以採取兩種方法。

第一個方法是一建立工具函式來匯出字體、匯入它，並在需要時套用其 `className`。這確保字體僅在渲染時才預載：

```ts filename="app/fonts.ts" switcher
import { Inter, Roboto_Mono } from 'next/font/google'

export const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
})

export const roboto_mono = Roboto_Mono({
  subsets: ['latin'],
  display: 'swap',
})
```

```js filename="app/fonts.js" switcher
import { Inter, Roboto_Mono } from 'next/font/google'

export const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
})

export const roboto_mono = Roboto_Mono({
  subsets: ['latin'],
  display: 'swap',
})
```

或者，您可以建立 [CSS 變數 (CSS variable)](/docs/app/api-reference/components/font#variable) 並使用您喜歡的 CSS 方案：

```tsx filename="app/layout.tsx" switcher
import { Inter, Roboto_Mono } from 'next/font/google'
import styles from './global.css'

const inter = Inter({
  subsets: ['latin'],
  variable: '--font-inter',
  display: 'swap',
})

const roboto_mono = Roboto_Mono({
  subsets: ['latin'],
  variable: '--font-roboto-mono',
  display: 'swap',
})

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={`${inter.variable} ${roboto_mono.variable}`}>
      <body>
        <h1>My App</h1>
        <div>{children}</div>
      </body>
    </html>
  )
}
```

```js
      weight: '700',
      style: 'italic',
    },
  ],
})
```

請查看 [字體 API 參考 (Font API Reference)](/docs/app/api-reference/components/font) 以獲取更多資訊。

### 搭配 Tailwind CSS

`next/font` 使用 [CSS 變數 (CSS variables)](/docs/app/api-reference/components/font#css-variables) 與 [Tailwind CSS](https://tailwindcss.com/) 無縫整合。

以下範例中，我們使用來自 `next/font/google` 的 `Inter` 和 `Roboto_Mono` 字體（您可以使用任何 Google 字體或本機字體）。使用 `variable` 選項定義 CSS 變數名稱，例如分別為 `inter` 和 `roboto_mono`。然後，將 `inter.variable` 和 `roboto_mono.variable` 套用至您的 HTML 文件。

> **小提示**：您可以根據偏好、樣式需求或專案需求，將這些變數加入 `<html>` 或 `<body>` 標籤。

<AppOnly>

```tsx filename="app/layout.tsx" switcher
import { Inter, Roboto_Mono } from 'next/font/google'

const inter = Inter({
  subsets: ['latin'],
  display: 'swap',
  variable: '--font-inter',
})

const roboto_mono = Roboto_Mono({
  subsets: ['latin'],
  display: 'swap',
  variable: '--font-roboto-mono',
})

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html
      lang="en"
      className={`${inter.variable} ${roboto_mono.variable} antialiased`}
    >
      <body>{children}</body>
    </html>
  )
}
```

### 套用樣式

您可以透過三種方式套用字體樣式：

- [`className`](#classname)
- [`style`](#style-1)
- [CSS 變數 (CSS Variables)](#css-variables)

#### `className`

已載入字體的回傳唯讀 CSS `className`，以供傳遞給 HTML 元素。

```tsx
<p className={inter.className}>Hello, Next.js!</p>
```

#### `style`

已載入字體的回傳唯讀 CSS `style` 物件，以供傳遞給 HTML 元素，包含 `style.fontFamily` 以存取字體族名稱和置換字體。

```tsx
<p style={inter.style}>Hello World</p>
```

#### CSS 變數

如果您想在外部樣式表中設定樣式並在其中指定其他選項，請使用 CSS 變數方法。

除了匯入字體，也匯入定義 CSS 變數的 CSS 檔案，並將字體載入器物件的 `variable` 選項設定如下：

```tsx filename="app/page.tsx" switcher
import { Inter } from 'next/font/google'
import styles from '../styles/component.module.css'

const inter = Inter({
  variable: '--font-inter',
})
```

```jsx filename="app/page.js" switcher
import { Inter } from 'next/font/google'
import styles from '../styles/component.module.css'

const inter = Inter({
  variable: '--font-inter',
})
```

若要使用字體，將您想要樣式化文字的父容器之 `className` 設定為字體載入器的 `variable` 值，並將文字的 `className` 設定為外部 CSS 檔案中的 `styles` 屬性。

```tsx filename="app/page.tsx" switcher
<main className={inter.variable}>
  <p className={styles.text}>Hello World</p>
</main>
```

```jsx filename="app/page.js" switcher
<main className={inter.variable}>
  <p className={styles.text}>Hello World</p>
</main>
```

在 `component.module.css` CSS 檔案中定義 `text` 選擇器類別如下：

```css filename="styles/component.module.css"
.text {
  font-family: var(--font-inter);
  font-weight: 200;
  font-style: italic;
}
```

在上面的範例中，文字 `Hello World` 是使用 `Inter` 字體和所產生的字體置換來樣式化，設定 `font-weight: 200` 和 `font-style: italic`。

### 使用字體定義檔案

每次匯入字體定義檔案 `localFont` 或 Google 字體時，該字體都會在您的應用程式作為一個執行個體託管。因此，如果您需要在多個地方使用相同字體，您應該在一個地方載入它，並在需要處匯入相關字體物件。這是使用字體定義檔案完成的。

例如，在應用程式目錄根索引區中的 `styles` 資料夾建立 `fonts.ts` 檔案。

然後，如下所示指定您的字體定義：

```ts filename="styles/fonts.ts" switcher
import { Inter, Lora, Source_Sans_3 } from 'next/font/google'
import localFont from 'next/font/local'

// define your variable fonts
const inter = Inter()
const lora = Lora()
// define 2 weights of a non-variable font
const sourceCodePro400 = Source_Sans_3({ weight: '400' })
const sourceCodePro700 = Source_Sans_3({ weight: '700' })
// define a custom local font where GreatVibes-Regular.ttf is stored in the styles folder
const greatVibes = localFont({ src: './GreatVibes-Regular.ttf' })

export { inter, lora, sourceCodePro400, sourceCodePro700, greatVibes }
```

```js filename="styles/fonts.js" switcher
import { Inter, Lora, Source_Sans_3 } from 'next/font/google'
import localFont from 'next/font/local'

// define your variable fonts
const inter = Inter()
const lora = Lora()
// define 2 weights of a non-variable font
const sourceCodePro400 = Source_Sans_3({ weight: '400' })
const sourceCodePro700 = Source_Sans_3({ weight: '700' })
// define a custom local font where GreatVibes-Regular.ttf is stored in the styles folder
const greatVibes = localFont({ src: './GreatVibes-Regular.ttf' })

export { inter, lora, sourceCodePro400, sourceCodePro700, greatVibes }
```

您現在可以在程式碼中使用這些定義如下：

```tsx filename="app/page.tsx" switcher
import { inter, lora, sourceCodePro700, greatVibes } from '../styles/fonts'

export default function Page() {
  return (
    <div>
      <p className={inter.className}>Hello world using Inter font</p>
      <p style={lora.style}>Hello world using Lora font</p>
      <p className={sourceCodePro700.className}>
        Hello world using Source_Sans_3 font with weight 700
      </p>
      <p className={greatVibes.className}>My title in Great Vibes font</p>
    </div>
  )
}
```

```jsx filename="app/page.js" switcher
import { inter, lora, sourceCodePro700, greatVibes } from '../styles/fonts'

  return (
    <div>
      <p className={inter.className}>Hello world using Inter font</p>

        Hello world using Source_Sans_3 font with weight 700
      </p>
      <p className={greatVibes.className}>My title in Great Vibes font</p>
    </div>
  )
}
```

為了更輕鬆地存取程式碼中的字體定義，您可以在 `tsconfig.json` 或 `jsconfig.json` 檔案中定義路徑別名，如下所示：

```json filename="tsconfig.json"
{
  "compilerOptions": {
    "paths": {
      "@/fonts": ["./styles/fonts"]
    }
  }
}
```

您現在可以匯入任何字體定義如下：

```tsx filename="app/about/page.tsx" switcher

```

```jsx filename="app/about/page.js" switcher

```

### 預載

<AppOnly>

當字體函式在您的網站的頁面呼叫時，它並非全域可用且在所有路徑上預載。相反地，字體僅在關聯路徑上預載，取決於使用它的檔案類型：

- 如果它是 [唯一頁面](/docs/app/api-reference/file-conventions/page)，它會在該頁面的唯一路徑上預載。
- 如果它是 [佈局](/docs/app/api-reference/file-conventions/layout)，它會在所有被佈路由路徑上預載。
- 如果它是 [根佈局 (root layout)](/docs/app/api-reference/file-conventions/layout#root-layout)，它會在所有路徑上預載。

</AppOnly>

<PagesOnly>

當字體函式在您的網站的頁面呼叫時，它並非全域可用且在所有路徑上預載。相反地，字體僅在關聯路徑上預載，取決於使用它的檔案類型：

- 如果它是 [唯一頁面 (unique page)](/docs/pages/building-your-application/routing/pages-and-layouts)，它會在該頁面的唯一路徑上預載
- 如果它在 [自訂 App](/docs/pages/building-your-application/routing/custom-app) 中，它會在 `/pages` 下網站的所有路徑上預載

</PagesOnly>
