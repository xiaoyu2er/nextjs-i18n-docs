<Form> 組件擴充了 HTML <form> 元素，以提供 <AppOnly>[**預先載入 (prefetching)**](/docs/app/getting-started/linking-and-navigating#prefetching) 的 [載入 UI](/docs/app/api-reference/file-conventions/loading)，</AppOnly> 在提交時進行 **客戶端導航**，並支援 **漸進增強**。

它對於更新 URL 搜尋參數的表單非常有用的，因為它減少了實現上述功能所需的样板代碼。

基本用法：

<AppOnly>

```tsx filename="/app/ui/search.tsx" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      {/* On submission, the input value will be appended to
          the URL, e.g. /search?query=abc */}
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

```jsx filename="/app/ui/search.js" switcher
import Form from 'next/form'

export default function Search() {
  return (
    <Form action="/search">
      {/* On submission, the input value will be appended to
          the URL, e.g. /search?query=abc */}
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

</AppOnly>

## 參考

<Form> 組件的行為取決於是否為 `action` prop 傳入 `string` 或 `function`。

<AppOnly>

- 當 `action` 為 **字串** 時，`<Form>` 的行為類似於使用 **`GET`** 方法的本地 HTML 表單。表單資料會編碼為 URL 搜尋參數，當表單提交時，會導航至指定的 URL。此外，Next.js 會:
  - [在表單可見時預先載入](/docs/app/getting-started/linking-and-navigating#prefetching) 路徑，這會預先載入共用 UI（例如 `layout.js` 和 `loading.js`），從而實現更快的導航。
  - 在表單提交時執行 [客戶端導航](/docs/app/getting-started/linking-and-navigating#client-side-transitions) 而非全頁面重新整理。這將保留共用的 UI 和客戶端狀態。
- 當 `action` 為 **函式** (伺服器操作 (Server Action)) 時，`<Form>` 的行為類似於 [React 表單](https://react.dev/reference/react-dom/components/form)，在表單提交時執行該操作。

</AppOnly>

<PagesOnly>

- 當 `action` 為 **字串** 時，`<Form>` 的行為類似於使用 **`GET`** 方法的本地 HTML 表單。表單資料會編碼為 URL 搜尋參數，當表單提交時，會導航至指定的 URL。此外，Next.js 會執行 [客戶端導航](/docs/app/getting-started/linking-and-navigating#client-side-transitions) 而非全頁面重新整理，在表單提交時。這將保留共用的 UI 和客戶端狀態。

</PagesOnly>

### `action` (string) 屬性

<PagesOnly>

當 `action` 為字串時，<Form> 組件支援以下屬性:

| 屬性        | 範例            | 型別                            | 必需 |
| --------- | ------------------ | ------------------------------- | -------- |
| `action`  | `action="/search"` | `string` (URL 或相對路徑) | 是      |
| `replace` | `replace={false}`  | `boolean`                       | -        |
| `scroll`  | `scroll={true}`    | `boolean`                       | -        |

- **`action`**: 表單提交時要導航到的 URL 或路徑。
  - 空字串 `""` 將導航至同一個路由並更新搜尋參數。
- **`replace`**: 取代目前的歷史狀態，而不是推入新的歷史狀態到 [瀏覽器的歷史 (History)](https://developer.mozilla.org/en-US/docs/Web/API/History_API) 堆疊。預設為 `false`。
- **`scroll`**: 控制導航時的捲動行為。預設為 `true`，這表示它會將頁面捲動至新路由的頂部，並為反向和正向導航維護捲動位置。

</PagesOnly>

<AppOnly>

當 `action` 為字串時，<Form> 組件支援以下屬性:

| 屬性         | 範例            | 型別                            | 必需 |
| ---------- | ------------------ | ------------------------------- | -------- |
| `action`   | `action="/search"` | `string` (URL 或相對路徑) | 是      |
| `replace`  | `replace={false}`  | `boolean`                       | -        |
| `scroll`   | `scroll={true}`    | `boolean`                       | -        |
| `prefetch` | `prefetch={true}`  | `boolean`                       | -        |

- **`action`**: 表單提交時要導航到的 URL 或路徑。
  - 空字串 `""` 將導航至同一個路由並更新搜尋參數。
- **`replace`**: 取代目前的歷史狀態，而不是推入新的歷史狀態到 [瀏覽器的歷史 (History)](https://developer.mozilla.org/en-US/docs/Web/API/History_API) 堆疊。預設為 `false`。
- **`scroll`**: 控制導航時的捲動行為。預設為 `true`，這表示它會將頁面捲動至新路由的頂部，並為反向和正向導航維護捲動位置。
- **`prefetch`**: 控制當表單出現在使用者視區時是否預先載入路徑。預設為 `true`。

### `action` (function) 屬性

當 `action` 為函式時，<Form> 組件支援以下屬性:

| 屬性     | 範例              | 型別                       | 必需 |
| -------- | ------------------- | -------------------------- | -------- |
| `action` | `action={myAction}` | `function` (伺服器操作 (Server Action)) | 是      |

- **`action`**: 表單提交時要呼叫的伺服器操作 (Server Action)。詳情請參閱 [React 文件](https://react.dev/reference/react-dom/components/form#props)。

> **值得一提**: 當 `action` 為函式時，`replace` 和 `scroll` 屬性將被忽略。

</AppOnly>

### 注意事項

<AppOnly>

- **`formAction`**: 可用於 `<button>` 或 `<input type="submit">` 欄位以覆蓋 `action` prop。Next.js 將執行客戶端導航，然而，此方法不支援預先載入。
  - 使用 [`basePath`](/docs/app/api-reference/config/next-config-js/basePath) 時，也必須將其包含在 `formAction` 路徑中。例如 `formAction="/base-path/search"`。
- **`key`**: 傳遞 `key` prop 給字串 `action` 是不支援的。如果您想觸發重新整理或執行變更，考慮改用函式 `action`。

</AppOnly>

- **`onSubmit`**: 可用來處理表單提交邏輯。然而，呼叫 `event.preventDefault()` 將覆蓋 `<Form>` 行為，例如導航至指定 URL。
- **[方式 (method)](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#method#https...)**, **[格式類型 (encType)](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#enctype)**, **[目標 (target)](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#target)**: 不支援，因為它們會覆蓋 `<Form>` 行為。
  - 同樣地，`formMethod`, `formEncType` 和 `formTarget` 可用來分別覆蓋 `method`, `encType` 和 `target` 屬性，使用它們將回退到原生瀏覽器行為。
  - 如果您需要使用這些屬性，請使用 HTML `<form>` 元素。
- **`<input type="file">`**: 當 `action` 為字串時使用此輸入類型的行為將與瀏覽器行為一致，提交的是檔名而非檔案物件。

<AppOnly>

## 範例

### 導向搜尋結果頁的搜尋表單

您可以透過將路徑作為 `action` 傳遞來建立一個導航至搜尋結果頁的搜尋表單：

```tsx filename="/app/page.tsx" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

```jsx filename="/app/page.js" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

當使用者更新查詢輸入欄位並提交表單時，表單資料將被編碼為 URL 的搜尋參數，例如 `/search?query=abc`。

> **值得一提**: 如果您傳入空字串 `""` 至 `action`，表單將導航至同一個路由並帶有更新的搜尋參數。

在結果頁上，可以使用 [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) `page.js` prop 來存取查詢參數，並用它來從外部來源撷取資料。

```tsx filename="/app/search/page.tsx" switcher
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const results = await getSearchResults((await searchParams).query)

  return <div>...</div>
}
```

```jsx filename="/app/search/page.js" switcher
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({ searchParams }) {
  const results = await getSearchResults((await searchParams).query)

  return <div>...</div>
}
```

當 `<Form>` 在使用者的視區中可見時，`/search` 頁面上的共用 UI（例如 `layout.js` 和 `loading.js`）將被預先載入。在提交時，表單將立即導航至新路線並顯示載入 UI，同時撷取結果。您可以使用 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 設計後備 UI：

```tsx filename="/app/search/loading.tsx" switcher
export default function Loading() {
  return <div>Loading...</div>
}
```

```jsx filename="/app/search/loading.js" switcher
export default function Loading() {
  return <div>Loading...</div>
}
```

為了涵蓋共用 UI 尚未載入的情況，您可以使用 [`useFormStatus`](https://react.dev/reference/react-dom/hooks/useFormStatus) 向使用者顯示即時回饋。

首先，建立一個當表單等待時顯示載入狀態的組件：

```tsx filename="/app/ui/search-button.tsx" switcher
'use client'
import { useFormStatus } from 'react-dom'

export default function SearchButton() {
  const status = useFormStatus()
  return (
    <button type="submit">{status.pending ? 'Searching...' : 'Search'}</button>
  )
}
```

```jsx filename="/app/ui/search-button.js" switcher
'use client'
import { useFormStatus } from 'react-dom'

export default function SearchButton() {
  const status = useFormStatus()
  return (
    <button type="submit">{status.pending ? 'Searching...' : 'Search'}</button>
  )
}
```

接著，更新搜尋表單頁面以使用 `SearchButton` 組件：

```tsx filename="/app/page.tsx" switcher
import Form from 'next/form'
import { SearchButton } from '@/ui/search-button'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <SearchButton />
    </Form>
  )
}
```

```jsx filename="/app/ui/search-button.js" switcher
import Form from 'next/form'
import { SearchButton } from '@/ui/search-button'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <SearchButton />
    </Form>
  )
}
```

### 使用伺服器操作 (Server Actions) 進行變更

您可以將函式傳遞至 `action` prop 來執行變更。

```tsx filename="/app/posts/create/page.tsx" switcher
import Form from 'next/form'
import { createPost } from '@/posts/actions'

export default function Page() {
  return (
    <Form action={createPost}>
      <input name="title" />
      {/* ... */}
      <button type="submit">Create Post</button>
    </Form>
  )
}
```

```jsx filename="/app/posts/create/page.js" switcher
import Form from 'next/form'
import { createPost } from '@/posts/actions'

export default function Page() {
  return (
    <Form action={createPost}>
      <input name="title" />
      {/* ... */}
      <button type="submit">Create Post</button>
    </Form>
  )
}
```

變更後，常見的做法是重新導引至新資源。您可以使用 `next/navigation` 的 [`redirect`](/docs/app/guides/redirecting) 函式來導航至新的貼文頁面。

> **值得一提**: 由於直到 `action` 執行後才知道表單提交的 "目的地"，`<Form>` 無法自動預先載入共用 UI。

```tsx filename="/app/posts/actions.ts" switcher
'use server'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  // Create a new post
  // ...

  // Redirect to the new post
  redirect(`/posts/${data.id}`)
}
```

```jsx filename="/app/posts/actions.js" switcher
'use server'
import { redirect } from 'next/navigation'

export async function createPost(formData) {
  // Create a new post
  // ...

  // Redirect to the new post
  redirect(`/posts/${data.id}`)
}
```

接著，在新頁面中，您可以使用 `params` prop 來撷取資料：

```tsx filename="/app/posts/[id]/page.tsx" switcher
import { getPost } from '@/posts/data'

export default async function PostPage({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const data = await getPost(id)

  return (
    <div>
      <h1>{data.title}</h1>
      {/* ... */}
    </div>
  )
}
```

```jsx filename="/app/posts/[id]/page.js" switcher
import { getPost } from '@/posts/data'

export default async function PostPage({ params }) {
  const { id } = await params
  const data = await getPost(id)

  return (
    <div>
      <h1>{data.title}</h1>
      {/* ... */}
    </div>
  )
}
```
