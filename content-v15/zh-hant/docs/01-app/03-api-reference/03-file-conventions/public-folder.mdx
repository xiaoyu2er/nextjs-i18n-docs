Next.js 可以在根目錄下的 `public` 資料夾中提供靜態檔案，例如圖片。`public` 內部的檔案則可以由您的程式碼從基礎 URL (`/`) 開始引用。

例如，檔案 `public/avatars/me.png` 可以通過造訪 `/avatars/me.png` 路徑查看。顯示該圖片的程式碼可能如下：

```jsx filename="avatar.js"
import Image from 'next/image'

export function Avatar({ id, alt }) {
  return <Image src={`/avatars/${id}.png`} alt={alt} width="64" height="64" />
}

export function AvatarOfMe() {
  return <Avatar id="me" alt="A portrait of me" />
}
```

## 快取

Next.js 無法安全地快取 `public` 資料夾中的資源，因為它們可能會變更。預設套用的快取標頭如下：

```jsx
Cache-Control: public, max-age=0
```

## Robots.txt、Favicon 及其他

<PagesOnly>

此資料夾也適用於 `robots.txt`、`favicon.ico`、Google 網站驗證，以及任何其他靜態檔案（包括 `.html`）。但請確保不要擁有與 `pages/` 目錄中的檔案同名之靜態檔案，否則會產生錯誤。[閱讀更多](/docs/messages/conflicting-public-file-page)。

</PagesOnly>

<AppOnly>

對於靜態元數據 (metadata) 檔案，如 `robots.txt`、`favicon.ico` 等，您應該在 `app` 資料夾內使用 [特殊元數據 (metadata) 檔案](/docs/app/api-reference/file-conventions/metadata)。

</AppOnly>
