title: Metadata Files API 參考
nav_title: Metadata Files

---

文件的這個部分涵蓋了 **Metadata 檔案規範**。基於檔案的 metadata 可以通過向路由區段添加特殊的 metadata 檔案來定義。

每個檔案規範都可以通過靜態檔案定義（例如：`opengraph-image.jpg`），或使用程式碼生成檔案的動態變體（例如：`opengraph-image.js`）。

一旦檔案被定義，Next.js 將會自動提供該檔案（生產環境包含雜湊值以用於快取），並用正確的 metadata 更新相關的 head 元素，例如資源的 URL、檔案類型和圖片尺寸。

> **好消息**：
>
> - 特殊路由處理文件如 [`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap)、[`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image) 和 [`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons)，以及其他 [metadata 檔案](/docs/app/api-reference/file-conventions/metadata) 預設會進行快取。
> - 如果與 [`middleware.ts`](/docs/app/api-reference/file-conventions/middleware) 一併使用，請 [設定匹配器](/docs/app/api-reference/file-conventions/middleware#matcher) 以排除 metadata 檔案。
