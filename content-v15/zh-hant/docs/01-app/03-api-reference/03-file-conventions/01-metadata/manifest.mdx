新增或產生一個 `manifest.(json|webmanifest)` 檔案，該檔案需符合 [Web Manifest 規範](https://developer.mozilla.org/docs/Web/Manifest)，並放置於 `app` 目錄的 **根目錄** 中，以向瀏覽器提供關於您網頁應用程式的資訊。

## 靜態 Manifest 檔案

```json filename="app/manifest.json | app/manifest.webmanifest"
{
  "name": "My Next.js Application",
  "short_name": "Next.js App",
  "description": "An application built with Next.js",
  "start_url": "/"
  // ...
}
```

## 產生 Manifest 檔案

新增一個 `manifest.js` 或 `manifest.ts` 檔案，該檔案需回傳 [`Manifest` 物件](#manifest-object)。

> 值得注意：`manifest.js` 是一種特殊的 Route Handler (路由處理器)，預設會快取，除非它使用了 [動態 API (Dynamic API)](/docs/app/guides/caching#dynamic-apis) 或 [動態配置 (dynamic config)](/docs/app/guides/caching#segment-config-options) 選項。

```ts filename="app/manifest.ts" switcher
import type { MetadataRoute } from 'next'

export default function manifest(): MetadataRoute.Manifest {
  return {
    name: 'Next.js App',
    short_name: 'Next.js App',
    description: 'Next.js App',
    start_url: '/',
    display: 'standalone',
    background_color: '#fff',
    theme_color: '#fff',
    icons: [
      {
        src: '/favicon.ico',
        sizes: 'any',
        type: 'image/x-icon',
      },
    ],
  }
}
```

```js filename="app/manifest.js" switcher
export default function manifest() {
  return {
    name: 'Next.js App',
    short_name: 'Next.js App',
    description: 'Next.js App',
    start_url: '/',
    display: 'standalone',
    background_color: '#fff',
    theme_color: '#fff',
    icons: [
      {
        src: '/favicon.ico',
        sizes: 'any',
        type: 'image/x-icon',
      },
    ],
  }
}
```

### Manifest 物件

Manifest 物件包含一份廣泛的選項列表，可能因新的網頁標準而更新。若要查詢所有目前的選項資訊，若您使用 [TypeScript](/docs/app/api-reference/config/typescript#ide-plugin)，請參閱程式碼編輯器中的 `MetadataRoute.Manifest` 類型，或參閱 [MDN](https://developer.mozilla.org/docs/Web/Manifest) 文件。
