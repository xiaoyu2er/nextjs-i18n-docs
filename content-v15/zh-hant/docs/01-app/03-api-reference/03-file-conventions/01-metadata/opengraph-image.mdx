 `opengraph-image` 和 `twitter-image` 檔案慣例允許您為路由路段設定 Open Graph 和 Twitter 圖片。

當用戶分享您的網站連結時，它們非常有用，可用於設定顯示在社交網路和訊息應用程式中的圖片。

有兩種方式可以設定 Open Graph 和 Twitter 圖片：

- [使用圖片檔案 (.jpg, .png, .gif)](#image-files-jpg-png-gif)
- [使用程式碼生成圖片 (.js, .ts, .tsx)](#generate-images-using-code-js-ts-tsx)

## 圖片檔案 (.jpg, .png, .gif)

 透過將 `opengraph-image` 或 `twitter-image` 圖片檔案放置於路段中，使用圖片檔案來設定路由路段的共用圖片。

Next.js 將評估該檔案，並自動將適當的標籤新增到您的應用程式的 `<head>` 元素中。

| 檔案慣例 | 支援的檔案類型 |
| ----------------------------------------------- | ------------------------------- |
| [`opengraph-image`](#opengraph-image) | .jpg, .jpeg, .png, .gif |
| [`twitter-image`](#twitter-image) | .jpg, .jpeg, .png, .gif |
| [`opengraph-image.alt`](#opengraph-imagealttxt) | .txt |
| [`twitter-image.alt`](#twitter-imagealttxt) | .txt |

> **值得了解**：
>
> `twitter-image` 檔案大小不得超過 [5MB](https://developer.x.com/en/docs/x-for-websites/cards/overview/summary)，且 `opengraph-image` 檔案大小不得超過 [8MB](https://developers.facebook.com/docs/sharing/webmasters/images)。如果圖片檔案大小超過這些限制，建置將會失敗。

### opengraph-image

 將 `opengraph-image.(jpg|jpeg|png|gif)` 圖片檔案新增至任何路由路段。

```html filename="<head> output"
<meta property="og:image" content="<generated>" />
<meta property="og:image:type" content="<generated>" />
<meta property="og:image:width" content="<generated>" />
<meta property="og:image:height" content="<generated>" />
```

### twitter-image

 將 `twitter-image.(jpg|jpeg|png|gif)` 圖片檔案新增至任何路由路段。

```html filename="<head> output"
<meta name="twitter:image" content="<generated>" />
<meta name="twitter:image:type" content="<generated>" />
<meta name="twitter:image:width" content="<generated>" />
<meta name="twitter:image:height" content="<generated>" />
```

### opengraph-image.alt.txt

 在與 `opengraph-image.(jpg|jpeg|png|gif)` 圖片相同的路由路段中，新增隨附的 `opengraph-image.alt.txt` 檔案。

```txt filename="opengraph-image.alt.txt"
About Acme
```

```html filename="<head> output"
<meta property="og:image:alt" content="About Acme" />
```

### twitter-image.alt.txt

 在與 `twitter-image.(jpg|jpeg|png|gif)` 圖片相同的路由路段中，新增隨附的 `twitter-image.alt.txt` 檔案。

```txt filename="twitter-image.alt.txt"
About Acme
```

```html filename="<head> output"
<meta property="twitter:image:alt" content="About Acme" />
```

## 使用程式碼生成圖片 (.js, .ts, .tsx)

 除了使用 [圖片檔案](#image-files-jpg-png-gif) 外，您還可以程式化地 **生成** 圖片。

 透過建立預設匯出一個函式的 `opengraph-image` 或 `twitter-image` 路由，來生成路由路段的共用圖片。

| 檔案慣例 | 支援的檔案類型 |
| ----------------- | -------------------- |
| `opengraph-image` | .js, .ts, .tsx |
| `twitter-image` | .js, .ts, .tsx |

> **值得了解**：
>
> - 預設情況下，生成的圖片會進行 [**靜態最佳化**](/docs/app/getting-started/partial-prerendering#static-rendering)（在建置時生成並快取），除非使用了 [動態 API](/docs/app/getting-started/partial-prerendering#dynamic-rendering#dynamic-apis) 或未快取的數據。
> - 您可以在同一個檔案中使用 [`generateImageMetadata`](/docs/app/api-reference/functions/generate-image-metadata) 生成多個圖片。
> - `opengraph-image.js` 和 `twitter-image.js` 是特殊的 Route Handlers，預設會進行快取，除非它使用了 [動態 API](/docs/app/guides/caching#dynamic-apis) 或 [dynamic config](/docs/app/guides/caching#segment-config-options) 選項。

 生成圖片 simplest 的方式是使用 `next/og` 中的 [ImageResponse](/docs/app/api-reference/functions/image-response) API。

```tsx filename="app/about/opengraph-image.tsx" switcher
import { ImageResponse } from 'next/og'
import { readFile } from 'node:fs/promises'
import { join } from 'node:path'

// Image metadata
export const alt = 'About Acme'
export const size = {
  width: 1200,
  height: 630,
}

export const contentType = 'image/png'

// Image generation
export default async function Image() {
  // Font loading, process.cwd() is Next.js project directory
  const interSemiBold = await readFile(
    join(process.cwd(), 'assets/Inter-SemiBold.ttf')
  )

  return new ImageResponse(
    (
      // ImageResponse JSX element
      <div
        style={{
          fontSize: 128,
          background: 'white',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
        }}
      >
        About Acme
      </div>
    ),
    // ImageResponse options
    {
      // For convenience, we can re-use the exported opengraph-image
      // size config to also set the ImageResponse's width and height.
      ...size,
      fonts: [
        {
          name: 'Inter',
          data: interSemiBold,
          style: 'normal',
          weight: 400,
        },
      ],
    }
  )
}
```

### 屬性 (Props)

預設匯出的函數收到以下屬性：

#### `params` (選填)

 包含從根路段到 `opengraph-image` 或 `twitter-image` 所在路段之 [動態路由參數](/docs/app/api-reference/file-conventions/dynamic-routes) 物件。

```tsx filename="app/shop/[slug]/opengraph-image.tsx" switcher
export default function Image({ params }: { params: { slug: string } }) {
  // ...
}
```

```jsx filename="app/shop/[slug]/opengraph-image.js" switcher
export default function Image({ params }) {
  // ...
}
```

| 路由 | URL | params |
| ------------------------------------------ | ----------- | ------------------------- |
| `app/shop/opengraph-image.js` | /shop | undefined |
| `app/shop/[slug]/opengraph-image.js` | /shop/1 | { slug: '1' } |
| `app/shop/[tag]/[item]/opengraph-image.js` | /shop/1/2 | { tag: '1', item: '2' } |

### 回傳值

預設匯出的函數應回傳 `Blob` | `ArrayBuffer` | `TypedArray` | `DataView` | `ReadableStream` | `Response`。

> **貼心提示**: `ImageResponse` 符合此回傳類型。

### 組態匯出 (Config exports)

 您可以選擇性地從 `opengraph-image` 或 `twitter-image` 路由匯出 `alt`、`size` 和 `contentType` 變數，以配置圖片的元數據。

| 選項 | 類型 |
| ----------------------------- | --------------------------------------------------------------------------------------------------------------- |
| [`alt`](#alt) | string |
| [`size`](#size) | { width: number; height: number } |
| [`contentType`](#contenttype) | string - [image MIME type](https://developer.mozilla.org/docs/Web/HTTP/Basics_of_HTTP/MIME_types#image_types) |


#### 路由區段組態 (Route Segment Config)

 `opengraph-image` 和 `twitter-image` 是專門的 [Route Handlers](/docs/app/api-reference/file-conventions/route)，可以使用與 Pages 和 Layouts 相同的 [路由路段配置](/docs/app/api-reference/file-conventions/route-segment-config) 選項。

### 範例

#### 使用外部資料

 此範例使用 `params` 物件和外部數據來生成圖片。

> **值得了解**：
> 預設情況下，此生成的圖片將進行 [statically optimized](/docs/app/getting-started/partial-prerendering#static-rendering)。您可以配置單個 `fetch` options 或路由路段的 [options](/docs/app/api-reference/file-conventions/route-segment-config#revalidate) 來更改此行為。

```tsx filename="app/posts/[slug]/opengraph-image.tsx" switcher
import { ImageResponse } from 'next/og'

export const alt = 'About Acme'
export const size = {
  width: 1200,
  height: 630,
}
export const contentType = 'image/png'

export default async function Image({ params }: { params: { slug: string } }) {
  const post = await fetch(`https://.../posts/${params.slug}`).then((res) =>
    res.json()
  )

  return new ImageResponse(
    (
      <div
        style={{
          fontSize: 48,
          background: 'white',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
        }}
      >
        {post.title}
      </div>
    ),
    {
      ...size,
    }
  )
}
```


#### 使用 Node.js 執行環境與本機資源

 這些範例使用 Node.js 執行環境從檔案系統擷取本機圖片，並將其作為 base64 字串或 `ArrayBuffer` 傳遞到 `<img>` `src` 屬性。將本機資源放置在專案根部的相對路徑，而非範例源檔案。

```tsx filename="app/opengraph-image.tsx" switcher
import { ImageResponse } from 'next/og'
import { join } from 'node:path'
import { readFile } from 'node:fs/promises'

export default async function Image() {
  const logoData = await readFile(join(process.cwd(), 'logo.png'), 'base64')
  const logoSrc = `data:image/png;base64,${logoData}`

  return new ImageResponse(
    (
      <div
        style={{
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
        }}
      >
        <img src={logoSrc} height="100" />
      </div>
    )
  )
}
```


 將 `ArrayBuffer` 傳遞到 `<img>` 元素的 `src` 屬性並非 HTML 規範的一部分。`next/og` 使用的渲染引擎支援它，但因為 TypeScript 定義遵循規範，您需要 `@ts-expect-error` 指令或類似方法才能使用此 [功能](https://github.com/vercel/satori/issues/606#issuecomment-2144000453)。

```tsx filename="app/opengraph-image.tsx" switcher
import { ImageResponse } from 'next/og'
import { join } from 'node:path'
import { readFile } from 'node:fs/promises'

export default async function Image() {
  const logoData = await readFile(join(process.cwd(), 'logo.png'))
  const logoSrc = Uint8Array.from(logoData).buffer

  return new ImageResponse(
    (
      <div
        style={{
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
        }}
      >
        {/* @ts-expect-error Satori accepts ArrayBuffer/typed arrays for <img src> at runtime */}
        <img src={logoSrc} height="100" />
      </div>
    )
  )
}
```

