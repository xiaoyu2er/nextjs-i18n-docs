除了將 Next.js 特殊的 `app` 或 `pages` 目錄放在專案根目錄之外，Next.js 也支援將應用程式程式碼放置在 `src` 文件夾下的常見模式。

這將應用程式程式碼與主要位於專案根目錄的專案組態檔案區隔開來，這是某些個人和團隊偏好的做法。

若要使用 `src` 文件夾，請分別將 `app` Router 資料夾或 `pages` Router 資料夾移至 `src/app` 或 `src/pages`。

> **值得了解**：
>
> - `/public` 目錄應保留在專案根目錄。
> - 組態檔案（例如 `package.json`、`next.config.js` 和 `tsconfig.json`）應保留在專案根目錄。
> - `.env.*` 檔案應保留在專案根目錄。
> - 如果根目錄中存在 `app` 或 `pages`，將忽略 `src/app` 或 `src/pages`。
> - 如果您使用 `src`，可能也會將其他應用程式資料夾（如 `/components` 或 `/lib`）移動。
> - 如果您使用 Middleware，請確保它放置在 `src` 文件夾內。
> - 如果您使用 Tailwind CSS，則需要在 [內容部分](https://tailwindcss.com/docs/content-configuration) 中的 `tailwind.config.js` 檔案添加 `/src` 前綴。
> - 如果您使用如 `@/*` 的 TypeScript 匯入路徑，則應更新 `tsconfig.json` 中的 `paths` 物件以包含 `src/`。
