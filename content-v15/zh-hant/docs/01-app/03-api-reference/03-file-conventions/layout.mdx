layout 檔案是用來在您的 Next.js 應用程式中定義佈局的。

```tsx filename="app/dashboard/layout.tsx" switcher
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return <section>{children}</section>
}
```

```jsx filename="app/dashboard/layout.js" switcher
export default function DashboardLayout({ children }) {
  return <section>{children}</section>
}
```

一個**根佈局 (Root Layout)** 是根 `app` 目錄中最上層的佈局。它用於定義 `<html>` 和 `<body>` 標籤以及其他全域共享的 UI。

```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

## 參考

### 屬性 (Props)

#### `children` (必填)

佈局元件 (Components) 應該接受並使用 `children` prop。在渲染期間，`children` 將被填充為該佈局所包裹的途徑區段 (route segments)。主要將是子層 [佈局](/docs/app/api-reference/file-conventions/page) (如果存在) 或 [頁面](/docs/app/api-reference/file-conventions/page) 的元件，但也可能是其他特殊檔案，例如 [載入器](/docs/app/api-reference/file-conventions/loading) 或 [錯誤](/docs/app/getting-started/error-handling)，視適用情況而定。

#### `params` (選填)

是一個 Promise，解析後會包含一個物件，該物件包含從根區段到該佈局的所有 [動態途徑區段參數](/docs/app/api-reference/file-conventions/dynamic-routes)。

```tsx filename="app/dashboard/[team]/layout.tsx" switcher
export default async function Layout({
  children,
  params,
}: {
  children: React.ReactNode
  params: Promise<{ team: string }>
}) {
  const { team } = await params
}
```

```jsx filename="app/dashboard/[team]/layout.js" switcher
export default async function Layout({ children, params }) {
  const { team } = await params
}
```

| 範例途徑                          | URL            | `params`                           |
| --------------------------------- | -------------- | ---------------------------------- |
| `app/dashboard/[team]/layout.js`  | `/dashboard/1` | `Promise<{ team: '1' }>`           |
| `app/shop/[tag]/[item]/layout.js` | `/shop/1/2`    | `Promise<{ tag: '1', item: '2' }>` |
| `app/blog/[...slug]/layout.js`    | `/blog/1/2`    | `Promise<{ slug: ['1', '2'] }>`    |

*   由於 `params` prop 是一個 Promise。您必須使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函式來存取這些值。
    *   在第 14 版及更早版本中，`params` 是同步的 prop。為了協助向後相容，您在 Next.js 15 中仍可以同步存取它，但此行為將在未來被淘汰。

### 佈局 Props 輔助函式

您可以使用 `LayoutProps` 為佈局輸入類型，以獲得強型別的 `params` 和從目錄結構推斷命名的插槽。`LayoutProps` 是一個全域可用的輔助函式。

```tsx filename="app/dashboard/layout.tsx" switcher
export default function Layout(props: LayoutProps<'/dashboard'>) {
  return (
    <section>
      {props.children}
      {/* If you have app/dashboard/@analytics, it appears as a typed slot: */}
      {/* {props.analytics} */}
    </section>
  )
}
```

> **值得了解**:

### 根佈局

`app` 目錄**必須**包含一個**根佈局**，這是根 `app` 目錄中位於最上層的佈局。通常，根佈局是 `app/layout.js`。

```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html>
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html>
      <body>{children}</body>
    </html>
  )
}
```

*   根佈局**必須**定義 `<html>` 和 `<body>` 標籤。
    *   您**不應該**手動將 `<head>` 標籤（例如 `<title>` 和 `<meta>`）新增到根佈局中。相反地，您應該使用 [Metadata API](/docs/app/getting-started/metadata-and-og-images)，它會自動處理進階需求，例如串流 (streaming) 和去重 `<head>` 元素。
*   您可以使用 [途徑群組](/docs/app/api-reference/file-conventions/route-groups) 來建立**多個根佈局**。
    *   在**多個根佈局之間**導航將導致**完整頁面載入**（與客戶端導航相反）。例如，從使用 `app/(shop)/layout.js` 的 `/cart` 導航到使用 `app/(marketing)/layout.js` 的 `/blog` 將導致完整頁面載入。這**僅**適用於多個根佈局。
*   根佈局可以位於**動態區段**下，例如在使用 `app/[lang]/layout.js` 實作 [國際化](/docs/app/guides/internationalization) 時。

## 注意事項

### Request 物件

導覽期間，佈局在客戶端快取，以避免不必要的伺服器請求。

[佈局](/docs/app/api-reference/file-conventions/layout) 不會重新渲染。它們可以被快取並重用，以避免在頁面之間導航時進行不必要的運算。透過限制佈局存取 raw request，Next.js 可以防止執行佈局中可能緩慢或昂貴的使用者程式碼，這可能會負面影響效能。

若要存取請求物件，您可以在 [伺服器元件 (Server Components)](/docs/app/getting-started/server-and-client-components) 和 Functions 中使用 [`headers`](/docs/app/api-reference/functions/headers) 和 [`cookies`](/docs/app/api-reference/functions/cookies) APIs。

```tsx filename="app/shop/layout.tsx" switcher
import { cookies } from 'next/headers'

export default async function Layout({ children }) {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

```jsx filename="app/shop/layout.js" switcher
import { cookies } from 'next/headers'

export default async function Layout({ children }) {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

### 查詢參數

佈局在導航時不會重新渲染，因此無法存取會過期的 search params。

若要存取更新的查詢參數，您可以使用頁面 [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) prop，或在客戶端元件 (Client Component) 中使用 [`useSearchParams`](/docs/app/api-reference/functions/use-search-params) hook 來讀取它們。由於客戶端元件 (Client Components) 在導航時會重新渲染，因此它們可以存取最新的查詢參數。

```tsx filename="app/ui/search.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function Search() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  return '...'
}
```

```jsx filename="app/ui/search.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function Search() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  return '...'
}
```

```tsx filename="app/shop/layout.tsx" switcher
import Search from '@/app/ui/search'

export default function Layout({ children }) {
  return (
    <>
      <Search />
      {children}
    </>
  )
}
```

```jsx filename="app/shop/layout.js" switcher

```

### 路徑名稱

佈局在導航時不會重新渲染，因此無法存取會過期的 pathname。

若要存取目前的 pathname，您可以使用 [`usePathname`](/docs/app/api-reference/functions/use-pathname) hook 在客戶端元件中讀取它。由於客戶端元件在導航期間會重新渲染，因此它們可以存取最新的 pathname。

```tsx filename="app/ui/breadcrumbs.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'

// Simplified breadcrumbs logic
export default function Breadcrumbs() {
  const pathname = usePathname()
  const segments = pathname.split('/')

  return (
    <nav>
      {segments.map((segment, index) => (
        <span key={index}>
          {' > '}
          {segment}
        </span>
      ))}
    </nav>
  )
}
```


### 擷取資料

佈局無法將資料傳遞給其 `children`。不過，您可以在途徑中多次擷取相同的資料，並使用 React [`cache`](https://react.dev/reference/react/cache) 來去重請求而不影響效能。

或者，當使用 Next.js 的 [`fetch`](/docs/app/api-reference/functions/fetch) 時，請求會自動去重。

```tsx filename="app/lib/data.ts" switcher
export async function getUser(id: string) {
  const res = await fetch(`https://.../users/${id}`)
  return res.json()
}
```

```tsx filename="app/dashboard/layout.tsx" switcher
import { getUser } from '@/app/lib/data'
import { UserName } from '@/app/ui/user-name'

export default async function Layout({ children }) {
  const user = await getUser('1')

  return (
    <>
      <nav>
        {/* ... */}
        <UserName user={user.name} />
      </nav>
      {children}
    </>
  )
}
```

```jsx filename="app/dashboard/layout.js" switcher
import { getUser } from '@/app/lib/data'
```

### 存取子區段

佈局無法存取位於其下方的途徑區段。若要存取所有途徑區段，您可以在客戶端元件中使用 [`useSelectedLayoutSegment`](/docs/app/api-reference/functions/use-selected-layout-segment) 或 [`useSelectedLayoutSegments`](/docs/app/api-reference/functions/use-selected-layout-segments)。

```tsx filename="app/ui/nav-link.tsx" switcher
'use client'

import Link from 'next/link'
import { useSelectedLayoutSegment } from 'next/navigation'

export default function NavLink({
  slug,
  children,
}: {
  slug: string
  children: React.ReactNode
}) {
  const segment = useSelectedLayoutSegment()
  const isActive = slug === segment

  return (
    <Link
      href={`/blog/${slug}`}
      // Change style depending on whether the link is active
      style={{ fontWeight: isActive ? 'bold' : 'normal' }}
    >
      {children}
    </Link>
  )
}
```

```jsx filename="app/ui/nav-link.js" switcher
'use client'

import Link from 'next/link'
import { useSelectedLayoutSegment } from 'next/navigation'

export default function NavLinks({ slug, children }) {
  const segment = useSelectedLayoutSegment()
  const isActive = slug === segment

  return (
    <Link
      href={`/blog/${slug}`}
      style={{ fontWeight: isActive ? 'bold' : 'normal' }}
    >
      {children}
    </Link>
  )
}
```

```tsx filename="app/blog/layout.tsx" switcher
import { NavLink } from './nav-link'
import getPosts from './get-posts'

export default async function Layout({
  children,
}: {
  children: React.ReactNode
}) {
  const featuredPosts = await getPosts()
  return (
    <div>
      {featuredPosts.map((post) => (
        <div key={post.id}>
          <NavLink slug={post.slug}>{post.title}</NavLink>
        </div>
      ))}
      <div>{children}</div>
    </div>
  )
}
```

## 範例

### 元數據 (Metadata)

您可以使用 [`metadata` 物件](/docs/app/api-reference/functions/generate-metadata#the-metadata-object) 或 [`generateMetadata` 函式](/docs/app/api-reference/functions/generate-metadata#generatemetadata-function) 來修改 `<head>` HTML 元素，例如 `title` 和 `meta`。

```tsx filename="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Next.js',
}

export default function Layout({ children }: { children: React.ReactNode }) {
  return '...'
}
```

```jsx filename="app/layout.js" switcher
export const metadata = {
  title: 'Next.js',
}

export default function Layout({ children }) {
  return '...'
}
```

> **值得了解**: 您**不應該**手動將 `<head>` 標籤（例如 `<title>` 和 `<meta>`）新增到根佈局中。相反地，請使用 [Metadata APIs](/docs/app/api-reference/functions/generate-metadata)，它會自動處理進階需求，例如串流 (streaming) 和去重 `<head>` 元素。

### 已選導覽連結

您可以使用 [`usePathname`](/docs/app/api-reference/functions/use-pathname) hook 來確定導覽連結是否為已選中狀態。

由於 `usePathname` 是一個客戶端 hook (client hook)，因此您需要將導覽連結提取到客戶端元件 (Client Component) 中，該元件可以匯入到您的佈局中：

```tsx filename="app/ui/nav-links.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'
import Link from 'next/link'

export function NavLinks() {
  const pathname = usePathname()

  return (
    <nav>
      <Link className={`link ${pathname === '/' ? 'active' : ''}`} href="/">
        Home
      </Link>

      <Link
        className={`link ${pathname === '/about' ? 'active' : ''}`}
        href="/about"
      >
        About
      </Link>
    </nav>
  )
}
```

```jsx filename="app/ui/nav-links.js" switcher
'use client'

import { usePathname } from 'next/navigation'

export function Links() {
```

### 根據 `params` 顯示內容

使用 [動態途徑區段](/docs/app/api-reference/file-conventions/dynamic-routes)，您可以根據 `params` prop 顯示或擷取特定內容。

```tsx filename="app/dashboard/layout.tsx" switcher
export default async function DashboardLayout({
  children,
  params,
}: {
  children: React.ReactNode
  params: Promise<{ team: string }>
}) {
  const { team } = await params

  return (
    <section>
      <header>
        <h1>Welcome to {team}'s Dashboard</h1>
      </header>
      <main>{children}</main>
    </section>
  )
}
```

```jsx filename="app/dashboard/layout.js" switcher
export default async function DashboardLayout({ children, params }) {
  const { team } = await params

  return (
    <section>
      <header>
        <h1>Welcome to {team}'s Dashboard</h1>
      </header>
      <main>{children}</main>
    </section>
  )
}
```

### 在客戶端元件中讀取 `params`

若在客戶端元件 (Client Component) 中 (無法為 `async`) 使用 `params`，您可以使用 React 的 [`use`](https://react.dev/reference/react/use) 函式來讀取 Promise：

```tsx filename="app/page.tsx" switcher
'use client'

import { use } from 'react'

export default function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = use(params)
}
```

```js filename="app/page.js" switcher
'use client'

import { use } from 'react'

export default function Page({ params }) {
  const { slug } = use(params)
}
```
