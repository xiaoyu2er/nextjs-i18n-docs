> 本頁列出的選項在 [`cacheComponents`](/docs/app/api-reference/config/next-config-js/cacheComponents) 旗標啟用時會處於停用狀態，且最終將被棄用。

路由段選項允許您通過直接導出以下變量來配置 [頁面](/docs/app/api-reference/file-conventions/layout)、[佈局](/docs/app/api-reference/file-conventions/layout) 或 [路由處理程序](/docs/app/api-reference/file-conventions/route) 的行為：

| 選項                                  | 類型                                                                                                                      | 預設值                    |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- | -------------------------- |
| [`experimental_ppr`](#experimental_ppr) | `boolean`                                                                                                                 |                          |
| [`dynamic`](#dynamic)                   | `'auto' \| 'force-dynamic' \| 'error' \| 'force-static'`                                                                  | `'auto'`                   |
| [`dynamicParams`](#dynamicparams)       | `boolean`                                                                                                                 | `true`                     |
| [`revalidate`](#revalidate)             | `false \| 0 \| number`                                                                                                    | `false`                    |
| [`fetchCache`](#fetchcache)             | `'auto' \| 'default-cache' \| 'only-cache' \| 'force-cache' \| 'force-no-store' \| 'default-no-store' \| 'only-no-store'` | `'auto'`                   |
| [`runtime`](#runtime)                   | `'nodejs' \| 'edge'`                                                                                                      | `'nodejs'`                 |
| [`preferredRegion`](#preferredregion)   | `'auto' \| 'global' \| 'home' \| string \| string[]`                                                                      | `'auto'`                   |
| [`maxDuration`](#maxduration)           | `number`                                                                                                                  | 由部署平台設置   |

## 選項

### `experimental_ppr`

為佈局或頁面啟用 [部分預渲染 (PPR)](/docs/app/getting-started/partial-prerendering)。

```tsx filename="layout.tsx | page.tsx " switcher
export const experimental_ppr = true
// true | false
```

```jsx filename="layout.js | page.js " switcher
export const experimental_ppr = true
// true | false
```

### `dynamic`

將佈局或頁面的動態行為更改為完全靜態或完全動態。

```tsx filename="layout.tsx | page.tsx | route.ts" switcher
export const dynamic = 'auto'
// 'auto' | 'force-dynamic' | 'error' | 'force-static'
```

```js filename="layout.js | page.js | route.js" switcher
export const dynamic = 'auto'
// 'auto' | 'force-dynamic' | 'error' | 'force-static'
```

> **小貼士**: 在 `app` 目錄中的新模型優先考慮在 `fetch` 請求層級進行細粒度的緩存控制，而非 `pages` 目錄中頁面級別的 `getServerSideProps` 和 `getStaticProps` 的二進制全有或全無模型。`dynamic` 選項是一種可選回退到先前模型的快捷方式，並提供了更簡單的遷移路徑。

- **`auto`**（預設）：將盡可能多地緩存而不阻止任何組件選擇動態行為的預設選項。
- **`force-dynamic`**：強制 [動態渲染](/docs/app/getting-started/partial-prerendering#dynamic-rendering)，這將導致路由在請求時為每個用戶進行渲染。此選項等同於：

  - 將佈局或頁面中每個 `fetch()` 請求的選項設置為 `{ cache: 'no-store', next: { revalidate: 0 } }`。
  - 將段配置設置為 `export const fetchCache = 'force-no-store'`

- **`error`**：強制靜態渲染並緩存佈局或頁面的數據，如果任何組件使用 [動態 API](/docs/app/getting-started/partial-prerendering#dynamic-rendering) 或未緩存數據則會導致錯誤。此選項等同於：
  - `pages` 目錄中的 `getStaticProps()`。
  - 將佈局或頁面中每個 `fetch()` 請求的選項設置為 `{ cache: 'force-cache' }`。
  - 將段配置設置為 `fetchCache = 'only-cache'`。
- **`force-static`**：強制靜態渲染並緩存佈局或頁面的數據，通過強制 [`cookies`](/docs/app/api-reference/functions/cookies)、[`headers()`](/docs/app/api-reference/functions/headers) 和 [`useSearchParams()`](/docs/app/api-reference/functions/use-search-params) 返回空值。可以在使用 `force-static` 渲染的頁面或佈局中進行 [`revalidate`](#revalidate)、[`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 或 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag)。

> **小貼士**:

### `dynamicParams`

控制當訪問未使用 [generateStaticParams](/docs/app/api-reference/functions/generate-static-params) 生成的動態段時會發生什麼。

```tsx filename="layout.tsx | page.tsx" switcher
export const dynamicParams = true // true | false
```

```js filename="layout.js | page.js | route.js" switcher
export const dynamicParams = true // true | false
```

- **`true`**（預設）：`generateStaticParams` 中未包含的動態段將按需生成。
- **`false`**：`generateStaticParams` 中未包含的動態段將返回 404。

> **小貼士**:
>
> - 此選項替換了 `pages` 目錄中 `getStaticPaths` 的 `fallback: true | false | blocking` 選項。
> - 要以靜態方式在首次訪問時渲染所有路徑，您需要在 `generateStaticParams` 中返回一個空數組，或使用 `export const dynamic = 'force-static'`。
> - 當 `dynamicParams = true` 時，該段使用 [Streaming Server Rendering](/docs/app/getting-started/linking-and-navigating#streaming)。

### `revalidate`

為佈局或頁面設置默認重新驗證時間。此選項不會覆蓋個別 `fetch` 請求設置的 `revalidate` 值。

```tsx filename="layout.tsx | page.tsx | route.ts" switcher
export const revalidate = false
// false | 0 | number
```

```js filename="layout.js | page.js | route.js" switcher
export const revalidate = false
// false | 0 | number
```

- **`false`**（預設）：將任何將其 `cache` 選項設置為 `'force-cache'` 或在 [動態 API](/docs/app/getting-started/partial-prerendering#dynamic-rendering#dynamic-apis) 使用前發現的 `fetch` 請求進行緩存的默設規則。在語義上等效於 `revalidate: Infinity`，這意味著資源應無限緩存。仍可以為個別 `fetch` 請求使用 `cache: 'no-store'` 或 `revalidate: 0` 避免被緩存，並使路由動態渲染。或者將 `revalidate` 設置為比路由默認值低的正數以增加路由的重新驗證頻率。
- **`0`**：確保即使在未發現任何動態 API 或未緩存數據請求時，佈局或頁面始終 [動態渲染](/docs/app/getting-started/partial-prerendering#dynamic-rendering)。此選項將不設置 `cache` 選項的 `fetch` 請求的默設值更改為 `'no-store'`，但保留選擇 `'force-cache'` 或使用正數 `revalidate` 的 `fetch` 請求。
- **`number`**：（以秒為單位）將佈局或頁面的默認重新驗證頻率設置為 `n` 秒。

> **小貼士**:
>
> - 重新驗證值需要是靜態可分析的。例如 `revalidate = 600` 是有效的，但 `revalidate = 60 * 10` 不是。
> - 在 `runtime = 'edge'` 時，重新驗證值不可用。
> - 在開發模式下，頁面始終按需渲染且永不被緩存。這允許您立即看到更改，而無需等待重新驗證期。

#### 重新驗證頻率

- 單一路由中每個佈局和頁面的最低 `revalidate` 值將決定整個 _路由_ 的重新驗證頻率。這確保子頁面與父佈局的重新驗證頻率相同。
- 個別 `fetch` 請求可以設置比路由默認 `revalidate` 更低的 `revalidate` 以增加整個路由的重新驗證頻率。這允許您根據某些條件動態選擇加入更頻繁地重新驗證某些路由。

#### `fetchCache`

<details>

  <summary>這是一個進階選項，僅在需要覆蓋默認行為時使用。</summary>

默認情況下，Next.js **會緩存**在 [動態 API](/docs/app/getting-started/partial-prerendering#dynamic-rendering#dynamic-apis) 使用之前可達的任何 `fetch()` 請求，且 **不會緩存** 動態 API 使用後發出的 `fetch` 請求。

`fetchCache` 允許您覆蓋佈局或頁面中所有 `fetch` 請求的默認 `cache` 選項。

```tsx filename="layout.tsx | page.tsx | route.ts" switcher
export const fetchCache = 'auto'
// 'auto' | 'default-cache' | 'only-cache'
// 'force-cache' | 'force-no-store' | 'default-no-store' | 'only-no-store'
```

```js filename="layout.js | page.js | route.js" switcher
export const fetchCache = 'auto'
// 'auto' | 'default-cache' | 'only-cache'
// 'force-cache' | 'force-no-store' | 'default-no-store' | 'only-no-store'
```

- **`auto`**（預設）：動態 API 之前可達 `fetch` 請求的默認選項，使用它們提供的 `cache` 選項進行緩存，且不在動態 API 之後緩存 `fetch` 請求。
- **`default-cache`**：允許將任何 `cache` 選項傳遞給 `fetch`，但如果不提供選項，則將 `cache` 選項設置為 `'force-cache'`。這意味著即動態 API 之後的 `fetch` 請求也被視為靜態。
- **`only-cache`**：確保所有 `fetch` 請求選擇加入緩存，將默設值更改為 `cache: 'force-cache'`，如果不提供選項並導致任何 `fetch` 請求使用 `cache: 'no-store'` 時出現錯誤。
- **`force-cache`**：確保所有 `fetch` 請求選擇加入緩存，將所有 `fetch` 請求的 `cache` 選項設置為 `'force-cache'`。
- **`default-no-store`**：允許將任何 `cache` 選項傳遞給 `fetch`，但如果不提供選項，則將 `cache` 選項設置為 `'no-store'`。這意味著即動態 API 之前的 `fetch` 請求也被視為動態。
- **`only-no-store`**：確保所有 `fetch` 請求選擇退出緩存，將默設值更改為 `cache: 'no-store'`，如果不提供選項並導致任何 `fetch` 請求使用 `cache: 'force-cache'` 時出現錯誤。
- **`force-no-store`**：確保所有 `fetch` 請求選擇退出緩存，將所有 `fetch` 請求的 `cache` 選項設置為 `'no-store'`。這強制所有 `fetch` 請求在每次請求時都重新獲取，即使它們提供 `'force-cache'` 選項。

#### 跨路由段行為

- 單一路由中每個佈局和頁面的每個設置選項需要相互兼容。
  - 如果同時提供了 `'only-cache'` 和 `'force-cache'`，則 `'force-cache'` 生效。如果同時提供了 `'only-no-store'` 和 `'force-no-store'`，則 `'force-no-store'` 生效。強制選項更改了跨路由的行為，因此具有 `'force-*'` 的單一段落會防止由 `'only-*'` 引起的錯誤。
  - `'only-*'` 和 `'force-*'` 選項的意圖是保證整個路由完全靜態或完全動態。這意味著：
    - 單一路由中同時結合 `'only-cache'` 和 `'only-no-store'` 是不允許的。
    - 單一路由中同時結合 `'force-cache'` 和 `'force-no-store'` 是不允許的。
  - 如果子節點提供了 `'auto'` 或 `'*-cache'`，父節點不能提供 `'default-no-store'`，因為這可能導致相同的 `fetch` 具有不同的行為。
- 通常建議將共享的父佈局保留為 `'auto'`，並在子段分叉處自訂選項。

</details>

### `runtime`

我們建議使用 Node.js 運行時來渲染您的應用程序，並使用 Edge 運行時來設置中間件。

```tsx filename="layout.tsx | page.tsx | route.ts" switcher
export const runtime = 'nodejs'
// 'nodejs' | 'edge'
```

```js filename="layout.js | page.js | route.js" switcher
export const runtime = 'nodejs'
// 'nodejs' | 'edge'
```

- **`nodejs`**（預設）
- **`edge`**

### `preferredRegion`

```tsx filename="layout.tsx | page.tsx | route.ts" switcher
export const preferredRegion = 'auto'
// 'auto' | 'global' | 'home' | ['iad1', 'sfo1']
```

```js filename="layout.js | page.js | route.js" switcher
export const preferredRegion = 'auto'
// 'auto' | 'global' | 'home' | ['iad1', 'sfo1']
```

對 `preferredRegion` 的支持以及支持的區域取決於您的部署平台。

> **小貼士**:
>
> - 如果不指定 `preferredRegion`，它將繼承最近的父佈局的選項。
> - 根佈局的默認為所有 (`all`) 區域。

### `maxDuration`

默認情況下，Next.js 不限制服務器端邏輯的執行（渲染頁面的或處理 API）。部署平臺可使用 Next.js 構建輸出中的 `maxDuration` 來添加特定的執行限制。

**注意**: 此設置需要 Next.js `13.4.10` 或更高版本。

```tsx filename="layout.tsx | page.tsx | route.ts" switcher
export const maxDuration = 5
```

```js filename="layout.js | page.js | route.js" switcher
export const maxDuration = 5
```

> **小貼士**:
>
> - 如果使用 [Server Actions](/docs/app/getting-started/updating-data)，在頁面級別設置 `maxDuration` 以更改頁面上使用的所有 Server Actions 的默認超時時間。

### `generateStaticParams`

`generateStaticParams` 函數可與 [動態路由段](/docs/app/api-reference/file-conventions/dynamic-routes) 配合使用，以定義在構建時靜態生成而不是在請求時按需生成的路由段參數列表。

查看 [API 參考](/docs/app/api-reference/functions/generate-static-params) 以獲取更多詳細信息。
