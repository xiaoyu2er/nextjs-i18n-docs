當您無法提前得知確切的路由區段名稱，並希望從動態資料建立路由時，您可以使用在請求時填充或建置時預先渲染的動態區段。

## 慣例

動態區段可以透過將資料夾名稱用方括號括起來建立：`[folderName]`。例如，部落格可能包含以下路由 `app/blog/[slug]/page.js`，其中 `[slug]` 是部落格文章的動態區段。

```tsx filename="app/blog/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  return <div>My Post: {slug}</div>
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
  return <div>My Post: {slug}</div>
}
```

動態區段作為 `params` prop 傳遞給 [`layout`](/docs/app/api-reference/file-conventions/layout)、[`page`](/docs/app/api-reference/file-conventions/page)、[`route`](/docs/app/api-reference/file-conventions/route) 和 [`generateMetadata`](/docs/app/api-reference/functions/generate-metadata#generatemetadata-function) 函式。

| Route                     | Example URL | params        |
| ------------------------- | ----------- | ------------- |
| `app/blog/[slug]/page.js` | `/blog/a`   | `{ slug: 'a' }` |
| `app/blog/[slug]/page.js` | `/blog/b`   | `{ slug: 'b' }` |
| `app/blog/[slug]/page.js` | `/blog/c`   | `{ slug: 'c' }` |

### In Client Components

在客戶端元件的 **page** 中，可以透過 [`use`](https://react.dev/reference/react/use) 存取來自 props 的動態區段。

```tsx filename="app/blog/[slug]/page.tsx" switcher
'use client'
import { use } from 'react'

export default function BlogPostPage({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = use(params)

  return (
    <div>
      <p>{slug}</p>
    </div>
  )
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
'use client'
import { use } from 'react'
import { useParams } from 'next/navigation'

export default function BlogPostPage({ params }) {
  const { slug } = use(params)

  return (
    <div>
      <p>{slug}</p>
    </div>
  )
}
```

或者，客戶端元件也可使用 [`useParams`](/docs/app/api-reference/functions/use-params) hook 來在整個客戶端元件樹狀結構中存取 `params`。

### Catch-all Segments

可以透過在方括號內加入省略號 `[...folderName]` 將動態區段擴展至 **全捕捉** 後續區段。

例如，`app/shop/[...slug]/page.js` 將匹配 `/shop/clothes`，但也匹配 `/shop/clothes/tops`、`/shop/clothes/tops/t-shirts` 等等。

| Route                        | Example URL   | params                    |
| ---------------------------- | ------------- | ------------------------- |
| `app/shop/[...slug]/page.js` | `/shop/a`     | `{ slug: ['a'] }`         |
| `app/shop/[...slug]/page.js` | `/shop/a/b`   | `{ slug: ['a', 'b'] }`    |
| `app/shop/[...slug]/page.js` | `/shop/a/b/c` | `{ slug: ['a', 'b', 'c'] }` |

### Optional Catch-all Segments

可透過將參數包含在雙重方括號中，將全捕捉區段設定為 **可選**：`[[...folderName]]`。

例如，`app/shop/[[...slug]]/page.js` 除了匹配 `/shop/clothes`、`/shop/clothes/tops`、`/shop/clothes/tops/t-shirts` 之外，也將匹配 `/shop`。

**全捕捉**與**可選全捕捉**區段的區別在於，可選版本也會匹配不帶參數的路由（如上例中的 `/shop`）。

| Route                          | Example URL   | params                    |
| ------------------------------ | ------------- | ------------------------- |
| `app/shop/[[...slug]]/page.js` | `/shop`       | `{ slug: undefined }`     |
| `app/shop/[[...slug]]/page.js` | `/shop/a`     | `{ slug: ['a'] }`         |
| `app/shop/[[...slug]]/page.js` | `/shop/a/b`   | `{ slug: ['a', 'b'] }`    |
| `app/shop/[[...slug]]/page.js` | `/shop/a/b/c` | `{ slug: ['a', 'b', 'c'] }` |

### TypeScript

使用 TypeScript 時，您可以根據配置的路由區段為 `params` 新增類型 — 使用 [`PageProps<'/route'>`](/docs/app/api-reference/file-conventions/page#page-props-helper)、[`LayoutProps<'/route'>`](/docs/app/api-reference/file-conventions/layout#layout-props-helper) 或 [`RouteContext<'/route'>`](/docs/app/api-reference/file-conventions/route#route-context-helper) 分別為 `page`、`layout` 和 `route` 中的 `params` 輸入類型。

路由 `params` 值定義為 `string`、`string[]` 或 `undefined`（用於可選全捕捉區段），因為這些值直到執行時才知道。使用者可以在網址列輸入任何 URL，這些寬鬆的類型有助於確保您的應用程式程式碼能處理所有這些可能的案例。

| Route                               | params Type Definition |
| ----------------------------------- | ---------------------- |
| `app/blog/[slug]/page.js`           | `{ slug: string }`     |
| `app/shop/[...slug]/page.js`        | `{ slug: string[] }`   |
| `app/shop/[[...slug]]/page.js`      | `{ slug?: string[] }`  |
| `app/[categoryId]/[itemId]/page.js` | `{ categoryId: string, itemId: string }` |

如果您正在處理一個 `params` 只能具有有限有效值的路由，例如具有已知語言代碼集之 `[locale]` 參數，您可以使用執行時驗證來處理使用者可能輸入的任何無效參數，並讓應用程式其餘部分使用您已知集合中的較窄類型。

```tsx filename="/app/[locale]/page.tsx"
import { notFound } from 'next/navigation'
import type { Locale } from '@i18n/types'
import { isValidLocale } from '@i18n/utils'

function assertValidLocale(value: string): asserts value is Locale {
  if (!isValidLocale(value)) notFound()
}

export default async function Page(props: PageProps<'/[locale]'>) {
  const { locale } = await props.params // locale is typed as string
  assertValidLocale(locale)
  // locale is now typed as Locale
}
```

## 行為

- 由於 `params` prop 是 Promise。您必須使用 `async`/`await` 或 React 的 use function 來存取值。
  - 在版本 14 及更早版本中，`params` 是同步 prop。為了協助向後相容性，您可以在 Next.js 15 中繼續同步存取它，但此行為未來將被棄用。

## 範例

### With `generateStaticParams`

[`generateStaticParams`](/docs/app/api-reference/functions/generate-static-params) function 可用於在建置時 [靜態生成](/docs/app/getting-started/partial-prerendering#static-rendering) 路由，而不是在請求時按需生成。

```tsx filename="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

在 `generateStaticParams` function 內部使用 `fetch` 時，請求會 [自動去重複](/docs/app/guides/caching#request-memoization)。這可避免對相同的資料 Layouts、Pages 和其他 `generateStaticParams` function 進行多次網路呼叫，加快建置時間。
