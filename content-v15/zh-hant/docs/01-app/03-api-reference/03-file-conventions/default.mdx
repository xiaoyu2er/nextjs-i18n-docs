`default.js` 檔案用於在 [平行路由 (Parallel Routes)](/docs/app/api-reference/file-conventions/parallel-routes) 中渲染備援，當 Next.js 在完整頁面載入後無法恢復 [插槽 (slot)](/docs/app/api-reference/file-conventions/parallel-routes#slots) 的當前活動狀態時。

在 [軟式導航 (soft navigation)](/docs/app/getting-started/linking-and-navigating#client-side-transitions) 期間，Next.js 會追蹤每個插槽的當前 _狀態_ (子頁面)。然而，對於硬式導航 (完整頁面載入)，Next.js 無法恢復當前活動狀態。在這種情況下，不匹配目前 URL 的子頁面可以渲染 `default.js` 檔案。

請考慮以下資料夾結構。`@team` 插槽有 `settings` 頁面，但 `@analytics` 則沒有。

當導航至 `/settings` 時，`@team` 插槽將渲染 `settings` 頁面，同時保持 `@analytics` 插槽目前的活動頁面。

重新整理時，Next.js 將為 `@analytics` 渲染 `default.js`。如果 `default.js` 不存在，則會渲染 `404` 作為替代。

此外，由於 `children` 是隱式插槽，您還需要建立 `default.js` 檔案，以在 Next.js 無法還原父層頁面的活動狀態時，為 `children` 渲染備援。

## 參考

### `params` (選填)

一個 Promise，解析後包含從根段 (root segment) 到插槽子頁面的 [動態路由參數 (dynamic route parameters)](/docs/app/api-reference/file-conventions/dynamic-routes) 的物件。例如：

```tsx filename="app/[artist]/@sidebar/default.js" switcher
export default async function Default({
  params,
}: {
  params: Promise<{ artist: string }>
}) {
  const { artist } = await params
}
```

```jsx filename="app/[artist]/@sidebar/default.js" switcher
export default async function Default({ params }) {
  const { artist } = await params
}
```

| 範例 | 網址 | `params`                                     |
| ------------------------------------------ | ------------ | -------------------------------------------- |
| `app/[artist]/@sidebar/default.js`         | `/zack`      | `Promise<{ artist: 'zack' }>`                |
| `app/[artist]/[album]/@sidebar/default.js` | `/zack/next` | `Promise<{ artist: 'zack', album: 'next' }>` |

*   由於 `params` prop 是一個 Promise。您必須使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函式來存取這些值。
    *   在第 14 版及更早版本中，`params` 是同步的 prop。為了協助向後相容，您在 Next.js 15 中仍可以同步存取它，但此行為將在未來被淘汰。
