特殊檔案 `loading.js` 可協助您使用 [React Suspense](https://react.dev/reference/react/Suspense) 建立有意義的載入使用者介面。透過此命名規範，您可以在路線區段內容串流傳輸時，從伺服器顯示 [即時載入狀態](#instant-loading-states)。一旦完成，新內容會自動替換。

```tsx filename="app/feed/loading.tsx" switcher
export default function Loading() {
  // Or a custom loading skeleton component
  return <p>Loading...</p>
}
```

```jsx filename="app/feed/loading.js" switcher
export default function Loading() {
  // Or a custom loading skeleton component
  return <p>Loading...</p>
}
```

在 `loading.js` 檔案中，您可以新增任何輕量級的載入使用者介面。您可能會發現使用 [React Developer Tools](https://react.dev/learn/react-developer-tools) 來手動切換 Suspense 邊界很有幫助。

預設情況下，此檔案為一個 [伺服器組件](/docs/app/getting-started/server-and-client-components)，但也能夠透過 `"use client"` 指令作為客戶端組件使用。

## 參考

### 參數

載入使用者介面組件不支援任何參數。

## 行為

### 導航

- 後備使用者介面已 [預先載入](/docs/app/getting-started/linking-and-navigating#prefetching)，這使得導航立即進行，除非預先載入尚未完成。
- 導航可被中斷，這意味著切換路由時不需要等待當前路由的內容完全載入即可導航至其他路由。
- 共用佈局在新增路由區段載入時仍保持互動性。

### 即時載入狀態

即時載入狀態是指在開始導航時立即顯示的後備使用者介面。您可以預先渲染載入指示器，例如骨架或旋轉圖示，或是未來螢幕的一部分（例如封面圖片、標題等）但有意義的部分。這能幫助用戶瞭解應用程式正在回應，並提供更好的使用者體驗。

在資料夾內新增 `loading.js` 檔案即可建立載入狀態。

### 搜尋引擎優化 (SEO)

- Next.js 將等待 [`generateMetadata`](/docs/app/api-reference/functions/generate-metadata) 內的數據獲取完成後，再將使用者介面串流傳輸至用戶端。這保證了串流回應的第一部分包含 `<head>` 標籤。
- 由於串流是伺服器渲染的，因此不會影響 SEO。您可以使用 Google 的 [Rich Results Test](https://search.google.com/test/rich-results) 工具，查看頁面在 Google 網絡爬蟲眼中的呈現方式，並查看序列化 HTML（[來源](https://web.dev/rendering-on-the-web/#seo-considerations)）。

### 狀態代碼

當進行串流時，將傳回 `200` 狀態代碼，表示請求已成功。

伺服器仍可在串流內容本身中向用戶端傳遞錯誤或問題，例如，當使用 [`redirect`](/docs/app/api-reference/functions/redirect) 或 [`notFound`](/docs/app/api-reference/functions/not-found) 時。由於回應標頭已發送給用戶端，因此無法更新回應的狀態代碼。這不會影響 SEO。

### 瀏覽器限制

[某些瀏覽器](https://bugs.webkit.org/show_bug.cgi?id=252413) 會緩存串流回應。您可能會在看到回應超過 1024 位元組之前都看不到串流回應。這通常只會影響 "Hello World" 範例應用程式，但不會影響應用程式。

## 範例

### 使用 Suspense 進行串流

除了 `loading.js`，您也可以為自己的使用者介面組件手動建立 Suspense 邊界。App Router 支援使用 [Suspense](https://react.dev/reference/react/Suspense) 進行串流。

`<Suspense>` 的運作方式是包裝執行非同步動作的組件（例如：獲取數據），在其進行顯示後備使用者介面（例如：骨架、旋轉圖示），然後在動作完成後替換為您自己的組件。

```tsx filename="app/dashboard/page.tsx" switcher
import { Suspense } from 'react'
import { PostFeed, Weather } from './Components'

export default function Posts() {
  return (
    <section>
      <Suspense fallback={<p>Loading feed...</p>}>
        <PostFeed />
      </Suspense>
      <Suspense fallback={<p>Loading weather...</p>}>
        <Weather />
      </Suspense>
    </section>
  )
}
```

```jsx filename="app/dashboard/page.js" switcher
import { Suspense } from 'react'
import { PostFeed, Weather } from './Components'

export default function Posts() {
  return (
    <section>
      <Suspense fallback={<p>Loading feed...</p>}>
        <PostFeed />
      </Suspense>
      <Suspense fallback={<p>Loading weather...</p>}>
        <Weather />
      </Suspense>
    </section>
  )
}
```

透過使用 Suspense，您可以獲得以下優勢：

1. **串流式伺服器渲染** - 從伺服器逐步渲染 HTML 至用戶端。
2. **選擇性水合 (Selective Hydration)** - React 根據使用者互動優先決定先讓哪些組件具互動性。

若要查看更多 Suspense 範例和用途，請參閱 [React 文件](https://react.dev/reference/react/Suspense)。
