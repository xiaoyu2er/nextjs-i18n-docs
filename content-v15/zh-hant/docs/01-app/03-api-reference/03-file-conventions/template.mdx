**模板** 檔案類似於 [佈局 (layout)](/docs/app/getting-started/layouts-and-pages#creating-a-layout)，它會包圍佈局或頁面。與跨路由持久存在並保持狀態的佈局不同，模板會被賦予唯一的 key，這意味著子 Client Components 在導航時會重置它們的狀態。

當您需要以下情況時，它們非常有用：

- 在導航時重新同步 `useEffect`。
- 在導航時重置子 Client Components 的狀態。例如，一個輸入欄位。
- 更改預設框架行為。例如，佈局內的 Suspense boundaries 僅在首次載入時顯示 fallback，而模板則在每次導航時顯示。

## 慣例

模板可以通過從 `template.js` 檔案匯出預設 React 元件來定義。該元件應該接受 `children` 屬性。

```tsx filename="app/template.tsx" switcher
export default function Template({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>
}
```

```jsx filename="app/template.js" switcher
export default function Template({ children }) {
  return <div>{children}</div>
}
```

就嵌套而言，`template.js` 是在佈局和其子元件之間呈現。這是簡化的輸出：

```jsx filename="Output"
<Layout>
  {/* Note that the template is given a unique key. */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```

## Props (屬性)

### children (必填)

模板接受 `children` 屬性。

```jsx filename="Output"
<Layout>
  {/* Note that the template is automatically given a unique key. */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```

## 行為

- **伺服器元件 (Server Components)**：預設情況下，模板是伺服器元件。
- **導航時重新掛載**：模板會自動接收唯一的 key。導航到新路由會導致模板及其子元件重新掛載。
- **狀態重置**：模板內的任何 Client Components 都會在導航時重置狀態。
- **Effect 重新執行**：如 `useEffect` 的 Effect 會在元件重新掛載時重新同步。
- **DOM 重置**：模板內的 DOM 元素會被完全重新建立。
