`revalidateTag` 允許您針對特定的快取標籤，按需 [清除已快取的資料](/docs/app/guides/caching)。

## 使用方法

`revalidateTag` 可以在伺服器函式 (Server Functions) 和路由處理程式 (Route Handlers) 中呼叫。

`revalidateTag` 無法在用戶端元件 (Client Components) 或中間件 (Middleware) 中呼叫，因為它僅在伺服器環境中運作。

> **值得注意**：`revalidateTag` 會將標記的資料標記為過期，但新數據僅在下次訪問使用該標籤的頁面時才會被獲取。這意味著呼叫 `revalidateTag` 不會立即觸發大量重新驗證。無效化僅在下次訪問使用該標籤的任何頁面時發生。

## 參數

```tsx
revalidateTag(tag: string): void;
```

- `tag`: 代表與您想要重新驗證的資料關聯的快取標籤的字串。不得超過 256 個字元。此值為大小寫敏感。

您可以按照以下方式為 `fetch` 添加標籤：

```tsx
fetch(url, { next: { tags: [...] } });
```

## 返回值

`revalidateTag` 不會傳回任何值。

## 與 revalidatePath 的關係

`revalidateTag` 會使使用這些標籤的所有頁面上的帶有特定標籤的資料無效，而 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 則會使特定的頁面或佈局路徑無效。

> **值得注意**：這些函式具有不同的用途，為了確保全面的資料一致性，可能需要搭配使用。有關詳細範例和考量，請參閱 [與 revalidateTag 的關係](/docs/app/api-reference/functions/revalidatePath#relationship-with-revalidatetag)。

## 範例

### 伺服器動作

```ts filename="app/actions.ts" switcher
'use server'

import { revalidateTag } from 'next/cache'

export default async function submit() {
  await addPost()
  revalidateTag('posts')
}
```

```js filename="app/actions.js" switcher
'use server'

import { revalidateTag } from 'next/cache'

export default async function submit() {

}
```

### 路由處理程式

```ts filename="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { revalidateTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')

  if (tag) {
    revalidateTag(tag)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing tag to revalidate',
  })
}
```
