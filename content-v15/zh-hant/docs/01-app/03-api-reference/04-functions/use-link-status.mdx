`useLinkStatus` 掛鉤讓您追蹤 `<Link>` 的 **pending** 狀態。當導航到新路由完成時，您可以用它向用戶顯示內聯視覺反饋（例如旋轉圖標或文字閃爍）。

`useLinkStatus` 在有以下情況時非常有用：

- 停用或進行中的 [Prefetching (預先加載)](/docs/app/getting-started/linking-and-navigating#prefetching)，這表示導航已鎖定。
- 目標路由是動態的 **且** 不包含 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 檔案，原本允許即時導航。

```tsx filename="app/loading-indicator.tsx" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return pending ? (
    <div role="status" aria-label="Loading" className="spinner" />
  ) : null
}
```

```jsx filename="app/loading-indicator.js" switcher

import { useLinkStatus } from 'next/link'

```

```tsx filename="app/header.tsx" switcher
import Link from 'next/link'
import LoadingIndicator from './loading-indicator'

export default function Header() {
  return (
    <header>
      <Link href="/dashboard" prefetch={false}>
        Dashboard <LoadingIndicator />
      </Link>
    </header>
  )
}
```

> **值得了解**：
>
> - `<Link>` 組件的子組件中必須使用 `useLinkStatus`。
> - 當 `Link` 組件上設定了 `prefetch={false}` 時，此掛鉤最有幫助。
> - 如果已預先加載連結的路由，則會跳過 pending 狀態。
> - 當連續快速點擊多個連結時，僅顯示最後一個連結的 pending 狀態。
> - 此掛鉤不支援 Pages Router，並永遠會傳回 `{ pending: false }`。

## 參數

```tsx
const { pending } = useLinkStatus()
```

`useLinkStatus` 不需要任何參數。

## 返回值

`useLinkStatus` 傳回一個包含單一屬性的物件：

| 屬性 | 類型 | 描述 |
| -------- | ------- | -------------------------------------------- |
| pending | 布林值 | history 更新前為 `true`，更新後為 `false` |

## 範例

### 內聯載入指標

在預先加載完成之前用戶點擊連結時，添加指示導航正在進行的視覺反饋是有益的。

```tsx filename="app/components/loading-indicator.tsx" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return pending ? (
    <div role="status" aria-label="Loading" className="spinner" />
  ) : null
}
```

```jsx filename="app/components/loading-indicator.js" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return pending ? (
    <div role="status" aria-label="Loading" className="spinner" />
  ) : null
}
```

```tsx filename="app/shop/layout.tsx" switcher
import Link from 'next/link'
import LoadingIndicator from './components/loading-indicator'

const links = [
  { href: '/shop/electronics', label: 'Electronics' },
  { href: '/shop/clothing', label: 'Clothing' },
  { href: '/shop/books', label: 'Books' },
]

function Menubar() {
  return (
    <div>
      {links.map((link) => (
        <Link key={link.label} href={link.href}>
          {link.label} <LoadingIndicator />
        </Link>
      ))}
    </div>
  )
}

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <Menubar />
      {children}
    </div>
  )
}
```

## 優雅地處理快速導航

如果導航到新路由的速度很快，用戶可能會看到不必要的載入指標閃爍。改善使用者體驗的一種方法是只在導航耗時完成時才顯示載入指標，即添加初始動畫延遲（例如 100ms）並將動畫设为不可見（例如 `opacity: 0`）。

```css filename="app/styles/global.css"
.spinner {
  /* ... */
  opacity: 0;
  animation:
    fadeIn 500ms 100ms forwards,
    rotate 1s linear infinite;
}

@keyframes fadeIn {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

@keyframes rotate {
  to {
    transform: rotate(360deg);
  }
}
```
