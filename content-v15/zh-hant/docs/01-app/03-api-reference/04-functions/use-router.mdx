The `useRouter` hook allows you to programmatically change routes inside [Client Components](/docs/app/getting-started/server-and-client-components).

> **Recommendation:** Use the [`<Link>` component](/docs/app/api-reference/components/link) for navigation unless you have a specific requirement for using `useRouter`.

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button type="button" onClick={() => router.push('/dashboard')}>
      Dashboard
    </button>
  )
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button type="button" onClick={() => router.push('/dashboard')}>
      Dashboard
    </button>
  )
}
```

## `useRouter()`

- `router.push(href: string, { scroll: boolean }): 執行對指定路由的客戶端導航。將新條目添加到 [瀏覽器歷史記錄 (Browser's History)](https://developer.mozilla.org/docs/Web/API/History_API) 堆疊中。
- `router.replace(href: string, { scroll: boolean }): 執行對指定路由的客戶端導航，不向 [瀏覽器歷史記錄堆疊 (Browser's History Stack)](https://developer.mozilla.org/docs/Web/API/History_API) 添加新條目。
- `router.refresh()`: 刷新當前路由。向服務器發出新請求，重新獲取數據請求，並重新呈現服務端組件。客戶端將合併更新的 React 服務端組件負載，而不會丟失未受影響的客戶端 React（例如 `useState`）或瀏覽器狀態（例如滾動位置）。
- `router.prefetch(href: string, options?: { onInvalidate?: () => void })`: [預先獲取 (Prefetch)](/docs/app/getting-started/linking-and-navigating#prefetching) 指定路由以實現更快的客戶端轉換。可選的 `onInvalidate` 回調函數會在 [預先獲取的數據過時](/docs/app/guides/prefetching#extending-or-ejecting-link) 時調用。
- `router.back()`: 在瀏覽器歷史記錄堆疊中返回上一個路由。
- `router.forward()`: 在瀏覽器歷史記錄堆疊中向前導航到下一個頁面。

> **Good to know**:
>
> - 你不得向 `router.push` 或 `router.replace` 發送不受信任或未清理的 URL，因為這可能會使你的網站面臨跨站腳本 (XSS) 漏洞。例如，發送到 `router.push` 或 `router.replace` 的 `javascript:` URL 會在你的頁面上下文中執行。
> - `<Link>` 組件會在路由進入視口時自動預先獲取路由。
> - 如果獲取請求已緩存，`refresh()` 可能會產生相同的結果。其他動態 APIs 如 `cookies` 和 `headers` 也可能會更改響應。
> - `onInvalidate` 回調函數在每次預先獲取請求中最多調用一次。它指示何時你可能想為更新的路由數據觸發新的預先獲取。

### 從 `next/router` 遷移

- 當使用 App Router 時，`useRouter` hook 應從 `next/navigation` 導入，而不是 `next/router`
- `pathname` 字符串已刪除，並由 [`usePathname()`](/docs/app/api-reference/functions/use-pathname) 替換
- `query` 對象已刪除，並由 [`useSearchParams()`](/docs/app/api-reference/functions/use-search-params) 替換
- `router.events` 已更換。[查看下文](#router-events)

[View the full migration guide](/docs/app/guides/migrating/app-router-migration).

## 範例

### 路由事件

你可以通過組合其他客戶端組件 hook（例如 `usePathname` 和 `useSearchParams`）來監聽頁面更改。

```jsx filename="app/components/navigation-events.js"
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

export function NavigationEvents() {
  const pathname = usePathname()
  const searchParams = useSearchParams()

  useEffect(() => {
    const url = `${pathname}?${searchParams}`
    console.log(url)
    // You can now use the current URL
    // ...
  }, [pathname, searchParams])

  return '...'
}
```

Which can be imported into a layout.

```jsx filename="app/layout.js" highlight={2,10-12}
import { Suspense } from 'react'
import { NavigationEvents } from './components/navigation-events'

export default function Layout({ children }) {
  return (
    <html lang="en">
      <body>
        {children}

        <Suspense fallback={null}>
          <NavigationEvents />
        </Suspense>
      </body>
    </html>
  )
}
```

> **Good to know`: `<NavigationEvents>` is wrapped in a [`Suspense` boundary](/docs/app/api-reference/file-conventions/loading#examples) because[\`useSearchParams()\`](/docs/app/api-reference/functions/use-search-params) causes client-side rendering up to the closest `Suspense` boundary during [static rendering](/docs/app/getting-started/partial-prerendering#static-rendering). [Learn more](/docs/app/api-reference/functions/use-search-params#behavior).
>
> **小貼士：** `<NavigationEvents>` 被包裹在 [`Suspense` 邊界 (Suspense Boundary)](/docs/app/api-reference/file-conventions/loading#examples) 中，因為 [`useSearchParams()`](/docs/app/api-reference/functions/use-search-params) 會在 [靜態渲染 (Static Rendering)](/docs/app/getting-started/partial-prerendering#static-rendering) 期間導致客戶端渲染直到最近的 `Suspense` 邊界。[了解更多](/docs/app/api-reference/functions/use-search-params#behavior).

### 禁用滾動至頂部

預設情況下，Next.js 會在導航到新路由時滾動至頁面頂部。你可以透過向 `router.push()` 或 `router.replace()` 傳遞 `scroll: false` 來禁用此行為。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button
      type="button"
      onClick={() => router.push('/dashboard', { scroll: false })}
    >
      Dashboard
    </button>
  )
}
```

```jsx filename="app/example-client-component.jsx" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button
      type="button"
      onClick={() => router.push('/dashboard', { scroll: false })}
    >
      Dashboard
    </button>
  )
}
```
