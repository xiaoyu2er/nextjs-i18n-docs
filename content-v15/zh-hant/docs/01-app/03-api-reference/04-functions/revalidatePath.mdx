`revalidatePath` 允許您按需使特定路徑的 [緩存資料](/docs/app/guides/caching) 無效。

## 使用方法

`revalidatePath` 可在伺服器函數和路由處理程式 (Route Handlers) 中呼叫。

`revalidatePath` 無法在客戶端組件或中間件 (Middleware) 中呼叫，因為它僅適用於伺服器環境。

> **小知識**:
>
> - **伺服器函數 (Server Functions)**：立即更新 UI（如果正在檢視受影響的路徑）。目前，它還會在再次導航時導致所有先前造訪過的頁面重新整理。這種行為是暫時的，未來將更新為僅套用到特定路徑。
> - **路由處理程式 (Route Handlers)**：標記路徑以供重新驗證。重新驗證會在下次造訪指定路徑時完成。這意味著使用動態路徑段呼叫 `revalidatePath` 不會立即觸發大量重新驗證。無效化只會發生在下次造訪路徑時。

## 參數

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`: 要重新驗證的資料對應的路徑模式，例如 `/product/[slug]`，或特定 URL，`/product/123`。不要附加 `/page` 或 `/layout`，改為使用 `type` 參數。不得超過 1024 個字元。此值區分大小寫。
- `type`: (選用) `'page'` 或 `'layout'` 字串，用於變更要重新驗證的路徑類型。如果 `path` 包含動態段，例如 `/product/[slug]`, 此參數為必要條件。如果 `path` 是特定 URL，`/product/1`, 省略 `type`。

當您想要重新整理 [單一頁面](#revalidating-a-specific-url) 時，請使用特定 URL。使用路徑模式加上 `type` 來重新整理 [多個 URL](#revalidating-a-page-path)。

## 返回值

`revalidatePath` 不會傳回任何值。

## 可無效化的項目

路徑參數可以指向頁面、佈局或路由處理程式 (Route Handlers)：

- **頁面 (Pages)**：使特定頁面無效
- **佈局 (Layouts)**：使佈局（該段落的 `layout.tsx`）無效，其下方所有的嵌套佈局以及它們下方的所有頁面也隨之無效
- **路由處理程式 (Route Handlers)**：使路由處理程式中存取的資料快取 (Data Cache) 條目無效。例如 `revalidatePath("/api/data")` 會使此 GET 處理程式無效：

```ts filename="app/api/data/route.ts"
export async function GET() {
  const data = await fetch('https://api.vercel.app/blog', {
    cache: 'force-cache',
  })

  return Response.json(await data.json())
}
```

## 與 `revalidateTag` 的關係

`revalidatePath` 和 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag) 有不同的用途：

- **`revalidatePath`**：使特定的頁面或佈局路徑無效
- **`revalidateTag`**：使所有使用這些標籤的頁面中具有特定標籤的資料無效

當您呼叫 `revalidatePath` 時，只有指定的路徑在下一次造訪時會獲取新資料。使用相同資料標籤的其他頁面將繼續服務緩存資料，直到這些特定標籤也被重新驗證為止：

```tsx
// Page A: /blog
const posts = await fetch('https://api.vercel.app/blog', {
  next: { tags: ['posts'] },
})

// Page B: /dashboard
const recentPosts = await fetch('https://api.vercel.app/blog?limit=5', {
  next: { tags: ['posts'] },
})
```

呼叫 `revalidatePath('/blog')` 後：

- **頁面 A (/blog)**：顯示新資料（頁面已重新渲染）
- **頁面 B (/dashboard)**：仍然顯示過時的資料（快取標籤 'posts' 未無效化）

### 建立重新驗證工具

`revalidatePath` 和 `revalidateTag` 是互補的原生功能，通常於工具函數中搭配使用，以確保應用程式中的資料一致性：

```ts
'use server'

import { revalidatePath, revalidateTag } from 'next/cache'

export async function updatePost() {
  await updatePostInDatabase()

  revalidatePath('/blog') // Refresh the blog page
  revalidateTag('posts') // Refresh all pages using 'posts' tag
}
```

此模式確保特定頁面與任何其他使用相同資料的頁面都能保持一致。

## 範例

### 重新驗證特定 URL

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

此行為將使特定 URL 在下次頁面造訪時無效化以便重新驗證。

### 重新驗證頁面路徑

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// or with route groups
revalidatePath('/(main)/blog/[slug]', 'page')
```

此行為將使任何符合提供的 `page` 檔案的 URL 在下次頁面造訪時無效化以便重新驗證。這 _不會_ 使特定頁面下方的頁面無效化。例如，`/blog/[slug]` 不會無效化 `/blog/[slug]/[author]`。

### 重新驗證佈局路徑

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// or with route groups
revalidatePath('/(main)/post/[slug]', 'layout')
```

此行為將使任何符合提供的 `layout` 檔案的 URL 在下次頁面造訪時無效化以便重新驗證。這將導致與該佈局相同的下方頁面在下次造訪時被無效化並重新驗證。例如，在上面的情況下，`/blog/[slug]/[another]` 也會在下次造訪時被無效化並重新驗證。

### 重新驗證所有資料

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

此行為將清除客戶端路由快取 (Client-side Router Cache)，並使資料快取 (Data Cache) 在下次頁面造訪時無效化以便重新驗證。

### 伺服器函數

```ts filename="app/actions.ts" switcher
'use server'

import { revalidatePath } from 'next/cache'

export default async function submit() {
  await submitForm()
  revalidatePath('/')
}
```

### 路由處理程式

```ts filename="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
