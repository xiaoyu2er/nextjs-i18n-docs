---
title: 不穩定重發 (unstable_rethrow)
description: unstable_rethrow 函數的 API 參考。
version: unstable
---

`unstable_rethrow` 可用於避免在嘗試處理應用程式代碼中拋出的錯誤時，捕獲 Next.js 拋出的內部錯誤。

例如，調用 `notFound` 函數會拋出 Next.js 內部錯誤並渲染 [`not-found.js`](/docs/app/api-reference/file-conventions/not-found) 組件。然而，如果用於 `try/catch` 語句的 `try` 區塊內，錯誤將被捕獲，從而阻止 `not-found.js` 渲染：

```tsx filename="@/app/ui/component.tsx"
import { notFound } from 'next/navigation'

export default async function Page() {
  try {
    const post = await fetch('https://.../posts/1').then((res) => {
      if (res.status === 404) notFound()
      if (!res.ok) throw new Error(res.statusText)
      return res.json()
    })
  } catch (err) {
    console.error(err)
  }
}
```

您可以使用 `unstable_rethrow` API 重新拋出內部錯誤並繼續執行預期行為：

```tsx filename="@/app/ui/component.tsx"
import { notFound, unstable_rethrow } from 'next/navigation'

export default async function Page() {
  try {
    const post = await fetch('https://.../posts/1').then((res) => {
      if (res.status === 404) notFound()
      if (!res.ok) throw new Error(res.statusText)
      return res.json()
    })
  } catch (err) {
    unstable_rethrow(err)
    console.error(err)
  }
}
```

以下 Next.js API 依賴於拋出錯誤，這些錯誤應該被重新拋出並由 Next.js 本身處理：

- [`notFound()`](/docs/app/api-reference/functions/not-found)
- [`redirect()`](/docs/app/guides/redirecting#redirect-function)
- [`permanentRedirect()`](/docs/app/guides/redirecting#permanentredirect-function)

如果路由段設定為除非是靜態否則拋出錯誤，動態 API 呼叫也將拋出錯誤，開發者同樣不應捕獲該錯誤。請注意，部分預渲染 (PPR) 也會影響此行為。這些 API 包括：

- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional)
- `fetch(..., { cache: 'no-store' })`
- `fetch(..., { next: { revalidate: 0 } })`

> **小提示**:
>
> - 此方法應在 `catch` 區塊頂部調用，並將錯誤物件作為唯一參數傳入。它也可用於 Promise 的 `.catch` 處理程序內。
> - 如果您封裝了拋出錯誤的 API 呼叫，並讓**呼叫者**處理異常，則可能可以避免使用 `unstable_rethrow`。
> - 僅當您捕獲的異常可能同時包含應用程式錯誤和框架控制的異常（如 `redirect()` 或 `notFound()`）時，才使用 `unstable_rethrow`。
> - 任何資源清理（如清除間隔、計時器等）必須在調用 `unstable_rethrow` 之前發生，或在 `finally` 區塊內進行。
>
