`cookies` 是一個 **async** 函式，允許您在 [Server Components](/docs/app/getting-started/server-and-client-components) 中讀取 HTTP 請求進入的 cookie，並讀取/寫入 [Server Actions](/docs/app/getting-started/updating-data) 或 [Route Handlers](/docs/app/api-reference/file-conventions/route) 中的請求外發 cookie。

```tsx filename="app/page.tsx" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

```js filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

## 參考

### 方法

以下可用方法：

| 方法                      | 回傳類型      | 描述                                                                     |
| --------------------------- | ---------------- | ------------------------------------------------------------------------------- |
| `get('name')`               | Object           | 接受 cookie 名稱並返回包含名稱和值的物件。            |
| `getAll()`                  | Array of objects | 返回所有名稱匹配的 cookie 列表。                         |
| `has('name')`               | Boolean          | 接受 cookie 名稱並根據 cookie 是否存在返回布林值。      |
| `set(name, value, options)` | -                | 接受 cookie 名稱、值和選項，並設定外發請求 cookie。 |
| `delete(name)`              | -                | 接受 cookie 名稱並刪除 cookie。                                   |
| `clear()`                   | -                | 刪除所有 cookie。                                                            |
| `toString()`                | String           | 返回 cookie 的字串表示。                                 |

### 選項

設定 cookie 時，支援 `options` 物件中的以下屬性：

| 選項        | 類型                                   | 描述                                                                        |
| ------------- | -------------------------------------- | ---------------------------------------------------------------------------------- |
| `name`        | String                                 | 指定 cookie 的名稱。                                                  |
| `value`       | String                                 | 指定要儲存在 cookie 中的值。                                    |
| `expires`     | Date                                   | 定義 cookie 過期的確切日期。                                |
| `maxAge`      | Number                                 | 設定 cookie 的生命週期（以秒為單位）。                                             |
| `domain`      | String                                 | 指定 cookie 可用的網域。                                |
| `path`        | String, 預設：'/'                 | 將 cookie 的範圍限制在網域內的特定路徑。                    |
| `secure`      | Boolean                                | 確保 cookie 僅透過 HTTPS 連線發送以增強安全性。         |
| `httpOnly`    | Boolean                                | 將 cookie 限制為 HTTP 請求，防止客戶端存取。              |
| `sameSite`    | Boolean, 'lax', 'strict', 'none' | 控制 cookie 的跨站請求行為。                                 |
| `priority`    | String ("low", "medium", "high") | 指定 cookie 的優先級                                                    |
| `partitioned` | Boolean                                | 指示 cookie 是否為 [分割](https://github.com/privacycg/CHIPS)。 |

唯一具有預設值的選項是 `path`。

要了解這些選項的更多資訊，請參閱 [MDN 文件](https://developer.mozilla.org/en-US/docs/Web/HTTP/Cookies)。

## 小知識

- `cookies` 是一個 **非同步** 函式，它會傳回 promise。您必須使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函式來存取 cookies。
  - 在 14 及更早版本中，`cookies` 是一個同步函式。為了協助向下相容，您可以在 Next.js 15 中繼續同步存取它，但此行為未來將被棄用。
- `cookies` 是一個 [動態 API](/docs/app/getting-started/partial-prerendering#dynamic-rendering)，其返回值無法提前預知。在佈局或頁面中使用它將使路由選擇 [動態渲染](/docs/app/getting-started/partial-prerendering#dynamic-rendering)。
- `.delete` 方法僅可在以下情況呼叫：
  - 在 [Server Action](/docs/app/getting-started/updating-data) 或 [Route Handler](/docs/app/api-reference/file-conventions/route) 中。
  - 如果它屬於 `.set` 呼叫所用的相同網域。對於萬用字域名，特定子網域必須完全匹配。此外，程式碼必須在與要刪除的 cookie 相同通訊協定（HTTP 或 HTTPS）上執行。
- HTTP 不允許在串流開始後設定 cookie，因此您必須在 [Server Action](/docs/app/getting-started/updating-data) 或 [Route Handler](/docs/app/api-reference/file-conventions/route) 中使用 `.set`。

## 理解 Server Components 中的 Cookie 行為

當在 Server Components 中使用 cookie 時，了解 cookie 本質上是一種客戶端儲存機制非常重要：

- **讀取 cookie** 在 Server Components 中可行，因為您正在存取客戶端瀏覽器透過 HTTP 請求標頭發送給伺服器的 cookie 資料。
- **設定 cookie** 無法直接在 Server Component 中完成，即使使用 Route Handler 或 Server Action。這是因為 cookie 實際上是儲存在瀏覽器中，而不是伺服器上。

伺服器僅能發送指示（透過 `Set-Cookie` 標頭）來告訴瀏覽器儲存 cookie —— 實際儲存發生在客戶端。這就是為什麼修改狀態的 cookie 操作（`.set`, `.delete`, `.clear`）必須在 Route Handler 或 Server Action 中執行，以便正確設定回應標頭。

## 理解 Server Actions 中的 Cookie 行為

當您在 Server Action 中設定或刪除 cookie 後，Next.js 會重新繪製伺服器上的目前頁面及其佈局，以便 UI 反映新的 cookie 值。請參閱 [快取指南](/docs/app/guides/caching#cookies)。

UI 不會解除掛載，但依賴來自伺服器資料的效果將會重新執行。

要同時刷新快取的數據，請在動作中呼叫 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 或 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag)。

## 範例

### 取得 cookie

您可以使用 `(await cookies()).get('name')` 方法來取得單個 cookie：

```tsx filename="app/page.tsx" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

```jsx filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

### 取得所有 cookie

您可以使用 `(await cookies()).getAll()` 方法來取得所有名稱匹配的 cookie。如果未指定 `name`，則返回所有可用的 cookie。

```tsx filename="app/page.tsx" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  return cookieStore.getAll().map((cookie) => (
    <div key={cookie.name}>
      <p>Name: {cookie.name}</p>
      <p>Value: {cookie.value}</p>
    </div>
  ))
}
```

```jsx filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  return cookieStore.getAll().map((cookie) => (
    <div key={cookie.name}>
      <p>Name: {cookie.name}</p>
      <p>Value: {cookie.value}</p>
    </div>
  ))
}
```

### 設定 cookie

您可以在 [Server Action](/docs/app/getting-started/updating-data) 或 [Route Handler](/docs/app/api-reference/file-conventions/route) 中使用 `(await cookies()).set(name, value, options)` 方法來設定 cookie。[`options` 物件](#options) 是可選的。

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function create(data) {
  const cookieStore = await cookies()

  cookieStore.set('name', 'lee')
  // or
  cookieStore.set('name', 'lee', { secure: true })
  // or
  cookieStore.set({
    name: 'name',
    value: 'lee',
    httpOnly: true,
    path: '/',
  })
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function create(data) {
  const cookieStore = await cookies()

  cookieStore.set('name', 'lee')
  // or
  cookieStore.set('name', 'lee', { secure: true })
  // or
  cookieStore.set({
    name: 'name',
    value: 'lee',
    httpOnly: true,
    path: '/',
  })
}
```

### 檢查 cookie 是否存在

您可以使用 `(await cookies()).has(name)` 方法檢查是否存在 cookie：

```tsx filename="app/page.ts" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const hasCookie = cookieStore.has('theme')
  return '...'
}
```

```jsx filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const hasCookie = cookieStore.has('theme')
  return '...'
}
```

### 刪除 Cookie

有三種方法可以刪除 cookie。

使用 `delete()` 方法：

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function delete(data) {
  (await cookies()).delete('name')
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function delete(data) {
  (await cookies()).delete('name')
}
```

設定具有相同名稱和空值的新 cookie：

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function delete(data) {
  (await cookies()).set('name', '')
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function delete(data) {
  (await cookies()).set('name', '')
}
```

將 `maxAge` 設定為 0 將立即使 cookie 過期。`maxAge` 接受以秒為單位的值。

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function delete(data) {
  (await cookies()).set('name', 'value', { maxAge: 0 })
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function delete(data) {
  (await cookies()).set('name', 'value', { maxAge: 0 })
``
}
```
