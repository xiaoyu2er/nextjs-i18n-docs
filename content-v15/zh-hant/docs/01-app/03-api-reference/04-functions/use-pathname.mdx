`usePathname` 是一個 **Client Component** hook，讓您可以讀取目前 URL 的 **pathname**。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'

export default function ExampleClientComponent() {
  const pathname = usePathname()
  return <p>Current pathname: {pathname}</p>
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { usePathname } from 'next/navigation'

export default function ExampleClientComponent() {
  const pathname = usePathname()
  return <p>Current pathname: {pathname}</p>
}
```

`usePathname` 刻意要求使用 [Client Component](/docs/app/getting-started/server-and-client-components)。值得注意的是，Client Components 並非去最佳化。它們是 [Server Components](/docs/app/getting-started/server-and-client-components) 架構中不可或缺的一部分。

例如，帶有 `usePathname` 的 Client Component 會在初次頁面載入時被渲染為 HTML。當導航至新路由時，此組件不需要重新獲取。取而代之，此組件僅下載一次（在客戶端 JavaScript 封包中），並根據當前狀態重新渲染。

> **小知識**：
>
> - 無法從 [Server Component](/docs/app/getting-started/server-and-client-components) 讀取當前 URL。此設計是有意為之，以便在頁面導航之間保留佈局狀態。
> - 相容模式：
>   - 當 [fallback route](/docs/pages/api-reference/functions/get-static-paths#fallback-true) 正在渲染時，或者 `pages` 目錄頁面已被 Next.js [自動靜態最佳化](/docs/pages/building-your-application/rendering/automatic-static-optimization) 且路由器尚未就緒時，`usePathname` 可能會返回 `null`。
>   - 當在 [`next.config`](/docs/app/api-reference/config/next-config-js/rewrites) 或 [`Middleware`](/docs/app/api-reference/file-conventions/middleware) 中使用 `usePathname` 進行重寫時，也必須使用 `useState` 和 `useEffect`，以避免 hydration 不匹配錯誤。
>   - 如果您專案中同時偵測到 `app` 和 `pages` 目錄，Next.js 將自動更新您的類型。

## 參數

```tsx
const pathname = usePathname()
```

`usePathname` 不接收任何參數。

## 返回值

`usePathname` 返回當前 URL 的 pathname 字串。例如：

| URL                 | 傳回的值        |
| ------------------- | --------------------- |
| `/`                 | `'/'`                 |
| `/dashboard`        | `'/dashboard'`        |
| `/dashboard?v=2`    | `'/dashboard'`        |
| `/blog/hello-world` | `'/blog/hello-world'` |

## 範例

### 針對路由變更執行操作

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

function ExampleClientComponent() {
  const pathname = usePathname()
  const searchParams = useSearchParams()
  useEffect(() => {
    // Do something here...
  }, [pathname, searchParams])
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

function ExampleClientComponent() {
  const pathname = usePathname()
  const searchParams = useSearchParams()
  useEffect(() => {
    // Do something here...
  }, [pathname, searchParams])
}
```

### 使用重寫時避免 hydration 不匹配

當頁面被預先渲染時，HTML 會為原始 pathname 生成。如果通過 `next.config` 或 `Middleware` 使用重寫來存取頁面，瀏覽器 URL 可能會不同，並且 `usePathname()` 會在客戶端讀取重寫後的 pathname。

為了避免 hydration 不匹配，設計 UI 時應確保只有少量、隔離的部分依賴客戶端 pathname。在伺服器渲染時呈現穩定的 fallback，並在掛載後更新該部分。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useEffect, useState } from 'react'
import { usePathname } from 'next/navigation'

export default function PathnameBadge() {
  const pathname = usePathname()
  const [clientPathname, setClientPathname] = useState('')

  useEffect(() => {
    setClientPathname(pathname)
  }, [pathname])

  return (
    <p>
      Current pathname: <span>{clientPathname}</span>
    </p>
  )
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useEffect, useState } from 'react'
import { usePathname } from 'next/navigation'

export default function PathnameBadge() {
  const pathname = usePathname()
  const [clientPathname, setClientPathname] = useState('')

  useEffect(() => {
    setClientPathname(pathname)
  }, [pathname])

  return (
    <p>
      Current pathname: <span>{clientPathname}</span>
    </p>
  )
}
```
