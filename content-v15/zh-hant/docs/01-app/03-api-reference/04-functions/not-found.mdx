「notFound」函式允許您在路由段中渲染 [`not-found 檔案`](/docs/app/api-reference/file-conventions/not-found)，並注入 `<meta name="robots" content="noindex" />` 標籤。

## notFound()

調用 `notFound()` 函式會拋出 `NEXT_HTTP_ERROR_FALLBACK;404` 錯誤，並終止發生該錯誤的路由段之渲染。指定 [**not-found 檔案**](/docs/app/api-reference/file-conventions/not-found) 允許您透過在路由段內渲染「找不到」的 UI 來優雅地處理此類錯誤。

```jsx filename="app/user/[id]/page.js"
import { notFound } from 'next/navigation'

async function fetchUser(id) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({ params }) {
  const { id } = await params
  const user = await fetchUser(id)

  if (!user) {
    notFound()
  }

  // ...
}
```

> **小知識**：由於使用了 TypeScript 的 [`never`](https://www.typescriptlang.org/docs/handbook/2/functions.html#never) 類型，`notFound()` 不需要您使用 `return notFound()`。
