cacheLife 函數用於設定函數或元件的快取生命週期。它應與 [`use cache`](/docs/app/api-reference/directives/use-cache) 指令搭配使用，且位於該函數或元件的範圍內。

## 使用方法

要使用 cacheLife，請在您的 next.config.js 檔案中啟用 [`cacheComponents` 標誌](/docs/app/api-reference/config/next-config-js/cacheComponents)：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    cacheComponents: true,
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
const nextConfig = {
  experimental: {
    cacheComponents: true,
  },
}

export default nextConfig
```

接著，在該函數或元件的範圍內匯入並調用 cacheLife 函數：

```tsx filename="app/page.tsx" highlight={5} switcher
'use cache'
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default async function Page() {
  cacheLife('hours')
  return <div>Page</div>
}
```

```jsx filename="app/page.js" highlight={5} switcher
'use cache'
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default async function Page() {
  cacheLife('hours')
  return <div>Page</div>
}
```

## 參考

### 預設快取設定檔

Next.js 提供一組基於不同時間範圍的命名快取設定檔。如果您沒有在 cacheLife 函數中與 use cache 指令一同指定快取設定檔，Next.js 將自動套用預設快取設定檔。

不過，我們建議在使用 use cache 指令時始終加入快取設定檔，以明確定義快取行為。

| **設定檔** | `stale`   | `revalidate` | `expire` | **描述**                                                          |
| ----------- | --------- | ------------ | -------- | ----------------------------------------------------------------- |
| `default`   | 5 分鐘    | 15 分鐘      | 1 年     | 預設設定檔，適用於不需要頻繁更新的内容                            |
| `seconds`   | 0         | 1 秒         | 1 秒     | 適用於需要近乎即時更新的快速變更内容                              |
| `minutes`   | 5 分鐘    | 1 分鐘       | 1 小時   | 適用於一小時內頻繁更新的内容                                      |
| `hours`     | 5 分鐘    | 1 小時       | 1 天     | 適用於每天更新但可稍許滯後的内容                                  |
| `days`      | 5 分鐘    | 1 天         | 1 週     | 適用於每週更新但可能為一天前內容                                  |
| `weeks`     | 5 分鐘    | 1 週         | 30 天    | 適用於每月更新但可能為一週前內容                                  |
| `max`       | 5 分鐘    | 30 天        | 1 年     | 適用於極少需要更新的極穩定内容                                    |

用於參考快取設定檔的字串值本身不攜帶具體意義；它們作為語意標籤使用。這讓您能更好地理解和管理程式庫中的快取内容。

> **實用知識：** 更新 [`staleTimes`](/docs/app/api-reference/config/next-config-js/staleTimes) 和 [`expireTime`](/docs/app/api-reference/config/next-config-js/expireTime) 設定選項也會更新預設快取設定檔的 `stale` 和 `expire` 屬性。

### 自訂快取設定檔

您可以透過在您的 next.config.ts 檔案中的 [`cacheLife`](/docs/app/api-reference/config/next-config-js/cacheLife) 選項中新增自訂快取設定檔來進行設定。

快取設定檔是包含以下屬性的一般物件：

| **屬性**     | **值**       | **描述**                                                                                                             | **需求**                             |
| ------------ | ------------ | -------------------------------------------------------------------------------------------------------------------- | ------------------------------------ |
| `stale`      | `number`     | 客戶端應快儲值而不檢查伺服器的時間長度。                                                                             | 選填                                 |
| `revalidate` | `number`     | 快取應在伺服器上重新整理的頻率；重新校正期間可能提供滯後值。                                                         | 選填                                 |
| `expire`     | `number`     | 值可保持滯後狀態的最大時間長度，之後切換到動態抓取；必須比 `revalidate` 長。                                         | 選填 - 必須比 `revalidate` 長        |

“stale”屬性與 [`staleTimes`](/docs/app/api-reference/config/next-config-js/staleTimes) 設定不同，它特別控制客戶端 Router 快取。雖然 `staleTimes` 是全球設定，會影響動態和静态資料的所有執行個體，但 `cacheLife` 設定允許您針對每個函數或路由定義“滯後”時間。

> **實用知識：** “stale”屬性不設定 `Cache-control: max-age` 標頭。它反而控制客戶端 Router 快取。

## 範例

### 定義可複用的快取設定檔

您可以在您的 next.config.ts 檔案中定義它們來建立可複用的快取設定檔。選擇適合您使用案例的名稱，並設定 `stale`、`revalidate` 和 `expire` 屬性的值。您可以建立任意数量的自訂快取設定檔。每個設定檔均可透過其名稱作為字串值傳遞給 `cacheLife` 函數來參考。

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    cacheComponents: true,
    cacheLife: {
      biweekly: {
        stale: 60 * 60 * 24 * 14, // 14 days
        revalidate: 60 * 60 * 24, // 1 day
        expire: 60 * 60 * 24 * 14, // 14 days
      },
    },
  },
}

module.exports = nextConfig
```

```js filename="next.config.js" switcher
const nextConfig = {
  experimental: {
    cacheComponents: true,
    cacheLife: {
      biweekly: {
        stale: 60 * 60 * 24 * 14, // 14 days
        revalidate: 60 * 60 * 24, // 1 day
        expire: 60 * 60 * 24 * 14, // 14 days
      },
    },
  },
}

module.exports = nextConfig
```

上述範例快取 14 天，每天檢查更新，並在 14 天后使快取過期。然後您可以在應用程式中透過其名稱引用此設定檔：

```tsx filename="app/page.tsx" highlight={5}
'use cache'
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default async function Page() {
  cacheLife('biweekly')
  return <div>Page</div>
}
```

### 覆寫預設快取設定檔

雖然預設快取設定檔提供了一種思考快取輸出中任何特定部分的新舊程度的有用方式，您可能偏好不同的命名設定檔，以更好地符合您應用程式的快取策略。

您可以建立與預設相同名稱的新設定，來覆寫預設命名快取設定檔。

下方範例顯示如何覆寫預設的「days」快取設定檔：

```ts filename="next.config.ts"
const nextConfig = {
  experimental: {
    cacheComponents: true,
    cacheLife: {
      days: {
        stale: 3600, // 1 hour
        revalidate: 900, // 15 minutes
        expire: 86400, // 1 day
      },
    },
  },
}

module.exports = nextConfig
```

### 行內定義快取設定檔

對於特定使用案例，您可以透過將物件傳遞給 `cacheLife` 函數來設定自訂快取設定檔：

```tsx filename="app/page.tsx" highlight={5-9} switcher
'use cache'
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default async function Page() {
  cacheLife({
    stale: 3600, // 1 hour
    revalidate: 900, // 15 minutes
    expire: 86400, // 1 day
  })

  return <div>Page</div>
}
```

```jsx filename="app/page.js" highlight={5-9} switcher
'use cache'
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default async function Page() {
  cacheLife({
    stale: 3600, // 1 hour
    revalidate: 900, // 15 minutes
    expire: 86400, // 1 day
  })

  return <div>Page</div>
}
```

此行內快取設定檔僅適用於建立它的函數或檔案。如果您想在整個應用程式中重複使用相同的設定檔，您可以將 [設定新增](#defining-reusable-cache-profiles) 到您的 `next.config.ts` 檔案的 `cacheLife` 屬性中。

### `use cache` 和 `cacheLife` 的巢狀使用

當在相同路由或元件樹中定義多個快取行為時，如果內部快取指定了它們自己的 `cacheLife` 設定檔，外部快取將遵守其中最短的快取時間。**這僅適用於外部快取沒有定義其自己的明確 `cacheLife` 設定檔 的情況。**

例如，如果您將 `use cache` 指令新增到您的頁面，而未指定快取設定檔，預設快取設定檔將被隱式套用 (`cacheLife("default")`)。如果頁面匯入的元件也使用帶有其自己快取設定檔的 `use cache` 指令，則外部和內部快取設定檔會進行比較，並將設定檔中設定的最短時間套用。

```tsx filename="app/components/parent.tsx" highlight={5,6}
// Parent component
import { unstable_cacheLife as cacheLife } from 'next/cache'
import { ChildComponent } from './child'

export async function ParentComponent() {
  'use cache'
  cacheLife('days')

  return (
    <div>
      <ChildComponent />
    </div>
  )
}
```

在另一個單獨的檔案中，我們定義了匯入的子元件：

```tsx filename="app/components/child.tsx" highlight={4,5}
// Child component
import { unstable_cacheLife as cacheLife } from 'next/cache'

export async function ChildComponent() {
  'use cache'
  cacheLife('hours')
  return <div>Child Content</div>

  // This component's cache will respect the shorter 'hours' profile
}
```
