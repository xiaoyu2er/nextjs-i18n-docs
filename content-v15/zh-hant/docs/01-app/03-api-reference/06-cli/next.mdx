Next.js CLI 允許您開發、構建、啟動您的應用程序等操作。

基本用法：

```bash filename="Terminal"
npx next [command] [options]
```

## 參考

下列選項可用：

| Options             | Description                        |
| ------------------- | ---------------------------------- |
| `-h` or `--help`    | 顯示所有可用選項        |
| `-v` or `--version` | 輸出 Next.js 版本號 |

### 命令

下列命令可用：

| Command                                | Description                                                                                                                                                                                                                        |
| -------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [`dev`](#next-dev-options)             | 以開發模式啟動 Next.js，包含熱模組載入 (HMR)、錯誤報告等功能。                                                                                                                                                                                                                     |
| [`build`](#next-build-options)         | 創建應用程序的優化生產構建。顯示關於每個路由的信息。                                                                                                                                                                                                |
| [`start`](#next-start-options)         | 以生產模式啟動 Next.js。應用程序應先使用 `next build` 進行編譯。                                                                                                                                                                                                     |
| [`info`](#next-info-options)           | 打印有關當前系統的相關詳情，可用於報告 Next.js 錯誤。                                                                                                                                                                                                         |
| [`lint`](#next-lint-options)           | 為 `/src`, `/app`, `/pages`, `/components`, and `/lib` 目錄中的所有文件運行 ESLint。如果應用程序中尚未配置 ESLint，它還提供定向設置以安裝任何必需的依賴項。                                                                                                                |
| [`telemetry`](#next-telemetry-options) | 允許您啟用或禁用 Next.js 的完全匿名遥測收集。                                                                                                                                                                                                                |
| [`typegen`](#next-typegen-options)     | 無需執行完整構建即可為路由、頁面、佈局和路由處理程序生成 TypeScript 定義。                                                                                                                                                                                      |

> **小知識**: 運行 `next` 而沒有命令是 `next dev` 的別名。

### `next dev` 選項

`next dev` 以開發模式啟動應用程序，包含熱模組載入 (HMR)、錯誤報告等功能。運行 `next dev` 時可用以下選項：

| Option                                   | Description                                                                                                                                          |
| ---------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `-h, --help`                             | 顯示所有可用選項。                                                                                                                          |
| `[directory]`                            | 構建應用程序時使用的目錄。如果未提供，則使用當前目錄。                                                                                                                           |
| `--turbopack`                            | 使用 [Turbopack](/docs/app/api-reference/turbopack) 啟動開發模式。也可作為 `--turbo` 使用。                                           |
| `-p` or `--port <port>`                  | 指定啟動應用程序時使用的端口號。默認值：3000，環境變量：PORT                                                                    |
| `-H`or `--hostname <hostname>`           | 指定啟動應用程序時使用的主機名。對於使應用程序在網絡上可用於其他設備很有用。默認值：0.0.0.0 |
| `--experimental-https`                   | 使用 HTTPS 啟動服務器並生成自簽名證書。                                                                                |
| `--experimental-https-key <path>`        | HTTPS 密鑰文件的路徑。                                                                                                                            |
| `--experimental-https-cert <path>`       | HTTPS 證書文件的路徑。                                                                                                                    |
| `--experimental-https-ca <path>`         | HTTPS 證書頒發機構文件的路徑。                                                                                                          |
| `--experimental-upload-trace <traceUrl>` | 將調試追蹤的子集報告到遠程 HTTP URL。                                                                                        |

### `next build` 選項

`next build` 創建應用程序的優化生產構建。輸出顯示有關每個路由的信息。例如：

```bash filename="Terminal"
Route (app)                              Size     First Load JS
┌ ○ /_not-found                          0 B               0 kB
└ ƒ /products/[id]                       0 B               0 kB

○  (Static)   prerendered as static content
ƒ  (Dynamic)  server-rendered on demand
```

- **大小**: 在客戶端導航到頁面時下載的資源大小。每個路由的大小僅包含其依賴項。
- **首次加載 JS**: 從服務器訪問頁面時下載的資源大小。所有 JS 共享的量顯示為單獨的指標。

這兩個值均 [使用 gzip 壓縮](/docs/app/api-reference/config/next-config-js/compress)。第一次加載顯示為綠色、黃色或紅色。目標是綠色，以獲得流暢的應用程序性能。

下列選項可用於 `next build` 命令：

| Option                             | Description                                                                                                                                                                        |
| ---------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `-h, --help`                       | 顯示所有可用選項。                                                                                                                                                        |
| `[directory]`                      | 構建應用程序時使用的目錄。如果未提供，則使用當前目錄。                                                                                |
| `--turbopack`                      | 使用 [Turbopack](/docs/app/api-reference/turbopack)（測試版）進行構建。也可作為 `--turbo` 使用。                                                                                    |
| `-d` or `--debug`                  | 啟用更詳細的構建輸出。啟用了此標志後，將顯示其他構建輸出，如重寫、重定向和標頭。                                                                                                                                                                           |
|                                    |
| `--profile`                        | 啟用 React 的 [生產剖析](https://react.dev/reference/react/Profiler)。                                                                                              |
| `--no-lint`                        | 禁用代碼檢查。_註：在 Next 16 中將從 `next build` 中移除代碼檢查。如果您使用的 Next 15.5+ 且代碼檢查工具不是 `eslint`，則構建期間不會發生代碼檢查。_ |
| `--no-mangling`                    | 禁用 [名稱修改](https://en.wikipedia.org/wiki/Name_mangling)。這可能會影響性能，僅應用於調試目的。                                      |
| `--experimental-app-only`          | 僅構建 App Router 路由。                                                                                                                                                     |
| `--experimental-build-mode [mode]` | 使用實驗性構建模式。（選擇："compile", "generate"，默認："default"                                                                                              |
| `--debug-prerender`                | 調試開發過程中的預渲染錯誤。                                                                                                                                                                                                             |

### `next start` 選項

`next start` 以生產模式啟動應用程序。應用程序應先使用 [`next build`](#next-build-options) 進行編譯。

下列選項可用於 `next start` 命令：

| Option                                  | Description                                                                                                     |
| --------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| `-h` or `--help`                        | 顯示所有可用選項。                                                                                     |
| `[directory]`                           | 啟動應用程序時使用的目錄。如果未提供目錄，則使用當前目錄。 |
| `-p` or `--port <port>`                 | 指定啟動應用程序時使用的端口號。(default: 3000, env: PORT)                             |
| `-H` or `--hostname <hostname>`         | 指定啟動應用程序時使用的主機名 (default: 0.0.0.0)。                                        |
| `--keepAliveTimeout <keepAliveTimeout>` | 指定在關閉非活動連接之前等待的最大毫秒數。                     |

### `next info` 選項

`next info` 打印有關當前系統的相關詳情，可用於在開啟 [GitHub issue](https://github.com/vercel/next.js/issues) 時報告 Next.js 錯誤。此信息包括操作系統平台/架構/版本、編譯器 (Node.js, npm, Yarn, pnpm)、套件版本 (`next`, `react`, `react-dom`) 等。

下列選項可用於 `next info` 命令：

| `--verbose`      | 收集用於調試的額外信息。 |

### `next lint` 選項

> **警告**: 此選項已棄用，將在 Next 16 中移除。可用的 [codemod](/blog/next-15-5#next-lint-deprecation) 可用於遷移到 ESLint CLI。

`next lint` 為 `pages/`, `app/`, `components/`, `lib/`, and `src/` 目錄中的所有文件運行 ESLint。如果應用程序中尚未配置 ESLint，它還提供定向設置以安裝任何必需的依賴項。

下列選項可用於 `next lint` 命令：

| Option                                                | Description                                                                                                   |
| ----------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| `[directory]`                                         | 代碼檢查應用程序時使用的基本目錄。如果未提供，則使用當前目錄。       |
| `-d, --dir, <dirs...>`                                | 包含要運行 ESLint 的目錄。                                                             |
| `--file, <files...>`                                  | 包含要運行 ESLint 的文件。                                                                        |
| `--ext, [exts...]`                                    | 指定 JavaScript 文件擴展名。(default: [".js", ".mjs", ".cjs", ".jsx", ".ts", ".mts", ".cts", ".tsx"]) |
| `-c, --config, <config>`                              | 使用此配置文件，覆蓋所有其他配置選項。                                     |
| `--resolve-plugins-relative-to, <rprt>`               | 指定插件解析目錄。                                                    |
| `--strict`                                            | 使用 Next.js 嚴格配置文件創建 `.eslintrc.json` 文件。                                       |
| `--rulesdir, <rulesdir...>`                           | 使用此目录中的附加規則。                                                                 |
| `--fix`                                               | 自動修復代碼檢查問題。                                                                             |
| `--fix-type <fixType>`                                | 指定修復類型 (例如，problem, suggestion, layout)。                                      |
| `--ignore-path <path>`                                | 指定忽略的文件。                                                                                     |
| `--no-ignore <path>`                                  | 禁用 `--ignore-path` 選項。                                                                          |
| `--quiet`                                             | 僅報告錯誤。                                                                                          |
| `--max-warnings [maxWarnings]`                        | 指定在觸發非零退出代碼之前允許的警告數量。(default: -1)                          |
| `-o, --output-file, <outputFile>`                     | 指定用於撰寫報告的文件。                                                                            |
| `-f, --format, <format>`                              | 使用特定的輸出格式。                                                                                |
| `--no-inline-config`                                  | 防止註釋更改配置或規則。                                                              |
| `--report-unused-disable-directives-severity <level>` | 指定未使用的 eslint-disable 指令的嚴重性級別。(choices: "error", "off", "warn")                |
| `--no-cache`                                          | 禁用緩存。                                                                                             |
| `--cache-location, <cacheLocation>`                   | 指定緩存位置。                                                                                 |
| `--cache-strategy, [cacheStrategy]`                   | 指定用於檢測緩存中更改文件的策略。(default: "metadata")                     |
| `--error-on-unmatched-pattern`                        | 當任何文件模式不匹配時報告錯誤。                                                          |
| `-h, --help`                                          | 顯示此信息。                                                                                        |

### `next telemetry` 選項

Next.js 會收集有關一般使用情况的**完全匿名**遥測數據。參與此匿名計劃是可選的，如果您不希望共享信息，可以選擇退出。

下列選項可用於 `next telemetry` 命令：

| Option       | Description                             |
| ------------ | --------------------------------------- |
| `-h, --help` | 顯示所有可用選項。             |
| `--enable`   | 啟用 Next.js 的遥測收集。  |
| `--disable`  | 禁用 Next.js 的遥測收集。 |

了解有關 [Telemetry](/telemetry) 的更多資訊。

### `next typegen` 選項

`next typegen` 無需執行完整構建即可為您的應用程序路由生成 TypeScript 定義。這對於 IDE 自動完成和 CI 路由使用類型檢查很有用。

此前，路由類型僅在 `next dev` 或 `next build` 期間生成，這意味著直接運行 `tsc --noEmit` 不會驗證您的路由類型。現在您可以獨立生成類型並在外部驗證它們：

```bash filename="Terminal"
# Generate route types first, then validate with TypeScript
next typegen && tsc --noEmit

# Or in CI workflows for type checking without building
next typegen && npm run type-check
```

下列選項可用於 `next typegen` 命令：

| Option        | Description                                                                                  |
| ------------- | -------------------------------------------------------------------------------------------- |
| `-h, --help`  | 顯示所有可用選項。                                                                  |
| `[directory]` | 生成類型時使用的目錄。如果未提供，則使用當前目錄。 |

輸出文件寫入 `<distDir>/types`（默認值：`.next/types`）：

```bash filename="Terminal"
next typegen
# or for a specific app
next typegen ./apps/web
```

> **小知識**: `next typegen` 使用生產構建階段加載您的 Next.js 配置 (`next.config.js`, `next.config.mjs`, or `next.config.ts`)。確保任何必需的環境變量和依賴項可用，以便配置可以正確加載。

## 範例

### 調試預渲染錯誤

如果您在 `next build` 期間遇到預渲染錯誤，可以通過傳遞 `--debug-prerender` 標誌來獲取更詳細的輸出：

```bash filename="Terminal"
next build --debug-prerender
```

這啟用了幾個實驗性選項以方便調試：

- 禁用服務器代碼壓縮：
  - `experimental.serverMinification = false`
  - `experimental.turbopackMinify = false`
- 為服務器束生成源映射：
  - `experimental.serverSourceMaps = true`
- 在用於預渲染的子進程中啟用源映射使用：
  - `experimental.enablePrerenderSourceMaps = true`
- 即使在首次預渲染錯誤後繼續構建，以便您可以一次性查看所有問題：
  - `experimental.prerenderEarlyExit = false`

這有助於在構建輸出中顯示更易讀的堆棧追蹤和代碼框架。

> **警告**: `--debug-prerender` 僅用於開發環境調試。切勿將使用 `--debug-prerender` 生成的構建部署到生產環境，因為它可能會影響性能。

### 更改默認端口

默認情況下，Next.js 在開發期間和與 `next start` 一起使用 `http://localhost:3000`。默認端口可以使用 `-p` 選項更改，如下所示：

```bash filename="Terminal"
next dev -p 4000
```

或者使用 `PORT` 環境變量：

```bash filename="Terminal"
PORT=4000 next dev
```

> **小知識**: `PORT` 不能設置在 `.env` 中，因為 HTTP 服務器啟動發生在任何其他代碼初始化之前。

### 開發期間使用 HTTPS

對於某些使用情況，如 webhooks 或身份驗證，您可以使用 [HTTPS](https://developer.mozilla.org/en-US/docs/Glossary/HTTPS) 在 `localhost` 上擁有安全環境。Next.js 可以使用 `--experimental-https` 標誌通過 `next dev` 生成自簽名證書：

```bash filename="Terminal"
next dev --experimental-https
```

使用生成的證書，Next.js 開發服務器將存在於 `https://localhost:3000`。默認端口 `3000` 用於指定 `-p`, `--port` 或 `PORT` 之外。

您也可以使用 `--experimental-https-key` 和 `--experimental-https-cert` 提供自定義證書和密鑰。此外，您也可以使用 `--experimental-https-ca` 提供自定義 CA 證書。

```bash filename="Terminal"
next dev --experimental-https --experimental-https-key ./certificates/localhost-key.pem --experimental-https-cert ./certificates/localhost.pem
```

`next dev --experimental-https` 僅打算用於開發，並使用 [`mkcert`](https://github.com/FiloSottile/mkcert) 創建本地可信證書。在生產環境中，請使用可信機構發放的適當證書。

### 為下游代理配置超時時間

當在下游代理後部署 Next.js 時（例如 AWS ELB/ALB 等負載平衡器），建議將 Next 的底層 HTTP 服務器配置為 [keep-alive timeouts](https://nodejs.org/api/http.html#http_server_keepalivetimeout)，這些超時時間應**大於**下游代理的超時時間。否則，一旦達到給定 TCP 連接的 keep-alive 超時時間，Node.js 將立即終止該連接而不會通知下游代理。這會導致代理在嘗試重用 Node.js 已經終止的連接時出現代理錯誤。

要為生產 Next.js 服務器配置超時值，將 `--keepAliveTimeout`（以毫秒為單位）傳遞給 `next start`，如下所示：

```bash filename="Terminal"
next start --keepAliveTimeout 70000
```

### 傳遞 Node.js 參數

您可以將任何 [node 參數](https://nodejs.org/api/cli.html#cli_node_options_options) 傳遞給 `next` 命令。例如：

```bash filename="Terminal"
NODE_OPTIONS='--throw-deprecation' next
NODE_OPTIONS='-r esm' next
NODE_OPTIONS='--inspect' next
```
