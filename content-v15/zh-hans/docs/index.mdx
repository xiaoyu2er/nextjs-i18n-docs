## Next.js 是什么？

{/* md5:7ad4b46907228cf4d9bc9b456f4196de */}
Next.js 是一个用于构建全栈 Web 应用程序的 React 框架。您可以使用 React 组件来构建用户界面，并使用 Next.js 获得额外的功能和优化。

{/* md5:b7c00f71e72e9c53d070405975d559bd */}
它还自动配置了打包工具和编译器等底层工具。您可以专注于构建您的产品并快速发布。

{/* md5:f8d841a7a8b9e6ce15326e7be301b195 */}
无论您是独立开发者还是大型团队的一员，Next.js 都可以帮助您构建交互式、动态且快速的 React 应用程序。

{/* md5:10e4f1d0ab4bce2a33eaf5bf3228db64 */}

## App Router 和 Pages Router

{/* md5:1193d153da1771dc493c0d5e1f2aa194 */}
Next.js 有两种不同的路由器：

{/* md5:8fe5729ee77652ed173a68e92d63409d */}
- **App Router**：较新的路由器，支持新的 React 功能，如 Server Components。
- **Pages Router**：原始路由器，仍然受支持并不断改进。

{/* md5:0de9683e043e20bdfda4c9f663a12787 */}
在侧边栏顶部，您会看到一个下拉菜单，允许您在 [App Router](/docs/app) 和 [Pages Router](/docs/pages) 文档之间切换。

{/* md5:28ed78f1352cee1cdc46d534c4126b66 */}
### React 版本处理

{/* md5:b53e4411e240ec758911ed3ca95d237a */}
App Router 和 Pages Router 以不同方式处理 React 版本：

{/* md5:57eefae57412f67207057db4e9a1d7d7 */}
- **App Router**：使用内置的 [React canary 版本](https://react.dev/blog/2023/05/03/react-canaries)，其中包含所有稳定的 React 19 更改，以及在 React 新版本发布前在框架中验证的新功能。
- **Pages Router**：使用项目 `package.json` 中安装的 React 版本。

{/* md5:b239111894a56710d169dbd55d4b82b8 */}
这种方法确保新 React 功能在 App Router 中可靠运行，同时保持现有 Pages Router 应用程序的向后兼容性。

{/* md5:8cedc8b4f7a0d3e4314976fee16e90d5 */}
