## 路由处理器

{/* md5:1f2350e14e74a80ddb07369e7e1884f3 */}
路由处理程序允许你使用 Web [Request](https://developer.mozilla.org/docs/Web/API/Request) 和 [Response](https://developer.mozilla.org/docs/Web/API/Response) API 为给定路由创建自定义请求处理器。

{/* md5:edb9307e0efa362489bb223cf862b7cd */}
<Image
  alt="Route.js 特殊文件"
  srcLight="/docs/light/route-special-file.png"
  srcDark="/docs/dark/route-special-file.png"
  width="1600"
  height="444"
/>

{/* md5:58e5d4ba83149b9f1ff6a4441903774c */}
> **须知**：路由处理程序仅在 `app` 目录内可用。它们等同于 `pages` 目录中的 [API 路由](/docs/pages/building-your-application/routing/api-routes)，这意味着你 **不需要** 同时使用 API 路由和路由处理程序。

{/* md5:ff415fa3d6765fa125a3ca15e4134935 */}
### 约定

{/* md5:4926376402d2f176bf5e3648536968af */}
路由处理程序在 `app` 目录中的 [`route.js|ts` 文件](/docs/app/api-reference/file-conventions/route)里定义：

```ts filename="app/api/route.ts" switcher
export async function GET(request: Request) {}
```

```js filename="app/api/route.js" switcher
export async function GET(request) {}
```

{/* md5:65f4b1d8415d5be30c44bfc1d1abe758 */}
路由处理程序可以嵌套在 `app` 目录内的任何位置，类似于 `page.js` 和 `layout.js`。但 **不能** 在与 `page.js` 相同的路由段层级存在 `route.js` 文件。

{/* md5:0ed04aa6361e0c904e16ff5a72879e99 */}
### 支持的 HTTP 方法

{/* md5:cfdf7c06834129a4abd9fab1899e17eb */}
支持以下 [HTTP 方法](https://developer.mozilla.org/docs/Web/HTTP/Methods)：`GET`、`POST`、`PUT`、`PATCH`、`DELETE`、`HEAD` 和 `OPTIONS`。如果调用了不支持的方法，Next.js 将返回 `405 Method Not Allowed` 响应。

{/* md5:24565e9141f64ace4ebc3f51f5fa3e0d */}
### 扩展的 NextRequest 和 NextResponse API

{/* md5:a9577e413bb06df30f0c9a5f11a06825 */}
除了支持原生的 [Request](https://developer.mozilla.org/docs/Web/API/Request) 和 [Response](https://developer.mozilla.org/docs/Web/API/Response) API 之外，Next.js 还通过 [`NextRequest`](/docs/app/api-reference/functions/next-request) 和 [`NextResponse`](/docs/app/api-reference/functions/next-response) 对它们进行了扩展，为高级用例提供便捷的辅助功能。

{/* md5:5a3bb9f6010f6e06ba627b963a7a5d11 */}
### 缓存

{/* md5:6bb8ee763fa043c3f55d84cbe5560c48 */}
路由处理程序默认不缓存。但是，你可以选择对 `GET` 方法进行缓存。其他支持的 HTTP 方法则**不会**被缓存。要缓存 `GET` 方法，请在你的路由处理程序文件中使用[路由配置选项](/docs/app/api-reference/file-conventions/route-segment-config#dynamic)，例如 `export const dynamic = 'force-static'`。

```ts filename="app/items/route.ts" switcher
export const dynamic = 'force-static'

export async function GET() {
  const res = await fetch('https://data.mongodb-api.com/...', {
    headers: {
      'Content-Type': 'application/json',
      'API-Key': process.env.DATA_API_KEY,
    },
  })
  const data = await res.json()

  return Response.json({ data })
}
```

```js filename="app/items/route.js" switcher
export const dynamic = 'force-static'

export async function GET() {
  const res = await fetch('https://data.mongodb-api.com/...', {
    headers: {
      'Content-Type': 'application/json',
      'API-Key': process.env.DATA_API_KEY,
    },
  })
  const data = await res.json()

  return Response.json({ data })
}
```

{/* md5:80153cad4d93e9c2009b79d6ca8ff995 */}
> **须知**：即使其他支持的 HTTP 方法与缓存的 `GET` 方法放在同一个文件中，它们也 **不会** 被缓存。

{/* md5:35ad30a0d38828ac5b7b25e130ae7e13 */}
### 特殊路由处理器

{/* md5:3d47e8e20cdfd4cd660966cdf6e5b4d0 */}
像 [`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap)、[`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image)、[`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons) 以及其他[元数据文件](/docs/app/api-reference/file-conventions/metadata) 这样的特殊路由处理程序默认是静态的，除非它们使用了动态 API 或动态配置选项。

{/* md5:79a35015855befa48727700083d8e4b0 */}
### 路由解析

{/* md5:a595e8d8d9f8574c38e730576d165744 */}
你可以将 `route` 视为最低级别的路由原语。

{/* md5:7704ce210e8bdefcab24d23385ef2576 */}
- 它们 **不** 参与布局或客户端导航，不像 `page`。
- 在与 `page.js` 相同的路由上 **不能** 存在 `route.js` 文件。

{/* md5:68c76a5d41f10510685b276b3ff557c2 */}
| 页面                  | 路由               | 结果                       |
| --------------------- | ------------------ | -------------------------- |
| `app/page.js`         | `app/route.js`     | <Cross size={18} /> 冲突   |
| `app/page.js`         | `app/api/route.js` | <Check size={18} /> 有效   |
| `app/[user]/page.js`  | `app/api/route.js` | <Check size={18} /> 有效   |

{/* md5:0172da729ce2a0492062e444b9bd0887 */}
每个 `route.js` 或 `page.js` 文件会接管该路由的所有 HTTP 方法。

```ts filename="app/page.ts" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}

// Conflict
// `app/route.ts`
export async function POST(request: Request) {}
```

```js filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}

// Conflict
// `app/route.js`
export async function POST(request) {}
```

### 路由上下文明助器

{/* md5:6f8bf681bf769431cb4f1825e20137d2 */}
在 TypeScript 中，你可以使用全局可用的 [`RouteContext`](/docs/app/api-reference/file-conventions/route#route-context-helper) 帮助器来为路由处理程序的 `context` 参数添加类型：

```ts filename="app/users/[id]/route.ts" switcher
import type { NextRequest } from 'next/server'

export async function GET(_req: NextRequest, ctx: RouteContext<'/users/[id]'>) {
  const { id } = await ctx.params
  return Response.json({ id })
}
```

{/* md5:8a9e4f6e2a181a859e9634c478858ee4 */}
> **须知**
>
> - 类型会在 `next dev`、`next build` 或 `next typegen` 期间生成。

## 中间件

{/* md5:4ff945566714a0a50b1d6418fa295c93 */}
中间件允许你在请求完成前运行代码。然后，基于传入的请求，你可以通过重写、重定向、修改请求或响应头或直接响应来修改响应。

{/* md5:17001dcfa8a16f2bf03bbdc0f529ad78 */}
### 使用场景

{/* md5:1ff8a19eb27134a34d00dc67e7bb56db */}
中间件在以下常见场景中非常有效：

{/* md5:27d4f82273436acb6c042d3c010d25f5 */}
- 读取部分传入请求后进行快速重定向
- 基于 A/B 测试或实验重写到不同的页面
- 为所有页面或页面子集修改请求头

{/* md5:6f2bc7114b71f550368b4519c3201b4e */}
中间件*不*适用于：

{/* md5:9b9040127d250832c6e3043f4b934349 */}
- 缓慢的数据获取
- 会话管理

{/* md5:da718ba3b7443fea22008b5ce47be0fc */}
使用 `options.cache`、`options.next.revalidate` 或 `options.next.tags` 的 fetch 请求在中间件中不会生效。

{/* md5:ff415fa3d6765fa125a3ca15e4134935 */}
### 约定

{/* md5:c60bf98643cda19258fa603417a478eb */}
在项目根目录下创建一个 `middleware.ts`（或 `.js`）文件，如果适用则在 `src` 目录下创建，以确保它与 `pages` 或 `app` 处于同一层级。

{/* md5:f8cdab3765c0eccf55ff529811cd6bd5 */}
> **注意**：虽然每个项目只支持一个 `middleware.ts` 文件，但你仍然可以将中间件逻辑组织为模块。将中间件功能分解到独立的 `.ts` 或 `.js` 文件中，并导入到主 `middleware.ts` 文件中。这允许更清晰地管理特定路由的中间件，并在 `middleware.ts` 中集中控制。通过强制执行单个中间件文件，可以简化配置，防止潜在的冲突，并通过避免多层中间件层来优化性能。

{/* md5:5eed7483978ab0c9d6496e63c9f65268 */}
### 示例

```ts filename="middleware.ts" switcher
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

// This function can be marked `async` if using `await` inside
export function middleware(request: NextRequest) {
  return NextResponse.redirect(new URL('/home', request.url))
}

// See "Matching Paths" below to learn more
export const config = {
  matcher: '/about/:path*',
}
```

```js filename="middleware.js" switcher
import { NextResponse } from 'next/server'

// This function can be marked `async` if using `await` inside
export function middleware(request) {
  return NextResponse.redirect(new URL('/home', request.url))
}

// See "Matching Paths" below to learn more
export const config = {
  matcher: '/about/:path*',
}
```
