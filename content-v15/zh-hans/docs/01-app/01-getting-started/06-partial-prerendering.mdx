{/* md5:1c09abcd21527f75db47430a822c8258 */}
部分预渲染 (Partial Prerendering, PPR) 是一种渲染策略，允许您在同一条路由中组合静态和动态内容。这在支持个性化动态数据的同时，也提升了初始页面性能。

{/* md5:bc3fd96dc81037e7b720e717d332b88a */}

  srcLight="/learn/light/thinking-in-ppr.png"
  srcDark="/learn/dark/thinking-in-ppr.png"
  width="1600"
  height="632"
/>

{/* md5:c6d51bb3440cbb0b336e957132d93bcc */}
当用户访问一条路由时：

{/* md5:d2c7ca316ed45d4a5aaec094b49ea60c */}
- 服务器发送包含静态内容的**外壳 (shell)**，确保快速的初始加载。
- 外壳为异步加载的动态内容留下**空洞 (holes)**。
- 动态空洞**并行流式传输**，减少了页面的整体加载时间。

{/* md5:3199aaf0c1ae2f7c92efc6d61f2b971a */}
> **🎥 观看：** 为什么选择 PPR 及其工作原理 → [YouTube (10 分钟)](https://www.youtube.com/watch?v=MTcPrTIBkpA)。

## 部分预渲染如何工作？

{/* md5:a365e2f2396ae80d16235e96798b9b1f */}
要理解部分预渲染，熟悉 Next.js 中可用的渲染策略会有所帮助。

{/* md5:97b330e96f0fd60dfcc45a2da9739387 */}
### 静态渲染

{/* md5:08e41d988dc412de5240ca9eb2ab56be */}
通过静态渲染，HTML 会提前生成——无论是在构建时还是通过[重新验证](/docs/app/guides/incremental-static-regeneration)。结果会被缓存，并在用户和请求间共享。

{/* md5:af8efd5795e2fad3f2fcc25f16ccade7 */}
在部分预渲染中，Next.js 会为路由预渲染一个**静态外壳**。这可以包括布局以及任何其他不依赖于请求时数据的组件。

{/* md5:2d0f785900e46f4f7b9c6ad4d35cc57f */}
### 动态渲染

{/* md5:276313e317c7abd959ab154f5fbe42ce */}
通过动态渲染，HTML 在**请求时**生成。这使得您可以根据请求时的数据提供个性化内容。

{/* md5:301c391af6810368129e7619b440d7d5 */}
如果组件使用了以下 API，则该组件会变为动态组件：

{/* md5:970fb26b27673c2731e45a6db179947f */}
- [`cookies`](/docs/app/api-reference/functions/cookies)
- [`headers`](/docs/app/api-reference/functions/headers)
- [`connection`](/docs/app/api-reference/functions/connection)
- [`draftMode`](/docs/app/api-reference/functions/draft-mode)
- [`searchParams` prop](/docs/app/api-reference/file-conventions/page#searchparams-optional)
- [`unstable_noStore`](/docs/app/api-reference/functions/unstable_noStore)
- [`fetch`](/docs/app/api-reference/functions/fetch) with `{ cache: 'no-store' }`

{/* md5:dea39b4732740e2b7f0e19ea50ee6108 */}
在部分预渲染中，使用这些 API 会抛出一个特殊的 React 错误，通知 Next.js 该组件无法进行静态渲染，从而导致构建错误。您可以使用 [Suspense](#suspense) 边界来包裹您的组件，以将渲染推迟到运行时。

{/* md5:88dcefb21edd2ade584d0e262767aa57 */}
### 悬念 (Suspense)

{/* md5:bcd2262970bc6c2239823b8b00ca2f5f */}
React [Suspense](https://react.dev/reference/react/Suspense) 用于将应用程序的部分渲染推迟到满足某些条件。

{/* md5:9f0b345eff168575612b9b9c5b866628 */}
在部分预渲染中，Suspense 用于在组件树中标记**动态边界**。

{/* md5:55290dd2c9f3aed68ce510b9ba7f807d */}
在构建时，Next.js 预渲染静态内容和 `fallback` UI。动态内容会被**推迟 (postponed)**，直到用户请求该路由。

{/* md5:4b322571f7dc678956315062b8640e1c */}
将组件包裹在 Suspense 中并不会使组件本身变为动态（决定动态性的是您的 API 使用），而是将 Suspense 用作一个边界，封装动态内容并启用[流式传输](#streaming)

```jsx filename="app/page.js"
import { Suspense } from 'react'
import StaticComponent from './StaticComponent'
import DynamicComponent from './DynamicComponent'
import Fallback from './Fallback'

export const experimental_ppr = true

export default function Page() {
  return (
    <>
      <StaticComponent />
      <Suspense fallback={<Fallback />}>
        <DynamicComponent />
      </Suspense>
    </>
  )
}
```

{/* md5:24436b4c367c4fcdec8aeccfdb492ed8 */}
### 流式传输

{/* md5:a61e07b8b1e0d6fa5b68932d5b208455 */}
流式传输将路由分割成块 (chunks)，并在它们准备就绪时逐步将它们流式传输到客户端。这允许用户在全部内容渲染完成之前立即看到页面的部分内容。

为了减少网络开销，完整的响应——包括静态 HTML 和流式传输的动态部分——通过**单个 HTTP 请求**发送。这避免了额外的往返，并同时改善了初始加载和整体性能。

{/* md5:27d02787aec292e55c5c27491ec66ab6 */}
## 启用部分预渲染

{/* md5:b17af40dd0e08c2f8cd7c14ebd0b6d0e */}
您可以通过将 [`ppr`](https://rc.nextjs.org/docs/app/api-reference/next-config-js/ppr) 选项添加到您的 `next.config.ts` 文件中来启用 PPR：

```ts filename="next.config.ts" highlight={5} switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    ppr: 'incremental',
  },
}

export default nextConfig
```

```js filename="next.config.js" highlight={4} switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    ppr: 'incremental',
  },
}
```

{/* md5:ec195b17458d049d1ba98342d82f6088 */}
`'incremental'` 值允许您为特定路由采用 PPR：

```tsx filename="/app/dashboard/layout.tsx" switcher
export const experimental_ppr = true

export default function Layout({ children }: { children: React.ReactNode }) {
  // ...
}
```

```jsx filename="/app/dashboard/layout.jsx" switcher
export const experimental_ppr = true

export default function Layout({ children }) {
  // ...
}
```

{/* md5:f826740945c4a46bab3c6ab9b1c7f0b2 */}
没有 `experimental_ppr` 的路由将默认为 `false`，并且不会使用 PPR 进行预渲染。您需要为每个路由显式选择加入 PPR。

{/* md5:439aa0ce7f8baf3a0f7f74ca3dbf1a5e */}
> **须知**：
>
> - `experimental_ppr` 将应用于路由段的所有子级，包括嵌套布局和页面。您不需要将其添加到每个文件，只需添加到路由的顶级段。
> - 要禁用子段的 PPR，您可以在子段中将 `experimental_ppr` 设置为 `false`。

{/* md5:e57da1656ead1413f55464d7577686d2 */}

### 动态 API

{/* md5:8de9eb02467ea945253fa20c868ffd91 */}
当使用需要查看传入请求的动态 API 时，Next.js 会为该路由选择动态渲染。要继续使用 PPR，请将组件用 Suspense 包裹。例如，`<User />` 组件是动态的，因为它使用了 `cookies` API：

```jsx filename="app/user.js" switcher
import { cookies } from 'next/headers'

export async function User() {
  const session = (await cookies()).get('session')?.value
  return '...'
}
```

```tsx filename="app/user.tsx" switcher
import { cookies } from 'next/headers'

export async function User() {
  const session = (await cookies()).get('session')?.value
  return '...'
}
```

{/* md5:78870a244a58254c37d2058a54bec96b */}
`<User />` 组件将被流式传输，而 `<Page />` 内部的任何其他内容将被预渲染并成为静态外壳的一部分。

```tsx filename="app/page.tsx" switcher
import { Suspense } from 'react'
import { User, AvatarSkeleton } from './user'

export const experimental_ppr = true

export default function Page() {
  return (
    <section>
      <h1>This will be prerendered</h1>
      <Suspense fallback={<AvatarSkeleton />}>
        <User />
      </Suspense>
    </section>
  )
}
```

```jsx filename="app/page.js" switcher
import { Suspense } from 'react'
import { User, AvatarSkeleton } from './user'

export default function Page() {
  return (

      <h1>This will be prerendered</h1>

      </Suspense>
    </section>

}
```

### 传递动态属性

{/* md5:f8e7cac83c9e5b96e78b570d2bc04918 */}
组件仅在访问值时才会选择动态渲染。例如，如果您从 `<Page />` 组件读取 `searchParams`，您可以将此值作为 prop 转发给另一个组件：

```tsx filename="app/page.tsx" switcher
import { Table, TableSkeleton } from './table'
import { Suspense } from 'react'

export default function Page({
  searchParams,
}: {
  searchParams: Promise<{ sort: string }>
}) {
  return (
    <section>
      <h1>This will be prerendered</h1>
      <Suspense fallback={<TableSkeleton />}>
        <Table searchParams={searchParams} />
      </Suspense>
    </section>
  )
}
```

```jsx filename="app/page.js" switcher
import { Table, TableSkeleton } from './table'
import { Suspense } from 'react'

export default function Page({ searchParams }) {
  return (
    <section>
      <h1>This will be prerendered</h1>
      <Suspense fallback={<TableSkeleton />}>
        <Table searchParams={searchParams} />
      </Suspense>
    </section>
  )
}
```

{/* md5:b96d24eeeaf83a3a9acd7cc759f7d25e */}
在表格组件内部，访问 `searchParams` 的值将使该组件变为动态组件，而页面的其余部分将被预渲染。

```tsx filename="app/table.tsx" switcher
export async function Table({
  searchParams,
}: {
  searchParams: Promise<{ sort: string }>
}) {
  const sort = (await searchParams).sort === 'true'
  return '...'
}
```
