Next.js 使用**基于文件系统的路由**，这意味着你可以使用文件夹和文件来定义路由。本页将指导你如何创建布局和页面，并在它们之间进行链接。

{/* md5:6cf9848b84ff2b270a901a3309315b13 */}
## 创建页面

{/* md5:add3d5efa3ae8951d982c840a836d786 */}

## 创建布局

{/* md5:dc24c2fd35cc939626dd36c6a3da5b62 */}
布局是在多个页面之间**共享**的 UI。在导航过程中，布局会保留状态、保持交互性，并且不会重新渲染。

{/* md5:a38b8d180692b7d0fc157d567b50fd21 */}
你可以通过从 [`layout` 文件](/docs/app/api-reference/file-conventions/layout)默认导出一个 React 组件来定义布局。该组件应接受一个 `children` prop，它可以是一个页面或另一个[布局](#嵌套布局)。

{/* md5:293878aab0dbe4ef889ec3db4652baba */}
例如，要创建一个接受你的索引页作为子级的布局，在 `app` 目录内添加一个 `layout` 文件：

{/* md5:30cf8e5ccb982f4a7ccc85a0e8ed6db5 */}
<Image
  alt="layout.js 特殊文件"
  srcLight="/docs/light/layout-special-file.png"
  srcDark="/docs/dark/layout-special-file.png"
  width="1600"
  height="363"
/>

```tsx filename="app/layout.tsx" switcher
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        {/* Layout UI */}
        {/* Place children where you want to render a page or nested layout */}
        <main>{children}</main>
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function DashboardLayout({ children }) {
  return (
    <html lang="en">
      <body>
        {/* Layout UI */}
        {/* Place children where you want to render a page or nested layout */}
        <main>{children}</main>
      </body>
    </html>
  )
}
```

{/* md5:25a6c1940450c4ed504a436295a514e7 */}
上面的布局被称为[根布局](/docs/app/api-reference/file-conventions/layout#root-layout)，因为它定义在 `app` 目录的根部。根布局是**必需的**，并且必须包含 `html` 和 `body` 标签。

{/* md5:ea6640672ebdc450b7e565b653d946e9 */}
## 创建嵌套路由

{/* md5:458e0d07bb0acb7df9657879a1593abf */}
嵌套路由是由多个 URL 段组成的路由。例如，`/blog/[slug]` 路由由三个段组成：

{/* md5:27b42d8694e9f890de5d77a3039a6427 */}
- `/` (根段)
- `blog` (段)
- `[slug]` (叶段)

{/* md5:02d03820278b2a9a238f8d367669a317 */}
在 Next.js 中：

{/* md5:e10b533d20f740c19aa34a7a65ccda64 */}
- **文件夹**用于定义映射到 URL 段的路由段。
- **文件**（如 `page` 和 `layout`）用于为段创建要显示的 UI。

{/* md5:71054635e7ec8137ed5a0dbfbf16f7e7 */}
要创建嵌套路由，你可以将文件夹相互嵌套。例如，要为 `/blog` 添加一个路由，在 `app` 目录中创建一个名为 `blog` 的文件夹。然后，为了使 `/blog` 可公开访问，添加一个 `page.tsx` 文件：

{/* md5:319b10c53543ff33992fa0fe61419d19 */}
<Image
  alt="显示 blog 文件夹和 page.js 文件的文件层级结构"
  srcLight="/docs/light/blog-nested-route.png"
  srcDark="/docs/dark/blog-nested-route.png"
  width="1600"
  height="525"
/>

```tsx filename="app/blog/page.tsx" switcher
// Dummy imports
import { getPosts } from '@/lib/posts'
import { Post } from '@/ui/post'

export default async function Page() {
  const posts = await getPosts()

  return (
    <ul>
      {posts.map((post) => (
        <Post key={post.id} post={post} />
      ))}
    </ul>
  )
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
// Dummy imports
import { getPosts } from '@/lib/posts'
import { Post } from '@/ui/post'

export default async function Page() {
  const posts = await getPosts()

  return (
    <ul>
      {posts.map((post) => (
        <Post key={post.id} post={post} />
      ))}
    </ul>
  )
}
```

## 嵌套布局

{/* md5:6e3d35b098e9ed04029a420bb829c779 */}
默认情况下，文件夹层级中的布局也会被嵌套，这意味着它们通过 `children` prop 包裹子布局。你可以通过在特定路由段（文件夹）内添加 `layout` 来嵌套布局。

{/* md5:3cc7d0bad017a6b6cda3eaa649b11f53 */}
例如，要为 `/blog` 路由创建一个布局，在 `blog` 文件夹内添加一个新的 `layout` 文件。

{/* md5:2077c02a602b5623fbecf849a4dc0086 */}
<Image
  alt="显示根布局包裹 blog 布局的文件层级结构"
  srcLight="/docs/light/nested-layouts.png"
  srcDark="/docs/dark/nested-layouts.png"
  width="1600"
  height="768"
/>

```tsx filename="app/blog/layout.tsx" switcher
export default function BlogLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return <section>{children}</section>
}
```

```jsx filename="app/blog/layout.js" switcher
export default function BlogLayout({ children }) {
  return <section>{children}</section>
}
```

{/* md5:ae1bec3851fce6eeeb22c692fc43e070 */}
如果你将上面的两个布局结合起来，根布局 (`app/layout.js`) 将包裹 blog 布局 (`app/blog/layout.js`)，而 blog 布局又会包裹博客页面 (`app/blog/page.js`) 和博客文章页面 (`app/blog/[slug]/page.js`)。

## 创建动态段

{/* md5:d58db62fa4eb6e912937273664b359f5 */}
[动态段](/docs/app/api-reference/file-conventions/dynamic-routes)允许你创建从数据生成的路由。例如，不是为每篇博客文章手动创建路由，你可以创建一个动态段来根据博客文章数据生成路由。

{/* md5:9e586b5d2b346cd317f51402049ac4ad */}
要创建一个动态段，将段（文件夹）名称用方括号包裹：`[segmentName]`。例如，在 `app/blog/[slug]/page.tsx` 路由中，`[slug]` 就是动态段。

```tsx filename="app/blog/[slug]/page.tsx" switcher
export default async function BlogPostPage({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  const post = await getPost(slug)

  return (
    <div>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </div>
  )
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
export default async function BlogPostPage({ params }) {
  const { slug } = await params
  const post = await getPost(slug)

  return (
    <div>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </div>
  )
}
```

{/* md5:45fad1f105043878f4a8a6da10fe1f82 */}
了解更多关于 [动态段](/docs/app/api-reference/file-conventions/dynamic-routes) 和 [`params`](/docs/app/api-reference/file-conventions/page#params-optional) 属性的信息。

{/* md5:8d12d1d34ff9dff6bfc078a261db3efd */}
嵌套在 [动态段中的布局](/docs/app/api-reference/file-conventions/layout#params-optional) 也可以访问 `params` 属性。

## 使用搜索参数渲染

{/* md5:5e1f4beb2b750122d0274fecc1a9b76d */}
在服务器组件的 **页面** 中，你可以使用 [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) 属性访问搜索参数：

```tsx filename="app/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
}
```

```jsx filename="app/page.jsx" switcher
export default async function Page({ searchParams }) {
  const filters = (await searchParams).filters
}
```

{/* md5:044f31a46d3c30b002926f24c4a93c79 */}
使用 `searchParams` 使你的页面进入 [**动态渲染**](/docs/app/getting-started/partial-prerendering#dynamic-rendering)，因为它需要一个传入请求来读取搜索参数。

{/* md5:a407e532c57cd832315ed3cf933b6e50 */}
客户端组件可以使用 [`useSearchParams`](/docs/app/api-reference/functions/use-search-params) 钩子读取搜索参数。

{/* md5:ed25767298972f0922009f8e91e3f14c */}
在 [静态渲染](/docs/app/api-reference/functions/use-search-params#static-rendering) 和 [动态渲染](/docs/app/api-reference/functions/use-search-params#dynamic-rendering) 路由中了解更多关于 `useSearchParams` 的信息。

{/* md5:7fa8875fad4cc35a40d577ea790f444e */}
### 何时使用什么

{/* md5:b4165c2f36d13a2bbbd442857023374f */}
- 当你需要搜索参数来**加载页面数据**时（例如分页、从数据库过滤），请使用 `searchParams` 属性。
- 当搜索参数**仅在客户端**使用时（例如过滤已通过 props 加载的列表），请使用 `useSearchParams`。
- 为了提高性能，你可以在 **回调或事件处理程序** 中使用 `new URLSearchParams(window.location.search)` 来读取搜索参数，而不会触发重新渲染。

{/* md5:8e8167e45c1de1fa7813a9f12c6ba879 */}
## 页面之间的链接

{/* md5:109257e02291f08e65b580e7c0cdeed0 */}
你可以使用 [`<Link>` 组件](/docs/app/api-reference/components/link) 在路由之间导航。`<Link>` 是一个内置的 Next.js 组件，它扩展了 HTML `<a>` 标签以提供[预取](/docs/app/getting-started/linking-and-navigating#prefetching)和[客户端导航](/docs/app/getting-started/linking-and-navigating#client-side-transitions)功能。

{/* md5:8a3bb4e17fc2ef7287ef720c0962df03 */}
例如，要生成一个博客文章列表，从 `next/link` 导入 `<Link>` 并向组件传递一个 `href` prop：

```tsx filename="app/ui/post.tsx" highlight={1,10} switcher
import Link from 'next/link'

export default async function Post({ post }) {
  const posts = await getPosts()

  return (
    <ul>
      {posts.map((post) => (
        <li key={post.slug}>
          <Link href={`/blog/${post.slug}`}>{post.title}</Link>
        </li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/ui/post.js" highlight={1,10}  switcher
import Link from 'next/link'

export default async function Post({ post }) {
  const posts = await getPosts()

  return (
    <ul>
      {posts.map((post) => (
        <li key={post.slug}>
          <Link href={`/blog/${post.slug}`}>{post.title}</Link>
        </li>
      ))}
    </ul>
  )
}
```

{/* md5:2baf8828e2de35aeb120424d8315fe8f */}
> **提示**：`<Link>` 是在 Next.js 中路由间导航的主要方式。你也可以使用 [`useRouter` Hook](/docs/app/api-reference/functions/use-router) 进行更高级的导航。

{/* md5:7b711d1d61611567b5b973a754ca2f68 */}
## 路由 Props 辅助工具

{/* md5:4ee00a40c599b8f5d0b0576da6d274f1 */}
Next.js 提供了实用程序类型，可以根据你的路由结构推断 `params` 和命名插槽：

{/* md5:9f1e076ad7f6665d4e573ae1a4c8efd0 */}
- [**PageProps**](/docs/app/api-reference/file-conventions/page#page-props-helper)：`page` 组件的属性，包括 `params` 和 `searchParams`。
- [**LayoutProps**](/docs/app/api-reference/file-conventions/layout#layout-props-helper)：`layout` 组件的属性，包括 `children` 和任何命名插槽（例如 `@analytics` 这样的文件夹）。

{/* md5:09373984032462848c7dc1deeb724840 */}
这些是全局可用的辅助工具，在运行 `next dev`、`next build` 或 [`next typegen`](/docs/app/api-reference/cli/next#next-typegen-options) 时生成。

```tsx filename="app/blog/[slug]/page.tsx"
export default async function Page(props: PageProps<'/blog/[slug]'>) {
  const { slug } = await props.params
  return <h1>Blog post: {slug}</h1>
}
```

```tsx filename="app/dashboard/layout.tsx"
export default function Layout(props: LayoutProps<'/dashboard'>) {
  return (
    <section>
      {props.children}
      {/* If you have app/dashboard/@analytics, it appears as a typed slot: */}
      {/* {props.analytics} */}
    </section>
  )
}
```

{/* md5:0f24b500a86316778fbb7492be6436d8 */}
> **小贴士**
>
> - 静态路由将 `params` 解析为 `{}`。
> - `PageProps`, `LayoutProps` 是全局辅助工具 — 无需导入。
> - 类型是在 `next dev`, `next build` 或 `next typegen` 期间生成的。
