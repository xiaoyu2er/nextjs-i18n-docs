## 获取数据

{/* md5:83ee120aaa5af43f777ca789ca049fee */}
### 服务器组件

{/* md5:0a3079dfd1e8956927937fe42cd9a7d0 */}
你可以使用以下方式在服务器组件中获取数据：

{/* md5:3de18b19ae1f8029784d2f1a10758be3 */}
1.  [`fetch` API](#with-the-fetch-api)
2.  [ORM 或数据库](#with-an-orm-or-database)

{/* md5:d193e2e1620bfb486f861f169f4c8eee */}
#### 使用 `fetch` API

{/* md5:92a1b8af026728eaea3d6acdab39ea97 */}
要使用 `fetch` API 获取数据，请将你的组件转换为异步函数，并等待 `fetch` 调用。例如：

```tsx filename="app/blog/page.tsx" switcher
export default async function Page() {
  const data = await fetch('https://api.vercel.app/blog')
  const posts = await data.json()
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/blog/page.js" switcher
export default async function Page() {
  const data = await fetch('https://api.vercel.app/blog')
  const posts = await data.json()
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

{/* md5:d4d00d901440777f252f74808242f907 */}
> **请注意：**
>
> - 默认情况下，`fetch` 响应不会被缓存。但是，Next.js 会 [预渲染](/docs/app/getting-started/partial-prerendering#static-rendering) 路由，并且输出将被缓存以提高性能。如果你想选择 [动态渲染](/docs/app/getting-started/partial-prerendering#dynamic-rendering)，请使用 `{ cache: 'no-store' }` 选项。请参阅 [`fetch` API 参考](/docs/app/api-reference/functions/fetch)。
> - 在开发过程中，你可以记录 `fetch` 调用以获得更好的可见性和调试能力。请参阅 [`logging` API 参考](/docs/app/api-reference/config/next-config-js/logging)。

{/* md5:4958e75141aa786d51ae5c99cee57158 */}
#### 使用 ORM 或数据库

{/* md5:75ceacf371658785e6bab57a1dfa9995 */}
由于服务器组件在服务器上渲染，你可以安全地使用 ORM 或数据库客户端进行数据库查询。将你的组件转换为异步函数，并等待调用：

```tsx filename="app/blog/page.tsx" switcher
import { db, posts } from '@/lib/db'

export default async function Page() {
  const allPosts = await db.select().from(posts)
  return (
    <ul>
      {allPosts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/blog/page.js" switcher
import { db, posts } from '@/lib/db'

export default async function Page() {
  const allPosts = await db.select().from(posts)
  return (
    <ul>
      {allPosts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

### 客户端组件

{/* md5:f83415be6e85feb143af410b1fcd82ec */}
有两种方式可以在客户端组件中获取数据：

{/* md5:93f3f8f50bad27d8a4950c4b782a3ed1 */}
1.  使用 React 的 [`use` hook](https://react.dev/reference/react/use)
2.  使用社区库，如 [SWR](https://swr.vercel.app/) 或 [React Query](https://tanstack.com/query/latest)

{/* md5:89245ed5556a696e0651c50e99ab92e4 */}
#### 使用 `use` Hook 流式传输数据

{/* md5:5e819ecd33d6ac624ed4940a28c12490 */}
你可以使用 React 的 [`use` hook](https://react.dev/reference/react/use) 将数据从服务器 [流式传输](#streaming) 到客户端。首先在你的服务器组件中获取数据，然后将 promise 作为 prop 传递给你的客户端组件：

```tsx filename="app/blog/page.tsx" switcher
import Posts from '@/app/ui/posts'
import { Suspense } from 'react'

export default function Page() {
  // Don't await the data fetching function
  const posts = getPosts()

  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Posts posts={posts} />
    </Suspense>
  )
}
```

```jsx filename="app/blog/page.js" switcher
import Posts from '@/app/ui/posts'
import { Suspense } from 'react'

export default function Page() {
  // Don't await the data fetching function
  const posts = getPosts()

  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Posts posts={posts} />
    </Suspense>
  )
}
```

{/* md5:8df65800b529b8d278d827b65af776ac */}
然后，在你的客户端组件中，使用 `use` hook 来读取 promise：

```tsx filename="app/ui/posts.tsx" switcher
'use client'
import { use } from 'react'

export default function Posts({
  posts,
}: {
  posts: Promise<{ id: string; title: string }[]>
}) {
  const allPosts = use(posts)

  return (
    <ul>
      {allPosts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/ui/posts.js" switcher
'use client'
import { use } from 'react'

export default function Posts({ posts }) {
  const allPosts = use(posts)

  return (
    <ul>
      {allPosts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

{/* md5:de74c792662b34966dd1a8b8c76a2fa2 */}
在上面的例子中，`<Posts>` 组件被包裹在 [`<Suspense>` 边界](https://react.dev/reference/react/Suspense) 内。这意味着在 promise 被解析时，将显示 fallback 内容。了解更多关于 [流式传输](#streaming) 的信息。

{/* md5:5ffb6e628430ae7984c4197bfc50fc4d */}
#### 社区库

{/* md5:e523116be6a70a60678149f9438de07e */}
你可以使用像 [SWR](https://swr.vercel.app/) 或 [React Query](https://tanstack.com/query/latest) 这样的社区库在客户端组件中获取数据。这些库有自己的缓存、流式传输和其他功能的语义。例如，使用 SWR：

```tsx filename="app/blog/page.tsx" switcher
'use client'
import useSWR from 'swr'

const fetcher = (url) => fetch(url).then((r) => r.json())

export default function BlogPage() {
  const { data, error, isLoading } = useSWR(
    'https://api.vercel.app/blog',
    fetcher
  )

  if (isLoading) return <div>Loading...</div>
  if (error) return <div>Error: {error.message}</div>

  return (
    <ul>
      {data.map((post: { id: string; title: string }) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/blog/page.js" switcher
'use client'

import useSWR from 'swr'

const fetcher = (url) => fetch(url).then((r) => r.json())

export default function BlogPage() {
  const { data, error, isLoading } = useSWR(

    fetcher
  )

  if (isLoading) return <div>Loading...</div>
  if (error) return <div>Error: {error.message}</div>

  return (
    <ul>
      {data.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

## 去重请求和缓存数据

{/* md5:8d397107398a5b94ce53f2240af0fdbb */}
一种对 `fetch` 请求去重的方式是使用 [请求 Memoization](/docs/app/guides/caching#request-memoization)。使用此机制，在单次渲染过程中使用 `GET` 或 `HEAD` 且 URL 和选项相同的 `fetch` 调用会被合并为一个请求。这是自动发生的，你可以通过向 `fetch` 传递 Abort 信号来 [选择退出](/docs/app/guides/caching#opting-out)。

{/* md5:5f0ce19c1315944c8406568dd1023447 */}
请求 Memoization 的范围仅限于单个请求的生命周期。

{/* md5:cf307fa8c43d1e4863a87f6ba8bf004f */}
你也可以通过使用 Next.js 的 [数据缓存](/docs/app/guides/caching#data-cache) 对 `fetch` 请求进行去重，例如在 `fetch` 选项中设置 `cache: 'force-cache'`。

{/* md5:f6f351e0e15fe2ebe1219af17f0c97b8 */}
数据缓存允许在当前渲染过程和传入请求之间共享数据。

{/* md5:313348c9a2de4a2e394bc43468ff8451 */}
如果你不_使用_ `fetch`，而是直接使用 ORM 或数据库，你可以使用 [React `cache`](https://react.dev/reference/react/cache) 函数包装你的数据访问。

```tsx filename="app/lib/data.ts" switcher
import { cache } from 'react'
import { db, posts, eq } from '@/lib/db'

export const getPost = cache(async (id: string) => {
  const post = await db.query.posts.findFirst({
    where: eq(posts.id, parseInt(id)),
  })
})
```

```jsx filename="app/lib/data.js" switcher
import { cache } from 'react'
import { db, posts, eq } from '@/lib/db'
import { notFound } from 'next/navigation'

export const getPost = cache(async (id) => {
  const post = await db.query.posts.findFirst({
    where: eq(posts.id, parseInt(id)),
  })
})
```

## 流式传输

{/* md5:12620378df3007368dce78a4ca0db1cf */}
> **警告：** 下面的内容假设已在你的应用程序中启用 [`cacheComponents` 配置选项](/docs/app/api-reference/config/next-config-js/cacheComponents)。该标志是在 Next.js 15 canary 中引入的。

{/* md5:6a4331fa88fc422ddabd3537e13ab170 */}
当在服务器组件中使用 `async/await` 时，Next.js 将选择 [动态渲染](/docs/app/getting-started/partial-prerendering#dynamic-rendering)。这意味着对于每个用户请求，数据都会在服务器上获取和渲染。如果有任何慢速数据请求，整个路由将被阻止渲染。

{/* md5:b44845fc49629df0977b349ad7d4fd34 */}
为了改善初始加载时间和用户体验，你可以使用流式传输将页面的 HTML 分解为更小的块，并逐步将这些块从服务器发送到客户端。

### 使用 `<Suspense>`

{/* md5:19e4f25c881c81ae532157c1970e2e63 */}
`<Suspense>` 允许你更精细地控制页面的哪些部分进行流式传输。例如，你可以立即显示 `<Suspense>` 边界之外的任何页面内容，并在边界内流式传输博客文章列表。

```tsx filename="app/blog/page.tsx" switcher
import { Suspense } from 'react'
import BlogList from '@/components/BlogList'
import BlogListSkeleton from '@/components/BlogListSkeleton'

export default function BlogPage() {
  return (
    <div>
      {/* This content will be sent to the client immediately */}
      <header>
        <h1>Welcome to the Blog</h1>
        <p>Read the latest posts below.</p>
      </header>
      <main>
        {/* Any content wrapped in a <Suspense> boundary will be streamed */}
        <Suspense fallback={<BlogListSkeleton />}>
          <BlogList />
        </Suspense>
      </main>
    </div>
  )
}
```


### 创建有意义的加载状态

{/* md5:e74d44c2456c80bd9b813227840a2549 */}

{/* md5:c6c0e7beb33d014b366de37737efb47f */}

在某些情况下，你可能希望使用这种模式，因为一个获取操作依赖于另一个操作的结果。

{/* md5:0067c9f40f457c416ed2f855d4745a19 */}

```tsx filename="app/artist/[username]/page.tsx" switcher
export default async function Page({

}: {
  params: Promise<{ username: string }>
}) {
  const { username } = await params
  // Get artist information
  const artist = await getArtist(username)

  return (
    <>
      <h1>{artist.name}</h1>
      {/* Show fallback UI while the Playlists component is loading */}
      <Suspense fallback={<div>Loading...</div>}>
        {/* Pass the artist ID to the Playlists component */}
        <Playlists artistID={artist.id} />
      </Suspense>
    </>
  )
}

async function Playlists({ artistID }: { artistID: string }) {
  // Use the artist ID to fetch playlists
  const playlists = await getArtistPlaylists(artistID)

  return (
    <ul>
      {playlists.map((playlist) => (
        <li key={playlist.id}>{playlist.name}</li>
      ))}
    </ul>
  )
}
```

### 并行数据获取

{/* md5:10068c6a3610fa54799213768dc73a93 */}
当路由中的数据请求被急切地发起并同时开始时，就会发生并行数据获取。

{/* md5:e6cb65ca61a808febfd1efa13d69165a */}
默认情况下，[布局和页面](/docs/app/getting-started/layouts-and-pages) 是并行渲染的。因此，每个段都会尽快开始获取数据。

{/* md5:347c86dc26b322a6057da9e0f0680549 */}
然而，在_任何_组件内，如果多个 `async`/`await` 请求一个接一个放置，它们可能仍然是顺序的。例如，`getAlbums` 将被阻塞，直到 `getArtist` 被解析：

```tsx filename="app/artist/[username]/page.tsx" switcher
import { getArtist, getAlbums } from '@/app/lib/data'

export default async function Page({ params }) {
  // These requests will be sequential
  const { username } = await params
  const artist = await getArtist(username)
  const albums = await getAlbums(username)
  return <div>{artist.name}</div>
}
```

{/* md5:2ec7a28b86804d19870da3080a233da1 */}
你可以通过在使用数据的组件外部定义请求，并将它们一起解析（例如使用 [`Promise.all`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise/all)）来并行发起请求：

```tsx filename="app/artist/[username]/page.tsx" highlight={3,8,23} switcher
import Albums from './albums'

async function getArtist(username: string) {
  const res = await fetch(`https://api.example.com/artist/${username}`)
  return res.json()
}

async function getAlbums(username: string) {
  const res = await fetch(`https://api.example.com/artist/${username}/albums`)
  return res.json()
}

export default async function Page({
  params,
}: {
  params: Promise<{ username: string }>
}) {
  const { username } = await params
  const artistData = getArtist(username)
  const albumsData = getAlbums(username)

  // Initiate both requests in parallel
  const [artist, albums] = await Promise.all([artistData, albumsData])

  return (
```

### 预加载数据

{/* md5:38ab9bde6f5cc1a9399ba70c8cf584e4 */}
你可以通过创建一个工具函数来预加载数据，你可以在阻塞请求之上急切地调用它。`<Item>` 基于 `checkIsAvailable()` 函数有条件地渲染。

{/* md5:db9efa857215887d5dcc3cc9ee1f07e0 */}
你可以在 `checkIsAvailable()` 之前调用 `preload()` 来急切地发起 `<Item/>` 的数据依赖。到 `<Item/>` 被渲染时，它的数据已经被获取了。

```tsx filename="app/item/[id]/page.tsx" switcher
import { getItem, checkIsAvailable } from '@/lib/data'

export default async function Page({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  // starting loading item data
  preload(id)
  // perform another asynchronous task
  const isAvailable = await checkIsAvailable()

  return isAvailable ? <Item id={id} /> : null
}

export const preload = (id: string) => {
  // void evaluates the given expression and returns undefined
  // https://developer.mozilla.org/docs/Web/JavaScript/Reference/Operators/void
  void getItem(id)
}
export async function Item({ id }: { id: string }) {
  const result = await getItem(id)
  // ...
}
```
