{/* md5:068c9405821810ccc150cca18a639687 */}
默认情况下，布局和页面是[服务器组件](https://react.dev/reference/rsc/server-components)，这允许您在服务器上获取数据并渲染部分 UI，可选地缓存结果，然后将其流式传输到客户端。当您需要交互性或浏览器 API 时，可以使用[客户端组件](https://react.dev/reference/rsc/use-client)来添加功能。

{/* md5:1ff0ef42818847fad112b0661321c628 */}
本页面解释了 Next.js 中服务器组件和客户端组件的工作原理及其使用场景，并提供了如何在您的应用程序中组合使用的示例。

{/* md5:31578e8d9eeb281821b63735fe8b33a0 */}
## 何时使用服务器组件和客户端组件？

{/* md5:e5147225738eccf3dadc8a633f55e37f */}
客户端和服务器环境具有不同的能力。服务器组件和客户端组件允许您根据用例在各个环境中运行逻辑。

{/* md5:ae0d6e40f51d0f8a1c3b5d09d6452f3e */}
当您需要以下功能时，请使用**客户端组件**：

{/* md5:5fb5389e7dc2a7e3e31e8820acd598e5 */}
- [状态](https://react.dev/learn/managing-state)和[事件处理器](https://react.dev/learn/responding-to-events)。例如 `onClick`、`onChange`。
- [生命周期逻辑](https://react.dev/learn/lifecycle-of-reactive-effects)。例如 `useEffect`。
- 仅限浏览器的 API。例如 `localStorage`、`window`、`Navigator.geolocation` 等。
- [自定义 Hook](https://react.dev/learn/reusing-logic-with-custom-hooks)。

{/* md5:feca068bd4cb25a5dda71aad0e16cd78 */}
当您需要以下功能时，请使用**服务器组件**：

{/* md5:ab66e84e73f84b1fb4a69ac463de80c9 */}
- 从靠近数据源的数据库或 API 获取数据。
- 使用 API 密钥、令牌和其他密钥，而无需将其暴露给客户端。
- 减少发送到浏览器的 JavaScript 代码量。
- 改善[首次内容绘制 (FCP)](https://web.dev/fcp/)，并逐步将内容流式传输到客户端。

{/* md5:e0c1ce3a1123c88557f177345993c36c */}
例如，`<Page>` 组件是一个服务器组件，它获取关于帖子的数据，并将其作为 props 传递给处理客户端交互的 `<LikeButton>`。

```tsx filename="app/[id]/page.tsx" highlight={1,12} switcher
import LikeButton from '@/app/ui/like-button'
import { getPost } from '@/lib/data'

export default async function Page({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const post = await getPost(id)

  return (
    <div>
      <main>
        <h1>{post.title}</h1>
        {/* ... */}
        <LikeButton likes={post.likes} />
      </main>
    </div>
  )
}
```

```jsx filename="app/[id]/page.js" highlight={1,12} switcher
import LikeButton from '@/app/ui/like-button'

        <h1>{post.title}</h1>
        {/* ... */}
        <LikeButton likes={post.likes} />
      </main>
    </div>
  )
}
```

```tsx filename="app/ui/like-button.tsx" highlight={1} switcher
  // ...
}
```

## Next.js 中的服务器组件和客户端组件如何工作？

{/* md5:5248dbeb2d38c589b3156e65c9775e0c */}
### 在服务器上

{/* md5:fc2a197ea4ddbfe55170a9ad112b03d1 */}
在服务器上，Next.js 使用 React 的 API 来编排渲染。渲染工作按单个路由段（[布局和页面](/docs/app/getting-started/layouts-and-pages)）被拆分为多个块：

{/* md5:82eef625f5f39305201d0bc363d13f1f */}
- **服务器组件**被渲染成一种称为 React 服务器组件负载（RSC Payload）的特殊数据格式。
- **客户端组件**和 RSC 负载被用于[预渲染](/docs/app/getting-started/partial-prerendering#how-does-partial-prerendering-work) HTML。

{/* md5:fbbbbf4d991b93832979bca835d4dc89 */}
> **什么是 React 服务器组件负载 (RSC)？**
>
> RSC 负载是已渲染的 React 服务器组件树的紧凑二进制表示。它被客户端上的 React 用来更新浏览器的 DOM。RSC 负载包含：
>
> - 服务器组件的渲染结果
> - 应在何处渲染客户端组件的占位符及其 JavaScript 文件的引用
> - 从服务器组件传递给客户端组件的任何 props

{/* md5:eb452800c0cdd415ab52552374b97743 */}
### 在客户端上（首次加载）

{/* md5:0cc5560cb1881c3c4f104ff1ecff204c */}
然后，在客户端上：

{/* md5:b0a4b98fc3ecbe4592731309fd29377a */}
1. **HTML** 用于立即向用户显示路由的快速非交互式预览。
2. **RSC 负载**用于协调客户端和服务器组件树。
3. **JavaScript** 用于水合（hydrate）客户端组件并使应用程序具有交互性。

{/* md5:7114bcc67f55bc86807a58b98d7630e9 */}
> **什么是水合（hydration）？**
>
> 水合是 React 将[事件处理器](https://react.dev/learn/responding-to-events)附加到 DOM 以使静态 HTML 具有交互性的过程。

{/* md5:b84ab8e986becc52a57bdc14a719cc29 */}
### 后续导航

{/* md5:1ebe8a122dbb7475d6e5a6b9dfae2729 */}
在后续导航时：

{/* md5:4a75718f318e94686d9a4b4246ddb917 */}
- **RSC 负载**会被预取并缓存，以实现即时导航。
- **客户端组件**完全在客户端渲染，无需服务器渲染的 HTML。

{/* md5:e57da1656ead1413f55464d7577686d2 */}
## 示例

### 使用客户端组件

{/* md5:bb785fb53bb016500b7ecab2dd5d1089 */}
您可以通过在文件顶部、导入语句上方添加 [`"use client"`](https://react.dev/reference/react/use-client) 指令来创建客户端组件。

```tsx filename="app/ui/counter.tsx" highlight={1} switcher
'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>{count} likes</p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  )
}
```

```jsx filename="app/ui/counter.tsx" highlight={1} switcher
'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>{count} likes</p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  )
}
```

{/* md5:f399b8aaf34eaefbab092b52a313de19 */}
`"use client"` 用于在服务器模块图（树）和客户端模块图之间声明一个**边界**。

{/* md5:942e229d78a128ed545be4b1c76ce880 */}
一旦一个文件被标记为 `"use client"`，**它的所有导入和子组件都被视为客户端包的一部分**。这意味着您不需要为每个旨在客户端使用的组件都添加该指令。

{/* md5:b677ae13fd5d747043c5ad23d2daf66a */}
### 减少 JS 包大小

{/* md5:1426eb4c059a876997b13b543ade49b4 */}
为了减少客户端 JavaScript 包的大小，请将 `'use client'` 添加到特定的交互式组件，而不是将大部分 UI 标记为客户端组件。

{/* md5:d030bcb385611a98637b5e7f1ad5d777 */}
例如，`<Layout>` 组件主要包含静态元素（如 Logo 和导航链接），但也包含一个交互式搜索栏。`<Search />` 是交互式的，需要是一个客户端组件，但布局的其余部分可以保持为服务器组件。

```tsx filename="app/layout.tsx" highlight={12} switcher
// Client Component
import Search from './search'
// Server Component
import Logo from './logo'

// Layout is a Server Component by default
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <>
      <nav>
        <Logo />
        <Search />
      </nav>
      <main>{children}</main>
    </>
  )
}
```

```jsx filename="app/layout.js" highlight={12} switcher
// Client Component
import Search from './search'
// Server Component
import Logo from './logo'

// Layout is a Server Component by default
export default function Layout({ children }) {
  return (
    <>
      <nav>
        <Logo />
        <Search />
      </nav>
      <main>{children}</main>
    </>
  )
}
```

```tsx filename="app/ui/search.tsx" highlight={1} switcher
'use client'

export default function Search() {
  // ...
}
```

```jsx filename="app/ui/search.js" highlight={1} switcher
'use client'

export default function Search() {
  // ...
}
```

### 交错使用服务器和客户端组件

{/* md5:c450392d22540b07c1e889e67ddd9867 */}
您可以将服务器组件作为 prop 传递给客户端组件。这允许您在视觉上将服务器渲染的 UI 嵌套在客户端组件内部。

{/* md5:7f981270761acf037d103e02dc7e2736 */}
一个常见的模式是使用 `children` 在 `<ClientComponent>` 中创建一个*插槽*。例如，在使用客户端状态切换可见性的 `<Modal>` 组件内部，放置一个在服务器上获取数据的 `<Cart>` 组件。

```tsx filename="app/ui/modal.tsx" switcher
'use client'

export default function Modal({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>
}
```

```jsx filename="app/ui/modal.js" switcher
'use client'

export default function Modal({ children }) {
  return <div>{children}</div>
}
```

{/* md5:b7c60d06c09712fba4ec0eaf17949df7 */}
然后，在父级服务器组件（例如 `<Page>`）中，您可以将 `<Cart>` 作为 `<Modal>` 的子组件传递：

```tsx filename="app/page.tsx"  highlight={7} switcher
import Modal from './ui/modal'
import Cart from './ui/cart'

export default function Page() {
  return (
    <Modal>
      <Cart />
    </Modal>
  )
}
```

```jsx filename="app/page.js" highlight={7} switcher
import Modal from './ui/modal'
import Cart from './ui/cart'

export default function Page() {
  return (
    <Modal>
      <Cart />
    </Modal>
  )
}
```

{/* md5:135c2e7fbc39b8b7ffb26bc8b3706b52 */}
在这种模式下，所有服务器组件都将提前在服务器上渲染，包括那些作为 prop 传递的组件。生成的 RSC 负载将包含应在组件树中何处渲染客户端组件的引用。

### 上下文提供程序

{/* md5:b884255224d983831c11b880e15a1141 */}
[React 上下文](https://react.dev/learn/passing-data-deeply-with-context)通常用于共享全局状态，如当前主题。但是，React 上下文在服务器组件中不受支持。

{/* md5:fd61dda73b4849270069b64692a174c8 */}
要使用上下文，请创建一个接受 `children` 的客户端组件：

```tsx filename="app/theme-provider.tsx" switcher
'use client'

import { createContext } from 'react'

export const ThemeContext = createContext({})

export default function ThemeProvider({
  children,
}: {
  children: React.ReactNode
}) {
  return <ThemeContext.Provider value="dark">{children}</ThemeContext.Provider>
}
```

```jsx filename="app/theme-provider.js" switcher
'use client'

import { createContext } from 'react'

export const ThemeContext = createContext({})

  return <ThemeContext.Provider value="dark">{children}</ThemeContext.Provider>
}
```

{/* md5:67c69af7a26b50a8234dabbac9115fd7 */}
然后，将其导入到一个服务器组件中（例如 `layout`）：

```tsx filename="app/layout.tsx" switcher
import ThemeProvider from './theme-provider'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (

      <body>

    </html>

```

```jsx filename="app/layout.js" switcher

```

现在，您的服务器组件将能够直接渲染您的提供者，并且您应用程序中的所有其他客户端组件都将能够消费此上下文。

{/* md5:01655a395959d91416a9bd4a3b3d4f8d */}
> **请注意**：您应该尽可能在树的深处渲染提供者——注意 `ThemeProvider` 只包裹了 `{children}`，而不是整个 `<html>` 文档。这使得 Next.js 更容易优化服务器组件的静态部分。

### 第三方组件

{/* md5:ed9ceb9a94b892ba6efc8f1c87cda533 */}
当使用依赖于仅限客户端功能的第三方组件时，您可以将其包装在客户端组件中以确保其按预期工作。

{/* md5:31e7af6bdadcfca84a6a184db215499c */}
例如，`<Carousel />` 可以从 `acme-carousel` 包导入。该组件使用了 `useState`，但它尚未包含 `"use client"` 指令。

{/* md5:39a11c258baf61f20c253e5b267459ed */}
如果您在客户端组件中使用 `<Carousel />`，它将按预期工作：

```tsx filename="app/gallery.tsx" switcher
'use client'

import { useState } from 'react'
import { Carousel } from 'acme-carousel'

export default function Gallery() {
  const [isOpen, setIsOpen] = useState(false)

  return (
    <div>
      <button onClick={() => setIsOpen(true)}>View pictures</button>
      {/* Works, since Carousel is used within a Client Component */}
      {isOpen && <Carousel />}
    </div>
  )
}
```

```jsx filename="app/gallery.js" switcher
'use client'

import { useState } from 'react'
import { Carousel } from 'acme-carousel'

export default function Gallery() {
  const [isOpen, setIsOpen] = useState(false)

  return (
    <div>
      <button onClick={() => setIsOpen(true)}>View pictures</button>
      {/*  Works, since Carousel is used within a Client Component */}
      {isOpen && <Carousel />}
    </div>
  )
}
```

{/* md5:d7f7f8d223bf1581565006a3bd776ef5 */}
但是，如果您尝试直接在服务器组件中使用它，您将看到错误。这是因为 Next.js 不知道 `<Carousel />` 使用了仅限客户端的功能。

{/* md5:3d39b196ac4feb635945ceb1f0884741 */}
为了解决这个问题，您可以将依赖于仅限客户端功能的第三方组件包装在您自己的客户端组件中：

```tsx filename="app/carousel.tsx" switcher
'use client'

import { Carousel } from 'acme-carousel'

export default Carousel
```

```jsx filename="app/carousel.js" switcher
'use client'

export default Carousel
```

{/* md5:f1b12b9ed2d33430c5db33af660d70b1 */}
现在，您可以直接在服务器组件中使用 `<Carousel />`：

```tsx filename="app/page.tsx" switcher

  return (
    <div>
      <p>View pictures</p>

    </div>
  )
}
```

```jsx filename="app/page.js" switcher

  return (

      <p>View pictures</p>
      {/*  Works, since Carousel is a Client Component */}

```

{/* md5:31e301031bfd8ebc6984d19ba5f175ea */}
> **给库作者的建议**
>
> 如果您正在构建组件库，请将 `"use client"` 指令添加到依赖于仅限客户端功能的入口点。这允许您的用户将组件导入服务器组件，而无需创建包装器。
>
> 值得注意的是，一些打包器可能会去除 `"use client"` 指令。您可以在 [React Wrap Balancer](https://github.com/shuding/react-wrap-balancer/blob/main/tsup.config.ts#L10-L13) 和 [Vercel Analytics](https://github.com/vercel/analytics/blob/main/packages/web/tsup.config.js#L26-L30) 仓库中找到如何配置 esbuild 以包含 `"use client"` 指令的示例。

### 防止环境污染

{/* md5:cb0d6a20918720e5dada707daace051c */}
JavaScript 模块可以在服务器组件和客户端组件模块之间共享。这意味着有可能意外地将仅限服务器的代码导入客户端。例如，考虑以下函数：

```ts filename="lib/data.ts" switcher
export async function getData() {
  const res = await fetch('https://external-service.com/data', {
    headers: {
      authorization: process.env.API_KEY,
    },
  })

  return res.json()
}
```

```js filename="lib/data.js" switcher
export async function getData() {
  const res = await fetch('https://external-service.com/data', {
    headers: {
      authorization: process.env.API_KEY,
    },
  })

  return res.json()
}
```

{/* md5:e3b892a4ed2a2f7e56d845f3bdf1a774 */}
此函数包含一个绝不应暴露给客户端的 `API_KEY`。

{/* md5:39e158ec34fa26f95d49728e94eeae37 */}
在 Next.js 中，只有以 `NEXT_PUBLIC_` 为前缀的环境变量才会包含在客户端包中。如果变量没有前缀，Next.js 会将其替换为空字符串。

{/* md5:91a2ee3d32a713a91cd1444df1605d94 */}
因此，即使 `getData()` 可以被导入并在客户端上执行，它也不会按预期工作。

{/* md5:87937bbf2eac00068268f491d931879c */}
为了防止在客户端组件中意外使用，您可以使用 [`server-only` 包](https://www.npmjs.com/package/server-only)。

{/* md5:380b3377f6091567add13fbb1aa4e2ba */}
然后，将包导入包含仅限服务器代码的文件：

```js filename="lib/data.js"
import 'server-only'

export async function getData() {
  const res = await fetch('https://external-service.com/data', {
    headers: {
```
