{/* md5:d79825b0c38e985dcd6420b0652629bd */}
在 Next.js 中，路由默认在服务器上渲染。这通常意味着客户端必须等待服务器响应才能显示新路由。Next.js 内置了 [预取]、[流式传输] 和 [客户端转换](#client-side-transitions)，确保导航保持快速和响应灵敏。

{/* md5:8a1b98b32e8428c30e08ae33d906373c */}
本指南解释了 Next.js 中导航的工作原理，以及如何针对[动态路由](#dynamic-routes-without-loadingtsx)和[慢速网络](#slow-networks)进行优化。

{/* md5:abc175caedce7e9e8a42c91359598c4c */}
## 导航工作原理

{/* md5:0e015354845b322765f3ef6692a04faf */}
要理解 Next.js 中导航的工作原理，了解以下概念会有所帮助：

{/* md5:0d093453405521d7c86fae76c96c69db */}
- [服务器端渲染](#server-rendering)
- [预取](#prefetching)
- [流式传输](#streaming)
- [客户端转换](#client-side-transitions)

{/* md5:a6002beebed41858329e1576a8b31552 */}
### 服务器端渲染

{/* md5:1a29a5cce69b505e1743ee20770efde1 */}
在 Next.js 中，[布局和页面](/docs/app/getting-started/layouts-and-pages)默认是 [React 服务器组件](https://react.dev/reference/rsc/server-components)。在初始导航和后续导航时，[服务器组件负载](/docs/app/getting-started/server-and-client-components#how-do-server-and-client-components-work-in-nextjs)会先在服务器上生成，然后再发送到客户端。

{/* md5:bc4fbf3a757263558f67725cc9d412a6 */}
根据_何时_发生，服务器端渲染有两种类型：

{/* md5:1fd6e1791505c492b281ff391d7313ec */}
- **静态渲染（或预渲染）** 在构建时或在[重新验证](/docs/app/getting-started/caching-and-revalidating)期间发生，其结果会被缓存。
- **动态渲染** 在请求时，响应客户端请求而发生。

{/* md5:780d6a4bd63e0fc4b8ae5725e6a2fc6c */}
服务器端渲染的权衡在于，在显示新路由之前，客户端必须等待服务器响应。Next.js 通过[预取](#prefetching)用户可能访问的路由并执行[客户端转换](#client-side-transitions)来解决这一延迟。

{/* md5:5e26ae16a4fa860008d53d787046adb9 */}
> **提示**：初始访问也会生成 HTML。

{/* md5:46383162eb69ac9e660f22215b3e26cd */}
### 预取

{/* md5:5099fb099f45e6594ceffed402c08e32 */}
预取是在用户导航到路由之前，在后台加载该路由的过程。这使得应用程序中的路由间导航感觉即时，因为当用户点击链接时，渲染下一个路由的数据已经可以在客户端使用。

{/* md5:3d3b4b2267f2ee62971b83edd392b203 */}
当使用 [`<Link>` 组件](/docs/app/api-reference/components/link) 链接的路由进入用户视口时，Next.js 会自动预取它们。

```tsx filename="app/layout.tsx" switcher
import Link from 'next/link'

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <body>
        <nav>
          {/* Prefetched when the link is hovered or enters the viewport */}
          <Link href="/blog">Blog</Link>
          {/* No prefetching */}
          <a href="/contact">Contact</a>
        </nav>
        {children}
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import Link from 'next/link'

export default function Layout() {
  return (
    <html>
      <body>
        <nav>
          {/* Prefetched when the link is hovered or enters the viewport */}
          <Link href="/blog">Blog</Link>
          {/* No prefetching */}
          <a href="/contact">Contact</a>
        </nav>
        {children}
      </body>
    </html>
  )
}
```

{/* md5:f4695ca1bc6997942403855194aca95e */}
预取路由的多少取决于它是静态还是动态的：

{/* md5:00f4618fb07789064aae1dda97b9ef3a */}
- **静态路由**：会预取整个路由。
- **动态路由**：跳过预取，或者如果存在 [`loading.tsx`](/docs/app/api-reference/file-conventions/loading)，则会部分预取路由。

{/* md5:c09ffe644a226d642ce8e3f0e6cc7e32 */}
通过跳过或部分预取动态路由，Next.js 避免了对用户可能永远不会访问的路由在服务器上进行不必要的工作。然而，在导航前等待服务器响应可能会给用户留下应用没有响应的印象。

### 流式传输

{/* md5:2775653161bf652f21e035af6170a39c */}
流式传输允许服务器在动态路由的各个部分准备就绪后立即发送到客户端，而不是等待整个路由渲染完成。这意味着用户能更快地看到内容，即使页面的部分仍在加载。

{/* md5:1c7391b7ae6102d15d2cae2fc5050ead */}
对于动态路由，这意味着它们可以被**部分预取**。也就是说，共享布局和加载骨架屏可以提前请求。

{/* md5:b1d9493c84a6673547eb0c4054b5e9f2 */}
> **提示**：你也可以使用 [`<Suspense>`](https://react.dev/reference/react/Suspense) 为嵌套组件创建加载 UI。

{/* md5:030cb6c4b7b555f4b51faf39498aa0b9 */}
`loading.tsx` 的好处：

{/* md5:2f6d2d6d83b037573d44f1e9a1632478 */}
- 为用户提供即时导航和视觉反馈。
- 共享布局保持交互性，导航可中断。
- 改善核心 Web 指标：[TTFB](https://web.dev/articles/ttfb)、[FCP](https://web.dev/articles/fcp) 和 [TTI](https://web.dev/articles/tti)。

{/* md5:136a0974abf84b8f950896fcc7a42da1 */}
为了进一步改善导航体验，Next.js 使用 `<Link>` 组件执行[客户端转换](#client-side-transitions)。

{/* md5:2b9c4b210d4e481c520694ade60cc93d */}
### 客户端转换

{/* md5:9993c0877dd9b3e3f28ac51075b4a454 */}
传统上，导航到服务器渲染的页面会触发完整的页面加载。这会清除状态、重置滚动位置并阻止交互性。

{/* md5:5eef9697bfb6773e7d9f6af45532b1df */}
Next.js 通过使用 `<Link>` 组件进行客户端转换来避免这种情况。它不是重新加载页面，而是动态更新内容：

{/* md5:6c48c284bccd63a823773ef2496ab111 */}
- 保留所有共享布局和 UI。
- 用预取的加载状态或新页面（如果可用）替换当前页面。

{/* md5:e225f7c72da726626b566ff4ec69248d */}
客户端转换使得服务器渲染的应用程序_感觉_像客户端渲染的应用程序。当与[预取](#prefetching)和[流式传输](#streaming)结合使用时，即使对于动态路由也能实现快速转换。

{/* md5:7f1938955166f633a2cb0c30157250c1 */}
## 什么会导致转换缓慢？

{/* md5:c3a52bede18756f7cc3c7af69d66ec74 */}
这些 Next.js 优化使导航快速且响应迅速。然而，在某些条件下，转换仍然可能_感觉_缓慢。以下是一些常见原因及如何改善用户体验：

{/* md5:85e58ec3714ff5f5d98e9e0026a734b9 */}
### 没有 `loading.tsx` 的动态路由

{/* md5:3b80e73837266bd1b6edfe00aa1e11d3 */}
当导航到动态路由时，客户端必须等待服务器响应才能显示结果。这可能会让用户觉得应用没有响应。

{/* md5:f3d3dc80f04d5f71b678781d839644b8 */}
我们建议在动态路由中添加 `loading.tsx`，以启用部分预取、触发即时导航，并在路由渲染时显示加载 UI。

```tsx filename="app/blog/[slug]/loading.tsx" switcher
export default function Loading() {
  return <LoadingSkeleton />
}
```

```jsx filename="app/blog/[slug]/loading.js" switcher
export default function Loading() {
  return <LoadingSkeleton />
}
```

{/* md5:01bf03efc95d394360400952a6ddc2ff */}
> **提示**：在开发模式下，你可以使用 Next.js 开发者工具来识别路由是静态的还是动态的。更多信息请参阅 [`devIndicators`](/docs/app/api-reference/config/next-config-js/devIndicators)。

{/* md5:4fca114d7b219b54b1d9cf6a30d3bfda */}
### 没有 `generateStaticParams` 的动态段

{/* md5:cbc8b713e214b96807df93041c7e1033 */}
如果[动态段](/docs/app/api-reference/file-conventions/dynamic-routes)可以预渲染，但由于缺少 [`generateStaticParams`](/docs/app/api-reference/functions/generate-static-params) 而没有预渲染，该路由将在请求时回退到动态渲染。

{/* md5:4b33b6b375997fc756cd2306287d973a */}
通过添加 `generateStaticParams` 确保路由在构建时静态生成：

```tsx filename="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  // ...
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))

export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

{/* md5:5e119302383bda2c8c0cc51120283ccd */}
### 慢速网络

{/* md5:a117327e7e60ac1aea3073bcdd4e2f11 */}
在缓慢或不稳定的网络上，预取可能在用户点击链接前无法完成。这会影响静态和动态路由。在这些情况下，`loading.js` 回退可能不会立即出现，因为它还没有被预取。

{/* md5:f0252a74582e2b68e373f76f72739512 */}
为了改善感知性能，你可以使用 [`useLinkStatus` 钩子](/docs/app/api-reference/functions/use-link-status) 在转换进行时向用户显示内联视觉反馈（如链接上的旋转器或文本闪烁）。

```tsx filename="app/ui/loading-indicator.tsx" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return pending ? (
    <div role="status" aria-label="Loading" className="spinner" />
  ) : null
}
```

```jsx filename="app/ui/loading-indicator.js" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return pending ? (
    <div role="status" aria-label="Loading" className="spinner" />
  ) : null
}
```

{/* md5:ba478e4099cd5e03b1b038759ff34411 */}
你可以通过添加初始动画延迟（例如 100ms）并以不可见状态（例如 `opacity: 0`）启动动画来“防抖”加载指示器。这意味着只有当导航时间超过指定延迟时，加载指示器才会显示。

```css
.spinner {
  /* ... */
  opacity: 0;
  animation:
    fadeIn 500ms 100ms forwards,
    rotate 1s linear infinite;
}

@keyframes fadeIn {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

@keyframes rotate {
  to {
    transform: rotate(360deg);
  }
}
```

{/* md5:5ebe6b0cedd458158a9f94552ed816f3 */}
> **提示**：你可以使用其他视觉反馈模式，如进度条。在此处查看示例：[https://github.com/vercel/react-transition-progress](https://github.com/vercel/react-transition-progress)。

{/* md5:760af26d7e02c5ccf4482c17319a1e93 */}
### 禁用预取

{/* md5:8a1f3829eff9133414bd31f240153894 */}
你可以通过在 `<Link>` 组件上将 `prefetch` 属性设置为 `false` 来选择退出预取。这在渲染大量链接列表（例如无限滚动表格）时，对于避免不必要的资源使用非常有用。

```tsx
<Link prefetch={false} href="/blog">
  Blog
</Link>
```

{/* md5:f3e2d484d3cb1e42410605e831ffcb90 */}
然而，禁用预取会带来权衡：

{/* md5:0ddb33225f5fe9a1c994ad8dfb972a9f */}
- **静态路由** 将仅在用户点击链接时获取。
- **动态路由** 将需要先在服务器上渲染，然后客户端才能导航到它。

{/* md5:ddf02447b0f28055c0f5df24c45b05e8 */}
为了在不完全禁用预取的情况下减少资源使用，您可以仅悬停时预取。这会限制预取到用户更可能访问的路由，而不是视口中的所有链接。

```tsx filename="app/ui/hover-prefetch-link.tsx" switcher
'use client'

import Link from 'next/link'
import { useState } from 'react'

function HoverPrefetchLink({
  href,
  children,
}: {
  href: string
  children: React.ReactNode
}) {
  const [active, setActive] = useState(false)

  return (
    <Link
      href={href}
      prefetch={active ? null : false}
      onMouseEnter={() => setActive(true)}
    >
      {children}
    </Link>
  )
}
```


### 水合未完成

{/* md5:100ab3d4c112ace7a914bf6e6bd983c0 */}
`<Link>` 是一个客户端组件，必须在水合之后才能预取路由。在初始访问时，大型 JavaScript 包可能会延迟水合，阻止预取立即开始。

{/* md5:4fdbadfd83d2689132e3409e5af7ada0 */}
React 通过选择性水合来缓解这一问题，你还可以通过以下方式进一步改善：

{/* md5:7332b9d89d5f198dac49bc0a79d88e5d */}
- 使用 [`@next/bundle-analyzer`](/docs/app/guides/package-bundling#analyzing-javascript-bundles) 插件识别并移除大型依赖项以减少包大小。
- 在可能的情况下，将逻辑从客户端移到服务器。请参阅[服务器和客户端组件](/docs/app/getting-started/server-and-client-components)文档以获取指导。

### 原生 History API

{/* md5:7d78d59bbe59dd47c4f896fba3edae8d */}
Next.js 允许你使用原生的 [`window.history.pushState`](https://developer.mozilla.org/en-US/docs/Web/API/History/pushState) 和 [`window.history.replaceState`](https://developer.mozilla.org/en-US/docs/Web/API/History/replaceState) 方法来更新浏览器的历史记录堆栈，而无需重新加载页面。

{/* md5:8514d98850d2323a83a568d15ed84a22 */}
`pushState` 和 `replaceState` 调用会与 Next.js 路由器集成，允许你与 [`usePathname`](/docs/app/api-reference/functions/use-pathname) 和 [`useSearchParams`](/docs/app/api-reference/functions/use-search-params) 同步。

{/* md5:6f3b3f283a8ad64605e76601438624c9 */}
#### `window.history.pushState`

{/* md5:7ae43a94ca0a007d10af623977efc98e */}
用于向浏览器历史记录堆栈添加新条目。用户可以导航回上一个状态。例如，对产品列表进行排序：

```tsx fileName="app/ui/sort-products.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SortProducts() {
  const searchParams = useSearchParams()

  function updateSorting(sortOrder: string) {
    const params = new URLSearchParams(searchParams.toString())
    params.set('sort', sortOrder)
    window.history.pushState(null, '', `?${params.toString()}`)
  }

  return (
    <>
      <button onClick={() => updateSorting('asc')}>Sort Ascending</button>
      <button onClick={() => updateSorting('desc')}>Sort Descending</button>
    </>
  )
}
```


#### `window.history.replaceState`

{/* md5:6dc5fbadf90b7ec57c1cb22803482df7 */}
用于替换浏览器历史记录堆栈上的当前条目。用户无法导航回上一个状态。例如，切换应用程序的区域设置：

```tsx fileName="app/ui/locale-switcher.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'

export function LocaleSwitcher() {
  const pathname = usePathname()

  function switchLocale(locale: string) {
    // e.g. '/en/about' or '/fr/contact'
    const newPath = `/${locale}${pathname}`
    window.history.replaceState(null, '', newPath)
  }

  return (
    <>
      <button onClick={() => switchLocale('en')}>English</button>
      <button onClick={() => switchLocale('fr')}>French</button>
    </>
  )
}
```
