在 React 和 Next.js 中，您可以编写几种不同类型的测试，每种都有其特定的目的和使用场景。本页概述了可用于测试应用程序的测试类型和常用工具。

## 测试类型

- **单元测试 (Unit Testing)** 涉及在隔离环境中测试单个单元（或代码块）。在 React 中，一个单元可以是单个函数、Hook 或组件。
  - **组件测试 (Component Testing)** 是单元测试的一个更聚焦的版本，其主要测试对象是 React 组件。这可能涉及测试组件如何渲染、它们与 props 的交互以及它们在响应用户事件时的行为。
  - **集成测试 (Integration Testing)** 涉及测试多个单元如何协同工作。这可以是组件、Hook 和函数的组合。
- **端到端测试 (End-to-End Testing)** 涉及在模拟真实用户场景（如浏览器）的环境中测试用户流程。这意味着在类似生产的环境中测试特定任务（例如注册流程）。
- **快照测试 (Snapshot Testing)** 涉及捕获组件的渲染输出并将其保存到快照文件。当测试运行时，组件的当前渲染输出会与保存的快照进行比较。快照中的变化用于指示行为中的意外变化。

## 异步服务端组件 (Async Server Components)

由于 `async` 服务器组件在 React 生态系统中相对较新，一些工具尚不完全支持它们。在此期间，对于 `async` 组件，我们建议使用**端到端测试 (End-to-End Testing)** 而非**单元测试 (Unit Testing)**。
