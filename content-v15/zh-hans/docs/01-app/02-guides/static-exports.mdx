Next.js 支持从静态站点或单页面应用（SPA）开始，之后可选择升级到需要服务器的功能。

{/* md5:445cca43a323c631e2eccf357f227cd6 */}
当运行 `next build` 时，Next.js 会为每个路由生成一个 HTML 文件。通过将严格的 SPA 拆分为独立的 HTML 文件，Next.js 可以避免在客户端加载不必要的 JavaScript 代码，从而减小包体积并实现更快的页面加载。

{/* md5:30b21cc19f26d6b3a4658a1828333c0b */}
由于 Next.js 支持这种静态导出，因此它可以部署和托管在任何能够提供 HTML/CSS/JS 静态资源的 Web 服务器上。

{/* md5:4de9ac2b7349e74e6eeddaa69cdc8303 */}
## 配置

{/* md5:8d37a0a58a5e8e994a32085ffa494cbc */}
要启用静态导出，请更改 `next.config.js` 中的输出模式：

```js filename="next.config.js" highlight={5}
/**
 * @type {import('next').NextConfig}
 */
const nextConfig = {
  output: 'export',

  // Optional: Change links `/me` -> `/me/` and emit `/me.html` -> `/me/index.html`
  // trailingSlash: true,

  // Optional: Prevent automatic `/me` -> `/me/`, instead preserve `href`
  // skipTrailingSlashRedirect: true,

  // Optional: Change the output directory `out` -> `dist`
  // distDir: 'dist',
}

module.exports = nextConfig
```

{/* md5:596580a6d350f55642bcbdef3b5af7f0 */}
运行 `next build` 后，Next.js 将创建一个包含应用程序 HTML/CSS/JS 资源的 `out` 文件夹。

<PagesOnly>

{/* md5:7e594302d30ee2119279c4ff54358afe */}
您可以利用 [`getStaticProps`](/docs/pages/building-your-application/data-fetching/get-static-props) 和 [`getStaticPaths`](/docs/pages/building-your-application/data-fetching/get-static-paths) 来为 `pages` 目录中的每个页面（或为[动态路由](/docs/app/api-reference/file-conventions/dynamic-routes)生成更多页面）生成 HTML 文件。

</PagesOnly>

## 支持的功能

{/* md5:8be0ccc644aae0e84d6f5e597c1827ae */}
Next.js 的核心设计支持静态导出。

{/* md5:83ee120aaa5af43f777ca789ca049fee */}
### 服务器组件

{/* md5:aa91ce276fa2ecf7bedc29902575dc47 */}
当您运行 `next build` 生成静态导出时，在 `app` 目录中使用的服务器组件将在构建期间运行，类似于传统的静态站点生成。

{/* md5:1cffab85e17d9631ff3510474d72c788 */}
生成的组件将被渲染为用于初始页面加载的静态 HTML 和用于客户端路由间导航的静态负载。使用静态导出时，您的服务器组件无需进行任何更改，除非它们使用了[动态服务器功能](#unsupported-features)。

```tsx filename="app/page.tsx" switcher
export default async function Page() {
  // This fetch will run on the server during `next build`
  const res = await fetch('https://api.example.com/...')
  const data = await res.json()

  return <main>...</main>
}
```

{/* md5:3ca9363922963f170ad24f8f5eea2a29 */}
### 客户端组件

{/* md5:36b47b3e964d5d4c556f4e90ab6d8b83 */}
如果您想在客户端执行数据获取，可以使用带有 [SWR](https://github.com/vercel/swr) 的客户端组件来缓存请求。

```tsx filename="app/other/page.tsx" switcher
'use client'

import useSWR from 'swr'

const fetcher = (url: string) => fetch(url).then((r) => r.json())

export default function Page() {
  const { data, error } = useSWR(
    `https://jsonplaceholder.typicode.com/posts/1`,
    fetcher
  )
  if (error) return 'Failed to load'
  if (!data) return 'Loading...'

  return data.title
}
```

```jsx filename="app/other/page.js" switcher
'use client'

import useSWR from 'swr'

const fetcher = (url) => fetch(url).then((r) => r.json())

export default function Page() {
  const { data, error } = useSWR(
    `https://jsonplaceholder.typicode.com/posts/1`,
    fetcher
  )
  if (error) return 'Failed to load'
  if (!data) return 'Loading...'

  return data.title
}
```

{/* md5:46eca1c871d0e56c7a46831aa00147ba */}
由于路由转换发生在客户端，这表现得像传统的 SPA。例如，以下索引路由允许您在客户端导航到不同的帖子：

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <>
      <h1>Index Page</h1>
      <hr />
      <ul>
        <li>
          <Link href="/post/1">Post 1</Link>
        </li>
        <li>
          <Link href="/post/2">Post 2</Link>
        </li>
      </ul>
    </>
  )
}
```

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <>
      <h1>Index Page</h1>
      <p>
        <Link href="/other">Other Page</Link>
      </p>
    </>
  )
}
```

### 图像优化

{/* md5:4ad24729782f6acc9535817e3b9e7e8b */}
通过在 `next.config.js` 中定义自定义图像加载器，[图像优化](/docs/app/api-reference/components/image) 功能可以通过 `next/image` 与静态导出一起使用。例如，您可以使用像 Cloudinary 这样的服务来优化图像：

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  images: {
    loader: 'custom',
    loaderFile: './my-loader.ts',
  },
}

module.exports = nextConfig
```

{/* md5:d2478acbe063d2443e07ae4148689906 */}
此自定义加载器将定义如何从远程源获取图像。例如，以下加载器将为 Cloudinary 构建 URL：

```ts filename="my-loader.ts" switcher
export default function cloudinaryLoader({
  src,
  width,
  quality,
}: {
  src: string
  width: number
  quality?: number
}) {
  const params = ['f_auto', 'c_limit', `w_${width}`, `q_${quality || 'auto'}`]
  return `https://res.cloudinary.com/demo/image/upload/${params.join(
    ','
  )}${src}`
}
```

```js filename="my-loader.js" switcher
export default function cloudinaryLoader({ src, width, quality }) {
  const params = ['f_auto', 'c_limit', `w_${width}`, `q_${quality || 'auto'}`]
  return `https://res.cloudinary.com/demo/image/upload/${params.join(
    ','
  )}${src}`
}
```

{/* md5:936a3f899bf4b91055fb709b4d6d09f3 */}
然后，您可以在应用程序中使用 `next/image`，定义指向 Cloudinary 中图像的相对路径：

```tsx filename="app/page.tsx" switcher
import Image from 'next/image'

export default function Page() {
  return <Image alt="turtles" src="/turtles.jpg" width={300} height={300} />
}
```

```jsx filename="app/page.js" switcher
import Image from 'next/image'

export default function Page() {
  return <Image alt="turtles" src="/turtles.jpg" width={300} height={300} />
}
```

### 路由处理程序

{/* md5:3c879458ceb4534b09f5efa7348c01d4 */}
运行 `next build` 时，路由处理程序将渲染静态响应。仅支持 `GET` HTTP 方法。这可用于从缓存或未缓存的数据生成静态 HTML、JSON、TXT 或其他文件。例如：

```ts filename="app/data.json/route.ts" switcher
export async function GET() {
  return Response.json({ name: 'Lee' })
}
```

```js filename="app/data.json/route.js" switcher
export async function GET() {
  return Response.json({ name: 'Lee' })
}
```

{/* md5:9661020feef06f1fe5433bd39bed3eca */}
上面的文件 `app/data.json/route.ts` 将在 `next build` 期间渲染为静态文件，生成包含 `{ name: 'Lee' }` 的 `data.json`。

{/* md5:a4c2e4fbf1261f1a5bd32359ac721761 */}
如果您需要从传入的请求中读取动态值，则不能使用静态导出。

{/* md5:bdc61b56259624ac6d631d9caf5c05c0 */}
### 浏览器 API

{/* md5:05dcecce3533e081ce925abbdc2ea281 */}
客户端组件在 `next build` 期间被预渲染为 HTML。由于像 `window`、`localStorage` 和 `navigator` 这样的 [Web API](https://developer.mozilla.org/docs/Web/API) 在服务器上不可用，因此您需要仅在浏览器中运行时安全地访问这些 API。例如：

```jsx
'use client';

import { useEffect } from 'react';

export default function ClientComponent() {
  useEffect(() => {
    // You now have access to `window`
    console.log(window.innerHeight);
  }, [])

  return ...;
}
```

## 部署

{/* md5:2ca358f172b2549b7785a90954eb294f */}
通过静态导出，Next.js 可以部署和托管在任何能够提供 HTML/CSS/JS 静态资源的 Web 服务器上。

{/* md5:4fdf2d54ac345956fe2fadae5e199518 */}
运行 `next build` 时，Next.js 会将静态导出生成到 `out` 文件夹中。例如，假设您有以下路由：

{/* md5:db093521ee7aff5fce5ccfebc54b48ff */}
- `/`
- `/blog/[id]`

{/* md5:ef50a782661dae45bd09799e2b107462 */}
运行 `next build` 后，Next.js 将生成以下文件：

{/* md5:7375719bab90332c0d9b65dad7c441e4 */}
- `/out/index.html`
- `/out/404.html`
- `/out/blog/post-1.html`
- `/out/blog/post-2.html`

{/* md5:086c6cd3694b40294346afbf45a29ab4 */}
如果您使用像 Nginx 这样的静态主机，您可以配置从传入请求到正确文件的重写规则：

```nginx filename="nginx.conf"
server {
  listen 80;
  server_name acme.com;

  root /var/www/out;

  location / {
      try_files $uri $uri.html $uri/ =404;
  }

  # This is necessary when `trailingSlash: false`.
  # You can omit this when `trailingSlash: true`.
  location /blog/ {
      rewrite ^/blog/(.*)$ /blog/$1.html break;
  }

  error_page 404 /404.html;
  location = /404.html {
      internal;
  }
}
```
