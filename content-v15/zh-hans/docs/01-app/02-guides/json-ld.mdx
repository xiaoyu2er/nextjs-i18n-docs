---
title: 如何在 Next.js 应用中实现 JSON-LD
nav_title: JSON-LD
description: 了解如何向 Next.js 应用添加 JSON-LD，以向搜索引擎和 AI 描述您的内容。
---

[JSON-LD](https://json-ld.org/) 是一种结构化数据格式，可供搜索引擎和 AI 使用，以帮助它们理解页面除纯内容之外的结构。例如，您可以使用它来描述人、事件、组织、电影、书籍、食谱以及许多其他类型的实体。

我们目前对于 JSON-LD 的建议是将结构化数据渲染为您 `layout.js` 或 `page.js` 组件中的一个 `<script>` 标签。

以下代码片段使用了 `JSON.stringify`，它不会清理用于 XSS 注入的恶意字符串。为了防止此类漏洞，您可以从 `JSON-LD` 载荷中清除 `HTML` 标签，例如，将字符 `<` 替换为其 unicode 等效形式 `\u003c`。

请查阅您所在组织推荐的清理潜在危险字符串的方法，或使用社区维护的 `JSON.stringify` 替代方案，例如 [serialize-javascript](https://www.npmjs.com/package/serialize-javascript)。

```tsx filename="app/products/[id]/page.tsx" switcher
export default async function Page({ params }) {
  const { id } = await params
  const product = await getProduct(id)

  const jsonLd = {
    '@context': 'https://schema.org',
    '@type': 'Product',
    name: product.name,
    image: product.image,
    description: product.description,
  }

  return (
    <section>
      {/* Add JSON-LD to your page */}
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{
          __html: JSON.stringify(jsonLd).replace(/</g, '\\u003c'),
        }}
      />
      {/* ... */}
    </section>
  )
}
```

您可以使用 Google 的 [Rich Results Test](https://search.google.com/test/rich-results) 或通用的 [Schema Markup Validator](https://validator.schema.org/) 来验证和测试您的结构化数据。

您可以使用社区包（如 [`schema-dts`](https://www.npmjs.com/package/schema-dts)）通过 TypeScript 为您的 JSON-LD 添加类型：

```tsx
import { Product, WithContext } from 'schema-dts'

const jsonLd: WithContext<Product> = {
  '@context': 'https://schema.org',
  '@type': 'Product',
  name: 'Next.js Sticker',
  image: 'https://nextjs.org/imgs/sticker.png',
  description: 'Dynamic at the speed of static.',
}
```
