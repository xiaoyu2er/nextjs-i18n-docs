{/* md5:ccaecab859f627ea2b7e1e2c129b4fd7 */}
Next.js 支持"前端后端" (Backend for Frontend) 模式。这使您可以创建公开的端点来处理 HTTP 请求并返回任何内容类型——不仅仅是 HTML。您还可以访问数据源并执行副作用，例如更新远程数据。

{/* md5:ccb12e39031311af02576202f8bd98df */}
如果您开始一个新项目，使用带有 `--api` 标志的 `create-next-app` 会自动在新项目的 `app/` 文件夹中包含一个示例 `route.ts`，展示如何创建 API 端点。

```bash filename="Terminal"
npx create-next-app@latest --api
```

{/* md5:05b271a2875792fed8049874453e6bfd */}
> **须知**：Next.js 的后端功能并非完整的后端替代方案。它们充当一个 API 层，该层：
>
> - 可公开访问
> - 处理任何 HTTP 请求
> - 可以返回任何内容类型

{/* md5:3adcc2a8be67c4fdd8bbd1781220cbb0 */}
要实现此模式，请使用：

{/* md5:57bf0e18a2e3c4321200615eae3add4f */}
- [路由处理器](/docs/app/api-reference/file-conventions/route)
- [`middleware`](/docs/app/api-reference/file-conventions/middleware)
- 在 Pages 路由中，[API 路由](/docs/pages/building-your-application/routing/api-routes)

{/* md5:d15ce64f280a3e3e1a97cb189852d7ab */}
## 公开端点

{/* md5:1f9d8725c42a8de06846043e2f21bccf */}
路由处理器是公开的 HTTP 端点。任何客户端都可以访问它们。

{/* md5:4a4e25e7798520a866d8f3d4147c4a89 */}
使用 `route.ts` 或 `route.js` 文件约定创建路由处理器：

```ts filename="/app/api/route.ts" switcher
export function GET(request: Request) {}
```

```js filename="/app/api/route.js" switcher
export function GET(request) {}
```

{/* md5:4368f52e8d23e95d0227cbb894d3e5a0 */}
这将处理发送到 `/api` 的 `GET` 请求。

{/* md5:3d4b3c6650314e92fa76686b1452557a */}
对可能抛出异常的操作使用 `try/catch` 块：

```ts filename="/app/api/route.ts" switcher
import { submit } from '@/lib/submit'

export async function POST(request: Request) {
  try {
    await submit(request)
    return new Response(null, { status: 204 })
  } catch (reason) {
    const message =
      reason instanceof Error ? reason.message : 'Unexpected error'

    return new Response(message, { status: 500 })
  }
}
```


## 内容类型

{/* md5:41501dbda76150f2a85c6ba04c60f338 */}
路由处理器允许您提供非 UI 响应，包括 JSON、XML、图像、文件和纯文本。

{/* md5:9ca5af4e52b7c8c0f3c21bdd0ee63c18 */}
Next.js 为常见端点使用文件约定：

{/* md5:4e83b3596c43e12f2d46f2c26c07f1ca */}

- [`manifest.json`](/docs/app/api-reference/file-conventions/metadata/manifest)
- [`robots.txt`](/docs/app/api-reference/file-conventions/metadata/robots)

{/* md5:dbb02c66b83267b2e006de34ea78cef7 */}

- `.well-known`

```ts filename="/app/api/send-email/route.ts" switcher
import { sendMail, validateInputs } from '@/lib/email-transporter'

export async function POST(request: Request) {
  const formData = await request.formData()
  const email = formData.get('email')
  const contents = formData.get('contents')

  try {
    await validateInputs({ email, contents })
    const info = await sendMail({ email, contents })

    return Response.json({ messageId: info.messageId })
  } catch (reason) {
    const message =
      reason instanceof Error ? reason.message : 'Unexpected exception'

    return new Response(message, { status: 500 })
  }
}
```

## 操作数据

{/* md5:2bd49db380012edbc063545e30e35889 */}
路由处理器可以转换、过滤和聚合来自一个或多个来源的数据。这使逻辑远离前端，并避免暴露内部系统。

{/* md5:bf81f385a7cc4b5b5f613a7f4b67da3e */}
您还可以将繁重的计算卸载到服务器，并减少客户端的电池和数据使用。

```ts file="/app/api/weather/route.ts" switcher
import { parseWeatherData } from '@/lib/weather'

export async function POST(request: Request) {
  const body = await request.json()
  const searchParams = new URLSearchParams({ lat: body.lat, lng: body.lng })

  try {
    const weatherResponse = await fetch(`${weatherEndpoint}?${searchParams}`)

    if (!weatherResponse.ok) {
      /* handle error */
    }

    const weatherData = await weatherResponse.text()
    const payload = parseWeatherData.asJSON(weatherData)

    return new Response(payload, { status: 200 })
  } catch (reason) {
    const message =
      reason instanceof Error ? reason.message : 'Unexpected exception'

    return new Response(message, { status: 500 })
  }
}
```

## 代理到后端

{/* md5:f057c9398a128b1db3e12b83a94df73f */}
您可以将路由处理器用作另一个后端的代理。在转发请求之前添加验证逻辑。

```ts filename="/app/api/[...slug]/route.ts" switcher
import { isValidRequest } from '@/lib/utils'

export async function POST(request: Request, { params }) {
  const clonedRequest = request.clone()
  const isValid = await isValidRequest(clonedRequest)

  if (!isValid) {
    return new Response(null, { status: 400, statusText: 'Bad Request' })
  }

  const { slug } = await params
  const pathname = slug.join('/')
  const proxyURL = new URL(pathname, 'https://nextjs.org')
  const proxyRequest = new Request(proxyURL, request)

  try {
    return fetch(proxyRequest)
  } catch (reason) {
    const message =
      reason instanceof Error ? reason.message : 'Unexpected exception'

    return new Response(message, { status: 500 })
  }
}
```

## NextRequest 和 NextResponse

{/* md5:2189c64b916d734b3241ad7898edcd38 */}
Next.js 扩展了 [`Request`](https://developer.mozilla.org/en-US/docs/Web/API/Request) 和 [`Response`](https://developer.mozilla.org/en-US/docs/Web/API/Response) Web API，提供简化常用操作的方法。这些扩展在路由处理器和中间件中均可用。

{/* md5:ef047735d34f5b6088bb490bbded3d2a */}
两者都提供了用于读取和操作 cookie 的方法。

{/* md5:ebef2f557be7b8ea3ae45fff774e0744 */}
`NextRequest` 包含 [`nextUrl`](/docs/app/api-reference/functions/next-request#nexturl) 属性，它暴露了传入请求的解析值，例如，它使访问请求路径名和搜索参数变得更容易。

{/* md5:7c1243e6daa2fba6b15a754703314fc8 */}
`NextResponse` 提供了诸如 `next()`、`json()`、`redirect()` 和 `rewrite()` 等辅助方法。

{/* md5:912cf7cd6e0bf3534cb1bfadf5cbd13a */}
您可以将 `NextRequest` 传递给任何期望 `Request` 的函数。同样，您可以在期望 `Response` 的地方返回 `NextResponse`。

```ts filename="/app/echo-pathname/route.ts" switcher
import { type NextRequest, NextResponse } from 'next/server'

export async function GET(request: NextRequest) {
  const nextUrl = request.nextUrl

  if (nextUrl.searchParams.get('redirect')) {
    return NextResponse.redirect(new URL('/', request.url))
  }

  if (nextUrl.searchParams.get('rewrite')) {
    return NextResponse.rewrite(new URL('/', request.url))
  }

  return NextResponse.json({ pathname: nextUrl.pathname })
}
```


## Webhook 和回调 URL

{/* md5:1623774054dc35aa7f57e9869bc64cc3 */}
使用路由处理器接收来自第三方应用程序的事件通知。

{/* md5:abd7a0c887ca932fde11b356b1da4010 */}
例如，当内容在 CMS 中更改时重新验证路由。配置 CMS 在更改时调用特定端点。

```ts filename="/app/webhook/route.ts" switcher
import { type NextRequest, NextResponse } from 'next/server'

export async function GET(request: NextRequest) {
  const token = request.nextUrl.searchParams.get('token')

  if (token !== process.env.REVALIDATE_SECRET_TOKEN) {
    return NextResponse.json({ success: false }, { status: 401 })
  }

  const tag = request.nextUrl.searchParams.get('tag')

  if (!tag) {
    return NextResponse.json({ success: false }, { status: 400 })
  }

  revalidateTag(tag)

  return NextResponse.json({ success: true })
}
```

回调 URL 是另一个用例。当用户完成第三方流程后，第三方会将其发送到回调 URL。使用路由处理器验证响应并决定将用户重定向到哪里。

```ts filename="/app/auth/callback/route.ts" switcher
import { type NextRequest, NextResponse } from 'next/server'

export async function GET(request: NextRequest) {
  const token = request.nextUrl.searchParams.get('session_token')
  const redirectUrl = request.nextUrl.searchParams.get('redirect_url')

  const response = NextResponse.redirect(new URL(redirectUrl, request.url))

  response.cookies.set({
    value: token,
    name: '_token',
    path: '/',
    secure: true,
    httpOnly: true,
    expires: undefined, // session cookie
  })

  return response
}
```

## 中间件

{/* md5:d17bc59d9e4aadda664649720a50d942 */}
每个项目只允许一个中间件文件。使用 `config.matcher` 来定位特定路径。了解更多有关 [中间件](/docs/app/api-reference/file-conventions/middleware) 的信息。

{/* md5:03bf1052c94199e89a2b3071596e37f2 */}
使用 `middleware` 在请求到达路由路径之前生成响应。

```ts filename="middleware.ts" switcher
import { isAuthenticated } from '@lib/auth'

export const config = {
  matcher: '/api/:function*',
}

export function middleware(request: Request) {
  if (!isAuthenticated(request)) {
    return Response.json(
      { success: false, message: 'authentication failed' },
      { status: 401 }
    )
  }
}
```

### 处理头部

{/* md5:a6fb2010fde4a808624079dec6b68eb7 */}
要慎重决定头部的去向，并避免将传入的请求头部直接传递到传出的响应中。

{/* md5:d7c90d173fa4f5472b90bb7a26cf7431 */}
- **上游请求头**：在中间件 (`Middleware`) 中，`NextResponse.next({ request: { headers } })` 会修改服务器接收的头部，并且不将其暴露给客户端。
- **响应头**：`new Response(..., { headers })`, `NextResponse.json(..., { headers })`, `NextResponse.next({ headers })`, or `response.headers.set(...)` 将头部发送回客户端。如果敏感值附加到这些头部，它们将对客户端可见。

{/* md5:6227eaaff415ce2077a76f09c6b01cdb */}
在 [中间件中的 NextResponse 头部](/docs/app/api-reference/functions/next-response#next) 了解更多。

{/* md5:e4566ce3255ad04c48b0a8daba7d1769 */}
### 速率限制

{/* md5:85ebdd83092a96c4f95234b4b849a7a0 */}
您可以在 Next.js 后端实现速率限制。除了基于代码的检查外，请启用主机提供的任何速率限制功能。

```ts filename="/app/resource/route.ts" switcher
import { NextResponse } from 'next/server'
import { checkRateLimit } from '@/lib/rate-limit'

export async function POST(request: Request) {
  const { rateLimited } = await checkRateLimit(request)

  if (rateLimited) {
    return NextResponse.json({ error: 'Rate limit exceeded' }, { status: 429 })
  }

  return new Response(null, { status: 204 })
}
```

```js filename="/app/resource/route.js" switcher
import { NextResponse } from 'next/server'
import { checkRateLimit } from '@/lib/rate-limit'

export async function POST(request) {
  const { rateLimited } = await checkRateLimit(request)

  if (rateLimited) {
    return NextResponse.json({ error: 'Rate limit exceeded' }, { status: 429 })
  }

  return new Response(null, { status: 204 })
}
```

### 验证负载

{/* md5:544a55e1c0de4b3952cc5634b34ad8f0 */}
永远不要信任传入的请求数据。在使用前验证内容类型和大小，并针对 XSS 进行清理。

{/* md5:163efdaf704a88051949df709ebf86b1 */}
使用超时来防止滥用并保护服务器资源。

{/* md5:d2a76b473fcfd33b0864dd2735fd64ab */}
将用户生成的静态资源存储在专用服务中。尽可能从浏览器上传它们，并将返回的 URI 存储在数据库中以减少请求大小。

## 库模式

{/* md5:8803cc513effe4c2b040ca542510f17f */}
社区库通常为路由处理器使用工厂模式。

```ts filename="/app/api/[...path]/route.ts"
import { createHandler } from 'third-party-library'

const handler = createHandler({
  /* library-specific options */
})

export const GET = handler
// or
export { handler as POST }
```

{/* md5:fd6b9301fb1bdb9c0f5ec31da64b5a7a */}
这为 `GET` 和 `POST` 请求创建了一个共享的处理程序。该库根据请求中的 `method` 和 `pathname` 自定义行为。

{/* md5:650816df18a8dbcbd7f447942dd20983 */}
库还可以提供 `middleware` 工厂。

```ts filename="middleware.ts"
import { createMiddleware } from 'third-party-library'

export default createMiddleware()
```

## 注意事项

{/* md5:83ee120aaa5af43f777ca789ca049fee */}
### 服务器组件

{/* md5:c1afdff99366dba8b75ee2bc84e6e0b6 */}
在服务器组件中直接从数据源获取数据，而不是通过路由处理器。

{/* md5:b24fb2fe072ab435e124e62f278cc7de */}
对于构建时预渲染的服务器组件，使用路由处理器将导致构建步骤失败。这是因为在构建过程中没有服务器监听这些请求。

{/* md5:f4bfc92c8758c08fb7ac2ed832d52eff */}
对于按需渲染的服务器组件，从路由处理器获取数据会更慢，因为处理器和渲染过程之间存在额外的 HTTP 往返。

{/* md5:cc10f97aa0262d52ff5f33d3092c6438 */}
> 服务器端的 `fetch` 请求使用绝对 URL。这意味着需要一次 HTTP 往返到外部服务器。在开发期间，您自己的开发服务器充当外部服务器。在构建时没有服务器，而在运行时，服务器通过您的公开域名可用。

{/* md5:607fc16a35fd08d835a338d8b644c2f4 */}
服务器组件涵盖了大多数数据获取需求。然而，客户端获取数据可能对于以下情况是必要的：

{/* md5:3edf9bb66031594f9debdac99ab6b15b */}
- 依赖于客户端专属 Web API 的数据：
  - 地理定位 API
  - 存储 API
  - 音频 API
  - 文件 API
- 频繁轮询的数据

{/* md5:8886049ea040ee11837e1ed5d6db56e5 */}
对于这些情况，请使用像 [`swr`](https://swr.vercel.app/) 或 [`react-query`](https://tanstack.com/query/latest/docs/framework/react/overview) 这样的社区库。

{/* md5:3b127b0053980bbf90887067a5d66013 */}
### 服务器操作

{/* md5:ee9157110b1842d9e411473a7e87aa80 */}
服务器操作允许您从客户端运行服务器端代码。它们的主要目的是从您的前端客户端变更数据。

{/* md5:a2b12b0ea661f592daa4410c956a4bde */}
服务器操作是排队执行的。将它们用于数据获取会引入顺序执行。

{/* md5:a9af0a900f734d1e2dc662a49d07d975 */}
### `export` 模式

{/* md5:eb36770dcb4570b70a4fd8b301f03fcc */}
`export` 模式输出一个没有运行时服务器的静态站点。需要 Next.js 运行时的功能[不受支持](/docs/app/guides/static-exports#unsupported-features)，因为此模式生成静态站点，没有运行时服务器。

{/* md5:14f2e6695366e15a890069e43b2e4fc6 */}
在 `export` 模式下，仅支持 `GET` 路由处理器，配合 [`dynamic`](/docs/app/api-reference/file-conventions/route-segment-config#dynamic) 路由部分配置，设置为 `'force-static'`。

{/* md5:7c919513bd65a271e1236fe3f5e9a7a4 */}
这可用于生成静态 HTML、JSON、TXT 或其他文件。

```js filename="app/hello-world/route.ts"
export const dynamic = 'force-static'

export function GET() {
  return new Response('Hello World', { status: 200 })
}
```

### 部署环境

{/* md5:cf005b560a0b8648b0ffd4e99e19260d */}
某些主机将路由处理器部署为 lambda 函数。这意味着：

{/* md5:4d4c01d69187be736f1b812cab2ea5a2 */}
- 路由处理器无法在请求之间共享数据。
- 环境可能不支持写入文件系统。
- 长时间运行的处理器可能会因超时而终止。
- WebSocket 将无法工作，因为连接会在超时后关闭，或在响应生成后关闭。
