对外部包进行打包可以显著提升应用程序的性能。<AppOnly>默认情况下，在服务器组件和路由处理器内导入的包会由 Next.js 自动打包。本页将引导您了解如何分析并进一步优化包打包。</AppOnly> <PagesOnly>默认情况下，导入到应用程序中的包不会被打包。这可能会影响性能，或者如果外部包没有预打包（例如，从 monorepo 或 `node_modules` 导入），则可能无法正常工作。本页将引导您了解如何分析和配置包打包。</PagesOnly>

## 分析 JavaScript 包

{/* md5:a5405080179b1f4907122fbd14d63149 */}
[`@next/bundle-analyzer`](https://www.npmjs.com/package/@next/bundle-analyzer) 是 Next.js 的一个插件，可帮助您管理应用程序包的大小。它会生成每个包及其依赖项大小的可视化报告。您可以利用这些信息来移除大型依赖项、拆分代码或进行[懒加载](/docs/app/guides/lazy-loading)。

{/* md5:1e4be0c3d7aa5c750e2bc8d1034930d1 */}
### 安装

{/* md5:e89f6c6ac00a78e53431557c251e6497 */}
运行以下命令来安装该插件：

```bash
npm i @next/bundle-analyzer
# or
yarn add @next/bundle-analyzer
# or
pnpm add @next/bundle-analyzer
```

{/* md5:59078e3a125a81d6f8b1ffecd365ae8f */}
然后，将包分析器的设置添加到您的 `next.config.js` 文件中。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {}

const withBundleAnalyzer = require('@next/bundle-analyzer')({
  enabled: process.env.ANALYZE === 'true',
})

module.exports = withBundleAnalyzer(nextConfig)
```

{/* md5:981cfcbaee0a2b5b108285cd6a2c86f0 */}
### 生成报告

{/* md5:26c254d056adde646abd0a92eff4e90a */}
运行以下命令来分析您的包：

```bash
ANALYZE=true npm run build
# or
ANALYZE=true yarn build
# or
ANALYZE=true pnpm build
```

{/* md5:155f001da938c392e752a06d2a8e0ee7 */}
报告将在您的浏览器中打开三个新标签页，您可以进行检查。定期评估应用程序的包有助于您随时间推移维护应用程序的性能。

{/* md5:1d066f2f1a64b6f595bc5c12f026d3d2 */}
## 优化包导入

{/* md5:225ffa0d905f0b9e1410d0a4c5fc7a02 */}
某些包，例如图标库，可能会导出数百个模块，这可能会在开发和生产环境中引发性能问题。

{/* md5:cea8a00390849a2b011bcf1fb942efe4 */}
您可以通过在 `next.config.js` 中添加 [`optimizePackageImports`](/docs/app/api-reference/config/next-config-js/optimizePackageImports) 选项来优化这些包的导入方式。此选项将仅加载您_实际_使用的模块，同时仍为您提供使用多个命名导出编写导入语句的便利。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    optimizePackageImports: ['icon-library'],
  },
}

module.exports = nextConfig
```

{/* md5:f2f81e3a513bd27ed2bf80b1d62364fb */}
Next.js 也会自动优化某些库，因此无需将它们包含在 optimizePackageImports 列表中。请参阅[完整列表](/docs/app/api-reference/config/next-config-js/optimizePackageImports)。

## 打包特定包

{/* md5:d99a3e0dc313aff43c7cb4106745922a */}
要打包特定包，您可以在 `next.config.js` 中使用 [`transpilePackages`](/docs/app/api-reference/config/next-config-js/transpilePackages) 选项。此选项对于打包未预打包的外部包非常有用，例如，在 monorepo 中或从 `node_modules` 导入的包。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  transpilePackages: ['package-name'],
}

module.exports = nextConfig
```

{/* md5:7ffac2889d3be7799269d52941325cc9 */}
## 打包所有包

{/* md5:0e13564b083e094ef42fe35ed384e583 */}
要自动打包所有包（App Router 中的默认行为），您可以在 `next.config.js` 中使用 [`bundlePagesRouterDependencies`](/docs/pages/api-reference/config/next-config-js/bundlePagesRouterDependencies) 选项。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  bundlePagesRouterDependencies: true,
}

module.exports = nextConfig
```

{/* md5:e59e53e28f467d2404f1d790b87d23f1 */}
## 将特定包排除在打包之外

{/* md5:121140fad22bf5aeadc9bd695ab0cc95 */}
如果您启用了 [`bundlePagesRouterDependencies`](/docs/pages/api-reference/config/next-config-js/bundlePagesRouterDependencies) 选项，您可以使用 `next.config.js` 中的 [`serverExternalPackages`](/docs/pages/api-reference/config/next-config-js/serverExternalPackages) 选项将特定包排除在自动打包之外：

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  // Automatically bundle external packages in the Pages Router:
  bundlePagesRouterDependencies: true,
  // Opt specific packages out of bundling for both App and Pages Router:
  serverExternalPackages: ['package-name'],
}

module.exports = nextConfig
```

</PagesOnly>

## 将特定包排除在打包之外

{/* md5:2ed6a951f6a867cb971833900f031ef1 */}
由于在服务器组件和路由处理器内导入的包会由 Next.js 自动打包，您可以使用 `next.config.js` 中的 [`serverExternalPackages`](/docs/app/api-reference/config/next-config-js/serverExternalPackages) 选项将特定包排除在打包之外。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  serverExternalPackages: ['package-name'],
}

module.exports = nextConfig
```

{/* md5:047c9700fff747e148cb177324f13196 */}
Next.js 包含一个当前正在努力提升兼容性的流行包列表，这些包会被自动排除在外。请参阅[完整列表](/docs/app/api-reference/config/next-config-js/serverExternalPackages)。

</AppOnly>
