在将您的 Next.js 应用部署到生产环境之前，为了获得最佳的用户体验、性能和安全性，您应该考虑实施一些优化措施和遵循一些模式。

{/* md5:c8605f9abcd9e52131fed4c5ed46e543 */}
本页提供了最佳实践，您可以将其作为在[构建应用](#during-development)和[准备投入生产](#before-going-to-production)时的参考，同时也要了解[自动的 Next.js 优化](#automatic-optimizations)。

## 自动优化

{/* md5:0a60172f64a4dd71dc88c76b716b577f */}
这些 Next.js 优化默认启用，无需配置：

<AppOnly>

{/* md5:4d98913a43c70fc1206948fd4f0fc552 */}
- **[服务器组件](/docs/app/getting-started/server-and-client-components):** Next.js 默认使用服务器组件。服务器组件在服务器上运行，不需要在客户端渲染时依赖 JavaScript。因此，它们不会影响客户端 JavaScript 包的大小。您可以根据需要随时使用 [客户端组件](/docs/app/getting-started/server-and-client-components) 来实现交互性。
- **[代码分割](/docs/app/getting-started/linking-and-navigating#how-navigation-works):** 服务器组件支持按路由段自动进行代码分割。您也可以酌情考虑 [懒加载](/docs/app/guides/lazy-loading) 客户端组件和第三方库。
- **[预获取](/docs/app/getting-started/linking-and-navigating#prefetching):** 当指向新路由的链接进入用户视口时，Next.js 会在后台预获取该路由。这使得导航到新路由几乎是瞬时的。您可以酌情选择退出预获取。
- **[静态渲染](/docs/app/getting-started/partial-prerendering#static-rendering):** Next.js 在构建时在服务器上静态渲染服务器组件和客户端组件，并缓存渲染结果以提高应用性能。您可以酌情选择 [动态渲染](/docs/app/getting-started/partial-prerendering#dynamic-rendering) 特定路由。 {/* TODO: Update when PPR is stable */}
- **[缓存](/docs/app/guides/caching):** Next.js 缓存数据请求、服务器组件和客户端组件的渲染结果、静态资源等，以减少向服务器、数据库和后端服务发起的网络请求数量。您可以酌情选择退出缓存。

</AppOnly>

<PagesOnly>

{/* md5:0402919abc49392948d96c4141262f60 */}
- **[代码分割](/docs/pages/building-your-application/routing/pages-and-layouts)：** Next.js 按页面自动分割您的应用代码。这意味着导航时只加载当前页面所需的代码。您也可以酌情考虑对第三方库进行[懒加载](/docs/pages/guides/lazy-loading)。
- **[预获取](/docs/pages/api-reference/components/link#prefetch)：** 当指向新路由的链接进入用户视口时，Next.js 会在后台预获取该路由。这使得导航到新路由几乎是瞬时的。您可以酌情选择退出预获取。
- **[自动静态优化](/docs/pages/building-your-application/rendering/automatic-static-optimization)：** 如果页面没有阻塞性的数据需求，Next.js 会自动将其确定为静态页面（可以预渲染）。优化后的页面可以被缓存，并从多个 CDN 位置提供给最终用户。您可以酌情选择[服务端渲染](/docs/pages/building-your-application/data-fetching/get-server-side-props)。

</PagesOnly>

{/* md5:09fb1ff557cc7ee12a8fc5ce7fe42099 */}
这些默认设置旨在提高应用性能，并减少每次网络请求的数据传输成本和数量。

{/* md5:012408002e610cab2016fcda7a6295d8 */}
## 开发阶段优化

{/* md5:ad721ea134bf793e8b94c0b5837f9287 */}
在构建应用时，我们建议使用以下功能来确保最佳性能和用户体验：

{/* md5:071bf8ed6c5ce023c3a4b3d06ecd7656 */}
### 路由与渲染

<AppOnly>

{/* md5:bab35078732491c50545884bb2120435 */}
- **[布局](/docs/app/api-reference/file-conventions/layout)：** 使用布局在页面间共享 UI，并在导航时启用[部分渲染](/docs/app/getting-started/linking-and-navigating#client-side-transitions)。
- **[`<Link>` 组件](/docs/app/api-reference/components/link)：** 使用 `<Link>` 组件进行[客户端导航和预获取](/docs/app/getting-started/linking-and-navigating#how-navigation-works)。
- **[错误处理](/docs/app/getting-started/error-handling)：** 通过创建自定义错误页面，在生产环境中优雅地处理[捕获所有错误](/docs/app/getting-started/error-handling)和[404 错误](/docs/app/api-reference/file-conventions/not-found)。
- **[客户端和服务端组件](/docs/app/getting-started/server-and-client-components#examples)：** 遵循推荐的服务端和客户端组件组合模式，并检查您的 [`"use client"` 边界](/docs/app/getting-started/server-and-client-components#examples#moving-client-components-down-the-tree)的放置位置，以避免不必要地增加客户端 JavaScript 包大小。
- **[动态 API](/docs/app/getting-started/partial-prerendering#dynamic-rendering)：** 请注意，像 [`cookies`](/docs/app/api-reference/functions/cookies) 和 [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) 属性这样的动态 API 会使整个路由进入[动态渲染](/docs/app/getting-started/partial-prerendering#dynamic-rendering)（如果在[根布局](/docs/app/api-reference/file-conventions/layout#root-layout)中使用，则会影响整个应用）。请确保动态 API 的使用是故意的，并酌情将它们包装在 `<Suspense>` 边界中。

{/* md5:5d37545425b3694f5b8f5209196b32c3 */}
> **请注意**：[部分预渲染（实验性）](/blog/next-14#partial-prerendering-preview) 将允许路由的某些部分是动态的，而无需使整个路由都进入动态渲染。

</AppOnly>

<PagesOnly>

{/* md5:774467ca0d74a5224bf30ce3dec8d4c3 */}
- **[`<Link>` 组件](/docs/pages/building-your-application/routing/linking-and-navigating)：** 使用 `<Link>` 组件进行客户端导航和预获取。
- **[自定义错误](/docs/pages/building-your-application/routing/custom-error)：** 优雅地处理 [500](/docs/pages/building-your-application/routing/custom-error#500-page) 和 [404 错误](/docs/pages/building-your-application/routing/custom-error#404-page)。

</PagesOnly>

{/* md5:e92fb9d40842618892b1b8c11df4dba7 */}
### 数据获取与缓存

<AppOnly>

{/* md5:9a7c260050d325198310e5f8ba9cd7fc */}
- **[服务器组件](/docs/app/getting-started/fetching-data):** 利用服务器组件在服务器上获取数据的优势。
- **[路由处理程序](/docs/app/api-reference/file-conventions/route):** 使用路由处理程序从客户端组件访问后端资源。但不要从服务器组件调用路由处理程序，以避免额外的服务器请求。
- **[流式传输](/docs/app/api-reference/file-conventions/loading):** 使用 Loading UI 和 React Suspense 逐步将 UI 从服务器发送到客户端，并在获取数据时防止整个路由阻塞。
- **[并行数据获取](/docs/app/getting-started/fetching-data#parallel-data-fetching):** 通过并行获取数据减少网络瀑布流，酌情操作。另外，酌情考虑 [预加载数据](/docs/app/getting-started/fetching-data#preloading-data)。
- **[数据缓存](/docs/app/guides/caching#data-cache):** 验证您的数据请求是否被缓存，并酌情选择启用缓存。确保不使用 `fetch` 的请求被 [缓存](/docs/app/api-reference/functions/unstable_cache)。
- **[静态图片](/docs/app/api-reference/file-conventions/public-folder):** 使用 `public` 目录自动缓存应用的静态资源，例如图片。

</AppOnly>

<PagesOnly>

{/* md5:7603c574484907574b0009bbb8bd1070 */}
- **[API 路由](/docs/pages/building-your-application/routing/api-routes)：** 使用路由处理程序访问后端资源，并防止敏感密钥暴露给客户端。
- **[数据缓存](/docs/pages/building-your-application/data-fetching/get-static-props)：** 验证您的数据请求是否被缓存，并酌情选择启用缓存。确保不使用 `getStaticProps` 的请求在适当时被缓存。
- **[增量静态再生](/docs/pages/guides/incremental-static-regeneration)：** 使用增量静态再生在页面构建后更新静态页面，而无需重建整个站点。
- **[静态图片](/docs/pages/api-reference/file-conventions/public-folder)：** 使用 `public` 目录自动缓存应用的静态资源，例如图片。

</PagesOnly>

{/* md5:d7d3ccb90bb3284234eaa5c5b18b6b73 */}
### UI与无障碍访问

<AppOnly>

{/* md5:17625bb68dbfa1854583683ddb0ed262 */}
- **[表单与验证](/docs/app/guides/forms):** 使用服务端操作 (Server Actions) 处理表单提交、服务端验证和错误处理。

</AppOnly>

{/* md5:0c0f99b036c6cad83375f8b5d42679f9 */}
- **[字体模块](/docs/app/api-reference/components/font)：** 通过使用字体模块优化字体，该模块自动托管您的字体文件与其他静态资源，移除外部网络请求，并减少[布局偏移](https://web.dev/articles/cls)。
- **[`<Image>` 组件](/docs/app/api-reference/components/image)：** 通过使用图像组件优化图像，该组件自动优化图像，防止布局偏移，并以 WebP 等现代格式提供它们。
- **[`<Script>` 组件](/docs/app/guides/scripts)：** 通过使用脚本组件优化第三方脚本，该组件自动延迟脚本并阻止它们阻塞主线程。
- **[ESLint](/docs/architecture/accessibility#linting)：** 使用内置的 `eslint-plugin-jsx-a11y` 插件尽早发现可访问性问题。

{/* md5:868ffd32a26c2fdab23a3013369f77d5 */}
### 安全性

<AppOnly>

{/* md5:c98e7477e325dcce1adee2ded5e10bc1 */}
- **[数据标记](/docs/app/api-reference/config/next-config-js/taint):** 通过标记数据对象和/或特定值，防止敏感数据暴露给客户端。
- **[服务器操作](/docs/app/getting-started/updating-data):** 确保用户已获授权调用服务器操作。查阅建议的 [安全实践](/blog/security-nextjs-server-components-actions)。

</AppOnly>

{/* md5:c3b31d91ae9c55c75e466167bc0cbe01 */}
- **[环境变量](/docs/app/guides/environment-variables)：** 确保您的 `.env.*` 文件已添加到 `.gitignore`，并且只有公共变量以 `NEXT_PUBLIC_` 为前缀。
- **[内容安全策略](/docs/app/guides/content-security-policy)：** 考虑添加内容安全策略，以保护您的应用免受跨站脚本、点击劫持和其他代码注入攻击等各种安全威胁。

{/* md5:5be1244abdd6c64b799b985e21ab2282 */}
### 元数据与SEO

<AppOnly>

{/* md5:c1da878650e2a606d90728b1dbe75978 */}

- **[开放图谱 (OG) 图片](/docs/app/api-reference/file-conventions/metadata/opengraph-image)：** 创建 OG 图片，为应用的社交分享做好准备。
- **[站点地图](/docs/app/api-reference/functions/generate-sitemaps) 和 [Robots](/docs/app/api-reference/file-conventions/metadata/robots)：** 通过生成站点地图和 robots 文件帮助搜索引擎抓取和索引您的页面。

</AppOnly>

</PagesOnly>
