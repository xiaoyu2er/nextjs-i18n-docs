{/* md5:5535accdc24b80cd760c735324405229 */}
[Markdown](https://daringfireball.net/projects/markdown/syntax) 是一种轻量级标记语言，用于格式化文本。它允许你使用纯文本语法进行编写，并将其转换为结构有效的 HTML。它通常用于在网站和博客上撰写内容。

{/* md5:57cff3c677fe97a7612c1ccf130a667b */}
你编写...

```md
I **love** using [Next.js](https://nextjs.org/)
```

{/* md5:b1282d58a4dcde0a2015d98ad33afd4c */}
输出：

```html
<p>I <strong>love</strong> using <a href="https://nextjs.org/">Next.js</a></p>
```

{/* md5:adc00cd30343f93493b5b231952dd2a6 */}
[MDX](https://mdxjs.com/) 是 Markdown 的超集，它允许你直接在 Markdown 文件中编写 [JSX](https://react.dev/learn/writing-markup-with-jsx)。这是一种强大的方式，可以在你的内容中添加动态交互性并嵌入 React 组件。

{/* md5:c0e3c61b53175ae11d8a09108bd94c4a */}
Next.js 可以支持应用程序内的本地 MDX 内容，也可以支持在服务器上动态获取的远程 MDX 文件。Next.js 插件负责将 Markdown 和 React 组件转换为 HTML，包括支持在服务器组件中使用（这是应用路由中的默认方式）。

{/* md5:dc08cb087c33900cb2cf8c590d72197e */}
> **请注意**：请查看 [Portfolio Starter Kit](https://vercel.com/templates/next.js/portfolio-starter-kit) 模板以获取一个完整的工作示例。

{/* md5:3b673c0e5c7e7f3cc9d885fb5cc473e2 */}
## 安装依赖

{/* md5:528add7c70ed136f5fe84132b3e60adc */}
`@next/mdx` 包以及相关包用于配置 Next.js，使其能够处理 Markdown 和 MDX。**它从本地文件中获取数据**，允许你在 `/pages` 或 `/app` 目录中直接创建带有 `.md` 或 `.mdx` 扩展名的页面。

{/* md5:93db51185c530b72e82b9e2307cdce1f */}
安装这些包以使用 Next.js 渲染 MDX：

```bash filename="Terminal"
npm install @next/mdx @mdx-js/loader @mdx-js/react @types/mdx
```

{/* md5:7ede515a6027e8fc52ca85c89b9b2ff1 */}
## 配置 `next.config.mjs`

{/* md5:bff51f490e39fe55578b0be6b8b359e5 */}
更新项目根目录下的 `next.config.mjs` 文件以配置其使用 MDX：

```js filename="next.config.mjs"
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  // Configure `pageExtensions` to include markdown and MDX files
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
  // Optionally, add any other Next.js config below
}

const withMDX = createMDX({
  // Add markdown plugins here, as desired
})

// Merge MDX config with Next.js config
export default withMDX(nextConfig)
```

{/* md5:1ace28da4b8cf496ba6130b2c3055d0f */}
这允许 `.mdx` 文件在你的应用程序中充当页面、路由或导入。

{/* md5:97b98ce983249c18159a800118e17337 */}
### 处理 `.md` 文件

{/* md5:8c86e4e1d82c28118d2c0b287d1b9791 */}
默认情况下，`next/mdx` 只编译扩展名为 `.mdx` 的文件。要使用 webpack 处理 `.md` 文件，请更新 `extension` 选项：

```js filename="next.config.mjs"
const withMDX = createMDX({
  extension: /\.(md|mdx)$/,
})
```

## 添加 `mdx-components.tsx` 文件

{/* md5:ca677eaa41ef6f4ce41dc52965c38b4d */}
在项目根目录创建一个 `mdx-components.tsx`（或 `.js`）文件以定义全局 MDX 组件。例如，与 `pages` 或 `app` 同级，或者在适用的情况下放在 `src` 内部。

```tsx filename="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'

const components: MDXComponents = {}

export function useMDXComponents(): MDXComponents {
  return components
}
```

```js filename="mdx-components.js" switcher
const components = {}

export function useMDXComponents() {
  return components
}
```

{/* md5:3ae09c8432f0b4a0fc77415841024b6e */}
> **请注意**：
>
> - `mdx-components.tsx` 是在应用路由中使用 `@next/mdx` **必需的**，没有它将无法工作。
> - 了解更多关于 [`mdx-components.tsx` 文件约定](/docs/app/api-reference/file-conventions/mdx-components)。
> - 学习如何 [使用自定义样式和组件](#using-custom-styles-and-components)。

## 渲染 MDX

{/* md5:47d2abf6f3ff7f16623b73dc532fb66b */}
你可以使用 Next.js 的基于文件的路由来渲染 MDX，或者将 MDX 文件导入到其他页面中。

{/* md5:67d52a9de9d723e8a3518154e5748f9d */}
### 使用基于文件的路由

{/* md5:466dc794b70dcfcccfcaddc5f80e3c5d */}
当使用基于文件的路由时，你可以像使用任何其他页面一样使用 MDX 页面。

<AppOnly>

### 使用动态导入

{/* md5:c6c97cc05603813fc1c8b9d0afb554d3 */}
你可以导入动态的 MDX 组件，而不是为 MDX 文件使用文件系统路由。

{/* md5:268c9d09652f4128a5d03650546fcb4e */}
例如，你可以有一个动态路由段，它从单独的目录加载 MDX 组件：

{/* md5:9ccc8a90f9722bbaf3481ae4fcef83b4 */}
<Image
  alt="动态 MDX 组件的路由段"
  srcLight="/docs/light/mdx-files.png"
  srcDark="/docs/dark/mdx-files.png"
  width="1600"
  height="849"
/>

{/* md5:7a4d14129e2730b66c50db3ccc8c1aa5 */}
[`generateStaticParams`](/docs/app/api-reference/functions/generate-static-params) 可用于预渲染提供的路由。通过将 `dynamicParams` 标记为 `false`，访问未在 `generateStaticParams` 中定义的路由将返回 404。

```tsx filename="app/blog/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  const { default: Post } = await import(`@/content/${slug}.mdx`)

  return <Post />
}

export function generateStaticParams() {
  return [{ slug: 'welcome' }, { slug: 'about' }]
}

export const dynamicParams = false
```

```jsx filename="app/blog/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
  const { default: Post } = await import(`@/content/${slug}.mdx`)

  return <Post />
}

export function generateStaticParams() {
  return [{ slug: 'welcome' }, { slug: 'about' }]
}

export const dynamicParams = false
```

{/* md5:df10870bad7da907bcaa6a119e78030c */}
> **请注意**：确保在导入中指定 `.mdx` 文件扩展名。虽然不是必须使用 [模块路径别名](/docs/app/getting-started/installation#set-up-absolute-imports-and-module-path-aliases)（例如 `@/content`），但它确实可以简化你的导入路径。

## 使用自定义样式和组件

{/* md5:586bfa74091e8265e5eaaf33df425ade */}
Markdown 在渲染时映射到原生 HTML 元素。例如，编写以下 Markdown：

```md
## This is a heading

This is a list in markdown:

- One
- Two
- Three
```

{/* md5:923b11bc9fa5172bce0a39c7a311cb3b */}
生成以下 HTML：

```html
<h2>This is a heading</h2>

<p>This is a list in markdown:</p>

<ul>
  <li>One</li>
  <li>Two</li>
  <li>Three</li>
</ul>
```

{/* md5:bbfe5ca8aef86d6b335d4b45852883c0 */}
要为你的 Markdown 添加样式，你可以提供映射到生成的 HTML 元素的自定义组件。样式和组件可以在全局、局部以及使用共享布局实现。

{/* md5:1e0facbf5eb688108e0ed4cc1d289b35 */}
### 全局样式和组件

{/* md5:c63998905a2a95b35f8f2a2ca1a78acd */}
在 `mdx-components.tsx` 中添加样式和组件将影响你应用程序中的 *所有* MDX 文件。

```tsx filename="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'
import Image, { ImageProps } from 'next/image'

// This file allows you to provide custom React components
// to be used in MDX files. You can import and use any
// React component you want, including inline styles,
// components from other libraries, and more.

const components = {
  // Allows customizing built-in components, e.g. to add styling.
  h1: ({ children }) => (
    <h1 style={{ color: 'red', fontSize: '48px' }}>{children}</h1>
  ),
  img: (props) => (
    <Image
      sizes="100vw"
      style={{ width: '100%', height: 'auto' }}
      {...(props as ImageProps)}
    />
  ),
} satisfies MDXComponents

export function useMDXComponents(): MDXComponents {
  return components
}
```

```js filename="mdx-components.js" switcher
import Image from 'next/image'

// This file allows you to provide custom React components
// to be used in MDX files. You can import and use any
// React component you want, including inline styles,
// components from other libraries, and more.

const components = {
  // Allows customizing built-in components, e.g. to add styling.
  h1: ({ children }) => (
    <h1 style={{ color: 'red', fontSize: '48px' }}>{children}</h1>
  ),
  img: (props) => (
    <Image sizes="100vw" style={{ width: '100%', height: 'auto' }} {...props} />
  ),
}

export function useMDXComponents() {
  return components
}
```

### 局部样式和组件

{/* md5:1e20fcf9b54a30c9c848132fea8c6107 */}
你可以通过将它们传递给导入的 MDX 组件来将局部样式和组件应用于特定页面。这些将与[全局样式和组件](#global-styles-and-components)合并并覆盖它们。

<AppOnly>

```tsx filename="app/mdx-page/page.tsx" switcher
import Welcome from '@/markdown/welcome.mdx'

function CustomH1({ children }) {
  return <h1 style={{ color: 'blue', fontSize: '100px' }}>{children}</h1>
}

const overrideComponents = {
  h1: CustomH1,
}

export default function Page() {
  return <Welcome components={overrideComponents} />
}
```

### 使用 Tailwind 排版插件

{/* md5:70578747bd3af34e72de92317cd3d5b4 */}

{/* md5:5459cdea8900f3f3068239f3d075f058 */}

## Frontmatter

{/* md5:4c37ea22b29e8bde25e5aaf571650b3e */}
Frontmatter 是一种类似 YAML 的键/值配对，可用于存储页面数据。`@next/mdx` **默认不支持** frontmatter，但有许多解决方案可以为你的 MDX 内容添加 frontmatter，例如：

{/* md5:9e86a6553d59a40223fe896ab5fd785a */}

## remark 和 rehype 插件

{/* md5:d77286af3aa3cd0c51e0b9ad404ca4e6 */}
你可以选择提供 remark 和 rehype 插件来转换 MDX 内容。

{/* md5:7c892154d2209b32fd29cc2a9eaaee5d */}
例如，你可以使用 [`remark-gfm`](https://github.com/remarkjs/remark-gfm) 来支持 GitHub 风格的 Markdown。

{/* md5:89c48497b0d0b258e860dd839019dca8 */}
由于 remark 和 rehype 生态系统仅支持 ESM，你需要使用 `next.config.mjs` 或 `next.config.ts` 作为配置文件。

```js filename="next.config.mjs"
import remarkGfm from 'remark-gfm'
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  // Allow .mdx extensions for files
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
  // Optionally, add any other Next.js config below
}

const withMDX = createMDX({
  // Add markdown plugins here, as desired
  options: {
    remarkPlugins: [remarkGfm],
    rehypePlugins: [],
  },
})

// Combine MDX and Next.js config
export default withMDX(nextConfig)
```

### 在 Turbopack 中使用插件

{/* md5:5c74fd7f86f75ca5570c11c173b03f9d */}
要在 [Turbopack](/docs/app/api-reference/turbopack) 中使用插件，请升级到最新的 `@next/mdx` 并使用字符串指定插件名称：

```js filename="next.config.mjs"
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
}

const withMDX = createMDX({
  options: {
    remarkPlugins: [
      // Without options
      'remark-gfm',
      // With options
      ['remark-toc', { heading: 'The Table' }],
    ],
    rehypePlugins: [
      // Without options
      'rehype-slug',
      // With options
      ['rehype-katex', { strict: true, throwOnError: true }],
    ],
  },
})

export default withMDX(nextConfig)
```

## 远程 MDX

{/* md5:d9eed43ca9bfaa29e24f69c70cea2bf0 */}
如果你的 MDX 文件或内容存储在*其他地方*，你可以在服务器上动态获取它。这对于存储在 CMS、数据库或其他任何地方的内容很有用。用于此目的的一个社区包是 [`next-mdx-remote-client`](https://github.com/ipikuka/next-mdx-remote-client?tab=readme-ov-file#the-part-associated-with-nextjs-app-router)。

{/* md5:0a117fabf7f1aed420c0ca4cff464ae7 */}
> **请注意**：请谨慎操作。MDX 编译为 JavaScript 并在服务器上执行。你应该只从可信来源获取 MDX 内容，否则这可能导致远程代码执行 (RCE)。

{/* md5:c1d7eb88c7631565152633342e104a28 */}
以下示例使用 `next-mdx-remote-client`：

<AppOnly>

```tsx filename="app/mdx-page-remote/page.tsx" switcher
import { MDXRemote } from 'next-mdx-remote-client/rsc'

export default async function RemoteMdxPage() {
  // MDX text - can be from a database, CMS, fetch, anywhere...
  const res = await fetch('https://...')
  const markdown = await res.text()
  return <MDXRemote source={markdown} />
}
```


## 深入解析：如何将 Markdown 转换为 HTML？

{/* md5:6d8dc43b29687d6bdf16530ba3db7c11 */}
React 本身并不理解 Markdown。Markdown 纯文本需要先转换为 HTML。这可以通过 `remark` 和 `rehype` 来实现。

{/* md5:5676bde865b62604fa2ec857a6e1e7c0 */}
`remark` 是围绕 Markdown 的工具生态系统。`rehype` 也是如此，但用于 HTML。例如，以下代码片段将 Markdown 转换为 HTML：

```js
import { unified } from 'unified'
import remarkParse from 'remark-parse'
import remarkRehype from 'remark-rehype'
import rehypeSanitize from 'rehype-sanitize'
import rehypeStringify from 'rehype-stringify'

main()

async function main() {
  const file = await unified()
    .use(remarkParse) // Convert into markdown AST
    .use(remarkRehype) // Transform to HTML AST
    .use(rehypeSanitize) // Sanitize HTML input
    .use(rehypeStringify) // Convert AST into serialized HTML
    .process('Hello, Next.js!')

  console.log(String(file)) // <p>Hello, Next.js!</p>
}
```

{/* md5:2e89ec5fb7d2c0fd0e931b6d83727f36 */}
`remark` 和 `rehype` 生态系统包含用于[语法高亮](https://github.com/atomiks/rehype-pretty-code)、[链接标题](https://github.com/rehypejs/rehype-autolink-headings)、[生成目录](https://github.com/remarkjs/remark-toc)等的插件。

{/* md5:1e8a200415b60318331865ee3551383a */}
如上所述，当使用 `@next/mdx` 时，你**不需要**直接使用 `remark` 或 `rehype`，因为它会为你处理。我们在此描述它是为了更深入地了解 `@next/mdx` 包在底层做了什么。

{/* md5:c60a60dc700a408e8597c031d67b16e7 */}
## 使用基于 Rust 的 MDX 编译器（实验性）

{/* md5:92c95165c8a17c88b9565112f4fef682 */}
Next.js 支持一个新的用 Rust 编写的 MDX 编译器。此编译器仍处于实验阶段，不建议用于生产环境。要使用新编译器，你需要在将 `next.config.js` 传递给 `withMDX` 时进行配置：

```js filename="next.config.js"
module.exports = withMDX({
  experimental: {
    mdxRs: true,
  },
})
```

{/* md5:0233a0185cdb1e4d1bb9affbb3dfabda */}
`mdxRs` 也接受一个对象来配置如何转换 mdx 文件。

```js filename="next.config.js"
module.exports = withMDX({
  experimental: {
    mdxRs: {
      jsxRuntime?: string            // Custom jsx runtime
      jsxImportSource?: string       // Custom jsx import source,
      mdxType?: 'gfm' | 'commonmark' // Configure what kind of mdx syntax will be used to parse & transform
    },
  },
})
```
