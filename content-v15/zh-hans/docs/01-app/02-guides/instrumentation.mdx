插桩 (Instrumentation) 是使用代码将监控和日志记录工具集成到您的应用程序中的过程。这使您能够跟踪应用程序的性能和行为，并在生产环境中调试问题。

{/* md5:45dcaa5a371e659f8b3d273badd01cd9 */}
## 约定

{/* md5:f2e54c2e75a23772acb9827fc88b7c82 */}
要设置插桩 (instrumentation)，请在项目的**根目录**（如果使用了 [`src`](/docs/app/api-reference/file-conventions/src-folder) 文件夹，则在该文件夹内）创建 `instrumentation.ts|js` 文件。

{/* md5:5f3ff160f8a67332cab1975cce5a690b */}
然后，在该文件中导出一个 `register` 函数。当启动新的 Next.js 服务器实例时，该函数将被调用**一次**。

{/* md5:b3f10c28a04384a423d9c50739d5d357 */}
例如，要在 Next.js 中结合使用 [OpenTelemetry](https://opentelemetry.io/) 和 [@vercel/otel](https://vercel.com/docs/observability/otel-overview)：

```ts filename="instrumentation.ts" switcher
import { registerOTel } from '@vercel/otel'

export function register() {
  registerOTel('next-app')
}
```

```js filename="instrumentation.js" switcher
import { registerOTel } from '@vercel/otel'

export function register() {
  registerOTel('next-app')
}
```

{/* md5:3b1d338df8d779d3d8d47decde52cf66 */}
请参阅 [带有 OpenTelemetry 的 Next.js 示例](https://github.com/vercel/next.js/tree/canary/examples/with-opentelemetry) 以获取完整实现。

{/* md5:7ef683b2b3c92b7e674526b3b9567494 */}
> **须知**:
>
> - `instrumentation` 文件应位于项目的根目录，而不是 `app` 或 `pages` 目录内。如果您使用 `src` 文件夹，请将文件放在 `src` 内与 `pages` 和 `app` 同级的位置。
> - 如果您使用 [`pageExtensions` 配置选项](/docs/app/api-reference/config/next-config-js/pageExtensions) 来添加后缀，您还需要更新 `instrumentation` 的文件名以匹配该后缀。

{/* md5:e57da1656ead1413f55464d7577686d2 */}
## 示例

{/* md5:6a69a148284ca8abbe930246da6ab542 */}
### 导入有副作用的文件

{/* md5:235f35f70de04bfb63848b3b74fdc8a5 */}
有时，在代码中导入一个文件可能很有用，因为它会引起副作用。例如，您可能导入一个定义了一组全局变量的文件，但从未在代码中显式使用该导入的文件。您仍然可以访问该包已声明的全局变量。

{/* md5:49c45ce073b2fbdd1a4a9a759bdc3691 */}
我们建议在 `register` 函数内使用 JavaScript `import` 语法导入文件。以下示例演示了在 `register` 函数中 `import` 的基本用法：

```ts filename="instrumentation.ts" switcher
export async function register() {
  await import('package-with-side-effect')
}
```

```js filename="instrumentation.js" switcher
export async function register() {
  await import('package-with-side-effect')
}
```

{/* md5:9c954a25e8f1b5251d0800f495310368 */}
> **须知：**
>
> 我们建议在 `register` 函数内部导入文件，而不是在文件顶部全局导入。通过这样做，您可以将所有副作用集中在代码中的一个位置，并避免因在文件顶部全局导入而产生的任何意外后果。

{/* md5:23f1133504b64bafbd30620a52da2e8b */}
### 导入运行时特定代码

{/* md5:64f6e01b91ecbd8eaea6559a4b182bd2 */}
Next.js 会在所有环境中调用 `register`，因此，有条件地导入任何不支持特定运行时（例如 [Edge 或 Node.js](/docs/app/api-reference/edge)）的代码非常重要。您可以使用 `NEXT_RUNTIME` 环境变量来获取当前环境：

```ts filename="instrumentation.ts" switcher
export async function register() {
  if (process.env.NEXT_RUNTIME === 'nodejs') {
    await import('./instrumentation-node')
  }

  if (process.env.NEXT_RUNTIME === 'edge') {
    await import('./instrumentation-edge')
  }
}
```

```js filename="instrumentation.js" switcher
export async function register() {
  if (process.env.NEXT_RUNTIME === 'nodejs') {
    await import('./instrumentation-node')
  }

  if (process.env.NEXT_RUNTIME === 'edge') {
    await import('./instrumentation-edge')
  }
}
```
