### 布局脚本

{/* md5:22d7271d6543cfe056417412f8ef455e */}
要为多个路由加载第三方脚本，请导入 `next/script` 并直接在您的布局组件中包含该脚本：

```tsx filename="app/dashboard/layout.tsx" switcher
import Script from 'next/script'

export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <>
      <section>{children}</section>
      <Script src="https://example.com/script.js" />
    </>
  )
}
```

```jsx filename="app/dashboard/layout.js" switcher
import Script from 'next/script'

export default function DashboardLayout({ children }) {
  return (
    <>
      <section>{children}</section>
      <Script src="https://example.com/script.js" />
    </>
  )
}
```

{/* md5:ae545c2e14529d530f3f7071bfa9fbab */}
当用户访问该文件夹路由（例如 `dashboard/page.js`）或任何嵌套路由（例如 `dashboard/settings/page.js`）时，就会获取该第三方脚本。Next.js 将确保该脚本**只加载一次**，即使用户在同一布局下的多个路由之间导航也是如此。

</AppOnly>

### 应用脚本

<AppOnly>

{/* md5:1ce509ad6072cde7c6eb4ec26a9d9177 */}
要为所有路由加载第三方脚本，请导入 `next/script` 并直接在您的根布局中包含该脚本：

```tsx filename="app/layout.tsx" switcher
import Script from 'next/script'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
      <Script src="https://example.com/script.js" />
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import Script from 'next/script'

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
      <Script src="https://example.com/script.js" />
    </html>
  )
}
```

</AppOnly>

<PagesOnly>

{/* md5:deb7f2bd73450252985be4668d6f465a */}
要为所有路由加载第三方脚本，请导入 `next/script` 并直接在您的自定义 `_app` 中包含该脚本：

```jsx filename="pages/_app.js"
import Script from 'next/script'

export default function MyApp({ Component, pageProps }) {
  return (
    <>
      <Component {...pageProps} />
      <Script src="https://example.com/script.js" />
    </>
  )
}
```

</PagesOnly>

{/* md5:2abef1ea74e19d8d34725ea98185369d */}
此脚本将在访问您应用中的*任何*路由时加载和执行。Next.js 将确保该脚本**只加载一次**，即使用户在多个页面之间导航也是如此。

{/* md5:099e4911e372dc1fce4b1b5bc86015cb */}
> **建议**：我们建议仅在特定页面或布局中包含第三方脚本，以尽量减少对性能的任何不必要影响。

{/* md5:8a4dccfa46b2a35a3f9b2541dbc71743 */}
### 加载策略

{/* md5:b93fa9d8ab2402fd38d96708eb78b60c */}
尽管 `next/script` 的默认行为允许您在任何页面或布局中加载第三方脚本，但您可以通过使用 `strategy` 属性来微调其加载行为：

{/* md5:120b3d86a6faf35db6645d97081a3f2e */}
- `beforeInteractive`：在任何 Next.js 代码之前以及任何页面水合 (hydration) 发生之前加载脚本。
- `afterInteractive`：（**默认**）在页面发生部分水合后尽早加载脚本。
- `lazyOnload`：在浏览器空闲时间稍后加载脚本。
- `worker`：（实验性）在 Web Worker 中加载脚本。

{/* md5:19491110b7eeda1cbde61e1938f14863 */}
请参阅 [`next/script`](/docs/app/api-reference/components/script#strategy) API 参考文档，以了解更多关于每种策略及其用例的信息。

{/* md5:2abf123b805b3158acb1327c92663815 */}
### 将脚本卸载到 Web Worker（实验性）

{/* md5:90b5ac03297c0b5e9a27232c797a71be */}
> **警告：** `worker` 策略尚不稳定，并且尚不能与应用路由 (App Router) 一起使用。请谨慎使用。

{/* md5:271ee56043ebe853c8a1dfdfb27a3fbc */}
使用 `worker` 策略的脚本会被卸载并在一个使用 [Partytown](https://partytown.builder.io/) 的 Web Worker 中执行。这可以通过将主线程专用于您应用的其余代码来提高您站点的性能。

{/* md5:86bd4912e111d4caf7b44781e0c09a97 */}
此策略仍处于实验阶段，只有在 `next.config.js` 中启用了 `nextScriptWorkers` 标志时才能使用：

```js filename="next.config.js"
module.exports = {
  experimental: {
    nextScriptWorkers: true,
  },
}
```

{/* md5:d9a951653dc82182c2c264c5acff6d75 */}
然后，运行 `next`（通常是 `npm run dev` 或 `yarn dev`），Next.js 将引导您完成所需包的安装以完成设置：

```bash filename="Terminal"
npm run dev
```

{/* md5:0bd5e956415932ec79764132dc4f988f */}
您会看到类似这样的指示：请通过运行 `npm install @builder.io/partytown` 来安装 Partytown

{/* md5:28f3bc95545a5eac49a89f1aa9606187 */}
设置完成后，定义 `strategy="worker"` 将在您的应用程序中自动实例化 Partytown 并将脚本卸载到 Web Worker。

```tsx filename="pages/home.tsx" switcher
import Script from 'next/script'

export default function Home() {
  return (
    <>
      <Script src="https://example.com/script.js" strategy="worker" />
    </>
  )
}
```

```jsx filename="pages/home.js" switcher
import Script from 'next/script'

export default function Home() {
  return (
    <>
      <Script src="https://example.com/script.js" strategy="worker" />
    </>
  )
}
```

{/* md5:89fe3e80ee13ddb1478b67db931b8c5b */}
在 Web Worker 中加载第三方脚本时，需要考虑一些权衡取舍。请参阅 Partytown 的 [tradeoffs](https://partytown.builder.io/trade-offs) 文档以获取更多信息。

#### 使用自定义 Partytown 配置

{/* md5:7eca07ce0de534d1a0540e584c3a620b */}
尽管 `worker` 策略不需要任何额外的配置即可工作，但 Partytown 支持使用配置对象来修改其部分设置，包括启用 `debug` 模式以及转发事件和触发器。

{/* md5:bf7a81f52a686c3d1cf6ee85fa44c183 */}
如果您想添加额外的配置选项，可以在 [自定义 `_document.js`](/docs/pages/building-your-application/routing/custom-document) 中使用的 `<Head />` 组件内包含它：

```jsx filename="_pages/document.jsx"
import { Html, Head, Main, NextScript } from 'next/document'

export default function Document() {
  return (
    <Html>
      <Head>
        <script
          data-partytown-config
          dangerouslySetInnerHTML={{
            __html: `
              partytown = {
                lib: "/_next/static/~partytown/",
                debug: true
              };
            `,
          }}
        />
      </Head>
      <body>
        <Main />
        <NextScript />
      </body>
    </Html>
  )
}
```

{/* md5:9d3375aefd16a549b0142ce805debd42 */}
为了修改 Partytown 的配置，必须满足以下条件：

{/* md5:b25d0fb1406dc741dfabfba9ff20e7a3 */}
1.  必须使用 `data-partytown-config` 属性才能覆盖 Next.js 使用的默认配置
2.  除非您决定将 Partytown 的库文件保存在单独的目录中，否则必须在配置对象中包含 `lib: "/_next/static/~partytown/"` 属性和值，以让 Partytown 知道 Next.js 将必要的静态文件存储在何处。

{/* md5:b6a58b71745d155fc488121342e9e813 */}
> **注意**：如果您正在使用 [资源前缀 (asset prefix)](/docs/pages/api-reference/config/next-config-js/assetPrefix) 并且想要修改 Partytown 的默认配置，则必须将其作为 `lib` 路径的一部分包含在内。

{/* md5:23ee56f2e619eb703904211c87699ba4 */}
查看 Partytown 的 [配置选项](https://partytown.builder.io/configuration) 以查看可以添加的其他属性的完整列表。

</PagesOnly>

### 内联脚本

{/* md5:e54ce27f030e2a9a1289c2aafe8fd922 */}
Script 组件也支持内联脚本，即不从外部文件加载的脚本。可以通过将 JavaScript 放在大括号内来编写它们：

```jsx
<Script id="show-banner">
  {`document.getElementById('banner').classList.remove('hidden')`}
</Script>
```

{/* md5:1b63474dc087e6c2ff585ca0993a73ca */}
或者通过使用 `dangerouslySetInnerHTML` 属性：

```jsx
<Script
  id="show-banner"
  dangerouslySetInnerHTML={{
    __html: `document.getElementById('banner').classList.remove('hidden')`,
  }}
/>
```

{/* md5:fdec4cf2fd2b600a4af8b337d667278c */}
> **警告**：必须为内联脚本分配一个 `id` 属性，以便 Next.js 跟踪和优化该脚本。

{/* md5:f0c4633ea602c413bc88df83a4b3fe28 */}
### 执行额外代码

{/* md5:6321ec0a31996108e5a25b7c48791efc */}
事件处理器可以与 Script 组件一起使用，以在某个事件发生后执行额外的代码：

{/* md5:8052291a61a9b9f686a5dd716f38c208 */}
- `onLoad`：在脚本加载完成后执行代码。
- `onReady`：在脚本加载完成后以及每次组件挂载时执行代码。
- `onError`：如果脚本加载失败则执行代码。

<AppOnly>

{/* md5:46c9be216e63a2bd0a0f70e645b27f6d */}
这些处理器仅在 `next/script` 被导入并用于 [客户端组件 (Client Component)](/docs/app/getting-started/server-and-client-components) 内部时才有效，其中 `"use client"` 被定义为代码的第一行：

```tsx filename="app/page.tsx" switcher
'use client'

import Script from 'next/script'

export default function Page() {
  return (
    <>
      <Script
        src="https://example.com/script.js"
        onLoad={() => {
          console.log('Script has loaded')
        }}
      />
    </>
  )
}
```


### 附加属性

{/* md5:17e97947c0b63316834c20ed3ee9d403 */}
有许多 DOM 属性可以分配给 `<script>` 元素，而 Script 组件并未使用它们，例如 [`nonce`](https://developer.mozilla.org/docs/Web/HTML/Global_attributes/nonce) 或 [自定义数据属性](https://developer.mozilla.org/docs/Web/HTML/Global_attributes/data-*)。包含任何附加属性都将自动将其转发到包含在 HTML 中的最终、优化后的 `<script>` 元素。

<AppOnly>

```tsx filename="app/page.tsx" switcher
import Script from 'next/script'

export default function Page() {
  return (
    <>
      <Script
        src="https://example.com/script.js"
        id="example-script"
        nonce="XUENAJFW"
        data-test="script"
      />
    </>
  )
}
```

```jsx filename="app/page.js" switcher
import Script from 'next/script'

export default function Page() {
  return (
    <>
      <Script
        src="https://example.com/script.js"
        id="example-script"
        nonce="XUENAJFW"
        data-test="script"
      />
    </>
  )
}
```

</AppOnly>
