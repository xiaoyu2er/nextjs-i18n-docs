## 从 13 升级至 14

要更新到 Next.js 版本 14，请使用你偏好的包管理器运行以下命令：

```bash filename="Terminal"
npm i next@next-14 react@18 react-dom@18 && npm i eslint-config-next@next-14 -D
```

```bash filename="Terminal"
yarn add next@next-14 react@18 react-dom@18 && yarn add eslint-config-next@next-14 -D
```

```bash filename="Terminal"
pnpm i next@next-14 react@18 react-dom@18 && pnpm i eslint-config-next@next-14 -D
```

```bash filename="Terminal"
bun add next@next-14 react@18 react-dom@18 && bun add eslint-config-next@next-14 -D
```

> **请注意：** 如果你正在使用 TypeScript，请确保同时将 `@types/react` 和 `@types/react-dom` 升级到其最新版本。

### v14 版本概要

- 最低 Node.js 版本已从 16.14 提升至 18.17，因为 16.x 已达到生命周期终点。
- `next export` 命令已被移除，取而代之的是 `output: 'export'` 配置。更多信息请参阅[文档](https://nextjs.org/docs/app/guides/static-exports)。
- 用于 `ImageResponse` 的 `next/server` 导入已更名为 `next/og`。可以使用 [codemod](/docs/app/guides/upgrading/codemods#next-og-import) 来安全、自动地重命名你的导入。
- `@next/font` 包已被完全移除，取而代之的是内置的 `next/font`。可以使用 [codemod](/docs/app/guides/upgrading/codemods#built-in-next-font) 来安全、自动地重命名你的导入。
- `next-swc` 的 WASM 目标已被移除。
