## 设置 CDN

要设置 [CDN](https://en.wikipedia.org/wiki/Content_delivery_network)，您可以设置资产前缀，并将 CDN 的源服务器配置为解析到托管 Next.js 的域名。

打开 `next.config.mjs`，并基于 [阶段 (phase)](/docs/app/api-reference/config/next-config-js#async-configuration) 添加 `assetPrefix` 配置：

```js filename="next.config.mjs"
// @ts-check
import { PHASE_DEVELOPMENT_SERVER } from 'next/constants'

export default (phase) => {
  const isDev = phase === PHASE_DEVELOPMENT_SERVER
  /**
   * @type {import('next').NextConfig}
   */
  const nextConfig = {
    assetPrefix: isDev ? undefined : 'https://cdn.mydomain.com',
  }
  return nextConfig
}
```

Next.js 将自动在从其 `/_next/` 路径（`.next/static/` 文件夹）加载的 JavaScript 和 CSS 文件中使用您的资产前缀。例如，使用上述配置，以下 JS 块的请求：

```
/_next/static/chunks/4b9b41aaa062cbbfeff4add70f256968c51ece5d.4d708494b3aed70c04f0.js
```

则会变为：

```
https://cdn.mydomain.com/_next/static/chunks/4b9b41aaa062cbbfeff4add70f256968c51ece5d.4d708494b3aed70c04f0.js
```

将文件上传到特定 CDN 的确切配置将取决于您选择的 CDN。您需要在 CDN 上托管的唯一年份夹是 `.next/static/` 的内容，根据上面的 URL 请求显示，应将其上传为 `_next/static/`。**请上传 `.next/` 文件夹的其余部分**（Wait, "Do not upload" -> `不要上传`），因为您不应向公众暴露您的服务器代码和其他配置。

虽然 `assetPrefix` 覆盖了前往 `_next/static` 的请求，但它不影响以下路径：

<AppOnly>

- [public](/docs/app/api-reference/file-conventions/public-folder) 文件夹中的文件；如果您想通过 CDN 托管这些资产，则需要自己引入前缀

</AppOnly>

<PagesOnly>

- [public](/docs/pages/api-reference/file-conventions/public-folder) 文件夹中的文件；如果您想通过 CDN 托管这些资产，则需要自己引入前缀
- `getServerSideProps` 页面的 `/_next/data/` 请求。由于这些请求不是静态的，它们将始终针对主域发出。
- `getStaticProps` 页面的 `/_next/data/` 请求。为了支持 [增量静态生成 (Incremental Static Generation)](/docs/pages/guides/incremental-static-regeneration)，即使您未使用它（为了保持一致性），这些请求也将始终针对主域发出。

</PagesOnly>
