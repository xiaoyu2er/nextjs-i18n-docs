description: 配置在查看当前路由时提供开发上下文的屏幕指示器 (On-screen Indicator) 选项。
---

`devIndicators` 允许您配置屏幕指示器，该指示器可在开发过程中为您提供当前路由的上下文信息。

```ts filename="Types"
  devIndicators: false | {
    position?: 'bottom-right'
    | 'bottom-left'
    | 'top-right'
    | 'top-left', // defaults to 'bottom-left',
  },
```

将 `devIndicators` 设置为 `false` 会隐藏该指示器，但 Next.js 仍会继续显示任何遇到的构建或运行时错误。

## 故障排除

### 指示器未将路由标记为静态 (Static)

如果您期望某个路由是静态 (Static) 的，但指示器将其标记为动态 (Dynamic)，则该路由可能已选择退出静态渲染 (Static Rendering)。

您可以使用 `next build --debug` 构建应用程序并检查终端中的输出来确认路由是 [静态 (Static)](/docs/app/getting-started/partial-prerendering#static-rendering) 还是 [动态 (Dynamic)](/docs/app/getting-started/partial-prerendering#dynamic-rendering)。静态 (Static)（或预渲染）路由将显示 ○ 符号，而动态 (Dynamic) 路由将显示 ƒ 符号。例如：

```bash filename="Build Output"
Route (app)                              Size     First Load JS
┌ ○ /_not-found                          0 B               0 kB
└ ƒ /products/[id]                       0 B               0 kB

○  (Static)   prerendered as static content
ƒ  (Dynamic)  server-rendered on demand
```

<AppOnly>

路由选择退出静态 (Static) 渲染有两个原因：

- 存在依赖运行时信息的 [动态 (Dynamic) API](/docs/app/getting-started/partial-prerendering#dynamic-rendering)。
- 存在 [未缓存 (Uncached) 的数据请求](/docs/app/getting-started/fetching-data)，例如对 ORM 或数据库驱动的调用。

检查您的路由是否存在这些情况，如果您无法静态渲染该路由，则考虑使用 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 或 [`<Suspense />`](https://react.dev/reference/react/Suspense) 来利用 [流式传输 (Streaming)](/docs/app/getting-started/linking-and-navigating#streaming)。

</AppOnly>

<PagesOnly>

从页面导出 [`getServerSideProps`](/docs/pages/building-your-application/data-fetching/get-server-side-props) 或 [`getInitialProps`](/docs/pages/api-reference/functions/get-initial-props) 时，它将被标记为动态 (Dynamic)。

</PagesOnly>
