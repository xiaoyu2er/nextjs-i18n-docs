Next.js 中的 `cacheComponents` 标志是一项实验性功能，它会导致 App Router 中的数据获取操作除非被明确缓存，否则会被排除在预渲染之外。这对于优化 Server Components 中动态数据获取的性能很有用。

如果您的应用程序需要在运行时获取实时数据，而不是从预渲染缓存中提供服务，那么此功能将非常有用。

预期与 [`use cache`](/docs/app/api-reference/directives/use-cache) 配合使用，默认情况下您的数据获取将在运行时发生，除非您使用 `use cache` 在页面、函数或组件级别定义应用程序的特定部分进行缓存。

## 使用方法

要启用 `cacheComponents` 标志，请在您的 `next.config.ts` 文件的 `experimental` 部分将其设置为 `true`：

```ts filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    cacheComponents: true,
  },
}

export default nextConfig
```

```ts filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    cacheComponents: true,
  },
}

export default nextConfig
```

当启用 `cacheComponents` 时，您可以使用以下缓存函数和配置：

- [`use cache` 指令](/docs/app/api-reference/directives/use-cache)
- 与 `use cache` 配合使用的 [`cacheLife` 函数](/docs/app/api-reference/config/next-config-js/cacheLife)
- [`cacheTag` 函数](/docs/app/api-reference/functions/cacheTag)

- [`use cache` 指令](/docs/app/api-reference/directives/use-cache)
- 与 `use cache` 配合使用的 [`cacheLife` 函数](/docs/app/api-reference/config/next-config-js/cacheLife)
- [`cacheTag` 函数](/docs/app/api-reference/functions/cacheTag)

## 注意

- 尽管 `cacheComponents` 可以通过确保运行时获取实时数据来优化性能，但与提供预渲染内容相比，它也可能引入额外的延迟。
