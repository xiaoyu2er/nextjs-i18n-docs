The `cacheComponents` flag 是 Next.js 中的一个实验性功能，除非明确缓存，否则它会导致 App Router 中的数据获取操作被排除在预渲染之外。这对于优化 Server Components 中动态数据获取的性能很有用。

如果您的应用需要在运行时获取最新数据，而不是从预渲染的缓存中提供内容，那么它非常有用。

预计应将其与 [`use cache`](/docs/app/api-reference/directives/use-cache) 配合使用，以便默认情况下您的数据获取在运行时发生，除非您使用 `use cache` 在页面、函数或组件级别定义应用程序的特定部分进行缓存。

## 使用方法

要启用 `cacheComponents` 标志，请在 `next.config.ts` 文件的 `experimental` 部分将其设置为 `true`：

```ts filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    cacheComponents: true,
  },
}

export default nextConfig
```

启用 `cacheComponents` 后，您可以使用以下缓存函数和配置：

- [`use cache` 指令](/docs/app/api-reference/directives/use-cache)
- 与 `use cache` 配合使用的 [`cacheLife` 函数](/docs/app/api-reference/config/next-config-js/cacheLife)
- [`cacheTag` 函数](/docs/app/api-reference/functions/cacheTag)

## 注意

- 虽然 `cacheComponents` 可以通过确保在运行时获取最新数据来优化性能，但相比于提供预渲染的内容，它也可能引入额外的延迟。
