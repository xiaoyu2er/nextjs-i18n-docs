---

`staleTimes` 是一个实验性功能，它允许在 [客户端路由缓存 (client-side router cache)](/docs/app/guides/caching#client-side-router-cache) 中缓存页面片段。

您可以通过设置实验性的 `staleTimes` 标志来启用此实验性功能并提供自定义的重新验证时间：

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

`static` 和 `dynamic` 属性对应于根据不同类型的[链接预取](/docs/app/api-reference/components/link#prefetch)所设定的时间段（以秒为单位）。

- `dynamic` 属性用于页面既未静态生成也未完全预取时（例如使用 `prefetch={true}`）。
  - 默认值：0 秒（不缓存）
- `static` 属性用于静态生成页面，或者 `Link` 的 `prefetch` 属性设置为 `true` 时，或者调用 [`router.prefetch`](/docs/app/guides/caching#routerprefetch) 时。
  - 默认值：5 分钟

> **值得了解：**
>
> - [`loading 边界文件`](/docs/app/api-reference/file-conventions/loading) 被视为在此配置中定义的 `static` 周期内可重用。
> - 这不影响 [部分渲染](/docs/app/getting-started/linking-and-navigating#client-side-transitions)，**这意味着共享布局不会在每次导航时自动重新获取，只有变化的页面片段会。**
> - 这不会更改 [前进/后退缓存 (back/forward caching)](/docs/app/guides/caching#client-side-router-cache) 行为，以防止布局偏移并避免浏览器滚动位置丢失。

您可以在此处了解更多关于 [客户端路由缓存 (Client Router Cache)](/docs/app/guides/caching#client-side-router-cache) 的信息。

### 版本历史

| 版本      | 变更                                                        |
| --------- | ----------------------------------------------------------- |
| `v15.0.0` | `dynamic` `staleTimes` 的默认值从 30 秒更改为 0 秒。         |
| `v14.2.0` | 引入了实验性的 `staleTimes` 功能。                           |
