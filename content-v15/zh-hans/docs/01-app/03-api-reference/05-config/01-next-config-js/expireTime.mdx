您可以为启用 ISR 的页面指定自定义的 `stale-while-revalidate` 过期时间，供 CDN 在 `Cache-Control` 头中使用。

打开 `next.config.js` 并添加 `expireTime` 配置：

```js filename="next.config.js"
module.exports = {
  // one hour in seconds
  expireTime: 3600,
}
```

现在，发送 `Cache-Control` 头时，过期时间将根据特定的 revalidate 周期计算。

例如，如果某路径的 revalidate 设置为 15 分钟，且过期时间为 1 小时，生成的 `Cache-Control` 头将包含 `s-maxage=900, stale-while-revalidate=2700`，使其能够在配置的过期时间减去 15 分钟后仍保持陈旧。
