> **请注意**：从 Next.js 13.5.1 开始，Strict Mode 在使用 `app` 路由时默认为 `true`，所以上面的配置仅适用于 `pages`。你仍然可以通过设置 `reactStrictMode: false` 来禁用 Strict Mode。

> **建议**：我们强烈建议你在 Next.js 应用中启用 Strict Mode，以便更好地为 React 的未来做好准备。

React 的 [Strict Mode](https://react.dev/reference/react/StrictMode) 是一项仅用于开发模式的功能，用于突出显示应用程序中潜在的问题。它有助于识别不安全的生命周期、遗留 API 的使用以及其他诸多功能。

Next.js 运行时已符合 Strict Mode 标准。要启用 Strict Mode，请在你的 `next.config.js` 中配置以下选项：

```js filename="next.config.js"
module.exports = {
  reactStrictMode: true,
}
```

如果你或你的团队尚未准备好在整个应用程序中使用 Strict Mode，也完全可以！你可以使用 `<React.StrictMode>` 来按页面逐步迁移。
