Next.js 在开发过程中不会自动阻止跨域请求，但在未来主版本中默认将阻止，以防止未经授权访问仅在开发模式下可用的内部资源/端点。

要配置 Next.js 应用程序以允许来自服务器初始化时设置的主机名（默认为 `localhost`）之外的源进行请求，你可以使用 `allowedDevOrigins` 配置选项。

`allowedDevOrigins` 允许你设置可在开发模式中使用的额外源。例如，若要使用 `local-origin.dev`（而非仅 `localhost`），请打开 `next.config.js` 并添加 `allowedDevOrigins` 配置：

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```
