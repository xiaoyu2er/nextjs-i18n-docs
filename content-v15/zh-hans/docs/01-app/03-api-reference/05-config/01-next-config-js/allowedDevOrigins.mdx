在开发期间，Next.js 不会自动阻止跨域请求，但为了防止未经授权请求开发模式下可用的内部资源/端点，Next.js 在未来的主要版本中将默认阻止此类请求。

若要配置 Next.js 应用，使其允许来自服务器初始化时主机名（默认为 localhost）之外的其他源发起请求，您可以使用 `allowedDevOrigins` 配置选项。

`allowedDevOrigins` 允许您设置可以在开发模式中使用的其他源。例如，若要使用 `local-origin.dev` 替代仅使用 `localhost`，请打开 `next.config.js` 并添加 `allowedDevOrigins` 配置：

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```
