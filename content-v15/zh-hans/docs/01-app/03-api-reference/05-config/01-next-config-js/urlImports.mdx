URL 导入是一项实验性功能，允许您直接从外部服务器（而非本地磁盘）导入模块。

> **警告**：仅使用您信任的域名来下载并在您的机器上执行代码。在该功能被标记为稳定版之前，请谨慎使用，保持警惕。

要启用此功能，请在 `next.config.js` 中添加允许的 URL 前缀：

```js filename="next.config.js"
module.exports = {
  experimental: {
    urlImports: ['https://example.com/assets/', 'https://cdn.skypack.dev'],
  },
}
```

然后，您就可以直接从 URL 导入模块了：

```js
import { a, b, c } from 'https://example.com/assets/some/module.js'
```

URL 导入可以在所有可以使用常规包导入的地方使用。

## 安全模型

此功能的设计将**安全性放在首位**。首先，我们添加了一个实验性标志，要求您明确允许接受 URL 导入的域名。我们正致力于通过限制 URL 导入在 [Edge Runtime](/docs/app/api-reference/edge) 的浏览器沙箱中执行来进一步增强安全性。

## 锁定文件

当使用 URL 导入时，Next.js 将创建一个包含锁定文件和已获取资源的 `next.lock` 目录。此目录**必须提交到 Git**，不能被 `.gitignore` 忽略。

- 当运行 `next dev` 时，Next.js 将下载所有新发现的 URL 导入并将其添加到您的锁定文件中。
- 当运行 `next build` 时，Next.js 将仅使用锁定文件来构建生产环境的应用程序。

通常情况下，不需要任何网络请求，任何过时的锁定文件都会导致构建失败。一个例外是响应头为 `Cache-Control: no-cache` 的资源。这些资源在锁定文件中会有一个 `no-cache` 条目，并且在每次构建时都会从网络获取。

## 示例

### Skypack

```js
import confetti from 'https://cdn.skypack.dev/canvas-confetti'
import { useEffect } from 'react'

export default () => {
  useEffect(() => {
    confetti()
  })
  return <p>Hello</p>
}
```

### 静态图像导入

```js
import Image from 'next/image'
import logo from 'https://example.com/assets/logo.png'

export default () => (
  <div>
    <Image src={logo} placeholder="blur" />
  </div>
)
```

### CSS 中的 URL

```css
.className {
  background: url('https://example.com/assets/hero.jpg');
}
```

### 资源导入

```js
const logo = new URL('https://example.com/assets/file.txt', import.meta.url)

console.log(logo.pathname)

// prints "/_next/static/media/file.a9727b5d.txt"
```
