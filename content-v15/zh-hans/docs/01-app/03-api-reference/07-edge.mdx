Next.js 有两种服务器运行时 (server runtime) 可供您在应用程序中使用：

{/* md5:fadf5982ad33ecae0ce584940fcaaf07 */}
- **Node.js 运行时 (Node.js Runtime)**（默认），它可以访问所有 Node.js API，用于渲染您的应用程序。
- **边缘运行时 (Edge Runtime)**，它包含一组更有限的 [API](#reference)，用于 [中间件 (Middleware)](/docs/app/api-reference/file-conventions/middleware)。

{/* md5:7cbd511c4ab51a1f1f7835b0deb23e60 */}
## 注意事项

{/* md5:00ef802d33b29e67488f9b6a2262a2e8 */}
- 边缘运行时不支持所有 Node.js API。某些包可能无法按预期工作。
- 边缘运行时支持增量静态再生 (Incremental Static Regeneration, ISR)。
- 根据您的部署适配器，两种运行时都可以支持 [流式传输 (streaming)](/docs/app/api-reference/file-conventions/loading)。

{/* md5:c3399052227a4ce4d5bc2084381aac1f */}

### 不支持的 API

{/* md5:3a9af02ae6a2ae4a851f226b6cc706bb */}
边缘运行时有一些限制，包括：

{/* md5:b75b33f8db510315f1a5ddab59aa96c9 */}
- **不支持**原生 Node.js API。例如，您无法读取或写入文件系统。
- `node_modules` **可以**使用，只要它们实现 ES 模块且不使用原生 Node.js API。
- 直接调用 `require` 是**不允许**的。请改用 ES 模块。

{/* md5:a6319a9f66599b1fc64851bdd4c1b418 */}
以下 JavaScript 语言特性被禁用，并且**将无法工作**：

{/* md5:6eab7c8887fed829cb50f59f5dad6a95 */}
| API                                                                                                                             | 描述                                     |
| ------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------- |
| [`eval`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/eval)                                       | 计算表示为字符串的 JavaScript 代码       |
| [`new Function(evalString)`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Function)               | 创建一个将代码作为参数提供的新函数       |
| [`WebAssembly.compile`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/compile)         | 从缓冲区源编译 WebAssembly 模块          |
| [`WebAssembly.instantiate`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/instantiate) | 从缓冲区源编译并实例化 WebAssembly 模块  |

{/* md5:0a6ab0a1ea5f0f4be33f38504788d163 */}
在极少数情况下，您的代码可能包含（或导入）一些动态代码评估语句，这些语句**在运行时无法到达**，并且无法通过树摇 (treeshaking) 移除。您可以通过中间件配置放宽检查，以允许特定文件：

```javascript filename="middleware.ts"
export const config = {
  unstable_allowDynamic: [
    // allows a single file
    '/lib/utilities.js',
    // use a glob to allow anything in the function-bind 3rd party module
    '**/node_modules/function-bind/**',
  ],
}
```

{/* md5:b3a9f3fdf240fa57c1b0cb9169810a72 */}
`unstable_allowDynamic` 是一个 [glob](https://github.com/micromatch/micromatch#matching-features) 或 glob 数组，用于忽略特定文件的动态代码评估。glob 模式相对于您的应用程序根文件夹。

{/* md5:49b72b631a7f61d5a3e7abd8f68fd1e7 */}
请注意，如果这些语句在边缘上执行，**它们将抛出错误并导致运行时错误**。
