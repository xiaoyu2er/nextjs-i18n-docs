Turbopack 是一个为 **JavaScript** 和 **TypeScript** 优化的**增量打包器**，用 **Rust** 编写，并内置于 **Next.js** 中。您可以在 Pages Router 和 App Router 中使用 Turbopack，以获得**快得多**的本地开发体验。

{/* md5:0391689e686dce093f01225dbb93db1b */}
## 为什么选择 Turbopack?

{/* md5:5281fa82c0ffebd5014a4080e39ae79c */}
我们构建 Turbopack 是为了推动 Next.js 的性能，包括：

{/* md5:f11e2486492e64a84356f9f6a91dfcf7 */}
- **统一图谱：** Next.js 支持多种输出环境（例如客户端和服务器）。管理多个编译器并将打包结果拼接在一起可能很繁琐。Turbopack 为所有环境使用**单一、统一的图谱**。
- **打包 vs 原生 ESM：** 一些工具在开发中跳过打包，依赖浏览器的原生 ESM。这对于小型应用程序效果很好，但由于过多的网络请求，可能会减慢大型应用程序的速度。Turbopack 在开发中进行**打包**，但采用优化方式以保持大型应用程序的快速。
- **增量计算：** Turbopack 将工作并行化到多个核心上，并将结果**缓存**到函数级别。一旦某项工作完成，Turbopack 就不会重复执行。
- **延迟打包：** Turbopack 仅打包开发服务器实际请求的内容。这种延迟方法可以减少初始编译时间和内存使用。

{/* md5:da8327dd95c41f51c1dea5e58a4d1d17 */}
## 快速开始

{/* md5:5f34bf230ded6cce5f0c1a022473a157 */}
要在您的 Next.js 项目中启用 Turbopack，请在 `package.json` 文件的 `dev` 和 `build` 脚本中添加 `--turbopack` 标志：

```json filename="package.json" highlight={3}
{
  "scripts": {
    "dev": "next dev --turbopack",
    "build": "next build --turbopack",
    "start": "next start"
  }
}
```

{/* md5:af19eb5ee3711a8a910036a57fd51250 */}
目前，`dev` 的 Turbopack 是稳定的，而 `build` 处于 Beta 阶段。随着 Turbopack 趋向稳定，我们正在积极开发生产环境支持。

{/* md5:b2d2e25ff786b50e04136f1f42f93729 */}

## 与 webpack 的已知差距

{/* md5:153391dd6af2455e1a9f7f709e23807a */}
在迁移应用程序时，了解 webpack 和 Turbopack 之间许多重要的行为差异非常重要。通常，对于新应用程序来说，这些差异不太重要。

{/* md5:07ed0adb8ce24fd3b8018ebc52264b75 */}
### CSS 模块排序

{/* md5:177a8d681950053d4eaa08d5863736d7 */}
Turbopack 将遵循 JS 导入顺序来排列那些没有其他顺序的 [CSS modules](/docs/app/getting-started/css#css-modules)。例如：

```jsx filename="components/BlogPost.jsx"
import utilStyles from './utils.module.css'
import buttonStyles from './button.module.css'
export default function BlogPost() {
  return (
    <div className={utilStyles.container}>
      <button className={buttonStyles.primary}>Click me</button>
    </div>
  )
}
```

{/* md5:7e24f6a8f1f0efa50c30e360cd0ac2e8 */}
在此示例中，Turbopack 将确保在生成的 CSS 块中，`utils.module.css` 出现在 `button.module.css` 之前，遵循导入顺序。

{/* md5:d693591b7132196ca66a77ecf2407a39 */}
Webpack 通常也会这样做，但在某些情况下会忽略 JS 推断的顺序，例如如果它推断 JS 文件是无副作用的。

{/* md5:3157c9f70b7dc4ce3adc103d2ffe2341 */}
当采用 Turbopack 时，如果应用程序依赖于任意顺序，这可能会导致细微的渲染变化。通常，解决方案很简单，例如让 `button.module.css` `@import utils.module.css` 来强制顺序，或者识别冲突的规则并更改它们，使它们不针对相同的属性。

{/* md5:7ca053f4a59c474f8b5795fdcf574e82 */}
### 包体积

{/* md5:22d0f6620a5e2addd68879f6b3985638 */}
Turbopack 还没有与 webpack 中的 [Inner Graph Optimization](https://webpack.js.org/configuration/optimization/#optimizationinnergraph) 等效的功能。这种优化对于摇树优化大型模块很有用。例如：

```js filename=large.module.js
import heavy from 'some-heavy-dependency.js'

export function usesHeavy() {
  return heavy.run()
}

export const CONSTANT_VALUE = 3
```

{/* md5:719a780e55ee587bfb45c107c44176e1 */}
如果应用程序只使用 `CONSTANT_VALUE`，Turbopack 会检测到这一点并删除 `usesHeavy` 导出，但不会删除相应的 `import`。但是，启用 `optimization.innerGraph = true` 选项后，webpack 也可以删除 `import`。

{/* md5:e6534f7c8cf0492bd1c1fa3bf9ad0a45 */}
我们计划在 Turbopack 中提供与 `innerGraph` 优化等效的功能，但它仍在开发中。如果您受到此差距的影响，请考虑手动拆分这些模块。

{/* md5:0a6376a3576c2ceb2d642eb0b403cf34 */}
### 构建缓存

{/* md5:ed450fb577b6ba8df87dbf70f5ede766 */}
Webpack 支持[磁盘构建缓存](https://webpack.js.org/configuration/cache/#cache)以加速构建。我们计划在 Turbopack 中支持类似的功能，但它还没有准备好。在 `next@canary` 版本中，您可以通过启用 [`experimental.turbopackPersistentCaching` 标志](/docs/app/api-reference/config/next-config-js/turbopackPersistentCaching) 来尝试我们的解决方案。

{/* md5:d078a9af22d8d45a2d0a39ea29cd5149 */}
> **注意：** 出于这个原因，在比较 webpack 和 Turbopack 性能时，请确保在构建之间删除 `.next` 文件夹，以获得公平的比较。

{/* md5:c85bb20ec2e14d172dc59135583cbd19 */}
### Webpack 插件

{/* md5:beefcb93047352f66d78f239d6ed98d5 */}
Turbopack 不支持 webpack 插件。这会影响依赖 webpack 插件系统进行集成的第三方工具。我们支持 [webpack 加载器](/docs/app/api-reference/config/next-config-js/turbopack#configuring-webpack-loaders)。如果您依赖 webpack 插件，则需要寻找 Turbopack 兼容的替代方案，或者继续使用 webpack，直到可用等效功能。

{/* md5:fd4253855af14a53c6b7ea73f1f5c958 */}
## 不支持且未计划的功能

{/* md5:f9e527b45152a2f7efb7ca2b7943682f */}
一些功能尚未实现或未计划：

{/* md5:39faa47c0ee5cbe088a9855388ea5a90 */}
- **遗留 CSS 模块功能**
  - 独立的 `:local` 和 `:global` 伪类（仅支持函数变体 `:global(...)`）。
  - `@value` 规则（已被 CSS 变量取代）。
  - `:import` 和 `:export` ICSS 规则。
  - `.module.css` 中组合 `.css` 文件的 `composes`。在 webpack 中，这会将 `.css` 文件视为 CSS 模块，而在 Turbopack 中，`.css` 文件将始终是全局的。这意味着，如果您想在 CSS 模块中使用 `composes`，您需要将 `.css` 文件更改为 `.module.css` 文件。
  - CSS 模块中的 `@import` 将 `.css` 作为 CSS 模块导入。在 webpack 中，这会将 `.css` 文件视为 CSS 模块，而在 Turbopack 中，`.css` 文件将始终是全局的。这意味着，如果您想在 CSS 模块中使用 `@import`，您需要将 `.css` 文件更改为 `.module.css` 文件。
- **`next.config.js` 中的 `webpack()` 配置**
  Turbopack 取代了 webpack，因此不识别 `webpack()` 配置。请改用 [`turbopack` 配置](/docs/app/api-reference/config/next-config-js/turbopack)。
- **AMP**
  未计划在 Next.js 中支持 Turbopack。
- **Yarn PnP**
  未计划在 Next.js 中支持 Turbopack。
- **`experimental.urlImports`**
  不支持 Turbopack。
- **`experimental.esmExternals`**
  不支持。Turbopack 不支持 Next.js 中遗留的 `esmExternals` 配置。
- **一些 Next.js 实验性标志**
  - `experimental.nextScriptWorkers`
  - `experimental.sri.algorithm`
  - `experimental.fallbackNodePolyfills`
    我们计划将来实现这些。

{/* md5:f5de8a20cf6b72bda82748d4493f8a76 */}
有关每个功能标志及其状态的完整详细细分，请参阅 [Turbopack API 参考](/docs/app/api-reference/config/next-config-js/turbopack)。

{/* md5:4de9ac2b7349e74e6eeddaa69cdc8303 */}
## 配置

{/* md5:6df919eb4ea654671890f0ca1061d6c2 */}
可以通过 `next.config.js`（或 `next.config.ts`）中的 `turbopack` 键来配置 Turbopack。配置选项包括：

{/* md5:20e3fdc2a7972c71db9be83a990dfa46 */}
- **`rules`**
  定义额外的 [webpack 加载器](/docs/app/api-reference/config/next-config-js/turbopack#configuring-webpack-loaders) 用于文件转换。
- **`resolveAlias`**
  创建手动别名（类似于 webpack 中的 `resolve.alias`）。
- **`resolveExtensions`**
  更改或扩展用于模块解析的文件扩展名。
- **`moduleIds`**
  设置模块 ID 的生成方式（`'named'` vs `'deterministic'`）。
- **`memoryLimit`**
  为 Turbopack 设置内存限制（以字节为单位）。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    // Example: adding an alias and custom file extension
    resolveAlias: {
      underscore: 'lodash',
    },
    resolveExtensions: ['.mdx', '.tsx', '.ts', '.jsx', '.js', '.json'],
  },
}
```

{/* md5:05252d9e3b0463dc2a70e132a67ada19 */}
有关更深入的配置示例，请参阅 [Turbopack 配置文档](/docs/app/api-reference/config/next-config-js/turbopack)。

## 生成跟踪文件以进行性能调试

{/* md5:a2870319dea2e2dd3349127351a50671 */}
如果您遇到性能或内存问题，并希望帮助 Next.js 团队进行诊断，您可以通过在开发命令后附加 `NEXT_TURBOPACK_TRACING=1` 来生成跟踪文件：

```bash
NEXT_TURBOPACK_TRACING=1 next dev --turbopack
```

{/* md5:8723cc0a7725f50c8cab1ea13f0959bb */}
这将生成一个 `.next/trace-turbopack` 文件。在 [Next.js 仓库](https://github.com/vercel/next.js)中创建 GitHub issue 时请包含该文件，以帮助我们进行调查。

## 总结

{/* md5:c57aa32510cc78ace4ff7ca1d91a9a54 */}
Turbopack 是一个基于 **Rust** 的、**增量式**打包器，旨在使本地开发和构建变得快速——尤其适用于大型应用程序。它集成在 Next.js 中，提供零配置的 CSS、React 和 TypeScript 支持。

{/* md5:33115274dc14815d3197352b8d1dc0ee */}
请继续关注更多更新，我们将持续改进 Turbopack 并添加生产构建支持。与此同时，尝试使用 `next dev --turbopack` 并告诉我们您的反馈。
