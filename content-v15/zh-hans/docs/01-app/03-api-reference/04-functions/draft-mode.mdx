`draftMode` 是一个 **异步** 函数，允许你启用和禁用 [预览模式 (Draft Mode)](/docs/app/guides/draft-mode)，以及检查在 [服务端组件 (Server Component)](/docs/app/getting-started/server-and-client-components) 中是否启用了预览模式。

```tsx filename="app/page.ts" switcher
import { draftMode } from 'next/headers'

export default async function Page() {
  const { isEnabled } = await draftMode()
}
```

```jsx filename="app/page.js" switcher
import { draftMode } from 'next/headers'

export default async function Page() {
  const { isEnabled } = await draftMode()
}
```

## 参考

可以使用以下方法和属性：

| 方法/属性   | 描述                                                                 |
| ----------- | -------------------------------------------------------------------- |
| `isEnabled` | 一个布尔值，指示预览模式是否已启用。                                  |
| `enable()`  | 通过设置一个 cookie (`__prerender_bypass`) 在路由处理器 (Route Handler) 中启用预览模式。 |
| `disable()` | 通过删除 cookie 在路由处理器 (Route Handler) 中禁用预览模式。          |

## 小知识

- `draftMode` 是一个返回 Promise 的 **异步** 函数。你必须使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函数。
  - 在版本 14 及更早版本中，`draftMode` 是一个同步函数。为了帮助向后兼容，你仍然可以在 Next.js 15 中以同步方式访问它，但此行为将在未来版本中被弃用。
- 每次运行 `next build` 时都会生成一个新的绕过 cookie 值。这确保了绕过 cookie 无法被猜测。
- 要通过 HTTP 在本地测试预览模式，你的浏览器需要允许第三方 cookie 和本地存储访问。

## 示例

### 启用预览模式 (Draft Mode)

要启用预览模式 (Draft Mode)，请创建一个新的 [路由处理器 (Route Handler)](/docs/app/api-reference/file-conventions/route) 并调用 `enable()` 方法：

```tsx filename="app/draft/route.ts" switcher
import { draftMode } from 'next/headers'

export async function GET(request: Request) {
  const draft = await draftMode()
  draft.enable()
  return new Response('Draft mode is enabled')
}
```

```js filename="app/draft/route.js" switcher
import { draftMode } from 'next/headers'

export async function GET(request) {
  const draft = await draftMode()
  draft.enable()
  return new Response('Draft mode is enabled')
}
```

### 禁用预览模式 (Draft Mode)

默认情况下，预览模式会话在浏览器关闭时结束。

要手动禁用预览模式 (Draft Mode)，请在你 的 [路由处理器 (Route Handler)](/docs/app/api-reference/file-conventions/route) 中调用 `disable()` 方法：

```tsx filename="app/draft/route.ts" switcher
import { draftMode } from 'next/headers'

export async function GET(request: Request) {
  const draft = await draftMode()
  draft.disable()
  return new Response('Draft mode is disabled')
}
```

```js filename="app/draft/route.js" switcher
import { draftMode } from 'next/headers'

export async function GET(request) {
  const draft = await draftMode()
  draft.disable()
  return new Response('Draft mode is disabled')
}
```

然后，发送一个请求来调用路由处理器。如果使用 [`<Link>` 组件](/docs/app/api-reference/components/link) 调用该路由，你必须传递 `prefetch={false}` 以防止在预取时意外删除 cookie。

### 检查预览模式 (Draft Mode) 是否已启用

你可以使用 `isEnabled` 属性在服务端组件中检查预览模式是否已启用：

```tsx filename="app/page.ts" switcher
import { draftMode } from 'next/headers'

export default async function Page() {
  const { isEnabled } = await draftMode()
  return (
    <main>
      <h1>My Blog Post</h1>
      <p>Draft Mode is currently {isEnabled ? 'Enabled' : 'Disabled'}</p>
    </main>
  )
}
```

```jsx filename="app/page.js" switcher
import { draftMode } from 'next/headers'

export default async function Page() {
  const { isEnabled } = await draftMode()
  return (
    <main>
      <h1>My Blog Post</h1>
      <p>Draft Mode is currently {isEnabled ? 'Enabled' : 'Disabled'}</p>
    </main>
  )
}
```
