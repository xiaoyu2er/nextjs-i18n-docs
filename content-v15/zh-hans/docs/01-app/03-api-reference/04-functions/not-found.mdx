`notFound` 函数允许你在路由段内渲染 [`not-found 文件`](/docs/app/api-reference/file-conventions/not-found)，同时也会注入一个 `<meta name="robots" content="noindex" />` 标签。

## `notFound()`

调用 `notFound()` 函数会抛出一个 `NEXT_HTTP_ERROR_FALLBACK;404` 错误，并终止其被抛出的路由段的渲染。通过指定一个 [**not-found** 文件](/docs/app/api-reference/file-conventions/not-found)，你可以在该段内优雅地处理此类错误，渲染一个"未找到"的用户界面。

```jsx filename="app/user/[id]/page.js"
import { notFound } from 'next/navigation'

async function fetchUser(id) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({ params }) {
  const { id } = await params
  const user = await fetchUser(id)

  if (!user) {
    notFound()
  }

  // ...
}
```

> **请注意**：由于使用了 TypeScript 的 [`never`](https://www.typescriptlang.org/docs/handbook/2/functions.html#never) 类型，`notFound()` 不需要你使用 `return notFound()`。
