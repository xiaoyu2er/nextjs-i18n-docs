`revalidatePath` 允许您按需使特定路径的 [缓存数据](/docs/app/guides/caching) 失效。

## 使用方法

`revalidatePath` 可以在服务器函数和路由处理器中调用。

`revalidatePath` 不能在客户端组件或中间件中调用，因为它仅在服务器环境中有效。

> **小知识**：
>
> - **服务器函数**：立即更新 UI（如果您正在查看受影响的路径）。目前，它还导致所有之前访问过的页面在再次导航时刷新。此行为是临时的，未来将更新为仅适用于特定路径。
> - **路由处理器**：标记路径以进行重新验证。重新验证会在下次访问指定路径时完成。这意味着使用动态路由段调用 `revalidatePath` 不会立即触发大量重新验证。失效仅在下次访问该路径时发生。

## 参数

```tsx
revalidatePath(path: string, type?: 'page' | 'layout'): void;
```

- `path`：可以是与您要重新验证的数据对应的路由模式，例如 `/product/[slug]`，也可以是特定的 URL，`/product/123`。请勿追加 `/page` 或 `/layout`，请使用 `type` 参数代替。不得超过 1024 个字符。此值区分大小写。
- `type`：（可选）`'page'` 或 `'layout'` 字符串，用于更改要重新验证的路径类型。如果 `path` 包含动态段，例如 `/product/[slug]`，则此参数是必需的。如果 `path` 是特定 URL，如 `/product/1`，请省略 `type`。

当您想要刷新 [单个页面](#revalidating-a-specific-url) 时使用特定 URL。使用路由模式加上 `type` 来刷新 [多个页面](#revalidating-a-page-path)。

## 返回

`revalidatePath` 不返回值。

## 可失效的内容

路径参数可以指向页面、布局或路由处理器：

- **页面**：使特定页面失效
- **布局**：使布局（该段落的 `layout.tsx`）、其下所有嵌套布局以及其下所有页面失效
- **路由处理器**：使在路由处理器内访问的数据缓存条目失效。例如 `revalidatePath("/api/data")` 会使此 GET 处理器失效：

```ts filename="app/api/data/route.ts"
export async function GET() {
  const data = await fetch('https://api.vercel.app/blog', {
    cache: 'force-cache',
  })

  return Response.json(await data.json())
}
```

## 与 revalidateTag 的关系

`revalidatePath` 和 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag) 有不同的用途：

- **`revalidatePath`**：使特定页面或布局路径失效
- **`revalidateTag`**：使所有使用该标签的页面中具有特定标签的数据失效

当您调用 `revalidatePath` 时，仅指定路径在下一次访问时获取新数据。使用相同数据标签的其他页面将继续提供缓存数据，直到这些特定标签也被重新验证：

```tsx
// Page A: /blog
const posts = await fetch('https://api.vercel.app/blog', {
  next: { tags: ['posts'] },
})

// Page B: /dashboard
const recentPosts = await fetch('https://api.vercel.app/blog?limit=5', {
  next: { tags: ['posts'] },
})
```

调用 `revalidatePath('/blog')` 后：

- **页面 A (/blog)**：显示新数据（页面已重新渲染）
- **页面 B (/dashboard)**：仍显示旧数据（缓存标签 'posts' 未失效）

### 构建重新验证工具

`revalidatePath` 和 `revalidateTag` 是互补的原语，经常在工具函数中一起使用，以确保应用程序范围内全面的数据一致性：

```ts
'use server'

import { revalidatePath, revalidateTag } from 'next/cache'

export async function updatePost() {
  await updatePostInDatabase()

  revalidatePath('/blog') // Refresh the blog page
  revalidateTag('posts') // Refresh all pages using 'posts' tag
}
```

此模式确保特定页面和任何使用相同数据的页面保持一致。

## 示例

### 重新验证特定 URL

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/post-1')
```

这将在下次页面访问时使一个特定 URL 失效以便重新验证。

### 重新验证页面路径

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'page')
// or with route groups
revalidatePath('/(main)/blog/[slug]', 'page')
```

这将在下次页面访问时使与提供的 `page` 文件匹配的任何 URL 失效以便重新验证。这**不会**使特定页面下属的页面失效。例如，`/blog/[slug]` 不会使 `/blog/[slug]/[author]` 失效。

### 重新验证布局路径

```ts
import { revalidatePath } from 'next/cache'
revalidatePath('/blog/[slug]', 'layout')
// or with route groups
revalidatePath('/(main)/post/[slug]', 'layout')
```

这将在下次页面访问时使与提供的 `layout` 文件匹配的任何 URL 失效以便重新验证。这将导致具有相同布局的下属页面在下一次访问时失效并重新验证。例如，在上述情况下，`/blog/[slug]/[another]` 也会在下一次访问时失效并重新验证。

### 重新验证所有数据

```ts
import { revalidatePath } from 'next/cache'

revalidatePath('/', 'layout')
```

这将清除客户端路由缓存，并使数据缓存失效以便在下次页面访问时重新验证。

### 服务器函数

```ts filename="app/actions.ts" switcher
'use server'

import { revalidatePath } from 'next/cache'

export default async function submit() {
  await submitForm()
  revalidatePath('/')
}
```

### 路由处理程序

```ts filename="app/api/revalidate/route.ts" switcher
import { revalidatePath } from 'next/cache'
import type { NextRequest } from 'next/server'

export async function GET(request: NextRequest) {
  const path = request.nextUrl.searchParams.get('path')

  if (path) {
    revalidatePath(path)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing path to revalidate',
  })
}
```
