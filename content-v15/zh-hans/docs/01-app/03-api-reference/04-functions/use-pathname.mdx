`usePathname` 是一个**客户端组件 (Client Component)** 钩子，它允许你读取当前 URL 的**路径名 (pathname)**。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'

export default function ExampleClientComponent() {
  const pathname = usePathname()
  return <p>Current pathname: {pathname}</p>
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { usePathname } from 'next/navigation'

export default function ExampleClientComponent() {
  const pathname = usePathname()
  return <p>Current pathname: {pathname}</p>
}
```

`usePathname` 故意要求使用[客户端组件 (Client Component)](/docs/app/getting-started/server-and-client-components)。重要的是要注意，客户端组件 (Client Components) 并非一种性能劣化。它们是[服务器组件 (Server Components)](/docs/app/getting-started/server-and-client-components) 架构不可或缺的一部分。

例如，一个使用 `usePathname` 的客户端组件 (Client Component) 将在初始页面加载时被渲染成 HTML。当导航到新路由时，此组件不需要重新获取。相反，该组件只会被下载一次（在客户端 JavaScript 包中），然后根据当前状态重新渲染。

> **值得了解**:
>
> - 不支持从 [服务器组件 (Server Component)](/docs/app/getting-started/server-and-client-components) 读取当前 URL。此设计是有意为之，以支持页面导航之间保留布局状态。
> - 兼容模式：
>   - 当渲染 [回退路由 (fallback route)](/docs/pages/api-reference/functions/get-static-paths#fallback-true) 时，`usePathname` 可能返回 `null`，或者当 Next.js 自动对 `pages` 目录页面进行 [自动静态优化 (automatically statically optimized)](/docs/pages/building-your-application/rendering/automatic-static-optimization) 且路由器尚未就绪时。
>   - 当在 [`next.config`](/docs/app/api-reference/config/next-config-js/rewrites) 或 [`Middleware`](/docs/app/api-reference/file-conventions/middleware) 中使用 `usePathname` 配合重写 (rewrites) 时，必须同时使用 `useState` 和 `useEffect` 以避免水合 (hydration) 不匹配错误。
>   - 如果 Next.js 检测到项目中既有 `app` 目录又有 `pages` 目录，它将自动更新你的类型。

## 参数

```tsx
const pathname = usePathname()
```

`usePathname` 不接受任何参数。

## 返回

`usePathname` 返回一个表示当前 URL 路径名的字符串。例如：

| URL                 | 返回值                |
| ------------------- | --------------------- |
| `/`                 | `'/'`                 |
| `/dashboard`        | `'/dashboard'`        |
| `/dashboard?v=2`    | `'/dashboard'`        |
| `/blog/hello-world` | `'/blog/hello-world'` |

## 示例

### 根据路由变化执行操作

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

function ExampleClientComponent() {
  const pathname = usePathname()
  const searchParams = useSearchParams()
  useEffect(() => {
    // Do something here...
  }, [pathname, searchParams])
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

function ExampleClientComponent() {
  const pathname = usePathname()
  const searchParams = useSearchParams()
  useEffect(() => {
    // Do something here...
  }, [pathname, searchParams])
}
```

### 在使用重写 (rewrites) 时避免水合 (hydration) 不匹配

当页面被预渲染 (pre-rendered) 时，HTML 是针对源路径名 (source pathname) 生成的。如果随后通过 `next.config` 或 `Middleware` 的重写 (rewrite) 访问该页面，浏览器 URL 可能会不同，并且 `usePathname()` 将在客户端读取重写后的路径名。

为了避免水合 (hydration) 不匹配，请设计用户界面，使得只有小部分隔离的部分依赖于客户端路径名。在服务器端渲染稳定的回退 (fallback)，并在挂载后更新该部分。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useEffect, useState } from 'react'
import { usePathname } from 'next/navigation'

export default function PathnameBadge() {
  const pathname = usePathname()
  const [clientPathname, setClientPathname] = useState('')

  useEffect(() => {
    setClientPathname(pathname)
  }, [pathname])

  return (
    <p>
      Current pathname: <span>{clientPathname}</span>
    </p>
  )
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useEffect, useState } from 'react'
import { usePathname } from 'next/navigation'

export default function PathnameBadge() {
  const pathname = usePathname()
  const [clientPathname, setClientPathname] = useState('')

  useEffect(() => {
    setClientPathname(pathname)
  }, [pathname])

  return (
    <p>
      Current pathname: <span>{clientPathname}</span>
    </p>
  )
}
```
