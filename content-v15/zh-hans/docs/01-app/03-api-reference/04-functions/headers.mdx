`headers` 是一个 **async** 函数，允许你从 [服务器组件](/docs/app/getting-started/server-and-client-components) **读取** HTTP 传入请求的 headers。

```tsx filename="app/page.tsx" switcher
import { headers } from 'next/headers'

export default async function Page() {
  const headersList = await headers()
  const userAgent = headersList.get('user-agent')
}
```

```jsx filename="app/page.js" switcher
import { headers } from 'next/headers'

export default async function Page() {
  const headersList = await headers()
  const userAgent = headersList.get('user-agent')
}
```

## 参考

### 参数

`headers` 不接受任何参数。

### 返回值

`headers` 返回一个 **只读的** [Web Headers](https://developer.mozilla.org/docs/Web/API/Headers) 对象。

- [`Headers.entries()`](https://developer.mozilla.org/docs/Web/API/Headers/entries)：返回一个 [`iterator`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Iteration_protocols)，允许遍历此对象中包含的所有键/值对。
- [`Headers.forEach()`](https://developer.mozilla.org/docs/Web/API/Headers/forEach)：对此 `Headers` 对象中的每个键/值对执行一次提供的函数。
- [`Headers.get()`](https://developer.mozilla.org/docs/Web/API/Headers/get)：返回一个 [`String`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/String)，表示具有给定名称的 header 在 `Headers` 对象中的所有值序列。
- [`Headers.has()`](https://developer.mozilla.org/docs/Web/API/Headers/has)：返回一个布尔值，表明 `Headers` 对象是否包含某个特定的 header。
- [`Headers.keys()`](https://developer.mozilla.org/docs/Web/API/Headers/keys)：返回一个 [`iterator`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Iteration_protocols)，允许你遍历此对象中包含的所有键/值对的键。
- [`Headers.values()`](https://developer.mozilla.org/docs/Web/API/Headers/values)：返回一个 [`iterator`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Iteration_protocols)，允许你遍历此对象中包含的所有键/值对的值。

## 小知识

- `headers` 是一个返回 Promise 的 **异步** 函数。你必须使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函数。
  - 在版本 14 及更早版本中，`headers` 是一个同步函数。为了向后兼容，你在 Next.js 15 中仍可以同步访问它，但此行为在未来将被弃用。
- 由于 `headers` 是只读的，你无法 `set`（设置）或 `delete`（删除）传出的请求 headers。
- `headers` 是一个 [动态 API](/docs/app/getting-started/partial-prerendering#dynamic-rendering#dynamic-apis)，其返回值无法预先知晓。使用它将使路由进入 **[动态渲染](/docs/app/getting-started/partial-prerendering#dynamic-rendering)** 模式。

## 示例

### 使用授权头 (Authorization header)

```jsx filename="app/page.js"
import { headers } from 'next/headers'

export default async function Page() {
  const authorization = (await headers()).get('authorization')
  const res = await fetch('...', {
    headers: { authorization }, // Forward the authorization header
  })
  const user = await res.json()

  return <h1>{user.name}</h1>
}
```
