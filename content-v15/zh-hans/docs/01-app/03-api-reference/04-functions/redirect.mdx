`redirect` 函数允许您重定向用户到另一个 URL。`redirect` 可在渲染 [服务端和客户端组件](/docs/app/getting-started/server-and-client-components)、[路由处理器](/docs/app/api-reference/file-conventions/route) 以及 [服务器操作](/docs/app/getting-started/updating-data) 时使用。

当在[流式上下文](/docs/app/getting-started/linking-and-navigating#streaming)中使用时，它会插入一个 `meta` 标签以在客户端发出重定向。在服务器操作中使用时，它会向调用方提供 303 HTTP 重定向响应。否则，它将向调用方提供 307 HTTP 重定向响应。

如果资源不存在，您可以改用 [`notFound` 函数](/docs/app/api-reference/functions/not-found)。

## 参考

### 参数

`redirect` 函数接受两个参数：

```js
redirect(path, type)
```

| 参数    | 类型                                                          | 描述                                               |
| ------- | ------------------------------------------------------------- | -------------------------------------------------- |
| `path`  | `string`                                                      | 要重定向到的 URL。可以是相对路径或绝对路径。       |
| `type`  | `'replace'` (默认) 或 `'push'` (在服务器操作中默认)           | 要执行的重定向类型。                               |

默认情况下，`redirect` 将在 [服务器操作](/docs/app/getting-started/updating-data) 中使用 `push`（向浏览器历史记录堆栈添加新条目），在其他地方使用 `replace`（替换浏览器历史记录堆栈中的当前 URL）。您可以通过指定 `type` 参数来覆盖此行为。

`RedirectType` 对象包含 `type` 参数的可用选项。

```ts
import { redirect, RedirectType } from 'next/navigation'

redirect('/redirect-to', RedirectType.replace)
// or
redirect('/redirect-to', RedirectType.push)
```

当在服务端组件中使用时，`type` 参数不起作用。

### 返回值

`redirect` 不返回值。

## 行为

- 在服务器操作和路由处理器中，使用 `try/catch` 语句时，`redirect` 应在 `try` 块 **外部** 调用。
- 如果您更希望返回 308（永久）HTTP 重定向而非 307（临时）HTTP 重定向，可以使用 [`permanentRedirect` 函数](/docs/app/api-reference/functions/permanentRedirect)。
- `redirect` 会抛出错误，因此使用 `try/catch` 语句时应在 `try` 块 **外部** 调用。
- `redirect` 可在渲染过程期间在客户端组件中调用，但不能在事件处理器中调用。可以使用 [`useRouter` 钩子](/docs/app/api-reference/functions/use-router)。
- `redirect` 也接受绝对 URL，可用于重定向到外部链接。
- 如果您想在渲染过程之前进行重定向，请使用 [`next.config.js`](/docs/app/guides/redirecting#redirects-in-nextconfigjs) 或 [中间件](/docs/app/guides/redirecting#nextresponseredirect-in-middleware)。

## 示例

### 服务端组件

调用 `redirect()` 函数会抛出一个 `NEXT_REDIRECT` 错误，并终止抛出该错误的路由段的渲染。

```tsx filename="app/team/[id]/page.tsx" switcher
import { redirect } from 'next/navigation'

async function fetchTeam(id: string) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const team = await fetchTeam(id)

  if (!team) {
    redirect('/login')
  }

  // ...
}
```

```jsx filename="app/team/[id]/page.js" switcher
import { redirect } from 'next/navigation'

async function fetchTeam(id) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({ params }) {
  const { id } = await params
  const team = await fetchTeam(id)

  if (!team) {
    redirect('/login')
  }

  // ...
}
```

> **请注意**: `redirect` 不需要你使用 `return redirect()`，因为它使用了 TypeScript 的 [`never`](https://www.typescriptlang.org/docs/handbook/2/functions.html#never) 类型。

### 客户端组件

`redirect` 可以直接在客户端组件中使用。

```tsx filename="components/client-redirect.tsx" switcher
'use client'

import { redirect, usePathname } from 'next/navigation'

export function ClientRedirect() {
  const pathname = usePathname()

  if (pathname.startsWith('/admin') && !pathname.includes('/login')) {
    redirect('/admin/login')
  }

  return <div>Login Page</div>
}
```

```jsx filename="components/client-redirect.jsx" switcher
'use client'

import { redirect, usePathname } from 'next/navigation'

export function ClientRedirect() {
  const pathname = usePathname()

  if (pathname.startsWith('/admin') && !pathname.includes('/login')) {
    redirect('/admin/login')
  }

  return <div>Login Page</div>
}
```

> **请注意**: 在服务器端渲染（SSR）期间，当在客户端组件中使用 `redirect` 时，它将执行服务器端重定向。

`redirect` 可以通过服务器操作在客户端组件中使用。如果你需要使用事件处理程序来重定向用户，可以使用 [`useRouter`](/docs/app/api-reference/functions/use-router) 钩子。

```tsx filename="app/client-redirect.tsx" switcher
'use client'

import { navigate } from './actions'

export function ClientRedirect() {
  return (
    <form action={navigate}>
      <input type="text" name="id" />
      <button>Submit</button>
    </form>
  )
}
```

```jsx filename="app/client-redirect.jsx" switcher
'use client'

import { navigate } from './actions'

export function ClientRedirect() {
  return (
    <form action={navigate}>
      <input type="text" name="id" />
      <button>Submit</button>
    </form>
  )
}
```

'use server'

import { redirect } from 'next/navigation'

export async function navigate(data: FormData) {
  redirect(`/posts/${data.get('id')}`)
}
```

```

## 常见问题

### 为什么 `redirect` 使用 307 和 308？

使用 `redirect()` 时，你可能会注意到它使用的状态码是 `307`（临时重定向）和 `308`（永久重定向）。传统上，`302` 用于临时重定向，`301` 用于永久重定向，但许多浏览器在使用 `302` 时会改变重定向的请求方法，从 `POST` 变为 `GET`，而不考虑原始请求的方法。

以下面从 `/users` 重定向到 `/people` 的例子为例：如果你向 `/users` 发出 `POST` 请求来创建新用户，并且遵循 `302` 临时重定向，请求方法将从 `POST` 变为 `GET`。这不合逻辑，因为要创建新用户，你应该向 `/people` 发出 `POST` 请求，而不是 `GET` 请求。

`307` 状态码的引入意味着请求方法会被保留为 `POST`。

- `302` - 临时重定向，会将请求方法从 `POST` 改为 `GET`
- `307` - 临时重定向，会将请求方法保留为 `POST`

`redirect()` 方法默认使用 `307`，而不是 `302` 临时重定向，这意味着你的请求将*始终*保留为 `POST` 请求。

[了解更多](https://developer.mozilla.org/docs/Web/HTTP/Redirections)关于 HTTP 重定向的信息。
