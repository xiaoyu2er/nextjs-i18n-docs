`permanentRedirect` 函数允许您将用户重定向到另一个 URL。`permanentRedirect` 可用于服务端组件、客户端组件、[路由处理器](/docs/app/api-reference/file-conventions/route) 和 [服务器操作](/docs/app/getting-started/updating-data)。

当在流式上下文中使用时，它将插入一个 `<meta>` 标签以在客户端发出重定向。当在服务器操作中使用时，它将向调用方提供 303 HTTP 重定向响应。否则，它将向调用方提供 308（永久）HTTP 重定向响应。

如果资源不存在，您可以改用 [`notFound` 函数](/docs/app/api-reference/functions/not-found)。

> **提示**：如果您希望返回 307（临时）HTTP 重定向而不是 308（永久）重定向，可以改用 [`redirect` 函数](/docs/app/api-reference/functions/redirect)。

## 参数

`permanentRedirect` 函数接受两个参数：

```js
permanentRedirect(path, type)
```

| 参数    | 类型                                                          | 描述                                               |
| ------- | ------------------------------------------------------------- | -------------------------------------------------- |
| `path`  | `string`                                                      | 要重定向到的 URL。可以是相对路径或绝对路径。       |
| `type`  | `'replace'` (默认) 或 `'push'` (在服务器操作中默认)           | 要执行的重定向类型。                               |

默认情况下，`permanentRedirect` 在 [服务器操作](/docs/app/getting-started/updating-data) 中将使用 `push`（向浏览器历史记录栈添加新条目），而在其他地方使用 `replace`（替换浏览器历史记录栈中的当前 URL）。您可以通过指定 `type` 参数来覆盖此行为。

当在服务端组件中使用时，`type` 参数不起作用。

## 返回

`permanentRedirect` 不返回值。

## 示例

调用 `permanentRedirect()` 函数会抛出一个 `NEXT_REDIRECT` 错误，并终止抛出该错误的路由段的渲染。

```jsx filename="app/team/[id]/page.js"
import { permanentRedirect } from 'next/navigation'

async function fetchTeam(id) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({ params }) {
  const { id } = await params
  const team = await fetchTeam(id)
  if (!team) {
    permanentRedirect('/login')
  }

  // ...
}
```

> **提示**：`permanentRedirect` 不需要您使用 `return permanentRedirect()`，因为它使用了 TypeScript 的 [`never`](https://www.typescriptlang.org/docs/handbook/2/functions.html#never) 类型。
