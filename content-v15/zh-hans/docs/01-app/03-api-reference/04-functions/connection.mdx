`connection()` 函数允许你指示渲染应等待传入的用户请求，然后再继续。

当一个组件没有使用[动态 API (Dynamic APIs)](/docs/app/getting-started/partial-prerendering#dynamic-rendering)，但你希望它在运行时被动态渲染而不是在构建时静态渲染时，这个函数非常有用。这种情况通常发生在你访问外部信息时，你有意希望改变渲染结果，例如 `Math.random()` 或 `new Date()`。

```ts filename="app/page.tsx" switcher
import { connection } from 'next/server'

export default async function Page() {
  await connection()
  // Everything below will be excluded from prerendering
  const rand = Math.random()
  return <span>{rand}</span>
}
```

```jsx filename="app/page.js" switcher
import { connection } from 'next/server'

export default async function Page() {
  await connection()
  // Everything below will be excluded from prerendering
  const rand = Math.random()
  return <span>{rand}</span>
}
```

## 参考

### 类型

```jsx
function connection(): Promise<void>
```

### 参数

- 该函数不接受任何参数。

### 返回值

- 该函数返回一个 `void` Promise。它并不用于被消费。

## 小知识

- `connection` 取代了 [`unstable_noStore`](/docs/app/api-reference/functions/unstable_noStore)，以更好地与 Next.js 的未来保持一致。
- 该函数仅在需要动态渲染且未使用常见动态 API 时才必要。
