`useLinkStatus` Hook 允许你追踪 `<Link>` 的 **挂起 (pending)** 状态。你可以使用它在导航到新路由完成之前，向用户显示内联的视觉反馈（如加载指示器或文本微光）。

`useLinkStatus` 在以下情况下很有用：

- [预取 (Prefetching)](/docs/app/getting-started/linking-and-navigating#prefetching) 被禁用或正在进行中，这意味着导航被阻塞。
- 目标路由是动态的，**并且** 不包含能够实现即时导航的 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 文件。

```tsx filename="app/loading-indicator.tsx" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return pending ? (
    <div role="status" aria-label="Loading" className="spinner" />
  ) : null
}
```

```jsx filename="app/loading-indicator.js" switcher

import { useLinkStatus } from 'next/link'

  const { pending } = useLinkStatus()

```

```tsx filename="app/header.tsx" switcher
import Link from 'next/link'
import LoadingIndicator from './loading-indicator'

export default function Header() {
  return (
    <header>
      <Link href="/dashboard" prefetch={false}>
        Dashboard <LoadingIndicator />
      </Link>
    </header>
  )
}
```


> **须知**：
>
> - `useLinkStatus` 必须在 `Link` 组件的后代组件中使用。
> - 当在 `Link` 组件上设置 `prefetch={false}` 时，此 Hook 最为有用。
> - 如果链接的路由已被预取，挂起状态将被跳过。
> - 当快速连续点击多个链接时，仅显示最后一个链接的挂起状态。
> - 此 Hook 在 Pages Router 中不受支持，并将始终返回 `{ pending: false }`。

## 参数

```tsx
const { pending } = useLinkStatus()
```

`useLinkStatus` 不接受任何参数。

## 返回

`useLinkStatus` 返回一个包含单个属性的对象：

| 属性    | 类型    | 描述                           |
| ------- | ------- | ------------------------------ |
| pending | boolean | 在历史记录更新前为 `true`，之后为 `false` |

## 示例

### 内联加载指示器

如果用户在预取完成前就点击了链接，添加导航正在进行的视觉反馈会很有帮助。

```tsx filename="app/components/loading-indicator.tsx" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return pending ? (
    <div role="status" aria-label="Loading" className="spinner" />
  ) : null
}
```

```jsx filename="app/components/loading-indicator.js" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return pending ? (
    <div role="status" aria-label="Loading" className="spinner" />
  ) : null
}
```

```tsx filename="app/shop/layout.tsx" switcher
import Link from 'next/link'
import LoadingIndicator from './components/loading-indicator'

const links = [
  { href: '/shop/electronics', label: 'Electronics' },
  { href: '/shop/clothing', label: 'Clothing' },
  { href: '/shop/books', label: 'Books' },
]

function Menubar() {
  return (
    <div>
      {links.map((link) => (
        <Link key={link.label} href={link.href}>
          {link.label} <LoadingIndicator />
        </Link>
      ))}
    </div>
  )
}

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <Menubar />
      {children}
    </div>
  )
}
```

## 优雅处理快速导航

如果导航到新路由很快，用户可能会看到加载指示器的不必要闪烁。改善用户体验的一种方法是在导航完成需要时间时才显示加载指示器，即添加初始动画延迟（例如 100ms）并以不可见状态开始动画（例如 `opacity: 0`）。

```css filename="app/styles/global.css"
.spinner {
  /* ... */
  opacity: 0;
  animation:
    fadeIn 500ms 100ms forwards,
    rotate 1s linear infinite;
}

@keyframes fadeIn {
  from {
    opacity: 0;
  }
  to {
    opacity: 1;
  }
}

@keyframes rotate {
  to {
    transform: rotate(360deg);
  }
}
```
