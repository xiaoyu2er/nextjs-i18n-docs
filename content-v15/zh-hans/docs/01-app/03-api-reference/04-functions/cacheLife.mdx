`cacheLife` 函数用于设置函数或组件的缓存生命周期。它应与 [`use cache`](/docs/app/api-reference/directives/use-cache) 指令一起使用，并位于函数或组件的作用域内。

## 使用方法

要使用 `cacheLife`，请在您的 `next.config.js` 文件中启用 [`cacheComponents` 标志](/docs/app/api-reference/config/next-config-js/cacheComponents)：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    cacheComponents: true,
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
const nextConfig = {
  experimental: {
    cacheComponents: true,
  },
}

export default nextConfig
```

然后，在函数或组件的作用域内导入并调用 `cacheLife` 函数：

```tsx filename="app/page.tsx" highlight={5} switcher
'use cache'
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default async function Page() {
  cacheLife('hours')
  return <div>Page</div>
}
```

```jsx filename="app/page.js" highlight={5} switcher
'use cache'
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default async function Page() {
  cacheLife('hours')
  return <div>Page</div>
}
```

## 参考

### 默认缓存配置

Next.js 提供了一套基于不同时间尺度的命名缓存配置。如果你在 `use cache` 指令旁的 `cacheLife` 函数中未指定缓存配置，Next.js 会自动应用 `default` 缓存配置。

然而，我们建议在使用 `use cache` 指令时始终添加一个缓存配置，以明确定义缓存行为。

| **配置**    | `stale`   | `revalidate` | `expire` | **描述**                     |
| ----------- | --------- | ------------ | -------- | ---------------------------- |
| `default`   | 5 分钟    | 15 分钟      | 1 年     | 默认配置，适用于不需要频繁更新的内容 |
| `seconds`   | 0         | 1 秒         | 1 秒     | 适用于需要近乎实时更新的快速变化内容 |
| `minutes`   | 5 分钟    | 1 分钟       | 1 小时   | 适用于在一小时内频繁更新的内容 |
| `hours`     | 5 分钟    | 1 小时       | 1 天     | 适用于每天更新但允许轻微过期的内容 |
| `days`      | 5 分钟    | 1 天         | 1 周     | 适用于每周更新但允许延迟一天的内容 |
| `weeks`     | 5 分钟    | 1 周         | 30 天    | 适用于每月更新但允许延迟一周的内容 |
| `max`       | 5 分钟    | 30 天        | 1 年     | 适用于很少需要更新的非常稳定的内容 |

用于引用缓存配置的字符串值本身不具有内在含义；它们仅作为语义标签。这使您能够更好地在代码库中理解和管理缓存内容。

> **须知：** 更新 [`staleTimes`](/docs/app/api-reference/config/next-config-js/staleTimes) 和 [`expireTime`](/docs/app/api-reference/config/next-config-js/expireTime) 配置选项也会更新 `default` 缓存配置的 `stale` 和 `expire` 属性。

### 自定义缓存配置

您可以通过在 `next.config.ts` 文件的 [`cacheLife`](/docs/app/api-reference/config/next-config-js/cacheLife) 选项中添加它们来配置自定义缓存配置。

缓存配置是包含以下属性的对象：

| **属性**     | **值**    | **描述**                                                                 | **要求**                            |
| ------------ | --------- | ------------------------------------------------------------------------ | ----------------------------------- |
| `stale`      | `number`  | 客户端在不检查服务器的情况下缓存值的持续时间。                             | 可选                                |
| `revalidate` | `number`  | 服务器上缓存应刷新的频率；在重新验证期间可能会提供过期值。                 | 可选                                |
| `expire`     | `number`  | 值在切换到动态获取之前可以保持过期的最大持续时间；必须长于 `revalidate`。 | 可选 - 必须长于 `revalidate`        |

`stale` 属性与 [`staleTimes`](/docs/app/api-reference/config/next-config-js/staleTimes) 设置的不同之处在于，它专门控制客户端路由缓存。`staleTimes` 是一个全局设置，影响所有动态和静态数据的实例，而 `cacheLife` 配置允许您按函数或路由定义“过期”时间。

> **须知**：`stale` 属性不设置 `Cache-control: max-age` 头。它控制的是客户端路由缓存。

## 示例

### 定义可重用的缓存配置

您可以通过在 `next.config.ts` 文件中定义它们来创建可重用的缓存配置。为您的用例选择一个合适的名称，并为 `stale`、`revalidate` 和 `expire` 属性设置值。您可以根据需要创建任意数量的自定义缓存配置。每个配置都可以通过其名称作为字符串值传递给 `cacheLife` 函数来引用。

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    cacheComponents: true,
    cacheLife: {
      biweekly: {
        stale: 60 * 60 * 24 * 14, // 14 days
        revalidate: 60 * 60 * 24, // 1 day
        expire: 60 * 60 * 24 * 14, // 14 days
      },
    },
  },
}

module.exports = nextConfig
```

```js filename="next.config.js" switcher
const nextConfig = {
  experimental: {
    cacheComponents: true,
    cacheLife: {
      biweekly: {
        stale: 60 * 60 * 24 * 14, // 14 days
        revalidate: 60 * 60 * 24, // 1 day
        expire: 60 * 60 * 24 * 14, // 14 days
      },
    },
  },
}

module.exports = nextConfig
```

上面的示例将缓存 14 天，每天检查更新，并在 14 天后使缓存过期。然后，您可以在整个应用程序中通过其名称引用此配置：

```tsx filename="app/page.tsx" highlight={5}
'use cache'
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default async function Page() {
  cacheLife('biweekly')
  return <div>Page</div>
}
```

### 覆盖默认缓存配置

虽然默认缓存配置提供了一种有用的方式来思考任何可缓存输出的任何部分可以有多新鲜或多陈旧，但您可能更喜欢不同的命名配置，以更好地与您的应用程序缓存策略保持一致。

您可以通过创建与默认配置同名的新配置来覆盖默认的命名缓存配置。

下面的示例展示了如何覆盖默认的 “days” 缓存配置：

```ts filename="next.config.ts"
const nextConfig = {
  experimental: {
    cacheComponents: true,
    cacheLife: {
      days: {
        stale: 3600, // 1 hour
        revalidate: 900, // 15 minutes
        expire: 86400, // 1 day
      },
    },
  },
}

module.exports = nextConfig
```

### 内联定义缓存配置

对于特定用例，您可以通过向 `cacheLife` 函数传递一个对象来设置自定义缓存配置：

```tsx filename="app/page.tsx" highlight={5-9} switcher
'use cache'
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default async function Page() {
  cacheLife({
    stale: 3600, // 1 hour
    revalidate: 900, // 15 minutes
    expire: 86400, // 1 day
  })

  return <div>Page</div>
}
```

```jsx filename="app/page.js" highlight={5-9} switcher
'use cache'
import { unstable_cacheLife as cacheLife } from 'next/cache'

export default async function Page() {
  cacheLife({
    stale: 3600, // 1 hour
    revalidate: 900, // 15 minutes
    expire: 86400, // 1 day
  })

  return <div>Page</div>
}
```

此内联缓存配置将仅应用于其创建时所在的函数或文件。如果您想在整个应用程序中重用相同的配置，可以[将该配置](#定义可重用的缓存配置)添加到 `next.config.ts` 文件的 `cacheLife` 属性中。

### `use cache` 和 `cacheLife` 的嵌套用法

当在同一路由或组件树中定义多个缓存行为时，如果内部缓存指定了自己的 `cacheLife` 配置，外部缓存将遵守其中最短的缓存持续时间。**这仅适用于外部缓存没有自己明确定义的 `cacheLife` 配置。**

例如，如果您向页面添加 `use cache` 指令但未指定缓存配置，则会隐式应用默认缓存配置（`cacheLife("default")`）。如果导入到该页面的组件也使用 `use cache` 指令并带有自己的缓存配置，则会比较外部和内部缓存配置，并应用配置中设置的最短持续时间。

```tsx filename="app/components/parent.tsx" highlight={5,6}
// Parent component
import { unstable_cacheLife as cacheLife } from 'next/cache'
import { ChildComponent } from './child'

export async function ParentComponent() {
  'use cache'
  cacheLife('days')

  return (
    <div>
      <ChildComponent />
    </div>
  )
}
```

并在单独的文件中定义导入的子组件：

```tsx filename="app/components/child.tsx" highlight={4,5}
// Child component
import { unstable_cacheLife as cacheLife } from 'next/cache'

export async function ChildComponent() {
  'use cache'
  cacheLife('hours')
  return <div>Child Content</div>

  // This component's cache will respect the shorter 'hours' profile
}
```
