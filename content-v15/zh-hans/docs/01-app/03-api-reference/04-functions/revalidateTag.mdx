title: revalidateTag

`revalidateTag` 允许您按需使特定缓存标签的 [缓存数据](/docs/app/guides/caching) 失效。

## 使用方法

`revalidateTag` 可以在服务端函数和路由处理程序中调用。

`revalidateTag` 无法在客户端组件或中间件中调用，因为它仅在服务器环境中生效。

> **值得了解**：`revalidateTag` 将带标签的数据标记为过期，但新数据仅在下次访问使用该标签的页面时才会获取。这意味着调用 `revalidateTag` 不会一次性立即触发大量重新验证。无效化操作仅在该标签被使用的页面下次被访问时发生。

## 参数

```tsx
revalidateTag(tag: string): void;
```

- `tag`: 代表要重新验证的数据所关联的缓存标签的字符串。长度不得超过 256 个字符。此值区分大小写。

您可以按以下方式向 `fetch` 添加标签：

```tsx
fetch(url, { next: { tags: [...] } });
```

## 返回

`revalidateTag` 不返回值。

## 与 revalidatePath 的关系

`revalidateTag` 会使所有使用该标签的页面中具有特定标签的数据失效，而 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 仅使特定的页面或布局路径失效。

> **值得了解**：这些函数服务于不同的目的，可能需要配合使用以确保数据完全一致。有关详细示例和注意事项，请参阅 [与 revalidateTag 的关系](/docs/app/api-reference/functions/revalidatePath#relationship-with-revalidatetag)。

## 示例

### 服务器操作

```ts filename="app/actions.ts" switcher
'use server'

import { revalidateTag } from 'next/cache'

export default async function submit() {
  await addPost()
  revalidateTag('posts')
}
```

```js filename="app/actions.js" switcher
'use server'

import { revalidateTag } from 'next/cache'

  revalidateTag('posts')
}
```

### 路由处理程序

```ts filename="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { revalidateTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')

  if (tag) {
    revalidateTag(tag)
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing tag to revalidate',
  })
}
```
