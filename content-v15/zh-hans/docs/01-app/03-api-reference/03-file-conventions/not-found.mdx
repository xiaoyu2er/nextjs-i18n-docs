Next.js 提供了两种处理未找到情况的约定：

{/* md5:107ea0a53fdba4f3a12ec8c51f74102d */}
- **`not-found.js`**：当您在路由段中调用 [`notFound`](/docs/app/api-reference/functions/not-found) 函数时使用。
- **`global-not-found.js`**：用于为整个应用程序中未匹配的路由定义全局 404 页面。这是在路由级别处理的，不依赖于渲染布局或页面。

{/* md5:0a24858ba00ceab3963f246b3de8687c */}
## `not-found.js`

{/* md5:ff90f692aa717f48d63ee57cd35fc545 */}
**not-found** 文件用于在路由段内抛出 [`notFound`](/docs/app/api-reference/functions/not-found) 函数时渲染 UI。除了提供自定义 UI 外，对于流式响应，Next.js 将返回 `200` HTTP 状态码，对于非流式响应则返回 `404`。

```tsx filename="app/not-found.tsx" switcher
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

```jsx filename="app/blog/not-found.js" switcher
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

## `global-not-found.js` (实验性)

{/* md5:3e2bc3317ad0e7a0a83452529dac7be5 */}
`global-not-found.js` 文件允许您为整个应用程序定义一个 404 页面。与在路由级别工作的 `not-found.js` 不同，此文件在请求的 URL 完全不匹配任何路由时使用。Next.js 会**跳过渲染**并直接返回此全局页面。

{/* md5:18c609b554e888660006312bd7d28fb4 */}
`global-not-found.js` 文件会绕过应用程序的正常渲染流程，这意味着您需要导入 404 页面所需的任何全局样式、字体或其他依赖项。

{/* md5:1f2e82d6a5a94b87c38010e5814206ea */}
> **需知**：使用较小的全局样式版本和更简单的字体系列可以提高此页面的性能。

{/* md5:0f688e02acaf775c4888c285c08d4c22 */}
当您无法使用 `layout.js` 和 `not-found.js` 的组合来构建 404 页面时，`global-not-found.js` 很有用。这可能发生在两种情况下：

{/* md5:a1ef8faa630870a7d40de863163f5e8a */}
- 您的应用程序有多个根布局（例如 `app/(admin)/layout.tsx` 和 `app/(shop)/layout.tsx`），因此没有单一的布局来组合全局 404 页面。
- 您的根布局是使用顶级动态段定义的（例如 `app/[country]/layout.tsx`），这使得组合一致的 404 页面变得更加困难。

{/* md5:4237431df97248d1a2cc3d28876d3f11 */}
要启用它，请在 `next.config.ts` 中添加 `globalNotFound` 标志：

```tsx filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    globalNotFound: true,
  },
}

export default nextConfig
```

{/* md5:54b9622e6f56f388ac08ec412063d439 */}
然后，在 `app` 目录的根目录创建一个文件：`app/global-not-found.js`：

```tsx filename="app/global-not-found.tsx" switcher
// Import global styles and fonts
import './globals.css'
import { Inter } from 'next/font/google'
import type { Metadata } from 'next'

const inter = Inter({ subsets: ['latin'] })

export const metadata: Metadata = {
  title: '404 - Page Not Found',
  description: 'The page you are looking for does not exist.',
}

export default function GlobalNotFound() {
  return (
    <html lang="en" className={inter.className}>
      <body>
        <h1>404 - Page Not Found</h1>
        <p>This page does not exist.</p>
      </body>
    </html>
  )
}
```


## 属性

{/* md5:7757a028bf9f37d06bc1391b6328d485 */}
`not-found.js` 或 `global-not-found.js` 组件不接受任何属性。

{/* md5:a4e5ed22ec8696910d6154de74d5ee9a */}
> **需知**：除了捕获预期的 `notFound()` 错误外，根目录的 `app/not-found.js` 和 `app/global-not-found.js` 文件还处理应用程序中所有未匹配的 URL。这意味着访问您的应用程序未处理的 URL 的用户将看到导出的 UI。

{/* md5:e57da1656ead1413f55464d7577686d2 */}

### 数据获取

{/* md5:082409ac97964fa1e1fe1f9c770c0214 */}
默认情况下，`not-found` 是一个服务器组件。你可以将其标记为 `async` 以获取并显示数据：

```tsx filename="app/not-found.tsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

```jsx filename="app/not-found.jsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

### 元数据

{/* md5:f9b55bbba81756eaaa9fe90757a8b6a9 */}
对于 `global-not-found.js`，您可以导出一个 `metadata` 对象或一个 [`generateMetadata`](/docs/app/api-reference/functions/generate-metadata) 函数来自定义 404 页面的 `<title>`、`<meta>` 和其他头部标签：

{/* md5:fdae45272b8e3ce9a30600a19848513b */}
> **需知**：Next.js 会自动为返回 404 状态码的页面注入 `<meta name="robots" content="noindex" />`，包括 `global-not-found.js` 页面。

```tsx filename="app/global-not-found.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Not Found',
  description: 'The page you are looking for does not exist.',
}

export default function GlobalNotFound() {
  return (
    <html lang="en">
      <body>
        <div>
          <h1>Not Found</h1>
          <p>The page you are looking for does not exist.</p>
        </div>
      </body>
    </html>
  )
}
```
