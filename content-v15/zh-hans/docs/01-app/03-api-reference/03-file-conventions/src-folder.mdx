作为在项目根目录使用特殊的 Next.js `app` 或 `pages` 目录的替代方案，Next.js 也支持将应用代码放置在 `src` 文件夹下的常见模式。

这可以将应用代码与主要位于项目根目录的项目配置文件分离开来，这是某些个人和团队所偏好的方式。

要使用 `src` 文件夹，请将 `app` 路由文件夹或 `pages` 路由文件夹分别移动到 `src/app` 或 `src/pages`。

> **请注意**:
>
> - `/public` 目录应保持在项目根目录下。
> - `package.json`、`next.config.js` 和 `tsconfig.json` 等配置文件应保持在项目根目录下。
> - `.env.*` 文件应保持在项目根目录下。
> - 如果根目录下存在 `app` 或 `pages`，则 `src/app` 或 `src/pages` 将被忽略。
> - 如果你使用 `src`，你可能还需要移动其他应用文件夹，例如 `/components` 或 `/lib`。
> - 如果你使用中间件，请确保它放置在 `src` 文件夹内。
> - 如果你使用 Tailwind CSS，你需要在 [content 部分](https://tailwindcss.com/docs/content-configuration) 的 `tailwind.config.js` 文件中添加 `/src` 前缀。
> - 如果你正在使用 TypeScript 路径进行导入（例如 `@/*`），你应该更新 `tsconfig.json` 中的 `paths` 对象，以包含 `src/`。
