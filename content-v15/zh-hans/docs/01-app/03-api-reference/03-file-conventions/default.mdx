{/* md5:e7fff2a35435896fe7ab7c120e2bf845 */}
`default.js` 文件用于在 [平行路由](/docs/app/api-reference/file-conventions/parallel-routes) 中渲染回退内容，当 Next.js 在全页面加载后无法恢复 [插槽](/docs/app/api-reference/file-conventions/parallel-routes#slots) 的活动状态时。

{/* md5:704ffed26d5eeccd985a0eb2d6614c0d */}
在 [软导航](/docs/app/getting-started/linking-and-navigating#client-side-transitions) 期间，Next.js 会跟踪每个插槽的活动 _状态_（子页面）。然而，对于硬导航（全页面加载），Next.js 无法恢复活动状态。在这种情况下，可以为与当前 URL 不匹配的子页面渲染 `default.js` 文件。

{/* md5:72d1b332cd9ce65982e4391cf2f0d32e */}
考虑以下文件夹结构。`@team` 插槽有一个 `settings` 页面，但 `@analytics` 没有。

当导航到 `/settings` 时，`@team` 插槽将渲染 `settings` 页面，同时保持 `@analytics` 插槽当前活动页面。

{/* md5:a0f6486d2225d005e5aa9ed04bdb7dbd */}
刷新页面时，Next.js 将为 `@analytics` 渲染一个 `default.js`。如果 `default.js` 不存在，则渲染 `404`。

{/* md5:6fa48007fab8b5aa728f7446fcfec1e6 */}
此外，由于 `children` 是一个隐式插槽，当 Next.js 无法恢复父页面的活动状态时，您也需要创建一个 `default.js` 文件来为 `children` 渲染回退内容。

### `params`（可选）

{/* md5:9ac085ca5071cd13a080e53d4eaf66de */}
一个 Promise，解析为一个对象，包含从根段到插槽子页面的 [动态路由参数](/docs/app/api-reference/file-conventions/dynamic-routes)。例如：

```tsx filename="app/[artist]/@sidebar/default.js" switcher
export default async function Default({
  params,
}: {
  params: Promise<{ artist: string }>
}) {
  const { artist } = await params
}
```

```jsx filename="app/[artist]/@sidebar/default.js" switcher
export default async function Default({ params }) {
  const { artist } = await params
}
```

{/* md5:c2a7e7eca995fe6fbcd83bb3ae0bc3a4 */}
| 示例 | URL | `params` |
| --- | --- | --- |
| `app/[artist]/@sidebar/default.js` | `/zack` | `Promise<{ artist: 'zack' }>` |
| `app/[artist]/[album]/@sidebar/default.js` | `/zack/next` | `Promise<{ artist: 'zack', album: 'next' }>` |

{/* md5:43317055be6b7a9d7bcc5ad50e978dd4 */}
- 由于 `params` 属性是一个 Promise，您必须使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函数来访问其值。
  - 在版本 14 及更早版本中，`params` 是一个同步属性。为了向后兼容，您仍然可以在 Next.js 15 中同步访问它，但此行为将在未来被弃用。
