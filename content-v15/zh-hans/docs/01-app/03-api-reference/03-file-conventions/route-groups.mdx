title: 路由组

路由组是一种文件夹约定，允许您按类别或团队组织路由。

{/* md5:45dcaa5a371e659f8b3d273badd01cd9 */}
## 约定

{/* md5:159683ab6b5c0078048372b129ac35dc */}
通过将文件夹名称用括号括起来，可以创建一个路由组：`(folderName)`。

{/* md5:ed754f0a5d153a7a0b51d689e6d14309 */}
此约定表示该文件夹用于组织目的，**不应包含**在路由的 URL 路径中。

## 使用场景

{/* md5:abceb10494ac53684e2f90080690891e */}
- 按团队、关注点或功能组织路由。
- 定义多个[根布局](/docs/app/api-reference/file-conventions/layout#root-layout)。
- 选择特定的路由段共享一个布局，同时排除其他段。

{/* md5:7cbd511c4ab51a1f1f7835b0deb23e60 */}
## 注意事项

{/* md5:67fabb91da46c3c9713affc85578646f */}
- **整页加载**: 如果您在使用不同根布局的路由之间导航，它将触发整页重新加载。例如，从使用 `app/(shop)/layout.js` 的 `/cart` 导航到使用 `app/(marketing)/layout.js` 的 `/blog`。这**仅**适用于多个根布局。
- **路径冲突**: 不同组中的路由不应解析到相同的 URL 路径。例如，`(marketing)/about/page.js` 和 `(shop)/about/page.js` 都将解析到 `/about` 并导致错误。
- **顶层根布局**: 如果您使用多个根布局而没有顶层的 `layout.js` 文件，请确保您的主页路由 (/) 在其中一个路由组中定义，例如 `app/(marketing)/page.js`。
