**模板**文件类似于[布局](/docs/app/getting-started/layouts-and-pages#creating-a-layout)，它包裹布局或页面。与在不同路由间保持状态且持久存在的布局不同，模板会被赋予一个唯一的键 (key)，这意味着子级客户端组件 (Client Components) 会在导航时重置它们的状态。

当您需要以下情况时，它们非常有用：

- 在导航时重新同步 `useEffect`。
- 在导航时重置子客户端组件的状态。例如，一个输入字段。
- 改变默认的框架行为。例如，布局内的 Suspense 边界仅在首次加载时显示回退 (fallback)，而模板则在每次导航时都显示它。

## 约定

可以通过从 `template.js` 文件导出一个默认的 React 组件来定义模板。该组件应接受一个 `children` 属性。

```tsx filename="app/template.tsx" switcher
export default function Template({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>
}
```

```jsx filename="app/template.js" switcher
export default function Template({ children }) {
  return <div>{children}</div>
}
```

在嵌套关系方面，`template.js` 在布局和其子内容之间渲染。以下是一个简化的输出：

```jsx filename="Output"
<Layout>
  {/* Note that the template is given a unique key. */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```

## 属性

### `children` (必填)

模板接受一个 `children` 属性。

```jsx filename="Output"
<Layout>
  {/* Note that the template is automatically given a unique key. */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```

## 行为

- **服务器组件 (Server Components)**：默认情况下，模板是服务器组件。
- **导航时重新挂载**：模板会自动接收一个唯一的键。导航到新路由会导致模板及其子组件重新挂载。
- **状态重置**：模板内的任何客户端组件都会在导航时重置其状态。
- **效果重新运行**：像 `useEffect` 这样的效果会随着组件重新挂载而重新同步。
- **DOM 重置**：模板内的 DOM 元素会被完全重新创建。
