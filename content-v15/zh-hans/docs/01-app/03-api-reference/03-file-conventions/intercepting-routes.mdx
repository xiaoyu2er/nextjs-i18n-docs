拦截路由允许您在当前布局中加载应用程序另一部分的路由。当您希望在用户不切换到不同上下文的情况下显示某个路由的内容时，这种路由范式非常有用。

例如，当在信息流中点击照片时，您可以在模态框中显示该照片，并覆盖在信息流之上。在这种情况下，Next.js 会拦截 `/photo/123` 路由，屏蔽 URL，并将其覆盖在 `/feed` 之上。

## 约定

拦截路由可以使用 `(..)` 约定来定义，该约定类似于相对路径约定 `../`，但用于路由段。

您可以使用：

- `(.)` 来匹配**同一层级**的段
- `(..)` 来匹配**上一层级**的段
- `(..)(..)` 来匹配**上两层**的段
- `(...)` 来匹配从 **`app` 根目录**开始的段

例如，您可以通过创建一个 `(..)photo` 目录，来在 `feed` 段内拦截 `photo` 段。

<Image

> **提示：** `(..)` 约定基于_路由段_，而非文件系统。例如，它不考虑 [并行路由 (Parallel Routes)](/docs/app/api-reference/file-conventions/parallel-routes) 中的 `@slot` 文件夹。

## 示例

### 模态框

拦截路由可以与[并行路由 (Parallel Routes)](/docs/app/api-reference/file-conventions/parallel-routes) 一起使用来创建模态框。这使您能够解决构建模态框时常见的挑战，例如：

- 使模态框内容**可通过 URL 分享**。
- 在页面刷新时**保留上下文**，而不是关闭模态框。
- 在向后导航时**关闭模态框**，而不是转到上一个路由。
- 在向前导航时**重新打开模态框**。

考虑以下 UI 模式：用户可以通过客户端导航从图库中打开照片模态框，或者通过可分享的 URL 直接导航到照片页面：

<Image

  srcLight="/docs/light/intercepted-routes-modal-example.png"
  srcDark="/docs/dark/intercepted-routes-modal-example.png"

在上面的示例中，`photo` 段的路径可以使用 `(..)` 匹配器，因为 `@modal` 是一个插槽 (slot)，**而不是**一个段。这意味着 `photo` 路由在路由层级上只高一级，尽管在文件系统层级上高了两级。

请参阅[并行路由 (Parallel Routes)](/docs/app/api-reference/file-conventions/parallel-routes#modals) 文档以获取分步示例，或查看我们的[图片库示例](https://github.com/vercel-labs/nextgram)。

> **值得注意：**
>
> - 其他示例可能包括在顶部导航栏中打开登录模态框，同时也有一个专门的 `/login` 页面；或者在侧边模态框中打开购物车。
