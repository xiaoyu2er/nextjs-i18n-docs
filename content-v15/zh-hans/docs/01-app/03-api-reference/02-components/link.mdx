`<Link>` 是一个 React 组件，它扩展了 HTML `<a>` 元素，提供了路由间的[预取 (prefetching)](/docs/app/getting-started/linking-and-navigating#prefetching) 和客户端导航功能。它是在 Next.js 中进行路由间导航的主要方式。

{/* md5:ffd610bf3bc0b18ce3a7eb7606809d07 */}
基本用法：

<AppOnly>

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

```jsx filename="app/page.js" switcher

export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>

```

</AppOnly>

### `scroll`

{/* md5:ab7671fa8759ec8dd716490db5499e36 */}
**默认为 `true`。** Next.js 中 `<Link>` 的默认滚动行为**是保持滚动位置**，类似于浏览器处理前进和后退导航的方式。当你导航到新的[页面 (Page)](/docs/app/api-reference/file-conventions/page) 时，只要该页面在视口中可见，滚动位置就会保持不变。但是，如果该页面在视口中不可见，Next.js 将滚动到第一个页面元素的顶部。

{/* md5:972c2f23ef99a9d910bd30621ed8a95a */}
当 `scroll = {false}` 时，Next.js 将不会尝试滚动到第一个页面元素。

{/* md5:48b600b86f72e63054f69ab068b32dbd */}
> **请注意**：Next.js 在管理滚动行为之前会检查 `scroll: false`。如果启用了滚动，它会识别导航的相关 DOM 节点并检查每个顶级元素。所有不可滚动的元素和那些没有渲染 HTML 的元素都会被跳过，这包括 sticky 或 fixed 定位的元素，以及像 `getBoundingClientRect` 计算的那样不可见的元素。然后 Next.js 会继续检查同级元素，直到找到一个在视口中可见的可滚动元素。

<AppOnly>

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" scroll={false}>
      Dashboard
    </Link>
  )
}
```


### `prefetch`

<AppOnly>

{/* md5:238cada402a3ffd59b9accd4279294a3 */}
当 `<Link />` 组件进入用户视口（初始加载或通过滚动）时，会发生预取。Next.js 会预取并加载链接的路由（由 `href` 表示）及其数据到后台，以提升客户端导航的性能。如果预取的数据在用户将鼠标悬停在 `<Link />` 上时已过期，Next.js 将尝试再次预取它。**预取功能仅在生产环境中启用**。

{/* md5:f23dc64129f080941401e662f893b318 */}
可以将以下值传递给 `prefetch` 属性：

{/* md5:8965d74b736e84b862a0e243dd597010 */}
- **`"auto"` 或 `null` (默认)**：预取行为取决于路由是静态的还是动态的。对于静态路由，将预取完整路由（包括所有其数据）。对于动态路由，将预取部分路由，直到最近的带有 [`loading.js`](/docs/app/api-reference/file-conventions/loading#instant-loading-states) 边界的分段。
- `true`：将为静态和动态路由预取完整路由。
- `false`：预取永远不会在进入视口或悬停时发生。

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</AppOnly>

### `legacyBehavior`

{/* md5:12f45fa9f90e58242ae490fc37665f78 */}
> **警告**：`legacyBehavior` 属性将在 Next.js v16 中被移除。要采用新的 `<Link>` 行为，请移除用作 `<Link>` 子元素的任何 `<a>` 标签。提供了一个 [codemod](/docs/app/guides/upgrading/codemods#new-link) 来帮助你自动更新代码库。

{/* md5:cd93ec9e11ee7c27632cb7fe6536a048 */}
从版本 13 开始，不再需要将 `<a>` 元素作为 `<Link>` 组件的子元素。如果由于兼容性原因你仍然需要旧行为，可以添加 `legacyBehavior` 属性。

{/* md5:111122f94642cee95b11d1bdd4ae7a9e */}
> **请注意**：当 `legacyBehavior` 未设置为 `true` 时，所有 [`锚点 (anchor)`](https://developer.mozilla.org/docs/Web/HTML/Element/a) 标签属性（如 `className`、`onClick` 等）也可以传递给 `next/link`。

### `locale`

{/* md5:82421d8152ddc21d866b1943cbcebc4e */}
活动的语言环境会自动添加在前面。`locale` 允许提供一个不同的语言环境。当设为 `false` 时，`href` 必须包含语言环境，因为默认行为被禁用了。

```tsx filename="pages/index.tsx" switcher
import Link from 'next/link'

export default function Home() {
  return (
    <>
      {/* Default behavior: locale is prepended */}
      <Link href="/dashboard">Dashboard (with locale)</Link>

      {/* Disable locale prepending */}
      <Link href="/dashboard" locale={false}>
        Dashboard (without locale)
      </Link>

      {/* Specify a different locale */}
      <Link href="/dashboard" locale="fr">
        Dashboard (French)
      </Link>
    </>
  )
}
```

### `onNavigate`

{/* md5:dec169c1fa535697750a5ef8d8ec73ff */}
在客户端导航期间调用的事件处理程序。该处理程序接收一个事件对象，其中包含一个 `preventDefault()` 方法，允许你在需要时取消导航。

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link
      href="/dashboard"
      onNavigate={(e) => {
        // Only executes during SPA navigation
        console.log('Navigating...')

        // Optionally prevent navigation
        // e.preventDefault()
      }}
    >
      Dashboard
    </Link>
  )
}
```

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link
      href="/dashboard"
      onNavigate={(e) => {
        // Only executes during SPA navigation
        console.log('Navigating...')

        // Optionally prevent navigation
        // e.preventDefault()
      }}
    >
      Dashboard
    </Link>
  )
}
```

{/* md5:d15f7b19b2f3184407c1c2272b1a2195 */}
> **请注意**：虽然 `onClick` 和 `onNavigate` 看起来相似，但它们的用途不同。`onClick` 对所有点击事件执行，而 `onNavigate` 仅在客户端导航期间运行。一些关键区别如下：
>
> - 当使用修饰键（`Ctrl`/`Cmd` + 点击）时，`onClick` 会执行但 `onNavigate` 不会，因为 Next.js 会阻止新标签页的默认导航行为。
> - 外部 URL 不会触发 `onNavigate`，因为它仅用于客户端和同源导航。
> - 带有 `download` 属性的链接会与 `onClick` 一起工作，但不会与 `onNavigate` 一起工作，因为浏览器会将链接的 URL 视为下载。

### Linking to dynamic route segments

{/* md5:aa3ff270fed0279c92a3cb8371a765ea */}
当链接到[动态段](/docs/app/api-reference/file-conventions/dynamic-routes)时，你可以使用[模板字符串和插值](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Template_literals)来生成链接列表。例如，生成一个博客文章列表：

```tsx filename="app/blog/post-list.tsx" switcher

interface Post {
  id: number
  title: string
  slug: string

export default function PostList({ posts }: { posts: Post[] }) {
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>

        </li>

    </ul>
  )
}
```

### Checking active links

{/* md5:c61327b3761b741a95c1bba3c83b0561 */}
你可以使用 [`usePathname()`](/docs/app/api-reference/functions/use-pathname) 来确定一个链接是否是活动的。例如，要向活动链接添加一个类，你可以检查当前的 `pathname` 是否与链接的 `href` 匹配：

```tsx filename="app/ui/nav-links.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'
import Link from 'next/link'

export function Links() {
  const pathname = usePathname()

  return (
    <nav>
      <Link className={`link ${pathname === '/' ? 'active' : ''}`} href="/">
        Home
      </Link>

      <Link
        className={`link ${pathname === '/about' ? 'active' : ''}`}
        href="/about"
      >
        About
      </Link>
    </nav>
  )
}
```

### Linking to dynamic route segments

{/* md5:a022e9ade91763b93a384d36db714e93 */}
对于[动态路由段](/docs/pages/building-your-application/routing/dynamic-routes#convention)，使用模板字面量来创建链接路径会很方便。

{/* md5:46b3411fb40b8cf65e696e6744a77ec7 */}
例如，你可以生成一个指向动态路由 `pages/blog/[slug].js` 的链接列表：

```tsx filename="pages/blog/index.tsx" switcher
import Link from 'next/link'

function Posts({ posts }) {
  return (
    <ul>
```

### Prefetching links in Middleware

{/* md5:329ec6f967cd380e87ab192c4f07745e */}
使用 [Middleware](/docs/app/api-reference/file-conventions/middleware) 进行身份验证或通过重写用户到不同页面的其他用途是很常见的。为了使 `<Link />` 组件能够正确通过 Middlewares 重写预取链接，你需要告诉 Next.js 要显示的 URL 和要预取的 URL。这是必要的，以避免不必要的中间件获取请求来确定正确的预取路由。

{/* md5:d3170064f188a622c1fa3b5e36c2659b */}

```ts filename="middleware.ts" switcher
import { NextResponse } from 'next/server'

export function middleware(request: Request) {
  const nextUrl = request.nextUrl
  if (nextUrl.pathname === '/dashboard') {
    if (request.cookies.authToken) {
      return NextResponse.rewrite(new URL('/auth/dashboard', request.url))
    } else {
      return NextResponse.rewrite(new URL('/public/dashboard', request.url))
    }
  }
}
```

### Blocking navigation

{/* md5:db0f61941a080ec42b9dafc73fd63dd9 */}

```tsx filename="app/contexts/navigation-blocker.tsx" switcher
import { createContext, useState, useContext } from 'react'

interface NavigationBlockerContextType {
  isBlocked: boolean
  setIsBlocked: (isBlocked: boolean) => void
}

export const NavigationBlockerContext =
  createContext<NavigationBlockerContextType>({
    isBlocked: false,
    setIsBlocked: () => {},
  })

export function NavigationBlockerProvider({
  children,
}: {
  children: React.ReactNode
}) {
  const [isBlocked, setIsBlocked] = useState(false)

  return (
    <NavigationBlockerContext.Provider value={{ isBlocked, setIsBlocked }}>
      {children}
    </NavigationBlockerContext.Provider>
  )
}

export function useNavigationBlocker() {
  return useContext(NavigationBlockerContext)
}
```
