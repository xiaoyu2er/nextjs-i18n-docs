{/* md5:e9036f0dd1a1accc19c358c21f1a23a7 */}
`<Form>` 组件扩展了 HTML `<form>` 元素，以提供<AppOnly>[**预取**](/docs/app/getting-started/linking-and-navigating#prefetching) [加载 UI](/docs/app/api-reference/file-conventions/loading)、</AppOnly>提交时的**客户端导航**和**渐进增强**。

{/* md5:3ffc25d17ce6c58d2358edd7be844158 */}
对于更新 URL 搜索参数的表单来说，它很有用，因为它减少了实现上述功能所需的样板代码。

{/* md5:ffd610bf3bc0b18ce3a7eb7606809d07 */}
基本用法：

<AppOnly>

```tsx filename="/app/ui/search.tsx" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      {/* On submission, the input value will be appended to
          the URL, e.g. /search?query=abc */}
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

```jsx filename="/app/ui/search.js" switcher
import Form from 'next/form'

export default function Search() {
  return (

          the URL, e.g. /search?query=abc */}
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

## 参考

{/* md5:ae7cb6184c23d753d00c145b0847236e */}
`<Form>` 组件的行为取决于 `action` 属性传递的是一个 `string` 还是一个 `function`。

<AppOnly>

{/* md5:afb81970580f7ebcf55b48f77d0943c4 */}
- 当 `action` 是一个**字符串**时，`<Form>` 的行为类似于使用 **`GET`** 方法的原生 HTML 表单。表单数据被编码为 URL 中的搜索参数，提交表单时，它会导航到指定的 URL。此外，Next.js：
  - 当表单可见时，会[预取](/docs/app/getting-started/linking-and-navigating#prefetching)该路径，这会预加载共享 UI（例如 `layout.js` 和 `loading.js`），从而实现更快的导航。
  - 提交表单时，执行[客户端导航](/docs/app/getting-started/linking-and-navigating#client-side-transitions)而不是完全重新加载页面。这保留了共享 UI 和客户端状态。
- 当 `action` 是一个**函数**（Server Action）时，`<Form>` 的行为类似于 [React 表单](https://react.dev/reference/react-dom/components/form)，在表单提交时执行该操作。

</AppOnly>

<PagesOnly>

{/* md5:46f9be0f66f13d114a38dbf5e1e08c1e */}

### `action` (string) 属性

<PagesOnly>

{/* md5:28fa45507beac15525aeeaf9c72c33fd */}
当 `action` 是字符串时，`<Form>` 组件支持以下属性：

{/* md5:aa740ac7d05d1be53553c2e2c79eb54a */}
| 属性      | 示例             | 类型                             | 必填 |
| --------- | ---------------- | -------------------------------- | ---- |
| `action`  | `action="/search"` | `string` (URL 或相对路径)        | 是   |
| `replace` | `replace={false}`  | `boolean`                        | -    |
| `scroll`  | `scroll={true}`    | `boolean`                        | -    |

{/* md5:c9f695602bbc6c5279b1f03e2ccdbbb6 */}
- **`action`**: 表单提交时要导航到的 URL 或路径。
  - 空字符串 `""` 将导航到具有更新后的搜索参数的相同路由。
- **`replace`**: 替换当前历史记录状态，而不是将其推入[浏览器历史记录](https://developer.mozilla.org/en-US/docs/Web/API/History_API)堆栈。默认为 `false`。
- **`scroll`**: 控制导航期间的滚动行为。默认为 `true`，这意味着它将滚动到新路由的顶部，并为后退和前进导航保持滚动位置。

</PagesOnly>

<AppOnly>

{/* md5:28fa45507beac15525aeeaf9c72c33fd */}
当 `action` 是字符串时，`<Form>` 组件支持以下属性：

{/* md5:594d7c70deea39df9eaa988cd1a75f2f */}
| 属性       | 示例             | 类型                             | 必填 |
| ---------- | ---------------- | -------------------------------- | ---- |
| `action`   | `action="/search"` | `string` (URL 或相对路径)        | 是   |
| `replace`  | `replace={false}`  | `boolean`                        | -    |
| `scroll`   | `scroll={true}`    | `boolean`                        | -    |
| `prefetch` | `prefetch={true}`  | `boolean`                        | -    |

{/* md5:17fdf351a7f60fea0e97c3050b7a8457 */}
- **`action`**: 表单提交时要导航到的 URL 或路径。
  - 空字符串 `""` 将导航到具有更新后的搜索参数的相同路由。
- **`replace`**: 替换当前历史记录状态，而不是将其推入[浏览器历史记录](https://developer.mozilla.org/en-US/docs/Web/API/History_API)堆栈。默认为 `false`。
- **`scroll`**: 控制导航期间的滚动行为。默认为 `true`，这意味着它将滚动到新路由的顶部，并为后退和前进导航保持滚动位置。
- **`prefetch`**: 控制当表单在用户视口中可见时是否应预取路径。默认为 `true`。

### `action` (function) 属性

{/* md5:bc803a6d46a8360b6666630db017f058 */}
当 `action` 是函数时，`<Form>` 组件支持以下属性：

{/* md5:8164834a9b56db35bd06bb48cbb7bc60 */}
| 属性     | 示例              | 类型                        | 必填 |
| -------- | ----------------- | --------------------------- | ---- |
| `action` | `action={myAction}` | `function` (Server Action)  | 是   |

{/* md5:14c37350cf0d2d69402dd42a9da1e71a */}
- **`action`**: 表单提交时要调用的 Server Action。更多信息请参阅 [React 文档](https://react.dev/reference/react-dom/components/form#props)。

{/* md5:ef03d8733e79c0658118277fda580d97 */}
> **请注意**：当 `action` 是函数时，`replace` 和 `scroll` 属性将被忽略。

</AppOnly>

{/* md5:1078984dc69ddba1ccac792621503bb9 */}
### 注意事项

<AppOnly>

{/* md5:5da645c28f643c35369025e5898d2335 */}
- **`formAction`**: 可用于 `<button>` 或 `<input type="submit">` 字段以覆盖 `action` 属性。Next.js 将执行客户端导航，但是，此方法不支持预取。
  - 当使用 [`basePath`](/docs/app/api-reference/config/next-config-js/basePath) 时，您也必须在 `formAction` 路径中包含它。例如 `formAction="/base-path/search"`。
- **`key`**: 不支持向字符串 `action` 传递 `key` 属性。如果您希望触发重新渲染或执行变更操作，请考虑使用函数 `action` 代替。

</AppOnly>

{/* md5:60323e99543f0f8ca7de5f73266e60eb */}
- **`onSubmit`**: 可用于处理表单提交逻辑。但是，调用 `event.preventDefault()` 将覆盖 `<Form>` 的行为，例如导航到指定的 URL。
- **[`method`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#method)、[`encType`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#enctype)、[`target`](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/form#target)**: 不受支持，因为它们会覆盖 `<Form>` 的行为。
  - 类似地，`formMethod`、`formEncType` 和 `formTarget` 可以分别用于覆盖 `method`、`encType` 和 `target` 属性，使用它们将回退到原生浏览器行为。
  - 如果您需要使用这些属性，请改用 HTML `<form>` 元素。
- **`<input type="file">`**: 当 `action` 是字符串时使用此输入类型，将通过提交文件名而不是文件对象来匹配浏览器行为。

<AppOnly>

### Search form that leads to a search result page

{/* md5:81b439e7b2dd6bc591f65546828ea076 */}
您可以通过将路径作为 `action` 传递来创建一个导航到搜索结果页的搜索表单：

```tsx filename="/app/page.tsx" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

```jsx filename="/app/page.js" switcher
import Form from 'next/form'

export default function Page() {
  return (
    <Form action="/search">
      <input name="query" />
      <button type="submit">Submit</button>
    </Form>
  )
}
```

{/* md5:457d7b8f088761fac36dd60309daeae8 */}
当用户更新查询输入字段并提交表单时，表单数据将被编码为 URL 中的搜索参数，例如 `/search?query=abc`。

{/* md5:a640774a19dfad736bee66a482519eeb */}
> **请注意**：如果您向 `action` 传递空字符串 `""`，表单将导航到具有更新后的搜索参数的相同路由。

{/* md5:a75fdee7f9cf5a963f0fe2ac87ed8132 */}
在结果页面，您可以使用 [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) `page.js` 属性访问查询，并使用它从外部源获取数据。

```tsx filename="/app/search/page.tsx" switcher
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const results = await getSearchResults((await searchParams).query)

  return <div>...</div>
}
```

```jsx filename="/app/search/page.js" switcher
import { getSearchResults } from '@/lib/search'

export default async function SearchPage({ searchParams }) {
  const results = await getSearchResults((await searchParams).query)

  return <div>...</div>
}
```

{/* md5:c51f4833a66b8b83f9b20db3ebb31e90 */}
当 `<Form>` 在用户视口中可见时，`/search` 页面上的共享 UI（例如 `layout.js` 和 `loading.js`）将被预取。提交时，表单将立即导航到新路由，并在获取结果时显示加载 UI。您可以使用 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 设计回退 UI：

```tsx filename="/app/search/loading.tsx" switcher
export default function Loading() {
  return <div>Loading...</div>
}
```

```jsx filename="/app/search/loading.js" switcher
export default function Loading() {
  return <div>Loading...</div>
}
```

{/* md5:cd91a316ac8967800341b64e0212346e */}
为了覆盖共享 UI 尚未加载的情况，您可以使用 [`useFormStatus`](https://react.dev/reference/react-dom/hooks/useFormStatus) 向用户显示即时反馈。

{/* md5:d8e0f4830485fb68e69d38719591b399 */}
首先，创建一个在表单挂起时显示加载状态的组件：

```tsx filename="/app/ui/search-button.tsx" switcher
'use client'
import { useFormStatus } from 'react-dom'

export default function SearchButton() {
  const status = useFormStatus()
  return (
    <button type="submit">{status.pending ? 'Searching...' : 'Search'}</button>
  )
}
```

```jsx filename="/app/ui/search-button.js" switcher

```

### Mutations with Server Actions

{/* md5:a8328efa1843ff11b1f187e15143d813 */}
您可以通过向 `action` 属性传递一个函数来执行变更操作。

```tsx filename="/app/posts/create/page.tsx" switcher
import Form from 'next/form'
import { createPost } from '@/posts/actions'

export default function Page() {
  return (
    <Form action={createPost}>
      <input name="title" />
      {/* ... */}
      <button type="submit">Create Post</button>
    </Form>
  )
}
```

{/* md5:ce7c241d796f0ce76314a4e639ed5795 */}
变更后，通常需要重定向到新资源。您可以使用 `next/navigation` 中的 [`redirect`](/docs/app/guides/redirecting) 函数导航到新帖子页面。

{/* md5:efce9e7ea2560ac1fd48b651d2c8847e */}
> **请注意**：由于表单提交的"目标"在操作执行前是未知的，`<Form>` 无法自动预取共享 UI。

```tsx filename="/app/posts/actions.ts" switcher
'use server'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  // Create a new post
  // ...

  // Redirect to the new post
  redirect(`/posts/${data.id}`)
}
```

```jsx filename="/app/posts/actions.js" switcher
'use server'
import { redirect } from 'next/navigation'

export async function createPost(formData) {
  // Create a new post
  // ...

  // Redirect to the new post

```

import { getPost } from '@/posts/data'
