{/* md5:d02d5d4e3f99adb7180db074ee933b95 */}
`'use client'` 指令声明了一个**客户端**渲染组件的入口点，在创建需要客户端 JavaScript 能力（例如状态管理、事件处理和访问浏览器 API）的交互式用户界面（UI）时应该使用它。这是 React 的一个功能。

{/* md5:81fb99e561dcc8a1152bca5cf59bc6e5 */}
> **须知：**
>
> 您不需要在每个包含客户端组件（Client Components）的文件中添加 `'use client'` 指令。您只需要将其添加到您希望直接在服务端组件（Server Components）内渲染的组件文件中。`'use client'` 指令定义了客户端-服务端的[边界](https://nextjs.org/docs/app/building-your-application/rendering#network-boundary)，从此类文件导出的组件作为客户端的入口点。

{/* md5:36e7e62aaa19ae92537b906bf2eae752 */}
## 使用方法

{/* md5:be7325368615e68e4a9886c99d645d4e */}
要声明客户端组件的入口点，请在文件的**顶部**、所有导入之前添加 `'use client'` 指令：

```tsx filename="app/components/counter.tsx" highlight={1} switcher
'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  )
}
```

```jsx filename="app/components/counter.js" highlight={1} switcher
'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  )
}
```

{/* md5:a617e0829e18092e17e5858ba933d22e */}
使用 `'use client'` 指令时，客户端组件的 props 必须是[可序列化的](https://react.dev/reference/rsc/use-client#serializable-types)。这意味着当 React 将数据从服务端发送到客户端时，props 需要处于 React 可以序列化的格式。

```tsx filename="app/components/counter.tsx" highlight={4} switcher
'use client'

export default function Counter({
  onClick /* ❌ Function is not serializable */,
}) {
  return (
    <div>
      <button onClick={onClick}>Increment</button>
    </div>
  )
}
```

```jsx filename="app/components/counter.js" highlight={4} switcher
'use client'

export default function Counter({
  onClick /* ❌ Function is not serializable */,
}) {
  return (
    <div>
      <button onClick={onClick}>Increment</button>
    </div>
  )
}
```

## 在服务端组件中嵌套客户端组件

{/* md5:0107481c6a69b081fbd59c4ea0ccf2a5 */}
结合使用服务端组件和客户端组件，可以构建既高性能又具有交互性的应用程序：

{/* md5:c702cae2cc88df588ca523c9c229dd7e */}
1. **服务端组件**：用于静态内容、数据获取和对 SEO 友好的元素。
2. **客户端组件**：用于需要状态、副作用（effects）或浏览器 API 的交互式元素。
3. **组件组合**：根据需要将客户端组件嵌套在服务端组件内部，以实现服务端和客户端逻辑的清晰分离。

{/* md5:ea8ebf645588fe3f13e376c68337f3ad */}
在以下示例中：

{/* md5:1d5ca70207d80d3f15b9615ba5c55026 */}
- `Header` 是一个处理静态内容的服务端组件。
- `Counter` 是一个在页面内启用交互性的客户端组件。

```tsx filename="app/page.tsx" highlight={2,8} switcher
import Header from './header'
import Counter from './counter' // This is a Client Component

export default function Page() {
  return (
    <div>
      <Header />
      <Counter />
    </div>
  )
}
```

```jsx filename="app/page.js" highlight={2,8} switcher
import Header from './header'
import Counter from './counter' // This is a Client Component

export default function Page() {
  return (
    <div>
      <Header />
      <Counter />
    </div>
  )
}
```
