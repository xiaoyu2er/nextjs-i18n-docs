`use cache` 指令允许您将路由、React 组件或函数标记为可缓存。它可以在文件顶部使用，表示该文件中的所有导出都应被缓存；也可以内联在函数或组件顶部，以缓存返回值。

## 使用方法

`use cache` 目前是一项实验性功能。要启用它，请在 `next.config.ts` 文件中添加 [`useCache`](/docs/app/api-reference/config/next-config-js/useCache) 选项：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    useCache: true,
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    useCache: true,
  },
}

module.exports = nextConfig
```

> **了解更多信息：** `use cache` 也可以通过 [`cacheComponents`](/docs/app/api-reference/config/next-config-js/cacheComponents) 选项启用。

然后，在文件、组件或函数级别添加 `use cache`：

```tsx
// File level
'use cache'

export default async function Page() {
  // ...
}

// Component level
export async function MyComponent() {
  'use cache'
  return <></>
}

// Function level
export async function getData() {
  'use cache'
  const data = await fetch('/api/data')
  return data
}
```

## "use cache" 如何工作

### 缓存键

缓存条目的键是使用其输入的序列化版本生成的，其中包括：

- 构建 ID (为每个构建生成)
- 函数 ID (函数的唯一安全标识符)
- [可序列化](https://react.dev/reference/rsc/use-server#serializable-parameters-and-return-values)的函数参数（或 props）。

传递给缓存函数的参数以及它从父作用域读取的任何值都会自动成为键的一部分。这意味着，只要输入相同，就会重用相同的缓存条目。

## 不可序列化的参数

任何非序列化的参数、props 或封闭值都会在缓存函数内部变成引用，并且只能被传递，无法被检查或修改。这些非序列化值将在请求时被填充，并且不会成为缓存键的一部分。

例如，一个缓存函数可以将 JSX 作为 `children` prop 传入并返回 `<div>{children}</div>`，但它无法内省实际的 `children` 对象。这允许您在缓存组件内部嵌套非缓存内容。

```tsx filename="app/ui/cached-component.tsx" switcher
function CachedComponent({ children }: { children: ReactNode }) {
  'use cache'
  return <div>{children}</div>
}
```

```jsx filename="app/ui/cached-component.js" switcher
function CachedComponent({ children }) {
  'use cache'
  return <div>{children}</div>
}
```

## 返回值

可缓存函数的返回值必须是可序列化的。这确保了缓存的数据可以被正确存储和检索。

## `use cache` 在构建时

当在 [布局](/docs/app/api-reference/file-conventions/layout) 或 [页面](/docs/app/api-reference/file-conventions/page) 顶部使用时，路由段将被预渲染，允许稍后进行[重新验证](#during-revalidation)。

这意味着 `use cache` 不能与 [`cookies`](/docs/app/api-reference/functions/cookies) 或 [`headers`](/docs/app/api-reference/functions/headers) 等[请求时 API](/docs/app/getting-started/partial-prerendering#dynamic-rendering) 一起使用。

## `use cache` 在运行时

在 **服务器端**，单个组件或函数的缓存条目将被缓存在内存中。

然后，在 **客户端**，从服务器缓存返回的任何内容都将存储在浏览器的内存中，有效期为会话期间或直到[重新验证](#during-revalidation)。

## 重新验证期间

默认情况下，`use cache` 具有 **15 分钟** 的服务器端重新验证周期。虽然这个周期对于不需要频繁更新的内容很有用，但您可以使用 `cacheLife` 和 `cacheTag` API 来配置各个缓存条目应在何时重新验证。

- [`cacheLife`](/docs/app/api-reference/functions/cacheLife)：配置缓存条目的生命周期。
- [`cacheTag`](/docs/app/api-reference/functions/cacheTag)：创建标签以进行按需重新验证。

这两个 API 都跨客户端和服务器缓存层集成，这意味着您可以在一个地方配置缓存语义，并使它们在所有地方生效。

更多信息，请参阅 [`cacheLife`](/docs/app/api-reference/functions/cacheLife) 和 [`cacheTag`](/docs/app/api-reference/functions/cacheTag) API 文档。

## 示例

### 使用 `use cache` 缓存整个路由

要预渲染整个路由，请将 `use cache` 添加到 **同时** `layout` 和 `page` 文件的顶部。这些段中的每一个都被视为应用程序中的单独入口点，并将被独立缓存。

```tsx filename="app/layout.tsx" switcher
'use cache'

export default function Layout({ children }: { children: ReactNode }) {
  return <div>{children}</div>
}
```

```jsx filename="app/page.tsx" switcher
'use cache'

export default function Layout({ children }) {
  return <div>{children}</div>
}
```

在 `page` 文件中导入和嵌套的任何组件都将继承 `page` 的缓存行为。

```tsx filename="app/page.tsx" switcher
'use cache'

async function Users() {
  const users = await fetch('/api/users')
  // loop through users
}

export default function Page() {
  return (
    <main>
      <Users />
    </main>
  )
}
```

```jsx filename="app/page.js" switcher
'use cache'

async function Users() {
  const users = await fetch('/api/users')
  // loop through users
}

export default function Page() {
  return (
    <main>
      <Users />
    </main>
  )
}
```

> **请注意**：
>
> - 如果 `use cache` 仅添加到 `layout` 或 `page` 中，则仅该路由段及其导入的组件将被缓存。
> - 如果路由中任何嵌套的子组件使用了 [动态 API](/docs/app/getting-started/partial-prerendering#dynamic-rendering)，则该路由将退出预渲染。

### 使用 `use cache` 缓存组件的输出

您可以在组件级别使用 `use cache` 来缓存在该组件内执行的任何获取请求或计算。只要序列化的 props 在每个实例中产生相同的值，缓存条目就会被重用。

```tsx filename="app/components/bookings.tsx" highlight={2} switcher
export async function Bookings({ type = 'haircut' }: BookingsProps) {
  'use cache'
  async function getBookingsData() {
    const data = await fetch(`/api/bookings?type=${encodeURIComponent(type)}`)
    return data
  }
  return //...
}

interface BookingsProps {
  type: string
}
```

```jsx filename="app/components/bookings.js" highlight={2} switcher
export async function Bookings({ type = 'haircut' }) {
  'use cache'
  async function getBookingsData() {
    const data = await fetch(`/api/bookings?type=${encodeURIComponent(type)}`)
    return data
  }
  return //...
}
```

### 使用 `use cache` 缓存函数输出

因为您可以将 `use cache` 添加到任何异步函数，所以您不仅限于缓存组件或路由。您可能希望缓存网络请求、数据库查询或缓慢的计算。

```tsx filename="app/actions.ts" highlight={2} switcher
export async function getData() {
  'use cache'

  const data = await fetch('/api/data')
  return data
}
```

```jsx filename="app/actions.js" highlight={2} switcher
export async function getData() {
  'use cache'

  const data = await fetch('/api/data')
  return data
}
```

### 交错

如果您需要将非序列化参数传递给可缓存函数，可以将它们作为 `children` 传递。这意味着 `children` 引用可以改变，而不会影响缓存条目。

```tsx filename="app/page.tsx" switcher
export default async function Page() {
  const uncachedData = await getData()
  return (
    <CacheComponent>
      <DynamicComponent data={uncachedData} />
    </CacheComponent>
  )
}

async function CacheComponent({ children }: { children: ReactNode }) {
  'use cache'
  const cachedData = await fetch('/api/cached-data')
  return (
    <div>
      <PrerenderedComponent data={cachedData} />
      {children}
    </div>
  )
}
```

```jsx filename="app/page.js" switcher
export default async function Page() {
  const uncachedData = await getData()
  return (
    <CacheComponent>
      <DynamicComponent data={uncachedData} />
    </CacheComponent>
  )
}

async function CacheComponent({ children }) {
  'use cache'
  const cachedData = await fetch('/api/cached-data')
  return (
    <div>
      <PrerenderedComponent data={cachedData} />
      {children}
    </div>
  )
}
```

您也可以通过缓存组件将服务器操作 (Server Actions) 传递给客户端组件，而无需在可缓存函数内部调用它们。

```tsx filename="app/page.tsx" switcher
import ClientComponent from './ClientComponent'

export default async function Page() {
  const performUpdate = async () => {
    'use server'
    // Perform some server-side update
    await db.update(...)
  }

  return <CacheComponent performUpdate={performUpdate} />
}

async function CachedComponent({
  performUpdate,
}: {
  performUpdate: () => Promise<void>
}) {
  'use cache'
  // Do not call performUpdate here
  return <ClientComponent action={performUpdate} />
}
```

```jsx filename="app/page.js" switcher
import ClientComponent from './ClientComponent'

export default async function Page() {
  const performUpdate = async () => {
    'use server'
    // Perform some server-side update
    await db.update(...)
  }

  return <CacheComponent performUpdate={performUpdate} />
}

async function CachedComponent({ performUpdate }) {
  'use cache'
  // Do not call performUpdate here
  return <ClientComponent action={performUpdate} />
}
```

```tsx filename="app/ClientComponent.tsx" switcher
'use client'

export default function ClientComponent({
  action,
}: {
  action: () => Promise<void>
}) {
  return <button onClick={action}>Update</button>
}
```

```jsx filename="app/ClientComponent.js" switcher
'use client'

export default function ClientComponent({ action }) {
  return <button onClick={action}>Update</button>
}
```
