---
title: 快速刷新
description: 快速刷新是一种热模块重载体验，可让您对 React 组件的编辑获得即时反馈。
---

{/* md5:14f7e54e6e6e85110f15e40f55792426 */}
快速刷新是集成到 Next.js 中的 React 功能，它允许您在保存文件更改时实时重新加载浏览器页面，同时保持临时的客户端状态。它在所有 Next.js 9.4 或更高版本的应用程序中默认启用。启用快速刷新后，大多数编辑应在**一秒内**可见。

{/* md5:d3d8186c44a628d780942a910680f466 */}
## 工作原理

{/* md5:972e94a29818b054d690ecf4d6134f3b */}
- 如果您编辑一个**仅导出 React 组件**的文件，快速刷新将仅更新该文件的代码，并重新渲染您的组件。您可以编辑该文件中的任何内容，包括样式、渲染逻辑、事件处理程序或效果。
- 如果您编辑的文件导出的内容**不是** React 组件，快速刷新将重新运行该文件以及导入它的其他文件。因此，如果 `Button.js` 和 `Modal.js` 都导入了 `theme.js`，编辑 `theme.js` 将同时更新这两个组件。
- 最后，如果您编辑的文件被**React 树之外的文件**导入，快速刷新将**回退到完全重新加载**。您可能有一个文件用于渲染 React 组件，但同时导出一个被**非 React 组件**导入的值。例如，您的组件可能还导出了一个常量，并且一个非 React 工具文件导入了它。在这种情况下，建议将该常量迁移到单独的文件中，并在两个文件中都导入它。这将重新启用快速刷新。其他情况通常也可以类似的方式解决。

{/* md5:2d553bd788d358e9ce56f3a843c8185b */}
## 错误恢复能力

{/* md5:23157fff09453a7b5f17e4acc6f5dbfd */}
### 语法错误

{/* md5:76b145bbf9d21c41b4b40c81b5fe5772 */}
如果在开发过程中出现语法错误，您可以修复它并再次保存文件。错误将自动消失，因此您无需重新加载应用程序。**您不会丢失组件状态**。

{/* md5:4db305749144e702182294e843aa7c86 */}
### 运行时错误

{/* md5:d9aa20dfa205b580d1bde91d8a19c2cb */}
如果您犯了导致组件内部运行错误的错误，您将看到一个上下文覆盖层。修复错误将自动关闭覆盖层，而无需重新加载应用程序。

{/* md5:6efbc988481e026d49d358adf9256dd9 */}
如果错误不是在渲染期间发生的，则组件状态将被保留。如果错误确实发生在渲染期间，React 将使用更新的代码重新挂载您的应用程序。

{/* md5:3a279e3e9dfeee9342a27e76b2864dc8 */}
如果您的应用程序中有[错误边界](https://react.dev/reference/react/Component#catching-rendering-errors-with-an-error-boundary)（这是生产环境中实现优雅故障的良好实践），它们将在渲染错误后的下一次编辑中重试渲染。这意味着拥有错误边界可以防止您始终被重置为根应用程序状态。但是请记住，错误边界不应该过于细粒度。它们在 React 的生产环境中使用，应始终经过精心设计。

{/* md5:7e9d7d6dd4b947cbc70b118e54686f17 */}
## 限制

{/* md5:2371ed64f4b9c90abd8e2bc217a6e77b */}
快速刷新尝试保留正在编辑的组件中的本地 React 状态，但仅在安全的情况下这样做。以下是每次编辑文件时本地状态可能会重置的几个原因：

{/* md5:796cba3dd12a26b7b0eb329813a13eab */}
- 类组件不会保留本地状态（只有函数组件和 Hooks 会保留状态）。
- 您正在编辑的文件可能除了 React 组件之外还有其他导出。
- 有时，文件会导出调用高阶组件（如 `HOC(WrappedComponent)`）的结果。如果返回的组件是类，其状态将被重置。
- 匿名箭头函数（如 `export default () => <div />;`）会导致快速刷新不保留本地组件状态。对于大型代码库，您可以使用我们的 [`name-default-component` 代码转换](/docs/pages/guides/upgrading/codemods#name-default-component)。

{/* md5:640d4f1bd354078feef2d24054a4d428 */}
随着您的代码库越来越多地迁移到函数组件和 Hooks，您可以期望在更多情况下保留状态。

{/* md5:bac4affa39d5a746bdfd7512c124dddd */}
## 提示

{/* md5:48e6d52d0d42eade7dd253a6af7476f6 */}
- 默认情况下，快速刷新会保留函数组件（和 Hooks）中的 React 本地状态。
- 有时您可能希望**强制**重置状态并重新挂载组件。例如，如果您正在调整仅在挂载时发生的动画，这会很有用。为此，您可以在正在编辑的文件中的任何位置添加 `// @refresh reset`。此指令是文件本地的，并指示快速刷新在每次编辑时重新挂载该文件中定义的组件。
- 您可以在开发期间编辑的组件中添加 `console.log` 或 `debugger;`。
- 请记住，导入是区分大小写的。如果您的导入与实际文件名不匹配，快速刷新和完全刷新都可能失败。例如，`'./header'` 与 `'./Header'`。

{/* md5:371d6ca2d9365b5709dd06677c2a92de */}
## 快速刷新与 Hooks

{/* md5:b17b2eb69c7e9f27eed90fd34a751d45 */}
在可能的情况下，快速刷新会尝试在编辑之间保留组件的状态。特别是，只要您不更改 `useState` 和 `useRef` 的参数或 Hook 调用的顺序，它们就会保留其先前值。

{/* md5:d52a9337bdb928fbcbdd00941a455b5c */}
具有依赖项的 Hook——例如 `useEffect`、`useMemo` 和 `useCallback`——在快速刷新期间将**始终**更新。在快速刷新发生时，它们的依赖项列表将被忽略。

{/* md5:7013fbc9f1a1c79b65e72c34b93da71a */}
例如，当您将 `useMemo(() => x * 2, [x])` 编辑为 `useMemo(() => x * 10, [x])` 时，即使 `x`（依赖项）没有更改，它也会重新运行。如果 React 不这样做，您的编辑就不会反映在屏幕上！

{/* md5:a7e42eeae107afb420847c33d5c4b53b */}
有时，这可能会导致意外的结果。例如，即使是一个依赖项数组为空的 `useEffect` 在快速刷新期间仍然会重新运行一次。

{/* md5:439d3444352093dc2554ecb4872d3427 */}
然而，编写能够适应 `useEffect` 偶尔重新运行的代码，即使没有快速刷新也是一个良好的实践。这将使您稍后更容易向其中引入新的依赖项，并且它由 [React 严格模式](/docs/pages/api-reference/config/next-config-js/reactStrictMode) 强制执行，我们强烈建议启用它。
