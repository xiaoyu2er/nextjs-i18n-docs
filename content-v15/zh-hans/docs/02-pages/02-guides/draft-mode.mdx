{/* md5:f186c36c546b52cb6ba7032dd8b560d1 */}
在 [页面文档](/docs/pages/building-your-application/routing/pages-and-layouts) 和 [数据获取文档](/docs/pages/building-your-application/data-fetching) 中，我们讨论了如何使用 `getStaticProps` 和 `getStaticPaths` 在构建时预渲染页面（**静态生成**）。

{/* md5:f7e5cba870f3f421b0c314a70bbff575 */}
当页面从无头 CMS 获取数据时，静态生成非常有用。但如果您正在无头 CMS 上撰写草稿并希望立即在页面上查看，这种方式就不理想了。此时您会希望 Next.js 在**请求时**而非构建时渲染这些页面，并获取草稿内容而非已发布内容。您需要 Next.js 仅在此特定情况下绕过静态生成。

{/* md5:1f17e9a46b42bb25653bcc4a0e468fc0 */}
Next.js 的**草稿模式 (Draft Mode)** 功能正是为此而生。以下是使用指南。

{/* md5:b73d28b62e6c0013d8032cdc50200a72 */}
## 第一步：创建并访问 API 路由

{/* md5:9fd1dde45fd3fb759797e1042184553c */}
> 如果您不熟悉 Next.js 的 API 路由，请先查看 [API 路由文档](/docs/pages/building-your-application/routing/api-routes)。

{/* md5:13228baecc8d3af8a85c0ad8098072ba */}
首先创建 **API 路由**，可任意命名（例如 `pages/api/draft.ts`）。

{/* md5:53b5e85240378934f335039aeacafee2 */}
在此 API 路由中，需在响应对象上调用 `setDraftMode`。

```js
export default function handler(req, res) {
  // ...
  res.setDraftMode({ enable: true })
  // ...
}
```

{/* md5:0e674be0d47c60c25fa6c7f3effe2126 */}
这将设置一个**Cookie**来启用草稿模式。后续携带此 Cookie 的请求将触发**草稿模式**，从而改变静态生成页面的行为（下文详述）。

{/* md5:2ac5687a1c44dc5b7604a2fefcdab840 */}
您可以通过手动创建如下 API 路由并从浏览器手动访问来进行测试：

```ts filename="pages/api/draft.ts"
// simple example for testing it manually from your browser.
export default function handler(req, res) {
  res.setDraftMode({ enable: true })
  res.end('Draft mode is enabled')
}
```

{/* md5:c748843592f29e734d735917944b5770 */}
如果在浏览器开发者工具中访问 `/api/draft`，会注意到响应头中有 `Set-Cookie` 字段，其值为名为 `__prerender_bypass` 的 Cookie。

{/* md5:311844498aeb3c63a085e1a865f67de6 */}
### 从无头 CMS 安全访问

{/* md5:57f7aa969e8957d2ff4a8d9311d2511c */}
实际上，您会希望从无头 CMS **安全地**调用此 API 路由。具体步骤会根据您使用的无头 CMS 而有所不同，但以下是一些常见的步骤。

{/* md5:78e74a0ad65bc03163fe5cb0b6bc6d7f */}
此方案假设您的无头 CMS 支持设置**自定义草稿 URL**。若不支持，您仍可用此方法保护草稿 URL，但需手动构建和访问。

{/* md5:ddc95365c5621c7d0c4e0db6e5556987 */}
**首先**，使用任意令牌生成器创建**密钥字符串**。该密钥仅由 Next.js 应用和无头 CMS 知晓，可防止无 CMS 访问权限者获取草稿 URL。

{/* md5:b02a1500de5b46dbd053f0e6f5b9256f */}
**其次**，若 CMS 支持自定义草稿 URL，请将以下内容设为草稿 URL（假设草稿 API 路由位于 `pages/api/draft.ts`）：

```bash filename="Terminal"
https://<your-site>/api/draft?secret=<token>&slug=<path>
```

{/* md5:5fc424c4633c7ae771b7d7e01bfc11e7 */}
- `<your-site>` 替换为部署域名
- `<token>` 替换为生成的密钥
- `<path>` 替换为目标页面路径（如欲查看 `/posts/foo`，则使用 `&slug=/posts/foo`）

{/* md5:7c95b073eea892dca49403628360b2c7 */}
某些 CMS 允许在草稿 URL 中包含变量，从而动态设置 `<path>`，例如：`&slug=/posts/{entry.fields.slug}`

{/* md5:d8eb18a938ce6082d30bfc5feed31e7c */}
**最后**，在草稿 API 路由中：

{/* md5:07a7f8900e510c1d58bf745a14d93391 */}
- 检查密钥是否匹配及 `slug` 参数是否存在（否则请求应失败）
- 调用 `res.setDraftMode`
- 重定向浏览器到 `slug` 指定路径（下例使用 [307 重定向](https://developer.mozilla.org/docs/Web/HTTP/Status/307)）

```js
export default async (req, res) => {
  // Check the secret and next parameters
  // This secret should only be known to this API route and the CMS
  if (req.query.secret !== 'MY_SECRET_TOKEN' || !req.query.slug) {
    return res.status(401).json({ message: 'Invalid token' })
  }

  // Fetch the headless CMS to check if the provided `slug` exists
  // getPostBySlug would implement the required fetching logic to the headless CMS
  const post = await getPostBySlug(req.query.slug)

  // If the slug doesn't exist prevent draft mode from being enabled
  if (!post) {
    return res.status(401).json({ message: 'Invalid slug' })
  }

  // Enable Draft Mode by setting the cookie
  res.setDraftMode({ enable: true })

  // Redirect to the path from the fetched post
  // We don't redirect to req.query.slug as that might lead to open redirect vulnerabilities
  res.redirect(post.slug)
}
```

{/* md5:642bc525206422e87bb8d52cb7e80f0e */}
如果成功，浏览器将被重定向到你想要查看的路径，并带上草稿模式 cookie。

{/* md5:047b638d5103f03e4854777ad1dbb042 */}
## 第二步：更新 `getStaticProps`

{/* md5:54c46fde3bfcf00f55b1a88d6ad59bce */}
接下来需更新 `getStaticProps` 以支持草稿模式。

{/* md5:d4dfcb8181b2051ef23c56e63694d707 */}
若请求的页面具有 `getStaticProps` 且已设置 Cookie（通过 `res.setDraftMode`），则 `getStaticProps` 将在**请求时**被调用（而非构建时）。

{/* md5:6dbe5aa831b881ad4fda2a16a65dd7ae */}
此外，其调用时将接收 `context` 对象，其中 `context.draftMode` 为 `true`。

```js
export async function getStaticProps(context) {
  if (context.draftMode) {
    // dynamic data
  }
}
```

{/* md5:5e0a46770cb8f733d93cca9c11b1eeb4 */}
由于我们在草稿 API 路由中使用了 `res.setDraftMode`，故 `context.draftMode` 将为 `true`。

{/* md5:c7e63c1f68146b8620ea8bf7238aa6de */}
如果您还使用 `getStaticPaths`，则 `context.params` 也将可用。

{/* md5:bc8ed986a666f2cdbe2f2bb9190b08f8 */}
### 获取草稿数据

{/* md5:f4f34a503cfabf99eb59c6b3933444eb */}
可根据 `context.draftMode` 修改 `getStaticProps` 以获取不同数据。

{/* md5:0982d44ab030ae114740a134e7b6bcea */}
例如，无头 CMS 可能有专用的草稿文章 API 端点。此时可如下修改端点 URL：

```js
export async function getStaticProps(context) {
  const url = context.draftMode
    ? 'https://draft.example.com'
    : 'https://production.example.com'
  const res = await fetch(url)
  // ...
}
```

{/* md5:acbb89a404e50560d906462d4a3d1b94 */}
完成！现在从无头 CMS 或手动访问草稿 API 路由（携带 `secret` 和 `slug`）即可查看草稿内容。当您更新草稿但未发布时，也能立即查看。

{/* md5:7406e78cd3821db7545bc45ce4f7c9f5 */}
将此 URL 设为无头 CMS 的草稿 URL 或手动访问：

```bash filename="Terminal"
https://<your-site>/api/draft?secret=<token>&slug=<path>
```

{/* md5:23bb3c7081fd6e014d15fe6321edd97d */}

### 清除草稿模式 Cookie

{/* md5:4b4d11a3287fd1fe3ec9f89546823235 */}
默认情况下，预览模式会话在浏览器关闭时结束。

{/* md5:b51c0415c6280a19e3a2fb135943fb9b */}
要手动清除 Cookie，可创建调用 `setDraftMode({ enable: false })` 的 API 路由：

```ts filename="pages/api/disable-draft.ts"
export default function handler(req, res) {
  res.setDraftMode({ enable: false })
}
```

{/* md5:39d8a34ba9f0eecb4f4a6bcea0c495aa */}
然后向 `/api/disable-draft` 发送请求。若使用 [`next/link`](/docs/pages/api-reference/components/link) 调用此路由，必须传递 `prefetch={false}` 以防预取时意外删除 Cookie。

{/* md5:2a963ad9477c85313a9bc7234120e889 */}
### 与 `getServerSideProps` 一起使用

{/* md5:460f4fa01fca10cb2504f3a85e212920 */}
草稿模式可与 `getServerSideProps` 协同工作，在[上下文对象](/docs/pages/api-reference/functions/get-server-side-props#context-parameter)中以 `draftMode` 键存在。

{/* md5:b94878a1a40ae099f48573568729b5f2 */}
> **须知**：使用草稿模式时不应设置 `Cache-Control` 标头，因其无法被绕过。建议改用 [ISR](/docs/pages/guides/incremental-static-regeneration)。

{/* md5:091cab05d5f1a6dfbde8d5face32acff */}
### 与 API 路由一起使用

{/* md5:2fd471b40f23828e581b4ce4555d3b44 */}
API 路由可通过请求对象的 `draftMode` 访问该状态。例如：

```js
export default function myApiRoute(req, res) {
  if (req.draftMode) {
    // get draft data
  }
}
```

{/* md5:dd9f4a814e2b9f17e6f72208faab35a1 */}
### 每个 `next build` 唯一

{/* md5:990a06417308b238df381b724ec71a8b */}
每次运行 `next build` 都会生成新的绕过 Cookie 值。

{/* md5:bece7549070eef1ca95604c20c212dd8 */}
这确保了绕过 Cookie 无法被猜测。

{/* md5:373895d6565d2b79ba4ab0867d71337f */}
> **须知**：要通过 HTTP 在本地测试草稿模式，浏览器需允许第三方 Cookie 和本地存储访问。
