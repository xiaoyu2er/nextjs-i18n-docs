{/* md5:af64632cd4817d2e98a3f3a864e7a0e5 */}
自 `v10.0.0` 版本起，Next.js 已内置支持国际化（[i18n](https://en.wikipedia.org/wiki/Internationalization_and_localization#Naming)）路由功能。您只需提供语言环境列表、默认语言环境以及特定域名的语言环境配置，Next.js 便会自动处理路由转换。

{/* md5:3c89bd8ed842ef0c0a8689f823b32a63 */}
i18n 路由支持目前旨在通过对路由和语言环境解析的简化，来补充现有的 i18n 库解决方案，如 [`react-intl`](https://formatjs.io/docs/getting-started/installation)、[`react-i18next`](https://react.i18next.com/)、[`lingui`](https://lingui.dev/)、[`rosetta`](https://github.com/lukeed/rosetta)、[`next-intl`](https://github.com/amannn/next-intl)、[`next-translate`](https://github.com/aralroca/next-translate)、[`next-multilingual`](https://github.com/Avansai/next-multilingual)、[`tolgee`](https://tolgee.io/integrations/next)、[`paraglide-next`](https://inlang.com/m/osslbuzt/paraglide-next-i18n)、[`next-intlayer`](https://intlayer.org/doc/environment/nextjs/next-with-page-router) 等。

{/* md5:da8327dd95c41f51c1dea5e58a4d1d17 */}
## 快速开始

{/* md5:fa4dbb381ce6743076569a997ac6ff67 */}
首先在 `next.config.js` 文件中添加 `i18n` 配置。

{/* md5:e81dae2062a4d01666e6d8dfe03392ad */}
语言环境需采用 [UTS 语言标识符](https://www.unicode.org/reports/tr35/tr35-59/tr35.html#Identifiers) 这一标准化格式。

{/* md5:b28f54bf51969faa7d74b914d121b154 */}
通常语言标识符由语言代码、地区代码和文字代码组成，格式为 `语言-地区-文字`。其中地区代码和文字代码为可选项。例如：

{/* md5:ed3d0e01b5342e09c2dd1904c8864f47 */}
- `en-US` - 美国英语
- `nl-NL` - 荷兰荷兰语
- `nl` - 荷兰语（无特定地区）

{/* md5:1cb846cdd046ead80de565d922d08246 */}
当用户访问的语言环境（如 `nl-BE`）未在配置中列出时，系统会尝试回退到 `nl`（如果存在），否则将使用默认语言环境。因此，若您不打算支持某个国家的所有地区变体，最佳实践是包含该国家的基础语言代码作为回退选项。

```js filename="next.config.js"
module.exports = {
  i18n: {
    // These are all the locales you want to support in
    // your application
    locales: ['en-US', 'fr', 'nl-NL'],
    // This is the default locale you want to be used when visiting
    // a non-locale prefixed path e.g. `/hello`
    defaultLocale: 'en-US',
    // This is a list of locale domains and the default locale they
    // should handle (these are only required when setting up domain routing)
    // Note: subdomains must be included in the domain value to be matched e.g. "fr.example.com".
    domains: [
      {
        domain: 'example.com',
        defaultLocale: 'en-US',
      },
      {
        domain: 'example.nl',
        defaultLocale: 'nl-NL',
      },
      {
        domain: 'example.fr',
        defaultLocale: 'fr',
        // an optional http field can also be used to test
        // locale domains locally with http instead of https
        http: true,
      },
    ],
  },
}
```

{/* md5:e75b2f5a3fa797355e8e89f475a41ea1 */}
## 语言环境策略

{/* md5:32d7d3731b140b70db4f424e99306546 */}
Next.js 提供两种语言环境处理策略：子路径路由和域名路由。

{/* md5:025e0176864af8aa3db7b40850e627e7 */}
### 子路径路由

{/* md5:63d14e7896383adff0bd00c2294f198d */}
子路径路由将语言代码置于 URL 路径中。

```js filename="next.config.js"
module.exports = {
  i18n: {
    locales: ['en-US', 'fr', 'nl-NL'],
    defaultLocale: 'en-US',
  },
}
```

{/* md5:a028fd83458067087792ec7d84ef984d */}
使用上述配置时，`en-US`、`fr` 和 `nl-NL` 将成为可路由的语言环境，其中 `en-US` 为默认语言环境。假设您有一个 `pages/blog.js` 页面，则以下 URL 将生效：

{/* md5:a93ac37d71cce76379a7457d46a9b1fa */}
- `/blog`
- `/fr/blog`
- `/nl-nl/blog`

{/* md5:b2150cac698bd08db76b8f1d28ef752b */}
默认语言环境不包含前缀。

{/* md5:b6b23d0d7c7f8c09869f70fbbc260b31 */}
### 域名路由

{/* md5:464e6f8510585b89cef500a6897980e9 */}
通过域名路由，您可以将不同语言环境配置到不同域名：

```js filename="next.config.js"
module.exports = {
  i18n: {
    locales: ['en-US', 'fr', 'nl-NL', 'nl-BE'],
    defaultLocale: 'en-US',

    domains: [
      {
        // Note: subdomains must be included in the domain value to be matched
        // e.g. www.example.com should be used if that is the expected hostname
        domain: 'example.com',
        defaultLocale: 'en-US',
      },
      {
        domain: 'example.fr',
        defaultLocale: 'fr',
      },
      {
        domain: 'example.nl',
        defaultLocale: 'nl-NL',
        // specify other locales that should be redirected
        // to this domain
        locales: ['nl-BE'],
      },
    ],
  },
}
```

{/* md5:b52c0effd82482d2b6348a8106ba7c59 */}
例如对于 `pages/blog.js` 页面，以下 URL 将生效：

{/* md5:ceded9c7d4400e81053ded47aa44774b */}
- `example.com/blog`
- `www.example.com/blog`
- `example.fr/blog`
- `example.nl/blog`
- `example.nl/nl-BE/blog`

{/* md5:822bc05880da2341b188a83df8a3306e */}
## 自动语言环境检测

{/* md5:50db74294066e8dc91d56f89daaa56a1 */}
当用户访问应用根路径（通常为 `/`）时，Next.js 会根据 [`Accept-Language`](https://developer.mozilla.org/docs/Web/HTTP/Headers/Accept-Language) 请求头和当前域名自动检测用户偏好的语言环境。

{/* md5:689bfcba6b34d8d8517b54c6cf3efc49 */}
若检测到非默认语言环境，用户将被重定向至：

{/* md5:b102d05c2c749ebda038e500e5e8cc82 */}
- **使用子路径路由时**：带语言前缀的路径
- **使用域名路由时**：配置了该语言环境为默认值的域名

{/* md5:e728b1b5f305e71111782307fefc4abe */}
例如在使用域名路由时，若带有 `Accept-Language: fr;q=0.9` 请求头的用户访问 `example.com`，将被重定向至 `example.fr`，因为该域名默认处理 `fr` 语言环境。

{/* md5:1b9308d1538567eac4db74f956cd9741 */}
使用子路径路由时，用户将被重定向至 `/fr`。

{/* md5:bb2fd851e958b659912e582faccd8d5a */}
### 为默认语言添加前缀

{/* md5:4a1d67998f026c12de0259795e927b54 */}
配合 Next.js 12 和 [Middleware](/docs/pages/api-reference/file-conventions/middleware)，我们可以通过一个 [变通方案](https://github.com/vercel/next.js/discussions/18419) 为默认语言环境添加前缀。

{/* md5:6c9082ea248ca0c8bb8d121c4b25a6cf */}
例如，以下 `next.config.js` 文件支持多种语言，请注意其中特意添加了 `"default"` 语言环境：

```js filename="next.config.js"
module.exports = {
  i18n: {
    locales: ['default', 'en', 'de', 'fr'],
    defaultLocale: 'default',
    localeDetection: false,
  },
  trailingSlash: true,
}
```

{/* md5:ea01fac34a5b882399ad7aba9a10346f */}
接下来，我们可以使用 [Middleware](/docs/pages/api-reference/file-conventions/middleware) 来添加自定义路由规则：

```ts filename="middleware.ts"
import { NextRequest, NextResponse } from 'next/server'

const PUBLIC_FILE = /\.(.*)$/

export async function middleware(req: NextRequest) {
  if (
    req.nextUrl.pathname.startsWith('/_next') ||
    req.nextUrl.pathname.includes('/api/') ||
    PUBLIC_FILE.test(req.nextUrl.pathname)
  ) {
    return
  }

  if (req.nextUrl.locale === 'default') {
    const locale = req.cookies.get('NEXT_LOCALE')?.value || 'en'

    return NextResponse.redirect(
      new URL(`/${locale}${req.nextUrl.pathname}${req.nextUrl.search}`, req.url)
    )
  }
}
```

{/* md5:c9b796cc4862f8698ca3615e2a00ab49 */}
此 [Middleware](/docs/pages/api-reference/file-conventions/middleware) 会跳过为 [API 路由](/docs/pages/building-your-application/routing/api-routes) 和 [公共资源](/docs/pages/api-reference/file-conventions/public-folder)（如字体或图像）文件添加默认前缀。如果请求针对默认语言环境，我们将重定向到我们的前缀 `/en`。

{/* md5:a9c8993a3576a6c70ea05c33512719f1 */}
### 禁用自动语言检测

{/* md5:741732f679209f1fe598df29d059b78b */}
可通过以下配置禁用自动语言检测：

```js filename="next.config.js"
module.exports = {
  i18n: {
    localeDetection: false,
  },
}
```

{/* md5:f34bfebf7275c1f580dfb6e14d8ceaba */}
当 `localeDetection` 设为 `false` 时，Next.js 将不再根据用户偏好语言自动重定向，仅提供从基于语言的域名或路径中检测到的语言环境信息。

{/* md5:1df901d2e3635f04c2a3eb015fe1a04d */}
## 访问语言环境信息

{/* md5:c369efc1d53f05c228866291f3b552e3 */}
您可以通过 Next.js 路由访问语言环境信息。例如使用 [`useRouter()`](/docs/pages/api-reference/functions/use-router) 钩子时，以下属性可用：

{/* md5:c6b82d0542b89c58027543cc0bc117e5 */}
- `locale` - 当前活动的语言环境
- `locales` - 所有配置的语言环境
- `defaultLocale` - 配置的默认语言环境

{/* md5:c89a65e776d0473005c6443bd016f478 */}
当使用 `getStaticProps` 或 `getServerSideProps` 对页面进行 [预渲染](/docs/pages/building-your-application/rendering/static-site-generation) 时，语言环境信息将在提供给函数的 [上下文](/docs/pages/building-your-application/data-fetching/get-static-props) 中提供。

{/* md5:3d3c10423a545231862b5cae3e8b2b2d */}
使用 `getStaticPaths` 时，配置的语言环境会通过上下文参数的 `locales` 属性提供，默认语言环境通过 `defaultLocale` 属性提供。

{/* md5:d5cf05a16daf6bdbc89165aae2e55d2a */}
## 语言环境切换

{/* md5:e0efc02adc653cca8b5d9a1b67996735 */}
您可以使用 `next/link` 或 `next/router` 实现语言环境切换。

{/* md5:bf81446cd7da33e0fa6b13b60da61846 */}
对于 `next/link`，可通过 `locale` 属性指定要切换的目标语言环境。若不提供 `locale` 属性，客户端切换时将使用当前活动语言环境。例如：

```jsx
import Link from 'next/link'

export default function IndexPage(props) {
  return (
    <Link href="/another" locale="fr">
      To /fr/another
    </Link>
  )
}
```

{/* md5:7631fffe3ac8ebcc0ed2248d55944e48 */}
直接使用 `next/router` 方法时，可通过过渡选项指定目标语言环境。例如：

```jsx
import { useRouter } from 'next/router'

export default function IndexPage(props) {
  const router = useRouter()

  return (
    <div
      onClick={() => {
        router.push('/another', '/another', { locale: 'fr' })
      }}
    >
      to /fr/another
    </div>
  )
}
```

{/* md5:766f7536ba79e802b5193e8a4f433fca */}
注意：若需在保留所有路由信息（如 [动态路由](/docs/pages/building-your-application/routing/dynamic-routes) 查询参数或隐藏的 href 查询值）的同时仅切换语言环境，可将 `href` 参数作为对象传递：

```jsx
import { useRouter } from 'next/router'
const router = useRouter()
const { pathname, asPath, query } = router
// change just the locale and maintain all other route information including href's query
router.push({ pathname, query }, asPath, { locale: nextLocale })
```

{/* md5:0cb6b0114552d6e98a957780325f8b8d */}
有关 `router.push` 对象结构的更多信息，请参阅 [此处](/docs/pages/api-reference/functions/use-router#with-url-object)。

{/* md5:8677add81591d71054a8cde087c977ef */}
若 href 已包含语言环境前缀，可通过以下方式禁用自动处理：

```jsx
import Link from 'next/link'

export default function IndexPage(props) {
  return (
    <Link href="/fr/another" locale={false}>
      To /fr/another
    </Link>
  )
}
```

{/* md5:d6786ca1965f09009a95d83595825c34 */}
## 利用 `NEXT_LOCALE` Cookie

{/* md5:cd3932f862ab9bf18343b51c2fdb9f41 */}
Next.js 支持设置 `NEXT_LOCALE=语言代码` Cookie，其优先级高于 accept-language 请求头。可通过语言切换器设置此 Cookie，当用户再次访问站点时，从 `/` 重定向时会优先使用 Cookie 中指定的语言环境。

{/* md5:f6a9f67aa94127a5b9d819afe8a4fb16 */}
例如，若用户请求头中的 accept-language 偏好为 `fr`，但设置了 `NEXT_LOCALE=en` Cookie，则访问 `/` 时将重定向至 `en` 语言环境，直到 Cookie 被删除或过期。

{/* md5:707c0bffa3073f91b467abfceca0205f */}
## 搜索引擎优化

{/* md5:dedaa4be40c73f4dc7f1e8d46248341d */}
由于 Next.js 知晓用户访问的语言环境，它会自动为 `<html>` 标签添加 `lang` 属性。

{/* md5:579e7796e228ab1fd100309f9d42f3bd */}
Next.js 不了解页面的语言变体，因此需要您使用 [`next/head`](/docs/pages/api-reference/components/head) 手动添加 `hreflang` 元标签。有关 `hreflang` 的更多信息，请参阅 [Google 站长文档](https://support.google.com/webmasters/answer/189077)。

{/* md5:37872443bf21432b16827f475e97a5a6 */}
## 静态生成如何运作？

{/* md5:630e66e4d04d8dd58d493b154bbd2aaf */}
> 注意：国际化路由不与 [`output: 'export'`](/docs/pages/guides/static-exports) 兼容，因其不依赖 Next.js 路由层。不使用 `output: 'export'` 的混合式 Next.js 应用可获得完整支持。

{/* md5:fc91d9b6de019117cae962c024ede6e3 */}
### 动态路由与 `getStaticProps` 页面

{/* md5:14b76c6c3e10713f928287014b4635d3 */}
对于使用 `getStaticProps` 的 [动态路由](/docs/pages/building-your-application/routing/dynamic-routes) 页面，所有需要预渲染的语言变体都需通过 [`getStaticPaths`](/docs/pages/building-your-application/data-fetching/get-static-paths) 返回。除了为 `paths` 返回的 `params` 对象外，还可指定 `locale` 字段以声明要渲染的语言环境。例如：

```jsx filename="pages/blog/[slug].js"
export const getStaticPaths = ({ locales }) => {
  return {
    paths: [
      // if no `locale` is provided only the defaultLocale will be generated
      { params: { slug: 'post-1' }, locale: 'en-US' },
      { params: { slug: 'post-1' }, locale: 'fr' },
    ],
    fallback: true,
  }
}
```

{/* md5:fb8d06ae8763e3402372a6e221c28117 */}
对于 [自动静态优化](/docs/pages/building-your-application/rendering/automatic-static-optimization) 和非动态的 `getStaticProps` 页面，**每个语言环境都会生成对应的页面版本**。这需要特别注意，因为根据配置的语言环境数量，可能会显著增加构建时间。

{/* md5:266ab03ede131240684ef011f701d115 */}
例如若配置了 50 个语言环境，且有 10 个非动态页面使用 `getStaticProps`，则 `getStaticProps` 将被调用 500 次。每次构建时会生成这 10 个页面的 50 个语言版本。

{/* md5:3dd9cad5e7c2027570f16d2cb726c60c */}
要减少动态页面的构建时间，可使用 [`fallback` 模式](/docs/pages/api-reference/functions/get-static-paths#fallback-true)。这样您只需在 `getStaticPaths` 中返回最受欢迎的路径和语言环境进行预渲染，剩余页面将在运行时按需生成。

{/* md5:8358793acd0892551ddf8883f3714d91 */}
### 自动静态优化页面

{/* md5:0ab87c485b408067726fab361c6e5e9a */}
对于 [自动静态优化](/docs/pages/building-your-application/rendering/automatic-static-optimization) 的页面，每个语言环境都会生成对应的页面版本。

{/* md5:d0452424ab8fbcc641eccbc499296501 */}
### 非动态 getStaticProps 页面

{/* md5:66d7d1d3c1ff5a6cbb0c7a65b9a7b4d2 */}
对于非动态的 `getStaticProps` 页面，如上所述会为每个语言环境生成一个版本。`getStaticProps` 会在渲染每个 `locale` 时被调用。如果您希望某个语言环境不进行预渲染，可以从 `getStaticProps` 返回 `notFound: true`，这样该页面的变体将不会被生成。

```js
export async function getStaticProps({ locale }) {
  // Call an external API endpoint to get posts.
  // You can use any data fetching library
  const res = await fetch(`https://.../posts?locale=${locale}`)
  const posts = await res.json()

  if (posts.length === 0) {
    return {
      notFound: true,
    }
  }

  // By returning { props: posts }, the Blog component
  // will receive `posts` as a prop at build time
  return {
    props: {
      posts,
    },
  }
}
```

{/* md5:d00d35253114d251a93f3c8de6cab4b7 */}
## i18n 配置限制

{/* md5:930243270fd0e6fc60001917b90f6ff1 */}
- `locales`：最多支持 100 个语言环境
- `domains`：最多支持 100 个域名配置项

{/* md5:6e9d56cc096d84c37d602ca808e2adc9 */}
> **小提示**：这些限制最初是为了防止潜在的 [构建时的性能问题](#dynamic-routes-and-getstaticprops-pages) 而添加的。您可以在 Next.js 12 中使用 [Middleware](/docs/pages/api-reference/file-conventions/middleware) 通过自定义路由绕过这些限制。
