## Default Behavior

Next.js 使用 PostCSS 编译其 [内置 CSS 支持](/docs/app/getting-started/css) 的 CSS。

开箱即用，无需配置，Next.js 使用以下转换来编译 CSS：

- [Autoprefixer](https://github.com/postcss/autoprefixer) 自动为 CSS 规则添加浏览器供应商前缀（兼容 IE11）。
- [跨浏览器 Flexbox 错误](https://github.com/philipwalton/flexbugs) 已被修正以符合 [规范](https://www.w3.org/TR/css-flexbox-1/) 的行为。
- 新的 CSS 功能会自动编译以支持 Internet Explorer 11 兼容性：
  - [`all`](https://developer.mozilla.org/docs/Web/CSS/all) 属性
  - [Break Properties](https://developer.mozilla.org/docs/Web/CSS/break-after)
  - [`font-variant`](https://developer.mozilla.org/docs/Web/CSS/font-variant) 属性
  - [Gap Properties](https://developer.mozilla.org/docs/Web/CSS/gap)
  - [Media Query Ranges](https://developer.mozilla.org/docs/Web/CSS/Media_Queries/Using_media_queries#Syntax_improvements_in_Level_4)

默认情况下，[CSS Grid](https://www.w3.org/TR/css-grid-1/) 和 [自定义属性](https://developer.mozilla.org/docs/Web/CSS/var)（CSS 变量）**不会被编译**以支持 IE11。

要在 IE11 中编译 [CSS Grid 布局](https://developer.mozilla.org/docs/Web/CSS/grid)，您可以在 CSS 文件顶部添加以下注释：

```css
/* autoprefixer grid: autoplace */
```

您也可以通过配置 autoprefixer（如下所示展开）在整个项目中启用 [CSS Grid 布局](https://developer.mozilla.org/docs/Web/CSS/grid) 的 IE11 支持。有关更多信息，请参阅下面的 [“自定义插件”](#customizing-plugins) 部分。

<details>

  <summary>点击查看启用 CSS Grid 布局的配置</summary>

```json filename="postcss.config.json"
{
  "plugins": [
    "postcss-flexbugs-fixes",
    [
      "postcss-preset-env",
      {
        "autoprefixer": {
          "flexbox": "no-2009",
          "grid": "autoplace"
        },
        "stage": 3,
        "features": {
          "custom-properties": false
        }
      }
    ]
  ]
}
```

</details>

CSS 变量不会被编译，因为 [无法安全地做到这一点](https://github.com/MadLittleMods/postcss-css-variables#caveats)。如果您必须使用变量，请考虑使用类似 [Sass 变量](https://sass-lang.com/documentation/variables) 的内容，这些变量会被 [Sass](https://sass-lang.com/) 编译掉。

## 自定义目标浏览器

Next.js 允许您通过 [Browserslist](https://github.com/browserslist/browserslist) 配置目标浏览器（适用于 [Autoprefixer](https://github.com/postcss/autoprefixer) 和编译后的 CSS 功能）。

要自定义 browserslist，请在 `package.json` 中创建一个 `browserslist` 键，如下所示：

```json filename="package.json"
{
  "browserslist": [">0.3%", "not dead", "not op_mini all"]
}
```

您可以使用 [browsersl.ist](https://browsersl.ist/?q=%3E0.3%25%2C+not+ie+11%2C+not+dead%2C+not+op_mini+all) 工具可视化您正在针对哪些浏览器。

## CSS Modules

无配置即可支持 CSS 模块。要为文件启用 CSS 模块，请将文件重命名为 `.module.css` 扩展名。

您可以在此处了解 [Next.js 的 CSS 模块支持](/docs/app/getting-started/css)。

## Customizing Plugins

> **警告**：当您定义自定义 PostCSS 配置文件时，Next.js 将**完全禁用** [默认行为](#default-behavior)。
> 请务必手动配置所有需要编译的功能，包括 [Autoprefixer](https://github.com/postcss/autoprefixer)。
> 您还需要手动安装自定义配置中包含的任何插件，例如 `npm install postcss-flexbugs-fixes postcss-preset-env`。

要自定义 PostCSS 配置，请在项目根目录中创建一个 `postcss.config.json` 文件。

这是 Next.js 使用的默认配置：

```json filename="postcss.config.json"
{
  "plugins": [
    "postcss-flexbugs-fixes",
    [
      "postcss-preset-env",
      {
        "autoprefixer": {
          "flexbox": "no-2009"
        },
        "stage": 3,
        "features": {
          "custom-properties": false
        }
      }
    ]
  ]
}
```

> **了解**：Next.js 还允许文件命名为 `.postcssrc.json`，或者从 `package.json` 中的 `postcss` 键读取。

也可以通过 `postcss.config.js` 文件配置 PostCSS，这在您想根据环境条件包含插件时很有用：

```js filename="postcss.config.js"
module.exports = {
  plugins:
    process.env.NODE_ENV === 'production'
      ? [
          'postcss-flexbugs-fixes',
          [
            'postcss-preset-env',
            {
              autoprefixer: {
                flexbox: 'no-2009',
              },
              stage: 3,
              features: {
                'custom-properties': false,
              },
            },
          ],
        ]
      : [
          // No transformations in development
        ],
}
```

> **了解**：Next.js 还允许文件命名为 `.postcssrc.js`。

请**不要使用 `require()`** 导入 PostCSS 插件。插件必须作为字符串提供。

> **了解**：如果您的 `postcss.config.js` 需要支持同一项目中的其他非 Next.js 工具，则必须使用可互操作的基于对象的格式代替：
>
> ```js
> module.exports = {
>   plugins: {
>     'postcss-flexbugs-fixes': {},
>     'postcss-preset-env': {
>       autoprefixer: {
>         flexbox: 'no-2009',
>       },
>       stage: 3,
>       features: {
>         'custom-properties': false,
>       },
>     },
>   },
> }
> ```
