AMP とは？[](#what-is-amp)
----------------------------

AMP は、レンダリングのオーバーヘッドを最小化し、主要な検索エンジンで拡張された動作と一緒にインデックス付けできる、高パフォーマンスなウェブサイトの構築のための標準です。例えば、AMP ページは Google のモバイル検索結果に直接読み込まれ、ライトニングアイコンでマークされます。

![Next.js の Google 検索結果](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-8-1/next-amp-result.png)

Next.js の Google 検索結果

AMP HTML は、より信頼性の高いパフォーマンスと優れたスケーラビリティを実現するためにいくつかの制限を課す、HTML のより厳格なバージョンです。いくつかの HTML タグは、対応する HTML タグよりも優れたユーザーエクスペリエンスを提供するために、AMP 固有の HTML タグに置き換えられます。例えば、[`amp-img`](https://amp.dev/documentation/components/amp-img) タグは、まだ対応していないブラウザでも完全な `srcset` サポートを提供します。

AMP ページは、それに対応する検索エンジンによって自動的に発見されます。これらの検索エンジンは通常、[AMP Cache](https://amp.dev/documentation/guides-and-tutorials/learn/amp-caches-and-cors/how_amp_pages_are_cached) (例えば [Google](https://developers.google.com/amp/cache/) や [Bing](https://www.bing.com/webmaster/help/bing-amp-cache-bc1c884c)) を実装しています。AMP キャッシュは、ページを検索結果からより高速にレンダリングするのに役立ちます。

Next.js における AMP
--------------------------------

本日、Next.js のエクスペリエンスを AMP ページの作成に拡張したことを発表できることを嬉しく思います。これにより、React コンポーネントの力を活用して AMP ページを作成できるようになります。AMP サポートはページベースで定義されるため、AMP の段階的な採用が可能になります。Next.js で AMP を有効にするには 2 つの方法があります：ハイブリッド AMP ページまたは AMP ファースト ページです。

ハイブリッド AMP ページ
-------------------------------------

ハイブリッド AMP ページを使用すると、従来のページの AMP バージョンを併用して、検索エンジンがモバイル検索結果でページの AMP バージョンを表示できるようにし、既存のページのバージョンを維持できます。これにより、アプリケーションに AMP を使用しながら、メインのユーザーエクスペリエンスには Next.js の機能であるクライアントサイドルーティングなどを引き続き利用することができます。

ハイブリッド AMP ページに最適インする場合は、`withAmp` 関数を使用し、`hybrid: true` オプションを提供してください。

pages/index.js

```
function HomePage() {
  return <p>Welcome to AMP + Next.js.</p>;
}
 
export default withAmp(HomePage, { hybrid: true });
```

生産環境で使用されているハイブリッド AMP パターンの 1 つの例は Reddit です。各スレッドは [Google の AMP Cache](https://developers.google.com/amp/cache/) に保存されており、モバイルウェブ全体で高速なユーザーエクスペリエンスを提供しながら、デスクトップには Reddit のフルバージョンを提供し、その後のナビゲーションを行います。

### AMP ファースト ページ[](#amp-first-pages)

AMP ファースト ページは、ウェブサイトの主要なトラフィックおよび検索エンジントラフィックに提供されます。AMP ファースト ページを使用することで、デスクトップも含めて、AMP の高速なエクスペリエンスをメインのウェブサイトにもたらします。

AMP ファースト ページを実装するには、ページを `withAmp` 関数でラップします。

pages/index.js

```
import { withAmp } from 'next/amp';
function HomePage() {
  return <p>Welcome to AMP + Next.js.</p>;
}
 
export default withAmp(HomePage);
```

Next.js ランタイムが必要ない場合、AMP ファースト ページを使用すると開発がスピードアップする可能性があります。AMP ファーストを使用すると、ページを構築するために AMP 互換コンポーネントのみを使用する必要があります。AMP ファースト ページは現在、[nextjs.org/docs](https://nextjs.org/docs) および [nextjs.org/blog](https://nextjs.org/blog) で本番環境で使用されています。

### コンポーネント内での AMP レンダリングの区別 [](#distinguishing-amp-rendering-in-components)

プロジェクト内のすべての React コンポーネントで `useAmp` を使用して、AMP モードと非 AMP レンダリングモードの区別を行うことができます。これにより、`<img>` と `<amp-img>` の両方でロジックを共有する `<Image>` コンポーネントを実装できます。

components/image.js

```
import { useAmp } from 'next/amp';
 
export function Image({ src, width, height }) {
  const isAmp = useAmp();
  return isAmp ? (
    <amp-img src={src} width={width} height={height} />
  ) : (
    <img src={src} width={width} height={height} />
  );
}
```

pages/index.js

```
import { withAmp } from 'next/amp';
import { Image } from '../components/image';
 
function HomePage() {
  return (
    <>
      <p>Hello! Welcome to AMP + Next.js.</p>
      <Image src="https://placehold.it/120" width="120" height="120" />
    </>
  );
}
 
export default withAmp(HomePage, { hybrid: true });
```

AMP 開発のための自動再読み込み [](#automatic-reloading-for-amp-development)
-----------------------------------------------------------------------------------

開発中は、ホットモジュール置換 (Hot-module-replacement) を使用する代わりに、現在表示しているページのすべての変更を追跡し、変更があった場合にのみページを再読み込みします。ホットモジュール置換ではなくフル再読み込みを使用する理由は、常に AMP ページの新鮮なサーバーサイドレンダリングが表示されていることを確認するためです。

AMP 検証 [](#amp-validation)
---------------------------------

有効な AMP ページのみが生成されるようにするには、開発中 [`amphtml-validator`](https://www.npmjs.com/package/amphtml-validator) で自動的に検証します。エラーと警告は Next.js を起動したターミナルに表示されます。

ページは `next export` 中にも検証され、問題はすべてターミナルに印刷されます。AMP エラーは、出力が有効な AMP ではないため、`next export` の失敗を引き起こします。
