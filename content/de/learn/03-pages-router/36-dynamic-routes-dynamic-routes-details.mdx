### [Externe API abfragen oder Datenbank abfragen](#fetch-external-api-or-query-database)

Ă„hnlich wie [`getStaticProps`](/docs/basic-features/data-fetching#getstaticprops-static-generation) kann auch [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) Daten aus beliebigen Datenquellen abrufen. In unserem Beispiel kĂ¶nnte `getAllPostIds` (das von [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) verwendet wird) von einem externen API-Endpunkt abrufen:

```

  // Anstelle des Dateisystems,
  // werden Post-Daten von einem externen API-Endpunkt abgerufen
  const res = await fetch('..');
  const posts = await res.json();
  return posts.map((post) => {
    return {
      params: {
        id: post.id,
      },
    };
  });
}
```

### [Entwicklung vs. Produktion](#development-vs-production)

*   In der **Entwicklung** (`npm run dev` oder `yarn dev`) wird [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) bei _jeder Anfrage_ ausgefĂĽhrt.
*   In der **Produktion** wird [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) zur _Build-Zeit_ ausgefĂĽhrt.

### [Fallback](#fallback)

Erinnern Sie sich, dass wir `fallback: false` aus [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) zurĂĽckgegeben haben. Was bedeutet das?

Wenn [`fallback` auf `false`](/docs/basic-features/data-fetching#fallback-false) gesetzt ist, fĂĽhren alle Pfade, die nicht von [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) zurĂĽckgegeben werden, zu einer **404-Seite**.

Wenn [`fallback` auf `true`](/docs/basic-features/data-fetching#fallback-true) gesetzt ist, Ă¤ndert sich das Verhalten von [`getStaticProps`](/docs/basic-features/data-fetching#getstaticprops-static-generation):

*   Die von [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) zurĂĽckgegebenen Pfade werden zur Build-Zeit als HTML gerendert.
*   Pfade, die zur Build-Zeit nicht generiert wurden, fĂĽhren **nicht** zu einer 404-Seite. Stattdessen serviert Next.js eine "Fallback"-Version der Seite bei der ersten Anfrage an einen solchen Pfad.
*   Im Hintergrund generiert Next.js den angefragten Pfad statisch. Nachfolgende Anfragen an denselben Pfad erhalten die generierte Seite, genau wie andere Seiten, die zur Build-Zeit vorgerendert wurden.

Wenn [`fallback` auf `blocking`](/docs/basic-features/data-fetching#fallback-blocking) gesetzt ist, werden neue Pfade serverseitig mit `getStaticProps` gerendert und fĂĽr zukĂĽnftige Anfragen zwischengespeichert, sodass dies nur einmal pro Pfad geschieht.

Dies geht ĂĽber den Rahmen unserer Lektionen hinaus, aber Sie kĂ¶nnen mehr ĂĽber `fallback: true` und `fallback: 'blocking'` in der [`fallback`-Dokumentation](/docs/api-reference/data-fetching/get-static-paths#fallback-false) erfahren.

### [Catch-all-Routen](#catch-all-routes)

Dynamische Routen kĂ¶nnen erweitert werden, um alle Pfade abzufangen, indem drei Punkte (`...`) innerhalb der Klammern hinzugefĂĽgt werden. Zum Beispiel:

*   `pages/posts/[...id].js` passt auf `/posts/a`, aber auch auf `/posts/a/b`, `/posts/a/b/c` und so weiter.

Wenn Sie dies tun, mĂĽssen Sie in [`getStaticPaths`](/docs/basic-features/data-fetching#getstaticpaths-static-generation) ein Array als Wert des `id`-SchlĂĽssels zurĂĽckgeben, wie folgt:

```
return [
  {
    params: {
      // Statisch generiert /posts/a/b/c
      id: ['a', 'b', 'c'],
    },
  },
  //...
];
```

Und `params.id` wird in [`getStaticProps`](/docs/basic-features/data-fetching#getstaticprops-static-generation) ein Array sein:

```
export async function getStaticProps({ params }) {
  // params.id wird wie ['a', 'b', 'c'] sein
}
```

Werfen Sie einen Blick in die [Dokumentation zu Catch-all-Routen](/docs/routing/dynamic-routes#catch-all-routes), um mehr zu erfahren.

### [404-Seiten](#404-pages)

Um eine [benutzerdefinierte 404-Seite](/docs/advanced-features/custom-error-page#404-page) zu erstellen, erstellen Sie `pages/404.js`. Diese Datei wird zur Build-Zeit statisch generiert.

```
// pages/404.js
export default function Custom404() {
  return <h1>404 - Seite nicht gefunden</h1>;
}
```
