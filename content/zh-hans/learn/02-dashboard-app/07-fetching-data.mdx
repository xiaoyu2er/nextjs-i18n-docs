既然你已经创建并填充了数据库，让我们讨论一下获取应用程序数据的不同方式，并构建仪表板概述页面。

选择如何获取数据 [](#choosing-how-to-fetch-data)
---------------------------------------------------------

### API 层 (API layer) [](#api-layer)

API 是应用程序代码和数据库之间的中介层。有几种情况你会使用 API：

*   如果你使用的是提供 API 的第三方服务。
*   如果你在客户端获取数据，你想要有一个在服务器上运行的 API 层 (API layer)，以避免向客户端暴露你的数据库密钥。

在 Next.js 中，你可以使用 [Route Handlers](/docs/app/building-your-application/routing/route-handlers) 创建 API 端点。

### 数据库查询 (Database queries) [](#database-queries)

当你在创建全栈应用程序时，你还编写与数据库交互的逻辑。对于像 Postgres [这样的关系数据库](https://aws.amazon.com/relational-database/)，你可以使用 SQL 或 [ORM](https://vercel.com/docs/storage/vercel-postgres/using-an-orm) 来实现这一点。

有几种情况你需要编写数据库查询：

*   在创建 API 端点时，你需要编写与数据库交互的逻辑。
* 如果你正在使用 React 服务器组件 (React Server Components)（在服务器上获取数据），你可以跳过 API 层，直接查询数据库，而不用担心向客户端暴露数据库密钥。

让我们了解更多关于 React 服务器组件 (React Server Components) 的知识。

### 使用 服务器组件 (Server Components) 获取数据 [](#using-server-components-to-fetch-data)

默认情况下，Next.js 应用程序使用 **React 服务器组件 (React Server Components)**。使用 服务器组件 (Server Components) 获取数据是一种相对较新的方法，并且有一些好处：

*   服务器组件 (Server Components) 支持 JavaScript Promises，原生地为数据获取等异步任务提供解决方案。你可以使用 `async/await` 语法，而无需 `useEffect`、`useState` 或其他数据获取库。
* 服务器组件 (Server Components) 在服务器上运行，因此你可以将昂贵的数据获取和逻辑保留在服务器上，仅将结果发送到客户端。
* 由于服务器组件 (Server Components) 在服务器上运行，你可以直接查询数据库而无需额外的 API 层。这可以节省你编写和维护额外代码的时间。

### 使用 SQL [](#using-sql)

对于你的仪表板应用程序，你将使用 [postgres.js](https://github.com/porsager/postgres) 库和 SQL 编写数据库查询。我们使用 SQL 有几个原因：

*   SQL 是查询关系数据库的行业标准（例如，ORM 在底层生成 SQL）。
* 对 SQL 有基本了解可以帮助你理解关系数据库的基础知识，从而让你能够将知识应用到其他工具上。
* SQL 非常通用，允许你获取和操纵特定数据。
* `postgres.js` 库提供了针对 [SQL 注入](https://github.com/porsager/postgres?tab=readme-ov-file#query-parameters) 的保护。

如果你以前没用过 SQL，别担心——我们已经为你提供了查询。

前往 `/app/lib/data.ts`。在这里你会看到我们正在使用 `postgres`。`sql` [function](https://github.com/porsager/postgres) 允许你查询数据库：

/app/lib/data.ts

```
import postgres from 'postgres';
 
const sql = postgres(process.env.POSTGRES_URL!, { ssl: 'require' });
 
// ...
```

你可以在服务器上的任意位置调用 `sql`，就像在 服务器组件 (Server Components) 中一样。但为了方便你更轻松地导航组件，我们将所有数据查询保留在 `data.ts` 文件中，你可以将它们导入到组件中。

> **注意：** 如果你在章节 6 中使用了您自己的数据库提供商，则需要更新数据库查询以适配您的提供商。你可以在 `/app/lib/data.ts` 中找到这些查询。

获取 **`<LatestInvoices/>`** 的数据 [](#fetching-data-for-latestinvoices)
------------------------------------------------------------------------------

对于 `<LatestInvoices />` 组件，我们需要获取按日期排序的前 5 张最新发票。

你可以获取所有发票并使用 JavaScript 进行排序。这目前不是问题，因为我们的数据量很小，但随着应用程序的增长，这可能会显著增加每次请求传输的数据量和用于排序所需的 JavaScript 代码。

与其在内存中对最新发票进行排序，你可以使用 SQL 查询仅获取最后 5 张发票。例如，这是来自你的 `data.ts` 文件的 SQL 查询：

/app/lib/data.ts

```
// Fetch the last 5 invoices, sorted by date
const data = await sql<LatestInvoiceRaw[]>`
  SELECT invoices.amount, customers.name, customers.image_url, customers.email
  FROM invoices
  JOIN customers ON invoices.customer_id = customers.id
  ORDER BY invoices.date DESC
  LIMIT 5`;
```

在你的页面中，导入 `fetchLatestInvoices` 函数：

/app/dashboard/page.tsx

```
import { Card } from '@/app/ui/dashboard/cards';
import RevenueChart from '@/app/ui/dashboard/revenue-chart';
import LatestInvoices from '@/app/ui/dashboard/latest-invoices';
import { lusitana } from '@/app/ui/fonts';
import { fetchRevenue, fetchLatestInvoices } from '@/app/lib/data';
 
export default async function Page() {
  const revenue = await fetchRevenue();
  const latestInvoices = await fetchLatestInvoices();
  // ...
}
```

然后，取消注释 `<LatestInvoices />` 组件。你还需要在位于 `/app/ui/dashboard/latest-invoices` 的 `<LatestInvoices />` 组件本身中取消相关代码的注释。

如果你访问 localhost，你应该会看到只有最后 5 条是从数据库返回的。希望你现在开始明白直接查询数据库的优势了！

什么是请求瀑布流？ (request waterfalls) [](#what-are-request-waterfalls)
------------------------------------------------------------

“瀑布流”指的是一系列依赖前序请求完成的网络请求。在数据获取的情况下，每个请求只有在之前的请求返回数据后才能开始。

![显示顺序数据获取和并行数据获取的时间图](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/sequential-parallel-data-fetching.png)![显示顺序数据获取和并行数据获取的时间图](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fsequential-parallel-data-fetching.png&w=3840&q=75)

例如，在 `fetchLatestInvoices()` 开始运行之前，我们需要等待 `fetchRevenue()` 执行，以此类推。

/app/dashboard/page.tsx

```
const revenue = await fetchRevenue();
const latestInvoices = await fetchLatestInvoices(); // wait for fetchRevenue() to finish
const {
  numberOfInvoices,
  numberOfCustomers,
  totalPaidInvoices,
  totalPendingInvoices,
} = await fetchCardData(); // wait for fetchLatestInvoices() to finish
```

这种模式并不一定是坏的。可能存在你想要瀑布流的情况，因为希望在提出下一个请求之前满足某个条件。例如，你可能想先获取用户的 ID 和个人资料信息。一旦有了 ID，你可能接着获取他们的朋友列表。在这种情况下，每个请求都取决于前一个请求返回的数据。

然而，这种行为也可能是不自觉的，并影响性能。

并行数据获取 [](#parallel-data-fetching)
-------------------------------------------------

避免瀑布流的一种常见方法是同时发起所有数据请求——并行执行。

在 JavaScript 中，你可以使用 [`Promise.all()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise/all) 或 [`Promise.allSettled()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise/allSettled) 函数同时发起所有 promise。例如，在 `data.ts` 中，我们在 `fetchCardData()` 函数中使用了 `Promise.all()`：

/app/lib/data.ts

```
export async function fetchCardData() {
  try {
    const invoiceCountPromise = sql`SELECT COUNT(*) FROM invoices`;
    const customerCountPromise = sql`SELECT COUNT(*) FROM customers`;
    const invoiceStatusPromise = sql`SELECT
         SUM(CASE WHEN status = 'paid' THEN amount ELSE 0 END) AS "paid",
         SUM(CASE WHEN status = 'pending' THEN amount ELSE 0 END) AS "pending"
         FROM invoices`;
 
    const data = await Promise.all([
      invoiceCountPromise,
      customerCountPromise,
      invoiceStatusPromise,
    ]);
    // ...
  }
}
```

通过使用该模式，你可以：

*   同时开始执行所有数据获取，这比在瀑布流中等待每个请求完成更快。
* 使用可应用于任何库或框架的原生 JavaScript 模式。

然而，仅依赖这种 JavaScript 模式有一个**缺点 (disadvantage)**：如果一个数据请求比其他的慢怎么办？让我们在下一章中了解更多。
