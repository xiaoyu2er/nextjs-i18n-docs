创建新项目 [](#creating-a-new-project)
-------------------------------------------------

我们推荐使用 [`pnpm`](https://pnpm.io/) 作为你的包管理器，因为它比 `npm` 或 `yarn` 更快且更高效。如果你没有安装 `pnpm`，可以通过运行以下命令全局安装：

终端

```
npm install -g pnpm
```

要创建 Next.js 应用，打开你的终端，[`cd`](https://developer.mozilla.org/en-US/docs/Learn/Tools_and_testing/Understanding_client-side_tools/Command_line#basic_built-in_terminal_commands) 到你希望存放项目的文件夹，然后运行以下命令：

终端

```
npx create-next-app@latest nextjs-dashboard --example "https://github.com/vercel/next-learn/tree/main/dashboard/starter-example" --use-pnpm
```

此命令使用了 [`create-next-app`](/docs/app/api-reference/create-next-app)，这是一个为你设置 Next.js 应用的命令行 (CLI) 工具。在上述命令中，你还使用了 `--example` 标志，配合本课程的 [入门示例](https://github.com/vercel/next-learn/tree/main/dashboard/starter-example)。

探索项目 [](#exploring-the-project)
-----------------------------------------------

不同于从零开始编写代码的教程，本课程的大部分代码已经为你写好了。这更好地反映了现实世界中的开发，在那里你很可能是在现有的代码库中工作。

我们的目标是帮助你专注于学习 Next.js 的主要功能，而无需编写 _all_ 的应用代码。

安装后，在你的代码编辑器中打开该项目，然后导航到 `nextjs-dashboard`。

终端

```
cd nextjs-dashboard
```

让我们花一些时间探索这个项目。

### 文件夹结构 [](#folder-structure)

你会注意到这个项目有以下文件夹结构：

![仪表板项目的文件夹结构，显示了主要文件夹和文件：app, public 和配置文件。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/learn-folder-structure.png)![仪表板项目的文件夹结构，显示了主要文件夹和文件：app, public 和配置文件。](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Flearn-folder-structure.png&w=3840&q=75)

*   **`/app`**: 包含你应用的所有路由、组件和逻辑，你主要从这里开始工作。
*   **`/app/lib`**: 包含你应用中使用的函数，例如可复用的工具函数和数据获取函数。
*   **`/app/ui`**: 包含你应用的所有 UI 组件，例如卡片、表格和表单。为了节省时间，我们已预先为这些组件设置了样式。
*   **`/public`**: 包含你应用的所有静态资源，例如图片。
*   **配置文件**: 你还会注意到应用根目录中有 `next.config.ts` 等配置文件。大多数这些文件在使用 `create-next-app` 启动新项目时已创建并预配置。在本课程中你不需要修改它们。

随意探索这些文件夹，如果你暂时不理解代码所做的一切，也不要担心。

### 占位数据 [](#placeholder-data)

在构建用户界面时，有一些占位数据是有帮助的。如果数据库或 API 还不可用，你可以：

*   使用 JSON 格式或 JavaScript 对象的占位数据。
*   使用像 [mockAPI](https://mockapi.io/) 这样的第三方服务。

对于这个项目，我们已在 `app/lib/placeholder-data.ts` 中提供了一些占位数据。文件中的每个 JavaScript 对象代表你的数据库中的一张表。例如，对于发票表：

/app/lib/placeholder-data.ts

```
const invoices = [
  {
    customer_id: customers[0].id,
    amount: 15795,
    status: 'pending',
    date: '2022-12-06',
  },
  {
    customer_id: customers[1].id,
    amount: 20348,
    status: 'pending',
    date: '2022-11-14',
  },
  // ...
];
```

在 [设置数据库](/learn/dashboard-app/setting-up-your-database) 章节中，你将使用这些数据来 _初始化 (seed)_ 你的数据库（即用一些初始数据填充它）。

### TypeScript[](#typescript)

可能还会注意到大多数文件都有 `.ts` 或 `.tsx` 后缀。这是因为项目是使用 TypeScript 编写的。我们要创建一个反映现代网络环境的课程。

如果你不知道 TypeScript 也没关系 - 我们需要时会提供 TypeScript 代码片段。

现在，看一下 `/app/lib/definitions.ts` 文件。这里，我们手动定义了将从数据库返回的类型。例如，发票表有以下类型：

/app/lib/definitions.ts

```
export type Invoice = {
  id: string;
  customer_id: string;
  amount: number;
  date: string;
  // In TypeScript, this is called a string union type.
  // It means that the "status" property can only be one of the two strings: 'pending' or 'paid'.
  status: 'pending' | 'paid';
};
```

使用 TypeScript，你可以确保不会意外地将错误的数据格式传递给组件或数据库，例如向发票 `amount` 传递 `string` 而不是 `number`。

> **如果你是 TypeScript 开发者：**
> 
> *   我们手动声明数据类型，但为了更好的类型安全，我们推荐 [Prisma](https://www.prisma.io/) 或 [Drizzle](https://orm.drizzle.team/)，它们会根据你的数据库架构自动生成类型。
> *   Next.js 会检测你的项目是否使用 TypeScript，并自动安装必要的包和配置。Next.js 还自带了一个 [TypeScript 插件](https://nextjs.org/docs/app/building-your-application/configuring/typescript#typescript-plugin) 供你的代码编辑器使用，以帮助自动补全和类型安全。

启动开发服务器 [](#running-the-development-server)
-----------------------------------------------------------------

运行 `pnpm i` 安装项目的包。

终端

```
pnpm i
```

然后是 `pnpm dev` 启动开发服务器。

终端

```
pnpm dev
```

 `pnpm dev` 在端口 `3000` 上启动你的 Next.js 开发服务器。让我们检查一下它是否正常工作。

在你的浏览器中打开 [http://localhost:3000](http://localhost:3000/)。你的主页应该看起来像这样，这是故意未样式的：
