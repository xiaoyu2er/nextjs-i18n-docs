什么是 Server Actions？[](#what-are-server-actions)
----------------------------------------------------

React Server Actions 允许你直接在服务器上运行异步代码。它们消除了创建 API 端点来变更数据的需要。相反，你编写在服务器上执行的异步函数，并且可以从你的客户端或服务器组件中调用。

安全是 Web 应用的首要任务，因为它们容易受到各种威胁。这就是 Server Actions 发挥作用的地方。它们包含诸如加密闭包、严格输入检查、错误消息哈希、主机限制等功能——所有这些功能协同工作，显著增强你的应用程序安全性。

使用表单与 Server Actions[](#using-forms-with-server-actions)
-------------------------------------------------------------------

在 React 中，你可以使用 `<form>` 元素中的 `action` 属性来调用操作。该操作将自动接收原生的 [FormData](https://developer.mozilla.org/en-US/docs/Web/API/FormData) 对象，其中包含捕获的数据。

例如：

```
// Server Component
export default function Page() {
  // Action
  async function create(formData: FormData) {
    'use server';
 
    // Logic to mutate data...
  }
 
  // Invoke the action using the "action" attribute
  return <form action={create}>...</form>;
}
```

在服务器组件内部调用 Server Action 的一个优势是渐进式增强——即使 JavaScript 尚未在客户端加载，表单也能正常工作。例如，在较慢的网络连接下。

Next.js 与 Server Actions[](#nextjs-with-server-actions)
----------------------------------------------------------

Server Actions 也与 Next.js [缓存](https://nextjs.org/docs/app/building-your-application/caching)深度集成。当通过 Server Action 提交表单时，你不仅可以使用该操作来变更数据，还可以使用 `revalidatePath` 和 `revalidateTag` 等 API 来重新验证相关的缓存。

让我们看看它们是如何协同工作的！

创建发票 [](#creating-an-invoice)
-------------------------------------------

以下是创建新发票的步骤：

1.  创建一个表单来捕获用户输入。
2.  创建一个 Server Action 并从表单调用它。
3.  在你的 Server Action 内部，从 `formData` 对象中提取数据。
4.  验证并准备要插入数据库的数据。
5.  插入数据并处理任何错误。
6.  重新验证缓存并将用户重定向回发票页面。

### 2\. 创建 Server Action[](#2-create-a-server-action)

很好，现在让我们创建一个在表单提交时调用的 Server Action。

导航到你的 `lib/` 目录并创建一个名为 `actions.ts` 的新文件。在此文件顶部，添加 React [`use server`](https://react.dev/reference/react/use-server) 指令：

/app/lib/actions.ts

```
'use server';
```

通过添加 `'use server'`，你将文件中所有导出的函数标记为 Server Actions。然后可以在客户端和服务器组件中导入并使用这些服务器函数。此文件中包含的任何未使用的函数将自动从最终应用程序包中移除。

你也可以通过在操作内部添加 `"use server"` 来直接在服务器组件内部编写 Server Actions。但在本课程中，我们将它们全部组织在一个单独的文件中。我们建议为你的操作创建一个单独的文件。

在你的 `actions.ts` 文件中，创建一个新的异步函数，该函数接受 `formData`：

/app/lib/actions.ts

```
'use server';
 
export async function createInvoice(formData: FormData) {}
```

然后，在你的 `<Form>` 组件中，从你的 `actions.ts` 文件导入 `createInvoice`。向 `<form>` 元素添加 `action` 属性，并调用 `createInvoice` 操作。

/app/ui/invoices/create-form.tsx

```
import { CustomerField } from '@/app/lib/definitions';
import Link from 'next/link';
import {
  CheckIcon,
  ClockIcon,
  CurrencyDollarIcon,
  UserCircleIcon,
} from '@heroicons/react/24/outline';
import { Button } from '@/app/ui/button';
import { createInvoice } from '@/app/lib/actions';
 
export default function Form({
  customers,
}: {
  customers: CustomerField[];
}) {
  return (
    <form action={createInvoice}>
      // ...
  )
}
```

> **须知**：在 HTML 中，你会将 URL 传递给 `action` 属性。此 URL 是表单数据应提交到的目标（通常是 API 端点）。
> 
> 然而，在 React 中，`action` 属性被视为一个特殊的 prop —— 这意味着 React 在其基础上进行了扩展，允许调用操作。
> 
> 在幕后，Server Actions 会创建一个 `POST` API 端点。这就是为什么在使用 Server Actions 时，你不需要手动创建 API 端点。

### 3\. 从 `formData` 中提取数据 [](#3-extract-the-data-from-formdata)

回到你的 `actions.ts` 文件，你需要提取 `formData` 的值，有[几种方法](https://developer.mozilla.org/en-US/docs/Web/API/FormData)可以做到这一点。对于这个例子，让我们使用 [`.get(name)`](https://developer.mozilla.org/en-US/docs/Web/API/FormData/get) 方法。

/app/lib/actions.ts

```
'use server';
 
export async function createInvoice(formData: FormData) {
  const rawFormData = {
    customerId: formData.get('customerId'),
    amount: formData.get('amount'),
    status: formData.get('status'),
  };
  // Test it out:
  console.log(rawFormData);
}
```

> **提示：** 如果你处理的是包含许多字段的表单，你可能需要考虑将 [`entries()`](https://developer.mozilla.org/en-US/docs/Web/API/FormData/entries) 方法与 JavaScript 的 [`Object.fromEntries()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Object/fromEntries) 结合使用。

要检查一切是否正确连接，试试这个表单。提交后，你应该在你的**终端**（而不是浏览器）中看到刚刚输入到表单中的数据记录。

现在你的数据已经是对象形状了，处理起来会容易得多。

### 4\. 验证并准备数据 [](#4-validate-and-prepare-the-data)

在将表单数据发送到数据库之前，你需要确保其格式正确且类型正确。如果你还记得本课程前面的内容，你的发票表期望的数据格式如下：

/app/lib/definitions.ts

```
export type Invoice = {
  id: string; // Will be created on the database
  customer_id: string;
  amount: number; // Stored in cents
  status: 'pending' | 'paid';
  date: string;
};
```

到目前为止，你只有来自表单的 `customer_id`、`amount` 和 `status`。

#### 类型验证和转换 [](#type-validation-and-coercion)

验证来自表单的数据是否与数据库中预期的类型一致非常重要。例如，如果你在操作内部添加一个 `console.log`：

```
console.log(typeof rawFormData.amount);
```

你会注意到 `amount` 是 `string` 类型，而不是 `number`。这是因为 `type="number"` 的 `input` 元素实际上返回的是字符串，而不是数字！

为了处理类型验证，你有几个选择。虽然你可以手动验证类型，但使用类型验证库可以节省你的时间和精力。对于你的示例，我们将使用 [Zod](https://zod.dev/)，这是一个以 TypeScript 为先的验证库，可以简化此任务。

在你的 `actions.ts` 文件中，导入 Zod 并定义一个与表单对象形状匹配的模式。此模式将在将 `formData` 保存到数据库之前对其进行验证。

/app/lib/actions.ts

```
'use server';
 
import { z } from 'zod';
 
const FormSchema = z.object({
  id: z.string(),
  customerId: z.string(),
  amount: z.coerce.number(),
  status: z.enum(['pending', 'paid']),
  date: z.string(),
});
 
const CreateInvoice = FormSchema.omit({ id: true, date: true });
 
export async function createInvoice(formData: FormData) {
  // ...
}
```

`amount` 字段被特别设置为从字符串强制转换（更改）为数字，同时验证其类型。

然后，你可以将 `rawFormData` 传递给 `CreateInvoice` 来验证类型：

/app/lib/actions.ts

```
// ...
export async function createInvoice(formData: FormData) {
  const { customerId, amount, status } = CreateInvoice.parse({
    customerId: formData.get('customerId'),
    amount: formData.get('amount'),
    status: formData.get('status'),
  });
}
```

#### 以美分为单位存储值 [](#storing-values-in-cents)

通常，在数据库中以分为单位存储货币值是一个好习惯，以消除 JavaScript 浮点错误并确保更高的准确性。

让我们将金额转换为分：

/app/lib/actions.ts

```
// ...
export async function createInvoice(formData: FormData) {
  const { customerId, amount, status } = CreateInvoice.parse({
    customerId: formData.get('customerId'),
    amount: formData.get('amount'),
    status: formData.get('status'),
  });
  const amountInCents = amount * 100;
}
```

#### 创建新日期 [](#creating-new-dates)

最后，让我们为发票的创建日期创建一个格式为“YYYY-MM-DD”的新日期：

/app/lib/actions.ts

```
// ...
export async function createInvoice(formData: FormData) {
  const { customerId, amount, status } = CreateInvoice.parse({
    customerId: formData.get('customerId'),
    amount: formData.get('amount'),
    status: formData.get('status'),
  });
  const amountInCents = amount * 100;
  const date = new Date().toISOString().split('T')[0];
}
```

### 5\. 将数据插入数据库 [](#5-inserting-the-data-into-your-database)

现在你已经有了数据库所需的所有值，你可以创建一个 SQL 查询来将新发票插入到数据库中，并传入变量：

/app/lib/actions.ts

```
import { z } from 'zod';
import postgres from 'postgres';
 
const sql = postgres(process.env.POSTGRES_URL!, { ssl: 'require' });
 
// ...
 
export async function createInvoice(formData: FormData) {
  const { customerId, amount, status } = CreateInvoice.parse({
    customerId: formData.get('customerId'),
    amount: formData.get('amount'),
    status: formData.get('status'),
  });
  const amountInCents = amount * 100;
  const date = new Date().toISOString().split('T')[0];
 
  await sql`
    INSERT INTO invoices (customer_id, amount, status, date)
    VALUES (${customerId}, ${amountInCents}, ${status}, ${date})
  `;
}
```

现在，我们还没有处理任何错误。我们将在下一章讨论这个问题。现在，让我们进入下一步。

### 6\. 重新验证并重定向 [](#6-revalidate-and-redirect)

Next.js 有一个客户端路由器缓存，会在用户的浏览器中存储路由片段一段时间。结合[预取](/docs/app/building-your-application/routing/linking-and-navigating#1-prefetching)，此缓存确保用户可以在路由之间快速导航，同时减少对服务器的请求数量。

由于你正在更新发票路由中显示的数据，你需要清除此缓存并触发对服务器的新请求。你可以使用 Next.js 中的 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 函数来做到这一点：

/app/lib/actions.ts

```
'use server';
 
import { z } from 'zod';
import { revalidatePath } from 'next/cache';
import postgres from 'postgres';
 
const sql = postgres(process.env.POSTGRES_URL!, { ssl: 'require' });
 
// ...
 
export async function createInvoice(formData: FormData) {
  const { customerId, amount, status } = CreateInvoice.parse({
    customerId: formData.get('customerId'),
    amount: formData.get('amount'),
    status: formData.get('status'),
  });
  const amountInCents = amount * 100;
  const date = new Date().toISOString().split('T')[0];
 
  await sql`
    INSERT INTO invoices (customer_id, amount, status, date)
    VALUES (${customerId}, ${amountInCents}, ${status}, ${date})
  `;
 
  revalidatePath('/dashboard/invoices');
}
```

一旦数据库更新完毕，`/dashboard/invoices` 路径将被重新验证，并将从服务器获取新数据。

此时，你还需要将用户重定向回 `/dashboard/invoices` 页面。你可以使用 Next.js 中的 [`redirect`](/docs/app/api-reference/functions/redirect) 函数来做到这一点：

/app/lib/actions.ts

```
'use server';
 
import { z } from 'zod';
import { revalidatePath } from 'next/cache';
import { redirect } from 'next/navigation';
import postgres from 'postgres';
 
const sql = postgres(process.env.POSTGRES_URL!, { ssl: 'require' });
 
// ...
 
export async function createInvoice(formData: FormData) {
  // ...
 
  revalidatePath('/dashboard/invoices');
  redirect('/dashboard/invoices');
}
```

恭喜！你刚刚实现了你的第一个 Server Action。通过添加一个新发票来测试一下，如果一切正常：

1.  提交后，你应该被重定向到 `/dashboard/invoices` 路由。
2.  你应该在表格顶部看到新发票。

更新发票 [](#updating-an-invoice)
-------------------------------------------

更新发票表单类似于创建发票表单，只是你需要传递发票 `id` 来更新数据库中的记录。让我们看看如何获取和传递发票 `id`。

以下是更新发票的步骤：

1.  创建一个包含发票 `id` 的新动态路由段。
2.  从页面参数中读取发票 `id`。
3.  从数据库中获取特定发票。
4.  用发票数据预填充表单。
5.  在数据库中更新发票数据。

### 1\. 使用发票 `id` 创建动态路由片段 [](#1-create-a-dynamic-route-segment-with-the-invoice-id)

在你的 `/invoices` 文件夹中，创建一个名为 `[id]` 的新动态路由，然后创建一个名为 `edit` 的新路由，其中包含一个 `page.tsx` 文件。你的文件结构应该如下所示：

回到你的 `<Page>` 组件，粘贴以下代码：

### 3\. 获取特定发票 [](#3-fetch-the-specific-invoice)

然后：

*   导入一个名为 `fetchInvoiceById` 的新函数，并将 `id` 作为参数传递。
*   导入 `fetchCustomers` 来获取下拉菜单的客户名称。

你可以使用 `Promise.all` 来并行获取发票和客户：

/dashboard/invoices/[id]/edit/page.tsx

```
import Form from '@/app/ui/invoices/edit-form';
import Breadcrumbs from '@/app/ui/invoices/breadcrumbs';
import { fetchInvoiceById, fetchCustomers } from '@/app/lib/data';
 
export default async function Page(props: { params: Promise<{ id: string }> }) {
  const params = await props.params;
  const id = params.id;
  const [invoice, customers] = await Promise.all([
    fetchInvoiceById(id),
    fetchCustomers(),
  ]);
  // ...
}
```

你会在终端中看到一个关于 `invoice` prop 的临时 TypeScript 错误，因为 `invoice` 可能是 `undefined`。现在先不用担心，当你在下一章添加错误处理时，你会解决它。

很好！现在，测试一切是否正确连接。访问 [http://localhost:3000/dashboard/invoices](http://localhost:3000/dashboard/invoices) 并点击铅笔图标来编辑发票。导航后，你应该看到一个预填充了发票详细信息的表单：

![带有面包屑导航和表单的编辑发票页面](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/edit-invoice-page.png)![带有面包屑导航和表单的编辑发票页面](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fedit-invoice-page.png&w=1920&q=75)

URL 也应该更新为包含一个 `id`，如下所示：`http://localhost:3000/dashboard/invoice/uuid/edit`

> **UUID 与自增主键**
> 
> 我们使用 UUID 而不是自增主键（例如 1, 2, 3 等）。这使得 URL 更长；然而，UUID 消除了 ID 冲突的风险，是全局唯一的，并降低了枚举攻击的风险——这使其成为大型数据库的理想选择。
> 
> 但是，如果你更喜欢更简洁的 URL，你可能会倾向于使用自增主键。

### 4\. 将 `id` 传递给 Server Action[](#4-pass-the-id-to-the-server-action)

最后，你需要将 `id` 传递给 Server Action，以便可以更新数据库中的正确记录。你**不能**像这样将 `id` 作为参数传递：

/app/ui/invoices/edit-form.tsx

```
// Passing an id as argument won't work
<form action={updateInvoice(id)}>
```

相反，你可以使用 JS `bind` 将 `id` 传递给 Server Action。这将确保传递给 Server Action 的任何值都被编码。

/app/ui/invoices/edit-form.tsx

```
// ...
import { updateInvoice } from '@/app/lib/actions';
 
export default function EditInvoiceForm({
  invoice,
  customers,
}: {
  invoice: InvoiceForm;
  customers: CustomerField[];
}) {
  const updateInvoiceWithId = updateInvoice.bind(null, invoice.id);
 
  return <form action={updateInvoiceWithId}>{/* ... */}</form>;
}
```

> **注意：** 在表单中使用隐藏输入字段也有效（例如 `<input type="hidden" name="id" value={invoice.id} />`）。但是，值将以纯文本形式出现在 HTML 源代码中，这对于敏感数据来说并不理想。

然后，在你的 `actions.ts` 文件中，创建一个新操作 `updateInvoice`：

/app/lib/actions.ts

```
// Use Zod to update the expected types
const UpdateInvoice = FormSchema.omit({ id: true, date: true });
 
// ...
 
export async function updateInvoice(id: string, formData: FormData) {
  const { customerId, amount, status } = UpdateInvoice.parse({
    customerId: formData.get('customerId'),
    amount: formData.get('amount'),
    status: formData.get('status'),
  });
 
  const amountInCents = amount * 100;
 
  await sql`
    UPDATE invoices
    SET customer_id = ${customerId}, amount = ${amountInCents}, status = ${status}
    WHERE id = ${id}
  `;
 
  revalidatePath('/dashboard/invoices');
  redirect('/dashboard/invoices');
}
```

与 `createInvoice` 操作类似，这里你正在：

1.  从 `formData` 提取数据。
2.  使用 Zod 验证类型。
3.  将金额转换为分。
4.  将变量传递给你的 SQL 查询。
5.  调用 `revalidatePath` 来清除客户端缓存并发出新的服务器请求。
6.  调用 `redirect` 将用户重定向到发票页面。

通过编辑发票来测试一下。提交表单后，你应该被重定向到发票页面，并且发票应该被更新。

删除发票 [](#deleting-an-invoice)
-------------------------------------------

要使用 Server Action 删除发票，请将删除按钮包装在 `<form>` 元素中，并使用 `bind` 将 `id` 传递给 Server Action：

/app/ui/invoices/buttons.tsx

```
import { deleteInvoice } from '@/app/lib/actions';
 
// ...
 
export function DeleteInvoice({ id }: { id: string }) {
  const deleteInvoiceWithId = deleteInvoice.bind(null, id);
 
  return (
    <form action={deleteInvoiceWithId}>
      <button type="submit" className="rounded-md border p-2 hover:bg-gray-100">
        <span className="sr-only">Delete</span>
        <TrashIcon className="w-4" />
      </button>
    </form>
  );
}
```

在你的 `actions.ts` 文件内部，创建一个名为 `deleteInvoice` 的新操作。

/app/lib/actions.ts

```
export async function deleteInvoice(id: string) {
  await sql`DELETE FROM invoices WHERE id = ${id}`;
  revalidatePath('/dashboard/invoices');
}
```

由于此操作在 `/dashboard/invoices` 路径中被调用，你不需要调用 `redirect`。调用 `revalidatePath` 将触发一个新的服务器请求并重新渲染表格。
