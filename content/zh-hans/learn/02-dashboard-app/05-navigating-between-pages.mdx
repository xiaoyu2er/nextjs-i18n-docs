在上一个章节中，你创建了仪表板布局和各页面。现在，让我们一起添加一些链接，以便用户可以在仪表板路由之间进行导航。

为何要优化导航？[](#why-optimize-navigation)
----------------------------------------------------

链接页面时，传统上使用 <a> HTML 元素。目前，侧边栏链接使用的是 <a> 元素，但请注意在浏览器中导航主页、账单和客户页面时会发生什么。

你看到了吗？

每次页面导航时都会出现整页刷新！

<Link> 组件[](#the-link-component)
---------------------------------------------

在 Next.js 中，你可以使用 <Link /> 组件来链接应用中的页面。<Link> 允许你进行 [客户端导航](/docs/app/building-your-application/routing/linking-and-navigating#how-routing-and-navigation-works)

要使用 <Link /> 组件，打开 /app/ui/dashboard/nav-links.tsx，并从 [next/link](/docs/app/api-reference/components/link) 导入 Link 组件。然后，将 <a> 标签替换为 Link：

/app/ui/dashboard/nav-links.tsx

```
import {
  UserGroupIcon,
  HomeIcon,
  DocumentDuplicateIcon,
} from '@heroicons/react/24/outline';
import Link from 'next/link';
 
// ...
 
export default function NavLinks() {
  return (
    <>
      {links.map((link) => {
        const LinkIcon = link.icon;
        return (
          <Link
            key={link.name}
            href={link.href}
            className="flex h-[48px] grow items-center justify-center gap-2 rounded-md bg-gray-50 p-3 text-sm font-medium hover:bg-sky-100 hover:text-blue-600 md:flex-none md:justify-start md:p-2 md:px-3"
          >
            <LinkIcon className="w-6" />
            <p className="hidden md:block">{link.name}</p>
          </Link>
        );
      })}
    </>
  );
}
```

如您所见，`Link` 组件的使用方式与 `<a>` 标签类似，但需要使用 `<Link href="…">` 替代 `<a href="…">`。

保存更改并在本地主机上检查它是否有效。现在你应该能够在页面之间导航而无需看到整页刷新。虽然应用程序的某些部分是在服务器端渲染的，但没有整页刷新，使其感觉像原生 Web 应用。为什么会这样？

### 自动代码拆分和预取[](#automatic-code-splitting-and-prefetching)

为了改善导航体验，Next.js 会根据路由段自动对你的应用进行代码拆分。这与传统的 React 单页应用 [SPA](https://nextjs.org/docs/app/building-your-application/upgrading/single-page-applications) 不同，传统的 SPA 在首次页面加载时浏览器会加载所有应用代码。

按路由拆分代码意味着页面变得独立。如果某个页面抛出错误，应用程序的其余部分仍会正常工作。这也减少了浏览器需要解析的代码量，使应用程序运行更快。

此外，在生产环境中，当 [<Link>](/docs/api-reference/next/link) 组件出现在浏览器视口中时，Next.js 会自动在后台 **预取** 链接路由的代码。当用户点击链接时，目标页面的代码已经在后台加载完毕，这就是页面切换近乎瞬间的原因！

深入了解 [导航的工作原理](/docs/app/building-your-application/routing/linking-and-navigating#how-routing-and-navigation-works)。

模式：显示活动链接[](#pattern-showing-active-links)
--------------------------------------------------------------

常见的 UI 模式是显示一个活动链接，以指示用户当前所在的页面。要做到这一点，你需要从 URL 获取用户的当前路径。Next.js 提供了一个名为 [usePathname()](/docs/app/api-reference/functions/use-pathname) 的钩子，你可以用它来检查路径并实现此模式。

由于 [usePathname()](https://nextjs.org/docs/app/api-reference/functions/use-pathname) 是 React 钩子，你需要将 nav-links.tsx 转换为客户端组件。在文件顶部添加 React 的 "use client" 指令，然后从 next/navigation 导入 usePathname()：

/app/ui/dashboard/nav-links.tsx

```
'use client';
 
import {
  UserGroupIcon,
  HomeIcon,
  DocumentDuplicateIcon,
} from '@heroicons/react/24/outline';
import Link from 'next/link';
import { usePathname } from 'next/navigation';
 
// ...
```

接下来，在你的 <NavLinks /> 组件内部，将路径分配给名为 pathname 的变量：

/app/ui/dashboard/nav-links.tsx

```
export default function NavLinks() {
  const pathname = usePathname();
  // ...
}
```

> **注意**：nav-links.tsx 对于 Next.js 来说并不是特殊文件——它可以命名为任何你喜欢的名字。如果你重命名它，请确保相应地更新导入语句。

你可以使用在 [CSS 样式](/learn/dashboard-app/css-styling) 章节中介绍的 clsx 库，在链接激活时条件地应用类名。当 link.href 与 pathname 匹配时，链接应显示为蓝色文本和浅蓝色背景。

这是 nav-links.tsx 的最终代码：

/app/ui/dashboard/nav-links.tsx

```
'use client';
 
import {
  UserGroupIcon,
  HomeIcon,
  DocumentDuplicateIcon,
} from '@heroicons/react/24/outline';
import Link from 'next/link';
import { usePathname } from 'next/navigation';
import clsx from 'clsx';
 
// ...
 
export default function NavLinks() {
  const pathname = usePathname();
 
  return (
    <>
      {links.map((link) => {
        const LinkIcon = link.icon;
        return (
          <Link
            key={link.name}
            href={link.href}
            className={clsx(
              'flex h-[48px] grow items-center justify-center gap-2 rounded-md bg-gray-50 p-3 text-sm font-medium hover:bg-sky-100 hover:text-blue-600 md:flex-none md:justify-start md:p-2 md:px-3',
              {
                'bg-sky-100 text-blue-600': pathname === link.href,
              },
            )}
          >
            <LinkIcon className="w-6" />
            <p className="hidden md:block">{link.name}</p>
          </Link>
        );
      })}
    </>
  );
}
```

保存并检查你的本地主机。你现在应该看到活动链接高亮显示为蓝色。
