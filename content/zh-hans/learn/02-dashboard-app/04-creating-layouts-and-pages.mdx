到目前为止，您的应用程序只有一个主页。让我们学习如何使用 **布局 (layouts)** 和 **页面 (pages)** 创建更多路由。

嵌套路由 (Nested routing)
---------------------------------

Next.js 使用文件系统路由，其中 **文件夹 (folders)** 用于创建嵌套路由。每个文件夹代表一个映射到 **URL 段 (URL segment)** 的 **路由段 (route segment)**。

![图表展示了文件夹如何映射到 URL 段](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/folders-to-url-segments.png)![图表展示了文件夹如何映射到 URL 段](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Ffolders-to-url-segments.png&w=3840&q=75)

您可以使用 `layout.tsx` 和 `page.tsx` 文件为每个路由创建独立的 UI。

`page.tsx` 是一个特殊的 Next.js 文件，它导出一个 React 组件，路由需要它才能可访问。在您的应用程序中，您已经有一个页面文件：`/app/page.tsx` - 这是与路由 `/` 关联的主页。

要创建嵌套路由，您可以将文件夹互相嵌套，并在其中添加 `page.tsx` 文件。例如：

![图表展示了添加名为 dashboard 的文件夹如何创建新路由 '/dashboard'](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/dashboard-route.png)![图表展示了添加名为 dashboard 的文件夹如何创建新路由 '/dashboard'](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fdashboard-route.png&w=3840&q=75)

`/app/dashboard/page.tsx` 与 `/dashboard` 路径关联。让我们创建页面来看看它是如何工作的！

创建仪表板页面 (Creating the dashboard page)
-----------------------------------------------------------

在 `/app` 内创建一个名为 `dashboard` 的新文件夹。然后，在 `dashboard` 文件夹内创建一个新的 `page.tsx` 文件，内容如下：

/app/dashboard/page.tsx

```
export default function Page() {
  return <p>Dashboard Page</p>;
}
```

现在，请确保开发服务器正在运行，并访问 [http://localhost:3000/dashboard](http://localhost:3000/dashboard)。您应该会看到“仪表板页面 (Dashboard Page)”文本。

这就是您在 Next.js 中创建不同页面的方法：使用文件夹创建新的路由段，并在其中添加 `page` 文件。

通过为 `page` 文件指定特殊名称，Next.js 允许您将 UI 组件、测试文件和其他相关代码 [共存 (colocate)](/docs/app/getting-started/project-structure#colocation) 到您的路由中。只有 `page` 文件内的内容才会公开可访问。例如，`/ui` 和 `/lib` 文件夹与您的路由一起 _共存 (colocated)_ 在 `/app` 文件夹内。

创建仪表板布局 (Creating the dashboard layout)
---------------------------------------------------------------

仪表板具有某种跨多个页面共享的导航。在 Next.js 中，您可以使用特殊的 `layout.tsx` 文件创建跨多个页面共享的 UI。让我们为仪表板页面创建一个布局！

在 `/dashboard` 文件夹内，添加一个新文件，名为 `layout.tsx`，然后粘贴以下代码：

/app/dashboard/layout.tsx

```
import SideNav from '@/app/ui/dashboard/sidenav';
 
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div className="flex h-screen flex-col md:flex-row md:overflow-hidden">
      <div className="w-full flex-none md:w-64">
        <SideNav />
      </div>
      <div className="grow p-6 md:overflow-y-auto md:p-12">{children}</div>
    </div>
  );
}
```

这段代码中有一些内容，让我们分解一下：

首先，您正在将 `<SideNav />` 组件导入到布局中。导入到此文件中的任何组件都将成为布局的一部分。

`<Layout />` 组件接收一个 `children` prop。这个子组件可以是页面或另一个布局。在您的情况下，`/dashboard` 内的页面将自动嵌套在 `<Layout />` 内，如下所示：

![带有仪表板布局嵌套仪表板页面作为子项的文件夹结构](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/shared-layout.png)![带有仪表板布局嵌套仪表板页面作为子项的文件夹结构](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fshared-layout.png&w=3840&q=75)

通过保存您的更改并检查您的本地主机来确保所有内容正常工作。您应该会看到以下内容：

![带有侧边导航和主内容区域的仪表板页面](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/shared-layout-page.png)![带有侧边导航和主内容区域的仪表板页面](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fshared-layout-page.png&w=1920&q=75)

在 Next.js 中使用布局的一个好处是，在导航时，只有页面组件会更新，而布局不会重新渲染。这被称为 [部分渲染 (partial rendering)](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering)，它在页面之间切换时保留布局中的客户端 React 状态。

根布局 (Root layout)
---------------------------

在第 3 章中，您将 `Inter` 字体导入到了另一个布局中：`/app/layout.tsx`。作为提醒：

/app/layout.tsx

```
import '@/app/ui/global.css';
import { inter } from '@/app/ui/fonts';
 
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body className={`${inter.className} antialiased`}>{children}</body>
    </html>
  );
}
```

这被称为 [根布局](/docs/app/api-reference/file-conventions/layout#root-layouts)，在每个 Next.js 应用程序中都是必需的。您添加到根布局的任何 UI 都将在应用程序中的**所有**页面之间共享。您可以使用根布局来修改您的 `<html>` 和 `<body>` 标签，并添加元数据（您将在[后面的章节](/learn/dashboard-app/adding-metadata)中了解更多关于元数据的知识）。

由于您刚刚创建的新布局 (`/app/dashboard/layout.tsx`) 是仪表板页面独有的，因此您不需要在上述根布局中添加任何 UI。
