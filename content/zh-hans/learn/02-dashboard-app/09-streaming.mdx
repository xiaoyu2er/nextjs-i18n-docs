在上一章中，您了解了 Next.js 的不同渲染方法。我们还讨论了慢速数据获取如何影响您应用程序的性能。让我们看看当有慢速数据请求时，您可以如何提高用户体验。

什么是流式传输？[](#what-is-streaming)
----------------------------------------

流式传输是一种数据传输技术，允许您将路由拆分为更小的“块”，并随着它们的就绪程度，从服务器逐步流式传输到客户端。

![显示顺序数据获取和并行数据获取时间的图表](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/server-rendering-with-streaming.png)![显示顺序数据获取和并行数据获取时间的图表](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fserver-rendering-with-streaming.png&w=3840&q=75)

通过流式传输，您可以防止慢速数据请求阻塞整个页面。这样，用户无需等待所有数据加载完成，就能看到并与页面的部分进行交互。

![显示顺序数据获取和并行数据获取时间的图表](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/server-rendering-with-streaming-chart.png)![显示顺序数据获取和并行数据获取时间的图表](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fserver-rendering-with-streaming-chart.png&w=3840&q=75)

流式传输与 React 的组件模型配合良好，因为每个组件都可以被视为一个 _块_。

在 Next.js 中实现流式传输有两种方法：

1.  在页面级别，使用 `loading.tsx` 文件（它会为您创建 `<Suspense>`）。
2.  在组件级别，使用 `<Suspense>` 进行更细粒度的控制。

让我们看看这是如何工作的。

使用 `loading.tsx` 流式传输整个页面[](#streaming-a-whole-page-with-loadingtsx)
------------------------------------------------------------------------------------

在 `/app/dashboard` 文件夹中，创建一个名为 `loading.tsx` 的新文件：

/app/dashboard/loading.tsx

```
export default function Loading() {
  return <div>Loading...</div>;
}
```

刷新 [http://localhost:3000/dashboard](http://localhost:3000/dashboard)，您现在应该看到的是：

![带有“Loading...”文本的仪表板页面](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/loading-page.png)![带有“Loading...”文本的仪表板页面](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Floading-page.png&w=1920&q=75)

这里有几点说明：

1.  `loading.tsx` 是一个基于 React Suspense 构建的特殊 Next.js 文件。它允许您创建后备 UI，以便在页面内容加载时作为替代品显示。
2.  由于 `<SideNav>` 是静态的，它会立即显示。用户可以在加载动态内容时与 `<SideNav>` 交互。
3.  用户不必等到页面加载完成才能离开（这称为可中断导航）。

恭喜！您刚才实现了流式传输。但我们可以做更多来改善用户体验。让我们显示加载骨架屏而不是 `Loading…` 文本。

### 添加加载骨架屏[](#adding-loading-skeletons)

加载骨架屏是 UI 的简化版本。许多网站将它们用作占位符（或后备内容），以向用户表明内容正在加载。您在 `loading.tsx` 中添加的任何 UI 都将作为静态文件的一部分嵌入，并优先发送。然后，其余的动态内容将从服务器流式传输到客户端。

在您的 `loading.tsx` 文件中，导入一个新组件 `<DashboardSkeleton>`：

/app/dashboard/loading.tsx

```
import DashboardSkeleton from '@/app/ui/skeletons';
 
export default function Loading() {
  return <DashboardSkeleton />;
}
```

然后，刷新 [http://localhost:3000/dashboard](http://localhost:3000/dashboard)，您现在应该看到的是：

![带有加载骨架屏的仪表板页面](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/loading-page-with-skeleton.png)![带有加载骨架屏的仪表板页面](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Floading-page-with-skeleton.png&w=1920&q=75)

### 使用路由组修复加载骨架屏问题[](#fixing-the-loading-skeleton-bug-with-route-groups)

目前，您的加载骨架屏将应用于发票页面。

由于 `loading.tsx` 在文件系统中比 `/invoices/page.tsx` 和 `/customers/page.tsx` 高一级，因此它也应用于这些页面。

我们可以使用 [路由组](https://nextjs.org/docs/app/building-your-application/routing/route-groups) 来更改此内容。在 dashboard 文件夹内创建一个名为 `/(overview)` 的新文件夹。然后，将您的 `loading.tsx` 和 `page.tsx` 文件移入该文件夹：

![显示如何使用括号创建路由组的文件夹结构](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/route-group.png)![显示如何使用括号创建路由组的文件夹结构](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Froute-group.png&w=3840&q=75)

现在，`loading.tsx` 文件将仅适用于您的仪表板概览页面。

路由组允许您将文件组织成逻辑组，而不影响 URL 路径结构。当您使用括号 `()` 创建新文件夹时，名称将不包含在 URL 路径中。因此 `/dashboard/(overview)/page.tsx` 变为 `/dashboard`。

在此，您正在使用路由组来确保 `loading.tsx` 仅适用于您的仪表板概览页面。但是，您也可以将路由组用于将应用程序分离为部分（例如 `(marketing)` 路由和 `(shop)` 路由）或按团队分离大型应用。

### 流式传输组件[](#streaming-a-component)

到目前为止，您正在流式传输整个页面。但您也可以更细粒度地使用 React Suspense 流式传输特定组件。

Suspense 允许您推迟渲染应用程序的某些部分，直到满足某些条件（例如数据加载）。您可以在 Suspense 中包装您的动态组件。然后，传递一个后备组件，在动态组件加载时显示。

如果您还记得那个慢速数据请求 `fetchRevenue()`，这就是放慢整个页面的请求。您可以使用 Suspense 来只流式传输该组件，并立即显示页面的其余 UI，而不是阻塞整个页面。

为此，您需要将数据获取移动到组件中，让我们更新代码以查看其外观：

从 `/dashboard/(overview)/page.tsx` 中删除所有 `fetchRevenue()` 实例及其数据：

/app/dashboard/(overview)/page.tsx

```
import { Card } from '@/app/ui/dashboard/cards';
import RevenueChart from '@/app/ui/dashboard/revenue-chart';
import LatestInvoices from '@/app/ui/dashboard/latest-invoices';

  return (
    // ...

```

然后，从 React 导入 `<Suspense>`，并将其包裹在 `<RevenueChart />` 周围。您可以传递一个名为 `<RevenueChartSkeleton>` 的后备组件。

/app/dashboard/(overview)/page.tsx

```
import { Card } from '@/app/ui/dashboard/cards';
import RevenueChart from '@/app/ui/dashboard/revenue-chart';
import LatestInvoices from '@/app/ui/dashboard/latest-invoices';
import { lusitana } from '@/app/ui/fonts';
import { fetchLatestInvoices, fetchCardData } from '@/app/lib/data';
import { Suspense } from 'react';
import { RevenueChartSkeleton } from '@/app/ui/skeletons';
 
export default async function Page() {
  const latestInvoices = await fetchLatestInvoices();
  const {
    numberOfInvoices,
    numberOfCustomers,
    totalPaidInvoices,
    totalPendingInvoices,
  } = await fetchCardData();
 
  return (
    <main>
      <h1 className={`${lusitana.className} mb-4 text-xl md:text-2xl`}>
        Dashboard
      </h1>
      <div className="grid gap-6 sm:grid-cols-2 lg:grid-cols-4">
        <Card title="Collected" value={totalPaidInvoices} type="collected" />
        <Card title="Pending" value={totalPendingInvoices} type="pending" />
        <Card title="Total Invoices" value={numberOfInvoices} type="invoices" />
        <Card
          title="Total Customers"
          value={numberOfCustomers}
          type="customers"
        />
      </div>
      <div className="mt-6 grid grid-cols-1 gap-6 md:grid-cols-4 lg:grid-cols-8">
        <Suspense fallback={<RevenueChartSkeleton />}>
          <RevenueChart />
        </Suspense>
        <LatestInvoices latestInvoices={latestInvoices} />
      </div>
    </main>
  );
}
```

组件分组[](#grouping-components)
-------------------------------------------

太好了！你快成功了，现在你需要将 `<Card>` 组件包裹在 Suspense 中。你可以获取每个单独卡片的数据，但这可能会导致卡片加载时出现 _弹出_ 效果，这对用户来说可能在视觉上很突兀。

那么，你会如何解决这个问题？

为了创建更明显的 _交错_ 效果，您可以使用包装组件对卡片进行分组。这意味着静态 `<SideNav/>` 将首先显示，然后是卡片，依此类推。

在您的 `page.tsx` 文件中：

1.  删除您的 `<Card>` 组件。
2.  删除 `fetchCardData()` 函数。
3.  导入名为 `<CardWrapper />` 的新 **包装** 组件。
4.  导入名为 `<CardsSkeleton />` 的新 **骨架** 组件。
5.  在 Suspense 中包裹 `<CardWrapper />`。

/app/dashboard/(overview)/page.tsx

```
import CardWrapper from '@/app/ui/dashboard/cards';
// ...
import {
  RevenueChartSkeleton,
  LatestInvoicesSkeleton,
  CardsSkeleton,
} from '@/app/ui/skeletons';
 
export default async function Page() {
  return (
    <main>
      <h1 className={`${lusitana.className} mb-4 text-xl md:text-2xl`}>
        Dashboard
      </h1>
      <div className="grid gap-6 sm:grid-cols-2 lg:grid-cols-4">
        <Suspense fallback={<CardsSkeleton />}>
          <CardWrapper />
        </Suspense>
      </div>
      // ...
    </main>
  );
}
```

然后，进入文件 `/app/ui/dashboard/cards.tsx`，导入 `fetchCardData()` 函数，并在 `<CardWrapper/>` 组件中调用它。请确保取消注释该组件中的任何必要代码。

/app/ui/dashboard/cards.tsx

```
// ...
import { fetchCardData } from '@/app/lib/data';
 
// ...
 
export default async function CardWrapper() {
  const {
    numberOfInvoices,
    numberOfCustomers,
    totalPaidInvoices,
    totalPendingInvoices,
  } = await fetchCardData();
 
  return (
    <>
      <Card title="Collected" value={totalPaidInvoices} type="collected" />
      <Card title="Pending" value={totalPendingInvoices} type="pending" />
      <Card title="Total Invoices" value={numberOfInvoices} type="invoices" />
      <Card
        title="Total Customers"
        value={numberOfCustomers}
        type="customers"
      />
    </>
  );
}
```

刷新页面，您应该看到所有卡片同时加载。当您希望多个组件同时加载时，可以使用此模式。

决定放置 Suspense 边界的位置[](#deciding-where-to-place-your-suspense-boundaries)
-----------------------------------------------------------------------------------------------------

您放置 Suspense 边界的位置取决于以下几点：

1.  您希望用户如何在流式传输过程中体验页面。
2.  您想要优先考虑的内容。
3.  组件是否依赖数据获取。

看看您的仪表板页面，有什么你会做得不一样的地方吗？

别担心。没有标准答案。

*   您可以流式传输**整个页面**，就像我们使用 `loading.tsx` 所做的那样……但如果其中一个组件的数据获取缓慢，这可能会导致更长的加载时间。
*   您可以单独流式传输**每个组件**……但这可能会导致 UI _弹出_ 到屏幕上。
*   您也可以通过流式传输**页面部分**来创建 _交错_ 效果。但您需要创建包装组件。

您放置 Suspense 边界的位置会根据您的应用而有所不同。通常，将数据获取移动到需要它的组件中，然后将这些组件包装在 Suspense 中是一个良好的做法。但是如果您的应用需要流式传输部分或整个页面，这样做也没有问题。

不要害怕尝试 Suspense，看看什么最有效，它是一个强大的 API，可以帮助您创建更愉快的用户体验。
