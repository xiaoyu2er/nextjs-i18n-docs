在继续处理仪表板之前，你需要一些数据。在本章中，你将通过 [Vercel 的应用市场集成](https://vercel.com/marketplace?category=storage) 之一设置 PostgreSQL 数据库。如果你已经熟悉 PostgreSQL 并且希望使用自己的数据库提供商，你可以跳过本章自行设置。否则，让我们继续吧！

创建 GitHub 仓库 [](#create-a-github-repository)
---------------------------------------------------------

首先，如果你还没有这样做，请将你的仓库推送到 GitHub。这将使你更容易设置数据库并进行部署。

如果你需要帮助设置仓库，请查看 [GitHub 上的这份指南](https://help.github.com/en/github/getting-started-with-github/create-a-repo)。

> **小贴士**:
> 
> *   你也可以使用其他 Git 提供商，例如 GitLab 或 Bitbucket。
> *   如果你是 GitHub 新手，我们推荐使用 [GitHub Desktop 应用](https://desktop.github.com/) 以简化开发工作流程。

创建 Vercel 账户 [](#create-a-vercel-account)
---------------------------------------------------

访问 [vercel.com/signup](https://vercel.com/signup) 来创建账户。选择免费的 "hobby" 计划。选择 **继续通过 GitHub** 以连接你的 GitHub 和 Vercel 账户。

创建 Postgres 数据库 [](#create-a-postgres-database)
---------------------------------------------------------

接下来，要设置数据库，请点击 **继续前往仪表板**，并从你的项目仪表板中选择 **存储** 选项卡。选择 **创建数据库**。根据你的 Vercel 账户创建时间，你可能会看到如 Neon 或 Supabase 等选项。选择你喜欢的提供商并点击 **继续**。

![连接存储屏幕，显示 Postgres 选项以及 KV、Blob 和 Edge Config](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/create-database.png)![连接存储屏幕，显示 Postgres 选项以及 KV、Blob 和 Edge Config](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fcreate-database.png&w=1920&q=75)

选择你的区域和存储计划（如果需要）。所有 Vercel 项目的 [默认区域](https://vercel.com/docs/functions/configuring-functions/region) 是 **华盛顿特区 (iad1)**，如果可用，我们建议选择该区域以降低数据请求的 [延迟](https://developer.mozilla.org/en-US/docs/Web/Performance/Understanding_latency)。

![显示数据库名称和区域的数据库创建模态框](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/database-region.png)![显示数据库名称和区域的数据库创建模态框](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fdatabase-region.png&w=1920&q=75)

连接后，导航到 `.env.local` 选项卡，点击 **显示密钥** 并 **复制片段**。在复制之前，请确保你已显示密钥。

![.env.local 选项卡，显示隐藏的数据库密钥](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/database-dashboard.png)![.env.local 选项卡，显示隐藏的数据库密钥](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fdatabase-dashboard.png&w=1920&q=75)

导航到你的代码编辑器并将 `.env.example` 文件重命名为 **`.env`**。粘贴从 Vercel 复制的内容。

> **重要：** 转到你的 `.gitignore` 文件，确保 `.env` 在忽略的文件中，以防止在推送到 GitHub 时泄露你的数据库密钥。

为数据库填充数据 [](#seed-your-database)
-----------------------------------------

既然你的数据库已创建，让我们用一些初始数据填充它。

我们包含了一个你可以在浏览器中访问的 API，它将运行种子脚本，用一组初始数据填充数据库。

该脚本使用 **SQL** 创建表，并使用 `placeholder-data.ts` 文件中的数据在创建后填充它们。

确保你的本地开发服务器正在通过 `pnpm run dev` 运行，并在浏览器中导航到 [`localhost:3000/seed`](http://localhost:3000/seed)。完成后，你将在浏览器中看到一条消息“数据库成功填充”。完成后，你可以删除此文件。

> **故障排除**:
> 
> *   在将其复制到 `.env` 文件之前，请确保显示你的数据库密钥。
> *   该脚本使用 `bcrypt` 对用户密码进行哈希处理，如果 `bcrypt` 与你的环境不兼容，你可以更新脚本以使用 [`bcryptjs`](https://www.npmjs.com/package/bcryptjs) 替代。
> *   如果在为数据库填充数据时遇到任何问题并希望重新运行脚本，你可以在数据库查询界面中运行 `DROP TABLE tablename` 来删除任何现有表。有关更多详细信息，请参阅下面的 [执行查询部分](#executing-queries)。但请注意，此命令将删除表及其所有数据。在示例应用中这样做是可以的，因为你使用的是占位数据，但在生产应用中你不应该运行此命令。

执行查询 [](#executing-queries)
---------------------------------------

让我们执行一个查询以确保一切按预期运行。我们将使用另一个路由处理器 `app/query/route.ts` 来查询数据库。在这个文件中，你会找到一个 `listInvoices()` 函数，它包含以下 SQL 查询。

```
SELECT invoices.amount, customers.name
FROM invoices
JOIN customers ON invoices.customer_id = customers.id
WHERE invoices.amount = 666;
```

取消注释该文件，删除 `Response.json()` 代码块，并在浏览器中导航到 [`localhost:3000/query`](http://localhost:3000/query)。你应该会看到返回了一个发票的 `amount` 和 `name`。
