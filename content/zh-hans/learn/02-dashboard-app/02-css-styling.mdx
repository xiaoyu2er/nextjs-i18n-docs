目前，您的主页还没有任何样式。让我们看看可以为您的 Next.js 应用添加样式的不同方法。

全局样式 [](#global-styles)
-------------------------------

如果您查看 `/app/ui` 文件夹，您将看到一个名为 `global.css` 的文件。您可以使用此文件为应用程序中的 **所有** 路由添加 CSS 规则 - 例如 CSS 重置规则、HTML 元素（如链接）的全站样式，等等。

您可以在应用程序的任何组件中导入 `global.css`，但通常的惯例是将其添加到顶层组件中。在 Next.js 中，这就是 [根布局](/docs/app/api-reference/file-conventions/layout#root-layouts)（稍后详细说明）。

通过导航到 `/app/layout.tsx` 并导入 `global.css` 文件来为应用程序添加全局样式：

/app/layout.tsx

```
import '@/app/ui/global.css';
 
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

开发服务器仍在运行时，保存更改并在浏览器中预览。您的主页现在应该看起来像这样：

![带有“Acme”标志、说明和登录链接的已样式化页面。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/home-page-with-tailwind.png)![带有“Acme”标志、说明和登录链接的已样式化页面。](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fhome-page-with-tailwind.png&w=1920&q=75)

但等一下，您并没有添加任何 CSS 规则，样式是从哪里来的？

如果您查看 `global.css` 内部，您会注意到一些 `@tailwind` 指令：

/app/ui/global.css

```
@tailwind base;
@tailwind components;
@tailwind utilities;
```

Tailwind [](#tailwind)
---------------------

[Tailwind](https://tailwindcss.com/) 是一个 CSS 框架，它可以让您直接在 React 代码中快速编写 [实用程序类](https://tailwindcss.com/docs/utility-first)，从而加快开发流程。

在 Tailwind 中，您通过添加类名来样式化元素。例如，添加 ``"text-blue-500"`` 将使 `<h1>` 文本变为蓝色：

```
<h1 className="text-blue-500">I'm blue!</h1>
```

虽然 CSS 样式是共享的，但每个类单独应用于每个元素。这意味着如果您添加或删除元素，您无需担心维护单独的样式表、样式冲突，或者应用程序扩展时 CSS 捆绑包的大小增长。

当您使用 `create-next-app` 启动新项目时，Next.js 会询问您是否要使用 Tailwind。如果您选择 `是`，Next.js 将自动安装必要的软件包并在您的应用程序中配置 Tailwind。

如果您查看 `/app/page.tsx`，您将看到我们在示例中使用了 Tailwind 类。

/app/page.tsx

```
import AcmeLogo from '@/app/ui/acme-logo';
import { ArrowRightIcon } from '@heroicons/react/24/outline';
import Link from 'next/link';
 
export default function Page() {
  return (
    // These are Tailwind classes:
    <main className="flex min-h-screen flex-col p-6">
      <div className="flex h-20 shrink-0 items-end rounded-lg bg-blue-500 p-4 md:h-52">
    // ...
  )
}
```

如果这是您第一次使用 Tailwind，请不要担心。为了节省时间，我们已经为您使用的所有组件添加了样式。

让我们试试 Tailwind！复制下面的代码并将其粘贴到 `/app/page.tsx` 中的 `<p>` 元素上方：

/app/page.tsx

```
<div
  className="relative w-0 h-0 border-l-[15px] border-r-[15px] border-b-[26px] border-l-transparent border-r-transparent border-b-black"
/>

```

如果您更喜欢编写传统 CSS 规则或将样式与 JSX 分开 - CSS 模块是一个很好的替代方案。

CSS 模块 [](#css-modules)
---------------------------

[CSS 模块](/docs/app/getting-started/css#css-modules) 允许你通过自动生成唯一的类名将 CSS 的范围限定到组件，这样你也不需要担心样式冲突。

我们将在本课程中继续使用 Tailwind，但让我们花点时间看看如何使用 CSS 模块实现上述测验中的相同结果。

在 `/app/ui` 内部，创建一个名为 `home.module.css` 的新文件，并添加以下 CSS 规则：

/app/ui/home.module.css

```
.shape {
  height: 0;
  width: 0;
  border-bottom: 30px solid black;
  border-left: 20px solid transparent;
  border-right: 20px solid transparent;
}
```

然后，在您的 `/app/page.tsx` 文件中导入样式，并将您添加的 `<div>` 中的 Tailwind 类名替换为 `styles.shape`：

/app/page.tsx

```
import AcmeLogo from '@/app/ui/acme-logo';
import { ArrowRightIcon } from '@heroicons/react/24/outline';
import Link from 'next/link';
import styles from '@/app/ui/home.module.css';
 
export default function Page() {
  return (
    <main className="flex min-h-screen flex-col p-6">
      <div className={styles.shape} />
    // ...
  )
}
```

保存您的更改并在浏览器中预览。您应该看到与之前相同的形状。

Tailwind 和 CSS 模块是样式化 Next.js 应用程序的两种最常见方法。使用哪一个完全是个人偏好问题 - 您甚至可以在同一个应用程序中同时使用两者！

使用 `clsx` 库切换类名 [](#using-the-clsx-library-to-toggle-class-names)
-----------------------------------------------------------------------------------------------

在某些情况下，您可能需要根据状态或其他条件有条件地样式化一个元素。

[clsx](https://www.npmjs.com/package/clsx) 是一个库，可以让您轻松切换类名。我们建议您查看 [文档](https://github.com/lukeed/clsx) 以获取更多详细信息，但基本用法如下：

*   假设您想创建一个接受 `status` 的 `InvoiceStatus` 组件。状态可以是 `'pending'` 或 `'paid'`。
*   如果它是 `'paid'`，您希望颜色为绿色。如果它是 `'pending'`，您希望颜色为灰色。

您可以使用 `clsx` 有条件地应用类，如下所示：

/app/ui/invoices/status.tsx

```
import clsx from 'clsx';
 
export default function InvoiceStatus({ status }: { status: string }) {
  return (
    <span
      className={clsx(
        'inline-flex items-center rounded-full px-2 py-1 text-sm',
        {
          'bg-gray-100 text-gray-500': status === 'pending',
          'bg-green-500 text-white': status === 'paid',
        },
      )}
    >
    // ...
)}
```

其他样式解决方案 [](#other-styling-solutions)
---------------------------------------------------

除了我们讨论的方法之外，您还可以使用以下方法样式化您的 Next.js 应用程序：

*   Sass，允许您导入 `.css` 和 `.scss` 文件。
*   CSS-in-JS 库，例如 [styled-jsx](https://github.com/vercel/styled-jsx), [styled-components](https://github.com/vercel/next.js/tree/canary/examples/with-styled-components), 和 [emotion](https://github.com/vercel/next.js/tree/canary/examples/with-emotion)。

查看 [CSS 文档](/docs/app/building-your-application/styling) 以获取更多信息。
