在上一章中，您为目标仪表板概览页面获取了数据。然而，我们简要讨论了当前设置的两个限制：

1.  数据请求正在创建非预期的瀑布流。
2.  仪表板是静态的，因此任何数据更新都不会反映到您的应用程序中。

### 什么是静态渲染？[](#what-is-static-rendering)

在静态渲染中，数据获取和渲染发生在构建时（部署时）或 [重新验证数据](/docs/app/building-your-application/data-fetching/fetching-caching-and-revalidating#revalidating-data) 时。

每当用户访问您的应用程序时，都会提供缓存的结果。静态渲染有几个好处：

*   **更快的网站** - 预渲染的內容可以在部署到 [Vercel](https://vercel.com/) 等平台时进行缓存和全球分发。这确保世界各地的用户可以更快、更可靠地访问您的网站内容。
*   **降低服务器负载** - 因为内容已缓存，您的服务器不必为每个用户请求动态生成內容。这可以降低计算成本。
*   **SEO** - 预渲染的内容更容易被搜索引擎爬虫索引，因为页面加载时内容已经可用。这可以提高搜索引擎排名。

静态渲染适用于**没有数据**或**跨用户共享数据**的 UI，例如静态博客文章或产品页面。它可能不适合具有定期更新的个性化数据的仪表板。

静态渲染的对立面是动态渲染。

什么是动态渲染？[](#what-is-dynamic-rendering)
--------------------------------------------------------

在动态渲染中，内容在每个用户的**请求时**（当用户访问页面时）在服务端渲染。动态渲染有几个好处：

*   **实时数据** - 动态渲染允许您的应用程序显示实时或频繁更新的数据。这对于数据经常变化的应用程序非常理想。
*   **用户特定内容** - 提供个性化内容（如仪表板或用户资料）更容易，并可根据用户交互更新数据。
*   **请求时信息** - 动态渲染允许您获取仅在请求时才能获取的信息，例如 cookies 或 URL 搜索参数。

模拟慢速数据获取[](#simulating-a-slow-data-fetch)
-------------------------------------------------------------

我们正在构建的仪表板应用程序是动态的。

然而，上一章中还提到了一个问题。如果一个数据请求比其他请求慢怎么办？

让我们模拟一个慢速数据获取。在 `app/lib/data.ts` 中，取消注释 `fetchRevenue()` 内部的 `console.log` 和 `setTimeout`：

/app/lib/data.ts

```
export async function fetchRevenue() {
  try {
    // We artificially delay a response for demo purposes.
    // Don't do this in production :)
    console.log('Fetching revenue data...');
    await new Promise((resolve) => setTimeout(resolve, 3000));
 
    const data = await sql<Revenue[]>`SELECT * FROM revenue`;
 
    console.log('Data fetch completed after 3 seconds.');
 
    return data;
  } catch (error) {
    console.error('Database Error:', error);
    throw new Error('Failed to fetch revenue data.');
  }
}
```

现在在标签页中打开 [http://localhost:3000/dashboard/](http://localhost:3000/dashboard/)，并注意页面加载需要更长时间。在终端中，您还应该看到以下消息：

```
Fetching revenue data...
Data fetch completed after 3 seconds.
```

在这里，您添加了 3 秒的人工延迟以模拟慢速数据获取。结果就是，现在整个页面在数据获取期间被阻塞，无法向访客显示 UI。这就引出了开发人员必须解决的一个常见挑战：

在动态渲染中，**您的应用程序速度仅取决于最慢的数据获取速度。**
