为什么优化字体？[](#why-optimize-fonts)
------------------------------------------

字体在网站设计中起着重要作用，但在项目中自定义字体可能会影响性能，如果需要获取和加载字体文件的话。

[累积布局偏移 (Cumulative Layout Shift)](https://vercel.com/blog/how-core-web-vitals-affect-seo) 是 Google 用来评估网站性能和用户体验的指标。对于字体来说，布局偏移发生在浏览器最初使用回退或系统字体渲染文本，然后在加载后将其替换为自定义字体时。这种替换可能导致文本大小、间距或布局发生变化，导致周围元素移动。

![显示页面初始加载的模拟 UI，随后自定义字体加载时出现布局偏移。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/font-layout-shift.png)![显示页面初始加载的模拟 UI，随后自定义字体加载时出现布局偏移。](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Ffont-layout-shift.png&w=3840&q=75)

当你使用 `next/font` 模块时，Next.js 会自动优化应用中的字体。它在构建时下载字体文件，并将它们作为其他静态资源之一进行托管。这意味着当用户访问你的应用时，不会增加额外的网络请求，从而不会影响性能。

添加主字体 [](#adding-a-primary-font)
-----------------------------------------------

让我们为你的应用添加一个自定义 Google 字体，看看这是如何工作的。

在 `/app/ui` 文件夹中，创建一个名为 `fonts.ts` 的新文件。你将使用此文件来维护将在整个应用的各个部分中使用的字体。

从 `next/font/google` 模块导入 `Inter` 字体 - 这将是你的主字体。然后，指定你想要加载的 [子集](https://fonts.google.com/knowledge/glossary/subsetting)。在这种情况下，是 `'latin'`：

/app/ui/fonts.ts

```
import { Inter } from 'next/font/google';
 
export const inter = Inter({ subsets: ['latin'] });
```

最后，在 `/app/layout.tsx` 中的 `<body>` 元素上添加该字体：

/app/layout.tsx

```
import '@/app/ui/global.css';
import { inter } from '@/app/ui/fonts';
 
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body className={`${inter.className} antialiased`}>{children}</body>
    </html>
  );
}
```

通过将 `Inter` 添加到 `<body>` 元素，字体将应用于整个应用。在这里，你还添加了 Tailwind [`antialiased`](https://tailwindcss.com/docs/font-smoothing) 类来平滑字体。这并不是必须使用的类，但它能提升视觉效果。

打开你的浏览器，开发者工具 (Dev Tools) 并选择 `<body>` 元素。你应该在样式中看到 `Inter` 和 `Inter_Fallback` 已应用。

为什么要优化图片？[](#why-optimize-images)
--------------------------------------------

Next.js 可以在顶级 [`/public`](/docs/app/building-your-application/optimizing/static-assets) 文件夹下提供 **静态资源**，如图片。`/public` 文件夹内的文件可以在应用中被引用。

使用标准 HTML 时，你会这样添加图片：

```
<img
  src="/hero.png"
  alt="Screenshots of the dashboard project showing desktop version"
/>
```

然而，这意味着你必须手动执行以下操作：

* 确保图片在不同的屏幕尺寸上具有响应性。
* 为不同的设备指定图片尺寸。
* 防止图片加载时的布局偏移。
* 对位于用户视口之外的图片进行延迟加载。

图片优化是 Web 开发中的一个大话题，甚至可以被视为一个专业领域。与其手动实现这些优化，你可以使用 `next/image` 组件来自动优化你的图片。

<Image> 组件 [](#the-image-component)
-----------------------------------------------

`<Image>` 组件是 HTML `<img>` 标签的扩展，并具有自动图片优化功能，例如：

* 在图片加载时自动防止布局偏移。
* 调整图片尺寸，避免向视口较小的设备发送大图片。
* 默认情况下延迟加载图片（图片进入视口时才加载）。
* 当浏览器支持时，提供现代格式的图片，如 [WebP](https://developer.mozilla.org/en-US/docs/Web/Media/Formats/Image_types#webp) 和 [AVIF](https://developer.mozilla.org/en-US/docs/Web/Media/Formats/Image_types#avif_image)。

添加桌面横幅图片 [](#adding-the-desktop-hero-image)
---------------------------------------------------------------

让我们使用 `<Image>` 组件。如果你查看 `/public` 文件夹内部，你会看到有两个图片：`hero-desktop.png` 和 `hero-mobile.png`。这两张图片完全不同，它们取决于用户的设备是桌面还是移动而显示。

在你的 `/app/page.tsx` 文件中，从 [`next/image`](https://nextjs.org/docs/api-reference/next/image) 导入组件。然后，在注释下方添加图片：

/app/page.tsx

```
import AcmeLogo from '@/app/ui/acme-logo';
import { ArrowRightIcon } from '@heroicons/react/24/outline';
import Link from 'next/link';
import { lusitana } from '@/app/ui/fonts';
import Image from 'next/image';
 
export default function Page() {
  return (
    // ...
    <div className="flex items-center justify-center p-6 md:w-3/5 md:px-28 md:py-12">
      {/* Add Hero Images Here */}
      <Image
        src="/hero-desktop.png"
        width={1000}
        height={760}
        className="hidden md:block"
        alt="Screenshots of the dashboard project showing desktop version"
      />
    </div>
    //...
  );
}
```

在这里，你将 `width` 设置为 `1000`，`height` 设置为 `760` 像素。最好将图片的 `width` 和 `height` 设置为防止布局偏移，它们应该是与源图片 **完全相同** 的宽高比。这些值不是图片渲染的尺寸，而是实际使用的图片文件的尺寸，以便了解宽高比。

你还会注意到 `hidden` 类用于在移动屏幕上从 DOM 中移除图片，而 `md:block` 用于在桌面屏幕上显示图片。
