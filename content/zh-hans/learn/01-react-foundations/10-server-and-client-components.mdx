要理解服务端组件和客户端组件如何工作，熟悉两个基础的 Web 概念会有所帮助：

*   应用程序代码可以执行到的 [环境 (Environments)](#server-and-client-environments)：服务端和客户端。
*   分隔服务端和客户端代码的 [网络边界 (Network Boundary)](#network-boundary)。

服务端和客户端环境 [](#server-and-client-environments)
-----------------------------------------------------------------

在 Web 应用的上下文中：

![展示了左侧的浏览器和右侧的服务端，由网络边界分隔的示意图](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/learn-client-and-server-environments.png)![展示了左侧的浏览器和右侧的服务端，由网络边界分隔的示意图](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Flearn-client-and-server-environments.png&w=3840&q=75)

*   **客户端 (Client)**：指用户设备上发送请求以获取应用代码给服务端的浏览器。然后它将服务器返回的响应转换为用户可交互的界面。
*   **服务端 (Server)**：指存储应用代码、接收来自客户端的请求、执行计算并返回相应响应的数据中心计算机。

每种环境都有各自的功能和能力限制。例如，通过将渲染和数据获取移至服务端，您可以减少发送给客户端的代码量，从而提升应用性能。但是，正如之前所学的，要使您的 UI 具有交互性，则必须在客户端更新 DOM。

因此，您为服务端和客户端编写的代码并不总是一样的。某些操作（例如数据获取或管理用户状态）更适合在其中一种环境中进行。

网络边界 [](#network-boundary)
-------------------------------------

**网络边界 (Network Boundary)** 是一条概念上的线，用于分隔不同的环境。

在 React 中，您可以决定在组件树中何处放置网络边界。例如，您可以在服务端获取数据并渲染用户的帖子（使用服务端组件），然后在客户端为每个帖子渲染交互式的 `LikeButton`（使用客户端组件）。

同样，您可以创建一个 `Nav` 组件，该组件在服务端渲染并在页面之间共享，但如果您想显示链接的激活状态，则可以在客户端渲染 `Links` 列表。

![显示组件树的示意图，其布局包含 3 个子组件：Nav、Page 和 Footer。Page 组件有 2 个子组件：Posts 和 LikeButton。Posts 组件在服务端渲染，LikeButton 组件在客户端渲染。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/learn-client-server-modules.png)![显示组件树的示意图，其布局包含 3 个子组件：Nav、Page 和 Footer。Page 组件有 2 个子组件：Posts 和 LikeButton。Posts 组件在服务端渲染，LikeButton 组件在客户端渲染。](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Flearn-client-server-modules.png&w=3840&q=75)

在底层，组件被分为两个模块图。**服务端模块图（或树）**包含所有在服务端渲染的服务端组件，而**客户端模块图（或树）**包含所有客户端组件。

渲染服务端组件后，会发送一种称为 **React 服务端组件负载 (RSC)** 的特殊数据格式到客户端。RSC 负载包含：

1.  服务端组件的渲染结果。
2.  客户端组件应渲染位置的占位符（或空洞）及其 JavaScript 文件的引用。

React 使用这些信息来合并服务端和客户端组件，并在客户端更新 DOM。

让我们看看这是如何工作的。

使用客户端组件 [](#using-client-components)
---------------------------------------------------

正如您在上一章所学的，Next.js 默认使用服务端组件——这是为了提高应用性能，意味着您无需采取额外步骤即可采用它们。

回顾一下浏览器中的错误，Next.js 在警告您尝试在服务端组件内部使用 `useState`。您可以将交互式的 "Like" 按钮移动到客户端组件来解决此问题。

在 `app` 文件夹内创建一个名为 `like-button.js` 的新文件，导出 `LikeButton` 组件：

/app/like-button.js

```
export default function LikeButton() {}
```

将 `<button>` 元素和 `handleClick()` 函数从 `page.js` 移动到您新的 `LikeButton` 组件中：

/app/like-button.js

```
export default function LikeButton() {
  function handleClick() {
    setLikes(likes + 1);
  }
 
  return <button onClick={handleClick}>Like ({likes})</button>;
}
```

接下来，移动 `likes` 状态和导入语句：

/app/like-button.js

```
import { useState } from 'react';
 
export default function LikeButton() {
  const [likes, setLikes] = useState(0);
 
  function handleClick() {
    setLikes(likes + 1);
  }
 
  return <button onClick={handleClick}>Like ({likes})</button>;
}
```

现在，为了使 `LikeButton` 成为客户端组件，在文件顶部添加 React 的 `'use client'` 指令。这告诉 React 在客户端渲染该组件。

/app/like-button.js

```
'use client';
 
import { useState } from 'react';
 
export default function LikeButton() {
  const [likes, setLikes] = useState(0);
 
  function handleClick() {
    setLikes(likes + 1);
  }
 
  return <button onClick={handleClick}>Like ({likes})</button>;
}
```

回到您的 `page.js` 文件，将 `LikeButton` 组件导入到您的页面中：

/app/page.js

```
import LikeButton from './like-button';
 
function Header({ title }) {
  return <h1>{title ? title : 'Default title'}</h1>;
}
 
export default function HomePage() {
  const names = ['Ada Lovelace', 'Grace Hopper', 'Margaret Hamilton'];
 
  return (
    <div>
      <Header title="Develop. Preview. Ship." />
      <ul>
        {names.map((name) => (
          <li key={name}>{name}</li>
        ))}
      </ul>
      <LikeButton />
    </div>
  );
}
```

保存这两个文件并在浏览器中查看您的应用。既然没有错误了，一旦您更改并保存，您应该会注意到浏览器会自动更新以反映更改。

此功能称为 [快速刷新 (Fast Refresh)](/docs/architecture/fast-refresh)。它会针对您进行任何编辑立即提供反馈，并且已随 Next.js 预配置好。
