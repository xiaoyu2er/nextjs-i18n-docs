在本章中，我们将开始构建我们的项目，通过使用 JavaScript 和 DOM 方法向您的项目添加 `h1` 标签。

打开您的代码编辑器，创建一个新 `index.html` 文件。在 HTML 文件中，添加以下代码：

index.html

```
<html>
  <body>
    <div></div>
  </body>
</html>
```

然后给 `div` 分配一个唯一的 `id`，以便您稍后定位它。

index.html

```
<html>
  <body>
    <div id="app"></div>
  </body>
</html>
```

要在 HTML 文件中编写 JavaScript，添加一个 `script` 标签：

index.html

```
<html>
  <body>
    <div id="app"></div>
    <script type="text/javascript"></script>
  </body>
</html>
```

现在，在 `script` 标签内，您可以使用 DOM 方法，[`getElementById()`](https://developer.mozilla.org/en-US/docs/Web/API/Document/getElementById)，通过 `id` 选择 `<div>` 元素：

index.html

```
<html>
  <body>
    <div id="app"></div>
    <script type="text/javascript">
      const app = document.getElementById('app');
    </script>
  </body>
</html>
```

您可以继续使用 DOM 方法来创建一个新 `<h1>` 元素：

index.html

```
<html>
  <body>
    <div id="app"></div>
    <script type="text/javascript">
      // Select the div element with 'app' id
      const app = document.getElementById('app');
 
      // Create a new H1 element
      const header = document.createElement('h1');
 
      // Create a new text node for the H1 element
      const text = 'Develop. Preview. Ship.';
      const headerContent = document.createTextNode(text);
 
      // Append the text to the H1 element
      header.appendChild(headerContent);
 
      // Place the H1 element inside the div
      app.appendChild(header);
    </script>
  </body>
</html>
```

为确保一切正常运行，请在您选择的浏览器中打开 HTML 文件。您应该看到一个 `h1` 标签，上面写着“Develop. Preview. Ship.”。

HTML vs. 文档对象模型 [](#html-vs-the-dom)
------------------------------------

如果您查看 [浏览器开发者工具](https://developer.mozilla.org/docs/Learn/Common_questions/Tools_and_setup/What_are_browser_developer_tools) 中的 DOM 元素，您会注意到 DOM 包括了 `<h1>` 元素。页面的 DOM 与源代码不同——换句话说，就是您创建的原始 HTML 文件。

![显示渲染后的 DOM 元素与源代码（HTML）之间差异的并排两张图示](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/learn-dom-and-source.png)![显示渲染后的 DOM 元素与源代码（HTML）之间差异的并排两张图示](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Flearn-dom-and-source.png&w=3840&q=75)

这是因为 HTML 代表**初始页面内容**，而 DOM 代表**更新的页面内容**，这是由您编写的 JavaScript 代码更改后的内容。

使用纯 JavaScript 更新 DOM 非常强大，但也非常冗长。您编写了所有这些代码来添加一个带有文本的 `<h1>` 元素：

index.html

```
<script type="text/javascript">
  const app = document.getElementById('app');
  const header = document.createElement('h1');
  const text = 'Develop. Preview. Ship.';
  const headerContent = document.createTextNode(text);
  header.appendChild(headerContent);
  app.appendChild(header);
</script>
```

随着应用或团队规模的扩大，以这种方式构建应用变得越来越具有挑战性。

使用这种方法，开发人员花费大量时间编写指令来告诉计算机**如何**做事。但是，描述您想展示**什么**，并让计算机弄清楚**如何**更新 DOM，这难道不好吗？

命令式 vs. 声明式编程 [](#imperative-vs-declarative-programming)
--------------------------------------------------------------------------------

上面的代码是**命令式** **编程**的一个很好的例子。您正在编写用户界面应该如何更新的步骤。但在构建用户界面时，声明式方法通常更受欢迎，因为它可以加快开发过程。不需要编写 DOM 方法，如果开发人员能够声明他们想展示**什么**（在这种情况下，是一个带有文本的 `h1` 标签），那就更有用了。

换句话说，**命令式编程**就像给厨师提供有关如何制作披萨的逐步说明。**声明式编程**就像订购披萨而不关心制作披萨的步骤。🍕

[React](https://react.dev/) 是一个流行的声明式库，您可以用来构建用户界面。

React：一个声明式 UI 库 [](#react-a-declarative-ui-library)
------------------------------------------------------------------

作为开发人员，您可以告诉 React 您想要用户界面发生什么，React 会弄清楚**如何**更新 DOM 的步骤来代替您。

在下一节中，我们将探讨如何开始使用 React。
