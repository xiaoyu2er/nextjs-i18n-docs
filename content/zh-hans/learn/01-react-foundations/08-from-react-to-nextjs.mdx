到目前为止，我们探讨了如何开始学习 React。这是最终代码的样子。如果你从这里开始，将这段代码粘贴到代码编辑器中的 `index.html` 文件中。

index.html

```
<html>
  <body>
    <div id="app"></div>
 
    <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
    <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
 
    <script type="text/jsx">
      const app = document.getElementById("app")
 
      function Header({ title }) {
        return <h1>{title ? title : "Default title"}</h1>
      }
 
      function HomePage() {
        const names = ["Ada Lovelace", "Grace Hopper", "Margaret Hamilton"]
 
        const [likes, setLikes] = React.useState(0)
 
        function handleClick() {
          setLikes(likes + 1)
        }
 
        return (
          <div>
            <Header title="Develop. Preview. Ship." />
            <ul>
              {names.map((name) => (
                <li key={name}>{name}</li>
              ))}
            </ul>
 
            <button onClick={handleClick}>Like ({likes})</button>
          </div>
        )
      }
 
      const root = ReactDOM.createRoot(app);
      root.render(<HomePage />);
    </script>
  </body>
</html>
```

在最近几章中，你了解了三个重要的 React 概念：**组件 (components)**、**属性 (props)** 和 **状态 (state)**。在这几方面打下坚实基础将帮助你开始构建 React 应用程序。

就学习 React 而言，**最好的学习方式就是动手构建**。你可以通过使用 `<script>` 和你目前所学，将小组件逐渐集成到现有网站中。然而，许多开发者发现 React 所带来的用户和开发生验非常有价值，足以投入其中，直接用 React 编写整个前端应用程序。

从 React 到 Next.js[](#from-react-to-nextjs)
----------------------------------------------

尽管 React 在构建 UI 方面表现出色，但要独立将该 UI 构建为功能完备且可扩展的应用程序仍需付出一些努力。此外，还有一些需要框架的新 React 功能，例如服务端组件 (Server Components) 和客户端组件 (Client Components)。好消息是，Next.js 处理了大部分配置和设置工作，并拥有额外的功能来帮助你构建 React 应用程序。

接下来，我们将把示例从 React 迁移到 Next.js，讨论 Next.js 的工作原理，并介绍服务端组件 (Server Components) 和客户端组件 (Client Components) 之间的区别。
