当你在项目中使用 Next.js 时，你不再需要从 [unpkg.com](http://unpkg.com) 加载 `react` 和 `react-dom` 脚本。相反，你可以通过 `npm` 或你首选的包管理器在本地安装这些包。

> **注意**：使用 Next.js 时，你需要在机器上安装 **Node.js 20.9** 或更高版本（[查看最低版本要求](/docs/getting-started/installation)），你可以 [在此下载](https://nodejs.org/en/)。

为此，在 `index.html` 文件所在的目录中创建一个新文件，名为 `package.json`，内容为空对象 `{}`。

package.json

```
{}
```

在你的 [终端](https://code.visualstudio.com/docs/terminal/basics) 中，在项目根目录下运行以下命令：

终端

```
npm install react@latest react-dom@latest next@latest
```

安装完成后，你应该能在 `package.json` 文件中看到你的项目依赖项列表：

package.json

```
{
  "dependencies": {
    "next": "^14.0.3",
    "react": "^18.3.1",
    "react-dom": "^18.3.1"
  }
}
```

如果你的版本比上面显示的更高，也不要担心，只要安装了 `next`、`react` 和 `react-dom` 包，你就可以继续了。

你还会注意到一个名为 `package-lock.json` 的新文件，其中包含每个包确切版本的详细信息。

回到 `index.html` 文件，你可以删除以下代码：

1.  `<html>` 和 `<body>` 标签。
2.  `id` 为 `app` 的 `<div>` 元素。
3.  `react` 和 `react-dom` 脚本，因为你已通过 NPM 安装它们。
4.  `Babel` 脚本，因为 Next.js 拥有一个编译器，可将 JSX 转换为浏览器可理解的 JavaScript。
5.  `<script type="text/jsx">` 标签。
6.  `document.getElementById()` 和 `ReactDom.createRoot()` 方法。
7.  `React.useState(0)` 函数中的 `React.` 部分

删除上述行后，将以下导入添加到文件顶部：

index.html

```
import { useState } from 'react';
```

你的代码应该如下所示：

index.html

```
import { useState } from 'react';
 
function Header({ title }) {
  return <h1>{title ? title : 'Default title'}</h1>;
}
 
function HomePage() {
  const names = ['Ada Lovelace', 'Grace Hopper', 'Margaret Hamilton'];
 
  const [likes, setLikes] = useState(0);
 
  function handleClick() {
    setLikes(likes + 1);
  }
 
  return (
    <div>
      <Header title="Develop. Preview. Ship." />
      <ul>
        {names.map((name) => (
          <li key={name}>{name}</li>
        ))}
      </ul>
 
      <button onClick={handleClick}>Like ({likes})</button>
    </div>
  );
}
```

HTML 文件中剩下的唯一代码是 JSX，因此你可以将文件类型从 `.html` 更改为 `.js` 或 `.jsx`。

创建你的第一个页面 [](#creating-your-first-page)
-----------------------------------------------------

Next.js 使用文件系统路由。这意味着你不需要使用代码来定义你的应用路由，而是可以使用文件夹和文件。

以下是你在 Next.js 中创建第一个页面的方法：

1.  创建一个名为 [app](/docs/app/building-your-application/routing#the-app-router) 的新文件夹，并将 `index.js` 文件移入其中。
2.  将你的 `index.js` 文件重命名为 `page.js`。这将作为你应用的首页。
3.  向你的 `<HomePage>` 组件添加 `export default`，以帮助 Next.js 区分应将哪个组件渲染为该页面的主要组件。

`getInitialProps` 在服务端*和*客户端运行。此 API 扩展了 React 组件，允许您创建一个 `Promise` 并将结果转发给组件的 `props`。

```
import { useState } from 'react';
 
function Header({ title }) {
  return <h1>{title ? title : 'Default title'}</h1>;
}
 
export default function HomePage() {
  // ...
}
```

启动开发服务器 [](#running-the-development-server)
-----------------------------------------------------------------

接下来，让我们启动开发服务器，以便你在开发时能在新的页面中看到更改。在你的 `package.json` 文件中添加 `"next dev"` 脚本：

package.json

```
{
  "scripts": {
    "dev": "next dev"
  },
  "dependencies": {
    "next": "^14.0.3",
    "react": "^18.3.1",
    "react-dom": "^18.3.1"
  }
}
```

在终端中运行 `npm run dev` 查看发生的情况。你将会注意到以下两点：

1.  当你访问 [localhost:3000](http://localhost:3000) 时，你应该会看到以下错误：

![Next.js 错误消息：你正在导入一个需要 `useState` 的组件。它仅在 客户端组件 (Client component) 中有效...](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/learn-usestate-rsc-error.png)![Next.js 错误消息：你正在导入一个需要 `useState` 的组件。它仅在 客户端组件 (Client component) 中有效...](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Flearn-usestate-rsc-error.png&w=3840&q=75)

这是因为 Next.js 使用 React 服务端组件 (React Server Components)，这是一项允许 React 在服务器端渲染的新功能。服务端组件 (Server Components) 不支持 `useState`，因此你需要改用客户端组件 (Client Component)。

在下一章中，我们将讨论服务端组件和客户端组件之间的主要差异，并修复此错误。

2.  在 `app` 文件夹内创建了一个名为 `layout.js` 的新文件。这是你应用的主要布局。你可以用它来添加在多个页面间共享的 UI 元素（例如导航栏、页脚等）。

/app/layout.js

```
export const metadata = {
  title: 'Next.js',
  description: 'Generated by Next.js',
};
 
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

总结 [](#summary)
-------------------

回顾目前的迁移情况，你已经可能能感受到使用 Next.js 的好处了：

* 你移除了 React 和 Babel 脚本；这让你体验到了无需再关心的复杂工具和配置。
* 你创建了你的第一个页面。
