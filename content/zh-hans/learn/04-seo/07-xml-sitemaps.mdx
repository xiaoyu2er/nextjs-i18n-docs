**站点地图** 是与 Google 沟通最简单的方式。它们指示了属于您网站的 URL 以及更新的时间，以便 Google 可以更容易地发现新内容并更高效地抓取您的网站。

虽然 XML 站点地图是最知名且最常用的一种，但它们也可以通过 [RSS](https://developers.google.com/search/docs/advanced/sitemaps/build-sitemap) 或 [Atom](https://developers.google.com/search/docs/advanced/sitemaps/build-sitemap) 创建，如果您更偏好最大程度简单性，甚至可以通过 [文本](https://developers.google.com/search/docs/advanced/sitemaps/build-sitemap) 文件创建。

站点地图是一种文件，您可以在其中提供有关您网站上的页面、视频和其他文件的信息，以及它们之间的关系。像 Google 这样的搜索引擎读取此文件，以更智能地抓取您的网站。

根据 [Google](https://developers.google.com/search/docs/advanced/sitemaps/overview) 的说法：

如果满足以下条件，您可能需要站点地图：

*   **您的网站非常大。** 结果，Google 网络爬虫更有可能忽略抓取您的一些新页面或最近更新的页面。
*   **您的网站拥有大量孤立或未相互充分链接的内容页面档案。** 如果您的网站页面没有自然地相互引用，您可以在站点地图中列出它们，以确保 Google 不会忽略您的一些页面。
*   **您的网站是新创建的，且外部链接很少。** Googlebot 和其他网络爬虫通过从一个页面跟随链接到另一个页面来浏览网络。结果，如果没有其他网站链接到您的页面，Google 可能无法发现您的页面。
*   **您的网站有大量富媒体内容（视频、图片）或在 Google News 中显示。** 如果提供，Google 可以在适当的时候将站点地图中的附加信息纳入搜索考虑。

虽然站点地图对于出色的搜索引擎性能来说不是强制性的，但它们可以促进对机器人的抓取和索引，因此您的内容会被更快收录并相应地排名。

强烈建议使用站点地图并在网站上填充新内容时使其保持动态。静态站点地图也是有效的，但它们可能对 Google 用途较小，因为它们不便于持续发现。

### 如何将站点地图添加到 Next.js 项目 [](#how-to-add-sitemaps-to-a-nextjs-project)

有两种选项：

**手动**: 如果您有一个相对简单且静态的网站，您可以在项目的 `public` 目录中手动创建 `sitemap.xml`：

```
   <!-- public/sitemap.xml -->
   <xml version="1.0" encoding="UTF-8">
   <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
     <url>
       <loc>http://www.example.com/foo</loc>
       <lastmod>2021-06-01</lastmod>
     </url>
   </urlset>
   </xml>
```

**动态**: 如果您的网站是动态的，您可以利用 `getServerSideProps` 按需生成 XML 站点地图。

我们可以在 pages 目录下创建一个新的页面，例如 `pages/sitemap.xml.js`。这个页面的目标是调用我们的 API 获取数据，以便了解我们动态页面的 URL。然后我们将编写一个 XML 文件作为 `/sitemap.xml` 的响应。

这是一个示例，您可以亲自尝试：

```
//pages/sitemap.xml.js
const EXTERNAL_DATA_URL = 'https://jsonplaceholder.typicode.com/posts';
 
function generateSiteMap(posts) {
  return `<?xml version="1.0" encoding="UTF-8"?>
   <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
     <!--We manually set the two URLs we know already-->
     <url>
       <loc>https://jsonplaceholder.typicode.com</loc>
     </url>
     <url>
       <loc>https://jsonplaceholder.typicode.com/guide</loc>
     </url>
     ${posts
       .map(({ id }) => {
         return `
       <url>
           <loc>${`${EXTERNAL_DATA_URL}/${id}`}</loc>
       </url>
     `;
       })
       .join('')}
   </urlset>
 `;
}
 
function SiteMap() {
  // getServerSideProps will do the heavy lifting
}
 
export async function getServerSideProps({ res }) {
  // We make an API call to gather the URLs for our site
  const request = await fetch(EXTERNAL_DATA_URL);
  const posts = await request.json();
 
  // We generate the XML sitemap with the posts data
  const sitemap = generateSiteMap(posts);
 
  res.setHeader('Content-Type', 'text/xml');
  // we send the XML to the browser
  res.write(sitemap);
  res.end();
 
  return {
    props: {},
  };
}
 
export default SiteMap;
```
