正如我们在上一节中所见，核心网页标准 (Core Web Vitals) 关注用户体验的各个方面，通过加载性能 (最大内容绘制 (Largest Contentful Paint))、交互性 (首次输入延迟 (First Input Delay)) 和视觉稳定性 (累积布局偏移 (Cumulative Layout Shift))。

在本课程中，我们将重点关注如何使用名为 [Lighthouse](/docs/api-reference/cli#production) 的模拟环境来测量核心网页标准 (Core Web Vitals)。

**注意**: 在本课程中，我们将使用 [Chrome Dev Tools](https://developers.google.com/web/tools/lighthouse?hl=en#devtools)。不过，有 [多种方法](/docs/api-reference/cli#production) 可以运行 Lighthouse。

Lighthouse 的工作方式是运行一系列针对所提供 URL 的检查。基于这些检查，它会生成一份报告，评估页面的表现。如果存在需要改进之处，报告将提供如何改进的洞察。

让我们来看看两个 Lighthouse 报告的示例，以查看具有健康核心网页标准 (Core Web Vitals) 的网站与不具有健康标准的网站之间的区别。

### 优化示例 [](#optimized-example)

为了查看 Lighthouse 如何工作的示例，我们将使用我们的 [主页](https://nextjs.org)。

*   打开 [Chrome](https://www.google.com/chrome/)。
*   在无痕窗口中，导航到 `https://nextjs.org`。
*   打开 DevTools 并点击 **Lighthouse** 标签。
*   点击 **生成报告**。

这将运行一个报告，应该少于 60 秒。

**注意**: 在无痕窗口中运行报告很重要，因为第三方插件会影响您的报告。

此外，广告拦截器可能会阻止脚本加载，从而导致审计不完整。考虑设置干净的 [persona](/docs/api-reference/cli#production) 来测量性能。

### 未优化示例 [](#unoptimized-example)

为了本教程的目的，我们创建了一个未进行任何优化的应用程序。

#### 项目设置 [](#project-setup)

这是一个基本的未优化应用程序，允许访客做两件事：搜索国家以检索其人口，并点击按钮阅读弹出模态框。此应用程序旨在模拟在大型应用程序上工作的现实情况，其中无法避免使用第三方库。

#### 下载 Starter Code[](#download-starter-code)

```
npx create-next-app@latest nextjs-lighthouse --use-npm --example "https://github.com/vercel/next-learn/tree/main/seo"
```

#### 运行生产构建 [](#run-production-build)

为了从 Lighthouse 获得准确的报告，您的应用程序必须始终使用生产构建进行测试。要运行生产构建，请进入项目目录：

```
cd nextjs-lighthouse
```

通过运行 `next build` 构建应用程序，并通过运行 `next start` 在 [生产模式](/docs/api-reference/cli#production) 下启动服务器。

```
npm run build && npm run start
```

让我们使用 Chrome DevTools 运行 Lighthouse 报告。报告完成后，让我们开始优化网站并改进核心指标 (vitals)。
