### 嵌套路由

文件夹定义 URL 段。嵌套文件夹即嵌套段。任意级别的布局都会包装其子段。当存在 `page` 或 `route` 文件时，路由变为公开。

| 路径                        | URL 模式        | 备注                         |
| --------------------------- | --------------- | ---------------------------- |
| `app/layout.tsx`            | —               | 根布局包装所有路由           |
| `app/blog/layout.tsx`       | —               | 包装 `/blog` 及其后代路由    |
| `app/page.tsx`              | `/`             | 公开路由                     |
| `app/blog/page.tsx`         | `/blog`         | 公开路由                     |
| `app/blog/authors/page.tsx` | `/blog/authors` | 公开路由                     |

### 动态路由

使用方括号对段进行参数化。使用 `[segment]` 表示单个参数，`[...segment]` 表示捕获所有，`[[...segment]]` 表示可选捕获所有。通过 [`params`](/docs/app/api-reference/file-conventions/page#params-optional) 属性访问值。

| 路径                            | URL 模式                                                               |
| ------------------------------- | --------------------------------------------------------------------- |
| `app/blog/[slug]/page.tsx`      | `/blog/my-first-post`                                                 |
| `app/shop/[...slug]/page.tsx`   | `/shop/clothing`, `/shop/clothing/shirts`                             |
| `app/docs/[[...slug]]/page.tsx` | `/docs`, `/docs/layouts-and-pages`, `/docs/api-reference/use-router`  |

### 路由组与私有文件夹

使用路由组 [`(group)`](/docs/app/api-reference/file-conventions/route-groups#convention) 在不更改 URL 的情况下组织代码，并使用私有文件夹 [`_folder`](#private-folders) 放置非路由文件。

| 路径                            | URL 模式 | 备注                                     |
| ------------------------------- | -------- | ---------------------------------------- |
| `app/(marketing)/page.tsx`      | `/`      | 组名从 URL 中省略                        |
| `app/(shop)/cart/page.tsx`      | `/cart`  | 在 `(shop)` 内共享布局                   |
| `app/blog/_components/Post.tsx` | —        | 不可路由；UI 工具的安全存放处            |
| `app/blog/_lib/data.ts`         | —        | 不可路由；工具函数的安全存放处           |

### 并行和拦截路由

这些功能适用于特定的 UI 模式，例如基于插槽的布局或模态框路由。

使用 `@slot` 表示由父布局渲染的命名插槽。使用拦截模式在当前布局内渲染另一个路由而不更改 URL，例如，将详细信息视图显示为列表上方的模态框。

| 模式 (文档)                                                                                      | 含义                 | 典型用例                             |
| ---------------------------------------------------------------------------------------------- | -------------------- | ------------------------------------ |
| [`@folder`](/docs/app/api-reference/file-conventions/parallel-routes#slots)                     | 命名插槽             | 侧边栏 + 主内容                      |
| [`(.)folder`](/docs/app/api-reference/file-conventions/intercepting-routes#convention)          | 拦截同级             | 在模态框中预览同级路由               |
| [`(..)folder`](/docs/app/api-reference/file-conventions/intercepting-routes#convention)         | 拦截父级             | 将父级的子级作为叠加层打开           |
| [`(..)(..)folder`](/docs/app/api-reference/file-conventions/intercepting-routes#convention)     | 拦截两级             | 深度嵌套的叠加层                     |
| [`(...)folder`](/docs/app/api-reference/file-conventions/intercepting-routes#convention)        | 从根拦截             | 在当前视图中显示任意路由             |

### 组织你的项目

Next.js 对于你如何组织和放置项目文件是**无固定看法的**。但它确实提供了几种功能来帮助你组织项目。

### 组件层次结构

在特殊文件中定义的组件按特定层次结构渲染：

- `layout.js`
- `template.js`
- `error.js` (React 错误边界)
- `loading.js` (React 悬念边界)
- `not-found.js` (用于 "未找到" UI 的 React 错误边界)
- `page.js` 或嵌套的 `layout.js`

### 就近存放

在 `app` 目录中，嵌套文件夹定义了路由结构。每个文件夹代表一个路由段，该段映射到 URL 路径中的相应段。

然而，即使路由结构是通过文件夹定义的，一个路由在向路由段添加 `page.js` 或 `route.js` 文件之前**不是公开可访问的**。

<Image
  alt="显示在向路由段添加 page.js 或 route.js 文件之前，路由不是公开可访问的示意图。"
  srcLight="/docs/light/project-organization-not-routable.png"
  srcDark="/docs/dark/project-organization-not-routable.png"
  width="1600"
  height="444"
/>

而且，即使路由变为公开可访问的，也只有 `page.js` 或 `route.js` **返回的内容**会被发送到客户端。

<Image
  alt="显示 page.js 和 route.js 文件如何使路由公开可访问的示意图。"
  srcLight="/docs/light/project-organization-routable.png"
  srcDark="/docs/dark/project-organization-routable.png"
  width="1600"
  height="687"
/>

这意味着**项目文件**可以**安全地放置**在 `app` 目录中的路由段内，而不会意外地变为可路由的。

<Image
  alt="显示即使段包含 page.js 或 route.js 文件，放置的项目文件也不是可路由的示意图。"
  srcLight="/docs/light/project-organization-colocation.png"

> **值得注意的是**：虽然你**可以**将项目文件放置在 `app` 中，但你**并非必须**这样做。如果你愿意，你可以[将它们保留在 `app` 目录之外](#store-project-files-outside-of-app)。

### 私有文件夹

可以通过在文件夹名称前加下划线来创建私有文件夹：`_folderName`

这表明该文件夹是私有的实现细节，不应被路由系统考虑，从而**将该文件夹及其所有子文件夹排除在路由之外**。

<Image
  alt="使用私有文件夹的示例文件夹结构"
  srcLight="/docs/light/project-organization-private-folders.png"
  srcDark="/docs/dark/project-organization-private-folders.png"
  width="1600"
  height="849"
/>

由于 `app` 目录中的文件默认可以[安全地放置](#colocation)，私有文件夹不是放置所必需的。然而，它们可用于：

- 将 UI 逻辑与路由逻辑分离。
- 在项目和 Next.js 生态系统中一致地组织内部文件。
- 在代码编辑器中排序和分组文件。
- 避免与未来的 Next.js 文件约定发生潜在的命名冲突。

> **值得注意的是**：
>
> - 虽然这不是框架约定，但你也可以考虑使用相同的下划线模式标记私有文件夹之外的文件为“私有”。
> - 你可以通过在文件夹名称前加 `%5F`（下划线的 URL 编码形式）来创建以下划线开头的 URL 段：`%5FfolderName`。
> - 如果你不使用私有文件夹，了解 Next.js [特殊文件约定](/docs/app/getting-started/project-structure#routing-files)将有助于防止意外的命名冲突。

### 路由组

可以通过将文件夹名包裹在括号中来创建路由组：`(folderName)`

这表明该文件夹用于组织目的，**不应包含**在路由的 URL 路径中。

<Image
  alt="使用路由组的示例文件夹结构"
  srcLight="/docs/light/project-organization-route-groups.png"
  srcDark="/docs/dark/project-organization-route-groups.png"
  width="1600"
  height="849"
/>

路由组可用于：

- 按站点部分、意图或团队组织路由。例如，营销页面、管理员页面等。
- 在同一路由段级别启用嵌套布局：
  - [在同一段中创建多个嵌套布局，包括多个根布局](#creating-multiple-root-layouts)
  - [将布局添加到公共段中的一部分路由](#opting-specific-segments-into-a-layout)

> **值得注意的是**：在我们的下面示例中，我们使用 `components` 和 `lib` 文件夹作为通用占位符，它们的命名没有特殊的框架意义，你的项目可能使用其他文件夹，如 `ui`、`utils`、`hooks`、`styles` 等。
