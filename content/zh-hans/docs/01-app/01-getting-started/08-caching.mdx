> 本页介绍使用 [缓存组件 (Cache Components)](/docs/app/api-reference/config/next-config-js/cacheComponents) 进行缓存，该功能通过在你的 `next.config.ts` 文件中设置 [`cacheComponents: true`](/docs/app/api-reference/config/next-config-js/cacheComponents) 来启用。如果你不使用缓存组件 (Cache Components)，请参阅[缓存与重新验证（先前模型）](/docs/app/guides/caching-without-cache-components)指南。

缓存 (Caching) 是一种存储数据获取和其他计算结果的技术，以便未来对相同数据的请求可以更快地响应，而无需重复执行工作。

## 启用缓存组件 (Cache Components)

你可以通过将 [`cacheComponents`](/docs/app/api-reference/config/next-config-js/cacheComponents) 选项添加到你的 Next.js 配置文件来启用缓存组件 (Cache Components)：

```ts filename="next.config.ts" highlight={4} switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  cacheComponents: true,
}

export default nextConfig
```

```js filename="next.config.js" highlight={3} switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  cacheComponents: true,
}

module.exports = nextConfig
```

> **值得注意的是：** 当启用缓存组件 (Cache Components) 时，`GET` 路由处理程序 (Route Handlers) 遵循与页面相同的预渲染模型。详情请参阅[使用缓存组件的路由处理程序](/docs/app/getting-started/route-handlers#with-cache-components)。

## 使用方法

[`use cache`](/docs/app/api-reference/directives/use-cache) 指令用于缓存异步函数和组件的返回值。你可以在两个级别应用它：

- **数据级 (Data-level)**：缓存一个获取或计算数据的函数（例如，`getProducts()`, `getUser(id)`）。
- **UI 级 (UI-level)**：缓存整个组件或页面（例如，`async function BlogPosts()`）。

> 参数和任何来自父作用域的闭合值会自动成为[缓存键 (cache key)](/docs/app/api-reference/directives/use-cache#cache-keys) 的一部分，这意味着不同的输入将产生单独的缓存条目。这使得个性化或参数化的缓存内容成为可能。有关什么可以被缓存以及参数如何工作的详细信息，请参阅[序列化要求和约束](/docs/app/api-reference/directives/use-cache#constraints)。

### 数据级缓存 (Data-level caching)

要缓存一个获取数据的异步函数，请在函数体顶部添加 `use cache` 指令：

```tsx filename="app/lib/data.ts" highlight={3,4,5}
import { cacheLife } from 'next/cache'

export async function getUsers() {
  'use cache'
  cacheLife('hours')
  return db.query('SELECT * FROM users')
}
```

数据级缓存 (Data-level caching) 在相同数据被多个组件使用，或者你希望独立于 UI 缓存数据时非常有用。

### UI 级 缓存 (UI-level caching)

要缓存整个组件、页面或布局，请在组件或页面体顶部添加 `use cache` 指令：

```tsx filename="app/page.tsx" highlight={1,4,5}
import { cacheLife } from 'next/cache'

export default async function Page() {
  'use cache'
  cacheLife('hours')

  const users = await db.query('SELECT * FROM users')

  return (
    <ul>
      {users.map((user) => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  )
}
```

> 如果你在文件顶部添加 "`use cache`"，文件中的所有导出函数都将被缓存。

### 流式传输未缓存数据 (Streaming uncached data)

对于从异步源（如 API、数据库或任何其他异步操作）获取数据，并且每次请求都需要新鲜数据的组件，请不要使用 `"use cache"`。

相反，将组件包裹在 [`<Suspense>`](https://react.dev/reference/react/Suspense) 中并提供一个回退 UI。在请求时，React 首先渲染回退内容，然后在异步工作完成后流式传输解析后的内容。

```tsx filename="page.tsx"
import { Suspense } from 'react'

async function LatestPosts() {
  const data = await fetch('https://api.example.com/posts')
  const posts = await data.json()
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}

export default function Page() {
  return (
    <>
      <h1>My Blog</h1>
      <Suspense fallback={<p>Loading posts...</p>}>
        <LatestPosts />
      </Suspense>
    </>
  )
}
```

回退 UI (`<p>Loading posts...</p>`) 被包含在静态外壳中，而组件的内容则在请求时流式传输进来。

## 使用运行时 API (Runtime APIs)

请求时 API 需要仅在用户发起请求时才可用的信息。这些包括：

- [`cookies`](/docs/app/api-reference/functions/cookies) - 用户的 Cookie 数据
- [`headers`](/docs/app/api-reference/functions/headers) - 请求头
- [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) - URL 查询参数
- [`params`](/docs/app/api-reference/file-conventions/page#params-optional) - 动态路由参数（除非通过 [`generateStaticParams`](/docs/app/api-reference/functions/generate-static-params) 提供了至少一个样本）。

访问运行时 API 的组件应包裹在 `<Suspense>` 中：

```tsx filename="page.tsx"
import { cookies } from 'next/headers'
import { Suspense } from 'react'

async function UserGreeting() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')?.value || 'light'
  return <p>Your theme: {theme}</p>
}

export default function Page() {
  return (
    <>
      <h1>Dashboard</h1>
      <Suspense fallback={<p>Loading...</p>}>
        <UserGreeting />
      </Suspense>
    </>
  )
}
```

### 将运行时值传递给缓存函数 (Passing runtime values to cached functions)

你可以从运行时 API 提取值，并将其作为参数传递给缓存函数：

```tsx filename="app/profile/page.tsx"
import { cookies } from 'next/headers'
import { Suspense } from 'react'

export default function Page() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <ProfileContent />
    </Suspense>
  )
}

// Component (not cached) reads runtime data
async function ProfileContent() {
  const session = (await cookies()).get('session')?.value
  return <CachedContent sessionId={session} />
}

// Cached component receives extracted value as a prop
async function CachedContent({ sessionId }: { sessionId: string }) {
  'use cache'
  // sessionId becomes part of the cache key
  const data = await fetchUserData(sessionId)
  return <div>{data}</div>
}
```

在请求时，`CachedContent` 如果没有找到匹配的缓存条目则会执行，并为具有相同 `sessionId` 的未来请求存储结果。

## 处理非确定性操作 (Non-deterministic operations)

像 `Math.random()`、`Date.now()` 或 `crypto.randomUUID()` 这样的操作每次执行都会产生不同的值。缓存组件 (Cache Components) 要求你显式处理这些情况。

**为了为每个请求生成唯一值**，通过在这些操作之前调用 [`connection()`](/docs/app/api-reference/functions/connection) 来推迟到请求时，并将组件包裹在 `<Suspense>` 中：

```tsx filename="page.tsx"
import { connection } from 'next/server'
import { Suspense } from 'react'

async function UniqueContent() {
  await connection()
  const uuid = crypto.randomUUID()
  return <p>Request ID: {uuid}</p>
}

export default function Page() {
  return (
    <Suspense fallback={<p>Loading...</p>}>
      <UniqueContent />
    </Suspense>
  )
}
```

或者，你可以**缓存结果**，以便所有用户在重新验证之前看到相同的值：

```tsx filename="page.tsx"
export default async function Page() {
  'use cache'
  const buildId = crypto.randomUUID()
  return <p>Build ID: {buildId}</p>
}
```

## 处理确定性操作 (Deterministic operations)

像同步 I/O、模块导入和纯计算这样的操作可以在预渲染期间完成。仅使用这些操作的组件，其渲染输出会自动包含在静态 HTML 外壳中。

```tsx filename="page.tsx"
import fs from 'node:fs'

export default async function Page() {
  const content = fs.readFileSync('./config.json', 'utf-8')
  const constants = await import('./constants.json')
  const processed = JSON.parse(content).items.map((item) => item.value * 2)

  return (
    <div>
      <h1>{constants.appName}</h1>
      <ul>
        {processed.map((value, i) => (
          <li key={i}>{value}</li>
        ))}
      </ul>
    </div>
  )
}
```

## 渲染工作原理 (How rendering works)

在构建时，Next.js 会渲染你路由的组件树。每个组件如何处理取决于它使用的 API：

- [`use cache`](#usage)：结果被缓存并包含在静态外壳中
- [`<Suspense>`](#streaming-uncached-data)：回退 UI 包含在静态外壳中，而内容在请求时流式传输
- [确定性操作](#working-with-deterministic-operations)：如纯计算和模块导入，会自动包含在静态外壳中

这会生成一个静态外壳，包含用于初始页面加载的 HTML 和用于客户端导航的序列化 [RSC 载荷](/docs/app/getting-started/server-and-client-components#on-the-server)，确保无论用户直接导航到 URL 还是从另一个页面过渡，浏览器都能即时接收到完全渲染的内容。

### 禁用静态外壳 (Opting out of the static shell)

在根布局中，在文档主体上方放置一个具有空回退 (empty fallback) 的 `<Suspense>` 边界会导致整个应用推迟到请求时执行。由于回退为空，没有可立即发送的静态外壳，因此每个请求都会阻塞，直到页面完全渲染。要将其限制在特定路由，请使用[多个根布局](/docs/app/api-reference/file-conventions/layout#root-layout)。

```tsx filename="app/layout.tsx" highlight={1,10-12}
import { Suspense } from 'react'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html>
      <Suspense fallback={null}>
        <body>{children}</body>
      </Suspense>
    </html>
  )
}
```

> **值得注意的是**：当 `generateViewport` 访问未缓存的动态数据时，也适用相同的模式。有关详细示例，请参阅[使用缓存组件的视口 (Viewport)](/docs/app/api-reference/functions/generate-viewport#with-cache-components)。

### 综合运用 (Putting it all together)

以下是一个完整的示例，展示了静态内容、缓存的动态内容和流式传输的动态内容在单个页面上协同工作：

```tsx filename="app/blog/page.tsx"
import { Suspense } from 'react'
import { cookies } from 'next/headers'
import { cacheLife, cacheTag, updateTag } from 'next/cache'
import Link from 'next/link'

export default function BlogPage() {
  return (
    <>
      {/* Static content - prerendered automatically */}
      <header>
        <h1>Our Blog</h1>
        <nav>
          <Link href="/">Home</Link> | <Link href="/about">About</Link>
        </nav>
      </header>

      {/* Cached dynamic content - included in the static shell */}
      <BlogPosts />

      {/* Runtime dynamic content - streams at request time */}
      <Suspense fallback={<p>Loading your preferences...</p>}>
        <UserPreferences />
      </Suspense>

      {/* Mutation - server action that revalidates the cache */}
      <Suspense fallback={<p>Loading...</p>}>
        <CreatePost />
      </Suspense>
    </>
  )
}

// Everyone sees the same blog posts (revalidated every hour)
async function BlogPosts() {
  'use cache'
  cacheLife('hours')
  cacheTag('posts')

  const res = await fetch('https://api.vercel.app/blog')
  const posts = await res.json()

  return (
    <section>
      <h2>Latest Posts</h2>
      <ul>
        {posts.slice(0, 5).map((post: any) => (
          <li key={post.id}>
            <h3>{post.title}</h3>
            <p>
              By {post.author} on {post.date}
            </p>
          </li>
        ))}
      </ul>
    </section>
  )
}

// Personalized per user based on their cookie
async function UserPreferences() {
  const theme = (await cookies()).get('theme')?.value || 'light'
  const favoriteCategory = (await cookies()).get('category')?.value

  return (
    <aside>
      <p>Your theme: {theme}</p>
      {favoriteCategory && <p>Favorite category: {favoriteCategory}</p>}
    </aside>
  )
}

// Admin-only form that creates a post and revalidates the cache
async function CreatePost() {
  const isAdmin = (await cookies()).get('role')?.value === 'admin'
  if (!isAdmin) return null

  async function createPost(formData: FormData) {
    'use server'
    await db.post.create({ data: { title: formData.get('title') } })
    updateTag('posts')
  }

  return (
    <form action={createPost}>
      <input name="title" placeholder="Post title" required />
      <button type="submit">Publish</button>
    </form>
  )
}
```

在预渲染期间，页眉（静态）和博客文章（使用 `use cache` 缓存）连同用户偏好的回退 UI 成为静态外壳的一部分。只有个性化的偏好在请求时流式传输进来。当管理员发布新文章时，[`updateTag`](/docs/app/getting-started/revalidating#updatetag) 调用会立即过期博客文章缓存，以便下一个访客能看到它。

> **值得注意的是：** `generateMetadata` 和 `generateViewport` 会单独跟踪运行时数据访问。有关如何处理此情况，请参阅[使用缓存组件的元数据](/docs/app/api-reference/functions/generate-metadata#with-cache-components)和[使用缓存组件的视口](/docs/app/api-reference/functions/generate-viewport#with-cache-components)。
