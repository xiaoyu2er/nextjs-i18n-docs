`use cache` 指令允许您将路由、React 组件或函数标记为可缓存。它可以在文件顶部使用，表示该文件中的所有导出都应被缓存；也可以内联在函数或组件顶部，以缓存返回值。

> **须知：**
>
> - 要使用 cookies 或 headers，请在缓存作用域之外读取它们，并将值作为参数传递。这是推荐的模式。
> - 如果内存缓存不足以满足运行时数据需求，[`'use cache: remote'`](/docs/app/api-reference/directives/use-cache-remote) 允许平台提供专用的缓存处理程序，尽管它需要一次网络往返来检查缓存，并且通常会产生平台费用。
> - 对于合规性要求，或者当您无法重构代码将运行时数据作为参数传递给 `use cache` 作用域时，请参阅 [`'use cache: private'`](/docs/app/api-reference/directives/use-cache-private)。

## 使用方法

`use cache` 是一个缓存组件功能。要启用它，请将 [`cacheComponents`](/docs/app/api-reference/config/next-config-js/cacheComponents) 选项添加到您的 `next.config.ts` 文件中：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  cacheComponents: true,
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  cacheComponents: true,
}

module.exports = nextConfig
```

然后，在文件、组件或函数级别添加 `use cache`：

```tsx
// File level
'use cache'

export default async function Page() {
  // ...
}

// Component level
export async function MyComponent() {
  'use cache'
  return <></>
}

// Function level
export async function getData() {
  'use cache'
  const data = await fetch('/api/data')
  return data
}
```

> **须知**：在文件级别使用时，所有函数导出都必须是异步函数。

## "use cache" 如何工作

### 缓存键

缓存条目的键是使用其输入的序列化版本生成的，其中包括：

1. **构建 ID (Build ID)** - 每次构建唯一，更改此值会使所有缓存条目失效
2. **函数 ID (Function ID)** - 函数在代码库中位置和签名的安全哈希值
3. **可序列化参数 (Serializable arguments)** - 属性（对于组件）或函数参数
4. **HMR 刷新哈希 (HMR refresh hash)**（仅开发环境）- 在热模块替换时使缓存失效

当一个缓存的函数引用来自外部作用域的变量时，这些变量会被自动捕获并绑定为参数，成为缓存键的一部分。

```tsx filename="lib/data.ts"
async function Component({ userId }: { userId: string }) {
  const getData = async (filter: string) => {
    'use cache'
    // Cache key includes both userId (from closure) and filter (argument)
    return fetch(`/api/users/${userId}/data?filter=${filter}`)
  }

  return getData('active')
}
```

在上面的代码片段中，`userId` 从外部作用域捕获，`filter` 作为参数传递，因此两者都成为 `getData` 函数缓存键的一部分。这意味着不同的用户和过滤器组合将拥有独立的缓存条目。

## 序列化

缓存函数的参数及其返回值必须是可序列化的。

完整参考，请参阅：

- [可序列化参数](https://react.dev/reference/rsc/use-server#serializable-parameters-and-return-values) - 使用 **React 服务器组件** 序列化
- [可序列化返回类型](https://react.dev/reference/rsc/use-client#serializable-types) - 使用 **React 客户端组件** 序列化

> **须知：** 参数和返回值使用不同的序列化系统。服务器组件序列化（用于参数）比客户端组件序列化（用于返回值）更严格。这意味着您可以返回 JSX 元素，但不能接受它们作为参数，除非使用透传模式。

### 支持的数据类型

**参数：**

- 原始类型：`string`、`number`、`boolean`、`null`、`undefined`
- 普通对象：`{ key: value }`
- 数组：`[1, 2, 3]`
- Dates、Maps、Sets、TypedArrays、ArrayBuffers
- React 元素（仅作为透传）

**返回值：**

- 与参数相同，加上 JSX 元素

### 不支持的数据类型

- 类实例
- 函数（透传除外）
- Symbols、WeakMaps、WeakSets
- URL 实例

```tsx filename="app/components/user-card.tsx"
// Valid - primitives and plain objects
async function UserCard({
  id,
  config,
}: {
  id: string
  config: { theme: string }
}) {
  'use cache'
  return <div>{id}</div>
}

// Invalid - class instance
async function UserProfile({ user }: { user: UserClass }) {
  'use cache'
  // Error: Cannot serialize class instance
  return <div>{user.name}</div>
}
```

### 透传 (不可序列化的参数)

只要您不检视它们，就可以接受不可序列化的值。这支持了与 `children` 和 Server Actions 的组合模式：

```tsx filename="app/components/cached-wrapper.tsx"
async function CachedWrapper({ children }: { children: ReactNode }) {
  'use cache'
  // Don't read or modify children - just pass it through
  return (
    <div className="wrapper">
      <header>Cached Header</header>
      {children}
    </div>
  )
}

// Usage: children can be dynamic
export default function Page() {
  return (
    <CachedWrapper>
      <DynamicComponent /> {/* Not cached, passed through */}
    </CachedWrapper>
  )
}
```

您还可以通过缓存的组件传递 Server Actions：

```tsx filename="app/components/cached-form.tsx"
async function CachedForm({ action }: { action: () => Promise<void> }) {
  'use cache'
  // Don't call action here - just pass it through
  return <form action={action}>{/* ... */}</form>
}
```

## 限制

缓存的函数在隔离环境中执行。以下约束确保缓存行为保持可预测和安全。

### 请求时 API

缓存的函数和组件 **不能** 直接访问 `cookies()`、`headers()` 或 `searchParams` 等运行时 API。相反，请在缓存作用域之外读取这些值，并将它们作为参数传递。

### 运行时缓存注意事项

虽然 `use cache` 主要设计用于将未缓存的数据包含在静态外壳中，但它也可以使用内存中的 LRU（最近最少使用）存储在运行时缓存数据。

运行时缓存行为取决于您的宿主环境：

| 环境             | 运行时缓存行为                                                                                     |
| ---------------- | ------------------------------------------------------------------------------------------------ |
| **无服务器 (Serverless)**  | 缓存条目通常在请求之间不会持久化（每个请求可能是不同的实例）。构建时缓存正常工作。                  |
| **自托管 (Self-hosted)**   | 缓存条目在请求之间持久存在。使用 [`cacheMaxMemorySize`](/docs/app/api-reference/config/next-config-js/incrementalCacheHandlerPath) 控制缓存大小。 |

如果默认的内存缓存不够，请考虑 **[`use cache: remote`](/docs/app/api-reference/directives/use-cache-remote)**，它允许平台提供专用的缓存处理程序（如 Redis 或 KV 数据库）。这有助于减少对未针对您的总流量进行扩展的数据源的命中，但会产生费用（存储、网络延迟、平台费用）。

在极少数情况下，对于合规性要求，或者当您无法重构代码将运行时数据作为参数传递给 `use cache` 作用域时，您可能需要 [`use cache: private`](/docs/app/api-reference/directives/use-cache-private)。

### React.cache 隔离

[`React.cache`](https://react.dev/reference/react/cache) 在 `use cache` 边界内的隔离作用域中运行。在 `use cache` 函数外部通过 `React.cache` 存储的值在函数内部不可见。

这意味着您不能使用 `React.cache` 将数据传递到 `use cache` 作用域中：

```tsx
import { cache } from 'react'

const store = cache(() => ({ current: null as string | null }))

function Parent() {
  const shared = store()
  shared.current = 'value from parent'
  return <Child />
}

async function Child() {
  'use cache'
  const shared = store()
  // shared.current is null, not 'value from parent'
  // use cache has its own isolated React.cache scope
  return <div>{shared.current}</div>
}
```

这种隔离确保缓存的函数具有可预测、自包含的行为。要将数据传递到 `use cache` 作用域中，请改用函数参数。

## `use cache` 在运行时

在 **服务器** 上，缓存条目存储在内存中，并遵守您的 `cacheLife` 配置中的 `revalidate` 和 `expire` 时间。您可以通过在 `next.config.js` 文件中配置 [`cacheHandlers`](/docs/app/api-reference/config/next-config-js/cacheHandlers) 来自定义缓存存储。

在 **客户端** 上，来自服务器缓存的内容存储在浏览器的内存中，持续时间由 `stale` 时间定义。客户端路由器强制执行 **最少 30 秒的 stale 时间**，无论配置如何。

`x-nextjs-stale-time` 响应标头将缓存生存期从服务器传达给客户端，确保行为协调。

## 重新验证 (Revalidation)

默认情况下，`use cache` 使用 `default` 配置，其设置如下：

- **stale**：5 分钟（客户端）
- **revalidate**：15 分钟（服务器端）
- **expire**：永不过期

```tsx filename="lib/data.ts"
async function getData() {
  'use cache'
  // Implicitly uses default profile
  return fetch('/api/data')
}
```

### 自定义缓存生命周期

使用 [`cacheLife`](/docs/app/api-reference/functions/cacheLife) 函数来自定义缓存持续时间：

```tsx filename="lib/data.ts"
import { cacheLife } from 'next/cache'

async function getData() {
  'use cache'
  cacheLife('hours') // Use built-in 'hours' profile
  return fetch('/api/data')
}
```

### 按需重新验证

使用 [`cacheTag`](/docs/app/api-reference/functions/cacheTag)、[`updateTag`](/docs/app/api-reference/functions/updateTag) 或 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag) 进行按需缓存失效：

```tsx filename="lib/data.ts"
import { cacheTag } from 'next/cache'

async function getProducts() {
  'use cache'
  cacheTag('products')
  return fetch('/api/products')
}
```

```tsx filename="app/actions.ts"
'use server'

import { updateTag } from 'next/cache'

export async function updateProduct() {
  await db.products.update(...)
  updateTag('products') // Invalidates all 'products' caches
}
```

`cacheLife` 和 `cacheTag` 都集成了客户端和服务器缓存层，这意味着您只需在一个地方配置缓存语义，它们就会在所有地方应用。

## 示例

### 使用 `use cache` 缓存整个路由

要预渲染整个路由，请将 `use cache` 添加到 **同时** `layout` 和 `page` 文件的顶部。这些段中的每一个都被视为应用程序中的单独入口点，并将被独立缓存。

```tsx filename="app/layout.tsx" switcher
'use cache'

export default async function Layout({ children }: { children: ReactNode }) {
  return <div>{children}</div>
}
```

```jsx filename="app/layout.js" switcher
'use cache'

export default async function Layout({ children }) {
  return <div>{children}</div>
}
```

在 `page` 文件中导入和嵌套的任何组件都是与 `page` 关联的缓存输出的一部分。

```tsx filename="app/page.tsx" switcher
'use cache'

async function Users() {
  const users = await fetch('/api/users')
  // loop through users
}

export default async function Page() {
  return (
    <main>
      <Users />
    </main>
  )
}
```

```jsx filename="app/page.js" switcher
'use cache'

async function Users() {
  const users = await fetch('/api/users')
  // loop through users
}

export default async function Page() {
  return (
    <main>
      <Users />
    </main>
  )
}
```

> **须知**：
>
> - 如果只将 `use cache` 添加到 `layout` 或 `page`，则只有该路由段及其导入的任何组件会被缓存。

### 使用 `use cache` 缓存组件的输出

您可以在组件级别使用 `use cache` 来缓存在该组件内执行的任何获取请求或计算。只要序列化的 props 在每个实例中产生相同的值，缓存条目就会被重用。

```tsx filename="app/components/bookings.tsx" highlight={2} switcher
export async function Bookings({ type = 'haircut' }: BookingsProps) {
  'use cache'
  async function getBookingsData() {
    const data = await fetch(`/api/bookings?type=${encodeURIComponent(type)}`)
    return data
  }
  return //...
}

interface BookingsProps {
  type: string
}
```

```jsx filename="app/components/bookings.js" highlight={2} switcher
export async function Bookings({ type = 'haircut' }) {
  'use cache'
  async function getBookingsData() {
    const data = await fetch(`/api/bookings?type=${encodeURIComponent(type)}`)
    return data
  }
  return //...
}
```

### 使用 `use cache` 缓存函数输出

因为您可以将 `use cache` 添加到任何异步函数，所以您不仅限于缓存组件或路由。您可能希望缓存网络请求、数据库查询或缓慢的计算。

```tsx filename="app/actions.ts" highlight={2} switcher
export async function getData() {
  'use cache'

  const data = await fetch('/api/data')
  return data
}
```

```jsx filename="app/actions.js" highlight={2} switcher
export async function getData() {
  'use cache'

  const data = await fetch('/api/data')
  return data
}
```

### 交错

在 React 中，使用 `children` 或插槽进行组合是构建灵活组件的众所周知的模式。使用 `use cache` 时，您可以继续以这种方式组合 UI。作为 `children` 或其他组合插槽包含在返回的 JSX 中的任何内容，都将通过缓存的组件传递，而不会影响其缓存条目。

只要您不在可缓存函数本身的主体中直接引用任何 JSX 插槽，它们在返回输出中的存在就不会影响缓存条目。

```tsx filename="app/page.tsx" switcher
export default async function Page() {
  const uncachedData = await getData()
  return (
    // Pass compositional slots as props, e.g. header and children
    <CacheComponent header={<h1>Home</h1>}>
      {/* DynamicComponent is provided as the children slot */}
      <DynamicComponent data={uncachedData} />
    </CacheComponent>
  )
}

async function CacheComponent({
  header, // header: a compositional slot, injected as a prop
  children, // children: another slot for nested composition
}: {
  header: ReactNode
  children: ReactNode
}) {
  'use cache'
  const cachedData = await fetch('/api/cached-data')
  return (
    <div>
      {header}
      <PrerenderedComponent data={cachedData} />
      {children}
    </div>
  )
}
```

```jsx filename="app/page.js" switcher
export default async function Page() {
  const uncachedData = await getData()
  return (
    // Pass compositional slots as props, e.g. header and children
    <CacheComponent header={<h1>Home</h1>}>
      {/* DynamicComponent is provided as the children slot */}
      <DynamicComponent data={uncachedData} />
    </CacheComponent>
  )
}

async function CacheComponent({
  header, // header: a compositional slot, injected as a prop
  children, // children: another slot for nested composition
}) {
  'use cache'
  const cachedData = await fetch('/api/cached-data')
  return (
    <div>
      {header}
      <PrerenderedComponent data={cachedData} />
      {children}
    </div>
  )
}
```

您也可以通过缓存组件将服务器操作 (Server Actions) 传递给客户端组件，而无需在可缓存函数内部调用它们。

```tsx filename="app/page.tsx" switcher
import ClientComponent from './ClientComponent'

export default async function Page() {
  const performUpdate = async () => {
    'use server'
    // Perform some server-side update
    await db.update(...)
  }

  return <CachedComponent performUpdate={performUpdate} />
}

async function CachedComponent({
  performUpdate,
}: {
  performUpdate: () => Promise<void>
}) {
  'use cache'
  // Do not call performUpdate here
  return <ClientComponent action={performUpdate} />
}
```

```jsx filename="app/page.js" switcher
import ClientComponent from './ClientComponent'

export default async function Page() {

    // Perform some server-side update

}

async function CachedComponent({ performUpdate }) {
  'use cache'
  // Do not call performUpdate here
  return <ClientComponent action={performUpdate} />
}
```

```tsx filename="app/ClientComponent.tsx" switcher
'use client'

export default function ClientComponent({

  return <button onClick={action}>Update</button>
```


## 故障排除

### 调试缓存行为

#### 详细日志

设置 `NEXT_PRIVATE_DEBUG_CACHE=1` 以获取详细的缓存日志记录：

```bash
NEXT_PRIVATE_DEBUG_CACHE=1 npm run dev
# or for production
NEXT_PRIVATE_DEBUG_CACHE=1 npm run start
```

> **须知：** 此环境变量也会记录 ISR 和其他缓存机制。更多详情请参阅[验证正确的生产行为](/docs/app/guides/incremental-static-regeneration#verifying-correct-production-behavior)。

#### 控制台日志回放

在开发环境中，来自缓存函数的控制台日志会带有 `Cache` 前缀。

### 构建挂起 (缓存超时)

如果您的构建挂起，那是因为您正在访问在 `use cache` 边界外部创建的、解析为未缓存或运行时数据的 Promise。缓存函数等待无法在构建期间解析的数据，导致 50 秒后超时。

构建超时时，您将看到以下错误消息：

> Error: Filling a cache during prerender timed out, likely because request-specific arguments such as params, searchParams, cookies() or uncached data were used inside "use cache".

常见情况包括：将此类 Promise 作为 props 传递、通过闭包访问它们，或从共享存储（Maps）中检索它们。

> **须知：** 在 `use cache` 内部直接调用 `cookies()` 或 `headers()` 会立即失败，并显示[不同的错误](/docs/messages/next-request-in-use-cache)，而不是超时。

**将运行时数据 Promise 作为 props 传递：**

```tsx filename="app/page.tsx"
import { cookies } from 'next/headers'
import { Suspense } from 'react'

export default function Page() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Dynamic />
    </Suspense>
  )
}

async function Dynamic() {
  const cookieStore = cookies()
  return <Cached promise={cookieStore} /> // Build hangs
}

async function Cached({ promise }: { promise: Promise<unknown> }) {
  'use cache'
  const data = await promise // Waits for runtime data during build
  return <p>..</p>
}
```

在 `Dynamic` 组件中等待 `cookies` 存储，并将 cookie 值传递给 `Cached` 组件。

**共享去重存储：**

```tsx filename="app/page.tsx"
// Problem: Map stores dynamic Promises, accessed by cached code
import { Suspense } from 'react'

const cache = new Map<string, Promise<string>>()

export default function Page() {
  return (
    <>
      <Suspense fallback={<div>Loading...</div>}>
        <Dynamic id="data" />
      </Suspense>
      <Cached id="data" />
    </>
  )
}

async function Dynamic({ id }: { id: string }) {
  // Stores dynamic Promise in shared Map
  cache.set(
    id,
    fetch(`https://api.example.com/${id}`).then((r) => r.text())
  )
  return <p>Dynamic</p>
}

async function Cached({ id }: { id: string }) {
  'use cache'
  return <p>{await cache.get(id)}</p> // Build hangs - retrieves dynamic Promise
}
```

使用 Next.js 内置的 `fetch()` 去重，或者为缓存和未缓存的上下文使用单独的 Maps。
