`'use cache: private'` 指令允许函数在缓存作用域内访问运行时请求 API，如 `cookies()`、`headers()` 和 `searchParams`。但是，结果**永远不会存储在服务器上**，它们仅在浏览器内存中缓存，且不会在页面重新加载后保留。

在以下情况下请使用 `'use cache: private'`：

- 你希望缓存一个已访问运行时数据的函数，并且重构以[将运行时访问移到外部并将值作为参数传递](/docs/app/getting-started/caching#working-with-runtime-apis)不切实际。
- 合规要求禁止在服务器上存储某些数据，即使是暂时的。

由于此指令访问运行时数据，函数会在每次服务器渲染时执行，并且在[静态外壳](/docs/app/getting-started/caching#how-rendering-works)生成期间被排除运行。

**无法**为 `'use cache: private'` 配置自定义缓存处理程序。

有关不同缓存指令的比较，请参阅 [`use cache: remote` 与 `use cache` 和 `use cache: private` 的区别](/docs/app/api-reference/directives/use-cache-remote#how-use-cache-remote-differs-from-use-cache-and-use-cache-private)。

> **请注意**：此指令标记为 `experimental`，因为它依赖于运行时预取，该功能尚未稳定。运行时预取是一项即将推出的功能，它将使路由器能够预取到[静态外壳](/docs/app/getting-started/caching#how-rendering-works)之外的**任何**缓存作用域，而不仅限于私有缓存。

## 使用方法

要使用 `'use cache: private'`，请在你的 `next.config.ts` 文件中启用 [`cacheComponents`](/docs/app/api-reference/config/next-config-js/cacheComponents) 标志：

```tsx filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  cacheComponents: true,
}

export default nextConfig
```

```jsx filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  cacheComponents: true,
}

export default nextConfig
```

然后将 `'use cache: private'` 添加到你的函数中，并配置 `cacheLife`。

> **请注意**：此指令在路由处理程序中不可用。

### 基本示例

在此示例中，我们演示了你可以在 `'use cache: private'` 作用域内访问 cookies：

```tsx filename="app/product/[id]/page.tsx" switcher
import { Suspense } from 'react'
import { cookies } from 'next/headers'
import { cacheLife, cacheTag } from 'next/cache'

export async function generateStaticParams() {
  return [{ id: '1' }]
}

export default async function ProductPage({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params

  return (
    <div>
      <ProductDetails id={id} />
      <Suspense fallback={<div>Loading recommendations...</div>}>
        <Recommendations productId={id} />
      </Suspense>
    </div>
  )
}

async function Recommendations({ productId }: { productId: string }) {
  const recommendations = await getRecommendations(productId)

  return (
    <div>
      {recommendations.map((rec) => (
        <ProductCard key={rec.id} product={rec} />
      ))}
    </div>
  )
}

async function getRecommendations(productId: string) {
  'use cache: private'
  cacheTag(`recommendations-${productId}`)
  cacheLife({ stale: 60 })

  // Access cookies within private cache functions
  const sessionId = (await cookies()).get('session-id')?.value || 'guest'

  return getPersonalizedRecommendations(productId, sessionId)
}
```

> **请注意**：`stale` 时间必须至少为 30 秒，运行时预取才能正常工作。详情请参阅 [`cacheLife` 客户端缓存行为](/docs/app/api-reference/functions/cacheLife#client-cache-behavior)。

## 允许在私有缓存中使用的请求 API

以下特定于请求的 API 可在 `'use cache: private'` 函数内部使用：

| API            | 在 `use cache` 中是否允许 | 在 `'use cache: private'` 中是否允许 |
| -------------- | ---------------------- | --------------------------------- |
| `cookies()`    | 否                     | 是                               |
| `headers()`    | 否                     | 是                               |
| `searchParams` | 否                     | 是                               |
| `connection()` | 否                     | 否                                |

> **注意**：[`connection()`](https://nextjs.org/docs/app/api-reference/functions/connection) API 在 `use cache` 和 `'use cache: private'` 中均被禁止使用，因为它提供无法安全缓存的特定于连接的信息。
