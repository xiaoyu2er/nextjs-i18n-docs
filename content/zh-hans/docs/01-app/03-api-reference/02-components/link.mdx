`<Link>` 是一个 React 组件，它扩展了 HTML `<a>` 元素，提供了路由间的[预取 (prefetching)](/docs/app/getting-started/linking-and-navigating#prefetching) 和客户端导航功能。它是在 Next.js 中进行路由间导航的主要方式。

基本用法：

<AppOnly>

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

</AppOnly>

<PagesOnly>

```tsx filename="pages/index.tsx" switcher

  return <Link href="/dashboard">Dashboard</Link>
}
```

```jsx filename="pages/index.js" switcher

```

</PagesOnly>

### `scroll`

**默认为 `true`。** Next.js 中 `<Link>` 的默认滚动行为**是保持滚动位置**，类似于浏览器处理前进和后退导航的方式。当你导航到新的[页面 (Page)](/docs/app/api-reference/file-conventions/page) 时，只要该页面在视口中可见，滚动位置就会保持不变。但是，如果该页面在视口中不可见，Next.js 将滚动到第一个页面元素的顶部。

当 `scroll = {false}` 时，Next.js 将不会尝试滚动到第一个页面元素。

> **请注意**：Next.js 在管理滚动行为之前会检查 `scroll: false`。如果启用了滚动，它会识别导航的相关 DOM 节点并检查每个顶级元素。所有不可滚动的元素和那些没有渲染 HTML 的元素都会被跳过，这包括 sticky 或 fixed 定位的元素，以及像 `getBoundingClientRect` 计算的那样不可见的元素。然后 Next.js 会继续检查同级元素，直到找到一个在视口中可见的可滚动元素。

<AppOnly>

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" scroll={false}>
      Dashboard
    </Link>
  )
}
```


### `prefetch`

<AppOnly>

当 `<Link />` 组件进入用户视口（初始加载或通过滚动）时，会发生预取。Next.js 会预取并加载链接的路由（由 `href` 表示）及其数据到后台，以提升客户端导航的性能。如果预取的数据在用户将鼠标悬停在 `<Link />` 上时已过期，Next.js 将尝试再次预取它。**预取功能仅在生产环境中启用**。

可以将以下值传递给 `prefetch` 属性：

- **`"auto"` 或 `null` (默认)**：预取行为取决于路由是静态的还是动态的。对于静态路由，将预取完整路由（包括所有其数据）。对于动态路由，将预取部分路由，直到最近的带有 [`loading.js`](/docs/app/api-reference/file-conventions/loading#instant-loading-states) 边界的分段。
- `true`：将为静态和动态路由预取完整路由。
- `false`：预取永远不会在进入视口或悬停时发生。

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</AppOnly>

<PagesOnly>

当 `<Link />` 组件进入用户视口（初始加载或通过滚动）时，会发生预取。Next.js 会预取并加载链接的路由（由 `href` 表示）及其数据到后台，以提升客户端导航的性能。**预取功能仅在生产环境中启用**。

可以将以下值传递给 `prefetch` 属性：

- **`true`（默认值）**：将预取整个路由及其数据。
- `false`：在进入视口时不会发生预取，但在鼠标悬停时会发生预取。如果你想完全禁用鼠标悬停时的预取，可以考虑使用 `<a>` 标签或[渐进式采用](/docs/app/guides/migrating/app-router-migration)应用路由 (App Router)，它也支持禁用鼠标悬停时的预取。

### `locale`

活动的语言环境会自动添加在前面。`locale` 允许提供一个不同的语言环境。当设为 `false` 时，`href` 必须包含语言环境，因为默认行为被禁用了。

```tsx filename="pages/index.tsx" switcher
import Link from 'next/link'

export default function Home() {
  return (
    <>
      {/* Default behavior: locale is prepended */}
      <Link href="/dashboard">Dashboard (with locale)</Link>

      {/* Disable locale prepending */}
      <Link href="/dashboard" locale={false}>
        Dashboard (without locale)
      </Link>

      {/* Specify a different locale */}
      <Link href="/dashboard" locale="fr">
        Dashboard (French)
      </Link>
    </>
  )
}
```

```jsx filename="pages/index.js" switcher

export default function Home() {
```

### `as`

路径的可选修饰符，将在浏览器 URL 栏中显示。在 Next.js 9.5.3 之前，这用于动态路由，请查看我们[之前的文档](https://github.com/vercel/next.js/blob/v9.5.2/docs/api-reference/next/link.md#dynamic-routes)了解其工作方式。

当此路径与 `href` 中提供的路径不同时，将使用之前的 `href`/`as` 行为，如[之前的文档](https://github.com/vercel/next.js/blob/v9.5.2/docs/api-reference/next/link.md#dynamic-routes)所示。

</PagesOnly>

### `onNavigate`

在客户端导航期间调用的事件处理程序。该处理程序接收一个事件对象，其中包含一个 `preventDefault()` 方法，允许你在需要时取消导航。

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link
      href="/dashboard"
      onNavigate={(e) => {
        // Only executes during SPA navigation
        console.log('Navigating...')

        // Optionally prevent navigation
        // e.preventDefault()
      }}
    >
      Dashboard
    </Link>
  )
}
```

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link
      href="/dashboard"
      onNavigate={(e) => {
        // Only executes during SPA navigation
        console.log('Navigating...')

        // Optionally prevent navigation
        // e.preventDefault()
      }}
    >
      Dashboard
    </Link>
  )
}
```

> **请注意**：虽然 `onClick` 和 `onNavigate` 看起来相似，但它们的用途不同。`onClick` 对所有点击事件执行，而 `onNavigate` 仅在客户端导航期间运行。一些关键区别如下：
>
> - 当使用修饰键（`Ctrl`/`Cmd` + 点击）时，`onClick` 会执行但 `onNavigate` 不会，因为 Next.js 会阻止新标签页的默认导航行为。
> - 外部 URL 不会触发 `onNavigate`，因为它仅用于客户端和同源导航。
> - 带有 `download` 属性的链接会与 `onClick` 一起工作，但不会与 `onNavigate` 一起工作，因为浏览器会将链接的 URL 视为下载。

### `transitionTypes`

<AppOnly>

### Checking active links

你可以使用 [`usePathname()`](/docs/app/api-reference/functions/use-pathname) 来确定一个链接是否是活动的。例如，要向活动链接添加一个类，你可以检查当前的 `pathname` 是否与链接的 `href` 匹配：

```tsx filename="app/ui/nav-links.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'
import Link from 'next/link'

export function Links() {
  const pathname = usePathname()

  return (
    <nav>
      <Link className={`link ${pathname === '/' ? 'active' : ''}`} href="/">
        Home
      </Link>

      <Link
        className={`link ${pathname === '/about' ? 'active' : ''}`}
        href="/about"
      >
        About
      </Link>
    </nav>
  )
}
```

### Scrolling to an `id`

如果你想在导航时滚动到特定的 `id`，你可以在 URL 后附加 `#` 哈希链接，或者直接将哈希链接传递给 `href` 属性。这是可行的，因为 `<Link>` 会渲染为 `<a>` 元素。

```jsx
<Link href="/dashboard#settings">Settings</Link>

// Output
<a href="/dashboard#settings">Settings</a>
```

### Disable scrolling to the top of the page

<AppOnly>

Next.js 中 `<Link>` 的默认滚动行为**是保持滚动位置**，类似于浏览器处理前进和后退导航的方式。当你导航到新的[页面 (Page)](/docs/app/api-reference/file-conventions/page) 时，只要该页面在视口中可见，滚动位置就会保持不变。

但是，如果该页面在视口中不可见，Next.js 将滚动到第一个页面元素的顶部。如果你想禁用此行为，可以将 `scroll={false}` 传递给 `<Link>` 组件，或者将 `scroll: false` 传递给 `router.push()` 或 `router.replace()`。

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/#hashid" scroll={false}>
      Disables scrolling to the top
    </Link>
  )
}
```

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/#hashid" scroll={false}>
      Disables scrolling to the top
    </Link>
  )
}
```

使用 `router.push()` 或 `router.replace()`：

```jsx
// useRouter
import { useRouter } from 'next/navigation'

const router = useRouter()

router.push('/dashboard', { scroll: false })
```

</AppOnly>

<PagesOnly>

### 带有粘性头部时的滚动偏移

因为 Next.js 在寻找滚动目标时会跳过 sticky 和 fixed 定位的元素，内容在导航后可能会出现在粘性头部的后面。例如，如果你的布局有一个粘性头部：

```tsx filename="app/layout.tsx" switcher
import './globals.css'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        <header className="sticky top-0 h-16 bg-white">
          {/* Navigation */}
        </header>
        {children}
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import './globals.css'

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <header className="sticky top-0 h-16 bg-white">
          {/* Navigation */}
        </header>
        {children}
      </body>
    </html>
  )
}
```

你可以使用滚动容器上的 [`scroll-padding-top`](https://developer.mozilla.org/en-US/docs/Web/CSS/scroll-padding-top) 来考虑其高度：

```css filename="app/globals.css"
html {
  scroll-padding-top: 64px; /* Match the height of your sticky header */
}
```

这是一个浏览器 CSS 属性，用于偏移基于滚动的定位。它在 Next.js 使用原生 [`scrollIntoView()`](https://developer.mozilla.org/en-US/docs/Web/API/Element/scrollIntoView) API 时都会应用，包括哈希片段（`#id`）导航。或者，你可以在单个目标元素上使用 [`scroll-margin-top`](https://developer.mozilla.org/en-US/docs/Web/CSS/scroll-margin-top)，而不是设置全局偏移。

### 代理 (Proxy) 中的链接预取

使用 [代理](/docs/app/api-reference/file-conventions/proxy) 进行身份验证或将用户重写到不同页面是很常见的。为了让 `<Link />` 组件能正确预取通过代理重写的链接，你需要同时告诉 Next.js 要显示的 URL 和要预取的 URL。这是为了避免向代理发出不必要的 fetch 来知道要预取的正确路由。

例如，如果你想提供一个包含已验证用户视图和访客视图的 `/dashboard` 路由，你可以在你的代理中添加以下内容将用户重定向到正确的页面：

```ts filename="proxy.ts" switcher
import { NextResponse } from 'next/server'

export function proxy(request: Request) {
  const nextUrl = request.nextUrl
  if (nextUrl.pathname === '/dashboard') {
    if (request.cookies.authToken) {
      return NextResponse.rewrite(new URL('/auth/dashboard', request.url))
    } else {
      return NextResponse.rewrite(new URL('/public/dashboard', request.url))
    }
  }
}
```


### Blocking navigation

你可以使用 `onNavigate` 属性在满足某些条件时阻止导航，例如当表单有未保存的更改时。当你需要在应用中的多个组件中阻止导航时（例如防止在编辑表单时从任何链接导航），React Context 提供了一种干净的方式来共享这种阻止状态。首先，创建一个上下文来跟踪导航阻止状态：

```tsx filename="app/contexts/navigation-blocker.tsx" switcher
'use client'

import { createContext, useState, useContext } from 'react'

interface NavigationBlockerContextType {
  isBlocked: boolean
  setIsBlocked: (isBlocked: boolean) => void
}

export const NavigationBlockerContext =
  createContext<NavigationBlockerContextType>({
    isBlocked: false,
    setIsBlocked: () => {},
  })

export function NavigationBlockerProvider({
  children,
}: {
  children: React.ReactNode
}) {
  const [isBlocked, setIsBlocked] = useState(false)

  return (
    <NavigationBlockerContext.Provider value={{ isBlocked, setIsBlocked }}>
      {children}
    </NavigationBlockerContext.Provider>
  )
}

export function useNavigationBlocker() {
  return useContext(NavigationBlockerContext)
}
```

