Next.js 的 Image 组件扩展了 HTML 的 `<img>` 元素，用于自动进行图片优化。

```jsx filename="app/page.js"
import Image from 'next/image'

export default function Page() {
  return (
    <Image
      src="/profile.png"
      width={500}
      height={500}
      alt="Picture of the author"
    />
  )
}
```

<PagesOnly>

> **请注意**：如果您使用的是 13 之前的 Next.js 版本，请查阅 [next/legacy/image](/docs/pages/api-reference/components/image-legacy) 文档，因为该组件已被重命名。

</PagesOnly>

#### `src`

图片的源。可以是以下之一：

一个内部路径字符串。

```jsx
<Image src="/profile.png" />
```

一个绝对外部 URL（必须配置 [remotePatterns](#remotepatterns)）。

```jsx
<Image src="https://example.com/profile.png" />
```

一个静态导入。

```jsx
import profile from './profile.png'

export default function Page() {
  return <Image src={profile} />
}
```

> **需要注意**：出于安全原因，使用默认 [loader](#loader) 的图像优化 API 在获取 `src` 图像时 _不会_ 转发请求头。
> 如果 `src` 图像需要认证，请考虑使用 [unoptimized](#unoptimized) 属性来禁用图像优化。

#### `alt`

`alt` 属性用于为屏幕阅读器和搜索引擎描述图片。如果图片被禁用或加载图片时发生错误，它也是回退文本。

它应包含可以替换图片的文本，[且不改变页面的含义](https://html.spec.whatwg.org/multipage/images.html#general-guidelines)。它不是用来补充图片的，也不应重复图片上方或下方标题中已提供的信息。

如果图片是[纯装饰性的](https://html.spec.whatwg.org/multipage/images.html#a-purely-decorative-image-that-doesn't-add-any-information)或[不打算提供给用户](https://html.spec.whatwg.org/multipage/images.html#an-image-not-intended-for-the-user)，`alt` 属性应为空字符串 (`alt=""`)。

> 了解有关[图片无障碍指南](https://html.spec.whatwg.org/multipage/images.html#alt)的更多信息。

#### `width` and `height`

`width` 和 `height` 属性表示图片的[固有](https://developer.mozilla.org/en-US/docs/Glossary/Intrinsic_Size)尺寸（以像素为单位）。此属性用于推断浏览器使用的正确**宽高比**，以为图片预留空间并避免加载期间的布局偏移。它不决定图片的*渲染大小*，后者由 CSS 控制。

```jsx
<Image src="/profile.png" width={500} height={500} />
```

您**必须**同时设置 `width` 和 `height` 属性，除非：

- 图片是静态导入的。
- 图片具有 [`fill` 属性](#fill)。

如果高度和宽度未知，我们建议使用 [`fill` 属性](#fill)。

#### `fill`

一个布尔值，使图片扩展到父元素的大小。

```js
<Image src="/profile.png" fill={true} />
```

**定位**：

- 父元素**必须**设置 `position: "relative"`、`"fixed"` 或 `"absolute"`。
- 默认情况下，`<img>` 元素使用 `position: "absolute"`。

**对象适应**：

如果没有对图片应用样式，图片将拉伸以适应容器。您可以使用 `objectFit` 来控制裁剪和缩放。

- `"contain"`：图片将按比例缩小以适应容器并保持宽高比。
- `"cover"`：图片将填充容器并被裁剪。

> 了解有关 [`position`](https://developer.mozilla.org/en-US/docs/Web/CSS/position) 和 [`object-fit`](https://developer.mozilla.org/docs/Web/CSS/object-fit) 的更多信息。

#### `loader`

一个自定义函数，用于生成图片 URL。该函数接收以下参数，并返回图片的 URL 字符串：

- [`src`](#src)
- [`width`](#width-and-height)
- [`quality`](#quality)

<AppOnly>

```js
'use client'

import Image from 'next/image'

const imageLoader = ({ src, width, quality }) => {
  return `https://example.com/${src}?w=${width}&q=${quality || 75}`
}

export default function Page() {
  return (
    <Image
      loader={imageLoader}
      src="me.png"
      alt="Picture of the author"
      width={500}
      height={500}
    />
  )
}
```

> **请注意**：使用像 `onLoad` 这样接受函数的属性，需要使用[客户端组件](https://react.dev/reference/rsc/use-client)来序列化提供的函数。

</AppOnly>

<PagesOnly>

```js
import Image from 'next/image'

const imageLoader = ({ src, width, quality }) => {
  return `https://example.com/${src}?w=${width}&q=${quality || 75}`
}

export default function Page() {
  return (
    <Image
      loader={imageLoader}
      src="me.png"
      alt="Picture of the author"
      width={500}
      height={500}
    />
  )
}
```

</PagesOnly>

或者，您可以使用 `next.config.js` 中的 [loaderFile](#loaderfile) 配置来配置应用程序中每个 `next/image` 实例，而无需传递属性。

#### `sizes`

定义不同断点下图片的大小。供浏览器从生成的 `srcset` 中选择最合适的尺寸。

```jsx
import Image from 'next/image'

export default function Page() {
  return (
    <div className="grid-element">
      <Image
        fill
        src="/example.png"
        sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
      />
    </div>
  )
}
```

`sizes` 应在以下情况使用：

- 图片使用了 [`fill`](#fill) 属性
- 使用了 CSS 使图片响应式

如果缺少 `sizes`，浏览器会假定图片将与视口一样宽 (`100vw`)。这可能导致下载不必要的大尺寸图片。

此外，`sizes` 会影响 `srcset` 的生成方式：

- 没有 `sizes`：Next.js 生成有限的 `srcset`（例如 1x, 2x），适用于固定尺寸的图片。
- 有 `sizes`：Next.js 生成完整的 `srcset`（例如 640w, 750w 等），针对响应式布局进行了优化。

> 在 [web.dev](https://web.dev/learn/design/responsive-images/#sizes) 和 [mdn](https://developer.mozilla.org/docs/Web/HTML/Element/img#sizes) 上了解有关 `srcset` 和 `sizes` 的更多信息。

#### `quality`

一个介于 `1` 和 `100` 之间的整数，用于设置优化后图片的质量。较高的值会增加文件大小和视觉保真度。较低的值会减小文件大小，但可能会影响清晰度。

```jsx
// Default quality is 75
<Image quality={75} />
```

如果您已在 `next.config.js` 中配置了 [qualities](#qualities)，则该值必须匹配允许的条目之一。

> **请注意**：如果原始图片质量已经很低，设置高质量值将增加文件大小，而不会改善外观。

#### `style`

允许将 CSS 样式传递给底层的图片元素。

```jsx
const imageStyle = {
  borderRadius: '50%',
  border: '1px solid #fff',
  width: '100px',
  height: 'auto',
}

export default function ProfileImage() {
  return <Image src="..." style={imageStyle} />
}
```

> **请注意**：如果您使用 `style` 属性设置自定义宽度，请务必同时设置 `height: 'auto'` 以保持图片的宽高比。

#### `preload`

一个布尔值，指示是否应预加载图片。

```jsx
// Default preload is false
<Image preload={false} />
```

- `true`：通过在 `<head>` 中插入一个 `<link>` [预加载](https://web.dev/preload-responsive-images/)图像。
- `false`：不预加载图像。

**何时使用：**

- 图像是[最大内容绘制 (LCP)](https://nextjs.org/learn/seo/web-performance/lcp) 元素。
- 图像在首屏上方，通常是主视觉图像。
- 您希望在 `<head>` 中开始加载图像，而不是稍后在 `<body>` 中发现。

**何时不使用：**

- 当您有多个图像可能被视为 [最大内容绘制 (LCP)](https://nextjs.org/learn/seo/web-performance/lcp) 元素，具体取决于视口。
- 当使用了 `loading` 属性时。
- 当使用了 `fetchPriority` 属性时。

在大多数情况下，您应该使用 `loading="eager"` 或 `fetchPriority="high"` 而不是 `preload`。

#### `priority`

从 Next.js 16 开始，`priority` 属性已被弃用，建议使用 [`preload`](#preload) 属性，以使行为更清晰。

#### `loading`

控制图片何时开始加载。

```jsx
// Defaults to lazy
<Image loading="lazy" />
```

- `lazy`：延迟加载图片，直到它距离视口达到计算的距离。
- `eager`：立即加载图片，无论它在页面中的位置如何。

仅当您想确保图片立即加载时才使用 `eager`。

> 了解有关 [`loading` 属性](https://developer.mozilla.org/docs/Web/HTML/Element/img#loading)的更多信息。

#### `placeholder`

指定图片加载时要使用的占位符，以提高感知加载性能。

```jsx
// defaults to empty
<Image placeholder="empty" />
```

- `empty`：图片加载时没有占位符。
- `blur`：使用图片的模糊版本作为占位符。必须与 [`blurDataURL`](#blurdataurl) 属性一起使用。
- `data:image/...`：使用 [Data URL](https://developer.mozilla.org/docs/Web/HTTP/Basics_of_HTTP/Data_URIs) 作为占位符。

**示例：**

- [`blur` 占位符](https://image-component.nextjs.gallery/placeholder)
- [使用 `blurDataURL` 属性的闪烁效果](https://image-component.nextjs.gallery/shimmer)
- [使用 `blurDataURL` 属性的颜色效果](https://image-component.nextjs.gallery/color)

> 了解有关 [`placeholder` 属性](https://developer.mozilla.org/docs/Web/HTML/Element/img#placeholder)的更多信息。

#### `blurDataURL`

一个 [Data URL](https://developer.mozilla.org/docs/Web/HTTP/Basics_of_HTTP/Data_URIs)，在图片成功加载之前用作占位图片。可以自动设置或与 [`placeholder="blur"`](#placeholder) 属性一起使用。

```jsx
<Image placeholder="blur" blurDataURL="..." />
```

图片会自动放大和模糊，因此建议使用非常小的图片（10 像素或更小）。

**自动**

如果 `src` 是 `jpg`、`png`、`webp` 或 `avif` 文件的静态导入，`blurDataURL` 会自动添加——除非图片是动画图片。

**手动设置**

如果图片是动态的或远程的，您必须自己提供 `blurDataURL`。要生成一个，您可以使用：

- [像 png-pixel.com 这样的在线工具](https://png-pixel.com)
- [像 Plaiceholder 这样的库](https://github.com/joe-bell/plaiceholder)

较大的 `blurDataURL` 可能会损害性能。请保持其小而简单。

**示例：**

- [默认 `blurDataURL` 属性](https://image-component.nextjs.gallery/placeholder)
- [使用 `blurDataURL` 属性的颜色效果](https://image-component.nextjs.gallery/color)

#### `onLoad`

当图片完全加载且[占位符](#placeholder)已被移除时调用的回调函数。

```jsx
<Image onLoad={(e) => console.log(e.target.naturalWidth)} />
```

回调函数将被调用，并带有一个参数，该事件包含引用底层 `<img>` 元素的 `target`。

<AppOnly>

> **请注意**：使用像 `onLoad` 这样接受函数的属性，需要使用[客户端组件](https://react.dev/reference/rsc/use-client)来序列化提供的函数。

</AppOnly>

#### `onError`

如果图片加载失败，则调用的回调函数。

```jsx
<Image onError={(e) => console.error(e.target.id)} />
```

<AppOnly>

> **请注意**：使用像 `onError` 这样接受函数的属性，需要使用[客户端组件](https://react.dev/reference/rsc/use-client)来序列化提供的函数。

</AppOnly>

#### `unoptimized`

一个布尔值，指示是否应优化图片。这对于不需要优化的图片很有用，例如小图片（小于 1KB）、矢量图片 (SVG) 或动画图片 (GIF)。

```js
import Image from 'next/image'

const UnoptimizedImage = (props) => {
  // Default is false
  return <Image {...props} unoptimized />
}
```

- `true`：源图片将按原样从 `src` 提供，而不更改质量、大小或格式。
- `false`：源图片将被优化。

自 Next.js 12.3.0 起，可以通过更新 `next.config.js` 以下配置将此属性分配给所有图片：

```js filename="next.config.js"
module.exports = {
  images: {
    unoptimized: true,
  },
}
```

#### `overrideSrc`

当向 `<Image>` 组件提供 `src` 属性时，生成的 `<img>` 会自动同时生成 `srcset` 和 `src` 属性。

```jsx filename="input.js"
<Image src="/profile.jpg" />
```

```html filename="output.html"
<img
  srcset="
    /_next/image?url=%2Fprofile.jpg&w=640&q=75 1x,
    /_next/image?url=%2Fprofile.jpg&w=828&q=75 2x
  "
  src="/_next/image?url=%2Fprofile.jpg&w=828&q=75"
/>
```

在某些情况下，不希望生成 `src` 属性，您可能希望使用 `overrideSrc` 属性覆盖它。

例如，将现有网站从 `<img>` 升级到 `<Image>` 时，您可能希望出于 SEO 目的（如图片排名或避免重新抓取）保持相同的 `src` 属性。

```jsx filename="input.js"
<Image src="/profile.jpg" overrideSrc="/override.jpg" />
```

```html filename="output.html"
<img
  srcset="
    /_next/image?url=%2Fprofile.jpg&w=640&q=75 1x,
    /_next/image?url=%2Fprofile.jpg&w=828&q=75 2x
  "
  src="/override.jpg"
/>
```

#### `decoding`

给浏览器的一个提示，指示它是否应该在呈现其他内容更新之前等待图片解码完成。

```jsx
// Default is async
<Image decoding="async" />
```

- `async`：异步解码图片，并允许在解码完成之前渲染其他内容。
- `sync`：同步解码图片，以便与其他内容原子呈现。
- `auto`：无偏好。浏览器选择最佳方式。

> 了解有关 [`decoding` 属性](https://developer.mozilla.org/docs/Web/HTML/Element/img#decoding)的更多信息。

## 配置选项

您可以在 `next.config.js` 中配置 Image 组件。以下选项可用：

#### `localPatterns`

在您的 `next.config.js` 文件中使用 `localPatterns` 来允许优化来自特定本地路径的图片并阻止所有其他路径。

```js filename="next.config.js"
module.exports = {
  images: {
    localPatterns: [
      {
        pathname: '/assets/images/**',
        search: '',
      },
    ],
  },
}
```

上面的示例将确保 `next/image` 的 `src` 属性必须以 `/assets/images/` 开头，并且不能有查询字符串。尝试优化任何其他路径将响应 `400` Bad Request 错误。

#### `remotePatterns`

#### `remotePatterns`

在您的 `next.config.js` 文件中使用 `remotePatterns` 来允许来自特定外部路径的图片并阻止所有其他路径。这确保只有来自您账户的外部图片可以被提供。

```js filename="next.config.js"
module.exports = {
  images: {
    remotePatterns: [new URL('https://example.com/account123/**')],
  },
}
```

上面的示例将确保 `next/image` 的 `src` 属性必须以 `https://example.com/account123/` 开头，并且不能有查询字符串。任何其他协议、主机名、端口或不匹配的路径将响应 `400` Bad Request。

```js filename="next.config.js"
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'example.com',
        port: '',
        pathname: '/account123/**',
        search: '',
      },
    ],
  },
}
```

上面的示例将确保 `next/image` 的 `src` 属性必须以 `https://example.com/account123/` 开头，并且不能有查询字符串。任何其他协议、主机名、端口或不匹配的路径将响应 `400` Bad Request。

**通配符模式：**

通配符模式可用于 `pathname` 和 `hostname`，并具有以下语法：

- `*` 匹配单个路径段或子域名
- `**` 匹配路径末尾任意数量的路径段或开头的子域名。此语法不能在模式中间使用。

```js filename="next.config.js"
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: '**.example.com',
        port: '',
        search: '',
      },
    ],
  },
}
```

这允许像 `image.example.com` 这样的子域名。查询字符串和自定义端口仍然被阻止。

> **请注意**：当省略 `protocol`、`port`、`pathname` 或 `search` 时，会隐含通配符 `**`。不建议这样做，因为它可能允许恶意行为者优化您未打算优化的 URL。

**查询字符串**：

您还可以使用 `search` 属性限制查询字符串：

```js filename="next.config.js"
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'assets.example.com',
        search: '?v=1727111025337',
      },
    ],
  },
}
```

上面的示例将确保 `next/image` 的 `src` 属性必须以 `https://assets.example.com` 开头，并且必须具有确切的查询字符串 `?v=1727111025337`。任何其他协议或查询字符串将响应 `400` Bad Request。

#### `loaderFile`

#### `loaderFile`

`loaderFiles` 允许您使用自定义的图片优化服务代替 Next.js。

```js filename="next.config.js"
module.exports = {
  images: {
    loader: 'custom',
    loaderFile: './my/image/loader.js',
  },
}
```

路径必须相对于项目根目录。该文件必须导出一个返回 URL 字符串的默认函数：

<AppOnly>

#### `formats`

#### `formats`

`formats` 允许您指定要使用的图片格式列表。

```js filename="next.config.js"
module.exports = {
  images: {
    // Default
    formats: ['image/webp'],
  },
}
```

Next.js 通过请求的 `Accept` 头自动检测浏览器支持的图片格式，以确定最佳的输出格式。

如果 `Accept` 头匹配多个配置的格式，则使用数组中的第一个匹配项。因此，数组顺序很重要。如果没有匹配（或源图片是动画图片），它将使用原始图片的格式。

您可以启用 AVIF 支持，如果浏览器[不支持 AVIF](https://caniuse.com/avif)，它将回退到 src 图片的原始格式：

```js filename="next.config.js"
module.exports = {
  images: {
    formats: ['image/avif'],
  },
}
```

> **需要注意**：
>
> - 我们仍然建议在大多数用例中使用 WebP。
> - AVIF 通常编码时间长 50%，但比 WebP 压缩小 20%。这意味着图像第一次被请求时通常会更慢，但后续缓存的请求会更快。
> - 使用多种格式时，Next.js 会分别缓存每种格式。这意味着与使用单一格式相比，存储需求增加，因为图像的 AVIF 和 WebP 版本都将针对不同的浏览器支持进行存储。
> - 如果您在 Next.js 前面使用代理/CDN 自托管，则必须配置代理以转发 `Accept` 头。

```js filename="next.config.js"
module.exports = {
  images: {
    formats: ['image/avif', 'image/webp'],
  },
}
```

#### `minimumCacheTTL`

`minimumCacheTTL` 允许您配置缓存优化图片的生存时间（TTL）（以秒为单位）。在许多情况下，最好使用[静态图片导入](/docs/app/getting-started/images#local-images)，它会自动对文件内容进行哈希处理，并使用 `Cache-Control` 头的 `immutable` 永久缓存图片。

如果未提供配置，则使用以下默认值。

```js filename="next.config.js"
module.exports = {

    minimumCacheTTL: 14400, // 4 hours
  },
}
```

优化图片的过期时间（或更确切地说是最大有效期）由 `minimumCacheTTL` 或上游图片的 `Cache-Control` 头定义，以较大者为准。

```js filename="next.config.js"
module.exports = {
  images: {
    minimumCacheTTL: 2678400, // 31 days
  },
}
```

优化图片的过期时间（或更确切地说是最大有效期）由 `minimumCacheTTL` 或上游图片的 `Cache-Control` 头定义，以较大者为准。

如果您需要针对每张图片更改缓存行为，可以配置 [`headers`](/docs/app/api-reference/config/next-config-js/headers) 来设置上游图片（例如 `/some-asset.jpg`，而不是 `/_next/image` 本身）的 `Cache-Control` 头。

目前没有使缓存失效的机制，因此最好将 `minimumCacheTTL` 保持在较低水平。否则，您可能需要手动更改 [`src`](#src) 属性或删除缓存的文件 `<distDir>/cache/images`。

#### `dangerouslyAllowSVG`

`dangerouslyAllowSVG` 允许您提供 SVG 图片。

```js filename="next.config.js"
module.exports = {
  images: {
    dangerouslyAllowSVG: true,
  },
}
```

默认情况下，Next.js 不优化 SVG 图片，原因如下：

- SVG 是一种矢量格式，这意味着它可以无损调整大小。
- SVG 具有许多与 HTML/CSS 相同的功能，如果没有适当的[内容安全策略 (CSP) 头](/docs/app/api-reference/config/next-config-js/headers#content-security-policy)，可能会导致漏洞。

当已知 [`src`](#src) 属性是 SVG 时，我们建议使用 [`unoptimized`](#unoptimized) 属性。当 `src` 以 `".svg"` 结尾时，这会自动发生。

```jsx
<Image src="/my-image.svg" unoptimized />
```

此外，强烈建议同时设置 `contentDispositionType` 以强制浏览器下载图片，以及设置 `contentSecurityPolicy` 以防止嵌入在图片中的脚本执行。

```js filename="next.config.js"
module.exports = {
  images: {
    dangerouslyAllowSVG: true,
    contentDispositionType: 'attachment',
    contentSecurityPolicy: "default-src 'self'; script-src 'none'; sandbox;",
  },
}
```

`contentDispositionType` 允许您配置 [`Content-Disposition`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Disposition#as_a_response_header_for_the_main_body) 头。

```js filename="next.config.js"
module.exports = {
  images: {
    contentDispositionType: 'inline',
  },
```

#### `contentSecurityPolicy`

默认情况下，[loader](#loader) 将 [`Content-Disposition`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Disposition#as_a_response_header_for_the_main_body) 头设置为 `attachment` 以增加保护，因为该 API 可以提供任意的远程图片。

```js filename="next.config.js"
module.exports = {
  images: {
    contentSecurityPolicy: "default-src 'self'; script-src 'none'; sandbox;",
  },
}
```

默认情况下，[loader](#loader) 将 [`Content-Disposition`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Disposition#as_a_response_header_for_the_main_body) 头设置为 `attachment` 以增加保护，因为该 API 可以提供任意的远程图片。

默认值是 `attachment`，这会强制浏览器在直接访问时下载图片。当 [`dangerouslyAllowSVG`](#dangerouslyallowsvg) 为 true 时，这一点尤其重要。

您可以选择配置 `inline` 以允许浏览器在直接访问时渲染图片，而无需下载。

### `getImageProps`

`getImageProps` 函数可用于获取将传递给底层 `<img>` 元素的属性，然后将它们传递给另一个组件、样式、画布等。

```jsx
import { getImageProps } from 'next/image'

const { props } = getImageProps({
  src: 'https://example.com/image.jpg',

  width: 1200,
  height: 800,
})

function ImageWithCaption() {
  return (
    <figure>
      <img {...props} />
      <figcaption>A scenic mountain view</figcaption>
    </figure>
  )

```

这也避免了调用 React `useState()`，因此可以带来更好的性能，但它不能与 [`placeholder`](#placeholder) 属性一起使用，因为占位符永远不会被移除。

这个 `next/image` 组件使用浏览器原生的[懒加载](https://caniuse.com/loading-lazy-attr)，对于 Safari 15.4 之前的旧浏览器，它可能会回退到立即加载。当使用模糊占位符时，Safari 12 之前的旧浏览器将回退到空占位符。当使用 `width`/`height` 为 `auto` 的样式时，可能会在不[保持宽高比](https://caniuse.com/mdn-html_elements_img_aspect_ratio_computed_from_attributes)的 Safari 15 之前的旧浏览器上导致[布局偏移](https://web.dev/cls/)。更多详情，请参阅[此 MDN 视频](https://www.youtube.com/watch?v=4-d_SoCHeWE)。

### 设置图片样式

设置 Image 组件的样式类似于设置普通 `<img>` 元素的样式，但有一些准则需要记住：

使用 `className` 或 `style`，而不是 `styled-jsx`。在大多数情况下，我们建议使用 `className` 属性。这可以是导入的 [CSS 模块](/docs/app/getting-started/css)、[全局样式表](/docs/app/getting-started/css#global-css) 等。

```jsx
import styles from './styles.module.css'

export default function MyImage() {
  return <Image className={styles.image} src="/my-image.png" alt="My Image" />
}
```

您也可以使用 `style` 属性来分配内联样式。

```jsx
export default function MyImage() {
  return (
    <Image style={{ borderRadius: '8px' }} src="/my-image.png" alt="My Image" />
  )
}
```

当使用 `fill` 时，父元素必须有 `position: relative` 或 `display: block`。这对于在该布局模式下正确渲染图片元素是必要的。

```jsx
<div style={{ position: 'relative' }}>

  <Image fill src="/my-image.png" alt="My Image" />

</div>

```

您不能使用 [styled-jsx](/docs/app/guides/css-in-js)，因为它仅限于当前组件（除非您将样式标记为 `global`）。

### 远程图片

要使用远程图片，`src` 属性应为 URL 字符串。

```jsx filename="app/page.js"
import Image from 'next/image'

export default function Page() {
  return (
    <Image
      src="https://s3.amazonaws.com/my-bucket/profile.png"
      alt="Picture of the author"
      width={500}
      height={500}
    />
  )
}
```

由于 Next.js 在构建过程中无法访问远程文件，您需要手动提供 [`width`](/docs/app/api-reference/components/image#width-and-height)、[`height`](/docs/app/api-reference/components/image#width-and-height) 和可选的 [`blurDataURL`](/docs/app/api-reference/components/image#blurdataurl) 属性。

`width` 和 `height` 属性用于推断图片的正确宽高比，并避免图片加载时的布局偏移。`width` 和 `height` *不*决定图片文件的渲染大小。

为了安全地允许优化图片，请在 `next.config.js` 中定义支持的 URL 模式列表。尽可能具体以防止恶意使用。例如，以下配置将只允许来自特定 AWS S3 存储桶的图片：

```js filename="next.config.js"
module.exports = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 's3.amazonaws.com',
        port: '',
        pathname: '/my-bucket/**',
        search: '',
      },
    ],
  },
}
```

### 艺术方向

如果您想为移动设备和桌面设备显示不同的图片，有时称为[艺术方向](https://developer.mozilla.org/en-US/docs/Learn/HTML/Multimedia_and_embedding/Responsive_images#art_direction)，您可以向 `getImageProps()` 提供不同的 `src`、`width`、`height` 和 `quality` 属性。

```jsx filename="app/page.js"
import { getImageProps } from 'next/image'

export default function Home() {
  const common = { alt: 'Art Direction Example', sizes: '100vw' }
  const {
    props: { srcSet: desktop },
  } = getImageProps({
    ...common,
    width: 1440,
    height: 875,
    quality: 80,
    src: '/desktop.jpg',
  })
  const {
    props: { srcSet: mobile, ...rest },
  } = getImageProps({
    ...common,
    width: 750,
    height: 1334,
    quality: 70,
    src: '/mobile.jpg',
  })

  return (
    <picture>
      <source media="(min-width: 1000px)" srcSet={desktop} />
      <source media="(min-width: 500px)" srcSet={mobile} />
      <img {...rest} style={{ width: '100%', height: 'auto' }} />
    </picture>
  )
}
```

### 背景 CSS

您甚至可以将 `srcSet` 字符串转换为 [`image-set()`](https://developer.mozilla.org/en-US/docs/Web/CSS/image/image-set) CSS 函数来优化背景图片。

```jsx filename="app/page.js"
import { getImageProps } from 'next/image'

function getBackgroundImage(srcSet = '') {
  const imageSet = srcSet
    .split(', ')
    .map((str) => {
      const [url, dpi] = str.split(' ')
      return `url("${url}") ${dpi}`
    })
    .join(', ')
  return `image-set(${imageSet})`
}

export default function Home() {
  const {
    props: { srcSet },
  } = getImageProps({ alt: '', width: 128, height: 128, src: '/img.png' })
  const backgroundImage = getBackgroundImage(srcSet)
  const style = { height: '100vh', width: '100vw', backgroundImage }

  return (
    <main style={style}>
      <h1>Hello World</h1>
    </main>
  )
}
```
