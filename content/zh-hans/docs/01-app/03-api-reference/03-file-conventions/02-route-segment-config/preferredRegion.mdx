---
title: 首选区域 (preferredRegion)
description: preferredRegion 路由段配置选项的 API 参考。
---

 `preferredRegion` 选项允许您为路由段指定首选部署区域。该值将传递给您的部署平台。

```tsx filename="layout.tsx | page.tsx | route.ts" switcher
export const preferredRegion = // string || string[]
```

```js filename="layout.js | page.js | route.js" switcher
export const preferredRegion = // string || string[]
```

- **`string`**: 将路由部署到特定区域。可用的区域代码依平台而异。例如，`'iad1'`。
- **`string[]`**: 将路由部署到多个特定区域。路由将部署到列出的**所有**区域，而不是从列表中选择单个区域。例如，`['iad1', 'sfo1']`。

> **要知道**：
>
> - 如果未指定 `preferredRegion`，它将继承最近的父布局的选项。根布局默认为 `'auto'`。
> - 子段的值会覆盖父段的值，值不会合并。
> - Next.js 将区域值传递给部署平台。确切的行为和可用的区域代码依平台而异。请参阅您的部署平台文档以获取支持的值。

## Vercel

如果将 Next.js 部署在 Vercel 上，仅当设置 `export const runtime = 'edge'` 时才支持区域。可以传递以下选项：

- **`'auto'`**（默认）：使用默认区域。
- **`'global'`**: 优先将路由部署到所有可用区域。
- **`'home'`**: 优先将路由部署到主区域。

如果传入不支持的值，将抛出错误。
