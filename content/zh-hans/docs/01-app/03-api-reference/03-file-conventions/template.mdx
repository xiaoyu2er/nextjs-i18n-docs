**模板**文件类似于[布局](/docs/app/getting-started/layouts-and-pages#creating-a-layout)，它包裹布局或页面。与在不同路由间保持状态且持久存在的布局不同，模板会被赋予一个唯一的键 (key)，这意味着子级客户端组件 (Client Components) 会在导航时重置它们的状态。

当您需要以下情况时，它们非常有用：

- 在导航时重新同步 `useEffect`。
- 在导航时重置子客户端组件的状态。例如，一个输入字段。
- 改变默认的框架行为。例如，布局内的 Suspense 边界仅在首次加载时显示回退 (fallback)，而模板则在每次导航时都显示它。

## 约定

可以通过从 `template.js` 文件导出一个默认的 React 组件来定义模板。该组件应接受一个 `children` 属性。

```tsx filename="app/template.tsx" switcher
export default function Template({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>
}
```

```jsx filename="app/template.js" switcher
export default function Template({ children }) {
  return <div>{children}</div>
}
```

在嵌套关系方面，`template.js` 在布局和其子内容之间渲染。以下是一个简化的输出：

```jsx filename="Output"
<Layout>
  {/* Note that the template is given a unique key. */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```

在[组件层次结构](/docs/app/getting-started/project-structure#component-hierarchy)中，`template.js` 在 `layout.js` 和 `error.js` 之间渲染。它包裹 `error.js`、`loading.js`、`not-found.js` 和 `page.js`，但**不**包裹同一段内的 `layout.js`。

## 属性

### `children` (必填)

模板接受一个 `children` 属性。

```jsx filename="Output"
<Layout>
  {/* Note that the template is automatically given a unique key. */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```

## 行为

- **服务器组件**：默认情况下，模板是服务器组件 (Server Components)。
- **导航时**：模板在其自身的段级别接收一个唯一键。当该段（包括其动态参数）改变时，它们会重新挂载。在更深层段内的导航不会重新挂载更高级别的模板。搜索参数不会触发重新挂载。
- **状态重置**：模板内的任何客户端组件都会在导航时重置其状态。
- **效果重新运行**：像 `useEffect` 这样的效果会随着组件重新挂载而重新同步。
- **DOM 重置**：模板内的 DOM 元素会被完全重新创建。

### 导航和重新挂载期间的模板

本节说明模板在导航期间的行为。它逐步展示在每次路由变化时哪些模板会重新挂载以及原因。

使用以下项目树：

```
app
├── about
│   ├── page.tsx
├── blog
│   ├── [slug]
│   │   └── page.tsx
│   ├── page.tsx
│   └── template.tsx
├── layout.tsx
├── page.tsx
└── template.tsx
```

从 `/` 开始，React 树大致如下。

> 注意：示例中显示的 `key` 值是示意性的，您应用程序中的值可能不同。

```jsx filename="Output"
<RootLayout>
  {/* app/template.tsx */}
  <Template key="/">
    <Page />
  </Template>
</RootLayout>
```

导航到 `/about`（第一段改变），根模板的键改变，它重新挂载：

```jsx filename="Output"
<RootLayout>
  {/* app/template.tsx */}
  <Template key="/about">
    <AboutPage />
  </Template>
</RootLayout>
```

导航到 `/blog`（第一段改变），根模板的键改变，它重新挂载，同时博客级别的模板挂载：

```jsx filename="Output"
<RootLayout>
  {/* app/template.tsx (root) */}
  <Template key="/blog">
    {/* app/blog/template.tsx */}
    <Template key="/blog">
      <BlogIndexPage />
    </Template>
  </Template>
</RootLayout>
```

在同一第一段内导航到 `/blog/first-post`（子段改变），根模板的键不变，但博客级别的模板键改变，它重新挂载：

```jsx filename="Output"
<RootLayout>
  {/* app/template.tsx (root) */}
  <Template key="/blog">
    {/* app/blog/template.tsx */}
    {/* remounts because the child segment at this level changed */}
    <Template key="/blog/first-post">
      <BlogPostPage slug="first-post" />
    </Template>
  </Template>
</RootLayout>
```

导航到 `/blog/second-post`（相同的第一个段，不同的子段），根模板的键不变，但博客级别的模板键改变，它再次重新挂载：

```jsx filename="Output"
<RootLayout>
  {/* app/template.tsx (root) */}
  <Template key="/blog">
    {/* app/blog/template.tsx */}
    {/* remounts again due to changed child segment */}
    <Template key="/blog/second-post">
      <BlogPostPage slug="second-post" />
    </Template>
  </Template>
</RootLayout>
```
