本文档部分涵盖**元数据文件约定**。基于文件的元数据可以通过向路由段添加特殊元数据文件来定义。

每个文件约定都可以使用静态文件（例如 `opengraph-image.jpg`）或代码生成动态变体（例如 `opengraph-image.js`）来定义。

定义文件后，Next.js 将自动提供该文件（针对生产环境的缓存包含哈希）并更新相关的头元素以包含正确的元数据，例如资产的 URL、文件类型和图像大小。

> **值得了解**：

- 特殊路由处理器 (Route Handlers)，例如 [`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap)、[`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image) 和 [`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons)，以及其他 [元数据文件](/docs/app/api-reference/file-conventions/metadata) 默认会被缓存。
- 若与 [`proxy.ts`](/docs/app/api-reference/file-conventions/proxy) 一起使用，请 [配置匹配器 (matcher)](/docs/app/api-reference/file-conventions/proxy#matcher) 以排除元数据文件。
