在 `app` 目录的 **根** 部添加或生成一个 `manifest.(json|webmanifest)` 文件，该文件需符合 [Web Manifest 规范](https://developer.mozilla.org/docs/Web/Manifest)，以便向浏览器提供有关您 Web 应用程序的信息。

## 静态 Manifest 文件

```json filename="app/manifest.json | app/manifest.webmanifest"
{
  "name": "My Next.js Application",
  "short_name": "Next.js App",
  "description": "An application built with Next.js",
  "start_url": "/"
  // ...
}
```

## 生成 Manifest 文件

添加一个返回 [`Manifest` 对象](#manifest-object) 的 `manifest.js` 或 `manifest.ts` 文件。

> 小提示： `manifest.js` 是一个特殊的路由处理器 (Route Handlers)，默认会被缓存，除非它使用了 [运行时 API (Request-time API)](/docs/app/glossary#request-time-apis) 或 [动态配置 (dynamic config)](/docs/app/guides/caching-without-cache-components#dynamic) 选项。

```ts filename="app/manifest.ts" switcher
import type { MetadataRoute } from 'next'

export default function manifest(): MetadataRoute.Manifest {
  return {
    name: 'Next.js App',
    short_name: 'Next.js App',
    description: 'Next.js App',
    start_url: '/',
    display: 'standalone',
    background_color: '#fff',
    theme_color: '#fff',
    icons: [
      {
        src: '/favicon.ico',
        sizes: 'any',
        type: 'image/x-icon',
      },
    ],
  }
}
```

```js filename="app/manifest.js" switcher
export default function manifest() {
  return {
    name: 'Next.js App',
    short_name: 'Next.js App',
    description: 'Next.js App',
    start_url: '/',
    display: 'standalone',
    background_color: '#fff',
    theme_color: '#fff',
    icons: [
      {
        src: '/favicon.ico',
        sizes: 'any',
        type: 'image/x-icon',
      },
    ],
  }
}
```

### Manifest 对象

Manifest 对象包含广泛的选项列表，这些选项可能会因新的 Web 标准而更新。如果您使用 [TypeScript](/docs/app/api-reference/config/typescript#ide-plugin)，请查看代码编辑器中的 `MetadataRoute.Manifest` 类型，或者查看 [MDN](https://developer.mozilla.org/docs/Web/Manifest) 文档。
