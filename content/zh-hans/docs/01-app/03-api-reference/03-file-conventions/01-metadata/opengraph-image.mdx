opengraph-image 和 twitter-image 文件规范允许您为路由段设置 Open Graph 和 Twitter 图片。

在用户分享您的网站链接时，它们可用于设置社交媒体和即时通讯应用程序上显示的图片。

设置 Open Graph 和 Twitter 图片有两种方法：

- [使用图片文件 (.jpg, .png, .gif)](#image-files-jpg-png-gif)
- [使用代码生成图片 (.js, .ts, .tsx)](#generate-images-using-code-js-ts-tsx)

## 图片文件 (.jpg, .png, .gif)

在段中放置 opengraph-image 或 twitter-image 图片文件，以使用图片文件设置路由段的共享图片。

Next.js 将评估该文件，并自动将适当的标签添加到应用的 `<head>` 元素中。

| 文件规范                                 | 支持的文件类型            |
| ---------------------------------------- | ------------------------- |
| [`opengraph-image`](#opengraph-image)    | `.jpg`, `.jpeg`, `.png`, `.gif` |
| [`twitter-image`](#twitter-image)        | `.jpg`, `.jpeg`, `.png`, `.gif` |
| [`opengraph-image.alt`](#opengraph-imagealttxt) | `.txt`                |
| [`twitter-image.alt`](#twitter-imagealttxt)     | `.txt`                |

> **小提示**：
>
> `twitter-image` 文件大小不得超过 [5MB](https://developer.x.com/en/docs/x-for-websites/cards/overview/summary)，`opengraph-image` 文件大小不得超过 [8MB](https://developers.facebook.com/docs/sharing/webmasters/images)。如果图片文件大小超过这些限制，构建将会失败。

### `opengraph-image`

在任何路由段中添加 opengraph-image.(jpg|jpeg|png|gif) 图片文件。

```html filename="<head> output"
<meta property="og:image" content="<generated>" />
<meta property="og:image:type" content="<generated>" />
<meta property="og:image:width" content="<generated>" />
<meta property="og:image:height" content="<generated>" />
```

### `twitter-image`

在任何路由段中添加 twitter-image.(jpg|jpeg|png|gif) 图片文件。

```html filename="<head> output"
<meta name="twitter:image" content="<generated>" />
<meta name="twitter:image:type" content="<generated>" />
<meta name="twitter:image:width" content="<generated>" />
<meta name="twitter:image:height" content="<generated>" />
```

### `opengraph-image.alt.txt`

在同一个路由段中添加一个配套的 opengraph-image.alt.txt 文件，作为 opengraph-image.(jpg|jpeg|png|gif) 的替代文本。

```txt filename="opengraph-image.alt.txt"
About Acme
```

```html filename="<head> output"
<meta property="og:image:alt" content="About Acme" />
```

### `twitter-image.alt.txt`

在同一个路由段中添加一个配套的 twitter-image.alt.txt 文件，作为 twitter-image.(jpg|jpeg|png|gif) 的替代文本。

```txt filename="twitter-image.alt.txt"
About Acme
```

```html filename="<head> output"
<meta property="twitter:image:alt" content="About Acme" />
```

## 使用代码生成图片 (.js, .ts, .tsx)

除了使用 [静态图片文件 (literal image files)](#image-files-jpg-png-gif) 外，您还可以通过代码以编程方式生成图片。

通过创建一个默认导出函数的 opengraph-image 或 twitter-image 路由，来生成路由段的共享图片。

| 文件规范 | 支持的文件类型 |
| --- | --- |
| `opengraph-image` | `.js`, `.ts`, `.tsx` |
| `twitter-image` | `.js`, `.ts`, `.tsx` |

> **小知识**:
>
> - 默认情况下，生成的图片是 [**静态优化 (statically optimized)**](/docs/app/glossary#prerendering)（在构建时生成并缓存），除非它们使用了 [请求时 API (Request-time APIs)](/docs/app/glossary#request-time-apis) 或未缓存的数据。
> - 您可以使用 [`generateImageMetadata`](/docs/app/api-reference/functions/generate-image-metadata) 在同一文件中生成多个图片。
> - `opengraph-image.js` 和 `twitter-image.js` 是特殊的路由处理器 (Route Handlers)，默认会被缓存，除非它们使用了 [请求时 API (Request-time APIs)](/docs/app/glossary#request-time-apis) 或 [动态配置 (Dynamic Config)](/docs/app/guides/caching-without-cache-components#dynamic) 选项。

生成图片最简单的方法是使用 `next/og` 提供的 [ImageResponse](/docs/app/api-reference/functions/image-response) API。

```tsx filename="app/about/opengraph-image.tsx" switcher
import { ImageResponse } from 'next/og'
import { readFile } from 'node:fs/promises'
import { join } from 'node:path'

// Image metadata
export const alt = 'About Acme'
export const size = {
  width: 1200,
  height: 630,
}

export const contentType = 'image/png'

// Image generation
export default async function Image() {
  // Font loading, process.cwd() is Next.js project directory
  const interSemiBold = await readFile(
    join(process.cwd(), 'assets/Inter-SemiBold.ttf')
  )

  return new ImageResponse(
    (
      // ImageResponse JSX element
      <div
        style={{
          fontSize: 128,
          background: 'white',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
        }}
      >
        About Acme
      </div>
    ),
    // ImageResponse options
    {
      // For convenience, we can re-use the exported opengraph-image
      // size config to also set the ImageResponse's width and height.
      ...size,
      fonts: [
        {
          name: 'Inter',
          data: interSemiBold,
          style: 'normal',
          weight: 400,
        },
      ],
    }
  )
}
```

## 属性

默认导出的函数接收以下参数：

#### `params`（可选）

一个 Promise (承诺)，解析为包含从根段到 `opengraph-image` 或 `twitter-image` 所在段的 [动态路由参数 (dynamic route parameters)](/docs/app/api-reference/file-conventions/dynamic-routes) 对象的对象，且共置于该段中。

> **小知识**: 如果您使用 [`generateImageMetadata`](/docs/app/api-reference/functions/generate-image-metadata)，该函数还将接收一个 `id` 属性，它是一个 Promise (承诺)，解析为 `generateImageMetadata` 返回的项之一中的 `id` 值。

```tsx filename="app/shop/[slug]/opengraph-image.tsx" switcher
export default async function Image({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  // ...
}
```

```jsx filename="app/shop/[slug]/opengraph-image.js" switcher
export default async function Image({ params }) {
  const { slug } = await params
  // ...
}
```

| 路由                                      | URL         | `params`                           |
| ------------------------------------------ | ----------- | ---------------------------------- |
| `app/shop/opengraph-image.js`              | `/shop`     | `undefined`                        |
| `app/shop/[slug]/opengraph-image.js`       | `/shop/1`   | `Promise<{ slug: '1' }>`           |
| `app/shop/[tag]/[item]/opengraph-image.js` | `/shop/1/2` | `Promise<{ tag: '1', item: '2' }>` |

### 返回值

默认导出的函数应返回 `Blob` | `ArrayBuffer` | `TypedArray` | `DataView` | `ReadableStream` | `Response`。

> **小提示**：`ImageResponse` 符合此返回类型。

### 配置导出

您可以通过从 opengraph-image 或 twitter-image 路由导出 `alt`、`size` 和 `contentType` 变量来可选地配置图片的元数据。

| --- | --- |
| [`alt`](#alt) | `string` |

#### 路由段配置 (Route Segment Config)

opengraph-image 和 twitter-image 是专门的 [路由处理器 (Route Handlers)](/docs/app/api-reference/file-conventions/route)，可以使用与 Pages 和 Layouts 相同的路由段配置选项。

### 示例

#### 使用外部数据

此示例使用 `params` 对象和外部数据来生成图片。

> **小知识**:
> 默认情况下，此生成的图片将进行静态优化 (statically optimized)。您可以配置单独的 `fetch` [`选项 (options)`](/docs/app/api-reference/functions/fetch) 或路由段 [选项 (options)](/docs/app/guides/caching-without-cache-components#route-segment-config-revalidate) 来更改此行为。

```tsx filename="app/posts/[slug]/opengraph-image.tsx" switcher
import { ImageResponse } from 'next/og'

export const alt = 'About Acme'
export const size = {
  width: 1200,
  height: 630,
}
export const contentType = 'image/png'

export default async function Image({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  const post = await fetch(`https://.../posts/${slug}`).then((res) =>
    res.json()
  )

  return new ImageResponse(
    (
      <div
        style={{
          fontSize: 48,
          background: 'white',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
        }}
      >
        {post.title}
      </div>
    ),
    {
      ...size,
    }
  )
}
```


#### 使用 Node.js 运行时和本地资源

这些示例使用 Node.js 运行时从文件系统获取本地图片，并将其作为 base64 字符串或 `ArrayBuffer` 传递给 `<img>` 的 `src` 属性。请将本地资源相对于项目根目录放置，而不是示例源代码文件。

```tsx filename="app/opengraph-image.tsx" switcher
import { ImageResponse } from 'next/og'
import { join } from 'node:path'
import { readFile } from 'node:fs/promises'

export default async function Image() {
  const logoData = await readFile(join(process.cwd(), 'logo.png'), 'base64')
  const logoSrc = `data:image/png;base64,${logoData}`

  return new ImageResponse(
    (
      <div
        style={{
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
        }}
      >
        <img src={logoSrc} height="100" />
      </div>
    )
  )
}
```

```jsx filename="app/opengraph-image.js" switcher

        style={{

```

将 `ArrayBuffer` 传递给 `<img>` 元素的 `src` 属性不是 HTML 规范的一部分。`next/og` 使用的渲染引擎支持它，但由于 TypeScript 定义遵循规范，您需要使用 `@ts-expect-error` 指令或类似内容来使用此 [特性](https://github.com/vercel/satori/issues/606#issuecomment-2144000453)。

```tsx filename="app/opengraph-image.tsx" switcher
import { ImageResponse } from 'next/og'
import { join } from 'node:path'
import { readFile } from 'node:fs/promises'

export default async function Image() {
  const logoData = await readFile(join(process.cwd(), 'logo.png'))
  const logoSrc = Uint8Array.from(logoData).buffer

  return new ImageResponse(
    (
      <div
        style={{
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
        }}
      >
        {/* @ts-expect-error Satori accepts ArrayBuffer/typed arrays for <img src> at runtime */}
        <img src={logoSrc} height="100" />
      </div>
    )
  )
}
```

```jsx filename="app/opengraph-image.js" switcher

```
