`favicon`、`icon` 或 `apple-icon` 文件规范允许您为您的应用设置图标。

这对于添加出现在网页浏览器标签页、手机主屏幕和搜索引擎结果中的应用图标非常有用。

设置应用图标有两种方法：

- [使用图片文件 (.ico, .jpg, .png)](#image-files-ico-jpg-png)
- [使用代码生成图标 (.js, .ts, .tsx)](#generate-icons-using-code-js-ts-tsx)

## 图片文件 (.ico, .jpg, .png)

将 `favicon`、`icon` 或 `apple-icon` 图像文件放置在您的 `/app` 目录中，即可设置应用图标。`favicon` 图像只能位于 `app/` 的顶级位置。

Next.js 将评估该文件，并自动将适当的标签添加到应用的 `<head>` 元素中。

| 文件规范                      | 支持的文件类型                                           | 有效位置 |
| ----------------------------- | ------------------------------------------------------- | -------- |
| [`favicon`](#favicon)         | `.ico`                                                  | `app/`   |
| [`icon`](#icon)               | `.ico`, `.jpg`, `.jpeg`, `.png`, `.svg`                 | `app/**/*` |
| [`apple-icon`](#apple-icon)   | `.jpg`, `.jpeg`, `.png`                                 | `app/**/*` |

### favicon

在根 `/app` 路由段中放置一个 `favicon.ico` 图像文件。

```html filename="<head> output"
<link rel="icon" href="/favicon.ico" sizes="any" />
```

### icon

添加一个 `icon.(ico|jpg|jpeg|png|svg)` 图像文件。

```html filename="<head> output"
<link
  rel="icon"
  href="/icon?<generated>"
  type="image/<generated>"
  sizes="<generated>"
/>
```

### apple-icon

添加一个 `apple-icon.(jpg|jpeg|png)` 图像文件。

```html filename="<head> output"
<link
  rel="apple-touch-icon"
  href="/apple-icon?<generated>"
  type="image/<generated>"
  sizes="<generated>"
/>
```

> **小提示**:
>
> - 您可以通过在文件名中添加数字后缀来设置多个图标。例如，`icon1.png`、`icon2.png` 等。编号文件将按字典顺序排序。
> - `favicon` 只能在根 `/app` 段中设置。如果您需要更精细的控制，可以使用 [`icon`](#icon)。
> - 适当的 `<link>` 标签和属性，如 `rel`、`href`、`type` 和 `sizes` 取决于图标的类型和被评估文件的元数据。
> - 例如，32x32px 的 `.png` 文件将具有 `type="image/png"` 和 `sizes="32x32"` 属性。
> - 当扩展名为 `.svg` 或文件图像大小未确定时，会为图标添加 `sizes="any"`。有关更多详细信息，请参阅此 [favicon 手册](https://evilmartians.com/chronicles/how-to-favicon-in-2021-six-files-that-fit-most-needs)。

## 使用代码生成图标 (.js, .ts, .tsx)

除了使用 [图像文件](#image-files-ico-jpg-png) 外，您还可以使用代码以编程方式生成图标。

创建一个 `icon` 或 `apple-icon` 路由，并默认导出一个函数，即可生成应用图标。

| 文件规范 | 支持的文件类型        |
| -------- | --------------------- |
| `icon`   | `.js`, `.ts`, `.tsx`  |
| `apple-icon` | `.js`, `.ts`, `.tsx` |

生成图标最简单的方法是使用 `next/og` 的 [`ImageResponse`](/docs/app/api-reference/functions/image-response) API。

```tsx filename="app/icon.tsx" switcher
import { ImageResponse } from 'next/og'

// Image metadata
export const size = {
  width: 32,
  height: 32,
}
export const contentType = 'image/png'

// Image generation
export default function Icon() {
  return new ImageResponse(
    (
      // ImageResponse JSX element
      <div
        style={{
          fontSize: 24,
          background: 'black',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
          color: 'white',
        }}
      >
        A
      </div>
    ),
    // ImageResponse options
    {
      // For convenience, we can re-use the exported icons size metadata
      // config to also set the ImageResponse's width and height.
      ...size,
    }
  )
}
```

```jsx filename="app/icon.js" switcher
import { ImageResponse } from 'next/og'

// Image metadata
export const size = {
  width: 32,
  height: 32,
}
export const contentType = 'image/png'

// Image generation
export default function Icon() {
  return new ImageResponse(
    (
      // ImageResponse JSX element
      <div
        style={{
          fontSize: 24,
          background: 'black',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
          color: 'white',
        }}
      >
        A
      </div>
    ),
    // ImageResponse options
    {
      // For convenience, we can re-use the exported icons size metadata
      // config to also set the ImageResponse's width and height.
      ...size,
    }
  )
}
```

```html filename="<head> output"
<link rel="icon" href="/icon?<generated>" type="image/png" sizes="32x32" />
```

> **要知道**:
>
> - 默认情况下，生成的图标是 [**静态优化 (statically optimized)**](/docs/app/glossary#prerendering)（除非它们使用 [请求时 API (Request-time APIs)](/docs/app/glossary#request-time-apis) 或未缓存的数据）。
> - 您可以在同一文件中生成多个图标，使用 [`generateImageMetadata`](/docs/app/api-reference/functions/generate-image-metadata)。
> - 您不能生成 `favicon` 图标。请改用 [`icon`](#icon) 或 [favicon.ico](#favicon) 文件。
> - App 图标是特殊的路由处理器 (Route Handlers)，默认情况下会被缓存，除非它们使用 [请求时 API (Request-time API)](/docs/app/glossary#request-time-apis) 或 [动态配置 (dynamic config)](/docs/app/guides/caching-without-cache-components#dynamic) 选项。

## 属性

默认导出的函数接收以下参数：

#### `params`（可选）

这是一个位于 `icon` 或 `apple-icon` 所在段中的 Promise，解析后包含 [动态路由参数 (dynamic route parameters)](/docs/app/api-reference/file-conventions/dynamic-routes)。

> **小知识**: 如果您使用 [`generateImageMetadata`](/docs/app/api-reference/functions/generate-image-metadata)，该函数还将接收一个 `id` 属性，它是一个 Promise (承诺)，解析为 `generateImageMetadata` 返回的项之一中的 `id` 值。

```tsx filename="app/shop/[slug]/icon.tsx" switcher
export default async function Icon({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  // ...
}
```

```jsx filename="app/shop/[slug]/icon.js" switcher
export default async function Icon({ params }) {
  const { slug } = await params
  // ...
}
```

｜路由                           ｜URL         ｜params                           
｜ ------------------------------- ｜ ----------- ｜ ---------------------------------- 
｜ `app/shop/icon.js`               ｜ `/shop`     ｜ `undefined`                       
｜ `app/shop/[slug]/icon.js`        ｜ `/shop/1`   ｜ `Promise<{ slug: '1' }>`          
｜ `app/shop/[tag]/[item]/icon.js`  ｜ `/shop/1/2` ｜ `Promise<{ tag: '1', item: '2' }>` |

### 返回值

默认导出的函数应返回 `Blob` | `ArrayBuffer` | `TypedArray` | `DataView` | `ReadableStream` | `Response`。

> **小提示**：`ImageResponse` 符合此返回类型。

### 配置导出

您可以选择从 `icon` 或 `apple-icon` 路由中导出 `size` 和 `contentType` 变量来配置图标的元数据。

| 选项                        | 类型                                                                                                            |
| --------------------------- | --------------------------------------------------------------------------------------------------------------- |
| [`size`](#size)             | `{ width: number; height: number }`                                                                             |
| [`contentType`](#contenttype) | `string` - [图像 MIME 类型](https://developer.mozilla.org/docs/Web/HTTP/Basics_of_HTTP/MIME_types#image_types) |

#### size

```tsx filename="icon.tsx | apple-icon.tsx" switcher
export const size = { width: 32, height: 32 }

export default function Icon() {}
```

```jsx filename="icon.js | apple-icon.js" switcher
export const size = { width: 32, height: 32 }

export default function Icon() {}
```

```html filename="<head> output"
<link rel="icon" sizes="32x32" />
```

#### contentType

```tsx filename="icon.tsx | apple-icon.tsx" switcher
export const contentType = 'image/png'

export default function Icon() {}
```

```jsx filename="icon.js | apple-icon.js" switcher
export const contentType = 'image/png'

export default function Icon() {}
```

```html filename="<head> output"
<link rel="icon" type="image/png" />
```

#### 路由段配置 (Route Segment Config)

`icon` 和 `apple-icon` 是专门的 [路由处理程序](/docs/app/api-reference/file-conventions/route)，可以使用与 Pages 和 Layouts 相同的 [路由段配置](/docs/app/api-reference/file-conventions/route-segment-config) 选项。
