特殊文件 `loading.js` 可帮助你使用 [React Suspense (React Suspense)](https://react.dev/reference/react/Suspense) 创建有意义的加载 UI。通过此约定，你可以在路由段内容流式传输时，从服务器显示一个[即时加载状态](#instant-loading-states)。内容加载完成后，会自动替换为新内容。

```tsx filename="app/feed/loading.tsx" switcher
export default function Loading() {
  // Or a custom loading skeleton component
  return <p>Loading...</p>
}
```

```jsx filename="app/feed/loading.js" switcher
export default function Loading() {
  // Or a custom loading skeleton component
  return <p>Loading...</p>
}
```

在 `loading.js` 文件内部，你可以添加任何轻量级的加载 UI。你可能会发现，使用 [React Developer Tools (React 开发者工具)](https://react.dev/learn/react-developer-tools) 手动切换 Suspense 边界会很有帮助。

默认情况下，此文件是一个[服务器组件](/docs/app/getting-started/server-and-client-components) —— 但也可以通过 `"use client"` 指令用作客户端组件。

## 参考

### 参数

加载 UI 组件不接受任何参数。

## 行为

### 导航

- 回退 UI 是[预取的](/docs/app/getting-started/linking-and-navigating#prefetching)，除非预取尚未完成，否则导航是即时的。
- 导航是可中断的，意味着在导航到另一个路由之前，不需要等待当前路由的内容完全加载。
- 当新的路由段加载时，共享布局保持交互状态。

### 即时加载状态

即时加载状态是在导航时立即显示的回退 UI。你可以预渲染加载指示器，如骨架屏和旋转器，或者未来屏幕的一小部分但有意义的内容，如封面照片、标题等。这有助于用户理解应用正在响应，并提供更好的用户体验。

通过在文件夹内添加 `loading.js` 文件来创建加载状态。

在同一文件夹中，`loading.js` 将嵌套在 `layout.js` 内部。它会自动将 `page.js` 文件及其下的任何子组件包装在 `<Suspense>` 边界内。

在[组件层次结构](/docs/app/getting-started/project-structure#component-hierarchy)中，`loading.js` 将 `not-found.js`、`page.js` 和嵌套的 `layout.js` 文件包装在 `<Suspense>` 边界内。它**不**包装同一段内的 `layout.js`、`template.js` 或 `error.js`。

> **请注意**：如果布局访问了未缓存或运行时数据（例如 `cookies()`、`headers()` 或未缓存的 fetch），`loading.js` 将不会为其显示回退。
>
> - **没有 [缓存组件](/docs/app/getting-started/caching) 时：** 导航会阻塞，直到布局完成渲染。
> - **有 [缓存组件](/docs/app/getting-started/caching) 时：** 布局中未缓存或运行时的数据访问必须显式包装在 `<Suspense>` 中，否则 Next.js 会通过构建时错误进行引导。静态外壳会首先流式传输，未缓存的内容随后填充。
>
> 为确保即时导航，请将未缓存的数据获取从 `layout.js` 移至 `page.js`，或者将布局中的运行时数据访问包装在其自己的 `<Suspense>` 边界内。有关详细信息和示例，请参阅 [layout.js 注意事项](/docs/app/api-reference/file-conventions/layout#interaction-with-loadingjs)。

### 搜索引擎优化

- 对于只抓取静态 HTML，且无法像完整浏览器一样执行 JavaScript 的机器人（例如 Twitterbot），Next.js 会在流式传输 UI 之前解析 [`generateMetadata`](/docs/app/api-reference/functions/generate-metadata)，并将元数据放在初始 HTML 的 `<head>` 中。
- 否则，可以使用[流式元数据](/docs/app/api-reference/functions/generate-metadata#streaming-metadata)。Next.js 会自动检测用户代理，以在阻塞和流式行为之间进行选择。
- 由于流式传输是服务器端渲染，因此它不影响 SEO。你可以使用 Google 的 [富媒体搜索结果测试](https://search.google.com/test/rich-results) 工具来查看你的页面在 Google 网络爬虫中的呈现方式，并查看序列化的 HTML ([来源](https://web.dev/rendering-on-the-web/#seo-considerations))。

### 状态码

流式传输时，将返回 `200` 状态码以表示请求成功。

服务器仍然可以在流内容本身中向客户端传达错误或问题，例如，在使用 [`redirect`](/docs/app/api-reference/functions/redirect) 或 [`notFound`](/docs/app/api-reference/functions/not-found) 时。因为响应头已经发送到客户端，所以无法更新响应的状态码。

例如，当 404 页面流式传输到客户端时，Next.js 会在流式传输的 HTML 中包含一个 `<meta name="robots" content="noindex">` 标签。这可以防止搜索引擎索引该 URL，即使 HTTP 状态是 200。请参阅 Google 关于 [`robots` 元标签](https://developers.google.com/search/docs/crawling-indexing/robots-meta-tag) 的指南。

某些爬虫可能会将这些响应标记为“软 404”。在流式传输的情况下，这不会导致索引，因为页面在 HTML 中明确标记为 `noindex`。

如果你出于合规性或分析需要 404 状态码，请确保在响应主体开始流式传输之前资源存在，以便服务器可以设置 HTTP 状态码。

你可以在 [`proxy`](/docs/app/api-reference/file-conventions/proxy) 中运行此检查，以将缺失的 slug 重写到未找到路由，或[生成 404 响应](/docs/app/api-reference/file-conventions/proxy#producing-a-response)。保持代理检查快速，并避免在那里获取完整内容。

<details>

<summary>响应主体何时开始流式传输？</summary>

当 Suspense 回退渲染时（例如，`loading.tsx`）或当服务器组件在 `Suspense` 边界下暂停时，响应主体开始流式传输。请在这些边界之前以及在任何可能暂停的 `await` 之前放置 `notFound()`。

要开始流式传输，必须设置响应头。这就是为什么在流式传输开始后无法更改状态码。

</details>

### 浏览器限制

[某些浏览器](https://bugs.webkit.org/show_bug.cgi?id=252413) 会缓冲流式响应。在响应超过 1024 字节之前，你可能看不到流式传输的响应。这通常只影响 “hello world” 应用程序，而不影响真实的应用程序。

## 示例

### Suspense 流式传输

除了 `loading.js`，你还可以手动为自己的 UI 组件创建 Suspense 边界。App 路由器支持使用 [Suspense](https://react.dev/reference/react/Suspense) 进行流式传输。有关流式传输工作原理的更多信息，包括细粒度的 Suspense 模式、路由处理程序流式传输和基础设施注意事项，请参阅[流式传输指南](/docs/app/guides/streaming)。

`<Suspense>` 的工作原理是：包装一个执行异步操作（例如获取数据）的组件，在操作进行时显示回退 UI（例如骨架屏、旋转器），然后在操作完成后换入你的组件。

```tsx filename="app/dashboard/page.tsx" switcher
import { Suspense } from 'react'
import { PostFeed, Weather } from './Components'

export default function Posts() {
  return (
    <section>
      <Suspense fallback={<p>Loading feed...</p>}>
        <PostFeed />
      </Suspense>
      <Suspense fallback={<p>Loading weather...</p>}>
        <Weather />
      </Suspense>
    </section>
  )
}
```

```jsx filename="app/dashboard/page.js" switcher
import { Suspense } from 'react'
import { PostFeed, Weather } from './Components'

export default function Posts() {
  return (
    <section>
      <Suspense fallback={<p>Loading feed...</p>}>
        <PostFeed />
      </Suspense>
      <Suspense fallback={<p>Loading weather...</p>}>
        <Weather />
      </Suspense>
    </section>
  )
}
```

通过使用 Suspense，你可以获得以下好处：

1. **流式服务器渲染** - 从服务器到客户端逐步渲染 HTML。
2. **选择性水合** - React 根据用户交互，优先处理哪些组件首先变为可交互。

更多 Suspense 示例和用法，请参阅 [React 文档](https://react.dev/reference/react/Suspense)。
