---

`devIndicators` 允许您配置屏幕指示器，该指示器可在开发过程中为您提供当前路由的上下文信息。

```ts filename="Types"
  devIndicators: false | {
    position?: 'bottom-right'
    | 'bottom-left'
    | 'top-right'
    | 'top-left', // defaults to 'bottom-left',
  },
```

将 `devIndicators` 设置为 `false` 会隐藏该指示器，但 Next.js 仍会继续显示任何遇到的构建或运行时错误。

## 故障排除

### 指示器未将路由标记为静态 (Static)

如果您预期某个路由是静态的，但指示器将其标记为动态，那么该路由很可能已选择不进行预渲染。

您可以通过使用 `next build --debug` 构建应用程序并查看终端输出来确认某个路由是 [预渲染的](/docs/app/glossary#prerendering) 还是 [动态渲染的](/docs/app/glossary#dynamic-rendering)。静态（或预渲染）路由将显示 `○` 符号，而动态路由将显示 `ƒ` 符号。例如：

```bash filename="Build Output"
Route (app)
┌ ○ /_not-found
└ ƒ /products/[id]

○  (Static)   prerendered as static content
ƒ  (Dynamic)  server-rendered on demand
```

<AppOnly>

路由选择不进行预渲染的原因有两个：

- 存在 [请求时 API (Request-time APIs)](/docs/app/glossary#request-time-apis)，它们依赖于请求信息。
- [未缓存的数据请求](/docs/app/getting-started/fetching-data)，例如对 ORM 或数据库驱动程序的调用。

检查您的路由是否存在这些情况，如果您无法静态渲染该路由，则考虑使用 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 或 [`<Suspense />`](https://react.dev/reference/react/Suspense) 来利用 [流式传输 (Streaming)](/docs/app/getting-started/linking-and-navigating#streaming)。

</AppOnly>

<PagesOnly>

从页面导出 [`getServerSideProps`](/docs/pages/building-your-application/data-fetching/get-server-side-props) 或 [`getInitialProps`](/docs/pages/api-reference/functions/get-initial-props) 时，它将被标记为动态 (Dynamic)。

</PagesOnly>

## 版本历史

| 版本   | 变更                                                                             |
| --------- | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| v16.0.0 | `appIsrStatus`、`buildActivity` 和 `buildActivityPosition` 选项已被移除。                                                       |
| v15.2.0 | 使用新的 `position` 选项改进了屏幕指示器。`appIsrStatus`、`buildActivity` 和 `buildActivityPosition` 选项已被弃用。     |
| v15.0.0 | 使用 `appIsrStatus` 选项添加了屏幕静态指示器。                                                                               |
