---
title: React 编译器 (React Compiler)
description: 启用 React 编译器 来自动优化组件渲染。
---

Next.js 支持 [React 编译器 (React Compiler)](https://react.dev/learn/react-compiler/introduction)，该工具旨在通过自动优化组件渲染来提高性能。这减少了对使用 `useMemo` 和 `useCallback` 进行手动内存化的需求。

Next.js 包含一个用 SWC 编写的自定义性能优化，使 React Compiler 更高效。与其在每个文件上运行编译器，Next.js 会分析您的项目，仅将 React Compiler 应用于相关文件。这避免了不必要的工作，与单独使用 Babel 插件相比，构建速度更快。

## 工作原理

React Compiler 通过 Babel 插件运行。为了保持构建速度，Next.js 使用了一个自定义的 SWC 优化，仅将 React Compiler 应用于相关文件——例如包含 JSX 或 React Hooks 的文件。

这避免了编译所有内容，并将性能成本降至最低。与默认的基于 Rust 的编译器相比，您仍可能会看到构建速度稍慢，但影响很小且是局部的。

要使用它，请安装 `babel-plugin-react-compiler`：

```bash package="pnpm"
pnpm add -D babel-plugin-react-compiler
```

```bash package="npm"
npm install -D babel-plugin-react-compiler
```

```bash package="yarn"
yarn add -D babel-plugin-react-compiler
```

```bash package="bun"
bun add -D babel-plugin-react-compiler
```

然后，在 `next.config.js` 中添加 `reactCompiler` 选项：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  reactCompiler: true,
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactCompiler: true,
}

module.exports = nextConfig
```

## 注解 (Annotations)

您可以按以下方式配置编译器以在 "opt-in" 模式下运行：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  reactCompiler: {
    compilationMode: 'annotation',
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactCompiler: {
    compilationMode: 'annotation',
  },
}

module.exports = nextConfig
```

然后，您可以使用 React 提供的 "use memo" 指令注解特定组件或钩子以选择加入：

```ts filename="app/page.tsx" switcher
export default function Page() {
  'use memo'
  // ...
}
```

```js filename="app/page.js" switcher
export default function Page() {
  'use memo'
  // ...
}
```

> **注意：** 您还可以使用 React 的 "use no memo" 指令实现相反效果，以选择退出一个组件或钩子。
