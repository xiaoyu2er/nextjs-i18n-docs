---

`cacheComponents` 标志是 Next.js 中的一项功能，它会导致 App Router 中的数据获取操作在明确缓存之前被排除在预渲染之外。这对于优化服务器组件 (Server Components) 中未缓存数据获取的性能很有用。

如果您的应用需要在运行时获取新鲜数据，而不是从预渲染缓存中提供，则它非常有用。

预期与 [`use cache`](/docs/app/api-reference/directives/use-cache) 配合使用，默认情况下您的数据获取将在运行时发生，除非您使用 `use cache` 在页面、函数或组件级别定义应用程序的特定部分进行缓存。

## 使用方法

要启用 cacheComponents 标志，请在 next.config.ts 文件中将其设置为 `true`：

```ts filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  cacheComponents: true,
}

export default nextConfig
```

当启用 `cacheComponents` 时，您可以使用以下缓存函数和配置：

- [`use cache` 指令](/docs/app/api-reference/directives/use-cache)
- 与 `use cache` 配合使用的 [`cacheLife` 函数](/docs/app/api-reference/config/next-config-js/cacheLife)
- [`cacheTag` 函数](/docs/app/api-reference/functions/cacheTag)

- [`use cache` 指令](/docs/app/api-reference/directives/use-cache)
- 与 `use cache` 配合使用的 [`cacheLife` 函数](/docs/app/api-reference/config/next-config-js/cacheLife)
- [`cacheTag` 函数](/docs/app/api-reference/functions/cacheTag)

## 使用 Activity 进行导航

当 `cacheComponents` 启用时，Next.js 会使用 React 的 [`<Activity>`](https://react.dev/reference/react/Activity) 组件，在客户端导航期间保留组件状态。

在导航离开时，Next.js 不会卸载上一个路由，而是将 Activity 模式设置为 ["hidden"](https://react.dev/reference/react/Activity#activity)。这意味着：

- 在路由之间导航时保留组件状态
- 当导航返回时，上一个路由会以其完整状态重新出现
- 当路由被隐藏时，效果会被清理，并在路由再次可见时重新创建

这种行为通过在用户在路由之间来回导航时保持 UI 状态（表单输入、展开的部分等）来改善导航体验。

> **小知识**：Next.js 使用启发式方法以保持最近访问的几个路由处于 "hidden" 状态，而旧路由会从 DOM 中移除，以防止过度增长。

当组件保持挂载而不是卸载时，一些 UI 模式的行为会有所不同。请参阅 [保持 UI 状态指南](/docs/app/guides/preserving-ui-state) 以处理下拉菜单、对话框和测试等常见模式。
