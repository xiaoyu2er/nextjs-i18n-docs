默认情况下，当使用 `next start` 或自定义服务器时，Next.js 会使用 `gzip` 来压缩渲染后的内容和静态文件。这是为未配置压缩的应用程序提供的优化。如果您的应用程序通过自定义服务器 _已经_ 配置了压缩，Next.js 将不会添加压缩。

您可以通过检查响应中的 [`Accept-Encoding`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Accept-Encoding)（浏览器接受的选项）和 [`Content-Encoding`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Encoding)（当前使用的）头信息，来查看是否启用了压缩以及使用了哪种算法。

## 禁用压缩

要禁用**压缩**, 请将 `compress` 配置选项设置为 `false` :

```js filename="next.config.js"
module.exports = {
  compress: false,
}
```

除非你在服务器上配置了压缩，否则我们**不建议禁用压缩**, 因为压缩可以减少带宽使用并提升应用程序的性能。例如，你正在使用 [nginx](https://nginx.org/) 并希望切换到 `brotli`, 请将 `compress` 选项设置为 `false`, 以便让 nginx 处理压缩。
