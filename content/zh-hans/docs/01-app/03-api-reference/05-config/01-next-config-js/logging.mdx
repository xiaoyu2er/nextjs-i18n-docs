## 选项

<AppOnly>

### 数据获取 (Fetching)

当在 Next.js 开发模式下运行时，您可以配置日志级别以及是否将完整 URL 记录到控制台。

```js filename="next.config.js"
module.exports = {
  logging: {
    fetches: {
      fullUrl: true,
    },
  },
}
```

默认情况下，从 [服务器组件 HMR 缓存](/docs/app/api-reference/config/next-config-js/serverComponentsHmrCache) 恢复的任何 `fetch` 请求都不会被记录。但是，可以通过将 `logging.fetches.hmrRefreshes` 设置为 `true` 来启用此功能。

```js filename="next.config.js"
module.exports = {
  logging: {
    fetches: {
      hmrRefreshes: true,
    },
  },
}
```

### 服务器函数

[Server 函数 (Server Function)](https://react.dev/reference/rsc/server-functions) 的调用在开发期间默认会被记录。你可以将 `logging.serverFunctions` 设置为 `false` 来禁用此功能。

```js filename="next.config.js"
module.exports = {
  logging: {
    serverFunctions: false,
  },
}
```

启用时，终端会显示每个 Server 函数调用的函数名称、参数及持续时间：

```bash filename="Terminal"
POST /
  └─ ƒ myAction(arg1, arg2) in 5ms app/actions.ts
```

</AppOnly>

### 传入请求 (Incoming Requests)

默认情况下，所有传入请求在开发期间都会记录到控制台。您可以使用 `incomingRequests` 选项来决定忽略哪些请求。

```js filename="next.config.js"
module.exports = {
  logging: {
    incomingRequests: {
      ignore: [/\api\/v1\/health/],
    },
  },
}
```

由于这仅在开发时记录，此选项不会影响生产构建。

```js filename="next.config.js"
module.exports = {
  logging: {
    incomingRequests: false,
  },
}
```

### 浏览器控制台日志

在开发期间，你可以将浏览器控制台日志（例如 `console.log`、`console.warn`、`console.error`）转发到终端。这对于调试客户端代码非常有用，而无需检查浏览器的开发者工具。

```js filename="next.config.js"
module.exports = {
  logging: {
    browserToTerminal: true,
  },
}
```

#### 选项

`browserToTerminal` 选项接受以下值：

| 值     | 说明                                         |
| --------- | --------------------------------------------- |
| `'warn'`  | 仅转发警告和错误，默认值                     |
| `'error'` | 仅转发错误                                   |
| `true`    | 转发所有控制台输出（log、info、warn、error） |
| `false`   | 禁用浏览器日志转发                           |

```js filename="next.config.js"
module.exports = {
  logging: {
    browserToTerminal: 'warn',
  },
}
```

#### 源代码位置

启用时，浏览器日志默认包含源代码位置信息（文件路径和行号）。例如：

<AppOnly>

```tsx filename="app/page.tsx" highlight={8}
'use client'

export default function Home() {
  return (
    <button
      type="button"
      onClick={() => {
        console.log('Hello World')
      }}
    >
      Click me
    </button>
  )
}
```

点击按钮会将此消息打印到终端：

```bash filename="Terminal"
[browser] Hello World (app/page.tsx:8:17)
```

</AppOnly>

<PagesOnly>

```tsx filename="pages/index.tsx" highlight={6}
export default function Home() {
  return (
    <button
      type="button"
      onClick={() => {
        console.log('Hello World')
      }}
    >
      Click me
    </button>
  )
}
```

点击按钮会将此消息打印到终端：

```bash filename="Terminal"
[browser] Hello World (pages/index.tsx:6:17)
```

</PagesOnly>

### 禁用日志记录 (Disabling Logging)

此外，您可以通过将 `logging` 设置为 `false` 来禁用开发日志记录。

```js filename="next.config.js"
module.exports = {
  logging: false,
}
```

## 版本历史

| 版本   | 变更                                                                          |
| --------- | -------------------------------------------------------------------------------- |
| `v16.2.0` | `browserToTerminal` 已添加（从 `experimental.browserDebugInfoInTerminal` 移动） |
| `v15.4.0` | 引入 `experimental.browserDebugInfoInTerminal`                                 |
| `v15.2.0` | 添加 `incomingRequests`                                                         |
| `v15.0.0` | 添加 `logging: false` 选项，为 App Router 添加 `fetches.hmrRefreshes`         |
| `v14.0.0` | `logging.fetches` 针对 App Router 移入稳定版                                   |
