## 使用方法

Turbopack 文件系统缓存 (FileSystem Cache) 使 Turbopack 减少 `next dev` 或 `next build` 命令的工作量。启用后，Turbopack 将在各次构建之间向 `.next` 文件夹保存和恢复数据，从而显著加快后续的构建和开发会话。

> **了解要点：** 文件系统缓存 (FileSystem Cache) 功能对于开发已被视为稳定，对于生产构建处于实验阶段

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    // Enable filesystem caching for `next dev`
    turbopackFileSystemCacheForDev: true,
    // Enable filesystem caching for `next build`
    turbopackFileSystemCacheForBuild: true,
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    // Enable filesystem caching for `next dev`
    turbopackFileSystemCacheForDev: true,
    // Enable filesystem caching for `next build`
    turbopackFileSystemCacheForBuild: true,
  },
}

module.exports = nextConfig
```

## 版本变更

| 版本   | 变更                                                        |
| --------- | -------------------------------------------------------------- |
| `v16.1.0` | 文件系统缓存默认针对开发环境启用                                       |
| `v16.0.0` | Beta 版本发布，构建和开发使用独立的标志                                   |
| `v15.5.0` | 持久化缓存作为试验功能在 canary 版本上发布                              |
