## 设置 CDN

要设置 [CDN](https://en.wikipedia.org/wiki/Content_delivery_network)，您可以设置资源前缀并将 CDN 的源地址配置为解析到 Next.js 托管的域。

打开 `next.config.mjs` 并根据 [阶段 (phase)](/docs/app/api-reference/config/next-config-js#async-configuration) 添加 `assetPrefix` 配置：



```js filename="next.config.mjs"
// @ts-check
import { PHASE_DEVELOPMENT_SERVER } from 'next/constants'

export default (phase) => {
  const isDev = phase === PHASE_DEVELOPMENT_SERVER
  /**
   * @type {import('next').NextConfig}
   */
  const nextConfig = {
    assetPrefix: isDev ? undefined : 'https://cdn.mydomain.com',
  }
  return nextConfig
}
```

Next.js 将自动为您从 `/_next/` 路径 (`.next/static/` 文件夹) 加载的 JavaScript 和 CSS 文件使用资源前缀。例如，使用上述配置，以下对 JS 块的请求：

```
/_next/static/chunks/4b9b41aaa062cbbfeff4add70f256968c51ece5d.4d708494b3aed70c04f0.js
```

将变为：

```
https://cdn.mydomain.com/_next/static/chunks/4b9b41aaa062cbbfeff4add70f256968c51ece5d.4d708494b3aed70c04f0.js
```

将文件上传到特定 CDN 的确切配置取决于您选择的 CDN。您唯一需要在 CDN 上托管的文件夹是 `.next/static/` 的内容，正如上述 URL 请求指示的那样，应该上传为 `_next/static/`。**不要上传 `.next/` 文件夹的其余部分**，因为您不应向公众暴露服务器代码和其他配置。

虽然 `assetPrefix` 覆盖了对 `_next/static` 的请求，但它不影响以下路径：

<AppOnly>

- [public](/docs/app/api-reference/file-conventions/public-folder) 文件夹中的文件；如果您希望通过 CDN 托管这些资源，则需要自己引入前缀

</AppOnly>

<PagesOnly>

- [public](/docs/pages/api-reference/file-conventions/public-folder) 文件夹中的文件；如果您希望通过 CDN 托管这些资源，则需要自己引入前缀
- `/_next/data/` 对 `getServerSideProps` 页面的请求。这些请求将始终针对主域发出，因为它们不是静态的。
- `/_next/data/` 对 `getStaticProps` 页面的请求。这些请求将始终针对主域发出以支持 [Incremental Static Generation](/docs/pages/guides/incremental-static-regeneration)，即使您不使用它（为了保持一致性）。

</PagesOnly>
