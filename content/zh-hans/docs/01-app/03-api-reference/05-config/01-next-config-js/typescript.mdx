通过 `next.config.js` 中的 `typescript` 选项配置 TypeScript 行为：

```js filename="next.config.js"
module.exports = {
  typescript: {
    ignoreBuildErrors: false,
    tsconfigPath: 'tsconfig.json',
  },
}
```

## 选项

| 选项        | 类型    | 默认值            | 描述                                                     |
| ------------------- | --------- | ----------------- | ---------------------------------------------------------------- |
| `ignoreBuildErrors` | `boolean` | `false`           | 即使存在 TypeScript 错误，也允许生产构建完成。 |
| `tsconfigPath`      | `string`  | `'tsconfig.json'` | 自定义 `tsconfig.json` 文件的路径。                           |

## `ignoreBuildErrors`

当项目中存在 TypeScript 错误时，Next.js 会使您的**生产构建** (`next build`) 失败。

如果您希望 Next.js 即使在您的应用程序存在错误时也冒险生成生产代码，可以禁用内置类型检查步骤。

注意，这将完全跳过 TypeScript 类型检查步骤。它不会运行 TypeScript 并抑制错误，而是完全绕过了检查。

如果禁用，请确保您在构建或部署过程中运行类型检查，否则这可能非常危险。

```js filename="next.config.js"
module.exports = {
  typescript: {
    // !! WARN !!
    // Dangerously allow production builds to successfully complete even if
    // your project has type errors.
    // !! WARN !!
    ignoreBuildErrors: true,
  },
}
```

## `tsconfigPath`

为构建或工具使用不同的 TypeScript 配置文件：

```js filename="next.config.js"
module.exports = {
  typescript: {
    tsconfigPath: 'tsconfig.build.json',
  },
}
```

有关更多详细信息，请查看 [TypeScript 配置](/docs/app/api-reference/config/typescript#custom-tsconfig-path) 页面。
