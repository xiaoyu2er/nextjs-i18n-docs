title: useLightningcss
description: 启用实验性的 Lightning CSS 支持。
version: experimental
---

支持在 webpack 中使用 [Lightning CSS](https://lightningcss.dev) 的实验性功能。Lightning CSS 是一个用 Rust 编写的快速 CSS 转换器和压缩器。

如果未设置此选项，webpack 上的 Next.js 默认使用 [PostCSS](https://postcss.org/) 以及 [`postcss-preset-env`](https://www.npmjs.com/package/postcss-preset-env)。

自 Next 14.2 起，Turbopack 默认使用 Lightning CSS。此配置选项对 Turbopack 无效。Turbopack 始终使用 Lightning CSS。

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    useLightningcss: false, // default, ignored on Turbopack
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    useLightningcss: true, // disables PostCSS on webpack
  },
}

module.exports = nextConfig
```

## `lightningCssFeatures`

默认情况下，Lightning CSS 根据您的 [browserslist](https://browsersl.ist/) 目标决定编译哪些 CSS 功能。`lightningCssFeatures` 选项允许您覆盖此行为，强制特定功能始终被编译（`include`）或从不被编译（`exclude`），无视浏览器支持。

这适用于 webpack（当启用 `useLightningcss` 时）和 Turbopack。

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    useLightningcss: true,
    lightningCssFeatures: {
      // Always transpile these features, even if targets support them
      include: ['light-dark', 'oklab-colors'],
      // Never transpile these features, even if targets don't support them
      exclude: ['nesting'],
    },
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    useLightningcss: true,
    lightningCssFeatures: {
      // Always transpile these features, even if targets support them
      include: ['light-dark', 'oklab-colors'],
      // Never transpile these features, even if targets don't support them
      exclude: ['nesting'],
    },
  },
}

module.exports = nextConfig
```

### 选项

| 选项    | 类型       | 描述                                                                |
| --------- | ---------- | -------------------------------------------------------------------------- |
| `include` | `string[]` | 始终编译的功能，无视浏览器目标。               |
| `exclude` | `string[]` | 从不编译的功能，即使浏览器目标需要它们。 |

### 可用功能

单独功能：

| 功能名称                        | 描述                                          |
| ----------------------------------- | ---------------------------------------------------- |
| `nesting`                           | [CSS 嵌套 (CSS Nesting)](https://drafts.csswg.org/css-nesting/) |
| `not-selector-list`                 | 多个选择器的 `:not`                                |
| `dir-selector`                      | `:dir()` 选择器                                     |
| `lang-selector-list`                | 多个语言的 `:lang()`                                 |
| `is-selector`                       | `:is()` 选择器                                      |
| `text-decoration-thickness-percent` | `text-decoration-thickness` 中的百分比值             |
| `media-interval-syntax`             | 媒体查询范围区间语法                                  |
| `media-range-syntax`                | 媒体查询范围语法（width >= 600px）          |
| `custom-media-queries`              | `@custom-media` 规则                          |
| `clamp-function`                    | `clamp()` 函数                                      |
| `color-function`                    | `color()` 函数                                      |
| `oklab-colors`                      | `oklab()` 和 `oklch()` 颜色                     |
| `lab-colors`                        | `lab()` 和 `lch()` 颜色                           |
| `p3-colors`                         | Display P3 颜色                                     |
| `hex-alpha-colors`                  | 带 alpha 的 4 位和 8 位十六进制颜色           |
| `space-separated-color-notation`    | 空格分隔颜色表示法（rgb(0 0 0)）        |
| `font-family-system-ui`             | `system-ui` 字体家族                              |
| `double-position-gradients`         | 双位置渐变停止点                                     |
| `vendor-prefixes`                   | 厂商前缀属性和值                                       |
| `logical-properties`                | 逻辑属性和值                                         |
| `light-dark`                        | `light-dark()` 颜色函数                         |

组合组（启用多个功能的简写方式）：

| 组名称      | 包含内容                                                                                                                        |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `selectors`     | `nesting`, `not-selector-list`, `dir-selector`, `lang-selector-list`, `is-selector`                                             |
| `media-queries` | `media-interval-syntax`, `media-range-syntax`, `custom-media-queries`                                                           |
| `colors`        | `color-function`, `oklab-colors`, `lab-colors`, `p3-colors`, `hex-alpha-colors`, `space-separated-color-notation`, `light-dark` |

## 版本历史

| 版本  | 变更                                                                                                                                                                                      |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 16.2.0 | `lightningCssFeatures` 已添加。                                                                                                                                                                |
| 15.1.0 | `useSwcCss` 的支持从 Turbopack 中移除。                                                                                                                                          |
| 14.2.0 | Turbopack 的默认 CSS 处理器从 `@swc/css` 更改为 Lightning CSS。`useLightningcss` 在 Turbopack 上被忽略，并添加了旧版 `experimental.turbo.useSwcCss` 选项。 |
