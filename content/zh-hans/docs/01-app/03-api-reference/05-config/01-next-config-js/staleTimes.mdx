`staleTimes` 是一个实验性功能，允许在 [客户端缓存 (Client Cache)](/docs/app/glossary#client-cache) 中缓存页面片段。

您可以通过设置实验性的 `staleTimes` 标志来启用此实验性功能并提供自定义的重新验证时间：

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

`static` 和 `dynamic` 属性对应于根据不同类型的[链接预取](/docs/app/api-reference/components/link#prefetch)所设定的时间段（以秒为单位）。

- `dynamic` 属性用于页面既未静态生成也未完全预取（例如，通过 `prefetch={true}`）时。
  - 默认值：0 秒（未缓存）
- `static` 属性用于静态生成的页面，或当 `Link` 上的 `prefetch` 属性设置为 `true`，或调用 [`router.prefetch`](/docs/app/api-reference/functions/use-router) 时。
  - 默认值：5 分钟

> **提示：**
>
> - [加载边界 (Loading Boundaries)](/docs/app/api-reference/file-conventions/loading) 被视为在此配置定义的 `static` 期间可复用。
> - 这不影响 [部分渲染 (Partial Rendering)](/docs/app/getting-started/linking-and-navigating#client-side-transitions)，**意味着共享布局不会在每次导航时自动重新获取，只有变化的页面片段会。**
> - 这不会更改 [回退/前进缓存 (Back/Forward Caching)](/docs/app/glossary#client-cache) 行为，以防止布局偏移并避免丢失浏览器滚动位置。

### 版本历史

| 版本      | 变更                                                        |
| --------- | ----------------------------------------------------------- |
| `v15.0.0` | `dynamic` `staleTimes` 的默认值从 30 秒更改为 0 秒。         |
| `v14.2.0` | 引入了实验性的 `staleTimes` 功能。                           |
