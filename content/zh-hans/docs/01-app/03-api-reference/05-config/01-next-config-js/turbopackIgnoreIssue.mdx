`turbopack.ignoreIssue` 选项允许您过滤掉特定的 [Turbopack](/docs/app/api-reference/turbopack) 错误和警告，使其不出现在 CLI 输出或错误覆盖层中。这对于抑制已知但不影响您应用程序的警告非常有用，例如故意忽略的可选依赖项。

此选项仅在 Turbopack 下可用（`next dev --turbopack`）。

## 使用方法

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/vendor/**',
      },
    ],
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/vendor/**',
      },
    ],
  },
}

module.exports = nextConfig
```

## 选项

`ignoreIssue` 数组中的每个规则都是一个具有以下字段的对象：

| 字段                         | 类型               | 必填 | 描述                                |
| ----------------------------- | ------------------ | -------- | ------------------------------------------ |
| [`path`](#path)               | `string \| RegExp` | 是      | 用于匹配问题的文件路径 |
| [`title`](#title)             | `string \| RegExp` | 否       | 用于匹配问题标题            |
| [`description`](#description) | `string \| RegExp` | 否       | 用于匹配问题描述      |

当问题与规则中的 `path` **以及**所有其他指定字段匹配时，该问题将被抑制。如果仅提供 `path`，则匹配文件中的任何问题都会被抑制。

> **值得了解：** 问题标题和描述可能在 Turbopack 版本之间发生变化。`path` 字段通常稳定，但不能保证所有类型的问题都保持一致。如可能，优先使用更具体的 `path` 模式而不是 `title` 或 `description` 匹配。

### `path`

与问题所在的文件路径匹配的 **glob 模式**（字符串时）或 **正则表达式**。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    ignoreIssue: [
      // Glob pattern: suppress issues from any file under vendor/
      { path: '**/vendor/**' },
      // RegExp: suppress issues from files matching a pattern
      { path: /node_modules\/legacy-lib/ },
    ],
  },
}
```

### `title`

与问题标题匹配的 **精确字符串匹配**（字符串时）或 **正则表达式**。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/src/**',
        title: 'Module not found',
      },
    ],
  },
}
```

### `description`

与问题描述匹配的 **精确字符串匹配**（字符串时）或 **正则表达式**。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/src/**',
        description: /Cannot find module 'optional-dep'/,
      },
    ],
  },
}
```

## 示例

### 抑制可选依赖项的警告

如果您的代码在可选的 `require()` 调用周围使用了 `try/catch`，Turbopack 可能会报告 "Module not found" 警告。您可以抑制该警告：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/lib/optional-feature/**',
        title: 'Module not found',
      },
    ],
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/lib/optional-feature/**',
        title: 'Module not found',
      },
    ],
  },
}

module.exports = nextConfig
```

### 组合多个规则

您可以指定多个规则来抑制不同的问题：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    ignoreIssue: [
      { path: '**/vendor/**' },
      { path: '**/legacy/**', title: 'Module not found' },
      { path: /generated\//, description: /expected identifier/ },
    ],
  },
}
```
