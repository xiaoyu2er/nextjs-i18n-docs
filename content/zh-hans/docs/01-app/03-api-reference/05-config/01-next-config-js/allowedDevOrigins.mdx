Next.js 默认会阻止对仅开发环境的资源和端点的跨源请求，以防止未经授权的访问。

要配置 Next.js 应用以允许来自除服务器初始化时主机名（默认为 `localhost`）以外的来源的请求，请使用 `allowedDevOrigins` 配置选项。

`allowedDevOrigins` 允许您设置额外的来源，这些来源可以在开发模式下请求开发服务器。例如，要使用 `local-origin.dev` 代替仅使用 `localhost`，请打开 `next.config.js` 并添加 `allowedDevOrigins` 配置：

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```
