您可以指定一个名称来用作自定义构建目录，以替代默认的 `.next` 目录。

打开 `next.config.js` 并添加 `distDir` 配置：

``js filename="next.config.js"
module.exports = {
  distDir: 'build',
}
```

```js filename="next.config.js"
module.exports = {
  distDir: 'build',
}
```

现在如果您运行 `next build`，Next.js 将使用 `build` 目录代替默认的 `.next` 目录。

> `distDir` **不应** 超出您的项目目录。例如，`../build` 是 **无效** 目录。
