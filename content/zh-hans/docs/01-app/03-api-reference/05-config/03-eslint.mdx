Next.js 提供了一个 ESLint 配置包，[`eslint-config-next`](https://www.npmjs.com/package/eslint-config-next)，它可以轻松捕获应用程序中的常见问题。它包含了 [`@next/eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next) 插件，以及来自 [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react) 和 [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks) 的推荐规则集。

该包提供两种主要配置：

- **`eslint-config-next`**：包含 Next.js、React 和 React Hooks 规则的基准配置。同时支持 JavaScript 和 TypeScript 文件。
- **`eslint-config-next/core-web-vitals`**：包含基准配置的所有内容，并将影响 [核心网页指标 (Core Web Vitals)](https://web.dev/vitals/) 的规则从警告升级为错误。推荐用于大多数项目。

此外，对于 TypeScript 项目：

- **`eslint-config-next/typescript`**：添加来自 [`typescript-eslint`](https://typescript-eslint.io/) 的特定于 TypeScript 的 lint 规则。请与基准配置或 core-web-vitals 配置一起使用。

## 设置 ESLint

使用 ESLint CLI（扁平化配置）快速让 lint 工作起来：

1.  安装 ESLint 和 Next.js 配置：

    ```bash package="pnpm"
    pnpm add -D eslint eslint-config-next
    ```

    ```bash package="npm"
    npm i -D eslint eslint-config-next
    ```

    ```bash package="yarn"
    yarn add --dev eslint eslint-config-next
    ```

    ```bash package="bun"
    bun add -d eslint eslint-config-next
    ```

2.  创建包含 Next.js 配置的 `eslint.config.mjs`：

    ```js filename="eslint.config.mjs"
    import { defineConfig, globalIgnores } from 'eslint/config'
    import nextVitals from 'eslint-config-next/core-web-vitals'

    const eslintConfig = defineConfig([
      ...nextVitals,
      // 覆盖 eslint-config-next 的默认忽略项。
      globalIgnores([
        // eslint-config-next 的默认忽略项：
        '.next/**',
        'out/**',
        'build/**',
        'next-env.d.ts',
      ]),
    ])

    export default eslintConfig
    ```

3.  运行 ESLint：

    ```bash package="pnpm"
    pnpm exec eslint .
    ```

    ```bash package="npm"
    npx eslint .
    ```

    ```bash package="yarn"
    yarn eslint .
    ```

    ```bash package="bun"
    bunx eslint .
    ```

## 参考

`eslint-config-next` 包包含了来自以下 ESLint 插件的 `recommended` 规则集：

- [`eslint-plugin-react`](https://www.npmjs.com/package/eslint-plugin-react)
- [`eslint-plugin-react-hooks`](https://www.npmjs.com/package/eslint-plugin-react-hooks)
- [`@next/eslint-plugin-next`](https://www.npmjs.com/package/@next/eslint-plugin-next)

## 示例

### 在 monorepo 中指定根目录

如果您在根目录未安装 Next.js 的项目（如 monorepo）中使用 `@next/eslint-plugin-next`，您可以在 `eslint.config.mjs` 中使用 `settings` 属性告诉 `@next/eslint-plugin-next` 在哪里找到您的 Next.js 应用程序：

```js filename="eslint.config.mjs"
import { defineConfig } from 'eslint/config'
import eslintNextPlugin from '@next/eslint-plugin-next'

const eslintConfig = defineConfig([
  {
    files: ['**/*.{js,jsx,ts,tsx}'],
    plugins: {
      next: eslintNextPlugin,
    },
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  },
])

export default eslintConfig
```

`rootDir` 可以是一个路径（相对或绝对）、一个 glob（例如 `"packages/*/"`），或一个路径和/或 glob 的数组。

### 禁用规则

如果您想修改或禁用任何受支持插件（`react`、`react-hooks`、`next`）提供的规则，您可以在 `eslint.config.mjs` 中使用 `rules` 属性直接更改它们：

```js filename="eslint.config.mjs"
import { defineConfig, globalIgnores } from 'eslint/config'
import nextVitals from 'eslint-config-next/core-web-vitals'

const eslintConfig = defineConfig([
  ...nextVitals,
  {
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  },
  // Override default ignores of eslint-config-next.
  globalIgnores([
    // Default ignores of eslint-config-next:
    '.next/**',
    'out/**',
    'build/**',
    'next-env.d.ts',
  ]),
])

export default eslintConfig
```

### 使用 Core Web Vitals

在您的 ESLint 配置中启用 `eslint-config-next/core-web-vitals` 配置。

```js filename="eslint.config.mjs"
import { defineConfig, globalIgnores } from 'eslint/config'
import nextVitals from 'eslint-config-next/core-web-vitals'

const eslintConfig = defineConfig([
  ...nextVitals,
  // Override default ignores of eslint-config-next.
  globalIgnores([
    // Default ignores of eslint-config-next:
    '.next/**',
    'out/**',
    'build/**',
    'next-env.d.ts',
  ]),
])

export default eslintConfig
```

`eslint-config-next/core-web-vitals` 将 `@next/eslint-plugin-next` 中的某些 lint 规则从警告升级为错误，以帮助改善您的[核心网页指标 (Core Web Vitals)](https://web.dev/vitals/)。

> `eslint-config-next/core-web-vitals` 配置会自动包含在使用 [Create Next App](/docs/app/api-reference/cli/create-next-app) 构建的新应用程序中。

### 使用 TypeScript

除了 Next.js ESLint 规则外，`create-next-app --typescript` 还会通过 `eslint-config-next/typescript` 将特定于 TypeScript 的 lint 规则添加到您的配置中：

```js filename="eslint.config.mjs"
import { defineConfig, globalIgnores } from 'eslint/config'
import nextVitals from 'eslint-config-next/core-web-vitals'
import nextTs from 'eslint-config-next/typescript'

const eslintConfig = defineConfig([
  ...nextVitals,
  ...nextTs,
  // Override default ignores of eslint-config-next.
  globalIgnores([
    // Default ignores of eslint-config-next:
    '.next/**',
    'out/**',
    'build/**',
    'next-env.d.ts',
  ]),
])

export default eslintConfig
```

这些规则基于 [`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)。
更多详情请参阅 [typescript-eslint > 配置](https://typescript-eslint.io/linting/configs)。

### 使用 Prettier

ESLint 也包含代码格式化规则，这可能会与您现有的 [Prettier](https://prettier.io/) 设置冲突。我们建议在您的 ESLint 配置中包含 [eslint-config-prettier](https://github.com/prettier/eslint-config-prettier)，以使 ESLint 和 Prettier 协同工作。

首先，安装依赖项：

```bash package="pnpm"
pnpm add -D eslint-config-prettier
```

```bash package="npm"
npm i -D eslint-config-prettier
```

```bash package="yarn"
yarn add --dev eslint-config-prettier
```

```bash package="bun"
bun add -d eslint-config-prettier
```

然后，将 `prettier` 添加到您现有的 ESLint 配置中：

```js filename="eslint.config.mjs"
import { defineConfig, globalIgnores } from 'eslint/config'
import nextVitals from 'eslint-config-next/core-web-vitals'
import prettier from 'eslint-config-prettier/flat'

const eslintConfig = defineConfig([
  ...nextVitals,
  prettier,
  // Override default ignores of eslint-config-next.
  globalIgnores([
    // Default ignores of eslint-config-next:
    '.next/**',
    'out/**',
    'build/**',
    'next-env.d.ts',
  ]),
])

export default eslintConfig
```

### 在暂存的文件上运行 Lint

如果您想将 ESLint 与 [lint-staged](https://github.com/okonet/lint-staged) 一起使用，以对暂存的 git 文件运行 linter，请将以下内容添加到项目根目录的 `.lintstagedrc.js` 文件中：

```js filename=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `eslint --fix ${filenames
    .map((f) => `"${path.relative(process.cwd(), f)}"`)
    .join(' ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## 迁移现有配置

如果您的应用程序中已经配置了 ESLint，根据您的设置，有两种方法可以集成 Next.js lint 规则。

#### 直接使用插件

如果您已经配置了以下任何一项，请直接使用 `@next/eslint-plugin-next`：

- 单独安装或通过其他配置（如 `airbnb` 或 `react-app`）安装了冲突的插件：
  - `react`
  - `react-hooks`
  - `jsx-a11y`
  - `import`
- 与 Next.js 默认值不同的自定义 `parserOptions`（仅当您[自定义了 Babel 配置](/docs/pages/guides/babel)时）
- 带有自定义 Node.js 和/或 TypeScript [解析器](https://github.com/benmosher/eslint-plugin-import#resolvers)的 `eslint-plugin-import`

在这些情况下，直接使用 `@next/eslint-plugin-next` 以避免冲突：

首先，安装插件：

```bash package="pnpm"
pnpm add -D @next/eslint-plugin-next
```

```bash package="npm"
npm i -D @next/eslint-plugin-next
```

```bash package="yarn"
yarn add --dev @next/eslint-plugin-next
```

```bash package="bun"
bun add -d @next/eslint-plugin-next
```

然后将其添加到您的 ESLint 配置中：

```js filename="eslint.config.mjs"
import { defineConfig } from 'eslint/config'
import nextPlugin from '@next/eslint-plugin-next'

const eslintConfig = defineConfig([
  // Your other configurations...
  {
    files: ['**/*.{js,jsx,ts,tsx}'],
    plugins: {
      '@next/next': nextPlugin,
    },
    rules: {
      ...nextPlugin.configs.recommended.rules,
    },
  },
])

export default eslintConfig
```

这种方法消除了在多个配置中导入相同插件或解析器时可能发生的冲突或错误的风险。

#### 添加到现有配置

如果您要将 Next.js 添加到现有的 ESLint 设置中，请将 Next.js 配置展开到您的数组中：

```js filename="eslint.config.mjs"
import nextConfig from 'eslint-config-next/core-web-vitals'
// Your other config imports...

const eslintConfig = [
  // Your other configurations...
  ...nextConfig,
]

export default eslintConfig
```

当您展开 `...nextConfig` 时，您添加了多个包含文件模式、插件、规则、忽略项和解析器设置的配置对象。ESLint 按顺序应用配置，因此对于匹配的文件，后面的规则可以覆盖前面的规则。

> **请注意：** 这种方法适用于简单的设置。如果您的现有配置很复杂，具有冲突的特定文件模式或插件配置，请考虑直接使用插件（如上所示）以获得更精细的控制。

| 版本      | 变更                                                                                                                                                                                                                 |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `v16.0.0` | `next lint` 和 `eslint` next.config.js 选项已被移除，转而使用 ESLint CLI。提供了一个[代码迁移工具](/docs/app/guides/upgrading/codemods#migrate-from-next-lint-to-eslint-cli)来帮助您进行迁移。 |
