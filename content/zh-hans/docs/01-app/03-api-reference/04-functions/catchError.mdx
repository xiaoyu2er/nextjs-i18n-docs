unstable_catchError 函数创建一个组件，将其子元素包裹在错误边界中。它为 [`error.js`](/docs/app/api-reference/file-conventions/error) 文件格式提供了一种程序化替代方案，使您能够在组件树的任何位置实现组件级别的错误恢复。

unstable_catchError 可从 [客户端组件](/docs/app/getting-started/server-and-client-components) 中调用。

```tsx filename="app/custom-error-boundary.tsx" switcher
'use client'

import { unstable_catchError } from 'next/error'

function ErrorFallback(
  props: { title: string },
  { error, unstable_retry: retry }: { error: Error; unstable_retry: () => void }
) {
  return (
    <div>
      <h2>{props.title}</h2>
      <p>{error.message}</p>
      <button onClick={() => retry()}>Try again</button>
    </div>
  )
}

export default unstable_catchError(ErrorFallback)
```

```jsx filename="app/custom-error-boundary.js" switcher
'use client'

import { unstable_catchError } from 'next/error'

function ErrorFallback(props, { error, unstable_retry: retry }) {
  return (
    <div>

      <p>{error.message}</p>
      <button onClick={() => retry()}>Try again</button>
    </div>
  )
}

export default unstable_catchError(ErrorFallback)
```

## 参考

### 参数

unstable_catchError 接受一个参数：

```ts
const ErrorWrapper = unstable_catchError(fallback)
```

#### 降级内容 `fallback`

捕获错误时用于渲染错误 UI 的函数。它接收两个参数：

- `props` — 传递给包装组件的 props（不包括 `children`）。
- `errorInfo` — 包含错误信息的对象：

| 属性         | 类型                                                                                        | 描述                                                                                                                                              |
| ---------------- | ------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `error`          | [`Error`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Error) | 捕获到的错误实例。                                                                                                                      |
| `unstable_retry` | `() => void`                                                                                | 重新获取并重新呈现错误边界的内容。如果成功，降级内容将被重新呈现的结果替换。                            |
| `reset`          | `() => void`                                                                                | 重置错误状态并重新呈现，不重新获取。大多数情况下请使用 [`retry()`](/docs/app/api-reference/file-conventions/error#unstable_retry)。|

`fallback` 函数必须是客户端组件（或在带 [`use client`](/docs/app/getting-started/server-and-client-components) 的模块中定义）。

### 返回值

unstable_catchError 返回一个 React 组件，该组件会：

- 接受与降级内容的第一个参数相同的 props，以及 `children`。
- 将 `children` 包裹在错误边界中。
- 在 `children` 捕获错误时渲染 `fallback`。

## 示例

### 客户端组件

定义降级内容并使用返回的组件包裹 UI 的一部分：

```tsx filename="app/some-component.tsx" switcher
export default function Component({ children }: { children: React.ReactNode }) {
  return <ErrorWrapper title="Dashboard Error">{children}</ErrorWrapper>
}
```

```jsx filename="app/some-component.js" switcher
import ErrorWrapper from '../custom-error-boundary'

export default function Component({ children }) {
  return <ErrorWrapper title="Dashboard Error">{children}</ErrorWrapper>

```

### 错误恢复

使用 `retry()` 提示用户从错误中恢复。调用时，函数会重新获取并重新呈现错误边界的内容。如果成功，降级内容将被重新呈现的结果替换。

大多数情况下，请使用 `retry()` 而不是 `reset()`。`reset()` 函数仅清除错误状态并重新呈现，不重新获取，这意味着它无法从服务端组件错误中恢复。

```tsx filename="app/custom-error-boundary.tsx" highlight={9,12} switcher
'use client'

import { unstable_catchError } from 'next/error'

function ErrorFallback(
  props: {},
  {
    error,
    unstable_retry: retry,
    reset,
  }: { error: Error; unstable_retry: () => void; reset: () => void }
) {
  return (
    <div>
      <p>{error.message}</p>
      <button onClick={() => retry()}>Try again</button>
      <button onClick={() => reset()}>Reset</button>
    </div>
  )
}

export default unstable_catchError(ErrorFallback)
```

```jsx filename="app/custom-error-boundary.js" highlight={7,10} switcher
'use client'

import { unstable_catchError } from 'next/error'

function ErrorFallback(props, { error, unstable_retry: retry, reset }) {
  return (
    <div>
      <p>{error.message}</p>
      <button onClick={() => retry()}>Try again</button>
      <button onClick={() => reset()}>Reset</button>
    </div>
  )
}

export default unstable_catchError(ErrorFallback)
```

### 服务端渲染的错误降级内容

您可以以 prop 的形式传递服务端渲染内容来显示数据驱动的降级 UI。这通过将服务端组件渲染为 `React.ReactNode` prop 来实现，当捕获错误时，降级内容会显示它。

> **了解要点**：此模式会在每次页面渲染时立即渲染降级内容，即使没有发生错误也是如此。对于大多数用例，更简单的客户端降级内容就足够了。

```tsx filename="app/error-boundary.tsx" switcher
'use client'

import { unstable_catchError } from 'next/error'
import type { ErrorInfo } from 'next/error'

function ErrorFallback(
  props: { fallback: React.ReactNode },
  errorInfo: ErrorInfo
) {
  return props.fallback
}

export default unstable_catchError(ErrorFallback)
```

```jsx filename="app/error-boundary.js" switcher
'use client'

import { unstable_catchError } from 'next/error'

function ErrorFallback(props, errorInfo) {
  return props.fallback
}

export default unstable_catchError(ErrorFallback)
```

```tsx filename="app/some-component.tsx" switcher
import ErrorBoundary from '../error-boundary'

async function ErrorFallback() {
  const data = await getData()
  return <div>{data.message}</div>
}

export default function Component({ children }: { children: React.ReactNode }) {
  return <ErrorBoundary fallback={<ErrorFallback />}>{children}</ErrorBoundary>
}
```

```jsx filename="app/some-component.js" switcher
import ErrorBoundary from '../error-boundary'

async function ErrorFallback() {
  const data = await getData()
  return <div>{data.message}</div>
}

export default function Component({ children }) {
  return <ErrorBoundary fallback={<ErrorFallback />}>{children}</ErrorBoundary>
}
```

> **了解要点**：
>
> - 与局限于路由范围的 `error.js` 文件格式不同，`unstable_catchError` 可用于包裹组件树的任何部分以实现组件级别的错误恢复。
> - 传递给包装组件的 props 会转发到降级函数，使得用不同配置创建可重用的错误 UI 变得简单。
> - 您不需要用 `unstable_catchError` 包装 `error.js` 默认导出。[`error.js`](/docs/app/api-reference/file-conventions/error) 文件格式已经在 Next.js 提供的内置错误边界内渲染。
