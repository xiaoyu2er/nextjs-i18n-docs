``useLinkStatus`` 钩子允许您追踪 ``<Link>`` 的 **pending** 状态。用于细微的即时反馈，例如在导航完成期间，在被点击的链接上显示辉光效果。优先使用 ``loading.js`` 的路由级回退方案，以及预加载以实现即时过渡。

`useLinkStatus` 在以下情况下很有用：

- [预取 (Prefetching)](/docs/app/getting-started/linking-and-navigating#prefetching) 被禁用或正在进行中，这意味着导航被阻塞。
- 目标路由是动态的，**并且** 不包含能够实现即时导航的 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 文件。

```tsx filename="app/hint.tsx" switcher
'use client'

import Link from 'next/link'
import { useLinkStatus } from 'next/link'

function Hint() {
  const { pending } = useLinkStatus()
  return (
    <span aria-hidden className={`link-hint ${pending ? 'is-pending' : ''}`} />
  )
}

export default function Header() {
  return (
    <header>
      <Link href="/dashboard" prefetch={false}>
        <span className="label">Dashboard</span> <Hint />
      </Link>
    </header>
  )
}
```

> **小提示**:
>
> - ``useLinkStatus`` 必须用于 ``Link`` 组件的后代组件内
> - 当在 ``Link`` 组件上设置了 ``prefetch={false}`` 时，此钩子最为有用
> - 如果链接的路由已预加载，将跳过 pending 状态
> - 快速连续点击多个链接时，仅显示最后一个链接的 pending 状态
> - 此钩子在 Pages Router 中不受支持，始终返回 ``{ pending: false }``
> - 行内指示器容易引入布局偏移。优先使用固定大小、始终渲染的提示元素并切换其透明度，或使用动画。

## 可能不需要 ``useLinkStatus``

在添加即时反馈之前，请考虑是否：

- 目标路由是静态的且在生产环境中已预加载，因此等待阶段可能被跳过。
- 该路由包含 ``loading.js`` 文件，使用路由级回退可实现即时过渡。

导航通常很快。当您发现过渡缓慢时，可将 ``useLinkStatus`` 作为快速补丁，然后通过预加载或 ``loading.js`` 回退来修复根本原因并进行迭代。

## 参数

```tsx
const { pending } = useLinkStatus()
```

`useLinkStatus` 不接受任何参数。

## 返回

`useLinkStatus` 返回一个包含单个属性的对象：

| 属性    | 类型    | 描述                           |
| ------- | ------- | ------------------------------ |
| pending | boolean | 在历史记录更新前为 `true`，之后为 `false` |

## 示例

### 行内链接提示

添加一个细微、固定大小、不影响布局的提示，以便在预加载未完成时确认点击。

```tsx filename="app/components/loading-indicator.tsx" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return (
    <span aria-hidden className={`link-hint ${pending ? 'is-pending' : ''}`} />
  )
}
```

```jsx filename="app/components/loading-indicator.js" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return (
    <span aria-hidden className={`link-hint ${pending ? 'is-pending' : ''}`} />
  )
}
```

```tsx filename="app/shop/layout.tsx" switcher
import Link from 'next/link'
import LoadingIndicator from './components/loading-indicator'

const links = [
  { href: '/shop/electronics', label: 'Electronics' },
  { href: '/shop/clothing', label: 'Clothing' },
  { href: '/shop/books', label: 'Books' },
]

function Menubar() {
  return (
    <div>
      {links.map((link) => (
        <Link key={link.label} href={link.href}>
          <span className="label">{link.label}</span> <LoadingIndicator />
        </Link>
      ))}
    </div>
  )
}

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <Menubar />
      {children}
    </div>
  )
}
```

## 优雅处理快速导航

如果导航到新路由很快，用户可能会看到提示的不必要闪烁。一种改进用户体验的方式是仅当导航耗时完成时才显示提示，即添加初始动画延迟（例如 100ms）并设定动画从不可见状态开始（例如 ``opacity: 0``）。

```css filename="app/styles/global.css"
.link-hint {
  display: inline-block;
  width: 0.6em;
  height: 0.6em;
  margin-left: 0.25rem;
  border-radius: 9999px;
  background: currentColor;
  opacity: 0;
  visibility: hidden; /* reserve space without showing the hint */
}

.link-hint.is-pending {
  /* Animation 1: fade in after 100ms and keep final opacity */
  /* Animation 2: subtle pulsing while pending */
  visibility: visible;
  animation-name: fadeIn, pulse;
  animation-duration: 200ms, 1s;
  /* Appear only if navigation actually takes time */
  animation-delay: 100ms, 100ms;
  animation-timing-function: ease, ease-in-out;
  animation-iteration-count: 1, infinite;
  animation-fill-mode: forwards, none;
}

@keyframes fadeIn {
  to {
    opacity: 0.35;
  }
}
@keyframes pulse {
  50% {
    opacity: 0.15;
  }
}
```
