title: 更新标签

`updateTag` 允许您从 [Server Actions (服务器操作)](/docs/app/getting-started/mutating-data) 中按需更新特定缓存标签的缓存数据。

此函数专为 **read-your-own-writes (读写一致性)** 场景设计，在此场景中，用户做出更改（例如创建帖子），UI 会立即显示该更改，而不是过时数据。

## 使用方法

`updateTag` **仅** 可从 [Server Actions (服务器操作)](/docs/app/getting-started/mutating-data) 中调用。它不能用于 Route Handlers、Client Components 或其他任何上下文中。

如果您需要在 Route Handlers 或其他上下文中使缓存标签失效，请使用 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag) 代替。

> **要知道**: `updateTag` 会立即使指定标签的缓存数据过期。下一个请求将等待获取新数据，而不是从缓存提供过时内容，确保用户立即看到他们的更改。

## 参数

```tsx
updateTag(tag: string): void;
```

- `tag`: 表示与您要更新的数据关联的缓存标签的字符串。不得超过 256 个字符。此值区分大小写。

标签首先必须分配给缓存数据。您可以通过以下两种方法实现：

- 在 `fetch` 中结合使用 [`next.tags`](/docs/app/api-reference/functions/fetch) 选项来缓存外部 API 请求：

```tsx
fetch(url, { next: { tags: ['posts'] } })
```

- 在带有 `'use cache'` 指令的缓存函数或组件中使用 [`cacheTag`](/docs/app/api-reference/functions/cacheTag)：

```tsx
import { cacheTag } from 'next/cache'

async function getData() {
  'use cache'
  cacheTag('posts')
  // ...
}
```

## 返回

`updateTag` 不返回值。

## 与 revalidateTag 的区别

虽然 `updateTag` 和 `revalidateTag` 都会使缓存数据失效，但它们的目的不同：

- **`updateTag`**:
  - 仅限 Server Actions 中使用
  - 下一个请求等待新数据（不提供服务过时的内容）
  - 专为 read-your-own-writes（读写一致性）场景设计

- **`revalidateTag`**:
  - 可在 Server Actions 和 Route Handlers 中使用
  - 使用 `profile="max"`（推荐）：在后台获取新数据的同时提供缓存数据（stale-while-revalidate）
  - 使用自定义 profile：可配置任何缓存生命周期配置文件以进行高级使用
  - 未指定 profile：遗留行为，等同于 `updateTag`

## 示例

### 在 Server Action 中使用 read-your-own-writes

```ts filename="app/actions.ts" switcher
'use server'

import { updateTag } from 'next/cache'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  const title = formData.get('title')
  const content = formData.get('content')

  // Create the post in your database
  const post = await db.post.create({
    data: { title, content },
  })

  // Invalidate cache tags so the new post is immediately visible
  // 'posts' tag: affects any page that displays a list of posts
  updateTag('posts')
  // 'post-{id}' tag: affects the individual post detail page
  updateTag(`post-${post.id}`)

  // Redirect to the new post - user will see fresh data, not cached
  redirect(`/posts/${post.id}`)
}
```

### 在服务器操作 (Server Actions) 外部使用时报错

```ts filename="app/api/posts/route.ts" switcher
import { updateTag } from 'next/cache'

export async function POST() {
  // This will throw an error
  updateTag('posts')
  // Error: updateTag can only be called from within a Server Action

  // Use revalidateTag instead in Route Handlers
  revalidateTag('posts', 'max')
}
```

## 何时使用 updateTag

当满足以下条件时使用 `updateTag`：

- 您处于 Server Action 中
- 您需要为 read-your-own-writes（读写一致性）立即使缓存失效
- 您希望确保下一个请求能看到更新后的数据

当满足以下条件时使用 `revalidateTag`：

- 您处于 Route Handler 或其他非动作上下文中
- 您想要 stale-while-revalidate（带过期缓存的重验证）语义
- 您正在为缓存失效构建 webhook 或 API 端点
