title: revalidateTag

`revalidateTag` 允许您按需使特定缓存标签的缓存数据失效。

此函数适用于更新可以接受轻微延迟的内容，例如博客文章、产品目录或文档。用户可以在后台加载最新数据时接收到过时内容。

## 使用方法

`revalidateTag` 可以在服务端函数和路由处理程序中调用。

`revalidateTag` 不能在客户端组件或 Proxy 中调用，因为它仅在服务器环境中有效。

### 重新验证行为

重新验证行为取决于您是否提供第二个参数：

- **使用 `profile="max"` (推荐)**: 标签条目被标记为过时，下次访问带有该标签的资源时，它将使用 stale-while-revalidate 语义。这意味着在后台获取新内容的同时提供过时内容。
- **使用自定义缓存生命周期配置**: 对于高级用法，您可以指定应用程序定义的任何缓存生命周期配置，从而允许根据您特定的缓存需求定制自定义重新验证行为。
- **不提供第二个参数 (已弃用)**: 标签条目立即过期，下次对该资源的请求将变为阻塞式重新验证/缓存未命中。此行为现已弃用，您应改用 `profile="max"` 或迁移到 [`updateTag`](/docs/app/api-reference/functions/updateTag)。

> **值得注意的是**: 当使用 `profile="max"` 时，`revalidateTag` 将标记的数据标记为过时，但只有在使用该标签的页面下次被访问时才会获取新数据。这意味着调用 `revalidateTag` 不会立即一次性触发许多重新验证。仅在下次访问使用该标签的任何页面时才会发生失效。

## 参数

```ts
revalidateTag(tag: string, profile: string | { expire?: number }): void;
```

- `tag`: 表示与要重新验证的数据关联的缓存标签的字符串。不得超过 256 个字符。此值是区分大小写的。
- `profile`: 指定重新验证行为的字符串。推荐值为 `"max"`，提供 stale-while-revalidate 语义，或者 [`cacheLife`](/docs/app/api-reference/config/next-config-js/cacheLife) 中定义的任何其他默认或自定义配置。另外，您可以传递带有 `expire` 属性的对象以实现自定义过期行为。

标签必须首先分配给缓存的数据。您可以通过以下两种方式实现：

- 使用 `fetch` 的 [`next.tags`](/docs/app/api-reference/functions/fetch) 选项缓存外部 API 请求：

```tsx
fetch(url, { next: { tags: ['posts'] } })
```

- 在使用 `'use cache'` 指令的缓存函数或组件内部使用 [`cacheTag`](/docs/app/api-reference/functions/cacheTag)：

```tsx
import { cacheTag } from 'next/cache'

async function getData() {
  'use cache'
  cacheTag('posts')
  // ...
}
```

> **值得注意的是**: 单参数形式 `revalidateTag(tag)` 已弃用。如果抑制 TypeScript 错误，它目前可以工作，但这种行为可能会在将来的版本中移除。请更新为双参数签名。

## 返回

`revalidateTag` 不返回值。

## 与 revalidatePath 的关系

`revalidateTag` 会使所有使用该标签的页面中具有特定标签的数据失效，而 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 仅使特定的页面或布局路径失效。

> **值得注意的是**: 这些函数服务于不同的目的，可能需要一起使用以确保全面的数据一致性。有关详细的示例和注意事项，请参阅 [与 revalidateTag 和 updateTag 的关系](/docs/app/api-reference/functions/revalidatePath#relationship-with-revalidatetag-and-updatetag) 以获取更多信息。

## 示例

以下示例展示了如何在不同上下文中使用 `revalidateTag`。在这两种情况下，我们都使用 `profile="max"` 将数据标记为过时并使用 stale-while-revalidate 语义，这是大多数用例的推荐方法。

### 服务器操作

```ts filename="app/actions.ts" switcher
'use server'

import { revalidateTag } from 'next/cache'

export default async function submit() {
  await addPost()
  revalidateTag('posts', 'max')
}
```

```js filename="app/actions.js" switcher
'use server'

import { revalidateTag } from 'next/cache'

export default async function submit() {
  await addPost()
  revalidateTag('posts', 'max')
}
```

### 路由处理程序

```ts filename="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { revalidateTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')

  if (tag) {
    revalidateTag(tag, 'max')
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing tag to revalidate',
  })
}
```

```js filename="app/api/revalidate/route.js" switcher
import { revalidateTag } from 'next/cache'

export async function GET(request) {
  const tag = request.nextUrl.searchParams.get('tag')

  if (tag) {
    revalidateTag(tag, 'max')
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing tag to revalidate',
  })
}
```

> **值得注意的是**: 对于需要立即过期的 Webhook 或第三方服务，您可以将 `{ expire: 0 }` 作为第二个参数传递：`revalidateTag(tag, { expire: 0 })`。当外部系统调用您的路由处理器并要求数据立即过期时，这种模式是必要的。对于其他所有情况，建议在 Server Actions 中使用 [`updateTag`](/docs/app/api-reference/functions/updateTag) 进行即时更新。
