`usePathname` 是一个**客户端组件 (Client Component)** 钩子，它允许你读取当前 URL 的**路径名 (pathname)**。

> **值得了解**：当启用 [`cacheComponents`](/docs/app/api-reference/config/next-config-js/cacheComponents) 时，如果您的路由具有动态参数，`usePathname` 可能需要在其周围添加 `Suspense` 边界。如果您使用 `generateStaticParams`，则 `Suspense` 边界是可选的

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'

export default function ExampleClientComponent() {
  const pathname = usePathname()
  return <p>Current pathname: {pathname}</p>
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { usePathname } from 'next/navigation'

export default function ExampleClientComponent() {
  const pathname = usePathname()
  return <p>Current pathname: {pathname}</p>
}
```

`usePathname` 故意要求使用[客户端组件 (Client Component)](/docs/app/getting-started/server-and-client-components)。重要的是要注意，客户端组件 (Client Components) 并非一种性能劣化。它们是[服务器组件 (Server Components)](/docs/app/getting-started/server-and-client-components) 架构不可或缺的一部分。

例如，一个使用 `usePathname` 的客户端组件 (Client Component) 将在初始页面加载时被渲染成 HTML。当导航到新路由时，此组件不需要重新获取。相反，该组件只会被下载一次（在客户端 JavaScript 包中），然后根据当前状态重新渲染。

> **值得了解**:
>
> - 从 [服务器组件 (Server Component)](/docs/app/getting-started/server-and-client-components) 读取当前 URL 不受支持。此设计是有意为之，以支持布局状态在页面导航之间保持。
> - 如果您的页面正在静态预渲染，并且您的应用在 `next.config` 或 [重写 (rewrites)](/docs/app/api-reference/config/next-config-js/rewrites) 或 [Proxy](/docs/app/api-reference/file-conventions/proxy) 文件中，使用 `usePathname()` 读取路径名可能会导致水合不匹配错误（hydration mismatch errors）——因为初始值来自服务器，在路由后可能与实际浏览器路径名不匹配。请参阅我们的 [示例](#avoid-hydration-mismatch-with-rewrites) 以缓解此问题。

<details>

<summary>与 Pages Router 的兼容性</summary>

如果您使用的是 `usePathname` 的组件，并且它们被导入到 Pages Router 的路由中，请注意，如果路由器尚未初始化，`usePathname` 可能会返回 `null`。这种情况可能发生在 [回退路由 (fallback routes)](/docs/pages/api-reference/functions/get-static-paths#fallback-true) 期间或 Pages Router 中的 [自动静态优化 (Automatic Static Optimization)](https://nextjs.org/docs/pages/building-your-application/rendering/static#automatic-static-optimization) 期间。

为了增强路由系统之间的兼容性，如果您的项目同时包含 `app` 和 `pages` 目录，Next.js 将自动调整 `usePathname` 的返回类型。

</details>

## 参数

```tsx
const pathname = usePathname()
```

`usePathname` 不接受任何参数。

## 返回

`usePathname` 返回一个表示当前 URL 路径名的字符串。例如：

| URL                 | 返回值                |
| ------------------- | --------------------- |
| `/`                 | `'/'`                 |
| `/dashboard`        | `'/dashboard'`        |
| `/dashboard?v=2`    | `'/dashboard'`        |
| `/blog/hello-world` | `'/blog/hello-world'` |

## 示例

### 根据路由变化执行操作

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

function ExampleClientComponent() {
  const pathname = usePathname()
  const searchParams = useSearchParams()
  useEffect(() => {
    // Do something here...
  }, [pathname, searchParams])
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

function ExampleClientComponent() {
  const pathname = usePathname()
  const searchParams = useSearchParams()
  useEffect(() => {
    // Do something here...
  }, [pathname, searchParams])
}
```

### 在使用重写 (rewrites) 时避免水合 (hydration) 不匹配

当页面被预渲染时，HTML 基于源路径名生成。如果随后通过 `next.config` 或 `Proxy` 中的重写访问该页面，浏览器 URL 可能不同，`usePathname()` 将在客户端读取重写后的路径名。

为了避免水合 (hydration) 不匹配，请设计用户界面，使得只有小部分隔离的部分依赖于客户端路径名。在服务器端渲染稳定的回退 (fallback)，并在挂载后更新该部分。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useEffect, useState } from 'react'
import { usePathname } from 'next/navigation'

export default function PathnameBadge() {
  const pathname = usePathname()
  const [clientPathname, setClientPathname] = useState('')

  useEffect(() => {
    setClientPathname(pathname)
  }, [pathname])

  return (
    <p>
      Current pathname: <span>{clientPathname}</span>
    </p>
  )
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useEffect, useState } from 'react'
import { usePathname } from 'next/navigation'

export default function PathnameBadge() {
  const pathname = usePathname()
  const [clientPathname, setClientPathname] = useState('')

  useEffect(() => {
    setClientPathname(pathname)
  }, [pathname])

  return (
    <p>
      Current pathname: <span>{clientPathname}</span>
    </p>
  )
}
```
