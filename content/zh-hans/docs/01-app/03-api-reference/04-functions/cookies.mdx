`cookies` 是一个 **异步** 函数，允许你在 [服务器组件 (Server Components)](/docs/app/getting-started/server-and-client-components) 中读取 HTTP 传入请求的 cookies，并在 [服务器函数 (Server Functions)](/docs/app/getting-started/mutating-data) 或 [路由处理器 (Route Handlers)](/docs/app/api-reference/file-conventions/route) 中读取/写入传出请求的 cookies。

```tsx filename="app/page.tsx" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

```js filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

## 参考

### 方法

以下是可用的方法：

| 方法                      | 返回类型      | 描述                                                                     |
| --------------------------- | ---------------- | ------------------------------------------------------------------------------- |
| `get('name')`               | 对象           | 接受 cookie 名称并返回包含名称和值的对象。            |
| `getAll()`                  | 对象数组       | 返回所有匹配名称的 cookies 的列表。                         |
| `has('name')`               | 布尔值         | 接受 cookie 名称并返回表示该 cookie 是否存在结果的布尔值。      |
| `set(name, value, options)` | -                | 接受 cookie 名称、值和选项并设置传出请求的 cookie。 |
| `delete(name)`              | -                | 接受 cookie 名称并删除该 cookie。                                   |
| `toString()`                | 字符串         | 返回 cookies 的字符串表示。                                 |

### 选项

设置 Cookie 时，`options` 对象支持以下属性：

| 选项           | 类型                                    | 描述                                                                        |
| ------------- | -------------------------------------- | ---------------------------------------------------------------------------------- |
| `name`        | String                                 | 指定 cookie 的名称。                                                  |
| `value`       | String                                 | 指定要存储在 cookie 中的值。                                    |
| `expires`     | Date                                   | 定义 cookie 过期的确切日期。                                |
| `maxAge`      | Number                                 | 设置 cookie 的生存时间（秒）。                                             |
| `domain`      | String                                 | 指定 cookie 可用的域名。                                |
| `path`        | String, default: `'/'`                 | 将 cookie 的范围限制在域名内的特定路径。                    |
| `secure`      | Boolean                                | 确保 cookie 仅通过 HTTPS 连接发送以加强安全性。         |
| `httpOnly`    | Boolean                                | 限制 cookie 仅用于 HTTP 请求，防止客户端访问。              |
| `sameSite`    | Boolean, `'lax'`, `'strict'`, `'none'` | 控制 cookie 的跨站请求行为。                                 |
| `priority`    | String (`"low"`, `"medium"`, `"high"`) | 指定 cookie 的优先级                                                    |
| `partitioned` | Boolean                                | 指示 cookie 是否 [已分区](https://github.com/privacycg/CHIPS)。 |

唯一具有默认值的选项是 `path`。

要了解这些选项的更多信息，请参阅 [MDN 文档](https://developer.mozilla.org/en-US/docs/Web/HTTP/Cookies)。

## 小知识

- `cookies` 是一个 **异步** 函数，返回一个 Promise。你必须使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函数来访问 cookies。
  - 在 14 及更早版本中，`cookies` 是一个同步函数。为了协助向后兼容性，在 Next.js 15 中你仍然可以同步访问它，但这行为未来将被弃用。
- `cookies` 是一个 [请求时 (Request-time) API](/docs/app/glossary#request-time-apis)，其返回值无法提前知晓。在 layout 或 page 中使用它会将路由纳入 [动态渲染 (dynamic rendering)](/docs/app/glossary#dynamic-rendering)。
- `.delete` 方法仅可在以下情况调用：
  - 在 [服务器函数 (Server Function)](/docs/app/getting-started/mutating-data) 或 [路由处理器 (Route Handler)](/docs/app/api-reference/file-conventions/route) 中。
  - 如果它属于与 `.set` 调用相同的域。对于通配符域，特定的子域必须完全匹配。此外，代码必须在与要删除的 cookie 相同的协议 (HTTP 或 HTTPS) 上执行。
- HTTP 不允许在流式传输开始后设置 cookie，因此你必须在 [服务器函数 (Server Function)](/docs/app/getting-started/mutating-data) 或 [路由处理器 (Route Handler)](/docs/app/api-reference/file-conventions/route) 中使用 `.set`。

## 理解 Server Components 中的 Cookie 行为

在服务器组件中使用 Cookie 时，重要的是要理解 Cookie 本质上是一种客户端存储机制：

- **读取 cookies** 在 服务器组件 (Server Components) 中有效，因为你正在访问客户端浏览器在 HTTP 请求标头中发送到服务器的 cookie 数据。
- **设置 cookies** 在 Server Component 渲染期间不受支持。要修改 cookies，请从客户端调用 服务器函数 (Server Function) 或使用 路由处理器 (Route Handler)。

服务器只能发送指令（通过 `Set-Cookie` 标头）通知浏览器存储 cookies—实际存储发生在客户端。这就是为什么修改状态的 cookie 操作（`.set`, `.delete`）必须在 [服务器函数 (Server Function)](/docs/app/getting-started/mutating-data) 或 [路由处理器 (Route Handler)](/docs/app/api-reference/file-conventions/route) 中执行，因为这些地方可以正确设置响应标头。

## 理解 服务器函数 (Server Function) 中的 Cookie 行为

在 服务器函数 (Server Function) 中设置或删除 cookie 后，当该函数用作 [服务器操作 (Server Action)](/docs/app/getting-started/mutating-data#what-are-server-functions)（例如，传递给表单的 `action` 属性）时，Next.js 可以在单个服务器往返中返回更新的 UI 和新数据。请参见 [缓存和重新验证 (Caching and Revalidating)](/docs/app/getting-started/caching)。

UI 不会卸载，但依赖于来自服务器数据的 effects 将会重新运行。

要刷新缓存数据，也请在该函数内调用 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 或 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag)。

## 示例

### 获取 Cookie

你可以使用 `(await cookies()).get('name')` 方法来获取单个 Cookie：

```tsx filename="app/page.tsx" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

```jsx filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

### 获取所有 Cookie

你可以使用 `(await cookies()).getAll()` 方法来获取所有名称匹配的 Cookie。如果未指定 `name`，则返回所有可用的 Cookie。

```tsx filename="app/page.tsx" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  return cookieStore.getAll().map((cookie) => (
    <div key={cookie.name}>
      <p>Name: {cookie.name}</p>
      <p>Value: {cookie.value}</p>
    </div>
  ))
}
```

```jsx filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  return cookieStore.getAll().map((cookie) => (
    <div key={cookie.name}>
      <p>Name: {cookie.name}</p>
      <p>Value: {cookie.value}</p>
    </div>
  ))
}
```

### 设置 Cookie

你可以在 [服务器函数 (Server Function)](/docs/app/getting-started/mutating-data) 或 [路由处理器 (Route Handler)](/docs/app/api-reference/file-conventions/route) 中使用 `(await cookies()).set(name, value, options)` 方法设置 cookie。[`options` 对象](#options) 是可选的。

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function create(data) {
  const cookieStore = await cookies()

  cookieStore.set('name', 'lee')
  // or
  cookieStore.set('name', 'lee', { secure: true })
  // or
  cookieStore.set({
    name: 'name',
    value: 'lee',
    httpOnly: true,
    path: '/',
  })
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function create(data) {
  const cookieStore = await cookies()

  cookieStore.set('name', 'lee')
  // or
  cookieStore.set('name', 'lee', { secure: true })
  // or
  cookieStore.set({
    name: 'name',
    value: 'lee',
    httpOnly: true,
    path: '/',
  })
}
```

### 检查 Cookie 是否存在

你可以使用 `(await cookies()).has(name)` 方法来检查 Cookie 是否存在：

```tsx filename="app/page.ts" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const hasCookie = cookieStore.has('theme')
  return '...'
}
```

```jsx filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const hasCookie = cookieStore.has('theme')
  return '...'
}
```

### 删除 Cookie

有三种方法可以删除 Cookie。

使用 `delete()` 方法：

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.delete('name')
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.delete('name')
}
```

设置一个具有相同名称和空值的新 Cookie：

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.set('name', '')
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.set('name', '')
}
```

将 `maxAge` 设置为 0 将立即使 Cookie 过期。`maxAge` 接受以秒为单位的值。

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.set('name', 'value', { maxAge: 0 })
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.set('name', 'value', { maxAge: 0 })
}
```
