`useRouter` hook 允许你在[客户端组件](/docs/app/getting-started/server-and-client-components)中以编程方式更改路由。

> **建议：** 除非你有使用 `useRouter` 的特定需求，否则请使用 [`<Link>` 组件](/docs/app/api-reference/components/link) 进行导航。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button type="button" onClick={() => router.push('/dashboard')}>
      Dashboard
    </button>
  )
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button type="button" onClick={() => router.push('/dashboard')}>
      Dashboard
    </button>
  )
}
```

## `useRouter()`

- `router.push(href: string, { scroll: boolean, transitionTypes: string[] })`: 执行到指定路由的客户端导航。向 [浏览器历史记录栈](https://developer.mozilla.org/docs/Web/API/History_API) 添加新条目。可选的 `transitionTypes` 将在导航 Transition 期间传递给 [`React.addTransitionType`](https://react.dev/reference/react/addTransitionType)。
- `router.replace(href: string, { scroll: boolean, transitionTypes: string[] })`: 执行到指定路由的客户端导航，而不向浏览器历史记录栈添加新条目。可选的 `transitionTypes` 将在导航 Transition 期间传递给 [`React.addTransitionType`](https://react.dev/reference/react/addTransitionType)。
- `router.refresh()`: 刷新当前路由。向服务器发送新请求，重新获取数据请求，并重新渲染服务器组件。客户端将合并更新后的 React 服务器组件有效负载，同时保留未受影响的客户端 React 状态（如 `useState`）和浏览器状态（如滚动位置）。这会清除当前路由的 [客户端缓存](/docs/app/glossary#client-cache)，但**不会**使服务端缓存无效。请使用 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 或 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag) 来使服务端缓存数据无效。
- `router.prefetch(href: string, options?: { onInvalidate?: () => void })`: 为指定的路由进行 [预取](/docs/app/getting-started/linking-and-navigating#prefetching)，以实现更快的客户端过渡。可选的 `onInvalidate` 回调在 [预取数据过期时](/docs/app/guides/prefetching#extending-or-ejecting-link) 被调用。
- `router.back()`: 导航回浏览器历史记录栈中的上一路由。
- `router.forward()`: 导航到浏览器历史记录栈中的下一路由。

> **要知道**：
>
> - 不得向 `router.push` 或 `router.replace` 发送不可信或未经清洗的 URL，因为这可能会使您的站点面临跨站脚本 (XSS) 漏洞。例如，发送给 `router.push` 或 `router.replace` 的 `javascript:` URL 将在您的页面上下文中执行。
> - 当路由在视口变得可见时，`<Link>` 组件会自动预取路由。
> - 如果 `fetch` 请求被缓存，`refresh()` 可能会重新产生相同的结果。其他请求时 API（如 `cookies` 和 `headers`）也可能改变响应。
> - `onInvalidate` 回调在每个预取请求中最多调用一次。它指示您何时可能需要为更新的路由数据触发新的预取。

### 从 `next/router` 迁移

- 使用 App Router 时，`useRouter` hook 应从 `next/navigation` 导入，而不是从 `next/router` 导入。
- `pathname` 字符串已被移除，由 [`usePathname()`](/docs/app/api-reference/functions/use-pathname) 替代。
- `query` 对象已被移除，由 [`useSearchParams()`](/docs/app/api-reference/functions/use-search-params) 替代。
- `router.events` 已被替代。[见下文](#router-events)。

[查看完整的迁移指南](/docs/app/guides/migrating/app-router-migration)。

## 示例

### Router 事件

你可以通过组合其他客户端组件 hook（如 `usePathname` 和 `useSearchParams`）来监听页面变化。

```jsx filename="app/components/navigation-events.js"
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

export function NavigationEvents() {
  const pathname = usePathname()
  const searchParams = useSearchParams()

  useEffect(() => {
    const url = `${pathname}?${searchParams}`
    console.log(url)
    // You can now use the current URL
    // ...
  }, [pathname, searchParams])

  return '...'
}
```

然后可以在布局中导入它。

```jsx filename="app/layout.js" highlight={2,10-12}
import { Suspense } from 'react'
import { NavigationEvents } from './components/navigation-events'

export default function Layout({ children }) {
  return (
    <html lang="en">
      <body>
        {children}

        <Suspense fallback={null}>
          <NavigationEvents />
        </Suspense>
      </body>
    </html>
  )
}
```

> **要知道**: `<NavigationEvents>` 被 [`Suspense` 边界](/docs/app/api-reference/file-conventions/loading#examples) 包裹，因为 [`useSearchParams()`](/docs/app/api-reference/functions/use-search-params) 会导致客户端渲染直到最近的 `Suspense` 边界，这是在 [预渲染](/docs/app/glossary#prerendering) 期间。[了解更多](/docs/app/api-reference/functions/use-search-params#behavior)。

### 禁用自动滚动到顶部

默认情况下，Next.js 在导航到新路由时会将页面滚动到顶部。你可以通过向 `router.push()` 或 `router.replace()` 传递 `scroll: false` 来禁用此行为。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button
      type="button"
      onClick={() => router.push('/dashboard', { scroll: false })}
    >
      Dashboard
    </button>
  )
}
```

```jsx filename="app/example-client-component.jsx" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button
      type="button"
      onClick={() => router.push('/dashboard', { scroll: false })}
    >
      Dashboard
    </button>
  )
}
```
