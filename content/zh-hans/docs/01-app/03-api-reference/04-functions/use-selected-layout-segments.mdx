`useSelectedLayoutSegments` 是一个**客户端组件**钩子，允许你读取调用它的布局**下方**的活动路由段。

它对于在父布局中创建需要了解活动子段信息（例如面包屑导航）的 UI 非常有用。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useSelectedLayoutSegments } from 'next/navigation'

export default function ExampleClientComponent() {
  const segments = useSelectedLayoutSegments()

  return (
    <ul>
      {segments.map((segment, index) => (
        <li key={index}>{segment}</li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useSelectedLayoutSegments } from 'next/navigation'

export default function ExampleClientComponent() {
  const segments = useSelectedLayoutSegments()

  return (
    <ul>
      {segments.map((segment, index) => (
        <li key={index}>{segment}</li>
      ))}
    </ul>
  )
}
```

> **小提示**：
>
> - 由于 `useSelectedLayoutSegments` 是一个 [客户端组件 (Client Component)](/docs/app/getting-started/server-and-client-components) 钩子 (hook)，而布局 (Layout) 默认是 [服务端组件 (Server Components)](/docs/app/getting-started/server-and-client-components)，因此 `useSelectedLayoutSegments` 通常是通过导入到布局 (Layout) 中的客户端组件 (Client Component) 调用的。
> - 返回的路径段 (segments) 包含 [路由组 (Route Groups)](/docs/app/api-reference/file-conventions/route-groups)，您可能不希望它们出现在您的 UI 中。您可以使用 [`filter`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/filter) 数组方法来移除以方括号开头的项。
> - 对于 [捕获所有 (catch-all)](/docs/app/api-reference/file-conventions/dynamic-routes#catch-all-segments) 路由，匹配的路径段 (segments) 在数组中返回为单个连接的字符串。例如，给定 `app/blog/[...slug]/page.js`，从 `app/layout.js` 访问 `/blog/a/b/c` 时返回 `['blog', 'a/b/c']`，而不是 `['blog', 'a', 'b', 'c']`。

## 参数

```tsx
const segments = useSelectedLayoutSegments(parallelRoutesKey?: string)
```

`useSelectedLayoutSegments` _可选地_ 接受一个 [`parallelRoutesKey`](/docs/app/api-reference/file-conventions/parallel-routes#with-useselectedlayoutsegments)，它允许你读取该插槽内的活动路由段。

## 返回

`useSelectedLayoutSegments` 返回一个字符串数组，包含从调用该钩子的布局下一级的活动段。如果不存在，则返回空数组。

例如，给定以下布局和 URL，返回的段将是：

| Layout                    | 访问的 URL           | 返回的段                      |
| ------------------------- | -------------------- | ----------------------------- |
| `app/layout.js`           | `/`                  | `[]`                          |
| `app/layout.js`           | `/dashboard`         | `['dashboard']`               |
| `app/layout.js`           | `/dashboard/settings` | `['dashboard', 'settings']`   |
| `app/dashboard/layout.js` | `/dashboard`         | `[]`                          |
| `app/dashboard/layout.js` | `/dashboard/settings` | `['settings']`                |

对于 捕获所有 (catch-all) 路由 (`[...slug]`)，所有匹配的路径段 (segments) 在数组中作为单个连接的字符串返回：

| 布局 (Layout) | 访问的 URL | 返回的路径段 (segments) |
| -------------------- | ------------- | ------------------- |
| `app/layout.js` | `/blog/a/b/c` | `['blog', 'a/b/c']` |
| `app/blog/layout.js` | `/blog/a/b/c` | `['a/b/c']` |
