`useSearchParams` 是一个**客户端组件 (Client Component)** 钩子，允许您读取当前 URL 的**查询字符串 (query string)**。

`useSearchParams` 返回一个**只读 (read-only)** 版本的 [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams) 接口。

```tsx filename="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // URL -> `/dashboard?search=my-project`
  // `search` -> 'my-project'
  return <>Search: {search}</>
}
```

```jsx filename="app/dashboard/search-bar.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // URL -> `/dashboard?search=my-project`
  // `search` -> 'my-project'
  return <>Search: {search}</>
}
```

## 参数

```tsx
const searchParams = useSearchParams()
```

`useSearchParams` 不接受任何参数。

## 返回

`useSearchParams` 返回一个**只读 (read-only)** 版本的 [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams) 接口，该接口包含了用于读取 URL 查询字符串的实用方法：

- [`URLSearchParams.get()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/get)：返回与搜索参数关联的第一个值。例如：

  | URL                  | `searchParams.get("a")`                                                                                         |
  | -------------------- | --------------------------------------------------------------------------------------------------------------- |
  | `/dashboard?a=1`     | `'1'`                                                                                                           |
  | `/dashboard?a=`      | `''`                                                                                                            |
  | `/dashboard?b=3`     | `null`                                                                                                          |
  | `/dashboard?a=1&a=2` | `'1'` _- 使用 [`getAll()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/getAll) 获取所有值_ |

- [`URLSearchParams.has()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/has)：返回一个布尔值，指示给定参数是否存在。例如：

  | URL              | `searchParams.has("a")` |
  | ---------------- | ----------------------- |
  | `/dashboard?a=1` | `true`                  |
  | `/dashboard?b=3` | `false`                 |

- 了解更多关于 [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams) 的其他**只读 (read-only)** 方法，包括 [`getAll()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/getAll)、[`keys()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/keys)、[`values()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/values)、[`entries()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/entries)、[`forEach()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/forEach) 和 [`toString()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/toString)。

> **值得了解**：
>
> - `useSearchParams` 是一个 [客户端组件 (Client Component)](/docs/app/getting-started/server-and-client-components) 钩子且 **不支持** 在 [服务端组件 (Server Components)](/docs/app/getting-started/server-and-client-components) 中以防止 [部分渲染 (partial rendering)](/docs/app/getting-started/linking-and-navigating#client-side-transitions) 期间的陈旧值。
> - 如果您想基于 search params 在服务端组件中获取数据，通常更好的选择是读取相应页面的 [`searchParams` 属性](/docs/app/api-reference/file-conventions/page#searchparams-optional)。然后您可以通过 props 将其传递给该页面内的任何组件（服务端或客户端）。
> - 如果应用程序包含 `/pages` 目录，`useSearchParams` 将返回 `ReadonlyURLSearchParams | null`。`null` 值是为了兼容迁移期间，因为在没有使用 `getServerSideProps` 的页面预渲染期间无法知道 search params

## 行为

### 预渲染

如果路由被 [预渲染 (prerendered)](/docs/app/glossary#prerendering)，调用 `useSearchParams` 会导致直到最近的 [`Suspense` 边界](/docs/app/api-reference/file-conventions/loading#examples) 的客户端组件树被客户端渲染。

这使得路由的一部分可以被预渲染，而使用 `useSearchParams` 的动态部分则被客户端渲染。

我们建议将使用 `useSearchParams` 的客户端组件包裹在 `<Suspense/>` 边界中。这将允许其上方的任何客户端组件被预渲染并作为初始 HTML 的一部分发送。[示例](/docs/app/api-reference/functions/use-search-params#prerendering)。

例如：

```tsx filename="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // This will not be logged on the server during prerendering
  console.log(search)

  return <>Search: {search}</>
}
```

```jsx filename="app/dashboard/search-bar.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // This will not be logged on the server during prerendering
  console.log(search)

  return <>Search: {search}</>
}
```

```tsx filename="app/dashboard/page.tsx" switcher
import { Suspense } from 'react'
import SearchBar from './search-bar'

// This component passed as a fallback to the Suspense boundary
// will be rendered in place of the search bar in the initial HTML.
// When the value is available during React hydration the fallback
// will be replaced with the `<SearchBar>` component.
function SearchBarFallback() {
  return <>placeholder</>
}

export default function Page() {
  return (
    <>
      <nav>
        <Suspense fallback={<SearchBarFallback />}>
          <SearchBar />
        </Suspense>
      </nav>
      <h1>Dashboard</h1>
    </>
  )
}
```

> **值得了解**：
>
> - 在开发环境中，路由按需渲染，所以 `useSearchParams` 不会触发挂起，并且在没有 `Suspense` 的情况下事情看起来也能工作。
> - 在生产构建中，如果客户端组件中的静态页面调用了 `useSearchParams`，则必须将其包裹在 `Suspense` 边界中，否则构建将失败，并出现 [缺少 Suspense 边界 (with useSearchParams)](/docs/messages/missing-suspense-with-csr-bailout) 错误。
> - 如果您打算让路由动态渲染，建议在服务端组件中优先使用 [`connection`](/docs/app/api-reference/functions/connection) 函数来等待传入请求，这会排除下方的所有内容免于预渲染。参见 [动态渲染指南 (Dynamic Rendering)](/docs/app/glossary#dynamic-rendering) 了解什么使路由动态化。
> - 如果您已经处于服务端组件页面中，请考虑使用 [`searchParams` 属性](/docs/app/api-reference/file-conventions/page#searchparams-optional) 并将值传递给客户端组件。
> - 您也可以直接将页面的 [`searchParams` 属性](/docs/app/api-reference/file-conventions/page#searchparams-optional) 传递给客户端组件，并使用 React 的 `use()` 解包。尽管这会触发挂起，因此客户端组件应被 `Suspense` 边界包裹。

### 动态渲染

如果路由是动态渲染的，`useSearchParams` 将在客户端组件的初始服务端渲染期间在服务端上可用。

例如：

```tsx filename="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // This will be logged on the server during the initial render
  // and on the client on subsequent navigations.
  console.log(search)

  return <>Search: {search}</>
}
```

```jsx filename="app/dashboard/search-bar.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // This will be logged on the server during the initial render
  // and on the client on subsequent navigations.
  console.log(search)

  return <>Search: {search}</>
}
```

```tsx filename="app/dashboard/page.tsx" switcher
import { connection } from 'next/server'
import SearchBar from './search-bar'

export default async function Page() {
  await connection()
  return (
    <>
      <nav>
        <SearchBar />
      </nav>
      <h1>Dashboard</h1>
    </>
  )
}
```

```jsx filename="app/dashboard/page.js" switcher

import SearchBar from './search-bar'

  await connection()
  return (
    <>
      <nav>
        <SearchBar />
      </nav>
      <h1>Dashboard</h1>
    </>
  )
}
```

> **值得了解**：
>
> - 之前，在页面设置 `export const dynamic = 'force-dynamic'` 用于强制动态渲染。更推荐使用 [`connection()`](/docs/app/api-reference/functions/connection) 替代，因为它在语义上将动态渲染与传入请求相关联。

### 服务器组件

#### 页面 (Pages)

要在[页面 (Pages)](/docs/app/api-reference/file-conventions/page)（服务器组件）中访问搜索参数，请使用 [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) 属性。

#### 布局 (Layouts)

与页面不同，[布局 (Layouts)](/docs/app/api-reference/file-conventions/layout)（服务器组件）**不会**接收 `searchParams` 属性。这是因为共享布局在导航期间[不会被重新渲染](/docs/app/getting-started/linking-and-navigating#client-side-transitions)，这可能导致导航之间的 `searchParams` 过时。请查看[详细解释](/docs/app/api-reference/file-conventions/layout#query-params)。

相反，应使用页面的 [`searchParams`](/docs/app/api-reference/file-conventions/page) 属性，或在客户端组件中使用 [`useSearchParams`](/docs/app/api-reference/functions/use-search-params) 钩子，该组件会在客户端使用最新的 `searchParams` 重新渲染。

## 示例

### 更新 `searchParams`

您可以使用 [`useRouter`](/docs/app/api-reference/functions/use-router) 或 [`Link`](/docs/app/api-reference/components/link) 来设置新的 `searchParams`。在导航完成后，当前的 [`page.js`](/docs/app/api-reference/file-conventions/page) 将接收到更新的 [`searchParams` 属性](/docs/app/api-reference/file-conventions/page#searchparams-optional)。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

export default function ExampleClientComponent() {
  const router = useRouter()
  const pathname = usePathname()
  const searchParams = useSearchParams()

  // Get a new searchParams string by merging the current
  // searchParams with a provided key/value pair
  const createQueryString = useCallback(
    (name: string, value: string) => {
      const params = new URLSearchParams(searchParams.toString())
      params.set(name, value)

      return params.toString()
    },
    [searchParams]
  )

  return (
    <>
      <p>Sort By</p>

      {/* using useRouter */}
      <button
        onClick={() => {
          // <pathname>?sort=asc
          router.push(pathname + '?' + createQueryString('sort', 'asc'))
        }}
      >
        ASC
      </button>

      {/* using <Link> */}
      <Link
        href={
          // <pathname>?sort=desc
          pathname + '?' + createQueryString('sort', 'desc')
        }
      >
        DESC
      </Link>
    </>
  )
}
```
