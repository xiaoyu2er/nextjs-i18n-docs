# A

## 应用路由 (App Router)

Next.js 13 版本引入的路由器，基于 React 服务器组件 (React Server Components) 构建。它采用基于文件系统的路由，支持布局、嵌套路由、加载状态、错误处理等。了解更多请参阅[应用路由器文档](/docs/app)。

# B

## 构建时 (Build time)

您的应用程序正在被编译的阶段。在构建时，Next.js 会将您的代码转换为用于生产的优化文件，生成静态页面，并为部署准备资源。参见 [`next build` CLI 参考](/docs/app/api-reference/cli/next#next-build-options)。

# C

## 缓存组件 (Cache Components)

一项功能，通过 [`"use cache"` 指令](/docs/app/api-reference/directives/use-cache)实现组件和函数级别的缓存。缓存组件允许您在单个路由中混合使用静态、缓存和动态内容，通过预渲染一个可立即提供的静态 HTML 外壳，同时动态内容在就绪后流式传输。使用 [`cacheLife()`](/docs/app/api-reference/functions/cacheLife) 配置缓存时长，使用 [`cacheTag()`](/docs/app/api-reference/functions/cacheTag) 为缓存数据打标签，并使用 [`updateTag()`](/docs/app/api-reference/functions/updateTag) 按需失效。了解更多请参阅[缓存组件指南](/docs/app/getting-started/caching)。

## 全匹配段

可以使用 `[...folder]/page.js` 语法匹配多个 URL 部分的动态路由段。这些段会捕获所有剩余的 URL 段，对于实现文档站点或文件浏览器等功能非常有用。了解更多请参阅[动态路由段](/docs/app/api-reference/file-conventions/dynamic-routes#catch-all-segments)。

## 客户端包 (Client Bundles)

发送到浏览器的 JavaScript 包。Next.js 会根据[模块图](#module-graph)自动拆分这些包，以减少初始有效负载大小，并仅加载每个页面所需的代码。

## 客户端组件 (Client Component)

一种在浏览器中运行的 React 组件。在 Next.js 中，客户端组件也可以在初始页面生成期间在服务器上渲染。它们可以使用状态、效果、事件处理程序和浏览器 API，并通过在文件顶部标记 [`"use client"` 指令](#use-client-directive) 来标识。了解更多请参阅[服务器与客户端组件](/docs/app/getting-started/server-and-client-components)。

## 客户端导航 (Client-side navigation)

一种导航技术，页面内容会动态更新，而无需完全重新加载页面。Next.js 使用 [`<Link>` 组件](/docs/app/api-reference/components/link) 进行客户端导航，保持共享布局的交互性并保留浏览器状态。了解更多请参阅[链接与导航](/docs/app/getting-started/linking-and-navigating#client-side-transitions)。

## 客户端路由缓存

浏览器中的内存缓存，存储已访问和预取路由的 [RSC 有效负载](#rsc-payload)。在[客户端导航](#client-side-navigation)期间，Next.js 可以即时提供缓存的[布局](#layout)和[加载状态](#loading-ui)，而无需向服务器发送请求。页面默认不会被缓存，但在浏览器后退/前进导航期间会被重用。

客户端缓存在页面刷新时会被清除。可以通过 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag)、[`revalidatePath`](/docs/app/api-reference/functions/revalidatePath)、[`updateTag`](/docs/app/api-reference/functions/updateTag)、[`router.refresh`](/docs/app/api-reference/functions/use-router)、[`cookies.set`](/docs/app/api-reference/functions/cookies) 或 [`cookies.delete`](/docs/app/api-reference/functions/cookies) 以编程方式使其失效。

您可以使用 [`staleTimes`](/docs/app/api-reference/config/next-config-js/staleTimes) 全局配置客户端缓存时长，或通过 [`cacheLife`](/docs/app/api-reference/functions/cacheLife#client-cache-behavior) 中的 `stale` 属性按路由配置（推荐）。

## 代码分割 (Code Splitting)

根据路由将应用程序拆分为更小的 JavaScript 块的过程。Next.js 不会预先加载所有代码，而是只加载当前路由所需的代码，从而减少初始加载时间。Next.js 会根据路由自动执行代码分割。了解更多请参阅[打包指南](/docs/app/guides/package-bundling)。

# D

## 动态渲染 (Dynamic rendering)

组件在请求时而非[构建时](#build-time)渲染。当组件使用[请求时 API](#request-time-apis)时，它会变为动态的。

## 动态路由路径段 (Dynamic route segments)

在请求时根据数据生成的[路由段](#route-segment)。通过将文件夹名称用方括号括起来（例如 `[slug]`）来创建，它们允许您根据博客文章或产品页面等动态数据创建路由。了解更多请参阅[动态路由段](/docs/app/api-reference/file-conventions/dynamic-routes)。

# E

## 环境变量

在构建时或请求时可访问的配置值。在 Next.js 中，以 `NEXT_PUBLIC_` 为前缀的变量会暴露给浏览器，而其他变量仅在服务器端可用。了解更多请参阅[环境变量](/docs/app/guides/environment-variables)。

## 错误边界 (Error Boundary)

一种 React 组件，用于捕获其子组件树中的 JavaScript 错误并显示备用 UI。在 Next.js 中，创建一个 [`error.js` 文件](/docs/app/api-reference/file-conventions/error) 以自动将路由段包装在错误边界中。了解更多请参阅[错误处理](/docs/app/getting-started/error-handling)。

# F

## 字体优化 (Font Optimization)

使用 [`next/font`](/docs/app/api-reference/components/font) 进行自动字体优化。Next.js 自托管字体，消除布局偏移，并应用性能最佳实践。支持 Google Fonts 和本地字体文件。了解更多请参阅[字体](/docs/app/getting-started/fonts)。

## 文件系统缓存 (File-system caching)

Turbopack 的一项功能，可在运行之间将编译器产物存储在磁盘上，减少跨 `next dev` 或 `next build` 命令的工作量，从而显著加快编译时间。了解更多请参阅 [Turbopack 文件系统缓存](/docs/app/api-reference/config/next-config-js/turbopackFileSystemCache)。

# H

## 水合 (Hydration)

React 将事件处理程序附加到 DOM 以使服务器渲染的静态 HTML 具有交互性的过程。在水合期间，React 会协调服务器渲染的标记与客户端 JavaScript。了解更多请参阅[服务器与客户端组件](/docs/app/getting-started/server-and-client-components#how-do-server-and-client-components-work-in-nextjs)。

# I

## 导入别名 (Import Aliases)

为常用目录提供简写的自定义路径映射。导入别名减少了相对导入的复杂性，并使代码更易于阅读和维护。了解更多请参阅[绝对导入和模块路径别名](/docs/app/getting-started/installation#set-up-absolute-imports-and-module-path-aliases)。

## 增量静态再生 (ISR)

一种允许您在不重新构建整个站点的情况下更新静态内容的技术。ISR 使您可以在按页面的基础上使用静态生成，同时在流量进入时在后台重新验证页面。了解更多请参阅 [ISR 指南](/docs/app/guides/incremental-static-regeneration)。

> **请注意**: 在 Next.js 中，ISR 也被称为[重新验证](#revalidation)。

## 拦截路由 (Intercepting Routes)

一种路由模式，允许在当前布局中从应用程序的另一部分加载路由。适用于在不切换用户上下文的情况下显示内容（如模态框），同时保持 URL 可共享。了解更多请参阅[拦截路由](/docs/app/api-reference/file-conventions/intercepting-routes)。

## 图像优化

使用 [`<Image>` 组件](/docs/app/api-reference/components/image) 进行自动图像优化。Next.js 按需优化图像，以 WebP 等现代格式提供服务，并自动处理延迟加载和响应式尺寸。了解更多请参阅[图像](/docs/app/getting-started/images)。

# L

## 布局 (Layout)

在多个页面之间共享的 UI。布局会保持状态，保持交互性，并且在导航时不会重新渲染。通过从 [`layout.js` 文件](/docs/app/api-reference/file-conventions/layout) 导出 React 组件来定义。了解更多请参阅[布局与页面](/docs/app/getting-started/layouts-and-pages)。

## 加载 UI (Loading UI)

在[路由段](#route-segment)加载时显示的备用 UI。通过向文件夹添加 [`loading.js` 文件](/docs/app/api-reference/file-conventions/loading) 来创建，该文件会自动将页面包装在 [Suspense 边界](#suspense-boundary) 中。了解更多请参阅[加载 UI](/docs/app/api-reference/file-conventions/loading)。

# M

## 模块图 (Module Graph)

应用程序中文件依赖关系的图。每个文件（模块）是一个节点，导入/导出关系构成边。Next.js 分析此图以确定最佳的打包和代码分割策略。了解更多请参阅[服务器与客户端组件](/docs/app/getting-started/server-and-client-components#reducing-js-bundle-size)。

## 元数据

浏览器和搜索引擎使用的页面信息，如标题、描述和 Open Graph 图像。在 Next.js 中，使用布局或页面文件中的 [`metadata` 导出](/docs/app/api-reference/functions/generate-metadata) 或 [`generateMetadata` 函数](/docs/app/api-reference/functions/generate-metadata) 来定义元数据。了解更多请参阅[元数据和 OG 图像](/docs/app/getting-started/metadata-and-og-images)。

## 记忆化 (Memoization)

缓存函数的返回值，以便在一次渲染传递（请求）期间多次调用同一函数时只执行一次。在 Next.js 中，具有相同 URL 和选项的 `fetch` `GET` 请求会在服务器组件、布局、页面和 `generateMetadata`/`generateStaticParams` 之间自动记忆化（但不包括[路由处理程序](/docs/app/api-reference/file-conventions/route)，因为它们不是 React 组件树的一部分）。

对于非 `fetch` 操作，请使用 React [`cache`](https://react.dev/reference/react/cache) 函数。了解更多请参阅 [`fetch` API 参考](/docs/app/api-reference/functions/fetch)。

## 中间件

参见[代理 (Proxy)](#proxy)。

# N

## 未找到 (Not Found)

当路由不存在或调用 [`notFound()` 函数](/docs/app/api-reference/functions/not-found) 时显示的特殊组件。通过在应用目录中添加 [`not-found.js` 文件](/docs/app/api-reference/file-conventions/not-found) 来创建。了解更多请参阅[错误处理](/docs/app/getting-started/error-handling#not-found)。

# P

## 私有文件夹 (Private Folders)

以前缀下划线命名的文件夹（例如 `_components`），它们被排除在路由系统之外。这些文件夹用于代码组织和共享实用程序，而不会创建可访问的路由。了解更多请参阅[私有文件夹](/docs/app/getting-started/project-structure#private-folders)。

## 页面 (Page)

特定于路由的 UI。通过从 `app` 目录中的 [`page.js` 文件](/docs/app/api-reference/file-conventions/page) 导出 React 组件来定义。了解更多请参阅[布局与页面](/docs/app/getting-started/layouts-and-pages)。

## 并行路由 (Parallel Routes)

一种允许在同一布局中同时或有条件地渲染多个页面的模式。使用 `@folder` 约定命名插槽创建，适用于仪表板、模态框和复杂布局。了解更多请参阅[并行路由](/docs/app/api-reference/file-conventions/parallel-routes)。

## 部分预渲染 (PPR)

一种渲染优化，将预渲染和动态渲染结合在单个路由中。静态外壳立即提供，而动态内容在就绪后流式传输，提供两种渲染策略的最佳效果。了解更多请参阅[缓存组件](/docs/app/getting-started/caching)。

## 预取 (Prefetching)

在用户导航到路由之前在后台加载该路由。当链接进入视口时，Next.js 会自动预取使用 [`<Link>` 组件](/docs/app/api-reference/components/link) 链接的路由，使导航感觉即时。了解更多请参阅[预取指南](/docs/app/guides/prefetching)。

## 预渲染 (Prerendering)

组件在[构建时](#build-time)或在[重新验证](#revalidation)期间在后台渲染。结果是 HTML 和 [RSC 有效负载](#rsc-payload)，可以被缓存并从 CDN 提供。预渲染是未使用[请求时 API](#request-time-apis) 的组件的默认行为。

## 代理 (Proxy)

一个文件（[`proxy.js`](/docs/app/api-reference/file-conventions/proxy)），在请求完成之前在服务器上运行代码。用于实现服务器端逻辑，如日志记录、重定向和重写。以前称为中间件 (Middleware)。了解更多请参阅[代理指南](/docs/app/getting-started/proxy)。

# R

## 重定向 (Redirect)

将用户从一个 URL 发送到另一个 URL。在 Next.js 中，重定向可以在 [`next.config.js`](/docs/app/api-reference/config/next-config-js/redirects) 中配置，从[代理](/docs/app/api-reference/file-conventions/proxy) 返回，或使用 [`redirect()` 函数](/docs/app/api-reference/functions/redirect) 以编程方式触发。了解更多请参阅[重定向](/docs/app/guides/redirecting)。

## 请求时 API (Request-time APIs)

访问特定于请求的数据的函数，导致组件选择加入[动态渲染](#dynamic-rendering)。这些包括：

- [`cookies()`](/docs/app/api-reference/functions/cookies) - 访问请求 Cookie
- [`headers()`](/docs/app/api-reference/functions/headers) - 访问请求头
- [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) - 访问 URL 查询参数
- [`draftMode()`](/docs/app/api-reference/functions/draft-mode) - 启用或检查草稿模式

## 运行时渲染 (Runtime rendering)

参见[动态渲染](#dynamic-rendering)。

## 重新验证 (Revalidation)

更新缓存数据的过程。可以是基于时间的（使用 [`cacheLife()`](/docs/app/api-reference/functions/cacheLife) 设置缓存时长）或按需的（使用 [`cacheTag()`](/docs/app/api-reference/functions/cacheTag) 为数据打标签，然后使用 [`updateTag()`](/docs/app/api-reference/functions/updateTag) 使其失效）。了解更多请参阅[缓存与重新验证](/docs/app/getting-started/caching)。

## 重写 (Rewrite)

将传入的请求路径映射到不同的目标路径，而不更改浏览器中的 URL。在 [`next.config.js`](/docs/app/api-reference/config/next-config-js/rewrites) 中配置或从[代理](/docs/app/api-reference/file-conventions/proxy) 返回。适用于代理到外部服务或旧版 URL。

## 路由组 (Route Groups)

一种在不影响 URL 结构的情况下组织路由的方法。通过将文件夹名称用括号括起来（例如 `(marketing)`）来创建，路由组有助于组织相关路由并启用按组的[布局](#layout)。了解更多请参阅[路由组](/docs/app/api-reference/file-conventions/route-groups)。

## 路由处理器 (Route Handler)

处理特定路由的 HTTP 请求的函数，在 [`route.js` 文件](/docs/app/api-reference/file-conventions/route) 中定义。路由处理程序使用 Web Request 和 Response API，可以处理 `GET`、`POST`、`PUT`、`PATCH`、`DELETE`、`HEAD` 和 `OPTIONS` 方法。了解更多请参阅[路由处理程序](/docs/app/getting-started/route-handlers)。

## 路由段 (Route Segment)

URL 路径的一部分（两个斜杠之间），由 `app` 目录中的文件夹定义。每个文件夹代表 URL 结构中的一个段。了解更多请参阅[项目结构](/docs/app/getting-started/project-structure)。

## RSC 负载 (RSC Payload)

React 服务器组件有效负载——渲染后的 React 服务器组件树的紧凑二进制表示。包含服务器组件的渲染结果、客户端组件的占位符以及在它们之间传递的属性。了解更多请参阅[服务器与客户端组件](/docs/app/getting-started/server-and-client-components#how-do-server-and-client-components-work-in-nextjs)。

# S

## 服务器组件 (Server Component)

应用路由器中的默认组件类型。服务器组件在服务器上渲染，可以直接获取数据，并且不会增加客户端 JavaScript 包的大小。它们不能使用状态或浏览器 API。了解更多请参阅[服务器与客户端组件](/docs/app/getting-started/server-and-client-components)。

## 服务器动作 (Server Action)

作为属性传递给客户端组件或绑定到表单操作的[服务器函数](#server-function)。服务器动作通常用于表单提交和数据变更。了解更多请参阅[服务器动作与变更](/docs/app/getting-started/mutating-data)。

## 服务器函数 (Server Function)

在服务器上运行的异步函数，使用 [`"use server"` 指令](/docs/app/api-reference/directives/use-server) 标记。服务器函数可以从客户端组件调用。当作为属性传递给客户端组件或绑定到表单操作时，它们被称为[服务器动作](#server-action)。了解更多请参阅 [React 服务器函数](https://react.dev/reference/rsc/server-functions)。

## 静态导出 (Static Export)

一种生成包含 HTML、CSS 和 JavaScript 文件的完全静态站点的部署模式。通过在 `next.config.js` 中设置 `output: 'export'` 来启用。静态导出可以托管在任何静态文件服务器上，无需 Node.js 服务器。了解更多请参阅[静态导出](/docs/app/guides/static-exports)。

## 静态渲染 (Static rendering)

参见[预渲染](#prerendering)。

## 静态资源 (Static Assets)

图像、字体、视频和其他媒体等无需处理即可直接提供的文件。静态资源通常存储在 `public` 目录中，并通过其相对路径引用。了解更多请参阅[静态资源](/docs/app/api-reference/file-conventions/public-folder)。

## 静态外壳 (Static Shell)

页面的预渲染 HTML 结构，可立即提供给浏览器。通过[部分预渲染](#partial-prerendering-ppr)，静态外壳包含所有可静态渲染的内容，加上用于稍后流式传输的动态内容的 [Suspense 边界](#suspense-boundary) 回退。

## 流式传输

一种允许服务器在页面的各个部分准备就绪时将其发送到客户端的技术，而不是等待整个页面渲染完毕。通过 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 或手动 `<Suspense>` 边界自动启用。了解更多请参阅[流式传输指南](/docs/app/guides/streaming)。

## Suspense 边界 (Suspense boundary)

一个 React [`<Suspense>`](https://react.dev/reference/react/Suspense) 组件，用于包装异步内容并在加载时显示备用 UI。在 Next.js 中，Suspense 边界定义了[静态外壳](#static-shell)结束和[流式传输](#streaming)开始的位置，实现了[部分预渲染](#partial-prerendering-ppr)。

# T

## Turbopack

一个为 Next.js 构建的、基于 Rust 的快速打包器。Turbopack 是 `next dev` 的默认打包器，也可用于 `next build`。与 Webpack 相比，它提供了显著更快的编译时间。了解更多请参阅 [Turbopack](/docs/app/api-reference/turbopack)。

## 摇树 (Tree Shaking)

在构建过程中从 JavaScript 包中移除未使用代码的过程。Next.js 会自动对代码进行 tree shaking 以减少包的大小。了解更多请参阅[打包指南](/docs/app/guides/package-bundling)。

# U

## `"use cache"` 指令

一个指令，用于将组件或函数标记为可缓存。它可以放在文件顶部，表示该文件中的所有导出都是可缓存的，或者内联在函数或组件的顶部，将该特定作用域标记为可缓存。了解更多请参阅 [`"use cache"` 参考](/docs/app/api-reference/directives/use-cache)。

## `"use client"` 指令

一个特殊的 React 指令，用于标记服务器和客户端代码之间的边界。它必须放在文件顶部，在任何导入或其他代码之前。它指示 React 组件、辅助函数、变量声明以及所有导入的依赖项都应包含在[客户端包](#client-bundles)中。了解更多请参阅 [`"use client"` 参考](/docs/app/api-reference/directives/use-client)。

## `"use server"` 指令

一个指令，用于将函数标记为可以从客户端代码调用的[服务器函数](#server-function)。它可以放在文件顶部，表示该文件中的所有导出都是服务器函数，或者内联在函数的顶部，将该特定函数标记为服务器函数。了解更多请参阅 [`"use server"` 参考](/docs/app/api-reference/directives/use-server)。

# V

## 版本偏差 (Version skew)

在部署新版本的应用程序后，仍处于活动状态的客户端可能引用旧版本构建的 JavaScript、CSS 或数据。客户端和服务器版本之间的这种不匹配称为版本偏斜，它可能导致资源缺失、服务器动作错误和导航失败。Next.js 使用 [`deploymentId`](/docs/app/api-reference/config/next-config-js/deploymentId) 来检测和处理版本偏斜。了解更多请参阅[自托管 - 版本偏斜](/docs/app/guides/self-hosting#version-skew)。
