Next.js 在 `next` 包中附带了版本匹配的文档，使 AI 编码代理能够参考准确、最新的 API 和模式。项目根目录下的 `AGENTS.md` 文件会指引代理使用这些捆绑的文档，而非其训练数据。

## 工作原理

当你安装 `next` 后，Next.js 文档会被捆绑在 `node_modules/next/dist/docs/` 目录下。这些捆绑的文档反映了 [Next.js 文档站点](https://nextjs.org/docs)的结构：

```txt
node_modules/next/dist/docs/
├── 01-app/
│   ├── 01-getting-started/
│   ├── 02-guides/
│   └── 03-api-reference/
├── 02-pages/
├── 03-architecture/
└── index.mdx
```

这意味着代理始终可以访问与你安装版本相匹配的文档——无需网络请求或外部查找。

项目根目录下的 `AGENTS.md` 文件会告诉代理在编写任何代码之前先阅读这些捆绑的文档。大多数 AI 编码代理——包括 Claude Code、Cursor、GitHub Copilot 等——在启动会话时会自动读取 `AGENTS.md`。

## 快速开始

### 新项目

[`create-next-app`](/docs/app/api-reference/cli/create-next-app) 会自动生成 `AGENTS.md` 和 `CLAUDE.md`。无需额外设置：

```bash package="pnpm"
pnpm create next-app@canary
```

```bash package="npm"
npx create-next-app@canary
```

```bash package="yarn"
yarn create next-app@canary
```

```bash package="bun"
bun create next-app@canary
```

如果你不需要代理文件，可以传递 `--no-agents-md` 参数：

```bash
npx create-next-app@canary --no-agents-md
```

### 现有项目

请确保你的 Next.js 版本在 `v16.2.0-canary.37` 或更高，然后将以下文件添加到你的项目根目录。

`AGENTS.md` 包含代理将读取的指令：

```md filename="AGENTS.md"
<!-- BEGIN:nextjs-agent-rules -->

# Next.js: ALWAYS read docs before coding

Before any Next.js work, find and read the relevant doc in `node_modules/next/dist/docs/`. Your training data is outdated — the docs are the source of truth.

<!-- END:nextjs-agent-rules -->
```

`CLAUDE.md` 使用 `@` 导入语法来包含 `AGENTS.md`，这样 [Claude Code](https://docs.anthropic.com/en/docs/claude-code) 用户无需复制内容也能获得相同的指令：

```md filename="CLAUDE.md"
@AGENTS.md
```

## 理解 AGENTS.md

默认的 `AGENTS.md` 包含一条简洁明了的指令：**在编写代码前先阅读捆绑的文档**。这是有意为之的极简设计——目的是将代理从过时的训练数据引导至 `node_modules/next/dist/docs/` 中准确、版本匹配的文档。

`<!-- BEGIN:nextjs-agent-rules -->` 和 `<!-- END:nextjs-agent-rules -->` 注释标记界定了 Next.js 管理的部分。你可以在这些标记之外添加你自己项目的特定指令，而无需担心它们被未来的更新覆盖。

捆绑的文档包括 App Router 和 Pages Router 的指南、API 参考和文件约定。当代理遇到涉及路由、数据获取或任何其他 Next.js 功能的任务时，它可以在捆绑的文档中查找正确的 API，而不是依赖可能已过时的训练数据。

> **请注意：** 要了解捆绑文档和 `AGENTS.md` 如何提升代理在真实 Next.js 任务上的表现，请访问 [基准测试结果](https://nextjs.org/evals)。
