当启用 [缓存组件 (Cache Components)](/docs/app/api-reference/config/next-config-js/cacheComponents) 时，诸如 `dynamic`、`revalidate` 和 `fetchCache` 等路由段配置将被 [`use cache`](/docs/app/api-reference/directives/use-cache) 和 [`cacheLife`](/docs/app/api-reference/functions/cacheLife) 取代。

## `dynamic = "force-dynamic"`

**不需要。** 默认情况下所有页面都是动态的。

```tsx filename="app/page.tsx" switcher
// Before - No longer needed
export const dynamic = 'force-dynamic'

export default function Page() {
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// Before - No longer needed
export const dynamic = 'force-dynamic'

export default function Page() {
  return <div>...</div>
}
```

```tsx filename="app/page.tsx" switcher
// After - Just remove it
export default function Page() {
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// After - Just remove it
export default function Page() {
  return <div>...</div>
}
```

## `dynamic = "force-static"`

首先将其移除。在开发和构建时检测到未缓存或运行时数据访问时，Next.js 会抛出错误。否则，预渲染步骤会自动提取静态 HTML 外壳。

对于未缓存的数据访问，请尽可能靠近数据访问处添加 [`use cache`](/docs/app/api-reference/directives/use-cache)，并使用像 `'max'` 这样的长 [`cacheLife`](/docs/app/api-reference/functions/cacheLife) 以维持缓存行为。如果需要，可以在页面或布局的顶层添加它。

对于运行时数据访问（`cookies()`、`headers()` 等），错误信息会指引你将其包裹在 `<Suspense>` 中。由于你最初使用的是 `force-static`，因此必须移除运行时数据访问以避免任何请求时的工作。

```tsx filename="app/page.tsx" switcher
// Before
export const dynamic = 'force-static'

export default async function Page() {
  const data = await fetch('https://api.example.com/data')
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// Before
export const dynamic = 'force-static'

export default async function Page() {
  const data = await fetch('https://api.example.com/data')
  return <div>...</div>
}
```

```tsx filename="app/page.tsx" switcher
import { cacheLife } from 'next/cache'

// After - Use 'use cache' instead
export default async function Page() {
  'use cache'
  cacheLife('max')
  const data = await fetch('https://api.example.com/data')
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
import { cacheLife } from 'next/cache'

// After - Use 'use cache' instead
export default async function Page() {
  'use cache'
  cacheLife('max')
  const data = await fetch('https://api.example.com/data')
  return <div>...</div>
}
```

## `revalidate`

**替换为 `cacheLife`。** 使用 `cacheLife` 函数来定义缓存持续时间，而不是使用路由段配置。

```tsx filename="app/page.tsx" switcher
// Before
export const revalidate = 3600 // 1 hour

export default async function Page() {
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// Before
export const revalidate = 3600 // 1 hour

export default async function Page() {
  return <div>...</div>
}
```

```tsx filename="app/page.tsx" switcher
// After - Use cacheLife
import { cacheLife } from 'next/cache'

export default async function Page() {
  'use cache'
  cacheLife('hours')
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// After - Use cacheLife
import { cacheLife } from 'next/cache'

export default async function Page() {
  'use cache'
  cacheLife('hours')
  return <div>...</div>
}
```

## `fetchCache`

**不需要。** 使用 `use cache` 后，缓存作用域内的所有数据获取都会被自动缓存，使得 `fetchCache` 变得多余。

```tsx filename="app/page.tsx" switcher
// Before
export const fetchCache = 'force-cache'
```

```jsx filename="app/page.js" switcher
// Before
export const fetchCache = 'force-cache'
```

```tsx filename="app/page.tsx" switcher
// After - Use 'use cache' to control caching behavior
export default async function Page() {
  'use cache'
  // All fetches here are cached
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// After - Use 'use cache' to control caching behavior
export default async function Page() {
  'use cache'
  // All fetches here are cached
  return <div>...</div>
}
```

## `runtime = 'edge'`

**不支持。** 缓存组件需要 Node.js 运行时。通过移除 `runtime = 'edge'` 导出来切换到 Node.js 运行时（默认）。如果你需要为特定路由使用边缘行为，请改用 [代理 (Proxy)](/docs/app/api-reference/file-conventions/proxy)。
