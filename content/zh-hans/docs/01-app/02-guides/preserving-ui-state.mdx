> **需知：** 本指南假定已启用[缓存组件](/docs/app/getting-started/caching)。通过在你的 Next 配置文件中设置 [`cacheComponents: true`](/docs/app/api-reference/config/next-config-js/cacheComponents) 来启用它。

在缓存组件出现之前，在导航间保留页面级状态需要一些变通方法，例如将状态提升到[共享布局](/docs/app/getting-started/layouts-and-pages#nesting-layouts)或使用外部存储。有了缓存组件，Next.js 开箱即用地保留状态和 DOM。

Next.js 不是在导航时卸载页面，而是使用 React 的 [`<Activity>`](https://react.dev/reference/react/Activity) 组件来隐藏它们。DOM 节点保留在文档中（用 `display: none` 隐藏），因此 React 状态和 DOM 状态都被保留：表单草稿、滚动位置、展开的 `<details>` 元素、视频播放进度等等。

Next.js 最多保留 3 个路由。超过此数量，最旧的路由将被逐出并重新渲染。

> **需知：** 我们正在考虑提供退出策略以进行渐进式迁移。

## 选择需要保留的内容

Activity 默认保留所有组件状态和 DOM 状态。对于每一部分状态，你可以决定这是否是你的 UI 的正确行为。以下模式展示了常见场景以及如何处理两种情况。

### 可展开的 UI (下拉菜单，手风琴，面板)

当用户导航离开并返回时，Activity 会保留可展开元素的打开/关闭状态。

**何时保留：** 一个带有展开部分的侧边栏、一个 FAQ 手风琴或一个筛选面板。用户是有意设置了他们的视图，恢复它避免了重复操作。

**何时重置：** 一个由按钮点击触发的下拉菜单或弹出框。这些是瞬时交互，不是持久的视图状态。返回一个下拉菜单已打开的页面对用户不友好。

要重置瞬时的打开/关闭状态，在 `useLayoutEffect` 清理函数中关闭它：

```tsx highlight={8-13}
'use client'

import { useState, useLayoutEffect } from 'react'

function SettingsDropdown() {
  const [isOpen, setIsOpen] = useState(false)

  // Close dropdown when this component becomes hidden
  useLayoutEffect(() => {
    return () => {
      setIsOpen(false)
    }
  }, [])

  return (
    <div>
      <button onClick={() => setIsOpen((o) => !o)}>Options</button>
      {isOpen && (
        <ul>
          <li>
            <button>Edit Profile</button>
          </li>
          <li>
            <button>Change Password</button>
          </li>
        </ul>
      )}
    </div>
  )
}
```

当 Activity 隐藏此组件时，清理函数运行并重置 `isOpen`。当页面再次可见时，下拉菜单已关闭。使用 `useLayoutEffect` 确保清理在组件隐藏前同步运行，避免任何过时状态的闪烁。

你也可以使用 `Link` 的 [`onNavigate`](/docs/app/api-reference/components/link#onnavigate) 回调在点击导航链接时立即关闭下拉菜单。

### 对话框和初始化逻辑

Activity 保留对话框的打开/关闭状态。这也会影响基于该状态运行的 Effects。

**何时保留：** 一个用户正在操作的多步骤向导或设置面板。保留步骤和输入状态可以避免丢失进度。

**何时重置：** 一个每次打开时都运行初始化逻辑（如聚焦输入框）的对话框。如果用户在对话框打开时导航离开，Activity 会保留 `isDialogOpen: true`。再次打开它会将值设置为 `true`（当它已经是 `true` 时），因此不会发生状态变化，Effect 也不会重新运行。

考虑这个例子：

```tsx
'use client'

import { useState, useRef, useEffect } from 'react'

function ProductTab() {
  const [isDialogOpen, setIsDialogOpen] = useState(false)
  const inputRef = useRef<HTMLInputElement>(null)

  useEffect(() => {
    if (isDialogOpen) {
      inputRef.current?.focus()
    }
  }, [isDialogOpen])

  // ...
}
```

如果用户在对话框打开时导航离开，返回并打开对话框不会触发聚焦 Effect，因为 `isDialogOpen` 已经是 `true`。

要解决这个问题，可以从保留的组件状态之外的东西（比如搜索参数）派生对话框状态：

```tsx highlight={3,7-9,20,25}
'use client'

import { useSearchParams, useRouter } from 'next/navigation'
import { useEffect, useRef } from 'react'

function ProductTab() {
  const searchParams = useSearchParams()
  const router = useRouter()
  const isDialogOpen = searchParams.get('edit') === 'true'
  const inputRef = useRef<HTMLInputElement>(null)

  useEffect(() => {
    if (isDialogOpen) {
      inputRef.current?.focus()
    }
  }, [isDialogOpen])

  return (
    <div>
      <button onClick={() => router.push('?edit=true')}>Edit Product</button>

      {isDialogOpen && (
        <dialog open>
          <input ref={inputRef} placeholder="Product name" />
          <button onClick={() => router.replace('?', { scroll: false })}>
            Close
          </button>
        </dialog>
      )}
    </div>
  )
}
```

通过这种方法，`isDialogOpen` 从 URL 派生，而不是组件状态。当导航离开并返回时，搜索参数被清除（URL 改变了），所以 `isDialogOpen` 变为 `false`。打开对话框会设置参数，这改变了 `isDialogOpen` 并触发 Effect。

### 表单输入值

Activity 保留表单输入值：在字段中输入的文本、选择的选项、复选框状态。

**何时保留：** 一个带有筛选器的搜索页面、用户正在撰写的草稿，或带有未保存更改的设置表单。保留输入状态是最大的用户体验优势之一，因为用户不会丢失工作。

**何时重置：** 一个“创建新项目”页面，返回时应该重新开始，或成功提交后的联系表单。

当 Activity 隐藏组件时要重置表单字段，使用回调 ref：

```tsx
<form
  ref={(form) => {
    // Cleanup function - runs when Activity hides this component
    return () => form?.reset()
  }}
>
  {/* fields */}
</form>
```

当用户导航离开时，这会重置表单。

### 操作状态 (\

Activity 保留 [`useActionState`](https://react.dev/reference/react/useActionState) 结果：成功消息、错误消息以及操作返回的任何其他状态。

**何时保留：** 一个显示“票券兑换成功”的票券兑换表单，或一个显示“更改已保存”的设置表单。在返回页面时看到之前操作的结果是有用的确认，以便用户知道发生了什么。

**何时重置：** 一个每次访问都应重新开始的“新交易”流程，或一个在新上下文中过时的成功/错误消息会造成混淆的表单。

你可以把 `useActionState` 看作是一个允许副作用的 `useReducer`。它不必只处理表单提交；你可以向它派发任何操作。添加一个 `RESET` 操作为你提供了一种清晰的方式来在 Activity 隐藏组件时清除状态（参见 React 文档中的[重置状态](https://react.dev/reference/react/useActionState#reset-state)）：

```tsx highlight={5-6,9-21,26-35}
'use client'

import { useActionState, useLayoutEffect, useRef, startTransition } from 'react'

type Action = { type: 'SUBMIT'; data: FormData } | { type: 'RESET' }
type State = { success: boolean; error: string | null }

function CommentForm() {
  const [state, dispatch, isPending] = useActionState(
    async (prev: State, action: Action) => {
      if (action.type === 'RESET') {
        return { success: false, error: null }
      }
      // Handle the form submission
      const res = await saveComment(action.data)
      if (!res.ok) return { success: false, error: res.message }
      shouldReset.current = true
      return { success: true, error: null }
    },
    { success: false, error: null }
  )

  const shouldReset = useRef(false)

  // Dispatch RESET when Activity hides this component
  useLayoutEffect(() => {
    return () => {
      if (shouldReset.current) {
        shouldReset.current = false
        startTransition(() => {
          dispatch({ type: 'RESET' })
        })
      }
    }
  }, [dispatch])

  return (
    <form action={(formData) => dispatch({ type: 'SUBMIT', data: formData })}>
      <textarea name="comment" />
      <button type="submit" disabled={isPending}>
        {isPending ? 'Posting...' : 'Post Comment'}
      </button>
      {state.success && <p>Comment posted!</p>}
      {state.error && <p>{state.error}</p>}
    </form>
  )
}
```

以下是逐步发生的情况：

1.  用户提交表单。reducer 接收到带有 `FormData` 的 `SUBMIT` 操作，调用 `saveComment`，并返回 `{ success: true }`。它还设置 `shouldReset.current = true` 以标记需要重置。
2.  用户导航离开。Activity 隐藏组件并运行 `useLayoutEffect` 清理。因为 `shouldReset.current` 是 `true`，它派发一个 `RESET` 操作。
3.  Reducer 接收到 `RESET` 并返回初始状态 (`{ success: false, error: null }`)。过时的成功消息被清除。
4.  如果用户导航返回，表单已准备好接受新的提交。如果他们从未提交（步骤 1 未发生），`shouldReset.current` 仍然是 `false`，因此不派发 `RESET`。表单保持原样。

## 状态和认证

Activity 在导航间保留本地组件状态（`useState`，DOM 输入值），包括认证变化。这是标准的 React 行为：props 改变（例如收到一个新用户）会触发重新渲染，但不会重置现有状态。一个用户撰写的草稿不应被另一个用户看到。

对于登出流程，使用 `window.location.href` 而不是 `router.push` 会触发完整的页面重载，清除所有客户端状态。

要在不进行完整重载的情况下在用户更改时重置特定状态：

```tsx
'use client'

import { useState, useEffect, useRef } from 'react'

function UserScopedForm({ userId }: { userId: string | null }) {
  const [draft, setDraft] = useState('')
  const lastUserIdRef = useRef<string | null>(null)

  useEffect(() => {
    if (lastUserIdRef.current !== null && lastUserIdRef.current !== userId) {
      setDraft('') // Reset on user change
    }
    lastUserIdRef.current = userId
  }, [userId])

  return <textarea value={draft} onChange={(e) => setDraft(e.target.value)} />
}
```

或者，按用户 ID 对组件设置 key，让 React 处理重置：`<Form key={userId} />`。

## 全局样式

页面级样式（CSS 变量、z-index、全局类）在源组件被 Activity 隐藏时可能会影响可见页面。你很可能希望在隐藏时禁用它们：一个隐藏页面的强调色或 z-index 覆盖不应泄漏到可见页面。

使用回调 ref 来切换样式表的 `media` 属性：

```tsx
<style
  ref={(style) => {
    if (style) style.media = '' // Enable when visible
    return () => {
      if (style) style.media = 'not all' // Disable when hidden
    }
  }}
>
  {`:root { --page-accent: blue; }`}
</style>
```

或者在处理多个样式元素或更复杂的清理时使用 `useLayoutEffect`：

```tsx
'use client'

import { useLayoutEffect, useRef } from 'react'

function PageWithStyles() {
  const styleRef = useRef<HTMLStyleElement>(null)

  useLayoutEffect(() => {
    if (styleRef.current) styleRef.current.media = ''
    return () => {
      if (styleRef.current) styleRef.current.media = 'not all'
    }
  }, [])

  return <style ref={styleRef}>{`:root { --page-accent: blue; }`}</style>
}
```

当 Activity 隐藏组件时，清理设置 `media="not all"`，这会禁用样式表。当再次可见时，effect 重新运行并重置 `media` 以启用它。

## 测试

隐藏的 Activity 内容具有 `display: none`，但仍保留在文档中。这既适用于由缓存组件保留的路由，也适用于你直接用 `<Activity>` 隐藏的内容。它会影响使用 Playwright、Cypress 或 Puppeteer 等工具进行的端到端测试：

-   **DOM 查询可以找到隐藏元素。** 选择器可能会匹配到元素，无论其可见性如何。
-   **与隐藏元素的交互会失败或超时。** 大多数工具在交互前会等待元素变为可见。
-   **断言可能匹配到隐藏内容。** 在断言元素存在时，请明确指定可见性。

### 使用感知可见性的选择器

在 Playwright 中，`getByRole` 查询会自动按可见性过滤：

```ts
// Good - getByRole filters by visibility automatically
await page.getByRole('button', { name: 'Submit' }).click()
await page.getByRole('textbox', { name: 'Email' }).fill('test@example.com')

// Also good - getByLabel, getByPlaceholder filter by visibility
await page.getByLabel('Email').fill('test@example.com')
await page.getByPlaceholder('Search...').fill('query')
```

当 `getByRole` 不适用时，使用 `.locator()` 配合可见性过滤：

```ts
// Fallback - filter by visibility explicitly
await page.locator('.product-card').filter({ visible: true }).first().click()
await page
  .locator('[data-testid="timer"]')
  .filter({ visible: true })
  .textContent()

// Avoid - may match hidden elements in Activity boundaries
await page.locator('.product-card').first().click()
```

`getByRole` 对 Activity、标签页导航、手风琴以及任何其他将隐藏内容保留在 DOM 中的模式都具有鲁棒性。它查询可访问性树，该树排除了隐藏元素。对于其他测试工具，请查阅其文档以了解感知可见性的选择器。例如，Cypress 使用 `.should('be.visible')` 或 `{ visible: true }` 选项。

## 在组件中使用 Activity

缓存组件在路由级别自动使用 Activity，但你也可以在自己的组件中直接使用 `<Activity>`。这对于标签页、可展开面板或任何你希望隐藏内容而不卸载它的 UI 非常有用。

### 预渲染隐藏内容

Activity 可以预渲染用户尚未看到的内容。隐藏边界以较低优先级渲染。结合 Suspense，这允许你为用户接下来可能查看的内容预取数据。

一个服务器组件可以立即开始获取数据，并将 promise 传递给客户端组件。客户端组件使用 Activity 隐藏内容直到用户请求它，并使用 `use()` 在渲染时解析 promise：

```tsx filename="app/page.tsx"
import { Suspense } from 'react'
import { ExpandableComments } from './expandable-comments'

async function getCommentsData() {
  return db.comments.findMany()
}

export default function Page() {
  const commentsPromise = getCommentsData()

  return (
    <article>
      <h1>Post Title</h1>
      <p>Main content visible immediately...</p>

      <ExpandableComments commentsPromise={commentsPromise} />
    </article>
  )
}
```

```tsx filename="app/expandable-comments.tsx"
'use client'

import { Activity, Suspense, useState, use } from 'react'

type Comment = { id: string; text: string; author: string }

export function ExpandableComments({
  commentsPromise,
}: {
  commentsPromise: Promise<Comment[]>
}) {
  const [expanded, setExpanded] = useState(false)

  return (
    <>
      <button onClick={() => setExpanded((e) => !e)}>
        {expanded ? 'Hide Comments' : 'Show Comments'}
      </button>

      <Activity mode={expanded ? 'visible' : 'hidden'}>
        <Suspense fallback={<CommentsSkeleton />}>
          <Comments commentsPromise={commentsPromise} />
        </Suspense>
      </Activity>
    </>
  )
}

function Comments({
  commentsPromise,
}: {
  commentsPromise: Promise<Comment[]>
}) {
  const comments = use(commentsPromise)
  return (
    <ul>
      {comments.map((c) => (
        <li key={c.id}>{c.text}</li>
      ))}
    </ul>
  )
}

function CommentsSkeleton() {
  return <div>Loading comments...</div>
}
```

服务器组件立即开始获取评论并传递 promise。在隐藏状态下，数据以较低优先级流式传输。当用户点击“Show Comments”时，`Comments` 组件用 `use()` 解析 promise，内容立即出现。

### 效果 (Effect) 清理和媒体清理

当 Activity 隐藏内容时，React 会运行 effect 清理函数，就像在卸载时一样。这意味着如果你有适当的清理，计时器、订阅和媒体播放会自动暂停：

```tsx
'use client'

import { useEffect, useState } from 'react'

function LiveTimer() {
  const [count, setCount] = useState(0)

  useEffect(() => {
    const id = setInterval(() => setCount((c) => c + 1), 1000)
    return () => clearInterval(id) // Pauses when hidden
  }, [])

  return <p>Count: {count}</p>
}
```

对于像 `<video>` 和 `<audio>` 这样的媒体元素，`display: none` 不会停止播放。用 `useLayoutEffect` 添加显式清理：

```tsx
'use client'

import { useLayoutEffect, useRef } from 'react'

function VideoPlayer({ src }: { src: string }) {
  const videoRef = useRef<HTMLVideoElement>(null)

  useLayoutEffect(() => {
    const video = videoRef.current
    return () => {
      video?.pause() // Pauses when hidden, preserves playback position
    }
  }, [])

  return <video ref={videoRef} src={src} controls />
}
```

当组件再次变为可见时，effects 会重新运行，并且由于 DOM 节点从未被移除，播放位置得以保留。

### 区分初次挂载和再次显示

Effects 在每次从隐藏到可见的转换时都会运行，而不仅仅是初始挂载。如果你需要区分首次挂载与后续的可见性变化，使用一个 ref：

```tsx
'use client'

import { useEffect, useRef } from 'react'

function TrackedComponent() {
  const hasMountedRef = useRef(false)

  useEffect(() => {
    if (!hasMountedRef.current) {
      hasMountedRef.current = true
      console.log('First mount')
    } else {
      console.log('Became visible again')
    }
  }, [])

  return <div>...</div>
}
```

该 ref 在隐藏/显示周期中持续存在（ref 不会被清理），所以 `hasMountedRef.current` 在首次挂载后保持 `true`。每次 Activity 变为可见时，Effect 再次运行，但现在它会执行 `else` 分支。

## 示例

[Activity Patterns Demo](https://react-activity-patterns.vercel.app/)（[源码](https://github.com/vercel-labs/react-activity-patterns)）是一个启用了缓存组件的 Next.js 应用，包含三个路由。在它们之间导航以查看状态保留的实际效果：

-   **数据** — 可排序表格和可选择列表，在导航间保持状态，以及一个在后台预渲染的评论部分
-   **表单** — 带有 DOM 状态（`<details>`、复选框、文本输入）的筛选面板会持久保存，以及一个使用 `useLayoutEffect` 清理在提交后重置的新闻通讯表单
-   **副作用** — 一个在导航离开时暂停、返回时恢复的实况计时器，以及一个自动暂停并保留播放位置的视频播放器
