[模型上下文协议 (Model Context Protocol, MCP)](https://modelcontextprotocol.io) 是一项开放标准，允许 AI 代理和编码助手通过标准化接口与您的应用程序交互。

Next.js 16+ 包含 MCP 支持，使编码代理能够实时访问您应用程序的内部状态。要使用此功能，请安装 [`next-devtools-mcp`](https://www.npmjs.com/package/next-devtools-mcp) 包。

## 快速开始

**要求：** Next.js 16 或更高版本

将 `next-devtools-mcp` 添加到项目根目录的 `.mcp.json` 文件中：

```json filename=".mcp.json"
{
  "mcpServers": {
    "next-devtools": {
      "command": "npx",
      "args": ["-y", "next-devtools-mcp@latest"]
    }
  }
}
```

就这样！当您启动开发服务器时，`next-devtools-mcp` 将自动发现并连接到您正在运行的 Next.js 实例。

更多配置选项，请参阅 [next-devtools-mcp 代码仓库](https://github.com/vercel/next-devtools-mcp)。

## 能力

`next-devtools-mcp` 为编码代理提供了一套不断扩展的功能：

### 应用运行时访问

- **错误检测**：从开发服务器检索当前的构建错误、运行时错误和类型错误。
- **实时状态查询**：访问实时的应用程序状态和运行时信息。
- **页面元数据**：查询页面路由、组件和渲染详情。
- **服务器操作**：检查服务器操作和组件层次结构。
- **开发日志**：访问开发服务器日志和控制台输出。

### 开发工具

- **Next.js 知识库**：查询全面的 Next.js 文档和最佳实践。
- **迁移和升级工具**：通过 codemod 自动辅助升级到 Next.js 16。
- **缓存指南**：为缓存组件提供设置和配置协助。
- **浏览器测试**：集成 [Playwright MCP](https://github.com/microsoft/playwright-mcp)，用于在浏览器中验证页面。

> **注意：** Next.js 团队正在积极扩展这些功能。会定期添加新工具和新特性，以改善代理开发体验。

## 开发工作流

1.  启动您的 Next.js 开发服务器：

```bash package="pnpm"
pnpm dev
```

```bash package="npm"
npm run dev
```

```bash package="yarn"
yarn dev
```

```bash package="bun"
bun dev
```

2.  您的编码代理将通过 `next-devtools-mcp` 自动连接到正在运行的 Next.js 实例。

3.  在浏览器中打开您的应用程序以查看页面。

4.  向您的代理查询分析信息和诊断数据（请参见下面的示例）。

### 可用工具

通过 `next-devtools-mcp`，代理可以使用以下工具：

- **`get_errors`**：从开发服务器检索当前的构建错误、运行时错误和类型错误。
- **`get_logs`**：获取包含浏览器控制台日志和服务器输出的开发日志文件路径。
- **`get_page_metadata`**：获取特定页面的元数据，包括路由、组件和渲染信息。
- **`get_project_metadata`**：检索项目结构、配置和开发服务器 URL。
- **`get_routes`**：通过扫描文件系统获取所有将成为入口点的路由。按路由器类型（appRouter、pagesRouter）分组返回路由。动态段显示为 `[param]` 或 `[...slug]` 模式。
- **`get_server_action_by_id`**：通过其 ID 查找服务器操作，以定位源文件和函数名称。

## 与 Agent 配合使用

兼容 MCP 的编码代理可以自动发现并连接到您的 Next.js 开发服务器，为它们提供有关您项目的丰富上下文。

### Agent 辅助开发的好处

代理可以：

- **提出上下文感知的建议**：根据您现有的结构，推荐添加新功能的正确位置。
- **查询实时应用程序状态**：在开发期间检查当前配置、路由和中间件。
- **理解您的 App Router 页面布局**：准确知道渲染了哪个页面和布局。
- **提供准确的实现**：生成遵循您项目模式和约定的代码。

### 升级与最佳实践

获取有关 Next.js 概念和迁移的帮助：

```bash
User: "Help me upgrade my Next.js app to version 16"
```

代理将分析您当前的版本，指导您使用 codemod 进行自动迁移，并提供处理破坏性更改的分步说明。

提出概念性问题：

```bash
User: "When should I use 'use client' in App Router?"
```

代理将查询 Next.js 知识库，并结合您代码库中的示例，提供基于文档的解释。

## 工作原理

Next.js 16+ 在您的开发服务器内部，于 `/_next/mcp` 路径下包含一个内置的 MCP 端点。`next-devtools-mcp` 包会自动发现并与这些端点通信，使其能够：

- 连接到在不同端口上运行的多个 Next.js 实例。
- 将工具调用转发到适当的 Next.js 开发服务器。
- 为编码代理提供统一的接口。

这种架构将代理接口与内部实现解耦，使 `next-devtools-mcp` 能够在不同的 Next.js 项目中无缝工作。

## 故障排除

### MCP 服务器未连接

- 确保您使用的是 Next.js v16 或更高版本。
- 验证 `next-devtools-mcp` 已在您的 `.mcp.json` 中配置。
- 启动您的开发服务器：`npm run dev`。
- 如果开发服务器已在运行，请重启它。
- 检查您的编码代理是否已加载 MCP 服务器配置。
