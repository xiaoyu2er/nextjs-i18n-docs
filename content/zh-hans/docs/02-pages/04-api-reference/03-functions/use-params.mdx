`useParams` 是一个钩子，允许您读取由当前 URL 填充的路由 [动态参数](/docs/pages/building-your-application/routing/dynamic-routes)。

```tsx filename="pages/shop/[slug].tsx" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams<{ slug: string }>()

  if (!params) {
    // Render fallback UI while params are not yet available
    return null
  }

  // Route -> /shop/[slug]
  // URL -> /shop/shoes
  // `params` -> { slug: 'shoes' }
  return <>Shop: {params.slug}</>
}
```

```jsx filename="pages/shop/[slug].js" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams()

  if (!params) {
    // Render fallback UI while params are not yet available
    return null
  }

  // Route -> /shop/[slug]
  // URL -> /shop/shoes
  // `params` -> { slug: 'shoes' }
  return <>Shop: {params.slug}</>
}
```

## 参数

```tsx
const params = useParams()
```

`useParams` 不接受任何参数。

## 返回

`useParams` 返回一个包含当前路由填充的 [动态参数](/docs/pages/building-your-application/routing/dynamic-routes) 的对象，或者在 [预渲染](#behavior-during-prerendering) 期间为 `null`。

- 对象中的每个属性都是一个活动的动态段。
- 属性名是段的名称，属性值是该段填充的内容。
- 属性值要么是 `string`，要么是 `string` 的数组，具体取决于 [动态段的类型](/docs/pages/building-your-application/routing/dynamic-routes)。
- 如果路由不包含任何动态参数，`useParams` 返回一个空对象。

例如：

| 路由                            | URL         | `useParams()`             |
| ----------------------------- | ----------- | ------------------------- |
| `pages/shop/page.js`          | `/shop`     | `{}`                      |
| `pages/shop/[slug].js`        | `/shop/1`   | `{ slug: '1' }`           |
| `pages/shop/[tag]/[item].js`  | `/shop/1/2` | `{ tag: '1', item: '2' }` |
| `pages/shop/[...slug].js`     | `/shop/1/2` | `{ slug: ['1', '2'] }`    |

> **小提示**： `useParams` 是一个 [React 钩子](https://react.dev/learn#using-hooks)，无法与类一起使用。

## 行为

### 预渲染期间的行为

对于 [静态优化](/docs/pages/building-your-application/rendering/automatic-static-optimization) 的页面，`useParams` 在首次渲染时返回 `null`。在客户端水合后，一旦路由器准备就绪，值将更新为实际的参数。

这是因为在动态路由的静态生成期间，无法得知参数值。

```tsx filename="pages/shop/[slug].tsx" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams<{ slug: string }>()

  if (!params) {
    // Return a fallback UI while params are loading
    // This prevents hydration mismatches
    return <ShopPageSkeleton />
  }

  return <>Shop: {params.slug}</>
}
```

```jsx filename="pages/shop/[slug].js" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams()

  if (!params) {
    // Return a fallback UI while params are loading
    // This prevents hydration mismatches
    return <ShopPageSkeleton />
  }

  return <>Shop: {params.slug}</>
}
```

### 与 `getServerSideProps` 配合使用

使用 [`getServerSideProps`](/docs/pages/building-your-application/data-fetching/get-server-side-props) 时，页面在每个请求时都会进行服务器端渲染，`useParams` 将立即返回实际参数：

```tsx filename="pages/shop/[slug].tsx" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams<{ slug: string }>()

  // With getServerSideProps, this fallback is never rendered because
  // params is always available on the server. However, keeping
  // the fallback allows this component to be reused on other pages
  // that may not use getServerSideProps.
  if (!params) {
    return null
  }

  return <>Shop: {params.slug}</>
}

export async function getServerSideProps() {
  return { props: {} }
}
```

```jsx filename="pages/shop/[slug].js" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams()

  // With getServerSideProps, this fallback is never rendered because
  // params is always available on the server. However, keeping
  // the fallback allows this component to be reused on other pages
  // that may not use getServerSideProps.
  if (!params) {
    return null
  }

  return <>Shop: {params.slug}</>
}

export async function getServerSideProps() {
  return { props: {} }
}
```

### 与 `router.query` 的对比

`useParams` 仅返回动态路由参数，而 `useRouter` 中的 [`router.query`](/docs/pages/api-reference/functions/use-router#router-object) 包含动态参数和查询字符串参数。

```tsx filename="pages/shop/[slug].tsx" switcher
import { useRouter } from 'next/router'
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const router = useRouter()
  const params = useParams()

  // URL -> /shop/shoes?color=red

  // router.query -> { slug: 'shoes', color: 'red' }
  // params -> { slug: 'shoes' }

  // ...
}
```

```jsx filename="pages/shop/[slug].js" switcher
import { useRouter } from 'next/router'
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const router = useRouter()
  const params = useParams()

  // URL -> /shop/shoes?color=red

  // router.query -> { slug: 'shoes', color: 'red' }
  // params -> { slug: 'shoes' }

  // ...
}
```

## 示例

### 与 App Router 共享组件

来自 `next/navigation` 的 `useParams` 在 Pages Router 和 App Router 中均可用。这允许您创建在任一上下文中均可使用的共享组件：

```tsx filename="components/breadcrumb.tsx" switcher
import { useParams } from 'next/navigation'

// This component works in both pages/ and app/
export function Breadcrumb() {
  const params = useParams<{ slug: string }>()

  if (!params) {
    // Fallback for Pages Router during prerendering
    return <nav>Home / ...</nav>
  }

  return <nav>Home / {params.slug}</nav>
}
```

```jsx filename="components/breadcrumb.js" switcher
import { useParams } from 'next/navigation'

// This component works in both pages/ and app/
export function Breadcrumb() {
  const params = useParams()

  if (!params) {
    // Fallback for Pages Router during prerendering
    return <nav>Home / ...</nav>
  }

  return <nav>Home / {params.slug}</nav>
}
```

> **小提示**：当在 App Router 中使用此组件时，`useParams` 永远不会返回 `null`，因此不会渲染备用分支。
