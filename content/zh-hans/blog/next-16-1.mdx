Next.js 打包分析器（实验性）[](#nextjs-bundle-analyzer-experimental)
------------------------------------------------------------------------------

Next.js 16.1 包含一个新的实验性 [打包分析器](/docs/app/guides/package-bundling#nextjs-bundle-analyzer-experimental)，可与 Turbopack 配合使用。它有助于优化服务器和客户端代码的打包大小，提高 Core Web Vitals 指标，减少 Lambda 冷启动时间，并识别臃肿的依赖项。

终端

```
next experimental-analyze
```

针对 `next dev` 的 Turbopack 文件系统缓存现在已稳定并默认开启。编译器工件存储于磁盘，在重启开发服务器时带来显著更快的编译时间，特别对于大型项目。

包分析器与 Next.js 深度集成，允许您：

*   按路由过滤包
*   查看显示模块为何被包含在内的完整导入链
*   追踪服务器到客户端组件边界和动态导入中的导入
*   查看 CSS 和其他导入资产的大小

包分析器处于早期开发阶段，将在未来的版本中进一步改进。请在专门的 [GitHub 讨论](https://github.com/vercel/next.js/discussions/86731) 中分享您的反馈。

使用 `next dev --inspect` 简化调试[](#easier-debugging-with-next-dev---inspect)
---------------------------------------------------------------------------------------

通过将 `--inspect` 传递给 `next dev`，您现在可以启用 [Node.js 调试器](https://nodejs.org/en/learn/getting-started/debugging)。以前这需要传递 `NODE_OPTIONS=--inspect`，并且会将检查器附加到 Next.js 启动的所有进程，而不仅仅是运行您代码的进程。

改进了 `serverExternalPackages` 的处理[](#improved-handling-of-serverexternalpackages)
---------------------------------------------------------------------------------------------

Next.js 允许您使用 [`serverExternalPackages`](https://nextjs.org/docs/app/api-reference/config/next-config-js/serverExternalPackages) 保持依赖项不打包。以前，这仅在直接依赖项上可靠工作。如果您使用的库在内部依赖于诸如 `sqlite` 之类的内容，并且需要外部化 `sqlite`，您必须将其添加到自己的 `package.json` 中——即使它不是您的直接依赖项。这种变通方法泄露了内部实现细节，造成了维护负担，并可能导致多个包需要同一依赖项的不同版本时的版本冲突。

Next.js 16.1 为 Turbopack 修复了此问题，现在它能正确解析和外部化 `serverExternalPackages` 中的传递依赖项，无需额外配置。

其他更新[](#other-updates)
-------------------------------

*   **20MB 更小的安装**：Next.js 安装体积减少约 20MB，这得益于 Turbopack 文件系统缓存层的简化。
*   **新的 `next upgrade` 命令**：一个新的 [`next upgrade`](/docs/app/getting-started/upgrading#latest-version) 命令使升级更容易。此后，您只需运行此命令即可升级 Next.js 版本。
*   **MCP `get_routes` 工具**：[Next.js DevTools MCP 服务器](/docs/app/guides/mcp) 现在有一个 `get_routes` 工具，可获取应用程序中所有路由的完整列表。
*   **`generateStaticParams` 时机**：在 [`generateStaticParams`](/docs/app/api-reference/functions/generate-static-params) 上花费的时间现在作为开发中请求显示的计时数据的一部分进行记录。
*   **构建工作线程日志**：`next build` 的“收集页面数据”和“生成静态页面”现在记录所使用的工作线程数量。
*   **改进的异步导入打包**：Turbopack 改进了开发中异步导入的打包，以减少生成的块数，避免某些虽属病态但现实存在的案例。
*   **相对源映射路径**：Turbopack 现在为服务端代码生成带有相对文件路径的源映射，提高了与 Node.js 和其他生态系统工具的兼容性。
