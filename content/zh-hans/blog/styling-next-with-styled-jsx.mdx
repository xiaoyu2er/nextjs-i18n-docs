[Styled JSX](https://github.com/vercel/styled-jsx) 是一种 CSS-in-JS 库，允许您编写封装的和局部的 CSS 来为您的组件样式化。您所引入的一个组件的样式不会影响其他组件，这使您在添加、更改和删除样式时不必担心产生意外副作用。

开始使用 [](#getting-started)
-----------------------------------

Next.js 默认包含 Styled JSX，因此开始使用就像在一个现有的 React 元素中添加一个 `<style jsx>` 标签并在其中编写 CSS 一样简单：

取而代之的是，Next.js 会自动注入 CSS 文件。在生产环境中，Next.js 还自动在 CSS URL 中添加内容哈希，因此如果有更改，可确保您的最终用户永远不会获得文件的过时版本，并且具有引入不可变永久缓存的能力。

```
function Home() {
  return (
    <div className="container">
      <h1>Hello Next.js</h1>
      <p>Let's explore different ways to style Next.js apps</p>
      <style jsx>{`
        .container {
          margin: 50px;
        }
        p {
          color: blue;
        }
      `}</style>
    </div>
  );
}
 
export default Home;
```

在此示例中，我们正在为组件的容器元素和段落包含样式。即使我们使用通用选择器，样式也不会影响其他组件中具有 `container` 类名或 `<p>` 标签的元素。这是因为 Styled JSX 确保样式仅限定于该组件（通过应用附加的唯一类名到样式化元素）。

只需向 `<style>` 元素添加单个 `jsx` 属性，您就可以编写经过自动前缀并自动限定于组件的标准 CSS。`<style jsx>` 元素应放在组件的根元素内。

---------------------------------------------

大多数项目需要一些全局样式来样式化 body 元素或提供 CSS 重置。Styled JSX 允许我们使用 `<style jsx global>` 添加全局样式。例如：

取而代之的是，Next.js 会自动注入 CSS 文件。在生产环境中，Next.js 还自动在 CSS URL 中添加内容哈希，因此如果有更改，可确保您的最终用户永远不会获得文件的过时版本，并且具有引入不可变永久缓存的能力。

```
function Home() {
  return (
    <div className="container">
      <h1>Hello Next.js</h1>
      <p>Let's explore different ways to style Next.js apps</p>
      <style jsx>{`
        .container {
          margin: 50px;
        }
        p {
          color: blue;
        }
      `}</style>
      <style jsx global>{`
        p {
          font-size: 20px;
        }
      `}</style>
    </div>
  );
}
 
export default Home;
```

这将把 20px 的字体大小应用于此特定页面中所有的 `<p>` 标签。

若要将全局样式应用到应用中的所有页面，一个好方法是先创建一个带有全局样式的布局组件，然后用它包装所有页面。

使用布局组件提供了灵活性，可以对某些页面应用一组特定样式，同时仍然允许其他页面使用不同的样式：

components/Layout.js

```
function Layout(props) {
  return (
    <div className="page-layout">
      {props.children}
      <style jsx global>{`
        body {
          margin: 0;
          padding: 0;
          font-size: 18px;
          font-weight: 400;
          line-height: 1.8;
          color: #333;
          font-family: sans-serif;
        }
        h1 {
          font-weight: 700;
        }
        p {
          margin-bottom: 10px;
        }
      `}</style>
    </div>
  );
}
 
export default Layout;
```

在 Next.js 中，可以通过在 `pages/_app.js` 内创建自定义 [`App` 组件](/docs#custom-app)，导入 `Layout` 组件，然后将其添加到渲染方法中来一次性为所有页面加载布局，如下所示：

`pages/_app.js`

```
import React from 'react';
import App, { Container } from 'next/app';
import Layout from '../components/Layout';
 
class MyApp extends App {
  render() {
    const { Component, pageProps } = this.props;
 
    return (
      <Container>
        <Layout>
          <Component {...pageProps} />
        </Layout>
      </Container>
    );
  }
}
 
export default MyApp;
```

### 在外部文件中编写样式 [](#writing-styles-in-external-files)

我们也可以在组件外部的外部文件中编写样式。

例如，我们可以将 `Layout` 组件中的全局样式移动到一个单独的文件中，如下所示：

styles/global.js

```
import css from 'styled-jsx/css';
 
export default css.global`
  body {
    margin: 0;
    padding: 0;
    font-size: 18px;
    font-weight: 400;
    line-height: 1.8;
    color: #333;
    font-family: sans-serif;
  }
  h1 {
    font-weight: 700;
  }
  p {
    margin-bottom: 10px;
  }
`;
```

然后我们可以将样式导回 `Layout` 组件：

components/Layout.js

```
import globalStyles from '../styles/global.js';
 
function Layout(props) {
  return (
    <div className="page-layout">
      {props.children}
      <style jsx global>
        {globalStyles}
      </style>
    </div>
  );
}
 
export default Layout;
```

### 一次性全局选择器 [](#one-off-global-selectors)

我们使用 `<style jsx>` 添加到组件中的样式仅影响该组件内的元素，而不影响子组件。

有时候，我们可能需要覆盖子组件的特定样式。为此，Styled JSX 提供了 `:global()`，提供对 _一次性全局选择器_ 的访问。

例如，假设我们要有一个 `<Widget>` 组件，其中包含一个类名为 `btn` 的按钮。如果我们只想在小部件在主页导入时更改此按钮的颜色，我们可以像这样操作：

取而代之的是，Next.js 会自动注入 CSS 文件。在生产环境中，Next.js 还自动在 CSS URL 中添加内容哈希，因此如果有更改，可确保您的最终用户永远不会获得文件的过时版本，并且具有引入不可变永久缓存的能力。

```
import Widget from '../components/Widget';
 
function Home() {
  return (
    <div className="container">
      <h1>Hello Next.js</h1>
      <Widget />
      <style jsx>{`
        .container {
          margin: 50px;
        }
        .container :global(.btn) {
          background: #000;
          color: #fff;
        }
      `}</style>
    </div>
  );
}
 
export default Home;
```

### 动态样式 [](#dynamic-styles)

与其他解决方案相比，能够根据组件的 props 调整组件的样式是 `CSS-in-JS` 库的一大优势。

使用 Styled JSX 我们可以这样做：

components/Alert.js

```
function Alert(props) {
  return (
    <div className="alert">
      {props.children}
      <style jsx>{`
        .alert {
          display: inline-block;
          padding: 20px;
          border-radius: 8px;
        }
      `}</style>
      <style jsx>{`
        .alert {
          background: ${props.type == 'warning' ? '#fff3cd' : '#eee'};
        }
      `}</style>
    </div>
  );
}
 
export default Alert;
```

如果 `Alert` 组件传入了 `type` 为 `warning` 的 `type` 属性，例如：

```
<Alert type="warning">This is an important message</Alert>
```

该组件将具有橙色背景。未指定 `type` 属性时，背景将回退到默认灰色。

请注意我们将动态样式放置在了单独的 `<style jsx>` 标签中。这不是必需的，但**建议**将静态样式和动态样式分开，这样当 props 变化时只有动态部分会被重新计算。

根据 props 调整样式的另一种方法是基于 props 值应用不同的类名，如下所示：

components/Alert.js

```
function Alert(props) {
  return (
    <div className={props.type == 'warning' ? 'alert warning' : 'alert'}>
      {props.children}
      <style jsx>{`
        .alert {
          display: inline-block;
          padding: 20px;
          border-radius: 8px;
          background: #eee;
        }
        .alert.warning {
          background: #fff3cd;
        }
      `}</style>
    </div>
  );
}
 
export default Alert;
```

### 创建网站主题 [](#creating-a-site-theme)

主题可以是一个简单的对象，其中包含我们在应用中可能需要的最常见变量：

styles/theme.js

```
const theme = {
  fontFamily: {
    sansSerif: '-apple-system, "Helvetica Neue", Arial, sans-serif',
    mono: 'Menlo, Monaco, monospace',
  },
  colors: {
    text: '#333',
    background: '#fff',
    link: '#1eaaf1',
    linkHover: '#0d8ecf',
    border: '#ddd',
    warning: '#fff3cd',
    success: '#d4edda',
  },
};
 
export default theme;
```

然后我们在组件中导入这个主题文件，并用变量替换 _硬编码_ 值：

components/Layout.js

```
import theme from '../styles/theme';
 
function Layout(props) {
  return (
    <div className="page-wrapper">
      {props.children}
      <style jsx global>{`
        body {
          background: ${theme.colors.background};
          color: ${theme.colors.text};
          font-family: ${theme.fontFamily.sansSerif};
        }
      `}</style>
      <style jsx global>{`
        body {
          margin: 0;
          padding: 0;
          font-size: 18px;
          font-weight: 400;
          line-height: 1.8;
        }
        h1 {
          font-weight: 700;
        }
        p {
          margin-bottom: 10px;
        }
      `}</style>
    </div>
  );
}
 
export default Layout;
```

components/Alert.js

```
import theme from '../styles/theme';
 
function Alert(props) {
  return (
    <div className="alert">
      {props.children}
      <style jsx>{`
        .alert {
          display: inline-block;
          padding: 20px;
          border-radius: 8px;
        }
      `}</style>
      <style jsx>{`
        .alert {
          background: ${props.type == 'warning'
            ? theme.colors.warning
            : theme.colors.light};
        }
      `}</style>
    </div>
  );
}
 
export default Alert;
```
