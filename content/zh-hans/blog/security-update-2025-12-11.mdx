React 服务器组件 (React Server Components, RSC) 协议中发现另外两个漏洞。这些问题是在安全研究人员审查 [React2Shell](/blog/CVE-2025-66478) 补丁时发现的。**重要的是，这两个新漏洞均不允许远程代码执行 (Remote Code Execution, RCE)**。React2Shell 的补丁仍然完全有效。

这些漏洞源于上游 React 实现 ([CVE-2025-55183](https://www.cve.org/CVERecord?id=CVE-2025-55183), [CVE-2025-55184](https://www.cve.org/CVERecord?id=CVE-2025-55184))。本公告跟踪这些漏洞对使用 App Router 的 Next.js 应用的下游影响。更多详细信息，请参阅 [React 博客文章](https://react.dev/blog/2025/12/11/denial-of-service-and-source-code-exposure-in-react-server-components)。

> **补充说明：** CVE-2025-55184 的初始修复不完整。完整的修复已通过 [CVE-2025-67779](https://www.cve.org/CVERecord?id=CVE-2025-67779) 发布。如果您先前已升级到最初推荐的版本之一，请再次升级到下面列出的最新修复版本。

影响 [](#impact)
-----------------

### 拒绝服务 (DoS)：[CVE-2025-55184](https://www.cve.org/CVERecord?id=CVE-2025-55184) (高风险) [](#denial-of-service-cve-2025-55184-high-severity)

可以向任何 App Router 端点发送精心构造的 HTTP 请求，该请求在反序列化时会导致无限循环，挂起服务器进程并阻止后续 HTTP 请求得到服务。

> **注意：** 该漏洞的初始修复不完整。完整的修复已通过 [CVE-2025-67779](https://www.cve.org/CVERecord?id=CVE-2025-67779) 发布。此前已升级的用户必须再次升级到最新的修复版本。

### 源代码泄露 (Source Code Exposure)：[CVE-2025-55183](https://www.cve.org/CVERecord?id=CVE-2025-55183) (中风险) [](#source-code-exposure-cve-2025-55183-medium-severity)

精心构造的 HTTP 请求可导致服务器函数 (Server Function) 返回您的应用程序中其他 [服务器函数](https://react.dev/reference/rsc/server-functions) 的编译源代码。这可能会泄露业务逻辑。如果您的代码中直接定义了机密信息（而不是在运行时通过环境变量访问）并在服务器函数中引用了这些信息，机密信息也可能泄露。根据您的打包器配置，这些值可能会被内联到编译好的函数输出中。

受影响的 Next.js 版本及修复版本 [](#affected-and-fixed-nextjs-versions)
--------------------------------------------------------------------------

使用带 App Router 的 React 服务器组件的应用程序受到影响。下表显示了哪些版本受每种漏洞影响以及相应的修复版本：

| Version | DoS (CVE-2025-55184) | Source Code Exposure (CVE-2025-55183) | Fixed In |
| --- | --- | --- | --- |
| \>=13.3 | ✓ | — | Upgrade to 14.2.35 |
| 14.x | ✓ | — | 14.2.35 |
| 15.0.x | ✓ | ✓ | 15.0.7 |
| 15.1.x | ✓ | ✓ | 15.1.11 |
| 15.2.x | ✓ | ✓ | 15.2.8 |
| 15.3.x | ✓ | ✓ | 15.3.8 |
| 15.4.x | ✓ | ✓ | 15.4.10 |
| 15.5.x | ✓ | ✓ | 15.5.9 |
| 15.x canary | ✓ | ✓ | 15.6.0-canary.60 |
| 16.0.x | ✓ | ✓ | 16.0.10 |
| 16.x canary | ✓ | ✓ | 16.1.0-canary.19 |

Pages Router 应用程序不受影响，但我们仍建议升级到修复版本。

所需操作 [](#required-action)
--------------------------------

所有用户都应在其发布分支中升级到最新的修复版本：

如果您使用的是 Next.js >=13.3、14.0.x 或 14.1.x，请升级为最新的 14.2.x 版本。

终端

```
npm install next@14.2.35  # for 14.x
npm install next@15.0.7   # for 15.0.x
npm install next@15.1.11  # for 15.1.x
npm install next@15.2.8   # for 15.2.x
npm install next@15.3.8   # for 15.3.x
npm install next@15.4.10  # for 15.4.x
npm install next@15.5.9   # for 15.5.x
npm install next@16.0.10  # for 16.0.x
 
npm install next@15.6.0-canary.60   # for 15.x canary releases
npm install next@16.1.0-canary.19   # for 16.x canary releases
```

运行 `npx fix-react2shell-next` 以启动交互式工具，该工具可以检查版本并根据上述推荐版本执行确定性版本更新。有关详细信息，请查看 [GitHub 仓库](https://github.com/vercel-labs/fix-react2shell-next)。

终端

```
npx fix-react2shell-next
```

没有变通方法。必须升级到修复版本。
