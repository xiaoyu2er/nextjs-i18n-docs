我们在 [Next.js Conf](https://nextjs.org/conf) 上宣布，Next.js 13 (稳定版) 奠定了无限制动态的基础：

*   [**`app` 目录（Beta）:**](#new-app-directory-beta) 更简单、更快、更少的客户端 JavaScript。
    *   [**布局**](#layouts)
    *   [**React 服务端组件**](#server-components)
    *   [**流式传输**](#streaming)
*   **[Turbopack（Alpha）:](#introducing-turbopack-alpha)** 基于 Rust 的 Webpack 替代方案，速度快 700 倍。
*   **[新的 `next/image`:](#nextimage)** 带有原生浏览器延迟加载，速度更快。
*   **[新的 `@next/font`（Beta）:](#nextfont)** 自动自托管字体，零布局位移。
*   **[改进的 `next/link`:](#breaking-changes)** 简化的 API，自动包含 `<a>`。

Next.js 13 和 `pages` 目录是稳定且可用于生产的。今天就通过运行以下命令进行更新：

终端

```
npm i next@latest react@latest react-dom@latest eslint-config-next@latest
```

新的 `app` 目录（Beta）[](#new-app-directory-beta)
-----------------------------------------------------

今天，我们通过在 Next.js 中引入 `app` 目录来改进路由和布局体验，并与 React 的未来保持一致。这是对之前发布的 [布局 RFC](https://nextjs.org/blog/layouts-rfc) 以征求社区反馈的后续。

`app` 目录目前处于 Beta 阶段，我们不建议在生产环境使用。您可以使用 Next.js 13 配合 `pages` 目录使用稳定功能，如改进的 [`next/image`](#nextimage) 和 [`next/link`](#nextlink) 组件，并按自己的节奏选择加入 `app` 目录。`pages` 目录在可预见的未来将继续得到支持。

`app` 目录包括对以下内容的支持：

*   **[布局：](#layouts)** 在路由之间轻松共享 UI，同时保持状态并避免昂贵的重新渲染。
*   **[服务端组件：](#server-components)** 使服务端优先成为动态应用程序的默认值。
*   **[流式传输：](#streaming)** 显示即时加载状态，并在渲染时流式传输 UI 单元。
*   **[支持数据获取：](#data-fetching)** `async` 服务端组件和扩展的 `fetch` API 支持组件级数据获取。

![`app` 目录可以从您现有的 pages/ 目录中采用增量方式。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/layouts-rfc/app-folder.png)

`app` 目录可以从您现有的 pages/ 目录中增量采用。

### 布局[](#layouts)

`app/` 目录使得构建跨导航保持状态、避免昂贵重新渲染并启用高级路由模式的复杂界面变得容易。此外，您可以嵌套布局，并**将应用代码就近放置**到您的路由中，如组件、测试和样式。

![`app/` 目录可以从您现有的 pages/ 目录中增量采用。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-13/colocating-assets-in-the-app-directory.png)

`app/` 目录可以从您现有的 pages/ 目录中增量采用。

在 `app/` 内创建路由需要一个单一的文件 `page.js`：

`getInitialProps` 在服务端*和*客户端运行。此 API 扩展了 React 组件，允许您创建一个 `Promise` 并将结果转发给组件的 `props`。

```
// This file maps to the index route (/)
export default function Page() {
  return <h1>Hello, Next.js!</h1>;
}
```

然后您可以通过文件系统**定义布局**。布局在不同页面之间共享 UI。在导航时，布局保持状态，保持交互性，并且不会重新渲染。

app/blog/layout.js

```
export default function BlogLayout({ children }) {
  return <section>{children}</section>;
}
```

[了解更多关于布局和页面的信息](/docs/app/building-your-application/routing) 或 [部署示例以尝试](https://vercel.com/templates/next.js/app-directory)。

### 服务端组件[](#server-components)

`app/` 目录引入对 React 新 [服务端组件](/docs/app/building-your-application/rendering/server-components) 架构的支持。服务端和 [客户端组件](/docs/app/building-your-application/rendering/client-components) 分别利用服务器和客户端各自擅长的领域——允许您使用统一的编程模型构建快速、高度互动的应用程序，提供出色的开发体验。

通过服务端组件，我们正在为构建复杂界面奠定基础，同时**减少发送给客户端的 JavaScript 量**，实现更快的初始页面加载。

当加载路由时，Next.js 和 React 运行时将被加载，它是**可缓存**的并且在大小上是**可预测**。此运行时的规模不会随应用程序的增长而增加。此外，运行时是异步加载的，使得服务器生成的 HTML 可以在客户端渐进式增强。

[了解更多关于服务端组件的信息](/docs/app/building-your-application/rendering/server-components) 或 [部署示例以尝试](https://vercel.com/templates/next.js/app-directory)。

### 流式传输[](#streaming)

`app/` 目录引入了渐进式渲染并将已渲染的 UI 单元增量流式传输到客户端的功能。

通过 Next.js 中的服务端组件和 [嵌套布局](/docs/app/building-your-application/routing/pages-and-layouts)，您可以立即渲染不需要特定数据的页面部分，并对获取数据的部分显示 [加载状态](/docs/app/building-your-application/routing/loading-ui-and-streaming)。通过这种方法，用户不必等待整个页面加载完毕即可开始与之交互。

![您可以将应用代码（如组件、测试和样式）与您的路由就近放置。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-13/streaming.png)

您可以将应用代码（如组件、测试和样式）与您的路由就近放置。

部署到 Vercel 后，使用 `app/` 目录的 Next.js 13 应用将在 Node.js 和 Edge 运行时中默认流式传输响应，以改善性能。

[了解更多关于流式传输的信息](/docs/app/building-your-application/routing/loading-ui-and-streaming#what-is-streaming) 或 [部署示例以尝试](https://vercel.com/templates/next.js/app-directory)。

### 数据获取[](#data-fetching)

React 最近的 [Promise 支持 RFC](https://github.com/acdlite/rfcs/blob/first-class-promises/text/0000-first-class-support-for-promises.md) 引入了在组件中**获取数据和处理 Promise**的强大新方法：

`getInitialProps` 在服务端*和*客户端运行。此 API 扩展了 React 组件，允许您创建一个 `Promise` 并将结果转发给组件的 `props`。

```
async function getData() {
  const res = await fetch('https://api.example.com/...');
  // The return value is *not* serialized
  // You can return Date, Map, Set, etc.
  return res.json();
}
 
// This is an async Server Component
export default async function Page() {
  const data = await getData();
 
  return <main>{/* ... */}</main>;
}
```

原生的 `fetch` Web API 也在 React 和 Next.js 中进行了扩展。它 [自动去重获取请求](/docs/app/api-reference/functions/fetch) 并提供**一种**灵活的方式在组件级获取、缓存和重新验证数据。这意味着静态站点生成 (SSG)、服务端渲染 (SSR) 和增量静态再生 (ISR) 的所有优点现在都可以通过一种 API 获得：

```
// This request should be cached until manually invalidated.
// Similar to `getStaticProps`.
// `force-cache` is the default and can be omitted.
fetch(URL, { cache: 'force-cache' });
 
// This request should be refetched on every request.
// Similar to `getServerSideProps`.
fetch(URL, { cache: 'no-store' });
 
// This request should be cached with a lifetime of 10 seconds.
// Similar to `getStaticProps` with the `revalidate` option.
fetch(URL, { next: { revalidate: 10 } });
```

在 `app` 目录中，您可以在 [布局、页面](/docs/app/building-your-application/routing/pages-and-layouts) 和组件中获取数据——包括支持来自服务器的 [流式传输响应](#streaming)。

我们正在启用处理加载和错误状态的便捷方式，并在渲染时流式传输 UI。在将来的版本中，我们也将改进和简化数据变更。

![通过 app/ 目录，您可以使用新的特殊文件 loading.js 自动创建带有 Suspense 边界的即时加载 UI。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-13/streaming.png)

通过 app/ 目录，您可以使用新的特殊文件 `loading.js` 自动创建带有 Suspense 边界的即时加载 UI。

我们很期待与开源社区、包维护者以及为 React 生态系统做出贡献的其他公司合作，为 React 和 Next.js 的这个新时代而构建。将数据获取就近放置在组件内部并向客户端发送更少的 JavaScript 是我们在 `app/` 目录中很兴奋地纳入的两个重要的社区反馈。

[了解更多关于数据获取的信息](/docs/app/building-your-application/data-fetching/fetching-caching-and-revalidating) 或 [部署示例以尝试](https://vercel.com/templates/next.js/app-directory)。

Turbopack (Alpha) 介绍[](#introducing-turbopack-alpha)
-------------------------------------------------------------

Next.js 13 包括基于 [Turbopack](https://vercel.com/blog/turbopack)，这是基于 Rust 的 Webpack 继任者。

Webpack 已被下载超过 **30 亿次**。虽然它一直是构建 Web 的重要组成部分，但我们已达到了基于 JavaScript 工具链可能实现的最大性能限制。

在 Next.js 12 中，我们开始向原生 Rust 驱动的 tooling 过渡。我们首先从迁移 Babel 开始，结果转换速度快了 17 倍。然后，我们替换了 Terser，结果压缩速度快了 6 倍。是时候全面进军原生了。

使用 Next.js 13 中的 Turbopack 预览版将带来：

*   比 Webpack 更新快 **700 倍**
*   比 Vite 更新快 **10 倍**
*   比 Webpack 冷启动快 **4 倍**

![Turbopack 是我们基于 Rust 的 Webpack 继任者，在大型应用中热更新快 700 倍。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-13/turbopack.png)

Turbopack 是我们基于 Rust 的 Webpack 继任者，在大型应用中热更新快 700 倍。

[Turbopack](https://nextjs.org/docs/app/api-reference/turbopack) 仅在开发环境打包所需的最少资源，因此启动时间极快。在拥有 3,000 个模块的应用中，**Turbopack 启动只需 1.8 秒**。**Vite 需要 11.4 秒**，**Webpack 需要 16.5 秒**。

Turbopack 开箱即用地支持服务端组件、TypeScript、JSX、CSS 等。在预览版期间，[许多功能](https://nextjs.org/docs/app/api-reference/turbopack#supported-features)尚未得到支持。我们很想听到您使用 Turbopack 加速本地迭代的反馈。

> **注意：** Next.js 中的 Turbopack 目前仅支持 `next dev`。查看 [支持的功能](https://nextjs.org/docs/app/api-reference/turbopack#supported-features)。我们也在努力通过 Turbopack 添加对 `next build` 的支持。

在 Next.js 13 中通过使用 `next dev --turbo` 来尝试 Turbopack 预览版。

`next/image`[](#nextimage)
--------------------------

Next.js 13 引入了**强大的新 Image 组件**，让您无需布局位移即可轻松显示图像，并在需要时按需优化文件以提高性能。

在 Next.js 社区调查中，**70% 的受访者**告诉我们他们在生产环境中使用了 Next.js Image 组件，进而看到了**核心网页指标 (Core Web Vitals) 的改进**。通过 Next.js 13，我们进一步优化了 `next/image`。

新的 Image 组件：

*   发送更少的客户端 JavaScript
*   样式和配置更容易
*   更具可访问性，默认需要 `alt` 标签
*   与 Web 平台对齐
*   更快，因为原生延迟加载不需要水合

`getInitialProps` 在服务端*和*客户端运行。此 API 扩展了 React 组件，允许您创建一个 `Promise` 并将结果转发给组件的 `props`。

```
import Image from 'next/image';
import avatar from './lee.png';
 
export default function Home() {
  // "alt" is now required for improved accessibility
  // optional: image files can be colocated inside the app/ directory
  return <Image alt="leeerob" src={avatar} placeholder="blur" />;
}
```

[了解更多关于 Image 组件的信息](/docs/app/building-your-application/optimizing/images) 或 [部署示例以尝试](https://vercel.com/templates/next.js/nextjs-boilerplate)。

`@next/font`[](#nextfont)
-------------------------

Next.js 13 推出了**全新的字体系统**，可以：

*   自动优化您的字体，包括自定义字体
*   移除外部网络请求，以改善隐私和性能
*   任何字体文件的内置自动自托管
*   使用 CSS [`size-adjust`](https://web.dev/css-size-adjust/) 属性自动防止零布局位移

此新字体系统允许您在考虑性能和隐私的情况下方便地使用所有 Google 字体。CSS 和字体文件在构建时下载并与其他静态资源一起自托管。**浏览器不会向 Google 发送请求。**

app/layout.js / pages/\_app.js

```
import { Inter } from '@next/font/google';
 
const inter = Inter();
 
<html className={inter.className}></html>;
```

自定义字体也得到支持，包括对字体文件的自动自托管、缓存和预加载的支持。

app/layout.js / pages/\_app.js

```
import localFont from '@next/font/local';
 
const myFont = localFont({ src: './my-font.woff2' });
 
<html className={myFont.className}></html>;
```

您可以自定义字体加载体验的每个部分，同时确保优秀的性能和零布局位移，包括 `font-display`、预加载、回退等。

[了解更多关于新的 Font 组件的信息](/docs/app/building-your-application/optimizing/fonts) 或 [部署示例以尝试](https://vercel.com/templates/next.js/loading-web-fonts)。

`next/link`[](#nextlink)
------------------------

`next/link` 不再需要手动添加 `<a>` 作为子元素。

这最初在 [12.2](https://nextjs.org/blog/next-12-2) 中作为实验性功能添加，现在默认为默认。在 Next.js 13 中，`<Link>` 始终渲染 `<a>`，并允许您将 props 转发到底层标签。例如：

```
import Link from 'next/link'
 
// Next.js 12: `<a>` has to be nested otherwise it's excluded
<Link href="/about">
  <a>About</a>
</Link>
 
// Next.js 13: `<Link>` always renders `<a>`
<Link href="/about">
  About
</Link>
```

[了解更多关于改进的 Link 组件的信息](/docs/app/api-reference/components/link) 或 [部署示例以尝试](https://vercel.com/templates/next.js/nextjs-boilerplate)。

OG 图像生成[](#og-image-generation)
-------------------------------------------

社交卡片，也称为 Open Graph 图像，可以大幅提高内容点击的参与度，一些实验显示转化率提高了高达 40%。

静态社交卡片耗时、易错且难以维护。因此，社交卡片经常缺乏甚至被忽略。直到今天，需要个性化且实时计算的动态社交卡片仍然困难且昂贵。

我们创建了一个新的库 **`@vercel/og`**，与 Next.js 无缝配合生成动态社交卡片。

pages/api/og.jsx

```
import { ImageResponse } from '@vercel/og';
 
export const config = {
  runtime: 'experimental-edge',
};
 
export default function () {
  return new ImageResponse(
    (
      <div
        style={{
          display: 'flex',
          fontSize: 128,
          background: 'white',
          width: '100%',
          height: '100%',
        }}
      >
        Hello, World!
      </div>
    ),
  );
}
```

通过使用 Vercel Edge 函数、WebAssembly 和用于将 HTML 和 CSS 转换为图像的全新核心库以及利用 React 组件抽象，这种方法比现有解决方案**快 5 倍**。

[了解更多关于 OG 图像生成的信息](https://vercel.com/docs/og-image-generation) 或 [部署示例以尝试](https://vercel.com/new/clone?repository-url=https://github.com/vercel/examples/tree/main/edge-functions/vercel-og-nextjs&project-name=vercel-og-nextjs&repository-name=vercel-og-nextjs)。

Middleware API 更新[](#middleware-api-updates)
-------------------------------------------------

在 Next.js 12 中，我们引入了 Middleware 以实现对 Next.js 路由器的完全灵活性。我们听到了您对初始 API 设计的反馈，并添加了一些改进以提高开发体验并增加强大的新功能。

您现在可以更轻松地设置请求上的头部：

middleware.ts

```
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';
 
export function middleware(request: NextRequest) {
  // Clone the request headers and set a new header `x-version`
  const requestHeaders = new Headers(request.headers);
  requestHeaders.set('x-version', '13');
 
  // You can also set request headers in NextResponse.rewrite
  const response = NextResponse.next({
    request: {
      // New request headers
      headers: requestHeaders,
    },
  });
 
  // Set a new response header `x-version`
  response.headers.set('x-version', '13');
  return response;
}
```

您现在也可以直接从 Middleware 提供响应，而无需 `rewrite` 或 `redirect`。

middleware.ts

```
import { NextRequest, NextResponse } from 'next/server';
import { isAuthenticated } from '@lib/auth';
 
// Limit the middleware to paths starting with `/api/`
export const config = {
  matcher: '/api/:function*',
};
 
export function middleware(request: NextRequest) {
  // Call our authentication function to check the request
  if (!isAuthenticated(request)) {
    // Respond with JSON indicating an error message
    return NextResponse.json(
      {
        success: false,
        message: 'Auth failed',
      },
      {
        status: 401,
      },
    );
  }
}
```

从 Middleware 发送响应目前需要 `next.config.js` 内部的 `experimental.allowMiddlewareResponseBody` 配置选项。

破坏性更改[](#breaking-changes)
-------------------------------------

*   最低 React 版本已从 17.0.2 提升到 18.2.0。
*   最低 Node.js 版本已从 12.22.0 提升到 14.6.0，因为 12.x 已到达生命周期结束（[PR](https://github.com/vercel/next.js/pull/41482)）。
*   `swcMinify` 配置属性已从 `false` 更改为 `true`。有关更多信息，请参阅 [Next.js 编译器](/docs/architecture/nextjs-compiler)。
*   `next/image` 导入已重命名为 `next/legacy/image`。`next/future/image` 导入已重命名为 `next/image`。[代码转换工具可用](/docs/app/building-your-application/upgrading/codemods#next-image-to-legacy-image) 可以安全自动地重命名您的导入。
*   `next/link` 子元素不再可以是 `<a>`。添加 `legacyBehavior` prop 以使用旧版行为或移除 `<a>` 以进行升级。[代码转换工具可用](/docs/app/building-your-application/upgrading/codemods#new-link) 可以自动升级代码。
*   当 `User-Agent` 是机器人时，路由不再预取。
*   已移除 `next.config.js` 中已弃用的 `target` 选项。
*   支持的浏览器已更改，不再支持 Internet Explorer 并针对现代浏览器。您仍然可以使用 Browserslist 更改目标浏览器。
    *   Chrome 64+
    *   Edge 79+
    *   Firefox 67+
    *   Opera 51+
    *   Safari 12+

要了解更多，请查看 [升级指南](/docs/app/building-your-application/upgrading)。
