我们很高兴地推出 Next.js 10.2，其包含：

* **[更快的构建](#webpack-5)**：缓存后的后续构建快约 60%。
* **[更快的刷新](#webpack-5)**：刷新快 100ms 到 200ms。
* **[更快的启动](#improved-startup-performance)**：`next dev` 快约 24%。
* **[可访问性改进](#accessibility-improvements)**：屏幕阅读器现在会播报路由更改。
* **[更灵活的重定向和重写](#routing-based-on-headers-and-query-string-parameters)**：匹配任何请求头、Cookie 或查询字符串。
* **[自动 Web 字体优化](#automatic-webfont-optimization)**：通过内联字体 CSS 提升性能。

今天通过运行 `npm i next@latest` 进行更新。

Webpack 5 [](#webpack-5)
-----------------------

在 Next.js 10.1 中，我们改进了 [快速刷新 (Fast Refresh)](/blog/next-10-1#3x-faster-refresh) 并[缩短了安装时间](/blog/next-10-1#improved-installation-time)。我们很高兴地分享通过 webpack 5 实现的额外性能改进。

今天，我们正在将 webpack 5 的推广扩展到 Next.js 应用。从 Next.js 10.2 开始，所有在 `next.config.js` 中未使用自定义 webpack 配置的应用将自动使用 webpack 5。

启用 webpack 5 时，您将自动获得新功能和改进，例如：

* **改进的磁盘缓存**：磁盘缓存允许编译器在 `next build` 运行之间持久化工作。只有更改的文件会被重新编译，从而提高性能。Vercel 客户 [Scale](http://scale.com/) 观察到后续构建的**速度提升了 63%**。
* **改进的快速刷新 (Fast Refresh)**：Next.js 现在优先处理与快速刷新 (Fast Refresh) 相关的编译，每次刷新快 100ms 到 200ms。
* **改进的资产长期缓存**：多次 `next build` 运行之间的输出现在是确定性的，改善了生产环境中 JavaScript 资产的浏览器缓存。当页面内容不改变时，哈希值保持不变。
* **改进的摇树优化 (Tree Shaking)**：[CommonJS](https://en.wikipedia.org/wiki/CommonJS) 模块现在可以进行 [摇树优化 (Tree Shaking)](https://webpack.js.org/guides/tree-shaking/) 以自动删除未使用的代码。使用静态分析来确定无副作用的模块。`export *` 跟踪更多信息，不再标记默认导出为已使用。还启用了模块内部的摇树优化，允许仅用于未使用导出中的导入进行摇树优化。

我们努力确保从 webpack 4 到 5 的平滑过渡。Next.js 核心测试套件包含超过 **3,400 个集成测试**，在启用了 webpack 5 支持的每次拉取请求上运行。

如果您的应用程序有自定义 webpack 配置，我们建议遵循 [启用 webpack 5 的文档](/docs/messages/webpack5)。在 Next.js 中升级到 webpack 5 后，请 [与我们分享任何反馈](https://github.com/vercel/next.js/discussions/23498)。

改进的启动性能 [](#improved-startup-performance)
-------------------------------------------------------------

我们改进了 Next.js CLI 初始化，使得首次运行后 `next dev` 的引导时间**快达 24%**。例如，[vercel.com](https://vercel.com) 的 `next dev` 从 3.3 秒减少到 2.5 秒。

我们的使命是将您的本地开发体验变为实时（[快 20 倍](https://twitter.com/shuding_/status/1378086219708473344)）。请关注后续版本中的更多启动性能改进。

可访问性改进 [](#accessibility-improvements)
---------------------------------------------------------

路由更改现在默认向屏幕阅读器和辅助技术进行播报。

下面的示例显示了在客户端导航中，[macOS VoiceOver](https://www.apple.com/voiceover/info/guide/_1121.html) 播报标题“Real Data. Real Performance”。页面名称首先查找 `<h1>` 元素，若未找到则回退到 `document.title`，最后回退到 pathname。

基于请求头和查询字符串参数的路由 [](#routing-based-on-headers-and-query-string-parameters)
-------------------------------------------------------------------------------------------------------------

Next.js [重写](/docs/pages/api-reference/next-config-js/rewrites)、[重定向](/docs/pages/api-reference/next-config-js/redirects) 和 [请求头](/docs/pages/api-reference/next-config-js/headers) 现在支持新的 `has` 属性，允许您匹配传入的请求头、Cookie 和查询字符串。

Vercel 客户 [Joyn](https://www.joyn.de/) 使用 `has` 来优化内容的可发现性和性能。例如，您可以根据 [User-Agent](https://developer.mozilla.org/docs/Web/HTTP/Headers/User-Agent) 重定向旧版浏览器：

next.config.js

```
module.exports = {
  async redirects() {
    return [
      {
        source: '/:path((?!old-browser$).*)',
        has: [
          {
            type: 'header',
            key: 'User-Agent',
            value:
              'Mozilla/5.0 (compatible; MSIE 10.0; Windows Phone 8.0; Trident/6.0; IEMobile/10.0; ARM; Touch; Microsoft; Lumia 950)',
          },
        ],
        destination: '/old-browser',
        permanent: false,
      },
    ];
  },
};
```

另一个示例是根据用户位置进行重定向：

next.config.js

```
module.exports = {
  async redirects() {
    return [
      {
        source: '/:path((?!uk/).*)',
        has: [
          {
            type: 'header',
            key: 'x-vercel-ip-country',
            value: 'GB',
          },
        ],
        destination: '/uk/:path*',
        permanent: true,
      },
    ];
  },
};
```

最后，如果用户已登录，您可以进行重定向：

next.config.js

```
module.exports = {
  async redirects() {
    return [
      {
        source: '/',
        has: [
          {
            type: 'header',
            key: 'x-authorized',
            value: '(?<authorized>yes|true)',
          },
        ],
        destination: '/dashboard?authorized=:authorized',
        permanent: false,
      },
    ];
  },
};
```

想了解更多并探索更多示例，请查看 [重定向文档](/docs/pages/api-reference/next-config-js/redirects#header-cookie-and-query-matching)。

自动 Web 字体优化 [](#automatic-webfont-optimization)
-----------------------------------------------------------------

[82% 的桌面端网页](https://almanac.httparchive.org/en/2020/fonts) 使用网络字体。自定义字体对于品牌形象、设计以及跨浏览器/设备的显示一致性至关重要。然而，使用网络字体不应以牺牲性能为代价。

Next.js 现在在使用 Web 字体时支持自动 Web 字体优化。默认情况下，Next.js 将在构建时自动内联字体 CSS，消除获取字体声明的额外往返。这有助于提升首次内容绘制 (FCP) 和最大内容绘制 (LCP)。例如：

```
// Before
<link href="https://fonts.googleapis.com/css2?family=Inter" rel="stylesheet" />
 
// After
<style data-href="https://fonts.googleapis.com/css2?family=Inter">
  @font-face {
    font-family: 'Inter';
    font-style: normal;
    <!-- ... -->
  }
</style>
```

自动 Web 字体优化目前支持 Google Fonts，我们正在努力进一步支持其他字体提供商。我们还计划添加对 [加载策略](https://github.com/vercel/next.js/issues/21555) 和 `font-display` 值的控制。通过默认优化字体，我们能够以最小的额外努力帮助开发者交付更快的网站并改善其 [核心 Web 指标](https://vercel.com/blog/core-web-vitals)。

感谢我们的合作伙伴 Google 和 [Janicklas Ralph](https://github.com/janicklas-ralph) 帮助实现此功能。
