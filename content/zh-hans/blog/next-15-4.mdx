Next.js 15.4 在性能、稳定性和 Turbopack 兼容性方面进行了更新。本次发布的亮点包括：

*   [**Turbopack 构建**](/blog/next-15-4#turbopack-builds): `next build --turbopack` 实现了 100% 集成测试兼容性
*   [**稳定性改进**](/blog/next-15-4#notable-changes): 针对 Next.js 和 Turbopack 进行了大量的稳定性和性能改进

本文还包含即将在 Next.js 16 中推出的内容预览，这是我们下一个主要版本。

立即升级，或使用以下内容开始：

终端

```
# Use the automated upgrade CLI
npx @next/codemod@canary upgrade latest
 
# ...or upgrade manually
npm install next@latest react@latest react-dom@latest
 
# ...or start a new project
npx create-next-app@latest
```

Turbopack 构建 [](#turbopack-builds)
-------------------------------------

`next build --turbopack` 现在通过了 [**所有 8298 个集成测试**](https://areweturboyet.com/) 用于生产构建，并现用于 `vercel.com`。这是将 Turbopack 在 Next.js 中标记为稳定的关键里程碑。

团队当前的优先级是通过生产分块完成捆绑优化，并修复 Alpha 版本早期采用者报告的错误。我们还对 Turbopack 底层进行了大量的稳定性和性能改进。

我们有信心 Turbopack 很快就能用于生产环境，并致力于在 Next.js 16 中推出 Beta 版。如果您之前尝试过 Turbopack，现在是个不错的重试时机。如果遇到任何问题，请在 [GitHub Issues](https://github.com/vercel/next.js/issues) 上告诉我们。

展望未来：Next.js 16[](#looking-ahead-nextjs-16)
-----------------------------------------------------

在过去的几个月里，我们正在迭代 Next.js 的重大改进，重点关注 Turbopack 的开发者体验以及 App Router 的核心能力，包括 `PPR` 和 `use cache`。虽然我们尚未准备好默认启用这些功能，但我们打算为今年夏天的用户试用 Next.js 16 奠定基础。

以下是 Next.js 16 即将推出的内容预览：

*   **缓存组件 (beta)**：将实验性缓存功能（Dynamic IO、`use cache` 和部分预渲染）整合到一个统一的 `cacheComponents` 标志中，简化了性能优化策略。Next.js 16 的博文将详细介绍我们的愿景。
*   **Turbopack 构建 (beta)**：引入 `next build --turbopack`，完全通过集成测试，并在 vercel.com 等高流量站点的内部验证。准备好在 Next.js 16 中进行扩展的公共 Beta 测试。
*   **优化的客户端路由**：增强 App Router 导航，具有更智能的预取、更完善的缓存失效策略，减少了带宽使用，从而实现更快、响应更灵敏的导航。
*   **开发工具与调试**：添加 Route Info 以检查应用结构，并切换 `loading.tsx` 等组件。为终端添加实验性浏览器日志转发，支持 AI 驱动的调试工作流。
*   **Node.js 中间件 (稳定版)**：将之前实验性的 Node.js 运行时支持（最初在 Next.js 15.2 中引入）提升为 Middleware 的稳定状态。
*   **部署适配器 (Alpha)**：使开发人员能够为更精细地控制构建和部署目标创建自定义部署适配器。
*   **次要弃用与变更**：弃用 Node.js 18 支持、AMP，并更改部分 `next/image` API，同时提供适当的迁移指导。

我们的目标是让这次更新尽可能顺畅简单。我们谨慎地管理这些变更，以最大程度减少干扰，并将提供清晰的指导和工具以支持您过渡到 Next.js 16。

预览即将推出的功能 [](#preview-upcoming-features)
-------------------------------------------------------

您可以从今天开始使用 `canary` 频道并在 `next.config.js` 中启用以下标志来尝试许多这些功能：

next.config.ts

```
import type { NextConfig } from 'next';
 
const nextConfig: NextConfig = {
  experimental: {
    // Forward browser logs to the terminal for easier debugging
    browserDebugInfoInTerminal: true,
 
    // Enable new caching and pre-rendering behavior
    dynamicIO: true, // will be renamed to cacheComponents in Next.js 16
 
    // Activate new client-side router improvements
    clientSegmentCache: true,
 
    // Explore route composition and segment overrides via DevTools
    devtoolSegmentExplorer: true,
 
    // Enable support for `global-not-found`, which allows you to more easily define a global 404 page.
    globalNotFound: true,
 
    // Enable persistent caching for the turbopack dev server and build.
    turbopackPersistentCaching: true,
  },
};
 
export default nextConfig;
```
