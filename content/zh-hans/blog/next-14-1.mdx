改进自托管[](##improved-self-hosting)
-----------------------------------------------

我们听到了您的反馈，需要更清晰地了解如何使用 Node.js 服务器、Docker 容器或静态导出来自行部署 Next.js。我们全面改写了以下自部署文档：

*   [运行时环境变量](/docs/app/building-your-application/deploying#environment-variables)
*   [ISR 的自定义缓存配置](/docs/app/building-your-application/deploying#caching-and-isr)
*   [自定义图像优化](/docs/app/building-your-application/deploying#image-optimization)
*   [中间件](/docs/app/building-your-application/deploying#middleware)

随着 Next.js 14.1，我们还稳定了为增量静态再生 (Incremental Static Regeneration) 和 App Router 的更细粒度数据缓存提供自定义缓存处理程序：

next.config.js

```
module.exports = {
  cacheHandler: require.resolve('./cache-handler.js'),
  cacheMaxMemorySize: 0, // disable default in-memory caching
};
```

在使用容器编排平台（如 Kubernetes）时，使用此配置进行自部署很重要，因为每个 Pod 都会有一份缓存副本。使用自定义缓存处理程序可以确保托管 Next.js 应用的所有 Pod 之间的一致性。

例如，您可以将缓存值保存到任何地方，如 Redis 或 Memcached。我们要感谢 [`@neshca`](https://github.com/caching-tools/next-shared-cache) 提供的 [Redis 缓存处理程序适配器](https://github.com/vercel/next.js/tree/canary/examples/cache-handler-redis) 和示例。

Turbopack Improvements[](#turbopack-improvements)
-------------------------------------------------

我们继续致力于提高本地 Next.js 开发的可靠性和性能：

*   **可靠性：** Turbopack 已通过整个 Next.js 开发测试套件，并实际应用了 Vercel 的应用程序
*   **性能：** 改进了 Turbopack 的初始编译时间和快速刷新 (Fast Refresh) 时间
*   **内存使用：** 改进了 Turbopack 的内存使用

我们计划在下一个版本中稳定 `next dev --turbo`，目前仍将作为可选功能。

### 可靠性[](##reliability)

Next.js 配合 Turbopack 现已通过 **5,600 项开发测试 (94%)**，自上次更新以来增加了 600 项。您可以在 [areweturboyet.com](https://areweturboyet.com/) 上跟踪进度。

我们继续在 Vercel 的所有 Next.js 应用程序中实际应用 `next dev --turbo`，包括 [vercel.com](https://vercel.com) 和 [v0.app](https://v0.app)。所有开发这些应用程序的工程师每天都在使用 Turbopack。

我们发现并修复了许多使用 Turbopack 的非常庞大的 Next.js 应用程序的问题。针对这些修复，我们在 Next.js 现有的开发测试套件中添加了新的测试。

### 性能[](##performance)

对于大型 Next.js 应用程序（如 vercel.com），我们观察到：

*   本地服务器启动速度提升 **76.7%**
*   使用快速刷新 (Fast Refresh) 的代码更新速度提升 **96.3%**
*   无缓存的首次路由编译速度提升 **45.8%**（Turbopack 目前尚未提供磁盘缓存）

在 [v0.app](https://v0.app) 中，我们识别到可以优化 Turbopack 中 React 客户端组件的发现和打包方式——使首次编译速度提升 **61.5%**。这种性能提升也在 [vercel.com](https://vercel.com) 中观察到。

### 未来改进[](##future-improvements)

Turbopack 目前支持内存缓存，可以加快快速刷新 (Fast Refresh) 的增量编译时间。

然而，当前重启 Next.js 开发服务器时不会保留缓存。Turbopack 性能提升的下一步是 **磁盘缓存**，这将允许在重启开发服务器时保留缓存。

### 改进的错误信息和快速刷新[](##improved-error-messages-and-fast-refresh)

我们深知清晰的错误信息对本地开发体验的重要性。我们进行了一系列修复，以提高运行 `next dev` 时您看到的堆栈跟踪和错误信息的准确性。

*   之前显示类似 `webpack-internal` 的打包器错误的地方，现在会正确显示错误的源代码和受影响的文件。
*   在客户端组件中看到错误后，在编辑器中修复错误，快速刷新 (Fast Refresh) 不会清除错误屏幕，需要硬刷新。我们修复了此类问题的多个实例。例如，尝试从客户端组件导出 `metadata`。

### `window.history.pushState` 和 `window.history.replaceState`[](##windowhistorypushstate-and-windowhistoryreplacestate)

App Router 现在允许使用原生 [`pushState`](https://developer.mozilla.org/en-US/docs/Web/API/History/pushState) 和 [`replaceState`](https://developer.mozilla.org/en-US/docs/Web/API/History/replaceState) 方法更新浏览器的历史栈，而无需重新加载页面。

`pushState` 和 `replaceState` 调用会集成到 Next.js App Router，允许您与 [`usePathname`](/docs/app/api-reference/functions/use-pathname) 和 [`useSearchParams`](/docs/app/api-reference/functions/use-search-params) 同步。

这在需要立即更新 URL 时很有帮助，例如保存筛选器、排序顺序或其他希望在重新加载时保留的信息。

```
'use client';
 
import { useSearchParams } from 'next/navigation';
 
export default function SortProducts() {
  const searchParams = useSearchParams();
 
  function updateSorting(sortOrder: string) {
    const params = new URLSearchParams(searchParams.toString());
    params.set('sort', sortOrder);
    window.history.pushState(null, '', `?${params.toString()}`);
  }
 
  return (
    <>
      <button onClick={() => updateSorting('asc')}>Sort Ascending</button>
      <button onClick={() => updateSorting('desc')}>Sort Descending</button>
    </>
  );
}
```

了解更多关于在 Next.js 中使用 [原生 History API](/docs/app/building-your-application/routing/linking-and-navigating#using-the-native-history-api) 的内容。

`next/image` 支持 `<picture>` 和艺术导向[](##nextimage-support-for-picture-and-art-direction)
----------------------------------------------------------------------------------------------------------

Next.js Image 组件现在通过 `getImageProps()`（稳定版）支持更高级的用法，无需直接使用 `<Image>`。包括：

*   配合 [`background-image`](https://developer.mozilla.org/docs/Web/CSS/background-image) 或 [`image-set`](https://developer.mozilla.org/docs/Web/CSS/image/image-set) 使用
*   配合 canvas 的 [`context.drawImage()`](https://developer.mozilla.org/docs/Web/API/Canvas_API/Tutorial/Using_images) 或 `new Image()` 使用
*   配合 [`<picture>`](https://developer.mozilla.org/docs/Web/HTML/Element/picture) 媒体查询实现 [艺术指导](https://developer.mozilla.org/docs/Learn/HTML/Multimedia_and_embedding/Responsive_images#art_direction) 或 亮/暗模式图片切换

```
import { getImageProps } from 'next/image';
 
export default function Page() {
  const common = { alt: 'Hero', width: 800, height: 400 };
  const {
    props: { srcSet: dark },
  } = getImageProps({ ...common, src: '/dark.png' });
  const {
    props: { srcSet: light, ...rest },
  } = getImageProps({ ...common, src: '/light.png' });
 
  return (
    <picture>
      <source media="(prefers-color-scheme: dark)" srcSet={dark} />
      <source media="(prefers-color-scheme: light)" srcSet={light} />
      <img {...rest} />
    </picture>
  );
}
```

了解更多关于 [`getImageProps()`](/docs/app/api-reference/components/image#getimageprops) 的内容。

并行和拦截路由[](##parallel--intercepted-routes)
--------------------------------------------------------------

在 Next.js 14.1 中，我们对并行和拦截路由进行了 **20 项改进**。

在过去的两个版本中，我们专注于提高 Next.js 的性能和可靠性。现在，基于您的反馈，我们已经能够对 [并行路由](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes) 和 [拦截路由](https://nextjs.org/docs/app/building-your-application/routing/intercepting-routes) 进行诸多改进。特别值得一提的是，我们添加了对通配符路由和服务端操作的支持。

*   **并行路由** 允许在同一布局中同时或条件渲染一个或多个页面。对于应用中的高度动态部分（如社交网站的仪表板和动态信息流），可以使用并行路由实现复杂的路由模式。
*   **拦截路由** 允许在当前布局中加载应用程序其他部分的路由。例如，点击动态信息流中的照片时，可以在覆盖层中以模态框形式显示照片。在此情况下，Next.js 会拦截 `/photo/123` 路由，遮蔽 URL 并将其覆盖在 `/feed` 上。

了解更多关于 [并行](https://nextjs.org/docs/app/building-your-application/routing/parallel-routes) 和 [拦截路由](https://nextjs.org/docs/app/building-your-application/routing/intercepting-routes) 的内容，或 [查看示例](https://github.com/vercel/nextgram)。
