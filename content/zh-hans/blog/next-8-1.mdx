AMP 是什么？[](#what-is-amp)
----------------------------
AMP 是构建高性能网站的标准，它最大限度地减少了渲染开销，并且可以在知名的搜索引擎 (Search engines) 中进行索引，并提供增强功能。例如，AMP 页面 (AMP pages) 是直接加载到 Google 的移动搜索结果中的，并带有闪电符号标记。

![Next.js 谷歌搜索结果](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-8-1/next-amp-result.png)
Next.js 谷歌搜索结果

AMP HTML 是 HTML 的更严格版本，施加了一些限制，以实现更可靠的性能和更好的可扩展性。一些 HTML 标签被替换为 AMP 特定的 HTML 标签 (amp-specific HTML tags)，以提供比其对应的 HTML 标签更好的体验。例如，[`amp-img`](https://amp.dev/documentation/components/amp-img) 标签即使在尚未支持它的浏览器中也提供完整的 `srcset` 支持。

AMP 页面会被支持此标准的搜索引擎 (Search engines) 自动发现。这些搜索引擎通常实现了 [AMP 缓存 (AMP Cache)](https://amp.dev/documentation/guides-and-tutorials/learn/amp-caches-and-cors/how_amp_pages_are_cached)（例如 [Google](https://developers.google.com/amp/cache/) 和 [Bing](https://www.bing.com/webmaster/help/bing-amp-cache-bc1c884c)）。AMP 缓存有助于页面从其搜索结果中更快地渲染。

AMP 在 Next.js 中[](#amp-in-nextjs)
--------------------------------
今天，我们很高兴地宣布我们已将 Next.js 体验扩展至编写 AMP 页面。这意味着您现在可以利用 React 组件 (React components) 的功能来创建 AMP 页面。AMP 支持是在每个页面的基础上定义的，允许逐步采用 AMP。在 Next.js 中有两种启用 AMP 的方法：混合 AMP 页面或 AMP 优先页面。

混合 AMP 页面[](#hybrid-amp-pages)
-------------------------------------
混合 AMP 页面允许您拥有一个传统页面的 AMP 版本，这样搜索引擎可以在移动搜索结果中显示该页面的 AMP 版本，同时保留现有页面版本。这使得您可以在应用程序中使用 AMP，同时仍然利用诸如客户端路由 (client-side routing) 之类的 Next.js 功能作为主要的用户体验。

取而代之的是，Next.js 会自动注入 CSS 文件。在生产环境中，Next.js 还自动在 CSS URL 中添加内容哈希，因此如果有更改，可确保您的最终用户永远不会获得文件的过时版本，并且具有引入不可变永久缓存的能力。
```
function HomePage() {
  return <p>Welcome to AMP + Next.js.</p>;
}
 
export default withAmp(HomePage, { hybrid: true });
```

混合 AMP 模式在生产环境中的一个示例是 Reddit。每个线程都存储在 [Google 的 AMP 缓存 (Google's AMP Cache)](https://developers.google.com/amp/cache/) 中以在移动 Web 上提供快速的用户体验，同时仍然为桌面和后续导航提供完整版本的 Reddit。

### AMP 优先页面[](#amp-first-pages)
AMP 优先页面提供给网站的主要流量以及搜索引擎流量。通过使用 AMP 优先页面，我们将 AMP 的快速体验带给主网站，包括桌面端。

要实现 AMP 优先页面，请使用 `withAmp` 函数包装页面：

取而代之的是，Next.js 会自动注入 CSS 文件。在生产环境中，Next.js 还自动在 CSS URL 中添加内容哈希，因此如果有更改，可确保您的最终用户永远不会获得文件的过时版本，并且具有引入不可变永久缓存的能力。
```
import { withAmp } from 'next/amp';
function HomePage() {
  return <p>Welcome to AMP + Next.js.</p>;
}
 
export default withAmp(HomePage);
```

如果您不需要 Next.js 运行时，使用 AMP 优先页面可能有助于加快开发速度。AMP 优先要求您仅使用 AMP 兼容组件 (AMP compatible components) 来构建您的页面。AMP 优先页面目前已在 [nextjs.org/docs](https://nextjs.org/docs) 和 [nextjs.org/blog](https://nextjs.org/blog) 上使用。

### 在组件中区分 AMP 渲染[](#distinguishing-amp-rendering-in-components)
您的项目中的任何 React 组件都可以利用 `useAmp` 来区分 AMP 和非 AMP 渲染模式。这使得您可以实现一个 `<Image>` 组件，该组件在 `<img>` 和 `<amp-img>` 之间共享逻辑：

components/image.js
```
import { useAmp } from 'next/amp';
 
export function Image({ src, width, height }) {
  const isAmp = useAmp();
  return isAmp ? (
    <amp-img src={src} width={width} height={height} />
  ) : (
    <img src={src} width={width} height={height} />
  );
}
```

取而代之的是，Next.js 会自动注入 CSS 文件。在生产环境中，Next.js 还自动在 CSS URL 中添加内容哈希，因此如果有更改，可确保您的最终用户永远不会获得文件的过时版本，并且具有引入不可变永久缓存的能力。
```
import { withAmp } from 'next/amp';
import { Image } from '../components/image';
 
function HomePage() {
  return (
    <>
      <p>Hello! Welcome to AMP + Next.js.</p>
      <Image src="https://placehold.it/120" width="120" height="120" />
    </>
  );
}
 
export default withAmp(HomePage, { hybrid: true });
```

AMP 开发的自动重新加载[](#automatic-reloading-for-amp-development)
-----------------------------------------------------------------------------------
在开发期间，我们不使用热模块替换 (hot-module-replacement)，而是跟踪您当前页面的任何更改，仅在该页面发生更改时才会重新加载。我们使用全页面重新加载而不是热模块替换的原因是为了确保您始终查看 AMP 页面的最新服务端渲染 (Server-side render)。

AMP 验证[](#amp-validation)
---------------------------------
为了确保只生成有效的 AMP 页面，我们在开发过程中会自动使用 [`amphtml-validator`](https://www.npmjs.com/package/amphtml-validator) 验证它们。错误和警告将显示在您启动 Next.js 的终端中。

页面在 `next export` 期间也会进行验证，任何问题都将打印到终端。任何 AMP 错误都会导致 `next export` 失败，因为导出不是有效的 AMP。
