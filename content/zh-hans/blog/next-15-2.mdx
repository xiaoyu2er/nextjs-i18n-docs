*   [**重构的错误 UI 和改进的堆栈跟踪 (Redesigned Error UI and Improved Stack Traces)**](/blog/next-15-2#redesigned-error-ui-and-improved-stack-traces): 重构的调试体验

流式传输元数据 (Streaming Metadata){#streaming-metadata}
-----------------------------------------

在 `generateMetadata` 中经常需要获取动态数据或执行一些异步操作。在之前的 Next.js 版本中，这些元数据必须完成生成才能发送初始 UI 以便将其包含在文档 `<head>` 中。

这意味着对于许多具有快速初始 UI 可用的页面，初始渲染仍然受到不影响用户视觉看到的数据需求的延迟。我们在 15.2 版中改进了这一点，允许在 `generateMetadata` 完成之前将初始 UI 发送到浏览器。

![Next.js 开发工具首选项示例](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-15-2/metadata-light.png)![Next.js 开发工具首选项示例](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Fstatic%2Fblog%2Fnext-15-2%2Fmetadata-dark.png&w=3840&q=75)
Next.js 开发工具首选项示例

Turbopack 性能提升 (Turbopack Performance Improvements){#turbopack-performance-improvements}
-------------------------------------------------------------------------

Turbopack 已在 [Next.js 15](https://nextjs.org/blog/next-15#turbopack-dev) 中标记为稳定。

一直在改进 Turbopack 的性能，特别是在没有持久缓存的场景中。作为此次发布的一部分，我们引入了以下增强功能：

*   **更快的编译时间：** 早期采用者报告称，与 Next.js 15.1 相比，访问路由时的编译时间**快 57.6%**。
*   **减少内存使用：** 对于 [vercel.com](https://vercel.com/) 应用程序，我们观察到本地开发期间**内存使用减少了 30%**。
有了这些改进，Turbopack 在几乎所有情况下现在都应该比 Webpack 更快。如果您遇到不符合您应用程序的情况，请联系我们——我们要调查这些情况。

我们在持久缓存和生产构建方面也取得了进展。虽然这些功能尚未准备好进行实验性发布，但我们已开始在实际项目中进行测试。当它们可供更广泛使用时，我们将分享更详细的指标。

React 视图过渡 (React View Transitions)（实验性）{#react-view-transitions-experimental}
-----------------------------------------------------------------------------

我们添加了一个特性标志，以在 React 中启用新的实验性 View Transitions API。此新 API 允许您动画切换应用程序中的不同视图和组件。

要启用此功能，请在您的 `next.config.js` 中添加以下内容：

next.config.js

```
module.exports = {
  experimental: {
    viewTransition: true,
  },
};
```

> **注意：** 此功能实验性很强，在未来的版本中可能会更改。

有关如何使用此功能的更详细信息，请参阅 React 仓库中的 [原始 View Transition pull request](https://github.com/facebook/react/pull/31975)。这项工作建立在 View Transitions 的原生浏览器实现之上。

随着稳定性的推进，我们将发布更多文档和示例。

Node.js 中间件 (Node.js Middleware)（实验性）{#nodejs-middleware-experimental}
--------------------------------------------------------------------

一直在开发一个新的实验性标志，以允许为 Next.js Middleware 使用 Node.js 运行时。

要启用此功能，请在您的 `next.config.js` 中添加以下内容：

next.config.js

```
module.exports = {
  experimental: {
    nodeMiddleware: true,
  },
};
```

有关如何使用此功能的更详细信息，请参阅 React 仓库中的 [原始 View Transition pull request](https://github.com/facebook/react/pull/31975)。这项工作建立在 View Transitions 的原生浏览器实现之上。

随着稳定性的推进，我们将发布更多文档和示例。

```
import bcrypt from 'bcrypt';
 
const API_KEY_HASH = process.env.API_KEY_HASH; // Pre-hashed API key in env
 
export default async function middleware(req) {
  const apiKey = req.headers.get('x-api-key');
 
  if (!apiKey || !(await bcrypt.compare(apiKey, API_KEY_HASH))) {
    return new Response('Forbidden', { status: 403 });
  }
 
  console.log('API key validated');
}
 
export const config = {
  runtime: 'nodejs',
};
```

> **注意：** 此功能尚未建议用于生产环境。因此，除非您使用的是 `next@canary` 发布版本，否则 Next.js 将抛出错误。

我们借此机会改进和重塑 Middleware API 计划。如果您有任何建议或请求，请告诉我们。Node.js Middleware 是社区请求的首选，我们很高兴解决这个问题。
