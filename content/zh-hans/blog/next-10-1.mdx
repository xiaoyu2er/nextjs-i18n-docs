我们很高兴介绍 Next.js 10.1，其亮点包括：

*   **[3 倍更快的刷新](#3x-faster-refresh)**: 200 毫秒更快的刷新，无需任何更改。
*   **[安装时间优化](#improved-installation-time)**: 58% 更小的安装大小，56% 更少的依赖项。
*   **[`next/image` 改进](#nextimage-improvements)**: Apple Silicon (M1) 支持，以及更多布局和加载器选项。
*   **[Next.js Commerce Shopify 集成](#nextjs-commerce-shopify-integration)**: 用于可组合电商应用的灵活数据层。
*   **[自定义 500 页面](#custom-500-page)**: 在错误页面添加您自己的徽标和品牌。
*   **[严格加载 PostCSS 配置](#strict-postcss-configuration-loading)**: 与 Webpack 5 一起改善缓存。
*   **[`tsconfig.json` 中支持 `extends`](#support-for-extends-in-tsconfigjson)**: 面向大型 TypeScript 应用的配置文件扩展。

*   **[文档改进](#documentation-improvements)**: 增量采用、迁移和 Docker 部署。

3 倍更快的刷新 [](#3x-faster-refresh)
---------------------------------------

从 [v9.4](https://nextjs.org/blog/next-9-4) 开始，Next.js 将 Facebook 的 [Fast Refresh](/docs/architecture/fast-refresh) 引入开源，以便对对 React 组件进行的编辑提供即时反馈。

这意味着 Next.js 仅更新您编辑文件中的代码，并且仅重新渲染该组件，而不会丢失组件状态。这包括样式（内联、CSS-in-JS 或 CSS/Sass 模块）、标记、事件处理程序和效果（通过 `useEffect`）。

今天，我们对每一次编辑都将其提速 200 毫秒，且您在代码端无需任何更改。平均而言，这使得 Fast Refresh 速度提升 3 倍。

想获得更快的性能和更佳的构建时间吗？即刻通过新的 `future` 标志启用 Webpack 5。此标志为发行候选版本（RC），Webpack 5 很快将成为默认选项。启用此标志后，请 [与我们分享您的反馈](https://github.com/vercel/next.js/discussions/23498)。

next.config.js

```
module.exports = {
  future: {
    webpack5: true,
  },
};
```

若要了解更多关于 Webpack 5 如何以向后兼容方式部署到您的应用程序的信息，请查看 [文档](/docs/messages/webpack5)。

安装时间优化 [](#improved-installation-time)
---------------------------------------------------------

我们已优化 Next.js 的依赖图以减少安装时间和依赖大小。现在创建 Next.js 项目比以往任何时候都更快，无论是在本地还是在 CI/CD 流程中。

| | 10.0 | 10.1 | delta |
| --- | --- | --- | --- |
| 平均安装时间 | ~15s | ~5s | 3.0 倍快 |
| 安装大小 | 96.5MB | 39.9MB | 58% 更小 |
| 依赖项数量 | 424 | 187 | 56% 更小 |

作为优化依赖的目标之一，我们更新或替换了这些 package 作者标记为废弃的 npm 包——即使它们深度嵌套在依赖树中。接下来安装 Next.js 10.1 将不再产生任何包警告。

[`next/image` 改进](#nextimage-improvements)
----------------------------------------------------

### Apple Silicon 支持 [](#apple-silicon-support)

在 Next.js 10 [https://nextjs.org/blog/next-10]** (https://nextjs.org/blog/next-10) 发布时，Image Optimization 依赖于原生依赖项。这导致安装时间变慢，原生依赖项占据了 Next.js 总安装大小的 50%。此外，这还移除了安装后运行脚本并导致按需编译。

我们很高兴宣布，通过 `next/image` 自动图像处理现在由 [WebAssembly](https://web.dev/squoosh-v2/) 驱动。此优化使用 `next dev` 和 `next start` 开箱即用。通过使用 WASM，我们将安装大小减少了约 30MB，并**增加了对 [Apple Silicon](https://www.apple.com/mac/m1/) M1 MacBooks**的支持。

我们正继续利用 [SIMD](https://v8.dev/features/simd) 扩展和多线程技术来提高 WebAssembly 性能。

#### 额外布局 [](#additional-layouts)

根据您的反馈，我们为 `next/image` 添加了各种新布局和选项：

*   `layout=fill`: 您无需提供 `width` 和 `height`。([演示](https://image-component.nextjs.gallery/layout-fill))
*   `layout=fixed`: 原生 `img` 行为。([演示](https://image-component.nextjs.gallery/layout-fixed))
*   `layout=responsive`: 灵活宽度，固定宽高比。([演示](https://image-component.nextjs.gallery/layout-responsive))
*   `layout=intrinsic`: 不会增长，但可随宽高比缩小。([演示](https://image-component.nextjs.gallery/layout-intrinsic))
*   `objectFit=cover`: 背景图像。([演示](https://image-component.nextjs.gallery/background))

更多信息，请查看 [`layout` 文档](/docs/pages/api-reference/components/image#layout)。

#### 改进的图像优化支持 [](#improved-image-optimization-support)

我们扩展了 `next/image` 的[内置图像加载器](/docs/pages/building-your-application/optimizing/images#loader)，提供添加任何自定义加载器的能力。通过使用新的 `loader` 属性，您**可以与任何提供商或 CDN 优化图像**。

components/my-image.js

```
import Image from 'next/image';
 
const myLoader = ({ src, width, quality }) => {
  return `https://example.com/${src}?w=${width}&q=${quality || 75}`;
};
 
const MyImage = (props) => {
  return (
    <Image
      loader={myLoader}
      src="/me.png"
      alt="Picture of the author"
      width={500}
      height={500}
    />
  );
};
```

更多信息，请查看 [`next/image` 文档](/docs/pages/api-reference/components/image#loader)。

Next.js Commerce Shopify 集成 [](#nextjs-commerce-shopify-integration)
----------------------------------------------------------------------------

电子商务是采用 Next.js 增长最快的行业之一。Apple、Wal-Mart、McDonald's 和 Nike 等公司都信任 Next.js。

我们要为您提供构建电商应用的高性能起点，因此我们发布了 [Next.js Commerce](https://nextjs.org/commerce)。经过数千对 Shopify 的请求后，我们更新了 Next.js Commerce，加入了**与提供商无关的 UI** —— 使您能够引入您选择的任何无头电商解决方案。

在提供商之间切换仅需在配置中进行 [单行更改](https://github.com/vercel/commerce#configuration)。我们还创建了一个 Features API，允许您根据使用场景切换功能 —— 同时保留同类最佳的性能。

![Next.js Commerce 的全球 Lighthouse 指标。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-10-1/lighthouse.png)

Next.js Commerce 的全球 Lighthouse 指标。

只需几次点击，Next.js 开发者即可克隆、部署并完全自定义自己的电商商店。在 [nextjs.org/commerce](https://nextjs.org/commerce) 了解更多信息，并试用我们的 [Shopify 演示](https://shopify.demo.vercel.store/)。更多电商提供商的支持即将推出。

自定义 500 页面 [](#custom-500-page)
-----------------------------------

您现在可以创建一个自定义 `pages/500.js`，在显示错误时添加您自己的徽标和品牌。类似于 `pages/404.js`，此文件在构建时静态生成。

pages/500.js

```
export default function Custom500() {
  return <h1>500 - Server-side error occurred</h1>;
}
```

严格加载 PostCSS 配置 [](#strict-postcss-configuration-loading)
-----------------------------------------------------------------------------

如果您正在使用 PostCSS 和 Webpack 5，则可以通过新的 `future` 标志提高性能（通过缓存）。

next.config.js

```
module.exports = {
  future: {
    webpack5: true,
    strictPostcssConfiguration: true,
  },
};
```

今天就试用一下，并让我们知道您的反馈。更多信息，请查看 [PostCSS 配置文档](/docs/pages/building-your-application/configuring/post-css)。

`tsconfig.json` 中支持 `extends` [](#support-for-extends-in-tsconfigjson)
--------------------------------------------------------------------------------

您现在可以在 Next.js 使用的 `tsconfig.json` 中使用 `extends` 来扩展项目中的其他 `tsconfig.json`。例如，您可以如下扩展项目中的 `tsconfig.base.json`：

tsconfig.json

```
{
  "extends": "./tsconfig.base.json"
}
```

更多信息，请查看 [TypeScript 文档](/docs/pages/building-your-application/configuring/typescript)。

检测预览模式是否启用 [](#detect-when-preview-mode-is-enabled)
---------------------------------------------------------------------------

您现在可以检测到 Preview Mode 是否启用，这允许您仅对编辑器渲染某些组件。

components/MyComponent.js

```
import { useRouter } from 'next/router';
 
function MyComponent() {
  const { isPreview } = useRouter();
  return <>{isPreview ? <h1>Preview Mode Enabled</h1> : null}</>;
}
```

更多信息，请查看 [预览模式文档](/docs/pages/building-your-application/configuring/preview-mode)。

路由方法滚动至顶部 [](#router-methods-scroll-to-top)
-------------------------------------------------------------

使用 `next/link` 导航时，Next.js 将自动滚动到顶部。随着 Next.js 10.1 的 `router.push` 和 `router.replace` 也将自动滚动到顶部以匹配 `next/link` 行为。您可以使用 `scroll` 选项退出自动滚动。

更多信息，请查看 [`next/router` 文档](/docs/pages/api-reference/functions/use-router)。
