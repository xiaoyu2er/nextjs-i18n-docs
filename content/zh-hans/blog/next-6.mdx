今年，ZEIT Day 主题演讲一开始就重点介绍了我们的开源项目，其中包括展示 Next.js 的相关指标。在 [GitHub](https://github.com/vercel/next.js) 上，它已获得超过 25000 个 Star，且已有超过 10000 个网站由它驱动。我们对它的增长感到非常惊叹，也非常乐意看到越来越多项目依赖它。

今天，我们自豪地推出生产就绪的 **Next.js 6**，具有以下功能：

*   零配置静态导出。默认情况下无需 `next.config.js`
*   `_app.js`，支持页面过渡、错误边界等的扩展点
*   Babel 7 和 Fragment 语法 `<>` 支持
*   扩展集成测试套件，重点关注安全性
*   核心代码库中的 Flow 注解

除了 6.0 版本发布，我们还将在其专属主页 [nextjs.org](https://nextjs.org) 上展示 Next.js，其内容包括：

*   所有 Next.js 文档集中在一处。不再需要去 GitHub 查阅 README 文件
*   将 [https://learnnextjs.com](https://learnnextjs.com) 合并到 [https://nextjs.org/learn](https://nextjs.org/learn)
*   展示使用 Next.js 构建的最令人印象深刻的网站

静态 React 应用程序[](#static-react-applications)
-------------------------------------------------------

Next.js 专注于 **预渲染 (Pre-rendering)** 的理念，作为实现高性能的一种手段。预渲染有两种形式：

*   **服务器渲染**：每个请求都会触发渲染。因此，最终用户无需等待任何 JS 下载完成即可开始消耗数据
*   **静态渲染**：输出静态文件，可直接提供服务，而无需在服务器上执行任何代码

直到现在，Next.js 中的静态导出虽然功能强大，但使用不够简便。即使没有使用自定义路由，也需要设置 [手动路由映射](/docs#static-html-export)。

使用 Next.js 6，我们会根据您的 `pages/` 目录内容自动生成路由映射。如果您不使用高级自定义路由，则无需对 `next.config.js` 进行任何修改。只需运行：

```
next build
next export
```

例如，[查看这个静态部署到 [Vercel](https://vercel.com) 的网站](https://out-khozebgbnl.now.sh/)。该网站的 [源代码](https://github.com/vercel/next.js/tree/canary/examples/basic-css) 也可用。

App 组件[](#app-component)
-------------------------------

Next.js 提供了一个名为 [`_document.js`](https://github.com/vercel/next.js#custom-document) 的扩展点。如果定义，它允许您覆盖应用程序最顶层的文档，它会渲染 `<html>` 元素。

`_document.js` 允许强大的扩展，但它有一些严重的限制。例如，React 无法在客户端直接渲染 `<html>` 或 `<body>`，因此 `_document.js` 主要局限于初始预渲染阶段。

为了支持其他强大的用例，我们引入了 `_app.js`，它是包裹在每个页面外部的顶层组件。

![](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-6/document-app-differences.png)

> `_document.js` 和 `_app.js` 之间的部分区别

让我们看看定义 `_app.js` 所支持的某些用例。

### 页面过渡[](#page-transitions)

![](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-6/page-transitions.png)

> 页面过渡示例：[Xavier Cazalot](https://github.com/xavczen%22) 的 [`page-transitions-app-next.vercel.app`](https://page-transitions-app-next.vercel.app) ([源码](https://github.com/xavczen/nextjs-page-transitions))

不过，有了 `_app.js`，现在包含这些就变得更简单了。以下是一些示例：

### 更好的错误处理[](#better-error-handling)

React 提供了一个名为 `componentDidCatch` 的组件方法，它允许您捕获和处理在客户端由嵌套组件冒泡上来的异常。

在许多情况下，由于这些意外的异常性质，您可能希望在顶层同等地处理所有异常。

因此，`_app.js` 是定义 `componentDidCatch` 逻辑的好地方。这里有一个错误处理边界在实际中的 [示例](https://error-app-next.vercel.app) ([源代码](https://github.com/vercel/next.js/tree/canary/examples/with-componentdidcatch))

Babel 7[](#babel-7)
-------------------

我们已将 Babel 升级到最新版本：7。随之而来的是许多很棒的新功能和改进。

### JSX 片段[](#jsx-fragments)

React 16.2 引入了 `Fragment` API，它允许您列出一组元素，而无需将它们包装在任意的 HTML 元素（如 `<div>`）中：

```
render() {
  return <React.Fragment>
    <A />,
    <B />
  </React.Fragment>
}
```

编写这样的代码可能会很繁琐，使用 Next.js 6，您可以使用新的 JSX 片段语法来方便地创建片段：

```
render() {
  return <>
    <A />,
    <B />
  </>
}
```

### 嵌套的 .babelrc[](#nested-babelrc)

如果您的 Next.js 应用中有一个嵌套目录需要不同的 Babel 配置，现在可以在该目录中专门包含一个受限的 `.babelrc` 文件

```
src/
  .babelrc      # General .babelrc
  components/
    i18n/
      .babelrc  # This .babelrc only applies to this directory
```

### 一流的 TypeScript 支持[](#first-class-typescript-support)

当我们宣布 [Universal webpack](/blog/next-5) 时，我们指出可以通过 [ts-loader](https://github.com/TypeStrong/ts-loader) 使用 TypeScript，因为我们现在在服务器和客户端都运行 webpack。

Babel 7 具有内置的 TypeScript 支持（此前 Babel 仅支持 Flow）。

要使用它，只需安装 [@zeit/next-typescript](https://github.com/vercel/next-plugins/tree/master/packages/next-typescript/#readme) 的最新版本，或者查看 [此示例](https://github.com/vercel/next.js/tree/canary/examples/with-typescript)。
