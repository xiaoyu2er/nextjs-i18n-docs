Next.js 13.3 添加了社区广泛请求的功能，包括：

*   [**基于文件的元数据 API**](#file-based-metadata-api): 动态生成站点地图、robots 文件、网站图标等。
*   [**动态 Open Graph 图像**](#dynamic-open-graph-image-generation): 使用 JSX、HTML 和 CSS 生成 OG 图像。
*   [**App Router 的静态导出**](#static-export-for-app-router): 支持服务器组件的静态/单页应用 (SPA)。
*   [**并行路由与拦截**](#parallel-routes-and-interception): App Router 的高级路由功能。

今天就运行以下命令进行更新：

终端

```
npm i next@latest react@latest react-dom@latest eslint-config-next@latest
```

我们即将在下一次次要版本中宣布 App Router 稳定，并将重点转向优化性能、增强行为以及修复 bug。

虽然我们正在开发一些功能，例如 Mutations，但我们预计这些不会对其他 App Router 功能的 API 表面产生影响。我们很期待看到大家使用 App Router 构建的内容，[并期待您的反馈](https://github.com/vercel/next.js/discussions/41745)。

基于文件的元数据 API[](#file-based-metadata-api)
---------------------------------------------------

在 [Next.js 13.2](https://nextjs.org/blog/next-13-2#built-in-seo-support-with-new-metadata-api) 中，我们宣布了一种新的元数据 API，允许您通过从布局或页面导出 Metadata 对象来定义元数据（例如 HTML `head` 元素中的 `title`、`meta` 和 `link` 标签）。

`layout.js` 或 `page.js`

```
// either Static metadata
export const metadata = {
  title: 'Home',
};
// Output:
// <head>
//	 <title>Home</title>
// </head>
 
// or Dynamic metadata
export async function generateMetadata({ params, searchParams }) {
  const product = await getProduct(params.id);
  return { title: product.title };
}
// Output:
// <head>
//	 <title>My Unique Product</title>
// </head>
 
export default function Page() {}
```

除了基于配置（config-based）的元数据之外，元数据 API 现在支持新的文件约定，允许您更方便地自定义页面，以提升 SEO 和网页分享体验：

*   `opengraph-image.(jpg|png|svg)`
*   `twitter-image.(jpg|png|svg)`
*   `favicon.ico`
*   `icon.(ico|jpg|png|svg)`
*   `sitemap.(xml|js|jsx|ts|tsx)`
*   `robots.(txt|js|jsx|ts|tsx)`
*   `manifest.(json|js|jsx|ts|tsx)`

例如，您可以使用基于文件的元数据为您的应用添加 favicon，并为 `/about` 页面添加开放图谱图像：

```
app
├── favicon.ico
├── layout.js
├── page.js
└── about
    ├── opengraph-image.jpg
    └── page.js
```

Next.js 会在生产环境中自动为这些文件加上哈希值（用于文件名）以进行缓存，并更新相关的 `head` 元素，提供正确的元数据信息，如资源的 URL、文件类型和图像尺寸。

```
// Visiting "/"
<link rel="icon" href="<computedUrl>"/>
 
// Visiting "/about"
<link rel="icon" href="<computedUrl>"/>
<meta property="og:image" content="<computedUrl>" type="<computedType>" ... />
```

向应用程序中添加静态文件通常是最简单的方法，但在某些情况下，您可能需要动态创建文件。每种静态文件约定都有一个配套的动态 `(.js|.jsx|.ts|.tsx)` 变体，允许您编写代码来生成该文件。

例如，虽然您可以添加一个静态 `sitemap.xml` 文件，但大多数网站都有部分页面是使用外部数据源动态生成的。要生成动态站点地图，您可以添加一个返回动态路由数组的 `sitemap.js` 文件。

`app/sitemap.js`

```
export default async function sitemap() {
  const res = await fetch('https://.../posts');
  const allPosts = await res.json();
 
  const posts = allPosts.map((post) => ({
    url: `https://acme.com/blog/${post.slug}`,
    lastModified: post.publishedAt,
  }));
 
  const routes = ['', '/about', '/blog'].map((route) => ({
    url: `https://acme.com${route}`,
    lastModified: new Date().toISOString(),
  }));
 
  return [...routes, ...posts];
}
```

结合 [基于配置](/docs/app/api-reference/file-conventions/metadata#metadata-object) 和新文件选项，您现在拥有了涵盖静态和动态元数据的全面元数据 API。

元数据 API 在 13.3 版本中适用于 App Router (`app`)。它在 `pages` 目录中不可用。了解有关 [基于文件的元数据并查看 API 参考](/docs/app/api-reference/file-conventions/metadata#file-based-metadata)。

动态 Open Graph 图像生成 [](#dynamic-open-graph-image-generation)
---------------------------------------------------------------------------

六个月前，我们发布了 [@vercel/og](https://vercel.com/blog/introducing-vercel-og-image-generation-fast-dynamic-social-card-images) 和 [Satori](https://github.com/vercel/satori)，这两个库允许您使用 JSX、HTML 和 CSS 动态生成图像。

`@vercel/og` 在 [Next.js Conf](https://nextjs.org/conf) 上接受了考验，为每位与会者生成了超过 100,000 张动态票据图像。在 Vercel 客户中已被广泛采用，自发布以来下载量超过 900,000 次，我们很高兴能够在无需外部包的情况下，将所有 Next.js 应用都集成动态生成图像的功能。

您现在可以从 `next/server` 导入 `ImageResponse` 来生成图像：

`/app/about/opengraph-image.tsx`

```
import { ImageResponse } from 'next/server';
 
export const size = { width: 1200, height: 600 };
export const alt = 'About Acme';
export const contentType = 'image/png';
export const runtime = 'edge';
 
export default function og() {
  return new ImageResponse();
  // ...
}
```

`ImageResponse` 很好地集成了其他 Next.js API，包括路由处理程序和基于文件的元数据。例如，您可以在 `opengraph-image.tsx` 文件中使用 `ImageResponse` 在构建时或在请求时动态地生成 Open Graph 和 Twitter 图像。

了解更多关于 [Image Response API.](/docs/app/building-your-application/optimizing/metadata#dynamic-image-generation)

App Router 的静态导出 [](#static-export-for-app-router)
-------------------------------------------------------------

Next.js App Router 现在支持完全静态导出。

你可以从静态站点或单页应用 (SPA) 开始，然后以后根据需要升级为使用需要服务器的 Next.js 功能。

当运行 `next build` 时，Next.js 会为每个路由生成一个 HTML 文件。通过将严格的 SPA 拆分为独立的 HTML 文件，Next.js 可以避免在客户端加载不必要的 JavaScript 代码，从而减小包体积并实现更快的页面加载。

next.config.js

```
/**
 * @type {import('next').NextConfig}
 */
const nextConfig = {
  output: 'export',
};
 
module.exports = nextConfig;
```

静态导出与 `app` 路由器的新功能一起工作，包括静态路由处理程序、Open Graph 图像和 React 服务器组件。

例如，服务器组件将在构建期间运行，类似于传统的静态站点生成，将组件渲染为初始页面加载所需的静态 HTML，以及路由之间客户端导航所需的静态负载。

以前，要在 `pages` 目录中使用静态导出，您需要运行 `next export`。但是，使用 `next.config.js` 选项，当设置 `output: 'export'` 时，`next build` 将输出 `out` 目录。您可以为 `app` 路由器和 `pages` 目录使用相同的配置。这意味着不再需要 `next export`。

有了高级静态导出支持，您在开发过程中 (`next dev`) 可以更早地获得错误，例如在尝试使用需要服务器的动态函数（如 `cookies()` 或 `headers()`）时。

了解更多关于 [Static Export](/docs/app/building-your-application/deploying/static-exports)。

并行路由与拦截 [](#parallel-routes-and-interception)
---------------------------------------------------------------------

Next.js 13.3 引入了新的动态约定，允许您实现高级路由用例：**并行路由与拦截路由**。这些功能允许您在同一视图中显示多个页面，就像复杂仪表板或模态窗口那样。

通过并行路由，您可以在_同一视图_中同时渲染一个或多个页面，这些页面可以独立导航。它也可用于条件渲染页面。

并行路由使用命名的 **"插槽"** 创建。插槽使用 `@folder` 定义：

```
dashboard
├── @user
│   └── page.js
├── @team
│   └── page.js
├── layout.js
└── page.js
```

同一路线段中的布局将作为 props 接受插槽：

app/dashboard/layout.js

```
export default async function Layout({ children, user, team }) {
  const userType = getCurrentUserType();
 
  return (
    <>
      {userType === 'user' ? user : team}
      {children}
    </>
  );
}
```

在上面的示例中，`@user` 和 `@team` 并行路由插槽（显式）根据您的逻辑进行条件渲染。`children` 是一个隐式路由插槽，_不需要_映射到 `@folder`。例如，`dashboard/page.js` 相当于 `dashboard/@children/page.js`。

拦截路由允许您在当前布局内加载新路由，同时“掩盖”浏览器 URL。这在保持当前页面的上下文很重要时很有用，例如通过模态窗口展开信息流中的照片，而信息流保持在模态窗口的背景中。

拦截路由可以使用 `(..)` 定义，类似于相对路径 `../`。您也可以使用 `(...)` 创建相对于 `app` 目录的路径。

```
feed
├── @modal
│   └── (..)photo
│       └── [id]
│           └── page.tsx
├── page.tsx
└── layout.tsx
photo
└── [id]
    └── page.tsx
```

在上面的示例中，单击用户个人资料中的照片将在客户端导航期间在模态窗口中打开照片。然而，刷新或共享页面将以默认布局加载照片。

![并行路由与拦截实现了类似 Instagram 的模态路由。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-13-3/modal-routing.png)

并行路由与拦截实现了类似 Instagram 的模态路由。

这解决了您在创建模态窗口时可能遇到的挑战，例如使模态内容可通过 URL 共享、防止在刷新页面时丢失上下文，以及通过前向和后向导航关闭和重新打开模态窗口。

有关更多示例和行为，请参阅 [Parallel](/docs/app/building-your-application/routing/parallel-routes) 和 [拦截路由](/docs/app/building-your-application/routing/intercepting-routes) 文档。

[其他改进](#other-improvements)
-----------------------------------------

*   **设计更新：** Next.js 主页和展示页面的设计已更新。
*   **Turbopack：** 添加了中间件、所有 `next/font` 选项以及服务器组件流式传输的支持（即将推出 beta 版本，[查看演示](https://twitter.com/padmaia/status/1641130761775464448)）。我们还修复了在使用成熟 Next.js 应用（如 `vercel.com` 和 `nextjs.org`）时发现的额外 bug。[了解更多](/docs/architecture/turbopack)。
*   **`next.config.js` 的快速刷新：** 对 `next.config.js` 所做的更改现在会自动重启本地开发服务器。这扩展了对 `.env`、`.env.*`、`jsconfig.json`、`tsconfig.json` 配置文件的自动重新加载功能。
*   **可访问性：** App Router 现在包括来自 `pages` 的路由播报。此功能向屏幕阅读器等其他辅助技术宣布客户端路由转换。[了解更多](/docs/accessibility)。
*   **静态类型链接：** `next.config.js` 中设置的 `redirects` 和 `rewrites` 现在在类型检查期间会被考虑。[了解更多](/docs/app/building-your-application/configuring/typescript#statically-typed-links)。
*   **`create-next-app` 的 Tailwind CSS：** 在使用 `npx create-next-app@latest` 开始新项目时，您现在可以选择是否使用 Tailwind CSS，或者使用 `--tailwind` 标志，预先配置此样式解决方案。
*   **路由处理程序：** 使用 `export default` 而不是 [支持的 HTTP 动词](/docs/app/api-reference/file-conventions/route) 现在会在 `route.ts` 上抛出有用的错误。[了解有关路由处理程序](/docs/app/building-your-application/routing/route-handlers)。
*   **图像：** `next/image` 现在支持 [`fetchPriority="high"`](https://developer.mozilla.org/docs/Web/HTML/Element/img#:~:text=elementtiming%20attribute%20page.-,fetchpriority,-Experimental) 属性。
*   **元数据：** 13.2 版本中弃用的旧元数据 API（`head.js`）已被删除。相反，请使用 [Metadata](/docs/app/api-reference/file-conventions/metadata) API 内置的 SEO 支持。
*   **选择退出路由的文件夹：** 在文件夹前加上 `_` 可将其及其子段排除在路由之外。例如，`app/_dashboard/page.tsx` 将不可路由。
*   **App Router：** 我们添加了一个新的 `useParams` 客户端组件钩子，用于读取给定路由段的动态参数。[了解更多](/docs/app/api-reference/functions/use-params)。
*   **改进的样式表加载：** Next.js 现在实现了 React 的 Suspense CSS，修复了 CSS 加载和无样式内容闪现的许多问题，尤其是在导航期间。
*   **改进了未找到（Not Found）的处理：** 除了捕获预期的 `notFound()` 错误外，根 `app/not-found.js` 文件还将处理整个应用程序的所有未匹配 URL。这意味着访问您应用程序未处理 URL 的用户将看到由 `app/not-found.js` 文件导出的 UI。[了解更多](/docs/app/api-reference/file-conventions/not-found)。
*   **改进了客户端路由缓存：** `router.refresh()` 现在将使整个缓存失效，并且搜索参数现在成为缓存键的一部分，允许在两个搜索参数之间导航（例如 `/?search=leerob` 和 `/?search=tim`）以正确恢复依赖于参数的内容。
