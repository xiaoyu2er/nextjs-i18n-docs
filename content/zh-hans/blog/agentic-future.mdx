过去一年，我们致力于提升 Next.js 的 AI 智能体体验。在此期间，我们构建并下线了一个浏览器内智能体，推出了 MCP 集成，并领悟到更好的智能体支持实质性的解决方案是从智能体的视角出发。

### 智能体无法看见浏览器 [](#agents-couldnt-see-the-browser)

今年夏天早些时候，我们在改进 Next.js 开发者工具 (devtools) 时注意到一个模式。开发者会在浏览器中看到错误，复制详细信息，粘贴到 AI 编辑器中，并要求智能体修复它。

问题在于智能体无法看到浏览器。运行时错误 (runtime errors)、客户端警告和已渲染组件对它们来说都是不可见的。当用户说“修复错误”时，智能体不知道指的是什么错误。

我们的第一个响应是更新复制按钮以捕获结构化的错误数据。接着我们添加了一个功能，将浏览器日志转发到终端。这些小改动指向了一个更大的领悟。我们需要让 Next.js 本身对智能体可见。

### 尝试开发浏览器内智能体 [](#experimenting-with-an-in-browser-agent)

这引出了一个雄心勃勃的想法。如果我们在 Next.js 内部直接构建一个像智能开发者工具一样工作的智能体，会怎么样？

我们构建了一个名为 Vector 的浏览器内聊天智能体。类似于 `react-grab` 但集成了 Next.js，Vector 允许你选择页面上的元素，查看其源代码，并提示进行更改。它内置了 Next.js 最佳实践，以帮助智能体避免幻觉。

![显示错误检测和推荐修复的 Vector 界面。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/agentic-future/vector-both-theme.png)

_副标题：显示错误检测和推荐修复的 Vector 界面。_

Vector 很有用，但它与 Cursor 和 Claude Code 等通用编码智能体重叠。大多数开发者已经在所有项目中使用这些工具，而不仅仅是 Next.js。UI 选择功能让你能轻松指定要更改的确切内容，但这并不是人们每天都需要用的东西。

我们下线了 Vector，但保留了让它有价值的部分（结构化可见性和框架特定知识），并决定将其构建到 Next.js 本身之中。

### MCP 让 Next.js 状态对智能体可见 [](#mcp-made-nextjs-state-visible-to-agents)

2025 年 10 月 Next.js v16 发布前后，用户在使用智能体进行调试时遇到了困难。常见的提示是“修复错误”，要求智能体解决浏览器覆盖层中的问题。但智能体会请求页面 HTML，却没发现任何异常。

运行时失败、浏览器 JavaScript 错误和异步错误都存在于浏览器中，而不是 HTML 中。已渲染的页面、布局片段、路由和其他内部状态对智能体来说是不可见的。

MCP 为我们提供了一种暴露这些数据的方法。第一个版本展示了错误、路由和渲染片段等内部状态，但仅暴露数据还不够。智能体还需要发现正在运行的开发服务器并与之通信，这导致了 `next-devtools-mcp` 的诞生。

![Next.js MCP 响应“如何使用缓存？”的文档和上下文。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/agentic-future/answer-dark.png)

MCP 还集成了提示词和工具，以帮助进行升级和缓存组件迁移。如果你想了解更多，这里有一个关于 MCP 集成的详细演讲。

### 秘诀在于像智能体一样思考 [](#the-answer-is-to-think-like-an-agent)

MCP 证实了 Vector 教给我们的东西。智能体需要感知 Next.js 正在做什么，但这只是事情的一部分。更深层的教训是将智能体视为 Next.js 的一等用户，并从它们的视角出发。它们需要什么信息？何时需要？如何获取？

这种思维模式带来了实际的变化。如果智能体在开发期间读取终端输出，那么记录 Server Action (服务器操作) 调用和转发浏览器错误就能提供它们所需的线索。如果智能体难以理解训练数据中没有的框架概念，那么嵌入压缩的文档索引 `agents.md` 或提供结构化工作流（Next.js skills）能为它们提供比单纯文档更好的上下文。

这些问题贯穿了我们构建的一切。对可见性的需求促成了更好的日志记录，对知识的需求促成了 `agents.md`，对发现的需求促成了 MCP。当你将智能体视为一等用户并在它们所在的地方提供帮助时，调试就变成了代码、运行时和 AI 之间紧密的反馈循环。

### 接下来会怎样 [](#whats-next)

我们目前正在致力于使其更容易采用。你已经可以运行 `npx @next/codemod` 来为你的项目生成最新的文档索引，我们正在扩展评估套件以覆盖更多的 Next.js 16 API，以便衡量什么真正对智能体有帮助。从长远来看，我们希望将其内置到 `next dev` 中，这样智能体就能在没有设置的情况下自动获得正确的上下文。

我们很期待您的反馈和想法，关于如何让 Next.js 与智能体工作得更好。
