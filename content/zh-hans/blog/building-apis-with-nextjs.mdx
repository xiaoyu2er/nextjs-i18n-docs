1. 入门<a id="1-getting-started"></a>

### 1.1 创建 Next.js 应用<a id="11-create-a-nextjs-app"></a>

如果您是初次开始，可以使用以下命令创建新的 Next.js 项目：

终端

```
npx create-next-app@latest --api
```

> **注意：** `--api` 标志会自动在新项目的 `app/` 文件夹中包含一个示例 `route.ts`，展示如何创建 API 端点。

### 1.2 App Router (App Router) 与 Pages Router (Pages Router)<a id="12-app-router-vs-pages-router"></a>

*   **Pages Router (Pages Router)**：历史上，Next.js 使用 `pages/api/*` 进行 API 开发。这种方法依赖 Node.js 的 request/response 对象和类 Express API。
*   **App Router (App Router)（默认）**：在 Next.js 13 中引入，App Router 全面拥抱了 Web 标准 Request/Response (Request/Response) API。不再使用 `pages/api/*`，您可以在 `app/` 目录内的任何位置放置 `route.ts` 或 `route.js` 文件。

> **为什么切换？** App Router 的"Route Handlers (Route Handlers)"依赖于 [Web Platform Request/Response (Request/Response) APIs](https://developer.mozilla.org/en-US/docs/Web/API)，而不是 Node.js 特定的 API。这简化了学习过程，减少了摩擦，并有助于您在不同的工具之间复用知识。

2. 为什么（以及何时）使用 Next.js 构建 API<a id="2-why-and-when-to-build-apis-with-nextjs"></a>

1.  **为多个客户端构建公共 API**
    
    *   您可以构建一个公共 API，供 Next.js Web 应用、独立的移动应用或任何第三方服务使用。例如，您可能同时在 React 网站和 React Native 移动应用中从 /api/users 获取数据。
2.  **代理现有后端**
    
    *   有时您想将外部 [微服务](https://vercel.com/blog/how-vercel-adopted-microfrontends) 隐藏或集中在一个端点后面。Next.js Route Handlers (Route Handlers) 可以作为代理或中间层，连接另一个现有的后端。例如，您可能拦截请求、处理身份验证、转换数据，然后将请求传递给上游 API。
3.  **Webhooks (回调) 和集成**
    
    *   如果您接收外部回调或 Webhooks (例如来自 Stripe、GitHub、Twilio)，可以使用 Route Handlers (Route Handlers) 处理它们。
4.  **自定义身份验证**
    
    *   如果您需要会话、令牌或其他认证逻辑，您可以在 Next.js API 层中存储 Cookie (Cookies)、读取头文件，并用适当的数据响应。

> **注意：** 如果您只需要为自身的 Next.js 应用进行服务端数据获取（并且不需要在外部共享数据），Server Components (Server Components) 可能足以在渲染期间直接获取数据——不需要单独的 API 层。

3. 使用 Route Handlers (Route Handlers) 创建 API<a id="3-creating-an-api-with-route-handlers"></a>

### 3.1 基本文件设置<a id="31-basic-file-setup"></a>

在 App Router (`app/`) 中，创建一个代表您路由的文件夹，并在其中创建 `route.ts` 文件。

例如，要创建位于 `/api/users` 的端点：

```
app
└── api
    └── users
        └── route.ts
```

### 3.2 一个文件中的多种 HTTP (HTTP) 方法<a id="32-multiple-http-methods-in-one-file"></a>

与 Pages Router API 路由（只有一个默认导出）不同，您可以从同一个文件导出代表不同 HTTP (HTTP) 方法的多个函数。

app/api/users/route.ts

```
export async function GET(request: Request) {
  // For example, fetch data from your DB here
  const users = [
    { id: 1, name: 'Alice' },
    { id: 2, name: 'Bob' }
  ];
  return new Response(JSON.stringify(users), {
    status: 200,
    headers: { 'Content-Type': 'application/json' }
  });
}
 
export async function POST(request: Request) {
  // Parse the request body
  const body = await request.json();
  const { name } = body;
 
  // e.g. Insert new user into your DB
  const newUser = { id: Date.now(), name };
 
  return new Response(JSON.stringify(newUser), {
    status: 201,
    headers: { 'Content-Type': 'application/json' }
  });
}
```

现在，向 `/api/users` 发送 GET (GET) 请求将返回您的用户列表，而向同一 URL 发送 POST (POST) 请求将插入新用户。

4. 与 Web Web APIs (Web APIs) 协作<a id="4-working-with-web-apis"></a>

### 4.1 直接使用 Request (Request) 与 Response (Response)<a id="41-directly-using-request--response"></a>

默认情况下，您的 Route Handler 方法 (`GET`, `POST` 等) 接收一个标准的 [Request (Request)](https://developer.mozilla.org/docs/Web/API/Request) 对象，并且您必须返回一个标准的 [Response (Response)](https://developer.mozilla.org/docs/Web/API/Response) 对象。

### 4.2 查询参数<a id="42-query-parameters"></a>

app/api/search/route.ts

```
import { NextRequest } from 'next/server';
 
export function GET(request: NextRequest) {
  const searchParams = request.nextUrl.searchParams;
  const query = searchParams.get('query'); // e.g. `/api/search?query=hello`
 
  return new Response(
    JSON.stringify({ result: `You searched for: ${query}` }),
    {
      headers: { 'Content-Type': 'application/json' },
    },
  );
}
```

### 4.3 头文件和 Cookies (Cookies)<a id="43-headers-and-cookies"></a>

app/api/auth/route.ts

```
import { NextRequest } from 'next/server';
import { cookies, headers } from 'next/headers';
 
export async function GET(request: NextRequest) {
  // 1. Using 'next/headers' helpers
  const cookieStore = await cookies();
  const token = cookieStore.get('token');
 
  const headersList = await headers();
  const referer = headersList.get('referer');
 
  // 2. Using the standard Web APIs
  const userAgent = request.headers.get('user-agent');
 
  return new Response(JSON.stringify({ token, referer, userAgent }), {
    headers: { 'Content-Type': 'application/json' },
  });
}
```

`cookies()` 和 `headers()` 函数如果您计划在 Next.js 的其他服务端代码中重用共享逻辑，可能会很有帮助。您会注意到 Next.js 还提供更扩展了基础 Web APIs 的 `NextRequest` 和 `NextResponse`。

5. 动态路由<a id="5-dynamic-routes"></a>

要创建动态路径（例如 `/api/users/:id`），请使用文件夹结构中的 **Dynamic Segments (动态段)**：

```
app
└── api
    └── users
        └── [id]
            └── route.ts
```

此文件对应于 `/api/users/123` 这样的 URL，其中 `123` 被捕获为参数。

app/api/users/\[id\]/route.ts

```
import { NextRequest } from 'next/server';
 
export async function GET(
  request: NextRequest,
  { params }: { params: Promise<{ id: string }> },
) {
  const id = (await params).id;
  // e.g. Query a database for user with ID `id`
  return new Response(JSON.stringify({ id, name: `User ${id}` }), {
    status: 200,
    headers: { 'Content-Type': 'application/json' },
  });
}
 
export async function DELETE(
  request: NextRequest,
  { params }: { params: Promise<{ id: string }> },
) {
  const id = (await params).id;
  // e.g. Delete user with ID `id` in DB
  return new Response(null, { status: 204 });
}
```

这里，`params.id` 为您提供动态段。

6. 将 Next.js 用作代理 (Proxy) 或转发层<a id="6-using-nextjs-as-a-proxy-or-forwarding-layer"></a>

常见场景是 **代理 (Proxying)** 现有的后端服务。您可以进行身份验证、处理日志记录，或在发送数据到远程服务器或后端之前转换数据：

app/api/external/route.ts

```
import { NextRequest } from 'next/server';
 
export async function GET(request: NextRequest) {
  const response = await fetch('https://example.com/api/data', {
    // Optional: forward some headers, add auth tokens, etc.
    headers: { Authorization: `Bearer ${process.env.API_TOKEN}` },
  });
 
  // Transform or forward the response
  const data = await response.json();
  const transformed = { ...data, source: 'proxied-through-nextjs' };
 
  return new Response(JSON.stringify(transformed), {
    headers: { 'Content-Type': 'application/json' },
  });
}
```

现在，您的客户端只需调用 `/api/external`，Next.js 将处理其余部分。这有时也称为"Backend for Frontend"或 BFF。

7. 构建共享“中间件 (Middleware)"逻辑<a id="7-building-shared-middleware-logic"></a>

如果您想将相同的逻辑（例如身份验证检查、日志记录）应用于多个 Route Handlers，您可以创建可重用的函数来包装您的处理程序：

lib/with-auth.ts

```
import { NextRequest } from 'next/server';
 
type Handler = (req: NextRequest, context?: any) => Promise<Response>;
 
export function withAuth(handler: Handler): Handler {
  return async (req, context) => {
    const token = req.cookies.get('token')?.value;
    if (!token) {
      return new Response(JSON.stringify({ error: 'Unauthorized' }), {
        status: 401,
        headers: { 'Content-Type': 'application/json' },
      });
    }
 
    // If authenticated, call the original handler
    return handler(req, context);
  };
}
```

然后在您的 Route Handler 中：

app/api/secret/route.ts

```
import { NextRequest } from 'next/server';
import { withAuth } from '@/lib/with-auth';
 
async function secretGET(request: NextRequest) {
  return new Response(JSON.stringify({ secret: 'Here be dragons' }), {
    headers: { 'Content-Type': 'application/json' },
  });
}
 
export const GET = withAuth(secretGET);
```

8. 部署与"SPA 模式”注意事项<a id="8-deployment-and-spa-mode-considerations"></a>

### 8.1 标准 Node.js 部署<a id="81-standard-nodejs-deployment"></a>

使用 `next start` 的标准 Next.js 服务端部署使您可以使用 Route Handlers (Route Handlers)、Server Components (Server Components)、中间件 (Middleware) 等功能，同时利用动态的请求时信息。

无需其他配置。有关更多详细信息，请参阅 [Deploying (部署)](/docs/app/building-your-application/deploying)。

### 8.2 SPA/静态导出<a id="82-spastatic-export"></a>

Next.js 还支持将整个网站输出为 [静态单页应用 (SPA)](/docs/app/building-your-application/upgrading/single-page-applications)。

您可以通过设置来启用此功能：

next.config.ts

```
import type { NextConfig } from 'next';
 
const nextConfig: NextConfig = {
  output: 'export',
};
 
export default nextConfig;
```

在 **静态导出模式** 下，Next.js 将生成纯粹的静态 HTML、CSS 和 JS。**您无法运行服务端代码**（例如 API 端点）。如果您仍然需要 API，则必须单独托管它（例如，独立的 Node.js 服务器）。

> **注意：**
> 
> *   **GET Route Handlers (GET Route Handlers)** 如果它们不依赖动态请求数据，可以 [静态导出](/docs/app/building-your-application/deploying/static-exports#route-handlers)。它们将变为 `out` 文件夹中的静态文件。
> *   **所有其他服务端功能**（动态请求、重写 Cookies (Cookies) 等）在纯 SPA 导出中 **不支持**。

### 8.3 在 Vercel 上部署 API<a id="83-deploying-apis-on-vercel"></a>

如果您正在将 Next.js 应用程序部署到 Vercel，我们有一份关于 [部署 API](https://vercel.com/guides/hosting-backend-apis) 的指南。这包括通过 Vercel Firewall 进行 [程序化速率限制](https://vercel.com/docs/security/vercel-waf/rate-limiting-sdk) 等其他 Vercel 功能。Vercel 还提供 [Cron Jobs (计划任务)](https://vercel.com/docs/cron-jobs/manage-cron-jobs)，这在使用 API 方法时通常很需要。

9. 何时跳过创建 API 端点<a id="9-when-to-skip-creating-an-api-endpoint"></a>

使用 App Router (App Router) 的 **React Server Components (React Server Components)**，您可以直接在服务端获取数据，而无需公开端点：

app/users/page.tsx

```
// (Server Component)
export default async function UsersPage() {
  // This fetch runs on the server (no client-side code needed here)
  const res = await fetch('https://api.example.com/users');
  const data = await res.json();
 
  return (
    <main>
      <h1>Users</h1>
      <ul>
        {data.map((user: any) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </main>
  );
}
```

如果您的数据仅在 Next.js 应用内部使用，您可能根本不需要公共 API。

10. 整合所有内容<a id="10-putting-it-all-together"></a>

1.  **创建新的 Next.js 项目**：`npx create-next-app@latest --api`。
2.  **添加 Route Handlers (Route Handlers)**：在 `app/` 目录内（例如，`app/api/users/route.ts`）。
3.  **导出 HTTP 方法** (`GET`, `POST`, `PUT`, `DELETE` 等)：在同一个文件中。
4.  **使用 Web Web APIs (Web APIs) 标准**：与 `Request (Request)` 对象交互并返回 `Response (Response)`。
5.  **构建公共 API (API)**：如果您需要其他客户端消费您的数据，或代理后端服务。
6.  **获取**：从客户端获取新 API 路由（例如，在 Client Component (客户端组件) 内或带 `fetch('/api/...')`）。
7.  或者 **跳过创建 API**：如果 Server Components (服务端组件) 可以直接获取数据。
8.  **添加共享“中间件 (Middleware)"** 模式（例如，`withAuth()`）进行认证或其他重复逻辑。
9.  **部署 (Deploying)**：对于服务端功能，部署到支持 Node 的环境，或者如果您只需要静态 SPA，则 **静态导出**。

结论<a id="conclusion"></a>

使用 Next.js 的 **App Router (App Router)** 和 **Route Handlers (Route Handlers)** 为您提供了一种灵活、现代的方法来构建直接拥抱 **Web Web Platform (Web 平台)** 的 API。您可以：

*   **创建完整的公共 API (API)**：供 Web、移动或第三方客户端共享。
*   **代理 (Proxy)** 并自定义对现有外部服务的调用。
*   **实现** 用于认证、日志记录或任何重复逻辑的可重用“中间件 (Middleware)"层。
*   **动态路由**：使用 `[id]` 段文件夹结构请求路由。

常见问题解答<a id="frequently-asked-questions"></a>

### 关于 Server Actions (Server Actions) 呢？<a id="what-about-server-actions"></a>

您可以将 [Server Actions (Server Actions)](/docs/app/building-your-application/data-fetching/server-actions-and-mutations) 视为为客户端调用的自动生成的 POST API 路由。

它们专为突变操作而设计，例如创建、更新或删除数据。您调用 Server Action 就像调用普通 JavaScript 函数，而不是向定义的 API 路由发出明确的 `fetch`。

虽然有网络请求仍在发生，但不需要显式管理它。URL 路径是自动生成的并 [加密](/docs/app/building-your-application/data-fetching/server-actions-and-mutations#security)，因此您无法在浏览器中手动访问像 `/api/users` 这样的路由。

如果您计划使用 Server Actions (Server Actions) **并** 公开 API，我们建议将核心逻辑移至 [数据访问层 (Data Access Layer)](/blog/security-nextjs-server-components-actions)，并从 Server Action 和 API 路由双方调用相同的逻辑。

### 我能否在 Route Handlers (Route Handlers) 中使用 TypeScript？<a id="can-i-use-typescript-with-route-handlers"></a>

是的，您可以在 Route Handlers 中使用 TypeScript。例如，在您的 `route` 文件中定义 `Request (Request)` 和 `Response (Response)` 类型。

有关 [Next.js 与 TypeScript](/docs/app/api-reference/config/typescript) 的更多信息。
