1\. البداية[](#1-getting-started)
-----------------------------------------

### 1.1 إنشاء تطبيق Next.js[](#11-create-a-nextjs-app)

إذا كنت تبدأ من جديد، يمكنك إنشاء مشروع Next.js جديد باستخدام:

محفظة الأوامر (Terminal)

```
npx create-next-app@latest --api
```

> **ملاحظة:** يقوم الخيار `--api` تلقائيًا بتضمين مثال `route.ts` في مجلد `app/` الخاص بمشروعك الجديد، يوضح كيفية إنشاء نقطة طرفية لواجهة برمجية.

### 1.2 مقارنة بين App Router و Pages Router[](#12-app-router-vs-pages-router)

*   **مسارات الصفحات (Pages Router)**: تاريخياً، كانت Next.js تستخدم `pages/api/*` للواجهات البرمجية. اعتمدت هذه الطريقة على كائنات طلب/استجابة Node.js و واجهة برمجية تشبه Express.
*   **تطبيق المسار (App Router - الافتراضي)**: تم تقديمه في Next.js 13، يقبل تطبيق المسار بشكل كامل واجهات طلب/استجابة الويب القياسية. بدلاً من `pages/api/*`، يمكنك الآن وضع ملفات `route.ts` أو `route.js` في أي مكان داخل المجلد `app/`.

> **لماذا الانتقال؟** تعتمد "معالجات المسارات" في تطبيق المسار على [واجهة طلب/استجابة منصة الويب](https://developer.mozilla.org/en-US/docs/Web/API) بدلاً من واجهات محددة لـ Node.js. هذا يبسط التعلم، ويقلل الاحتكاك، ويساعدك على إعادة استخدام معرفتك عبر أدوات مختلفة.

2\. لماذا (ومتى) بناء واجهات برمجية باستخدام Next.js[](#2-why-and-when-to-build-apis-with-nextjs)
------------------------------------------------------------------------------------------

1.  **واجهة برمجية عامة لعدة عملاء**
    
    *   يمكنك بناء واجهة برمجية عامة يتم استهلاكها بواسطة تطبيق الويب الخاص بك في Next.js، أو تطبيق جوال منفصل، أو أي خدمة تابعة لجهة خارجية. على سبيل المثال، قد تجلب من /api/users كلًا في موقع React الخاص بك وفي تطبيق جوال React Native.
2.  **وسيط إلى خادم برمجي موجود**
    
    *   في بعض الأحيان تريد إخفاء أو دمج الخدمات المصغرة الخارجية [microservices](https://vercel.com/blog/how-vercel-adopted-microfrontends) خلف نقطة طرفية واحدة. يمكن أن تعمل معالجات مسارات Next.js كوسيط أو طبقة ثانية لخادم برمجي موجود. على سبيل المثال، قد تقوم بتقاطر الطلبات ومعالجة المصادقة وتحويل البيانات، ثم تمرير الطلب إلى واجهة برمجية تابعة للأعلى.
3.  **الإشارات المرسلة (Webhooks) والتكامل**
    
    *   إذا كنت تتلقى ردود فعل خارجية أو إشارات مرسلّة (مثل تلك من Stripe، GitHub، Twilio)، يمكنك معالجتها باستخدام معالجات المسارات.
4.  **مصادقة مخصصة**
    
    *   إذا كنت بحاجة إلى جلسات، رموز، أو منطق مصادقة آخر، يمكنك تخزين الكوكيز، وقراءة رؤوس الطلب، والرد بالبيانات المناسبة في طبقة واجهة برمجية في Next.js.

> **ملاحظة:** إذا كنت تحتاج فقط إلى جلب بيانات على جانب الخادم لتطبيق Next.js الخاص بك (وقد لا تحتاج إلى مشاركة هذه البيانات خارجيًا)، فقد تكون مكونات الخادم كافية لجلب البيانات مباشرةً أثناء العرض - لا يلزم وجود طبقة API منفصلة.

3\. إنشاء API باستخدام معالجات المسارات[](#3-creating-an-api-with-route-handlers)
---------------------------------------------------------------------------------

### 3.1 إعداد الملفات الأساسي[](#31-basic-file-setup)

في تطبيق المسار (`app/`)، أنشئ مجلداً يمثل المسار الخاص بك، وداخله ملف `route.ts`.

على سبيل المثال، لإنشاء نقطة طرفية عند `/api/users`:

```
app
└── api
    └── users
        └── route.ts
```

### 3.2 طرق HTTP متعددة في ملف واحد[](#32-multiple-http-methods-in-one-file)

على عكس مسارات API لمسارات الصفحات (التي كان لديها تصدير افتراضي واحد)، يمكنك تصدير دوال متعددة تمثل طرق HTTP مختلفة من نفس الملف.

app/api/users/route.ts

```
export async function GET(request: Request) {
  // For example, fetch data from your DB here
  const users = [
    { id: 1, name: 'Alice' },
    { id: 2, name: 'Bob' }
  ];
  return new Response(JSON.stringify(users), {
    status: 200,
    headers: { 'Content-Type': 'application/json' }
  });
}
 
export async function POST(request: Request) {
  // Parse the request body
  const body = await request.json();
  const { name } = body;
 
  // e.g. Insert new user into your DB
  const newUser = { id: Date.now(), name };
 
  return new Response(JSON.stringify(newUser), {
    status: 201,
    headers: { 'Content-Type': 'application/json' }
  });
}
```

الآن، عند إرسال طلب GET إلى `/api/users` يتم إرجاع قائمة المستخدمين الخاصة بك، بينما طلب `POST` لنفس الرابط سيقوم بإدخال واحد جديد.

4\. العمل مع واجهات الويب[](#4-working-with-web-apis)
-----------------------------------------------------

### 4.1 استخدام Request & Response مباشرةً[](#41-directly-using-request--response)

بشكل افتراضي، تستقبل طرق معالج المسار (`GET`، `POST`، الخ) كائن [Request](https://developer.mozilla.org/docs/Web/API/Request) قياسي، ويجب عليك إرجاع كائن [Response](https://developer.mozilla.org/docs/Web/API/Response) قياسي.

### 4.2 معاملات الاستعلام[](#42-query-parameters)

app/api/search/route.ts

```
import { NextRequest } from 'next/server';
 
export function GET(request: NextRequest) {
  const searchParams = request.nextUrl.searchParams;
  const query = searchParams.get('query'); // e.g. `/api/search?query=hello`
 
  return new Response(
    JSON.stringify({ result: `You searched for: ${query}` }),
    {
      headers: { 'Content-Type': 'application/json' },
    },
  );
}
```

### 4.3 رؤوس الطلب والكوكيز[](#43-headers-and-cookies)

app/api/auth/route.ts

```
import { NextRequest } from 'next/server';
import { cookies, headers } from 'next/headers';
 
export async function GET(request: NextRequest) {
  // 1. Using 'next/headers' helpers
  const cookieStore = await cookies();
  const token = cookieStore.get('token');
 
  const headersList = await headers();
  const referer = headersList.get('referer');
 
  // 2. Using the standard Web APIs
  const userAgent = request.headers.get('user-agent');
 
  return new Response(JSON.stringify({ token, referer, userAgent }), {
    headers: { 'Content-Type': 'application/json' },
  });
}
```

يمكن أن تكون الدوال `cookies()` و `headers()` مفيدة إذا كنت تخطط لإعادة استخدام منطق مشترك عبر أكواد أخرى على جانب الخادم في Next.js. ستلاحظ أن Next.js توفر أيضًا `NextRequest` و `NextResponse` التي توسع واجهات الويب الأساسية.

5\. المسارات الديناميكية[](#5-dynamic-routes)
---------------------------------------

لإنشاء مسارات ديناميكية (مثل `/api/users/:id`)، استخدم **الأقسام الديناميكية** في هيكل مجلدك:

```
app
└── api
    └── users
        └── [id]
            └── route.ts
```

يتوافق هذا الملف مع عنوان URL مثل `/api/users/123`، مع التقاط `123` كعامل.

app/api/users/\[id\]/route.ts

```
import { NextRequest } from 'next/server';
 
export async function GET(
  request: NextRequest,
  { params }: { params: Promise<{ id: string }> },
) {
  const id = (await params).id;
  // e.g. Query a database for user with ID `id`
  return new Response(JSON.stringify({ id, name: `User ${id}` }), {
    status: 200,
    headers: { 'Content-Type': 'application/json' },
  });
}
 
export async function DELETE(
  request: NextRequest,
  { params }: { params: Promise<{ id: string }> },
) {
  const id = (await params).id;
  // e.g. Delete user with ID `id` in DB
  return new Response(null, { status: 204 });
}
```

هنا، `params.id` يعطيك القسم الديناميكي.

6\. استخدام Next.js كطبقة وسيطة أو تحويل[](#6-using-nextjs-as-a-proxy-or-forwarding-layer)
--------------------------------------------------------------------------------------------------

السيناريو الشائع هو **وساطة** خدمة خادم برمجي موجودة. يمكنك تأمين الطلبات، ومعالجة السجلات، أو تحويل البيانات قبل إرسالها إلى خادم بعيد أو خادم برمجي:

app/api/external/route.ts

```
import { NextRequest } from 'next/server';
 
export async function GET(request: NextRequest) {
  const response = await fetch('https://example.com/api/data', {
    // Optional: forward some headers, add auth tokens, etc.
    headers: { Authorization: `Bearer ${process.env.API_TOKEN}` },
  });
 
  // Transform or forward the response
  const data = await response.json();
  const transformed = { ...data, source: 'proxied-through-nextjs' };
 
  return new Response(JSON.stringify(transformed), {
    headers: { 'Content-Type': 'application/json' },
  });
}
```

الآن يحتاج عملاؤك فقط إلى استدعاء `/api/external`، وسيقوم Next.js بالتعامل مع الباقي. يُطلق على هذا أيضًا أحيانًا اسم "خادم برمجي لواجهة أمامية" أو BFF.

7\. منطق "الوسيط" المشترك البناء[](#7-building-shared-middleware-logic)
-----------------------------------------------------------------------------

إذا كنت تريد تطبيق نفس المنطق (مثل فحوصات المصادقة، السجلات) عبر عدة معالجات مسار، يمكنك إنشاء دوال قابلة لإعادة الاستخدام التي تغلف معالجاتك:

lib/with-auth.ts

```
import { NextRequest } from 'next/server';
 
type Handler = (req: NextRequest, context?: any) => Promise<Response>;
 
export function withAuth(handler: Handler): Handler {
  return async (req, context) => {
    const token = req.cookies.get('token')?.value;
    if (!token) {
      return new Response(JSON.stringify({ error: 'Unauthorized' }), {
        status: 401,
        headers: { 'Content-Type': 'application/json' },
      });
    }
 
    // If authenticated, call the original handler
    return handler(req, context);
  };
}
```

ثم في معالج المسار الخاص بك:

app/api/secret/route.ts

```
import { NextRequest } from 'next/server';
import { withAuth } from '@/lib/with-auth';
 
async function secretGET(request: NextRequest) {
  return new Response(JSON.stringify({ secret: 'Here be dragons' }), {
    headers: { 'Content-Type': 'application/json' },
  });
}
 
export const GET = withAuth(secretGET);
```

8\. اعتبارات النشر و "وضع SPA"[](#8-deployment-and-spa-mode-considerations)
-----------------------------------------------------------------------------------------

### 8.1 النشر العادي لـ Node.js[](#81-standard-nodejs-deployment)

النشر العادي لخادم Next.js باستخدام `next start` يتيح لك استخدام ميزات مثل معالجات المسارات، مكونات الخادم، الوسائط، والمزيد - مع الاستفادة من معلومات الطلب الديناميكية.

لا يلزم أي إعداد إضافي. يُرجى الاطلاع على [النشر](/docs/app/building-your-application/deploying) لمزيد من التفاصيل.

### 8.2 تصدير Static / SPA[](#82-spastatic-export)

يدعم Next.js أيضًا إصدار موقعك بالكامل على أنه [تطبيق الصفحة الواحدة الثابت (SPA)](/docs/app/building-your-application/upgrading/single-page-applications).

يمكنك تمكين ذلك عن طريق تحديد:

next.config.ts

```
import type { NextConfig } from 'next';
 
const nextConfig: NextConfig = {
  output: 'export',
};
 
export default nextConfig;
```

في **وضع التصدير الثابت**، سيقوم Next.js بتوليد HTML و CSS و JS ثابتة بالكامل. **لا يمكنك تشغيل أكواد على جانب الخادم** (مثل نقاط طرف API). إذا كنت لا تزال بحاجة إلى API، فستحتاج إلى استضافته بشكل منفصل (مثل خادم Node.js مستقل).

> **ملاحظة**:
> 
> *   **معالجات مسار GET** [يمكن تصديرها بشكل ثابت](/docs/app/building-your-application/deploying/static-exports#route-handlers) إذا لم تعتمد على بيانات الطلب الديناميكية. تصبح ملفات ثابتة في مجلد out الخاص بك.
> *   **جميع الميزات الأخرى على الخادم** (الطلبات الديناميكية، إعادة كتابة الكوكيز، الخ) **غير مدعومة** في تصدير SPA النقي.

### 8.3 نشر الـ APIs على Vercel[](#83-deploying-apis-on-vercel)

إذا كنت تنشر تطبيق Next.js الخاص بك على Vercel، لدينا [دليل حول نشر الواجهات البرمجية](https://vercel.com/guides/hosting-backend-apis). يتضمن ذلك ميزات Vercel الأخرى مثل [محدودية المعدل البرمجية](https://vercel.com/docs/security/vercel-waf/rate-limiting-sdk) عبر جدار حماية Vercel. تقدم Vercel أيضًا [أعمال جدولة cron](https://vercel.com/docs/cron-jobs/manage-cron-jobs)، وهي مطلوبة بشكل شائع مع منهجيات API.

9\. متى يتخطى إنشاء نقطة طرفية API[](#9-when-to-skip-creating-an-api-endpoint)
-------------------------------------------------------------------------------------

مع **مكونات خادم React** الموجودة في تطبيق المسار، يمكنك جلب البيانات مباشرة على الخادم دون تعريض نقطة طرفية عامة:

app/users/page.tsx

```
// (Server Component)
export default async function UsersPage() {
  // This fetch runs on the server (no client-side code needed here)
  const res = await fetch('https://api.example.com/users');
  const data = await res.json();
 
  return (
    <main>
      <h1>Users</h1>
      <ul>
        {data.map((user: any) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </main>
  );
}
```

إذا كانت البيانات الخاصة بك تُستخدم فقط داخل تطبيق Next.js الخاص بك، فقد لا تحتاج على الإطلاق إلى واجهة برمجية عامة.

10\. تجميع كل شيء معاً[](#10-putting-it-all-together)
-----------------------------------------------------------

1.  **إنشاء مشروع Next.js جديد**: `npx create-next-app@latest --api`.
2.  **إضافة معالجات المسارات** داخل المجلد `app/` (مثل `app/api/users/route.ts`).
3.  **تصدير طرق HTTP** (`GET`، `POST`، `PUT`، `DELETE`، وما إلى ذلك) في نفس الملف.
4.  **استخدام واجهات الويب القياسية** للتفاعل مع كائن `Request` وإرجاع `Response`.
5.  **بناء واجهة برمجية عامة** إذا كنت تريد أن تستهلكها عملاء آخرون، أو لوساطة خدمة خلفية.
6.  **جلب** مسارات API الجديدة الخاصة بك من العميل (مثل داخل مكون عميل أو باستخدام `fetch('/api/...')`).
7.  أو **تخطي إنشاء API** تمامًا إذا كان يمكن لمكون الخادم مجرد جلب البيانات.
8.  **إضافة نمط "وسيط" مشترك** (مثل `withAuth()`) للمصادقة أو المنطق المتكرر الآخر.
9.  **النشر** إلى بيئة قابلة لـ Node.js للميزات الخادم، أو **تصدير** بشكل ثابت إذا كنت تحتاج فقط إلى SPA ثابت.

الخلاصة[](#conclusion)
-------------------------

يُمنحك استخدام **تطبيق المسار** في Next.js و **معالجات المسارات** طريقة مرنة وعصرية لبناء واجهات برمجية تعتنق **منصة الويب** مباشرة. يمكنك:

*   **إنشاء واجهة برمجية عامة كاملة** ليتم مشاركتها عبر الويب أو الجوال أو عملاء جهات خارجية.
*   **وسيط** وتخصيص المكالمات إلى الخدمات الخارجية الموجودة.
*   **تنفيذ** طبقة "وسيط" قابلة لإعادة الاستخدام للمصادقة أو السجلات أو أي منطق متكرر.
*   **توجيه الطلبات بشكل ديناميكي** باستخدام هيكل مجلد القسم `[id]`.

الأسئلة الأكثر شيوعاً[](#frequently-asked-questions)
---------------------------------------------------------

### ماذا عن Server Actions؟[](#what-about-server-actions)

يمكنك النظر إلى [إجراءات الخادم](/docs/app/building-your-application/data-fetching/server-actions-and-mutations) على أنها مسارات API `POST` تم توليدها تلقائيًا ويمكن استدعاؤها من العميل.

هي مصممة لعمليات التعديل، مثل إنشاء أو تحديث أو حذف البيانات. تقوم باستدعاء إجراء خادم كدالة جافا سكريبت عادية، مقابل إجراء `fetch` صريح لمسار API محدد.

في حين أن هناك _طلبًا شبكيًا_ يحدث بالفعل، لا تحتاج إلى إدارته بشكل صريح. يتم إنشاء عنوان URL تلقائيًا و [تشفيره](/docs/app/building-your-application/data-fetching/server-actions-and-mutations#security) ، لذا لا يمكنك الوصول يدويًا إلى مسار مثل `/api/users` في المتصفح.

إذا كنت تخطط لاستخدام إجراءات الخادم _و_ تعريض واجهة برمجية عامة، فنحن نوصي بنقل المنطق الأساسي إلى [طبقة الوصول إلى البيانات](/blog/security-nextjs-server-components-actions) واستدعاء نفس المنطق من كل من إجراء خادم ومسار API.

### هل يمكن استخدام TypeScript مع معالجات المسارات؟[](#can-i-use-typescript-with-route-handlers)

نعم، يمكنك استخدام TypeScript مع معالجات المسارات. على سبيل المثال، تحديد أنواع `Request` و `Response` في ملف `route` الخاص بك.

اعرف المزيد حول [TypeScript مع Next.js](/docs/app/api-reference/config/typescript).
