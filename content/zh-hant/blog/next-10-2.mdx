我們很興奮地介紹 Next.js 10.2，其中包含：

* **[更快建置](#webpack-5)**：帶快取功能的後續建置速度提升達 ~60%。
* **[更快重新整理](#webpack-5)**：重新整理速度快 100ms 到 200ms。
* **[更快啟動](#improved-startup-performance)**：`next dev` 執行速度提升達 ~24%。
* **[改善無障礙功能](#accessibility-improvements)**：讀屏軟體現在會朗讀路由變更。
* **[更靈活的重新導向與重写](#routing-based-on-headers-and-query-string-parameters)**：匹配任何標頭、Cookie 或查詢字串。
* **[自動 Web 字型最佳化](#automatic-webfont-optimization)**：透過內嵌字型 CSS 改善效能。

透過執行 `npm i next@latest` 立即更新。

Webpack 5[](#webpack-5)
-----------------------

隨著 Next.js 10.1，我們改進了 [快速重新整理 (Fast Refresh)](/blog/next-10-1#3x-faster-refresh) 並 [縮短安裝時間](/blog/next-10-1#improved-installation-time)。我們很興奮能分享透過 webpack 5 啟用的額外效能改進。

今天，我們將 webpack 5 的推出範圍擴大至 Next.js 套用程式。從 Next.js 10.2 開始，所有在 `next.config.js` 中未使用自訂 webpack 配置的套用程式，都將自動使用 webpack 5。

啟用 webpack 5 後，您將自動獲得新功能和改善，例如：

* **改善的磁碟快取：** 磁碟快取允許編譯器在 `next build` 執行之間保留工作。只有變更的檔案會重新編譯，從而提高效能。Vercel 客戶 [Scale](http://scale.com/) 在後續建置上觀察到 **63% 的速度提升**。
* **改善的快速重新整理：** Next.js 現在會優先處理與快速重新整理相關的編譯，每次重新整理速度快 100ms 到 200ms。
* **改善的資產長期快取：** 多個 `next build` 執行之間的輸出現在具有確定性，改善了生產環境中 JavaScript 資產的瀏覽器快取。當頁面內容未變更時，雜湊值保持不變。
* **改善的樹搖動 (Tree Shaking)：** [CommonJS](https://en.wikipedia.org/wiki/CommonJS) 模組現在可以 [樹搖動 (tree shaking)](https://webpack.js.org/guides/tree-shaking/) 來自動移除未使用的程式碼。使用靜態分析來確定無副作用的模組。`export *` 追蹤更多資訊，不再將預設匯出標記為已使用。內部模組樹搖動也已啟用，允許僅在未使用的匯出中使用的匯入被樹搖動。

我們努力確保從 webpack 4 到 5 的平順過渡。Next.js 核心測試套件包含超過 **3,400 個整合測試**，在啟用 webpack 5 支援的所有拉取請求上運行。

如果您的套用程式有自訂 webpack 配置，我們建議參閱 [啟用 webpack 5 的說明文件](/docs/messages/webpack5)。在 Next.js 中升級至 webpack 5 後，請 [與我們分享任何回饋](https://github.com/vercel/next.js/discussions/23498)。

啟動效能改善 [](#improved-startup-performance)
-------------------------------------------------------------


我們改善了 Next.js CLI 初始化，使 `next dev` 在首次運行後的啓動時間快 **24%**。例如，`vercel.com` 的 `next dev` 從 3.3s 減少到 2.5s。

我們的目標是打造即時體驗的本地開發環境 ([20 倍速度提升](https://twitter.com/shuding_/status/1378086219708473344))。敬請期待即將推出的版本中更多的啟動效能改善。

無障礙功能改善 [](#accessibility-improvements)
---------------------------------------------------------


路由變更現在預設會通知螢幕閱讀器和其他輔助技術。

下面的範例顯示，在客戶端導航時，[macOS VoiceOver](https://www.apple.com/voiceover/info/guide/_1121.html) 朗讀標題「Real Data. Real Performance」。頁面的名稱是透過先搜尋 `<h1>` 元素找到，如果找不到則回退到 `document.title`，最後是 pathname。

路由變更現在會自動通知。

基於標頭和查詢字串參數的路由 [](#routing-based-on-headers-and-query-string-parameters)
-------------------------------------------------------------------------------------------------------------


Next.js [重新寫入](/docs/pages/api-reference/next-config-js/rewrites)、[重新導向](/docs/pages/api-reference/next-config-js/redirects) 和 [標頭](/docs/pages/api-reference/next-config-js/headers) 現在支援新的 `has` 屬性，允許您匹配傳入的標頭、Cookie 和查詢字串。

Vercel 客戶 [Joyn](https://www.joyn.de/) 使用 `has` 來優化內容的可見度和效能。例如，您可以根據 [User-Agent](https://developer.mozilla.org/docs/Web/HTTP/Headers/User-Agent) 重導向舊瀏覽器：

next.config.js

```
module.exports = {
  async redirects() {
    return [
      {
        source: '/:path((?!old-browser$).*)',
        has: [
          {
            type: 'header',
            key: 'User-Agent',
            value:
              'Mozilla/5.0 (compatible; MSIE 10.0; Windows Phone 8.0; Trident/6.0; IEMobile/10.0; ARM; Touch; Microsoft; Lumia 950)',
          },
        ],
        destination: '/old-browser',
        permanent: false,
      },
    ];
  },
};
```

另一個範例是根據使用者的位置重導向：

next.config.js

```
module.exports = {
  async redirects() {
    return [
      {
        source: '/:path((?!uk/).*)',
        has: [
          {
            type: 'header',
            key: 'x-vercel-ip-country',
            value: 'GB',
          },
        ],
        destination: '/uk/:path*',
        permanent: true,
      },
    ];
  },
};
```

最後，如果使用者已登入，您可以進行重導向：

next.config.js

```
module.exports = {
  async redirects() {
    return [
      {
        source: '/',
        has: [
          {
            type: 'header',
            key: 'x-authorized',
            value: '(?<authorized>yes|true)',
          },
        ],
        destination: '/dashboard?authorized=:authorized',
        permanent: false,
      },
    ];
  },
};
```

若要了解更多並探索更多範例，請查看 [重新導向說明文件](/docs/pages/api-reference/next-config-js/redirects#header-cookie-and-query-matching)。

自動 Web 字型最佳化 [](#automatic-webfont-optimization)
-----------------------------------------------------------------


[82% 的桌面版網頁](https://almanac.httparchive.org/en/2020/fonts) 使用網路字體。自訂字體對於網站品牌、設計以及跨瀏覽器/裝置的一致性至關重要。然而，使用網路字體不應以效能為代價。

Next.js 現在在使用 Web 字型時支援自動 Web 字型最佳化。預設情況下，Next.js 會在建置時自動內嵌字型 CSS，消除額外的往返來取得字型宣告。這能改善首次內容繪製 (FCP) 和最大內容繪製 (LCP)。例如：

```
// Before
<link href="https://fonts.googleapis.com/css2?family=Inter" rel="stylesheet" />
 
// After
<style data-href="https://fonts.googleapis.com/css2?family=Inter">
  @font-face {
    font-family: 'Inter';
    font-style: normal;
    <!-- ... -->
  }
</style>
```

自動 Web 字型最佳化目前支援 Google Fonts，我們正努力將支援範圍擴大到其他字型供應商。我們也計劃增加對 [載入策略 (loading strategies)](https://github.com/vercel/next.js/issues/21555) 和 `font-display` 值的控制。透過預設最佳化字型，我們能夠幫助開發人員發布更快速的網站，並提升其 [核心網頁指標 (Core Web Vitals)](https://vercel.com/blog/core-web-vitals)，無需額外努力。

感謝我們的 Google 合作夥伴以及 [Janicklas Ralph](https://github.com/janicklas-ralph) 協助實現此功能。
