Next.js 13.3 添加了社區廣泛請求的功能，包括：

*   [**檔案為基礎的元數據 API**](#file-based-metadata-api): 動態生成 sitemaps、robots、favicons 等。
*   [**動態 Open Graph 圖片**](#dynamic-open-graph-image-generation): 使用 JSX、HTML 和 CSS 生成 OG 圖片。
*   [**App Router 靜態導出**](#static-export-for-app-router): 為 Server Components 提供靜態/單頁應用程式 (SPA) 支援。
*   [**並行路由與攔截**](#parallel-routes-and-interception): 為 App Router 提供進階路由功能。

今天更新，請運行:

終端機

```
npm i next@latest react@latest react-dom@latest eslint-config-next@latest
```

我們即將在下一次次要版本中標記 App Router 為穩定版本，並將重心轉向優化效能、增強行為以及修復 Bug。

雖然我們仍在處理一些功能（如 Mutations），但不預期這會影響 App Router 其他功能的 API 表面。我們很期待看到你們用 App Router 建構什麼，並 [期待你的回饋](https://github.com/vercel/next.js/discussions/41745)。

檔案為基礎的元數據 API[](#file-based-metadata-api)
---------------------------------------------------

在 [Next.js 13.2](https://nextjs.org/blog/next-13-2#built-in-seo-support-with-new-metadata-api) 中，我們宣布了新的元數據 API，允許您從佈局或頁面匯出元數據物件，以定義元數據（例如：在 HTML `head` 元素內使用 `title`、`meta` 和 `link` 標籤）。

layout.js 或 page.js

```
// either Static metadata
export const metadata = {
  title: 'Home',
};
// Output:
// <head>
//	 <title>Home</title>
// </head>
 
// or Dynamic metadata
export async function generateMetadata({ params, searchParams }) {
  const product = await getProduct(params.id);
  return { title: product.title };
}
// Output:
// <head>
//	 <title>My Unique Product</title>
// </head>
 
export default function Page() {}
```

除了基於組態的元數據外，元數據 API 現在支援新的檔案慣例，讓您能更方便地自訂頁面以改善網站 SEO 和分享：

*   opengraph-image.(jpg|png|svg)
*   twitter-image.(jpg|png|svg)
*   favicon.ico
*   icon.(ico|jpg|png|svg)
*   sitemap.(xml|js|jsx|ts|tsx)
*   robots.(txt|js|jsx|ts|tsx)
*   manifest.(json|js|jsx|ts|tsx)

例如，您可以使用檔案為基礎的元數據來增加 App 的 favicon 以及 /about 頁面的 Open Graph 圖片：

```
app
├── favicon.ico
├── layout.js
├── page.js
└── about
    ├── opengraph-image.jpg
    └── page.js
```

Next.js 在生產環境中將自動使用雜湊值（針對檔案名稱）服務這些檔案以用於快取，並更新相關的 `head` 元素，包含正確的元數據資訊，例如資產的 URL、檔案類型和圖片尺寸。

```
// Visiting "/"
<link rel="icon" href="<computedUrl>"/>
 
// Visiting "/about"
<link rel="icon" href="<computedUrl>"/>
<meta property="og:image" content="<computedUrl>" type="<computedType>" ... />
```

將靜態檔案加入應用程式通常是最簡單的方法，但在某些情況下您可能需要動態建立檔案。針對每個靜態檔案慣例，都有一個對應的動態 (`.js`|`.jsx`|`.ts`|`.tsx`) 變體，讓您能撰寫程式碼來生成檔案。

例如，雖然您可以加入靜態 sitemap.xml 檔案，但多數網站包含某些使用外部資料來源動態生成頁面的頁面。為了生成動態 Sitemap，您可以新增一筆 sitemap.js 檔案，該檔案會回傳動態路由的陣列。

app/sitemap.js

```
export default async function sitemap() {
  const res = await fetch('https://.../posts');
  const allPosts = await res.json();
 
  const posts = allPosts.map((post) => ({
    url: `https://acme.com/blog/${post.slug}`,
    lastModified: post.publishedAt,
  }));
 
  const routes = ['', '/about', '/blog'].map((route) => ({
    url: `https://acme.com${route}`,
    lastModified: new Date().toISOString(),
  }));
 
  return [...routes, ...posts];
}
```

透過 [基於組態的](/docs/app/api-reference/file-conventions/metadata#metadata-object) 和新的檔案為基礎選項，您現在擁有一個全面的元數據 API，涵蓋靜態和動態元數據。

元數據 API 在 13.3 版本中適用於 App Router (`app`)。它不適用於 `pages` 目錄。了解更多內容 [關於檔案為基礎的元數據並查看 API 參考](/docs/app/api-reference/file-conventions/metadata#file-based-metadata)。

動態 Open Graph 圖片生成[](#dynamic-open-graph-image-generation)
---------------------------------------------------------------------------

六個月前，我們發布了 [@vercel/og](https://vercel.com/blog/introducing-vercel-og-image-generation-fast-dynamic-social-card-images) 和 [Satori](https://github.com/vercel/satori)，這兩個函式庫允許您使用 JSX、HTML 和 CSS 動態生成圖片。

`@vercel/og` 在 [Next.js Conf](https://nextjs.org/conf) 的測試中備受考驗，為每位與會者生成了超過 100,000 張動態票券圖片。由於廣泛採用於 Vercel 客戶群，且自發布以來下載次數超過 900,000 次，我們很高興能無需外部套件，即可在所有 Next.js 應用程式中使用動態生成的圖片。

您現在可以從 `next/server` 匯入 `ImageResponse` 來生成圖片：

/app/about/opengraph-image.tsx

```
import { ImageResponse } from 'next/server';
 
export const size = { width: 1200, height: 600 };
export const alt = 'About Acme';
export const contentType = 'image/png';
export const runtime = 'edge';
 
export default function og() {
  return new ImageResponse();
  // ...
}
```

`ImageResponse` 與其他 Next.js API（包括 Route Handlers 和檔案為基礎的元數據）的整合非常自然。例如，您可以在 `opengraph-image.tsx` 檔案中使用 `ImageResponse` 來在建置時間或请求時間動態生成 Open Graph 和 Twitter 圖片。

關於 [Image Response API.](/docs/app/building-your-application/optimizing/metadata#dynamic-image-generation) 了解更多

App Router 靜態導出 [](#static-export-for-app-router)
-------------------------------------------------------------

Next.js App Router 現在支援完整的靜態導出。

您可以從靜態網站或單頁應用程式 (SPA) 開始，之後可選擇升級為使用 Next.js 需要伺服器的功能。

當執行 `next build` 時，Next.js 會為每個路由生成一個 HTML 檔案。透過將嚴格的 SPA 拆分為獨立的 HTML 檔案，Next.js 可以避免在客戶端載入不必要的 JavaScript 程式碼，減少套件大小並實現更快的頁面載入。

next.config.js

```
/**
 * @type {import('next').NextConfig}
 */
const nextConfig = {
  output: 'export',
};
 
module.exports = nextConfig;
```

靜態導出 (Static Export) 支援 `app` 路由的新功能，包括靜態路由處理器 (Route Handlers)、Open Graph 圖片和 React Server Components。

例如，Server Components 會在建置期間執行，類似傳統的靜態網站生成，將元件渲染為靜態 HTML 以用於初始頁面載入，並在路由間進行客戶端導航時提供靜態載荷。

過去，要在 `pages` 目錄中使用靜態導出，您需要執行 `next export`。但是，透過 `next.config.js` 選項，當設定 `output: 'export'` 時，`next build` 將輸出 `out` 目錄。您可以用相同的組態在 `app` 路由器和 `pages` 目錄中使用。這意味著不再需要 `next export`。

利用進階靜態導出支援，您將在開發過程（`next dev`）的早期收到錯誤，例如嘗試使用需要伺服器的動態函式（如 `cookies()` 或 `headers()`）時。

了解更多關於 [Static Export](/docs/app/building-your-application/deploying/static-exports)。

並行路由與攔截 [](#parallel-routes-and-interception)
---------------------------------------------------------------------

Next.js 13.3 引入了新的動態慣例，讓您可以實作進階的路由案例：**並行路由與攔截路由**。這些功能使您可以同一視圖中顯示多個頁面，如複雜的儀表板或視窗。

透過並行路由，您可以同時在同一視圖中渲染一或多個頁面，這些頁面可以獨立導航。它也可用於條件式渲染頁面。

並行路由是使用命名的 `slots` 建立的。`Slots` 使用 `@folder` 慣例定義：

```
dashboard
├── @user
│   └── page.js
├── @team
│   └── page.js
├── layout.js
└── page.js
```

相同路由元件中的佈局將 slots 作為 props 接受：

app/dashboard/layout.js

```
export default async function Layout({ children, user, team }) {
  const userType = getCurrentUserType();
 
  return (
    <>
      {userType === 'user' ? user : team}
      {children}
    </>
  );
}
```

在上述範例中，`@user` 和 `@team` 並行路由 slots（明確的）會根據您的邏輯條件式渲染。`children` 是一個隱含的路由 slot，不需要對映到 `@folder`。例如，`dashboard/page.js` 等同於 `dashboard/@children/page.js`。

攔截路由允許您在目前的佈局中載入新路由，同時「遮蔽」瀏覽器 URL。這在需要保持目前頁面情境時很有用，例如透過視窗在 Feed 中展開照片，而 Feed 保留在視窗背景的後方。

攔截路由可以使用 `(..)` 慣例定義，類似於相對路徑 `../`。您也可以使用 `(...)` 慣例建立相對於 `app` 目錄的路徑。

```
feed
├── @modal
│   └── (..)photo
│       └── [id]
│           └── page.tsx
├── page.tsx
└── layout.tsx
photo
└── [id]
    └── page.tsx
```

在上述範例中，點擊用戶檔案中的照片會在客戶端導航期間以視窗開啟照片。然而，重新整理或分享頁面會以預設佈局載入照片。

![並行路由與攔截使 Instagram 風格的視窗路由變為可能。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-13-3/modal-routing.png)

並行路由與攔截使 Instagram 風格的視窗路由變為可能。

這解決了建立視窗時可能遇到的挑戰，例如透過 URL 分享視窗內容、防止頁面重新整理時情境流失，以及使用向後和向前導航關閉與重新開啟視窗。

如需更多範例和行為，請參閱 [並行路由](/docs/app/building-your-application/routing/parallel-routes) 和 [攔截路由](/docs/app/building-your-application/routing/intercepting-routes) 文件。

[其他改進](#other-improvements)
-----------------------------------------

*   **設計更新：** Next.js [主頁](https://nextjs.org/) 和 [展示](https://nextjs.org/showcase) 已用新設計刷新。
*   **Turbopack：** 新增 Middleware、所有 `next/font` 選項，以及 Server Components 串流支援（它接近 beta 狀態 ([see demo](https://twitter.com/padmaia/status/1641130761775464448))）。我們也已修復了在使用 mature Next.js 應用程式（如 [vercel.com](https://vercel.com) 和 [nextjs.org](https://nextjs.org/)）時發現的其他 Bug。[了解更多](/docs/architecture/turbopack)。
*   **`next.config.js` 的快速重新整理：** 更改 `next.config.js` 現在會自動重新啟動本端開發伺服器。這擴大了 `.env`、`.env.*`、`jsconfig.json`、`tsconfig.json` 組態檔案的自動重新載入。
*   **無障礙功能：** App Router 現在包含了來自 `pages` 的路徑通告。此功能向螢幕閱讀器和其他輔助技術通告客戶端路徑轉換。[了解更多](/docs/accessibility)。
*   **靜態類型連結：** `next.config.js` 中設定的 `redirects` 和 `rewrites` 現在在型別檢查時段會被視為有效。[了解更多](/docs/app/building-your-application/configuring/typescript#statically-typed-links)。
*   **`create-next-app` 的 Tailwind CSS：** 使用 `npx create-next-app@latest` 開始新專案時，您現在可以選擇 Tailwind CSS，或使用 `--tailwind` 旗標，以此設定解決方案預先您的應用程式。
*   **路由處理：** 使用 `export default` 而非 [支援的 HTTP 動詞](/docs/app/api-reference/file-conventions/route) 現在會在使用 `route.ts` 時拋出友善錯誤。[了解 Route Handlers 更多內容](/docs/app/building-your-application/routing/route-handlers)。
*   **圖片：** `next/image` 現在支援 [`fetchPriority="high"`](https://developer.mozilla.org/docs/Web/HTML/Element/img#:~:text=elementtiming%20attribute%20page.-,fetchpriority,-Experimental) 屬性。
*   **元數據：** 先前用於元數據的 API (`head.js`)，在 13.2 中已棄用，已被移除。相反，請使用透過 [元數據](/docs/app/api-reference/file-conventions/metadata) API 內建的 SEO 支援。
*   **跳過路由的資料夾：** 以 `_` 前綴資料夾以跳過該資料夾及其任何子元件的路由。例如，`app/_dashboard/page.tsx` 將不可導路。
*   **App Router：** 我們新增了一個新的 `useParams` 客戶端元件 Hook，用於讀取給定路由元件的動態參數。[了解更多](/docs/app/api-reference/functions/use-params)。
*   **增強的樣式表載入：** Next.js 現在實作 React 的 Suspense CSS，修復許多 CSS 載入和未樣式內容閃爍的問題，特別是在導航期間。
*   **增強的 404 處理：** 除了捕獲預期的 `notFound()` 錯誤外，根目錄的 `app/not-found.js` 檔案也會處理整個應用程式的任何未對映 URL。這意味著造訪未被應用程式處理的 URL 的用戶，將顯示 `app/not-found.js` 檔案中導出的 UI。[了解更多](/docs/app/api-reference/file-conventions/not-found)。
*   **增強的客戶端路由快取：** `router.refresh()` 現在將使整個快取無效，搜尋參數現在成為快取金鑰的一部分，允許在兩個搜尋參數（例如 `/?search=leerob` 和 `/?search=tim`）之間導航，以正確還原依賴參數的內容。
