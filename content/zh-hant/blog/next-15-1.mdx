Next.js 15.1 帶來核心升級、新 API 和開發者體驗的改進。主要更新包括:

* [**React 19 (穩定)**](/blog/next-15-1#react-19-stable): 支援 React 19 已正式提供，同時適用於 Pages Router 與 App Router。
* [**改進的錯誤除錯**](/blog/next-15-1#improved-error-debugging): 增強開發者体验和更好的源地圖，適用於瀏覽器及終端機。
* [`after` (穩定)](/blog/next-15-1#after-stable): 新 API，可在回應結束串流後執行程式碼。
* [`forbidden` / `unauthorized` (實驗性)](/blog/next-15-1#forbidden-and-unauthorized-experimental): 新 API，可啟用更細粒度的驗證錯誤處理。

立即升級，或開始使用：

終端機

```
# Use the automated upgrade CLI
npx @next/codemod@canary upgrade latest
 
# ...or upgrade manually
npm install next@latest react@latest react-dom@latest
 
# ...or start a new project
npx create-next-app@latest
```

React 19 (穩定)[](#react-19-stable)
-------------------------------------

Next.js 15.1 現在完全支援 React 19:

* **對於 Pages 路由**：您現在可以使用 React 19 穩定版，無需 Release Candidate 或 Canary 發行版，並繼續支援 React 18。
* **對於應用程式路由**：我們將繼續提供內建的 React Canary 發行版。這些包含所有穩定的 React 19 變更，以及框架中正在驗證的新功能，以便在新 React 發行版之前。

自 Next.js 15 發布以來，React 19 的一個重大新增為「[sibling預加載 (sibling pre-warming)](https://react.dev/blog/2024/04/25/react-19-upgrade-guide#improvements-to-suspense)」。

如需 React 19 更新的詳細概覽，請參閱 [官方 React 19 博客文章](https://react.dev/blog/2024/12/05/react-19)。

改進的錯誤除錯[](#improved-error-debugging)
-----------------------------------------------------

我們在 Next.js 中改進了錯誤除錯，確保您可以迅速找到問題的來源，無論它們出現在終端機、瀏覽器還是附加的除錯工具。這些增強功能適用於 Webpack 和 Turbopack ([現在已隨著 Next.js 15 穩定](/blog/turbopack-for-development-stable)。

### 源地圖增強[](#source-maps-enhancements)

現在透過改進的源地圖使用，錯誤更容易追蹤到其起源。我們已實施 [源地圖的 `ignoreList` 屬性](https://developer.chrome.com/docs/devtools/x-google-ignore-list)，允許 Next.js 隱藏外部相依項的堆疊框架，讓您的應用程式程式碼成為主要焦點。

為了讓方法名稱的源地圖映射更準確，建議採用 Turbopack (現已穩定)，它比 Webpack 具有更好的源地圖處理和偵測能力。

> **對於庫作者**：建議在發布您的庫時，將 `ignoreList` 屬性填入源地圖，特別是如果它們被配置為外部 (例如，在 `serverExternalPackages` 配置中)。

### 縮減的堆疊框架[](#collapsed-stack-frames)

我們已改進了縮減堆疊框架的邏輯，以突出顯示您程式碼中最相關的部分。

* **在瀏覽器和錯誤覆蓋層**：第三方相依項的堆疊框架預設會隱藏，聚焦於您的應用程式程式碼。您可以在開發者工具或覆蓋層中點擊「顯示忽略的框架」來揭示隱藏的框架。
* **在終端機中**：第三方相依項框架預設也會縮減，錯誤格式化現在與瀏覽器輸出一致以提供一致的除錯體驗。如果需要完整堆疊追蹤，錯誤會在瀏覽器中重播，以確保您在開發過程中不會遺失重要資訊。

### 效能分析增強[](#enhanced-profiling)

被忽略的堆疊框架也會被內建瀏覽器效能分析器識別。這讓效能分析您的應用程式變得更容易，讓您可以準確定位程式碼中的慢速函式，而不受外部函式庫雜訊的干擾。

### 邊緣運行時改進[](#improved-with-the-edge-runtime)

使用邊緣運行時時，錯誤現在在所有開發環境中一致顯示，確保無縫的除錯。以前，記錄的錯誤只會包含訊息，而不包含堆疊。

`after` (穩定)[](#after-stable)
---------------------------------

`after()` API 在首次引入 Next.js 15 RC 後現已穩定。

`after()` 提供了一種方法，可以在回應完成向用戶流式傳輸後執行記錄、分析和其他系統同步工作，而不會阻塞主要回應。

### 主要變更[](#key-changes)

自引入以來，我們已穩定化 `after()` 並解決了反饋，包括:

* **改進對** 自建 Next.js 伺服器的 **支援**。
* 解決 `after()` 與其他 Next.js 功能交互時的 **錯誤修復**。
* **增強可扩展性**，讓其他平台可以注入自己的 `waitUntil()` 原語來驅動 `after()`。
* **支援運行時 API**，如伺服器動作和路由處理程式中的 `cookies()` 和 `headers()`。

`app/layout.js`

```
import { after } from 'next/server';
import { log } from '@/app/utils';
 
export default function Layout({ children }) {
  // Secondary task
  after(() => {
    log();
  });
 
  // Primary task
  return <>{children}</>;
}
```

在文件中讀取更多關於 [`after`](/docs/app/api-reference/functions/after) API 資訊與如何將其納入文件。

`forbidden` 和 `unauthorized` (實驗性)[](#forbidden-and-unauthorized-experimental)
-----------------------------------------------------------------------------------------

Next.js 15.1 包含兩個實驗性 API `forbidden()` 和 `unauthorized()`，基於社群反饋。

> **我們非常期待您的反饋** — 請在您的開發環境中嘗試並在這個 [討論串](https://github.com/vercel/next.js/discussions/73753) 中分享您的想法。

### 概覽[](#overview)

如果您熟悉應用程式路由，您很可能已使用 [`notFound()`](/docs/app/api-reference/file-conventions/not-found) 觸發 404 行為並搭配可自訂的 `not-found.tsx` 檔案。版本 15.1 中，我們將此方法延伸到權限錯誤:

• `forbidden()` 觸發 **403 錯誤** 並搭配可自訂的 UI `forbidden.tsx`。

• `unauthorized()` 觸發 **401 錯誤** 並搭配可自訂的 UI `unauthorized.tsx`。

> **值得注意**：與 `notFound()` 錯誤一樣，如果在初始回應標頭發送後觸發錯誤，狀態碼將為 `200`。[了解更多](/docs/app/building-your-application/routing/loading-ui-and-streaming#status-codes)。

### 啟用功能[](#enabling-the-feature)

由於此功能仍處於實驗階段，您需要在其 `next.config.ts` 檔案中啟用它:

next.config.ts

```
import type { NextConfig } from 'next';
 
const nextConfig: NextConfig = {
  experimental: {
    authInterrupts: true,
  },
};
 
export default nextConfig;
```

> **注意**：`next.config.ts` 支援是在 Next.js 15 中引入的。[了解更多](/docs/app/api-reference/config/next-config-js#typescript)。

### 使用 `forbidden()` 和 `unauthorized()`[](#using-forbidden-and-unauthorized)

您可以在伺服器動作、伺服器元件、用戶端元件或路由處理程式中使用 `forbidden()` 和 `unauthorized()`。這裡有個範例:

```
import { verifySession } from '@/app/lib/dal';
import { forbidden } from 'next/navigation';
 
export default async function AdminPage() {
  const session = await verifySession();
 
  // Check if the user has the 'admin' role
  if (session.role !== 'admin') {
    forbidden();
  }
 
  // Render the admin page for authorized users
  return <h1>Admin Page</h1>;
}
```

### 建立自訂錯誤頁面[](#creating-custom-error-pages)

要自訂錯誤頁面，請建立以下檔案:

app/forbidden.tsx

```
import Link from 'next/link';
 
export default function Forbidden() {
  return (
    <div>
      <h2>Forbidden</h2>
      <p>You are not authorized to access this resource.</p>
      <Link href="/">Return Home</Link>
    </div>
  );
}
```

app/unauthorized.tsx

```
import Link from 'next/link';
 
export default function Unauthorized() {
  return (
    <div>
      <h2>Unauthorized</h2>
      <p>Please log in to access this page.</p>
      <Link href="/login">Go to Login</Link>
    </div>
  );
}
```

我們想感謝 [Clerk](https://clerk.com) 通過 PR 提出此功能並協助我們製作 API 原型。在 15.2 穩定此功能之前，我們計劃向 API 添加更多功能和改進，以支援更廣泛的用例。

閱讀 [`unauthorized`](/docs/app/api-reference/functions/unauthorized) 和 [`forbidden`](/docs/app/api-reference/functions/forbidden) API 的文件以了解更多細節。
