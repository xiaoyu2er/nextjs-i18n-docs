AMP 是什麼？[](#what-is-amp)
----------------------------
AMP 是一種用於構建高性能網站的標準，可最大限度地減少渲染開銷，並能在知名的搜索引擎上以增強功能進行索引。例如，AMP 頁面會直接加載到 Google 的移動端搜索結果中，並標記為閃電圖標。

![Next.js 谷歌搜索結果](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-8-1/next-amp-result.png)
Next.js 谷歌搜索結果

AMP HTML 是 HTML 的一個更嚴格版本，它施加了一些限制以實現更可靠性能和更好的可擴展性。一些 HTML 標籤被替換為 AMP 特定的 HTML 標籤，以提供比其對應的 HTML 標籤更好的體驗。例如，[`amp-img`](https://amp.dev/documentation/components/amp-img) 標籤在支援 `srcset` 的瀏覽器中提供完整的 `srcset` 支援，即便在那些尚未支援的瀏覽器中。

AMP 頁面會被支援它的搜索引擎自動發現。這些搜索引擎通常會實施 [AMP 緩存](https://amp.dev/documentation/guides-and-tutorials/learn/amp-caches-and-cors/how_amp_pages_are_cached)（例如，[Google](https://developers.google.com/amp/cache/) 和 [Bing](https://www.bing.com/webmaster/help/bing-amp-cache-bc1c884c)）。AMP 緩存有助於頁面從其搜索結果中更快地渲染。

Next.js 中的 AMP[](#amp-in-nextjs)
--------------------------------
今天，我們很高興地宣布，我們已將 Next.js 體驗擴展到撰寫 AMP 頁面。這意味著您現在可以利用 React 組件的力量來創建 AMP 頁面。AMP 支援是基於每頁定義的，允許對 AMP 進行增量式採用。有兩種方法可以在 Next.js 中啟用 AMP：混合式 AMP 頁面或優先 AMP 頁面。

混合式 AMP 頁面[](#hybrid-amp-pages)
-------------------------------------
混合式 AMP 頁面允許您為傳統頁面提供配套的 AMP 版本，這樣搜索引擎可以在移動端搜索結果中顯示該頁面的 AMP 版本，同時保留頁面的現有版本。這使得能夠將 AMP 用於您的應用程序，同時仍然利用 Next.js 功能，例如用於主要用戶體驗的客戶端路由。

要選擇加入混合式 AMP 頁面，請使用 `withAmp` 函數並提供 `hybrid: true` 選項：
pages/index.js

```
function HomePage() {
  return <p>Welcome to AMP + Next.js.</p>;
}
 
export default withAmp(HomePage, { hybrid: true });
```

混合式 AMP 模式在生產環境中的一個使用範例是 Reddit。每個線程都存儲在 [Google 的 AMP 緩存](https://developers.google.com/amp/cache/) 中，以在移動端網絡上提供快速的用戶體驗，同時仍為桌面端提供 Reddit 的完整版本和後續導航。

### AMP 優先頁面[](#amp-first-pages)
AMP 優先頁面提供給網站的主要流量以及搜索引擎流量。通過使用 AMP 優先頁面，我們將 AMP 的快速體驗帶到了主網站，包括桌面端。

要實施 AMP 優先頁面，請使用 `withAmp` 函數包裝頁面：

混合式 AMP 模式在生產環境中的一個使用範例是 Reddit。每個線程都存儲在 [Google 的 AMP 緩存](https://developers.google.com/amp/cache/) 中，以在移動端網絡上提供快速的用戶體驗，同時仍為桌面端提供 Reddit 的完整版本和後續導航。

```
import { withAmp } from 'next/amp';
function HomePage() {
  return <p>Welcome to AMP + Next.js.</p>;
}
 
export default withAmp(HomePage);
```

如果您不需要 Next.js 運行時，使用 AMP 優先頁面可以幫助加快開發速度。AMP 優先頁面要求您僅使用與 AMP 兼容的組件來構建頁面。AMP 優先頁面目前已在 [nextjs.org/docs](https://nextjs.org/docs) 和 [nextjs.org/blog](https://nextjs.org/blog) 上投入生產使用。

### 在組件中區分 AMP 渲染[](#distinguishing-amp-rendering-in-components)任何 React 組件在您的項目都可以利用 `useAmp` 來區分 AMP 和非 AMP 渲染模式。這使得您能夠實現一個 `<Image>` 組件，在 `<img>` 和 `<amp-img>` 之間共享邏輯：

components/image.js

```
import { useAmp } from 'next/amp';
 
export function Image({ src, width, height }) {
  const isAmp = useAmp();
  return isAmp ? (
    <amp-img src={src} width={width} height={height} />
  ) : (
    <img src={src} width={width} height={height} />
  );
}
```

pages/index.js

```
import { withAmp } from 'next/amp';
import { Image } from '../components/image';
 
function HomePage() {
  return (
    <>
      <p>Hello! Welcome to AMP + Next.js.</p>
      <Image src="https://placehold.it/120" width="120" height="120" />
    </>
  );
}
 
export default withAmp(HomePage, { hybrid: true });
```

### AMP 開發的自動重新加載[](#automatic-reloading-for-amp-development)在開發模式下，我們不使用熱模塊替換，而是追蹤當前頁面的任何更改，僅在頁面發生變化時重新加載該頁面。我們使用完全重新加載而不是熱模塊替換的原因是為了確保您始終看到 AMP 頁面的最新服務端渲染。

### AMP 驗證[](#amp-validation)為了幫助確保只生成有效的 AMP 頁面，我們在開發過程中與 [`amphtml-validator`](https://www.npmjs.com/package/amphtml-validator) 一同自動驗證它們。錯誤和警告將出現在您啟動 Next.js 的終端中。

頁面在 `next export` 期間也會被驗證，任何問題都會打印到終端中。任何 AMP 錯誤都將導致 `next export` 失敗，因為該輸出不是有效的 AMP。
