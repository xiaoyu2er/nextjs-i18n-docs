### 1.1 建立 Next.js 應用程式 [](#11-create-a-nextjs-app)

如果您是初次開始，可以使用以下指令建立新的 Next.js 專案：
終端機

終端機

```
npx create-next-app@latest --api
```

> **注意：** `--api` 參數會自動在新專案的 `app/` 資料夾中包含範例 `route.ts`，展示如何建立 API 端點。

### 1.2 App Router 與 Pages Router [](#12-app-router-vs-pages-router)

*   **Pages Router**：歷史上，Next.js 使用 `pages/api/*` 處理 API。此方法依賴 Node.js 的 request/response 物件以及類似 Express 的 API。
*   **App Router（預設）**：App Router 於 Next.js 13 引入，完全採用 Web 標準的 Request/Response APIs。不再使用 `pages/api/*`，現在您可以在 `app/` 目錄內的任何位置放置 `route.ts` 或 `route.js` 檔案。

> **為何切換？** App Router 的"Route Handlers"依賴 [Web Platform Request/Response APIs](https://developer.mozilla.org/en-US/docs/Web/API) 而非 Node.js 特定 API。這簡化了學習流程，減少摩擦，並讓您能在不同工具間重複使用知識。

2\. 為何（及何時）使用 Next.js 建構 API [](#2-why-and-when-to-build-apis-with-nextjs)
------------------------------------------------------------------------------------------

1.  **為多個客戶端提供公共 API**
    
    *   您可以建立一個被您的 Next.js 網頁應用程式、獨立的移動應用程式或任何第三方服務使用的公共 API。例如，您可能在 React 網站和 React Native 移動應用程式中都從 `/api/users` 抓取資料。
2.  **對現有後端進行代理**
    
    *   有時您希望將外部 [微服務](https://vercel.com/blog/how-vercel-adopted-microfrontends) 隱藏或整合在單一端點後面。Next.js Route Handlers 可以作為代理或中間層指向另一個現有的後端。例如，您可能攔截請求、處理身份驗證、轉換資料，然後將請求傳遞給上遊 API。
3.  **Webhooks 和整合**
    
    *   如果您接收外部回呼或 webhook（例如來自 Stripe、GitHub、Twilio），您可以使用 Route Handlers 處理它們。
4.  **自訂身份驗證**
    
    *   如果您需要 sessions、tokens 或其他 auth 邏輯，您可以在 Next.js API 層中儲存 cookie，讀取標頭，並回應適當的資料。

> **注意：** 如果您只需要為自身的 Next.js 應用程式進行伺服器端資料抓取（且不需要在外部共享該資料），Server Components 可能足以在渲染期間直接抓取資料，無需獨立的 API 層。

### 3.2 單一檔案中多個 HTTP 方法 [](#32-multiple-http-methods-in-one-file)

與 Pages Router API 路由（具有單一預設匯出）不同，您可以從相同檔案匯出代表不同 HTTP 方法的多個函式。
app/api/users/route.ts

現在，發送 GET 請求到 `/api/users` 將回傳您的使用者清單，而相同的 URL 的 `POST` 請求將插入新使用者。

```
export async function GET(request: Request) {
  // For example, fetch data from your DB here
  const users = [
    { id: 1, name: 'Alice' },
    { id: 2, name: 'Bob' }
  ];
  return new Response(JSON.stringify(users), {
    status: 200,
    headers: { 'Content-Type': 'application/json' }
  });
}
 
export async function POST(request: Request) {
  // Parse the request body
  const body = await request.json();
  const { name } = body;
 
  // e.g. Insert new user into your DB
  const newUser = { id: Date.now(), name };
 
  return new Response(JSON.stringify(newUser), {
    status: 201,
    headers: { 'Content-Type': 'application/json' }
  });
}
```

現在，向 `/api/users`發送 GET 請求會返回列表中的用戶，而向同一 URL 發送 POST 請求將插入一條新記錄。

4\. 使用 Web APIs [](#4-working-with-web-apis)

### 4.1 直接使用 Request & Response [](#41-directly-using-request--response)

預設情況下，您的 Route Handler 方法（`GET`、`POST` 等）接收標準的 [Request](https://developer.mozilla.org/docs/Web/API/Request) 物件，您必須回傳標準的 [Response](https://developer.mozilla.org/docs/Web/API/Response) 物件。

### 4.2 查詢參數 [](#42-query-parameters)

app/api/search/route.ts

```
import { NextRequest } from 'next/server';
 
export function GET(request: NextRequest) {
  const searchParams = request.nextUrl.searchParams;
  const query = searchParams.get('query'); // e.g. `/api/search?query=hello`
 
  return new Response(
    JSON.stringify({ result: `You searched for: ${query}` }),
    {
      headers: { 'Content-Type': 'application/json' },
    },
  );
}
```

### 4.3 標頭和 Cookie [](#43-headers-and-cookies)

app/api/auth/route.ts

```
import { NextRequest } from 'next/server';
import { cookies, headers } from 'next/headers';
 
export async function GET(request: NextRequest) {
  // 1. Using 'next/headers' helpers
  const cookieStore = await cookies();
  const token = cookieStore.get('token');
 
  const headersList = await headers();
  const referer = headersList.get('referer');
 
  // 2. Using the standard Web APIs
  const userAgent = request.headers.get('user-agent');
 
  return new Response(JSON.stringify({ token, referer, userAgent }), {
    headers: { 'Content-Type': 'application/json' },
  });
}
```

如果計劃在 Next.js 中的其他伺服器端程式碼中重複使用共享邏輯，`cookies()` 和 `headers()` 函式會很有幫助。您會注意到 Next.js 也提供了 `NextRequest` 和 `NextResponse`，它們擴充了基礎 Web APIs。

5\. 動態路由 [](#5-dynamic-routes)
---------------------------------------

要建立動態路徑（例如 `/api/users/:id`），請在您的資料夾結構中使用 **Dynamic Segments**：

```
app
└── api
    └── users
        └── [id]
            └── route.ts
```

此檔案對應類似 `/api/users/123` 的 URL，`123` 被捕捉為參數。
app/api/users/[id]/route.ts

此處，`params.id` 提供動態區段。

```
import { NextRequest } from 'next/server';
 
export async function GET(
  request: NextRequest,
  { params }: { params: Promise<{ id: string }> },
) {
  const id = (await params).id;
  // e.g. Query a database for user with ID `id`
  return new Response(JSON.stringify({ id, name: `User ${id}` }), {
    status: 200,
    headers: { 'Content-Type': 'application/json' },
  });
}
 
export async function DELETE(
  request: NextRequest,
  { params }: { params: Promise<{ id: string }> },
) {
  const id = (await params).id;
  // e.g. Delete user with ID `id` in DB
  return new Response(null, { status: 204 });
}
```

此處，`params.id`會提供動態參數部分。

6\. 使用 Next.js 作為代理或轉發層 [](#6-using-nextjs-as-a-proxy-or-forwarding-layer)
--------------------------------------------------------------------------------------------------

常見的情境是 **代理** 現有的後端服務。您可以驗證請求、處理記錄，或在傳送請求到遠端伺服器或後端之前轉換資料：

app/api/external/route.ts

```
import { NextRequest } from 'next/server';
 
export async function GET(request: NextRequest) {
  const response = await fetch('https://example.com/api/data', {
    // Optional: forward some headers, add auth tokens, etc.
    headers: { Authorization: `Bearer ${process.env.API_TOKEN}` },
  });
 
  // Transform or forward the response
  const data = await response.json();
  const transformed = { ...data, source: 'proxied-through-nextjs' };
 
  return new Response(JSON.stringify(transformed), {
    headers: { 'Content-Type': 'application/json' },
  });
}
```

現在您的客戶端只需呼叫 `/api/external`，Next.js 會處理其餘部分。這有時也被稱為"為前端後置的後端" (Backend for Frontend) 或 BFF。

7\. 建立共享的“中間件”邏輯 [](#7-building-shared-middleware-logic)
-----------------------------------------------------------------------------

如果您想在多個 Route Handlers 上套用相同的邏輯（例如身份驗證檢查、記錄），您可以建立封裝您處理器的可重用最函式：

lib/with-auth.ts

```
import { NextRequest } from 'next/server';
 
type Handler = (req: NextRequest, context?: any) => Promise<Response>;
 
export function withAuth(handler: Handler): Handler {
  return async (req, context) => {
    const token = req.cookies.get('token')?.value;
    if (!token) {
      return new Response(JSON.stringify({ error: 'Unauthorized' }), {
        status: 401,
        headers: { 'Content-Type': 'application/json' },
      });
    }
 
    // If authenticated, call the original handler
    return handler(req, context);
  };
}
```

然後在您的 Route Handler 中：

app/api/secret/route.ts

```
import { NextRequest } from 'next/server';
import { withAuth } from '@/lib/with-auth';
 
async function secretGET(request: NextRequest) {
  return new Response(JSON.stringify({ secret: 'Here be dragons' }), {
    headers: { 'Content-Type': 'application/json' },
  });
}
 
export const GET = withAuth(secretGET);
```

### 8.2 SPA/靜態匯出 [](#82-spastatic-export)

Next.js 也支援將整個網站輸出為 [靜態 Single-Page Application (SPA)](/docs/app/building-your-application/upgrading/single-page-applications)。
您可以設定以下內容來啟用此功能：

您可以透過設定以下內容來啟用此功能：

next.config.ts

```
import type { NextConfig } from 'next';
 
const nextConfig: NextConfig = {
  output: 'export',
};
 
export default nextConfig;
```

在 **靜態匯出模式** 下，Next.js 將生成純靜態 HTML、CSS 和 JS。**您無法執行伺服器端程式碼**（如 API 端點）。如果您仍需要 API，則必須單獨_host_（例如，獨立的 Node.js 伺服器）。

> **注意：**
>
> *   **GET Route Handlers** [可以靜態匯出](/docs/app/building-your-application/deploying/static-exports#route-handlers) 如果它們不依賴動態請求資料。它們會變成您 `out` 資料夾中的靜態檔案。
> *   **所有其他伺服器功能**（動態請求、重寫 cookie 等）在純 SPA 匯出中**不支援**。

9\. 何時應跳過建立 API 端點 [](#9-when-to-skip-creating-an-api-endpoint)
-------------------------------------------------------------------------------------

隨著 App Router 的 **React 伺服器組件**，您可以在伺服器上直接抓取資料而無需暴露公共端點：

app/users/page.tsx

```
// (Server Component)
export default async function UsersPage() {
  // This fetch runs on the server (no client-side code needed here)
  const res = await fetch('https://api.example.com/users');
  const data = await res.json();
 
  return (
    <main>
      <h1>Users</h1>
      <ul>
        {data.map((user: any) => (
          <li key={user.id}>{user.name}</li>
        ))}
      </ul>
    </main>
  );
}
```

如果您的資料僅用於您的 Next.js 應用程式內部，您可能根本不需要公共 API。

10\. 整合所有內容 [](#10-putting-it-all-together)
-----------------------------------------------------------

1.  **建立新的 Next.js 專案**： `npx create-next-app@latest --api`。
2.  **在 `app/` 目錄內增添 Route Handlers**（例如 `app/api/users/route.ts`）。
3.  **匯出相同的檔案中的 HTTP 方法**（`GET`、`POST`、`PUT`、`DELETE` 等）。
4.  **使用 Web Standard APIs** 來與 `Request` 物件互動並回傳 `Response`。
5.  **建立公共 API** 如果您需要其他客戶端使用您的資料，或要代理後端服務。
6.  **從客戶端抓取** 您的新 API 路由（例如，在 Client Component 內或使用 `fetch('/api/...')`）。
7.  或者如果 Server Component 只要能抓取資料就 **跳過建立 API**。
8.  **增添共享的“中間件”模式**（例如 `withAuth()`）用於身份驗證或其他重複邏輯。
9.  **佈署** 到具有 Node.js 功能的环境以使用伺服器功能，或者如果您只需要靜態 SPA 則 **靜態匯出**。

結論 [](#conclusion)
-------------------------

使用 Next.js 的 **App Router** 和 **Route Handlers** 為您提供靈活、現代的 API 建構方式，直接採用 **Web Platform**。您可以：

*   **建立完整的公共 API** 供網路、移動設備或第三方客戶端分享。
*   **代理** 和自訂對現有外部服務的呼叫。
*   **實作** 可重用的"中間件"層，用於身份驗證、記錄或任何重複邏輯。
*   **動態路由** 請求使用 `[id]` 區段資料夾結構。

### 伺服器操作的情況呢？ [](#what-about-server-actions)

您可以將 [伺服器操作](/docs/app/building-your-application/data-fetching/server-actions-and-mutations) 視為可從客戶端呼叫的自動生成 `POST` API 路由。
它們專為變更操作設計，例如建立、更新或刪除資料。您像調用正常 JavaScript 函式一樣調用 Server Action，而非對定義的 API 路由進行顯式 `fetch`。
雖然 _仍有_ 網路請求正在發生，但不需要明確管理它。URL 路徑為自動生成並 [加密](/docs/app/building-your-application/data-fetching/server-actions-and-mutations#security)，因此您無法在瀏覽器中手動存取類似 `/api/users` 的路由。
如果您計劃使用伺服器操作 _並_ 暴露公共 API，我們建議將核心邏輯移至 [Data Access Layer](/blog/security-nextjs-server-components-actions)，並從 Server Action 和 API 路由調用相同的邏輯。

它們專為數據變異操作（例如新增、更新或刪除資料）設計。呼叫伺服器操作 (Server Action) 就像呼叫普通的 JavaScript 函數，而非向定義的 API 路由發出明確的 `fetch`。

雖然確實仍有網路請求進行中，但你無需明確管理它。URL 路徑是自動生成且 [加密](/docs/app/building-your-application/data-fetching/server-actions-and-mutations#security) 的，因此你無法在瀏覽器中手動存取像 `/api/users` 這樣的路由。

如果您計劃使用伺服器操作 (Server Actions) 且同時公開 API，我們建議將核心邏輯移至 [數據存取層 (Data Access Layer)](/blog/security-nextjs-server-components-actions) 並從 Server Action 和 API 路由同時呼叫相同的邏輯。
