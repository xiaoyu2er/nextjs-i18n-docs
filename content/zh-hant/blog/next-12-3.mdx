本次 Next.js 12.3 版本的發行包含了一些生活品質改進：

*   [**改進的 Fast Refresh：**](#improved-fast-refresh) `.env`, `jsconfig.json` 和 `tsconfig.json` 文件現在支持熱重載。
*   [**TypeScript Auto-Install：**](#typescript-auto-install) 添加 `.ts` 文件即可自動配置 TypeScript 並安裝相依性。
*   [**Image Component：**](#image-component) `next/future/image` 現在已穩定。
*   [**SWC Minifier：**](#swc-minifier-stable) 使用 Next.js 編譯器的 Minification 功能現在已穩定。
*   [**新 Router + Layouts Update：**](#new-router-and-layouts-update) 實現工作已開始，並已解釋新功能。

立即執行 `npm i next@latest` 進行更新。

TypeScript Auto-Install[](#typescript-auto-install)
---------------------------------------------------

Next.js 內建自動配置 TypeScript 的支援。過去，在現有專案中新增 TypeScript 需要手動安裝必要的相依性。

隨著本次發行，Next.js 會自動偵測專案中新增的 TypeScript 檔案，安裝必要的相依性，並新增 `tsconfig.json` 檔案。這在執行 `next dev` 和 `next build` 時運作。

影片：顯示 Next.js 自動安裝 TypeScript。

改進的 Fast Refresh[](#improved-fast-refresh)
-----------------------------------------------

快速遞迴對您的本地開發工作流程至關重要。Next.js 使用 [Fast Refresh](/docs/architecture/fast-refresh) 提供對 React 元件修變更的新穎反饋。按儲存，即可在瀏覽器中看到您的變更*_不需要*_重新載入頁面。

隨著 Next.js 12.3，一些設定檔在進行編輯時現在也會熱重載，包括：

*   `.env` 和 `env.*` 變體
*   `jsconfig.json`
*   `tsconfig.json`

現在，針對這些設定檔進行變更時，您*_不需要*_重新啟動本地開發伺服器。

影片：顯示 Next.js 熱重載 `\.env\` 文件。

Image Component[](#image-component)
-----------------------------------

### `next/future/image` 元件 (穩定)[](#nextfutureimage-component-stable)

在 Next.js 12.2 中，我們分享了一個**新**Image 元件的實驗性預覽，簡化了您的圖片樣式設定方式，並透過使用原生瀏覽器懶載入來提升效能。

新的 Image 元件現在已穩定，不再需要實驗性標記。

自上次發行以來，我們進行了進一步的改進：

*   新增支援可讓圖片填滿父元素 (`fill`) 佈局模式。
*   改進的模糊佔位圖像現在可與透明圖片搭配使用，不再需要 `<noscript>`，且 `next dev` 與 `next start` 一致。
*   在圖片載入時隱藏 `alt` 文字，以防止其他 Chrome 瀏覽器以外的輕微版面位移。
*   為提升無障礙性，要求 `alt` 屬性。
*   修復了在還原前發生錯誤時的 `onError` 屬性。
*   修復了僅有 `width` 或 `height` 屬性的靜態匯入圖片之分徑。
*   改善了使用不正確設定或配置時的錯誤訊息。

在未來，我們計劃將 `next/image` 更名為 `next/legacy/image`，並將 `next/future/image` 更名為 `next/image`。這將為新的 Next.js 專案提供良好的體驗，並允許現有專案使用我們的 `next/image` codemod 輕鬆遷移。我們將在下個主要版本到來前提供更多詳情。

若要瞭解如何使用它，請參閱 [文件](/docs/pages/building-your-application/optimizing/images)，其中包括遷移指南。

### Remote Patterns (穩定)[](#remote-patterns-stable)

`remotePatterns` 允許您在執行內建 Image Optimization API 時指定遠端圖片的通配符。它允許比現有的 `images.domains` 配置進行更強勁的比對，後者僅對網域名稱進行精確比對。

此選項現在已穩定，並可配合以下配置選項使用：

next.config.js

```
module.exports = {
  images: {
    remotePatterns: [
      {
        // The `src` property hostname must end with `.example.com`,
        // otherwise the API will respond with 400 Bad Request.
        protocol: 'https',
        hostname: '**.example.com',
      },
    ],
  },
};
```

### 停用 Image Optimization (穩定)[](#disable-image-optimization-stable)

Image Optimization API 需要伺服器根據需求即時優化圖片。這在使用 `next start` 自行託管或部署到 [Vercel](https://vercel.com/solutions/nextjs) 等平台時運作良好。但是，這不會與 `next export` 配合運作，因為它不包含用於優化圖片的伺服器。過去，您需要使用 `loader` 屬性設定第 3 方 Image Optimization 提供者，但現在您可以使用 `next.config.js` 完全停用 Image Optimization，適用於所有 `next/image` 實體。

此之前實驗性的選項現在已穩定，並可配合以下配置選項使用：

next.config.js

```
module.exports = {
  images: {
    unoptimized: true,
  },
};
```

SWC Minifier (穩定)[](#swc-minifier-stable)
---------------------------------------------

在 Next.js 12 中，我們引入了使用 SWC 的程式碼 Minification，作為 [Next.js Compiler](/docs/architecture/nextjs-compiler) 實驗性功能的一部分。早期結果顯示，使用 SWC 進行 Minification 比先前使用 Terser 的版本快 **7 倍**。

隨著本次發行，此功能已穩定，您可以使用以下 `next.config.js` 選項參予：

next.config.js

```
module.exports = {
  swcMinify: true,
};
```

我們計劃在下個主要版本中將 SWC minifier 設為預設值。

新 Router 和 Layouts Update[](#new-router-and-layouts-update)
---------------------------------------------------------------

今年早些時候，我們發布了 [Layouts RFC](https://nextjs.org/blog/layouts-rfc)，引入了自 2016 年推出以來 Next.js 的最大更新。這些變更包括建立在 React Server Components 之上的新 Router。實作工作已開始，我們正朝着在下個主要版本中公開 Beta 邁進。

今天，我們分享更多關於即將推出功能的資訊，包括：

*   [Route Groups](https://nextjs.org/blog/layouts-rfc#route-groups): 用於讓路由退出佈局，並在不影响 URL 路徑的情況下組織路由。
*   [Instant Loading States](https://nextjs.org/blog/layouts-rfc#instant-loading-states): 新增檔案慣例，可使用 Suspense 輕鬆新增載入 UI。
*   [Error Handling](https://nextjs.org/blog/layouts-rfc#error-handling): 新增檔案慣例，用於輕鬆新增巢狀 Error Boundaries 和錯誤 UI。
*   [Templates](https://nextjs.org/blog/layouts-rfc#templates): 新增檔案慣例，用於新增共用 UI，這會建立不同的執行緒且不共用狀態。
*   [Intercepting Routes](https://nextjs.org/blog/layouts-rfc#intercepting-routes), [Parallel Routes](https://nextjs.org/blog/layouts-rfc#dynamic-parallel-routes), 和 [Conditional Routes](https://nextjs.org/blog/layouts-rfc#conditional-routes): 允許您實作進階路由模式的新慣例。

此外，在 [不同的 RFC](https://github.com/vercel/next.js/discussions/27953) 中，我們正在探索在新增的 `app` 目錄中新增 **Global CSS Imports** 的支援。這將允許您使用匯入其自己的 `.css` 檔案的套件，無需額外設定。

請在 [Layouts RFC](https://nextjs.org/blog/layouts-rfc) 中閱讀更多關於這些新功能的資訊。

[其他改進](#other-improvements)
-----------------------------------------

*   [新文件](/docs/accessibility) 關於 Next.js 的內建無障礙功能。
*   `create-next-app` 模板現在自動使用您的系統偏好設定包含深色模式支援。
*   `src/` 現在是執行 [內建 ESLint 支援](/docs/pages/building-your-application/configuring/eslint) 時的預設 linting 目錄。
*   `next.config.js` 現在包含使用 `ajv` 的 Schema 驗證，幫助防止配置選項的誤用。
*   `next.config.js` 現在有一個實驗性配置旗標 `fallbackNodePolyfills: false`，以防止 Next.js 為瀏覽器 Polyfilling Node.js API（這會導致封包大小增加）。此選項將導致 Next.js 顯示建置時間錯誤，顯示不必要匯入來自何處。
