我們很興奮地發布 Next.js 12.1 中最受請求的功能之一：

*   [**按需 ISR (測試版):**](#on-demand-incremental-static-regeneration-beta) 使用 `getStaticProps` 即時重新驗證頁面。
*   [**擴展 SWC 支持:**](#improved-swc-support) `styled-components`、Relay 等更多支持。
*   [**`next/jest` 外掛:**](#zero-configuration-jest-plugin) 使用 SWC 的零配置 Jest 支持。
*   [**使用 SWC 更快進行代碼壓縮 (RC):**](#faster-minification-with-swc) 比 Terser 快 7 倍的壓縮速度。
*   [**自託管 Next.js 改進:**](#self-hosted-nextjs-improvements) Docker 鏡像小了約 80%。
*   [**React 18 與 Server Components (Alpha):**](#react-18-server-components-and-ssr-streaming-alpha) 穩定性和支持改進。

透過執行 `npm i next@latest` 立即更新。

按需增量靜態再生 (Beta)[](#on-demand-incremental-static-regeneration-beta)
---------------------------------------------------------------------------------------------------

Next.js 現在公開了一個函數 `unstable_revalidate()`，允許您重新驗證使用 `getStaticProps` 的單獨頁面。自我們在 Next.js 9.5 中引入 [增量靜態再生](/docs/pages/building-your-application/data-fetching/incremental-static-regeneration) (ISR) 以來，這是一個最受請求的功能之一。

自 ISR 發布以來，我們看到像 [Tripadvisor](https://vercel.com/customers/cruise-critic)、[Parachute](https://vercel.com/customers/parachute)、HashiCorp 等公司大幅縮短了構建時間，同時保持了出色的性能。然而，我們聽取了大家對基於時間間隔的重驗證的反饋，很高興能提供更多靈活性。

目前，如果您將 `revalidate` 時間設置為 `60`，所有訪問者都將在一分鐘內看到您網站生成的同一版本。在該分鐘過去之後，有人訪問該頁面是使緩存失效的唯一方法。您現在可以為特定頁面按需手動清除 Next.js 緩存。

這讓您在以下情況更新網站變得更容易：

*   來自您無頭 CMS 的內容已創建或更新
*   商務元數據更改 (價格、描述、類別、評論等)

pages/api/revalidate.js

```
export default async function handler(req, res) {
  // Check for secret to confirm this is a valid request
  if (req.query.secret !== process.env.MY_SECRET_TOKEN) {
    return res.status(401).json({ message: 'Invalid token' });
  }
 
  try {
    await res.unstable_revalidate('/path-to-revalidate');
    return res.json({ revalidated: true });
  } catch (err) {
    // If there was an error, Next.js will continue
    // to show the last successfully generated page
    return res.status(500).send('Error revalidating');
  }
}
```

在 `getStaticProps` 內部，您無需指定 `revalidate` 即可使用按需重驗證。如果省略 `revalidate`，Next.js 將使用默認值 `false` (不重驗證)，僅在調用 `unstable_revalidate()` 時按需重新驗證頁面。

增量靜態再生適用於任何支持 [Next.js 構建 API](/docs/pages/building-your-application/deploying#nextjs-build-api) (`next build`) 的提供者。從今天開始，部署到 Vercel 時，當將頁面推送到邊緣時，按需重驗證將在全球範圍內傳播，時間約為 300ms。

[查看我們的演示](https://on-demand-isr.vercel.app/) 以查看按需重驗證的實際效果並提供反饋。Svix (企業級 webhooks) 和 Clerk (身份驗證) 也 [創建了一個演示](https://www.svix.com/blog/vercel-on-demand-isr-and-svix) 的按需 ISR。

SWC 支持改進[](#improved-swc-support)
---------------------------------------------

我們希望讓每個 Next.js 應用在生產構建時更快，並在本地開發時獲得即時反饋。[Next.js 12](https://nextjs.org/blog/next-12) 引入了一個新的 [基於 Rust 的編譯器](/docs/architecture/nextjs-compiler)，利用了原生編譯的優勢。

在 12.1 中，我們已為 Next.js 編譯器添加了以下支持：

*   [styled-components](/docs/architecture/nextjs-compiler#styled-components)
*   [importSource](/docs/architecture/nextjs-compiler#importsource)
*   [legacy-decorators](/docs/architecture/nextjs-compiler#legacy-decorators)
*   [relay](/docs/architecture/nextjs-compiler#relay)
*   [remove-react-properties](/docs/architecture/nextjs-compiler#remove-react-properties)
*   [remove-console](/docs/architecture/nextjs-compiler#remove-console)

添加了上述六個轉換後，我們現在已使用新編譯器將最常用的 Babel 插件移植到 Rust。如果您希望看到其他插件，請在 [討論區告訴我們](https://github.com/vercel/next.js/discussions/30174)。此外，我們正在為通過 SWC 啟用的 [高性能 WebAssembly 插件](https://twitter.com/swc_rs/status/1492454606118752257) 構建系統。

零配置 Jest 外掛[](#zero-configuration-jest-plugin)
-----------------------------------------------------------------

Next.js 現在使用基於 Rust 的 Next.js 編譯器自動為您配置 Jest (`next/jest`) 以轉換文件，包括：

*   自動模擬樣式表 (`.css`、`.module.css` 及其 `.scss` 變體) 和圖片導入
*   將 `.env` (及其所有變體) 加載到 `process.env` 中
*   忽略測試解析和轉換中的 `node_modules`
*   忽略測試解析中的 `.next`
*   加載 `next.config.js` 以啟用 Next.js 編譯器轉換的標誌

[查看文檔](/docs/pages/building-your-application/optimizing/testing#setting-up-jest-with-the-rust-compiler) 了解更多，或 [從我們的 Jest 示例開始](https://github.com/vercel/next.js/tree/canary/examples/with-jest)。

使用 SWC 進行更快的代碼壓縮[](#faster-minification-with-swc)
-------------------------------------------------------------

在 [Next.js 12](https://nextjs.org/blog/next-12) 中，我們引入了作為 Next.js 編譯器一部分的 SWC 壓縮功能。早期結果顯示其比 Terser 快 7 倍。SWC 壓縮現在已隨 12.1 進入 Release Candidate (RC) 並將在 12.2 中成為預設值。

您可以在 `next.config.js` 中選擇使用 SWC 進行壓縮：

next.config.js

```
module.exports = {
  swcMinify: true,
};
```

請在 [討論區](https://github.com/vercel/next.js/discussions/30237) 告訴我們您的反饋。

更快的圖片優化[](#faster-image-optimization)
-------------------------------------------------------

內置的圖片優化 API 已更新以支持與 [ISR 頁面](/docs/pages/building-your-application/data-fetching/incremental-static-regeneration) 相同的模式，即圖片以過時狀態提供，並在後台 _重新驗證_ (也稱為 `stale-while-revalidate`)。

這對優化圖片而言是一次重大的性能提升。請參閱 [圖片緩存行為](/docs/pages/api-reference/components/image#caching-behavior) 了解更多。

自託管 Next.js 改進[](#self-hosted-nextjs-improvements)
--------------------------------------------------------------------

Next.js 現在可以自動創建一個 `standalone` 文件夾，僅複製生產部署所需的文件。這使得自託管 Next.js 應用的 **Docker 鏡像小了 80%**。

若要利用此自動複製功能，您可以在 `next.config.js` 中啟用它：

next.config.js

```
module.exports = {
  experimental: {
    outputStandalone: true,
  },
};
```

這會建立一個位於 `.next/standalone` 的資料夾，然後即可單獨部署，無需安裝 `node_modules`。

[查看文檔](/docs/pages/api-reference/next-config-js/output#automatically-copying-traced-files-experimental) 或從 [Docker 示例](https://github.com/vercel/next.js/tree/canary/examples/with-docker) 開始。我們現在還擁有一個支持根據環境加載不同 `.env` 文件的 [多環境 Docker 示例](https://github.com/vercel/next.js/tree/canary/examples/with-docker-multi-env)。

React 18、Server Components 與 SSR Streaming (Alpha)[](#react-18-server-components-and-ssr-streaming-alpha)
-------------------------------------------------------------------------------------------------------------

正如在 [Next.js Conf](https://www.youtube.com/watch?v=WlP2TB2ORL4) 上所示，我們一直在努力將 React 18、服務端懸懸 (Server-Side Suspense)、流式傳輸 SSR (Streaming SSR) 以及最終的 Server Components 帶入 Next.js。

Server Components 是 React 中的一個新功能，允許您通過分離服務端和客戶端代碼來減少 JavaScript 包大小。Server Components 允許開發人員構建跨越服務端和客戶端的應用，將客戶端應用的豐富交互性與傳統服務端渲染的改進性能相結合。

對於在 Next.js 中跟隨 [Server Components](https://github.com/vercel/next.js/discussions/31263) 從 alpha 過渡到 beta 的開發者，以下是一些最近的更新：

*   [HTML 中內嵌 Server Component 響應以避免二次請求](https://github.com/vercel/next.js/issues/30994)
*   [優化流式傳輸時的腳本加載，以配合 React 18 實現部分水合](https://github.com/vercel/next.js/issues/31338)
*   [支持 React 18 中的新 `useId` API](https://github.com/vercel/next.js/pull/31102)
*   [支持 `_app` 作為 Server Component](https://github.com/vercel/next.js/issues/30996)
*   [優化 Server Components 的代碼分割](https://github.com/vercel/next.js/pull/31968)
*   [優化流式傳輸性能](https://github.com/vercel/next.js/pull/30585)
*   [能夠在 Node.js 和 Edge 之間更改默認運行時](https://github.com/vercel/next.js/pull/34068) ([RFC](https://github.com/vercel/next.js/discussions/34179))

我們正努力讓您能夠在邊緣環境下運行整個 Next.js 應用程序的服務端渲染。您可以選擇哪些頁面應選擇加入 Edge Runtime，以便您準備好在需要時逐步將頁面從 Node.js 遷移。

我們很快將發布另一篇更深入探討 Edge Runtime 的文章。我們的兩個演示 ([next-server-components](https://github.com/vercel/next-server-components) 和 [next-rsc-demo](https://github.com/vercel/next-rsc-demo)) 都已更新為最新變化。

其他修復和改進[](#other-bug-fixes-and-improvements)
---------------------------------------------------------------------

*   使用帶有 `target="blank"` 的 `<a>` 元素不再通過 ESLint 警告使用 `next/link`。
*   `.d.ts` 文件不再被視為頁面。
*   在螢幕閱讀器宣佈頁面更改時，`document.title` 現在優先於 `h1`。
*   創建 `pages/index/[...dynamic].js` 現在可以工作了，之前由於 `index` 是保留關鍵字而無法實現。
*   當使用 `dynamic(() => import('./some-component'), { ssr: false })` 時，引入代碼將從服務端代碼中自動樹搖 (treeshaken)。
*   為了創建更小的安裝大小並提高安全性，我們正在努力預編譯更多依賴項。最新的添加是 `node-fetch`。
*   使用 [Middleware](/docs/pages/building-your-application/routing/middleware) 時的 Fast Refresh 改進。
*   通過我們的 [內建 ESLint 集成](/docs/pages/building-your-application/configuring/eslint) 支持 ESLint 8。
*   `styled-jsx` 已升級到 5.0，包括所有 `styled-jsx` 編譯錯誤的新幫助錯誤鏈接。
*   Edge Runtime：支持 `AbortController` 和 `AbortSignal`
*   Edge Runtime：添加了 `CryptoKey` 和 `globalThis.CryptoKey`。
*   大型 Next.js 應用程序最近看到 Fast Refresh 時間有 ~60% 的改進 (超過 1,000 個模塊被重新加載)。
*   如果 Fast Refresh 失敗並導致整頁重新加載，現在會顯示通知。
*   當使用 React 18 strict mode 時，宣佈的路由現在正確跳過宣佈初始頁面加載。
*   與 2021 年 12 月相比，我們在 Next.js 存儲庫上將開放問題數量降低了 [近 300 個](https://nextjs-issue-tracker.vercel.app/) (解決了近 1000 個問題)。
