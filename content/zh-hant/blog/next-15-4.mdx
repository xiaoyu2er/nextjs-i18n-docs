Next.js 15.4 包含效能、穩定性和 Turbopack 相容性的更新。此次發布的亮點包括：

*   [**Turbopack 建置**](/blog/next-15-4#turbopack-builds): `next build --turbopack` 的 100% 整合測試相容性
*   [**穩定性改進**](/blog/next-15-4#notable-changes): Next.js 和 Turbopack 的眾多穩定性和效能改進

這篇部落格文章還包括 Next.js 16（我們的下一次主要版本）的早期預覽。

立即升級，或開始使用：

終端機

```
# Use the automated upgrade CLI
npx @next/codemod@canary upgrade latest
 
# ...or upgrade manually
npm install next@latest react@latest react-dom@latest
 
# ...or start a new project
npx create-next-app@latest
```

Turbopack 建置 [](#turbopack-builds)
-------------------------------------

`next build --turbopack` 現在已通過 [**所有 8298 項整合測試**](https://areweturboyet.com/)，並用於生產建置，且已驅動 `vercel.com`。這是邁向標記 Turbopack 在 Next.js 中為穩定版的重要里程碑。

團隊目前的優先事項是完成透過生產環境分塊 (chunking) 的打包最佳化工作，並修復 Alpha 版本早期採用者報告的問題。我們也在內層為 Turbopack 進行了眾多穩定性和效能改進。

我們有信心 Turbopack 很快就能準備好用於生產環境，並且我們正朝向 Next.js 16 中的 Beta 版本努力發行之餘。如果您之前嘗試過 Turbopack，現在是再次嘗試的好時機。如果您遇到任何問題，請在 [GitHub Issues](https://github.com/vercel/next.js/issues) 上告知我們。

展望未来：Next.js 16 [](#looking-ahead-nextjs-16)
-----------------------------------------------------

過去幾個月中，我們一直在迭代 Next.js 的重大改進，重點關注透過 Turbopack 的開發者體驗以及 App Router 結合 `PPR` 和 `use cache` 的核心功能。雖然我們尚未準備好預設啟用這些功能，但我們正著手為用戶在今年夏天的 Next.js 16 中嘗試這些功能奠定基礎。

以下是 Next.js 16 即將推出的功能預覽：

*   **快取元件 (beta)**：將實驗性緩存功能（Dynamic IO、`use cache` 和 Partial Prerendering）整合到統一的 `cacheComponents` 旗標中，簡化效能最佳化策略。Next.js 16 的部落格文章將詳細說明我們的願景。
*   **Turbopack 建置 (beta)**：引入 `next build --turbopack`，完全通過整合測試，並在 vercel.com 等高流量網站內部驗證。準備在 Next.js 16 中擴大公開 Beta 使用。
*   **最佳化客戶端路由**：透過更聰明的預載 (prefetching)、改進的快取無效策略和減少頻寬使用，提升 App Router 導航，帶來更快、更具響應性的導航體驗。
*   **DevTools & 除錯**：新增路由資訊以檢查應用程式結構和切換元件如 `loading.tsx`。新增實驗性瀏覽器日誌轉發至終端，以支援 AI 驅動的除錯工作流程。
*   **Node.js 中間件 (穩定)**：將先前實驗性的 Middleware Node.js 執行階段支援（最初在 Next.js 15.2 中引入）提升為穩定狀態。
*   **部署配接器 (Alpha)**：讓開發者能夠建立自訂部署配接器以對建置和部署目標進行更細微的控制。
*   **次要的棄用和變更**：棄用對 Node.js 18 的支援、AMP，並變更部分 `next/image` API 以及相關的遷移指南。

我們的目標是盡可能使此次更新流暢且直觀。我們正小心地管理這些變更以減少干擾，並將提供明確的指導和工具以支援您遷移至 Next.js 16。

預覽即將推出的功能 [](#preview-upcoming-features)
-------------------------------------------------------

您可以從今天開始透過使用 `canary` 通道，並在 `next.config.js` 中啟用以下旗標，嘗試這些功能。

next.config.ts

```
import type { NextConfig } from 'next';
 
const nextConfig: NextConfig = {
  experimental: {
    // Forward browser logs to the terminal for easier debugging
    browserDebugInfoInTerminal: true,
 
    // Enable new caching and pre-rendering behavior
    dynamicIO: true, // will be renamed to cacheComponents in Next.js 16
 
    // Activate new client-side router improvements
    clientSegmentCache: true,
 
    // Explore route composition and segment overrides via DevTools
    devtoolSegmentExplorer: true,
 
    // Enable support for `global-not-found`, which allows you to more easily define a global 404 page.
    globalNotFound: true,
 
    // Enable persistent caching for the turbopack dev server and build.
    turbopackPersistentCaching: true,
  },
};
 
export default nextConfig;
```
