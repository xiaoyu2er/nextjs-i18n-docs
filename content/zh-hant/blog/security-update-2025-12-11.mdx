在 React 伺服器元件 (React Server Components, RSC) 通訊協定中已識別出兩項額外脆弱性。這些問題是在安全研究人員檢查 [React2Shell](/blog/CVE-2025-66478) 的修補程式時發現的。**重要的是，這兩項新問題都不允許遠端程式碼執行 (RCE)**。React2Shell 的修補程式仍然完全有效。

這些脆弱性源自上游 React 實作 ([CVE-2025-55183](https://www.cve.org/CVERecord?id=CVE-2025-55183), [CVE-2025-55184](https://www.cve.org/CVERecord?id=CVE-2025-55184))。本安全公告追蹤使用 App Router 的 Next.js 應用程式的下游影響。詳細訊息請參閱 [React 部落格文章](https://react.dev/blog/2025/12/11/denial-of-service-and-source-code-exposure-in-react-server-components)。

> **附錄：** CVE-2025-55184 的初始修補程式不完整。完整的修補程式已透過 [CVE-2025-67779](https://www.cve.org/CVERecord?id=CVE-2025-67779) 發布。如果您先前已升級至其中一個最初推薦的版本，請再次升級至下列列出的最新修補版本。

影響範圍 [](#impact)
-----------------

### 服務妨礙 (DoS): [CVE-2025-55184](https://www.cve.org/CVERecord?id=CVE-2025-55184) (高嚴重性) [](#denial-of-service-cve-2025-55184-high-severity)

可以傳送至任何 App Router 端點的特定構造 HTTP 請求，在反序列化時會導致無限迴圈，造成伺服器工作進程停擺，並阻止後續 HTTP 請求得到服務。

> **注意：** 此脆弱性的初始修補程式不完整。完整的修補程式已透過 [CVE-2025-67779](https://www.cve.org/CVERecord?id=CVE-2025-67779) 發布。先前已升級的使用者必須重新升級至最新的修補版本。

### 原始碼洩漏：[CVE-2025-55183](https://www.cve.org/CVERecord?id=CVE-2025-55183) (中嚴重性) [](#source-code-exposure-cve-2025-55183-medium-severity)

特定構造的 HTTP 請求會導致伺服器函式 (Server Function) 回傳應用程式中其他 [伺服器函式](https://react.dev/reference/rsc/server-functions) 的已編譯原始碼。這可能會洩露商業邏輯。若密鑰 (Secrets) 直接在程式碼中定義（而非在執行階段透過環境變數存取）並在伺服器函式中引用，這些密鑰也可能被洩漏。取決於您的打包工具 (Bundler) 設定，這些值可能會被內嵌到已編譯的函式輸出中。

受影響及已修復的 Next.js 版本 [](#affected-and-fixed-nextjs-versions)
--------------------------------------------------------------------------

使用 App Router 搭配 React 伺服器元件 (React Server Components) 的應用程式受影響。下表顯示各脆弱性影響哪些版本及其對應的修補方案：

| 版本 | DoS (CVE-2025-55184) | 原始碼洩漏 (CVE-2025-55183) | 修復版本 |
| --- | --- | --- | --- |
| >= 13.3 | ✓ | — | 升級至 14.2.35 |
| 14.x | ✓ | — | 14.2.35 |
| 15.0.x | ✓ | ✓ | 15.0.7 |
| 15.1.x | ✓ | ✓ | 15.1.11 |
| 15.2.x | ✓ | ✓ | 15.2.8 |
| 15.3.x | ✓ | ✓ | 15.3.8 |
| 15.4.x | ✓ | ✓ | 15.4.10 |
| 15.5.x | ✓ | ✓ | 15.5.9 |
| 15.x canary | ✓ | ✓ | 15.6.0-canary.60 |
| 16.0.x | ✓ | ✓ | 16.0.10 |
| 16.x canary | ✓ | ✓ | 16.1.0-canary.19 |

Pages Router 應用程式不受影響，但我們仍建議升級至已修補版本。

必要行動 [](#required-action)
-----------------------------------

所有使用者應升級至其版本線路中的最新修補版本：

如果您使用的是 Next.js >= 13.3、14.0.x 或 14.1.x，請升級至最新的 14.2.x 版本發行版。

終端機

```
npm install next@14.2.35  # for 14.x
npm install next@15.0.7   # for 15.0.x
npm install next@15.1.11  # for 15.1.x
npm install next@15.2.8   # for 15.2.x
npm install next@15.3.8   # for 15.3.x
npm install next@15.4.10  # for 15.4.x
npm install next@15.5.9   # for 15.5.x
npm install next@16.0.10  # for 16.0.x
 
npm install next@15.6.0-canary.60   # for 15.x canary releases
npm install next@16.1.0-canary.19   # for 16.x canary releases
```

執行 `npx fix-react2shell-next` 以啟動互動式工具，可檢查版本並依據上述推薦版本執行確定性的版本更新。完整詳細資訊請參閱 [GitHub 儲存庫](https://github.com/vercel-labs/fix-react2shell-next)。

終端機

```
npx fix-react2shell-next
```

沒有替代方案。必須升級至修補版本。
