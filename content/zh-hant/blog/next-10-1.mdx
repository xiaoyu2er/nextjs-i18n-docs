我們興奮地介紹 Next.js 10.1，其中包含:

*   **[3 倍更快的快速重新整理](#3x-faster-refresh)**: 每次編輯更新速度快 200ms，且無需進行任何變更。
*   **[改進的安裝時間](#improved-installation-time)**: 安裝尺寸小了 58%，依賴套件少了 56%。
*   **[`next/image` 改進](#nextimage-improvements)**: 支援 Apple Silicon (M1)，以及其他版面配置和載入器選項。
*   **[Next.js Commerce Shopify 整合](#nextjs-commerce-shopify-integration)**: 用於組合式電子商務應用程式的靈敏資料層。
*   **[自訂 500 頁面](#custom-500-page)**: 在錯誤頁面中加入您自己的徽標和品牌標識。
*   **[嚴格 PostCSS 配置載入](#strict-postcss-configuration-loading)**: 搭配 Webpack 5 改進的快取。
*   **[支援 `tsconfig.json` 中的 `extends`](#support-for-extends-in-tsconfigjson)**: 適用於大型 TypeScript 應用程式的延伸配置。

*   **[文件改進](#documentation-improvements)**: 逐步採用、遷移和 Docker 部署。

3 倍更快的快速重新整理 [](#3x-faster-refresh)
---------------------------------------

自 [v9.4](https://nextjs.org/blog/next-9-4) 開始，Next.js 將 Facebook 的 [Fast Refresh](/docs/architecture/fast-refresh) 開放原始碼化，以便對 React 組件的編輯提供即時反饋。

這意味著 Next.js 僅更新您編輯文件中的程式碼，並且僅重新渲染該組件，且不會遺失組件狀態。這包括樣式（內聯、CSS-in-JS 或 CSS/Sass Modules）、標記、事件處理函式和效果（透過 `useEffect`）。

今天，我們讓它在每次編輯時都**快 200ms**，而且您不需要進行任何程式碼變更。平均而言，這使得 Fast Refresh 快 3 倍。

想要更快的效能和更好的建置時間嗎？今天即可透過新的 `future` 旗標選擇加入 Webpack 5。此旗標為預覽版本 (RC)，Webpack 5 很快將成為預設。您啟用此旗標後，請 [與我們分享任何反饋](https://github.com/vercel/next.js/discussions/23498)。

next.config.js

```
module.exports = {
  future: {
    webpack5: true,
  },
};
```

欲進一步了解 Webpack 5 如何以向後相容的方式推出到您的應用程式，請查看 [文件](/docs/messages/webpack5)。

改進安裝時間 [](#improved-installation-time)
---------------------------------------------------------

我們已最佳化 Next.js 的依賴圖以縮短安裝時間並減小依賴大小。現在建立 Next.js 專案比以往更快，無論是在本地還是 CI/CD 流程中。

|  | 10.0 | 10.1 | delta |
| --- | --- | --- | --- |
| 平均安裝時間 | ~15s | ~5s | 快 3.0 倍 |
| 安裝尺寸 | 96.5MB | 39.9MB | 小了 58% |
| 依賴數量 | 424 | 187 | 小了 56% |

作為我們最佳化依賴套件目標的一部分，我們也更新和/或替換了被作者標記為棄用的 npm 套件，即使在依賴樹中深深巢嵌也是如此。現在安裝 fresh Next.js 10.1 將不產生任何套件警告。

[`next/image` 改進](#nextimage-improvements)
----------------------------------------------------

### Apple Silicon 支援 [](#apple-silicon-support)

隨著 [Next.js 10](https://nextjs.org/blog/next-10) 發布，影像最佳化由原生依賴驅動。這導致安裝時間較長，原生依賴消耗了 Next.js 總安裝尺寸的 50%。此外，這移除了安裝後運行腳本並導致按需編譯。

我們興奮地分享，透過 `next/image` 的自動影像最佳化現在由 [WebAssembly](https://web.dev/squoosh-v2/) 驅動。此優化的使用無需額外設定，可透過 `next dev` 和 `next start` 使用。透過使用 WASM，我們將安裝尺寸減少了約 30MB，並**新增支援 [Apple Silicon](https://www.apple.com/mac/m1/) M1 MacBooks**。

我們透過利用 [SIMD](https://v8.dev/features/simd) 擴充和執行緒繼續改善我們的 WebAssembly 效能。

#### 其他版面配置 [](#additional-layouts)

基於您的反饋，我們為 `next/image` 新增了多種版面配置和選項：

*   `layout=fill`: 您不需要提供 `width` 和 `height`。 ([範例](https://image-component.nextjs.gallery/layout-fill))
*   `layout=fixed`: 原生 `img` 行為。 ([範例](https://image-component.nextjs.gallery/layout-fixed))
*   `layout=responsive`: 彈性寬度並保持固定比例。 ([範例](https://image-component.nextjs.gallery/layout-responsive))
*   `layout=intrinsic`: 不會增長，但可縮小並保持固定比例。 ([範例](https://image-component.nextjs.gallery/layout-intrinsic))
*   `objectFit=cover`: 背景圖片。 ([範例](https://image-component.nextjs.gallery/background))

欲進一步了解，請查看 [`layout` 文件](/docs/pages/api-reference/components/image#layout)。

#### 改進的影像最佳化支援 [](#improved-image-optimization-support)

我們擴充了 `next/image` 的 [內建影像載入器](/docs/pages/building-your-application/optimizing/images#loader)，新增了新增任何自定義載入器的能力。透過使用新的 `loader` prop，您可以**使用任何提供者或 CDN 最佳化圖片**。

components/my-image.js

```
import Image from 'next/image';
 
const myLoader = ({ src, width, quality }) => {
  return `https://example.com/${src}?w=${width}&q=${quality || 75}`;
};
 
const MyImage = (props) => {
  return (
    <Image
      loader={myLoader}
      src="/me.png"
      alt="Picture of the author"
      width={500}
      height={500}
    />
  );
};
```

欲進一步了解，請查看 [`next/image` 文件](/docs/pages/api-reference/components/image#loader)。

Next.js Commerce Shopify 整合 [](#nextjs-commerce-shopify-integration)

電子商務是採用 Next.js 成長最快的產業之一。像 Apple、Wal-Mart、McDonald's 和 Nike 的公司都信任 Next.js。

我們希望提供一個建立電子商務應用程式的高效能起點，因此我們發布了 [Next.js Commerce](https://nextjs.org/commerce)。數千次針對 Shopify 的要求後，我們已更新 Next.js Commerce，採用**與提供者無關的使用者介面** — 讓您將任何您選擇的無頭電子商務解決方案引入。

在不同提供者之間切換僅是配置中的**單一列變更** (single-line change)。我們也建立了一個 Features API，允許您根據使用案例切換功能 — 同時保留最高等級的效能。

![Lighthouse 全球指標](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-10-1/lighthouse.png)

Next.js Commerce 在全球的 Lighthouse 指標。

幾次點擊後，Next.js 開發人員即可複製、部署並完全自訂自己的電子商務商店。在 [nextjs.org/commerce](https://nextjs.org/commerce) 了解更多，並嘗試我們的 [Shopify 範例](https://shopify.demo.vercel.store/)。更多電子商務提供者的支援即將推出。

自訂 500 頁面 [](#custom-500-page)

您現在可以建立自訂的 `pages/500.js`，以便在顯示錯誤時加入您自己的徽標和品牌標識。類似於 `pages/404.js`，此檔案在建置時靜態生成。

pages/500.js

```
export default function Custom500() {
  return <h1>500 - Server-side error occurred</h1>;
}
```

嚴格 PostCSS 配置載入 [](#strict-postcss-configuration-loading)

如果您使用 PostCSS 和 Webpack 5，您可以透過新的 `future` 旗標提升效能（透過快取）。

next.config.js

```
module.exports = {
  future: {
    webpack5: true,
    strictPostcssConfiguration: true,
  },
};
```

今天嘗試並告訴我們您的反饋。欲進一步了解，請查看 [PostCSS 配置文件](/docs/pages/building-your-application/configuring/post-css)。

`tsconfig.json` 中的 `extends` 支援 [](#support-for-extends-in-tsconfigjson)

您現在可以在 Next.js 使用的 `tsconfig.json` 中使用 `extends`，以擴展自您在專案中的其他 `tsconfig.json`。例如，您可以如下從 `tsconfig.base.json` 擴展：

tsconfig.json

```
{
  "extends": "./tsconfig.base.json"
}
```

欲進一步了解，請查看 [TypeScript 文件](/docs/pages/building-your-application/configuring/typescript)。

偵測預覽模式是否已啟用 [](#detect-when-preview-mode-is-enabled)

您現在可以偵測預覽模式是否已啟用，讓您僅為編輯者渲染特定組件。

components/MyComponent.js

```
import { useRouter } from 'next/router';
 
function MyComponent() {
  const { isPreview } = useRouter();
  return <>{isPreview ? <h1>Preview Mode Enabled</h1> : null}</>;
}
```

欲進一步了解，請查看 [預覽模式文件](/docs/pages/building-your-application/configuring/preview-mode)。

路由方法滾動至頂端 [](#router-methods-scroll-to-top)

當使用 `next/link` 導航時，Next.js 將自動滾動至頂端。使用 Next.js 10.1 `router.push` 和 `router.replace` 時，也將會自動滾動至頂端以匹配 `next/link` 的行為。您可以使用 `scroll` 選項取消訂閱自動滾動。

欲進一步了解，請查看 [`next/router` 文件](/docs/pages/api-reference/functions/use-router)。
