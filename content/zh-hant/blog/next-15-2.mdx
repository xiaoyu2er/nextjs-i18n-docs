Next.js 15.2 引入了除錯錯誤、元數據、Turbopack 及更多功能更新：

*   [**重新設計的錯誤 UI 與改進的堆疊追蹤**](/blog/next-15-2#redesigned-error-ui-and-improved-stack-traces): 經過重新設計的除錯體驗
*   [**流式元數據**](/blog/next-15-2#streaming-metadata): 異步元數據將不再阻礙頁面渲染或客戶端頁面過渡
*   [**Turbopack 效能改進**](/blog/next-15-2#turbopack-performance-improvements): 更快的編譯時間與降低的記憶體使用量
*   [**React 視圖過渡 (實驗性)**](/blog/next-15-2#react-view-transitions-experimental): React 新視圖過渡 API 的實驗性支援
*   [**Node.js 中間件 (實驗性)**](/blog/next-15-2#nodejs-middleware-experimental): 在 Middleware 中使用 Node.js 執行時的實驗性支援

立即升級，或開始使用：

終端機

```
# Use the automated upgrade CLI
npx @next/codemod@canary upgrade latest
 
# ...or upgrade manually
npm install next@latest react@latest react-dom@latest
 
# ...or start a new project
npx create-next-app@latest
```

重新設計的錯誤 UI 與改進的堆疊追蹤 [](#redesigned-error-ui-and-improved-stack-traces)
-----------------------------------------------------------------------------------------------

我們在您開發應用程式時遇到的錯誤中，加入了視覺與品質上的改進。讓我們來逐一了解每個改進領域：

### 錯誤覆蓋層 [](#error-overlay)

![Next.js 15.2 版本後錯誤覆蓋層的範例](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-15-2/bad-element-error-turbo2-light.png)![Next.js 15.2 版本後錯誤覆蓋層的範例](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Fstatic%2Fblog%2Fnext-15-2%2Fbad-element-error-turbo2-dark.png&w=3840&q=75)

Next.js 15.2 版本後錯誤覆蓋層的範例

我們重構了 Next.js 錯誤訊息的 UI 與呈現方式，使其更易於理解。新設計突顯錯誤的核心細節——如訊息、相關程式碼框架與呼叫堆疊——同時減少函式庫或相依項目的程式碼雜訊。這意味著您可以迅速找到問題根源，並更快開始修復。

利用 React 新引入的 [擁有者堆疊](https://react.dev/reference/react/captureOwnerStack) 功能，我們現在能夠提供更準確的錯誤來源資訊。Next.js 現在能夠顯示引發錯誤的子元件，跳過那些並非造成錯誤根源的中間元素。

我們也讓自訂您的指標偏好設定變得更容易，而無需添加其他配置。

![Next.js 開發工具偏好設定範例](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-15-2/preferences-light.png)![Next.js 開發工具偏好設定範例](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Fstatic%2Fblog%2Fnext-15-2%2Fpreferences-dark.png&w=1080&q=75)

Next.js 開發工具偏好設定範例

我們在錯誤覆蓋層底部加入了回饋區段，讓您可評分錯誤訊息的協助程度。您的意見能協助我們了解常見痛點，並改進錯誤訊息，讓除錯變得更容易。

### 開發指示器 [](#dev-indicator)

開發指示器的各種狀態，從渲染中到顯示額外資訊。

我們將開發資訊整合進了一個全新的簡潔指示器中，顯示如渲染模式與建置狀態等細節。

在編譯期間，當您瀏覽不同路由時，會注意到一個昏暗且帶有動畫效果的 Next.js 標誌。編譯完成且 React 開始渲染時，標誌會變亮，為您提供應用程式狀態的視覺提示。

開啟開發指示器現在會顯示：

*   您目前路由的渲染模式 (靜態/動態)
*   Turbopack 編譯狀態
*   目前生效的錯誤，可快速存取錯誤覆蓋層

未來的更新將擴充此選單，包含：

*   PPR (部分預渲染) 除錯工具
*   快取監視功能
*   其他開發者工具

這種統一的做法將所有關鍵的開發資訊整合在同一個易於存取的位置。我們將根據您的回饋在未來的版本中持續優化與擴充此功能。

流式元數據 [](#streaming-metadata)
-----------------------------------------

在 generateMetadata 中獲取動態資料或執行某些異步操作通常是必要的。在 Next.js 的舊版本中，這些元數據必須在初始 UI 發送之前完成生成，以便包含在文件 <head> 中。

這意味著對於許多頁面上，即使可以快速顯示初始 UI，首次繪製仍會因數據需求而延遲，而這些需求並不影響用戶視覺上看到的内容。我們在 15.2 中改進了這一點，允許在 generateMetadata 完成之前先將初始 UI 發送到瀏覽器。

![Next.js 開發工具偏好設定範例](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/static/blog/next-15-2/metadata-light.png)![Next.js 開發工具偏好設定範例](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Fstatic%2Fblog%2Fnext-15-2%2Fmetadata-dark.png&w=3840&q=75)

Next.js 開發工具偏好設定範例

不過，為了維持與那些期望在文件 <head> 中取得元數據的機器人（bots）與爬蟲的相容性，我們仍會暫緩發送 HTML 給某些機器人使用者代理。如果您需要更細微的控制權來決定哪些機器人接收此待遇，您可以透過 `next.config.js` 中的 `htmlLimitedBots` 選項自訂用來服務它們的正則表達式。

深入瞭解 [流式元數據](/docs/app/api-reference/functions/generate-metadata#streaming-metadata)。

Turbopack 效能改進 [](#turbopack-performance-improvements)
-------------------------------------------------------------------------

Turbopack 已隨著 [Next.js 15](https://nextjs.org/blog/next-15#turbopack-dev) 標記為穩定版。

我們一直在致力於提升 Turbopack 的效能，特別是在沒有持續快取的場景。作為此次發布的一部分，我們引入了以下增強：

*   **更快的編譯時間：** 早期採用者報告顯示，與 Next.js 15.1 相比，存取路由時的編譯時間最多快 **57.6%**。
*   **降低記憶體使用量：** 對於 [vercel.com](https://vercel.com/) 應用，我們在本地開發期間观察到 **30% 的記憶體使用量減少**。

有了這些改進，Turbopack 應該現在在所有情況下都比 Webpack 快。如果您遇到對於您的應用不成立的情況，請聯絡我們——我們希望調查這些案例。

我們也在持續快取與生產環境建置方面取得了進展。雖然這些功能尚未準備好進行實驗性發布，但我們已开始在真實專案中測試它們。一旦更廣泛可用時，我們將分享更詳細的指標。

React 視圖過渡 (實驗性) [](#react-view-transitions-experimental)
-----------------------------------------------------------------------------

我們已新增功能旗標（feature flag）以在 React 中啟用新的實驗性視圖過渡 API。此新 API 允許您在應用程式的不同視圖與元件之間進行動畫過渡。

若要啟用此功能，請在您的 `next.config.js` 中新增以下內容：

next.config.js

```
module.exports = {
  experimental: {
    viewTransition: true,
  },
};
```

> **注意：** 此功能高度實驗性，可能在未來的版本中變更。

有關如何使用此功能的詳細資訊，請參閱 React 倉庫中的原始視圖轉換 Pull Request。此工作是建立在 View Transitions 的自然瀏覽器實作之上。

隨著穩定性推進，我們將發布更多文件與範例。

Node.js 中間件 (實驗性) [](#nodejs-middleware-experimental)
--------------------------------------------------------------------

我們一直在開發新的實驗性旗標，以允許在 Next.js Middleware 中使用 Node.js 執行時。

若要啟用此功能，請在您的 `next.config.js` 中新增以下內容：

next.config.js

```
module.exports = {
  experimental: {
    nodeMiddleware: true,
  },
};
```

然後您可以在 Middleware `config` 導出中指定 Node.js 執行時：

middleware.js

```
import bcrypt from 'bcrypt';
 
const API_KEY_HASH = process.env.API_KEY_HASH; // Pre-hashed API key in env
 
export default async function middleware(req) {
  const apiKey = req.headers.get('x-api-key');
 
  if (!apiKey || !(await bcrypt.compare(apiKey, API_KEY_HASH))) {
    return new Response('Forbidden', { status: 403 });
  }
 
  console.log('API key validated');
}
 
export const config = {
  runtime: 'nodejs',
};
```

> **注意：** 此功能尚不建議用於生產環境。因此，除非您使用的是 `next@canary` 發行版而非穩定版，否則 Next.js 將拋出錯誤。

我們計劃利用此機會改進並重塑 Middleware API。如果您有任何建議或請求，請告訴我們。Node.js Middleware 是社群的主要請求之一，我們很高興能解決此問題。
