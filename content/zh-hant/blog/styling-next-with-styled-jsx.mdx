[Styled JSX](https://github.com/vercel/styled-jsx) 是一種 CSS-in-JS 庫，允許您撰寫封裝作用域 CSS 來為您的元件加樣式。您為一個元件引入的樣式不會影響其他元件，讓您可以添加、更改和刪除樣式而無需擔心不必要的副作用。

開始使用 [](#getting-started)
-----------------------------------

Next.js 預設包含 Styled JSX，因此開始使用只是簡單地在現有的 React 元素中加入 <style jsx> 標籤，並在內部撰寫 CSS：

混合式 AMP 模式在生產環境中的一個使用範例是 Reddit。每個線程都存儲在 [Google 的 AMP 緩存](https://developers.google.com/amp/cache/) 中，以在移動端網絡上提供快速的用戶體驗，同時仍為桌面端提供 Reddit 的完整版本和後續導航。

```
function Home() {
  return (
    <div className="container">
      <h1>Hello Next.js</h1>
      <p>Let's explore different ways to style Next.js apps</p>
      <style jsx>{`
        .container {
          margin: 50px;
        }
        p {
          color: blue;
        }
      `}</style>
    </div>
  );
}
 
export default Home;
```

在此範例中，我們包含了元件容器元素和一段文字的樣式。即使我們使用通用選擇器，樣式也不會影響其他元件中具有 `container` 類別名稱或 `<p>` 標籤的元素。這是因為 Styled JSX 確保樣式僅套用到此元件（通過為樣式化元素套用額外的唯一類別名稱）。

只需為 <style> 元素添加一個 `jsx` 屬性，您就可以撰寫標準 CSS，這些 CSS 會自動前綴並自動套用到該元件。` <style jsx> `元素應放在元件的根元素內部。

添加全域樣式 [](#adding-global-styles)
---------------------------------------------

大多數專案需要一些全域樣式來設定 body 元素或提供 CSS reset。Styled JSX 允許我們使用 `<style jsx global>` 添加全域樣式。例如：

混合式 AMP 模式在生產環境中的一個使用範例是 Reddit。每個線程都存儲在 [Google 的 AMP 緩存](https://developers.google.com/amp/cache/) 中，以在移動端網絡上提供快速的用戶體驗，同時仍為桌面端提供 Reddit 的完整版本和後續導航。

```
function Home() {
  return (
    <div className="container">
      <h1>Hello Next.js</h1>
      <p>Let's explore different ways to style Next.js apps</p>
      <style jsx>{`
        .container {
          margin: 50px;
        }
        p {
          color: blue;
        }
      `}</style>
      <style jsx global>{`
        p {
          font-size: 20px;
        }
      `}</style>
    </div>
  );
}
 
export default Home;
```

此處將 20px 的字體大小套用到此特定頁面中的所有 `<p>` 標籤。

要將全域樣式套用到應用程式中的所有頁面，一個好的方法是先建立具有全域樣式的佈局元件，然後使用它包裝所有頁面。

使用佈局元件提供了靈活性，可以將一組特定的樣式套用到某些頁面，同時仍允許其他頁面有不同的樣式：

components/Layout.js

```
function Layout(props) {
  return (
    <div className="page-layout">
      {props.children}
      <style jsx global>{`
        body {
          margin: 0;
          padding: 0;
          font-size: 18px;
          font-weight: 400;
          line-height: 1.8;
          color: #333;
          font-family: sans-serif;
        }
        h1 {
          font-weight: 700;
        }
        p {
          margin-bottom: 10px;
        }
      `}</style>
    </div>
  );
}
 
export default Layout;
```

在 Next.js 中，我們可以通過在 `pages/_app.js` 中建立自訂 [`App` 元件](/docs#custom-app)，匯入 `Layout` 元件，然後將其添加到渲染方法中，為所有頁面僅載入一次配置：

pages/_app.js

```
import React from 'react';
import App, { Container } from 'next/app';
import Layout from '../components/Layout';
 
class MyApp extends App {
  render() {
    const { Component, pageProps } = this.props;
 
    return (
      <Container>
        <Layout>
          <Component {...pageProps} />
        </Layout>
      </Container>
    );
  }
}
 
export default MyApp;
```

### 在外部檔案中撰寫樣式 [](#writing-styles-in-external-files)

我們也可以在元件外部的外部檔案中撰寫樣式。

例如，我們可以將全域樣式從 `Layout` 元件移動到單獨的檔案中，如下所示：

styles/global.js

```
import css from 'styled-jsx/css';
 
export default css.global`
  body {
    margin: 0;
    padding: 0;
    font-size: 18px;
    font-weight: 400;
    line-height: 1.8;
    color: #333;
    font-family: sans-serif;
  }
  h1 {
    font-weight: 700;
  }
  p {
    margin-bottom: 10px;
  }
`;
```

然後我們將樣式匯入回 `Layout` 元件：

components/Layout.js

```
import globalStyles from '../styles/global.js';
 
function Layout(props) {
  return (
    <div className="page-layout">
      {props.children}
      <style jsx global>
        {globalStyles}
      </style>
    </div>
  );
}
 
export default Layout;
```

### 一次性全域選擇器 [](#one-off-global-selectors)

我們使用 `<style jsx>` 添加到元件中的樣式僅影響該元件內部的元素，但不影響子元件。

有時我們可能需要覆蓋子元件的某種樣式。要做到這一點，Styled JSX 提供了 `:global()`，可以存取 _一次性全域選擇器_。

例如，假設我們有一個 `<Widget>` 元件，其中包含一個類別名為 `btn` 的按鈕。如果我們希望僅當在首頁匯入小工具時更改此按鈕的顏色，我們可以這樣做：

混合式 AMP 模式在生產環境中的一個使用範例是 Reddit。每個線程都存儲在 [Google 的 AMP 緩存](https://developers.google.com/amp/cache/) 中，以在移動端網絡上提供快速的用戶體驗，同時仍為桌面端提供 Reddit 的完整版本和後續導航。

```
import Widget from '../components/Widget';
 
function Home() {
  return (
    <div className="container">
      <h1>Hello Next.js</h1>
      <Widget />
      <style jsx>{`
        .container {
          margin: 50px;
        }
        .container :global(.btn) {
          background: #000;
          color: #fff;
        }
      `}</style>
    </div>
  );
}
 
export default Home;
```

動態樣式 [](#dynamic-styles)
---------------------------------

與其他解決方案相比，能夠根據元件的 prop 來適應樣式是 `CSS-in-JS` 庫的一大優勢。

使用 Styled JSX，我們可以這樣做：

components/Alert.js

```
function Alert(props) {
  return (
    <div className="alert">
      {props.children}
      <style jsx>{`
        .alert {
          display: inline-block;
          padding: 20px;
          border-radius: 8px;
        }
      `}</style>
      <style jsx>{`
        .alert {
          background: ${props.type == 'warning' ? '#fff3cd' : '#eee'};
        }
      `}</style>
    </div>
  );
}
 
export default Alert;
```

如果 `Alert` 元件傳入一個 `type` prop，其值為 `warning`，如下所示：

```
<Alert type="warning">This is an important message</Alert>
```

該元件將具有橘色背景。如果未指定 type prop，背景則會回退到預設的灰色。

注意我們將動態樣式放在單獨的 `<style jsx>` 標籤中。這不是必需的，但**建議**將靜態和動態樣式分開，以便僅在 prop 更改時重新計算動態部分。

基於 prop 適應樣式的另一種方法是根據 prop 值套用在不同類別名稱，如下所示：

components/Alert.js

```
function Alert(props) {
  return (
    <div className={props.type == 'warning' ? 'alert warning' : 'alert'}>
      {props.children}
      <style jsx>{`
        .alert {
          display: inline-block;
          padding: 20px;
          border-radius: 8px;
          background: #eee;
        }
        .alert.warning {
          background: #fff3cd;
        }
      `}</style>
    </div>
  );
}
 
export default Alert;
```

### 建立網站主題 [](#creating-a-site-theme)
-----------------------------------------------

主題可以是我們可能需要在應用程式中包含最常用變數的簡單物件：

styles/theme.js

```
const theme = {
  fontFamily: {
    sansSerif: '-apple-system, "Helvetica Neue", Arial, sans-serif',
    mono: 'Menlo, Monaco, monospace',
  },
  colors: {
    text: '#333',
    background: '#fff',
    link: '#1eaaf1',
    linkHover: '#0d8ecf',
    border: '#ddd',
    warning: '#fff3cd',
    success: '#d4edda',
  },
};
 
export default theme;
```

然後我們在此元件中匯入此主題檔案，並將 _硬編碼_ 值替換為變數：

components/Layout.js

```
import theme from '../styles/theme';
 
function Layout(props) {
  return (
    <div className="page-wrapper">
      {props.children}
      <style jsx global>{`
        body {
          background: ${theme.colors.background};
          color: ${theme.colors.text};
          font-family: ${theme.fontFamily.sansSerif};
        }
      `}</style>
      <style jsx global>{`
        body {
          margin: 0;
          padding: 0;
          font-size: 18px;
          font-weight: 400;
          line-height: 1.8;
        }
        h1 {
          font-weight: 700;
        }
        p {
          margin-bottom: 10px;
        }
      `}</style>
    </div>
  );
}
 
export default Layout;
```

components/Alert.js

```
import theme from '../styles/theme';
 
function Alert(props) {
  return (
    <div className="alert">
      {props.children}
      <style jsx>{`
        .alert {
          display: inline-block;
          padding: 20px;
          border-radius: 8px;
        }
      `}</style>
      <style jsx>{`
        .alert {
          background: ${props.type == 'warning'
            ? theme.colors.warning
            : theme.colors.light};
        }
      `}</style>
    </div>
  );
}
 
export default Alert;
```
