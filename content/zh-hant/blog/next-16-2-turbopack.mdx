Turbopack 成為 Next.js 預設打包器後兩個版本，我們專注於提升效能、修復錯誤以及增加一致性。以下是 [Next.js 16.2](/blog/next-16-2) 中發布的一些最新功能。

*   [**伺服器快速更新 (Server Fast Refresh)**](#server-fast-refresh): 細粒度的伺服器端熱載入
*   [**Web Worker 來源 (Web Worker Origin)**](#web-worker-origin): 增加 Worker 對 WASM 函式庫的支援
*   [**子資源完整性 (Subresource Integrity) 支援**](#subresource-integrity-support): JavaScript 檔案的子資源完整性 (SRI)
*   [**動態匯入的搖樹優化**](#tree-shaking-of-dynamic-imports): 從動態 `import()` 中移除未使用的匯出
*   [**內聯載入器配置**](#inline-loader-configuration): 透過匯入屬性設定單一匯入的載入器
*   [**Lightning CSS 配置**](#lightning-css-configuration): LightningCSS 實驗性配置選項
*   [**日誌篩選 (Log Filtering)**](#log-filtering): 使用 `ignoreIssue` 抑止雜訊日誌和警告
*   [**`postcss.config.ts` 支援**](#postcssconfigts-support): 使用 TypeScript 的 PostCSS 配置
*   [**效能改進和錯誤修復**](#performance-improvements-and-bug-fixes): 超過 200 項修改和錯誤修復

展望下一個版本，我們將專注於加快編譯器效能並減少記憶體使用。

伺服器快速更新 [](#server-fast-refresh)
-------------------------------------------

我們重構了開發期間伺服器代碼的重新載入方式。舊系統會清除變更模組及其匯入鏈中所有其他模組的 `require.cache`。此方法經常重新載入比必要更多的代碼，包括未變更的 `node_modules`。

新系統將用於瀏覽器的 Fast Refresh 方法帶到了你的伺服器代碼。Turbopack 對模組圖譜的知識意味著只有實際變更的模組會被重新載入，其餘保持完整。這使伺服器端熱載入顯著更高效。

在實際的 Next.js 應用程式中，我們看到應用程式重新整理速度加快了 67-100%，Next.js 內的編譯時間加快了 400-900%。這些結果從小規模的 "hello world" 啟動專案到像 `vercel.com` 這樣的大型網站都適用。

Web Worker 來源 [](#web-worker-origin)
---------------------------------------

之前，Web Worker 透過 `blob://` URL 初始化。這簡化了 Worker 的載入，但設置了一個空的 `location.origin` 值。試圖使用 `importScripts()` 或 `fetch()` 的 Web Worker 代碼（通常在第三方函式庫中）若不做變更，將無法解析請求。

使用更新後的 Worker 初始化代碼，`origin` 正確指向你的網域名稱，且相關聯的 fetch 請求成功。這應該能解決任何在先前版本中嘗試在 Worker 內部執行 WASM 代碼有困難的人士。

子資源完整性 (Subresource Integrity) 支援 [](#subresource-integrity-support)
---------------------------------------------------------------

Turbopack 現在支援 [子資源完整性 (SRI)](https://nextjs.org/docs/app/guides/content-security-policy#enabling-sri)。SRI 在建置時生成 JavaScript 檔案的加密雜湊值，允許瀏覽器驗證檔案是否未被修改。

瀏覽器提供一種稱為內容安全策略 (Content Security Policy) 的技術，允許網站限制可以執行的 JavaScript，從而消除整類的安全問題。但是，實施此技術的典型 `nonce` 基方法要求所有頁面都動態渲染，影響效能。子資源完整性是一種替代方案，它預先計算每個腳本的雜湊值，並只允許瀏覽器執行擁有核准雜湊值的腳本。

next.config.js

```
const nextConfig = {
  experimental: {
    sri: {
      algorithm: 'sha256',
    },
  },
};
 
module.exports = nextConfig;
```

動態匯入的搖樹優化 [](#tree-shaking-of-dynamic-imports)
-------------------------------------------------------------------

Turbopack 現在以與靜態匯入相同的方式對解構的動態匯入進行搖樹優化。未用的匯出從 bundle 中移除：

```
const { cat } = await import('./lib');
```

這現在就搖樹優化而言等同於靜態匯入 — 任何 `./lib` 中未使用的匯出都將被搖樹優化掉。

內聯載入器配置 [](#inline-loader-configuration)
-----------------------------------------------------------

Turbopack 現在支援透過 [匯入屬性](https://github.com/vercel/next.js/pull/89644) 進行單一匯入的載入器配置。取代透過 `turbopack.rules` 全域套用載入器，你可以使用 `with` 子句在單一匯入上進行配置：

```
import rawText from './data.txt' with {
  turbopackLoader: 'raw-loader',
  turbopackAs: '*.js',
};
import value from './data.js' with {
  turbopackLoader: 'string-replace-loader',
  turbopackLoaderOptions: '{"search":"PLACEHOLDER","replace":"replaced value"}',
};
```

這用於僅特定匯入需要特殊處理的情況，避免對同一檔案類型的其他匯入產生副作用。可用的屬性為 `turbopackLoader`、`turbopackLoaderOptions`、`turbopackAs` 和 `turbopackModuleType`。

如果可能，你仍應優先在 `next.config.ts` 中配置載入器，因為使用內聯載入器的代碼可移植性較差。此選項對於從插件或載入器生成的代碼更有用。

Lightning CSS 配置 [](#lightning-css-configuration)
-----------------------------------------------------------

Lightning CSS 是快、基於 Rust 的 CSS 轉換器，供 Turbopack 用於 CSS 壓縮和預補字元。它也允許在舊版瀏覽器上實施現代 CSS 功能，類似於 Babel 或 SWC 對 JavaScript 所做的事。之前，切換這些設置的唯一起方式是透過 Browserslist 配置。現在透過實驗性的 `lightningCssFeatures` 選項，你可以強制某些功能始終或從不進行編譯。

```
import type { NextConfig } from 'next';
 
const nextConfig: NextConfig = {
  experimental: {
    useLightningcss: true,
    lightningCssFeatures: {
      include: ['light-dark', 'oklab-colors'],
      exclude: ['nesting'],
    },
  },
};
 
export default nextConfig;
```

日誌篩選 [](#log-filtering)
-------------------------------

Turbopack 現在可以使用 `turbopack.ignoreIssue` 配置選項，從串流日誌中抑止雜訊或預期的警告。這有用於抑止來自第三方代碼、生成檔案的警告，以及隱藏 Next.js 錯誤覆蓋層中的預期錯誤。

你可以匹配從特定代碼路徑生成的日誌，以及篩選特定的標題/描述字串或模式。

```
import type { NextConfig } from 'next';
 
const nextConfig: NextConfig = {
  turbopack: {
    ignoreIssue: [
      { path: '**/vendor/**' },
      { path: 'app/**', title: 'Module not found' },
      { path: /generated\/.*\.ts/, description: /expected error/i },
    ],
  },
};
 
export default nextConfig;
```

`postcss.config.ts` 支援 [](#postcssconfigts-support)
-------------------------------------------------------

Turbopack 現在除了現有 `.js` 和 `.cjs` 變體外，也支援 `postcss.config.ts`。

效能改進和錯誤修復 [](#performance-improvements-and-bug-fixes)
---------------------------------------------------------------------------------

我們持續投資改進 Turbopack 內部。超過 200 項修改和錯誤修復已改善廣泛專案的穩定性和相容性。資料編碼、內部代表格式和雜湊演算法的優化已改善了記憶體使用和建置時間。我們讓錯誤日誌更清晰並擴張診斷功能，以幫助你更好地理解編譯器錯誤。我們也使用您的 GitHub Issues 作為指引，了解自 16.1 版本以來缺失哪些功能。
