# A

## 應用路由 (App Router)

Next.js 13 版本引入的路由，基於 React Server Components 構建。它使用基於文件系統的路由，支持佈局、嵌套路由、加載狀態、錯誤處理等。詳細內容請參見 [App Router 文檔](/docs/app)。

# B

## 構建時間 (Build time)

應用程式處於編譯階段。在構建期間，Next.js 將您的代碼轉換為用於生產環境的優化文件，生成靜態頁面，並準備部署資源。參見 [`next build`] CLI 參考文檔](/docs/app/api-reference/cli/next#next-build-options)。

# C

## 緩存組件 (Cache Components)

一個功能，使用 [`"use cache"`] 指令來實現組件級和函數級緩存。緩存組件允許您將靜態、緩存和動態內容混合在單一路由中，通過預渲染靜態 HTML 外殼並立即提供，同時在動態內容就緒時流式傳輸。使用 [`cacheLife()`] 配置緩存時長，使用 [`cacheTag()`] 標記緩存數據，使用 [`updateTag()`] 按需無效化。詳細內容請參見 [緩存組件指南](/docs/app/getting-started/caching)。

## 全捕捉區段

動態路由段，可使用 `[...folder]/page.js` 語法匹配多個 URL 部分。這些段捕獲所有剩餘的 URL 段，適用於實現文檔站點或文件瀏覽器等功能。詳細內容請參見 [動態路由段](/docs/app/api-reference/file-conventions/dynamic-routes#catch-all-segments)。

## 客戶端打包文件 (Client Bundles)

發送到瀏覽器的 JavaScript 打包文件。Next.js 基於 [模塊圖](#module-graph) 自動拆分這些文件，以減少初始負載大小並僅加載每頁面所需的代碼。

## 客戶端組件 (Client Component)

在瀏覽器中運行的 React 組件。在 Next.js 中，客戶端組件也可以在初始頁面生成期間在服务端渲染。它們可以使用狀態、效果、事件處理函數和瀏覽器 API，並在文件頂部標記 [`"use client"`] 指令。詳細內容請參見 [服務端與客戶端組件](/docs/app/getting-started/server-and-client-components)。

## 客戶端導航 (Client-side navigation)

一種導航技術，頁面內容動態更新而無需完整頁面刷新。Next.js 使用 [`<Link>` 組件](/docs/app/api-reference/components/link) 的客戶端導航，保持共享佈局的交互性並保留瀏覽器狀態。詳細內容請參見 [鏈接與導航](/docs/app/getting-started/linking-and-navigating#client-side-transitions)。

## 客戶端路由快取

瀏覽器中的內存緩存，存儲 [RSC 負載](#rsc-payload) 供已訪問和預取的路由使用。在 [客戶端導航](#client-side-navigation) 期間，Next.js 提供緩存的 [佈局](#layout) 和 [加載 UI](#loading-ui) 而無需服務器請求。默認情況下頁面未緩存，但在瀏覽器後退/前進導航期間會重用。

客戶端緩存在頁面刷新時被清除。它可以通過 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag), [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath), [`updateTag`](/docs/app/api-reference/functions/updateTag), [`router.refresh`](/docs/app/api-reference/functions/use-router), [`cookies.set`](/docs/app/api-reference/functions/cookies), 或者 [`cookies.delete`](/docs/app/api-reference/functions/cookies) 以編程方式無效化。

您可以使用 [`staleTimes`](/docs/app/api-reference/config/next-config-js/staleTimes) 全局配置客戶端緩存時長，或者通過 [`cacheLife`](/docs/app/api-reference/functions/cacheLife#client-cache-behavior) 中的 `stale` 屬性為每個路由配置（推薦）。

## 代碼分割 (Code Splitting)

根據路由將應用程式劃分為較小的 JavaScript 塊的過程。預先加載所有代碼相反，只加載當前路由所需的代碼，減少初始加載時間。Next.js 根據路由自動執行代碼分割。詳細內容請參見 [包打包指南](/docs/app/guides/package-bundling)。

# D

## 動態渲染 (Dynamic rendering)

當組件在請求時渲染而不是 [構建時間](#build-time)。當組件使用 [請求時 API](#request-time-apis) 時，它變為動態。

## 動態路由段 (Dynamic route segments)

在請求時從數據生成的 [路由段](#route-segment)。通過在花括號中包裹文件名來創建（例如，`[slug]`），允許您從動態數據如博客文章或產品頁面創建路由。詳細內容請參見 [動態路由段](/docs/app/api-reference/file-conventions/dynamic-routes)。

# E

## 環境變數

在構建時間或請求時間可訪問的配置值。在 Next.js 中，帶 `NEXT_PUBLIC_` 前綴的變量暴露給瀏覽器，而其他變量僅在服务端可用。詳細內容請參見 [環境變量](/docs/app/guides/environment-variables)。

## 錯誤邊界 (Error Boundary)

一個在子組件樹中捕獲 JavaScript 錯誤的 React 組件，並顯示回退 UI。在 Next.js 中，創建 [`error.js`] 文件以將路由段自動包裝在錯誤邊界中。詳細內容請參見 [錯誤處理](/docs/app/getting-started/error-handling)。

# F

## 字體優化 (Font Optimization)

使用 [`next/font`] 的自動字體優化。Next.js 自行託管字體，消除佈局偏移，並應用性能最佳實踐。適用於 Google Fonts 和本地字體文件。詳細內容請參見 [字體](/docs/app/getting-started/fonts)。

## 文件系統緩存 (File-system caching)

Turbopack 功能，將編譯器內容存儲在磁盤上，在 `next dev` 或 `next build` 命令之間減少工作量，顯著加快編譯時間。詳細內容請參見 [Turbopack 文件系統緩存](/docs/app/api-reference/config/next-config-js/turbopackFileSystemCache)。

# H

## 水合 (Hydration)

React 將事件處理程序附加到 DOM 的過程，使服務端渲染的靜態 HTML 具有交互性。在水合期間，React 將服務端渲染的標記與客戶端 JavaScript 進行協調。詳細內容請參見 [服務端與客戶端組件](/docs/app/getting-started/server-and-client-components#how-do-server-and-client-components-work-in-nextjs)。

# I

## 導入別名 (Import Aliases)

提供縮寫參考的自定義路徑映射，減少相對導入的複雜性，使代碼更易讀和易維護。詳細內容請參見 [絕對導入和模塊路徑別名](/docs/app/getting-started/installation#set-up-absolute-imports-and-module-path-aliases)。

## 增量靜態再生 (ISR)

一種允許您更新靜態內容而無需重新構建整個站點的技术。ISR 允許您按頁面級別使用靜態生成，同時在後台重新驗證頁面。詳細內容請參見 [ISR 指南](/docs/app/guides/incremental-static-regeneration)。

> **值得了解**: 在 Next.js 中，ISR 也稱為 [重新驗證](#revalidation)。

## 攔截路由 (Intercepting Routes)

一種路由模式，允許在當前佈局中從應用程式的另一部分加載路由。適合顯示內容（如模態框）而無需用戶切換上下文，同時保持 URL 可共享。詳細內容請參見 [攔截路由](/docs/app/api-reference/file-conventions/intercepting-routes)。

## 影像優化

使用 [`<Image>` 組件](/docs/app/api-reference/components/image) 的自動圖像優化。Next.js 按需優化圖像，提供 WebP 等現代格式，並自動處理惰性加載和響應式縮放。詳細內容請參見 [圖像](/docs/app/getting-started/images)。

# L

## 佈局 (Layout)

在多個頁面之間共享的 UI。佈局保存狀態，保持交互性，並且在導航時不重新渲染。定義為從 [`layout.js`] 文件導出 React 組件。詳細內容請參見 [佈局和頁面](/docs/app/getting-started/layouts-and-pages)。

## 加載 UI (Loading UI)

在 [路由段](#route-segment) 加載期間顯示的回退 UI。通過在文件夾中添加 [`loading.js`] 文件來創建，它自動將頁面包裝在 [Suspense 邊界](#suspense-boundary) 中。詳細內容請參見 [加載 UI](/docs/app/api-reference/file-conventions/loading)。

# M

## 模塊圖 (Module Graph)

應用程式中文件依賴關係的圖表。每個文件（模塊）是一個節點，導入/導出關係形成邊。Next.js 分析此圖以確定最佳的打包和代碼分割策略。詳細內容請參見 [服務端與客戶端組件](/docs/app/getting-started/server-and-client-components#reducing-js-bundle-size)。

## 元數據

用於瀏覽器和安全引擎的頁面信息，例如標題、描述和 Open Graph 圖像。在 Next.js 中，使用 [`metadata`] 導出或 [`generateMetadata`] 函數在佈局或頁面文件中定義元數據。詳細內容請參見 [元數據和 OG 圖像](/docs/app/getting-started/metadata-and-og-images)。

## 記憶化 (Memoization)

緩存函數返回值的過程，以便在渲染傳遞（請求）期間調用相同函數多次時只執行一次。在 Next.js 中，具有相同 URL 和選項的 `fetch` `GET` 請求在 Server Components, layouts, pages 和 generateMetadata/generateStaticParams 之間自動記憶化（但 [路由處理器](/docs/app/api-reference/file-conventions/route) 除外，因為它們不是 React 組件樹的一部分）。

對於非 `fetch` 操作，使用 React [`cache`](https://react.dev/reference/react/cache) 函數。詳細內容請參見 [`fetch` API 參考](/docs/app/api-reference/functions/fetch)。

## 中間件

參見 [代理](#proxy)。

# N

## 未找到 (Not Found)

在路由不存在或調用 [`notFound()`] 函數時顯示的特殊組件。通過在應用程序目錄中添加 [`not-found.js`] 文件創建。詳細內容請參見 [錯誤處理](/docs/app/getting-started/error-handling#not-found)。

# P

## 私有文件夾 (Private Folders)

以下劃線為前綴的文件夾（例如，`_components`），從路由系統中排除。這些文件夾用於代碼組織和共享實用工具而不創建可訪問的路由。詳細內容請參見 [私有文件夾](/docs/app/getting-started/project-structure#private-folders)。

## 頁面 (Page)

特定於路由的 UI。定義為從 `app` 目錄中的 [`page.js`] 文件導出 React 組件。詳細內容請參見 [佈局和頁面](/docs/app/getting-started/layouts-and-pages)。

## 平行路由 (Parallel Routes)

允許同時或條件渲染多個頁面的模式。使用帶有 `@folder` 約定的命名槽創建，適用於儀表板、模態框和複雜佈局。詳細內容請參見 [平行路由](/docs/app/api-reference/file-conventions/parallel-routes)。

## 部分預渲染 (PPR)

結合預渲染和動態渲染的渲染優化。靜態外殼立即提供，而動態內容在就緒時流式傳輸，提供兩種渲染策略的優點。詳細內容請參見 [緩存組件](/docs/app/getting-started/caching)。

## 預取 (Prefetching)

在用戶導航之前後台加載路由。Next.js 自動預取與 [`<Link>` 組件](/docs/app/api-reference/components/link) 鏈接的路由，當它們進入視口時，使導航感覺瞬間完成。詳細內容請參見 [預取指南](/docs/app/guides/prefetching)。

## 預渲染 (Prerendering)

當組件在 [構建時間](#build-time) 或 [重新驗證](#revalidation) 期間在後台渲染。結果是 HTML 和 [RSC 負載](#rsc-payload)，可以緩存並從 CDN 提供。預渲染是不使用 [請求時 API](#request-time-apis) 的組件的默認設置。

## 代理 (Proxy)

一個文件 ([`proxy.js`](/docs/app/api-reference/file-conventions/proxy))，在請求完成之前在服務器上運代碼。用於實現服務器端邏輯如日誌、重定向和重寫。以前稱為 Middleware。詳細內容請參見 [代理指南](/docs/app/getting-started/proxy)。

# R

## 重定向 (Redirect)

將用戶從一個 URL 發送到另一個 URL。在 Next.js 中，重定向可以在 [`next.config.js`] 中配置，從 [代理](/docs/app/api-reference/file-conventions/proxy) 返回，或使用 [`redirect()`] 函數程序化觸發。詳細內容請參見 [重定向](/docs/app/guides/redirecting)。

## 請求時 API (Request-time APIs)

訪問請求特定數據的函數，導致組件選擇 [動態渲染](#dynamic-rendering)。包括：

- [`cookies()`](/docs/app/api-reference/functions/cookies) - 訪問請求 Cookie
- [`headers()`](/docs/app/api-reference/functions/headers) - 訪問請求標頭
- [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) - 訪問 URL 查詢參數
- [`draftMode()`](/docs/app/api-reference/functions/draft-mode) - 啟用或檢查草稿模式

## 運行時渲染 (Runtime rendering)

參見 [動態渲染](#dynamic-rendering)。

## 重新驗證 (Revalidation)

更新緩存數據的過程。可能是基於時間的（使用 [`cacheLife()`](/docs/app/api-reference/functions/cacheLife) 設置緩存時長）或按需（使用 [`cacheTag()`](/docs/app/api-reference/functions/cacheTag) 標記數據，然後使用 [`updateTag()`](/docs/app/api-reference/functions/updateTag) 無效化）。詳細內容請參見 [緩存和重新驗證](/docs/app/getting-started/caching)。

## 重寫 (Rewrite)

將入站請求路徑映射到不同的目標路徑而不改變瀏覽器中的 URL。在 [`next.config.js`](/docs/app/api-reference/config/next-config-js/rewrites) 中配置或從 [代理](/docs/app/api-reference/file-conventions/proxy) 返回。適用於代理到外部服務或舊 URL。

## 路由組 (Route Groups)

一種組織路由而不影響 URL 結構的方法。通過在花括號中包裹文件名創建（例如，`(marketing)`），路由組幫助組織相關路由並啟用每組 [佈局](#layout)。詳細內容請參見 [路由組](/docs/app/api-reference/file-conventions/route-groups)。

## 路由處理器 (Route Handler)

處理特定路由 HTTP 請求的函數，定義在 [`route.js`] 文件中。路由處理器使用 Web Request 和 Response API，可處理 `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `HEAD`, 和 `OPTIONS` 方法。詳細內容請參見 [路由處理器](/docs/app/getting-started/route-handlers)。

## 路由段 (Route Segment)

URL 路徑的一部分（兩個斜線之間）由 `app` 目錄中的文件夾定義。每個文件夾代表 URL 結構中的一個段。詳細內容請參見 [項目結構](/docs/app/getting-started/project-structure)。

## RSC 負載 (RSC Payload)

React Server Component 負載——渲染的 React Server Component 樹的壓縮二進制表示。包含 Server Components 的渲染結果、Client Components 的佔位符以及它們之間傳遞的 props。詳細內容請參見 [服務端與客戶端組件](/docs/app/getting-started/server-and-client-components#how-do-server-and-client-components-work-in-nextjs)。

# S

## 服務端組件 (Server Component)

App Router 中的默認組件類型。服務端組件在服务端渲染，可直接獲取數據，且不增加客戶端 JavaScript 打包文件大小。它們不能使用狀態或瀏覽器 API。詳細內容請參見 [服務端與客戶端組件](/docs/app/getting-started/server-and-client-components)。

## 服務端操作 (Server Action)

傳遞給 Client Component 作為 prop 或綁定到表單動作的 [Server Function](#server-function)。Server Actions 常用於表單提交和數據變異。詳細內容請參見 [服務端操作與變異](/docs/app/getting-started/mutating-data)。

## 服務端函數 (Server Function)

在服務器上運行的異步函數，標記 [`"use server"`] 指令。Server Functions 可以從 Client Components 調用。當作為 prop 傳遞給 Client Component 或綁定到表單動作時，它們稱為 [Server Actions](#server-action)。詳細內容請參見 [React 服務端函數](https://react.dev/reference/rsc/server-functions)。

## 靜態導出 (Static Export)

生成完全靜態站點的部署模式，包含 HTML, CSS 和 JavaScript 文件。通過設置 `next.config.js` 中的 `output: 'export'` 啟用。靜態導出可以在沒有 Node.js 服務器的任何靜態文件服務器上託管。詳細內容請參見 [靜態導出](/docs/app/guides/static-exports)。

## 靜態渲染 (Static rendering)

參見 [預渲染](#prerendering)。

## 靜態資源 (Static Assets)

如圖像、字體、視頻和其他媒體的文件，直接提供而不處理。靜態資源通常存儲在 `public` 目錄中，通過相對路徑引用。詳細內容請參見 [靜態資源](/docs/app/api-reference/file-conventions/public-folder)。

## 靜態外殼 (Static Shell)

頁面的預渲染 HTML 結構，立即提供給瀏覽器。與 [部分預渲染](#partial-prerendering-ppr) 配合，靜態外殼包括所有靜態可渲染內容以及 [Suspense 邊界](#suspense-boundary) 回退，用於稍後流式傳輸的動態內容。

## 串流

一種允許服務端發送頁面部分給客戶端的技術，而非等待整個頁面渲染完成。通過 [`loading.js`] 或 manual `<Suspense>` 邊界自動啟用。詳細內容請參見 [流式傳輸指南](/docs/app/guides/streaming)。

## Suspense 邊界 (Suspense boundary)

React [`<Suspense>`](https://react.dev/reference/react/Suspense) 組件，包裝異步內容並在加載時顯示回退 UI。在 Next.js 中，Suspense 邊界定義 [靜態外殼](#static-shell) 結束和 [流式傳輸](#streaming) 開始的位置，启用 [部分預渲染](#partial-prerendering-ppr)。

# T

## Turbopack

為 Next.js 構建的高速、基於 Rust 的打包程序。Turbopack 是 `next dev` 的默認打包程序，並可用於 `next build`。與 Webpack 相比，它提供顯著更快的編譯時間。詳細內容請參見 [Turbopack](/docs/app/api-reference/turbopack)。

## 搖樹 (Tree Shaking)

在構建過程中移除未使用代碼的過程。Next.js 自動搖樹您的代碼以減少打包大小。詳細內容請參見 [包打包指南](/docs/app/guides/package-bundling)。

# U

## \`"use cache"\` 指令

標記組件或函數為可緩存的指令。可以放在文件頂部以指示文件中的所有導出都是可緩存的，或者在函數或組件頂部內聯放置以標記該特定範圍為可緩存。詳細內容請參見 [`"use cache"`] 參考文檔](/docs/app/api-reference/directives/use-cache)。

## \`"use client"\` 指令

標記服務端和客戶端代碼邊界的特殊 React 指令。它必須放在文件頂部，在任何導入或其他代碼之前。它指示 React 組件、輔助函數、變量聲明和所有導入的依賴項應包含在 [客戶端打包文件](#client-bundles) 中。詳細內容請參見 [`"use client"`] 參考文檔](/docs/app/api-reference/directives/use-client)。

## \`"use server"\` 指令

標記函數為 [Server Function](#server-function) 的指令，可從客戶端代碼調用。可以放在文件頂部以指示文件中的所有導出都是 Server Functions，或者在函數頂部內聯放置以標記該特定函數。詳細內容請參見 [`"use server"`] 參考文檔](/docs/app/api-reference/directives/use-server)。

# V

## 版本偏斜 (Version skew)

部署新版本的應用程序後，仍处于活動狀態的客戶端可能引用舊構建的 JavaScript, CSS 或數據。這種客戶端和服务端版本之間的 mismtach 稱為版本偏斜，可能導致資源丟失、Server Action 錯誤和導航失敗。Next.js 使用 [`deploymentId`](/docs/app/api-reference/config/next-config-js/deploymentId) 檢測和處理版本偏斜。詳細內容請參見 [自託管 - 版本偏斜](/docs/app/guides/self-hosting#version-skew)。
