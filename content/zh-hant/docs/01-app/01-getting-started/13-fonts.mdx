[`next/font`](/docs/app/api-reference/components/font) 模組會自動優化您的字體，並移除外部網路請求，以提升隱私和性能。

它包含任何字體檔案的**內建自建託管**。這意味著您可以無佈局偏移地最佳化載入網頁字體。

<AppOnly>

要開始使用 `next/font`，請從 [`next/font/local`](#local-fonts) 或 [`next/font/google`](#google-fonts) 匯入它，將其作為帶有適當選項的函式呼叫，並設定您要套用字體的元素的 `className`。例如：

```tsx filename="app/layout.tsx" highlight={1,3-5,9} switcher
import { Geist } from 'next/font/google'

const geist = Geist({
  subsets: ['latin'],
})

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={geist.className}>
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" highlight={1,3-5,9} switcher
import { Geist } from 'next/font/google'

const geist = Geist({
  subsets: ['latin'],
})

export default function Layout({ children }) {
  return (
    <html className={geist.className}>
      <body>{children}</body>
    </html>
  )
}
```

字體範圍限定於其使用的組件。若要将字體套用至整個應用程式，請將其新增至 [根佈局 (Root Layout)](/docs/app/api-reference/file-conventions/layout#root-layout)。

</AppOnly>

<PagesOnly>

要開始使用 `next/font`，請從 [`next/font/local`](#local-fonts) 或 [`next/font/google`](#google-fonts) 匯入它，使用適當的選項將其作為函式呼叫，並將您要套用該字體的元素設定 `className`。例如，您可以在 [自定義應用程式 (Custom App)](/docs/pages/building-your-application/routing/custom-app) (`pages/_app`) 中全域套用字體：

## Google 字體

您可以自動自建託管任何 Google 字體。字體作為靜態資源儲存，並從與您的部署相同的網域提供服務，這意味著當用戶造訪您的網站時，瀏覽器不會向 Google 發送任何請求。

要開始使用 Google 字體，請從 `next/font/google` 匯入您選擇的字體：

<AppOnly>

```tsx filename="app/layout.tsx" switcher
import { Geist } from 'next/font/google'

const geist = Geist({
  subsets: ['latin'],
})

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={geist.className}>
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import { Geist } from 'next/font/google'

const geist = Geist({
  subsets: ['latin'],
})

export default function RootLayout({ children }) {
  return (
    <html lang="en" className={geist.className}>
      <body>{children}</body>
    </html>
  )
}
```

</AppOnly>

我們建議使用 [可變字體 (variable fonts)](https://fonts.google.com/variablefonts) 以獲得最佳的性能和靈活性。但如果您無法使用可變字體，您將需要指定 `weight`：

## 本機字體

<AppOnly>

要使用本地字體，請從 `next/font/local` 匯入 `localFont` 函式，並指定本地字體檔案的 [`src`](/docs/app/api-reference/components/font#src)。路徑是相對於呼叫 `localFont` 的檔案所解析的。字體可以儲存在專案中的任何位置，包括 [`public`](/docs/app/api-reference/file-conventions/public-folder) 資料夾或 `app` 資料夾內部。例如，要使用儲存在 `app/fonts/` 中的字體：

```tsx filename="app/layout.tsx" switcher
import localFont from 'next/font/local'

const myFont = localFont({
  src: './my-font.woff2',
})

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en" className={myFont.className}>
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import localFont from 'next/font/local'

const myFont = localFont({
  src: './my-font.woff2',
})

export default function RootLayout({ children }) {
  return (
    <html lang="en" className={myFont.className}>
      <body>{children}</body>
    </html>
  )
}
```

</AppOnly>

<PagesOnly>

</PagesOnly>

如果您想為單一字體家族使用多個檔案，則 `src` 可以是陣列：

```js
const roboto = localFont({
  src: [
    {
      path: './Roboto-Regular.woff2',
      weight: '400',
      style: 'normal',
    },
    {
      path: './Roboto-Italic.woff2',
      weight: '400',
      style: 'italic',
    },
    {
      path: './Roboto-Bold.woff2',
      weight: '700',
      style: 'normal',
    },
    {
      path: './Roboto-BoldItalic.woff2',
      weight: '700',
      style: 'italic',
    },
  ],
})
```
