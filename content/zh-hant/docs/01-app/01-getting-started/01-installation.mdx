建立新的 Next.js 應用程式並在本地執行。

## 快速開始

1. 建立名為 `my-app` 的新的 Next.js 應用程式
2. `cd my-app` 並啟動開發伺服器。
3. 造訪 `http://localhost:3000`。

```bash package="pnpm"
pnpm create next-app@latest my-app --yes
cd my-app
pnpm dev
```

```bash package="npm"
npx create-next-app@latest my-app --yes
cd my-app
npm run dev
```

```bash package="yarn"
yarn create next-app@latest my-app --yes
cd my-app
yarn dev
```

```bash package="bun"
bun create next-app@latest my-app --yes
cd my-app
bun dev
```

- `--yes` 跳過使用已儲存偏好設定或預設值的提示。預設設定會啟用 TypeScript、Tailwind CSS、ESLint、App Router 和 Turbopack，並包含導入別名 `@/*`，以及附有 `AGENTS.md` (含有引用它的 `CLAUDE.md`)，以指導編碼代理程式撰寫最新的 Next.js 代碼。

</AppOnly>

## 系統要求

在開始之前，請確保您的開發環境符合以下要求：

- Node.js 最低版本： [20.9](https://nodejs.org/)
- 作業系統：macOS、Windows (包含 WSL) 和 Linux。

## 支援的瀏覽器

Next.js 支援現代瀏覽器，無需配置。

- Chrome 111+
- Edge 111+
- Firefox 111+
- Safari 16.4+

更多關於 [瀏覽器支援](/docs/architecture/supported-browsers) 的資訊，包括如何配置 polyfills 以及針對特定瀏覽器。

## 使用 CLI 建立

建立新的 Next.js 應用程式最快的方式是使用 [`create-next-app`](/docs/app/api-reference/cli/create-next-app)，它會為您自動設定所有內容。要建立專案，請執行：

```bash package="pnpm"
pnpm create next-app
```

```bash package="npm"
npx create-next-app@latest
```

```bash package="yarn"
yarn create next-app
```

```bash package="bun"
bun create next-app
```

安裝過程中，您將看到以下提示：

```txt filename="Terminal"
What is your project named? my-app
Would you like to use the recommended Next.js defaults?
    Yes, use recommended defaults - TypeScript, ESLint, Tailwind CSS, App Router, AGENTS.md
    No, reuse previous settings
    No, customize settings - Choose your own preferences
```

如果您選擇 `自訂設定`，您將看到以下提示：

```txt filename="Terminal"
Would you like to use TypeScript? No / Yes
Which linter would you like to use? ESLint / Biome / None
Would you like to use React Compiler? No / Yes
Would you like to use Tailwind CSS? No / Yes
Would you like your code inside a `src/` directory? No / Yes
Would you like to use App Router? (recommended) No / Yes
Would you like to customize the import alias (`@/*` by default)? No / Yes
What import alias would you like configured? @/*
Would you like to include AGENTS.md to guide coding agents to write up-to-date Next.js code? No / Yes
```

提示結束後，[`create-next-app`](/docs/app/api-reference/cli/create-next-app) 會建立包含您專案名稱的資料夾並安裝所需的相依套件。

## 手動安裝

若要手動建立新的 Next.js 應用程式，請安裝所需的套件：

```bash package="pnpm"
pnpm i next@latest react@latest react-dom@latest
```

```bash package="npm"
npm i next@latest react@latest react-dom@latest
```

```bash package="yarn"
yarn add next@latest react@latest react-dom@latest
```

```bash package="bun"
bun add next@latest react@latest react-dom@latest
```

> **小提示**:
>
> - `App Router` 內建使用 [React canary releases](https://react.dev/blog/2023/05/03/react-canaries)，其中包括所有穩定的 React 19 變更，以及正在框架中驗證的新功能，但您仍需在 package.json 中聲明 react 和 react-dom 以確保工具及生態系的相容性。
> - `Pages Router` 使用您 `package.json` 中的 React 版本。

接著，將以下指令腳本新增到您的 `package.json` 檔案中：

```json filename="package.json"
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "eslint",
    "lint:fix": "eslint --fix"
  }
}
```

這些腳本參照應用程式開發的不同階段：

- `next dev`: 啟動使用 Turbopack 的開發伺服器（預設打包器）。
- `next build`: 建置應用程式以用於生產環境。
- `next start`: 啟動生產環境伺服器。
- `eslint`: 執行 ESLint。

Turbopack 現在是預設打包器。若要使用 Webpack，請執行 `next dev --webpack` 或 `next build --webpack`。請參閱 [Turbopack 文件](/docs/app/api-reference/turbopack) 以取得詳細配置資訊。

### 建立 `app` 目錄

Next.js 使用檔案系統路由，這意味著應用程式中的路由取決於您如何結構化檔案。

建立 `app` 資料夾。接著，在 `app` 內部建立 `layout.tsx` 檔案。此檔案是 [根佈局](/docs/app/api-reference/file-conventions/layout#root-layout)。這是必需的，且必須包含 `<html>` 和 `<body>` 標籤。

```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

建立首頁 `app/page.tsx` 並加入一些初始內容：

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

`layout.tsx` 和 `page.tsx` 在用戶造訪應用程式根目錄 (`/`) 時都會被渲染。

### 建立 `pages` 目錄

Next.js 使用檔案系統路由，這意味著應用程式中的路由取決於您如何結構化檔案。

在您的專案根目錄建立 `pages` 目錄。接著，在您的 `pages` 資料夾內新增 `index.tsx` 檔案。這將是您的首頁 (`/`)：

```tsx filename="pages/index.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="pages/index.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

接著，在 `pages/` 內新增 `_app.tsx` 檔案以定義全域佈局。請參閱 [自訂 App 檔案](/docs/pages/building-your-application/routing/custom-app) 以了解更多。

```tsx filename="pages/_app.tsx" switcher
import type { AppProps } from 'next/app'

export default function App({ Component, pageProps }: AppProps) {
  return <Component {...pageProps} />
}
```

```jsx filename="pages/_app.js" switcher
export default function App({ Component, pageProps }) {
  return <Component {...pageProps} />
}
```

最後，在 `pages/` 內新增 `_document.tsx` 檔案以控制伺服器的初始回應。請參閱 [自訂 Document 檔案](/docs/pages/building-your-application/routing/custom-document) 以了解更多。

```tsx filename="pages/_document.tsx" switcher
import { Html, Head, Main, NextScript } from 'next/document'

export default function Document() {
  return (
    <Html>
      <Head />
      <body>
        <Main />
        <NextScript />
      </body>
    </Html>
  )
}
```

```jsx filename="pages/_document.js" switcher
import { Html, Head, Main, NextScript } from 'next/document'

export default function Document() {
  return (
    <Html>
      <Head />
      <body>
        <Main />
        <NextScript />
      </body>
    </Html>
  )
}
```

</PagesOnly>

### 建立 `public` 資料夾（可選）

在您的專案根目錄建立 [`public` 資料夾](/docs/app/api-reference/file-conventions/public-folder) 以儲存靜態資源，例如圖片、字型等。接著，程式碼可以從根 URL (`/`) 開頭引用 `public` 內的檔案。

接著，您可以使用根路徑 (`/`) 引用這些資源。例如，`public/profile.png` 可以引用為 `/profile.png`：

```tsx filename="app/page.tsx" highlight={4} switcher
import Image from 'next/image'

export default function Page() {
  return <Image src="/profile.png" alt="Profile" width={100} height={100} />
}
```

```jsx filename="app/page.js" highlight={4} switcher
import Image from 'next/image'

export default function Page() {
  return <Image src="/profile.png" alt="Profile" width={100} height={100} />
}
```

## 執行開發伺服器

1. 執行 `npm run dev` 以啟動伺服器。
2. 造訪 `http://localhost:3000` 以檢視您的應用程式。
3. 編輯 <AppOnly>`app/page.tsx`</AppOnly><PagesOnly>`pages/index.tsx`</PagesOnly> 檔案並儲存，即可在瀏覽器中看到更新結果。

## 設定 TypeScript

> TypeScript 最低版本：`v5.1.0`

Next.js 內建 TypeScript 支援。要在專案中加入 TypeScript，請將檔案重新命名為 `.ts` / `.tsx` 並執行 `next dev`。Next.js 會自動安裝必要的相依套件，並新增一個包含建議組態選項的 `tsconfig.json` 檔案。

## 設定程式碼檢查

Next.js 支援使用 ESLint 或 Biome 進行程式碼檢查。選擇一個程式碼檢查器，並透過 `package.json` 腳本直接執行它。

- 使用 **ESLint**（完整的規則）：

```json filename="package.json"
{
  "scripts": {
    "lint": "eslint",
    "lint:fix": "eslint --fix"
  }
}
```

- 或使用 **Biome**（快速程式碼檢查器 + 格式化器）：

```json filename="package.json"
{
  "scripts": {
    "lint": "biome check",
    "format": "biome format --write"
  }
}
```

如果您的專案先前使用了 `next lint`，請使用 codemod 將腳本遷移至 ESLint CLI：

```bash filename="Terminal"
npx @next/codemod@canary next-lint-to-eslint-cli .
```

如果您使用 ESLint，請建立明確的設定檔（建議使用 `eslint.config.mjs`）。ESLint 支援 [舊版的 `.eslintrc.*` 格式和較新的 `eslint.config.mjs` 格式](https://eslint.org/docs/latest/use/configure/configuration-files#configuring-eslint)。請參閱 [ESLint API 參考](/docs/app/api-reference/config/eslint#with-core-web-vitals) 以獲得推薦的設定方式。

> **小提示**：從 Next.js 16 開始，`next build` 不再自動執行程式碼檢查器。相反地，您可以透過 NPM 腳本來執行您的程式碼檢查。

請參閱 [ESLint Plugin](/docs/app/api-reference/config/eslint) 頁面以取得更多資訊。

## 設定絕對匯入和模組路徑別名

Next.js 內建支援 `tsconfig.json` 和 `jsconfig.json` 檔案的 `"paths"` 和 `"baseUrl"` 選項。

這些選項允許您將專案目錄別名為絕對路徑，使匯入模組更方便且更清潔。例如：

```jsx
// Before
import { Button } from '../../../components/button'

// After
import { Button } from '@/components/button'
```

若要設定絕對匯入，請將 `baseUrl` 組態選項新增到您的 `tsconfig.json` 或 `jsconfig.json` 檔案中。例如：

```json filename="tsconfig.json or jsconfig.json"
{
  "compilerOptions": {
    "baseUrl": "src/"
  }
}
```

除了設定 `baseUrl` 路徑外，您還可以使用 `"paths"` 選項來 `"alias"` 模組路徑。

例如，以下設定將 `@/components/*` 映射到 `components/*`：

```json filename="tsconfig.json or jsconfig.json"
{
  "compilerOptions": {
    "baseUrl": "src/",
    "paths": {
      "@/styles/*": ["styles/*"],
      "@/components/*": ["components/*"]
    }
  }
}
```

每個 `"paths"` 均是相對於 `baseUrl` 位置。
