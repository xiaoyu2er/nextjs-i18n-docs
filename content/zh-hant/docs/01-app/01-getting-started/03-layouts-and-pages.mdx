Next.js 使用**基於檔案系統的路由 (file-system based routing)**，意味著您可以使用資料夾和檔案來定義路由。本頁面將引導您如何建立佈局 (layout) 和頁面 (page)，並在其間建立連結。

## 建立頁面 (Creating a page)

**頁面 (page)** 是在特定路由上渲染的 UI。要建立頁面，請在 `app` 目錄中新增 [`page` 檔案](/docs/app/api-reference/file-conventions/page)，並預設匯出 React 組件。例如，要建立索引頁面 (`/`)：

  srcLight="/docs/light/page-special-file.png"
  srcDark="/docs/dark/page-special-file.png"
  width="1600"

/>

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello Next.js!</h1>
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello Next.js!</h1>

```

## 建立佈局 (Creating a layout)

佈局 (layout) 是在多個頁面之間**共用**的 UI。在導航時，佈局會保留狀態、保持互動性，並且不會重新渲染。

您可以透過從 [`layout` 檔案](/docs/app/api-reference/file-conventions/layout) 預設匯出 React 組件來定義佈局。該組件應接受 `children` prop，它可以是頁面或另一個 [佈局 (layout)](#nesting-layouts)。

例如，要建立一個將您的索引頁面作為子元素的佈局，請在 `app` 目錄中新增 `layout` 檔案：

<Image
  alt="layout.js 特殊檔案 (layout.js special file)"
  srcLight="/docs/light/layout-special-file.png"
  srcDark="/docs/dark/layout-special-file.png"
  width="1600"
  height="363"
/>

```tsx filename="app/layout.tsx" switcher
export default function DashboardLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        {/* Layout UI */}
        {/* Place children where you want to render a page or nested layout */}
        <main>{children}</main>
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function DashboardLayout({ children }) {
  return (
    <html lang="en">
      <body>
        {/* Layout UI */}
        {/* Place children where you want to render a page or nested layout */}
        <main>{children}</main>
      </body>
    </html>
  )
}
```

上述佈局稱為 [根佈局 (root layout)](/docs/app/api-reference/file-conventions/layout#root-layout)，因為它是在 `app` 目錄的根位置定義的。根佈局是**必要**的，且必須包含 `html` 和 `body` 標籤。

## 建立巢狀路由 (Creating a nested route)

巢狀路由是由多個 URL 段構成的路由。例如，`/blog/[slug]` 路由由三個段組成：

- `/` (根段 (Root Segment))
- `blog` (段 (Segment))
- `[slug]` (葉段 (Leaf Segment))

在 Next.js 中：

- **物件夾 (Folders)** 用於定義對應到 URL 段的路由段。
- **檔案 (Files)** (如 `page` 和 `layout`) 用於建立顯示給該段的 UI。

要建立巢狀路由，您可以將資料夾互相巢狀。例如，要新增 `/blog` 的路由，請在 `app` 目錄中建立名為 `blog` 的資料夾。接著，要使 `/blog` 公開可存取，請新增 `page.tsx` 檔案：

<Image
  alt="顯示 blog 資料夾和 page.js 檔案的檔案階層結構 (File hierarchy showing blog folder and a page.js file)"
  srcLight="/docs/light/blog-nested-route.png"
  srcDark="/docs/dark/blog-nested-route.png"
  width="1600"
  height="525"
/>

```tsx filename="app/blog/page.tsx" switcher
// Dummy imports
import { getPosts } from '@/lib/posts'
import { Post } from '@/ui/post'

export default async function Page() {
  const posts = await getPosts()

  return (
    <ul>
      {posts.map((post) => (
        <Post key={post.id} post={post} />
      ))}
    </ul>
  )
}
```

```jsx filename="app/blog/[slug]/page.js" switcher

import { Post } from '@/ui/post'

export default async function Page() {
  const posts = await getPosts()

  return (
    <ul>
      {posts.map((post) => (
        <Post key={post.id} post={post} />
      ))}
    </ul>
  )
}
```

您可以繼續巢狀資料夾以建立巢狀路由。例如，要為特定部落格文章建立路由，請在 `blog` 內建立新的 `[slug]` 資料夾，並新增 `page` 檔案：

## 巢狀佈局 (Nesting layouts)

預設情況下，資料夾層級中的佈局也是巢狀的，這意味著它們透過 `children` prop 包裹子佈局。您可以在特定路由段（資料夾）內新增 `layout` 來巢狀佈局。

例如，要為 `/blog` 路由建立佈局，請在 `blog` 資料夾內新增 `layout` 檔案。

<Image
  alt="顯示根佈局包裹部落格佈局的檔案階層結構 (File hierarchy showing root layout wrapping the blog layout)"
  srcLight="/docs/light/nested-layouts.png"
  srcDark="/docs/dark/nested-layouts.png"
  width="1600"
  height="768"
/>

```tsx filename="app/blog/layout.tsx" switcher
export default function BlogLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return <section>{children}</section>
}
```

```jsx filename="app/blog/layout.js" switcher
export default function BlogLayout({ children }) {
  return <section>{children}</section>
}
```

如果您將上述兩個佈局組合在一起，根佈局 (`app/layout.js`) 將包裹部落格佈局 (`app/blog/layout.js`)，而部落格佈局將包裹部落格 (`app/blog/page.js`) 和部落格文章頁面 (`app/blog/[slug]/page.js`)。

## 建立動態區段 (Creating a dynamic segment)

[動態區段 (Dynamic segments)](/docs/app/api-reference/file-conventions/dynamic-routes) 允許您建立從資料生成的路由。例如，與其為每篇部落格文章手動建立路由，您可以建立動態區段以基於部落格文章數據生成路由。

要建立動態區段，請將區段（資料夾）名稱括在方括號中：`[segmentName]`。例如，在 `app/blog/[slug]/page.tsx` 路由中，`[slug]` 是動態區段。

```tsx filename="app/blog/[slug]/page.tsx" switcher
export default async function BlogPostPage({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  const post = await getPost(slug)

  return (
    <div>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </div>
  )
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
export default async function BlogPostPage({ params }) {
  const { slug } = await params
  const post = await getPost(slug)

  return (
    <div>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </div>
  )
}
```

了解更多關於 [動態區段 (Dynamic Segments)](/docs/app/api-reference/file-conventions/dynamic-routes) 和 [`params`](/docs/app/api-reference/file-conventions/page#params-optional) props 的內容。

巢狀 [動態區段內的佈局 (layouts within Dynamic Segments)](/docs/app/api-reference/file-conventions/layout#params-optional) 也可以存取 `params` props。

## 使用搜尋參數渲染 (Rendering with search params)

在服務器組件 (Server Component) **頁面 (page)** 中，您可以使用 [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) prop 存取搜尋參數：

```tsx filename="app/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
}
```

```jsx filename="app/page.jsx" switcher
export default async function Page({ searchParams }) {
  const filters = (await searchParams).filters
}
```

使用 `searchParams` 會讓您的頁面採用 [**動態渲染 (dynamic rendering)**](/docs/app/glossary#dynamic-rendering)，因為它需要依賴請求才能讀取搜尋參數。

客戶端組件 (Client Components) 可以使用 [`useSearchParams`](/docs/app/api-reference/functions/use-search-params) hook 讀取搜尋參數。

了解更多關於 `useSearchParams` 的資訊，適用於 [預渲染](/docs/app/api-reference/functions/use-search-params#prerendering) 和 [動態渲染](/docs/app/api-reference/functions/use-search-params#dynamic-rendering) 路由。

### 如何使用以及時機 (What to use and when)

- 當您需要搜尋參數來**載入頁面資料**時（例如：分頁、資料庫篩選），請使用 `searchParams` prop。
- 當搜尋參數**僅在客戶端**使用時（例如：篩選已透過 Props 載入的列表），請使用 `useSearchParams`。
- 作為小幅優化，您可以在**回呼或事件處理程式**中使用 `new URLSearchParams(window.location.search)` 來讀取搜尋參數而不會觸發重新渲染。

## 頁面之間的連結 (Linking between pages)

您可以使用 [`<Link>` 組件](/docs/app/api-reference/components/link) 在路由之間進行導航。<Link> 是 Next.js 內建的組件，擴充了 HTML `<a>` 標籤，以提供 [預取](/docs/app/getting-started/linking-and-navigating#prefetching) 和 [客戶端導航](/docs/app/getting-started/linking-and-navigating#client-side-transitions)。

例如，要生成部落格文章列表，請從 `next/link` 匯入 `<Link>`，並將 `href` prop 傳遞給組件：

```tsx filename="app/ui/post.tsx" highlight={1,10} switcher
import Link from 'next/link'

export default async function Post({ post }) {
  const posts = await getPosts()

  return (
    <ul>
      {posts.map((post) => (
        <li key={post.slug}>
          <Link href={`/blog/${post.slug}`}>{post.title}</Link>
        </li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/ui/post.js" highlight={1,10}  switcher
import Link from 'next/link'

export default async function Post({ post }) {
  const posts = await getPosts()

  return (
    <ul>
      {posts.map((post) => (
        <li key={post.slug}>
          <Link href={`/blog/${post.slug}`}>{post.title}</Link>
        </li>
      ))}
    </ul>
  )
}
```

> **值得了解 (Good to know)**: `<Link>` 是在 Next.js 中在路由之間導航的主要方式。您也可以使用 [`useRouter` hook](/docs/app/api-reference/functions/use-router) 進行更進階的導航。

## 路由 Props 輔助工具 (Route Props Helpers)

Next.js 暴露輔助類型，可從您的路由結構中推論 `params` 和有名稱的插槽 (named slots)：

- [**PageProps**](/docs/app/api-reference/file-conventions/page#page-props-helper)：`page` 組件的 Props，包括 `params` 和 `searchParams`。
- [**LayoutProps**](/docs/app/api-reference/file-conventions/layout#layout-props-helper)：`layout` 組件的 Props，包括 `children` 及任何有名稱的插槽（例如：資料夾如 `@analytics`）。

這些是全域可用的輔助工具，在執行 `next dev`、`next build` 或 [`next typegen`](/docs/app/api-reference/cli/next#next-typegen-options) 時生成。

```tsx filename="app/blog/[slug]/page.tsx"
export default async function Page(props: PageProps<'/blog/[slug]'>) {
  const { slug } = await props.params
  return <h1>Blog post: {slug}</h1>
}
```

```tsx filename="app/dashboard/layout.tsx"
export default function Layout(props: LayoutProps<'/dashboard'>) {
  return (
    <section>
      {props.children}
      {/* If you have app/dashboard/@analytics, it appears as a typed slot: */}
      {/* {props.analytics} */}
    </section>
  )
}
```

> **值得了解 (Good to know)**
>
> - 靜態路由將 `params` 解析為 `{}`。
> - `PageProps`, `LayoutProps` 是全域輔助工具 — 不需要匯入。
> - 類型是在 `next dev`, `next build` 或 `next typegen` 期間生成的。
