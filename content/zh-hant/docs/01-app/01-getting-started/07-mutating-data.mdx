您可以在 Next.js 中使用 [React Server Functions](https://react.dev/reference/rsc/server-functions) 來變更資料。本頁將介紹如何 [建立](#creating-server-functions) 和 [呼叫](#invoking-server-functions) 伺服器函式 (Server Functions)。

## 伺服器函數 (Server Functions) 是什麼？

一個 **伺服器函式 (Server Function)** 是在 伺服器 上執行的非同步函式。您可以透過 網路請求 從客戶端呼叫它們，這就是為什麼它們必須是非同步的。

在 \`action\` 或變動的上下文中，它們也被稱為 **伺服器動作 (Server Actions)**。

根據慣例，伺服器動作 (Server Action) 是一個配合 [`startTransition`](https://react.dev/reference/react/startTransition) 使用的非同步函數。當函數符合下列情況時，這會自動發生：

- 透過 \`action\` prop 傳遞給 \`<form>\`。
- 透過 \`formAction\` prop 傳遞給 \`<button>\`。

當動作被呼叫時，Next.js 可以在單一 伺服器往返中同時傳回已更新的 UI 和 新資料。

在後端，動作使用 \`POST\` 方法，且只有這 HTTP 方法能呼叫它們。

> [!警告]
> 伺服器函式 (Server Functions) 可透過直接 POST 請求存取，而不僅限於您的應用程式 UI。請務必在每個 伺服器函式 (Server Function) 內驗證 驗證與授權。請參閱 [資料安全指南](/docs/app/guides/data-security#authentication-and-authorization) 以獲取建議的模式。

> **值得了解**：伺服器動作 (Server Action) 是 伺服器函式 (Server Function) 以特定方式使用（用於處理 表單 提交和變更）。伺服器函式 (Server Function) 是更廣泛的術語。

## 建立 伺服器函數 (Server Functions)

可以使用 [`use server`](https://react.dev/reference/rsc/use-server) 指示字元來定義伺服器函數 (Server Function)。您可以將指示字元放在 **非同步** 函數的開頭，以將該函數標記為伺服器函數 (Server Function)，或放在獨立檔案的開頭，以將該檔案的所有匯出標記為伺服器函數。

```ts filename="app/lib/actions.ts" switcher
import { auth } from '@/lib/auth'

export async function createPost(formData: FormData) {
  'use server'
  const session = await auth()
  if (!session?.user) {
    throw new Error('Unauthorized')
  }

  const title = formData.get('title')
  const content = formData.get('content')

  // Mutate data
  // Revalidate cache
}

export async function deletePost(formData: FormData) {

    throw new Error('Unauthorized')
  }

  const id = formData.get('id')
```

### 伺服器元件

伺服器函數 (Server Functions) 可以透過將 "use server" 指示字元加到函數體的開頭，內嵌在伺服器元件 (Server Components) 中：

```tsx filename="app/page.tsx" switcher
export default function Page() {
  // Server Action
  async function createPost(formData: FormData) {
    'use server'
    // ...
  }

  return <></>
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  // Server Action
  async function createPost(formData) {
    'use server'
    // ...
  }

  return <></>
}
```

> **值得了解：** 伺服器元件 (Server Components) 預設支援增強功能，這意味著即使 JavaScript 尚未載入或已停用，呼叫伺服器動作 (Server Actions) 的表單仍會被提交。

### 客戶端元件

無法在客戶端元件 (Client Components) 中定義伺服器函數 (Server Functions)。不過，您可以從頂部有 "use server" 指示字元的檔案匯入它們，從而在客戶端元件 (Client Components) 中呼叫它們：

```ts filename="app/actions.ts" switcher
'use server'

export async function createPost() {}
```

```js filename="app/actions.js" switcher
'use server'

export async function createPost() {}
```

```tsx filename="app/ui/button.tsx" switcher
'use client'

import { createPost } from '@/app/actions'

export function Button() {
  return <button formAction={createPost}>Create</button>
}
```

```jsx filename="app/ui/button.js" switcher
'use client'

import { createPost } from '@/app/actions'

export function Button() {
  return <button formAction={createPost}>Create</button>
}
```

> **值得了解：** 在客戶端元件 (Client Components) 中，如果 JavaScript 尚未載入，呼叫伺服器動作 (Server Actions) 的表單將佇列化提交，並將被優先處理以進行水合。水合後，瀏覽器在表單提交時不會重新整理。

### 將動作作為 Props 傳遞

您也可以將動作作為 prop 傳遞給客戶端元件：

```jsx
<ClientComponent updateItemAction={updateItem} />
```

```tsx filename="app/client-component.tsx" switcher
'use client'

export default function ClientComponent({
  updateItemAction,
}: {
  updateItemAction: (formData: FormData) => void
}) {
  return <form action={updateItemAction}>{/* ... */}</form>
}
```

```jsx filename="app/client-component.js" switcher
'use client'

export default function ClientComponent({ updateItemAction }) {
  return <form action={updateItemAction}>{/* ... */}</form>
}
```

## 調用 伺服器函數 (Server Functions)

您調用 伺服器函數 (Server Functions) 主要有兩種方式：

1. 伺服器與客戶端元件中的 [表單](#forms)
2. 客戶端元件中的 [事件處理器](#event-handlers) 與 [useEffect](#useeffect)

> **值得了解：** 伺服器函數 (Server Functions) 是為伺服器端變動設計的。客戶端目前一次分派並等待一次。這是實現細節，可能會更改。如果您需要並行資料獲取，請在伺服器元件 (Server Components) 中使用 [資料獲取](/docs/app/getting-started/fetching-data#server-components)，或單個伺服器函數 (Server Function) 或 [Route Handler](/docs/app/guides/backend-for-frontend#manipulating-data) 內執行並行工作。

### 表單

React 擴充 了 HTML [`<form>`](https://react.dev/reference/react-dom/components/form) 元素，允許透過 HTML action 屬性來呼叫 伺服器函式 (Server Function)。

當在表單中呼叫時，該函數會自動接收 [`FormData`](https://developer.mozilla.org/docs/Web/API/FormData/FormData) 物件。您可以使用原生的 [`FormData` 方法](https://developer.mozilla.org/en-US/docs/Web/API/FormData#instance_methods) 來提取資料：

```tsx filename="app/ui/form.tsx" switcher
import { createPost } from '@/app/actions'

export function Form() {
  return (
    <form action={createPost}>
      <input type="text" name="title" />
      <input type="text" name="content" />
      <button type="submit">Create</button>
    </form>
  )
}
```

### 事件處理器

您可以透過使用如 \`onClick\` 的事件處理器，在客戶端元件中呼叫 伺服器函數 (Server Function)。

```tsx filename="app/like-button.tsx" switcher
'use client'

import { incrementLike } from './actions'
import { useState } from 'react'

export default function LikeButton({ initialLikes }: { initialLikes: number }) {
  const [likes, setLikes] = useState(initialLikes)

  return (
    <>
      <p>Total Likes: {likes}</p>
      <button
        onClick={async () => {
          const updatedLikes = await incrementLike()
          setLikes(updatedLikes)
        }}
      >
        Like
      </button>
    </>
  )
}
```


## 範例

### 顯示掛起狀態

執行 伺服器函數 (Server Function) 時，您可以使用 React 的 [`useActionState`](https://react.dev/reference/react/useActionState) 顯示載入指示器。此 hook 回傳一個 \`pending\` 布林值：

```tsx filename="app/ui/button.tsx" switcher
'use client'

import { useActionState, startTransition } from 'react'
import { createPost } from '@/app/actions'
import { LoadingSpinner } from '@/app/ui/loading-spinner'

export function Button() {
  const [state, action, pending] = useActionState(createPost, false)

  return (
    <button onClick={() => startTransition(action)}>
      {pending ? <LoadingSpinner /> : 'Create Post'}
    </button>
  )
}
```

```jsx filename="app/ui/button.js" switcher
'use client'

import { useActionState, startTransition } from 'react'
import { createPost } from '@/app/actions'
import { LoadingSpinner } from '@/app/ui/loading-spinner'

export function Button() {
  const [state, action, pending] = useActionState(createPost, false)

  return (
    <button onClick={() => startTransition(action)}>
      {pending ? <LoadingSpinner /> : 'Create Post'}
    </button>
  )
}
```

### 重新整理資料

變更後，您可能希望 重新整理 目前頁面 以顯示 最新 資料。您可以在 伺服器動作 (Server Action) 中 從 `next/cache` 呼叫 [`refresh`](/docs/app/api-reference/functions/refresh) 來達成此目的：

```ts filename="app/lib/actions.ts" switcher
'use server'

import { auth } from '@/lib/auth'
import { refresh } from 'next/cache'

export async function updatePost(formData: FormData) {
  const session = await auth()
  if (!session?.user) {
    throw new Error('Unauthorized')
  }
  // Mutate data
  // ...

  refresh()
}
```

```js filename="app/lib/actions.js" switcher
'use server'

import { auth } from '@/lib/auth'
import { refresh } from 'next/cache'

export async function updatePost(formData) {
  const session = await auth()
  if (!session?.user) {
    throw new Error('Unauthorized')
  }
  // Mutate data
  // ...

  refresh()
}
```

這會 重新整理 客戶端 Router，確保 UI 反映 最新 狀態。refresh() 函式 不會 重新驗證 標籤數據。要重新驗證 標籤數據，請改用 [`updateTag`](/docs/app/api-reference/functions/updateTag) 或 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag)。

### 重新驗證資料

執行變更後，您可以在 伺服器函式 (Server Function) 中呼叫 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 或 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag) 來重新驗證 Next.js 快取 並顯示 已更新的 資料：

```ts filename="app/lib/actions.ts" switcher
import { auth } from '@/lib/auth'
import { revalidatePath } from 'next/cache'

export async function createPost(formData: FormData) {
  'use server'
  const session = await auth()
  if (!session?.user) {
    throw new Error('Unauthorized')
  }
  // Mutate data
  // ...

  revalidatePath('/posts')
}
```

```js filename="app/actions.js" switcher
import { auth } from '@/lib/auth'
import { revalidatePath } from 'next/cache'

export async function createPost(formData) {
  'use server'
  const session = await auth()
  if (!session?.user) {
    throw new Error('Unauthorized')
  }
  // Mutate data
  // ...
  revalidatePath('/posts')
}
```

### 變更後重新導向

變更後，您可能希望將 使用者 重新導向 到 不同的 頁面。您可以在 伺服器函式 (Server Function) 中呼叫 [`redirect`](/docs/app/api-reference/functions/redirect) 來達成此目的。

```ts filename="app/lib/actions.ts" switcher
'use server'

import { auth } from '@/lib/auth'
import { revalidatePath } from 'next/cache'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  const session = await auth()
  if (!session?.user) {
    throw new Error('Unauthorized')
  }
  // Mutate data
  // ...

  revalidatePath('/posts')
  redirect('/posts')
}
```

```js filename="app/actions.js" switcher
'use server'

import { auth } from '@/lib/auth'
import { revalidatePath } from 'next/cache'
import { redirect } from 'next/navigation'

export async function createPost(formData) {
  const session = await auth()
  if (!session?.user) {
    throw new Error('Unauthorized')
  }
  // Mutate data
  // ...

  revalidatePath('/posts')
  redirect('/posts')
}
```

呼叫 `redirect` [會發出](/docs/app/api-reference/functions/redirect#behavior) 一個由框架處理的控制流程例外。其後的任何程式碼都不會執行。如果您需要新資料，請事先呼叫 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 或 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag)。

### Cookie (Cookies)

您可以在 Server Action 內部使用 [`cookies`](/docs/app/api-reference/functions/cookies) API 來 `get`、`set` 和 `delete` cookies。

當您在 伺服器動作 (Server Action) 中 [設定或刪除](/docs/app/api-reference/functions/cookies#understanding-cookie-behavior-in-server-functions) Cookie 時，Next.js 會在 伺服器 上 重新渲染 目前 頁面 及其佈局，以便 **UI 反映 新的 Cookie 值**。

> **值得了解**：伺服器更新適用於目前的 React 樹系，依需要重新重繪、掛載或解除掛載元件。已重新重繪的元件的客戶端狀態得以保留，若其相依性變更，則效果會重新執行。

```ts filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function exampleAction() {
  const cookieStore = await cookies()

  // Get cookie
  cookieStore.get('name')?.value

  // Set cookie
  cookieStore.set('name', 'Delba')

  // Delete cookie
  cookieStore.delete('name')
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function exampleAction() {
  // Get cookie
  const cookieStore = await cookies()

  // Get cookie
  cookieStore.get('name')?.value

  // Set cookie
  cookieStore.set('name', 'Delba')

  // Delete cookie
  cookieStore.delete('name')
}
```

### useEffect

您可以使用 React [`useEffect`](https://react.dev/reference/react/useEffect) hook 在元件掛載或相依性變更時呼叫 伺服器動作 (Server Action)。這對於依賴全端事件或需要自動觸發的變動很有用。例如，用於應用程式快捷鍵的 `onKeyDown`、用於無限捲動的 intersection observer hook，或是在元件掛載時更新檢視計數：

```tsx filename="app/view-count.tsx" switcher
'use client'

import { incrementViews } from './actions'
import { useState, useEffect, useTransition } from 'react'

export default function ViewCount({ initialViews }: { initialViews: number }) {
  const [views, setViews] = useState(initialViews)
  const [isPending, startTransition] = useTransition()

  useEffect(() => {
    startTransition(async () => {
      const updatedViews = await incrementViews()
      setViews(updatedViews)
    })
  }, [])

  // You can use `isPending` to give users feedback
  return <p>Total Views: {views}</p>
}
```

