> 此頁面涵蓋了使用 [快取組件 (Cache Components)](/docs/app/api-reference/config/next-config-js/cacheComponents) 的重新驗證，這需要先將 [`cacheComponents: true`](/docs/app/api-reference/config/next-config-js/cacheComponents) 設定在您 `next.config.ts` 檔案中來啟用。如果您沒有使用 快取組件 (Cache Components)，請參閱 [快取與重新驗證（舊模型）](/docs/app/guides/caching-without-cache-components) 指南。

重新驗證是更新快取資料的流程。它讓您繼續提供快速、快取的回應，同時確保內容保持新鮮。有以下兩種策略：

- **時間基礎重新驗證**：使用 [`cacheLife`](#cachelife) 在設定的時間間隔後自動更新快取資料。
- **按需重新驗證**：在執行變更後，利用 [`revalidateTag`](#revalidatetag)、[`updateTag`](#updatetag) 或 [`revalidatePath`](#revalidatepath) 手動使快取資料無效。

## `cacheLife`

[`cacheLife`](/docs/app/api-reference/functions/cacheLife) 控制快取資料有效的時間。在 [`use cache`](/docs/app/api-reference/directives/use-cache) 範圍內使用它來設定快取生命週期。

```tsx filename="app/lib/data.ts" highlight={1,4,5}
import { cacheLife } from 'next/cache'

export async function getProducts() {
  'use cache'
  cacheLife('hours')
  return db.query('SELECT * FROM products')
}
```

`cacheLife` 接受配置文件名稱或自定義配置對象：

| 配置文件 | 過期時間 (stale) | 重新驗證時間 (revalidate) | 失效時間 (expire)    |
| --------- | ------- | ------------ | ----------- |
| `seconds` | 0       | 1s           | 60s         |
| `minutes` | 5m      | 1m           | 1h          |
| `hours`   | 5m      | 1h           | 1d          |
| `days`    | 5m      | 1d           | 1w          |
| `weeks`   | 5m      | 1w           | 30d         |
| `max`     | 5m      | 30d          | ~無限期 |

為了進行精細化控制，請傳遞一個對象：

```tsx highlight={2-6}
'use cache'
cacheLife({
  stale: 3600, // 1 hour until considered stale
  revalidate: 7200, // 2 hours until revalidated
  expire: 86400, // 1 day until expired
})
```

> **值得了解：** 當快取使用 `seconds` 配置文件、`revalidate: 0` 或 `expire` 小於 5 分鐘時，被視為「短暫使用的 (short-lived)」。短暫使用的快取會自動被排除在靜態生成之外，並變成動態空隙。請詳細參閱 [靜態生成行為 (Prerendering behavior)](/docs/app/api-reference/functions/cacheLife#prerendering-behavior)。

請參閱 [`cacheLife`](/docs/app/api-reference/functions/cacheLife) API 參考文檔以獲得所有配置文件和自定義配置選項。

## `cacheTag`

[`cacheTag`](/docs/app/api-reference/functions/cacheTag) 讓您標記快取資料以便按需無效化。請在 [`use cache`](/docs/app/api-reference/directives/use-cache) 範圍內使用它：

```tsx filename="app/lib/data.ts" switcher
import { cacheTag } from 'next/cache'

export async function getProducts() {
  'use cache'
  cacheTag('products')
  return db.query('SELECT * FROM products')
}
```

```jsx filename="app/lib/data.js" switcher
import { cacheTag } from 'next/cache'

export async function getProducts() {
  'use cache'
  cacheTag('products')
  return db.query('SELECT * FROM products')
}
```

標記後，使用 [`revalidateTag`](#revalidatetag) 或 [`updateTag`](#updatetag) 使快取無效。

請參閱 [`cacheTag`](/docs/app/api-reference/functions/cacheTag) API 參考文檔以了解更多。

## `revalidateTag` (重新驗證標籤)

`revalidateTag` 使用過期同時重新驗證 (stale-while-revalidate) 機制，按標籤使快取條目無效。這意味著過期內容會立即提供，同時新內容在背景載入。這非常適合更新可以稍作延遲的內容，例如部落格文章或產品目錄。

```tsx filename="app/lib/actions.ts" highlight={1,5} switcher
import { revalidateTag } from 'next/cache'

export async function updateUser(id: string) {
  // Mutate data
  revalidateTag('user', 'max') // Recommended: stale-while-revalidate
}
```

```jsx filename="app/lib/actions.js" highlight={1,5} switcher
import { revalidateTag } from 'next/cache'

export async function updateUser(id) {
  // Mutate data
  revalidateTag('user', 'max') // Recommended: stale-while-revalidate
}
```

您可以在多個函數中重用相同的標籤以一次性重新驗證它們。在 [伺服器動作 (Server Action)](/docs/app/getting-started/mutating-data) 或 [路由處理器 (Route Handler)](/docs/app/api-reference/file-conventions/route) 中呼叫 `revalidateTag`。

> **值得了解：** 第二個參數設定過期內容可以在背景載入新內容時被提供的時間長度。一旦過期，後續請求將停止直到新內容準備就緒。使用 `'max'` 可提供最長的過期時間窗口。

請參閱 [`revalidateTag` API 參考](/docs/app/api-reference/functions/revalidateTag) 以了解更多。

## `updateTag`

`updateTag` 立即使快取資料過期，適用於讀你剛寫 (Read-your-own-writes) 場景。這意味著用戶會立即看到他們的更改，而不看到過期內容。與 `revalidateTag` 不同，它只能在 [伺服器動作 (Server Actions)](/docs/app/getting-started/mutating-data) 中使用。

```tsx filename="app/lib/actions.ts" highlight={1,6} switcher
import { updateTag } from 'next/cache'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  const post = await db.post.create({
    data: {
      title: formData.get('title'),
      content: formData.get('content'),
    },
  })

  updateTag('posts')
  redirect(`/posts/${post.id}`)
}
```

```jsx filename="app/lib/actions.js" highlight={1,6} switcher
import { updateTag } from 'next/cache'
import { redirect } from 'next/navigation'

export async function createPost(formData) {
  const post = await db.post.create({
    data: {
      title: formData.get('title'),
      content: formData.get('content'),
    },
  })

  updateTag('posts')
  redirect(`/posts/${post.id}`)
}
```

|              | `updateTag`                                   | `revalidateTag`                      |
| ------------ | --------------------------------------------- | ------------------------------------ |
| **位置**    | 僅限伺服器動作 (Server Actions)                           | 伺服器動作和路由處理器 (Route Handlers)    |
| **行為** | 立即使快取過期                     | 過期同時重新驗證 (stale-while-revalidate)               |
| **使用場景** | 讀你剛寫 (Read-your-own-writes) | 背景重新整理 (可接受輕微延遲) |

請參閱 [`updateTag`](/docs/app/api-reference/functions/updateTag) API 參考文檔以了解更多。

## `revalidatePath` (重新驗證路由)

`revalidatePath` 使特定路由路徑的所有快取資料無效。當您想要重新驗證某個路由，而不知道哪些標籤與之相關聯時使用它。

```tsx filename="app/lib/actions.ts" highlight={1,5} switcher
import { revalidatePath } from 'next/cache'

export async function updateUser(id: string) {
  // Mutate data
  revalidatePath('/profile')
}
```

```jsx filename="app/lib/actions.js" highlight={1,5} switcher
import { revalidatePath } from 'next/cache'

export async function updateUser(id) {
  // Mutate data
  revalidatePath('/profile')
}
```

> **值得了解：** 如果可能，請優先使用基於標籤的重新驗證 (`revalidateTag` / `updateTag`) 而不是基於路徑的 —— 它更精確並可避免過度無效化。

請參閱 [`revalidatePath` API 參考](/docs/app/api-reference/functions/revalidatePath) 以了解更多。

## 我應該快取什麼？

快取不依賴 [執行階段資料 (runtime data)](/docs/app/getting-started/caching#working-with-runtime-apis) 的資料，並且您願意在一段時間內從快取提供。請使用 `cacheLife` 搭配 `use cache` 來描述此行為。

對於具有更新機制的内容管理系統，請使用具有較長快取持續時間的標籤，並在內容實際變更時依賴 `revalidateTag` 重新整理內容，而不是預先使快取過期。
