預設情況下，佈局 (layouts) 與頁面 (pages) 是 [伺服器元件 (Server Components)](https://react.dev/reference/rsc/server-components)，這讓您可以在伺服器上取得資料並渲染 UI 的部分，可選性地快取結果，然後串流傳輸到客戶端。當您需要互動性或瀏覽器 API 時，您可以使用 [客戶端元件 (Client Components)](https://react.dev/reference/rsc/use-client) 來疊加功能。

本頁解釋了 Next.js 中伺服器與客戶端元件的運作方式及使用時機，並包含如何將它們組合在您的應用程式中的範例。

## 何時使用伺服器元件 (Server Components) 與客戶端元件 (Client Components)?

客戶端和伺服器環境具有不同的功能。伺服器與客戶端元件允許您根據使用案例在各個環境中執行邏輯。

使用**客戶端元件 (Client Components)** 時您需要：

- [狀態 (State)](https://react.dev/learn/managing-state) 和 [事件處理程式 (event handlers)](https://react.dev/learn/responding-to-events)。例如 `onClick`、`onChange`。
- [生命週期邏輯 (lifecycle logic)](https://react.dev/learn/lifecycle-of-reactive-effects)。例如 `useEffect`。
- 僅限瀏覽器的 API。例如 `localStorage`、`window`、`Navigator.geolocation` 等。
- [自訂 Hook (Custom hooks)](https://react.dev/learn/reusing-logic-with-custom-hooks)。

使用**伺服器元件 (Server Components)** 時您需要：

- 從資料庫或接近來源的 API 取得資料。
- 使用 API 金鑰 (API keys)、Token 和其他機密，而不將其暴露給客戶端。
- 減少發送到瀏覽器的 JavaScript 量。
- 改進 [首次內容繪製 (First Contentful Paint (FCP))](https://web.dev/fcp/)，並逐漸串流內容給客戶端。

例如，`<Page>` 元件是一個從伺服器取得文章資料的伺服器元件，並將資料作為屬性 (Props) 傳遞給處理客戶端互動的 `<LikeButton>`。

```tsx filename="app/[id]/page.tsx" highlight={1,17} switcher
import LikeButton from '@/app/ui/like-button'
import { getPost } from '@/lib/data'

export default async function Page({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const post = await getPost(id)

  return (
    <div>
      <main>
        <h1>{post.title}</h1>
        {/* ... */}
        <LikeButton likes={post.likes} />
      </main>
    </div>
  )
}
```

```jsx filename="app/[id]/page.js" highlight={1,12} switcher
import LikeButton from '@/app/ui/like-button'
import { getPost } from '@/lib/data'

export default async function Page({ params }) {
  const post = await getPost(params.id)

  return (
    <div>
      <main>
        <h1>{post.title}</h1>
        {/* ... */}
        <LikeButton likes={post.likes} />
      </main>
    </div>
  )
}
```

```tsx filename="app/ui/like-button.tsx" highlight={1} switcher
'use client'

import { useState } from 'react'

export default function LikeButton({ likes }: { likes: number }) {
  // ...
}
```

```jsx filename="app/ui/like-button.js" highlight={1} switcher
'use client'

import { useState } from 'react'

export default function LikeButton({ likes }) {
  // ...
}
```

## 伺服器與客戶端元件 (Server and Client Components) 在 Next.js 中如何運作？

### 在伺服器上 (On the server)

在伺服器上，Next.js 使用 React 的 API 來協調渲染。渲染工作依據個別的路由區段（[佈局與頁面](/docs/app/getting-started/layouts-and-pages)）拆分成區塊：

- **伺服器組件 (Server Components)** 被渲染為一種特殊的資料格式，稱為 React 伺服器組件 Payload (RSC Payload)。
- **客戶端組件 (Client Components)** 與 RSC Payload 用於 [預先渲染](/docs/app/glossary#prerendering) HTML。

> **什麼是 React 伺服器元件負載 (React Server Component Payload / RSC)?**
>
> RSC Payload 是渲染後的 React 伺服器元件樹的緊湊二元表示。它由 React 在客戶端用於更新瀏覽器的 DOM。RSC Payload 包含：
>
> - 伺服器元件的渲染結果
> - 客戶端元件應在何處渲染的占位符，以及它們 JavaScript 檔案的參考
> - 從伺服器元件傳遞給客戶端元件的任何屬性 (Props)

### 在客戶端（初次載入） (On the client (first load))

接著，在客戶端：

1. **HTML** 用於立即向使用者顯示路由的快速非互動預覽。
2. **RSC Payload** 用於同步客戶端與伺服器元件樹。
3. **JavaScript** 用於對客戶端元件執行注入 (hydration) 並使應用程式具有互動性。

> **什麼是注入 (hydration)?**
>
> 注入 (hydration) 是 React 將 [事件處理程式 (event handlers)](https://react.dev/learn/responding-to-events) 附加到 DOM 以讓靜態 HTML 具有互動性的過程。

### 後續導覽 (Subsequent Navigations)

在後續導覽時：

- **RSC Payload** 會被預先取得並快取，以實現即時導覽。
- **客戶端元件 (Client Components)** 完全在客戶端渲染，沒有服務器渲染的 HTML。

## 範例

### 使用客戶端元件 (Client Components)

您可以在檔案頂部、匯入之前新增 [`"use client"`](https://react.dev/reference/react/use-client) 指示詞來建立客戶端元件。

```tsx filename="app/ui/counter.tsx" highlight={1} switcher
'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>{count} likes</p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  )
}
```

```jsx filename="app/ui/counter.js" highlight={1} switcher
'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>{count} likes</p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
    </div>
  )
}
```

`"use client"` 用於聲明伺服器與客戶端模組圖譜（樹）之間的**邊界 (boundary)**。

一旦檔案標記為 `"use client"`，**其所有匯入和子元件都被視為客戶端套件的一部分**。這意味著您不需要為每個旨在用於客戶端的元件新增此指示詞。

### 減少 JS 套件大小

為了減少客戶端 JavaScript 套件的體積，請將 `'use client'` 新增到特定的互動元件，而不是將您的 UI 大部分標記為客戶端元件。

例如，`<Layout>` 元件包含大部分靜態元素（如 Logo 和導航連結），但也包含互動式搜尋列。`<Search />` 是互動式的，需要成為客戶端元件，不過，佈局的其餘部分可以保持為伺服器元件。

```tsx filename="app/layout.tsx" highlight={12} switcher
// Client Component
import Search from './search'
// Server Component
import Logo from './logo'

// Layout is a Server Component by default
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <>
      <nav>
        <Logo />
        <Search />
      </nav>
      <main>{children}</main>
    </>
  )
}
```

```jsx filename="app/layout.js" highlight={12} switcher
// Client Component
import Search from './search'
// Server Component
import Logo from './logo'

// Layout is a Server Component by default
export default function Layout({ children }) {
  return (
    <>
      <nav>
        <Logo />
        <Search />
      </nav>
      <main>{children}</main>
    </>
  )
}
```

```tsx filename="app/ui/search.tsx" highlight={1} switcher
'use client'

export default function Search() {
  // ...
}
```

```jsx filename="app/ui/search.js" highlight={1} switcher
'use client'

export default function Search() {
  // ...
}
```

### 從伺服器傳遞資料到客戶端元件 (Client Components)

您可以使用屬性 (Props) 將資料從伺服器元件傳遞到客戶端元件。

```tsx filename="app/[id]/page.tsx" highlight={1,12} switcher
import LikeButton from '@/app/ui/like-button'
import { getPost } from '@/lib/data'

export default async function Page({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const post = await getPost(id)

  return <LikeButton likes={post.likes} />
}
```

```jsx filename="app/[id]/page.js" highlight={1,7} switcher
import LikeButton from '@/app/ui/like-button'
import { getPost } from '@/lib/data'

export default async function Page({ params }) {
  const post = await getPost(params.id)

  return <LikeButton likes={post.likes} />
}
```

```tsx filename="app/ui/like-button.tsx" highlight={1} switcher
'use client'

export default function LikeButton({ likes }: { likes: number }) {
  // ...
}
```

```jsx filename="app/ui/like-button.js" highlight={1} switcher
'use client'

export default function LikeButton({ likes }) {
  // ...
}
```

另外，您也可以使用 [`use` API](https://react.dev/reference/react/use) 從伺服器組件將資料串流至客戶端組件。請參閱 [範例](/docs/app/getting-started/fetching-data#streaming-data-with-the-use-api)。

> **了解這一點很好**：傳遞給客戶端元件的屬性 (Props) 需要可被 React [序列化 (serializable)](https://react.dev/reference/react/use-server#serializable-parameters-and-return-values)。

### 交錯使用伺服器與客戶端元件

您可以將伺服器元件作為屬性 (prop) 傳遞給客戶端元件。這允許您在客戶端元件中視覺嵌套伺服器渲染的 UI。

一個常見的模式是使用 `children` 在 `<ClientComponent>` 中建立 _槽 (slot)_。例如，一個在伺服器上取得資料的 `<Cart>` 元件，置於使用客戶端狀態來切換可視性的 `<Modal>` 元件內。

```tsx filename="app/ui/modal.tsx" switcher
'use client'

export default function Modal({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>
}
```

```jsx filename="app/ui/modal.js" switcher
'use client'

export default function Modal({ children }) {
  return <div>{children}</div>
}
```

接著，在父層伺服器元件中（例如 `<Page>`），您可以將 `<Cart>` 傳遞為 `<Modal>` 的子元件：

```tsx filename="app/page.tsx"  highlight={7} switcher
import Modal from './ui/modal'
import Cart from './ui/cart'

export default function Page() {
  return (
    <Modal>
      <Cart />
    </Modal>
  )
}
```

```jsx filename="app/page.js" highlight={7} switcher
import Modal from './ui/modal'
import Cart from './ui/cart'

export default function Page() {
  return (
    <Modal>
      <Cart />
    </Modal>
  )
}
```

在此模式中，所有伺服器元件都會提前在伺服器上渲染，包括那些作為屬性 (Props) 的元件。產生的 RSC Payload 將包含客戶端元件應在元件樹中何處渲染的參考內容。

### Context 提供者

[React Context](https://react.dev/learn/passing-data-deeply-with-context) 常用於共用全域狀態，例如目前的佈景主題。不過，React Context 在伺服器元件中不受支援。

要使用 Context，請建立一個接受 `children` 的客戶端元件：

```tsx filename="app/theme-provider.tsx" switcher
'use client'

import { createContext } from 'react'

export const ThemeContext = createContext({})

export default function ThemeProvider({
  children,
}: {
  children: React.ReactNode
}) {
  return <ThemeContext.Provider value="dark">{children}</ThemeContext.Provider>
}
```

```jsx filename="app/theme-provider.js" switcher
'use client'

import { createContext } from 'react'

export const ThemeContext = createContext({})

export default function ThemeProvider({ children }) {
  return <ThemeContext.Provider value="dark">{children}</ThemeContext.Provider>
}
```

接著，將其匯入到伺服器元件（例如 `layout`）：

```tsx filename="app/layout.tsx" switcher
import ThemeProvider from './theme-provider'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html>
      <body>
        <ThemeProvider>{children}</ThemeProvider>
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import ThemeProvider from './theme-provider'

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <ThemeProvider>{children}</ThemeProvider>
      </body>
    </html>
  )
}
```

您的伺服器元件現在將能夠直接渲染您的提供者，並且您的應用程式中所有其他客戶端元件將能夠使用此 Context。

> **了解這一點很好**：您應該盡可能將提供者渲染在樹的深度 — 注意 `ThemeProvider` 是如何只包裝 `{children}` 而不是整個 `<html>` 文件的。這使得 Next.js 更容易優化您的伺服器元件的靜態部分。

### 第三方元件

當使用依賴客戶端專屬功能的第三方元件時，您可以將其包裝在客戶端元件中，以確保它能正常運作。

例如，`<Carousel />` 可以從 `acme-carousel` 套件匯入。此元件使用 `useState`，但它還沒有 `"use client"` 指示詞。

如果您在客戶端元件中使用 `<Carousel />`，它將能正常運作：

```tsx filename="app/gallery.tsx" switcher
'use client'

import { useState } from 'react'
import { Carousel } from 'acme-carousel'

export default function Gallery() {
  const [isOpen, setIsOpen] = useState(false)

  return (
    <div>
      <button onClick={() => setIsOpen(true)}>View pictures</button>
      {/* Works, since Carousel is used within a Client Component */}
      {isOpen && <Carousel />}
    </div>
  )
}
```

```jsx filename="app/gallery.js" switcher
'use client'

import { useState } from 'react'
import { Carousel } from 'acme-carousel'

export default function Gallery() {
  const [isOpen, setIsOpen] = useState(false)

  return (
    <div>
      <button onClick={() => setIsOpen(true)}>View pictures</button>
      {/*  Works, since Carousel is used within a Client Component */}
      {isOpen && <Carousel />}
    </div>
  )
}
```

不過，如果您試著直接在伺服器元件中使用它，您將看到錯誤。這是因為 Next.js 不知道 `<Carousel />` 正在使用客戶端專屬功能。

要修復這個問題，您可以將依賴客戶端專屬功能的第三方元件包裝在您自己的客戶端元件中：

```tsx filename="app/carousel.tsx" switcher
'use client'

import { Carousel } from 'acme-carousel'

export default Carousel
```

```jsx filename="app/carousel.js" switcher
'use client'

import { Carousel } from 'acme-carousel'

export default Carousel
```

現在，您可以在伺服器元件中直接使用 `<Carousel />`：

```tsx filename="app/page.tsx" switcher
import Carousel from './carousel'

export default function Page() {
  return (
    <div>
      <p>View pictures</p>
      {/*  Works, since Carousel is a Client Component */}
      <Carousel />
    </div>
  )
}
```

```jsx filename="app/page.js" switcher
import Carousel from './carousel'

export default function Page() {
  return (
    <div>
      <p>View pictures</p>
      {/*  Works, since Carousel is a Client Component */}
      <Carousel />
    </div>
  )
}
```

> **給庫作者的建議**
>
> 若您在建立元件庫，請將 `"use client"` 指示詞新增到依賴客戶端專屬功能的入口點。這讓您的使用者能夠將元件匯入到伺服器元件中，而無需建立包裝器。
>
> 值得一提，某些打包器可能會移除 `"use client"` 指示詞。您可以在 [React Wrap Balancer](https://github.com/shuding/react-wrap-balancer/blob/main/tsup.config.ts#L10-L13) 和 [Vercel Analytics](https://github.com/vercel/analytics/blob/main/packages/web/tsup.config.js#L26-L30) 儲存庫中找到如何配置 esbuild 以包含 `"use client"` 指示詞的範例。

### 防止環境污染

JavaScript 模組可以在伺服器與客戶端元件模組之間共用。這意味著可能會意外將僅伺服器的程式碼匯入到客戶端。例如，考慮以下函式：

```ts filename="lib/data.ts" switcher
export async function getData() {
  const res = await fetch('https://external-service.com/data', {
    headers: {
      authorization: process.env.API_KEY,
    },
  })

  return res.json()
}
```

```js filename="lib/data.js" switcher
export async function getData() {
  const res = await fetch('https://external-service.com/data', {
    headers: {
      authorization: process.env.API_KEY,
    },
  })

  return res.json()
}
```

此函式包含一個 `API_KEY`，永遠不應將其暴露給客戶端。

在 Next.js 中，只有以前綴 `NEXT_PUBLIC_` 啟動的環境變數才會包含在客戶端套件中。如果變數沒有前綴，Next.js 會將它們替換為空字串。

因此，即使 `getData()` 可以在客戶端匯入並執行，它也不會如預期運作。

為了防止在客戶端元件中意外使用，您可以使用 [`server-only` 套件](https://www.npmjs.com/package/server-only)。

接著，將套件匯入到包含僅伺服器程式碼的檔案中：

```js filename="lib/data.js"
import 'server-only'

export async function getData() {
  const res = await fetch('https://external-service.com/data', {
    headers: {
      authorization: process.env.API_KEY,
    },
  })

  return res.json()
}
```

現在，如果您嘗試將模組匯入到客戶端元件中，將會出現建置時錯誤。

相應的 [`client-only` 套件](https://www.npmjs.com/package/client-only) 可用於標記包含客戶端專屬邏輯的模組，例如存取 `window` 物件的程式碼。

在 Next.js 中，安裝 `server-only` 或 `client-only` 是**可選的**。不過，如果您的 Linting 規則標記了多餘的相依性，您可以安裝它們以避免問題。

```bash package="npm"
npm install server-only
```

```bash package="yarn"
yarn add server-only
```

```bash package="pnpm"
pnpm add server-only
```

```bash package="bun"
bun add server-only
```

Next.js 內部處理 `server-only` 和 `client-only` 匯入，以便在模組在錯誤的環境中使用時提供較清晰的錯誤訊息。這些來自 NPM 的套件內容未被 Next.js 使用。

Next.js 也提供 `server-only` 和 `client-only` 的類型宣告，適用於啟用了 [`noUncheckedSideEffectImports`](https://www.typescriptlang.org/tsconfig/#noUncheckedSideEffectImports) 的 TypeScript 設定。
