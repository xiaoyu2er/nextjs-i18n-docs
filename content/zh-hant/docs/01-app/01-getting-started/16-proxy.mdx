## 代理 (Proxy)

> **小提示**：從 Next.js 16 開始，Middleware 現在稱為 Proxy 以更準確地反映其用途。功能保持不變。

Proxy 允許您在請求完成之前執行程式碼。接著，根據輸入請求，您可以透過重寫、重定向、修改請求或響應標頭，或直接回應來修改響應。

### 使用案例

Proxy 有效的常見場景包括：

- 修改所有頁面或部分頁面的標頭
- 根據 A/B 測試或實驗重寫至不同頁面
- 基於輸入請求屬性進行程式化重定向

對於簡單的重定向，請先考慮使用 `next.config.ts` 中的 [`redirects`](/docs/app/api-reference/config/next-config-js/redirects) 配置。當您需要存取請求資料或處理更複雜的邏輯時，才應使用 Proxy。

Proxy _不_ 適用於耗時的資料獲取。雖然 Proxy 有助於 [樂觀檢查 (Optimistic Checks)](/docs/app/guides/authentication#optimistic-checks-with-proxy-optional)（例如基於權限的重定向），但它不應用作完整的會話管理或授權解決方案。

在 Proxy 中使用帶有 `options.cache`、`options.next.revalidate` 或 `options.next.tags` 的 fetch 無效。

### 慣例

在專案根目錄或 `src`（如適用）內建立 `proxy.ts`（或 `.js`）檔案，使其與 `pages` 或 `app` 位於同一層級。

> **注意**：雖然每個專案只支援一個 `proxy.ts` 檔案，您仍可將您的 proxy 邏輯組織成模組。將 proxy 功能拆分到單獨的 `.ts` 或 `.js` 檔案中，並匯入主 `proxy.ts` 檔案。這讓您可以更清晰地管理路徑特定的 proxy，將其匯總在 `proxy.ts` 中以進行集中控制。強制實施單一 proxy 檔案可簡化配置，防止潛在衝突，並透過避免多個 proxy 層來優化效能。

### 範例

您可以將您的 proxy 函數導出為預設導出或命名的 `proxy` 導出：

```ts filename="proxy.ts" switcher
import { NextResponse } from 'next/server'
import type { NextRequest } from 'next/server'

// This function can be marked `async` if using `await` inside
export function proxy(request: NextRequest) {
  return NextResponse.redirect(new URL('/home', request.url))
}

// Alternatively, you can use a default export:
// export default function proxy(request: NextRequest) { ... }

export const config = {
  matcher: '/about/:path*',
}
```

```js filename="proxy.js" switcher
import { NextResponse } from 'next/server'

// This function can be marked `async` if using `await` inside
export function proxy(request) {
  return NextResponse.redirect(new URL('/home', request.url))
}

// Alternatively, you can use a default export:
// export default function proxy(request) { ... }

export const config = {
  matcher: '/about/:path*',
}
```

`matcher` 配置允許您篩選 Proxy 在特定路徑上執行。有關路徑匹配的更多詳細資訊，請參閱 [比對器 (Matcher)](/docs/app/api-reference/file-conventions/proxy#matcher) 文件。

請閱讀關於 [使用 `proxy`](/docs/app/guides/backend-for-frontend#proxy) 的更多內容，或參考 `proxy` [API 參考](/docs/app/api-reference/file-conventions/proxy)。
