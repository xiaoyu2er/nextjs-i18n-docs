本頁將引導您了解如何在 [伺服器 (Server)](#server-components) 和 [客戶端 (Client)](#client-components) 元件中獲取數據，以及如何 [串流 (stream)](#streaming) 那些依賴未緩衝數據的元件。

## 獲取資料

### 伺服器元件

您可以在伺服器元件 (Server Components) 中使用任何異步 I/O 獲取數據，例如：

1. [`fetch` API](#with-the-fetch-api)
2. [ORM 或資料庫](#with-an-orm-or-database)

#### 使用 `fetch` API

要使用 `fetch` API 獲取資料，請將您的元件轉換為非同步函式，並等待 `fetch` 呼叫。例如：

```tsx filename="app/blog/page.tsx" switcher
export default async function Page() {
  const data = await fetch('https://api.vercel.app/blog')
  const posts = await data.json()
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/blog/page.js" switcher
export default async function Page() {
  const data = await fetch('https://api.vercel.app/blog')
  const posts = await data.json()
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

> **好知識：**

> - React 元件樹中的相同 `fetch` 請求預設會 [被記憶化 (memoized)](/docs/app/glossary#memoization)，因此您可以在需要使用數據的元件中獲取數據，而不需要向下傳遞 props。
> - `fetch` 請求預設不會被緩衝，並且會阻塞頁面呈現直到請求完成。使用 [`use cache`](/docs/app/api-reference/directives/use-cache) 指令來緩衝結果，或者將獲取數據的元件包裹在 [`<Suspense>`](/docs/app/getting-started/caching#streaming-uncached-data) 中以在請求時間串流新數據。詳情請參見 [緩衝 (caching)](/docs/app/getting-started/caching)。
> - 在開發期間，您可以記錄 `fetch` 調用以提高可見性和除錯。請參見 [`logging` API 引用](/docs/app/api-reference/config/next-config-js/logging)。

#### 使用 ORM 或資料庫

由於伺服器元件 (Server Components) 是在伺服器上呈現，憑證和查詢邏輯不會包含在客戶端打包中，因此您可以使用 ORM 或數據庫客戶端安全地執行數據庫查詢。

```tsx filename="app/blog/page.tsx" switcher
import { db, posts } from '@/lib/db'

export default async function Page() {
  const allPosts = await db.select().from(posts)
  return (
    <ul>
      {allPosts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/blog/page.js" switcher
import { db, posts } from '@/lib/db'

export default async function Page() {
  const allPosts = await db.select().from(posts)
  return (
    <ul>
      {allPosts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

您仍應確保請求經過適當的認證和授權。有關保護伺服器端數據訪問的最佳實踐，請參見 [數據安全指南 (data security guide)](/docs/app/guides/data-security)。

### 串流

當您在伺服器元件 (Server Components) 中獲取數據時，數據會在每個請求時在伺服器上獲取並呈現。如果您有任何慢速數據請求，整個路由將在獲取所有數據之前無法呈現。

為了改善初始載入時間和用戶體驗，您可以將頁面分解為更小的 *塊 (chunks)* 並從伺服器漸進式地發送這些塊給客戶端。這稱為串流 (streaming)。請參見 [串流指南 (Streaming guide)](/docs/app/guides/streaming) 以更深入地了解串流的工作原理，包括 HTTP 協議、基礎設施考慮因素和性能權衡。

1. 使用 [`loading.js` 檔案](#with-loadingjs) 包圍頁面
2. 使用 [`<Suspense>`](#with-suspense) 包圍元件

#### 使用 `loading.js`

您可以在與頁面相同的資料夾中建立 `loading.js` 檔案，在獲取資料時串流 **整個頁面**。例如，要串流 `app/blog/page.js`，請將檔案新增到 `app/blog` 資料夾內。

```tsx filename="app/blog/loading.tsx" switcher
export default function Loading() {
  // Define the Loading UI here
  return <div>Loading...</div>
}
```

```jsx filename="app/blog/loading.js" switcher
export default function Loading() {
  // Define the Loading UI here

```

在導覽時，使用者會立即看到佈局和 [載入狀態](#creating-meaningful-loading-states)，而頁面正在渲染。待渲染完成後，新內容則會自動交換進來。

在幕後，`loading.js` 會 [嵌套在 `layout.js` 之中](/docs/app/getting-started/project-structure#component-hierarchy)，並自動將 `page.js` 文件和任何其子節點包裹在 `<Suspense>` 邊界中。

因此，存取未緩衝或執行期數據（例如 `cookies()`、`headers()` 或未緩衝的 fetch）的佈局不會回退到相同的路由段 `loading.js`。相反，它會阻塞導航直到佈局完成呈現。[緩衝元件 (Cache Components)](/docs/app/getting-started/caching) 透過在構建時錯誤來防止此情況。

若要修復此問題，請將未緩衝的存取單獨包裹在其自身的 [`<Suspense>`](#with-suspense) 邊界中並加上回退，或者將數據獲取移至 `page.js`，這樣 `loading.js` 就可以覆蓋它。請參見 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 了解更多詳情。

這就是為什麼雖然 `loading.js` 對於串流路由段效果良好，但建議在接近執行期或未緩衝數據存取的位置使用 `<Suspense>`。

#### 使用 `<Suspense>`

`<Suspense>` 讓您能更細緻地篩選要串流的頁面部分。例如，您可以立即顯示位於 `<Suspense>` 邊界之外的任何頁面內容，並將部落格文章清單串流至邊界內部。

```tsx filename="app/blog/page.tsx" switcher
import { Suspense } from 'react'
import BlogList from '@/components/BlogList'
import BlogListSkeleton from '@/components/BlogListSkeleton'

export default function BlogPage() {
  return (
    <div>
      {/* This content will be sent to the client immediately */}
      <header>
        <h1>Welcome to the Blog</h1>
        <p>Read the latest posts below.</p>
      </header>
      <main>
        {/* If there's any dynamic content inside this boundary, it will be streamed in */}
        <Suspense fallback={<BlogListSkeleton />}>
          <BlogList />
        </Suspense>
      </main>
    </div>
  )
}
```

```jsx filename="app/blog/page.js" switcher
import { Suspense } from 'react'
import BlogList from '@/components/BlogList'
import BlogListSkeleton from '@/components/BlogListSkeleton'

export default function BlogPage() {
  return (
    <div>
      {/* This content will be sent to the client immediately */}
      <header>
        <h1>Welcome to the Blog</h1>
        <p>Read the latest posts below.</p>
      </header>
      <main>
        {/* If there's any dynamic content inside this boundary, it will be streamed in */}
        <Suspense fallback={<BlogListSkeleton />}>
          <BlogList />
        </Suspense>
      </main>
    </div>
  )
}
```

#### 建立有意義的載入狀態

即時載入狀態是一種在導航後立即顯示給使用者的 fallback UI。為了提供最佳的使用者體驗，我們建議設計有意義的載入狀態，幫助使用者了解應用程式正在回應。例如，您可以使用骨架圖和旋轉器，或未來畫面中一小部分但有意義的內容，如封面照片、標題等。

在開發期間，您可以使用 [React Devtools](https://react.dev/learn/react-developer-tools) 預覽和檢查元件的載入狀態。

### 客戶端元件

在客戶端元件中有兩種獲取資料的方式：

1. React 的 [`use` API](https://react.dev/reference/react/use)
2. 社區庫如 [SWR](https://swr.vercel.app/) 或 [React Query](https://tanstack.com/query/latest)

#### 使用 `use` API 串流數據

您可以使用 React 的 [`use` API](https://react.dev/reference/react/use) 將數據從伺服器 [串流 (stream)](#streaming) 至客戶端。首先從您的伺服器元件 (Server component) 獲取數據，並將 promises 作為屬性傳遞給您的客戶端元件 (Client Component)：

```tsx filename="app/blog/page.tsx" switcher
import Posts from '@/app/ui/posts'
import { Suspense } from 'react'

export default function Page() {
  // Don't await the data fetching function
  const posts = getPosts()

  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Posts posts={posts} />
    </Suspense>
  )
}
```

```jsx filename="app/blog/page.js" switcher
import Posts from '@/app/ui/posts'
import { Suspense } from 'react'

export default function Page() {
  // Don't await the data fetching function
  const posts = getPosts()

  return (
    <Suspense fallback={<div>Loading...</div>}>
      <Posts posts={posts} />
    </Suspense>
  )
}
```

然後，在您的客戶端元件 (Client Component) 中，使用 `use` API 讀取 promise：

```tsx filename="app/ui/posts.tsx" switcher
'use client'
import { use } from 'react'

export default function Posts({
  posts,
}: {
  posts: Promise<{ id: string; title: string }[]>
}) {
  const allPosts = use(posts)

  return (
    <ul>
      {allPosts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/ui/posts.js" switcher
'use client'
import { use } from 'react'

export default function Posts({ posts }) {
  const allPosts = use(posts)

  return (
    <ul>
      {allPosts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

在上述範例中，`<Posts>` 元件被包在 [`<Suspense>`] 邊界](https://react.dev/reference/react/Suspense) 中。這意味著在 promise 正在解析時將顯示 fallback UI。請參閱 [串流](#streaming) 以了解更多。

#### 社群函式庫

您可以使用社群函式庫，如 [SWR](https://swr.vercel.app/) 或 [React Query](https://tanstack.com/query/latest) 在客戶端元件中獲取資料。這些函式庫有自己的快取、串流和其他功能的語義。例如，使用 SWR：

```tsx filename="app/blog/page.tsx" switcher
'use client'
import useSWR from 'swr'

const fetcher = (url) => fetch(url).then((r) => r.json())

export default function BlogPage() {
  const { data, error, isLoading } = useSWR(
    'https://api.vercel.app/blog',
    fetcher
  )

  if (isLoading) return <div>Loading...</div>
  if (error) return <div>Error: {error.message}</div>

  return (
    <ul>
      {data.map((post: { id: string; title: string }) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/blog/page.js" switcher
'use client'

import useSWR from 'swr'

const fetcher = (url) => fetch(url).then((r) => r.json())

export default function BlogPage() {
  const { data, error, isLoading } = useSWR(
    'https://api.vercel.app/blog',
    fetcher
  )

  if (isLoading) return <div>Loading...</div>
  if (error) return <div>Error: {error.message}</div>

  return (
    <ul>
      {data.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}
```

## 範例

### 順序資料獲取

序列式數據獲取發生在一個請求依賴另一個請求的數據時。

例如，`<Playlists>` 只有在 `<Artist>` 完成後才能獲取數據，因為它需要 `artistID`：

```tsx filename="app/artist/[username]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ username: string }>
}) {
  const { username } = await params
  // Get artist information
  const artist = await getArtist(username)

  return (
    <>
      <h1>{artist.name}</h1>
      {/* Show fallback UI while the Playlists component is loading */}
      <Suspense fallback={<div>Loading...</div>}>
        {/* Pass the artist ID to the Playlists component */}
        <Playlists artistID={artist.id} />
      </Suspense>
    </>
  )
}

async function Playlists({ artistID }: { artistID: string }) {
  // Use the artist ID to fetch playlists
  const playlists = await getArtistPlaylists(artistID)

  return (
    <ul>
      {playlists.map((playlist) => (
        <li key={playlist.id}>{playlist.name}</li>
      ))}
    </ul>
  )
}
```

```jsx filename="app/artist/[username]/page.js" switcher
export default async function Page({ params }) {
  const { username } = await params
  // Get artist information
  const artist = await getArtist(username)

  return (
    <>
      <h1>{artist.name}</h1>
      {/* Show fallback UI while the Playlists component is loading */}
      <Suspense fallback={<div>Loading...</div>}>
        {/* Pass the artist ID to the Playlists component */}
        <Playlists artistID={artist.id} />
      </Suspense>
    </>
  )
}

async function Playlists({ artistID }) {
  // Use the artist ID to fetch playlists
  const playlists = await getArtistPlaylists(artistID)

  return (
    <ul>
      {playlists.map((playlist) => (
        <li key={playlist.id}>{playlist.name}</li>
      ))}
    </ul>
  )
}
```

在此範例中，`<Suspense>` 允許在藝術家數據載入後串流播放列表。然而，頁面仍等待藝術家數據載入後才顯示任何內容。若要防止此情況，您可以將整個頁面元件包裹在 `<Suspense>` 邊界中（例如，使用 [`loading.js` 檔案](#with-loadingjs)），以便立即顯示載入狀態。

確保您的數據來源可以快速解析第一個請求，因為它會阻塞其他所有請求。如果您無法進一步優化請求，請考慮 [緩衝 (caching)](/docs/app/getting-started/caching) 結果（前提是數據變更不頻繁）。

### 並行資料獲取

並行資料獲取發生於路由中的資料請求被主動啓動且同時開始時。

預設情況下，[佈局和頁面](/docs/app/getting-started/layouts-and-pages) 會並行渲染。因此每個區段會盡快開始獲取資料。

然而，在 *任何* 元件內，如果 `async`/`await` 請求彼此排列在後，它們仍然會是順序的。例如，`getAlbums` 會在 `getArtist` 解析完成前暫停：

```tsx filename="app/artist/[username]/page.tsx" switcher
import { getArtist, getAlbums } from '@/app/lib/data'

export default async function Page({ params }) {
  // These requests will be sequential
  const { username } = await params
  const artist = await getArtist(username)
  const albums = await getAlbums(username)
  return <div>{artist.name}</div>
}
```

透過呼叫 `fetch` 開始多個請求，然後使用 [`Promise.all`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise/all) 等待它們。請求在呼叫 `fetch` 時立即開始。

```tsx filename="app/artist/[username]/page.tsx" highlight={3,8,24} switcher
import Albums from './albums'

async function getArtist(username: string) {
  const res = await fetch(`https://api.example.com/artist/${username}`)
  return res.json()
}

async function getAlbums(username: string) {
  const res = await fetch(`https://api.example.com/artist/${username}/albums`)
  return res.json()
}

export default async function Page({
  params,
}: {
  params: Promise<{ username: string }>
}) {
  const { username } = await params

  // Initiate requests
  const artistData = getArtist(username)
  const albumsData = getAlbums(username)

  const [artist, albums] = await Promise.all([artistData, albumsData])

  return (
    <>
      <h1>{artist.name}</h1>
      <Albums list={albums} />
    </>
  )
}
```

```jsx filename="app/artist/[username]/page.js" highlight={3,8,20} switcher
import Albums from './albums'

async function getArtist(username) {
  const res = await fetch(`https://api.example.com/artist/${username}`)
  return res.json()
}

async function getAlbums(username) {
  const res = await fetch(`https://api.example.com/artist/${username}/albums`)
  return res.json()
}

export default async function Page({ params }) {
  const { username } = await params

  // Initiate requests
  const artistData = getArtist(username)
  const albumsData = getAlbums(username)

  const [artist, albums] = await Promise.all([artistData, albumsData])

  return (
    <>
      <h1>{artist.name}</h1>
      <Albums list={albums} />
    </>
  )
}
```

> **小提示：** 如果使用 `Promise.all` 時其中一個請求失敗，整個作業將失敗。為了處理此情況，您可以改用 [`Promise.allSettled`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise/allSettled) 方法。

### 使用 Context 和 `React.cache` 共享數據

您可以透過將 [`React.cache`](https://react.dev/reference/react/cache) 與 Context 提供者結合，來在伺服器和客戶端元件 (Server and Client Components) 之間共享獲取的數據。

建立一個獲取數據的緩衝函式：

```ts filename="app/lib/user.ts" switcher
import { cache } from 'react'

export const getUser = cache(async () => {
  const res = await fetch('https://api.example.com/user')
  return res.json()
})
```

```js filename="app/lib/user.js" switcher
import { cache } from 'react'

export const getUser = cache(async () => {
  const res = await fetch('https://api.example.com/user')
  return res.json()
})
```

建立一個儲存 promise 的 Context 提供者：

```tsx filename="app/user-provider.tsx" switcher
'use client'

import { createContext } from 'react'

type User = {
  id: string
  name: string
}

export const UserContext = createContext<Promise<User> | null>(null)

export default function UserProvider({
  children,
  userPromise,
}: {
  children: React.ReactNode
  userPromise: Promise<User>
}) {
  return <UserContext value={userPromise}>{children}</UserContext>
}
```

```jsx filename="app/user-provider.js" switcher
'use client'

import { createContext } from 'react'

export const UserContext = createContext(null)

export default function UserProvider({ children, userPromise }) {
  return <UserContext value={userPromise}>{children}</UserContext>
}
```

在佈局 (layout) 中，將 promise 傳遞給提供者而無需等待：

```tsx filename="app/layout.tsx" switcher
import UserProvider from './user-provider'
import { getUser } from './lib/user'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  const userPromise = getUser() // Don't await

  return (
    <html>
      <body>
        <UserProvider userPromise={userPromise}>{children}</UserProvider>
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import UserProvider from './user-provider'
import { getUser } from './lib/user'

export default function RootLayout({ children }) {
  const userPromise = getUser() // Don't await

  return (
    <html>
      <body>
        <UserProvider userPromise={userPromise}>{children}</UserProvider>
      </body>
    </html>
  )
}
```

客戶端元件 (Client Components) 使用 [`use()`](https://react.dev/reference/react/use) 解析來自 Context 的 promise，並包裹在 `<Suspense>` 中以支援回退 UI：

```tsx filename="app/ui/profile.tsx" switcher
'use client'

import { use, useContext } from 'react'
import { UserContext } from '../user-provider'

export function Profile() {
  const userPromise = useContext(UserContext)
  if (!userPromise) {
    throw new Error('useContext must be used within a UserProvider')
  }
  const user = use(userPromise)
  return <p>Welcome, {user.name}</p>
}
```

伺服器元件 (Server Components) 也可以直接呼叫 `getUser()`。

```tsx filename="app/dashboard/page.tsx" switcher
import { getUser } from '../lib/user'

export default async function DashboardPage() {
  const user = await getUser() // Cached - same request, no duplicate fetch
  return <h1>Dashboard for {user.name}</h1>
}
```

```jsx filename="app/dashboard/page.js" switcher
import { getUser } from '../lib/user'

export default async function DashboardPage() {
  const user = await getUser() // Cached - same request, no duplicate fetch
  return <h1>Dashboard for {user.name}</h1>
}
```

由於 `getUser` 是與 `React.cache` 封裝的，同一請求內的多次呼叫均會返回相同記憶化結果，無論是在伺服器元件 (Server Components) 中直接呼叫，還是在客戶端元件 (Client Components) 中透過 Context 解析。

> **好知識**：`React.cache` 僅限於當前請求。每個請求都有獨立的記憶化作用域，請求之間無共享。
