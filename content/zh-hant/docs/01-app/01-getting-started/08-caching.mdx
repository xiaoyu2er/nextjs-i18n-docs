> 本頁介紹使用 [快取組件 (Cache Components)](/docs/app/api-reference/config/next-config-js/cacheComponents) 進行快取，該功能透過在 `next.config.ts` 檔案中設定 [`cacheComponents: true`](/docs/app/api-reference/config/next-config-js/cacheComponents) 來啟用。如果您未使用快取組件 (Cache Components)，請參閱 [快取與重新驗證 (Previous Model)](/docs/app/guides/caching-without-cache-components) 指南。

快取是一種技術，用於儲存資料撈取和其他運算的結果，以便未來對相同資料的請求可以服務得更快速，而無需重覆這項工作。

## 啟用快取組件 (Cache Components)

您可以在 Next config 檔案中加入 [`cacheComponents`](/docs/app/api-reference/config/next-config-js/cacheComponents) 選項來啟用快取組件 (Cache Components)：

```ts filename="next.config.ts" highlight={4} switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  cacheComponents: true,
}

export default nextConfig
```

```js filename="next.config.js" highlight={3} switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  cacheComponents: true,
}

module.exports = nextConfig
```

> **好知識：** 啟用快取組件 (Cache Components) 時，`GET` Route Handlers 遵循與頁面相同的預渲染 (Prerendering) 模型。詳情請參閱 [Route Handlers with Cache Components](/docs/app/getting-started/route-handlers#with-cache-components)。

## 使用方法

[`use cache`](/docs/app/api-reference/directives/use-cache) 指令會快取異步函式和組件的傳回值。您可以從兩個層級套用它：

- **資料層級**：快取撈取或運算資料的函式（例如：`getProducts()`，`getUser(id)`）
- **UI 層級**：快取整個組件或頁面（例如：`async function BlogPosts()`）

> 引數和來自上層作用域的任何封閉值都會自動成為 [快取索引鍵](/docs/app/api-reference/directives/use-cache#cache-keys) 的一部分，這意味著不同的輸入會產生單獨的快取項目。這支援個人化或參數化的快取內容。請參閱 [序列化要求和限制](/docs/app/api-reference/directives/use-cache#constraints) 以了解什麼是內容可以快取，以及引數如何運作。

### 資料層級快取

若要快取撈取資料的異步函式，請在函式主體頂部加入 `use cache` 指令：

```tsx filename="app/lib/data.ts" highlight={3,4,5}
import { cacheLife } from 'next/cache'

export async function getUsers() {
  'use cache'
  cacheLife('hours')
  return db.query('SELECT * FROM users')
}
```

當多個組件使用相同資料，或您希望與 UI 獨立快取資料時，資料層級快取非常有用。

### UI 層級快取

若要快取整個組件、頁面或佈局，請在組件或頁面主體的頂部加入 `use cache` 指令：

```tsx filename="app/page.tsx" highlight={1,4,5}
import { cacheLife } from 'next/cache'

export default async function Page() {
  'use cache'
  cacheLife('hours')

  const users = await db.query('SELECT * FROM users')

  return (
    <ul>
      {users.map((user) => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  )
}
```

> 如果在檔案頂部加入 "`use cache`"，檔案中所有匯出的函式都會被快取。

### 串流未快取的資料

對於從異步來源（如 API、資料庫或任何其他異步作業）撈取資料並需要每次請求都使用最新資料的組件，請勿使用 "use cache"。

取而代之的是，將組件包裝在 [`<Suspense>`](https://react.dev/reference/react/Suspense) 中並提供回退 UI。在請求時間，React 會先渲染回退內容，然後在異步工作完成後串流解析內容。

```tsx filename="page.tsx"
import { Suspense } from 'react'

async function LatestPosts() {
  const data = await fetch('https://api.example.com/posts')
  const posts = await data.json()
  return (
    <ul>
      {posts.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  )
}

export default function Page() {
  return (
    <>
      <h1>My Blog</h1>
      <Suspense fallback={<p>Loading posts...</p>}>
        <LatestPosts />
      </Suspense>
    </>
  )
}
```

回退內容 (`<p>Loading posts...</p>`) 包含在靜態外殼 (static shell) 中，而組件的內容則在請求時間串流。

## 使用執行階段 API

請求時間 API 需要僅在用戶發出請求時才可用的資訊。這些包括：

- [`cookies`](/docs/app/api-reference/functions/cookies) - 用戶的 cookie 資料
- [`headers`](/docs/app/api-reference/functions/headers) - 請求標頭
- [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) - URL 查詢參數
- [`params`](/docs/app/api-reference/file-conventions/page#params-optional) - 動態路由參數（除非透過 [`generateStaticParams`](/docs/app/api-reference/functions/generate-static-params) 提供至少一個範例）。

存取執行階段 API 的組件應該包裝在 <Suspense> 中：

```tsx filename="page.tsx"
import { cookies } from 'next/headers'
import { Suspense } from 'react'

async function UserGreeting() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')?.value || 'light'
  return <p>Your theme: {theme}</p>
}

export default function Page() {
  return (
    <>
      <h1>Dashboard</h1>
      <Suspense fallback={<p>Loading...</p>}>
        <UserGreeting />
      </Suspense>
    </>
  )
}
```

### 將執行時值傳遞給快取函式

您可以從執行階段 API 中提取值並將其作為參數傳遞給快取函式：

```tsx filename="app/profile/page.tsx"
import { cookies } from 'next/headers'
import { Suspense } from 'react'

export default function Page() {
  return (
    <Suspense fallback={<div>Loading...</div>}>
      <ProfileContent />
    </Suspense>
  )
}

// Component (not cached) reads runtime data
async function ProfileContent() {
  const session = (await cookies()).get('session')?.value
  return <CachedContent sessionId={session} />
}

// Cached component receives extracted value as a prop
async function CachedContent({ sessionId }: { sessionId: string }) {
  'use cache'
  // sessionId becomes part of the cache key
  const data = await fetchUserData(sessionId)
  return <div>{data}</div>
}
```

在請求時間，如果未找到匹配的快取項目，`CachedContent` 將執行，並儲存結果以供具有相同 `sessionId` 的未來請求使用。

## 處理非確定性作業

類似 `Math.random()`、`Date.now()` 或 `crypto.randomUUID()` 的作业在每次執行時會產生不同的值。快取組件 (Cache Components) 要求您明確處理這些作業。

**若要為每個請求產生唯一值**，請在這些作業之前呼叫 [`connection()`](/docs/app/api-reference/functions/connection) 以延遲至請求時間，並將組件包裝在 <Suspense> 中：

```tsx filename="page.tsx"
import { connection } from 'next/server'
import { Suspense } from 'react'

async function UniqueContent() {
  await connection()
  const uuid = crypto.randomUUID()
  return <p>Request ID: {uuid}</p>
}

export default function Page() {
  return (
    <Suspense fallback={<p>Loading...</p>}>
      <UniqueContent />
    </Suspense>
  )
}
```

或者，您可以 **快取結果**，讓所有用戶直到重新驗證 (revalidation) 前都看到相同的值：

```tsx filename="page.tsx"
export default async function Page() {
  'use cache'
  const buildId = crypto.randomUUID()
  return <p>Build ID: {buildId}</p>
}
```

## 處理確定性作業

類似同步 I/O、模組匯入和純計算的作業可以在預渲染期間完成。僅使用這些運作的組件，其渲染輸出會自動包含在靜態 HTML 外殼中。

```tsx filename="page.tsx"
import fs from 'node:fs'

export default async function Page() {
  const content = fs.readFileSync('./config.json', 'utf-8')
  const constants = await import('./constants.json')
  const processed = JSON.parse(content).items.map((item) => item.value * 2)

  return (
    <div>
      <h1>{constants.appName}</h1>
      <ul>
        {processed.map((value, i) => (
          <li key={i}>{value}</li>
        ))}
      </ul>
    </div>
  )
}
```

## 渲染運作方式

在建置時間，Next.js 會渲染您的路由組件樹。每個組件的處理方式取決於它使用的 API：

- [`use cache`](#usage)：結果會快取並包含在靜態外殼中
- [`<Suspense>`](#streaming-uncached-data)：回退 UI 包含在靜態外殼中，而內容在請求時間串流
- [確定性作業](#working-with-deterministic-operations)：如純計算和模組匯入自動包含在靜態外殼中

這會產生一個靜態外殼，包含用於初次頁面載入的 HTML，以及用於用戶端導覽的序列化的 [RSC Payload](/docs/app/getting-started/server-and-client-components#on-the-server)，確保無論用戶是直接瀏覽至 URL 還是從另一個頁面過渡，瀏覽器都能即時接收完全渲染的內容。

### 退出靜態外殼

在 Root Layout 中的 document body 上方放置一個具有空回退的 <Suspense> 邊界 (boundary) 會導致整個應用程式延遲至請求時間。由於回退為空，沒有靜態外殼可以立即傳送，因此每個請求都會阻塞，直到頁面完全渲染完成。若要將此限制於特定路由，請使用 [多個根佈局](/docs/app/api-reference/file-conventions/layout#root-layout)。

```tsx filename="app/layout.tsx" highlight={1,10-12}
import { Suspense } from 'react'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html>
      <Suspense fallback={null}>
        <body>{children}</body>
      </Suspense>
    </html>
  )
}
```

> **好知識**：當 `generateViewport` 存取未快取的動態資料時，此相同模式也適用。請參閱 [Viewport with Cache Components](/docs/app/api-reference/functions/generate-viewport#with-cache-components) 以獲取詳細範例。

### 整合所有內容

這裡是一個完整的範例，顯示靜態內容、快取的動態內容和串流的動態內容如何在單一頁面上協同運作：

```tsx filename="app/blog/page.tsx"
import { Suspense } from 'react'
import { cookies } from 'next/headers'
import { cacheLife, cacheTag, updateTag } from 'next/cache'
import Link from 'next/link'

export default function BlogPage() {
  return (
    <>
      {/* Static content - prerendered automatically */}
      <header>
        <h1>Our Blog</h1>
        <nav>
          <Link href="/">Home</Link> | <Link href="/about">About</Link>
        </nav>
      </header>

      {/* Cached dynamic content - included in the static shell */}
      <BlogPosts />

      {/* Runtime dynamic content - streams at request time */}
      <Suspense fallback={<p>Loading your preferences...</p>}>
        <UserPreferences />
      </Suspense>

      {/* Mutation - server action that revalidates the cache */}
      <Suspense fallback={<p>Loading...</p>}>
        <CreatePost />
      </Suspense>
    </>
  )
}

// Everyone sees the same blog posts (revalidated every hour)
async function BlogPosts() {
  'use cache'
  cacheLife('hours')
  cacheTag('posts')

  const res = await fetch('https://api.vercel.app/blog')
  const posts = await res.json()

  return (
    <section>
      <h2>Latest Posts</h2>
      <ul>
        {posts.slice(0, 5).map((post: any) => (
          <li key={post.id}>
            <h3>{post.title}</h3>
            <p>
              By {post.author} on {post.date}
            </p>
          </li>
        ))}
      </ul>
    </section>
  )
}

// Personalized per user based on their cookie
async function UserPreferences() {
  const theme = (await cookies()).get('theme')?.value || 'light'
  const favoriteCategory = (await cookies()).get('category')?.value

  return (
    <aside>
      <p>Your theme: {theme}</p>
      {favoriteCategory && <p>Favorite category: {favoriteCategory}</p>}
    </aside>
  )
}

// Admin-only form that creates a post and revalidates the cache
async function CreatePost() {
  const isAdmin = (await cookies()).get('role')?.value === 'admin'
  if (!isAdmin) return null

  async function createPost(formData: FormData) {
    'use server'
    await db.post.create({ data: { title: formData.get('title') } })
    updateTag('posts')
  }

  return (
    <form action={createPost}>
      <input name="title" placeholder="Post title" required />
      <button type="submit">Publish</button>
    </form>
  )
}
```

在預渲染期間，頁首（靜態）和部落格文章（使用 `use cache` 快取）成為靜態外殼的一部分，以及用於個人偏好的回退 UI。僅有個人的偏好設定會在請求時間串流。當管理員發布新文章時，[`updateTag`](/docs/app/getting-started/revalidating#updatetag) 呼叫會立即使部落格文章快取失效，以便下一位訪客看到它。

> **好知識**：`generateMetadata` 和 `generateViewport` 在頁面上分別記錄執行階段資料存取。請參閱 [Metadata with Cache Components](/docs/app/api-reference/functions/generate-metadata#with-cache-components) 和 [Viewport with Cache Components](/docs/app/api-reference/functions/generate-viewport#with-cache-components) 以了解如何處理此問題。
