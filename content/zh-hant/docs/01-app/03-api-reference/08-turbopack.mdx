Turbo pack 是一個針對 JavaScript 和 TypeScript 優化的 **增量打包器 (incremental bundler)**，由 Rust 編寫，並內建於 **Next.js**。您可以在 Pages 和 App Router 中使用 Turbopack，獲得 **更快速 (much faster)** 的本地開發體驗。

## 為什麼使用 Turbopack？

我們開發 Turbopack 是為了提升 Next.js 的效能，包括：

- **統一模組圖 (Unified Graph):** Next.js 支援多輸出環境（例如：客戶端和伺服器端）。管理多個編譯器和縫合 bundles 可能十分繁瑣。Turbopack 對於所有環境使用 **單一統一模組圖 (single, unified graph)**。
- **打包與原生 ESM (Bundling vs Native ESM):** 一些工具在開發期間跳過打包並依賴瀏覽器的原生 ESM。對於小型應用程式這工作良好，但因過多網路請求可能會降低大型應用程式的速度。Turbopack 在開發期間 **打包 (bundles)**，但採用優化方式以保持大型應用程式速度。
- **增量計算 (Incremental Computation):** Turbopack 跨核心平行化工作，並 **快取 (caches)** 結果至函式級別。一旦完成部分工作，Turbopack 不會重複它。
- **延遲打包 (Lazy Bundling):** Turbopack 僅打包開發伺服器實際請求的內容。這種延遲方法可以降低初始編譯時間和記憶體使用量。

## 支援的平台

Turbopack 需要特定平台的原始綁定。目前支援以下平台：

| 平台       | 架構    |
| -------------- | -------- |
| macOS (Darwin) | x64, ARM64 |
| Windows        | x64, ARM64 |
| Linux (glibc)  | x64, ARM64 |
| Linux (musl)   | x64, ARM64 |

在沒有原生綁定（例如 FreeBSD, OpenBSD）的平台上，Next.js 會回退到 WebAssembly (WASM) 綁定。WASM 綁定支援 SWC 核心功能，例如編譯和壓縮，但 **不支持 Turbopack**。在這些平台上，使用 `--webpack` 標識：

```bash
next dev --webpack
next build --webpack
```

## 開始使用

Turbopack 現在是 Next.js 中的**預設打包器**。使用 Turbopack 不需要任何配置：

```json filename="package.json" highlight={3}
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start"
  }
}
```

### 改用 Webpack

如果您需要使用 Webpack 而不是 Turbopack，您可以使用 `--webpack` 標誌來選擇：

```json filename="package.json"
{
  "scripts": {
    "dev": "next dev --webpack",
    "build": "next build --webpack",
    "start": "next start"
  }
}
```

### 魔法註解

Turbopack 支援與 Webpack 相容的魔法註解，用於控制導入行為。這些註解適用於動態 `import()`、`require()`、`require.resolve()` 和 `new Worker()` 表達式（不適用於靜態 `import` 語句）。

| 註解                   | Webpack | Turbopack | 描述                    |
| ------------------------- | ------- | --------- | ------------------------------ |
| `webpackIgnore: true`     | ✓       | ✓         | 跳過打包，保留導入 |
| `turbopackIgnore: true`   | ✗       | ✓         | 跳過打包（Turbopack 專有） |
| `turbopackOptional: true` | ✗       | ✓         | 屏蔽解析錯誤        |
| `webpackOptional: true`   | ✗       | ✗         | 不支援                  |

請參閱 [惰性載入](/docs/app/guides/lazy-loading#magic-comments) 以獲取用法範例。

## 與 webpack 的已知差距

在遷移應用程式時，webpack 和 Turbopack 之間存在一些重要的非微不足道行為差異。一般來說，這些對新應用程式較無影響。

### 檔案系統根目錄

Turbopack 使用根目錄來解析模組。專案根目錄之外的檔案將不會被解析。

例如，在連結專案根源外的依賴時（透過 `npm link`、`yarn link`、`pnpm link` 等），這些連結的檔案預設不會被解析。要解析這些檔案，您必須將 root 選項配置為專案和連結依賴項的父目錄。

您可以在 `next.config.js` 中使用 [turbopack.root](/docs/app/api-reference/config/next-config-js/turbopack#root-directory) 選項來配置檔案系統根目錄。

### CSS Module 排序

Turbopack 將遵循 JS 匯入順序來排序 [CSS modules](/docs/app/getting-started/css#css-modules)（ otherwise 無排序）。例如：

```jsx filename="components/BlogPost.jsx"
import utilStyles from './utils.module.css'
import buttonStyles from './button.module.css'
export default function BlogPost() {
  return (
    <div className={utilStyles.container}>
      <button className={buttonStyles.primary}>Click me</button>
    </div>
  )
}
```

在這個範例中，Turbopack 將確保 `utils.module.css` 的輸出順序在 `button.module.css` 之前，遵循匯入順序。

Webpack 通常也會這樣做，但有些情況下它會忽略 JS 推斷的排序，例如如果它推斷 JS 檔案沒有副作用。

這可能會導致採用 Turbopack 時產生微妙的渲染變更，如果應用程式依賴於任意排序。通常，解決方案很簡單，例如讓 `button.module.css` `@import utils.module.css` 來強制排序，或識別衝突規則並更改它們以避免針對相同的屬性。

### Sass node_modules 導入

Turbopack 預先支援導入 `node_modules` 中的 Sass 檔案。Webpack 支援用於此功能的遺留波浪 `~` 語法，Turbopack 不支援。

從：

```scss filename="styles/globals.scss"
@import '~bootstrap/dist/css/bootstrap.min.css';
```

若要：

```scss filename="styles/globals.scss"
@import 'bootstrap/dist/css/bootstrap.min.css';
```

如果您無法更新導入，您可以添加 `turbopack.resolveAlias` 配置來將 `~` 語法映射到實際路徑：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    resolveAlias: {
      '~*': '*',
    },
  },
}
```

### 建置快取

Webpack 支援 [磁碟構建緩存](https://webpack.js.org/configuration/cache/#cache) 以構建效能。Turbopack 提供類似的功能，目前處於 Beta 版。從 Next 16 開始，您可以通設置以下實驗性標誌來啟用 Turbopack 的檔案系統緩存：

- [`experimental.turbopackFileSystemCacheForDev`](/docs/app/api-reference/config/next-config-js/turbopackFileSystemCache) 預設啟用
- [`experimental.turbopackFileSystemCacheForBuild`](/docs/app/api-reference/config/next-config-js/turbopackFileSystemCache) 目前需選擇啟用

> **值得了解：** 基於此原因，在比較 Webpack 和 Turbopack 效能時，請務必在每次構建之間刪除 `.next` 資料夾以進行冷構建公平比較，或啟用 Turbopack 檔案系統緩存功能以比較熱構建。

### Webpack 外掛

Turbopack 不支援 webpack 外掛。這會影響依賴 webpack 外掛系統進行整合的第三方工具。我們支援 [webpack loaders](/docs/app/api-reference/config/next-config-js/turbopack#configuring-webpack-loaders)。如果您依賴 webpack 外掛，您需要尋找 Turbopack 相容的替代方案，或繼續使用 webpack 直到擁有功能等效的功能。

## 不支援且未規劃的功能

某些功能尚未實作或尚未規劃：

- **遺留 CSS 模組功能**
  - 獨立的 `:local` 和 `:global` 偽類（僅支援函數變體 `:global(...)`）。
  - `@value` 規則（已被 CSS 變數取代）。
  - `:import` 和 `:export` ICSS 規則。
  - `.module.css` 中的 `composes` 組合 `.css` 檔案。在 Webpack 中這會將 `.css` 視為 CSS 模組，而 Turbopack 將 `.css` 檔案始終視為全域。這意味著如果您想在 CSS 模組中使用 `composes`，您需要將 `.css` 檔案改為 `.module.css` 檔案。
  - CSS 模組中的 `@import` 將 `.css` 導入為 CSS 模組。在 Webpack 中這會將 `.css` 視為 CSS 模組，在 Turbopack 中 `.css` 將始終是全球的。這意味著如果您想在 CSS 模組中使用 `@import`，您需要將 `.css` 檔案更改為 `.module.css` 檔案。
- **`sassOptions.functions`**
  `sassOptions.functions` 中定義的自定義 Sass 函數不受支援。此功能允許定義可在編譯期間從 Sass 程式碼調用的 JavaScript 函數。Turbopack 基於 Rust 的架構無法直接執行透過 `sassOptions.functions` 傳遞的 JavaScript 函數，不像 Webpack 的基於 Node.js 的 sass-loader 完全在 JavaScript 中運行。如果您使用自定義 Sass 函數，您需要使用 Webpack 而不是 Turbopack。
- **`next.config.js` 中的 `webpack()` 配置**
  Turbopack 替換 Webpack，因此 `webpack()` 配置不被識別。請改用 [`turbopack` 配置](/docs/app/api-reference/config/next-config-js/turbopack)。
- **Yarn PnP**
  Next.js 中不計劃支援 Turbopack 的 Yarn PnP 支持。
- **`experimental.urlImports`**
  Turbopack 不計劃。
- **`experimental.esmExternals`**
  不計劃。Turbopack 不支援 Next.js 中的遺留 `esmExternals` 配置。
- **一些 Next.js 實驗性標誌**
  - `experimental.nextScriptWorkers`
  - `experimental.fallbackNodePolyfills`
    我們計劃在未來實現這些。

欲查看每個功能旗標及其狀態的完整詳細清單，請參閱 [Turbopack API 參考文獻](/docs/app/api-reference/config/next-config-js/turbopack)。

## 配置

Turbopack 可以透過 `turbopack` 鍵下的 `next.config.js`（或 `next.config.ts`）進行配置。配置選項包括：

- **`rules`**
  定義額外的 [Webpack 加載器](/docs/app/api-reference/config/next-config-js/turbopack#configuring-webpack-loaders) 進行檔案轉換。
- **`resolveAlias`**
  建立手動別名（類似 Webpack 的 `resolve.alias`）。
- **`resolveExtensions`**
  更改或延展模組解析的檔案副檔名。
- **[`ignoreIssue`](/docs/app/api-reference/config/next-config-js/turbopackIgnoreIssue)**
  屏蔽特定 Turbopack 錯誤和警告從 CLI 輸出和錯誤覆蓋層。

此外，`next.config.js` 中的 `experimental` 下還提供以下實驗性選項：

| 選項                                                                                                       | 描述                                                                            | 預設值 (dev) | 預設值 (build)               |
| ------------------------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------- | ------------- | ----------------------------- |
| [`turbopackFileSystemCacheForDev`](/docs/app/api-reference/config/next-config-js/turbopackFileSystemCache)   | 啟用開發伺服器的檔案系統緩存。                                            | `true`        | N/A                           |
| [`turbopackFileSystemCacheForBuild`](/docs/app/api-reference/config/next-config-js/turbopackFileSystemCache) | 啟用構建的檔案系統緩存。                                                    | N/A           | `false`                       |
| `turbopackMinify`                                                                                            | 啟用壓縮。                                                                   | `false`       | `true`                        |
| `turbopackSourceMaps`                                                                                        | 啟用原始碼映射。                                                                    | `true`        | `productionBrowserSourceMaps` |
| `turbopackInputSourceMaps`                                                                                   | 啟用從輸入檔案中提取原始碼映射。                                     | `true`        | `true`                        |
| `turbopackTreeShaking`                                                                                       | 使用進階模組碎片樹搖動 (Tree Shaking)，而非預設的僅重新匯出模式。 | `false`       | `false`                       |
| `turbopackRemoveUnusedImports`                                                                               | 啟用移除未使用的導入。需要 `turbopackRemoveUnusedExports`。               | `false`       | `true`                        |
| `turbopackRemoveUnusedExports`                                                                               | 啟用移除未使用的導出。                                                        | `false`       | `true`                        |
| `turbopackInferModuleSideEffects`                                                                            | 啟用本地分析以推斷無副作用模組以增加 Tree Shaking 效果。       | `true`        | `true`                        |
| `turbopackScopeHoisting`                                                                                     | 啟用作用域提升 (Scope Hoisting)。開發模式時總為關閉。                                    | `false`       | `true`                        |
| `turbopackClientSideNestedAsyncChunking`                                                                     | 啟用前端資產的嵌套非同步分包。                                   | `false`       | `true`                        |
| `turbopackServerSideNestedAsyncChunking`                                                                     | 啟用服務器端資產的嵌套非同步分包。                                   | `false`       | `false`                       |
| `turbopackImportTypeBytes`                                                                                   | 啟用對 ESM 導入的 `with {type: "bytes"}` 的支援。                             | `false`       | `false`                       |
| `turbopackUseBuiltinBabel`                                                                                   | 存在 Babel 配置文件時啟用自動 Babel 加載器配置。       | `true`        | `true`                        |
| `turbopackUseBuiltinSass`                                                                                    | 啟用自動 Sass 加載器配置。                                            | `true`        | `true`                        |


## 產生效能除錯的追蹤檔案

如果您遇到效能或記憶體問題並希望協助 Next.js 團隊診斷，您可以在開發命令中追加 `NEXT_TURBOPACK_TRACING=1` 來產生追蹤檔案：

```bash
NEXT_TURBOPACK_TRACING=1 next dev
```

這將產生一個 `.next/dev/trace-turbopack` 檔案。在 [Next.js 存儲庫](https://github.com/vercel/next.js) 上建立 GitHub issue 時請包含該檔案，以協助我們調查。

預設情況下，開發伺服器會輸出到 `.next/dev`。

## 摘要

Turbopack 是一個基於 **Rust (Rust-based)**、**增量 (incremental)** 的打包器，旨在加快本地開發和建置速度——尤其是大型應用程式。它整合至 Next.js 中，提供零配置的 CSS、React 和 TypeScript 支援。
