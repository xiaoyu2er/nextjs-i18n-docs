title: 路由群組 (Route Groups)
description: 路由群組可用來將 Next.js 應用程式劃分為不同的區塊。
---

路由群組 (Route Groups) 是資料夾慣例，可用來按類別或團隊組織路由。

## 慣例

路由群組可以透過將資料夾名稱包在括號內來建立： `(folderName)`。

此慣例表示該資料夾僅用於組織目的，且**不應**包含在路由的 URL 路徑中。

## 使用情境

- 按團隊、關注點或功能組織路由。
- 定義多個 [根佈局](/docs/app/api-reference/file-conventions/layout#root-layout)。
- 選擇特定的路由區段共用佈局，同時保留其他區段不共用。

## 注意事項

- **整頁加載**：如果您在需要使用不同根佈局的路由之間導航，它將觸發整頁重新加載。例如，從使用 `app/(shop)/layout.js` 的 `/cart` 導航到使用 `app/(marketing)/layout.js` 的 `/blog`。此**僅**適用於多個根佈局。
- **衝突路徑**：不同群組中的路由不應該解析到相同的 URL 路徑。例如，`(marketing)/about/page.js` 和 `(shop)/about/page.js` 都會解析到 `/about` 並導致錯誤。
- **頂層根佈局**：如果您在沒有頂層 `layout.js` 檔案的情況下使用多個根佈局，請確保您的主頁路由 (/) 定義在一個路由群組內，例如 `app/(marketing)/page.js`。
