截斷路由 (Intercepting Routes) 允許你在當前佈局中載入應用程式的其他部分的路由。這種路由範型在你想要顯示路由內容而不讓使用者切換到不同情境時很有用。

例如，當點擊動態消息 (feed) 中的照片時，你可以在模態視窗中顯示照片，覆蓋動態消息。在此情況下，Next.js 會截斷 `/photo/123` 路由，掩蓋 URL，並將其覆蓋在 `/feed` 之上。

## 慣例

截斷路由 (Intercepting Routes) 可以透過 `(..)` 慣例來定義，這類似於相對路徑慣例 `../`，但用於路由分段。

您可以使用：

- `(.)` 用於匹配**同一層級**的分段
- `(..)` 用於匹配**上一層**的分段
- `(..)(..)` 用於匹配**上兩層**的分段
- `(...)` 用於匹配來自 **根目錄** `app` 的分段

例如，你可以透過建立 `(..)photo` 資料夾，從 `feed` 分段中截斷 `photo` 分段。

<Image

> **實用資訊：** `(..)` 慣例是基於 _路由分段_，而非檔案系統。例如，它不會考慮 [平行路由](/docs/app/api-reference/file-conventions/parallel-routes) 中的 `@slot` 資料夾。

## 範例

### 模態視窗

截斷路由 (Intercepting Routes) 可與 [平行路由](/docs/app/api-reference/file-conventions/parallel-routes) 搭配使用來建立模態視窗。這讓你可以解決建立模態視窗時常見的問題，例如：

- 使模態視窗內容可 **透過 URL 共用**。
- **保留情境** 時重新整理頁面，而不是關閉模態視窗。
- **回溯導航時關閉模態視窗**，而不是返回上一個路由。
- **向前導航時重新開啟模態視窗**。

請參見以下 UI 範式，使用者可以使用用戶端導航從圖庫開啟照片模態視窗，或直接透過可共用 URL 導航至照片頁面：

<Image

  srcLight="/docs/light/intercepted-routes-modal-example.png"
  srcDark="/docs/dark/intercepted-routes-modal-example.png"

在上述範例中，`photo` 分段的路徑可以使用 `(..)` 配對器，因為 `@modal` 是插槽而 **不是** 分段。這意味著 `photo` 路由僅比當前高一層分段，儘管在檔案系統中高出兩層。

請參閱 [平行路由](/docs/app/api-reference/file-conventions/parallel-routes#modals) 文件以獲得逐步範例，或參閱我們的 [圖庫範例](https://github.com/vercel-labs/nextgram)。

> **實用資訊：**
>
> - 其他範例可能包括在頂端導覽列開啟登入模態視窗，同時擁有專用的 `/login` 頁面，或在側邊模態視窗中開啟購物車。
