**error** 檔案允許您處理非預期的執行階段錯誤並顯示後備 UI。

```tsx filename="app/dashboard/error.tsx" switcher
'use client' // Error boundaries must be Client Components

import { useEffect } from 'react'

export default function Error({
  error,
  unstable_retry,
}: {
  error: Error & { digest?: string }
  unstable_retry: () => void
}) {
  useEffect(() => {
    // Log the error to an error reporting service
    console.error(error)
  }, [error])

  return (
    <div>
      <h2>Something went wrong!</h2>
      <button
        onClick={
          // Attempt to recover by re-fetching and re-rendering the segment
          () => unstable_retry()
        }
      >
        Try again
      </button>
    </div>
  )
}
```

`error.js` 將路由區段及其嵌套子項目包裝在 [React 錯誤邊界 (React Error Boundary)](https://react.dev/reference/react/Component#catching-rendering-errors-with-an-error-boundary) 中。當錯誤在邊界內拋出時，`error` 組件會顯示為後備 UI。

> **好資訊**：

> - [React DevTools](https://react.dev/learn/react-developer-tools) 允許您切換錯誤邊界 (Error Boundaries) 以測試錯誤狀態。
> - 如果您希望錯誤冒泡至上層錯誤邊界，您可以在渲染 `error` 組件時 `throw` 錯誤。
> - 對於不繫於路由區塊（如 [`error.js`](/docs/app/api-reference/file-conventions/error)）的組件層級錯誤復原，請使用 [`unstable_catchError`](/docs/app/api-reference/functions/catchError) 函數。

在 [組件層級 (component hierarchy)](/docs/app/getting-started/project-structure#component-hierarchy) 中，`error.js` 會在 React 錯誤邊界 (Error Boundary) 中包裝 `loading.js`、`not-found.js`、`page.js` 以及巢狀的 `layout.js` 檔案。它 **不會** 包裝同一區塊中位於上方的 `layout.js` 或 `template.js`。若要處理根層級佈局 (root layout) 的錯誤，請使用 [`global-error.js`](/docs/app/api-reference/file-conventions/error#global-error)。

## 參考

### 屬性 (Props)

#### 錯誤 (error)

轉發至 `error.js` 客戶端組件 (Client Component) 的 [`錯誤 (Error)`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Error) 物件執行個體。

> **值得注意：** 在開發階段，轉發至客戶端的 `錯誤 (Error)` 物件將被序列化並包含原始錯誤的 `message` 以便於除錯。然而，**此行為在生產環境 (production) 中有所不同**，以避免將包含在錯誤中的潛在敏感詳細資料洩漏給使用者。

#### 錯誤訊息 (error.message)

- 從客戶端組件 (Client Component) 轉發的錯誤顯示原始 `錯誤訊息 (Error message)`。
- 從伺服器組件 (Server Component) 轉發的錯誤顯示帶有識別碼的通用訊息。這是為了防止洩漏敏感詳細資料。您可以使用識別碼，位於 `error.digest` 下，以匹配相應的伺服器端記錄 (server-side logs)。

#### 錯誤摘要 (error.digest)

錯誤拋出時自動生成的雜湊值。可用於在伺服器端記錄中匹配相應的錯誤。

#### `unstable_retry`

錯誤的原因有時可能是暫時的。在這些情況下，重試可能會解決問題。

錯誤組件可以使用 `unstable_retry()` 函數來提示使用者嘗試從錯誤中復原。執行時，此函數會嘗試重新取得並重新渲染錯誤邊界的內容。如果成功，備用錯誤組件將被重新渲染的結果取代。

```tsx filename="app/dashboard/error.tsx" switcher
'use client' // Error boundaries must be Client Components

export default function Error({
  error,
  unstable_retry,
}: {
  error: Error & { digest?: string }
  unstable_retry: () => void
}) {
  return (
    <div>
      <h2>Something went wrong!</h2>
      <button onClick={() => unstable_retry()}>Try again</button>
    </div>
  )
}
```

```jsx filename="app/dashboard/error.js" switcher
'use client' // Error boundaries must be Client Components

export default function Error({ error, unstable_retry }) {
  return (
    <div>
      <h2>Something went wrong!</h2>
      <button onClick={() => unstable_retry()}>Try again</button>
    </div>
  )
}
```

#### 重置 (reset)

在大多數情況下，您應該改用 [`unstable_retry()`](#unstable_retry)。但是，如果您有特定理由要清除錯誤狀態並重新渲染錯誤邊界的內容，而不重新取得內容，則可以使用 `reset()` 函數。

## 範例

### 全域錯誤 (Global Error)

雖然較少見，但您可以使用位於根 app 目錄的 `global-error.jsx` 在根佈局或模板中處理錯誤，即使利用了 [國際化 (internationalization)](/docs/app/guides/internationalization)。全域錯誤 UI 必須定義其自己的 `<html>` 和 `<body>` 標籤、全域樣式、字體或其他錯誤頁面所需的依賴項。啟用時，此檔案會取代根佈局或模板。

> **值得注意：** 錯誤邊界必須是 [客戶端組件 (Client Components)](/docs/app/getting-started/server-and-client-components#using-client-components)，這意味著 `metadata` 和 `generateMetadata` 匯出在 `global-error.jsx` 中不受支援。作為替代方案，您可以使用 React [`<title>`](https://react.dev/reference/react-dom/components/title) 組件。

```tsx filename="app/global-error.tsx" switcher
'use client' // Error boundaries must be Client Components

export default function GlobalError({
  error,
  unstable_retry,
}: {
  error: Error & { digest?: string }
  unstable_retry: () => void
}) {
  return (
    // global-error must include html and body tags
    <html>
      <body>
        <h2>Something went wrong!</h2>
        <button onClick={() => unstable_retry()}>Try again</button>
      </body>
    </html>
  )
}
```

```jsx filename="app/global-error.js" switcher

```

### 使用自訂錯誤邊界的優雅錯誤復原 (Graceful error recovery with a custom error boundary)

當客戶端渲染失敗時，顯示最後已知的伺服器渲染 UI 對於更好的使用者體驗可能很有用。

`GracefullyDegradingErrorBoundary` 是一個自訂錯誤邊界的範例，用於在錯誤發生之前抓取並保留目前的 HTML。如果發生渲染錯誤，它會重新渲染抓取的 HTML 並顯示持續的通知列以通知使用者。

```tsx filename="app/dashboard/error.tsx" switcher
'use client'

import React, { Component, ErrorInfo, ReactNode } from 'react'

interface ErrorBoundaryProps {
  children: ReactNode
  onError?: (error: Error, errorInfo: ErrorInfo) => void
}

interface ErrorBoundaryState {
  hasError: boolean
}

export class GracefullyDegradingErrorBoundary extends Component<
  ErrorBoundaryProps,
  ErrorBoundaryState
> {
  private contentRef: React.RefObject<HTMLDivElement | null>

  constructor(props: ErrorBoundaryProps) {
    super(props)
    this.state = { hasError: false }
    this.contentRef = React.createRef()
  }

  static getDerivedStateFromError(_: Error): ErrorBoundaryState {
    return { hasError: true }
  }

  componentDidCatch(error: Error, errorInfo: ErrorInfo) {
    if (this.props.onError) {
      this.props.onError(error, errorInfo)
    }
  }

  render() {
    if (this.state.hasError) {
      // Render the current HTML content without hydration
      return (
        <>
          <div
            ref={this.contentRef}
            suppressHydrationWarning
            dangerouslySetInnerHTML={{
              __html: this.contentRef.current?.innerHTML || '',
            }}
          />
          <div className="fixed bottom-0 left-0 right-0 bg-red-600 text-white py-4 px-6 text-center">
            <p className="font-semibold">
              An error occurred during page rendering
            </p>
          </div>
        </>
      )
    }

    return <div ref={this.contentRef}>{this.props.children}</div>
  }
}

export default GracefullyDegradingErrorBoundary
```
