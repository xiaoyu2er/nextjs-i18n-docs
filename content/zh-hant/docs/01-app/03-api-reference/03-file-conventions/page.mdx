「page」檔案允許您定義與特定路由**獨特**的 UI。您可以透過從該檔案預設匯出元件創建頁面：

```tsx filename="app/blog/[slug]/page.tsx" switcher
export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  return <h1>My Page</h1>
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
export default function Page({ params, searchParams }) {
  return <h1>My Page</h1>
}
```

## 小知識

- `page` 可以使用 `.js`、`.jsx` 或 `.tsx` 副檔名。
- `page` 始終是路由子樹的**葉節點**。
- 需要 `page` 檔案才能使路由片段**公開可訪問**。
- 頁面預設為 [伺服器組件 (Server Components)](https://react.dev/reference/rsc/server-components)，但可以設定為 [客戶端組件 (Client Component)](https://react.dev/reference/rsc/use-client)。
- 在 [組件層次結構](/docs/app/getting-started/project-structure#component-hierarchy) 中，`page.js` 是最內層的檔案慣例。它在同一片段中由 `loading.js`（Suspense 邊界）、`error.js`（錯誤邊界）、`template.js` 和 `layout.js` 包裹。

## 參考

### 屬性 (Props)

#### `params` (選填)

一個 promise，解析後為包含從根區段到該頁面的 [動態路由參數 (dynamic route parameters)](/docs/app/api-reference/file-conventions/dynamic-routes) 的物件。

```tsx filename="app/shop/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
}
```

```jsx filename="app/shop/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
}
```

| 範例路由 | URL | `params` |
| --- | --- | --- |
| `app/shop/[slug]/page.js` | `/shop/1` | `Promise<{ slug: '1' }>` |
| `app/shop/[category]/[item]/page.js` | `/shop/1/2` | `Promise<{ category: '1', item: '2' }>` |
| `app/shop/[...slug]/page.js` | `/shop/1/2` | `Promise<{ slug: ['1', '2'] }>` |

- 由於 `params` prop 是一個 promise，您必須使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函數來取得這些值。
  - 在 v14 及更早版本中，`params` 是一個同步 prop。為了幫助向下相容，您在 Next.js 15 中仍可同步取得它，但此行為將會在未來遭到棄用。

#### `searchParams` (選填)

一個 promise，解析後為包含 [搜尋參數 (search parameters)](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters) 的物件，取自目前 URL。範例：

```tsx filename="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = (await searchParams).filters
}
```

```jsx filename="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const filters = (await searchParams).filters
}
```

客戶端元件 (Client Component) **頁面**也可以使用 React 的 [`use`](https://react.dev/reference/react/use) hook 來取得 `searchParams`：

```tsx filename="app/shop/page.tsx" switcher
'use client'
import { use } from 'react'

export default function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const filters = use(searchParams).filters
}
```

```jsx filename="app/page.jsx" switcher
'use client'
import { use } from 'react'

export default function Page({ searchParams }) {
  const filters = use(searchParams).filters
}
```

| 範例 URL | `searchParams` |
| --- | --- |
| `/shop?a=1` | `Promise<{ a: '1' }>` |
| `/shop?a=1&b=2` | `Promise<{ a: '1', b: '2' }>` |
| `/shop?a=1&a=2` | `Promise<{ a: ['1', '2'] }>` |

- 由於 `searchParams` prop 是一個 Promise，您必須使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函數來存取值。
  - 在 14 及更早版本中，`searchParams` 是一個同步 prop。為了協助相容性，您仍可在 Next.js 15 中同步存取它，但此行為未來將會被棄用。
- `searchParams` 是一個 **[請求級 API (Request-time API)](/docs/app/glossary#request-time-apis)**，其值無法事先得知。使用它會使頁面在請求時選擇 **[動態渲染 (dynamic rendering)](/docs/app/glossary#dynamic-rendering)**。
- `searchParams` 是一個純 JavaScript 對象，而非 `URLSearchParams` 實例。

### Page Props 輔助工具

您可以搭配 `PageProps` 為頁面寫型別，從路由字串取得具有強烈型別的 `params` 和 `searchParams`。`PageProps` 是一個全域可用的輔助工具。

```tsx filename="app/blog/[slug]/page.tsx"
export default async function Page(props: PageProps<'/blog/[slug]'>) {
  const { slug } = await props.params
  const query = await props.searchParams
  return <h1>Blog Post: {slug}</h1>
}
```

> **相關資訊**
>
> - 使用字串路由 (literal route) （例如 `'/blog/[slug]'`）可啟用 `params` 的自動完成和嚴格型別。
> - 靜態路由解析 `params` 為 `{}`。
> - 型別在 `next dev`、`next build` 或是使用 `next typegen` 時產生。
> - 類型產生後，`PageProps` 輔助工具已全域可用。不需要匯入。

## 範例

### 根據 `params` 顯示內容

使用 [動態路由區段 (dynamic route segments)](/docs/app/api-reference/file-conventions/dynamic-routes)，您可以根據 `params` prop 顯示或擷取頁面的特定內容。

```tsx filename="app/blog/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
  return <h1>Blog Post: {slug}</h1>
}
```

### 使用 searchParams 處理過濾

您可以使用 `searchParams` prop，根據 URL 的查詢字串來處理過濾、分頁或排序。

```tsx filename="app/shop/page.tsx" switcher
export default async function Page({
  searchParams,
}: {
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

```jsx filename="app/shop/page.js" switcher
export default async function Page({ searchParams }) {
  const { page = '1', sort = 'asc', query = '' } = await searchParams

  return (
    <div>
      <h1>Product Listing</h1>
      <p>Search query: {query}</p>
      <p>Current page: {page}</p>
      <p>Sort order: {sort}</p>
    </div>
  )
}
```

### 在客戶端元件中讀取 searchParams 和 params

要在客戶端元件 (Client Component，無法為 async) 中使用 `searchParams` 和 `params`，您可以使用 React 的 [`use`](https://react.dev/reference/react/use) 函數來讀取 promise：

```tsx filename="app/page.tsx" switcher
'use client'

import { use } from 'react'

export default function Page({
  params,
  searchParams,
}: {
  params: Promise<{ slug: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```

```js filename="app/page.js" switcher
'use client'

import { use } from 'react'

export default function Page({ params, searchParams }) {
  const { slug } = use(params)
  const { query } = use(searchParams)
}
```
