**模板** 檔案類似於 [佈局 (layout)](/docs/app/getting-started/layouts-and-pages#creating-a-layout)，它會包圍佈局或頁面。與跨路由持久存在並保持狀態的佈局不同，模板會被賦予唯一的 key，這意味著子 Client Components 在導航時會重置它們的狀態。

當您需要以下情況時，它們非常有用：

- 在導航時重新同步 `useEffect`。
- 在導航時重置子 Client Components 的狀態。例如，一個輸入欄位。
- 更改預設框架行為。例如，佈局內的 Suspense boundaries 僅在首次載入時顯示 fallback，而模板則在每次導航時顯示。

## 慣例

模板可以通過從 `template.js` 檔案匯出預設 React 元件來定義。該元件應該接受 `children` 屬性。

```tsx filename="app/template.tsx" switcher
export default function Template({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>
}
```

```jsx filename="app/template.js" switcher
export default function Template({ children }) {
  return <div>{children}</div>
}
```

就嵌套而言，`template.js` 是在佈局和其子元件之間呈現。這是簡化的輸出：

```jsx filename="Output"
<Layout>
  {/* Note that the template is given a unique key. */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```

在 [組件層級結構](/docs/app/getting-started/project-structure#component-hierarchy) 中，`template.js` 在 `layout.js` 和 `error.js` 之間渲染。它包裝了 `error.js`、`loading.js`、`not-found.js` 和 `page.js`，但並**不**包裝該分段中的 `layout.js`。

## Props (屬性)

### children (必填)

模板接受 `children` 屬性。

```jsx filename="Output"
<Layout>
  {/* Note that the template is automatically given a unique key. */}
  <Template key={routeParam}>{children}</Template>
</Layout>
```

## 行為

- **伺服組件 (Server Components)**: 預設情況下，模板是伺服組件。
- **與導航相關**：模板會獲得其自身分段層級的唯一 key。當該分段（包括其動態參數）更改時，它們會重新掛載。更深層分段內的導航不會重新掛載高階模板。搜索參數不會觸發重新掛載。
- **狀態重置**：模板內的任何客戶端組件 (Client Component) 在導航時將重置其狀態。
- **副作用重跑**：像 `useEffect` 這樣的副作用會隨組件重新掛載而重新同步。
- **DOM 重置**：模板內的 DOM 元素將被完全重新創建。

### 導航和重新掛載期間的模板

本節闡述了模板在導航期間的行為。它逐步展示了每一處路由更改時哪些模板會重新掛載以及原因。

使用此專案樹狀結構：

```
app
├── about
│   ├── page.tsx
├── blog
│   ├── [slug]
│   │   └── page.tsx
│   ├── page.tsx
│   └── template.tsx
├── layout.tsx
├── page.tsx
└── template.tsx
```

從 `/` 開始，React 樹狀結構大致如下。

> 注意：範例中顯示的 `key` 值僅供說明，您的應用程式中的值可能不同。

```jsx filename="Output"
<RootLayout>
  {/* app/template.tsx */}
  <Template key="/">
    <Page />
  </Template>
</RootLayout>
```

導航至 `/about`（首段更改），根模板 key 更改，它重新掛載：

```jsx filename="Output"
<RootLayout>
  {/* app/template.tsx */}
  <Template key="/about">
    <AboutPage />
  </Template>
</RootLayout>
```

導航至 `/blog`（首段更改），根模板 key 更改，它重新掛載，並且部落格層級模板掛載：

```jsx filename="Output"
<RootLayout>
  {/* app/template.tsx (root) */}
  <Template key="/blog">
    {/* app/blog/template.tsx */}
    <Template key="/blog">
      <BlogIndexPage />
    </Template>
  </Template>
</RootLayout>
```

在同一首段內導航至 `/blog/first-post`（子分段更改），根模板 key 不會更改，但部落格層級模板 key 更改，它重新掛載：

```jsx filename="Output"
<RootLayout>
  {/* app/template.tsx (root) */}
  <Template key="/blog">
    {/* app/blog/template.tsx */}
    {/* remounts because the child segment at this level changed */}
    <Template key="/blog/first-post">
      <BlogPostPage slug="first-post" />
    </Template>
  </Template>
</RootLayout>
```

導航至 `/blog/second-post`（相同首段，不同子分段），根模板 key 不會更改，但部落格層級模板 key 更改，它再次重新掛載：

```jsx filename="Output"
<RootLayout>
  {/* app/template.tsx (root) */}
  <Template key="/blog">
    {/* app/blog/template.tsx */}
    {/* remounts again due to changed child segment */}
    <Template key="/blog/second-post">
      <BlogPostPage slug="second-post" />
    </Template>
  </Template>
</RootLayout>
```
