特殊檔案 `loading.js` 可協助您使用 [React Suspense](https://react.dev/reference/react/Suspense) 建立有意義的載入使用者介面。透過此命名規範，您可以在路線區段內容串流傳輸時，從伺服器顯示 [即時載入狀態](#instant-loading-states)。一旦完成，新內容會自動替換。

```tsx filename="app/feed/loading.tsx" switcher
export default function Loading() {

  return <p>Loading...</p>

```

```jsx filename="app/feed/loading.js" switcher
export default function Loading() {
  // Or a custom loading skeleton component
  return <p>Loading...</p>
}
```

在 `loading.js` 檔案中，您可以新增任何輕量級的載入使用者介面。您可能會發現使用 [React Developer Tools](https://react.dev/learn/react-developer-tools) 來手動切換 Suspense 邊界很有幫助。

預設情況下，此檔案為一個 [伺服器組件](/docs/app/getting-started/server-and-client-components)，但也能夠透過 `"use client"` 指令作為客戶端組件使用。

## 參考

### 參數

載入使用者介面組件不支援任何參數。

## 行為

### 導航

- 後備 UI 已 [預取](/docs/app/getting-started/linking-and-navigating#prefetching)，這使得導航立即進行，除非預取尚未完成。
- 導航是可中斷的，這意味著更改路由不必等待路由內容完全加載後再導航到另一個路由。
- 新的路由段加載時，通用佈局仍保持互動性。

### 即時載入狀態

即時加載狀態是在導航時立即顯示的後備 UI。您可以預先渲染加載指示器，例如骨架屏和旋轉器，或者螢幕上的一些小但有意義的部分，例如封面照片、標題等。這有助於用戶理解應用程式正在響應，並提供更好的用戶體驗。

在資料夾內新增 `loading.js` 檔案即可建立載入狀態。

在同一個資料夾中，`loading.js` 將嵌套在 `layout.js` 內部。它會自動將 `page.js` 檔案及其下任何子項目包裹在 `<Suspense>` 邊界內。

在 [組件層次結構](/docs/app/getting-started/project-structure#component-hierarchy) 中，`loading.js` 將 `not-found.js`、`page.js` 和嵌套的 `layout.js` 文件包裹在 `<Suspense>` 邊界中。它**不**包裹相同段中的 `layout.js`、`template.js` 或 `error.js`。

> **值得了解**：如果佈局存取未快取或運行時數據（例如 `cookies()`、`headers()` 或未快取請求），`loading.js` 將不會顯示其後備文件。
>
> - **沒有 [快取組件](/docs/app/getting-started/caching)**：導航會阻塞，直到佈局渲染完成。
> - **有 [快取組件](/docs/app/getting-started/caching)**：佈局中的未快取或運行時數據存取必須明確包裹在 `<Suspense>` 中，否則 Next.js 會編譯時引導您出錯。靜態外殼會先流式傳輸，未快取內容隨後填充。
>
> 為了確保即時導航，請將未快取數據獲取從 `layout.js` 移動到 `page.js`，或者將佈局中的運行時數據存取包裹在其自己的 `<Suspense>` 邊界中。查看 [layout.js 注意事項](/docs/app/api-reference/file-conventions/layout#interaction-with-loadingjs) 以獲取詳細資訊和範例。

### 搜尋引擎優化 (SEO)

- 對於僅爬取靜態 HTML 且無法像完整瀏覽器那樣執行 JavaScript 的 bots（例如 Twitterbot），Next.js 會在流式傳輸 UI 之前解析 [`generateMetadata`](/docs/app/api-reference/functions/generate-metadata)，並且元數據會放在初始 HTML 的 `<head>` 中。
- 否則，可使用 [流式傳輸元數據](/docs/app/api-reference/functions/generate-metadata#streaming-metadata)。Next.js 會自動檢測使用者代理，以在阻塞和流式傳輸行為之間切換。
- 由於流式傳輸是服務端渲染，它不影響 SEO。您可以使用 Google 的 [Rich Results Test](https://search.google.com/test/rich-results) 工具查看頁面對 Google 網絡爬蟲的顯示方式，並查看序列化 HTML（[來源](https://web.dev/rendering-on-the-web/#seo-considerations)）。

### 狀態代碼

當進行串流時，將傳回 `200` 狀態代碼，表示請求已成功。

服務器仍然可以在流式傳輸的內容本身中告知客戶端錯誤或問題，例如在使用 [`redirect`](/docs/app/api-reference/functions/redirect) 或 [`notFound`](/docs/app/api-reference/functions/not-found) 時。由於響應標頭已發送給客戶端，因此無法更新響應的狀態代碼。

例如，當 404 頁面流式傳輸給客戶端時，Next.js 會在流式傳輸的 HTML 中包含一個 `<meta name="robots" content="noindex">` 標記。這可以防止搜索引擎索引該 URL，即使 HTTP 狀態為 200。請參閱 Google 關於 [`robots` meta tag](https://developers.google.com/search/docs/crawling-indexing/robots-meta-tag) 的指南。

某些爬蟲可能將這些響應標記為“soft 404s”。在流式傳輸情況下，這不會導致索引化，因為頁面已在 HTML 中明確標記為 `noindex`。

如果您需要 404 狀態碼，出於合規性或分析目的，請確保在響應載體傳輸之前資源存在，這樣服務器才能設置 HTTP 狀態碼。

您可以在 [`proxy`](/docs/app/api-reference/file-conventions/proxy) 中運行此檢查，將缺失的 slug 重新寫入未找到的路由，或者 [生成 404 響應](/docs/app/api-reference/file-conventions/proxy#producing-a-response)。保持代理檢查快速，並避免在那裡獲取完整內容。

<details>

<summary>響應體何時開始流式傳輸？</summary>

當 Suspense 後備渲染時（例如 `loading.tsx`）或當服務器組件在 `Suspense` 邊界下掛起時，響應載體開始流式傳輸。在這些邊界之前放置 `notFound()`，並在可能掛起的任何 `await` 之前。

要開始流式傳輸，必須設置響應標頭。這就是為什麼在流式傳輸開始後無法更改狀態碼的原因。

</details>

### 瀏覽器限制

[某些瀏覽器](https://bugs.webkit.org/show_bug.cgi?id=252413) 會緩存串流回應。您可能會在看到回應超過 1024 位元組之前都看不到串流回應。這通常只會影響 "Hello World" 範例應用程式，但不會影響應用程式。

## 範例

### 使用 Suspense 進行串流

除了 `loading.js`，您還可以為自己的 UI 組件手動創建 Suspense 邊界。App Router 支援使用 [Suspense](https://react.dev/reference/react/Suspense) 進行流式傳輸。查看 [流式傳輸指南](/docs/app/guides/streaming) 了解更多關於流式傳輸工作原理的內容，包括細粒度的 Suspense 模式、路由處理器流式傳輸和基礎設施考慮。

`<Suspense>` 的運作方式是包裝執行非同步動作的組件（例如：獲取數據），在其進行顯示後備使用者介面（例如：骨架、旋轉圖示），然後在動作完成後替換為您自己的組件。

```tsx filename="app/dashboard/page.tsx" switcher
import { Suspense } from 'react'
import { PostFeed, Weather } from './Components'

export default function Posts() {
  return (
    <section>
      <Suspense fallback={<p>Loading feed...</p>}>
        <PostFeed />
      </Suspense>
      <Suspense fallback={<p>Loading weather...</p>}>
        <Weather />
      </Suspense>
    </section>
  )
}
```

```jsx filename="app/dashboard/page.js" switcher
import { Suspense } from 'react'
import { PostFeed, Weather } from './Components'

export default function Posts() {
  return (
    <section>
      <Suspense fallback={<p>Loading feed...</p>}>
        <PostFeed />
      </Suspense>
      <Suspense fallback={<p>Loading weather...</p>}>
        <Weather />
      </Suspense>
    </section>
  )
}
```

透過使用 Suspense，您可以獲得以下優勢：

1. **串流式伺服器渲染** - 從伺服器逐步渲染 HTML 至用戶端。
2. **選擇性水合 (Selective Hydration)** - React 根據使用者互動優先決定先讓哪些組件具互動性。

若要查看更多 Suspense 範例和用途，請參閱 [React 文件](https://react.dev/reference/react/Suspense)。
