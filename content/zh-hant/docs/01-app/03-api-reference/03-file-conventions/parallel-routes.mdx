平行路由允許您在相同的佈局中同時或條件性地渲染一個或多個頁面。它們對於應用程式的高度動態部分很有用，例如社交網站上的儀表板和資訊流。

例如，考慮一個儀表板，您可以使用平行路由來同時渲染 `team` 和 `analytics` 頁面：

## 慣例

### 插槽

平行路由是使用具名的 **插槽** 建立的。插槽透過 `@folder` 慣例來定義。例如，以下檔案結構定義了兩個插槽：`@analytics` 和 `@team`：

<Image
  alt="平行路由檔案系統結構"
  srcLight="/docs/light/parallel-routes-file-system.png"
  srcDark="/docs/dark/parallel-routes-file-system.png"
  width="1600"
  height="687"
/>

插槽作為 prop 傳遞給共享的父層佈局。以上範例中，`app/layout.js` 中的元件現在接受 `@analytics` 和 `@team` 插槽 prop，並可將它們與 `children` prop 並行渲染：

```tsx filename="app/layout.tsx" switcher
export default function Layout({
  children,
  team,
  analytics,
}: {
  children: React.ReactNode
  analytics: React.ReactNode
  team: React.ReactNode
}) {
  return (
    <>
      {children}
      {team}
      {analytics}
    </>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function Layout({ children, team, analytics }) {
  return (
    <>
      {children}
      {team}
      {analytics}
    </>
  )
}
```

然而，插槽 (slot) **並非**路由片段 (route segment)，也不會影響 URL 結構。例如，對於 `/@analytics/views`，URL 會是 `/views`，因為 `@analytics` 是一個插槽 (slot)。插槽會與一般的 [Page](/docs/app/api-reference/file-conventions/page) 組合起來，形成與該路由片段 (route segment) 關聯的主要頁面。因此，在同一個路由片段 (route segment) 層級上，您無法同時擁有獨立的 [預先渲染 (prerendered)](/docs/app/glossary#prerendering) 和 [動態渲染 (dynamically rendered)](/docs/app/glossary#dynamic-rendering) 插槽。如果其中一個插槽 (slot) 是動態的，該層級的所有插槽 (slot) 都必須是動態的。

> **值得了解**：
>
> - **children** prop 是一個不須對應到資料夾的隱式插槽。這意味著 `app/page.js` 等同於 `app/@children/page.js`。

### default.js

您可以定義 `default.js` 檔案，以便在初始載入或全頁面重新整理期間，為未匹配的插槽顯示替補。

請考慮以下資料夾結構。

<Image
  alt="平行路由 (Parallel Routes) 未匹配的路由"
  srcLight="/docs/light/parallel-routes-unmatched-routes.png"
  srcDark="/docs/dark/parallel-routes-unmatched-routes.png"
  width="1600"
  height="930"
/>

當導航至 `/settings` 時，`@team` 插槽將渲染 `/settings` 頁面，同時保留 `@analytics` 插槽目前活潑的頁面。

重新整理時，Next.js 將為 `@analytics` 渲染 `default.js`。如果 `default.js` 不存在，則會渲染 `404` 作為替代。

此外，由於 `children` 是隱式插槽，您還需要建立 `default.js` 檔案，以在 Next.js 無法還原父層頁面的活動狀態時，為 `children` 渲染備援。

## 行為

預設情況下，Next.js 會追蹤每個插槽的 *狀態*（或子頁面）。然而，插槽內渲染的內容將取決於導航的類型：

- [**軟導航**](/docs/app/getting-started/linking-and-navigating#client-side-transitions): 在客戶端導航期間，Next.js 將執行 [部分渲染](/docs/app/getting-started/linking-and-navigating#client-side-transitions)，更改插槽內的子頁面，同時保留其他插槽的活躍子頁面，即使它們與當前 URL 不符。
- **硬導航**：在完整頁面載入後（瀏覽器重新整理），Next.js 無法判斷與當前 URL 不符的插槽的活躍狀態。相反地，它會為未匹配的插槽渲染 [`default.js`](#defaultjs) 檔案，如果 `default.js` 不存在則渲染 `404`。

> **值得了解**：
>
> - 對於未匹配的路由 `404` 有助於確保您不會意外渲染您不打算渲染的平行路由頁面。

## 範例

### 使用 `useSelectedLayoutSegment(s)` 配合

兩者 [`useSelectedLayoutSegment`](/docs/app/api-reference/functions/use-selected-layout-segment) 和 [`useSelectedLayoutSegments`](/docs/app/api-reference/functions/use-selected-layout-segments) 都接受 `parallelRoutesKey` 參數，這允許您讀取插槽內的活躍路由區段。

```tsx filename="app/layout.tsx" switcher
'use client'

import { useSelectedLayoutSegment } from 'next/navigation'

export default function Layout({ auth }: { auth: React.ReactNode }) {
  const loginSegment = useSelectedLayoutSegment('auth')
  // ...
}
```

```jsx filename="app/layout.js" switcher
'use client'

import { useSelectedLayoutSegment } from 'next/navigation'

export default function Layout({ auth }) {
  const loginSegment = useSelectedLayoutSegment('auth')
  // ...
}
```

當使用者導航至 `app/@auth/login`（或 URL 欄中的 `/login`），`loginSegment` 將等於字串 `"login"`。

### 條件式路由

您可以使用平行路由根據特定條件（例如使用者角色）來條件式地渲染路由。例如，要為 `/admin` 或 `/user` 角色渲染不同的儀表板頁面：

<Image
  alt="條件式路由圖"
  srcLight="/docs/light/conditional-routes-ui.png"
  srcDark="/docs/dark/conditional-routes-ui.png"
  width="1600"
  height="898"
/>

```tsx filename="app/dashboard/layout.tsx" switcher
import { checkUserRole } from '@/lib/auth'

export default function Layout({
  user,
  admin,
}: {
  user: React.ReactNode
  admin: React.ReactNode
}) {
  const role = checkUserRole()
  return role === 'admin' ? admin : user
}
```

```jsx filename="app/dashboard/layout.js" switcher
import { checkUserRole } from '@/lib/auth'

export default function Layout({ user, admin }) {
  const role = checkUserRole()
  return role === 'admin' ? admin : user
}
```

### 標籤組

您可以在插槽中加入 `layout`，以允許使用者獨立導航該插槽。這非常適合建立標籤。

例如，`@analytics` 插槽具有兩個子頁面：`/page-views` 和 `/visitors`。

<Image
  alt="具有兩個子頁面和佈局的 Analytics 插槽"
  srcLight="/docs/light/parallel-routes-tab-groups.png"
  srcDark="/docs/dark/parallel-routes-tab-groups.png"
  width="1600"
  height="768"
/>

在 `@analytics` 中，建立 [`layout`](/docs/app/api-reference/file-conventions/layout) 檔案以在兩個頁面之間共用標籤：

```tsx filename="app/@analytics/layout.tsx" switcher
import Link from 'next/link'

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <>
      <nav>
        <Link href="/page-views">Page Views</Link>
        <Link href="/visitors">Visitors</Link>
      </nav>
      <div>{children}</div>
    </>
  )
}
```

```jsx filename="app/@analytics/layout.js" switcher
import Link from 'next/link'

export default function Layout({ children }) {
  return (
    <>
      <nav>
        <Link href="/page-views">Page Views</Link>
        <Link href="/visitors">Visitors</Link>
      </nav>
      <div>{children}</div>
    </>
  )
}
```

### 模態視窗

平行路由可以與 [攔截路由](/docs/app/api-reference/file-conventions/intercepting-routes) 搭配使用，來建立支援深度連結的模態視窗。這讓您可以解決建立模态視窗時的常見挑戰，例如：

- 使模態視窗內容可 **透過 URL 共用**。
- **保留情境** 時重新整理頁面，而不是關閉模態視窗。
- **回溯導航時關閉模態視窗**，而不是返回上一個路由。
- **向前導航時重新開啟模態視窗**。

請考慮以下 UI 模式，其中使用者可以使用客戶端導航從佈局開啟登入模態視窗，或存取獨立的 `/login` 頁面：

<Image
  alt="平行路由圖"
  srcLight="/docs/light/parallel-routes-auth-modal.png"
  srcDark="/docs/dark/parallel-routes-auth-modal.png"
  width="1600"
  height="687"
/>

要實作此模式，請先建立一個 `/login` 路由來渲染您的 **主要** 登入頁面。

<Image
  alt="平行路由圖"
  srcLight="/docs/light/parallel-routes-modal-login-page.png"
  srcDark="/docs/dark/parallel-routes-modal-login-page.png"
  width="1600"
  height="768"
/>

```tsx filename="app/login/page.tsx" switcher
import { Login } from '@/app/ui/login'

export default function Page() {
  return <Login />
}
```

```jsx filename="app/login/page.js" switcher
import { Login } from '@/app/ui/login'

export default function Page() {
  return <Login />
}
```

然後，在 `@auth` 插槽內，加入 [`default.js`](/docs/app/api-reference/file-conventions/default) 檔案回傳 `null`。這確保模態視窗在不活躍時不會渲染。

```tsx filename="app/@auth/default.tsx" switcher
export default function Default() {
  return null
}
```

```jsx filename="app/@auth/default.js" switcher
export default function Default() {
  return null
}
```

在您的 `@auth` 插槽內，透過將 `<Modal>` 元件及其子內容匯入到 `@auth/(.)login/page.tsx` 檔案來攔截 `/login` 路由，並將資料夾名稱更新為 `/@auth/(.)login/page.tsx`。

```tsx filename="app/@auth/(.)login/page.tsx" switcher
import { Modal } from '@/app/ui/modal'
import { Login } from '@/app/ui/login'

export default function Page() {
  return (
    <Modal>
      <Login />
    </Modal>
  )
}
```

```jsx filename="app/@auth/(.)login/page.js" switcher
import { Modal } from '@/app/ui/modal'
import { Login } from '@/app/ui/login'

export default function Page() {
  return (
    <Modal>
      <Login />
    </Modal>
  )
}
```

> **小貼士：**
>
> - 慣例 `(.)` 用於截獲路由。請參閱 [截獲路由 (Intercepting Routes)](/docs/app/api-reference/file-conventions/intercepting-routes#convention) 文件以獲取更多信息。
> - 通過將 <Modal> 功能與模態框內容 (`<Login>`) 分開，您可以確保模態框內的任何內容，例如 [表單 (forms)](/docs/app/guides/forms)，都是伺服器元件。請參閱 [混用客戶端與伺服器元件 (Interleaving Client and Server Components)](/docs/app/getting-started/server-and-client-components#interleaving-server-and-client-components) 獲取更多信息。

#### 開啟模態視窗

現在，您可以利用 Next.js 路由器來開啟和關閉模態視窗。這確保在模態視窗開啟、以及導航向前和向後時，URL 能正確更新。

要開啟模態視窗，請將 `@auth` 插槽作為 prop 傳遞給父層佈局，並與 `children` prop 並行渲染。

```tsx filename="app/layout.tsx" switcher
import Link from 'next/link'

export default function Layout({
  auth,
  children,
}: {
  auth: React.ReactNode
  children: React.ReactNode
}) {
  return (
    <>
      <nav>
        <Link href="/login">Open modal</Link>
      </nav>
      <div>{auth}</div>
      <div>{children}</div>
    </>
  )
}
```

```jsx filename="app/layout.js" switcher
import Link from 'next/link'

export default function Layout({ auth, children }) {
  return (
    <>
      <nav>
        <Link href="/login">Open modal</Link>
      </nav>
      <div>{auth}</div>
      <div>{children}</div>
    </>
  )
}
```

當使用者點擊 `<Link>` 時，將開啟模態視窗，而不是導航至 `/login` 頁面。但是，在重新整理或初始載入時，導航至 `/login` 會將使用者帶往主要登入頁面。

#### 關閉模態視窗

您可以透過呼叫 `router.back()` 或使用 `Link` 元件來關閉模態視窗。

```tsx filename="app/ui/modal.tsx" switcher
'use client'

import { useRouter } from 'next/navigation'

export function Modal({ children }: { children: React.ReactNode }) {
  const router = useRouter()

  return (
    <>
      <button
        onClick={() => {
          router.back()
        }}
      >
        Close modal
      </button>
      <div>{children}</div>
    </>
  )
}
```

```jsx filename="app/ui/modal.js" switcher
'use client'

import { useRouter } from 'next/navigation'

export function Modal({ children }) {
  const router = useRouter()

  return (
    <>
      <button
        onClick={() => {
          router.back()
        }}
      >
        Close modal
      </button>
      <div>{children}</div>
    </>
  )
}
```

當使用 `Link` 元件導航到不應該再渲染 `@auth` 插槽的頁面時，我們需要確保平行路由匹配到一個回傳 `null` 的元件。例如，當導航回根頁面時，我們建立一個 `@auth/page.tsx` 元件：

```tsx filename="app/ui/modal.tsx" switcher
import Link from 'next/link'

export function Modal({ children }: { children: React.ReactNode }) {
  return (
    <>
      <Link href="/">Close modal</Link>
      <div>{children}</div>
    </>
  )
}
```

```jsx filename="app/ui/modal.js" switcher
import Link from 'next/link'

export function Modal({ children }) {
  return (
    <>
      <Link href="/">Close modal</Link>
      <div>{children}</div>
    </>
  )
}
```

```tsx filename="app/@auth/page.tsx" switcher
export default function Page() {
  return null
}
```

```jsx filename="app/@auth/page.js" switcher
export default function Page() {
  return null
}
```

或者，如果您導航至任何其他頁面（例如 `/foo`、`/foo/bar` 等），您可以使用萬用字元插槽：

```tsx filename="app/@auth/[...catchAll]/page.tsx" switcher
export default function CatchAll() {
  return null
}
```

```jsx filename="app/@auth/[...catchAll]/page.js" switcher
export default function CatchAll() {
  return null
}
```

> **值得了解**：
>
> - 我們在 `@auth` 插槽中使用萬用字元路由來關閉模態視窗，這是基於平行路由如何運作的。由於導航至不再匹配插槽的路由時，客戶端導航仍會顯示，因此我們需要將插槽匹配到回傳 `null` 的路由以關閉模態視窗。
> - 其他範例可能包括在圖庫中開啟照片模態視窗，同時也有專屬的 `/photo/[id]` 頁面，或在側邊模態視窗中開啟購物車。
> - [查看範例](https://github.com/vercel-labs/nextgram) 說明使用攔截和並列路由的對話方塊。
