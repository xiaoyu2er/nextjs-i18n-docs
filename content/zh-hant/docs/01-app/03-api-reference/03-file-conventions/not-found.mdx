Next.js 提供了兩種處理找不到頁面情況的慣例：

- **`not-found.js`**: 用於呼叫路由區塊中的 [`notFound`](/docs/app/api-reference/functions/not-found) 函式時。
- **`global-not-found.js`**: 用於定義整個應用程式中未匹配路由的全域 404 頁面。此作業在路由層級處理，不依賴於渲染佈局或頁面。

## `not-found.js`

**not-found** 檔案是用來在路由區塊 (route segment) 中拋出 [`notFound`](/docs/app/api-reference/functions/not-found) 函式時渲染 UI。除了提供自訂的 UI 之外， Next.js 會為串流回應 (streamed responses) 返回 `200` HTTP 狀態碼，為非串流回應 (non-streamed responses) 返回 `404`（關於 SEO 的詳細資訊，請參閱 [狀態碼 (Status Codes)](/docs/app/api-reference/file-conventions/loading#status-codes)）。

```tsx filename="app/not-found.tsx" switcher
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

```jsx filename="app/blog/not-found.js" switcher
import Link from 'next/link'

export default function NotFound() {
  return (
    <div>
      <h2>Not Found</h2>
      <p>Could not find requested resource</p>
      <Link href="/">Return Home</Link>
    </div>
  )
}
```

在 [組件層級](/docs/app/getting-started/project-structure#component-hierarchy) 中， `not-found.js` 渲染於 `loading.js` 與 `page.js` 之間。它被來自同一區段中的 `loading.js` 的 <Suspense> 邊界與 `error.js` 的錯誤邊界包裹。

## `global-not-found.js` (實驗性功能)

**`global-not-found.js`** 檔案允許您定義整個應用程式的 404 頁面。與在路由層級運作的 **`not-found.js`** 不同，此檔案用於請求的 URL 未匹配任何路由的所有情況。Next.js **跳過渲染**並直接回傳此全域頁面。

**`global-not-found.js`** 檔案會繞過應用程式的正常渲染，這意味著您需要匯入您的 404 頁面所需的全域樣式、字型或其他相依性。

> **值得注意**：使用較小的全域樣式版本和更簡單的字型族可以改善此頁面的效能。

**`global-not-found.js`** 在無法使用 `layout.js` 和 `not-found.js` 的組合建立 404 頁面時很有用。這可能發生在兩種情況下：

- 您的應用程式有多個根佈局（例如 `app/(admin)/layout.tsx` 和 `app/(shop)/layout.tsx`），因此沒有單一的佈局來組合成全域 404。
- 您的根佈局是使用最高層級動態區段定義的（例如 `app/[country]/layout.tsx`），這使得組建一致的 404 頁面變得更困難。

要啟用此功能，請在 `next.config.ts` 中加入 `globalNotFound` 旗標：

```tsx filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    globalNotFound: true,
  },
}

export default nextConfig
```

然後，在 `app` 目錄的根目錄中建立檔案：`app/global-not-found.js`：

```tsx filename="app/global-not-found.tsx" switcher
// Import global styles and fonts
import './globals.css'
import { Inter } from 'next/font/google'
import type { Metadata } from 'next'

const inter = Inter({ subsets: ['latin'] })

export const metadata: Metadata = {
  title: '404 - Page Not Found',
  description: 'The page you are looking for does not exist.',
}

export default function GlobalNotFound() {
  return (
    <html lang="en" className={inter.className}>
      <body>
        <h1>404 - Page Not Found</h1>
        <p>This page does not exist.</p>
      </body>
    </html>
  )
}
```

```jsx filename="app/global-not-found.js" switcher
// Import global styles and fonts
import './globals.css'
import { Inter } from 'next/font/google'

const inter = Inter({ subsets: ['latin'] })

export const metadata = {
  title: '404 - Page Not Found',
  description: 'The page you are looking for does not exist.',

      <body>
        <h1>404 - Page Not Found</h1>
        <p>This page does not exist.</p>
      </body>
    </html>
  )
}
```

與 `not-found.js` 不同，此檔案必須回傳完整的 HTML 文件，包含 `<html>` 和 `<body>` 標籤。

## 參考

### 屬性 (Props)

**`not-found.js`** 或 **`global-not-found.js`** 元件不會接受任何 prop。

> **值得注意**：除了捕捉預期的 `notFound()` 錯誤外，根目錄 `app/not-found.js` 和 `app/global-not-found.js` 檔案會處理應用程式中任何未匹配的 URL。這意味著訪問應用程式未處理 URL 的用戶將會看到導出的使用者介面。

## 範例

### 資料獲取

預設情況下，`not-found` 是一個伺服器元件。您可以將其標記為 `async` 來抓取和顯示資料：

```tsx filename="app/not-found.tsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

```jsx filename="app/not-found.jsx" switcher
import Link from 'next/link'
import { headers } from 'next/headers'

export default async function NotFound() {
  const headersList = await headers()
  const domain = headersList.get('host')
  const data = await getSiteData(domain)
  return (
    <div>
      <h2>Not Found: {data.name}</h2>
      <p>Could not find requested resource</p>
      <p>
        View <Link href="/blog">all posts</Link>
      </p>
    </div>
  )
}
```

如果需要使用像 `usePathname` 這樣的 Client Component Hook 來根據路徑顯示內容，則必須在客戶端抓取資料。

### 元數據 (Metadata)

對於 **`global-not-found.js`**，您可以匯出 `metadata` 物件或 [`generateMetadata`](/docs/app/api-reference/functions/generate-metadata) 函式，以自訂您 404 頁面的 `<title>`、`<meta>` 和其他標頭標籤：

> **值得注意**：Next.js 會自動為回傳 404 狀態碼的頁面注入 `<meta name="robots" content="noindex" />`，包括 `global-not-found.js` 頁面。

```tsx filename="app/global-not-found.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'Not Found',
  description: 'The page you are looking for does not exist.',
}

export default function GlobalNotFound() {
  return (
    <html lang="en">
      <body>
        <div>
          <h1>Not Found</h1>
          <p>The page you are looking for does not exist.</p>
        </div>
      </body>
    </html>
  )
}
```

