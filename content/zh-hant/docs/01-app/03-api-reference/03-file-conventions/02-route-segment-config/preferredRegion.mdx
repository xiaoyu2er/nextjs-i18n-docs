`preferredRegion` 選項允許您指定路由區段的首選部署區域。此值將傳遞給您的部署平台。

```tsx filename="layout.tsx | page.tsx | route.ts" switcher
export const preferredRegion = // string || string[]
```

```js filename="layout.js | page.js | route.js" switcher
export const preferredRegion = // string || string[]
```

- **`string`**: 將路由部署至特定區域。可用的區域代碼取決於平台。例如，`'iad1'`。
- **`string[]`**: 將路由部署至多個指定區域。路由會部署至列表中列出的**所有**區域，而非從列表中選擇一個。例如，`['iad1', 'sfo1']`。

> **小提示**:
>
> - 如果未指定 `preferredRegion`，則會繼承最近的父佈局選項。根佈局的預設值為 `'auto'`。
> - 子區段的值會覆蓋父區段，值不會合併。
> - Next.js 會將區域值傳遞至部署平台。確切行為和可用的區域代碼取決於平台。請參考您部署平台的文件以獲取支援的值。

## Vercel

如果在 Vercel 上部署 Next.js，僅在設定 `export const runtime = 'edge'` 時支援區域。可以傳遞以下選項：

- **`'auto'`** (default): 使用預設區域。
- **`'global'`**: 優先部署路由至所有可用區域。
- **`'home'`**: 優先部署路由至主區域。

如果傳入不支援的值，將拋出錯誤。
