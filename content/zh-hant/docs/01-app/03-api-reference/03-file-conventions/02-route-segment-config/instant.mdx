`unstable_instant` 路由段配置 (route segment config) 將路由選入針對 instant 客戶端導航的驗證。Next.js 在開發期間和構建時檢查，確保緩存結構在該路由的每個可能入口點生成 instant [靜態外殼 (static shell)](/docs/app/glossary#static-shell)

> **值得了解**:
>
> - `unstable_instant` 導出 (export) 僅在 [cacheComponents](/docs/app/api-reference/config/next-config-js/cacheComponents) 啟用時有效。
> - `unstable_instant` 無法用於 客戶端組件 (Client Components)。它將拋出錯誤。

```tsx filename="layout.tsx | page.tsx" switcher
export const unstable_instant = {
  prefetch: 'static',
}

export default function Page() {
  return <div>...</div>
}
```

```jsx filename="layout.js | page.js" switcher
export const unstable_instant = {
  prefetch: 'static',
}

export default function Page() {
  return <div>...</div>
}
```

## 參考

### `prefetch`

控制驗證和預加載模式。

```tsx filename="page.tsx"
export const unstable_instant = {
  prefetch: 'static',
}
```

- **'static'**: 啟用驗證。預加載行為保持不變（預設為靜態）。讀取 cookie 或標頭的組件視為動態，必須位於 Suspense 後方。

### 停用 instant

設定 `false` 以將段豁免於驗證：

```tsx filename="app/dashboard/layout.tsx"
export const unstable_instant = false
```

## 驗證運作方式

`unstable_instant` 在路由中的每個共享佈局邊界 (layout boundary) 觸發驗證。驗證在開發期間（頁面載入和 HMR 更新時）以及構建時執行。錯誤會出現在開發錯誤覆蓋層 (dev error overlay) 中或導致構建失敗。

每個錯誤會識別會阻礙導航的組件。修復方法通常是使用 `use cache` 緩存數據，或將其包裝在 `<Suspense>` 邊界中。

## 檢查載入狀態

使用實驗性標誌啟用 DevTools 開關：

```js filename="next.config.js"
module.exports = {
  experimental: {
    instantNavigationDevToolsToggle: true,
  },
}
```

開啟 Next.js DevTools 並選擇 **Instant Navs**。有兩個選項可用：

- **頁面載入 (Page load)**：點擊 **Reload** 刷新頁面，並在動態數據進入之前，將其凍結為為此路由生成的初始靜態 UI。
- **客戶端導航 (Client navigation)**：一旦啟用，點擊應用程序中的任何鏈接將顯示該頁面的預加載 UI，而不是完整結果。

同時使用兩項檢查，確保您的載入狀態在首次訪問和導航時看起來正確。

## 已知問題：跨項目共享 cookie

DevTools 使用 `next-instant-navigation-testing` cookie 來將 UI 凍結在靜態外殼。由於 cookie 的範圍是域名而不是端埠，在相同域名（通常是 `localhost`）上運行多個項目意味著 cookie 會被跨項目共享，並可能導致意外行為。在切換項目時清除 cookie 或關閉 Instant Navs 面板以避免問題。

> **值得了解**: 這將在穩定該功能時得到修復。

## TypeScript

```tsx
type RuntimeSample = {
  cookies?: Array<{ name: string; value: string }>
  headers?: Array<[string, string]>
  params?: Record<string, string | string[]>
  searchParams?: Record<string, string | string[]>
}

type InstantConfig =
  | false
  | {
      prefetch: 'static'
      from?: string[]
      unstable_disableValidation?: boolean
    }
  | {
      prefetch: 'runtime'
      samples: RuntimeSample[]
      from?: string[]
      unstable_disableValidation?: boolean
    }

export const unstable_instant: InstantConfig = {
  prefetch: 'static',
}
```
