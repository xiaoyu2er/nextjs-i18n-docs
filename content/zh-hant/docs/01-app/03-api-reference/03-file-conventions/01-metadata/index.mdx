title: Metadata Files API 參考
nav_title: Metadata Files
description: Metadata 檔案規格的 API 文件。
---

文件的這個部分涵蓋了 **Metadata 檔案規範**。基於檔案的 metadata 可以通過向路由區段添加特殊的 metadata 檔案來定義。

每個檔案規範都可以通過靜態檔案定義（例如：`opengraph-image.jpg`），或使用程式碼生成檔案的動態變體（例如：`opengraph-image.js`）。

一旦檔案被定義，Next.js 將會自動提供該檔案（生產環境包含雜湊值以用於快取），並用正確的 metadata 更新相關的 head 元素，例如資源的 URL、檔案類型和圖片尺寸。

> **值得注意**：
>
> - 特殊路由處理器 (Route Handlers)，如 [`sitemap.ts`](/docs/app/api-reference/file-conventions/metadata/sitemap)、[`opengraph-image.tsx`](/docs/app/api-reference/file-conventions/metadata/opengraph-image) 及 [`icon.tsx`](/docs/app/api-reference/file-conventions/metadata/app-icons)，以及其他 [元數據檔案 (metadata files)](/docs/app/api-reference/file-conventions/metadata)，預設會被快取。
> - 如果與 [`proxy.ts`](/docs/app/api-reference/file-conventions/proxy) 一起使用，[設定匹配器 (matcher)](/docs/app/api-reference/file-conventions/proxy#matcher) 以排除這些元數據檔案。
