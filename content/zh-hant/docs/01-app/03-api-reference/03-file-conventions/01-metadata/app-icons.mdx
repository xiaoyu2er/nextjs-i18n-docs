使用 `favicon`、`icon` 或 `apple-icon` 的檔案規範可以為您的應用程式設定圖示。

它們對於新增顯示在網頁瀏覽器分頁、手機主螢幕以及搜尋引擎結果等位置的應用程式圖示十分有用。

設定應用程式圖示有兩種方式：

- [使用圖片檔案 (.ico, .jpg, .png)](#image-files-ico-jpg-png)
- [使用程式碼生成圖示 (.js, .ts, .tsx)](#generate-icons-using-code-js-ts-tsx)

## 圖片檔案 (.ico, .jpg, .png)

透過在 `/app` 目錄中放置 `favicon`、`icon` 或 `apple-icon` 圖片檔案來設定應用程式圖示。
`favicon` 圖片只能位於 `app/` 的頂層。

Next.js 將評估該檔案，並自動將適當的標籤新增到您的應用程式的 `<head>` 元素中。

| 檔案規範 | 支援的檔案類型 | 有效位置 |
| ------- | -------------- | ----------- |
| [`網站圖示 (favicon)`](#favicon) | `.ico` | `app/` |
| [`圖示 (icon)`](#icon) | `.ico`, `.jpg`, `.jpeg`, `.png`, `.svg` | `app/**/*` |
| [`蘋果圖示 (apple-icon)`](#apple-icon) | `.jpg`, `.jpeg`, `.png` | `app/**/*` |

### `網站圖示 (favicon)`

將 `favicon.ico` 圖片檔案新增至根路徑 `/app` 路由區段。

```html filename="<head> output"
<link rel="icon" href="/favicon.ico" sizes="any" />
```

### `圖示 (icon)`

新增 `icon.(ico|jpg|jpeg|png|svg)` 圖片檔案。

```html filename="<head> output"
<link
  rel="icon"
  href="/icon?<generated>"
  type="image/<generated>"
  sizes="<generated>"
/>
```

### `蘋果圖示 (apple-icon)`

新增 `apple-icon.(jpg|jpeg|png)` 圖片檔案。

```html filename="<head> output"
<link
  rel="apple-touch-icon"
  href="/apple-icon?<generated>"
  type="image/<generated>"
  sizes="<generated>"
/>
```

> **貼心提示**:
>
> - 您可以透過在檔案名稱末尾添加數字字尾來設定多個圖示。例如，`icon1.png`、`icon2.png` 等。編號檔案將按字典順序排序。
> - 網站圖示 (favicon) 僅能設定在根 `/app` 區段。如果需要更細粒度的控制，可以使用 [`圖示 (icon)`](#icon)。
> - 適當的 `<link>` 標籤和屬性（例如 `rel`、`href`、`type` 和 `sizes`）由檔案的圖示類型和元數據決定。
> - 例如，32 x 32 px 的 `.png` 檔案將具有 `type="image/png"` 和 `sizes="32x32"` 屬性。
> - 當副檔名為 `.svg` 或檔案的影像大小未確定時，會新增 `sizes="any"`。更多詳情請參閱此 [網站圖示指南](https://evilmartians.com/chronicles/how-to-favicon-in-2021-six-files-that-fit-most-needs)。

## 使用程式碼生成圖示 (.js, .ts, .tsx)

除了使用 [圖片檔案](#image-files-ico-jpg-png)，您還可以透過程式碼以編程方式 **生成** 圖示。

透過建立預設匯出函數的 `icon` 或 `apple-icon` 路由來生成應用程式圖示。

| 檔案規範 | 支援的檔案類型 |
| -------- | -------------- |
| `icon` | `.js`, `.ts`, `.tsx` |
| `apple-icon` | `.js`, `.ts`, `.tsx` |

生成圖示最簡單的方式是使用來自 `next/og` 的 [`ImageResponse`](/docs/app/api-reference/functions/image-response) API。

```tsx filename="app/icon.tsx" switcher
import { ImageResponse } from 'next/og'

// Image metadata
export const size = {
  width: 32,
  height: 32,
}
export const contentType = 'image/png'

// Image generation
export default function Icon() {
  return new ImageResponse(
    (
      // ImageResponse JSX element
      <div
        style={{
          fontSize: 24,
          background: 'black',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
          color: 'white',
        }}
      >
        A
      </div>
    ),
    // ImageResponse options
    {
      // For convenience, we can re-use the exported icons size metadata
      // config to also set the ImageResponse's width and height.
      ...size,
    }
  )
}
```

```jsx filename="app/icon.js" switcher
import { ImageResponse } from 'next/og'

// Image metadata
export const size = {
  width: 32,
  height: 32,
}
export const contentType = 'image/png'

// Image generation
export default function Icon() {
  return new ImageResponse(
    (
      // ImageResponse JSX element
      <div
        style={{
          fontSize: 24,
          background: 'black',
          width: '100%',
          height: '100%',
          display: 'flex',
          alignItems: 'center',
          justifyContent: 'center',
          color: 'white',
        }}
      >
        A
      </div>
    ),
    // ImageResponse options
    {
      // For convenience, we can re-use the exported icons size metadata
      // config to also set the ImageResponse's width and height.
      ...size,
    }
  )
}
```

```html filename="<head> output"
<link rel="icon" href="/icon?<generated>" type="image/png" sizes="32x32" />
```

 > **要知道**：
>
> - 預設情況下，生成的圖標都是 [**靜態優化 (statically optimized)**](/docs/app/glossary#prerendering)（在構建時生成並緩存），除非它們使用了 [**請求時 API (Request-time APIs)**](/docs/app/glossary#request-time-apis) 或未緩存數據。
> - 您可以使用 [`generateImageMetadata`](/docs/app/api-reference/functions/generate-image-metadata) 在同一文件中生成多個圖標。
> - 您無法生成 `favicon` 圖標。請改用 [`icon`](#icon) 或 [favicon.ico](#favicon) 文件。
> - App 圖標是特殊的 Route Handler，預設會進行緩存，除非它們使用了 [**請求時 API (Request-time API)**](/docs/app/glossary#request-time-apis) 或 [**動態配置 (dynamic config)**](/docs/app/guides/caching-without-cache-components#dynamic) 選項。

### 屬性 (Props)

預設匯出的函數收到以下屬性：

#### `params` (選填)

 一個 Promise 解析後返回一個對象，其中包含從根區塊到 `icon` 或 `apple-icon` 區塊的 [動態路徑參數 (dynamic route parameters)](/docs/app/api-reference/file-conventions/dynamic-routes)，該 Promise 就地位於其中。

 > **要知道**：如果您使用 [`generateImageMetadata`](/docs/app/api-reference/functions/generate-image-metadata)，函數還將接收一個 `id` prop (屬性)，這是一個 Promise，解析後會返回由 `generateImageMetadata` 返回的項目之一的 `id` 值。

```tsx filename="app/shop/[slug]/icon.tsx" switcher
export default async function Icon({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  // ...
}
```

```jsx filename="app/shop/[slug]/icon.js" switcher
export default async function Icon({ params }) {
  const { slug } = await params
  // ...
}
```

 | Route                           | URL         | `params`                           |
| ------------------------------- | ----------- | ---------------------------------- |
| `app/shop/icon.js`              | `/shop`     | `undefined`                        |
| `app/shop/[slug]/icon.js`       | `/shop/1`   | `Promise<{ slug: '1' }>`           |
| `app/shop/[tag]/[item]/icon.js` | `/shop/1/2` | `Promise<{ tag: '1', item: '2' }>` |

### 回傳值

預設匯出的函數應回傳 `Blob` | `ArrayBuffer` | `TypedArray` | `DataView` | `ReadableStream` | `Response`。

> **貼心提示**: `ImageResponse` 符合此回傳類型。

### 組態匯出 (Config exports)

您可以從 `icon` 或 `apple-icon` 路由選擇性匯出 `size` 和 `contentType` 變數，以設定圖示的元數據。

| 選項 | 類型 |
| ----------- | ----------------------------------------------- |
| [`size`](#size) | `{ width: number; height: number }` |
| [`contentType`](#contenttype) | `string` - [影像 MIME 類型](https://developer.mozilla.org/docs/Web/HTTP/Basics_of_HTTP/MIME_types#image_types) |

#### `size`

```tsx filename="icon.tsx | apple-icon.tsx" switcher
export const size = { width: 32, height: 32 }

export default function Icon() {}
```

```jsx filename="icon.js | apple-icon.js" switcher
export const size = { width: 32, height: 32 }

export default function Icon() {}
```

```html filename="<head> output"
<link rel="icon" sizes="32x32" />
```

#### `contentType`

```tsx filename="icon.tsx | apple-icon.tsx" switcher
export const contentType = 'image/png'

export default function Icon() {}
```

```jsx filename="icon.js | apple-icon.js" switcher
export const contentType = 'image/png'

export default function Icon() {}
```

```html filename="<head> output"
<link rel="icon" type="image/png" />
```

#### 路由區段組態 (Route Segment Config)

 `icon` 和 `apple-icon` 是專門的 [路由處理器 (Route Handlers)](/docs/app/api-reference/file-conventions/route)，可以與頁面 (Pages) 和佈局 (Layouts) 使用相同的 [路由區段組態 (route segment configuration)](/docs/app/api-reference/file-conventions/route-segment-config) 選項。
