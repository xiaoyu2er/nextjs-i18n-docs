除了將 Next.js 特殊的 `app` 或 `pages` 目錄放在專案根目錄之外，Next.js 也支援將應用程式程式碼放置在 `src` 文件夾下的常見模式。

這將應用程式程式碼與主要位於專案根目錄的專案組態檔案區隔開來，這是某些個人和團隊偏好的做法。

若要使用 `src` 文件夾，請分別將 `app` Router 資料夾或 `pages` Router 資料夾移至 `src/app` 或 `src/pages`。

> **值得注意**：
>
> - `/public` 目錄應保留在您的專案根目錄中。
> - 設定檔如 `package.json`, `next.config.js` 和 `tsconfig.json` 應保留在您的專案根目錄中。
> - `.env.*` 檔案應保留在您的專案根目錄中。
> - 如果根目錄中存在 `app` 或 `pages`，則 `src/app` 或 `src/pages` 將被忽略。
> - 如果您使用 `src`，您很可能也會移動其他應用程式資料夾，例如 `/components` 或 `/lib`。
> - 如果您使用代理 (Proxy)，請確保它放置在 `src` 資料夾內。
> - 如果您使用 Tailwind CSS，則需要在 [內容部分](https://tailwindcss.com/docs/content-configuration) 中為 `tailwind.config.js` 檔案添加 `/src` 前綴。
> - 如果您使用 TypeScript 路徑進行匯入，例如 `@/*`，您應該更新 `tsconfig.json` 中的 `paths` 物件以包含 `src/`。
