在你的 Next.js 應用程式中配置伺服器動作 (Server Actions) 行為的選項。

## `allowedOrigins`

從中可以調用伺服器動作 (Server Actions) 的額外安全來源網域列表。Next.js 會比對伺服器動作 (Server Action) 請求的來源與主機網域，確保它們匹配以阻止 CSRF 攻擊。如果未提供，則只允許相同來源。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */

module.exports = {
  experimental: {
    serverActions: {
      allowedOrigins: ['my-proxy.com', '*.my-proxy.com'],
    },
  },
}
```

## `bodySizeLimit`

預設情況下，傳送至伺服器動作 (Server Action) 的請求體最大大小為 1MB，以防止在解析大量數據時消耗過多伺服器資源，以及潛在的 DDoS 攻擊。

然而，你可以使用 `serverActions.bodySizeLimit` 選項配置此限制。它可以接受字節數或 bytes 支援的任何字串格式，例如 `1000`、`'500kb'` 或 `'3mb'`。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */

module.exports = {
  experimental: {
    serverActions: {
      bodySizeLimit: '2mb',
    },
  },
}
```

## 啟用伺服器動作 (Server Actions) (v13)

伺服器動作 (Server Actions) 在 Next.js 14 中成為穩定功能，並預設啟用。然而，如果你使用的是較早版本的 Next.js，可以將 `experimental.serverActions` 設定為 `true` 來啟用它們。

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const config = {
  experimental: {
    serverActions: true,
  },
}

module.exports = config
```
