一些常見請求的功能可作為外掛使用：

- [@next/mdx](https://github.com/vercel/next.js/tree/canary/packages/next-mdx)
- [@next/bundle-analyzer](https://github.com/vercel/next.js/tree/canary/packages/next-bundle-analyzer)

為了擴展我們對 `webpack` 的使用，您可以在 `next.config.js` 內定義一個函式來擴展其配置，如下所示：

```js filename="next.config.js"
module.exports = {
  webpack: (
    config,
    { buildId, dev, isServer, defaultLoaders, nextRuntime, webpack }
  ) => {
    // Important: return the modified config
    return config
  },
}
```

> `webpack` 函式會執行三次，兩次用於服務器 (nodejs / edge runtime)，一次用於客戶端。這允許您使用 `isServer` 屬性來區分客戶端和服務器的配置。

webpack 函式的第二個參數是一個包含以下屬性的物件：

- `buildId`: `String` - 構建 ID，用作構建之間唯一的標識符。
- `dev`: `Boolean` - 指示編譯是否在開發模式下進行。
- `isServer`: `Boolean` - 服務器端編譯時為 `true`，客戶端編譯時為 `false`。
- `nextRuntime`: `String | undefined` - 服務器端編譯的目標運行時；可以為 `"edge"` 或 `"nodejs"`，客戶端編譯時為 `undefined`。
- `defaultLoaders`: `Object` - Next.js 內部使用的預設加載器：
  - `babel`: `Object` - 預設 `babel-loader` 配置。

``defaultLoaders.babel`` 的使用範例：

```js
// Example config for adding a loader that depends on babel-loader
// This source was taken from the @next/mdx plugin source:
// https://github.com/vercel/next.js/tree/canary/packages/next-mdx
module.exports = {
  webpack: (config, options) => {
    config.module.rules.push({
      test: /\.mdx/,
      use: [
        options.defaultLoaders.babel,
        {
          loader: '@mdx-js/loader',
          options: pluginOptions.options,
        },
      ],
    })

    return config
  },
}
```

#### nextRuntime

請注意，當 `nextRuntime` 為 `"edge"` 或 `"nodejs"` 時，`isServer` 的值為 `true`。`nextRuntime` 設定為 `"edge"` 目前僅用於 Edge 運行時中的代理 (proxy) 和服務端組件 (Server Components)。
