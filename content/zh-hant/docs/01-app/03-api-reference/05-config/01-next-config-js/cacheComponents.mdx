---
title: 組件緩存 (cacheComponents)
description: 學習如何在 Next.js 中啟用 cacheComponents 旗標。
---

`cacheComponents` 旗標是 Next.js 中的功能，除非明確緩存，否則會導致 App Router 中的數據獲取操作被排除於預渲染之外。這對於優化 Server Components 中未緩存數據獲取的效能很有用。

如果您的應用程式需要在運行時獲取新數據，而不是從預渲染緩存提供數據，則這將很有用。

預計應與 [`use cache`](/docs/app/api-reference/directives/use-cache) 搭配使用，使得預設情況下您的數據獲取在執行階段發生，除非您以頁面、函式或元件層級使用 use cache 定義應用程式的特定部分來進行快取。

## 使用方法

要在您的 `next.config.ts` 檔案中啟用 `cacheComponents` 旗標，請將其設定為 `true`：

```ts filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  cacheComponents: true,
}

export default nextConfig
```

啟用 cacheComponents 後，您可以使用下列快取函式與設定：

- [使用快取 (use cache) 指令](/docs/app/api-reference/directives/use-cache)
- [`cacheLife` 函式](/docs/app/api-reference/config/next-config-js/cacheLife) 搭配 `use cache` 使用
- [`cacheTag` 函式](/docs/app/api-reference/functions/cacheTag)

## 使用 Activity 進行導航

當啟用 `cacheComponents` 時，Next.js 使用 React 的 [`<Activity>`](https://react.dev/reference/react/Activity) 組件，以在客戶端導航期間保留組件狀態。

當您導航離開時，Next.js 不會卸載先前的路由，而是將 Activity 模式設定為 [`"hidden"`](https://react.dev/reference/react/Activity#activity)。這代表：

- 在路由之間導航時，組件狀態會被保留
- 當您返回時，先前路由會以完整的狀態重新出現
- 當路由被隱藏時，副作用會被清理，並在其再次可見時重新創建

此行為透過在用戶於路由之間來回導航時維持 UI 狀態（如表單輸入或展開區段），從而改善導航體驗。

> **小知識**：Next.js 使用啟發式將幾個最近訪問的路由保持為 `"hidden"`，而較舊的路由則會從 DOM 中移除，以防止過度增長。

當組件保持掛載而非卸載時，某些 UI 模式的行為會有所不同。參閱 [保持 UI 狀態指南](/docs/app/guides/preserving-ui-state) 以了解如何處理下拉選單、對話框和測試等常見模式。
