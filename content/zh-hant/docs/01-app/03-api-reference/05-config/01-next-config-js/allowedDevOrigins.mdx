Next.js 預設會在開發期間阻擋對僅限開發使用的資源和端點發出的跨來源請求，以防止未授權的存取。

若要配置 Next.js 應用程式以允許來自伺服器初始化時的主機名（預設為 ` localhost `）之外的來源發出請求，請使用 `allowedDevOrigins` 配置選項。

`allowedDevOrigins` 會讓您能夠設定在開發模式下可以向開發伺服器發出請求的額外來源。例如，若要使用 `local-origin.dev` 而不僅僅是 ` localhost `，請開啟 `next.config.js` 並加入 `allowedDevOrigins` 配置：

```js filename="next.config.js"
module.exports = {
  allowedDevOrigins: ['local-origin.dev', '*.local-origin.dev'],
}
```
