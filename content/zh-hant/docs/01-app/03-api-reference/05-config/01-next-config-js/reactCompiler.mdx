title: React 編譯器 (reactCompiler)

---

Next.js 支援 [React 編譯器 (React Compiler)](https://react.dev/learn/react-compiler/introduction)，這是一種旨在透過自動最佳化元件渲染來提升效能的工具。這減少了使用 `useMemo` 和 `useCallback` 進行手動記憶化的需求。

Next.js 內建以 SWC 撰寫的自訂效能最佳化，使 React 編譯器更有效率。與在每個檔案上執行編譯器不同，Next.js 會分析您的專案，並僅將 React 編譯器套用於相關檔案。這避免了不必要的工作，並帶來比僅使用 Babel 插件更快的建置速度。

## 運作原理

React 編譯器是透過 Babel 插件運作的。為了保持建置速度，Next.js 使用自訂的 SWC 最佳化，僅將 React 編譯器套用於相關檔案——例如那些包含 JSX 或 React Hooks 的檔案。

這避免了編譯所有內容，並將效能成本保持在最低。與預設的基於 Rust 的編譯器相比，您可能會看到建置速度稍慢，但影響很小且局限於此。

要使用它，請安裝 `babel-plugin-react-compiler`：

```bash package="pnpm"
pnpm add -D babel-plugin-react-compiler
```

```bash package="npm"
npm install -D babel-plugin-react-compiler
```

```bash package="yarn"
yarn add -D babel-plugin-react-compiler
```

```bash package="bun"
bun add -D babel-plugin-react-compiler
```

接著，在 next.config.js 中加入 reactCompiler 選項：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  reactCompiler: true,
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactCompiler: true,
}

module.exports = nextConfig
```

## 註解

您可以將編譯器配置為在「選擇加入 (opt-in)」模式運行，如下所示：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  reactCompiler: {
    compilationMode: 'annotation',
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactCompiler: {
    compilationMode: 'annotation',
  },
}

module.exports = nextConfig
```

然後，您可以使用 React 的 `"use memo"` 指令來標註特定的元件或 Hook，以選擇加入：

```ts filename="app/page.tsx" switcher
export default function Page() {
  'use memo'
  // ...
}
```

```js filename="app/page.js" switcher
export default function Page() {
  'use memo'
  // ...
}
```

> **注意：** 您也可以使用 React 的 `"use no memo"` 指令來達到相反的效果，以取消元件或 Hook 的選擇加入。
