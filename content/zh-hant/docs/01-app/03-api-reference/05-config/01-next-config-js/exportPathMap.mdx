> 此功能專用於 `next export`，且目前 **棄用 (deprecated)**，建議改用搭配 `pages` 的 `getStaticPaths` 或搭配 `app` 的 `generateStaticParams`。

`exportPathMap` 讓您指定請求路徑到頁面目的地的映射，以供導出期間使用。在 `exportPathMap` 中定義的路徑在使用 [`next dev`](/docs/app/api-reference/cli/next#next-dev-options) 時也會可用。

讓我們從一個範例開始，為具有下列頁面的應用程式建立自訂 `exportPathMap`：

- `pages/index.js`
- `pages/about.js`
- `pages/post.js`

開啟 `next.config.js` 並新增下列 `exportPathMap` 設定：

```js filename="next.config.js"
module.exports = {
  exportPathMap: async function (
    defaultPathMap,
    { dev, dir, outDir, distDir, buildId }
  ) {
    return {
      '/': { page: '/' },
      '/about': { page: '/about' },
      '/p/hello-nextjs': { page: '/post', query: { title: 'hello-nextjs' } },
      '/p/learn-nextjs': { page: '/post', query: { title: 'learn-nextjs' } },
      '/p/deploy-nextjs': { page: '/post', query: { title: 'deploy-nextjs' } },
    }
  },
}
```

> **提示**: `exportPathMap` 中的 `query` 欄位不能與 [自動靜態最佳化頁面](/docs/pages/building-your-application/rendering/automatic-static-optimization) 或 [`getStaticProps` 頁面](/docs/pages/building-your-application/data-fetching/get-static-props) 搭配使用，因為它們在建置階段已編譯為 HTML 檔案，且在 `next export` 期間無法提供額外的查詢資訊。

接著頁面將被導出為 HTML 檔案，例如，`/about` 將變更為 `/about.html`。

`exportPathMap` 是一個 `async` 函數，接受 2 個引數：第一個是 `defaultPathMap`，這是 Next.js 使用的預設映射。第二個引數為包含以下內容的物件：

- `dev` - 當 `exportPathMap` 在開發模式下被呼叫時為 `true`。執行 `next export` 時為 `false`。在開發模式下 `exportPathMap` 用於定義路徑。
- `dir` - 專案目錄的絕對路徑
- `outDir` - `out/` 目錄的絕對路徑 ([可透過 `-o`](#customizing-the-output-directory) 設定)。當 `dev` 為 `true` 時，`outDir` 的值將為 `null`。
- `distDir` - `.next/` 目錄的絕對路徑 (可透過 [`distDir`](/docs/pages/api-reference/config/next-config-js/distDir) 設定)
- `buildId` - 生成的建置 ID

回傳的物件是一個頁面映射，其中 `key` 是 `pathname`，而 `value` 是一個接受下列欄位的物件：

- `page`: `String` - 要在 `pages` 目錄內渲染的頁面
- `query`: `Object` - 預先渲染時傳遞給 `getInitialProps` 的 `query` 物件。預設值為 `{}`

> 導出的 `pathname` 也可以是檔名（例如 `/readme.md`），但如果您提供的內容與 `.html` 不同，在服務其內容時您可能需要將 `Content-Type` 標頭設為 `text/html`。

## 新增尾隨斜線

可以設定 Next.js 將頁面導出為 `index.html` 檔案並要求尾隨斜線，`/about` 變更為 `/about/index.html` 且可透過 `/about/` 路由。這是 Next.js 9 之前的預設行為。

若要切回並新增尾隨斜線，請開啟 `next.config.js` 並啟用 `trailingSlash` 設定：

```js filename="next.config.js"
module.exports = {
  trailingSlash: true,
}
```
