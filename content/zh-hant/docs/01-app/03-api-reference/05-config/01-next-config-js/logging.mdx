## 選項

<AppOnly>

### 數據獲取

您可以配置記錄等級，以及在運行 Next.js 開發模式時，是否將完整 URL 記錄到控制台。

```js filename="next.config.js"
module.exports = {
  logging: {
    fetches: {
      fullUrl: true,
    },
  },
}
```

任何從 [伺服器組件 (Server Components) HMR 緩存](/docs/app/api-reference/config/next-config-js/serverComponentsHmrCache) 還原的 `fetch` 請求預設都不會記錄。不過，可以通過將 `logging.fetches.hmrRefreshes` 設置為 `true` 來啟用此功能。

```js filename="next.config.js"
module.exports = {
  logging: {
    fetches: {
      hmrRefreshes: true,
    },
  },
}
```

### 伺服器函式

[伺服器函數](https://react.dev/reference/rsc/server-functions) 呼叫在開發期間預設會被記錄。您可以將 `logging.serverFunctions` 設定為 `false` 來停用此功能。

```js filename="next.config.js"
module.exports = {
  logging: {
    serverFunctions: false,
  },
}
```

啟用後，終端機會顯示每個伺服器函數呼叫，包含函數名稱、參數和持續時間：

```bash filename="Terminal"
POST /
  └─ ƒ myAction(arg1, arg2) in 5ms app/actions.ts
```

</AppOnly>

### 傳入請求

預設情況下，所有傳入請求在開發模式下都會記錄到控制台。您可以使用 incomingRequests 選項來決定要忽略哪些請求。
由於這僅在開發模式下記錄，此選項不會影響生產構建。

```js filename="next.config.js"
module.exports = {
  logging: {
    incomingRequests: {
      ignore: [/\api\/v1\/health/],
    },
  },
}
```

或者，您也可以將 incomingRequests 設置為 false 來禁用傳入請求記錄。

```js filename="next.config.js"
module.exports = {
  logging: {
    incomingRequests: false,
  },
}
```

### 瀏覽器控制台日誌

您可以在開發期間將瀏覽器控制台日誌（例如 `console.log`、`console.warn`、`console.error`）轉發至終端機。這對於除錯客戶端程式碼很有用，無需檢查瀏覽器的開發者工具。

```js filename="next.config.js"
module.exports = {
  logging: {
    browserToTerminal: true,
  },
}
```

#### Options

`browserToTerminal` 選項接受以下值：

| Value     | Description                                         |
| --------- | --------------------------------------------------- |
| `'warn'`  | 預設僅轉發警告和錯誤                              |
| `'error'` | 僅轉發錯誤                                        |
| `true`    | 轉發所有控制台輸出 (log, info, warn, error)       |
| `false`   | 停用瀏覽器日誌轉發                                  |

```js filename="next.config.js"
module.exports = {
  logging: {
    browserToTerminal: 'warn',
  },
}
```

#### 來源位置

啟用後，瀏覽器日誌預設將包含來源位置資訊（檔案路徑和列號）。例如：

<AppOnly>

```tsx filename="app/page.tsx" highlight={8}
'use client'

export default function Home() {
  return (
    <button
      type="button"
      onClick={() => {
        console.log('Hello World')
      }}
    >
      Click me
    </button>
  )
}
```

點擊按鈕會將此訊息列印至終端機：

```bash filename="Terminal"
[browser] Hello World (app/page.tsx:8:17)
```

</AppOnly>

<PagesOnly>


### 禁用記錄

此外，您可以將 logging 設置為 false 來禁用開發記錄。

```js filename="next.config.js"
module.exports = {
  logging: false,
}
```
