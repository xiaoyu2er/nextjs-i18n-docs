如果您希望將快取頁面和資料持久化到耐久儲存，或在多個容器或您的 Next.js 應用程式的實體之間共用快取，您可以配置 Next.js 快取位置。

> **小提示**： `cacheHandler`（單數）配置專門用於 Next.js 的伺服器快取操作，例如存儲和重新驗證 ISR、路由處理器響應和優化圖像。它**不**用於 `'use cache'` 指令。對於 `'use cache'` 指令，請改用 [`cacheHandlers`](/docs/app/api-reference/config/next-config-js/cacheHandlers)（複數）替代。

```js filename="next.config.js"
module.exports = {
  cacheHandler: require.resolve('./cache-handler.js'),
  cacheMaxMemorySize: 0, // disable default in-memory caching
}
```

查看 [自訂快取處理器](/docs/app/guides/self-hosting#configuring-caching) 的範例，並了解更多關於實作的資訊。

## API 參考

快取處理器可實作以下方法：`get`、`set`、`revalidateTag` 和 `resetRequestCache`。

### `get()`

| 參數 | 類型 | 描述 |
| --------- | -------- | --------------------------------------- |
| `key` | `string` | 儲存值的鍵。 |
| `ctx` | `object` | 包含快取條目類型的上下文。 |

 `ctx` 參數包含一個 `kind` 屬性，用於指示正在獲取的快取條目類型。可能的值包括 `'APP_PAGE'`, `'APP_ROUTE'`, `'PAGES'`, `'FETCH'`, 和 `'IMAGE'`。

若找不到則回傳快取值或 `null`。

### `set()`

| 參數 | 類型             | 說明                              |
| --------- | -------------- | -------------------------------- |
| `key`     | `string`       | 儲存資料的鍵。 |

 `data` 對象包含一個 `kind` 屬性，用於指示快取條目的類型。對於圖像優化， `kind` 將為 `'IMAGE'`，數據將包含 `buffer`, `etag`, `extension` 和 `revalidate` 等屬性。

回傳 `Promise<void>`。

### `revalidateTag()`

| 參數 | 類型                   | 說明                   |
| --------- | ---------------------- | ----------------------------- |
| `tag`     | `string` or `string[]` | 要重新驗證的快取標籤。 |

回傳 `Promise<void>`。了解更多關於 [重新驗證資料](/docs/app/guides/incremental-static-regeneration) 或 [`revalidateTag()`](/docs/app/api-reference/functions/revalidateTag) 函式。

### `resetRequestCache()`

此方法會在下一個請求之前，重置單一請求的臨時記憶體快取。

回傳 `void`。

**小知識：**

- `revalidatePath` 是快取標籤之上的便利層。呼叫 `revalidatePath` 將呼叫您的 `revalidateTag` 函式，您可以選擇是否要根據路徑為快取鍵加上標籤。

## 圖像優化快取

 `cacheHandler` 也可用於快取來自 `next/image` 的優化圖像。要啟用此功能，請在 `next.config.js` 中將 `images.customCacheHandler` 設置為 `true`：

```js filename="next.config.js"
module.exports = {
  cacheHandler: require.resolve('./cache-handler.js'),
  images: {
    customCacheHandler: true,
  },
}
```

> **小提示**：此選配開關將在下一個主要版本中成為預設行為。現在設置它可讓您為圖像優化條目準備快取處理器。

您可以使用 `kind` 屬性區分不同的快取條目類型，並在需要時單獨處理圖像，例如實施清除策略或將圖像存儲在不同的位置。

處理圖像快取條目時， `kind` 將為 `'IMAGE'`，數據將包含 `buffer`, `etag`, `extension` 和 `revalidate` 屬性。

## 平台支援

| 部署選項                                                           | 支援狀態          |
| ------------------------------------------------------------------- | ----------------- |
| [Node.js 伺服器](/docs/app/getting-started/deploying#nodejs-server) | 是               |
| [Docker 容器](/docs/app/getting-started/deploying#docker)      | 是               |
| [靜態匯出](/docs/app/getting-started/deploying#static-export)  | 否                |
| [配接器](/docs/app/getting-started/deploying#adapters)            | 依平台而定 |

了解如何 [設定 ISR](/docs/app/guides/self-hosting#caching-and-isr) 時自行託管 Next.js。
