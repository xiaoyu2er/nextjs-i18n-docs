使用 `turbopack.ignoreIssue` 選項，您可以過濾特定的 [Turbopack](/docs/app/api-reference/turbopack) 錯誤和警告，使它們不出現在 CLI 輸出或錯誤覆蓋層 (Error Overlay) 中。這有助於抑制已知但不影響應用的警告，例如故意未解析的可選依賴 (optional dependencies)。

僅在使用 Turbopack 時才提供此選項 (`next dev --turbopack`)。

## 使用方法

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/vendor/**',
      },
    ],
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/vendor/**',
      },
    ],
  },
}

module.exports = nextConfig
```

## 選項

`ignoreIssue` 陣列中的每個規則均為包含下列欄位的物件：

| 欄位 | 類型 | 必要 | 說明 |
| ----------------------------- | ------------------ | -------- | ------------------------------------------ |
| [`path`](#path) | `string \|\| RegExp` | 是 | 與問題的文件路徑進行匹配 |
| [`title`](#title) | `string \|\| RegExp` | 否 | 與問題的標題進行匹配 |
| [`description`](#description) | `string \|\| RegExp` | 否 | 與問題的說明進行匹配 |

當問題與規則中的 `path` **且** 所有其他指定欄位匹配時，該問題會被抑制。如果僅提供了 `path`，則來自匹配文件的所有問題都會被抑制。

> **小提示：** 問題標題與說明可能在 Turbopack 版本之間變更。`path` 欄位通常穩定，但不能保證對所有問題類型保持一致。在可能的情況下，請優先使用更精確的 `path` 範式，而非 `title` 或 `description` 匹配。

### `path`

一個 **全域模式 (glob pattern)**（當為字串時）或 **正則表達式 (regex)**，用於與問題產生的文件路徑進行匹配。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    ignoreIssue: [
      // Glob pattern: suppress issues from any file under vendor/
      { path: '**/vendor/**' },
      // RegExp: suppress issues from files matching a pattern
      { path: /node_modules\/legacy-lib/ },
    ],
  },
}
```

### `title`

一個 **精確字串匹配**（當為字串時）或 **正則表達式 (regex)**，用於與問題標題進行匹配。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/src/**',
        title: 'Module not found',
      },
    ],
  },
}
```

### `description`

一個 **精確字串匹配**（當為字串時）或 **正則表達式 (regex)**，用於與問題說明進行匹配。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/src/**',
        description: /Cannot find module 'optional-dep'/,
      },
    ],
  },
}
```

## 範例

### 抑制可选依賴的警告

如果您的代碼在可选的 `require()` 呼叫周圍使用了 `try/catch`，Turbopack 可能會報告「找不到模組」的警告。您可以抑制它：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/lib/optional-feature/**',
        title: 'Module not found',
      },
    ],
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  turbopack: {
    ignoreIssue: [
      {
        path: '**/lib/optional-feature/**',
        title: 'Module not found',
      },
    ],
  },
}

module.exports = nextConfig
```

### 組合多個規則

您可以指定多個規則以抑制不同的問題：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    ignoreIssue: [
      { path: '**/vendor/**' },
      { path: '**/legacy/**', title: 'Module not found' },
      { path: /generated\//, description: /expected identifier/ },
    ],
  },
}
```
