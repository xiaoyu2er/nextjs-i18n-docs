使用 `next.config.js` 中的 `typescript` 選項來配置 TypeScript 行為：

```js filename="next.config.js"
module.exports = {
  typescript: {
    ignoreBuildErrors: false,
    tsconfigPath: 'tsconfig.json',
  },
}
```

## 選項

| 選項              | 類型      | 預設值           | 描述                                                      |
| ------------------- | --------- | ----------------- | ---------------------------------------------------------------- |
| `ignoreBuildErrors` | `boolean` | `false`           | 即使存在 TypeScript 錯誤，也允許生產構建完成。 |
| `tsconfigPath`      | `string`  | `'tsconfig.json'` | 自定義 `tsconfig.json` 文件的路徑。                           |

## `ignoreBuildErrors`

Next.js 在您的專案中存在 TypeScript 錯誤時，會失敗您的**生產建置**（`next build`）。

如果您希望在應用程式有錯誤的情況下仍危險地產出生產代碼，您可以停用內建的類型檢查步驟。

注意，這會完全跳過 TypeScript 類型檢查步驟。它並非執行 TypeScript 並靜默錯誤，而是完全繞過檢查步驟。

若停用，請確保您正在建置或部署流程中執行類型檢查，否則這可能會非常危險。

```js filename="next.config.js"
module.exports = {
  typescript: {
    // !! WARN !!
    // Dangerously allow production builds to successfully complete even if
    // your project has type errors.
    // !! WARN !!
    ignoreBuildErrors: true,
  },
}
```

## `tsconfigPath`

在構建或工具中使用不同的 TypeScript 配置文件：

```js filename="next.config.js"
module.exports = {
  typescript: {
    tsconfigPath: 'tsconfig.build.json',
  },
}
```

請參閱 [TypeScript 配置](/docs/app/api-reference/config/typescript#custom-tsconfig-path) 頁面以獲取更多詳情。
