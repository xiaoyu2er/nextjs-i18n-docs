Next.js 可以透過 `next.config.js` 檔案進行設定，這個檔案位於您的專案目錄根目錄中（例如與 `package.json` 同一層級），並預設匯出。

```js filename="next.config.js"
// @ts-check

/** @type {import('next').NextConfig} */
const nextConfig = {
  /* config options here */
}

module.exports = nextConfig
```

## ECMAScript 模組 (ECMAScript Modules)

`next.config.js` 是一個一般的 Node.js 模組，而不是 JSON 檔案。它由 Next.js 的伺服器與建置階段使用，不會包含在瀏覽器建置中。

如果您需要 [ECMAScript 模組](https://nodejs.org/api/esm.html)，您可以使用 `next.config.mjs`：

```js filename="next.config.mjs"
// @ts-check

/**
 * @type {import('next').NextConfig}
 */
const nextConfig = {
  /* config options here */
}

export default nextConfig
```

> **小提示**：具有 `.cjs` 或 `.cts` 副檔名的 `next.config` 目前**不**支援。

## 作為函式進行設定

您也可以使用函式：

```js filename="next.config.mjs"
// @ts-check

export default (phase, { defaultConfig }) => {
  /**
   * @type {import('next').NextConfig}
   */
  const nextConfig = {
    /* config options here */
  }
  return nextConfig
}
```

### 異步設定

自 Next.js 12.1.0 起，您就能使用異步函式：

```js filename="next.config.js"
// @ts-check

module.exports = async (phase, { defaultConfig }) => {
  /**
   * @type {import('next').NextConfig}
   */
  const nextConfig = {
    /* config options here */
  }
  return nextConfig
}
```

### 階段

`phase` 是設定載入時目前所處的階段環境。您可以查看 [可用階段](https://github.com/vercel/next.js/blob/5e6b008b561caf2710ab7be63320a3d549474a5b/packages/next/shared/lib/constants.ts#L19-L23)。階段可以從 `next/constants` 匯入：

```js filename="next.config.js"
// @ts-check

const { PHASE_DEVELOPMENT_SERVER } = require('next/constants')

module.exports = (phase, { defaultConfig }) => {
  if (phase === PHASE_DEVELOPMENT_SERVER) {
    return {
      /* development only config options here */
    }
  }

  return {
    /* config options for all phases except development here */
  }
}
```

## TypeScript

如果您的專案使用 TypeScript，則可以使用 `next.config.ts` 來在設定中使用 TypeScript：

```ts filename="next.config.ts"
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  /* config options here */
}

export default nextConfig
```

註解的行是您放置 `next.config.js` 允許的設定的地方，這些設定 [在此檔案中定義](https://github.com/vercel/next.js/blob/canary/packages/next/src/server/config-shared.ts)。

然而，這些設定並非都必須，也不需要理解每個設定的作用。相反地，搜尋您需要啟用或修改的功能，並查看此章節，它會告訴您需要執行的操作。

> 避免使用目標 Node.js 版本無法使用的新 JavaScript 功能。`next.config.js` 不會由 Webpack 或 Babel 解析。

## 單元測試 (實驗性功能)

從 Next.js 15.1 開始，`next/experimental/testing/server` 套件包含用於協助單元測試 `next.config.js` 檔案的工具。

`unstable_getResponseFromNextConfig` 函式會以提供的請求資訊運行 `next.config.js` 中的 [`headers`](/docs/app/api-reference/config/next-config-js/headers)、[`redirects`](/docs/app/api-reference/config/next-config-js/redirects) 和 [`rewrites`](/docs/app/api-reference/config/next-config-js/rewrites) 函式，並回傳包含路由結果的 `NextResponse`。

> 回應來自 `unstable_getResponseFromNextConfig` 僅考量 `next.config.js` 的欄位，並不考慮代理 (proxy) 或檔案系統路由 (filesystem routes)，因此生產環境的結果可能與單元測試的結果不同。

```js
import {
  getRedirectUrl,
  unstable_getResponseFromNextConfig,
} from 'next/experimental/testing/server'

const response = await unstable_getResponseFromNextConfig({
  url: 'https://nextjs.org/test',
  nextConfig: {
    async redirects() {
      return [{ source: '/test', destination: '/test2', permanent: false }]
    },
  },
})
expect(response.status).toEqual(307)
expect(getRedirectUrl(response)).toEqual('https://nextjs.org/test2')
```
