您可以為啟用 ISR 的頁面指定自訂的 `stale-while-revalidate` 過期時間，供 CDN 在 `Cache-Control` 標頭中使用。

開啟 `next.config.js` 並新增 `expireTime` 配置：

```js filename="next.config.js"
module.exports = {
  // one hour in seconds
  expireTime: 3600,
}
```

```js filename="next.config.js"
module.exports = {
  // one hour in seconds
  expireTime: 3600,
}
```

現在當傳送 `Cache-Control` 標頭時，過期時間將根據特定的 revalidate 週期進行計算。

例如，如果某個路徑的 revalidate 時間為 15 分鐘且過期時間為一小時，則生成的 `Cache-Control` 標頭將為 `s-maxage=900, stale-while-revalidate=2700`，這樣它就可以保持陳舊狀態，時長比配置的過期時間少 15 分鐘。
