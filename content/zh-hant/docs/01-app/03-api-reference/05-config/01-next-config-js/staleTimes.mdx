description: 學習如何覆蓋客戶端緩存的過期時間。

---

`staleTimes` 是一個實驗性功能，允許在 [客戶端緩存 (Client Cache)](/docs/app/glossary#client-cache) 中緩存頁面區段。

您可以透過設定實驗性 `staleTimes` 旗標來啟用此實驗性功能，並提供自訂的重新驗證時間：

```js filename="next.config.js"
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
}

module.exports = nextConfig
```

`static` 和 `dynamic` 屬性對應於基於不同類型 [連結預取 (link prefetching)](/docs/app/api-reference/components/link#prefetch) 的時間區間（以秒為單位）。

- 若頁面既非靜態生成也未完全預加載 (例如使用 `prefetch={true}`)，則使用 `dynamic` 屬性。
  - 預設值：0 秒 (未緩存)
- `static` 屬性用於靜態生成頁面，或當 `Link` 上的 `prefetch` prop 設為 `true`，或調用 [`router.prefetch`](/docs/app/api-reference/functions/use-router) 時。
  - 預設值：5 分鐘

> **小提示**：
>
> - 在此配置定義的 `static` 期間，[載入邊界 (Loading boundaries)](/docs/app/api-reference/file-conventions/loading) 被視為可重複使用。
> - 這不會影響 [部分渲染 (partial rendering)](/docs/app/getting-started/linking-and-navigating#client-side-transitions)，**意味著共用佈局不會在每次導航時自動重新獲取，僅有變更的頁面區段會重新獲取。**
> - 這不會改變 [後退/前進緩存 (back/forward caching)](/docs/app/glossary#client-cache) 的行為，以防止佈局偏移並防止遺失瀏覽器滾動位置。

| 版本 | 變更 |
| --------- | ---------------------------------------------------------- |
| `v15.0.0` | `dynamic` `staleTimes` 預設值從 30 秒改為 0 秒。|
