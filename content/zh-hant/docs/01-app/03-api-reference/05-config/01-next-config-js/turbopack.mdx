「turbopack」選項可讓您自訂 [Turbopack](/docs/app/api-reference/turbopack)，以轉換不同的檔案並更改模組解析方式。

>> **溫馨提示**：`turbopack` 選項在 Next.js 13.0.0 到 15.2.x 版本中先前命名為 `experimental.turbo`。`experimental.turbo` 選項將在 Next.js 16 中移除。
> 如果您使用的是較舊版本的 Next.js，請執行 `npx @next/codemod@latest next-experimental-turbo-to-turbopack .` 自動遷移您的配置。

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  turbopack: {
    // ...
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  turbopack: {
    // ...
  },
}

module.exports = nextConfig
```

> **要知道**：
>
> - 針對 Next.js 的 Turbopack 不需要裝載器 (loader) 或裝載器配置即可支援內建功能。Turbopack 內建支援 CSS 和編譯現代 JavaScript，因此如果您使用 `@babel/preset-env`，則不需要 `css-loader`、`postcss-loader` 或 `babel-loader`。


## 參考

### 選項

以下選項可供 `turbopack` 配置使用：

| 選項              | 描述                                                                                                                              |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- |
| `root`              | 設定應用程式根目錄。應為絕對路徑。                                                                         |
| `rules`             | 支援的 Webpack 載入器清單，以便在執行 Turbopack 時套用。                                                                  |
| `resolveAlias`      | 將別名匯入映射到要載入的模組。                                                                                   |
| `resolveExtensions` | 匯入檔案時解析的副檔名清單。                                                                                      |
| `debugIds`          | 在 JavaScript 套件和來源地圖中啟用 [偵錯 ID](https://github.com/tc39/ecma426/blob/main/proposals/debug-id.md) 的產生。 |

### 支援的裝載器 (loader)

下列裝載器已測試可與 Turbopack 的 webpack 裝載器 (loader) 實作配合運作，但即使未列於此處，許多其他 webpack 裝載器也應能正常運作：

- [`babel-loader`](https://www.npmjs.com/package/babel-loader) [_(若找到 Babel 配置檔案則自動設定)_](/docs/app/api-reference/turbopack#language-features)
- [`@svgr/webpack`](https://www.npmjs.com/package/@svgr/webpack)
- [`svg-inline-loader`](https://www.npmjs.com/package/svg-inline-loader)
- [`yaml-loader`](https://www.npmjs.com/package/yaml-loader)
- [`string-replace-loader`](https://www.npmjs.com/package/string-replace-loader)
- [`raw-loader`](https://www.npmjs.com/package/raw-loader)
- [`sass-loader`](https://www.npmjs.com/package/sass-loader) [_(自動設定)_](/docs/app/api-reference/turbopack#css-and-styling)
- [`graphql-tag/loader`](https://www.npmjs.com/package/graphql-tag)

#### 缺少 Webpack 載入器功能

Turbopack 使用 [`loader-runner`](https://github.com/webpack/loader-runner) 函式庫來執行 Webpack 載入器，它提供了大多數標準載入器 API。然而，某些功能不支援：

**模組載入**：

- [`importModule`](https://webpack.js.org/api/loaders/#thisimportmodule) - 不支援
- [`loadModule`](https://webpack.js.org/api/loaders/#thisloadmodule) - 不支援

**檔案系統和輸出**：

- [`fs`](https://webpack.js.org/api/loaders/#thisfs) - 部分支援：目前僅實作了 `fs.readFile`。
- [`emitFile`](https://webpack.js.org/api/loaders/#thisemitfile) - 不支援

**上下文屬性**：

- [`version`](https://webpack.js.org/api/loaders/#thisversion) - 不支援
- [`mode`](https://webpack.js.org/api/loaders/#thismode) - 不支援
- [`target`](https://webpack.js.org/api/loaders/#thistarget) - 不支援

**工具函式**：

- [`utils`](https://webpack.js.org/api/loaders/#thisutils) - 不支援
- [`resolve`](https://webpack.js.org/api/loaders/#thisresolve) - 不支援（改用 [`getResolve`](https://webpack.js.org/api/loaders/#thisgetresolve)）

如果您的載入器嚴重依賴這些功能之一，請提交問題報告。

## 範例

### 根目錄

Turbopack 使用根目錄來解析模組。專案根目錄之外的檔案將不會被解析。

檔案不專案根目錄外部解析的原因是為了改善快取驗證、減少檔案系統監控開銷，並減少所需的解析步驟數量。

Next.js 會自動偵測您的專案根目錄。這是透過尋找以下檔案之一來完成的：

- `pnpm-lock.yaml`
- `package-lock.json`
- `yarn.lock`
- `bun.lock`
- `bun.lockb`

如果您有不同的專案結構，例如如果您不使用工作區 (workspaces)，您可以手動設定 `root` 選項：

```js filename="next.config.js"
const path = require('path')
module.exports = {
  turbopack: {
    root: path.join(__dirname, '..'),
  },
}
```

要解析專案根目錄外部連結相依相依套件中的檔案（透過 `npm link`、`yarn link`、`pnpm link` 等），您必須將 `turbopack.root` 設定為專案和連結相依套件共同的父目錄。

雖然這擴大了檔案系統監控的範圍，但通常僅在開發期間且積極處理連結的套件時才有必要。

### 設定 webpack 裝載器 (loader)

如果您需要超過內建能力的裝載器 (loader) 支援，許多 webpack 裝載器已經與 Turbopack 配合運作。目前有一些限制：

- 僅實現了 webpack 裝載器 API 的核心子集。目前，對於一些熱門裝載器已足夠覆蓋範圍，我們將擴展未來的 API 支援。
- 僅支援傳回 JavaScript 程式碼的裝載器。轉換樣式表或圖片等檔案的裝載器目前並不支援。
- 傳遞給 webpack 裝載器的選項必須是純 JavaScript 原始值、物件和陣列。例如，無法將 `require()` 外掛模組作為選項值傳遞。

要配置載入器，請將您已安裝的載入器名稱及任何選項新增至 `next.config.js`，並映射檔案副檔名到載入器列表。規則依順序評估。

下方為使用 [`@svgr/webpack`](https://www.npmjs.com/package/@svgr/webpack) 裝載器的範例，此功能可讓您在匯入 `.svg` 檔案時將其渲染為 React 元件。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    rules: {
      '*.svg': {
        loaders: ['@svgr/webpack'],
        as: '*.js',
      },
    },
  },
}
```

>> **溫馨提示**：`rules` 物件中使用的萬用字串模式（Glob）依據檔案名稱比對，除非萬用字串包含 `/` 字元，這將導致它依據完整專案相對檔案路徑比對。Windows 檔案路徑已標準化使用 Unix 樣式 `/` 路徑分隔符。
>
> Turbopack 使用修改版的 [Rust `globset` 函式庫](https://docs.rs/globset/latest/globset/)。

對於需要配置選項的裝載器，您可以使用物件格式來替代字串：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    rules: {
      '*.svg': {
        loaders: [
          {
            loader: '@svgr/webpack',
            options: {
              icon: true,
            },
          },
        ],
        as: '*.js',
      },
    },
  },
}
```

>> **溫馨提示**：Next.js 版本 13.4.4 之前，`turbopack.rules` 命名為 `turbo.loaders`，且僅接受像 `.mdx` 這樣的檔案副檔名，而非 `*.mdx`。

### 進階 Webpack 載入器條件

您可以使用進階 `condition` 語法進一步限制載入器的執行位置：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    rules: {
      // '*' will match all file paths, but we restrict where our
      // rule runs with a condition.
      '*': {
        condition: {
          all: [
            // 'foreign' is a built-in condition.
            { not: 'foreign' },
            // 'path' can be a RegExp or a glob string. A RegExp matches
            // anywhere in the full project-relative file path.
            { path: /^img\/[0-9]{3}\// },
            {
              any: [
                { path: '*.svg' },
                // 'query' matches anywhere in the full query string,
                // which can be empty, or start with `?`.
                { query: /[?&]svgr(?=&|$)/ },
                // 'content' is always a RegExp, and can match
                // anywhere in the file.
                { content: /\<svg\W/ },
              ],
            },
          ],
        },
        loaders: ['@svgr/webpack'],
        as: '*.js',
      },
    },
  },
}
```

- 支援的布林運算子為 `{all: [...]}`，`{any: [...]}` 和 `{not: ...}`。
- 支援的自訂運算子為 `{path: string | RegExp}`，`{content: RegExp}`，`{query: string | RegExp}`，和 `{contentType: string | RegExp}`。如果在同一物件中指定多個運算子，它會當作隱含的 `and`。
  - `path` 比對專案相對檔案路徑。字串視為萬用字串模式，而 RegExp 可用於部分比對路徑。
  - `content` 比對檔案內容的任一個位置。
  - `query` 比對匯入的查詢字串（例如 `import './file?foo'` 中的 `?foo`）。字串必須完全匹配，而 RegExp 可用於部分比對查詢字串。
  - `contentType` 比對資源的 MIME 內容類型（例如來自 data URLs 如 `data:text/plain,...`）。字串視為萬用字串模式（例如 `text/*`，`image/*`），而 RegExp 可用於部分比對內容類型。

此外，還支援多種內建條件：

- `browser`：比對將在用戶端執行的程式碼。伺服器程式碼可使用 `{not: 'browser'}` 比對。
- `foreign`：比對 `node_modules` 中的程式碼，以及部分 Next.js 內部程式碼。通常您會希望將載入器限制為 `{not: 'foreign'}`。這可以透過減少載入器被呼叫的檔案數量來改善效能。
- `development`：使用 `next dev` 時比對。
- `production`：使用 `next build` 時比對。
- `node`：比對將在預設 Node.js 執行環境執行的程式碼。
- `edge-light`：比對將在 [Edge 執行環境](/docs/app/api-reference/edge) 執行的程式碼。

規則可以是物件或物件陣列。陣列通常用於模擬不相交的條件：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    rules: {
      '*.svg': [
        {
          condition: 'browser',
          loaders: ['@svgr/webpack'],
          as: '*.js',
        },
        {
          condition: { not: 'browser' },
          loaders: [require.resolve('./custom-svg-loader.js')],
          as: '*.js',
        },
      ],
    },
  },
}
```

>> **溫馨提示**：所有符合的規則依順序執行。

### 模組類型

您可以直接設定模組類型而無需使用載入器。這類似於 Webpack 的 [`type`](https://webpack.js.org/configuration/module/#ruletype) 選項。

```js filename="next.config.js"
module.exports = {
  turbopack: {
    rules: {
      '*.svg': {
        type: 'asset',
      },
    },
  },
}
```

當使用 `type: 'asset'` 時，匯入檔案會回傳其 URL：

```tsx filename="app/page.tsx"
import svgUrl from './icon.svg'

export default function Page() {
  return <img src={svgUrl} alt="Icon" />
}
```

`type` 選項可以與 `loaders` 結合使用 - 載入器會先執行，然後結果會根據指定的類型進行處理。

可用模組類型：

| 類型         | 描述                                              |
| ------------ | -------------------------------------------------------- |
| `asset`      | 輸出檔案並回傳 URL（類似 Webpack `asset/resource`） |
| `ecmascript` | 作為 JavaScript 處理                                    |
| `typescript` | 作為 TypeScript 處理                                    |
| `css`        | 作為 CSS 處理                                           |
| `css-module` | 作為 CSS 模組處理                                    |
| `wasm`       | 作為 WebAssembly 處理                                   |
| `raw`        | 以字串回傳原始內容                            |
| `bytes`      | 將內容內嵌為位元組                                 |

### 使用匯入屬性進行載入器內嵌配置

您可以使用 `with` 子句（匯入屬性）將 Turbopack 載入器套用至個別匯入。這是針對匯入指定而非透過 `turbopack.rules` 全域指定。

這在想要將載入器套用至特定匯入而不影響該類型的所有檔案時很有用。

```tsx filename="app/page.tsx"
// Apply a raw loader to import a .txt file as a JavaScript module
import rawText from '../data.txt' with { turbopackLoader: 'raw-loader', turbopackAs: '*.js' }

export default function Page() {
  return <p>{rawText}</p>
}
```

支援以下匯入屬性：

| 屬性                | 描述                                                                                                                   |
| ------------------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| `turbopackLoader`        | 要套用的載入器（例如：`'raw-loader'`）。                                                                                    |
| `turbopackLoaderOptions` | 載入選項的 JSON 字串（例如：`'{"search":"X","replace":"Y"}'`）。                                                        |
| `turbopackAs`            | 輸出的重新命名模式（與 `turbopack.rules[].as` 相同）。例如，`'*.js'` 將載入器輸出視為 JavaScript。 |
| `turbopackModuleType`    | 設定輸出的模組類型（與 `turbopack.rules[].type` 相同）。                                                        |

帶有選項的載入器會透過 `turbopackLoaderOptions` 傳遞 JSON 編碼的字串：

```tsx filename="app/page.tsx"
import value from '../data.js' with { turbopackLoader: 'string-replace-loader', turbopackLoaderOptions: '{"search":"PLACEHOLDER","replace":"replaced value"}' }
```

>> **溫馨提示**：帶有 `turbopackLoader` 的匯入屬性為 Turbopack 專用，Webpack 不支援。此功能需要匯入語句中使用 `with` 關鍵字（而非 `assert`）。

### 解析別名

Turbopack 可透過別名設定來修改模組解析，類似於 webpack 的 [`resolve.alias`](https://webpack.js.org/configuration/resolve/#resolvealias) 設定。

要配置解析別名，請在 `next.config.js` 中將匯入模式對應至其新目的地：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    resolveAlias: {
      underscore: 'lodash',
      mocha: { browser: 'mocha/browser-entry.js' },
    },
  },
}
```

這將 `underscore` 程式集的匯入別名設定為 `lodash` 程式集。換句話說，`import underscore from 'underscore'` 將載入 `lodash` 模組而非 `underscore`。

Turbopack 也透過此欄位支援條件式別名設定，類似於 Node.js 的 [條件式匯出](https://nodejs.org/docs/latest-v18.x/api/packages.html#conditional-exports)。目前僅支援 `browser` 條件。在上述情況下，當 Turbopack 目標為瀏覽器環境時，`mocha` 模組的匯入將被別名設定為 `mocha/browser-entry.js`。

### 解析自定義副檔名

Turbopack 可配置解析具有自定義副檔名的模組，類似於 webpack 的 [`resolve.extensions`](https://webpack.js.org/configuration/resolve/#resolveextensions) 設定。

要配置解析副檔名，請在 `next.config.js` 中使用 `resolveExtensions` 欄位：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    resolveExtensions: ['.mdx', '.tsx', '.ts', '.jsx', '.js', '.mjs', '.json'],
  },
}
```

這會以提供的清單覆蓋原始解析副檔名。請務必包含預設副檔名。

關於如何將應用程式從 webpack 迁移至 Turbopack 的更多資訊和指導，請參閱 [Turbopack 關於 webpack 相容性的文件](https://turbo.build/pack/docs/migrating-from-webpack)。

### 偵錯 ID

Turbopack 可配置為在 JavaScript 套件和來源地圖中產生 [偵錯 ID](https://github.com/tc39/ecma426/blob/main/proposals/debug-id.md)。

要配置偵錯 ID，請在 `next.config.js` 中使用 `debugIds` 欄位：

```js filename="next.config.js"
module.exports = {
  turbopack: {
    debugIds: true,
  },
}
```

選項會自動將偵錯 ID 的 polyfill 新增至 JavaScript 套件以確保相容性。偵錯 ID 可以在 `globalThis._debugIds` 全域變數中取得。
