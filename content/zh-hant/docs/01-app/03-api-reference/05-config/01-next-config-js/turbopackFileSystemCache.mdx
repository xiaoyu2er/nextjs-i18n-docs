## 使用方法

Turbopack 檔案系統緩存 能讓 Turbopack 在執行 `next dev` 或 `next build` 命令時減少工作量。啟用後，Turbopack 會在構建之間將資料儲存至並從 `.next` 資料夾還原，這能大幅加速後續構建和開發運作。

> **了解要點：** 檔案系統緩存功能 對開發環境穩定，但對於生產構建仍屬實驗性質

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  experimental: {
    // Enable filesystem caching for `next dev`
    turbopackFileSystemCacheForDev: true,
    // Enable filesystem caching for `next build`
    turbopackFileSystemCacheForBuild: true,
  },
}

export default nextConfig
```

```js filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    // Enable filesystem caching for `next dev`
    turbopackFileSystemCacheForDev: true,
    // Enable filesystem caching for `next build`
    turbopackFileSystemCacheForBuild: true,
  },
}

module.exports = nextConfig
```

## 版本變更

| 版本   | 變更                                                        |
| --------- | -------------------------------------------------------------- |
| `v16.1.0` | 檔案系統緩存 預設啟用於開發環境                       |
| `v16.0.0` | Beta 版本，為構建和 開發 設有獨立 標誌                       |
| `v15.5.0` | 持久緩存 在 canary 版本 上發布為實驗性功能                    |
