Next.js 提供 ESLint 設定套件 ``eslint-config-next``(https://www.npmjs.com/package/eslint-config-next)，它能輕鬆捕捉應用程式中的常見問題。它包含了 ``@next/eslint-plugin-next``(https://www.npmjs.com/package/@next/eslint-plugin-next) 外掛，以及 ``eslint-plugin-react``(https://www.npmjs.com/package/eslint-plugin-react) 和 ``eslint-plugin-react-hooks``(https://www.npmjs.com/package/eslint-plugin-react-hooks) 的推薦規則集。

此套件提供兩個主要設定：

- **``eslint-config-next``**: 包含 Next.js、React 和 React Hooks 規則的基礎配置。支援 JavaScript 和 TypeScript 檔案。
- **``eslint-config-next/core-web-vitals``**: 包含基礎配置中的所有內容，並加強會影響 [核心網頁指標 (Core Web Vitals)](https://web.dev/vitals/) 的規則，將警告升級為錯誤。建議大多數專案使用。

此外，針對 TypeScript 專案：

- **``eslint-config-next/typescript``**: 新增來自 ``typescript-eslint``(https://typescript-eslint.io/) 的 TypeScript 專屬檢查規則。應與基礎配置或 core-web-vitals 配置搭配使用。

## 設定 ESLint

使用 ESLint CLI（扁平配置）快速啟用檢查功能：

1. 安裝 ESLint 和 Next.js 配置：

   ```bash package="pnpm"
   pnpm add -D eslint eslint-config-next
   ```

   ```bash package="npm"
   npm i -D eslint eslint-config-next
   ```

   ```bash package="yarn"
   yarn add --dev eslint eslint-config-next
   ```

   ```bash package="bun"
   bun add -d eslint eslint-config-next
   ```

2. 建立 ``eslint.config.mjs`` 使用 Next.js 配置：

   ```js filename="eslint.config.mjs"
   import { defineConfig, globalIgnores } from 'eslint/config'
   import nextVitals from 'eslint-config-next/core-web-vitals'

   const eslintConfig = defineConfig([
     ...nextVitals,
     // 覆蓋 eslint-config-next 的預設忽略。
     globalIgnores([
       // eslint-config-next 預設忽略：
       '.next/**',
       'out/**',
       'build/**',
       'next-env.d.ts',
     ]),
   ])

   export default eslintConfig
   ```

3. 執行 ESLint：

   ```bash package="pnpm"
   pnpm exec eslint .
   ```

   ```bash package="npm"
   npx eslint .
   ```

   ```bash package="yarn"
   yarn eslint .
   ```

   ```bash package="bun"
   bunx eslint .
   ```

## 參考

- ``eslint-plugin-react``(https://www.npmjs.com/package/eslint-plugin-react)
- ``eslint-plugin-react-hooks``(https://www.npmjs.com/package/eslint-plugin-react-hooks)
- ``@next/eslint-plugin-next``(https://www.npmjs.com/package/@next/eslint-plugin-next)

## 範例

### 指定 monorepo 內的根目錄

如果您在未在根目錄安裝 Next.js 的專案中使用 ``@next/eslint-plugin-next``（例如單倉庫 (Monorepo)），您可以在 ``eslint.config.mjs`` 的 ``settings`` 屬性中告訴 ``@next/eslint-plugin-next`` 尋找 Next.js 應用程式的位置：

```js filename="eslint.config.mjs"
import { defineConfig } from 'eslint/config'
import eslintNextPlugin from '@next/eslint-plugin-next'

const eslintConfig = defineConfig([
  {
    files: ['**/*.{js,jsx,ts,tsx}'],
    plugins: {
      next: eslintNextPlugin,
    },
    settings: {
      next: {
        rootDir: 'packages/my-app/',
      },
    },
  },
])

export default eslintConfig
```

`rootDir` 可以是路徑（相對或絕對）、萬用字元（Glob）（即 `"packages/*/"`），或是路徑和/或萬用字元的陣列。

### 停用規則

如果您想修改或停用支援外掛（``react``, ``react-hooks``, ``next``）提供的任何規則，您可以在 ``eslint.config.mjs`` 中使用 ``rules`` 屬性直接修改它們：

```js filename="eslint.config.mjs"
import { defineConfig, globalIgnores } from 'eslint/config'
import nextVitals from 'eslint-config-next/core-web-vitals'

const eslintConfig = defineConfig([
  ...nextVitals,
  {
    rules: {
      'react/no-unescaped-entities': 'off',
      '@next/next/no-page-custom-font': 'off',
    },
  },
  // Override default ignores of eslint-config-next.
  globalIgnores([
    // Default ignores of eslint-config-next:
    '.next/**',
    'out/**',
    'build/**',
    'next-env.d.ts',
  ]),
])

export default eslintConfig
```

### 搭配 Core Web Vitals

在您的 ESLint 設定中啟用 ``eslint-config-next/core-web-vitals`` 配置。

```js filename="eslint.config.mjs"
import { defineConfig, globalIgnores } from 'eslint/config'
import nextVitals from 'eslint-config-next/core-web-vitals'

const eslintConfig = defineConfig([
  ...nextVitals,
  // Override default ignores of eslint-config-next.
  globalIgnores([
    // Default ignores of eslint-config-next:
    '.next/**',
    'out/**',
    'build/**',
    'next-env.d.ts',
  ]),
])

export default eslintConfig
```

``eslint-config-next/core-web-vitals`` 將 ``@next/eslint-plugin-next`` 中的某些檢查規則從警告升級為錯誤，以幫助改進您的 [核心網頁指標 (Core Web Vitals)](https://web.dev/vitals/) 指標。

> ``eslint-config-next/core-web-vitals`` 配置會自動包含在透過 [建立 Next App](/docs/app/api-reference/cli/create-next-app) 構建的新應用程式中。

### 使用 TypeScript

除了 Next.js ESLint 規則外，``create-next-app --typescript`` 也會將 ``eslint-config-next/typescript`` 的 TypeScript 專屬檢查規則新增到您的配置中：

```js filename="eslint.config.mjs"
import { defineConfig, globalIgnores } from 'eslint/config'
import nextVitals from 'eslint-config-next/core-web-vitals'
import nextTs from 'eslint-config-next/typescript'

const eslintConfig = defineConfig([
  ...nextVitals,
  ...nextTs,
  // Override default ignores of eslint-config-next.
  globalIgnores([
    // Default ignores of eslint-config-next:
    '.next/**',
    'out/**',
    'build/**',
    'next-env.d.ts',
  ]),
])

export default eslintConfig
```

這些規則基於 [`plugin:@typescript-eslint/recommended`](https://typescript-eslint.io/linting/configs#recommended)。更多詳情請參閱 [typescript-eslint > Configs](https://typescript-eslint.io/linting/configs)。

### 搭配 Prettier

ESLint 也包含程式碼格式化規則，這可能與您現有的 [Prettier](https://prettier.io/) 設定衝突。我們建議將 [eslint-config-prettier](https://github.com/prettier/eslint-config-prettier) 包含在您的 ESLint 設定中，讓 ESLint 和 Prettier 能協同工作。

首先，安裝相依套件：

```bash package="pnpm"
pnpm add -D eslint-config-prettier
```

```bash package="npm"
npm i -D eslint-config-prettier
```

```bash package="yarn"
yarn add --dev eslint-config-prettier
```

```bash package="bun"
bun add -d eslint-config-prettier
```

接著，將 `prettier` 添加到您現有的 ESLint 設定：

```js filename="eslint.config.mjs"
import { defineConfig, globalIgnores } from 'eslint/config'
import nextVitals from 'eslint-config-next/core-web-vitals'
import prettier from 'eslint-config-prettier/flat'

const eslintConfig = defineConfig([
  ...nextVitals,
  prettier,
  // Override default ignores of eslint-config-next.
  globalIgnores([
    // Default ignores of eslint-config-next:
    '.next/**',
    'out/**',
    'build/**',
    'next-env.d.ts',
  ]),
])

export default eslintConfig
```

### 在暫存檔案上執行檢查

如果您想使用 ESLint 搭配 [lint-staged](https://github.com/okonet/lint-staged) 對暫存的 Git 檔案執行檢查器，請在專案根目錄的 ``.lintstagedrc.js`` 檔案中新增以下內容：

```js filename=".lintstagedrc.js"
const path = require('path')

const buildEslintCommand = (filenames) =>
  `eslint --fix ${filenames
    .map((f) => `"${path.relative(process.cwd(), f)}"`)
    .join(' ')}`

module.exports = {
  '*.{js,jsx,ts,tsx}': [buildEslintCommand],
}
```

## 遷移現有設定

如果您的應用程式已設定好了 ESLint，有兩種方法可以整合 Next.js 檢查規則，取決於您的設定。

#### 直接使用外掛

如果您已配置了以下任何一項，請直接使用 ``@next/eslint-plugin-next``：

- 分別安裝或透過其他配置（如 ``airbnb`` 或 ``react-app``）安裝的衝突插件：
  - ``react``
  - ``react-hooks``
  - ``jsx-a11y``
  - ``import``
- 與 Next.js 預設不同的自訂 ``parserOptions``（僅當您 [自訂了 Babel 配置](/docs/pages/guides/babel) 時）
- 使用自訂 Node.js 和/或 TypeScript [解析器](https://github.com/benmosher/eslint-plugin-import#resolvers) 的 ``eslint-plugin-import``

在這些情況下，直接使用 ``@next/eslint-plugin-next`` 以避免衝突：

首先，安裝外掛：

```bash package="pnpm"
pnpm add -D @next/eslint-plugin-next
```

```bash package="npm"
npm i -D @next/eslint-plugin-next
```

```bash package="yarn"
yarn add --dev @next/eslint-plugin-next
```

```bash package="bun"
bun add -d @next/eslint-plugin-next
```

然後將其新增到您的 ESLint 設定：

```js filename="eslint.config.mjs"
import { defineConfig } from 'eslint/config'
import nextPlugin from '@next/eslint-plugin-next'

const eslintConfig = defineConfig([
  // Your other configurations...
  {
    files: ['**/*.{js,jsx,ts,tsx}'],
    plugins: {
      '@next/next': nextPlugin,
    },
    rules: {
      ...nextPlugin.configs.recommended.rules,
    },
  },
])

export default eslintConfig
```

此方法可消除當多個配置載入相同外掛或解析器時可能發生的碰撞或錯誤風險。

#### 新增至現有設定

如果您要將 Next.js 新增至現有的 ESLint 設定，請將 Next.js 配置散布到您的陣列中：

```js filename="eslint.config.mjs"
import nextConfig from 'eslint-config-next/core-web-vitals'
// Your other config imports...

const eslintConfig = [
  // Your other configurations...
  ...nextConfig,
]

export default eslintConfig
```

當您散布 ``...nextConfig`` 時，您會新增多個包含檔案模式、外掛、規則、忽略項和解析器設定之配置物件。ESLint 依序套用配置，因此較後的規則對於匹配的檔案可以覆蓋較早的規則。

> **值得注意：** 此方法對於簡單的設定運作良好。如果您有衝突的特定檔案模式或外掛設定的複雜現有配置，考慮直接使用外掛（如上所示）以獲得更細微的控制。

| 版本   | 變更                                                                                                                                                                                                             |
| --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- |
| ``v16.0.0`` | ``next lint`` 和 ``next.config.js`` 選項 ``eslint`` 選項已被移除，改用 ESLint CLI。[代碼轉換程式 (Codemod)](/docs/app/guides/upgrading/codemods#migrate-from-next-lint-to-eslint-cli) 可用以協助您遷移。 |
