---
title: 使用客戶端 (use client)
description: 學習如何使用 'use client' 指令在客戶端渲染組件。
---

`'use client'` 指令宣告了要在 **客戶端** 渲染的組件入口點，當建立需要客戶端 JavaScript 能力（例如狀態管理、事件處理以及瀏覽器 API 存取）的互動式使用者介面 (UI) 時應使用此指令。這是 React 的功能。

> **小知識：**
>
> 您不需要將 `'use client'` 指令加入每一包含客戶端組件 (Client Components) 的檔案。您只需要將其加入您要直接在伺服器組件 (Server Components) 內部渲染的組件對應檔案中。`'use client'` 指令定義了客戶端 - 伺服器 [邊界](https://nextjs.org/docs/app/building-your-application/rendering#network-boundary)，且從該檔案導出的組件作為客戶端的入口點。

## 使用方法

為了宣告 Client Components 的入口點，請將 `'use client'` 指令 **置於檔案開頭**，在所有匯入之前：

```tsx filename="app/components/counter.tsx" highlight={1} switcher
'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  )
}
```

```jsx filename="app/components/counter.js" highlight={1} switcher
'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  )
}
```

當使用 `'use client'` 指令時，Client Components 的 props 必須是 [可序列化 (serializable)](https://react.dev/reference/rsc/use-client#serializable-types) 的。這表示 props 需要採用 React 可以在從伺服器發送資料給客戶端時進行序列化的格式。

```tsx filename="app/components/counter.tsx" highlight={4} switcher
'use client'

export default function Counter({
  onClick /* ❌ Function is not serializable */,
}) {
  return (
    <div>
      <button onClick={onClick}>Increment</button>
    </div>
  )
}
```

```jsx filename="app/components/counter.js" highlight={4} switcher
'use client'

export default function Counter({
  onClick /* ❌ Function is not serializable */,
}) {
  return (
    <div>
      <button onClick={onClick}>Increment</button>
    </div>
  )
}
```

## 在 Server Components 中嵌套 Client Components

結合 Server Components 和 Client Components 可讓您建構兼具效能與互動性的應用程式：

1. **伺服器組件 (Server Components)**：用於靜態內容、資料取得 (data fetching) 以及對 SEO 友善的元素。
2. **客戶端組件 (Client Components)**：用於需要狀態 (state)、副作用 (effects) 或瀏覽器 APIs 的互動元素。
3. **組件組合 (Component composition)**：視需要將 Client Components 嵌套在 Server Components 內，以清楚區分伺服器與客戶端的邏輯。

在下列範例中：

- `Header` 是一個處理靜態內容的伺服器組件 (Server Component)。
- `Counter` 是一個啟用頁面內互動性的客戶端組件 (Client Component)。

```tsx filename="app/page.tsx" highlight={2,8} switcher
import Header from './header'
import Counter from './counter' // This is a Client Component

export default function Page() {
  return (
    <div>
      <Header />
      <Counter />
    </div>
  )
}
```

```jsx filename="app/page.js" highlight={2,8} switcher
import Header from './header'
import Counter from './counter' // This is a Client Component

export default function Page() {
  return (
    <div>
      <Header />
      <Counter />
    </div>
  )
}
```
