『use cache: private』指令允許函式在快取範圍內存取執行階段請求 API，例如 `cookies()`、`headers()` 和 `searchParams`。然而，結果**永遠不會儲存在伺服器上**，它們僅快取在瀏覽器的記憶體中，並且不會在頁面重新載入後保留。

當遇到下列情況時，請使用 `'use cache: private'`

- 您希望快取一個已存取執行階段數據的函式，但將 [將執行階段存取移出並作為引數傳遞值](/docs/app/getting-started/caching#working-with-runtime-apis) 的重構並不可行。
- 合規要求阻止在伺服器上儲存某些數據，即使是暫時性的。

由於該指令存取執行階段數據，此函式會在每次伺服器重新繪製時執行，且在 [靜態外殼](/docs/app/getting-started/caching#how-rendering-works) 生成期間不會執行。

為 `'use cache: private'` 配置自訂快取處理器是**不可能**的。

若要比較不同的快取指令，請參閱 [「use cache: remote」與「use cache」及「use cache: private」有何不同](/docs/app/api-reference/directives/use-cache-remote#how-use-cache-remote-differs-from-use-cache-and-use-cache-private)。

> **提示**：此指令標記為 `experimental`，因為它依賴執行階段預取，該功能尚未穩定。執行階段預取是一項即將推出的功能，允許路由器將 [靜態外殼](/docs/app/getting-started/caching#how-rendering-works) 之外的範圍預取到 **任何** 快取範圍中，而不僅限於私有快取。

## 使用方法

要使用 `'use cache: private'`，請在您的 `next.config.ts` 檔案中啟用 [`cacheComponents`](/docs/app/api-reference/config/next-config-js/cacheComponents) 標記：

```tsx filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const nextConfig: NextConfig = {
  cacheComponents: true,
}

export default nextConfig
```

```jsx filename="next.config.js" switcher
/** @type {import('next').NextConfig} */
const nextConfig = {
  cacheComponents: true,
}

export default nextConfig
```

接著在您的函式中加上 `'use cache: private'`，並搭配 `cacheLife` 配置。

> **提示**：此指令無法在 路由處理器 (Route Handlers) 中使用。

### 基本範例

在這個範例中，我們展示您可以在 `'use cache: private'` 範圍內存取 cookies:

```tsx filename="app/product/[id]/page.tsx" switcher
import { Suspense } from 'react'
import { cookies } from 'next/headers'
import { cacheLife, cacheTag } from 'next/cache'

export async function generateStaticParams() {
  return [{ id: '1' }]
}

export default async function ProductPage({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params

  return (
    <div>
      <ProductDetails id={id} />
      <Suspense fallback={<div>Loading recommendations...</div>}>
        <Recommendations productId={id} />
      </Suspense>
    </div>
  )
}

async function Recommendations({ productId }: { productId: string }) {
  const recommendations = await getRecommendations(productId)

  return (
    <div>
      {recommendations.map((rec) => (
        <ProductCard key={rec.id} product={rec} />
      ))}
    </div>
  )
}

async function getRecommendations(productId: string) {
  'use cache: private'
  cacheTag(`recommendations-${productId}`)
  cacheLife({ stale: 60 })

  // Access cookies within private cache functions
  const sessionId = (await cookies()).get('session-id')?.value || 'guest'

  return getPersonalizedRecommendations(productId, sessionId)
}
```

## 私有快取中允許的請求 API

以下請求特定的 API 可在 `'use cache: private'` 函式內部使用：

| API            | 在 `use cache` 中允許 | 在 `'use cache: private'` 中允許 |
| -------------- | ---------------------- | --------------------------------- |
| `cookies()`    | 否                     | 是                                |
| `headers()`    | 否                     | 是                                |
| `searchParams` | 否                     | 是                                |
| `connection()` | 否                     | 否                                |

> **注意**：[`connection()`](https://nextjs.org/docs/app/api-reference/functions/connection) API 在 `use cache` 和 `'use cache: private'` 中均被禁止，因為它提供的資訊與連線特定相關，無法安全快取。
