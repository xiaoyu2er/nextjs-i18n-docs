`useSearchParams` 是一個 **Client Component（客戶端組件）** 鉤子，讓您能夠讀取當前 URL 的 **query string（查詢字串）**。

`useSearchParams` 傳回 [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams) 介面的 **read-only（唯讀）** 版本。

```tsx filename="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // URL -> `/dashboard?search=my-project`
  // `search` -> 'my-project'
  return <>Search: {search}</>
}
```

```jsx filename="app/dashboard/search-bar.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // URL -> `/dashboard?search=my-project`
  // `search` -> 'my-project'
  return <>Search: {search}</>
}
```

## 參數

```tsx
const searchParams = useSearchParams()
```

`useSearchParams` 不需要任何參數。

## 返回值

`useSearchParams` 傳回 [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams) 介面的 **read-only（唯讀）** 版本，其中包括用於讀取 URL 的 **query string（查詢字串）** 的公用方法：

- [`URLSearchParams.get()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/get): 傳回與搜尋參數關聯的第一個值。例如：

  | URL                  | `searchParams.get("a")`                                                                                         |
  | -------------------- | --------------------------------------------------------------------------------------------------------------- |
  | `/dashboard?a=1`     | `'1'`                                                                                                           |
  | `/dashboard?a=`      | `''`                                                                                                            |
  | `/dashboard?b=3`     | `null`                                                                                                          |
  | `/dashboard?a=1&a=2` | `'1'` _- 使用 [`getAll()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/getAll) 來取得所有值_ |

- [`URLSearchParams.has()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/has): 傳回布林滙值，表示給定的參數是否存在。例如：

  | URL              | `searchParams.has("a")` |
  | ---------------- | ----------------------- |
  | `/dashboard?a=1` | `true`                  |
  | `/dashboard?b=3` | `false`                 |

- 了解更多有關其他 **read-only（唯讀）** 方法的 [`URLSearchParams`](https://developer.mozilla.org/docs/Web/API/URLSearchParams) 方法，包括 [`getAll()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/getAll), [`keys()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/keys), [`values()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/values), [`entries()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/entries), [`forEach()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/forEach), 和 [`toString()`](https://developer.mozilla.org/docs/Web/API/URLSearchParams/toString)。

> **提示**：
>
 - `useSearchParams` 是一個 [客戶端組件 (Client Component)](/docs/app/getting-started/server-and-client-components) 鉤子，並且**不支持**用於 [服務器組件 (Server Components)](/docs/app/getting-started/server-and-client-components)，以防止在 [部分渲染 (partial rendering)](/docs/app/getting-started/linking-and-navigating#client-side-transitions) 期間出現過時值。
 - 如果您希望基於搜索參數 (search params) 在服務器組件 (Server Component) 中獲取數據，通常更好的選項是讀取相應頁面的 [`searchParams` 屬性 (prop)](/docs/app/api-reference/file-conventions/page#searchparams-optional)。然後您可以通過屬性將其傳遞給該頁面內的任何組件（服務器或客戶端）。
 - 如果應用包含 `/pages` 目錄，`useSearchParams` 將返回 `ReadonlyURLSearchParams | null`。`null` 值是為了遷移兼容性，因為在預渲染 (prerendering) 頁面的搜索參數在頁面不使用 `getServerSideProps` 時無法得知。

## 行為

### 預渲染

如果路由是 [預渲染 (prerendering)](/docs/app/glossary#prerendering) 的，調用 `useSearchParams` 將導致最接近的 [`Suspense` 邊界 (boundary)](/docs/app/api-reference/file-conventions/loading#examples) 之上的客戶端組件樹進行客戶端渲染。

這允許路由的一部分被預渲染，而使用 `useSearchParams` 的動態部分則在客戶端渲染。

我們建議將使用 `useSearchParams` 的客戶端組件 (Client Component) 包裹在 `<Suspense/>` 邊界中。這允许其上方的任何客戶端組件被預渲染並作為初始 HTML 的一部分發送。[示例](/docs/app/api-reference/functions/use-search-params#prerendering)。

例如：

```tsx filename="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // This will not be logged on the server during prerendering
  console.log(search)

  return <>Search: {search}</>
}
```

```jsx filename="app/dashboard/search-bar.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // This will not be logged on the server during prerendering
  console.log(search)

  return <>Search: {search}</>
}
```

```tsx filename="app/dashboard/page.tsx" switcher
import { Suspense } from 'react'
import SearchBar from './search-bar'

// This component passed as a fallback to the Suspense boundary
// will be rendered in place of the search bar in the initial HTML.
// When the value is available during React hydration the fallback
// will be replaced with the `<SearchBar>` component.
function SearchBarFallback() {
  return <>placeholder</>
}

export default function Page() {
  return (
    <>
      <nav>
        <Suspense fallback={<SearchBarFallback />}>
          <SearchBar />
        </Suspense>
      </nav>
      <h1>Dashboard</h1>
    </>
  )
}
```

> **提示**：
>
 - 在開發環境中，路由是按需渲染的，所以 `useSearchParams` 不會懸掛，而且情況也可能看似運作正常，即使沒有 `Suspense`。
 - 在生產環境構建期間，從客戶端組件 (Client Component) 調用 `useSearchParams` 的靜態頁面必須包裹在 `Suspense` 邊界中，否則構建將因 [缺失 useSearchParams 的 Suspense 邊界 (Missing Suspense boundary with useSearchParams)](/docs/messages/missing-suspense-with-csr-bailout) 錯誤而失敗。
 - 如果您打算使路由動態渲染，優先在服務器組件 (Server Component) 中首先使用 [`connection`](/docs/app/api-reference/functions/connection) 函式等待傳入請求，此舉會將下方所有內容排除於預渲染之外。請參閱 [動態渲染指南 (Dynamic Rendering guide)](/docs/app/glossary#dynamic-rendering) 了解是什麼使路由變得動態。
 - 如果您已在服務器組件頁面 (Server Component Page) 中，請考慮使用 [`searchParams` 屬性 (prop)](/docs/app/api-reference/file-conventions/page#searchparams-optional) 並將值傳遞給客戶端組件。
 - 您也可以將頁面 [`searchParams` 屬性 (prop)](/docs/app/api-reference/file-conventions/page#searchparams-optional) 直接傳遞給客戶端組件，並使用 React 的 `use()` 解包。雖然這會懸掛，但客戶端組件應包裹在 `Suspense` 邊界中。

### 動態渲染 (Dynamic Rendering)

如果路由是動態渲染的，`useSearchParams` 將在客戶端組件 (Client Component) 的初始服務器渲染期間可在服務器上獲取。

例如：

```tsx filename="app/dashboard/search-bar.tsx" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // This will be logged on the server during the initial render
  // and on the client on subsequent navigations.
  console.log(search)

  return <>Search: {search}</>
}
```

```jsx filename="app/dashboard/search-bar.js" switcher
'use client'

import { useSearchParams } from 'next/navigation'

export default function SearchBar() {
  const searchParams = useSearchParams()

  const search = searchParams.get('search')

  // This will be logged on the server during the initial render
  // and on the client on subsequent navigations.
  console.log(search)

  return <>Search: {search}</>
}
```

```tsx filename="app/dashboard/page.tsx" switcher
import { connection } from 'next/server'
import SearchBar from './search-bar'

export default async function Page() {
  await connection()
  return (
    <>
      <nav>
        <SearchBar />
      </nav>
      <h1>Dashboard</h1>
    </>
  )
}
```

```jsx filename="app/dashboard/page.js" switcher

import SearchBar from './search-bar'

  await connection()
  return (
    <>
      <nav>
        <SearchBar />
      </nav>
      <h1>Dashboard</h1>
    </>
  )
}
```

> **提示**：
>
 - 先前，在頁面上設定 `export const dynamic = 'force-dynamic'` 是用來強制進行動態渲染。建議改用 [`connection()`](/docs/app/api-reference/functions/connection)，因為它在語義上將動態渲染與傳入請求聯繫起來。

### 伺服器元件

#### Pages（頁面）

要存取 [Pages（頁面）](/docs/app/api-reference/file-conventions/page)（Server Components）中的搜尋參數，請使用 [`searchParams`](/docs/app/api-reference/file-conventions/page#searchparams-optional) 屬性。

#### Layouts（佈局）

與 Pages 不同，[Layouts（佈局）](/docs/app/api-reference/file-conventions/layout)（Server Components）**不**接收 `searchParams` 屬性。這是因為共用佈局在 [導航期間不會重新渲染](/docs/app/getting-started/linking-and-navigating#client-side-transitions)，這可能導致導航之間的 `searchParams` 過時。請查看 [詳細說明](/docs/app/api-reference/file-conventions/layout#query-params)。

相反，請使用 Page [`searchParams`](/docs/app/api-reference/file-conventions/page) 屬性或 Client Component 中的 [`useSearchParams`](/docs/app/api-reference/functions/use-search-params) 鉤子，它會在客戶端使用最新的 `searchParams` 重新渲染。

## 範例

### 更新 `searchParams`

您可以使用 [`useRouter`](/docs/app/api-reference/functions/use-router) 或 [`Link`](/docs/app/api-reference/components/link) 來設定新的 `searchParams`。執行導航後，目前的 [`page.js`](/docs/app/api-reference/file-conventions/page) 將收到更新的 [`searchParams` prop](/docs/app/api-reference/file-conventions/page#searchparams-optional)。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

export default function ExampleClientComponent() {
  const router = useRouter()
  const pathname = usePathname()
  const searchParams = useSearchParams()

  // Get a new searchParams string by merging the current
  // searchParams with a provided key/value pair
  const createQueryString = useCallback(
    (name: string, value: string) => {
      const params = new URLSearchParams(searchParams.toString())
      params.set(name, value)

      return params.toString()
    },
    [searchParams]
  )

  return (
    <>
      <p>Sort By</p>

      {/* using useRouter */}
      <button
        onClick={() => {
          // <pathname>?sort=asc
          router.push(pathname + '?' + createQueryString('sort', 'asc'))
        }}
      >
        ASC
      </button>

      {/* using <Link> */}
      <Link
        href={
          // <pathname>?sort=desc
          pathname + '?' + createQueryString('sort', 'desc')
        }
      >
        DESC
      </Link>
    </>
  )
}
```
