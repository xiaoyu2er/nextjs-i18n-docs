**在 15 版本中，我們建議使用 [`connection`](/docs/app/api-reference/functions/connection) 而不是 `unstable_noStore`。**

`unstable_noStore` 可用於以聲明式方式停用預渲染，並指示特定組件不應被緩存。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```

> **小提示**：
>
> - `unstable_noStore` 等同於 `fetch` 上的 `cache: 'no-store'`
> - 較推薦使用 `unstable_noStore`，而非 `export const dynamic = 'force-dynamic'`，因為它更細粒度，且可針對每個單獨組件使用

- 在 [`unstable_cache`](/docs/app/api-reference/functions/unstable_cache) 內部使用 `unstable_noStore` 將不會停用靜態生成。取而代之的是，它會依賴快取配置來決定是否快取結果。

## 使用方法

如果您不希望向 `fetch` 傳遞其他選項，例如 `cache: 'no-store'`、`next: { revalidate: 0 }`，或在 `fetch` 不可用的情況下，您可以將 `noStore()` 用作這些情境的替代方案。

```jsx
import { unstable_noStore as noStore } from 'next/cache';

export default async function ServerComponent() {
  noStore();
  const result = await db.query(...);
  ...
}
```
