`updateTag` 讓您可以在 [Server Actions](/docs/app/getting-started/mutating-data) 內，按需更新特定緩存標籤的緩存數據。

此函式專為 **讀取自己寫入 (read-your-own-writes)** 情境設計，在這種情境中，用戶進行更改（例如建立文章），UI 會立即顯示更改，而非展示過期的數據。

## 使用方法

`updateTag` **僅限** 從 [Server Actions](/docs/app/getting-started/mutating-data) 內呼叫。它不能在 Route Handlers、Client Components 或任何其他情境中使用。

如果您需要在 Route Handlers 或其他情境中使緩存標籤無效，請改用 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag) 函式。

> **小提示**：`updateTag` 會立即使指定標籤的緩存數據失效。下一個請求將等待獲取新數據，而不是從緩存中提供過期的內容，確保用戶立即看到他們的更改。

## 參數

```tsx
updateTag(tag: string): void;
```

- `tag`: 代表您想要更新之數據的緩存標籤的字串。不得超過 256 個字元。此值區分大小寫。

標籤必須先分配給緩存數據。您可以透過以下方式完成：

- 使用 [`next.tags`](/docs/app/api-reference/functions/fetch) 選項搭配 `fetch` 來快取外部 API 請求：

```tsx
fetch(url, { next: { tags: ['posts'] } })
```

- 在帶有 `'use cache'` 直達式的緩存函式或元件中使用 [`cacheTag`](/docs/app/api-reference/functions/cacheTag)：

```tsx
import { cacheTag } from 'next/cache'

async function getData() {
  'use cache'
  cacheTag('posts')
  // ...
}
```

## 返回值

`updateTag` 不傳回任何值。

## 與 revalidateTag 的差異

雖然 `updateTag` 和 `revalidateTag` 都會使緩存數據失效，但它們的目的各異：

- **`updateTag`**：
  - 僅限於在 Server Actions 中使用
  - 下一個請求等待新數據（不服務過期的內容）
  - 專為 read-your-own-writes（讀取自己寫入）情境設計

- **`revalidateTag`**：
  - 可在 Server Actions 和 Route Handlers 中使用
  - 使用 `profile="max"`（推薦）：在背景獲取新數據的同時提供緩存數據（stale-while-revalidate（過期緩存重驗證））
  - 使用自定義配置：可配置為任何緩存生命週期設定以進行高級使用
  - 無配置：舊版行為，相當於 `updateTag`

## 範例

### 具有 read-your-own-writes（讀取自己寫入）的 Server Action

```ts filename="app/actions.ts" switcher
'use server'

import { updateTag } from 'next/cache'
import { redirect } from 'next/navigation'

export async function createPost(formData: FormData) {
  const title = formData.get('title')
  const content = formData.get('content')

  // Create the post in your database
  const post = await db.post.create({
    data: { title, content },
  })

  // Invalidate cache tags so the new post is immediately visible
  // 'posts' tag: affects any page that displays a list of posts
  updateTag('posts')
  // 'post-{id}' tag: affects the individual post detail page
  updateTag(`post-${post.id}`)

  // Redirect to the new post - user will see fresh data, not cached
  redirect(`/posts/${post.id}`)
}
```

### 在 Server Actions 外使用時出現錯誤

```ts filename="app/api/posts/route.ts" switcher
import { updateTag } from 'next/cache'

export async function POST() {
  // This will throw an error
  updateTag('posts')
  // Error: updateTag can only be called from within a Server Action

  // Use revalidateTag instead in Route Handlers
  revalidateTag('posts', 'max')
}
```

## 何時使用 updateTag

當以下情況時使用 updateTag：

- 您在 Server Action 內
- 您需要立即使緩存無效以實現 read-your-own-writes（讀取自己寫入）
- 您希望確保下一個請求看到的是更新後的數據

當以下情況時，請改用 revalidateTag：

- 您在 Route Handler 或其他非動作情境中
- 您希望實現 stale-while-revalidate（過期緩存重驗證）語義
- 您正在構建用於緩存無效化的 webhook 或 API 端點
