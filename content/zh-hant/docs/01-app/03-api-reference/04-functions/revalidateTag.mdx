`revalidateTag` 允許您即時無效化特定緩存標籤的緩存資料。

此函式特別適用於更新略有延遲亦可接受的內容，例如部落格文章、產品目錄或文件檔。使用者會收到陳舊的內容，而新資料則在背景載入中。

## 使用方法

`revalidateTag` 可以在伺服器函式 (Server Functions) 和路由處理程式 (Route Handlers) 中呼叫。

`revalidateTag` 無法在客戶端元件或代理程式中呼叫，因為它僅在伺服器環境中運作。

### 重新驗證行為

重新驗證行為取決於你是否提供了第二個參數：

- **使用 `profile="max"`（推薦）**: 標籤條目被標記為陳舊，下次造訪具有該標籤的資源時，將會使用 `stale-while-revalidate` 語意。這表示在背景獲取新內容的同時，會提供陳舊內容。
- **使用自定義緩存生命週期配置檔**: 針對進階用途，您可以指定應用程式定義的任何緩存生命週期配置檔，從而允許根據您的特定緩存需求定製重新驗證行為。
- **未提供第二個參數（已棄用）**: 標籤條目會立即過期，下次請求該資源時將會成為阻擋式的重新驗證/緩存未命中。此行為現已棄用，請改為使用 `profile="max"` 或遷移至 [`updateTag`](/docs/app/api-reference/functions/updateTag)。

> **小知識**: 當使用 `profile="max"` 時，`revalidateTag` 會將標記過的資料標註為陳舊，但新資料僅在下一次造訪使用該標籤的頁面時才會獲取。這表示呼叫 `revalidateTag` 不會立即觸發許多重新驗證。無效化僅在下次造訪使用該標籤的任何頁面時發生。

## 參數

```ts
revalidateTag(tag: string, profile: string | { expire?: number }): void;
```

- `tag`: 代表您想要重新驗證的資料相關的緩存標籤的字串。不得超過 256 個字元。此值為區分大小寫。
- `profile`: 指定重新驗證行為的字串。推薦值是 ` "max" `, 它提供 `stale-while-revalidate` 語意，或者 [`cacheLife`](/docs/app/api-reference/config/next-config-js/cacheLife) 中定義的其他預設或自定義配置檔中的任一值。或者，您也可以傳遞具有 `expire` 屬性之物件以進行自定義過期行為。

標籤必須先分配給緩存數據。您可以透過以下方式完成：

- 使用 [`next.tags`](/docs/app/api-reference/functions/fetch) 選項搭配 `fetch` 來快取外部 API 請求：

```tsx
fetch(url, { next: { tags: ['posts'] } })
```

- 在帶有 `'use cache'` 直達式的緩存函式或元件中使用 [`cacheTag`](/docs/app/api-reference/functions/cacheTag)：

```tsx
import { cacheTag } from 'next/cache'

async function getData() {
  'use cache'
  cacheTag('posts')
  // ...
}
```

> **小知識**: 單參數形式 `revalidateTag(tag)` 已棄用。如果 TypeScript 錯誤被抑制，目前仍有效用，但此行為可能會在未來版本中移除。請更新為雙參數簽章。

## 返回值

`revalidateTag` 不會傳回任何值。

## 與 revalidatePath 的關係

`revalidateTag` 會使使用這些標籤的所有頁面上的帶有特定標籤的資料無效，而 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 則會使特定的頁面或佈局路徑無效。

> **小知識**: 這些函式服務於不同目的，可能需要搭配使用以達成全面的資料一致性。關於詳細範例與注意事項，請參閱 [與 revalidateTag 和 updateTag 的關係](/docs/app/api-reference/functions/revalidatePath#relationship-with-revalidatetag-and-updatetag) 以了解更多信息。

## 範例

以下範例展示如何在不同情境中使用 `revalidateTag`。在兩種情況下，我們都使用 `profile="max"` 將資料標記為陳舊並使用 `stale-while-revalidate` 語意，這是最常用的使用情境中的推薦做法。

### 伺服器動作

```ts filename="app/actions.ts" switcher
'use server'

import { revalidateTag } from 'next/cache'

export default async function submit() {
  await addPost()
  revalidateTag('posts', 'max')
}
```

```js filename="app/actions.js" switcher
'use server'

import { revalidateTag } from 'next/cache'

export default async function submit() {
  await addPost()
  revalidateTag('posts', 'max')
}
```

### 路由處理程式

```ts filename="app/api/revalidate/route.ts" switcher
import type { NextRequest } from 'next/server'
import { revalidateTag } from 'next/cache'

export async function GET(request: NextRequest) {
  const tag = request.nextUrl.searchParams.get('tag')

  if (tag) {
    revalidateTag(tag, 'max')
    return Response.json({ revalidated: true, now: Date.now() })
  }

  return Response.json({
    revalidated: false,
    now: Date.now(),
    message: 'Missing tag to revalidate',
  })
}
```

```js filename="app/api/revalidate/route.js" switcher
import { revalidateTag } from 'next/cache'

export async function GET(request) {

  if (tag) {

  }

  return Response.json({
    revalidated: false,

  })

```

> **小知識**: 針對需要立即過期的 webhooks 或第三方服務，您可以將 `{ expire: 0 }` 作為第二個參數傳遞：`revalidateTag(tag, { expire: 0 })`。當外部系統呼叫您的路由處理程式並要求數據立即過期時，此模式是必要的。對於所有其他情況，建議在伺服器動作 (Server Actions) 中使用 [`updateTag`](/docs/app/api-reference/functions/updateTag) 進行立即更新。
