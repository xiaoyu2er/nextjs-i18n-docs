`redirect` 函式允許您將使用者重定向到另一個 URL。`redirect` 可在渲染時於 [伺服器與客戶端組件 (Server and Client Components)](/docs/app/getting-started/server-and-client-components)、[路由處理程式 (Route Handlers)](/docs/app/api-reference/file-conventions/route) 以及 [伺服器函式 (Server Functions)](/docs/app/getting-started/mutating-data) 中使用。

當在 [串流環境](/docs/app/getting-started/linking-and-navigating#streaming) 中使用时，它會插入一個 meta 標籤以在客戶端側發出重新導向。當在伺服器動作中使用時，它將向呼叫端提供 303 HTTP 重新導向回應。否則，它將向呼叫端提供 307 HTTP 重新導向回應。

如果資源不存在，您可以改用 notFound 函式。

## 參考

### 參數

「redirect」函數接受兩個參數：

```js
redirect(path, type)
```

| 參數 | 類型 | 描述 |
| --------- | ------------------------------------------------------------- | ----------------------------------------------------------- |
| `path` | `string` | 要重新導向的 URL。可以是相對或絕對路徑。 |
| `type` | `'replace'` (預設) 或 `'push'` (伺服器動作預設) | 要執行的重導向類型。 |

預設情況下，`redirect` 將在 [伺服器動作 (Server Actions)](/docs/app/getting-started/mutating-data) 中使用 `push`（向瀏覽器歷史堆疊添加新條目），而在其他地方使用 `replace`（替換瀏覽器歷史堆疊中的當前 URL）。您可以透過指定 `type` 參數來覆寫此行為。

「RedirectType」物件包含 `type` 參數的可用選項。

```ts
import { redirect, RedirectType } from 'next/navigation'

redirect('/redirect-to', RedirectType.replace)
// or
redirect('/redirect-to', RedirectType.push)
```

`type` 參數在伺服器組件中使用時無效。

### 回傳值

`redirect` 不會回傳任何值。

## 行為

- 在伺服器動作 (Server Actions) 和路由處理程式 (Route Handlers) 中，使用 `try/catch` 語句時，應在 `try` 區塊 **外部** 呼叫 `redirect`。
- 如果您傾向於傳回 308 (永久) HTTP 重定向而不是 307 (暫定)，則可使用 [`permanentRedirect` 函式](/docs/app/api-reference/functions/permanentRedirect) 代替。
- `redirect` 會拋出錯誤，因此在採用 `try/catch` 語句時，應在 `try` 區塊 **外部** 呼叫。
- `redirect` 可在渲染過程中於客戶端組件 (Client Components) 中呼叫，但不能在事件處理常式中使用。您可以改用 [`useRouter` 鉤子](/docs/app/api-reference/functions/use-router)。
- `redirect` 也接受絕對 URL，可用於重定向至外部連結。
- 如果您希望在渲染過程之前進行重定向，請使用 [`next.config.js`](/docs/app/guides/redirecting#redirects-in-nextconfigjs) 或 [Proxy](/docs/app/guides/redirecting#nextresponseredirect-in-proxy)。

## 範例

### 伺服器組件

調用 `redirect()` 函數會拋出 `NEXT_REDIRECT` 錯誤，並終止發生該錯誤的路徑段落的渲染。

```tsx filename="app/team/[id]/page.tsx" switcher
import { redirect } from 'next/navigation'

async function fetchTeam(id: string) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({
  params,
}: {
  params: Promise<{ id: string }>
}) {
  const { id } = await params
  const team = await fetchTeam(id)

  if (!team) {
    redirect('/login')
  }

  // ...
}
```

```jsx filename="app/team/[id]/page.js" switcher
import { redirect } from 'next/navigation'

async function fetchTeam(id) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({ params }) {
  const { id } = await params
  const team = await fetchTeam(id)

  if (!team) {
    redirect('/login')
  }

  // ...
}
```

> **小提示**：`redirect` 不需要您使用 `return redirect()`，因為它使用了 TypeScript [`never`](https://www.typescriptlang.org/docs/handbook/2/functions.html#never) 類型。

### 客戶端組件

`redirect` 可以直接在客戶端組件中使用。

```tsx filename="components/client-redirect.tsx" switcher
'use client'

import { redirect, usePathname } from 'next/navigation'

export function ClientRedirect() {
  const pathname = usePathname()

  if (pathname.startsWith('/admin') && !pathname.includes('/login')) {
    redirect('/admin/login')
  }

  return <div>Login Page</div>
}
```

```jsx filename="components/client-redirect.jsx" switcher
'use client'

import { redirect, usePathname } from 'next/navigation'

export function ClientRedirect() {
  const pathname = usePathname()

  if (pathname.startsWith('/admin') && !pathname.includes('/login')) {
    redirect('/admin/login')
  }

  return <div>Login Page</div>
}
```

> **小提示**：在伺服器端渲染 (SSR) 期間初次載入頁面時，若在客戶端組件中使用 `redirect`，它將執行伺服器端重新導向。

`redirect` 可透過伺服器動作在客戶端組件中使用。如果您需要使用事件處理函數來重新導向使用者，您可以使用 [`useRouter](/docs/app/api-reference/functions/use-router) 掛鉤。

```tsx filename="app/client-redirect.tsx" switcher
'use client'

import { navigate } from './actions'

export function ClientRedirect() {
  return (
    <form action={navigate}>
      <input type="text" name="id" />
      <button>Submit</button>
    </form>
  )
}
```

```jsx filename="app/client-redirect.jsx" switcher
'use client'

import { navigate } from './actions'

export function ClientRedirect() {
  return (
    <form action={navigate}>
      <input type="text" name="id" />
      <button>Submit</button>
    </form>
  )
}
```

```ts filename="app/actions.ts" switcher
'use server'

import { redirect } from 'next/navigation'

export async function navigate(data: FormData) {
  redirect(`/posts/${data.get('id')}`)
}
```

## 常見問題 (FAQ)

### 為什麼「redirect」使用 307 和 308？

使用 `redirect()` 時，您可能會注意到使用的狀態碼：暫時重新導向為 `307`，永久重新導向為 `308`。雖然傳統上使用 `302` 進行暫時重新導向，使用 `301` 進行永久重新導向，但在許多瀏覽器中，使用 `302` 時，無論來源請求方法如何，都會將重新導向的請求方法從 `POST` 更改為 `GET` 請求。

以下舉例說明從 `/users` 到 `/people` 的重新導向，如果您向 `/users` 發送 `POST` 請求以創建新用戶，並且遵循 `302` 暫時重新導向，請求方法將從 `POST` 更改為 `GET` 請求。這不合理，因為要創建新用戶，您應該向 `/people` 發送 `POST` 請求，而不是 `GET` 請求。

引入 `307` 狀態碼意味著請求方法將保留為 `POST`。

- `302` - 暫時重新導向，將把請求方法從 `POST` 更改為 `GET`
- `307` - 暫時重新導向，將保留請求方法為 `POST`

「redirect()」方法預設使用 `307`，而不是 `302` 暫時重新導向，這意味著您的請求將_始終_保留為 `POST` 請求。

[了解更多](https://developer.mozilla.org/docs/Web/HTTP/Redirections) 關於 HTTP 重新導向。
