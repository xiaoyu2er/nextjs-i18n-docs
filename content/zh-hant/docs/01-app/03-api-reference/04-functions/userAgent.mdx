\`userAgent\` 助手擴充了 [Web Request API](https://developer.mozilla.org/docs/Web/API/Request)，提供額外屬性與方法以與請求中的使用者代理物件進行互動。

```ts filename="proxy.ts" switcher
import { NextRequest, NextResponse, userAgent } from 'next/server'

export function proxy(request: NextRequest) {
  const url = request.nextUrl
  const { device } = userAgent(request)

  // device.type can be: 'mobile', 'tablet', 'console', 'smarttv',
  // 'wearable', 'embedded', or undefined (for desktop browsers)
  const viewport = device.type || 'desktop'

  url.searchParams.set('viewport', viewport)
  return NextResponse.rewrite(url)
}
```

## \`isBot\` (是否為機器人)

表示請求是否來自已知機器人的布林值。

## \`browser\` (瀏覽器)

包含請求中使用的瀏覽器資訊的物件。

- \`name\`: 表示瀏覽器名稱的字串，若無法識別則為 \`undefined\`。
- \`version\`: 表示瀏覽器版本的字串，或 \`undefined\`。

## \`device\` (裝置)

包含請求中使用的裝置資訊的物件。

- \`model\`: 表示裝置型號的字串，或 \`undefined\`。
- \`type\`: 表示裝置類型的字串，例如 \`console\`, \`mobile\`, \`tablet\`, \`smarttv\`, \`wearable\`, \`embedded\`, 或 \`undefined\`。
- \`vendor\`: 表示裝置廠商的字串，或 \`undefined\`。

## \`engine\` (瀏覽器引擎)

包含關於瀏覽器引擎資訊的物件。

- \`name\`: 表示引擎名稱的字串。可能的值包括：\`Amaya\`, \`Blink\`, \`EdgeHTML\`, \`Flow\`, \`Gecko\`, \`Goanna\`, \`iCab\`, \`KHTML\`, \`Links\`, \`Lynx\`, \`NetFront\`, \`NetSurf\`, \`Presto\`, \`Tasman\`, \`Trident\`, \`w3m\`, \`WebKit\` 或 \`undefined\`。
- \`version\`: 表示引擎版本的字串，或 \`undefined\`。

## \`os\` (作業系統)

包含關於作業系統資訊的物件。

- \`name\`: 表示作業系統名稱的字串，或 \`undefined\`。
- \`version\`: 表示作業系統版本的字串，或 \`undefined\`。

## \`cpu\` (CPU 架構)

包含關於 CPU 架構資訊的物件。

- \`architecture\`: 表示 CPU 架構的字串。可能的值包括：\`68k\`, \`amd64\`, \`arm\`, \`arm64\`, \`armhf\`, \`avr\`, \`ia32\`, \`ia64\`, \`irix\`, \`irix64\`, \`mips\`, \`mips64\`, \`pa-risc\`, \`ppc\`, \`sparc\`, \`sparc64\` 或 \`undefined\`
