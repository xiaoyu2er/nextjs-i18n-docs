您可以使用 `metadata` 物件或 `generateMetadata` 函數來定義元數據。

## `metadata` 物件

若要定義靜態元數據，請從 `layout.js` 或 `page.js` 檔案匯出 [`Metadata` 物件](#metadata-fields)。

```tsx filename="layout.tsx | page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: '...',
  description: '...',
}

export default function Page() {}
```

```jsx filename="layout.js | page.js" switcher
export const metadata = {
  title: '...',
  description: '...',
}

export default function Page() {}
```

> 請參見 [Metadata Fields](#metadata-fields) 以獲取支援選項的完整清單。

## `generateMetadata` 函數

動態元數據取決於**動態資訊**，例如目前的路由參數、外部資料，或父段中的 `metadata`，可以透過匯出返回 [`Metadata` 物件](#metadata-fields) 的 `generateMetadata` 函數來設定。

解析 `generateMetadata` 是頁面渲染的一部分。如果頁面可以進行靜態預渲染，且 `generateMetadata` 未引入動態行為，則產生的 metadata 將包含在頁面的初始 HTML 中。

否則，從 `generateMetadata` 解析的元數據可以在發送初始 UI 後 [進行流式傳輸](/docs/app/api-reference/functions/generate-metadata#streaming-metadata)。

```tsx filename="app/products/[id]/page.tsx" switcher
import type { Metadata, ResolvingMetadata } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  // read route params
  const { id } = await params

  // fetch data
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // optionally access and extend (rather than replace) parent metadata
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }: Props) {}
```

```jsx filename="app/products/[id]/page.js" switcher
export async function generateMetadata({ params, searchParams }, parent) {
  // read route params
  const { id } = await params

  // fetch data
  const product = await fetch(`https://.../${id}`).then((res) => res.json())

  // optionally access and extend (rather than replace) parent metadata
  const previousImages = (await parent).openGraph?.images || []

  return {
    title: product.title,
    openGraph: {
      images: ['/some-specific-page-image.jpg', ...previousImages],
    },
  }
}

export default function Page({ params, searchParams }) {}
```

若要為 `params` 和 `searchParams` 進行類型補全，您可以使用 [`PageProps<'/route'>`](/docs/app/api-reference/file-conventions/page#page-props-helper) 或 [`LayoutProps<'/route'>`](/docs/app/api-reference/file-conventions/layout#layout-props-helper) 分別針對頁面和佈局來為第一個引數標註類型。

>
> **值得注意**：
>
> - 可以在 `layout.js` 和 `page.js` 檔案中加入 `metadata`。
> - Next.js 會自動解析 `metadata`，並為頁面建立相關的 `<head>` 標籤。
> - `metadata` 物件和 `generateMetadata` 函式**僅支援在伺服器元件 (Server Components) 中匯出**。
> - 您不能從同一路由區段同時匯出 `metadata` 物件和 `generateMetadata` 函式。
> - `generateMetadata` 內的 `fetch` 請求會自動對 `generateMetadata`、`generateStaticParams`、佈局、頁面和伺服器元件之間的相同資料進行 [已快取 (memoized)](/docs/app/glossary#memoization)。
> - 若無法使用 `fetch`，可搭配 React [`cache`](https://react.dev/reference/react/cache) 功能。
> - [基於檔案的 metadata](/docs/app/api-reference/file-conventions/metadata) 具有較高優先級，將覆蓋 `metadata` 物件和 `generateMetadata` 函式。

## 為什麼 `generateMetadata` 僅支援伺服器元件 (Server Components)

`generateMetadata` 和 `metadata` 匯出僅支援在伺服器元件 (Server Components) 中使用，因為 `metadata` 必須在頁面元件渲染之前在伺服器上解析。這使得 Next.js 能夠將 `metadata` 包含在初始 HTML 回應中。

如果您需要使用客戶端元件功能，請將 `page.tsx` 保留為伺服器元件，並將客戶端元件邏輯移至單獨的文件：

```tsx filename="app/page.tsx" switcher
import type { Metadata } from 'next'
import { InteractiveComponent } from './interactive-component'

export const metadata: Metadata = {
  title: 'My Page',
}

export default function Page() {
  return <InteractiveComponent />
}
```

```jsx filename="app/page.js" switcher
import { InteractiveComponent } from './interactive-component'

export const metadata = {
  title: 'My Page',
}

export default function Page() {
  return <InteractiveComponent />
}
```

```tsx filename="app/interactive-component.tsx" switcher
'use client'

export function InteractiveComponent() {
  // Client-side interactivity (hooks, event handlers, etc.)
}
```

```jsx filename="app/interactive-component.js" switcher
'use client'

export function InteractiveComponent() {
  // Client-side interactivity (hooks, event handlers, etc.)
}
```

## 參考

### 參數

`generateMetadata` 函數接受以下參數：

- `props` - 包含目前路徑參數的物件：
  - `params` - 包含 [動態路徑參數](/docs/app/api-reference/file-conventions/dynamic-routes) 物件的對象，從根區段一直到 `generateMetadata` 被呼叫的區段。範例：

    | 路徑                           | URL         | `params`                  |
    | ------------------------------- | ----------- | ------------------------- |
    | `app/shop/[slug]/page.js`       | `/shop/1`   | `{ slug: '1' }`           |
    | `app/shop/[tag]/[item]/page.js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |
    | `app/shop/[...slug]/page.js`    | `/shop/1/2` | `{ slug: ['1', '2'] }`    |

  - `searchParams` - 包含目前 URL 的 [搜尋參數](https://developer.mozilla.org/docs/Learn/Common_questions/What_is_a_URL#parameters) 的物件。範例：

    | URL             | `searchParams`       |
    | --------------- | -------------------- |
    | `/shop?a=1`     | `{ a: '1' }`         |
    | `/shop?a=1&b=2` | `{ a: '1', b: '2' }` |
    | `/shop?a=1&a=2` | `{ a: ['1', '2'] }`  |

- `parent` - 父層路徑區段解析後的 `metadata` 的 Promise。

### 回傳值

`generateMetadata` 應返回包含一個或多個元數據欄位的 [`Metadata` 物件](#metadata-fields)。

>
> **值得注意**：
>
> - 如果 `metadata` 不依賴請求資訊，應使用靜態 [`metadata` 物件](#the-metadata-object) 而非 `generateMetadata`。
> - `fetch` 請求會自動對 `generateMetadata`、`generateStaticParams`、佈局、頁面和伺服器元件之間的相同資料進行快取。若無法使用 `fetch`，可搭配 React [`cache`](https://react.dev/reference/react/cache) 功能。
> - `searchParams` 僅在 `page.js` 區段中可用。
> - 也可在 `generateMetadata` 中使用 [`redirect()`](/docs/app/api-reference/functions/redirect) 和 [`notFound()`](/docs/app/api-reference/functions/not-found) Next.js 方法。

### Metadata Fields

支援以下欄位：

#### `title`

`title` 屬性用於設定文件標題。它可以定義為簡單的字串 或是可選的 [模板物件](#template)。

##### String

```jsx filename="layout.js | page.js"
export const metadata = {
  title: 'Next.js',
}
```

```html filename="<head> output" hideLineNumbers
<title>Next.js</title>
```

##### `default`

`title.default` 可用於為未定義 `title` 的子路由段提供**預設字串**。

```tsx filename="app/layout.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    default: 'Acme',
  },
}
```

```tsx filename="app/about/page.tsx"
import type { Metadata } from 'next'

export const metadata: Metadata = {}

// Output: <title>Acme</title>
```

##### `template`

`title.template` 可用於在定義於 **子** 路由段的 `titles` 加入前綴或後綴。

```tsx filename="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    template: '%s | Acme',
    default: 'Acme', // a default is required when creating a template
  },
}
```

```jsx filename="app/layout.js" switcher
export const metadata = {
  title: {
    template: '%s | Acme',
    default: 'Acme', // a default is required when creating a template
  },
}
```

```tsx filename="app/about/page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: 'About',
}

// Output: <title>About | Acme</title>
```

```jsx filename="app/about/page.js" switcher
export const metadata = {
  title: 'About',
}

// Output: <title>About | Acme</title>
```

> **值得瞭解**：
>
> - `title.template` 適用於 **子** 路由段，而不是它定義的段。這意味著：
>   - 當您新增 `title.template` 時，`title.default` 是 **必需的**。
>   - `layout.js` 中定義的 `title.template` 不會套用于同一路由段中 `page.js` 定義的 `title`。
>   - `page.js` 中定義的 `title.template` 無效，因為頁面始終是終止段（它沒有子路由段）。
> - 如果路由未定義 `title` 或 `title.default`，`title.template` 無效。

##### `absolute`

`title.absolute` 可用於提供 **忽略** 父段中設定的 `title.template` 的標題。

```tsx filename="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    template: '%s | Acme',
  },
}
```

```jsx filename="app/layout.js" switcher
export const metadata = {
  title: {
    template: '%s | Acme',
  },
}
```

```tsx filename="app/about/page.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  title: {
    absolute: 'About',
  },
}

// Output: <title>About</title>
```

```jsx filename="app/about/page.js" switcher
export const metadata = {
  title: {
    absolute: 'About',
  },
}

// Output: <title>About</title>
```

>
> **值得注意**：
>
> - `layout.js`
>   - `title` (字串) 和 `title.default` 定義子區段（未定義其 own `title`）的預設標題。若存在，它將擴充最接近父區段的 `title.template`。
>   - `title.absolute` 定義子區段的預設標題。它會忽略父區段的 `title.template`。
>   - `title.template` 為子區段定義新的標題範本。
> - `page.js`
>   - 如果頁面未定義其標題，將使用最接近父層解析的標題。
>   - `title` (字串) 定義路由標題。若存在，它將擴充最接近父區段的 `title.template`。
>   - `title.absolute` 定義路由標題。它會忽略父區段的 `title.template`。
>   - `title.template` 在 `page.js` 中無效，因為頁面始終是路由的終止區段。

#### `metadataBase`

`metadataBase` 是一個方便選項，用於設定需要完整限定 URL 的 `metadata` 欄位的基礎 URL 前綴。

- `metadataBase` 允許在 **目前路由段和下方** 定義的基於 URL 的 `metadata` 欄位使用**相對路徑**，而不需要原本必需的絕對 URL。
- 欄位的相對路徑將與 `metadataBase` 組合成為完整適用的 URL。

```jsx filename="layout.js | page.js"
export const metadata = {
  metadataBase: new URL('https://acme.com'),
  alternates: {
    canonical: '/',
    languages: {
      'en-US': '/en-US',
      'de-DE': '/de-DE',
    },
  },
  openGraph: {
    images: '/og-image.png',
  },
}
```

```html filename="<head> output" hideLineNumbers
<link rel="canonical" href="https://acme.com" />
<link rel="alternate" hreflang="en-US" href="https://acme.com/en-US" />
<link rel="alternate" hreflang="de-DE" href="https://acme.com/de-DE" />
<meta property="og:image" content="https://acme.com/og-image.png" />
```

> **值得瞭解**：
>
> - `metadataBase` 通常在根目錄 `app/layout.js` 中設定，以套用于所有路由的基於 URL 的 `metadata` 欄位。
> - 所有需要絕對 URL 的基於 URL 的 `metadata` 欄位都可以使用 `metadataBase` 選項進行設定。
> - `metadataBase` 可以包含子網域，例如 `https://app.acme.com` 或基礎路徑，例如 `https://acme.com/start/from/here`
> - 如果 `metadata` 欄位提供了絕對 URL，則將忽略 `metadataBase`。
> - 在未設定 `metadataBase` 的情況下在基於 URL 的 `metadata` 欄位中使用相對路徑將導致建構錯誤。
> - Next.js 會將 `metadataBase`（例如 `https://acme.com/`）和相對欄位（例如 `/path`）之間的冗餘斜線規範化為單一斜線（例如 `https://acme.com/path`）

#### URL 組合成

URL 組合成優先考慮開發者的意圖而非預設的目錄遍歷語義。

- `metadataBase` 和 `metadata` 欄位之間的尾隨斜線会被規範化。
- `metadata` 欄位中的「絕對」路徑（通常會替換整個 URL 路徑）被視為「相對」路徑（從 `metadataBase` 的結尾開始）。

例如，給定以下 `metadataBase`：

```tsx filename="app/layout.tsx" switcher
import type { Metadata } from 'next'

export const metadata: Metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

```jsx filename="app/layout.js" switcher
export const metadata = {
  metadataBase: new URL('https://acme.com'),
}
```

任何繼承上述 `metadataBase` 並設定其自身值的所有 `metadata` 欄位將按以下方式解析：

| `metadata` field                 | Resolved URL                     |
| -------------------------------- | -------------------------------- |
| `/`                              | `https://acme.com`               |
| `./`                             | `https://acme.com`               |
| `payments`                       | `https://acme.com/payments`      |
| `/payments`                      | `https://acme.com/payments`      |
| `./payments`                     | `https://acme.com/payments`      |
| `../payments`                    | `https://acme.com/payments`      |
| `https://beta.acme.com/payments` | `https://beta.acme.com/payments` |

### Resource hints

`<link>` 元素具有多個 `rel` 關鍵字，可用於提示瀏覽器可能會需要外部資源。瀏覽器使用此資訊來根據關鍵字套用預加載最佳化。

```tsx filename="app/preload-resources.tsx" switcher
  ReactDOM.prefetchDNS('...')

  return '...'
}
```

## 行為

### 預設欄位

即使路由未定義元數據，始終有兩個預設的 `meta` 標籤會被加入：

- [meta charset 標籤](https://developer.mozilla.org/docs/Web/HTML/Element/meta#attr-charset) 設定網站的字元編碼。
- [meta viewport 標籤](https://developer.mozilla.org/docs/Web/HTML/Viewport_meta_tag) 設定網站的視窗寬度和縮放比例，以適應不同的裝置。

```html
<meta charset="utf-8" />
<meta name="viewport" content="width=device-width, initial-scale=1" />
```

> **值得瞭解**：您可以覆寫預設的 [`viewport`](/docs/app/api-reference/functions/generate-metadata#viewport) meta 標籤。

### 流式元數據

流式元數據允許 Next.js 渲染並將初始 UI 發送給瀏覽器，而不必等待 `generateMetadata` 完成。

當 `generateMetadata` 解析時，生成的元數據標籤將附加到 `<body>` 標籤。我們已驗證，在執行 JavaScript 並檢查完整 DOM 的機器人（例如 `Googlebot`）可以正確解釋元數據。

對於**受限於 HTML 的機器人**，無法執行 JavaScript（例如 `facebookexternalhit`），元數據繼續阻擋頁面渲染。生成的元數據將可在 `<head>` 標籤中取得。

Next.js 自動偵測**受限於 HTML 的機器人**，透過檢視 User Agent 標頭。您可以在您的 Next.js 設定文件中，使用 [`htmlLimitedBots`](/docs/app/api-reference/config/next-config-js/htmlLimitedBots) 選項來覆寫預設的 [User Agent 清單](https://github.com/vercel/next.js/blob/canary/packages/next/src/shared/lib/router/utils/html-bots.ts)。

若要完全停用流式元數據：

```ts filename="next.config.ts" switcher
import type { NextConfig } from 'next'

const config: NextConfig = {
  htmlLimitedBots: /.*/,
}

export default config
```

```js filename="next.config.js" switcher
module.exports = {
  htmlLimitedBots: /.*/,
}
```

流式元數據透過降低 [TTFB](https://developer.mozilla.org/docs/Glossary/Time_to_first_byte) 來改善感知效能，並可有助於降低 [LCP](https://developer.mozilla.org/docs/Glossary/Largest_contentful_paint) 時間。

覆寫 `htmlLimitedBots` 可能會導致較長的回應時間。流式元數據是一個進階功能，預設設定應該足夠應付大多數情況。

### 與快取元件 (Cache Components)

當啟用 [快取元件 (Cache Components)](/docs/app/getting-started/caching) 時，`generateMetadata` 遵循與其他元件相同的規則。如果 `metadata` 存取執行階段資料 (`cookies()`, `headers()`, `params`, `searchParams`) 或執行未快取的資料擷取，則會延遲到請求時間。

Next.js 處理此情況的方式取決於您頁面的其餘部分：

- **如果其他部分也延遲至請求時間**：靜態預渲染產生靜態外殼，且 `metadata` 會與其他延遲的內容一起串流。
- **如果頁面或佈局在其他方面是可完全預渲染的**：Next.js 需要明確的選擇：如果可能則快取資料，或發出訊號表示延遲渲染是有意為之。

在頁面其餘部分完全可預渲染的同時，於執行階段流式傳輸 `metadata` 並不常見。為確保此行為是有意為之，將會發出錯誤來指出需要處理的頁面或佈局。

為解決此問題，您可以選擇兩種方案。如果 `metadata` 依賴外部資料但不依賴執行階段資料，請使用 `use cache`：

```tsx filename="app/page.tsx" highlight={2}
export async function generateMetadata() {
  'use cache'
  const { title, description } = await db.query('site-metadata')
  return { title, description }
}
```

如果 `metadata` 確實需要執行階段資料，請為您的頁面新增一個動態標記元件：

```tsx filename="app/page.tsx" highlight={11-14,19,31}
import { Suspense } from 'react'
import { cookies } from 'next/headers'
import { connection } from 'next/server'

export async function generateMetadata() {
  const token = (await cookies()).get('token')?.value
  // ... use token to fetch personalized metadata
  return { title: 'Personalized Title' }
}

const Connection = async () => {
  await connection()
  return null
}

async function DynamicMarker() {
  return (
    <Suspense>
      <Connection />
    </Suspense>
  )
}

export default function Page() {
  // DO NOT place await connection() here
  // doing so prevents the article tag content from
  // being included in the static shell
  return (
    <>
      <article>Static content</article>
      <DynamicMarker />
    </>
  )
}
```

`DynamicMarker` 元件不會渲染任何內容，但會告訴 Next.js 頁面包含有意動的動態內容。透過將其包裝在 Suspense 中，靜態內容仍可正常預渲染。

### Ordering

元數據依順序評估，從根段開始直到最接近最終 `page.js` 段的段。例如：

1. `app/layout.tsx` (Root Layout)
2. `app/blog/layout.tsx` (Nested Blog Layout)
3. `app/blog/[slug]/page.tsx` (Blog Page)

### Merging

依照 [評估順序](#ordering)，從同一路由中的多個段匯出的 Metadata 物件會以**淺層**方式合併在一起，形成路由的最終元數據輸出。重複的鍵會根據其順序被**替換**。

這意味著具有嵌套欄位的元數據，例如 [`openGraph`](/docs/app/api-reference/functions/generate-metadata#opengraph) 和 [`robots`](/docs/app/api-reference/functions/generate-metadata#robots)，在較早段中定義的會被最後一個定義它的段**覆蓋**。

#### Overwriting fields

```jsx filename="app/layout.js"
export const metadata = {
  title: 'Acme',
  openGraph: {
    title: 'Acme',
    description: 'Acme is a...',
  },
}
```

```jsx filename="app/blog/page.js"
export const metadata = {
  title: 'Blog',
  openGraph: {
    title: 'Blog',
  },
}

// Output:
// <title>Blog</title>
// <meta property="og:title" content="Blog" />
```

在上述範例中：

- `app/layout.js` 的 `title` 被 `app/blog/page.js` 中的 `title` **替換**。
- `app/layout.js` 的所有 `openGraph` 欄位在 `app/blog/page.js` 中都被**替換**，因為 `app/blog/page.js` 設定 `openGraph` 元數據。請注意缺少 `openGraph.description`。

如果您希望在覆蓋其他欄位時共用一些嵌套欄位，您可以將它們拉到變數中：

```jsx filename="app/shared-metadata.js"
export const openGraphImage = { images: ['http://...'] }
```

```jsx filename="app/page.js"
import { openGraphImage } from './shared-metadata'

export const metadata = {
  openGraph: {
    ...openGraphImage,
    title: 'Home',
  },
}
```

```jsx filename="app/about/page.js"
import { openGraphImage } from '../shared-metadata'

export const metadata = {
  openGraph: {
    ...openGraphImage,
    title: 'About',
  },
}
```

在上述範例中，OG 圖片在 `app/layout.js` 和 `app/about/page.js` 之間共用，而標題不同。

#### Inheriting fields

```jsx filename="app/layout.js"
export const metadata = {
  title: 'Acme',
  openGraph: {
    title: 'Acme',
    description: 'Acme is a...',
  },
}
```

```jsx filename="app/about/page.js"
export const metadata = {
  title: 'About',
}

// Output:
// <title>About</title>
// <meta property="og:title" content="Acme" />
// <meta property="og:description" content="Acme is a..." />
```

**Notes**

- `app/layout.js` 的 `title` 被 `app/about/page.js` 中的 `title` **替換**。
- `app/layout.js` 的所有 `openGraph` 欄位在 `app/about/page.js` 中都被**繼承**，因為 `app/about/page.js` 未設定 `openGraph` 元數據。
