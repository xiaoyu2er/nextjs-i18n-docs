`draftMode` 是一個 **async** 函式，允許您啟用並停用 [草稿模式](/docs/app/guides/draft-mode)，以及檢查在 [伺服器組件 (Server Component)](/docs/app/getting-started/server-and-client-components) 中的草稿模式是否已啟用。

```tsx filename="app/page.ts" switcher
import { draftMode } from 'next/headers'

export default async function Page() {
  const { isEnabled } = await draftMode()
}
```

```jsx filename="app/page.js" switcher
import { draftMode } from 'next/headers'

export default async function Page() {
  const { isEnabled } = await draftMode()
}
```

## 參考

下列方法和屬性可用：

| 方法      | 說明                                                                    |
| --------- | ----------------------------------------------------------------------- |
| `isEnabled` | 表示草稿模式是否已啟用的布林值。                                        |
| `enable()`  | 透過設定 cookie (`__prerender_bypass`) 在路由處理常式中啟用草稿模式。    |
| `disable()` | 透過刪除 cookie 在路由處理常式中停用草稿模式。                           |

## 小知識

- `draftMode` 是一個 **非同步 (asynchronous)** 函式，它會傳回 `Promise`。您必須使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函式。
  - 在 14 版本及更早版本中，`draftMode` 是同步函式。為了協助向下相容性，您在 Next.js 15 中仍可同步存取它，但此行為未來將會被停用。
- 每次執行 `next build` 時，都會產生新的 bypass cookie 值。這確保了 bypass cookie 無法被猜測。
- 要在本機透過 HTTP 測試草稿模式，您的瀏覽器必須允許第三方 Cookie 和本機 Storage 存取權限。

## 範例

### 啟用草稿模式

要啟用草稿模式，請建立新的 [路由處理常式 (Route Handler)](/docs/app/api-reference/file-conventions/route) 並呼叫 `enable()` 方法：

```tsx filename="app/draft/route.ts" switcher
import { draftMode } from 'next/headers'

export async function GET(request: Request) {
  const draft = await draftMode()
  draft.enable()
  return new Response('Draft mode is enabled')
}
```

```js filename="app/draft/route.js" switcher
import { draftMode } from 'next/headers'

export async function GET(request) {
  const draft = await draftMode()
  draft.enable()
  return new Response('Draft mode is enabled')
}
```

### 停用草稿模式

預設情況下，草稿模式工作階段會在瀏覽器關閉時結束。

要手動停用草稿模式，請在您的 [路由處理常式 (Route Handler)](/docs/app/api-reference/file-conventions/route) 中呼叫 `disable()` 方法：

```tsx filename="app/draft/route.ts" switcher
import { draftMode } from 'next/headers'

export async function GET(request: Request) {
  const draft = await draftMode()
  draft.disable()
  return new Response('Draft mode is disabled')
}
```

```js filename="app/draft/route.js" switcher
import { draftMode } from 'next/headers'

export async function GET(request) {
  const draft = await draftMode()
  draft.disable()
  return new Response('Draft mode is disabled')
}
```

然後，傳送請求以呼叫路由處理常式。如果您使用 [`<Link>` 元件](/docs/app/api-reference/components/link) 呼叫路由，您必須傳入 `prefetch={false}` 以防止在預取時意外刪除 cookie。

### 檢查草稿模式是否已啟用

您可以在 [伺服器組件 (Server Component)](/docs/app/getting-started/server-and-client-components) 中使用 `isEnabled` 屬性檢查草稿模式是否已啟用：

```tsx filename="app/page.ts" switcher
import { draftMode } from 'next/headers'

export default async function Page() {
  const { isEnabled } = await draftMode()
  return (
    <main>
      <h1>My Blog Post</h1>
      <p>Draft Mode is currently {isEnabled ? 'Enabled' : 'Disabled'}</p>
    </main>
  )
}
```

```jsx filename="app/page.js" switcher
import { draftMode } from 'next/headers'

export default async function Page() {
  const { isEnabled } = await draftMode()
  return (
    <main>
      <h1>My Blog Post</h1>
      <p>Draft Mode is currently {isEnabled ? 'Enabled' : 'Disabled'}</p>
    </main>
  )
}
```
