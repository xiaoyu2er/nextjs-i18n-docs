> **注意**：
> 此 API 已在 Next.js 16 中被 [`use cache`](/docs/app/api-reference/directives/use-cache) 取代。
> 我們建議採用 [緩存組件 (Cache Components)](/docs/app/getting-started/caching) 並使用 `use cache` 指令替換 `unstable_cache`。

`unstable_cache` 允許您將昂贵操作（例如資料庫查詢）的結果緩存並跨多個請求重用它們。

```jsx
import { getUser } from './data';
import { unstable_cache } from 'next/cache';

const getCachedUser = unstable_cache(
  async (id) => getUser(id),
  ['my-app-user']
);

export default async function Component({ userID }) {
  const user = await getCachedUser(userID);
  ...
}
```

> **值得了解**：
>
> - 在緩存作用域內存取 `headers` 或 `cookies` 等未緩存資料來源不支援。若需在緩存函式內存取這些資料，請在緩存函式外使用 `headers` 並將所需的未緩存資料作為引數傳遞。
> - 此 API 使用 Next.js 內建的緩存以跨請求和部署持久化結果。請參見 [緩存和重新驗證 (Caching and Revalidating)](/docs/app/getting-started/caching)。

## 參數

```jsx
const data = unstable_cache(fetchData, keyParts, options)()
```

- `fetchData`: 這是一個用於獲取您想緩存的數據的異步函數。它必須是一個返回 `Promise` 的函數。
- `keyParts`: 這是一個額外的金鑰陣列，進一步增加對緩存的識別。預設情況下，`unstable_cache` 已使用參數和您函數的字符串化版本作為緩存金鑰。在大多數情況下它是可選的；僅當您使用外部變量但未將其作為參數傳遞時才需要使用它。但是，如果您未將函數內部使用的閉包作為參數傳遞，則務必將其包含在內。
- `options`: 這是一個控制緩存行為的物件。它可以包含以下屬性：
  - `tags`: 可用於控制緩存無效化的標籤陣列。Next.js 不會使用它來唯一識別函數。
  - `revalidate`: 緩存應該重新驗證的秒數。省略或傳遞 `false` 以無限期緩存，直到調用匹配的 `revalidateTag()` 或 `revalidatePath()` 方法為止。

## 返回值

`unstable_cache` 回傳一個函數，當呼叫時，會返回一個 Promise，該 Promise 解析為緩存的數據。如果數據不在緩存中，提供的函數將被執行，其結果將被緩存並返回。

## 範例

```tsx filename="app/page.tsx" switcher
import { unstable_cache } from 'next/cache'

export default async function Page({
  params,
}: {
  params: Promise<{ userId: string }>
}) {
  const { userId } = await params
  const getCachedUser = unstable_cache(
    async () => {
      return { id: userId }
    },
    [userId], // add the user ID to the cache key
    {
      tags: ['users'],
      revalidate: 60,
    }
  )

  //...
}
```
