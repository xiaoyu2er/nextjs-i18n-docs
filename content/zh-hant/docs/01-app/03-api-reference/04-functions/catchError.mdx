The `unstable_catchError` 函式建立一個將子元件包裹在錯誤邊界中的組件。它提供了 [`error.js`](/docs/app/api-reference/file-conventions/error) 檔案慣用法的程式化替代方案，讓您可以在元件樹的任何位置實現元件層級的錯誤復原。

`unstable_catchError` 可從 [客戶端元件 (Client Components)](/docs/app/getting-started/server-and-client-components) 呼叫。

```tsx filename="app/custom-error-boundary.tsx" switcher
'use client'

import { unstable_catchError } from 'next/error'

function ErrorFallback(
  props: { title: string },
  { error, unstable_retry: retry }: { error: Error; unstable_retry: () => void }
) {
  return (
    <div>
      <h2>{props.title}</h2>
      <p>{error.message}</p>
      <button onClick={() => retry()}>Try again</button>
    </div>
  )
}

export default unstable_catchError(ErrorFallback)
```

```jsx filename="app/custom-error-boundary.js" switcher
'use client'

import { unstable_catchError } from 'next/error'

function ErrorFallback(props, { error, unstable_retry: retry }) {
  return (
    <div>
      <h2>{props.title}</h2>
      <p>{error.message}</p>
      <button onClick={() => retry()}>Try again</button>
    </div>
  )
}

export default unstable_catchError(ErrorFallback)
```

## 參考

### 參數

`unstable_catchError` 接受一個參數：

```ts
const ErrorWrapper = unstable_catchError(fallback)
```

#### `fallback`

當捕捉到錯誤時呈現錯誤 UI 的函式。它接收兩個參數：

- `props` — 傳遞給包裝器組件的 props（不包括 `children`）。
- `errorInfo` — 包含有關錯誤資訊的物件：

| 屬性 | 類型 | 描述 |
| --------- | ------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| `error` | [`Error`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Error) | 捕捉到的錯誤執行個體。 |
| `unstable_retry` | `() => void` | 重新擷取並重新渲染錯誤邊界的內容。如果成功，則以重新渲染的結果取代 fallback。 |
| `reset` | `() => void` | 重置錯誤狀態並重新渲染，但不重新擷取。大多數情況下請使用 [`retry()`](/docs/app/api-reference/file-conventions/error#unstable_retry)。 |

The `fallback` 函式必須是 客戶端元件 (Client Component)（或定義在 `'use client'` 模組中）。

### 回傳值

`unstable_catchError` 傳回一個 React 組件，該組件：

- 接受與 fallback 的第一個參數相同的 props，外加 `children`。
- 將 `children` 包裹在錯誤邊界中。
- 當 `children` 捕捉到錯誤時，呈現 `fallback`。

## 範例

### 客戶端組件

定義一個 fallback，並使用傳回的組件包裹 UI 的一部分：

```tsx filename="app/some-component.tsx" switcher
import ErrorWrapper from '../custom-error-boundary'

export default function Component({ children }: { children: React.ReactNode }) {
  return <ErrorWrapper title="Dashboard Error">{children}</ErrorWrapper>
}
```

```jsx filename="app/some-component.js" switcher
import ErrorWrapper from '../custom-error-boundary'

export default function Component({ children }) {
  return <ErrorWrapper title="Dashboard Error">{children}</ErrorWrapper>
}
```

### 從錯誤中復原

使用 `retry()` 提示使用者從錯誤中復原。呼叫時，函式會重新擷取並重新渲染錯誤邊界的內容。如果成功，則以重新渲染的結果取代 fallback。

大多數情況下，請使用 `retry()` 而非 `reset()`。`reset()` 函式僅清除錯誤狀態並重新渲染，但不重新擷取，這意味著它無法從 伺服器元件 (Server Component) 錯誤中復原。

```tsx filename="app/custom-error-boundary.tsx" highlight={9,12} switcher
'use client'

import { unstable_catchError } from 'next/error'

function ErrorFallback(
  props: {},
  {
    error,
    unstable_retry: retry,
    reset,
  }: { error: Error; unstable_retry: () => void; reset: () => void }
) {
  return (
    <div>
      <p>{error.message}</p>
      <button onClick={() => retry()}>Try again</button>
      <button onClick={() => reset()}>Reset</button>
    </div>
  )
}

export default unstable_catchError(ErrorFallback)
```

```jsx filename="app/custom-error-boundary.js" highlight={7,10} switcher
'use client'

import { unstable_catchError } from 'next/error'

function ErrorFallback(props, { error, unstable_retry: retry, reset }) {
  return (
    <div>
      <p>{error.message}</p>
      <button onClick={() => retry()}>Try again</button>
      <button onClick={() => reset()}>Reset</button>
    </div>
  )
}

export default unstable_catchError(ErrorFallback)
```

### 伺服器渲染 (SSR) 錯誤 fallback

您可以將伺服器渲染的內容作為 prop 傳遞以顯示資料驅動的 fallback UI。這是透過將 `伺服器元件 (Server Component)` 渲染為 `React.ReactNode` prop 來實現，當捕捉到錯誤時 fallback 會顯示它。

> **小提示**：此模式在每次頁面渲染時會急切地渲染 fallback，即使未發生錯誤。對於大多數使用案例，較簡單的客戶端 fallback 就足夠了。

```tsx filename="app/error-boundary.tsx" switcher
'use client'

import { unstable_catchError } from 'next/error'
import type { ErrorInfo } from 'next/error'

function ErrorFallback(
  props: { fallback: React.ReactNode },
  errorInfo: ErrorInfo
) {
  return props.fallback
}

export default unstable_catchError(ErrorFallback)
```

```jsx filename="app/error-boundary.js" switcher
'use client'

import { unstable_catchError } from 'next/error'

function ErrorFallback(props, errorInfo) {
  return props.fallback
}

export default unstable_catchError(ErrorFallback)
```

```tsx filename="app/some-component.tsx" switcher
import ErrorBoundary from '../error-boundary'

async function ErrorFallback() {
  const data = await getData()
  return <div>{data.message}</div>
}

export default function Component({ children }: { children: React.ReactNode }) {
  return <ErrorBoundary fallback={<ErrorFallback />}>{children}</ErrorBoundary>
}
```

```jsx filename="app/some-component.js" switcher
import ErrorBoundary from '../error-boundary'

async function ErrorFallback() {
  const data = await getData()
  return <div>{data.message}</div>
}

export default function Component({ children }) {
  return <ErrorBoundary fallback={<ErrorFallback />}>{children}</ErrorBoundary>
}
```

> **小提示**：
>
> - 與受限於 route segments 的 `error.js` 檔案慣用法不同，`unstable_catchError` 可用於包裹元件樹的任何部分以實現元件層級的錯誤復原。
> - 傳遞給包裝器組件的 props 會轉發至 fallback 函式，使得建立具有不同配置的可重復使用錯誤 UI 變得輕鬆。
> - 您不必將 `error.js` 預設匯出包裝在 `unstable_catchError` 中。[`error.js`](/docs/app/api-reference/file-conventions/error) 檔案慣用法本身已在 Next.js 提供的內建錯誤邊界中渲染。
