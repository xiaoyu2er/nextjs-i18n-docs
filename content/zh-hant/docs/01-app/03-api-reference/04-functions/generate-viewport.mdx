您可以使用靜態 `viewport` 物件或動態 `generateViewport` 函數自訂網頁的初始視口。

> **小提示**：
>
> - **viewport** 物件和 `generateViewport` 函數的匯出**僅支援用於伺服器元件 (Server Components)**。
> - 您無法從同一個路由區段匯出 `viewport` 物件和 `generateViewport` 函數。
> - 如果您正在從 `metadata` 匯出遷移，您可以使用 [metadata-to-viewport-export codemod](/docs/app/guides/upgrading/codemods#metadata-to-viewport-export) 來更新您的變更。

## `viewport` 物件

要定義視口選項，請從 `layout.jsx` 或 `page.jsx` 檔案匯出 `viewport` 物件。

```tsx filename="layout.tsx | page.tsx" switcher
import type { Viewport } from 'next'

export const viewport: Viewport = {
  themeColor: 'black',
}

export default function Page() {}
```

```jsx filename="layout.jsx | page.jsx" switcher
export const viewport = {
  themeColor: 'black',
}

export default function Page() {}
```

## `generateViewport` 函數

`generateViewport` 應傳回包含一個或多個視口欄位的 [`Viewport` 物件](#viewport-fields)。

```tsx filename="layout.tsx | page.tsx" switcher
export function generateViewport({ params }) {
  return {
    themeColor: '...',
  }
}
```

在 TypeScript 中，`params` 引數可以透過 [`PageProps<'/route'>`](/docs/app/api-reference/file-conventions/page#page-props-helper) 或 [`LayoutProps<'/route'>`](/docs/app/api-reference/file-conventions/layout#layout-props-helper) 進行類型設定，這取決於 `generateViewport` 是定義在哪裡。

```jsx filename="layout.js | page.js" switcher
export function generateViewport({ params }) {
  return {
    themeColor: '...',
  }
}
```

> **重點提示**:
>
> - 如果 viewport 不依賴請求資訊，應使用靜態 [`viewport` 物件](#the-viewport-object) 而非 `generateViewport` 來定義。

## 視口欄位

### `themeColor`

進一步了解 [`theme-color`](https://developer.mozilla.org/docs/Web/HTML/Element/meta/name/theme-color)。

**簡單主題顏色**

```tsx filename="layout.tsx | page.tsx" switcher
import type { Viewport } from 'next'

export const viewport: Viewport = {
  themeColor: 'black',
}
```

```jsx filename="layout.jsx | page.jsx" switcher
export const viewport = {
  themeColor: 'black',
}
```

```html filename="<head> output" hideLineNumbers
<meta name="theme-color" content="black" />
```

**具有 media 屬性**

```tsx filename="layout.tsx | page.tsx" switcher
import type { Viewport } from 'next'

export const viewport: Viewport = {
  themeColor: [
    { media: '(prefers-color-scheme: light)', color: 'cyan' },
    { media: '(prefers-color-scheme: dark)', color: 'black' },
  ],
}
```

```jsx filename="layout.jsx | page.jsx" switcher
export const viewport = {
  themeColor: [
    { media: '(prefers-color-scheme: light)', color: 'cyan' },
    { media: '(prefers-color-scheme: dark)', color: 'black' },
  ],
}
```

```html filename="<head> output" hideLineNumbers
<meta name="theme-color" media="(prefers-color-scheme: light)" content="cyan" />
<meta name="theme-color" media="(prefers-color-scheme: dark)" content="black" />
```

### `width`、`initialScale`、`maximumScale` 和 `userScalable`

> **小提示**：`viewport` meta 標籤會自動設定，通常不需要手動設定，因為預設值已足夠。不過，提供此資訊是為了完整性。

```tsx filename="layout.tsx | page.tsx" switcher
import type { Viewport } from 'next'

export const viewport: Viewport = {
  width: 'device-width',
  initialScale: 1,
  maximumScale: 1,
  userScalable: false,
  // Also supported but less commonly used
  // interactiveWidget: 'resizes-visual',
}
```

```jsx filename="layout.jsx | page.jsx" switcher
export const viewport = {
  width: 'device-width',
  initialScale: 1,
  maximumScale: 1,
  userScalable: false,
  // Also supported but less commonly used
  // interactiveWidget: 'resizes-visual',
}
```

```html filename="<head> output" hideLineNumbers
<meta
  name="viewport"
  content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no"
/>
```

## 搭配緩存組件 (Cache Components)

當啟用 [緩存組件 (Cache Components)](/docs/app/getting-started/caching) 時，`generateViewport` 遵循與其它組件相同的規則。如果 `viewport` 存取執行時資料（`cookies()`、`headers()`、`params`、`searchParams`）或進行未緩存資料抓取，它將延遲到請求時。

與元數據不同，`viewport` 無法串流，因為它影響初始頁面載入介面。如果 `generateViewport` 延遲到請求時，頁面將需要阻塞直到解析完成。

如果 `viewport` 依賴外部資料但不依賴執行時資料，請使用 `use cache`：

```tsx filename="app/layout.tsx" highlight={2}
export async function generateViewport() {
  'use cache'
  const { width, initialScale } = await db.query('viewport-size')
  return { width, initialScale }
}
```

如果 `viewport` 確實需要執行時資料，請將文件的 <body> 包裝在 Suspense 邊界 (Suspense boundary) 中，以表示整個路由均為動態：

```tsx filename="app/layout.tsx" highlight={1,13,17}
import { Suspense } from 'react'
import { cookies } from 'next/headers'

export async function generateViewport() {
  const cookieJar = await cookies()
  return {
    themeColor: cookieJar.get('theme-color')?.value,
  }
}

export default function RootLayout({ children }) {
  return (
    <Suspense>
      <html>
        <body>{children}</body>
      </html>
    </Suspense>
  )
}
```

緩存是可取的，因為它允許生成靜態外殼。將文件的 `body` 包裝在 `Suspense` 中表示，當請求到達時，沒有靜態外殼或內容可立即發送，這使得整個路由在每次請求時都要阻塞直到就緒。

> **重點提示**：請使用 [多個根佈局 (multiple root layouts)](/docs/app/api-reference/file-conventions/layout#root-layout) 將完全動態的 `viewport` 限制在特定路由，同時仍讓應用程式中的其他路由生成靜態外殼。

## 類型

您可以使用 `Viewport` 類型來為您的視口物件增加類型安全性。如果您在 IDE 中使用 [內建的 TypeScript 插件](/docs/app/api-reference/config/typescript)，您不需要手動新增類型，但如果您願意，仍然可以明確地新增它。

### `viewport` 物件

```tsx
import type { Viewport } from 'next'

export const viewport: Viewport = {
  themeColor: 'black',
}
```

### `generateViewport` 函數

#### 一般函數

```tsx
import type { Viewport } from 'next'

export function generateViewport(): Viewport {
  return {
    themeColor: 'black',
  }
}
```

#### 具有區段 props

```tsx
import type { Viewport } from 'next'

type Props = {
  params: Promise<{ id: string }>
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>
}

export function generateViewport({ params, searchParams }: Props): Viewport {
  return {
    themeColor: 'black',
  }
}

export default function Page({ params, searchParams }: Props) {}
```

#### JavaScript 專案

對於 JavaScript 專案，您可以使用 JSDoc 來增加類型安全性。

```js
/** @type {import("next").Viewport} */
export const viewport = {
  themeColor: 'black',
}
```
