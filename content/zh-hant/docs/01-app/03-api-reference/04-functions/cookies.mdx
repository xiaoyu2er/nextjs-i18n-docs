`cookies` 是一個 **async** 函數，允許您在 [服務器組件 (Server Components)](/docs/app/getting-started/server-and-client-components) 中讀取 HTTP 請求中的 Cookie，並在 [伺服器函數 (Server Functions)](/docs/app/getting-started/mutating-data) 或 [路由處理器 (Route Handlers)](/docs/app/api-reference/file-conventions/route) 中讀取或寫入發出的請求 Cookie。

```tsx filename="app/page.tsx" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

```js filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

## 參考

### 方法

以下可用方法：

| 方法 | 返回類型 | 描述 |
| ------------ | ------------ | -------------------------------------------------------------------------------------- |
| `get('name')` | Object | 接受 Cookie 名稱，並返回包含名稱和值的對象。 |
| `getAll()` | Array of objects | 返回所有匹配名稱的 Cookie 列表。 |
| `has('name')` | Boolean | 接受 Cookie 名稱，並根據 Cookie 是否存在返回布林值。 |
| `set(name, value, options)` | - | 接受 Cookie 名稱、值和選項，並設置發出的請求 Cookie。 |
| `delete(name)` | - | 接受 Cookie 名稱並刪除該 Cookie。 |
| `toString()` | String | 返回 Cookie 的字符串表示。 |

### 選項

設定 cookie 時，支援 `options` 物件中的以下屬性：

| 選項        | 類型                                   | 描述                                                                        |
| ------------- | -------------------------------------- | ---------------------------------------------------------------------------------- |
| `name`        | String                                 | 指定 cookie 的名稱。                                                  |
| `value`       | String                                 | 指定要儲存在 cookie 中的值。                                    |
| `expires`     | Date                                   | 定義 cookie 過期的確切日期。                                |
| `maxAge`      | Number                                 | 設定 cookie 的生命週期（以秒為單位）。                                             |
| `domain`      | String                                 | 指定 cookie 可用的網域。                                |
| `path`        | String, 預設：'/'                 | 將 cookie 的範圍限制在網域內的特定路徑。                    |
| `secure`      | Boolean                                | 確保 cookie 僅透過 HTTPS 連線發送以增強安全性。         |
| `httpOnly`    | Boolean                                | 將 cookie 限制為 HTTP 請求，防止客戶端存取。              |
| `sameSite`    | Boolean, 'lax', 'strict', 'none' | 控制 cookie 的跨站請求行為。                                 |
| `priority`    | String ("low", "medium", "high") | 指定 cookie 的優先級                                                    |
| `partitioned` | Boolean                                | 指示 cookie 是否為 [分割](https://github.com/privacycg/CHIPS)。 |

唯一具有預設值的選項是 `path`。

要了解這些選項的更多資訊，請參閱 [MDN 文件](https://developer.mozilla.org/en-US/docs/Web/HTTP/Cookies)。

## 小知識

- `cookies` 是一個 **異步 (async)** 函數，會返回一個 Promise。您必須使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 函數來訪問 Cookie。
  - 在版本 14 及更早版本中，`cookies` 是一個同步函數。為了幫助向後相容，您在 Next.js 15 中仍可同步訪問它，但此行為未來將被棄用。
- `cookies` 是一個 [請求時 API](/docs/app/glossary#request-time-apis)，其返回的 Value 無法預先得知。在佈局或頁面中使用它會選擇進入動態渲染的 Route，即啟用 [動態渲染](/docs/app/glossary#dynamic-rendering)。
- `.delete` 方法僅限於在以下情況調用：
  - 在 [伺服器函數 (Server Functions)](/docs/app/getting-started/mutating-data) 或 [路由處理器 (Route Handlers)](/docs/app/api-reference/file-conventions/route) 中。
  - 如果它屬於與 `.set` 呼叫相同的 Domain。對於通配符域，特定子網域必須是精確匹配。此外，程式碼必須在與要刪除的 Cookie 相同的協議 (HTTP 或 HTTPS) 上執行。
- HTTP 不允許在串流開始後設置 Cookie，因此您必須在 [伺服器函數 (Server Functions)](/docs/app/getting-started/mutating-data) 或 [路由處理器 (Route Handlers)](/docs/app/api-reference/file-conventions/route) 中使用 `.set`。

## 理解 Server Components 中的 Cookie 行為

當在 Server Components 中使用 cookie 時，了解 cookie 本質上是一種客戶端儲存機制非常重要：

- **讀取 Cookie** 在 Server 組件中運作，因為您正在存取客戶端瀏覽器在 HTTP 請求標頭中發送的 Cookie 數據。
- **設置 Cookie** 在 Server 組件渲染期間不支援。要修改 Cookie，請從客戶端調用 Server 函數或使用路由處理器。

服務器只能發送指令（通過 `Set-Cookie` 標頭）來通知瀏覽器存儲 Cookie — 實際存儲發生在客戶端。這就是為什麼修改狀態的 Cookie 操作（`.set`、`.delete`）必須在響應標頭可以正確設置的 Server 函數或路由處理器中執行。

## 理解 Server 函數 (Server Functions) 中的 Cookie 行為

在 Server 函數中設置或刪除 Cookie 之後，當函數用作 [Server Action (Server Actions)](/docs/app/getting-started/mutating-data#what-are-server-functions)（例如傳遞給表單的 `action` prop）時，Next.js 可以在單一服務器往返中返回更新的 UI 和新數據。參見 [緩存與重新驗證 (Caching and Revalidating)](/docs/app/getting-started/caching)。

UI 不會解除掛載，但依賴來自伺服器資料的效果將會重新執行。

若要刷新緩存的數據，請在函數內部調用 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 或 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag)。

## 範例

### 取得 cookie

您可以使用 `(await cookies()).get('name')` 方法來取得單個 cookie：

```tsx filename="app/page.tsx" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

```jsx filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const theme = cookieStore.get('theme')
  return '...'
}
```

### 取得所有 cookie

您可以使用 `(await cookies()).getAll()` 方法來取得所有名稱匹配的 cookie。如果未指定 `name`，則返回所有可用的 cookie。

```tsx filename="app/page.tsx" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  return cookieStore.getAll().map((cookie) => (
    <div key={cookie.name}>
      <p>Name: {cookie.name}</p>
      <p>Value: {cookie.value}</p>
    </div>
  ))
}
```

```jsx filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  return cookieStore.getAll().map((cookie) => (
    <div key={cookie.name}>
      <p>Name: {cookie.name}</p>
      <p>Value: {cookie.value}</p>
    </div>
  ))
}
```

### 設定 cookie

您可以在 [Server 函數 (Server Functions)](/docs/app/getting-started/mutating-data) 或 [路由處理器 (Route Handlers)](/docs/app/api-reference/file-conventions/route) 中使用 `(await cookies()).set(name, value, options)` 方法來設置 Cookie。[`options` 對象](#options) 是可選的。

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function create(data) {
  const cookieStore = await cookies()

  cookieStore.set('name', 'lee')
  // or
  cookieStore.set('name', 'lee', { secure: true })
  // or
  cookieStore.set({
    name: 'name',
    value: 'lee',
    httpOnly: true,
    path: '/',
  })
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function create(data) {
  const cookieStore = await cookies()

  cookieStore.set('name', 'lee')
  // or
  cookieStore.set('name', 'lee', { secure: true })
  // or
  cookieStore.set({
    name: 'name',
    value: 'lee',
    httpOnly: true,
    path: '/',
  })
}
```

### 檢查 cookie 是否存在

您可以使用 `(await cookies()).has(name)` 方法檢查是否存在 cookie：

```tsx filename="app/page.ts" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const hasCookie = cookieStore.has('theme')
  return '...'
}
```

```jsx filename="app/page.js" switcher
import { cookies } from 'next/headers'

export default async function Page() {
  const cookieStore = await cookies()
  const hasCookie = cookieStore.has('theme')
  return '...'
}
```

### 刪除 Cookie

有三種方法可以刪除 cookie。

使用 `delete()` 方法：

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.delete('name')
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.delete('name')
}
```

設定具有相同名稱和空值的新 cookie：

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.set('name', '')
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.set('name', '')
}
```

將 `maxAge` 設定為 0 將立即使 cookie 過期。`maxAge` 接受以秒為單位的值。

```tsx filename="app/actions.ts" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.set('name', 'value', { maxAge: 0 })
}
```

```js filename="app/actions.js" switcher
'use server'

import { cookies } from 'next/headers'

export async function deleteCookie(data) {
  const cookieStore = await cookies()
  cookieStore.set('name', 'value', { maxAge: 0 })
}
```
