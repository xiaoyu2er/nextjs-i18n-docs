---

`generateStaticParams` 函數可與 [動態路徑段 (dynamic route segments)](/docs/app/api-reference/file-conventions/dynamic-routes) 搭配使用，在構建時 [**靜態生成 (statically generate)**](/docs/app/glossary#prerendering) 路由，而非請求時按需生成。

`generateStaticParams` 可用於：

- [頁面 (Pages)](/docs/app/api-reference/file-conventions/page) (`page.tsx`/`page.js`)
- [佈局 (Layouts)](/docs/app/api-reference/file-conventions/layout) (`layout.tsx`/`layout.js`)
- [路徑處理器 (Route Handlers)](/docs/app/api-reference/file-conventions/route) (`route.ts`/`route.js`)

```tsx filename="app/blog/[slug]/page.tsx" switcher
// Return a list of `params` to populate the [slug] dynamic segment
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// Multiple versions of this page will be statically generated
// using the `params` returned by `generateStaticParams`
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  // ...
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
// Return a list of `params` to populate the [slug] dynamic segment
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}

// Multiple versions of this page will be statically generated
// using the `params` returned by `generateStaticParams`
export default async function Page({ params }) {
  const { slug } = await params
  // ...
}
```

> **小知識**：
>
> - 您可以使用 [`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config/dynamicParams) 路徑段配置選項，來控制當訪未由 `generateStaticParams` 生成的動態片段時會發生什麼。
> - 若要重新驗證 (ISR) [運行時 (runtime) 的路徑 (paths)](#all-paths-at-runtime)，您必須從 `generateStaticParams` 返回 [空數組 (an empty array)](#all-paths-at-build-time)，或使用 [`export const dynamic = 'force-static'`](/docs/app/guides/caching-without-cache-components#dynamic)。
> - 在 `next dev` 期間，當您導航到路由時會呼叫 `generateStaticParams`。
> - 在 `next build` 期間，`generateStaticParams` 會在生成對應的 Layouts 或 Pages 之前執行。
> - 在重新驗證 (ISR) 期間，不會再次呼叫 `generateStaticParams`。
> - `generateStaticParams` 取代了頁面路由器 (Pages Router) 中的 [`getStaticPaths`](/docs/pages/api-reference/functions/get-static-paths) 函數。

## 參數

`options.params` (可選)

如果路由中的多個動態部分使用 `generateStaticParams`，則子 `generateStaticParams` 函數會為父方產生的每個 `params` 集合執行一次。

`params` 物件包含來自父方 `generateStaticParams` 的已填充 `params`，可用來 [在子部分中生成 `params`](#multiple-dynamic-segments-in-a-route)。

## 返回值

`generateStaticParams` 應返回物件陣列，其中每個物件代表單一路由的已填充動態部分。

- 物件中的每個屬性都是要填充的路由動態部分。
- 屬性的名稱是部分名稱，屬性的值是用來填充該部分的內容。

| 範例路由 | `generateStaticParams` 返回類型 |
| -------------------------------- | ----------------------------------------- |
| `/product/[id]` | `{ id: string }[]` |
| `/products/[category]/[product]` | `{ category: string, product: string }[]` |
| `/products/[...slug]` | `{ slug: string[] }[]` |

## 範例

### 預渲染

#### 所有路徑均在建構時

若要於建構時靜態渲染所有路徑，請提供完整的 `generateStaticParams` 路徑列表：

```tsx filename="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  return posts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### 建構時部分路徑

若要於建構時靜態渲染部分路徑，並在執行時於首次訪問時渲染其餘路徑，請返回部分路徑列表：

```tsx filename="app/blog/[slug]/page.tsx" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // Render the first 10 posts at build time
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())

  // Render the first 10 posts at build time
  return posts.slice(0, 10).map((post) => ({
    slug: post.slug,
  }))
}
```

然後，透過使用 [`dynamicParams`](/docs/app/api-reference/file-conventions/route-segment-config/dynamicParams) 路徑段配置選項，您可以控制在未由 `generateStaticParams` 生成的動態片段被造訪時會發生什麼。

```tsx filename="app/blog/[slug]/page.tsx" switcher
// All posts besides the top 10 will be a 404
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

```jsx filename="app/blog/[slug]/page.js" switcher
// All posts besides the top 10 will be a 404
export const dynamicParams = false

export async function generateStaticParams() {
  const posts = await fetch('https://.../posts').then((res) => res.json())
  const topPosts = posts.slice(0, 10)

  return topPosts.map((post) => ({
    slug: post.slug,
  }))
}
```

#### 所有路徑均在執行時

若要首次造訪時靜態渲染所有路徑，請返回空數組 (構建時不會渲染任何路徑)，或使用 [`export const dynamic = 'force-static'`](/docs/app/guides/caching-without-cache-components#dynamic)：

```jsx filename="app/blog/[slug]/page.js"
export async function generateStaticParams() {
  return []
}
```

> **小知識**：
>
> - 您必須始終從 `generateStaticParams` 返回數組，即使它是空的。否則，路由將以動態方式渲染。

```jsx filename="app/changelog/[slug]/page.js"
export const dynamic = 'force-static'
```

#### 搭配快取元件 (Cache Components)

當搭配動態路由使用 [緩存組件 (Cache Components)](/docs/app/getting-started/caching) 時，`generateStaticParams` 必須返回 **至少一個參數 (at least one param)**。空數組會導致 [構建錯誤 (build error)](/docs/messages/empty-generate-static-params)。這讓緩存組件能驗證您的路由未錯誤地在運行時訪問 `cookies()`、`headers()` 或 `searchParams`。

> **小知識**：如果您在構建時不知道實際的參數值，您可以返回一個佔位參數（例如 `[{ slug: '__placeholder__' }]`）以進行驗證，然後在頁面中使用 `notFound()` 處理它。然而，這會阻止構建時驗證有效工作，並可能導致運行時錯誤。

詳情請參閱 [動態路由章節 (dynamic routes section)](/docs/app/api-reference/file-conventions/dynamic-routes#with-cache-components)。

### 配合路徑處理器 (Route Handlers)

您可以將 `generateStaticParams` 與 [路徑處理器 (Route Handlers)](/docs/app/api-reference/file-conventions/route) 搭配使用，在構建時靜態生成 API 響應：

```ts filename="app/api/posts/[id]/route.ts" switcher
export async function generateStaticParams() {
  return [{ id: '1' }, { id: '2' }, { id: '3' }]
}

export async function GET(
  request: Request,
  { params }: RouteContext<'/api/posts/[id]'>
) {
  const { id } = await params
  // This will be statically generated for IDs 1, 2, and 3
  return Response.json({ id, title: `Post ${id}` })
}
```

```js filename="app/api/posts/[id]/route.js" switcher
export async function generateStaticParams() {
  return [{ id: '1' }, { id: '2' }, { id: '3' }]
}

export async function GET(request, { params }) {
  const { id } = await params
  // This will be statically generated for IDs 1, 2, and 3
  return Response.json({ id, title: `Post ${id}` })
}
```

### 搭配緩存組件的 Route Handlers (Route Handlers with Cache Components)

當使用 [緩存組件 (Cache Components)](/docs/app/getting-started/caching) 時，請搭配 `use cache` 指令以實現最佳緩存：

```ts filename="app/api/posts/[id]/route.ts"
export async function generateStaticParams() {
  return [{ id: '1' }, { id: '2' }, { id: '3' }]
}

async function getPost(id: Promise<string>) {
  'use cache'
  const resolvedId = await id
  const response = await fetch(`https://api.example.com/posts/${resolvedId}`)
  return response.json()
}

export async function GET(
  request: Request,
  { params }: RouteContext<'/api/posts/[id]'>
) {
  const post = await getPost(params.then((p) => p.id))
  return Response.json(post)
}
```

更多詳情請參閱 [路徑處理器 (Route Handlers) 文件](/docs/app/api-reference/file-conventions/route#static-generation-with-generatestaticparams)。

### 未指定路徑不進行渲染

若要防止未指定的路徑在運行時進行預渲染，請在路徑段中添加 `export const dynamicParams = false` 選項。使用此配置選項時，僅會提供由 `generateStaticParams` 提供的路徑，未指定的路由將返回 404 或匹配（在 [全部路徑 (catch-all) 路由](/docs/app/api-reference/file-conventions/dynamic-routes#catch-all-segments) 的情況下）。

### 路由中的多個動態部分

您可以為當前佈局或頁面上方的動態部分生成參數，但 **不可以**為下方生成。例如，給定 `app/products/[category]/[product]` 路由：

- `app/products/[category]/[product]/page.js` 可以為 **[兩個]** `[category]` 和 `[product]` 生成參數。
- `app/products/[category]/layout.js` **僅**能為 `[category]` 生成參數。

為具有多個動態部分的路由生成參數有兩種方法：

#### 由下而上產生參數

由子路由部分生成多個動態部分。

```tsx filename="app/products/[category]/[product]/page.tsx" switcher
// Generate segments for both [category] and [product]
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

```jsx filename="app/products/[category]/[product]/page.js" switcher
// Generate segments for both [category] and [product]
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

#### 由上而下產生參數

先生成父部分，然後使用結果生成子部分。

```tsx filename="app/products/[category]/layout.tsx" switcher
// Generate segments for [category]
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({
  params,
}: {
  params: Promise<{ category: string }>
}) {
  // ...
}
```

```jsx filename="app/products/[category]/layout.js" switcher
// Generate segments for [category]
export async function generateStaticParams() {
  const products = await fetch('https://.../products').then((res) => res.json())

  return products.map((product) => ({
    category: product.category.slug,
  }))
}

export default function Layout({ params }) {
  // ...
}
```

子路由部分的 `generateStaticParams` 函數會為父 `generateStaticParams` 生成的每個部分執行一次。

子 `generateStaticParams` 函數可以使用父 `generateStaticParams` 函數返回的 `params` 來動態生成其自己的部分。

```tsx filename="app/products/[category]/[product]/page.tsx" switcher
// Generate segments for [product] using the `params` passed from
// the parent segment's `generateStaticParams` function
export async function generateStaticParams({
  params: { category },
}: {
  params: { category: string }
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({
  params,
}: {
  params: Promise<{ category: string; product: string }>
}) {
  // ...
}
```

```jsx filename="app/products/[category]/[product]/page.js" switcher
// Generate segments for [product] using the `params` passed from
// the parent segment's `generateStaticParams` function
export async function generateStaticParams({ params: { category } }) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}

export default function Page({ params }) {
  // ...
}
```

注意參數引數可以同步存取，且僅包含父部分的 `params`。

為了類型補全，您可以搭配 [`Page Props helper`](/docs/app/api-reference/file-conventions/page#page-props-helper) 或 [`Layout Props helper`](/docs/app/api-reference/file-conventions/layout#layout-props-helper) 使用 TypeScript 的 `Awaited` 協助工具：

```ts filename="app/products/[category]/[product]/page.tsx" switcher
export async function generateStaticParams({
  params: { category },
}: {
  params: Awaited<LayoutProps<'/products/[category]'>['params']>
}) {
  const products = await fetch(
    `https://.../products?category=${category}`
  ).then((res) => res.json())

  return products.map((product) => ({
    product: product.id,
  }))
}
```

> **小知識**：`fetch` 請求會針對相同數據在所有 `generate`-前綴的函數、佈局、頁面和服務器組件之間自動 [記憶化 (memoized)](/docs/app/glossary#memoization)。如果 `fetch` 不可用，則可使用 React [`cache`](https://react.dev/reference/react/cache)。
