The `useLinkStatus` 鉤子可讓您追蹤 `<Link>` 元件的**執行中**狀態。請使用它來提供微妙的內聯反饋，例如在點擊連結時，在連結上方顯示閃爍效果，同時完成導航。推薦使用 `loading.js` 提供路由層級的備援方案，並使用預取功能實現即時轉換。

`useLinkStatus` 在有以下情況時非常有用：

- 停用或進行中的 [Prefetching (預先加載)](/docs/app/getting-started/linking-and-navigating#prefetching)，這表示導航已鎖定。
- 目標路由是動態的 **且** 不包含 [`loading.js`](/docs/app/api-reference/file-conventions/loading) 檔案，原本允許即時導航。

```tsx filename="app/hint.tsx" switcher
'use client'

import Link from 'next/link'
import { useLinkStatus } from 'next/link'

function Hint() {
  const { pending } = useLinkStatus()
  return (
    <span aria-hidden className={`link-hint ${pending ? 'is-pending' : ''}`} />
  )
}

export default function Header() {
  return (
    <header>
      <Link href="/dashboard" prefetch={false}>
        <span className="label">Dashboard</span> <Hint />
      </Link>
    </header>
  )
}
```

> **小提示**：
>
> - `useLinkStatus` 必須在 `Link` 元件的子元件中使用
> - 當 `Link` 元件設定 `prefetch={false}` 時，此鉤子最有用的
> - 如果已預取連結的路由，則會跳過執行中狀態
> - 當連續快速點擊多個連結時，僅顯示最後一個連結的執行中狀態
> - 此鉤子不支援 Pages Router，且一律回傳 { pending: false }
> - 內聯指示器容易引入版面偏移。預備使用固定大小、持續渲染的提示元素並切換其透明度，或使用動畫。

## 您可能不需要使用 `useLinkStatus`

在新增內聯反饋之前，請考慮是否：

- 目標是靜態的且在生產環境中已預取，因此可能會跳過執行中階段。
- 該路由擁有 `loading.js` 檔案，啟用路由層級備援方案可實現即時轉換。

導航通常很快。當您識別出轉換延遲時，請將 `useLinkStatus` 作為快速補丁來使用，然後透過預取功能或 `loading.js` 備援方案來修正根本原因。

## 參數

```tsx
const { pending } = useLinkStatus()
```

`useLinkStatus` 不需要任何參數。

## 返回值

`useLinkStatus` 傳回一個包含單一屬性的物件：

| 屬性 | 類型 | 描述 |
| -------- | ------- | -------------------------------------------- |
| pending | 布林值 | history 更新前為 `true`，更新後為 `false` |

## 範例

### 內聯連結提示

新增微妙的固定大小提示，不會影響版面，以便在預取尚未完成時確認點擊。

```tsx filename="app/components/loading-indicator.tsx" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return (
    <span aria-hidden className={`link-hint ${pending ? 'is-pending' : ''}`} />
  )
}
```

```jsx filename="app/components/loading-indicator.js" switcher
'use client'

import { useLinkStatus } from 'next/link'

export default function LoadingIndicator() {
  const { pending } = useLinkStatus()
  return (
    <span aria-hidden className={`link-hint ${pending ? 'is-pending' : ''}`} />
  )
}
```

```tsx filename="app/shop/layout.tsx" switcher
import Link from 'next/link'
import LoadingIndicator from './components/loading-indicator'

const links = [
  { href: '/shop/electronics', label: 'Electronics' },
  { href: '/shop/clothing', label: 'Clothing' },
  { href: '/shop/books', label: 'Books' },
]

function Menubar() {
  return (
    <div>
      {links.map((link) => (
        <Link key={link.label} href={link.href}>
          <span className="label">{link.label}</span> <LoadingIndicator />
        </Link>
      ))}
    </div>
  )
}

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      <Menubar />
      {children}
    </div>
  )
}
```

## 優雅地處理快速導航

如果前往新路由的導航很快，使用者可能會看到不必要的提示閃現。其中一種改善使用者體驗的方式是僅在導航耗時完成時才顯示提示，方法是在初始動畫中加入延遲（例如 100ms）並在動畫開始時使其不可見（例如 `opacity: 0`）。

```css filename="app/styles/global.css"
.link-hint {
  display: inline-block;
  width: 0.6em;
  height: 0.6em;
  margin-left: 0.25rem;
  border-radius: 9999px;
  background: currentColor;
  opacity: 0;
  visibility: hidden; /* reserve space without showing the hint */
}

.link-hint.is-pending {
  /* Animation 1: fade in after 100ms and keep final opacity */
  /* Animation 2: subtle pulsing while pending */
  visibility: visible;
  animation-name: fadeIn, pulse;
  animation-duration: 200ms, 1s;
  /* Appear only if navigation actually takes time */
  animation-delay: 100ms, 100ms;
  animation-timing-function: ease, ease-in-out;
  animation-iteration-count: 1, infinite;
  animation-fill-mode: forwards, none;
}

@keyframes fadeIn {
  to {
    opacity: 0.35;
  }
}
@keyframes pulse {
  50% {
    opacity: 0.15;
  }
}
```
