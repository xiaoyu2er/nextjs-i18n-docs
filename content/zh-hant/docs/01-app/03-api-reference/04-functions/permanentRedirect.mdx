永久導向 `permanentRedirect` 函數允許您將使用者重新導向至另一個 URL。`permanentRedirect` 可在伺服器組件 (Server Components)、客戶端組件 (Client Components)、[路由處理程序 (Route Handlers)](/docs/app/api-reference/file-conventions/route) 和 [伺服器函式 (Server Functions)](/docs/app/getting-started/mutating-data) 中使用。

當在串流上下文中使用時，這會在客戶端插入 meta 標籤以發出重導向回應。當在伺服器動作中使用時，它將向調用者提供 303 HTTP 重導向回應。否則，它將向調用者提供 308 (永久) HTTP 重導向回應。

如果資源不存在，您可以改用 notFound 函式。

> **小知識**：如果您希望返回 307 (暫時) HTTP 重導向而不是 308 (永久)，則可以改用 redirect 函式。

## 參數

permanentRedirect 函式接受兩個參數：

```js
permanentRedirect(path, type)
```

| 參數 | 類型 | 描述 |
| --------- | ------------------------------------------------------------- | ----------------------------------------------------------- |
| `path` | `string` | 要重新導向的 URL。可以是相對或絕對路徑。 |
| `type` | `'replace'` (預設) 或 `'push'` (伺服器動作預設) | 要執行的重導向類型。 |

預設情況下，`permanentRedirect` 在 [伺服器操作 (Server Actions)](/docs/app/getting-started/mutating-data) 中將使用 `push`（即在瀏覽器歷史記錄堆疊中新增一筆新項目），而在其他地方則使用 `replace`（即在瀏覽器歷史記錄堆疊中取代當前 URL）。您可透過指定 `type` 參數來覆寫此行為。

「RedirectType」物件包含 `type` 參數的可用選項。

```ts
import { permanentRedirect, RedirectType } from 'next/navigation'

permanentRedirect('/redirect-to', RedirectType.replace)
// or
permanentRedirect('/redirect-to', RedirectType.push)
```

`type` 參數在伺服器組件中使用時無效。

## 返回值

`permanentRedirect` 不返回值。

## 範例

調用 `permanentRedirect()` 函式會引發 `NEXT_REDIRECT` 錯誤，並終止其被拋出的路由區塊的渲染。

```jsx filename="app/team/[id]/page.js"
import { permanentRedirect } from 'next/navigation'

async function fetchTeam(id) {
  const res = await fetch('https://...')
  if (!res.ok) return undefined
  return res.json()
}

export default async function Profile({ params }) {
  const { id } = await params
  const team = await fetchTeam(id)
  if (!team) {
    permanentRedirect('/login')
  }

  // ...
}
```

> **小知識**：由於 `permanentRedirect` 使用 TypeScript 的 [`never`](https://www.typescriptlang.org/docs/handbook/2/functions.html#never) 類型，因此不需要使用 `return permanentRedirect()`。
