`usePathname` 是一個 **Client Component** hook，讓您可以讀取目前 URL 的 **pathname**。

> **提示**：當 [`cacheComponents`](/docs/app/api-reference/config/next-config-js/cacheComponents) 啟用時，如果您的路由包含動態參數，`usePathname` 可能需要在其周圍包裹 `Suspense` 邊界。如果您使用 `generateStaticParams`，則 `Suspense` 邊界為可選。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'

export default function ExampleClientComponent() {
  const pathname = usePathname()
  return <p>Current pathname: {pathname}</p>
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { usePathname } from 'next/navigation'

export default function ExampleClientComponent() {
  const pathname = usePathname()
  return <p>Current pathname: {pathname}</p>
}
```

`usePathname` 刻意要求使用 [Client Component](/docs/app/getting-started/server-and-client-components)。值得注意的是，Client Components 並非去最佳化。它們是 [Server Components](/docs/app/getting-started/server-and-client-components) 架構中不可或缺的一部分。

例如，帶有 `usePathname` 的 Client Component 會在初次頁面載入時被渲染為 HTML。當導航至新路由時，此組件不需要重新獲取。取而代之，此組件僅下載一次（在客戶端 JavaScript 封包中），並根據當前狀態重新渲染。

> **提示**：
>
> - 從 [伺服器組件 (Server Component)](/docs/app/getting-started/server-and-client-components) 讀取當前 URL 不獲支援。此設計是有意為之，以確保頁面導航時保持佈局狀態。
> - 如果您的頁面正在進行靜態預渲染，且您的應用程式在 `next.config` 中具有 [重寫 (rewrites)](/docs/app/api-reference/config/next-config-js/rewrites) 或 [代理服務 (Proxy)](/docs/app/api-reference/file-conventions/proxy) 文件，使用 `usePathname()` 讀取 pathname 可能會導致 hydration mismatch (水合不匹配) 錯誤——因為初始值來自伺服器，並且在路由後可能不匹配實際的瀏覽器 pathname。請參閱我們的 [範例](#avoid-hydration-mismatch-with-rewrites) 以了解如何減輕此問題。

<details>

<summary>與 Pages Router (頁面路由) 的相容性</summary>

如果您有使用 `usePathname` 的組件，並且它們被匯入到 Pages Router (頁面路由) 中的路由內，請注意，如果路由尚未初始化，`usePathname` 可能會返回 null。這可能發生在任何情況下，例如 [回退路由 (fallback routes)](/docs/pages/api-reference/functions/get-static-paths#fallback-true) 或在 Pages Router 中的 [自動靜態優化 (Automatic Static Optimization)](https://nextjs.org/docs/pages/building-your-application/rendering/static#automatic-static-optimization) 期間。

為了增強路由系統之間的相容性，如果您的專案同時包含 `app` 和 `pages` 目錄，Next.js 將自動調整 `usePathname` 的返回類型。

</details>

## 參數

```tsx
const pathname = usePathname()
```

`usePathname` 不接收任何參數。

## 返回值

`usePathname` 返回當前 URL 的 pathname 字串。例如：

| URL                 | 傳回的值        |
| ------------------- | --------------------- |
| `/`                 | `'/'`                 |
| `/dashboard`        | `'/dashboard'`        |
| `/dashboard?v=2`    | `'/dashboard'`        |
| `/blog/hello-world` | `'/blog/hello-world'` |

## 範例

### 針對路由變更執行操作

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

function ExampleClientComponent() {
  const pathname = usePathname()
  const searchParams = useSearchParams()
  useEffect(() => {
    // Do something here...
  }, [pathname, searchParams])
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

function ExampleClientComponent() {
  const pathname = usePathname()
  const searchParams = useSearchParams()
  useEffect(() => {
    // Do something here...
  }, [pathname, searchParams])
}
```

### 使用重寫時避免 hydration 不匹配

當頁面被預渲染時，HTML 是為來源 pathname 生成的。如果隨後通過使用 `next.config` 或 `Proxy` 的重寫來訪問該頁面，瀏覽器 URL 可能不同，並且 `usePathname()` 將在客戶端讀取重寫後的 pathname。

為了避免 hydration 不匹配，設計 UI 時應確保只有少量、隔離的部分依賴客戶端 pathname。在伺服器渲染時呈現穩定的 fallback，並在掛載後更新該部分。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useEffect, useState } from 'react'
import { usePathname } from 'next/navigation'

export default function PathnameBadge() {
  const pathname = usePathname()
  const [clientPathname, setClientPathname] = useState('')

  useEffect(() => {
    setClientPathname(pathname)
  }, [pathname])

  return (
    <p>
      Current pathname: <span>{clientPathname}</span>
    </p>
  )
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useEffect, useState } from 'react'
import { usePathname } from 'next/navigation'

export default function PathnameBadge() {
  const pathname = usePathname()
  const [clientPathname, setClientPathname] = useState('')

  useEffect(() => {
    setClientPathname(pathname)
  }, [pathname])

  return (
    <p>
      Current pathname: <span>{clientPathname}</span>
    </p>
  )
}
```
