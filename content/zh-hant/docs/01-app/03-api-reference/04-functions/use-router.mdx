The `useRouter` hook allows you to programmatically change routes inside [Client Components](/docs/app/getting-started/server-and-client-components).

> **Recommendation:** Use the [`<Link>` component](/docs/app/api-reference/components/link) for navigation unless you have a specific requirement for using `useRouter`.

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button type="button" onClick={() => router.push('/dashboard')}>
      Dashboard
    </button>
  )
}
```

```jsx filename="app/example-client-component.js" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button type="button" onClick={() => router.push('/dashboard')}>
      Dashboard
    </button>
  )
}
```

## `useRouter()`

- `router.push(href: string, { scroll: boolean, transitionTypes: string[] })`: 執行至所提供路由的客戶端導航。將新條目添加至 [瀏覽器歷史堆疊](https://developer.mozilla.org/docs/Web/API/History_API)。可選的 `transitionTypes` 會在導航 過渡 (Transition) 期間傳遞至 [`React.addTransitionType`](https://react.dev/reference/react/addTransitionType)。
- `router.replace(href: string, { scroll: boolean, transitionTypes: string[] })`: 執行至所提供路由的客戶端導航，不將新條目添加至瀏覽器歷史堆疊。可選的 `transitionTypes` 會在導航 過渡 (Transition) 期間傳遞至 [`React.addTransitionType`](https://react.dev/reference/react/addTransitionType)。
- `router.refresh()`: 刷新當前路由。向服務器發送新請求、重新獲取數據請求並重新渲染服務器組件。客戶端將合併更新的 React Server Component 載荷，而不會丟棄受影響的客戶端 React (例如 `useState`) 或瀏覽器狀態 (例如 捲動位置)。這會清空 [客戶端緩存](/docs/app/glossary#client-cache) 當前路由，但 並 **不** 會使服務器端緩存失效。使用 [`revalidatePath`](/docs/app/api-reference/functions/revalidatePath) 或 [`revalidateTag`](/docs/app/api-reference/functions/revalidateTag) 使服務器端緩存數據失效。
- `router.prefetch(href: string, options?: { onInvalidate?: () => void })`: [預緩存](/docs/app/getting-started/linking-and-navigating#prefetching) 所提供路由以實現更快的客戶端過渡。可選的 `onInvalidate` 回調會在 [預緩存數據過期時](/docs/app/guides/prefetching#extending-or-ejecting-link) 被調用。
- `router.back()`: 導航至瀏覽器歷史堆疊中的上一個路由。
- `router.forward()`: 導航至瀏覽器歷史堆疊中的下一頁。

> **Good to know**：
>
> - 您不應向 `router.push` 或 `router.replace` 發送不受信任或未清理的 URL，此行為可能使您的網站面臨跨網站腳本 (XSS) 漏洞。例如，發送至 `router.push` 或 `router.replace` 的 `javascript:` URL 將在您的頁面上下文中執行。
> - `<Link>` 組件在進入視口時自動預緩存路由。
> - `refresh()` 可能復現相同結果，如果 `fetch` 請求被緩存。其他請求時機 API 如 `cookies` 和 `headers` 也可能改變回應。
> - `onInvalidate` 回調在每次預緩存請求中最多調用一次。它會通知您是否可能需要觸發新路由數據的預緩存。

### 從 `next/router` 遷移

- 當使用 App Router 時，`useRouter` hook 應從 `next/navigation` 導入，而不是 `next/router`
- `pathname` 字符串已刪除，並由 [`usePathname()`](/docs/app/api-reference/functions/use-pathname) 替換
- `query` 對象已刪除，並由 [`useSearchParams()`](/docs/app/api-reference/functions/use-search-params) 替換
- `router.events` 已更換。[查看下文](#router-events)

[View the full migration guide](/docs/app/guides/migrating/app-router-migration).

## 範例

### 路由事件

你可以通過組合其他客戶端組件 hook（例如 `usePathname` 和 `useSearchParams`）來監聽頁面更改。

```jsx filename="app/components/navigation-events.js"
'use client'

import { useEffect } from 'react'
import { usePathname, useSearchParams } from 'next/navigation'

export function NavigationEvents() {
  const pathname = usePathname()
  const searchParams = useSearchParams()

  useEffect(() => {
    const url = `${pathname}?${searchParams}`
    console.log(url)
    // You can now use the current URL
    // ...
  }, [pathname, searchParams])

  return '...'
}
```

Which can be imported into a layout.

```jsx filename="app/layout.js" highlight={2,10-12}
import { Suspense } from 'react'
import { NavigationEvents } from './components/navigation-events'

export default function Layout({ children }) {
  return (
    <html lang="en">
      <body>
        {children}

        <Suspense fallback={null}>
          <NavigationEvents />
        </Suspense>
      </body>
    </html>
  )
}
```

> **Good to know**： `<NavigationEvents>` 包裝在 [`Suspense` 邊界](/docs/app/api-reference/file-conventions/loading#examples) 中，因為 [`useSearchParams()`](/docs/app/api-reference/functions/use-search-params) 會導致直到最近的 `Suspense` 邊界為止的客戶端渲染，在 [預渲染](/docs/app/glossary#prerendering) 期間。[了解更多](/docs/app/api-reference/functions/use-search-params#behavior)。

### 禁用滾動至頂部

預設情況下，Next.js 會在導航到新路由時滾動至頁面頂部。你可以透過向 `router.push()` 或 `router.replace()` 傳遞 `scroll: false` 來禁用此行為。

```tsx filename="app/example-client-component.tsx" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button
      type="button"
      onClick={() => router.push('/dashboard', { scroll: false })}
    >
      Dashboard
    </button>
  )
}
```

```jsx filename="app/example-client-component.jsx" switcher
'use client'

import { useRouter } from 'next/navigation'

export default function Page() {
  const router = useRouter()

  return (
    <button
      type="button"
      onClick={() => router.push('/dashboard', { scroll: false })}
    >
      Dashboard
    </button>
  )
}
```
