Next.js 有兩種執行時環境可在您的應用程式中使用：

- **Node.js Runtime** （預設），即可存取所有 Node.js API 並用於渲染您的應用程式。
- **Edge Runtime** 包含較有限的 [API 集合](#reference)，用於 [Proxy](/docs/app/api-reference/file-conventions/proxy)。

## 注意事項

- 邊緣執行時不支援所有 Node.js APIs。某些套件可能無法按預期工作。
- 邊緣執行時不支援漸增式靜態生成 (ISR)。
- 兩種執行時環境均可支援 [串流](/docs/app/api-reference/file-conventions/loading)，視您的部署配接器而定。

### 不支援的 APIs

邊緣執行時有一些限制，包括：

- 原生 Node.js APIs **不支援**。例如，您無法讀取或寫入檔案系統。
- `node_modules` _可以_ 使用，只要它們實作 ES 模組且不使用原生 Node.js APIs。
- 直接呼叫 `require` 是**不允許**的。請改用 ES 模組。

以下 JavaScript 語言功能已停用，且**無法運作：**

| API                                                                                                                             | Description                                                         |
| ------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| [`eval`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/eval)                                       | 評估以字串表示的 JavaScript 程式碼                   |
| [`new Function(evalString)`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Function)               | 建立具有程式碼作為參數的新函式        |
| [`WebAssembly.compile`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/compile)         | 從緩衝區來源編譯 WebAssembly 模組                  |
| [`WebAssembly.instantiate`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/WebAssembly/instantiate) | 從緩衝區來源編譯並實體化 WebAssembly 模組 |

在罕見的情況下，您的程式碼可能包含（或匯入）一些動態程式碼評估語句，_無法在執行時存取_ 且無法透過 treeshaking 移除。
您可以在 Proxy 設定中放寬此檢查以允許特定檔案：

```javascript filename="proxy.ts"
export const config = {
  unstable_allowDynamic: [
    // allows a single file
    '/lib/utilities.js',
    // use a glob to allow anything in the function-bind 3rd party module
    '**/node_modules/function-bind/**',
  ],
}
```

`unstable_allowDynamic` 是一個 [glob](https://github.com/micromatch/micromatch#matching-features)，或者是 glob 的陣列，忽略特定檔案的動態程式碼評估。globs 是相對於您的應用程式根資料夾的相對路徑。

請注意，如果這些語句在邊緣環境上執行，_它們將拋出錯誤並導致執行時錯誤_。
