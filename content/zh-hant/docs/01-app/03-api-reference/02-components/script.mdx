## Props (屬性)

這是 Script 元件可用屬性的摘要：

| Prop (屬性)                   | Example                           | Type     | Required                              |
| ----------------------------- | --------------------------------- | -------- | ------------------------------------- |
| [`src`](#src)                 | `src="http://example.com/script"` | String   | 必要項目，除非使用內聯腳本             |
| [`strategy`](#strategy)       | `strategy="lazyOnload"`           | String   | -                                     |
| [`onLoad`](#onload)           | `onLoad={onLoadFunc}`             | Function | -                                     |
| [`onReady`](#onready)         | `onReady={onReadyFunc}`           | Function | -                                     |
| [`onError`](#onerror)         | `onError={onErrorFunc}`           | Function | -                                     |

## 必要屬性

`<Script />` 元件需要提供以下屬性。

### src

指定外部腳本 URL 的路徑字串。可以是絕對外部 URL 或內部路徑。除非使用內聯腳本，否則 `src` 屬性為必要項目。

## 選用屬性

`<Script />` 元件接受一些除了必要屬性之外的附加屬性。

### 載入策略 (strategy)

腳本的載入策略。有四種不同的策略可供使用：

- `beforeInteractive`：在 Next.js 程式碼及頁面水合發生前載入。
- `afterInteractive`：（**預設**）在頁面發生部分水合後，及早載入。
- `lazyOnload`：在瀏覽器閒置期間載入。
- `worker`：（實驗中）在網頁工作线程中載入。

### 互動前 (beforeInteractive)

使用 `beforeInteractive` 策略載入的腳本會從伺服器注入至初始 HTML，在載入任何 Next.js 模組之前下載，並依照放置順序執行。

標記此策略的腳本會在任何第一方程式碼之前預先載入及 fetched，但其執行**不會阻塞頁面水合**。

<AppOnly>

`beforeInteractive` 腳本必須放置在根佈局 (`app/layout.tsx`) 中，並設計為載入整个網站需要的腳本（即：應用程式中的任何頁面服務端載入時，腳本都會載入）。

</AppOnly>

<PagesOnly>

`beforeInteractive` 腳本必須放置在 `Document` 元件 (`pages/_document.js`) 中，並設計為載入整個網站需要的腳本（即：應用程式中的任何頁面服務端載入時，腳本都會載入）。

</PagesOnly>

**此策略僅應僅用於需要盡快 fetched 的關鍵腳本。**

<AppOnly>

```tsx filename="app/layout.tsx" switcher
import Script from 'next/script'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        {children}
        <Script
          src="https://example.com/script.js"
          strategy="beforeInteractive"
        />
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import Script from 'next/script'

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        {children}
        <Script
          src="https://example.com/script.js"
          strategy="beforeInteractive"
        />
      </body>
    </html>
  )
}
```

</AppOnly>

<PagesOnly>

```jsx filename="pages/_document.js"
import { Html, Head, Main, NextScript } from 'next/document'
import Script from 'next/script'

export default function Document() {
  return (
    <Html>
      <Head />
      <body>
        <Main />
        <NextScript />
        <Script
          src="https://example.com/script.js"
          strategy="beforeInteractive"
        />
      </body>
    </Html>
  )
}
```

</PagesOnly>

> **小知識**：具有 `beforeInteractive` 的腳本將始終注入至 HTML 文件的 `head` 中，與其在元件中的位置無關。

使用 `beforeInteractive` 盡快 fetched 的一些示例包括：

- 機器人偵測程式 (Bot detectors)
- Cookie 同意管理程式 (Cookie consent managers)

### 互動後 (afterInteractive)

使用 `afterInteractive` 策略的腳本將在 HTML 客戶端端注入，並在頁面發生部分 (或全部) 水合後載入。**這是 Script 元件的預設策略**，應用於需要盡快但不用於任何第一方 Next.js 程式碼之前的腳本。

`afterInteractive` 腳本可以放置在任何頁面或佈局中，僅在該頁面 (或頁面群組) 在瀏覽器中打開時才會載入並執行。

```jsx filename="app/page.js"
import Script from 'next/script'

export default function Page() {
  return (
    <>
      <Script src="https://example.com/script.js" strategy="afterInteractive" />
    </>
  )
}
```

適合 `afterInteractive` 的一些示例包括：

- 標籤管理 (Tag managers)
- 分析 (Analytics)

### 延遲載入 (lazyOnload)

使用 `lazyOnload` 策略的腳本會在瀏覽器閒置期間注入至 HTML 客戶端，並在頁面中所有資源 fetched 後載入。此策略應用於不需要及早載入的背景或低優先級腳本。

`lazyOnload` 腳本可以放置在任何頁面或佈局中，僅在該頁面 (或頁面群組) 在瀏覽器中打開時才會載入並執行。

```jsx filename="app/page.js"
import Script from 'next/script'

export default function Page() {
  return (
    <>
      <Script src="https://example.com/script.js" strategy="lazyOnload" />
    </>
  )
}
```

不需要立即載入，可使用 `lazyOnload` fetched 的腳本示例包括：

- 聊天支援外掛程式 (Chat support plugins)
- 社群媒體小工具 (Social media widgets)

### worker (工作線程)

> **警告**：`worker` 策略尚未穩定，且尚未與 App Router 一起使用。請謹慎使用。

使用 `worker` 策略的腳本會被 off-loaded 到網頁工作线程中以釋放主線程，並確保僅有關鍵的第一方資源在其中處理。雖然此策略可用於任何腳本，但它是一種進階案例，且不能保證支援所有第三方腳本。

要使用 `worker` 作為策略，`next.config.js` 必須啟用 `nextScriptWorkers` 旗標：

```js filename="next.config.js"
module.exports = {
  experimental: {
    nextScriptWorkers: true,
  },
}
```

`worker` 腳本目前**僅可用於 `pages/` 目錄**：

```tsx filename="pages/home.tsx" switcher
import Script from 'next/script'

export default function Home() {
  return (
    <>
      <Script src="https://example.com/script.js" strategy="worker" />
    </>
  )
}
```

```jsx filename="pages/home.js" switcher
import Script from 'next/script'

export default function Home() {
  return (
    <>
      <Script src="https://example.com/script.js" strategy="worker" />
    </>
  )
}
```

### onLoad (載入完成)

> **警告：** `onLoad` 尚不支援 Server Components，僅能於 Client Components 使用。此外，`onLoad` 無法與 `beforeInteractive` 一起使用 – 請考慮改用 `onReady`。

部分第三方腳本需要在腳本載入完成後執行 JavaScript 程式碼，以便實裝內容或呼叫 function。如果您正在使用 `afterInteractive` 或 `lazyOnload` 作為載入策略載入腳本，您可以使用 `onLoad` 屬性在它載入後執行程式碼。

這是一個在載入 Library 後僅執行 `lodash` 方法範例。

```tsx filename="app/page.tsx" switcher
'use client'

import Script from 'next/script'

export default function Page() {
  return (
    <>
      <Script
        src="https://cdnjs.cloudflare.com/ajax/libs/lodash.js/4.17.20/lodash.min.js"
        onLoad={() => {
          console.log(_.sample([1, 2, 3, 4]))
        }}
      />
    </>
  )
}
```

```jsx filename="app/page.js" switcher
'use client'

import Script from 'next/script'

export default function Page() {
  return (
    <>
      <Script
        src="https://cdnjs.cloudflare.com/ajax/libs/lodash.js/4.17.20/lodash.min.js"
        onLoad={() => {
          console.log(_.sample([1, 2, 3, 4]))
        }}
      />
    </>
  )
}
```

### onReady (就緒)

> **警告：** `onReady` 尚不支援 Server Components，僅能於 Client Components 使用。

部分第三方腳本需要在腳本載入完成後執行 JavaScript 程式碼，並且每次元件 mounted（例如 Route 導航後）。您可以在腳本初次載入時和之後每次元件重新 mounted 使用 `onReady` 屬性執行程式碼以響應腳本的 load event。

這是一個如何每次元件 mounted 時重新實裝 Google Maps JS embed 範例：

<AppOnly>

```tsx filename="app/page.tsx" switcher
'use client'

import { useRef } from 'react'
import Script from 'next/script'

export default function Page() {
  const mapRef = useRef()

  return (
    <>
      <div ref={mapRef}></div>
      <Script
        id="google-maps"
        src="https://maps.googleapis.com/maps/api/js"
        onReady={() => {
          new google.maps.Map(mapRef.current, {
            center: { lat: -34.397, lng: 150.644 },
            zoom: 8,
          })
        }}
      />
    </>
  )
}
```


### onError (錯誤)

> **警告：** `onError` 尚不支援 Server Components，僅能於 Client Components 使用。`onError` 無法與 `beforeInteractive` 載入策略一起使用。

有時捕捉腳本載入失敗很有幫助。這些錯誤可以用 `onError` 屬性處理：

<AppOnly>

```tsx filename="app/page.tsx" switcher
'use client'

import Script from 'next/script'

export default function Page() {
  return (
    <>
      <Script
        src="https://example.com/script.js"
        onError={(e: Error) => {
          console.error('Script failed to load', e)
        }}
      />
    </>
  )
}
```

```jsx filename="app/page.js" switcher
'use client'

import Script from 'next/script'

export default function Page() {
  return (
    <>
      <Script
        src="https://example.com/script.js"
        onError={(e) => {
          console.error('Script failed to load', e)
        }}
      />
    </>
  )
}
```

</AppOnly>
