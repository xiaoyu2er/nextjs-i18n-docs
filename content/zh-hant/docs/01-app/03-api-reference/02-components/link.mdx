`Link` 元件是延伸到 HTML `<a>` 元素的 React 元件，可為路由間提供 [預取 (prefetching)](/docs/app/getting-started/linking-and-navigating#prefetching) 和前端 (client-side) 導航。這是 Next.js 中路由之間導航的主要方式。

基本用法：

<AppOnly>

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

</AppOnly>

<PagesOnly>

```tsx filename="pages/index.tsx" switcher
import Link from 'next/link'

export default function Home() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

```jsx filename="pages/index.js" switcher
import Link from 'next/link'

export default function Home() {
  return <Link href="/dashboard">Dashboard</Link>
}
```

</PagesOnly>

### `scroll` (預設值為 true。)

Next.js 中的 `<Link>` 預設滾動行為 **是保持滾動位置**，類似瀏覽器處理後退和前進導航的方式。當您導航到新的 [頁面](/docs/app/api-reference/file-conventions/page) 時，只要頁面在視口內，滾動位置將保持不變。但是，如果頁面不在視口中，Next.js 將滾動到第一個頁面元素的頂部。

當 `scroll = {false}` 時，Next.js 將不嘗試滾動到第一個頁面元素。

> **小知識**：Next.js 會在管理滾動行為之前檢查 `scroll: false`。如果啟用了滾動，它會識別導航相關的 DOM 節點並檢查每個頂層元素。所有無法滾動的元素和那些沒有渲染 HTML 的元素都會被跳過，這包括固定或置位元素，以及非可見元素（如那些使用 `getBoundingClientRect` 計算的元素）。接著 Next.js 會遍歷兄弟節點，直到識別出視口中可見的滾動元素。

<AppOnly>

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" scroll={false}>
      Dashboard
    </Link>
  )
}
```

### `prefetch`

<AppOnly>

預取操作會發生在 `<Link />` 元件進入使用者視口時（初始時或透過滾動）。Next.js 會在背景中預取並載入所連結的路由（由 `href` 標記）及其資料，以改善前端導航的效能。如果使用者將滑鼠游標懸停在 `<Link />` 上時預取的資料已過期，Next.js 將嘗試再次預取它。**僅在生產環境中啟用預取**。

以下值可以傳遞給 `prefetch` 屬性：

- **`"auto"` 或 `null`（預設）**：預取行為取決於路由是静态還是動態。對於靜態路由，完整的預取路由（包括所有資料）。對於動態路由，預取將向下預取到具有 [`loading.js`](/docs/app/api-reference/file-conventions/loading#instant-loading-states) 邊界的最近段的動態路線。
- `true`：預取將為靜態和動態路由預取完整的路由。
- `false`：無論是進入視口還是懸停，都不會發生預取。

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/dashboard" prefetch={false}>
      Dashboard
    </Link>
  )
}
```

</AppOnly>

<PagesOnly>

### `locale` (語言)

預設行為會自動前置當前區域設定。`locale` 允許提供不同的語言。當設定為 `false` 時，`href` 必須包含區域設定，因為預設行為已停用。

```tsx filename="pages/index.tsx" switcher
import Link from 'next/link'

export default function Home() {
  return (
    <>
      {/* Default behavior: locale is prepended */}
      <Link href="/dashboard">Dashboard (with locale)</Link>

      {/* Disable locale prepending */}
      <Link href="/dashboard" locale={false}>
        Dashboard (without locale)
      </Link>

      {/* Specify a different locale */}
      <Link href="/dashboard" locale="fr">
        Dashboard (French)
      </Link>
    </>
  )
}
```

```jsx filename="pages/index.js" switcher

export default function Home() {
  return (
    <>

      <Link href="/dashboard">Dashboard (with locale)</Link>

      </Link>

      {/* Specify a different locale */}

      </Link>

}
```

顯示在瀏覽器 URL 列中的路徑的選擇性裝飾器。在 Next.js 9.5.3 之前，這用於動態路由，查看我們的 [以前文件](https://github.com/vercel/next.js/blob/v9.5.2/docs/api-reference/next/link.md#dynamic-routes) 以了解其運作方式。

當此路徑與 `href` 中提供的路徑不同時，將使用先前 `href`/`as` 行為，如 [以前的文件](https://github.com/vercel/next.js/blob/v9.5.2/docs/api-reference/next/link.md#dynamic-routes) 所示。

</PagesOnly>

### `onNavigate` (導航時的事件)

在前端導航期間呼叫的事件處理器。該處理器接收一個包含 `preventDefault()` 方法的事件物件，允許您在需要時取消導航。

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link
      href="/dashboard"
      onNavigate={(e) => {
        // Only executes during SPA navigation
        console.log('Navigating...')

        // Optionally prevent navigation
        // e.preventDefault()
      }}
    >
      Dashboard
    </Link>
  )
}
```

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link
      href="/dashboard"
      onNavigate={(e) => {
        // Only executes during SPA navigation
        console.log('Navigating...')

        // Optionally prevent navigation
        // e.preventDefault()
      }}
    >
      Dashboard
    </Link>
  )
}
```

> **小知識**：雖然 `onClick` 和 `onNavigate` 看起來相似，但它們服務於不同的目的。`onClick` 用於所有點擊事件，而 `onNavigate` 僅在前端導航期間執行。一些主要差異：
>
> - 使用修正鍵 (`Ctrl`/`Cmd` + Click) 時，`onClick` 會執行，但 `onNavigate` 不會，因為 Next.js 會阻止新標籤頁的預設導航。
> - 外部網址不會觸發 `onNavigate`，因為它僅適用於前端和同來源導航。
> - 具有 `download` 屬性的連結會與 `onClick` 合作，但不與 `onNavigate` 合作，因為瀏覽器將把該網址視為下載。

### 檢查當前連結

您可以使用 [`usePathname()`](/docs/app/api-reference/functions/use-pathname) 來判斷連結是否為當前連結。例如，要為當前連結新增類別，您可以檢查目前的 `pathname` 是否與連結的 `href` 相符：

```tsx filename="app/ui/nav-links.tsx" switcher
'use client'

import { usePathname } from 'next/navigation'
import Link from 'next/link'

export function Links() {
  const pathname = usePathname()

  return (
    <nav>
      <Link className={`link ${pathname === '/' ? 'active' : ''}`} href="/">
        Home
      </Link>

      <Link
        className={`link ${pathname === '/about' ? 'active' : ''}`}
        href="/about"
      >
        About
      </Link>
    </nav>
  )
}
```

### 滾動至 `id`

如果您想在導航時滾動到特定的 `id`，您可以在 URL 後加上 `#` 哈希連結，或者只是將哈希連結傳遞給 `href` 屬性。這之所以可能，是因為 `<Link>` 會渲染至 `<a>` 元素。

```jsx
<Link href="/dashboard#settings">Settings</Link>

// Output
<a href="/dashboard#settings">Settings</a>
```

### 傳遞 URL 物件

`Link` 也可以接收 URL 物件，並且它會自動格式化它以建立 URL 字串：

```tsx filename="pages/index.ts" switcher
import Link from 'next/link'

function Home() {
  return (
    <ul>
      <li>
        <Link
          href={{
            pathname: '/about',
            query: { name: 'test' },
          }}
        >
          About us
        </Link>
      </li>
      <li>
        <Link
          href={{
            pathname: '/blog/[slug]',
            query: { slug: 'my-post' },
          }}
        >
          Blog Post
        </Link>
      </li>
    </ul>
  )
}

export default Home
```

```jsx filename="pages/index.js" switcher
import Link from 'next/link'

function Home() {

    <ul>

        <Link
          href={{
            pathname: '/about',

        </Link>

        <Link
          href={{
            pathname: '/blog/[slug]',
```

上述範例包含一個連結至：

### 使用 replace 取代 url 而非 push

<AppOnly>

```tsx filename="app/page.js" switcher
import Link from 'next/link'

  return (
    <Link href="/about" replace>
      About us
    </Link>
  )
}
```

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/about" replace>
      About us
    </Link>
  )
```

### 停用頁面頂端的滾動

<AppOnly>

Next.js 中的 `<Link>` 預設滾動行為 **是保持滾動位置**，類似瀏覽器處理後退和前進導航的方式。當您導航到新的 [頁面](/docs/app/api-reference/file-conventions/page) 時，只要頁面在視口中，滾動位置將保持不變。

不過，如果頁面不在視口中，Next.js 將滾動到第一個頁面元素的頂部。如果您想停用此行為，您可以將 `scroll={false}` 傳遞給 `<Link>` 元件，或將 `scroll: false` 傳遞給 `router.push()` 或 `router.replace()`。

```jsx filename="app/page.js" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/#hashid" scroll={false}>
      Disables scrolling to the top
    </Link>
  )
}
```

```tsx filename="app/page.tsx" switcher
import Link from 'next/link'

export default function Page() {
  return (
    <Link href="/#hashid" scroll={false}>
      Disables scrolling to the top
    </Link>
  )
}
```

使用 `router.push()` 或 `router.replace()`：

```jsx
// useRouter
import { useRouter } from 'next/navigation'

const router = useRouter()

router.push('/dashboard', { scroll: false })
```

</AppOnly>

### 固定頁首的滾動偏移量 (Scroll offset with sticky headers)

因為 Next.js 在尋找滾動目標時會略過固定定位 (sticky/fixed) 元素，導航後內容可能會被固定頁首遮擋。
例如，如果你的佈局有固定頁首：

```tsx filename="app/layout.tsx" switcher
import './globals.css'

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        <header className="sticky top-0 h-16 bg-white">
          {/* Navigation */}
        </header>
        {children}
      </body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
import './globals.css'

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <header className="sticky top-0 h-16 bg-white">
          {/* Navigation */}
        </header>
        {children}
      </body>
    </html>

```

你可以使用 [`scroll-padding-top`](https://developer.mozilla.org/en-US/docs/Web/CSS/scroll-padding-top) 在滾動容器上處理其高度：

```css filename="app/globals.css"
html {
  scroll-padding-top: 64px; /* Match the height of your sticky header */
}
```

這是一個瀏覽器 CSS 屬性，用於偏移基於滾動的位置。
當 Next.js 使用原生的 [`scrollIntoView()`](https://developer.mozilla.org/en-US/docs/Web/API/Element/scrollIntoView) API 時，包括哈希片段 (`#id`) 導航，它也會套用。
或者，你可以對個別目標元素使用 [`scroll-margin-top`](https://developer.mozilla.org/en-US/docs/Web/CSS/scroll-margin-top)，而不設定全域偏移。

### Proxy 中的連結預讀 (Prefetching links in Proxy)

常見的作法是使用 [代理 (Proxy)](/docs/app/api-reference/file-conventions/proxy) 進行身份驗證或涉及將使用者重新導向至不同頁面的其他用途。
為了讓 `<Link />` 元件能正確預讀透過 Proxy 重新導向的連結，你需要告知 Next.js 顯示用的 URL 和預讀用的 URL。
這是為了避免不必要的代理請求以獲取正確的預讀路由。

例如，如果您想要提供帶有已驗證身份和訪客視圖的 ` /dashboard ` 路由，您可以在 Proxy 中添加以下內容以將使用者重新導向至正確的頁面：

```ts filename="proxy.ts" switcher
import { NextResponse } from 'next/server'

export function proxy(request: Request) {
  const nextUrl = request.nextUrl
  if (nextUrl.pathname === '/dashboard') {
    if (request.cookies.authToken) {
      return NextResponse.rewrite(new URL('/auth/dashboard', request.url))
    } else {
      return NextResponse.rewrite(new URL('/public/dashboard', request.url))
    }
  }
}
```

```js filename="proxy.js" switcher

    }
  }
}
```

### 封鎖導航

您可以使用 `onNavigate` 屬性在滿足某些條件（例如表單有未儲存變更）時封鎖導航。當您需要在應用程式的多個元件中封鎖導航（例如防止從任何連結進行導航，而表單正在編輯中），React Context 提供了一種清晰的方式來共享此封鎖狀態。首先，建立一個用來追蹤導航封鎖狀態的 Context：

```tsx filename="app/contexts/navigation-blocker.tsx" switcher
'use client'

import { createContext, useState, useContext } from 'react'

interface NavigationBlockerContextType {
  isBlocked: boolean
  setIsBlocked: (isBlocked: boolean) => void
}

export const NavigationBlockerContext =
  createContext<NavigationBlockerContextType>({
    isBlocked: false,
    setIsBlocked: () => {},
  })

export function NavigationBlockerProvider({
  children,
}: {
  children: React.ReactNode
}) {
  const [isBlocked, setIsBlocked] = useState(false)

  return (
    <NavigationBlockerContext.Provider value={{ isBlocked, setIsBlocked }}>
      {children}
    </NavigationBlockerContext.Provider>
  )
}

export function useNavigationBlocker() {
  return useContext(NavigationBlockerContext)
}
```

