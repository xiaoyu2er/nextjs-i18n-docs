`create-next-app` CLI 允許您使用預設範本或來自公共 GitHub 存放庫的 [範例](https://github.com/vercel/next.js/tree/canary/examples) 建立新的 Next.js 應用程式。這是開始使用 Next.js 最簡單的方法。

基本用法：

```bash package="pnpm"
pnpm create next-app [project-name] [options]
```

```bash package="npm"
npx create-next-app@latest [project-name] [options]
```

```bash package="yarn"
yarn create next-app [project-name] [options]
```

```bash package="bun"
bun create next-app [project-name] [options]
```

## 參考

下列選項可用：

| 選項                                 | 說明                                                           |
| --------------------------------------- | --------------------------------------------------------------------- |
| `-h` 或 `--help`                        | 顯示所有可用選項                                            |
| `-v` 或 `--version`                     | 輸出版本號                                             |
| `--no-*`                                | 否定預設選項。例如：`--no-ts`                                |
| `--ts` 或 `--typescript`                | 初始化為 TypeScript 專案（預設）                          |
| `--js` 或 `--javascript`                | 初始化為 JavaScript 專案                                    |
| `--tailwind`                            | 使用 Tailwind CSS 設定初始化（預設）                         |
| `--react-compiler`                      | 啟用 React Compiler 進行初始化                                |
| `--eslint`                              | 使用 ESLint 設定初始化                                         |
| `--biome`                               | 使用 Biome 設定初始化                                          |
| `--no-linter`                           | 跳過 Linter 設定                                             |
| `--app`                                 | 初始化為 App Router 專案                                   |
| `--api`                                 | 初始化僅包含路由處理程序的專案                         |
| `--src-dir`                             | 在 `src/` 目錄內初始化                                  |
| `--turbopack`                           | 在產生的 package.json 中強制啟用 Turbopack（預設已啟用） |
| `--webpack`                             | 在產生的 package.json 中強制啟用 Webpack                        |
| `--import-alias <alias-to-configure>`   | 指定要使用的匯入別名（預設 "@/\*"）                          |
| `--empty`                               | 初始化一個空白的專案                                           |
| `--use-npm`                             | 明確告知 CLI 使用 npm 引導應用程式        |
| `--use-pnpm`                            | 明確告知 CLI 使用 pnpm 引導應用程式       |
| `--use-yarn`                            | 明確告知 CLI 使用 Yarn 引導應用程式       |
| `--use-bun`                             | 明確告知 CLI 使用 Bun 引導應用程式        |
| `-e` 或 `--example [name] [github-url]` | 引導應用程式時使用的範例                                  |
| `--example-path <path-to-example>`      | 單獨指定範例路徑                            |
| `--reset-preferences`                   | 明確告知 CLI 重置任何儲存的偏好設定               |
| `--skip-install`                        | 明確告知 CLI 跳過安裝套件                   |
| `--disable-git`                         | 明確告知 CLI 停用 git 初始化                 |
| `--agents-md`                           | 包含 `AGENTS.md` 和 `CLAUDE.md` 以指導程式設計助手（預設）  |
| `--yes`                                 | 針對所有選項使用先前的偏好設定或預設值                  |

## 範例

### 使用預設範本

要使用預設範本建立新應用程式，請在終端機中執行以下命令：

```bash package="pnpm"
pnpm create next-app
```

```bash package="npm"
npx create-next-app@latest
```

```bash package="yarn"
yarn create next-app
```

```bash package="bun"
bun create next-app
```

安裝過程中，您將看到以下提示：

```txt filename="Terminal"
What is your project named? my-app
Would you like to use the recommended Next.js defaults?
    Yes, use recommended defaults - TypeScript, ESLint, Tailwind CSS, App Router, AGENTS.md
    No, reuse previous settings
    No, customize settings - Choose your own preferences
```

如果您選擇 `自訂設定`，您將看到以下提示：

```txt filename="Terminal"
Would you like to use TypeScript? No / Yes
Which linter would you like to use? ESLint / Biome / None
Would you like to use React Compiler? No / Yes
Would you like to use Tailwind CSS? No / Yes
Would you like your code inside a `src/` directory? No / Yes
Would you like to use App Router? (recommended) No / Yes
Would you like to customize the import alias (`@/*` by default)? No / Yes
What import alias would you like configured? @/*
Would you like to include AGENTS.md to guide coding agents to write up-to-date Next.js code? No / Yes
```

輸入提示後，`create-next-app` 會建立包含您專案名稱的資料夾並安裝所需的相依套件。

### 程式碼檢查器 (Linter) 選項

**ESLint**: 傳統且最主流的 JavaScript 程式碼檢查器。包含來自 `@next/eslint-plugin-next` 的 Next.js 特定規則。

**Biome**: 快速現代的程式碼檢查器與格式化工具，結合 ESLint 和 Prettier 的功能。內建 Next.js 和 React 領域支援，以達到最佳效能。

**無**: 完全跳過程式碼檢查器設定。您隨時都可以稍後新增程式碼檢查器。

一旦您回答了這些提示，就會根據您選擇的設定建立新專案。

### 使用官方 Next.js 範例

要使用官方 Next.js 範例建立新應用程式，請使用 `--example` 標誌。例如：

```bash package="pnpm"
pnpm create next-app --example [example-name] [your-project-name]
```

```bash package="npm"
npx create-next-app@latest --example [example-name] [your-project-name]
```

```bash package="yarn"
yarn create next-app --example [example-name] [your-project-name]
```

```bash package="bun"
bun create next-app --example [example-name] [your-project-name]
```

您可以在 [Next.js 存放庫](https://github.com/vercel/next.js/tree/canary/examples) 中查看所有可用範例的清單以及設定指令。

### 使用任何公共 GitHub 範例

要使用任何公共 GitHub 範例建立新應用程式，請使用 `--example` 選項並附上 GitHub 存放庫的 URL。例如：

```bash package="pnpm"
pnpm create next-app --example "https://github.com/.../" [your-project-name]
```

```bash package="npm"
npx create-next-app@latest --example "https://github.com/.../" [your-project-name]
```

```bash package="yarn"
yarn create next-app --example "https://github.com/.../" [your-project-name]
```

```bash package="bun"
bun create next-app --example "https://github.com/.../" [your-project-name]
```
