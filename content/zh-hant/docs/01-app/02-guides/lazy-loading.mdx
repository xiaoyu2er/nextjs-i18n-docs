[延遲加載 (Lazy loading)](https://developer.mozilla.org/docs/Web/Performance/Lazy_loading) 在 Next.js 中有助於改善應用程式的初始加載性能，通過減少渲染路由所需的 JavaScript 量。

<AppOnly>

它允許您延遲加載 **客戶端組件 (Client Components)** 和導入的庫，僅在需要時才將其包含在客戶端打包 (Client Bundle) 中。例如，您可能希望延遲加載彈出視窗，直到用戶點擊打開它。

在 Next.js 中實現延遲加載有兩種方法：

1. 使用 `next/dynamic` 的 [動態導入 (Dynamic Imports)](#nextdynamic)
2. 使用 [`React.lazy()`](https://react.dev/reference/react/lazy) 搭配 [Suspense](https://react.dev/reference/react/Suspense)

預設情況下，服務器組件 (Server Components) 會自動 [代碼分割 (Code splitting)](https://developer.mozilla.org/docs/Glossary/Code_splitting)，並可以使用 [流式傳輸 (Streaming)](/docs/app/guides/streaming) 將 UI 的一部分從服務器逐步發送給客戶端。延遲加載 (Lazy loading) 適用於客戶端組件 (Client Components)。

## `next/dynamic`

`next/dynamic` 是 [`React.lazy()`](https://react.dev/reference/react/lazy) 和 [Suspense](https://react.dev/reference/react/Suspense) 的合成。它在 `app` 和 `pages` 目錄中的行為相同，以允許漸進式遷移。

## 範例

### 引入客戶端組件 (Client Components)

```jsx filename="app/page.js"
'use client'

import { useState } from 'react'
import dynamic from 'next/dynamic'

// Client Components:
const ComponentA = dynamic(() => import('../components/A'))
const ComponentB = dynamic(() => import('../components/B'))
const ComponentC = dynamic(() => import('../components/C'), { ssr: false })

export default function ClientComponentExample() {
  const [showMore, setShowMore] = useState(false)

  return (
    <div>
      {/* Load immediately, but in a separate client bundle */}
      <ComponentA />

      {/* Load on demand, only when/if the condition is met */}
      {showMore && <ComponentB />}
      <button onClick={() => setShowMore(!showMore)}>Toggle</button>

      {/* Load only on the client side */}
      <ComponentC />
    </div>
  )
}
```

> **注意：** 當伺服器組件 (Server Component) 動態導入客戶端組件 (Client Component) 時，自動 [代碼分割 (Code splitting)](https://developer.mozilla.org/docs/Glossary/Code_splitting) 目前**不**受支援。

### 跳過 SSR

使用 `React.lazy()` 和 Suspense 時，客戶端組件預設會被 [預先渲染 (Prerendering)](https://github.com/reactwg/server-components/discussions/4)（伺服器渲染 (SSR)）。

> **注意：** `ssr: false` 選項僅對客戶端組件有效，請將其移至客戶端組件中以確保客戶端代碼分割正常運作。

若您想停用客戶端組件 (Client Component) 的預渲染 (Prerendering)，可以將 `ssr` 選項設為 `false`：

```jsx
const ComponentC = dynamic(() => import('../components/C'), { ssr: false })
```

### 引入伺服器組件 (Server Components)

如果您動態導入伺服器組件，只有作為伺服器組件的子項的客戶端組件會被延遲加載，而不是伺服器組件本身。
當您在伺服器組件中使用它時，它還有助於預加載靜態資源，例如 CSS。

```jsx filename="app/page.js"
import dynamic from 'next/dynamic'

// Server Component:
const ServerComponent = dynamic(() => import('../components/ServerComponent'))

export default function ServerComponentExample() {
  return (
    <div>
      <ServerComponent />
    </div>
  )
}
```

> **注意：** `ssr: false` 選項在伺服器組件中不受支援。如果您嘗試在伺服器組件中使用它，將看到錯誤。
> `ssr: false` 不允許在伺服器組件中使用 `next/dynamic`。請將其移至客戶端組件。

### 加載外部庫 (External Libraries)

外部庫可以使用 [`import()`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Operators/import) 函數按需加載。這個範例使用外部庫 `fuse.js` 進行模糊搜索。該模組僅在用戶輸入搜索輸入後才會在客戶端加載。

```jsx filename="app/page.js"
'use client'

import { useState } from 'react'

const names = ['Tim', 'Joe', 'Bel', 'Lee']

export default function Page() {
  const [results, setResults] = useState()

  return (
    <div>
      <input
        type="text"
        placeholder="Search"
        onChange={async (e) => {
          const { value } = e.currentTarget
          // Dynamically load fuse.js
          const Fuse = (await import('fuse.js')).default
          const fuse = new Fuse(names)

          setResults(fuse.search(value))
        }}
      />
      <pre>Results: {JSON.stringify(results, null, 2)}</pre>
    </div>
  )
}
```

### 添加自定義加載組件

```jsx filename="app/page.js"
'use client'

import dynamic from 'next/dynamic'

const WithCustomLoading = dynamic(
  () => import('../components/WithCustomLoading'),
  {
    loading: () => <p>Loading...</p>,
  }
)

export default function Page() {
  return (
    <div>
      {/* The loading component will be rendered while  <WithCustomLoading/> is loading */}
      <WithCustomLoading />
    </div>
  )
}
```

### 引入命名導出 (Named Exports)

要動態導入命名導出，您可以將它從 [`import()`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Operators/import) 函數返回的 Promise 中返回：

```jsx filename="components/hello.js"
'use client'

export function Hello() {
  return <p>Hello!</p>
}
```

```jsx filename="app/page.js"
import dynamic from 'next/dynamic'

const ClientComponent = dynamic(() =>
  import('../components/hello').then((mod) => mod.Hello)
)
```

## 魔法註釋 (Magic Comments)

Next.js 支援魔法註釋 (Magic Comments) 來控制打包工具處理動態導入 (Dynamic Imports) 的方式。這些註釋可用於動態 `import()`、`require()`、`require.resolve()` 及 `new Worker()` 表達式。

> **好知：** 魔法註釋不適用於靜態 `import` 語句 (`import x from 'y'`)。它們僅適用於動態表達式。

### `webpackIgnore` / `turbopackIgnore`

使用這些註釋來跳過動態導入的打包。導入表達式將保持原樣保留在輸出中，適用於僅在運行時使用的模組：

```js
// Skip bundling - import happens at runtime
const runtime = await import(/* webpackIgnore: true */ 'runtime-module')

// Turbopack-specific variant
const plugin = await import(/* turbopackIgnore: true */ pluginPath)

// Also works with require
const mod = require(/* webpackIgnore: true */ 'runtime-module')
```

### `turbopackOptional` (僅限 Turbopack)

使用此註釋可在模組可能不存在時抑制構建錯誤。若模組缺失，導入仍會在運行時拋出異常：

```js
// No build error if './optional-feature' doesn't exist
// Runtime will throw MODULE_NOT_FOUND if executed
const feature = await import(/* turbopackOptional: true */ './optional-feature')

// Also works with require
const mod = require(/* turbopackOptional: true */ './optional-module')
```

這對於以下情況很有用：

- 可能未安裝的條件性功能
- 模組可選的插件系統
- 部分文件可能尚不存在的漸進式遷移

> **好知：** 不支援 `webpackOptional`。使用 Turbopack 時請改用 `turbopackOptional`。

## `next/dynamic`

`next/dynamic` 是 [`React.lazy()`](https://react.dev/reference/react/lazy) 和 [Suspense](https://react.dev/reference/react/Suspense) 的合成。它在 `app` 和 `pages` 目錄中的行為相同，以允許漸進式遷移。

在下方的範例中，使用 `next/dynamic` 後，Header 組件將不會包含在頁面的初始 JavaScript bundle 中。頁面將先渲染 Suspense `fallback`，待 `Suspense` 邊界解析後再渲染 `Header` 組件。

```jsx
import dynamic from 'next/dynamic'

const DynamicHeader = dynamic(() => import('../components/header'), {
  loading: () => <p>Loading...</p>,
})

export default function Home() {
  return <DynamicHeader />
}
```

> **小提示 (Good to know)**： 在 `import('path/to/component')` 中，路徑必須明確寫出。它不能是模板字符串或變量。此外，`import()` 必須在 `dynamic()` 調用內，以便 Next.js 能夠將 webpack 打包/模組 ID 與特定的 `dynamic()` 調用匹配，並在渲染之前預加載它們。`dynamic()` 不能在 React 渲染內部使用，因為它需要在模組的頂層標記才能預加載正常工作，這類似於 `React.lazy`。

## 範例

### 使用具名導入 (Named Exports)

要動態導入命名導出，您可以將其從 [Promise](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Promise) 返回的 [`import()`](https://github.com/tc39/proposal-dynamic-import#example) 中返回：

```jsx filename="components/hello.js"
export function Hello() {
  return <p>Hello!</p>
}

// pages/index.js
import dynamic from 'next/dynamic'

const DynamicComponent = dynamic(() =>
  import('../components/hello').then((mod) => mod.Hello)
)
```

### 不啟用 SSR (No SSR)

要在客戶端加載組件，您可以使用 `ssr` 選項來停用伺服器渲染。如果外部依賴或組件依賴於像 `window` 這類的瀏覽器 API，這很有用。

```jsx
'use client'

import dynamic from 'next/dynamic'

const DynamicHeader = dynamic(() => import('../components/header'), {
  ssr: false,
})
```

### 使用外部庫 (External Libraries)

這個範例使用外部庫 `fuse.js` 進行模糊搜索。該模組僅在用戶輸入搜索輸入後才會在瀏覽器中加載。

```jsx
import { useState } from 'react'

const names = ['Tim', 'Joe', 'Bel', 'Lee']

export default function Page() {
  const [results, setResults] = useState()

  return (
    <div>
      <input
        type="text"
        placeholder="Search"
        onChange={async (e) => {
          const { value } = e.currentTarget
          // Dynamically load fuse.js
          const Fuse = (await import('fuse.js')).default
          const fuse = new Fuse(names)

          setResults(fuse.search(value))
        }}
      />
      <pre>Results: {JSON.stringify(results, null, 2)}</pre>
    </div>
  )
}
```
