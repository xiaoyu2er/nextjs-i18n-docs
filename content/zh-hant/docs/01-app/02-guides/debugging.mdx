本文件說明如何使用具有完整原始碼圖表 (source maps) 支援的 [VS Code 除錯器 (Debugger)](https://code.visualstudio.com/docs/editor/debugging)、[Chrome DevTools](https://developers.google.com/web/tools/chrome-devtools) 或 [Firefox DevTools](https://firefox-source-docs.mozilla.org/devtools-user/) 來除錯您的 Next.js 前端與後端程式碼。

任何可以掛載至 Node.js 的除錯器也可用於除錯 Next.js 應用程式。您可以在 Node.js [除錯指南 (Debugging Guide)](https://nodejs.org/en/docs/guides/debugging-getting-started/) 中找到更多詳細資訊。

## 使用 VS Code 除錯

在專案根目錄建立名為 `.vscode/launch.json` 的檔案，並輸入以下內容：

```json filename="launch.json"
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Next.js: debug server-side",
      "type": "node-terminal",
      "request": "launch",
      "command": "npm run dev -- --inspect"
    },
    {
      "name": "Next.js: debug client-side",
      "type": "chrome",
      "request": "launch",
      "url": "http://localhost:3000"
    },
    {
      "name": "Next.js: debug client-side (Firefox)",
      "type": "firefox",
      "request": "launch",
      "url": "http://localhost:3000",
      "reAttach": true,
      "pathMappings": [
        {
          "url": "webpack://_N_E",
          "path": "${workspaceFolder}"
        }
      ]
    },
    {
      "name": "Next.js: debug full stack",
      "type": "node",
      "request": "launch",
      "program": "${workspaceFolder}/node_modules/next/dist/bin/next",
      "runtimeArgs": ["--inspect"],
      "skipFiles": ["<node_internals>/**"],
      "serverReadyAction": {
        "action": "debugWithEdge",
        "killOnServerStop": true,
        "pattern": "- Local:.+(https?://.+)",
        "uriFormat": "%s",
        "webRoot": "${workspaceFolder}"
      }
    }
  ]
}
```

> **注意**：若要在 VS Code 中使用 Firefox 除錯功能，您需要安裝 [Firefox 除錯器擴充套件](https://marketplace.visualstudio.com/items?itemName=firefox-devtools.vscode-firefox-debug)。

如果您使用 Yarn，可將 `npm run dev` 替換為 `yarn dev`；如果您使用 pnpm，則可替換為 `pnpm dev`。

在「Next.js: debug full stack」配置中，`serverReadyAction.action` 指定伺服器就緒時要開啟哪種瀏覽器。`debugWithEdge` 代表啟動 Edge 瀏覽器。如果您使用的是 Chrome，請將此值更改為 `debugWithChrome`。

如果您正在 [更改應用程式啟動時使用的埠號碼](/docs/pages/api-reference/cli/next#next-dev-options)，請將 `http://localhost:3000` 中的 `3000` 替換為您使用的埠號。

如果您從根目錄以外的目錄執行 Next.js（例如，如果您使用的是 Turborepo），則需要將 `cwd` 新增至端側和全端除錯任務（tasks）。例如：`"cwd": "${workspaceFolder}/apps/web"`。

現在請前往 `除錯面板 (Debug panel)`（在 Windows/Linux 上按 `Ctrl+Shift+D`，在 macOS 上按 `⇧+⌘+D`），選擇 `啟動配置 (launch configuration)`，接著按 `F5` 或在 `命令面板 (Command Palette)` 中選擇 **除錯 (Debug)：開始除錯 (Start Debugging)** 以開始您的 `除錯工作階段 (debugging session)`。

## 在 JetBrains WebStorm 中使用除錯器

點擊列出執行階段配置的下拉式選單，然後點擊 `編輯配置... (Edit Configurations...)`。建立一個 `JavaScript Debug` 除錯配置，並將 `http://localhost:3000` 設為 URL。根據您的需求進行自訂（例如：用於除錯的瀏覽器、儲存為專案檔案），然後點擊 `確定 (OK)`。執行此除錯配置，預設的瀏覽器應該會自動開啟。此時，您應該有 2 個處於除錯模式的應用程式：NextJS Node 應用程式，以及客戶端/瀏覽器應用程式。

## 使用瀏覽器開發工具 (Browser DevTools) 除錯

### 客戶端程式碼

像往常一樣執行 `next dev`、`npm run dev` 或 `yarn dev` 以啟動開發伺服器。伺服器啟動後，在您首選的瀏覽器中開啟 `http://localhost:3000`（或您的其他 URL）。

針對 Chrome：

- 開啟 Chrome 的開發人員工具 (Developer Tools)（在 Windows/Linux 上按 `Ctrl+Shift+J`，在 macOS 上按 `⌥+⌘+I`）
- 前往 **原始碼 (Sources)** 選項卡

針對 Firefox：

- 開啟 Firefox 的開發人員工具 (Developer Tools)（在 Windows/Linux 上按 `Ctrl+Shift+I`，在 macOS 上按 `⌥+⌘+I`）
- 前往 **除錯器 (Debugger)** 選項卡

在任何瀏覽器中，只要客戶端程式碼遇到 [`debugger`](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Statements/debugger) 陳述式，程式執行就會暫停，該檔案就會出現在除錯區域。您也可以搜尋檔案以手動設定停留點 (breakpoints)：

- 在 Chrome 中：在 Windows/Linux 上按 `Ctrl+P`，或在 macOS 上按 `⌘+P`
- 在 Firefox 中：在 Windows/Linux 上按 `Ctrl+P`，或在 macOS 上按 `⌘+P`，或使用左側面板中的檔案樹狀結構

注意，搜尋時您的原始碼檔案路徑將以 `webpack://_N_E/./` 開頭。

### React 開發人員工具 (React Developer Tools)

針對 React 特定的除錯，請安裝 [React 開發人員工具 (React Developer Tools)](https://react.dev/learn/react-developer-tools) 瀏覽器擴充套件。此不可或缺的工具可協助您：

- 檢查 React 元件
- 編輯 props 與 state
- 識別效能問題

### 伺服器端程式碼

要使用瀏覽器開發者工具除錯伺服器端的 Next.js 程式碼，您需要傳遞 `--inspect` 參數：

```bash package="pnpm"
pnpm dev --inspect
```

```bash package="npm"
npm run dev -- --inspect
```

```bash package="yarn"
yarn dev --inspect
```

```bash package="bun"
bun run dev --inspect
```

`--inspect` 的值會傳遞到底層的 Node.js 進程。請參閱 [`--inspect` 文件](https://nodejs.org/api/cli.html#--inspecthostport) 以了解進階使用方法。

> **值得注意**：使用 `--inspect=0.0.0.0` 以允許在非 localhost 環境（例如在 Docker 容器中執行應用程式時）進行遠端除錯存取。

使用 `--inspect` 參數啟動 Next.js 服務器將類似如下：

```bash filename="Terminal"
Debugger listening on ws://127.0.0.1:9229/0cf90313-350d-4466-a748-cd60f4e47c95
For help, see: https://nodejs.org/en/docs/inspector
ready - started server on 0.0.0.0:3000, url: http://localhost:3000
```

針對 Chrome：

1. 開啟新分頁並訪問 `chrome://inspect`
1. 在 **遠端目標 (Remote Target)** 區域尋找您的 Next.js 應用程式
1. 點擊 **檢查 (inspect)** 以開啟獨立的 **開發者工具 (DevTools)** 視窗
1. 進入 **來源 (Sources)** 標籤

針對 Firefox：

1. 開啟新分頁並訪問 `about:debugging`
1. 在左側邊欄點擊 **This Firefox**
1. 在 **遠端目標 (Remote Targets)** 下方，尋找您的 Next.js 應用程式
1. 點擊 **檢查 (Inspect)** 以開啟除錯器
1. 進入 **除錯器 (Debugger)** 標籤

除錯伺服器端程式碼的運作方式與客戶端除錯類似。搜尋檔案（`Ctrl+P`/`⌘+P`）時，您的原始碼檔案路徑將以 `webpack://{application-name}/./` 開頭（`{application-name}` 會根據 `package.json` 檔案中的應用程式名稱替換為實際應用名稱）。

要使用 `--inspect-brk` 或 `--inspect-wait`，您必須改用 `NODE_OPTIONS` 指定。例如：`NODE_OPTIONS=--inspect-brk next dev`。

### 使用瀏覽器開發工具檢查伺服器錯誤

當您遇到錯誤時，檢查原始碼可幫助追蹤錯誤的根本原因。

Next.js 會在錯誤覆蓋層上的 Next.js 版本指示器下方顯示 Node.js 圖示。點擊該圖示後，DevTools URL 即會複製到您的剪貼簿。您可以使用該 URL 開啟新的瀏覽器分頁，以檢查 Next.js 伺服器處理程序。

### 在 Windows 上除錯

請確保您的機器上已停用 Windows Defender。此外部服務會檢查 _每一次檔案讀取_，據報告在 `next dev` 時會大幅增加快速重新整理 (Fast Refresh) 的時間。這是已知問題，與 Next.js 無關，但它確實會影響 Next.js 開發。
