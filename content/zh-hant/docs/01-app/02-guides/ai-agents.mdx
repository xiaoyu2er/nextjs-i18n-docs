Next.js 隨附版本匹配的文件於 `next` 套件內，讓 AI 程式碼代理 (AI coding agents) 能參考準確、最新的 API 和範式。您專案根目錄的 `AGENTS.md` 檔案會引導代理使用這些捆綁文件，而非其訓練資料。

## 運作原理

安裝 `next` 時，Next.js 文件會捆綁在 `node_modules/next/dist/docs/`。捆綁文件的結構與 [Next.js 文件網站](https://nextjs.org/docs) 相同：

```txt
node_modules/next/dist/docs/
├── 01-app/
│   ├── 01-getting-started/
│   ├── 02-guides/
│   └── 03-api-reference/
├── 02-pages/
├── 03-architecture/
└── index.mdx
```

這表示代理隨時都能存取與其安裝版本相符的文件 —— 無需網路請求或外部查找。

您專案根目錄的 `AGENTS.md` 檔案會指示代理在編寫任何程式碼前先閱讀這些捆綁文件。大多數 AI 程式碼代理 (AI coding agents) —— 包括 Claude Code、Cursor、GitHub Copilot 及其他 —— 在啟動工作階段時都會自動讀取 `AGENTS.md`。

## 開始使用

### 新專案

[`create-next-app`](/docs/app/api-reference/cli/create-next-app) 會自動產生 `AGENTS.md` 和 `CLAUDE.md`。不需要額外的設定：

```bash package="pnpm"
pnpm create next-app@canary
```

```bash package="npm"
npx create-next-app@canary
```

```bash package="yarn"
yarn create next-app@canary
```

```bash package="bun"
bun create next-app@canary
```

如果您不想使用代理檔案，請傳入 `--no-agents-md`

```bash
npx create-next-app@canary --no-agents-md
```

### 現有專案

請確認您使用的是 Next.js `v16.2.0-canary.37` 或更新版本，然後將以下檔案新增至您專案的根目錄。

`AGENTS.md` 包含代理將會閱讀的指示：

```md filename="AGENTS.md"
<!-- BEGIN:nextjs-agent-rules -->

# Next.js: ALWAYS read docs before coding

Before any Next.js work, find and read the relevant doc in `node_modules/next/dist/docs/`. Your training data is outdated — the docs are the source of truth.

<!-- END:nextjs-agent-rules -->
```

`CLAUDE.md` 使用 `@` 匯入語法來包含 `AGENTS.md`，因此 [Claude Code](https://docs.anthropic.com/en/docs/claude-code) 使用者可以在不重複內容的情況下獲得相同的指示：

```md filename="CLAUDE.md"
@AGENTS.md
```

## 理解 AGENTS.md

預設的 `AGENTS.md` 包含單一、聚焦的指示：**在編寫程式碼前先閱讀捆綁文件**。這是有意的簡潔 —— 目標是將代理從過時的訓練資料重新導向至 `node_modules/next/dist/docs/` 中準確、與版本相符的文件。

`<!-- BEGIN:nextjs-agent-rules -->` 與 `<!-- END:nextjs-agent-rules -->` 註解標記用來界定由 Next.js 管理的區段。您可以在這些標記之外加入專案專屬指示，無需擔心它們會被未來的更新覆蓋。

捆綁文件包含指南、API 參考以及 App Router 和 Pages Router 的檔案命名規範。當代理遇到涉及路由、資料取得或任何其他 Next.js 功能的任務時，它可以從捆綁文件中查閱正確的 API，而非依賴可能過時的訓練資料。

> **提示：** 若想知道捆綁文件和 `AGENTS.md` 如何提升代理在真實 Next.js 任務上的效能，請瀏覽 [效能測試結果](https://nextjs.org/evals)。
