當 [快取元件 (Cache Components)](/docs/app/api-reference/config/next-config-js/cacheComponents) 啟用時，像 `dynamic`、`revalidate` 和 `fetchCache` 這樣的路由區段配置將由 [`使用快取 (use cache)`](/docs/app/api-reference/directives/use-cache) 和 [`cacheLife (快取配置)`](/docs/app/api-reference/functions/cacheLife) 取代。

## `dynamic = "force-dynamic"`

**無需設定。** 所有頁面預設為動態。

```tsx filename="app/page.tsx" switcher
// Before - No longer needed
export const dynamic = 'force-dynamic'

export default function Page() {
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// Before - No longer needed
export const dynamic = 'force-dynamic'

export default function Page() {
  return <div>...</div>
}
```

```tsx filename="app/page.tsx" switcher
// After - Just remove it
export default function Page() {
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// After - Just remove it
export default function Page() {
  return <div>...</div>
}
```

## `dynamic = "force-static"`

首先將其移除。當在開發和構建期間檢測到未處理的未緩存或執行期資料存取時，Next.js 會拋出錯誤。否則，預先渲染步驟會自動提取靜態 HTML 外殼。

對於未緩存的資料存取，在盡可能靠近資料存取的地方加入 [`使用快取 (use cache)`](/docs/app/api-reference/directives/use-cache)，並搭配長效 [`cacheLife (快取配置)`](/docs/app/api-reference/functions/cacheLife)（如 `'max'`）以維持緩存行為。如有需要，請將其加在頁面或佈局的開頭。

對於執行期資料存取（`cookies()`、`headers()` 等），錯誤將指示你用 `<Suspense>` 將其包裝。由於您是從使用 `force-static` 開始，您必須移除執行期資料存取以防止任何請求時的工作。

```tsx filename="app/page.tsx" switcher
// Before
export const dynamic = 'force-static'

export default async function Page() {
  const data = await fetch('https://api.example.com/data')
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// Before
export const dynamic = 'force-static'

export default async function Page() {
  const data = await fetch('https://api.example.com/data')
  return <div>...</div>
}
```

```tsx filename="app/page.tsx" switcher
import { cacheLife } from 'next/cache'

// After - Use 'use cache' instead
export default async function Page() {
  'use cache'
  cacheLife('max')
  const data = await fetch('https://api.example.com/data')
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
import { cacheLife } from 'next/cache'

// After - Use 'use cache' instead
export default async function Page() {
  'use cache'
  cacheLife('max')
  const data = await fetch('https://api.example.com/data')
  return <div>...</div>
}
```

## `revalidate`

**用 `cacheLife` 取代。** 使用 `cacheLife` 函數來定義緩存期限，而不是路由區段配置。

```tsx filename="app/page.tsx" switcher
// Before
export const revalidate = 3600 // 1 hour

export default async function Page() {
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// Before
export const revalidate = 3600 // 1 hour

export default async function Page() {
  return <div>...</div>
}
```

```tsx filename="app/page.tsx" switcher
// After - Use cacheLife
import { cacheLife } from 'next/cache'

export default async function Page() {
  'use cache'
  cacheLife('hours')
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// After - Use cacheLife
import { cacheLife } from 'next/cache'

export default async function Page() {
  'use cache'
  cacheLife('hours')
  return <div>...</div>
}
```

## `fetchCache`

**無需設定。** 使用 `use cache` 時，緩存範圍內的所有資料抓取都會自動快取，使得 `fetchCache` 變得無需設定。

```tsx filename="app/page.tsx" switcher
// Before
export const fetchCache = 'force-cache'
```

```jsx filename="app/page.js" switcher
// Before
export const fetchCache = 'force-cache'
```

```tsx filename="app/page.tsx" switcher
// After - Use 'use cache' to control caching behavior
export default async function Page() {
  'use cache'
  // All fetches here are cached
  return <div>...</div>
}
```

```jsx filename="app/page.js" switcher
// After - Use 'use cache' to control caching behavior
export default async function Page() {
  'use cache'
  // All fetches here are cached
  return <div>...</div>
}
```

## `runtime = 'edge'`

**不受支援。** 快取元件需要 Node.js 執行環境。透過移除 `runtime = 'edge'` 匯出改用 Node.js 執行環境（預設值）。如果您需要特定路由的邊緣行為，請改用 `[代理 (Proxy)](/docs/app/api-reference/file-conventions/proxy)`。
