> **值得注意：** 本指南假設 [快取組件 (Cache Components)](/docs/app/getting-started/caching) 已啟用。在您的 Next 配置文件中設置 [`cacheComponents: true`](/docs/app/api-reference/config/next-config-js/cacheComponents) 以啟用它。

在快取組件 (Cache Components) 之前，跨導航保留頁面級別狀態需要變通方法，例如將狀態提升到 [共用佈局](/docs/app/getting-started/layouts-and-pages#nesting-layouts) 或使用外部儲存。使用快取組件 (Cache Components) 後，Next.js 預設即保留狀態和 DOM。

Next.js 不取消載入頁面，而是使用 React 的 [`<Activity>`](https://react.dev/reference/react/Activity) 組件來隱藏它們。DOM 節點保留在文件中（使用 `display: none` 隱藏），因此 React 狀態和 DOM 狀態都會保留：表單草稿、捲動位置、展開的 `<details>` 元素、影片播放進度等。

Next.js 最多保留 3 條路由。超過此數量後，最舊的路由會被剔除並重新載入。

> **值得注意：** 正在考慮逐次遷移的選擇退出策略。

## 選擇保留內容

預設情況下，Activity 保留所有組件狀態和 DOM 狀態。對於每個狀態，您需決定這是否適合您的 UI。以下範例展示常見情境及如何處理雙方情況。

### 可擴展 UI（下拉選單、手風琴、面板）

當用戶導航離開並返回時，Activity 保留可擴展元素的展開/關閉狀態。

**何時保留：** 包含展開區塊的側邊欄、FAQ 手風琴或篩選面板。用戶有意設定視圖，恢復它們可避免重複工作。

**何時重置：** 由按鈕點擊觸發的下拉選單或彈出式視窗 (popover)。這是暫時性互動，非持久化視圖狀態。返回頁面時下拉選單已展開並不友善用戶。

若要重置暫時性的展開/關閉狀態，請在 `useLayoutEffect` 清除函數中關閉它：

```tsx highlight={8-13}
'use client'

import { useState, useLayoutEffect } from 'react'

function SettingsDropdown() {
  const [isOpen, setIsOpen] = useState(false)

  // Close dropdown when this component becomes hidden
  useLayoutEffect(() => {
    return () => {
      setIsOpen(false)
    }
  }, [])

  return (
    <div>
      <button onClick={() => setIsOpen((o) => !o)}>Options</button>
      {isOpen && (
        <ul>
          <li>
            <button>Edit Profile</button>
          </li>
          <li>
            <button>Change Password</button>
          </li>
        </ul>
      )}
    </div>
  )
}
```

當 Activity 隱藏此組件時，清除函數執行並重置 `isOpen`。當頁面再次可見時，下拉選單已關閉。使用 `useLayoutEffect` 確保清除工作在組件隱藏前同步執行，避免任何陳舊狀態的閃現。

您也可以使用 `Link` 的 [`onNavigate`](/docs/app/api-reference/components/link#onnavigate) 回呼，當導航連結被點擊時立即關閉下拉選單。

### 對話框與初始化邏輯

Activity 保留對話框的展開/關閉狀態。這也會影響基於該狀態執行的效果 (Effects)。

**何時保留：** 多步驟向導或用戶正在積極處理的設定面板。保留步驟和輸入狀態可避免遺失進度。

**何時重置：** 對話框每次打開時執行初始化邏輯（例如聚焦輸入框）。如果用戶在對話框打開時導航離開，Activity 會保留 `isDialogOpen: true`。再次打開時，由於它已經是 `true`，狀態不會改變，因此效果 (Effect) 不會重新執行。

考慮此範例：

```tsx
'use client'

import { useState, useRef, useEffect } from 'react'

function ProductTab() {
  const [isDialogOpen, setIsDialogOpen] = useState(false)
  const inputRef = useRef<HTMLInputElement>(null)

  useEffect(() => {
    if (isDialogOpen) {
      inputRef.current?.focus()
    }
  }, [isDialogOpen])

  // ...
}
```

如果用戶在對話框打開時離開，返回並打開對話框將不會觸發聚焦效果 (Focus Effect)，因為 `isDialogOpen` 已經是 `true`。

要修復此問題，請從保留的組件狀態外部（例如搜尋參數）派生對話框狀態：

```tsx highlight={3,7-9,20,25}
'use client'

import { useSearchParams, useRouter } from 'next/navigation'
import { useEffect, useRef } from 'react'

function ProductTab() {
  const searchParams = useSearchParams()
  const router = useRouter()
  const isDialogOpen = searchParams.get('edit') === 'true'
  const inputRef = useRef<HTMLInputElement>(null)

  useEffect(() => {
    if (isDialogOpen) {
      inputRef.current?.focus()
    }
  }, [isDialogOpen])

  return (
    <div>
      <button onClick={() => router.push('?edit=true')}>Edit Product</button>

      {isDialogOpen && (
        <dialog open>
          <input ref={inputRef} placeholder="Product name" />
          <button onClick={() => router.replace('?', { scroll: false })}>
            Close
          </button>
        </dialog>
      )}
    </div>
  )
}
```

使用此方法，`isDialogOpen` 從 URL 派生而非組件狀態。當導航離開並返回時，搜尋參數被清空（URL 已變更），因此 `isDialogOpen` 變成 `false`。打開對話框會設置參數，這會改變 `isDialogOpen` 並觸發效果 (Effect)。

### 表單輸入值

Activity 保留表單輸入值：輸入欄位的文字、選項、核取方塊狀態。

**何時保留：** 包含篩選的搜尋頁面、用戶正在撰寫的草稿，或有未儲存變更的設定表單。保留輸入狀態是主要的 UX 優勢之一，因為用戶不會遺失工作。

**何時重置：** 返回時應重新開始的「建立新項目」頁面，或提交成功後的聯絡表單。

要在 Activity 隱藏組件時重置表單欄位，請使用回呼參照 (callback ref)：

```tsx
<form
  ref={(form) => {
    // Cleanup function - runs when Activity hides this component
    return () => form?.reset()
  }}
>
  {/* fields */}
</form>
```

這會在用戶導航離開時重置表單。

### 動作狀態（`useActionState`）

Activity 保留 [`useActionState`](https://react.dev/reference/react/useActionState) 結果：成功訊息、錯誤訊息，以及操作返回的任何其他狀態。

**何時保留：** 顯示「票券已成功兌換」的票券兌換表單，或顯示「變更已儲存」的設定表單。返回頁面時看到先前操作的結果是有用的確認，讓用戶了解發生了什麼。

**何時重置：** 應該每次到訪時皆重新開始的「新交易」流程，或在新情境下過期成功/錯誤訊息會令人困惑的表單。

您可以將 `useActionState` 視為允許副作用的 `useReducer`。它不必僅處理表單提交；您可以向其中發送任何操作。添加 `RESET` 操作為 Activity 隱藏組件時清除狀態提供了一種清潔的方式（參見 React 文件中的 [重置狀態](https://react.dev/reference/react/useActionState#reset-state)）：

```tsx highlight={5-6,9-21,26-35}
'use client'

import { useActionState, useLayoutEffect, useRef, startTransition } from 'react'

type Action = { type: 'SUBMIT'; data: FormData } | { type: 'RESET' }
type State = { success: boolean; error: string | null }

function CommentForm() {
  const [state, dispatch, isPending] = useActionState(
    async (prev: State, action: Action) => {
      if (action.type === 'RESET') {
        return { success: false, error: null }
      }
      // Handle the form submission
      const res = await saveComment(action.data)
      if (!res.ok) return { success: false, error: res.message }
      shouldReset.current = true
      return { success: true, error: null }
    },
    { success: false, error: null }
  )

  const shouldReset = useRef(false)

  // Dispatch RESET when Activity hides this component
  useLayoutEffect(() => {
    return () => {
      if (shouldReset.current) {
        shouldReset.current = false
        startTransition(() => {
          dispatch({ type: 'RESET' })
        })
      }
    }
  }, [dispatch])

  return (
    <form action={(formData) => dispatch({ type: 'SUBMIT', data: formData })}>
      <textarea name="comment" />
      <button type="submit" disabled={isPending}>
        {isPending ? 'Posting...' : 'Post Comment'}
      </button>
      {state.success && <p>Comment posted!</p>}
      {state.error && <p>{state.error}</p>}
    </form>
  )
}
```

以下是逐步發生的情況：

1. 用戶提交表單。reducer 接收帶有 `FormData` 的 `SUBMIT` 操作，呼叫 `saveComment`，並返回 `{ success: true }`。它還將 `shouldReset.current = true` 設為 `true`，標記需要重置。
2. 用戶導航離開。Activity 隱藏組件並執行 `useLayoutEffect` 清除工作。因為 `shouldReset.current` 為 `true`，因此執行 `RESET` 操作。
3. reducer 接收 `RESET` 並返回初始狀態（`{ success: false, error: null }`）。過期的成功訊息被清除。
4. 如果用戶導航返回，表單已準備好進行新提交。如果他們從未提交（步驟 1 未發生），`shouldReset.current` 仍為 `false`，因此不執行 `RESET`。表單保持原狀。

## 狀態與認證

Activity 在導航之間保留本地組件狀態（`useState`、DOM 輸入值），包括認證變更。這是標準的 React 行為：Props 變更（例如收到新用戶）會觸發重新渲染，但不會重置現有狀態。一個用戶撰寫的草稿不應該對另一個用戶可見。

對於退出流程，使用 `window.location.href` 而非 `router.push` 會觸發整頁重新載入，清除所有客戶端狀態。

若要在不重新載入的情況下在用戶變更時重置特定狀態：

```tsx
'use client'

import { useState, useEffect, useRef } from 'react'

function UserScopedForm({ userId }: { userId: string | null }) {
  const [draft, setDraft] = useState('')
  const lastUserIdRef = useRef<string | null>(null)

  useEffect(() => {
    if (lastUserIdRef.current !== null && lastUserIdRef.current !== userId) {
      setDraft('') // Reset on user change
    }
    lastUserIdRef.current = userId
  }, [userId])

  return <textarea value={draft} onChange={(e) => setDraft(e.target.value)} />
}
```

或者，按用戶 ID 對組件進行鍵值設定，讓 React 處理重置：`<Form key={userId} />`。

## 全域樣式

當源頭組件被 Activity 隱藏時，頁面級別樣式（CSS 變數、z-index、全域類別）可能影響可見頁面。您很可能希望在隱藏時停用它們：隱藏頁面的強調色或 z-index 覆寫不應該洩漏到可見頁面。

使用回呼參照 (callback ref) 切換樣式表的 `media` 屬性：

```tsx
<style
  ref={(style) => {
    if (style) style.media = '' // Enable when visible
    return () => {
      if (style) style.media = 'not all' // Disable when hidden
    }
  }}
>
  {`:root { --page-accent: blue; }`}
</style>
```

或者在管理多個樣式元素或更複雜的清除工作時使用 `useLayoutEffect`：

```tsx
'use client'

import { useLayoutEffect, useRef } from 'react'

function PageWithStyles() {
  const styleRef = useRef<HTMLStyleElement>(null)

  useLayoutEffect(() => {
    if (styleRef.current) styleRef.current.media = ''
    return () => {
      if (styleRef.current) styleRef.current.media = 'not all'
    }
  }, [])

  return <style ref={styleRef}>{`:root { --page-accent: blue; }`}</style>
}
```

當 Activity 隱藏組件時，清除工作會將 `media` 設為 `not all`，這會停用樣式表。當再次可見時，效果會重新執行並將 `media` 重置為啟用。

## 測試

隱藏的 Activity 內容具有 `display: none` 但保留在文件中。這適用於 Cache Components 保留的路由和您直接使用 `<Activity>` 隱藏的內容。它影響使用 Playwright、Cypress 或 Puppeteer 等工具的端到端測試：

- **DOM 查詢可找到隱藏元素。** 選擇器可能匹配元素，無論其可見性如何。
- **與隱藏元素的互動可能失敗或超時。** 大多數工具會在互動前等待元素成為可見。
- **斷言可能匹配隱藏內容。** 在斷言元素存在時，請明確指定可見性。

### 使用了解可見性的選擇器

在 Playwright 中，`getByRole` 查詢會自動按可見性篩選：

```ts
// Good - getByRole filters by visibility automatically
await page.getByRole('button', { name: 'Submit' }).click()
await page.getByRole('textbox', { name: 'Email' }).fill('test@example.com')

// Also good - getByLabel, getByPlaceholder filter by visibility
await page.getByLabel('Email').fill('test@example.com')
await page.getByPlaceholder('Search...').fill('query')
```

當 `getByRole` 不適合時，使用 `.locator()` 搭配可見性篩選：

```ts
// Fallback - filter by visibility explicitly
await page.locator('.product-card').filter({ visible: true }).first().click()
await page
  .locator('[data-testid="timer"]')
  .filter({ visible: true })
  .textContent()

// Avoid - may match hidden elements in Activity boundaries
await page.locator('.product-card').first().click()
```

`getByRole` 對 Activity、分頁導航、手風琴以及其他任何在 DOM 中保留隱藏內容的模式具有魯棒性。它查詢無障礙樹 (accessibility tree)，該樹排除隱藏元素。對於其他測試工具，請查看其文件以了解了解可見性的選擇器。例如，Cypress 使用 `.should('be.visible')` 或 `{ visible: true }` 選項。

## 在組件中使用 Activity

快取組件 (Cache Components) 在路由層級自動使用 Activity，但您也可以直接在組件中使用 `<Activity>`。這對標籤頁、可擴展面板或您希望隱藏內容但不撤除 (unmount) 任何 UI 很有用。

### 預渲染隱藏內容

Activity 可以預先載入用戶尚未看到過的內容。隱藏邊界以較低優先級渲染。搭配 Suspense，這讓您能預取用戶可能會查看的下一部分內容的資料。

伺服器元件可以立即開始擷取資料並將 Promise 傳遞給客戶端組件。客戶端組件使用 Activity 隱藏內容，直到用戶請求，並使用 `use()` 在渲染時解析 Promise：

```tsx filename="app/page.tsx"
import { Suspense } from 'react'
import { ExpandableComments } from './expandable-comments'

async function getCommentsData() {
  return db.comments.findMany()
}

export default function Page() {
  const commentsPromise = getCommentsData()

  return (
    <article>
      <h1>Post Title</h1>
      <p>Main content visible immediately...</p>

      <ExpandableComments commentsPromise={commentsPromise} />
    </article>
  )
}
```

```tsx filename="app/expandable-comments.tsx"
'use client'

import { Activity, Suspense, useState, use } from 'react'

type Comment = { id: string; text: string; author: string }

export function ExpandableComments({
  commentsPromise,
}: {
  commentsPromise: Promise<Comment[]>
}) {
  const [expanded, setExpanded] = useState(false)

  return (
    <>
      <button onClick={() => setExpanded((e) => !e)}>
        {expanded ? 'Hide Comments' : 'Show Comments'}
      </button>

      <Activity mode={expanded ? 'visible' : 'hidden'}>
        <Suspense fallback={<CommentsSkeleton />}>
          <Comments commentsPromise={commentsPromise} />
        </Suspense>
      </Activity>
    </>
  )
}

function Comments({
  commentsPromise,
}: {
  commentsPromise: Promise<Comment[]>
}) {
  const comments = use(commentsPromise)
  return (
    <ul>
      {comments.map((c) => (
        <li key={c.id}>{c.text}</li>
      ))}
    </ul>
  )
}

function CommentsSkeleton() {
  return <div>Loading comments...</div>
}
```

伺服器元件立即開始擷取評論並傳遞 Promise。隱藏時，資料以較低優先級串流。當用戶點擊「顯示評論」時，`Comments` 組件使用 `use()` 解析 Promise，內容立即顯示。

### 效果與媒體清除

當 Activity 隱藏內容時，React 執行效果清除函數，就像在撤除 (unmount) 時一樣。這意味著如果您有正確的清除工作，計時器、訂閱和媒體播放將自動暫停：

```tsx
'use client'

import { useEffect, useState } from 'react'

function LiveTimer() {
  const [count, setCount] = useState(0)

  useEffect(() => {
    const id = setInterval(() => setCount((c) => c + 1), 1000)
    return () => clearInterval(id) // Pauses when hidden
  }, [])

  return <p>Count: {count}</p>
}
```

對於 `<video>` 和 `<audio>` 等媒體元素，`display: none` 不會停止播放。使用 `useLayoutEffect` 添加明確清除：

```tsx
'use client'

import { useLayoutEffect, useRef } from 'react'

function VideoPlayer({ src }: { src: string }) {
  const videoRef = useRef<HTMLVideoElement>(null)

  useLayoutEffect(() => {
    const video = videoRef.current
    return () => {
      video?.pause() // Pauses when hidden, preserves playback position
    }
  }, [])

  return <video ref={videoRef} src={src} controls />
}
```

當組件再次可見時，效果會重新執行，由於 DOM 節點從未被移除，播放位置得以保留。

### 區分初次載入與重新顯示

效果會在每次隱藏轉可見的轉換時執行，不僅僅是初次載入。如果您需要區分初次載入與後續可見性變更，請使用參照 (ref)。

```tsx
'use client'

import { useEffect, useRef } from 'react'

function TrackedComponent() {
  const hasMountedRef = useRef(false)

  useEffect(() => {
    if (!hasMountedRef.current) {
      hasMountedRef.current = true
      console.log('First mount')
    } else {
      console.log('Became visible again')
    }
  }, [])

  return <div>...</div>
}
```

參照在隱藏/顯示循環中保持存在（參照不會被清除），因此 `hasMountedRef.current` 在初次載入後保持為 `true`。每次 Activity 成為可見時，效果都會重新執行，但現在它會執行 `else` 分支。

## 範例

此 [活動演示 (Activity Patterns Demo)](https://react-activity-patterns.vercel.app/) ([源碼](https://github.com/vercel-labs/react-activity-patterns)) 是一個啟用了快取組件 (Cache Components) 並具有三個路由的 Next.js 應用。在它們之間導航以查看狀態保留的操作：

- **資料** — 可排序的表格和可選擇的列表，其狀態在導航間保留，加上在背景預渲染的評論區塊
- **表單** — 具有 DOM 狀態（`<details>`、核取方塊、文字輸入）的篩選面板，會持續保留，以及在提交後使用 `useLayoutEffect` 清除重置的電子報表單
- **副作用** — 在您導航離開時暫停、返回時恢復的實時計時器，以及自動暫停並保留播放位置的影片播放器
