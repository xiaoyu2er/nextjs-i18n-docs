[Markdown](https://daringfireball.net/projects/markdown/syntax) 是一種輕量級的標記語言，用於格式化文字。它允許您使用純文字語法撰寫並將其轉換為結構正確的 HTML。它常用於撰寫網站和部落格上的內容。

您撰寫...

```md
I **love** using [Next.js](https://nextjs.org/)
```

輸出：

```html
<p>I <strong>love</strong> using <a href="https://nextjs.org/">Next.js</a></p>
```

[MDX](https://mdxjs.com/) 是 Markdown 的超集，可讓您直接在 Markdown 檔案中撰寫 [JSX](https://react.dev/learn/writing-markup-with-jsx)。這是一個強而有力的方法，可為您加入動態互動並嵌入 React 元件。

Next.js 可支援應用程式內部的本地 MDX 內容，以及從伺服器動態獲取的遠端 MDX 檔案。Next.js 外掛負責將 Markdown 和 React 元件轉換為 HTML，包括支援在伺服器元件中使用的功能（App Router 的預設設定）。

> **小知識**：參閱 [Portfolio Starter Kit](https://vercel.com/templates/next.js/portfolio-starter-kit) 範本，以查看完整範例。

## 安裝相依套件

`@next/mdx` 套件及相關套件用於配置 Next.js，使其能處理 Markdown 和 MDX。**它從本地檔案取得資料**，讓您能在 `/pages` 或 `/app` 目錄中直接建立以 `.md` 或 `.mdx` 為副檔名的頁面。

安裝這些套件以使用 Next.js 渲染 MDX：

```bash package="pnpm"
pnpm add @next/mdx @mdx-js/loader @mdx-js/react @types/mdx
```

```bash package="npm"
npm install @next/mdx @mdx-js/loader @mdx-js/react @types/mdx
```

```bash package="yarn"
yarn add @next/mdx @mdx-js/loader @mdx-js/react @types/mdx
```

```bash package="bun"
bun add @next/mdx @mdx-js/loader @mdx-js/react @types/mdx
```

## 配置 `next.config.mjs`

更新專案根目錄處的 `next.config.mjs` 檔案以配置其使用 MDX：

```js filename="next.config.mjs"
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  // Configure `pageExtensions` to include markdown and MDX files
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
  // Optionally, add any other Next.js config below
}

const withMDX = createMDX({
  // Add markdown plugins here, as desired
})

// Merge MDX config with Next.js config
export default withMDX(nextConfig)
```

這允許 `.mdx` 檔案在您的應用程式中作為頁面、路由或匯入檔案使用。

### 處理 `.md` 檔案

預設情況下，`next/mdx` 僅編譯副檔名為 `.mdx` 的檔案。要使用 Webpack 處理 `.md` 檔案，請更新 `extension` 選項：

```js filename="next.config.mjs"
const withMDX = createMDX({
  extension: /\.(md|mdx)$/,
})
```

## 新增 `mdx-components.tsx` 檔案

在專案根目錄中建立 `mdx-components.tsx`（或 `.js`）檔案以定義全域 MDX 元件。例如，與 `pages` 或 `app` 同層級，或在 `src` 內（如適用）。

```tsx filename="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'

const components: MDXComponents = {}

export function useMDXComponents(): MDXComponents {
  return components
}
```

```js filename="mdx-components.js" switcher
const components = {}

export function useMDXComponents() {
  return components
}
```

> **小知識**：
>
> - `mdx-components.tsx` 是 **必需的**，以在 App Router 中搭配 `@next/mdx` 使用，缺少它無法運作。
> - 參閱 [`mdx-components.tsx` 檔案慣例](/docs/app/api-reference/file-conventions/mdx-components) 的詳細資訊。
> - 學習如何使用 [自訂樣式和元件](#using-custom-styles-and-components)。

## 渲染 MDX

您可以使用 Next.js 的基於檔案的路由來渲染 MDX，或透過將 MDX 檔案匯入到其他頁面。

### 使用基於檔案的路由

使用基於檔案的路由時，您可以像使用其他頁面一樣使用 MDX 頁面。

<AppOnly>

在 App Router 應用程式中，這包括能夠使用 [元數據 (Metadata)](/docs/app/getting-started/metadata-and-og-images)。

建立新的 MDX 頁面在 `/app` 目錄內：

```txt
  my-project
  ├── app
  │   └── mdx-page
  │       └── page.(mdx/md)
  |── mdx-components.(tsx/js)
  └── package.json
```

</AppOnly>

<PagesOnly>

在 `/pages` 目錄內建立新的 MDX 頁面：

```txt
  my-project
  |── mdx-components.(tsx/js)
  ├── pages
  │   └── mdx-page.(mdx/md)
  └── package.json
```

</PagesOnly>

您可以在這些檔案中使用 MDX，甚至直接匯入 React 元件到您的 MDX 頁面：

```mdx
import { MyComponent } from 'my-component'

# Welcome to my MDX page!

This is some **bold** and _italics_ text.

This is a list in markdown:

- One
- Two
- Three

Checkout my React component:

<MyComponent />
```

瀏覽至 `/mdx-page` 路由應會顯示您的渲染後 MDX 頁面。

### 使用動態匯入

您可以匯入動態 MDX 元件，而不使用 MDX 檔案的檔案系統路由。

例如，您可以有一個動態的路由區段，從獨立目錄載入 MDX 元件：

<Image
  alt="Route segments for dynamic MDX components"
  srcLight="/docs/light/mdx-files.png"
  srcDark="/docs/dark/mdx-files.png"
  width="1600"
  height="849"
/>

[`generateStaticParams`](/docs/app/api-reference/functions/generate-static-params) 可用於預先渲染提供的路由。將 `dynamicParams` 標記為 `false` 後，存取 `generateStaticParams` 中未定義的路由將返回 404。

```tsx filename="app/blog/[slug]/page.tsx" switcher
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>
}) {
  const { slug } = await params
  const { default: Post } = await import(`@/content/${slug}.mdx`)

  return <Post />
}

export function generateStaticParams() {
  return [{ slug: 'welcome' }, { slug: 'about' }]
}

export const dynamicParams = false
```

```jsx filename="app/blog/[slug]/page.js" switcher
export default async function Page({ params }) {
  const { slug } = await params
  const { default: Post } = await import(`@/content/${slug}.mdx`)

  return <Post />
}

export function generateStaticParams() {
  return [{ slug: 'welcome' }, { slug: 'about' }]
}

export const dynamicParams = false
```

> **小知識**：確保在匯入時指定 `.mdx` 檔案副檔名。雖然使用 [模組路徑別名](/docs/app/getting-started/installation#set-up-absolute-imports-and-module-path-aliases)（例如，`@/content`）非必須，但這可簡化您的匯入路徑。

</AppOnly>

## 使用自訂樣式和元件

Markdown 在渲染時會對應至原生 HTML 元素。例如，撰寫以下 Markdown：

```md
## This is a heading

This is a list in markdown:

- One
- Two
- Three
```

會產生以下 HTML：

```html
<h2>This is a heading</h2>

<p>This is a list in markdown:</p>

<ul>
  <li>One</li>
  <li>Two</li>
  <li>Three</li>
</ul>
```

要設計您的 Markdown，您可以提供自訂元件，這些元件會對應至產生的 HTML 元素。樣式和元件可全域、本地實施，並與共用版面配置搭配使用。

### 全域樣式和元件

在 `mdx-components.tsx` 中加入樣式和元件將影響您應用程式中的 _所有_ MDX 檔案。

```tsx filename="mdx-components.tsx" switcher
import type { MDXComponents } from 'mdx/types'
import Image, { ImageProps } from 'next/image'

// This file allows you to provide custom React components
// to be used in MDX files. You can import and use any
// React component you want, including inline styles,
// components from other libraries, and more.

const components = {
  // Allows customizing built-in components, e.g. to add styling.
  h1: ({ children }) => (
    <h1 style={{ color: 'red', fontSize: '48px' }}>{children}</h1>
  ),
  img: (props) => (
    <Image
      sizes="100vw"
      style={{ width: '100%', height: 'auto' }}
      {...(props as ImageProps)}
    />
  ),
} satisfies MDXComponents

export function useMDXComponents(): MDXComponents {
  return components
}
```

```js filename="mdx-components.js" switcher
import Image from 'next/image'

// This file allows you to provide custom React components
// to be used in MDX files. You can import and use any
// React component you want, including inline styles,
// components from other libraries, and more.

const components = {
  // Allows customizing built-in components, e.g. to add styling.
  h1: ({ children }) => (
    <h1 style={{ color: 'red', fontSize: '48px' }}>{children}</h1>
  ),
  img: (props) => (
    <Image sizes="100vw" style={{ width: '100%', height: 'auto' }} {...props} />
  ),
}

export function useMDXComponents() {
  return components
}
```

### 本地樣式和元件

您可以透過將樣式和元件傳入匯入的 MDX 元件，將本地樣式和元件套用至特定頁面。這些將合併並覆蓋 [全域樣式和元件](#global-styles-and-components)。

## 前置內容 (Frontmatter)

Frontmatter 是一種類似 YAML 的鍵值配對，可用來儲存關於頁面的資料。`@next/mdx` 預設**不**支援 frontmatter，不過有幾種方法可以將 frontmatter 新增到您的 MDX 內容中，例如：

- [remark-frontmatter](https://github.com/remarkjs/remark-frontmatter)
- [remark-mdx-frontmatter](https://github.com/remcohaszing/remark-mdx-frontmatter)
- [gray-matter](https://github.com/jonschlinkert/gray-matter)

`@next/mdx` **允許**您像使用任何 JavaScript 元件一樣使用匯出：

```mdx filename="content/blog-post.mdx" switcher
export const metadata = {
  author: 'John Doe',
}

# Blog post
```

現在可以在 MDX 檔案之外參閱元數據 (Metadata)：

<AppOnly>

```tsx filename="app/blog/page.tsx" switcher
import BlogPost, { metadata } from '@/content/blog-post.mdx'

export default function Page() {
  console.log('metadata: ', metadata)
  //=> { author: 'John Doe' }
  return <BlogPost />
}
```

```jsx filename="app/blog/page.js" switcher
import BlogPost, { metadata } from '@/content/blog-post.mdx'

export default function Page() {
  console.log('metadata: ', metadata)
  //=> { author: 'John Doe' }

```

</AppOnly>

<PagesOnly>


## remark 和 rehype 外掛

您可以選擇提供 remark 和 rehype 外掛以轉換 MDX 內容。

例如，您可以使用 [`remark-gfm`](https://github.com/remarkjs/remark-gfm) 來支援 GitHub 慣用 Markdown。

由於 remark 和 rehype 生態系僅支援 ESM，您需要使用 `next.config.mjs` 或 `next.config.ts` 作為設定檔案。

```js filename="next.config.mjs"
import remarkGfm from 'remark-gfm'
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  // Allow .mdx extensions for files
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
  // Optionally, add any other Next.js config below
}

const withMDX = createMDX({
  // Add markdown plugins here, as desired
  options: {
    remarkPlugins: [remarkGfm],
    rehypePlugins: [],
  },
})

// Combine MDX and Next.js config
export default withMDX(nextConfig)
```

### 在 Turbopack 中使用外掛

要搭配 [Turbopack](/docs/app/api-reference/turbopack) 使用外掛，請升級至最新版本的 `@next/mdx` 並使用字串指定外掛名稱：

```js filename="next.config.mjs"
import createMDX from '@next/mdx'

/** @type {import('next').NextConfig} */
const nextConfig = {
  pageExtensions: ['js', 'jsx', 'md', 'mdx', 'ts', 'tsx'],
}

const withMDX = createMDX({
  options: {
    remarkPlugins: [
      // Without options
      'remark-gfm',
      // With options
      ['remark-toc', { heading: 'The Table' }],
    ],
    rehypePlugins: [
      // Without options
      'rehype-slug',
      // With options
      ['rehype-katex', { strict: true, throwOnError: true }],
    ],
  },
})

export default withMDX(nextConfig)
```

> **小知識**：
>
> 尚無法與 [Turbopack](/docs/app/api-reference/turbopack) 搭配無可序列化選項的 remark 和 rehype 外掛使用，因為 JavaScript 函式無法傳遞給 Rust。

## 深入探討：如何將 Markdown 轉換為 HTML？

React 原生並不了解 Markdown。Markdown 純文字必須先轉換為 HTML。這可使用 `remark` 和 `rehype` 達成。

`remark` 是以 Markdown 為核心的工具生態系。`rehype` 則是相同概念，但針對 HTML。例如，以下程式碼片段將 Markdown 轉換為 HTML：

```js
import { unified } from 'unified'
import remarkParse from 'remark-parse'
import remarkRehype from 'remark-rehype'
import rehypeSanitize from 'rehype-sanitize'
import rehypeStringify from 'rehype-stringify'

main()

async function main() {
  const file = await unified()
    .use(remarkParse) // Convert into markdown AST
    .use(remarkRehype) // Transform to HTML AST
    .use(rehypeSanitize) // Sanitize HTML input
    .use(rehypeStringify) // Convert AST into serialized HTML
    .process('Hello, Next.js!')

  console.log(String(file)) // <p>Hello, Next.js!</p>
}
```

`remark` 和 `rehype` 生態系包含以下外掛：[語法突顯](https://github.com/atomiks/rehype-pretty-code)、[連結標題](https://github.com/rehypejs/rehype-autolink-headings)、[生成目錄](https://github.com/remarkjs/remark-toc) 等。

當使用上述的 `@next/mdx` 時，您**不需要**直接使用 `remark` 或 `rehype`，因為它已為您處理。我們在此描述它是為了幫助您更深入理解 `@next/mdx` 套件底層運作方式。

## 使用基於 Rust 的 MDX 編譯器 (實驗性)

Next.js 支援新編寫的 Rust MDX 編譯器。此編譯器仍處於實驗階段，不建議用於生產環境。要使用新編譯器，在將其傳遞給 `withMDX` 時需要配置 `next.config.js`：

```js filename="next.config.js"
module.exports = withMDX({
  experimental: {
    mdxRs: true,
  },
})
```

`mdxRs` 也接受物件以配置如何轉換 MDX 檔案。

```js filename="next.config.js"
module.exports = withMDX({
  experimental: {
    mdxRs: {
      jsxRuntime?: string            // Custom jsx runtime
      jsxImportSource?: string       // Custom jsx import source,
      mdxType?: 'gfm' | 'commonmark' // Configure what kind of mdx syntax will be used to parse & transform
    },
  },
})
```
