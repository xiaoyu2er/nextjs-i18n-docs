[模型上下文協議 (MCP)](https://modelcontextprotocol.io) 是一個開放標準，允許 AI 代理和編碼助手通過標準化介面與您的應用程式進行互動。

Next.js 16+ 包含了 MCP 支持，允許編碼代理實時訪問您的應用程式內部。要使用此功能，請安裝 [`next-devtools-mcp`](https://www.npmjs.com/package/next-devtools-mcp) 包。

## 開始使用

**需求：** Next.js 16 或更高版本

將 `next-devtools-mcp` 添加到專案根目錄的 `.mcp.json` 檔案中：

```json filename=".mcp.json"
{
  "mcpServers": {
    "next-devtools": {
      "command": "npx",
      "args": ["-y", "next-devtools-mcp@latest"]
    }
  }
}
```

就這樣！當您啟動開發伺服器時，`next-devtools-mcp` 將自動發現並連接到您的運行中 Next.js 實例。

更多配置選項，請參閱 [next-devtools-mcp 倉庫](https://github.com/vercel/next-devtools-mcp)。

## 功能

next-devtools-mcp 為編碼代理提供了一套不斷增長的功能：

### 應用程式執行階段訪問

- **錯誤偵測 (Error Detection)**：從開發伺服器擷取目前的建置錯誤、執行階段錯誤和類型錯誤
- **即時狀態查詢 (Live State Queries)**：訪問實時應用程式狀態和執行階段資訊
- **頁面元資料 (Page Metadata)**：查詢頁面路由、組件和渲染詳情
- **伺服器操作 (Server Actions)**：檢查組件階層和伺服器操作
- **開發日誌 (Development Logs)**：訪問開發伺服器日誌和控制台輸出

### 開發工具

- **Next.js 知識庫**：查詢完整的 Next.js 文件與最佳實踐
- **升級與移植工具**：使用 Codemods 自動協助升級到 Next.js 16
- **快取指南**：快取組件 (Cache Components) 的設定與配置輔助
- **瀏覽器測試**：[Playwright MCP](https://github.com/microsoft/playwright-mcp) 整合，用於在瀏覽器中驗證頁面

> **注意：** Next.js 團隊正在積極擴展這些功能。新工具和功能會定期添加，以改善代理開發體驗。

## 開發工作流程

1. 啟動 Next.js 開發伺服器：

```bash package="pnpm"
pnpm dev
```

```bash package="npm"
npm run dev
```

```bash package="yarn"
yarn dev
```

```bash package="bun"
bun dev
```

2. 您的編碼代理將自動通過 `next-devtools-mcp` 連接到運行中的 Next.js 實例

3. 在瀏覽器中打開您的應用程式以查看頁面

4. 向代理查詢洞察和診斷資訊（請參見以下範例）

### 可用的工具

透過 `next-devtools-mcp`，代理可以使用以下工具：

- **`get_errors`**：從您的開發伺服器擷取目前的建置錯誤、執行階段錯誤和類型錯誤
- **`get_logs`**：獲取包含瀏覽器控制台日誌和伺服器輸出的開發日誌檔案路徑
- **`get_page_metadata`**：獲取特定頁面的元資料，包括路由、組件和渲染資訊
- **`get_project_metadata`**：擷取專案結構、配置和開發伺服器 URL
- **`get_routes`**：通過掃描檔案系統獲取所有將成為入口點的路由。按路由類型（appRouter, pagesRouter）分組返回路由。動態片段顯示為 `[param]` 或 `[...slug]` 模式
- **`get_server_action_by_id`**：透過 ID 查找伺服器操作 (Server Actions)，以找到源檔案和函數名稱

## 與代理配合使用

與 MCP 相容的編碼代理可以自動發現並連接到您的 Next.js 開發伺服器，為它們提供有關您專案的豐富上下文。

### 代理輔助開發的優勢

代理可以：

- **做出上下文感知建議**：根據您的現有結構建議添加新功能的適當位置
- **查詢即時應用程式狀態**：在開發期間檢查當前配置、路由和中間件
- **理解您的 App Router 頁面佈局**：明確知道具體渲染了哪些頁面和佈局
- **提供準確實現**：生成遵循您專案模式和規範的程式碼


- 透過 `next-devtools-mcp` 查詢您的運行中的 Next.js 應用程式
- 擷取目前的建置錯誤、執行階段錯誤和類型錯誤

### 升級與最佳實踐

獲取關於 Next.js 概念和遷移的協助：

```bash
User: "Help me upgrade my Next.js app to version 16"
```

代理將分析您的當前版本，透過 codemods 引導您完成自動化遷移，并提供處理破壞性變化的逐步說明。

詢問概念性問題：

```bash
User: "When should I use 'use client' in App Router?"
```

代理將查詢 Next.js 知識庫，并提供基於文件的説明，以及來自您程式碼庫的範例。

## 運作原理

Next.js 16+ 包含了在 `/_next/mcp` 的內建 MCP 端點，該端點運行在您的開發伺服器內。`next-devtools-mcp` 包會自動發現並與這些端點溝通，使其能夠：

- 連接到在不同通訊埠上運行的多個 Next.js 實例
- 將工具調用轉發到適當的 Next.js 開發伺服器
- 為編碼代理提供統一介面

此架構將代理介面與內部實現解耦，使 next-devtools-mcp 能夠在不同 Next.js 專案之間無縫運行。

## 疑難排解

### MCP 伺服器未連接

- 確保您使用的是 Next.js v16 或更高版本
- 確認 `.mcp.json` 中已配置 `next-devtools-mcp`
- 啟動您的開發伺服器：`npm run dev`
- 如果伺服器已經在運行，請重新啟動開發伺服器
- 檢查您的編碼代理是否已載入 MCP 伺服器配置
