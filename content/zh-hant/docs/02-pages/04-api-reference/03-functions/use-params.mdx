`useParams` 是一個 Hook，讓您能讀取由目前 URL 填入的路由 [動態參數](/docs/pages/building-your-application/routing/dynamic-routes)。

```tsx filename="pages/shop/[slug].tsx" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams<{ slug: string }>()

  if (!params) {
    // Render fallback UI while params are not yet available
    return null
  }

  // Route -> /shop/[slug]
  // URL -> /shop/shoes
  // `params` -> { slug: 'shoes' }
  return <>Shop: {params.slug}</>
}
```

```jsx filename="pages/shop/[slug].js" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams()

  if (!params) {
    // Render fallback UI while params are not yet available
    return null
  }

  // Route -> /shop/[slug]
  // URL -> /shop/shoes
  // `params` -> { slug: 'shoes' }
  return <>Shop: {params.slug}</>
}
```

## 參數

```tsx
const params = useParams()
```

`useParams` 沒有任何參數。

## 返回值

`useParams` 回傳一個包含目前路由已填入的 [動態參數](/docs/pages/building-your-application/routing/dynamic-routes) 的物件，或在 [預渲染](#behavior-during-prerendering) 期間回傳 `null`。

- 物件中的每個屬性都是啟用的動態段。
- 屬性名稱是段的名稱，屬性值則為該段填入的內容。
- 屬性值將是 `string` 或 `string` 陣列，視 [動態段的類型](/docs/pages/building-your-application/routing/dynamic-routes) 而定。
- 如果路由不包含任何動態參數，`useParams` 則會回傳空物件。

例如：

| 路由                         | URL         | `useParams()`             |
| ---------------------------- | ----------- | ------------------------- |
| `pages/shop/page.js`         | `/shop`     | `{}`                      |
| `pages/shop/[slug].js`       | `/shop/1`   | `{ slug: '1' }`           |
| `pages/shop/[tag]/[item].js` | `/shop/1/2` | `{ tag: '1', item: '2' }` |
| `pages/shop/[...slug].js`    | `/shop/1/2` | `{ slug: ['1', '2'] }`    |

> **小提示**：`useParams` 是一個 [React Hook](https://react.dev/learn#using-hooks)，無法與類別一同使用。

## 行為

### 預渲染期間的行為

對於 [靜態優化](/docs/pages/building-your-application/rendering/automatic-static-optimization) 的頁面，`useParams` 在初始渲染時會回傳 `null`。經過水合後，當路由準備就緒，值便會更新為實際參數。

這是因為動態路由在靜態生成期間無法得知參數。

```tsx filename="pages/shop/[slug].tsx" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams<{ slug: string }>()

  if (!params) {
    // Return a fallback UI while params are loading
    // This prevents hydration mismatches
    return <ShopPageSkeleton />
  }

  return <>Shop: {params.slug}</>
}
```

```jsx filename="pages/shop/[slug].js" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams()

  if (!params) {
    // Return a fallback UI while params are loading
    // This prevents hydration mismatches
    return <ShopPageSkeleton />
  }

  return <>Shop: {params.slug}</>
}
```

### 與 `getServerSideProps` 搭配使用

當使用 [`getServerSideProps`](/docs/pages/building-your-application/data-fetching/get-server-side-props) 時，頁面會在每次請求時進行伺服器渲染，且 `useParams` 會立即回傳實際參數：

```tsx filename="pages/shop/[slug].tsx" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams<{ slug: string }>()

  // With getServerSideProps, this fallback is never rendered because
  // params is always available on the server. However, keeping
  // the fallback allows this component to be reused on other pages
  // that may not use getServerSideProps.
  if (!params) {
    return null
  }

  return <>Shop: {params.slug}</>
}

export async function getServerSideProps() {
  return { props: {} }
}
```

```jsx filename="pages/shop/[slug].js" switcher
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const params = useParams()

  // With getServerSideProps, this fallback is never rendered because
  // params is always available on the server. However, keeping
  // the fallback allows this component to be reused on other pages
  // that may not use getServerSideProps.
  if (!params) {
    return null
  }

  return <>Shop: {params.slug}</>
}

export async function getServerSideProps() {
  return { props: {} }
}
```

### 與 `router.query` 比較

`useParams` 僅回傳動態路由參數，而 `useRouter` 中的 [`router.query`](/docs/pages/api-reference/functions/use-router#router-object) 則包含動態參數和查詢字串參數。

```tsx filename="pages/shop/[slug].tsx" switcher
import { useRouter } from 'next/router'
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const router = useRouter()
  const params = useParams()

  // URL -> /shop/shoes?color=red

  // router.query -> { slug: 'shoes', color: 'red' }
  // params -> { slug: 'shoes' }

  // ...
}
```

```jsx filename="pages/shop/[slug].js" switcher
import { useRouter } from 'next/router'
import { useParams } from 'next/navigation'

export default function ShopPage() {
  const router = useRouter()
  const params = useParams()

  // URL -> /shop/shoes?color=red

  // router.query -> { slug: 'shoes', color: 'red' }
  // params -> { slug: 'shoes' }

  // ...
}
```

## 範例

### 與 App Router 共用元件

`next/navigation` 中的 `useParams` 同時適用於 Pages Router 和 App Router。這使您能夠建立在任何環境下都能運作的共用元件：

```tsx filename="components/breadcrumb.tsx" switcher
import { useParams } from 'next/navigation'

// This component works in both pages/ and app/
export function Breadcrumb() {
  const params = useParams<{ slug: string }>()

  if (!params) {
    // Fallback for Pages Router during prerendering
    return <nav>Home / ...</nav>
  }

  return <nav>Home / {params.slug}</nav>
}
```

```jsx filename="components/breadcrumb.js" switcher
import { useParams } from 'next/navigation'

// This component works in both pages/ and app/
export function Breadcrumb() {
  const params = useParams()

  if (!params) {
    // Fallback for Pages Router during prerendering
    return <nav>Home / ...</nav>
  }

  return <nav>Home / {params.slug}</nav>
}
```

> **小提示**：在 App Router 中使用此元件時，`useParams` 絕不會回傳 `null`，因此不會渲染 fallback 分支。
