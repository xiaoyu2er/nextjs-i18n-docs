要了解服務端組件與客户端組件如何運作，熟悉兩個基礎的網頁概念很有幫助：

*   [環境](#server-and-client-environments)：你的應用程式程式碼可以在哪些環境中執行：服務端與客户端。
*   [網路邊界](#network-boundary)：分隔服務端和客户端程式碼的邊界。

服務端與客户端環境[](#server-and-client-environments)
-----------------------------------------------------------------

在 Web 應用程式的脈絡下：

![圖示顯示左側為網頁瀏覽器，右側為服務端，中間有網路邊界分隔。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/learn-client-and-server-environments.png)![圖示顯示左側為網頁瀏覽器，右側為服務端，中間有網路邊界分隔。](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Flearn-client-and-server-environments.png&w=3840&q=75)

*   **客户端**是指使用者裝置上的瀏覽器，它向服務端發送請求以獲取應用程式程式碼。隨後，它將從服務端接收到的回應轉化為使用者可以互動的介面。
*   **服務端**是指資料中心中的電腦，它儲存你的應用程式程式碼，接收來自客户端的請求，執行計算，然後發送適當的回應。

每個環境都有其自己的功能與限制。例如，將渲染和數據獲取移到服務端，你可以減少發送給客户端的程式碼量，這能提升應用程式的性能。但是如你之前學到的，若要讓你的 UI 具備互動性，你需要在客户端更新 DOM。

因此，你為服務端和客户端撰寫的程式碼並不一定總是一樣的。某些操作（例如：數據獲取或管理使用者狀態）較適合在某種環境中進行，而非另一種。

網路邊界[](#network-boundary)
-------------------------------------

**網路邊界**是一個概念性的分界線，用來區分不同的環境。

在 React 中，你可以選擇在元件樹中的哪個位置放置網路邊界。例如，你可以在服務端獲取數據並渲染用戶的文章（使用服務端組件），然後在客户端為每篇文章渲染互動的 `LikeButton`（使用客户端組件）。

同樣地，你可以建立一個 `Nav` 元件，在服務端渲染並在各頁面間共用，但如果你想顯示連結的當前狀態，你可以在客户端渲染 `Links` 列表。

![元件樹顯示一個版面配置，其孩子為 3 個元件：Nav、Page 和 Footer。Page 元件有 2 個孩子：Posts 和 LikeButton。Posts 元件在服務端渲染，LikeButton 元件在客户端渲染。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/learn-client-server-modules.png)![元件樹顯示一個版面配置，其孩子為 3 個元件：Nav、Page 和 Footer。Page 元件有 2 個孩子：Posts 和 LikeButton。Posts 元件在服務端渲染，LikeButton 元件在客户端渲染。](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Flearn-client-server-modules.png&w=3840&q=75)

在幕後，元件被劃分為兩個模組圖譜。**服務端模組圖譜（或樹）(server module graph (or tree))** 包含所有在服務端渲染的服務端組件，而 **客户端模組圖譜（或樹）(client module graph (or tree))** 包含所有客户端組件。

服務端組件渲染後，一個稱為 **React 服務端組件數據包 (RSC)** 的專有數據格式會傳送到客户端。RSC 數據中包含：

1. 服務端組件的渲染結果。
2. 客户端組件應渲染的位置佔位符（或空位），及其 JavaScript 檔案的引用。

React 使用這些資訊來整合服務端組件與客户端組件，並在客户端更新 DOM。

讓我們來看看如何運作。

使用客户端組件[](#using-client-components)
---------------------------------------------------

正如你在上一章學到的，Next.js 預設使用服務端組件——這是為了提升你的應用程式性能，意味著你不需要額外的步驟來採用它們。

回想一下瀏覽器中的錯誤訊息，Next.js 警告你可能試圖在服務端組件內部使用 `useState`。你可以透過將互動的 "Like" 按鈕移動到客户端組件來修正這個問題。

在 `app` 資料夾中新建一個名為 `like-button.js` 的檔案，匯出 `LikeButton` 元件：

/app/like-button.js

```
export default function LikeButton() {}
```

將 `<button>` 元素和 `handleClick()` 函式從 `page.js` 移動到你的新 `LikeButton` 元件中：

/app/like-button.js

```
export default function LikeButton() {
  function handleClick() {
    setLikes(likes + 1);
  }
 
  return <button onClick={handleClick}>Like ({likes})</button>;
}
```

接下來，移動 `likes` 狀態和匯入指令：

/app/like-button.js

```
import { useState } from 'react';
 
export default function LikeButton() {
  const [likes, setLikes] = useState(0);
 
  function handleClick() {
    setLikes(likes + 1);
  }
 
  return <button onClick={handleClick}>Like ({likes})</button>;
}
```

現在，要讓 `LikeButton` 成為客户端組件，在檔案頂部加入 React 的 `use client` 指令。這會告訴 React 在客户端渲染該元件。

/app/like-button.js

```
'use client';
 
import { useState } from 'react';
 
export default function LikeButton() {
  const [likes, setLikes] = useState(0);
 
  function handleClick() {
    setLikes(likes + 1);
  }
 
  return <button onClick={handleClick}>Like ({likes})</button>;
}
```

回到你的 `page.js` 檔案，將 `LikeButton` 元件匯入頁面：

/app/page.js

```
import LikeButton from './like-button';
 
function Header({ title }) {
  return <h1>{title ? title : 'Default title'}</h1>;
}
 
export default function HomePage() {
  const names = ['Ada Lovelace', 'Grace Hopper', 'Margaret Hamilton'];
 
  return (
    <div>
      <Header title="Develop. Preview. Ship." />
      <ul>
        {names.map((name) => (
          <li key={name}>{name}</li>
        ))}
      </ul>
      <LikeButton />
    </div>
  );
}
```

儲存這兩個檔案並在瀏覽器中查看你的應用程式。現在沒有錯誤了，當你進行更改並儲存時，你應該會注意到瀏覽器自動更新以反映更改。

這個功能稱為 [快速重新整理](/docs/architecture/fast-refresh)。它提供你對任何編輯的即時回饋，並已在 Next.js 中預先設定。
