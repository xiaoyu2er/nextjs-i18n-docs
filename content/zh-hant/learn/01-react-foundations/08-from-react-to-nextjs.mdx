到目前為止，我們探討了如何開始使用 React。這是最終程式碼的外觀。如果您從這裡開始，請將此程式碼貼上到程式碼編輯器中的 `index.html` 檔案。

index.html

```
<html>
  <body>
    <div id="app"></div>
 
    <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
    <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
 
    <script type="text/jsx">
      const app = document.getElementById("app")
 
      function Header({ title }) {
        return <h1>{title ? title : "Default title"}</h1>
      }
 
      function HomePage() {
        const names = ["Ada Lovelace", "Grace Hopper", "Margaret Hamilton"]
 
        const [likes, setLikes] = React.useState(0)
 
        function handleClick() {
          setLikes(likes + 1)
        }
 
        return (
          <div>
            <Header title="Develop. Preview. Ship." />
            <ul>
              {names.map((name) => (
                <li key={name}>{name}</li>
              ))}
            </ul>
 
            <button onClick={handleClick}>Like ({likes})</button>
          </div>
        )
      }
 
      const root = ReactDOM.createRoot(app);
      root.render(<HomePage />);
    </script>
  </body>
</html>
```

在最近幾章節中，您會認識到三個重要的 React 概念：**元件 (components)**、**props** 和 **狀態 (state)**。在這些方面擁有堅實的基礎將幫助您開始建構 React 應用程式。

學習 React 時，**建構是最好的學習方式**。您可以逐漸採用 React，使用 `<script>` 和目前所學，將小型元件加入到現有網站。然而，許多開發者發現 React 所提供的使用者和開發者體驗非常有價值，足以讓他們直接投入，用 React 撰寫整個前端應用程式。

從 React 到 Next.js[](#from-react-to-nextjs)
----------------------------------------------

儘管 React 擅長於建構使用者介面 (UI)，但若獨立將該 UI 建構為運作良好的可擴展應用程式，仍需投入一些心力。還有較新的 React 功能，如伺服器元件 (Server Components) 和用戶端元件 (Client Components)，需要框架支援。好消息是 Next.js 處理了大部分設定與組態，並具有額外功能協助您建構 React 應用程式。

接下來，我們將範例從 React 遷移至 Next.js，討論 Next.js 的運作方式，並為您介紹伺服器元件 (Server Components) 和用戶端元件 (Client Components) 之間的差異。
