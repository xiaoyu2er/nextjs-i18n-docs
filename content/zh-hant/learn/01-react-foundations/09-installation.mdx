當您在專案中使用 Next.js 時，您不再需要從 [unpkg.com](http://unpkg.com) 載入 `react` 和 `react-dom` 腳本。相反地，您可以使用 `npm` 或您偏愛的套件管理員本地安裝這些套件。

> **注意**：要使用 Next.js，您的機器上必須安裝 Node.js 版本 **20.9** 或更高（[查看最小版本要求](/docs/getting-started/installation)），您可以在此 [下載](https://nodejs.org/en/)。

為此，請在 `index.html` 檔案所在的目錄建立一個新的檔案，名為 `package.json`，並使用空物件 `{}`。

package.json

```
{}
```

在您的 [終端機](https://code.visualstudio.com/docs/terminal/basics) 中，於專案根部執行以下指令：

終端機

```
npm install react@latest react-dom@latest next@latest
```

安裝完成後，您應該能夠在 `package.json` 檔案中看到您的專案相依性列表：

package.json

```
{
  "dependencies": {
    "next": "^14.0.3",
    "react": "^18.3.1",
    "react-dom": "^18.3.1"
  }
}
```

如果您使用的是比上述版本更新的版本，請不要擔心，只要已安裝 `next`、`react` 和 `react-dom` 套件，就可以開始了。

您也會注意到一個名為 `package-lock.json` 的新檔案，其中包含每個套件準確版本的詳細資訊。

回到 `index.html` 檔案，您可以刪除以下程式碼：

1.  `<html>` 和 `<body>` 標籤。
2.  `id` 為 `app` 的 `<div>` 元素。
3.  `react` 和 `react-dom` 腳本，因為您已使用 NPM 安裝它們。
4.  `Babel` 腳本，因為 Next.js 擁有一個編譯器，可將 JSX 轉換為瀏覽器可以理解的有效 JavaScript。
5.  `<script type="text/jsx">` 標籤。
6.  `document.getElementById()` 和 `ReactDom.createRoot()` 方法。
7.  `React.useState(0)` 函式中的 `React.` 部分

刪除上述行之後，將以下匯入新增到檔案的頂部：

index.html

```
import { useState } from 'react';
```

您的程式碼應如下所示：

index.html

```
import { useState } from 'react';
 
function Header({ title }) {
  return <h1>{title ? title : 'Default title'}</h1>;
}
 
function HomePage() {
  const names = ['Ada Lovelace', 'Grace Hopper', 'Margaret Hamilton'];
 
  const [likes, setLikes] = useState(0);
 
  function handleClick() {
    setLikes(likes + 1);
  }
 
  return (
    <div>
      <Header title="Develop. Preview. Ship." />
      <ul>
        {names.map((name) => (
          <li key={name}>{name}</li>
        ))}
      </ul>
 
      <button onClick={handleClick}>Like ({likes})</button>
    </div>
  );
}
```

HTML 檔案中僅剩的程式碼是 JSX，因此您可以將檔案類型從 `.html` 更改為 `.js` 或 `.jsx`。

建立您的第一個頁面 [](#creating-your-first-page)
-----------------------------------------------------

Next.js 使用檔案系統路由。這意味著您不必使用程式碼來定義應用程式的路由，而是可以使用資料夾和檔案。

以下是如何在 Next.js 中建立您的第一個頁面：

1.  建立一個名為 [app](/docs/app/building-your-application/routing#the-app-router) 的新資料夾，並將 `index.js` 檔案移至其中。
2.  將您的 `index.js` 檔案重新命名為 `page.js`。這將是您的應用程式主頁。
3.  為您的 `<HomePage>` 元件加上 `export default`，以幫助 Next.js 區分要渲染為頁面主要元件的元件。

app/page.js

```
import { useState } from 'react';
 
function Header({ title }) {
  return <h1>{title ? title : 'Default title'}</h1>;
}
 
export default function HomePage() {
  // ...
}
```

運行開發伺服器 [](#running-the-development-server)
------------------------------------------------------------------

接下來，讓我們執行您的開發伺服器，以便您在開發時可以看到新頁面的變更。在您的 `package.json` 檔案中添加 `"next dev"` 腳本：

package.json

```
{
  "scripts": {
    "dev": "next dev"
  },
  "dependencies": {
    "next": "^14.0.3",
    "react": "^18.3.1",
    "react-dom": "^18.3.1"
  }
}
```

在您的終端機中執行 `npm run dev` 檢查發生什麼事。您會注意到兩件事：

1.  當您瀏覽至 [localhost:3000](http://localhost:3000) 時，您應該會看到以下錯誤：

![Next.js 錯誤訊息：您正在導入一個需要使用 useState 的元件。它僅適用於客戶端 (Client) 元件...](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/learn-usestate-rsc-error.png)![Next.js 錯誤訊息：您正在導入一個需要使用 useState 的元件。它僅適用於客戶端 (Client) 元件...](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Flearn-usestate-rsc-error.png&w=3840&q=75)

這是因為 Next.js 使用 React Server Components（伺服器元件），這是一項新功能，允許 React 在伺服器上渲染。伺服器元件不支援 `useState`，因此您必須改用客戶端元件 (Client Component)。

在下一章中，我們將討論伺服器與客戶端 (Client) 元件之間的主要差異，並修復此錯誤。

2.  一個名為 `layout.js` 的新檔案已自動建立於 `app` 資料夾內。這是您的應用程式的主要佈局。您可以使用它來新增跨所有頁面共享的 UI 元素（例如導覽列、頁尾等）。

/app/layout.js

```
export const metadata = {
  title: 'Next.js',
  description: 'Generated by Next.js',
};
 
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

摘要 [](#summary)
-------------------

回顧到目前為止的遷移過程，您可能已經感受到使用 Next.js 的好處：

*   您移除了 React 和 Babel 腳本；這讓您體驗到不再需要操心的複雜工具與配置。
*   您建立了您的第一個頁面。
