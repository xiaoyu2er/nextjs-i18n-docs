在本章中，我們將開始使用 JavaScript 和 DOM 方法來建立專案，在您的專案中加入 `h1` 標籤。

開啟您的程式碼編輯器，並建立一個新的 `index.html` 檔案。在 HTML 檔案內，加入以下程式碼：

index.html

```
<html>
  <body>
    <div></div>
  </body>
</html>
```

接著給 `div` 一個唯一的 `id`，以便您稍後能夠針對它。

index.html

```
<html>
  <body>
    <div id="app"></div>
  </body>
</html>
```

要在 HTML 檔案內部撰寫 JavaScript，請加入 `script` 標籤：

index.html

```
<html>
  <body>
    <div id="app"></div>
    <script type="text/javascript"></script>
  </body>
</html>
```

現在，在 `script` 標籤內部，您可以使用 DOM 方法 [`getElementById()`](https://developer.mozilla.org/en-US/docs/Web/API/Document/getElementById) 透過其 `id` 選取 `<div>` 元素：

index.html

```
<html>
  <body>
    <div id="app"></div>
    <script type="text/javascript">
      const app = document.getElementById('app');
    </script>
  </body>
</html>
```

您可以繼續使用 DOM 方法來建立新的 <h1> 元素：

index.html

```
<html>
  <body>
    <div id="app"></div>
    <script type="text/javascript">
      // Select the div element with 'app' id
      const app = document.getElementById('app');
 
      // Create a new H1 element
      const header = document.createElement('h1');
 
      // Create a new text node for the H1 element
      const text = 'Develop. Preview. Ship.';
      const headerContent = document.createTextNode(text);
 
      // Append the text to the H1 element
      header.appendChild(headerContent);
 
      // Place the H1 element inside the div
      app.appendChild(header);
    </script>
  </body>
</html>
```

為了確保一切運作正常，請在您選擇的瀏覽器中開啟您的 HTML 檔案。您應該會看到一個 h1 標籤，上面顯示「Develop。Preview。Ship.」。

HTML vs. DOM[](#html-vs-the-dom)
------------------------------------

如果您在 [瀏覽器開發者工具](https://developer.mozilla.org/docs/Learn/Common_questions/Tools_and_setup/What_are_browser_developer_tools) 內部查看 DOM 元素，您會注意到 DOM 包含了 `<h1>` 元素。頁面的 DOM 與原始碼不同——換句話說，也就是您建立的原始 HTML 檔案。

![並排顯示渲染後的 DOM 元素與原始碼 (HTML) 差異的兩張圖](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/learn-dom-and-source.png)![並排顯示渲染後的 DOM 元素與原始碼 (HTML) 差異的兩張圖](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Flearn-dom-and-source.png&w=3840&q=75)

這是因為 HTML 代表**頁面的初始內容**，而 DOM 代表**更新後的頁面內容**，這是您撰寫的 JavaScript 程式碼所進行的更改。

使用純 JavaScript 更新 DOM 非常強大但繁瑣。您已撰寫了所有這些程式碼來為一些文字加入一個 <h1> 元素：

index.html

```
<script type="text/javascript">
  const app = document.getElementById('app');
  const header = document.createElement('h1');
  const text = 'Develop. Preview. Ship.';
  const headerContent = document.createTextNode(text);
  header.appendChild(headerContent);
  app.appendChild(header);
</script>
```

隨著應用程式或團隊規模的擴大，以此方式建構應用程式的難度會逐漸增加。

使用此方法時，開發者會花費大量時間撰寫指示來告訴電腦應該**如何**執行。但能否描述您想顯示的**內容**，讓電腦自行判斷**如何**更新 DOM 會更好嗎？

命令式與宣告式程式設計[](#imperative-vs-declarative-programming)
--------------------------------------------------------------------------------

上述程式碼是**命令式** **程式設計**的良好範例。您正在撰寫**如何**更新使用者介面的步驟。但在建構使用者介面時，通常較偏好宣告式方法，因為它可以加快開發流程。與其撰寫 DOM 方法，如果開發者能夠宣告他們想顯示的**內容**會更有幫助（在此情況下，帶有文字的 `h1` 標籤）。

換句話說，**命令式程式設計**就像給廚師逐步指令來製作披薩。**宣告式程式設計**就像點披薩，而不關心製作披薩所需的步驟。🍕

[React](https://react.dev/) 是一個流行的宣告式函式庫，可用來建立使用者介面。

React：宣告式 UI 函式庫[](#react-a-declarative-ui-library)
------------------------------------------------------------------

作為開發者，您可以告訴 React 您希望使用者介面發生什麼，React 會替您找出**如何**更新 DOM 的步驟。
