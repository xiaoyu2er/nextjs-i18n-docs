### 未優化的圖片 [](#unoptimized-images)

使用普通 HTML，我們就這樣添加了我們的 Hero 圖片：

```
<img src="large-image.jpg" alt="Large Image" />
```

然而，這意味著我們必須手動優化一些事情，例如：

*   確保我們的圖片在不同螢幕尺寸上是響應式的。
*   使用第三方工具或庫來優化我們的圖片。
*   當圖片進入視口時進行延遲載入

### 圖片組件 [](#the-image-component)

Next 為我們提供了一個 [圖片組件](/docs/api-reference/next/image)，它可以開箱即用地處理這些最佳化。

`next/image` 是 HTML `img` 元素的延伸，專為現代網頁進化而來。

這意味著，使用 `next/image` 可以自動完成調整大小、最佳化以及以現代格式（例如 WebP，當瀏覽器支援時）提供圖片。

該組件可避免向視口較小的設備發送大尺寸圖片，並允許 Next.js 採用未來的圖片格式，並將這些圖片服務給支援其的瀏覽器。

自動圖片優化適用於任何圖片來源。即使圖片託管在外部資料源（如 CMS），仍然可以進行優化。

### 自動圖片優化如何運作？[](#how-does-automatic-image-optimization-work)

#### 按需最佳化 [](#on-demand-optimization)

Next.js 不是在構建時間最佳化圖片，而是在用戶請求時按需最佳化圖片。與靜態網站生成器和僅靜態的解決方案不同，無論發送十張圖片還是十億張圖片，構建時間都不會增加。

#### 延遲載入的圖片 [](#lazy-loaded-images)

圖片預設會延遲載入。視口外的圖片不會對頁面速度評分造成負面影響。圖片僅在進入視口時才會載入。

#### 避免累積佈局偏移 (CLS) [](#avoids-cls)

圖片始終會被渲染，以避免累積佈局偏移 (CLS)。

### [使用 Image 元件](#using-the-image-component)

讓我們使用 `next/image` 更新應用程式以顯示我們的 Hero 圖片。`height` 和 `width` 屬性應為預期的渲染大小，且與原始圖片具有相同的長寬比。

開啟 `pages/index.js` 檔案並新增一個導入

在檔案開頭新增來自 `next/image` 的 `Image`：

```
import Image from 'next/image';
```

接著，將 Hero `img` 替換為 `Image` 組件：

```
// Before
<img src="large-image.jpg" alt="Large Image" />
 
// After
<Image src="/large-image.jpg" alt="Large Image" width={3048} height={2024} />
```

頁腳中也有一張圖片。我們也來替換這個：

```
// Before
<img src="/vercel.svg" alt="Vercel Logo" />
 
// After
<Image src="/vercel.svg" alt="Vercel Logo" width={72} height={16} />
```

最後，在 Chrome DevTools 中運行另一個 Lighthouse 報告並比較您的結果。
