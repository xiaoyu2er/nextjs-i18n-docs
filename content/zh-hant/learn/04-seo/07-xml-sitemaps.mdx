**網站地圖 (Sitemaps)** 是與 Google 溝通最簡單的方式。它們指示屬於您網站的 URL 及其更新情況，以便 Google 可以輕鬆檢測新內容並更有效率地爬取您的網站。

即使 XML 網站地圖 (Sitemaps) 是最為知名且常用的，但如果偏好最大程度簡潔，也可以透過 [RSS](https://developers.google.com/search/docs/advanced/sitemaps/build-sitemap) 或 [Atom](https://developers.google.com/search/docs/advanced/sitemaps/build-sitemap) 甚至 [文本](https://developers.google.com/search/docs/advanced/sitemaps/build-sitemap) 檔案來建立。

網站地圖 (Sitemap) 是一個檔案，您可以在其中提供有關您網站上的頁面、影片和其他檔案及其相互關係的資訊。像 Google 這樣的搜尋引擎會讀取這個檔案，以更智慧地爬取您的網站。

根據 [Google](https://developers.google.com/search/docs/advanced/sitemaps/overview)：

如果您符合以下情況，可能需要網站地圖：

*   **您的網站非常大。** 因此，Google 網頁爬蟲更有可能忽略抓取您的一些新頁面或最近更新的頁面。
*   **您的網站擁有大量孤立或彼此之間連結不佳的內容頁面存檔。** 如果您的網站頁面不能自然地相互引用，您可以在網站地圖中列出它們，以確保 Google 不會忽略您的某些頁面。
*   **您的網站是新的，且外部連結很少。** Googlebot 和其他網頁爬蟲透過從一個頁面跳转至另一個頁面來瀏覽網路。因此，如果沒有其他網站链接到您的頁面，Google 可能無法發現您的頁面。
*   **您的網站擁有大量富媒體內容（影片、圖片）或在 Google News 中顯示。** 如果提供，Google 可以在適當的情況下將網站地圖中的額外資訊納入搜尋結果。

雖然網站地圖對於良好的搜尋引擎效能並非強制要求，但它們有助於促進機器人的爬取和索引，因此您的內容將被更快地收錄並相應地排名。

強烈建議使用網站地圖，並且在網站填充新內容時使其保持動態。靜態網站地圖也是有效的，但它們可能對 Google 效用較小，因為它們無法用於持續發現目的。

### 如何將網站地圖新增到 Next.js 專案 [](#how-to-add-sitemaps-to-a-nextjs-project)

有兩種選擇：

**手動**：如果您有一個相對簡單且靜態的網站，您可以在專案的 `public` 目錄中建立 `sitemap.xml`：

```
   <!-- public/sitemap.xml -->
   <xml version="1.0" encoding="UTF-8">
   <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
     <url>
       <loc>http://www.example.com/foo</loc>
       <lastmod>2021-06-01</lastmod>
     </url>
   </urlset>
   </xml>
```

**動態**：如果您的網站是動態的，您可以利用 `getServerSideProps` 來按需生成 XML 網站地圖。

我們可以在 pages 目錄內建立一個新頁面，例如 `pages/sitemap.xml.js`。此頁面的目的是呼叫我们的 API 獲取資料，以便我們知道動態頁面的 URL。然後我們將編寫一個 XML 檔案作為 `/sitemap.xml` 的回應。

以下是您可以親自嘗試的例子：

```
//pages/sitemap.xml.js
const EXTERNAL_DATA_URL = 'https://jsonplaceholder.typicode.com/posts';
 
function generateSiteMap(posts) {
  return `<?xml version="1.0" encoding="UTF-8"?>
   <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
     <!--We manually set the two URLs we know already-->
     <url>
       <loc>https://jsonplaceholder.typicode.com</loc>
     </url>
     <url>
       <loc>https://jsonplaceholder.typicode.com/guide</loc>
     </url>
     ${posts
       .map(({ id }) => {
         return `
       <url>
           <loc>${`${EXTERNAL_DATA_URL}/${id}`}</loc>
       </url>
     `;
       })
       .join('')}
   </urlset>
 `;
}
 
function SiteMap() {
  // getServerSideProps will do the heavy lifting
}
 
export async function getServerSideProps({ res }) {
  // We make an API call to gather the URLs for our site
  const request = await fetch(EXTERNAL_DATA_URL);
  const posts = await request.json();
 
  // We generate the XML sitemap with the posts data
  const sitemap = generateSiteMap(posts);
 
  res.setHeader('Content-Type', 'text/xml');
  // we send the XML to the browser
  res.write(sitemap);
  res.end();
 
  return {
    props: {},
  };
}
 
export default SiteMap;
```
