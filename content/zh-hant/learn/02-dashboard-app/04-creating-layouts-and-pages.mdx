到目前為止，您的應用程式只有一個首頁。讓我們學習如何使用 **佈局 (layouts)** 和 **頁面 (pages)** 建立更多路由。

嵌套路由 [](#nested-routing)
---------------------------------

Next.js 使用基於檔案系統的路由，其中 **資料夾 (folders)** 用於建立嵌套路由。每個資料夾代表一個 **路由區段 (route segment)**，對應到 **URL 區段 (URL segment)**。

![顯示資料夾如何對應到 URL 區段的圖示](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/folders-to-url-segments.png)![顯示資料夾如何對應到 URL 區段的圖示](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Ffolders-to-url-segments.png&w=3840&q=75)

您可以使用 `layout.tsx` 和 `page.tsx` 檔案為每個路由建立独立 UI。

`page.tsx` 是一個特殊的 Next.js 檔案，用於 exports 一個 React 組件，且對於可訪問該路由是必需的。在您的應用程式中，您已經有一個頁面檔案：`/app/page.tsx` - 這是與路由 `/` 關聯的首頁。

要建立嵌套路由，您可以將資料夾互相嵌套，並在它們內部加入 `page.tsx` 檔案。例如：

![顯示新增名為 dashboard 的資料夾如何建立新路由 '/dashboard' 的圖示](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/dashboard-route.png)![顯示新增名為 dashboard 的資料夾如何建立新路由 '/dashboard' 的圖示](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fdashboard-route.png&w=3840&q=75)

`/app/dashboard/page.tsx` 與 `/dashboard` 路徑關聯。讓我們建立頁面來看看它如何運作！

建立儀表板頁面 [](#creating-the-dashboard-page)
-----------------------------------------------------------

在 `/app` 內部建立一個名為 `dashboard` 的資料夾。接著，在 `dashboard` 資料夾內部建立一個新的 `page.tsx` 檔案，內容如下：

/app/dashboard/page.tsx

```
export default function Page() {
  return <p>Dashboard Page</p>;
}
```

現在，請確認開發伺服器正在執行，並造訪 [http://localhost:3000/dashboard](http://localhost:3000/dashboard)。您應該會看到「Dashboard Page」文字。

這就是如何在 Next.js 中建立不同頁面的方法：使用資料夾建立新的路由區段，並在內部加入 `page` 檔案。

透過為 `page` 檔案保留特別的名稱，Next.js 讓您能將 UI 組件、測試檔案和其他相關程式碼與您的路由 [置於同一處](/docs/app/getting-started/project-structure#colocation)。只有 `page` 檔案內容會公開可訪問。例如，`/ui` 和 `/lib` 資料夾與您的路由一起 _置於同一處_ 在 `/app` 資料夾內。

建立儀表板佈局 [](#creating-the-dashboard-layout)
---------------------------------------------------------------

儀表板通常擁有一種類型的導航，可跨多個頁面共享。在 Next.js 中，您可以使用特殊的 `layout.tsx` 檔案來建立跨多個頁面共享的 UI。讓我們為儀表板頁面建立佈局！

在 `/dashboard` 資料夾內部，加入一個名為 `layout.tsx` 的新檔案，貼上以下程式碼：

/app/dashboard/layout.tsx

```
import SideNav from '@/app/ui/dashboard/sidenav';
 
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div className="flex h-screen flex-col md:flex-row md:overflow-hidden">
      <div className="w-full flex-none md:w-64">
        <SideNav />
      </div>
      <div className="grow p-6 md:overflow-y-auto md:p-12">{children}</div>
    </div>
  );
}
```

這段程式碼中有幾個動作，讓我們逐一解析：

首先，您將 `<SideNav />` 組件導入佈局中。您在此檔案中導入的任何組件都將成為佈局的一部分。

`<Layout />` 組件接收 `children` prop。這個子元素可以是頁面或其他佈局。在您的情況下，`/dashboard` 內的頁面將自動嵌套在 `<Layout />` 內部，如下所示：

![包含透過佈局嵌套儀表板頁面作為子元素的資料夾結構](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/shared-layout.png)![包含透過佈局嵌套儀表板頁面作為子元素的資料夾結構](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fshared-layout.png&w=3840&q=75)

儲存您的變更並檢查您的本機主機，即可確認所有項目是否正常運作。您應該會看到以下內容：

![帶有側邊導航欄和主內容區的儀表板頁面](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/shared-layout-page.png)![帶有側邊導航欄和主內容區的儀表板頁面](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fshared-layout-page.png&w=1920&q=75)

在 Next.js 中使用佈局的一個好處是，在導航時，只有頁面組件會更新，而佈局不會重新渲染。這稱為 [部分渲染](/docs/app/building-your-application/routing/linking-and-navigating#4-partial-rendering)，它能在頁面之間切換時保留佈局中的客户端 React 狀態。

![顯示儀表板佈局嵌套儀表板頁面的資料夾結構，但在導航時只有頁面 UI 交換](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/partial-rendering-dashboard.png)![顯示儀表板佈局嵌套儀表板頁面的資料夾結構，但在導航時只有頁面 UI 交換](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fpartial-rendering-dashboard.png&w=3840&q=75)

根佈局 [](#root-layout)
---------------------------

在第 3 章，您將 `Inter` 字型導入另一個佈局：`/app/layout.tsx`。回顧一下：

/app/layout.tsx

```
import '@/app/ui/global.css';
import { inter } from '@/app/ui/fonts';
 
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body className={`${inter.className} antialiased`}>{children}</body>
    </html>
  );
}
```

這稱為 [根佈局](/docs/app/api-reference/file-conventions/layout#root-layouts)，且在每個 Next.js 應用程式中都是必需的。添加到根佈局的任何 UI 都將與您的應用程式中的**所有**頁面共享。您可以使用根佈局來修改您的 `<html>` 和 `<body>` 標籤，並添加元數據 (您將在 [稍後的章節](/learn/dashboard-app/adding-metadata) 中了解更多關於元數據的內容)。

由於您剛剛建立的新佈局 (`/app/dashboard/layout.tsx`) 僅適用於儀表板頁面，因此無需將任何 UI 添加到上述根佈局中。
