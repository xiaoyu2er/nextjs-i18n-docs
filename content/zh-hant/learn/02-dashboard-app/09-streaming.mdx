在前一章中，你學習了 Next.js 的不同渲染方法。我們也討論了慢速資料擷取如何影響應用程式的性能。讓我們來看看如何當有慢速資料請求時，如何改善使用者體驗。

什麼是串流？[](#what-is-streaming)
----------------------------------------

串流（Streaming）是一種資料傳輸技術，允許你將路由分解成較小的「區塊」，並隨著準備就緒從伺服器逐步串流到客戶端。

![顯示時間與序列式資料擷取和並列資料擷取的圖表](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/server-rendering-with-streaming.png)![顯示時間與序列式資料擷取和並列資料擷取的圖表](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fserver-rendering-with-streaming.png&w=3840&q=75)

透過串流，你可以防止慢速的資料請求阻塞整個頁面。這樣使用者可以看到並與頁面的一部分進行互動，而無需等待所有資料載入後才將任何 UI 顯示給使用者。

![顯示串列式與並行資料撷取時間的圖表](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/server-rendering-with-streaming-chart.png)![顯示串列式與並行資料撷取時間的圖表](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fserver-rendering-with-streaming-chart.png&w=3840&q=75)

串流與 React 的元件模型運作良好，因為每個元件都可以被視為一個 _區塊_。

在 Next.js 中實現串流有兩種方式：

1.  在頁面層級，使用 `loading.tsx` 檔案（它會為您建立 `<Suspense>`）。
2.  在元件層級，使用 `<Suspense>` 以進行更細粒度的控制。

讓我們來看看如何運作。

### 使用 `loading.tsx` 串流整個頁面[](#streaming-a-whole-page-with-loadingtsx)
------------------------------------------------------------------------------------

在 `/app/dashboard` 資料夾中，建立一個名為 `loading.tsx` 的新檔案：

/app/dashboard/loading.tsx

```
export default function Loading() {
  return <div>Loading...</div>;
}
```

重新整理 [http://localhost:3000/dashboard](http://localhost:3000/dashboard)，你現在應該會看到：

![帶有「Loading...」文字的儀表板頁面](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/loading-page.png)![帶有「Loading...」文字的儀表板頁面](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Floading-page.png&w=1920&q=75)

這裡發生了一些事情：

1.  `loading.tsx` 是 Next.js 的特殊檔案，建立在 React Suspense 之上。它允許您建立後 UI 以在頁面內容載入時顯示。
2.  由於 `<SideNav>` 是靜態的，所以會立即顯示。使用者可以在動態內容載入時與 `<SideNav>` 互動。
3.  使用者無需等待頁面完成載入即可導航離開（這叫做中斷式導航）。

恭喜！你剛剛實作了串流。但我們還可以做得更多來改善使用者體驗。讓我們顯示載入骨架，而不是 `Loading…` 文字。

### 加入載入骨架[](#adding-loading-skeletons)

載入骨架（Loading Skeleton）是 UI 的簡化版本。許多網站使用它們作為預留位置（或後備方案），以告知使用者內容正在載入。你在 `loading.tsx` 中新增的任何 UI 都會嵌入為靜態檔案的一部分，並先發送。接著，剩餘的動態內容將從伺服器串流到客戶端。

在你的 `loading.tsx` 檔案中，匯入一個名為 `<DashboardSkeleton>` 的新元件：

/app/dashboard/loading.tsx

```
import DashboardSkeleton from '@/app/ui/skeletons';
 
export default function Loading() {
  return <DashboardSkeleton />;
}
```

接著，重新整理 [http://localhost:3000/dashboard](http://localhost:3000/dashboard)，你現在應該會看到：

![帶有載入骨架的儀表板頁面](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/loading-page-with-skeleton.png)![帶有載入骨架的儀表板頁面](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Floading-page-with-skeleton.png&w=1920&q=75)

### 使用路由群組修正載入骨架錯誤[](#fixing-the-loading-skeleton-bug-with-route-groups)

目前，你的載入骨架將套用至發票。

由於 `loading.tsx` 在檔案系統中比 `/invoices/page.tsx` 和 `/customers/page.tsx` 高一層，所以它也適用於這些頁面。

我們可以使用 [路由群組](https://nextjs.org/docs/app/building-your-application/routing/route-groups) 來變更這一點。在儀表板資料夾內建立一個名為 `/(overview)` 的新資料夾。接著，將你的 `loading.tsx` 和 `page.tsx` 檔案移動到資料夾內：

![顯示如何使用括號建立路由群組的資料夾結構](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/route-group.png)![顯示如何使用括號建立路由群組的資料夾結構](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Froute-group.png&w=3840&q=75)

現在，`loading.tsx` 檔案只會套用至你的儀表板總覽頁面。

路由群組允許你將檔案組織成邏輯群組，而不影響 URL 路徑結構。當你使用括號 `()` 建立新資料夾時，名稱不會包含在 URL 路徑中。因此 `/dashboard/(overview)/page.tsx` 變成 `/dashboard`。

這裡，你使用路由群組確保 `loading.tsx` 僅套用至你的儀表板總覽頁面。不過，你也可以使用路由群組將應用程式區段化（例如 `(marketing)` 路由和 `(shop)` 路由）或由團隊區分，適用於較大的應用程式。

### 串流元件[](#streaming-a-component)

到目前為止，你正在串流整個頁面。但你也可以更細粒度地透過 React Suspense 串流特定元件。

Suspense 允許你延遲渲染應用程式的部分內容，直到滿足某些條件（例如：資料已載入）。你可以將動態元件包裝在 Suspense 中。接著，傳遞一個後備元件，以便在動態元件載入時顯示。

如果你記得慢速資料請求 `fetchRevenue()`，這是導致整個頁面變慢的請求。與其阻塞整個頁面，你可以使用 Suspense 僅串流此元件，並立即顯示其餘頁面的 UI。

為了做到這一點，你必須將資料獲取移動到元件中，讓我們更新程式碼來看看這會是什麼樣子：

從 `/dashboard/(overview)/page.tsx` 中刪除所有 `fetchRevenue()` 及其資料的實例：

/app/dashboard/(overview)/page.tsx

```
import { Card } from '@/app/ui/dashboard/cards';
import RevenueChart from '@/app/ui/dashboard/revenue-chart';
import LatestInvoices from '@/app/ui/dashboard/latest-invoices';
import { lusitana } from '@/app/ui/fonts';
import { fetchLatestInvoices, fetchCardData } from '@/app/lib/data'; // remove fetchRevenue
 
export default async function Page() {
  const revenue = await fetchRevenue() // delete this line
  const latestInvoices = await fetchLatestInvoices();
  const {
    numberOfInvoices,
    numberOfCustomers,
    totalPaidInvoices,
    totalPendingInvoices,
  } = await fetchCardData();
 
  return (
    // ...
  );
}
```

接著，從 React 匯入 `<Suspense>`，並將其包覆在 `<RevenueChart />` 之外。你可以傳遞一個名為 `<RevenueChartSkeleton>` 的後備元件給它。

/app/dashboard/(overview)/page.tsx

```
import { Card } from '@/app/ui/dashboard/cards';
import RevenueChart from '@/app/ui/dashboard/revenue-chart';
import LatestInvoices from '@/app/ui/dashboard/latest-invoices';
import { lusitana } from '@/app/ui/fonts';
import { fetchLatestInvoices, fetchCardData } from '@/app/lib/data';
import { Suspense } from 'react';
import { RevenueChartSkeleton } from '@/app/ui/skeletons';
 
export default async function Page() {
  const latestInvoices = await fetchLatestInvoices();
  const {
    numberOfInvoices,
    numberOfCustomers,
    totalPaidInvoices,
    totalPendingInvoices,
  } = await fetchCardData();
 
  return (
    <main>
      <h1 className={`${lusitana.className} mb-4 text-xl md:text-2xl`}>
        Dashboard
      </h1>
      <div className="grid gap-6 sm:grid-cols-2 lg:grid-cols-4">
        <Card title="Collected" value={totalPaidInvoices} type="collected" />
        <Card title="Pending" value={totalPendingInvoices} type="pending" />
        <Card title="Total Invoices" value={numberOfInvoices} type="invoices" />
        <Card
          title="Total Customers"
          value={numberOfCustomers}
          type="customers"
        />
      </div>
      <div className="mt-6 grid grid-cols-1 gap-6 md:grid-cols-4 lg:grid-cols-8">
        <Suspense fallback={<RevenueChartSkeleton />}>
          <RevenueChart />
        </Suspense>
        <LatestInvoices latestInvoices={latestInvoices} />
      </div>
    </main>
  );
}
```

最後，更新 `<RevenueChart>` 元件以獲取自己的資料，並移除傳入的 prop：

元件分組 [](#grouping-components)
-------------------------------------------

太好了！你就快完成了，現在你需要將 `<Card>` 元件包裝在 Suspense 中。你可以為每個單獨的卡片擷取資料，但這可能會導致卡片載入時的 _跳彈_ 效果，這對使用者來說視覺上可能會造成震撼。

那麼，你會如何解決這個問題？

為了创建更 _延遲_ 的效果，你可以使用包裝元件將卡片分組。這意味著靜態的 `<SideNav/>` 將首先顯示，接著是卡片，等等。

在你的 `page.tsx` 檔案中：

1.  刪除你的 `<Card>` 元件。
2.  刪除 `fetchCardData()` 函式。
3.  匯入一個新的 **包裝** 元件，名為 `<CardWrapper />`。
4.  匯入一個新的 **骨架** 元件，名為 `<CardsSkeleton />`。
5.  在 Suspense 中包覆 `<CardWrapper />`。

/app/dashboard/(overview)/page.tsx

```
import CardWrapper from '@/app/ui/dashboard/cards';
// ...
import {
  RevenueChartSkeleton,
  LatestInvoicesSkeleton,
  CardsSkeleton,
} from '@/app/ui/skeletons';
 
export default async function Page() {
  return (
    <main>
      <h1 className={`${lusitana.className} mb-4 text-xl md:text-2xl`}>
        Dashboard
      </h1>
      <div className="grid gap-6 sm:grid-cols-2 lg:grid-cols-4">
        <Suspense fallback={<CardsSkeleton />}>
          <CardWrapper />
        </Suspense>
      </div>
      // ...
    </main>
  );
}
```

接著，進入檔案 `/app/ui/dashboard/cards.tsx`，匯入 `fetchCardData()` 函式，並在 `<CardWrapper/>` 元件內調用它。請務必取消註解此元件中的任何必要程式碼。

/app/ui/dashboard/cards.tsx

```
// ...
import { fetchCardData } from '@/app/lib/data';
 
// ...
 
export default async function CardWrapper() {
  const {
    numberOfInvoices,
    numberOfCustomers,
    totalPaidInvoices,
    totalPendingInvoices,
  } = await fetchCardData();
 
  return (
    <>
      <Card title="Collected" value={totalPaidInvoices} type="collected" />
      <Card title="Pending" value={totalPendingInvoices} type="pending" />
      <Card title="Total Invoices" value={numberOfInvoices} type="invoices" />
      <Card
        title="Total Customers"
        value={numberOfCustomers}
        type="customers"
      />
    </>
  );
}
```

重新整理頁面，你應該會看到所有卡片同時載入。當你想要多個元件同時載入時，可以使用這個模式。

決定安放 Suspense 邊界的位置 [](#deciding-where-to-place-your-suspense-boundaries)
-----------------------------------------------------------------------------------------------------

你安放 Suspense 邊界的位置將取決於幾件事：

1.  你想讓使用者在串流時如何體驗頁面。
2.  你想優先處理什麼內容。
3.  如果元件依賴於資料擷取。

看看你的儀表板頁面，有什麼你會做得不同的嗎？

別擔心。沒有正確的答案。

*   你可以像我們使用 `loading.tsx` 一樣串流**整個頁面**...但這若有一個元件的資料擷取較慢，可能會導致較長的載入時間。
*   你可以個別串流**每個元件**...但這可能會導致 UI _跳彈_ 到螢幕上。
*   你也可以透過串流**頁面區段**創建 _延遲_ 效果。但你必須建立包裝元件。

你安放 Suspense 邊界的位置將視應用程式而異。一般而言，良好的實踐是將資料擷取移至所需的元件，然後將這些元件包裝在 Suspense 中。但如果你的應用程式需要串流區段或整個頁面，那麼這樣做是沒有問題的。

不要害怕嘗試 Suspense 並查看效果最好的方式，它是一個強大的 API，可以幫助你創建更令人愉悅的使用者體驗。
