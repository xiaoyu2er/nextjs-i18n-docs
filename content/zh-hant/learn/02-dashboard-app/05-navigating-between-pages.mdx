在上一章中，您已建立儀表板佈局和頁面。現在，讓我們添加一些連結，允許用戶在儀表板路徑之間進行導覽。

為什麼要優化導覽？[](#why-optimize-navigation)
----------------------------------------------------

要在頁面之間建立連結，傳統上會使用 <a> HTML 元素。目前，側邊欄連結使用 <a> 元素，但請注意當您在瀏覽器中在首頁、發票和客戶頁面之間導覽時會發生什麼情況。

您看到了嗎？

每次頁面導覽時都會進行完整頁面重新整理！

<
`Link`
`> 元件 [](#the-link-component)
---------------------------------------------

在 Next.js 中，您可以使用 <Link /> 元件來連結應用程式中的頁面。<Link> 允許您使用 JavaScript 進行 [客戶端導航 (Client-Side Navigation)](/docs/app/building-your-application/routing/linking-and-navigating#how-routing-and-navigation-works)。

若要使用 <
`Link`
>` 元件，開啟 <
`/app/ui/dashboard/nav-links.tsx
`>，並從 [<
`next/link
`] (/docs/app/api-reference/components/link) 匯入 Link 元件。接著，將 <a> 標籤替換為 <
`Link`>
：

/app/ui/dashboard/nav-links.tsx

```
import {
  UserGroupIcon,
  HomeIcon,
  DocumentDuplicateIcon,
} from '@heroicons/react/24/outline';
import Link from 'next/link';
 
// ...
 
export default function NavLinks() {
  return (
    <>
      {links.map((link) => {
        const LinkIcon = link.icon;
        return (
          <Link
            key={link.name}
            href={link.href}
            className="flex h-[48px] grow items-center justify-center gap-2 rounded-md bg-gray-50 p-3 text-sm font-medium hover:bg-sky-100 hover:text-blue-600 md:flex-none md:justify-start md:p-2 md:px-3"
          >
            <LinkIcon className="w-6" />
            <p className="hidden md:block">{link.name}</p>
          </Link>
        );
      })}
    </>
  );
}
```

如您所見，`Link` 組件的使用方式與 `<a>` 標籤類似，但您使用的是 `<Link href="…">` 而非 `<a href="…">`。

儲存您的變更並檢查在本地端是否運作良好。您現在應該能夠在頁面之間進行導覽而無需看到完整重新整理。雖然應用程式的某些部分在伺服器上渲染，但沒有完整頁面重新整理，這讓它感覺像本地網頁應用程式。這是為什麼？

### 自動程式碼分割與預讀 [](#automatic-code-splitting-and-prefetching)

為了改善導覽體驗，Next.js 會依路由段自動為您的應用程式進行程式碼分割。這與傳統的 React [單一頁面應用程式 (SPA)](https://nextjs.org/docs/app/building-your-application/upgrading/single-page-applications) 不同，傳統模式下瀏覽器會在初次頁面載入時載入您的所有應用程式程式碼。

依路徑分割程式碼意味著頁面變得獨立。如果某個頁面發生錯誤，應用程式的其他部分仍會運作。這也讓瀏覽器需要解析的程式碼較少，這使得您的應用程式速度更快。

此外，在生產環境中，只要 [Link](/docs/api-reference/next/link) 組件出現在瀏覽器視口，Next.js 就會自動在背景 **預讀** 已連結路徑的程式碼。當用戶點擊連結時，目標頁面的程式碼已在背景中載入完成，這正是頁面過渡幾乎無延遲的原因！

了解更多關於 [導覽如何運作](/docs/app/building-your-application/routing/linking-and-navigating#how-routing-and-navigation-works)。

設計模式：顯示作用中連結 [](#pattern-showing-active-links)
--------------------------------------------------------------

常見的 UI 設計慣例是顯示作用中連結，以指示使用者目前所在的頁面。要做到這一點，您需要從 URL 取得使用者目前的路徑。Next.js 提供了一個名為 usePathname()[/docs/app/api-reference/functions/use-pathname] 的 Hook，您可以使用它來檢查路徑並實作此模式。

由於 [usePathname()](https://nextjs.org/docs/app/api-reference/functions/use-pathname) 是 React Hook，您需要將 nav-links.tsx 轉換為 Client Component。在檔案頂部新增 React 的 "use client" 指令，然後從 next/navigation 匯入 usePathname()：

/app/ui/dashboard/nav-links.tsx

```
'use client';
 
import {
  UserGroupIcon,
  HomeIcon,
  DocumentDuplicateIcon,
} from '@heroicons/react/24/outline';
import Link from 'next/link';
import { usePathname } from 'next/navigation';
 
// ...
```

接下來，將路徑指定給 <NavLinks /> 組件內的變數 pathname：

/app/ui/dashboard/nav-links.tsx

```
export default function NavLinks() {
  const pathname = usePathname();
  // ...
}
```

> **注意**：nav-links.tsx 不是 Next.js 的特殊檔案 — 它可以是任何您想要的名稱。如果您重新命名它，請確保相應地更新匯入語句。

您可以在 CSS 樣式](/learn/dashboard-app/css-styling) 章節中介紹的 clsx 函式庫用於在連結為作用中時條件式套用類別名稱。當 link.href 與 pathname 匹配時，連結應以藍色文字和淺藍色背景顯示。

這是 nav-links.tsx 的最終程式碼：

/app/ui/dashboard/nav-links.tsx

```
'use client';
 
import {
  UserGroupIcon,
  HomeIcon,
  DocumentDuplicateIcon,
} from '@heroicons/react/24/outline';
import Link from 'next/link';
import { usePathname } from 'next/navigation';
import clsx from 'clsx';
 
// ...
 
export default function NavLinks() {
  const pathname = usePathname();
 
  return (
    <>
      {links.map((link) => {
        const LinkIcon = link.icon;
        return (
          <Link
            key={link.name}
            href={link.href}
            className={clsx(
              'flex h-[48px] grow items-center justify-center gap-2 rounded-md bg-gray-50 p-3 text-sm font-medium hover:bg-sky-100 hover:text-blue-600 md:flex-none md:justify-start md:p-2 md:px-3',
              {
                'bg-sky-100 text-blue-600': pathname === link.href,
              },
            )}
          >
            <LinkIcon className="w-6" />
            <p className="hidden md:block">{link.name}</p>
          </Link>
        );
      })}
    </>
  );
}
```

儲存並檢查您的本地端。您現在應該會看到作用中連結以藍色高亮顯示。
