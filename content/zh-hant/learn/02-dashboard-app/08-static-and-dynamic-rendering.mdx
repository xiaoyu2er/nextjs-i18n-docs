在上一章中，您為儀表板概覽頁面檢索了數據。不過，我們簡要討論了當前設置的兩個限制：

1. 數據請求造成了無意的數據瀑布。
2. 儀表板是靜態的，因此任何數據更新都不會反映到您的應用程式中。

### 什麼是靜態渲染？[](#what-is-static-rendering)

使用靜態渲染時，數據檢索和渲染會在服務器上於構建時（部署時）或重新驗證數據時 [重新驗證數據](/docs/app/building-your-application/data-fetching/fetching-caching-and-revalidating#revalidating-data) 發生。

無論用戶何時訪問您的應用程式，都會提供緩存結果。靜態渲染有幾個好處：

*   **更快網站** - 預先渲染的內容在部署到 [Vercel](https://vercel.com/) 等平台時可以進行緩存並全球分發。這確保全球用戶可以更快速、可靠地訪問您的網站內容。
*   **減少服務器負載** - 由於內容已緩存，伺服器不需要為每個用戶請求動態生成內容。這可以降低計算成本。
*   **SEO** - 預先渲染的內容更容易讓搜索引擎爬蟲索引，因為頁面加載時內容已可用。這可以帶來更好的搜索引擎排名。

靜態渲染適用於 **沒有數據** 或 **跨用戶共享的數據** 的 UI，例如靜態博客文章或產品頁面。對於擁有定期更新的個人化數據的儀表板，它可能並不合適。

靜態渲染的相反是動態渲染。

什麼是動態渲染？[](#what-is-dynamic-rendering)
--------------------------------------------------------

使用動態渲染時，內容會在 **請求時**（當用戶訪問頁面時）於服務器上為每個用戶渲染。動態渲染有幾個好處：

*   **即時數據** - 動態渲染允許您的應用程式顯示即時或頻繁更新的數據。這非常適合數據經常變更的應用程式。
*   **特定用戶的內容** - 更容易提供個人化內容，例如儀表板或用戶個人資料，並根據用戶互動更新數據。
*   **請求時信息** - 動態渲染允許您獲取只能在請求時獲知的信息，例如 Cookie 或 URL 搜尋參數。

模擬緩慢的數據獲取 [](#simulating-a-slow-data-fetch)
-------------------------------------------------------------

我們正在構建的儀表板應用程式是動態的。

不過，上一章提及的問題仍然存在。如果其中一個數據請求比其他請求更慢，會發生什麼情況？

讓我們模擬緩慢的數據獲取。在 `app/lib/data.ts` 中，取消註釋 `fetchRevenue()` 函數內的 `console.log` 和 `setTimeout`：

/app/lib/data.ts

```
export async function fetchRevenue() {
  try {
    // We artificially delay a response for demo purposes.
    // Don't do this in production :)
    console.log('Fetching revenue data...');
    await new Promise((resolve) => setTimeout(resolve, 3000));
 
    const data = await sql<Revenue[]>`SELECT * FROM revenue`;
 
    console.log('Data fetch completed after 3 seconds.');
 
    return data;
  } catch (error) {
    console.error('Database Error:', error);
    throw new Error('Failed to fetch revenue data.');
  }
}
```

現在在一個新分頁中打開 [http://localhost:3000/dashboard/](http://localhost:3000/dashboard/)，並注意頁面加載時間變長了。在終端中，您還應該看到以下消息：

```
Fetching revenue data...
Data fetch completed after 3 seconds.
```

在此處，您添加了人工的 3 秒延遲，用以模擬緩慢的數據獲取。結果是，現在數據獲取期間，您的整個頁面無法向訪客顯示 UI。這引出了開發人員必須解決的常見挑戰：

使用動態渲染時，**您的應用程式速度僅取決於最慢的數據獲取。**
