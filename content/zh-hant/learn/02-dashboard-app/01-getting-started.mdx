建立新專案 [](#creating-a-new-project)
-------------------------------------------------

我們建議使用 [`pnpm`](https://pnpm.io/) 作為套件管理員，因為它比 `npm` 或 `yarn` 更快且更有效率。如果您尚未安裝 `pnpm`，您可以透過執行以下命令全局安裝：

終端機

```
npm install -g pnpm
```

要建立 Next.js 應用程式，請開啟終端機，[`cd`](https://developer.mozilla.org/en-US/docs/Learn/Tools_and_testing/Understanding_client-side_tools/Command_line#basic_built-in_terminal_commands) 到您想要存放專案的資料夾，並執行以下命令：

終端機

```
npx create-next-app@latest nextjs-dashboard --example "https://github.com/vercel/next-learn/tree/main/dashboard/starter-example" --use-pnpm
```

此命令使用了 [`create-next-app`](/docs/app/api-reference/create-next-app)，這是一個命令列介面 (CLI) 工具，會為您設定 Next.js 應用程式。在上述命令中，您還使用了 `--example` 旗標搭配本課程的 [啟動範例](https://github.com/vercel/next-learn/tree/main/dashboard/starter-example)。

探索專案 [](#exploring-the-project)
-----------------------------------------------

與讓您從零開始編寫程式碼的教學不同，本課程的程式碼大部分已經由您撰寫完成。這更能反映實際開發情境，在該情境中，您極有可能會與現有的程式碼庫合作。

我們的目標是協助您專注學習 Next.js 的主要功能，而不必撰寫 _所有_ 應用程式的所有程式碼。

安裝完成後，在您的編輯器中開啟專案，並瀏覽至 `nextjs-dashboard`。

終端機

```
cd nextjs-dashboard
```

讓我們花些時間探索這個專案。

### 資料夾結構 [](#folder-structure)

您會發現該專案具有以下資料夾結構：

![儀表板專案的資料夾結構，顯示主要資料夾與檔案：app、public 以及設定檔。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/learn-folder-structure.png)![儀表板專案的資料夾結構，顯示主要資料夾與檔案：app、public 以及設定檔。](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Flearn-folder-structure.png&w=3840&q=75)

*   **`/app`**：包含應用程式的所有路由、元件和邏輯，您將主要在此處工作。
*   **`/app/lib`**：包含應用程式中使用的所有函式，例如可複用的工具函式與資料取得函式。
*   **`/app/ui`**：包含應用程式的所有 UI 元件，例如卡片、表格與表單。為了節省時間，我們已為您預先設定了這些元件的樣式。
*   **`/public`**：包含應用程式的所有靜態資源，例如圖片。
*   **設定檔**：您也會注意到設定檔，例如位於應用程式根層的 `next.config.ts`。這些檔案多數是在使用 `create-next-app` 開始新專案時建立並預先設定的。在課程中您不需要修改它們。

請自由探索這些資料夾，如果目前還無法理解程式碼所做的所有事情，也不用擔心。

### 置換資料 [](#placeholder-data)

當您在建構使用者介面時，擁有某些置換資料會很有幫助。如果資料庫或 API 尚未可用，您可以：

*   使用 JSON 格式的置換資料或 JavaScript 物件。
*   使用如 [mockAPI](https://mockapi.io/) 的第三方服務。

在本專案中，我們已在 `app/lib/placeholder-data.ts` 中提供了些許置換資料。檔案中的每個 JavaScript 物件代表資料庫中的一個資料表。例如，針對發票資料表：

/app/lib/placeholder-data.ts

```
const invoices = [
  {
    customer_id: customers[0].id,
    amount: 15795,
    status: 'pending',
    date: '2022-12-06',
  },
  {
    customer_id: customers[1].id,
    amount: 20348,
    status: 'pending',
    date: '2022-11-14',
  },
  // ...
];
```

在 [設定您的資料庫](/learn/dashboard-app/setting-up-your-database) 章節中，您將使用這些資料來 _種子化 (seed)_ 您的資料庫（使用一些初始資料填充它）。

### TypeScript[](#typescript)

您可能也會注意到大多數檔案具有 `.ts` 或 `.tsx` 副檔名。這是因為專案是使用 TypeScript 撰寫的。我們希望建立一堂反映現代網路環境的課程。

如果您不懂 TypeScript 也沒關係 - 我們會在需要時提供 TypeScript 程式碼片段。

現在，請檢視 `/app/lib/definitions.ts` 檔案。在這裡，我們手動定義將從資料庫傳回的型別。例如，發票資料表具有以下型別：

/app/lib/definitions.ts

```
export type Invoice = {
  id: string;
  customer_id: string;
  amount: number;
  date: string;
  // In TypeScript, this is called a string union type.
  // It means that the "status" property can only be one of the two strings: 'pending' or 'paid'.
  status: 'pending' | 'paid';
};
```

透過使用 TypeScript，您可以確保不會不小心將錯誤的資料格式傳遞給您的元件或資料庫，例如在發票 `amount` 中傳遞 `string` 而不是 `number`。

>
> **如果您是一位 TypeScript 開發者：**
> 
> *   我們手動宣告資料型別，但為了更佳的型別安全性，我們推薦使用 [Prisma](https://www.prisma.io/) 或 [Drizzle](https://orm.drizzle.team/)，它們會根據您的資料庫結構自動生成型別。
> *   Next.js 會偵測您的專案是否使用 TypeScript，並自動安裝必要的套件和設定。Next.js 也隨附了一個 [TypeScript 外掛](https://nextjs.org/docs/app/building-your-application/configuring/typescript#typescript-plugin) 供您的程式碼編輯器使用，以協助自動完成和型別安全性。

運行開發伺服器 [](#running-the-development-server)
------------------------------------------------------------------

執行 `pnpm i` 以安裝專案套件。

終端機

```
pnpm i
```

接著執行 `pnpm dev` 以啟動開發伺服器。

終端機

```
pnpm dev
```

`pnpm dev` 會在連接埠 `3000` 啟動您的 Next.js 開發伺服器。讓我們檢查它是否正常運作。

在您的瀏覽器中開啟 [http://localhost:3000](http://localhost:3000/)。您的首頁應該看起來像這樣，這是刻意未經樣式設定的：

![未經樣式設定的頁面，標題為「Acme」，包含描述和登入連結。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/acme-unstyled.png)![未經樣式設定的頁面，標題為「Acme」，包含描述和登入連結。](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Facme-unstyled.png&w=3840&q=75)
