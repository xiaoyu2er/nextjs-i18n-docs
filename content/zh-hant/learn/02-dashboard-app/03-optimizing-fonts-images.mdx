在上一章中，您已學會如何為 Next.js 應用程式進行樣式設計。讓我們繼續完善首頁，加入自訂字體與主圖。

## 為什麼要最佳化字體？[](#why-optimize-fonts)
------------------------------------------

字體在網站設計中扮演重要角色，但在專案中使用自訂字體可能會影響效能，因為字體檔案需要被抓取和載入。

[累積佈局偏移 (Cumulative Layout Shift)](https://vercel.com/blog/how-core-web-vitals-affect-seo) 是 Google 用來評估網站效能與使用者體驗的指標。使用字體時，當瀏覽器最初以備用或系統字體渲染文字，之後再替換為自訂字體時，就會發生佈局偏移。這種替換可能會導致文字大小、間距或佈局改變，進而推擠周圍的元素。

![顯示頁面初始載入，隨後在自訂字體載入時發生佈局偏移的模擬 UI。](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/font-layout-shift.png)![顯示頁面初始載入，隨後在自訂字體載入時發生佈局偏移的模擬 UI。](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Ffont-layout-shift.png&w=3840&q=75)

Next.js 在您使用 `next/font` 模組時會自動最佳化應用程式中的字體。它會在建構時下載字體檔案，並隨其他靜態資源一同託管。這意味著當使用者造訪您的應用程式時，不會有針對字體的額外網路請求，從而影響效能。

## 新增主要字體[](#adding-a-primary-font)
-----------------------------------------------

讓我們將自訂 Google 字體加入您的應用程式，看看它是如何運作的。

在您的 `/app/ui` 資料夾中，建立一個名為 `fonts.ts` 的新檔案。您將使用此檔案來存放應用程式中將使用的字體。

從 `next/font/google` 模組匯入 `Inter` 字體 — 這將是您的主要字體。接著，指定您想載入的 [子集 (subset)](https://fonts.google.com/knowledge/glossary/subsetting)。在這個案例中，使用 `'latin'`：

/app/ui/fonts.ts

```
import { Inter } from 'next/font/google';
 
export const inter = Inter({ subsets: ['latin'] });
```

最後，將字體加入到 `/app/layout.tsx` 中的 `<body>` 元素：

/app/layout.tsx

```
import '@/app/ui/global.css';
import { inter } from '@/app/ui/fonts';
 
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body className={`${inter.className} antialiased`}>{children}</body>
    </html>
  );
}
```

將 `Inter` 加入到 `<body>` 元素，字體將套用至整個應用程式。此處您也加入了 Tailwind [抗鋸齒 (antialiased)](https://tailwindcss.com/docs/font-smoothing) 類別，這能让字體更平滑。使用此類別雖非必須，但能增添細節效果。

導覽至您的瀏覽器，開啟開發人員工具並選擇 `body` 元素。您應該看到 `Inter` 和 `Inter_Fallback` 現在已套用於樣式之下。

## 為什麼要最佳化圖片？[](#why-optimize-images)
--------------------------------------------

Next.js 可以在頂級 [`/public`](/docs/app/building-your-application/optimizing/static-assets) 資料夾下提供 **靜態資源**，例如圖片。`/public` 內的檔案可以在您的應用程式中參照。

使用一般 HTML 時，您會如下方式加入圖片：

```
<img
  src="/hero.png"
  alt="Screenshots of the dashboard project showing desktop version"
/>
```

然而，這意味著您必須手動：

*   確保您的圖片在不同螢幕尺寸下為響應式。
*   指定適合不同裝置的圖片尺寸。
*   防止圖片載入時的佈局偏移。
*   對使用者視口外的圖片進行惰性載入。

圖片最佳化是網頁開發中的一大主題，甚至可視為一門專門技術。與其手動實作這些最佳化，您可以使用 `next/image` 元件來自動最佳化您的圖片。

## `<Image>` 元件[](#the-image-component)
-----------------------------------------------

`<Image>` 元件是 HTML `<img>` 標籤的擴充，並隨自動圖片最佳化功能，例如：

*   圖片載入時自動防止佈局偏移。
*   調整圖片大小，避免向視窗較小的裝置傳送大型圖片。
*   預設進行圖片惰性載入（圖片隨著進入視口而載入）。
*   在瀏覽器支援時，提供現代格式的圖片，如 [WebP](https://developer.mozilla.org/en-US/docs/Web/Media/Formats/Image_types#webp) 和 [AVIF](https://developer.mozilla.org/en-US/docs/Web/Media/Formats/Image_types#avif_image)。

## 新增桌面主圖[](#adding-the-desktop-hero-image)
---------------------------------------------------------------

讓我們使用 `<Image>` 元件。如果您查看 `/public` 資料夾內部，會看到有兩張圖片：`hero-desktop.png` 和 `hero-mobile.png`。這兩張圖片完全不同，將根據使用者的裝置是桌面還是行動裝置來顯示。

在您的 `/app/page.tsx` 檔案中，從 [`next/image`](https://nextjs.org/docs/api-reference/next/image) 匯入該元件。接著，在註解下方加入該圖片：

/app/page.tsx

```
import AcmeLogo from '@/app/ui/acme-logo';
import { ArrowRightIcon } from '@heroicons/react/24/outline';
import Link from 'next/link';
import { lusitana } from '@/app/ui/fonts';
import Image from 'next/image';
 
export default function Page() {
  return (
    // ...
    <div className="flex items-center justify-center p-6 md:w-3/5 md:px-28 md:py-12">
      {/* Add Hero Images Here */}
      <Image
        src="/hero-desktop.png"
        width={1000}
        height={760}
        className="hidden md:block"
        alt="Screenshots of the dashboard project showing desktop version"
      />
    </div>
    //...
  );
}
```

這裡，您將 `width` 設定為 `1000`，`height` 設定為 `760` 像素。將圖片的 `width` 和 `height` 設定良好的做法是為了防止佈局偏移，這些比例應與原始圖片 **完全一致**。這些數值 *不是* 圖片渲染的大小，而是實際用來理解比例的图片檔案大小。

您也會注意到 `hidden` 類別用於在行動裝置上將圖片從 DOM 移除，而 `md:block` 則用於在桌面裝置上顯示圖片。

這就是您目前的首頁應該有的樣貌：

--------------------------------------------------------------------------------

現在輪到您了！在您剛加入的圖片下方，為 `hero-mobile.png` 加入另一個 `<Image>` 元件。
