既然您已建立並初始化了資料庫，讓我們探討可以為應用程式獲取資料的不同方式，並構建您的儀表板概覽頁面。

選擇如何獲取資料 [](#choosing-how-to-fetch-data)
---------------------------------------------------------

### API 層 [](#api-layer)

API 是應用程式程式碼和資料庫之間的中介層。您可能考慮使用 API 的幾種情況如下：

*   如果您使用第三方服務且該服務提供 API。
*   如果您要在用戶端抓取資料，您需要一個在伺服器上運行的 API 層，以避免將您的資料庫金鑰暴露在用戶端。

在 Next.js 中，您可以使用 [路由處理常式](/docs/app/building-your-application/routing/route-handlers) 來建立 API 端點。

### 資料庫查詢 [](#database-queries)

當您建立全端應用程式時，您還需要編寫與資料庫互動的邏輯。對於 [關聯式資料庫](https://aws.amazon.com/relational-database/) (如 Postgres)，您可以使用 SQL 或 [ORM](https://vercel.com/docs/storage/vercel-postgres/using-an-orm) 來完成此操作。

有幾種情況您必須編寫資料庫查詢：

*   在建立您的 API 端點時，您需要編寫與資料庫互動的邏輯。
*   如果您使用 React 伺服器組件 (在伺服器上抓取資料)，您可以跳過 API 層，直接查詢您的資料庫，而無需擔心將資料庫金鑰暴露在用戶端。

讓我們更深入地了解 React 伺服器組件。

### 使用伺服器組件獲取資料 [](#using-server-components-to-fetch-data)

預設情況下，Next.js 應用程式使用 **React 伺服器組件**。使用伺服器組件獲取資料是一種相對較新的方法，使用它們有幾項好處：

*   伺服器組件支援 JavaScript Promise，為非同步任務 (如資料抓取) 提供原生解決方案。您可以使用 `async/await` 語法，無需 `useEffect`、`useState` 或其他資料抓取套件。
*   伺服器組件在伺服器上運行，因此您可以將昂貴的資料抓取和邏輯保留在伺服器上，僅將結果發送至用戶端。
*   由於伺服器組件在伺服器上運行，您可以直接使用資料庫查詢，而不需要額外的 API 層。這節省了您編寫和維護額外程式碼的時間。

### 使用 SQL [](#using-sql)

對於您的儀表板應用程式，您將使用 [postgres.js](https://github.com/porsager/postgres) 套件和 SQL 編寫資料庫查詢。我們使用 SQL 的原因有幾點：

*   SQL 是用於查询關聯式資料庫的產業標準 (例如，ORM 會在底層產生 SQL)。
*   具備基本的 SQL 理解有助於您了解關聯式資料庫的底層原理，讓您能將相關知識套用到其他工具。
*   SQL 具有靈活性，允許您抓取和操作特定資料。
*   `postgres.js` 套件可防止 [SQL 注入](https://github.com/porsager/postgres?tab=readme-ov-file#query-parameters)。

如果您以前未使用過 SQL，請別擔心──我們已為您提供了查詢語句。

請前往 `/app/lib/data.ts`。您會發現我們正在使用 `postgres`。`sql` [函式](https://github.com/porsager/postgres) 允許您查詢您的資料庫：

/app/lib/data.ts

```
import postgres from 'postgres';
 
const sql = postgres(process.env.POSTGRES_URL!, { ssl: 'require' });
 
// ...
```

您可以在伺服器上任意位置呼叫 `sql`，就像伺服器組件一樣。但為了讓您在組件間的導覽更加方便，我們將所有資料查詢都保留在 `data.ts` 檔案中，並可將它們匯入至組件中。

> **注意：** 如果您在第六章使用了自己的資料庫提供者，您可能需要更新資料庫查詢以配合您的提供者。您可以在 `/app/lib/data.ts` 中找到這些查詢。

獲取 **`<LatestInvoices/>`** 的資料 [](#fetching-data-for-latestinvoices)
------------------------------------------------------------------------------

對於 `<LatestInvoices />` 組件，我們需要取得最新 5 份帳單，並按日期排序。

您可以抓取所有帳單並使用 JavaScript 進行排序。由於我們的資料量不大，這不成問題，但隨著應用程式成長，這可能會大幅增加每個請求的資料傳輸量以及用於排序所需的 JavaScript。

您可以在記憶體中排序最新帳單，或者使用 SQL 查詢來僅獲取最後 5 份帳單。例如，這是您 `data.ts` 檔案中的 SQL 查詢：

/app/lib/data.ts

```
// Fetch the last 5 invoices, sorted by date
const data = await sql<LatestInvoiceRaw[]>`
  SELECT invoices.amount, customers.name, customers.image_url, customers.email
  FROM invoices
  JOIN customers ON invoices.customer_id = customers.id
  ORDER BY invoices.date DESC
  LIMIT 5`;
```

在您的頁面中，匯入 `fetchLatestInvoices` 函式：

/app/dashboard/page.tsx

```
import { Card } from '@/app/ui/dashboard/cards';
import RevenueChart from '@/app/ui/dashboard/revenue-chart';
import LatestInvoices from '@/app/ui/dashboard/latest-invoices';
import { lusitana } from '@/app/ui/fonts';
import { fetchRevenue, fetchLatestInvoices } from '@/app/lib/data';
 
export default async function Page() {
  const revenue = await fetchRevenue();
  const latestInvoices = await fetchLatestInvoices();
  // ...
}
```

接著，取消註解 `<LatestInvoices />` 組件。您還需要在位於 `/app/ui/dashboard/latest-invoices` 的 `<LatestInvoices />` 組件本身中取消註解相關程式碼。

如果您造訪您的本機端點，您應該會看到只有最後 5 筆是從資料庫回傳的。希望您開始看到直接查詢您的資料庫的優勢了！

/app/lib/data.ts

```
const invoiceCountPromise = sql`SELECT COUNT(*) FROM invoices`;
const customerCountPromise = sql`SELECT COUNT(*) FROM customers`;
```

什麼是請求瀑布流？ [](#what-are-request-waterfalls)
------------------------------------------------------------

「瀑布流」指的是依賴前置請求完成的一系列網路請求。在獲取資料的情況下，每個請求必須等到前一個請求回傳資料後才能開始。

![顯示順序資料抓取與平行資料抓取時間的圖解](https://h8DxKfmAPhn8O0p3.public.blob.vercel-storage.com/learn/light/sequential-parallel-data-fetching.png)![顯示順序資料抓取與平行資料抓取時間的圖解](/_next/image?url=https%3A%2F%2Fh8DxKfmAPhn8O0p3.public.blob.vercel-storage.com%2Flearn%2Fdark%2Fsequential-parallel-data-fetching.png&w=3840&q=75)

例如，我們必須等待 `fetchRevenue()` 執行後，`fetchLatestInvoices()` 才能開始運作，以此類推。

/app/dashboard/page.tsx

```
const revenue = await fetchRevenue();
const latestInvoices = await fetchLatestInvoices(); // wait for fetchRevenue() to finish
const {
  numberOfInvoices,
  numberOfCustomers,
  totalPaidInvoices,
  totalPendingInvoices,
} = await fetchCardData(); // wait for fetchLatestInvoices() to finish
```

這種模式未必是壞事。可能有需要瀑布流的情況，您希望某個條件得到滿足才進行下一個請求。例如，您可能先想獲取使用者的 ID 和個人資料。一旦有了 ID，您可能再接續抓取其朋友列表。在這種情況下，每個請求都取決於前一個請求回傳的資料。

然而，這種行為也可能是不由自主的，並影響效能。

並行資料抓取 [](#parallel-data-fetching)
-------------------------------------------------

避免瀑布流的常見方法是同時啟動所有資料請求──進行並行請求。

在 JavaScript 中，您可以使用 [`Promise.all()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise/all) 或 [`Promise.allSettled()`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise/allSettled) 函式來同時啟動所有 Promise。例如，在 `data.ts` 中，我們在 `fetchCardData()` 函式中使用 `Promise.all()`：

/app/lib/data.ts

```
export async function fetchCardData() {
  try {
    const invoiceCountPromise = sql`SELECT COUNT(*) FROM invoices`;
    const customerCountPromise = sql`SELECT COUNT(*) FROM customers`;
    const invoiceStatusPromise = sql`SELECT
         SUM(CASE WHEN status = 'paid' THEN amount ELSE 0 END) AS "paid",
         SUM(CASE WHEN status = 'pending' THEN amount ELSE 0 END) AS "pending"
         FROM invoices`;
 
    const data = await Promise.all([
      invoiceCountPromise,
      customerCountPromise,
      invoiceStatusPromise,
    ]);
    // ...
  }
}
```

使用這種模式，您可以：

*   同時開始執行所有資料抓取，這比等待每個請求在水流模式完成要快。
*   使用可套用到任何套件或框架的原生 JavaScript 模式。

然而，單靠這個 JavaScript 模式有一個 **劣勢**：如果有一個資料請求比所有其他請求慢，該怎麼辦？讓我們在下一章了解更多。
