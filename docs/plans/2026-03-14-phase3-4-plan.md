## Phase 4：CI/CD 改造

### 当前 CI/CD 流程

```
sync-docs.yml                         translate-docs.yml
─────────────                         ──────────────────
1. Clone nextjs canary/v14/v13        1. Build translate package
2. rsync → content/en/docs/           2. Run old translate command
3. Run crawler (blog/learn)              (DeepSeek API, file-level, git timestamp)
4. process-file-changes.js            3. Create PR
   (handle renames/deletes)
5. Create PR
```

### 新 CI/CD 流程

```
sync-and-translate.yml (合并为一个 workflow)
──────────────────────
1. Clone nextjs canary/v14/v13
2. rsync → content/en/docs/
3. Run crawler (blog/learn)
4. Run batch.pipeline.ts
   - 解析 content/en/ → AST → MD5
   - 查缓存 → 拼装 → 翻译未缓存的
   - 校验 → 更新缓存
   - 写入 tmp/{lang}/ → 全量替换 content/{lang}/
5. Commit cache + translated files
6. Create PR

不再需要：
- process-file-changes.js (不用追踪 rename/delete)
- 旧的 translate package 的 main.ts/openai.ts/utils.ts
- git timestamp 比较逻辑
```

### Task 4.1：创建新的 translate-docs action

**Files:**
- Modify: `.github/actions/translate-docs/action.yml`

替换旧的翻译命令为新的 batch.pipeline.ts：

```yaml
- name: Run incremental translation
  shell: bash
  run: |
    node --import tsx packages/translate/src/batch.pipeline.ts \
      --pattern "**/*.mdx" \
      --concurrency ${{ inputs.concurrency || '10' }} \
      --output-dir tmp/translate-output \
      --lang ${{ inputs.target_language || 'zh-hans' }} \
      --lang-name "$LANG_NAME" \
      --api-type openai \
      --api-base-url "https://openrouter.ai/api/v1" \
      --api-key "${{ inputs.api_key }}" \
      --model "${{ inputs.model || 'openrouter/hunter-alpha' }}"
  env:
    LANG_NAME: "Simplified Chinese"  # Will need mapping from locale code
```

### Task 4.2：添加全量替换步骤

```yaml
- name: Replace translated content
  shell: bash
  run: |
    # For each language directory in output
    for lang_dir in tmp/translate-output/*/; do
      lang=$(basename "$lang_dir")
      if [ "$lang" != "cache" ] && [ "$lang" != "assembled" ] && [ "$lang" != "raw-translated" ] && [ "$lang" != "diffs" ]; then
        echo "Replacing content/$lang/ with translated output"
        rm -rf "apps/docs/content/$lang/"
        cp -r "$lang_dir" "apps/docs/content/$lang/"
      fi
    done
```

### Task 4.3：更新 sync-docs workflow

从 `sync-docs.yml` 中移除 `process-file-changes.js` 步骤（不再需要追踪 rename/delete）。

### Task 4.4：多语言支持

batch.pipeline.ts 已经支持 `--lang` 和 `--lang-name` 参数。在 CI 中循环所有语言：

```yaml
- name: Translate all languages
  shell: bash
  run: |
    # Read languages from translation.config.mjs or a simpler config
    for lang in zh-hans zh-hant ja es de fr ru ar; do
      echo "Translating $lang..."
      node --import tsx packages/translate/src/batch.pipeline.ts \
        --lang "$lang" \
        --lang-name "$(get_lang_name $lang)" \
        ...
    done
```

或者并行翻译多语言（每种语言一个 job）。

### Task 4.5：缓存持久化 in CI

## 可选优化

### 合并 sync + translate 为一个 workflow

目前是两个独立 workflow（sync-docs → PR → merge → translate-docs → PR）。可以合并为：
1. sync-docs 下载英文文档
2. 立即运行翻译
3. 一个 PR 包含英文更新 + 翻译更新

好处：减少一次 PR 合并周期，翻译更及时。

### 按变化量决定是否翻译

如果 sync-docs 没有检测到英文文档变化，跳过翻译步骤。

### MDX 错误自动重试

对 MDX 验证失败的文件，自动清除缓存并重试一次翻译。
